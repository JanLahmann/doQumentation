Ang operator backpropagation ay isang teknik na nagsasangkot ng pag-absorb ng mga operasyon mula sa dulo ng isang quantum circuit papasok sa isang Pauli operator, na karaniwang nagpapababa ng depth ng circuit sa kapalit ng karagdagang mga term sa operator. Ang layunin ay i-backpropagate ang pinakamaraming bahagi ng circuit hangga't maaari nang hindi pinapayagan ang operator na lumaki nang labis.

Isang paraan para payagan ang mas malalim na backpropagation sa circuit habang pinipigilan ang operator na lumaki nang labis ay ang pag-truncate ng mga term na may maliliit na coefficient, sa halip na idagdag ang mga ito sa operator. Ang pag-truncate ng mga term ay maaaring magresulta sa mas kaunting quantum circuits na kailangang i-execute, ngunit ang paggawa nito ay nagdudulot ng ilang error sa panghuling pagkalkula ng expectation value na proporsyonal sa magnitude ng mga coefficient ng na-truncate na mga term.
Sa tutorial na ito, ipapatupad natin ang isang [Qiskit pattern](https://quantum.cloud.ibm.com/docs/guides/serverless#qiskit-patterns-with-quantum-serverless) para sa pag-simulate ng quantum dynamics ng isang Heisenberg spin chain gamit ang operator backpropagation:

- **Hakbang 1: I-map sa quantum problem**
    - I-map ang time-evolved Hamiltonian sa isang quantum circuit
- **Hakbang 2: I-optimize ang problema**
    - I-slice ang circuit
    - <font color="#0F62FE">I-backpropagate ang mga slice mula sa circuit patungo sa isang Pauli observable</font>
    - Pagsamahin ang mga natitirang slice sa iisang circuit
    - I-transpile ang circuit para sa backend
- **Hakbang 3: Mag-execute ng mga eksperimento**
    - Kalkulahin ang expectation value gamit ang pinaikling circuit at pinalawak na observable kasama ang [StatevectorEstimator](https://quantum.cloud.ibm.com/docs/api/qiskit/qiskit.primitives.StatevectorEstimator) para sa kaginhawaan sa notebook na ito
- **Hakbang 4: I-reconstruct ang mga resulta**
    - Wala.

**Tandaan:** Maluwag na inilarawan ng Qiskit ang [layers](https://quantum.cloud.ibm.com/docs/api/qiskit/qiskit.dagcircuit.DAGCircuit) bilang mga depth-1 na partisyon ng circuit sa lahat ng qubit. Ginagamit ng package na ito ang terminong **slices** para ilarawan ang mga layer ng arbitrary na depth. Ang function na [qiskit_addon_obp.backpropagate](https://qiskit.github.io/qiskit-addon-obp/stubs/qiskit_addon_obp.backpropagate.html) ay dinisenyo para i-backpropagate ang buong mga slice nang isa-isa, kaya ang pagpili kung paano i-slice ang quantum circuit ay maaaring magkaroon ng malaking epekto sa kung gaano kahusay ang pagganap ng backpropagation para sa isang partikular na problema. Matututunan mo ang higit pa tungkol sa **slices** sa ibaba.
## Hakbang 1: I-map sa quantum problem
### I-map ang time-evolution ng isang quantum Heisenberg model sa isang quantum experiment.

Nagbibigay ang [qiskit_addon_utils](https://qiskit.github.io/qiskit-addon-utils/) package ng ilang reusable na functionality para sa iba't ibang layunin.

Ang module nitong [qiskit_addon_utils.problem_generators](https://qiskit.github.io/qiskit-addon-utils/stubs/qiskit_addon_utils.problem_generators.html) ay nagbibigay ng mga function para mag-generate ng mga Heisenberg-like na Hamiltonian sa isang ibinigay na connectivity graph.
Ang graph na ito ay maaaring maging [rustworkx.PyGraph](https://www.rustworkx.org/apiref/rustworkx.PyGraph.html) o [CouplingMap](https://quantum.cloud.ibm.com/docs/api/qiskit/qiskit.transpiler.CouplingMap), na ginagawa itong madaling gamitin sa mga workflow na nakasentro sa Qiskit.

Sa sumusunod, una munang mag-generate tayo ng heavy-hex na `CouplingMap` kung saan mag-uukit tayo ng linear chain ng 10 qubit. Tandaan na ang mga indeks ng bagong `reduced_coupling_map` na ito ay zero-based muli.

Susunod, mag-generate tayo ng isang Pauli operator na nag-mo-modelo ng isang Heisenberg XYZ Hamiltonian.

$$\hat{H} = \sum_{(j,k)\in E} (J_{x} \sigma_j^{x} \sigma_{k}^{x} +
    J_{y} \sigma_j^{y} \sigma_{k}^{y} + J_{z} \sigma_j^{z} \sigma_{k}^{z}) +
    \sum_{j\in V} (h_{x} \sigma_j^{x} + h_{y} \sigma_j^{y} + h_{z} \sigma_j^{z})$$

Kung saan ang $G(V,E)$ ay ang graph ng ibinigay na coupling map.

```python
import numpy as np
from qiskit_addon_utils.problem_generators import generate_xyz_hamiltonian

# Get a qubit operator describing the Heisenberg XYZ model
hamiltonian = generate_xyz_hamiltonian(
    reduced_coupling_map,
    coupling_constants=(np.pi / 8, np.pi / 4, np.pi / 2),
    ext_magnetic_field=(np.pi / 3, np.pi / 6, np.pi / 9),
)
print(hamiltonian)
```

## Hakbang 2: I-optimize ang problema
### Gumawa ng mga circuit slice para i-backpropagate

Tandaan, ang ``backpropagate`` function ay mag-ba-backpropagate ng buong mga circuit slice nang isa-isa, kaya ang pagpili kung paano mag-slice ay maaaring magkaroon ng epekto sa kung gaano kahusay ang pagganap ng backpropagation para sa isang partikular na problema. Dito, gagrupuhin natin ang mga gate ng parehong uri sa mga slice gamit ang [slice_by_gate_types](https://qiskit.github.io/qiskit-addon-utils/stubs/qiskit_addon_utils.slicing.slice_by_gate_types.html) function.

Para sa mas detalyadong talakayan tungkol sa circuit slicing, tingnan ang [how-to guide](https://qiskit.github.io/qiskit-addon-utils/how_tos/create_circuit_slices.html) na ito ng [qiskit-addon-utils](https://qiskit.github.io/qiskit-addon-utils/index.html) package.

```python
from qiskit_addon_utils.slicing import slice_by_gate_types

slices = slice_by_gate_types(circuit)
print(f"Separated the circuit into {len(slices)} slices.")
```

```text
Separated the circuit into 18 slices.
```

### Limitahan kung gaano kalaki ang maaaring lumaking operator sa panahon ng backpropagation

Sa panahon ng backpropagation, ang bilang ng mga term sa operator ay karaniwang mabilis na lalapit sa $4^N$, kung saan ang $N$ ay ang bilang ng mga qubit. Maaaring limitahan ang laki ng operator sa pamamagitan ng pagtukoy ng ``operator_budget`` kwarg ng ``backpropagate`` function, na tumatanggap ng isang [OperatorBudget](https://qiskit.github.io/qiskit-addon-obp/stubs/qiskit_addon_obp.utils.simplify.OperatorBudget.html) instance.

Dito, tinutukoy natin na dapat huminto ang backpropagation kapag ang bilang ng qubit-wise commuting Pauli group sa operator ay lumampas na sa 8.

```python
from qiskit_addon_obp.utils.simplify import OperatorBudget

op_budget = OperatorBudget(max_qwc_groups=8)
```

### I-backpropagate ang mga slice mula sa circuit

Una, tutukuyin natin ang Pauli-Z observable sa qubit 0, at mag-ba-backpropagate tayo ng mga slice mula sa time-evolution circuit hanggang sa hindi na maaaring pagsamahin ang mga term sa observable sa 8 o mas kaunting qubit-wise commuting Pauli group.

Sa ibaba, makikita mo na nag-backpropagate tayo ng 7 slice ngunit 6 lang sa 8 na inilaan na Pauli group ang ginamit. Ipinapahiwatig nito na ang pag-backpropagate ng isa pang slice ay magiging sanhi ng paglampas ng bilang ng Pauli group sa 8. Maaari nating i-verify na ganito ang kaso sa pamamagitan ng pagsusuri sa ibinalik na metadata.

```python
from qiskit.quantum_info import SparsePauliOp
from qiskit_addon_obp import backpropagate
from qiskit_addon_utils.slicing import combine_slices

# Specify a single-qubit observable
observable = SparsePauliOp("IIIIIIIIIZ")

# Backpropagate slices onto the observable
bp_obs, remaining_slices, metadata = backpropagate(observable, slices, operator_budget=op_budget)
# Recombine the slices remaining after backpropagation
bp_circuit = combine_slices(remaining_slices, include_barriers=True)

print(f"Backpropagated {metadata.num_backpropagated_slices} slices.")
print(
    f"New observable has {len(bp_obs.paulis)} terms, which can be combined into {len(bp_obs.group_commuting(qubit_wise=True))} groups."
)
print(
    f"Note that backpropagating one more slice would result in {metadata.backpropagation_history[-1].num_paulis[0]} terms "
    f"across {metadata.backpropagation_history[-1].num_qwc_groups} groups."
)
print("The remaining circuit after backpropagation looks as follows:")
bp_circuit.draw("mpl", scale=0.6)
```

```text
Backpropagated 7 slices.
New observable has 18 terms, which can be combined into 8 groups.
Note that backpropagating one more slice would result in 27 terms across 12 groups.
The remaining circuit after backpropagation looks as follows:
```

Susunod, tutukuyin natin ang parehong problema na may parehong mga limitasyon sa laki ng output observable. Ngunit sa pagkakataong ito, maglalaan tayo ng error budget sa bawat slice gamit ang [setup_budet](https://qiskit.github.io/qiskit-addon-obp/stubs/qiskit_addon_obp.utils.truncating.setup_budget.html) function. Ang mga Pauli term na may maliliit na coefficient ay ita-truncate mula sa bawat slice hanggang mapuno ang error budget, at ang natitirang budget ay idadagdag sa budget ng susunod na slice.

Para ma-enable ang truncation na ito, kailangan nating i-setup ang ating error budget ganito:

```python
from qiskit_addon_obp.utils.truncating import setup_budget

truncation_error_budget = setup_budget(max_error_per_slice=0.005)
```

Tandaan na sa pag-allot ng `5e-3` na error bawat slice para sa truncation, magagawa nating alisin pa ang 3 slice mula sa circuit, habang nananatili sa loob ng orihinal na budget na 8 commuting Pauli group sa observable. Bilang default, ginagamit ng `backpropagate` ang L1 norm ng mga na-truncate na coefficient para limitahan ang kabuuang error na naidulot mula sa truncation. Para sa iba pang mga opsyon, sumangguni sa [how-to guide sa pagtukoy ng p_norm](https://qiskit.github.io/qiskit-addon-obp/how_tos/bound_error_using_p_norm.html).

Sa partikular na halimbawang ito kung saan nag-backpropagate tayo ng 10 slice, ang kabuuang truncation error ay hindi dapat lumampas sa ``(5e-3 error/slice) * (10 slices) = 5e-2``.
Para sa karagdagang talakayan tungkol sa pamamahagi ng error budget sa mga slice, tingnan ang [how-to guide na ito](https://qiskit.github.io/qiskit-addon-obp/how_tos/truncate_operator_terms.html).

```python
# Run the same experiment but truncate observable terms with small coefficients
bp_obs_trunc, remaining_slices_trunc, metadata = backpropagate(
    observable, slices, operator_budget=op_budget, truncation_error_budget=truncation_error_budget
)

# Recombine the slices remaining after backpropagation
bp_circuit_trunc = combine_slices(remaining_slices_trunc, include_barriers=True)

print(f"Backpropagated {metadata.num_backpropagated_slices} slices.")
print(
    f"New observable has {len(bp_obs_trunc.paulis)} terms, which can be combined into {len(bp_obs_trunc.group_commuting(qubit_wise=True))} groups.\n"
    f"After truncation, the error in our observable is bounded by {metadata.accumulated_error(0):.3e}"
)
print(
    f"Note that backpropagating one more slice would result in {metadata.backpropagation_history[-1].num_paulis[0]} terms "
    f"across {metadata.backpropagation_history[-1].num_qwc_groups} groups."
)
print("The remaining circuit after backpropagation looks as follows:")
bp_circuit_trunc.draw("mpl", scale=0.6)
```

### Ngayong mayroon na tayong mga pinaikling ansatz at pinalawak na mga observable, maaari na nating i-transpile ang ating mga eksperimento sa backend.

Gagamitin natin dito ang 14-qubit na [FakeMelbourneV2](https://quantum.cloud.ibm.com/docs/api/qiskit-ibm-runtime/fake-provider-fake-melbourne-v2) mula sa [qiskit-ibm-runtime](https://quantum.cloud.ibm.com/docs/api/qiskit-ibm-runtime) para ipakita kung paano mag-transpile sa isang QPU backend.

```python
from qiskit.transpiler.preset_passmanagers import generate_preset_pass_manager
from qiskit_ibm_runtime.fake_provider import FakeMelbourneV2

# Specify a backend and a pass manager for transpilation
backend = FakeMelbourneV2()
pm = generate_preset_pass_manager(backend=backend, optimization_level=1)

# Transpile original experiment
circuit_isa = pm.run(circuit)
observable_isa = observable.apply_layout(circuit_isa.layout)

# Transpile backpropagated experiment
bp_circuit_isa = pm.run(bp_circuit)
bp_obs_isa = bp_obs.apply_layout(bp_circuit_isa.layout)

# Transpile the backpropagated experiment with truncated observable terms
bp_circuit_trunc_isa = pm.run(bp_circuit_trunc)
bp_obs_trunc_isa = bp_obs_trunc.apply_layout(bp_circuit_trunc_isa.layout)
```

## Hakbang 3: Mag-execute ng mga quantum experiment
### Kalkulahin ang expectation value

Sa wakas, maaari na tayong mag-run ng mga na-backpropagate na eksperimento at ikumpara ang mga ito sa buong eksperimento gamit ang walang ingay na [StatevectorEstimator](https://quantum.cloud.ibm.com/docs/api/qiskit/qiskit.primitives.StatevectorEstimator). Makikita natin na ang na-backpropagate na expectation value nang walang truncation ay katumbas ng eksaktong halaga sa loob ng mga limitasyon ng numerical precision.

Ang expectation value sa operator na may mga na-truncate na term ay may ilang error sa order na ``1e-4``, na nasa loob ng inaasahang tolerance.

**Tandaan:** Gumagamit tayo ng statevector-based na ``Estimator`` primitive para ilarawan ang epekto ng truncation sa output. Para mag-run sa backend kung saan na-transpile ang mga eksperimento sa Hakbang 2, mag-iimport ang isa ng [EstimatorV2](https://quantum.cloud.ibm.com/docs/api/qiskit-ibm-runtime/estimator-v2) mula sa ``qiskit-ibm-runtime`` at ipapasa ang backend instance sa constructor.

```python
from qiskit.primitives import StatevectorEstimator as Estimator

estimator = Estimator()

# Run the experiments using Estimator primitive
result_exact = estimator.run([(circuit_isa, observable_isa)]).result()[0].data.evs.item()
result_bp = estimator.run([(bp_circuit_isa, bp_obs_isa)]).result()[0].data.evs.item()
result_bp_trunc = (
    estimator.run([(bp_circuit_trunc_isa, bp_obs_trunc_isa)]).result()[0].data.evs.item()
)

print(f"Exact expectation value: {result_exact}")
print(f"Backpropagated expectation value: {result_bp}")
print(f"Backpropagated expectation value with truncation: {result_bp_trunc}")
print(f"    - Expected Error for truncated observable: {metadata.accumulated_error(0):.3e}")
print(f"    - Observed Error for truncated observable: {abs(result_exact - result_bp_trunc):.3e}")
```

```text
Exact expectation value: 0.8854160687717507
Backpropagated expectation value: 0.8854160687717532
Backpropagated expectation value with truncation: 0.8850236647156059
    - Expected Error for truncated observable: 4.933e-02
    - Observed Error for truncated observable: 3.924e-04
```

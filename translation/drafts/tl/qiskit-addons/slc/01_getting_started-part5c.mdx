#### c. Compute backward bounds

Ang bahaging ito ng noise prediction ay sinusuri kung paano maaaring makaapekto ang isang error sa isang partikular na layer sa input state na $\rho$. Ang `compute_backward_bounds` function ay unang binaliktad ang circuit, tinanggal ang mga measurement gate, at pagkatapos ay nagpatuloy sa katulad na pagsusuri tulad ng ginawa para sa forward-bound computations.

```python
backward_bounds = compute_backward_bounds(

    num_processes=slc_num_processes,
    timeout=slc_timeout,
)
```

#### Preview ng merged bounds nang walang learned noise rates

Ang `merged_bounds` function ay tinutukoy ang punto sa circuit kung saan ang paglipat mula sa backward bounds patungong forward bounds ay nagpapaliit sa kabuuang tinantyang bias sa gustong observable. Ang bias na ito ay kinukuwenta bilang kabuuan ng mga backward-bound contributions para sa lahat ng noise location bago ang puntong iyon, kasama ang mga forward-bound contributions para sa lahat ng noise location pagkatapos nito. Sa kasalukuyan, ginagawa ito nang pantay para sa lahat ng qubit.

> **Mahalagang Tala**: Ang punto ng paglipat mula sa forward patungong backward bounds ay depende sa mga learned noise rates.

```python
merged_bounds = merge_bounds(
    boxed_circuit,
    forward_bounds_tighter,
    backward_bounds,
    noise_model_rates,
)
```

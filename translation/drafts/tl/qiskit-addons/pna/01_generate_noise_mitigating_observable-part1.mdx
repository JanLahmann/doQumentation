### Ano ang propagated noise absorption (PNA)?

***Isang technique para mabawasan ang mga gate error sa pamamagitan ng pagpapalaganap ng observable sa pamamagitan ng inverse noise channel na nakakaapekto sa mga 2-qubit gate, na nagreresulta sa isang noise-mitigating observable.***
Ang mga 2Q gate sa eksperimentong gusto nating patakbuhin ay maapektuhan ng malaking noise.
![Noisy experiment](../images/noisy_expt.png)
Kung matututunan natin ang noise model, maaari nating ilapat ang inverse nito at kanselahin ang noise.
![Noise-mitigated experiment](../images/noise_mitigated_expt.png)
Sa halip na i-implement ang inverse noise channel sa pamamagitan ng pag-sample nito sa QPU tulad ng sa PEC, maaari nating i-implement ito nang klasikal sa measured observable gamit ang Pauli propagation. Nagreresulta ito sa isang mas kumplikadong observable na, kapag sinukat, ay may epekto ng pagbabawas ng natutunan na gate noise.
![PNA overview](../images/pna_overview.png)
### Buuin ang mirrored Trotter circuit at observable

Para sa eksperimentong ito, pag-aaralan natin ang time dynamics ng isang 30-site na kicked Ising model sa isang 1D spin chain. Ang Hamiltonian na isinasaalang-alang ay:

$H = -J\sum\limits_{\langle i,j \rangle} Z_iZ_j + h\sum\limits_iX_i$,

kung saan ang $J>0$ ay naglalarawan ng coupling ng nearest-neighbor spins, $i<j$, at ang global transverse field, $h$, ay nakatakda sa $\frac{\pi}{8}$. Habang lumalayo ang $h$ mula sa isang Clifford angle (ibig sabihin, $\theta=n\frac{\pi}{2}, n \in \mathbb{Z}$), nagiging mas mahirap na ipalaganap ang mga anti-noise generator sa buong circuit.

Para sa pagpili ng observable, isasaalang-alang natin ang average single-site magnetization, $\frac{1}{N} \sum_{i=1}^{N} \langle z_i \rangle$, kung saan ang $N$ ay ang bilang ng mga site.

```python
import numpy as np
from qiskit import QuantumCircuit
from qiskit.quantum_info import Pauli, SparsePauliOp

num_qubits = 30
num_trotter_steps = 10
rx_angle = np.pi / 8

# Avg single-site magnetization
id_pauli = Pauli("I" * num_qubits)
observable = SparsePauliOp([id_pauli.dot(Pauli("Z"), [i]) for i in range(num_qubits)]) / num_qubits

# Implement Trotterized kicked-Ising model
circuit = QuantumCircuit(num_qubits)
for _step in range(num_trotter_steps):
    circuit.rx(rx_angle, range(num_qubits))
    for first_qubit in (1, 2):
        for idx in range(first_qubit, num_qubits, 2):
            # equivalent to Rzz(-pi/2):
            circuit.sdg([idx - 1, idx])
            circuit.cz(idx - 1, idx)
circuit.compose(circuit.inverse(), inplace=True)
circuit.measure_active()
circuit.draw("mpl", fold=-1)
```

### I-twirl ang mga 2-qubit gate layer at measurement at hanapin ang mga natatanging layer

Tinitiyak natin dito na ang pass manager ay nag-anotasyon sa mga kahon gamit ang mga anotasyong ``Twirl`` at ``InjectNoise``, na nagbibigay-daan sa atin na matuto ng noise na makakaapekto sa ating circuit at iugnay ang noise na iyon sa kaukulang circuit layer nito.

- ``enable_gates/enable_measure: True``: I-box ang lahat ng 2q gate layer at terminal measurement. Ang mga single qubit gate ay iiwan na left-dressed sa loob ng mga kahon.
- ``measure_annotations: all`` Isama ang mga anotasyong `Twirl` at `ChangeBasis` sa measurement box
- ``twirling_strategy: active``: I-twirl ang lahat ng aktibong qubit sa bawat kahon na naglalaman ng mga entangling gate
- ``inject_noise_targets: gates``: Dapat idagdag ang mga anotasyong ``InjectNoise`` sa lahat ng ``Twirl``-annotated na kahon na naglalaman ng mga entangling gate
- ``inject_noise_strategy: uniform_modification``: Lahat ng noise layer ay dapat na naka-scale nang pantay-pantay.

```python
from samplomatic.transpiler import generate_boxing_pass_manager

# Box up circuit with Twirl and InjectNoise annotations
pm = generate_boxing_pass_manager(
    enable_gates=True,
    enable_measures=True,
    measure_annotations="all",
    twirling_strategy="active",
    inject_noise_targets="gates",
    inject_noise_strategy="uniform_modification",
    remove_barriers=True,
)
boxed_circuit = pm.run(isa_circuit)
```

```python
draw_circ = QuantumCircuit(boxed_circuit.num_qubits)
```

### Buuin ang template circuit at samplex, tukuyin kung paano sasa-sample ang circuit

Nagdadagdag din tayo dito ng mga spectator at post-selection measurement, na kailangan para maisagawa ang post-selection sa mga sample na output mula sa ``Executor``.

```python
import samplomatic
from qiskit.transpiler import PassManager
from qiskit_addon_utils.noise_management.post_selection.transpiler.passes import (
    AddPostSelectionMeasures,
    AddSpectatorMeasures,
)

# Build template circuit and samplex for later use with the "Executor"
template_circuit, samplex = samplomatic.build(boxed_circuit)

# Add post-selection instructions to the template circuit
post_selection_pm = PassManager(
    [
        AddSpectatorMeasures(backend.coupling_map),
        AddPostSelectionMeasures(x_pulse_type="rx"),
    ]
)
template_circuit = post_selection_pm.run(template_circuit)
```

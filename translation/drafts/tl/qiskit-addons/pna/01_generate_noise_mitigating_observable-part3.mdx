#### I-transform ang mga measurement bases sa canonical form

Susunod, hahanapin natin ang pinakamaliit na set ng mga bases na kailangang sukatin para masaklaw ang bawat Pauli term sa sinusukat na observable (***maraming observables ang maaaring sukatin nang sabay-sabay kung mag-commute sila qubit-wise***). Dahil sinusukat lang natin ang mga term sa ating orihinal na observable, na siyang kabuuan ng lahat ng single-`Z` Paulis, isang basis lang ang kailangan -- ang all-`Z` basis.

Bukod sa paghahanap ng set ng Pauli measurement bases, kailangan din nating i-map ang mga Pauli term na ito sa canonical form na inaasahan ng ``Executor`` primitive. Para sa karagdagang impormasyon tungkol sa canonical qubit ordering, bisitahin ang [samplomatic docs](https://qiskit.github.io/samplomatic/guides/samplex_io.html#qubit-ordering-convention).

```python
from qiskit_addon_utils.exp_vals.measurement_bases import get_measurement_bases

meas_box = boxed_circuit.data[-1]
canonical_qubits = [
    idx for idx, qubit in enumerate(boxed_circuit.qubits) if qubit in meas_box.qubits
]
c_2_p = {c: p for c, p in enumerate(canonical_qubits)}  # canonical -> physical
p_2_v = {p: v for v, p in enumerate(layout)}  # physical -> virtual
c_2_v = {c: p_2_v[p] for c, p in c_2_p.items()}  # canonical -> virtual
meas_bases, bases_reverser = get_measurement_bases(obs_tilde_virtual)
meas_bases_canonical = [
    np.array([base[c_2_v[c]] for c in range(num_qubits)], dtype=np.uint8) for base in meas_bases
]
```

#### Tukuyin kung paano mag-sample sa ``QuantumProgram``

Dito sa ``QuantumProgram`` tinutukoy natin kung paano i-sample ang eksperimento:

- ``template_circuit``: Ang circuit na naglalaman ng lahat ng gates na kailangan para maipatupad ang lahat ng gustong randomizations (mula sa twirling randomizations, parameters, atbp.).
- ``samplex``: Isang object na nagde-define ng probability distribution sa lahat ng posibleng circuit randomizations na isa-sample.
- ``samplex_arguments``: Mga binding na kailangan para ganap na matukoy ang samplex
    - ``basis_changes``: Dito tinutukoy ang set ng mga bases na susukatin na sasaklaw sa lahat ng Pauli terms sa sinusukat na observable.
    - ``noise_scales.ref``: Itinatakda natin ang scale ng bawat noise layer sa `0.0` para maiwasan ang pagpasok ng karagdagang noise sa ating mga sample
    - ``pauli_lindblad_maps``: Kailangan kung ``noise_scales`` ay ipinasa. Ini-map lang nito ang mga noise layer sa kaugnay na noise model.
- ``shape``: Isang shape tuple para palawakin ang implicit shape na tinukoy ng ``samplex_arguments``. Ang mga non-trivial axes na idinagdag ng extension na ito ay nagbibilang ng mga randomization.

```python
from qiskit_ibm_runtime import QuantumProgram

# Control the # of shots during execution
shots_per_randomization_exec = 64
num_randomizations_exec = 6144

# Zero out the noise to prevent noise from being injected during execution.
# We only added InjectNoise annotations so PNA could associate the noise
# to layers in the circuit
samplex_inputs = {f"noise_scales.{ref}": 0.0 for ref in refs_to_noise_models}
samplex_inputs |= {"pauli_lindblad_maps": refs_to_noise_models}

# Specify the bases to measure
bases_broadcastable = np.expand_dims(np.array(meas_bases_canonical), axis=1)
samplex_inputs |= {"basis_changes": {"basis0": bases_broadcastable}}

# Convert samplex_inputs into a dict to pass to QuantumProgram
samplex_arguments = samplex.inputs().make_broadcastable().bind(**samplex_inputs)

# Instantiate the QuantumProgram with the specified parameters
program = QuantumProgram(shots=shots_per_randomization_exec)
program.append(
    circuit=template_circuit,
    samplex=samplex,
    samplex_arguments=samplex_arguments,
    shape=(num_randomizations_exec),
)
```

#### I-sample ang circuit gamit ang ``Executor`` primitive prototype

Ngayong natukoy na natin ang ating ``QuantumProgram``, madali na lang ang pagpapatakbo ng eksperimento. I-instantiate lang natin ang ``Executor`` object, ibigay ang backend, at patakbuhin ang program.

```python
from qiskit_ibm_runtime import Executor

# Execute (sample) the circuit
executor = Executor(backend)
job_exec = executor.run(program)
exec_results = job_exec.result()
```

#### I-post-process ang mga sample para makalkula ang error-mitigated expectation value

Para makalkula ang error-mitigated expectation value, gagawin natin ang sumusunod:

- Kalkulahin ang mga TREX scaling factor batay sa natutunang noise na nakaka-apekto sa mga measurement
- Gumawa ng mask para mapanatili lang ang mga post-selected na sample
- Gamitin ang ``executor_expectation_values`` function mula sa ``qiskit-addon-utils`` para pagsamahin ang lahat ng data sa isang error-mitigated expectation value.

```python
from qiskit_addon_utils.exp_vals.expectation_values import executor_expectation_values

# Computing the TREX factors

    circuit=template_circuit, coupling_map=backend.coupling_map
```

#### Alamin ang noise

Bago tayo mag-run ng mga eksperimento, pinag-aaralan natin ang noise model na nakakaapekto sa mga entangling gate at measurement sa circuit. Kailangan ng tumpak na noise model para epektibong mapigilan ang error. Ang pag-aaral ng noise bago pa man i-execute ang mga eksperimento ay nagbibigay ng pinakamataas na pagkakataon na ang noise model ay talagang tutugma sa aktwal na noise na nakakaapekto sa mga gate habang nag-eexecute.

Bago natin matutunan ang noise, kailangan nating hanapin ang mga natatanging 2-qubit layer sa ating circuit, para mabawasan ang bilang ng mga shot na kailangan para matutunan ang noise ng buong circuit. Ginagamit natin ang ``find_unique_box_instructions`` mula sa ``samplomatic`` para makuha ang mga natatanging layer mula sa boxed circuit, kasama na ang measurement layer. Ito ang mga layer na ipinasa natin sa noise learner.

Kapag alam na natin ang mga layer, matututunan na natin ang noise. May ilang mga parameter na dapat nating isaalang-alang:

- `num_randomizations`: Ang bilang ng mga random circuit na gagamitin bawat learning circuit configuration
- `shots_per_randomization`: Kabuuang bilang ng mga shot na gagamitin bawat random learning circuit
- `layer_pair_depths`: Ang mga circuit depth (sinusukat bilang bilang ng mga pair) na gagamitin sa mga learning experiment.
- `post_selection`: Gagamit tayo ng edge-based post-selection habang nag-aaral gamit ang mga `rx` gate para ipatupad ang mga post-measurement pulse

```python
from qiskit_ibm_runtime.noise_learner_v3.noise_learner_v3 import NoiseLearnerV3
from qiskit_ibm_runtime.options import NoiseLearnerV3Options
from samplomatic.utils import find_unique_box_instructions

# Load noise learner data from a shared job
load_saved_nl_result = True

# Noise learning parameters
num_randomizations_nl = 64
shots_per_randomization_nl = 128
strategy = "edge"
enable_postsel = True
x_pulse_type = "rx"

# Find the unique instructions (layers) from boxed-up circuit
unique_2q_layers_and_meas = find_unique_box_instructions(
    boxed_circuit, normalize_annotations=None, undress_boxes=True
)

noise_learner_params = {
    "num_randomizations": num_randomizations_nl,
    "shots_per_randomization": shots_per_randomization_nl,
    "layer_pair_depths": [1, 2, 4, 8, 12, 16, 24, 32, 40, 48],
    "post_selection": {
        "enable": enable_postsel,
        "strategy": strategy,
        "x_pulse_type": x_pulse_type,
    },
    "experimental": {},
}
# set the options
noise_learner_options = NoiseLearnerV3Options(**noise_learner_params)

# run the noise learner job
noise_learner = NoiseLearnerV3(backend, noise_learner_options)
noise_learner_job = noise_learner.run(unique_2q_layers_and_meas)
noise_learner_result = noise_learner_job.result()

nl_metadata = noise_learner_params | {"layout": layout}
```

#### I-associate ang mga circuit box sa natutunan na noise

Dito, gumagawa tayo ng mapping sa pagitan ng mga ``InjectNoise`` reference ID ng bawat box at ng natutunan na noise model (`PauliLindbladMap`) na nakakaapekto sa mga entangling gate sa loob ng box na iyon.

```python
from samplomatic.annotations import InjectNoise
from samplomatic.utils import get_annotation

# map inject noise refs to pauli lindblad maps
refs_to_noise_models = {}
for instruction, result in zip(unique_2q_layers_and_meas, noise_learner_result, strict=False):
    if inject_noise_annot := get_annotation(instruction.operation, InjectNoise):
        refs_to_noise_models[inject_noise_annot.ref] = result.to_pauli_lindblad_map()
```

#### I-propagate ang observable sa pamamagitan ng natutunan na anti-noise para makakuha ng noise-mitigating observable

Tulad ng tinalakay sa itaas, ginagawa ito sa dalawang hakbang. Una, pino-propagate natin ang isang anti-noise generator hanggang sa dulo ng circuit. Pagkatapos nito, pino-propagate natin ang observable sa pamamagitan ng evolved generator na iyon. Inuulit ang prosesong ito para sa bawat anti-noise generator sa circuit. Sa implementasyong ito, ang bawat generator sa isang partikular na layer ay pino-propagate nang sabay-sabay hanggang sa dulo ng circuit. Bukod dito, ginagamit ang Python multiprocessing para isagawa nang sabay ang forward-propagation ng anti-noise at ang back-propagation ng observable. Pinipigilan nito ang pag-iipon ng mga evolved generator sa memory at pina-maximize din ang paggamit ng compute resources.

Kapag nag-run ng PNA, palagi kang kailangang magbigay ng noisy circuit at observable. Kung ang iyong noisy circuit ay isang boxed circuit na may mga `InjectNoise` annotation, kailangan mong ibigay ang mapping na ginawa natin sa hakbang sa itaas. Maaari rin itong ipasa ang isang hindi boxed na circuit na naglalaman ng mga ``PauliLindbladError`` instruction mula sa ``qiskit-aer``. Sa ganoong kaso, hindi na kailangang ibigay ang ``refs_to_noise_models``. Bukod sa mga pangunahing input, gusto ng mga user na isaalang-alang ang:

- `max_err_terms`: Ang bilang ng mga term na itatago sa bawat anti-noise generator habang ito ay pino-propagate nang pasulong. Ang pagpapahintulot na mas malaki ito ay karaniwang nagpapataas ng katumpakan, ngunit hindi ginagarantiyahan na magiging monotonic ang gawi na ito.
- `max_obs_terms`: Ang bilang ng mga term na itatago sa noise-mitigating observable, $\tilde{O}$, habang ito ay pino-propagate nang pabalik sa pamamagitan ng evolved anti-noise. Ang mas malalaking value ay karaniwang nagpapataas ng katumpakan, ngunit hindi rin ginagarantiyahan na monotonic ito.
- `num_processes`: Ang bilang ng mga core na ilalaan sa proseso. Tandaan, ang mga generator ay pino-propagate nang pasulong at inilalapat sa observable nang sabay-sabay.
- `search_step`: Ang back-propagation step ay gumagamit ng greedy method para approximate na i-conjugate ang dalawang operator sa Pauli basis. Ang method na ito ay maaaring mapabilis sa pamamagitan ng pagpapalaki ng ``search_step``. Tingnan ang [pauli-prop docs](https://qiskit.github.io/pauli-prop/) para sa karagdagang impormasyon.
- `num_to_measure`: Kahit na ang variable na ito ay hindi input sa ``generate_noise_mitigating_observable``, ginagamit natin ito para kontrolin kung ilang term mula sa $\tilde{O}$ ang gusto nating aktwal na sukatin. Dito, sukatin lang natin ang pinakamataas na 30 term, na siyang mga orihinal na term sa ating observable. Ang mga term ay na-rescale na ngayon upang ang pagsukat sa kanila ay magkaroon ng epekto ng pag-mitigate ng natutunan na gate noise. Kahit na 30 lang na term mula sa $\tilde{O}$ ang ating sinusukat, madalas pa rin itong kapaki-pakinabang na hayaang lumaki ito, dahil pinapataas nito ang katumpakan ng mga scaling factor ng mga nangungunang term.

```python
from qiskit_addon_pna import generate_noise_mitigating_observable

# PNA parameters
num_processes = 8
max_err_terms = 10_000
max_obs_terms = 10_000
num_to_measure = num_qubits

obs_tilde_isa = generate_noise_mitigating_observable(
    boxed_circuit,
    isa_observable,
    refs_to_noise_models,
    max_err_terms=max_err_terms,
    max_obs_terms=max_obs_terms,
    num_processes=num_processes,
    print_progress=True,
    search_step=8,
)
p_2_v = {p: v for v, p in enumerate(layout)}
obs_tilde_virtual = SparsePauliOp.from_sparse_list(
    [
        (pstr, [p_2_v[p] for p in p_qubits], coeff)
        for (pstr, p_qubits, coeff) in obs_tilde_isa.to_sparse_list()
    ],
    num_qubits=num_qubits,
)
obs_tilde_virtual = obs_tilde_virtual[np.argsort(np.abs(obs_tilde_virtual.coeffs))[::-1]][
    :num_to_measure
]
```

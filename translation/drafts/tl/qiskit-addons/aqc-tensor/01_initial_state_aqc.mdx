Sa notebook na ito, daanan natin ang mga hakbang ng isang [Qiskit pattern](https://quantum.cloud.ibm.com/docs/guides/intro-to-patterns) habang ginagamit ang **approximate quantum compilation with tensor networks (AQC-Tensor)** para makamit ang mas mababang lalim ng circuit kaysa sa karaniwang kailangan para gawin ang Trotter evolution.

Ito ang mga hakbang na gagawin natin:

- **Hakbang 1: Mag-map sa quantum na problema**
    - I-initialize ang Hamiltonian at observable(s) ng ating problema
    - <font color='#0F62FE'>Gumawa ng target tensor-network state para sa unang bahagi ng circuit</font>
    - <font color='#0F62FE'>Gumawa ng mababang-lalim na circuit na nagtatantya sa bahaging ini-compress</font>
    - <font color='#0F62FE'>Gumawa ng pangkalahatang ansatz mula sa circuit na iyon</font>
    - <font color='#0F62FE'>I-optimize ang mga parameter para mailapit ang ansatz nang hangga't maaari sa target</font>
    - <font color='#0F62FE'>Idagdag ang mga susunod na Trotter step sa na-optimize na ansatz</font>
- **Hakbang 2: I-optimize para sa target na hardware**
    - I-transpile ang circuit para sa hardware
- **Hakbang 3: Magsagawa ng mga eksperimento**
    - Gumamit ng fake backend para sa kaginhawaan
- **Hakbang 4: I-reconstruct ang mga resulta**
    - Hindi naaangkop; sa halip, i-output lang natin ang nasukat na observable
## Hakbang 1: Mag-map sa quantum circuit at operator

### Mag-set up ng modelo ng Hamiltonian at observable

Sa notebook na ito, ginagamit natin ang Ising model sa isang bilog na may 10 site:
$$
\hat{\mathcal{H}}_{\text{Ising}} = \sum_{i=1}^{10} J_{i,(i+1)} Z_i Z_{(i+1)} + h_i X_i \, ,
$$
kung saan ang mga periodic boundary condition ay nagpapahiwatig na para sa $i=10$ nakukuha natin ang $i+1=11\rightarrow1$, ang $J$ ay ang lakas ng coupling sa pagitan ng dalawang site at ang $h$ ay ang panlabas na magnetic field.


```python
from qiskit.transpiler import CouplingMap
from qiskit_addon_utils.problem_generators import generate_xyz_hamiltonian

# Generate some coupling map to use for this example
coupling_map = CouplingMap.from_heavy_hex(3, bidirectional=False)

# Choose a 10-qubit circle on this coupling map
reduced_coupling_map = coupling_map.reduce([0, 13, 1, 14, 10, 16, 4, 15, 3, 9])

# Get a qubit operator describing the Ising field model
hamiltonian = generate_xyz_hamiltonian(
    reduced_coupling_map,
    coupling_constants=(0.0, 0.0, 1.0),
    ext_magnetic_field=(0.4, 0.0, 0.0),
)
```

Ang observable na susukatin natin ay ang kabuuang magnetization.

```python
from qiskit.quantum_info import SparsePauliOp

L = reduced_coupling_map.size()
observable = SparsePauliOp.from_sparse_list([("Z", [i], 1 / L / 2) for i in range(L)], num_qubits=L)
```

### Tukuyin kung gaano kalaki ng time evolution ang isasimulate nang klasikal

Ang ating pangkalahatang layunin ay i-simulate ang time evolution ng modelong Hamiltonian sa itaas. Ginagawa natin ito sa pamamagitan ng Trotter evolution, na hahati-hatiin natin sa dalawang bahagi:

1. Isang unang bahagi na maaaring i-simulate gamit ang matrix product states (MPS). "Ico-compile" natin ang bahaging ito gamit ang AQC ayon sa inilalahad sa https://arxiv.org/abs/2301.08609.
2. Isang kasunod na bahagi ng circuit na isasagawa sa hardware.
Plano nating gamitin ang AQC-Tensor para i-compress ang ating time evolution circuit hanggang sa oras na $t=4$, pagkatapos ay mag-evolve gamit ang mga ordinaryong Trotter step hanggang $t=5$.
### Gumawa ng mga circuit bago at pagkatapos ng split

Ngayong napili na nating hatiin sa $t=4$, gagawa tayo ng dalawang circuit:

1. Isang "target" circuit para sa AQC na bahagi ng evolution, mula $t_i=0$ hanggang $t_f=4$. Dahil ito ay isina-simulate ng isang tensor-network simulator, ang bilang ng mga layer ay nakakaapekto sa oras ng pagpapatakbo sa pamamagitan lamang ng isang constant na factor, kaya maaari na tayong gumamit ng maluwag na bilang ng mga layer para mabawasan ang Trotter error.

```python
from qiskit.synthesis import SuzukiTrotter
from qiskit_addon_utils.problem_generators import generate_time_evolution_circuit

aqc_evolution_time = 4.0
aqc_target_num_trotter_steps = 45

aqc_target_circuit = generate_time_evolution_circuit(
    hamiltonian,
    synthesis=SuzukiTrotter(reps=aqc_target_num_trotter_steps),
    time=aqc_evolution_time,
)
```

2. Isang kasunod na evolution circuit, na nag-e-evolve mula $t_i=4$ hanggang $t_f=5$. Dahil ito ay tatakbo sa quantum hardware, mas mainam na gumamit ng kaunti lang na Trotter layer hangga't maaari.

```python
subsequent_evolution_time = 1.0
subsequent_num_trotter_steps = 5

subsequent_circuit = generate_time_evolution_circuit(
    hamiltonian,
    synthesis=SuzukiTrotter(reps=subsequent_num_trotter_steps),
    time=subsequent_evolution_time,
)
```

Para sa paghahambing mamaya, gumawa rin tayo ng ikatlong circuit: isa na nag-e-evolve para sa `aqc_evolution_time` ngunit may parehong evolution time bawat Trotter step tulad ng kasunod na circuit. Ito ang circuit na gagamitin sana natin kung hindi tayo gumamit ng maluwag na bilang ng Trotter steps para sa target circuit. Ito ang ating itatawagin bilang _comparison circuit_.

```python
aqc_comparison_num_trotter_steps = int(
    subsequent_num_trotter_steps / subsequent_evolution_time * aqc_evolution_time
)
aqc_comparison_num_trotter_steps
```

```python
comparison_circuit = generate_time_evolution_circuit(
    hamiltonian,
    synthesis=SuzukiTrotter(reps=aqc_comparison_num_trotter_steps),
    time=aqc_evolution_time,
)
```

### Gumawa ng ansatz at mga paunang parameter mula sa isang Trotter circuit na may mas kaunting hakbang

Una, gumagawa tayo ng "mabuting" circuit na may parehong evolution time tulad ng target circuit, ngunit may mas kaunting Trotter step (at kaya mas kaunting mga layer).

Pagkatapos ay ipinasa natin ang "mabuting" circuit na ito sa `generate_ansatz_from_circuit` na function ng AQC-Tensor. Sinusuri ng function na ito ang two-qubit connectivity ng circuit at nagbabalik ng dalawang bagay:
1. isang pangkalahatang, parametrized na ansatz circuit na may parehong two-qubit connectivity tulad ng input circuit; at,
2. mga parameter na, kapag ipinasok sa ansatz, nagbubunga ng input (mabuting) circuit.

Sa lalong madaling panahon, kukuhanin natin ang mga parameter na ito at paulit-ulit na ia-adjust ang mga ito para mailapit ang ansatz circuit nang hangga't maaari sa target MPS.

```python
from qiskit_addon_aqc_tensor import generate_ansatz_from_circuit

aqc_ansatz_num_trotter_steps = 5

aqc_good_circuit = generate_time_evolution_circuit(
    hamiltonian,
    synthesis=SuzukiTrotter(reps=aqc_ansatz_num_trotter_steps),
    time=aqc_evolution_time,
)

aqc_ansatz, aqc_initial_parameters = generate_ansatz_from_circuit(
    aqc_good_circuit, qubits_initially_zero=True
)
aqc_ansatz.draw("mpl", fold=-1)
```

### Pumili ng mga setting para sa tensor network simulation

Dito, ginagamit natin ang tensor network simulator na batay sa [quimb](http://quimb.readthedocs.io/). Sa halimbawang ito, ginagamit natin ang matrix-product state (MPS) simulator ng quimb, at ginagamit natin ang [JAX](https://docs.jax.dev/en/latest/) para sa automatic differentiation. Tingnan ang [API documentation](../stubs/qiskit_addon_aqc_tensor.simulation.quimb.QuimbSimulator.rst) para sa karagdagang impormasyon kung paano gamitin ang quimb simulator.

```python
from functools import partial

import quimb.tensor

from qiskit_addon_aqc_tensor.simulation.quimb import QuimbSimulator

simulator_settings = QuimbSimulator(
    partial(quimb.tensor.CircuitMPS, max_bond=100, cutoff=1e-8),
    autodiff_backend="jax",
)
```

### Bumuo ng matrix-product state representation ng AQC target state

Susunod, bumubuo tayo ng matrix-product representation ng state na tatantiyahin ng AQC.

```python
from qiskit_addon_aqc_tensor.simulation import tensornetwork_from_circuit

aqc_target_mps = tensornetwork_from_circuit(aqc_target_circuit, simulator_settings)
```

Tandaan na dahil pumili tayo ng maluwag na bilang ng Trotter step para sa target state, mas kaunti talaga itong Trotter error kaysa sa comparison circuit. Maaari nating kalkulahin ang fidelity ($| \langle \psi_1 | \psi_2 \rangle |^2$) ng state na inihanda ng comparison circuit kumpara sa target state:

```python
from qiskit_addon_aqc_tensor.simulation import compute_overlap

comparison_mps = tensornetwork_from_circuit(comparison_circuit, simulator_settings)
comparison_fidelity = abs(compute_overlap(comparison_mps, aqc_target_mps)) ** 2
comparison_fidelity
```

### I-optimize ang mga parameter ng ansatz gamit ang mga kalkulasyon ng MPS

Dito, mino-minimize natin ang pinakasimpleng posibleng cost function, `MaximizeStateFidelity`, sa pamamagitan ng paggamit ng L-BFGS optimizer mula sa scipy.

Pumipili tayo ng stopping point para sa fidelity na magiging mas mataas kaysa sa magiging resulta ng comparison circuit nang hindi gumagamit ng AQC. Kapag naabot na ito, ang compressed circuit ay may mas kaunting Trotter error _at_ mas mababang lalim kaysa sa orihinal na circuit. Kapag may mas maraming oras sa pagpoproseso, maaaring magsagawa ng karagdagang mga optimization step para mapataas pa ang fidelity.

```python
from scipy.optimize import OptimizeResult, minimize

from qiskit_addon_aqc_tensor.objective import MaximizeStateFidelity

objective = MaximizeStateFidelity(aqc_target_mps, aqc_ansatz, simulator_settings)

stopping_point = 1 - comparison_fidelity

def callback(intermediate_result: OptimizeResult):
    print(f"Intermediate result: Fidelity {1 - intermediate_result.fun:.8}")
    if intermediate_result.fun < stopping_point:
        # Good enough for now
        raise StopIteration

result = minimize(
    objective.loss_function,
    aqc_initial_parameters,
    method="L-BFGS-B",
    jac=True,
    options={"maxiter": 100},
    callback=callback,
)
if result.status not in (
    0,
    1,
    99,
):  # 0 => success; 1 => max iterations reached; 99 => early termination via StopIteration
    raise RuntimeError(f"Optimization failed: {result.message} (status={result.status})")

print(f"Done after {result.nit} iterations.")
aqc_final_parameters = result.x
```

## Hakbang 2: Mag-transpile para sa pagpapatakbo sa target na hardware

Sa Hakbang 2 ng isang [Qiskit pattern](https://quantum.cloud.ibm.com/docs/guides/intro-to-patterns), ini-transpile natin ang circuit na ito at ang anumang nais na observable(s) para sa pagpapatakbo sa isang target na device. Dito ay gumagamit tayo ng fake backend na ibinibigay ng `qiskit-ibm-runtime`.

```python
from qiskit import transpile
from qiskit_ibm_runtime.fake_provider import FakeMelbourneV2

backend = FakeMelbourneV2()

isa_circuit = transpile(final_circuit, backend)
isa_observable = observable.apply_layout(isa_circuit.layout)
```

Ang resultang ISA circuit ay maaaring ipadala para sa pagpapatakbo sa backend (hakbang 3 ng isang [Qiskit pattern](https://quantum.cloud.ibm.com/docs/guides/intro-to-patterns)).
## Hakbang 3: Magsagawa sa quantum hardware

```python
from qiskit_ibm_runtime import EstimatorV2 as Estimator

estimator = Estimator(backend)
job = estimator.run([(isa_circuit, isa_observable)])
pub_result = job.result()[0]
```

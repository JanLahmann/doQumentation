Sa tutorial na ito, ibabalik natin ang mga inaasahang halaga ng isang pitong-qubit na circuit sa pamamagitan ng paghati nito sa dalawang apat na qubit na circuit gamit ang wire cutting.

Ito ang mga hakbang na gagawin natin sa [Qiskit pattern](https://quantum.cloud.ibm.com/docs/guides/intro-to-patterns) na ito:

- **Hakbang 1: I-map ang problema sa mga quantum circuit at operator**:
    - I-map ang hamiltonian sa isang quantum circuit.
- **Hakbang 2: I-optimize para sa target na hardware** [_Gumagamit ng cutting addon_]:
    - <font color='#0F62FE'>I-cut ang circuit at observable.</font>
    - I-transpile ang mga subexperiment para sa hardware.
- **Hakbang 3: Isagawa sa target na hardware**:
    - Patakbuhin ang mga subexperiment na nakuha sa Hakbang 2 gamit ang `Sampler` primitive.
- **Hakbang 4: I-post-process ang mga resulta** [_Gumagamit ng cutting addon_]:
    - <font color='#0F62FE'>Pagsamahin ang mga resulta ng Hakbang 3 para mabawi ang inaasahang halaga ng observable na pinag-uusapan.</font>

## Hakbang 2: I-optimize

### Gumawa ng bagong circuit kung saan nailagay na ang mga `Move` na instruksyon sa mga gustong lokasyon ng cut

Dahil sa circuit sa itaas, gusto naming maglagay ng dalawang wire cut sa gitnang linya ng qubit, para mahati ang circuit sa dalawang circuit na tig-apat na qubit bawat isa. Isang paraan para gawin ito ay ang manu-manong paglalagay ng dalawang-qubit na `Move` na instruksyon na naglilipat ng estado mula sa isang qubit wire papunta sa isa pa. Ang `Move` na instruksyon ay katumbas ng isang reset na operasyon sa ikalawang qubit, na sinusundan ng isang SWAP gate. Ang epekto ng instruksyong ito ay ilipat ang estado ng unang (source) qubit papunta sa ikalawang (destination) qubit, habang itinatatapon ang papasok na estado ng ikalawang qubit. Para gumana ito nang tama, mahalaga na ang ikalawang (destination) qubit ay walang entanglement sa natitirang bahagi ng sistema; kung hindi man, ang reset na operasyon ay magiging sanhi ng bahagyang pagbagsak ng estado ng natitirang bahagi ng sistema.

Dito, gumawa tayo ng bagong circuit na may isang karagdagang qubit at ang mga `Move` na operasyon na nakalagay na. Sa halimbawang ito, nakakagamit ulit tayo ng isang qubit: ang source qubit ng unang `Move` ay nagiging destination qubit ng pangalawang `Move` na operasyon.

Tandaan: Bilang alternatibo sa direktang paggamit ng mga `Move` na instruksyon, maaaring piliin ng isa na markahan ang mga wire cut gamit ang isang single-qubit na `CutWire` na instruksyon. Mayroon ang `cut_wires` na function para i-transform ang mga `CutWire` sa mga `Move` na instruksyon sa mga bagong inilalaang qubit. Gayunpaman, kumpara sa manu-manong paraan, hindi pinapayagan ng awtomatikong paraang ito ang muling paggamit ng mga qubit wire. Tingnan ang `CutWire` na [how-to guide](../how-tos/how_to_specify_cut_wires.ipynb) para sa mga detalye.

```python
from qiskit_addon_cutting.instructions import Move

qc_1 = QuantumCircuit(8)
for i in [*range(4), *range(5, 8)]:
    qc_1.rx(np.pi / 4, i)
qc_1.cx(0, 3)
qc_1.cx(1, 3)
qc_1.cx(2, 3)
qc_1.append(Move(), [3, 4])
qc_1.cx(4, 5)
qc_1.cx(4, 6)
qc_1.cx(4, 7)
qc_1.append(Move(), [4, 3])
qc_1.cx(0, 3)
qc_1.cx(1, 3)
qc_1.cx(2, 3)

qc_1.draw("mpl")
```

### Buuin ang mga subexperiment para patakbuhin sa backend

Tinatanggap ng `generate_cutting_experiments` ang mga argumento na `circuits`/`observables` bilang mga dictionary na nagma-map ng mga qubit partition label sa kani-kanilang `subcircuit`/`subobservables`.

Para ma-simulate ang inaasahang halaga ng full-sized na circuit, maraming subexperiment ang binubuo mula sa joint quasiprobability distribution ng mga decomposed na gate at pagkatapos ay isinasagawa sa isa o higit pang mga backend. Ang bilang ng mga sample na kinukuha mula sa distribution ay kinokontrol ng `num_samples`, at isang pinagsama-samang coefficient ang ibinibigay para sa bawat natatanging sample. Para sa karagdagang impormasyon kung paano kinakalkula ang mga coefficient, sumangguni sa [mga paliwanag na materyal](../explanation/index.rst).

```python
from qiskit_addon_cutting import generate_cutting_experiments

subexperiments, coefficients = generate_cutting_experiments(
    circuits=subcircuits, observables=subobservables, num_samples=np.inf
)
```

### Pumili ng backend

Gumagamit tayo dito ng isang pekeng backend, na magreresulta sa Qiskit Runtime na tatakbo sa lokal na mode (ibig sabihin, sa isang lokal na simulator).

```python
from qiskit_ibm_runtime.fake_provider import FakeManilaV2

backend = FakeManilaV2()
```

### Ihanda ang mga subexperiment para sa backend

Kailangan nating i-transpile ang mga circuit na may aming backend bilang target bago isumite ang mga ito sa Qiskit Runtime.

```python
from qiskit.transpiler import generate_preset_pass_manager

# Transpile the subexperiments to ISA circuits
pass_manager = generate_preset_pass_manager(optimization_level=1, backend=backend)
isa_subexperiments = {
    label: pass_manager.run(partition_subexpts)
    for label, partition_subexpts in subexperiments.items()
}
```

## Hakbang 3: Isagawa

### Patakbuhin ang mga subexperiment gamit ang Qiskit Runtime Sampler primitive

```python
from qiskit_ibm_runtime import SamplerV2, Batch

# Submit each partition's subexperiments to the Qiskit Runtime Sampler
# primitive, in a single batch so that the jobs will run back-to-back.
with Batch(backend=backend) as batch:
    sampler = SamplerV2(mode=batch)
    jobs = {
        label: sampler.run(subsystem_subexpts, shots=2**12)
        for label, subsystem_subexpts in isa_subexperiments.items()
    }
```

## Hakbang 4: I-post-process

### Bawiin ang inaasahang halaga

Bawiin ang mga inaasahang halaga para sa bawat termino ng observable at pagsamahin ang mga ito para mabawi ang inaasahang halaga para sa orihinal na observable.

```python
from qiskit_addon_cutting import reconstruct_expectation_values

reconstructed_expval_terms = reconstruct_expectation_values(
    results,
    coefficients,
    subobservables,
)
reconstructed_expval = np.dot(reconstructed_expval_terms, observable.coeffs)
```

### Ihambing ang binawing inaasahang halaga sa eksaktong inaasahang halaga mula sa orihinal na circuit at observable

```python
from qiskit_aer.primitives import EstimatorV2

estimator = EstimatorV2()
exact_expval = estimator.run([(qc_0, observable)]).result()[0].data.evs
print(f"Reconstructed expectation value: {np.real(np.round(reconstructed_expval, 8))}")
print(f"Exact expectation value: {np.round(exact_expval, 8)}")
print(f"Error in estimation: {np.real(np.round(reconstructed_expval-exact_expval, 8))}")
print(
    f"Relative error in estimation: {np.real(np.round((reconstructed_expval-exact_expval) / exact_expval, 8))}"
)
```

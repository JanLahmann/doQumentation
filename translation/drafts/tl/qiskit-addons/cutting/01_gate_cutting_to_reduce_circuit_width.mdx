Sa notebook na ito, dadaanan natin ang mga hakbang ng isang [Qiskit pattern](https://quantum.cloud.ibm.com/docs/guides/intro-to-patterns) habang ginagamit ang **circuit cutting** para mabawasan ang bilang ng mga qubit sa isang circuit.  Magcu-cut tayo ng mga gate para ma-reconstruct ang expectation value ng isang four-qubit circuit gamit lang ang mga two-qubit na eksperimento.

Ito ang mga hakbang na gagawin natin:

- **Hakbang 1: I-map ang problema sa mga quantum circuit at operator**:
    - I-map ang hamiltonian sa isang quantum circuit.
- **Hakbang 2: I-optimize para sa target na hardware** [_Ginagamit ang cutting addon_]:
    - <font color='#0F62FE'>I-cut ang circuit at observable.</font>
    - I-transpile ang mga subexperiment para sa hardware.
- **Hakbang 3: Isagawa sa target na hardware**:
    - Patakbuhin ang mga subexperiment na nakuha sa Hakbang 2 gamit ang `Sampler` primitive.
- **Hakbang 4: I-post-process ang mga resulta** [_Ginagamit ang cutting addon_]:
    - <font color='#0F62FE'>Pagsamahin ang mga resulta ng Hakbang 3 para ma-reconstruct ang expectation value ng observable na pinag-uusapan.</font>

## Hakbang 2: I-optimize

### Paghiwalayin ang circuit at observable ayon sa tinukoy na partisyon ng mga qubit

Ang bawat label sa `partition_labels` ay tumutugma sa `circuit` qubit na nasa parehong index. Ang mga qubit na may parehong partition label ay ipipin-grupo, at ang mga non-local na gate na sumasaklaw sa mahigit isang partisyon ay icu-cut.

**Tandaan:** Ang ``observables`` kwarg sa `partition_problem` ay may uri na `PauliList`. Ang mga coefficient at phase ng observable term ay binabalewala sa panahon ng decomposition ng problema at pagpapatupad ng mga subexperiment. Maaari itong muling ilapat sa panahon ng reconstruction ng expectation value.

```python
from qiskit_addon_cutting import partition_problem

partitioned_problem = partition_problem(
    circuit=qc, partition_labels="AABB", observables=observable.paulis
)
subcircuits = partitioned_problem.subcircuits
subobservables = partitioned_problem.subobservables
bases = partitioned_problem.bases
```

### Buuin ang mga subexperiment na ipapatakbo sa backend

Tinatanggap ng `generate_cutting_experiments` ang mga `circuits`/`observables` args bilang mga dictionary na nag-ma-map ng mga qubit partition label sa kani-kanilang `subcircuit`/`subobservables`.

Para ma-simulate ang expectation value ng full-sized na circuit, maraming subexperiment ang ginagawa mula sa joint quasiprobability distribution ng mga na-decompose na gate at pagkatapos ay isinasagawa sa isa o higit pang mga backend. Ang bilang ng mga sample na kinukuha mula sa distribution ay kinokontrol ng `num_samples`, at isang pinagsama-samang coefficient ang ibinibigay para sa bawat natatanging sample. Para sa karagdagang impormasyon kung paano kinakalkula ang mga coefficient, tingnan ang [explanatory material](../explanation/index.rst).

```python
from qiskit_addon_cutting import generate_cutting_experiments

subexperiments, coefficients = generate_cutting_experiments(
    circuits=subcircuits, observables=subobservables, num_samples=np.inf
)
```

### Pumili ng backend

Gumagamit tayo dito ng fake backend, na magreresulta sa pagpapatakbo ng Qiskit Runtime sa local mode (ibig sabihin, sa isang local simulator).

```python
from qiskit_ibm_runtime.fake_provider import FakeManilaV2

backend = FakeManilaV2()
```

### Ihanda ang mga subexperiment para sa backend

Kailangan nating i-transpile ang mga circuit gamit ang ating backend bilang target bago isumite ang mga ito sa Qiskit Runtime.

```python
from qiskit.transpiler import generate_preset_pass_manager

# Transpile the subexperiments to ISA circuits
pass_manager = generate_preset_pass_manager(optimization_level=1, backend=backend)
isa_subexperiments = {
    label: pass_manager.run(partition_subexpts)
    for label, partition_subexpts in subexperiments.items()
}
```

## Hakbang 3: Isagawa

### Patakbuhin ang mga subexperiment gamit ang Qiskit Runtime Sampler primitive

```python
from qiskit_ibm_runtime import SamplerV2, Batch

# Submit each partition's subexperiments to the Qiskit Runtime Sampler
# primitive, in a single batch so that the jobs will run back-to-back.
with Batch(backend=backend) as batch:
    sampler = SamplerV2(mode=batch)
    jobs = {
        label: sampler.run(subsystem_subexpts, shots=2**12)
        for label, subsystem_subexpts in isa_subexperiments.items()
    }
```

## Hakbang 4: I-post-process

### I-reconstruct ang expectation value

I-reconstruct ang mga expectation value para sa bawat observable term at pagsamahin ang mga ito para ma-reconstruct ang expectation value para sa orihinal na observable.

```python
from qiskit_addon_cutting import reconstruct_expectation_values

# Get expectation values for each observable term
reconstructed_expval_terms = reconstruct_expectation_values(
    results,
    coefficients,
    subobservables,
)

# Reconstruct final expectation value
reconstructed_expval = np.dot(reconstructed_expval_terms, observable.coeffs)
```

### Ikumpara ang na-reconstruct na expectation value sa eksaktong expectation value mula sa orihinal na circuit at observable

```python
from qiskit_aer.primitives import EstimatorV2

estimator = EstimatorV2()
exact_expval = estimator.run([(qc, observable)]).result()[0].data.evs
print(f"Reconstructed expectation value: {np.real(np.round(reconstructed_expval, 8))}")
print(f"Exact expectation value: {np.round(exact_expval, 8)}")
print(f"Error in estimation: {np.real(np.round(reconstructed_expval-exact_expval, 8))}")
print(
    f"Relative error in estimation: {np.real(np.round((reconstructed_expval-exact_expval) / exact_expval, 8))}"
)
```

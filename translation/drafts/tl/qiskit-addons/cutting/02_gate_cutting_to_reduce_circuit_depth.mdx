Sa tutorial na ito, babawasan natin ang lalim ng isang circuit sa pamamagitan ng pagputol ng mga malalayong gate, na maiiwasan ang mga swap gate na idadagdag ng routing.

Ito ang mga hakbang na gagawin natin sa [pattern na ito ng Qiskit](https://quantum.cloud.ibm.com/docs/guides/intro-to-patterns):

- **Hakbang 1: I-map ang problema sa mga quantum circuit at operator**:
    - I-map ang hamiltonian sa isang quantum circuit.
- **Hakbang 2: I-optimize para sa target na hardware** [_Gumagamit ng cutting addon_]:
    - <font color='#0F62FE'>Putulin ang circuit at observable.</font>
    - I-transpile ang mga subexperiment para sa hardware.
- **Hakbang 3: Isagawa sa target na hardware**:
    - Patakbuhin ang mga subexperiment mula sa Hakbang 2 gamit ang `Sampler` primitive.
- **Hakbang 4: I-post-process ang mga resulta** [_Gumagamit ng cutting addon_]:
    - <font color='#0F62FE'>Pagsamahin ang mga resulta ng Hakbang 3 para ma-reconstruct ang expectation value ng observable.</font>

## Hakbang 2: I-optimize

### Tukuyin ang isang backend

Puwede kang magbigay ng fake backend o hardware backend mula sa Qiskit Runtime.

```python
from qiskit_ibm_runtime.fake_provider import FakeManilaV2

backend = FakeManilaV2()
```

### I-transpile ang circuit, tingnan ang mga swap, at tandaan ang lalim

Pumipili tayo ng layout na nangangailangan ng dalawang swap para maisagawa ang mga gate sa pagitan ng qubit 3 at 0, at isa pang dalawang swap para maibalik ang mga qubit sa kanilang mga paunang posisyon.

```python
from qiskit.transpiler import generate_preset_pass_manager

pass_manager = generate_preset_pass_manager(
    optimization_level=1, backend=backend, initial_layout=[0, 1, 2, 3]
)

transpiled_qc = pass_manager.run(circuit)
print(f"Transpiled circuit depth: {transpiled_qc.depth(lambda x: len(x.qubits) >= 2)}")
```

```text

```

transpiled_qc.draw("mpl", scale=0.4, idle_wires=False, fold=-1)

### Palitan ang mga malalayong gate ng `TwoQubitQPDGate` sa pamamagitan ng pagtukoy ng kanilang mga index

Papalitan ng `cut_gates` ang mga gate sa mga tinukoy na index ng `TwoQubitQPDGate`, at magbabalik din ito ng listahan ng mga `QPDBasis` na instance — isa para sa bawat gate decomposition.

```python
from qiskit_addon_cutting import cut_gates

# Find the indices of the distant gates
cut_indices = [
    i
    for i, instruction in enumerate(circuit.data)
    if {circuit.find_bit(q)[0] for q in instruction.qubits} == {0, 3}
]

# Decompose distant CNOTs into TwoQubitQPDGate instances
qpd_circuit, bases = cut_gates(circuit, cut_indices)

qpd_circuit.draw("mpl", scale=0.8)
```

![Quantum circuit diagram](./_02_gate_cutting_to_reduce_circuit_depth_imgs/output_3.png)

### Buuin ang mga subexperiment para patakbuhin sa backend

Tinatanggap ng `generate_cutting_experiments` ang isang circuit na naglalaman ng mga `TwoQubitQPDGate` at mga observable bilang `PauliList`.

Para ma-simulate ang expectation value ng buong circuit, maraming subexperiment ang binubuo mula sa joint quasiprobability distribution ng mga na-decompose na gate at isinasagawa sa isa o higit pang mga backend. Kinokontrol ng `num_samples` ang bilang ng mga sample na kinukuha mula sa distribution, at isang pinagsamang coefficient ang ibinibigay para sa bawat natatanging sample. Para sa karagdagang impormasyon kung paano kinakalkula ang mga coefficient, tingnan ang [explanatory material](../explanation/index.rst).

**Tandaan:** Ang ``observables`` kwarg ng `generate_cutting_experiments` ay may uri na `PauliList`. Ang mga coefficient at phase ng observable term ay hindi isinasaalang-alang sa panahon ng decomposition ng problema at pagpapatupad ng mga subexperiment. Maaari itong muling mailapat sa panahon ng reconstruction ng expectation value.

```python
import numpy as np
from qiskit_addon_cutting import generate_cutting_experiments

# Generate the subexperiments and sampling coefficients
subexperiments, coefficients = generate_cutting_experiments(
    circuits=qpd_circuit, observables=observable.paulis, num_samples=np.inf
)
```

### Kalkulahin ang sampling overhead para sa mga napiling cut

Dito nagputol tayo ng tatlong CNOT gate, na nagresulta sa sampling overhead na $9^3$.

Para sa karagdagang impormasyon tungkol sa sampling overhead na dulot ng circuit cutting, tingnan ang [explanatory material](../explanation/index.rst).

```python
print(f"Sampling overhead: {np.prod([basis.overhead for basis in bases])}")
```

```text
Sampling overhead: 729.0
```

### Ipakita na ang mga QPD subexperiment ay magiging mas mababaw pagkatapos putulin ang mga malalayong gate

Narito ang isang halimbawa ng arbitrary na piniling subexperiment na nabuo mula sa QPD circuit. Ang lalim nito ay nabawasan ng mahigit kalahati. Maraming probabilistic na subexperiment ang kailangang buuin at suriin para ma-reconstruct ang expectation value ng mas malalim na circuit.

```python
# Transpile the decomposed circuit to the same layout
transpiled_qpd_circuit = pass_manager.run(subexperiments[100])

print(
    f"Original circuit depth after transpile: {transpiled_qc.depth(lambda x: len(x.qubits) >= 2)}"
)
print(
    f"QPD subexperiment depth after transpile: {transpiled_qpd_circuit.depth(lambda x: len(x.qubits) >= 2)}"
)
transpiled_qpd_circuit.draw("mpl", scale=0.8, idle_wires=False, fold=-1)
```

```text
Original circuit depth after transpile: 30
QPD subexperiment depth after transpile: 7
```

### Ihanda ang mga subexperiment para sa backend

```python
# Transpile the subeperiments to the backend's instruction set architecture (ISA)
isa_subexperiments = pass_manager.run(subexperiments)
```

## Hakbang 3: Isagawa

### Patakbuhin ang mga subexperiment gamit ang Qiskit Runtime Sampler primitive

```python
from qiskit_ibm_runtime import SamplerV2

# Set up the Qiskit Runtime Sampler primitive.  For a fake backend, this will use a local simulator.
sampler = SamplerV2(backend)

# Submit the subexperiments
job = sampler.run(isa_subexperiments)
```

```python
# Retrieve the results
results = job.result()
```

## Hakbang 4: I-post-process

### I-reconstruct ang expectation value

I-reconstruct ang mga expectation value para sa bawat observable term at pagsamahin ang mga ito para ma-reconstruct ang expectation value ng orihinal na observable.

```python
from qiskit_addon_cutting import reconstruct_expectation_values

reconstructed_expval_terms = reconstruct_expectation_values(
    results,
    coefficients,
    observable.paulis,
)
# Reconstruct final expectation value
reconstructed_expval = np.dot(reconstructed_expval_terms, observable.coeffs)
```

### Ikumpara ang na-reconstruct na expectation value sa eksaktong expectation value mula sa orihinal na circuit at observable

```python
from qiskit_aer.primitives import EstimatorV2

estimator = EstimatorV2()
exact_expval = estimator.run([(circuit, observable)]).result()[0].data.evs
print(f"Reconstructed expectation value: {np.real(np.round(reconstructed_expval, 8))}")
print(f"Exact expectation value: {np.round(exact_expval, 8)}")
print(f"Error in estimation: {np.real(np.round(reconstructed_expval-exact_expval, 8))}")
print(
    f"Relative error in estimation: {np.real(np.round((reconstructed_expval-exact_expval) / exact_expval, 8))}"
)
```

```text
Reconstructed expectation value: 0.44018555
Exact expectation value: 0.50497603
Error in estimation: -0.06479049
Relative error in estimation: -0.12830408
```

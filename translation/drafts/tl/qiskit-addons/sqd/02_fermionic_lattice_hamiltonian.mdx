Sa tutorial na ito, ipinapatupad natin ang isang [Qiskit pattern](https://quantum.cloud.ibm.com/docs/guides/intro-to-patterns) na nagpapakita kung paano i-post-process ang mga maingay na quantum sample para mahanap ang isang pagtatantya ng ground state ng isang Fermionic lattice Hamiltonian na kilala bilang single-impurity Anderson model. Susundin natin ang isang sample-based quantum diagonalization na paraan para iproseso ang mga sample mula sa isang set ng ``16``-qubit na Krylov basis states sa paglipas ng papalaking agwat ng oras. Ang mga state na ito ay naisasagawa sa quantum device gamit ang Trotterization ng time evolution. Para matugunan ang epekto ng quantum noise, ginagamit ang configuration recovery technique. Sa pagpapalagay ng isang magandang initial state at sparsity ng ground state, [napatunayan na ang pamamaraang ito ay mag-co-converge nang mahusay](https://arxiv.org/abs/2501.09702).

Ang pattern ay maaaring ilarawan sa apat na hakbang:

1. **Hakbang 1: Mag-map sa quantum problem**
    - Gumawa ng isang set ng Krylov basis states (ibig sabihin, Trotterized time-evolution circuits) sa paglipas ng papalaking agwat ng oras para sa pagtatantya ng ground state
2. **Hakbang 2: I-optimize ang problema**
    - I-transpile ang mga circuit para sa backend
3. **Hakbang 3: Magsagawa ng mga eksperimento**
    - Mag-draw ng mga sample mula sa mga circuit gamit ang ``Sampler`` primitive
4. **Hakbang 4: I-post-process ang mga resulta**
   - Self-consistent configuration recovery loop
       - I-post-process ang buong set ng bitstring samples, gamit ang naunang kaalaman sa bilang ng particle at ang average orbital occupancy na kinakalkula sa pinakabagong iteration
       - Probabilistically gumawa ng mga batch ng subsamples mula sa mga na-recover na bitstrings
       - I-project at i-diagonalize ang Fermionic lattice Hamiltonian sa bawat sampled subspace
       - I-save ang pinakamababang ground state energy na nahanap sa lahat ng batch at i-update ang avg orbital occupancy
### Hakbang 1: Mag-map ng problema sa isang quantum circuit
Una, gagawa tayo ng one- at two-body Hamiltonians na naglalarawan ng one-dimensional single-impurity Anderson model (SIAM) na may ``7`` bath sites (``8`` electron sa ``8`` orbital). Ginagamit ang modelong ito para ilarawan ang mga magnetic impurity na nakalagay sa loob ng mga metal.

Pagkatapos ay gagawa tayo ng ``16``-qubit na Trotter circuits na ginagamit para makabuo ng quantum Krylov subspace.

```python
import numpy as np

n_bath = 7  # number of bath sites

V = 1  # hybridization amplitude
t = 1  # bath hopping amplitude
U = 10  # Impurity onsite repulsion
eps = -U / 2  # Chemical potential for the impurity

# Place the impurity on the first qubit
impurity_index = 0

# One body matrix elements in the "position" basis
h1e = -t * np.diag(np.ones(n_bath), k=1) - t * np.diag(np.ones(n_bath), k=-1)
h1e[impurity_index, impurity_index + 1] = -V
h1e[impurity_index + 1, impurity_index] = -V
h1e[impurity_index, impurity_index] = eps

# Two body matrix elements in the "position" basis
h2e = np.zeros((n_bath + 1, n_bath + 1, n_bath + 1, n_bath + 1))
h2e[impurity_index, impurity_index, impurity_index, impurity_index] = U
```

Susunod, gagawa tayo ng quantum Krylov subspace gamit ang isang set ng Trotterized quantum circuits. Dito ay gumagawa tayo ng mga helper para sa pagbuo ng initial (reference) state pati na rin ang time evolution para sa one- at two-body na bahagi ng Hamiltonian. Para sa mas detalyadong paglalarawan ng modelong ito at kung paano idinisenyo ang mga circuit, mangyaring sumangguni sa [papel](https://arxiv.org/abs/2501.09702).

```python
import ffsim
import scipy
from qiskit import QuantumCircuit, QuantumRegister
from qiskit.circuit.library import CPhaseGate, XGate, XXPlusYYGate

n_modes = n_bath + 1
nelec = (n_modes // 2, n_modes // 2)

dt = 0.2
Utar = scipy.linalg.expm(-1j * dt * h1e)

# The reference state
def initial_state(q_circuit, norb, nocc):
    """Prepare an initial state."""
    for i in range(nocc):
        q_circuit.append(XGate(), [i])
        q_circuit.append(XGate(), [norb + i])
    rot = XXPlusYYGate(np.pi / 2, -np.pi / 2)

    for i in range(3):
        for j in range(nocc - i - 1, nocc + i, 2):
            q_circuit.append(rot, [j, j + 1])
            q_circuit.append(rot, [norb + j, norb + j + 1])
    q_circuit.append(rot, [j + 1, j + 2])
    q_circuit.append(rot, [norb + j + 1, norb + j + 2])

# The one-body time evolution
free_fermion_evolution = ffsim.qiskit.OrbitalRotationJW(n_modes, Utar)

# The two-body time evolution
def append_diagonal_evolution(dt, U, impurity_qubit, num_orb, q_circuit):
    """Append two-body time evolution to a quantum circuit."""
    if U != 0:
        q_circuit.append(
            CPhaseGate(-dt / 2 * U),
            [impurity_qubit, impurity_qubit + num_orb],
        )
```

Gumawa ng ``d`` na time-evolved states na tumutukoy sa quantum Krylov subspace. Dito, pinili nating ``d=8``. Ang error mula sa pag-sample ng Krylov basis states ay nag-co-converge habang tumataas ang ``d``. Tandaan na ang mga partikular ng problem instance na ito ay nagpapahintulot ng mahusay na compilation ng one-body evolution gamit ang `OrbitalRotationJW`; gayunpaman, sa pangkalahatan, maaaring gumamit ng Qiskit's [PauliEvolutionGate](https://quantum.cloud.ibm.com/docs/api/qiskit/qiskit.circuit.library.PauliEvolutionGate) para ipatupad ang Trotterized time evolution ng buong Hamiltonian.

```python
# Generate the initial state
qubits = QuantumRegister(2 * n_modes, name="q")
init_state = QuantumCircuit(qubits)
initial_state(init_state, n_modes, n_modes // 2)
init_state.draw("mpl", scale=0.4, fold=-1)

d = 8  # Number of Krylov basis states
circuits = []
for i in range(d):
    circ = init_state.copy()
    circuits.append(circ)
    for _ in range(i):
        append_diagonal_evolution(dt, U, impurity_index, n_modes, circ)
        circ.append(free_fermion_evolution, qubits)
        append_diagonal_evolution(dt, U, impurity_index, n_modes, circ)
    circ.measure_all()
```

### Hakbang 2: I-optimize ang problema
Pagkatapos nating gawin ang mga Trotterized circuit, io-optimize natin ang mga ito para sa isang target na hardware. Kailangan nating piliin ang hardware device na gagamitin bago ang optimization. Gagamit tayo ng fake na 127-qubit backend mula sa ``qiskit_ibm_runtime`` para gayahin ang isang tunay na device. Para tumakbo sa isang tunay na QPU, palitan lang ang fake backend ng isang tunay na backend. Tingnan ang [Qiskit IBM Runtime docs](https://quantum.cloud.ibm.com/docs/guides/get-started-with-primitives#get-started-with-sampler) para sa karagdagang impormasyon.

```python
from qiskit_ibm_runtime.fake_provider.backends import FakeSherbrooke

backend = FakeSherbrooke()
```

Susunod, ita-transpile natin ang mga circuit sa target na backend gamit ang Qiskit.

```python
from qiskit.transpiler import generate_preset_pass_manager

# The circuit needs to be transpiled to the AerSimulator target
pass_manager = generate_preset_pass_manager(optimization_level=3, backend=backend)
isa_circuits = pass_manager.run(circuits)
```

### Hakbang 3: Magsagawa ng mga eksperimento
Pagkatapos i-optimize ang mga circuit para sa hardware execution, handa na tayong patakbuhin ang mga ito sa target na hardware at mangolekta ng mga sample para sa ground state energy estimation. Dito ay gumagamit tayo ng ``SamplerV2`` mula sa ``qiskit-ibm-runtime`` para gayahin ang maingay na mga sample mula sa ``ibm_sherbrooke`` backend. Pagkatapos ay pinagsasama natin ang mga counts mula sa bawat isa sa mga Krylov basis states sa isang solong counts dictionary at ini-plot ang nangungunang 20 pinakakaraniwang na-sample na bitstrings.

***Tandaan: Kailangan ang [Qiskit Aer](https://qiskit.github.io/qiskit-aer/index.html) para gayahin ang mga sample mula sa mga transpiled circuit.***

```python
from qiskit.primitives import BitArray
from qiskit.visualization import plot_histogram
from qiskit_ibm_runtime import SamplerV2 as Sampler

# Sample from the circuits
noisy_sampler = Sampler(backend, options={"simulator": {"seed_simulator": 24}})
job = noisy_sampler.run(isa_circuits, shots=500)

# Combine the counts from the individual Trotter circuits
bit_array = BitArray.concatenate_shots([result.data.meas for result in job.result()])

plot_histogram(bit_array.get_counts(), number_to_keep=20)
```

### Hakbang 4: I-post-process ang mga resulta
Ngayon, pinapatakbo natin ang SQD algorithm gamit ang `diagonalize_fermionic_hamiltonian` function. Tingnan ang [API documentation](../apidocs/qiskit_addon_sqd.fermion.rst#qiskit_addon_sqd.fermion.diagonalize_fermionic_hamiltonian) para sa mga paliwanag ng mga argumento ng function na ito.

```python
from qiskit_addon_sqd.fermion import SCIResult, diagonalize_fermionic_hamiltonian

# List to capture intermediate results
result_history = []

def callback(results: list[SCIResult]):
    result_history.append(results)
    iteration = len(result_history)
    print(f"Iteration {iteration}")
    for i, result in enumerate(results):
        print(f"\tSubsample {i}")
        print(f"\t\tEnergy: {result.energy}")
        print(f"\t\tSubspace dimension: {np.prod(result.sci_state.amplitudes.shape)}")

rng = np.random.default_rng(24)
result = diagonalize_fermionic_hamiltonian(
    h1e,
    h2e,
    bit_array,
    samples_per_batch=300,
    norb=n_modes,
    nelec=nelec,
    num_batches=3,
    max_iterations=10,
    symmetrize_spin=True,
    callback=callback,
    seed=rng,
)
```

Ngayon, ipo-plot natin ang mga resulta. Ang unang plot ay nagpapakita na pagkatapos ng ilang iteration, nakukuha natin ang eksaktong ground state energy.

Ang halimbawang ito ay maliit lang kaya kaya nating tuklasin ang buong Hilbert space, gaya ng makikita sa mga print statement sa itaas. Tandaan, ang buong Hilbert space ay may dimensyon na ``(num_orbitals choose nelec_a) * (num_orbitals choose nelec_b)``. Kaya para sa problemang ito: `(8 choose 4)**2 = 4900`. Ang mga dimensyon ng subspace ay lumalaki dahil sa pinahusay na configuration recovery, at dahil din sa katotohanan na ang SQD algorithm ay nagdadala ng mahahalagang configuration mula sa isang iteration patungo sa susunod. Sa huling iteration, nagda-diagonalize na tayo sa buong Hilbert space.

Ang ikalawang plot ay nagpapakita ng average occupancy ng bawat spatial orbital sa lahat ng solusyon ng mga batch. Makikita natin na may mataas na posibilidad na ang bawat orbital ay naglalaman ng isang electron.

```python
import matplotlib.pyplot as plt

exact_energy = -13.422491814605827
min_es = [min(result, key=lambda res: res.energy).energy for result in result_history]
min_id, min_e = min(enumerate(min_es), key=lambda x: x[1])

# Data for energies plot
x1 = range(len(result_history))
yt1 = list(np.arange(-13.5, -13.1, 0.1))
ytl = [f"{i:.1f}" for i in yt1]

# Data for avg spatial orbital occupancy
y2 = np.sum(result.orbital_occupancies, axis=0)
```

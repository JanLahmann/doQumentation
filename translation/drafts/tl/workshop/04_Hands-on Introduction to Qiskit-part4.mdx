## 3.1 Amplitude encoding

Ngayong nakita na natin kung paano gumawa ng quantum circuits,흥미로운 na tuklasin kung paano natin ma-encode ang klasikal na impormasyon sa mga quantum state. Isang makapangyarihang paraan ang amplitude encoding, kung saan ang mga amplitude ng isang quantum state ang kumakatawan sa mga component ng isang klasikal na vector.

Tingnan natin ang isang simpleng halimbawa. Sabihin nating gusto nating i-encode ang klasikal na vector

$$
\vec{x} = \begin{bmatrix} x_0 \\ x_1 \\ x_2 \\ x_3 \end{bmatrix}
$$

sa isang quantum state ng dalawang qubit. Ang layunin ay ihanda ang quantum state:

$$
\ket{\psi} = x_0\ket{00} + x_1\ket{01} + x_2\ket{10} + x_3\ket{11}
$$
kung saan $ x_0, x_1, x_2, x_3 \in \mathbb{R} $ (o $ \mathbb{C} $) at ang vector ay na-normalize para:

$$
|x_0|^2 + |x_1|^2 + |x_2|^2 + |x_3|^2 = 1
$$

Ngayon, titingnan natin ang partikular na halimbawa: $ \vec{x} = [0.8924,  0.3696, 0.2391, 0.0990] $

Ang katumbas na quantum state ay:

$$
\begin{aligned}
\ket{\psi} &= 0.8924\,\ket{00}
+ 0.3696\,\ket{01}
+ 0.2391\,\ket{10}
+ 0.0990\,\ket{11}
\end{aligned}
$$
Ang state na ito ay maaaring ihanda gamit ang kombinasyon ng mga rotation gate na $R_y$ na may mga anggulo na $\pi/6$ at $\pi/4$ para sa qubit 0 at 1 ayon sa pagkakasunod

```python
from qiskit import QuantumCircuit
from qiskit_aer import AerSimulator
import numpy as np

qc = QuantumCircuit(2)

qc.ry(np.pi / 6, 0)
qc.ry(np.pi / 4, 1)

simulator = AerSimulator()
qc.save_statevector()
result = simulator.run(qc).result()
statevector = result.get_statevector()

print("Statevector:", statevector)
qc.draw(output="mpl")
```

```text
             0.09904576+0.j],
            dims=(2, 2))
```

## 3.2 Angle encoding at parametrized circuits

Isang partikular na kawili-wiling paraan ng pag-encode ng impormasyon sa isang quantum computer ay ang pagdidisenyo ng isang quantum circuit na naglalaman ng ilang rotational angle na $\vec{\theta}$ o mga parameter na maaaring i-tune para kumatawan sa isang pamilya ng mga function na $f(\vec{\theta})$. Halimbawa, tingnan natin ang sumusunod na parametrized quantum circuit:

```python
from qiskit import QuantumCircuit
from qiskit.circuit import Parameter

# Define a symbolic parameter
theta = Parameter("θ")

qc = QuantumCircuit(2)
# We applied a parametrized RX gate
qc.rx(theta, 0)
qc.cx(0, 1)
qc.draw("mpl")
```

![Quantum circuit diagram](./_04_Hands-on Introduction to Qiskit_imgs/output_25.png)

Sa matematika, maaari nating suriin kung anong pamilya ng mga function ang maaari nating katawanin gamit ang circuit na ito:

$$ 
\text{CNOT}_{01} \, R_x^{\{0\}}(\theta) |00\rangle = \text{CNOT}_{01} \left( \cos(\theta/2)\ket{00} - i\sin(\theta/2)\ket{10} \right) = \cos(\theta/2)\ket{00} - i\sin(\theta/2)\ket{11}
$$
Malinaw na limitado ang bilang ng mga state na maaari nating katawanin gamit ang quantum circuit na ito, dahil hindi natin makukuha ang mga state na $\ket{10}$ o $\ket{01}$ halimbawa. Gayunpaman, lumalaki ang pamilya ng mga state na maaari nating katawanin kapag nagdagdag tayo ng mas maraming rotasyon sa tamang mga lugar:

```python
from qiskit import QuantumCircuit
from qiskit.circuit import Parameter

# Define a symbolic parameter
theta1 = Parameter("θ1")
theta2 = Parameter("θ2")

qc = QuantumCircuit(2)
qc.rx(theta1, 0)
qc.rx(theta2, 1)
qc.cx(0, 1)
qc.draw("mpl")
```

![Quantum circuit diagram](./_04_Hands-on Introduction to Qiskit_imgs/output_26.png)

Sa kasong ito, ang mga quantum state na katatawainin natin ay:

$$
\begin{align*}
\text{CNOT}_{01} \, R_x^{\{1}}(\theta_2) R_x^{\{0}}(\theta_1) \ket{00}
&= \text{CNOT}_{01} \, R_x^{\{1}}(\theta_2)\left( \cos(\theta_1/2)\ket{00} - i\sin(\theta_1/2)\ket{10} \right) \\
&= \text{CNOT}_{01}\left( \cos(\theta_1/2)\cos(\theta_2/2)\ket{00} - i\cos(\theta_1/2)\sin(\theta_2/2)\ket{01} \right. \\
&\quad \left. - i\sin(\theta_1/2)\cos(\theta_2/2)\ket{10} + \sin(\theta_1/2)\sin(\theta_2/2)\ket{11} \right) \\
&= \cos(\theta_1/2)\cos(\theta_2/2)\ket{00} - i\cos(\theta_1/2)\sin(\theta_2/2)\ket{01} \\
&\quad + \sin(\theta_1/2)\sin(\theta_2/2)\ket{10} - i\sin(\theta_1/2)\cos(\theta_2/2)\ket{11} 
\end{align*}
$$
Makikita natin na ang circuit na ito ay bumubuo ng mas malawak na pamilya ng mga quantum state kumpara sa nauna. Sa partikular, kaya na nitong gumawa ng mga state na may hindi-zero na amplitude para sa $\ket{01}$ o $\ket{10}$ na hindi posible sa circuit sa itaas. Gayunpaman, hindi pa rin universal quantum state generator ang circuit na ito, kahit na maaaring sapat ang bisa nito para sa pagdidisenyo ng mga circuit na may ilang flexibility sa pagkatawan ng ilang mga function. Sa pangkalahatan, habang mas maraming independyenteng parameter (anggulo) ang ipinakilala, mas lumalaki ang kakayahan ng circuit na mag-approximate ng mga arbitrary na quantum state.
## Ansatzes at Circuit library

Ang ganitong uri ng parametrized quantum circuit ay maaaring gamitin para gumawa ng mga [Ansatz](https://quantum.cloud.ibm.com/learning/en/courses/quantum-chem-with-vqe/ansatz), mga trial quantum state na naglalayong mag-approximate ng solusyon sa isang problema. Ang mga Ansatz na ito ay isang sentral na bahagi ng [Variational Quantum Algorithms](https://quantum.cloud.ibm.com/learning/en/courses/variational-algorithm-design/variational-algorithms), isang klase ng hybrid quantum-klasikal na mga algorithm na gumagamit ng quantum computer para suriin ang isang cost function at isang klasikal na optimizer para i-minimize ito. Tatalakayin natin nang mas detalyado ang mga paksang ito sa susunod na Unit, pero sa ngayon, ipakikilala natin kung paano gumawa ng simpleng ansatz gamit ang [Circuit library sa Qiskit](https://quantum.cloud.ibm.com/docs/en/api/qiskit/circuit_library). 

```python
from qiskit.circuit.library import efficient_su2

SU2_ansatz = efficient_su2(4, su2_gates=["rx", "y"], entanglement="linear", reps=1)
SU2_ansatz.decompose().draw(output="mpl")
```

# Bonus: Quantum Teleportation
Kapag narinig mo ang terminong quantum teleportation, maaaring maiisip mo ang futuristic na teknolohiya mula sa sci-fi na nagpapaalis ng isang bagay sa isang lugar at nagpapalitaw nito sa malayo. Pero ang quantum teleportation ay hindi ganoon. Sa katunayan, ang nate-teleport ay hindi bagay — kundi impormasyon.

Pinapayagan ng quantum teleportation ang paglipat ng quantum state ng isang qubit mula sa isang lokasyon patungo sa isa pa. Kahit na tila agaran ang paglipat na ito, hindi nito nilalabag ang mga batas ng pisika. Paano iyon posible? Sige, tuklasin natin!

Ang quantum teleportation ay isang protocol na nagbibigay-daan sa isang nagpapadala (Alice) na maipadala ang state na $|\psi\rangle$ ng isang qubit na `q` sa isang tatanggap (Bob) gamit ang dalawang pangunahing mapagkukunan: isang shared entangled pair ng mga qubit na `a` at `b` at dalawang bit ng klasikal na komunikasyon na `c0` at `c1`. 

Ang kailangan ng protocol ay:
*   `q`: Ang qubit ni Alice, una sa state na $|\psi\rangle$ na gusto nating i-teleport.
*   `a`: Ang kalahati ni Alice ng shared entangled pair.
*   `b`: Ang kalahati ni Bob ng shared entangled pair.
*   `c0`, `c1`: Mga klasikal na bit para itago ang mga resulta ng pagsukat ni Alice.

At paano ito gumagana? Ganito ang daloy

1.  **Ihanda ang state ni Alice na $|\psi\rangle$ sa `q`.** Gagawa tayo ng partikular na state tulad ng $|+\rangle$ para sa pag-verify.
2.  **Gumawa ng entanglement:** Bumuo ng Bell pair sa pagitan ng `a` at `b`.
3.  **Mga operasyon ni Alice:** Nagsasagawa si Alice ng "Bell measurement" sa kanyang dalawang qubit (`q` at `a`) at iniimbak ang mga klasikal na resulta sa `c0` at `c1`.
4.  **Klasikal na komunikasyon:** Ipinapadala ni Alice ang kanyang dalawang klasikal na bit (`c0`, `c1`) kay Bob.
5.  **Mga koreksyon ni Bob:** Nag-aaplay si Bob ng mga partikular na quantum gate (X at/o Z) sa kanyang qubit (`b`), nakondisyon sa mga value ng `c0` at `c1` na natanggap niya.

Kung lahat ay nagawa nang tama, magtatapos ang qubit ni Bob na `b` sa state na $|\psi\rangle$, ang orihinal na state ng `q` ni Alice!

Para sa mas malalim na paliwanag at pagtuklas ng quantum teleportation, kasama na ang matematikang paliwanag kung bakit gumagana ang protocol na ito, maaari kang sumangguni sa mga IBM Quantum Learning resource: [Quantum Teleportation](https://quantum.cloud.ibm.com/learning/courses/basics-of-quantum-information/entanglement-in-action/quantum-teleportation). Ito ay bahagi ng kursong [Basics of Quantum Information](https://quantum.cloud.ibm.com/learning/courses/basics-of-quantum-information).

```python

import matplotlib.pyplot as plt
from qiskit import QuantumCircuit, QuantumRegister, ClassicalRegister
from qiskit_aer import AerSimulator
from qiskit.visualization import plot_histogram, plot_bloch_multivector

# Define individual quantum registers for each qubit
q = QuantumRegister(1, name='q')  # message qubit
a = QuantumRegister(1, name='a')  # Alice's entangled qubit
b = QuantumRegister(1, name='b')  # Bob's entangled qubit

# Classical register for Alice's measurements
cr_alice = ClassicalRegister(2, name='c_alice')

# Create quantum circuit
teleport_qc = QuantumCircuit(q, a, b, cr_alice, name='Teleportation')

# Step 1: Prepare message state |+⟩ on q
teleport_qc.h(q[0])
teleport_qc.barrier()

# Step 2: Create entanglement between a and b
teleport_qc.h(a[0])
teleport_qc.cx(a[0], b[0])
teleport_qc.barrier()

# Step 3: Alice's Bell measurement
teleport_qc.cx(q[0], a[0])
teleport_qc.h(q[0])
teleport_qc.barrier()

# Step 4: Alice measures q and a
teleport_qc.measure(q[0], cr_alice[0])
teleport_qc.measure(a[0], cr_alice[1])
teleport_qc.barrier()

# Step 5: Bob's conditional measurements
with teleport_qc.if_test((cr_alice[1], 1)):
    teleport_qc.x(b[0])
with teleport_qc.if_test((cr_alice[0], 1)):
    teleport_qc.z(b[0])

# Draw the circuit
teleport_qc.draw(output='mpl')
```

![Quantum circuit diagram](./_04_Hands-on Introduction to Qiskit_imgs/output_28.png)

Pagkatapos isagawa ang protocol, lumitaw ang isang mahalagang tanong: paano natin mave-verify na natagumpay ang teleportation? Hindi natin direktang "makikita" ang state ng qubit ni Bob pagkatapos ng protocol. Gayunpaman, dahil *inihanda* natin ang paunang state ni Alice na $|\psi\rangle$ (pinili natin ang $|+\rangle$), maaari tayong gumamit ng espesyal na uri ng simulation para tingnan kung natapos ang qubit ni Bob na `b` sa parehong state.

Gagamitin natin ang `AerSimulator` na may `save_statevector` para tingnan kung natapos ang qubit ni Bob na `b` sa orihinal na state ni Alice ($|+\rangle$). Kinakalkula ng simulator na ito ang panghuling quantum state vector,
at pagkatapos ay kinikilala ito gamit ang `plot_bloch_multivector` para ma-visualize ang qubit ni Bob (`b`) kumpara sa paunang state ni Alice (`q`).

```python
# Simulate the teleportation circuit
sv_simulator = AerSimulator(method='statevector')
teleport_qc_sv = teleport_qc.copy()
teleport_qc_sv.save_statevector()

# Execute the circuit on the statevector simulator
job_sv = sv_simulator.run(teleport_qc_sv)
result_sv = job_sv.result()

# Get the final statevector
final_statevector = result_sv.get_statevector()
print("Visualizing final qubit states:")
display(plot_bloch_multivector(final_statevector))
print("Note that Alice's qubits have collapsed to |00⟩, |01⟩, |10⟩, or |11⟩, while Bob's qubit is in the original state |+⟩.")
```

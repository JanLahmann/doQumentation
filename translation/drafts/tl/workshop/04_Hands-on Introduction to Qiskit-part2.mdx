## 2.1 I-map ang problema sa quantum-native na format

Sa isang quantum program, ang *quantum circuits* ang katutubong format para kumatawan sa mga quantum na instruksyon, at ang *operators* naman ang kumakatawan sa mga observable na susukatin. Kapag gumagawa ng circuit, karaniwang gagawa ka ng bagong [`QuantumCircuit`](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.QuantumCircuit#quantumcircuit-class) na object, tapos daragdagan mo ito ng mga instruksyon sa sunud-sunod na paraan.

Ang sumusunod na code cell ay gumagawa ng circuit na nagpo-produce ng GHZ state — isang state kung saan tatlong qubits ay ganap na magkaka-entangle.

<div class="alert alert-info">

  Ginagamit ng Qiskit SDK ang LSb 0 bit numbering kung saan ang $n^{th}$ na digit ay may halaga na $1 \ll n$ o $2^n$. Para sa mas detalyadong impormasyon, tingnan ang [Bit-ordering in the Qiskit SDK](https://docs.quantum.ibm.com/guides/bit-ordering) na paksa.

</div>

```python
# Create a GHZ state circuit

qc = QuantumCircuit(3)
qc.h(0)
qc.cx(0,1)
qc.cx(0,2)
# Draw the circuit
qc.draw("mpl")
```

![Quantum circuit diagram](./_04_Hands-on Introduction to Qiskit_imgs/output_15.png)

Tingnan ang [`QuantumCircuit`](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.QuantumCircuit#quantumcircuit-class) sa dokumentasyon para sa lahat ng available na operasyon.

Kapag gumagawa ng quantum circuits, kailangan mo ring isaalang-alang kung anong uri ng datos ang gusto mong ibalik pagkatapos ng execution. Dalawang paraan ang ibinibigay ng Qiskit para ibalik ang datos: makakakuha ka ng probability distribution para sa isang set ng qubits na pipiliin mong sukatin, o makakakuha ka ng expectation value ng isang observable. Ihanda ang iyong workload para sukatin ang iyong circuit sa isa sa dalawang paraan na ito gamit ang [Qiskit primitives](https://docs.quantum.ibm.com/guides/get-started-with-primitives) (ipapaliwanag nang detalyado sa Hakbang 3).

Sa halimbawang ito, sinusukat ang mga expectation value sa pamamagitan ng `qiskit.quantum_info` submodule, na tinutukoy gamit ang mga operator (mga mathematical na object na ginagamit para kumatawan sa isang aksyon o proseso na nagbabago ng quantum state). Ang sumusunod na code cell ay gumagawa ng anim na three-qubit na Pauli operator: `ZZZ`, `ZZX`, `ZII`, `XXI`, `ZZI` at `III`.

```python
# Set up six different observables.

observables_labels = ["ZZZ", "ZZX", "ZII", "XXI", "ZZI", "III"]

observables = [SparsePauliOp(label) for label in observables_labels]
print(observables)
```

```text
[SparsePauliOp(['ZZZ'],
              coeffs=[1.+0.j]), SparsePauliOp(['ZZX'],
              coeffs=[1.+0.j]), SparsePauliOp(['ZII'],
              coeffs=[1.+0.j]), SparsePauliOp(['XXI'],
              coeffs=[1.+0.j]), SparsePauliOp(['ZZI'],
              coeffs=[1.+0.j]), SparsePauliOp(['III'],
              coeffs=[1.+0.j])]
```

<div class="alert alert-info">

  Dito, ang `ZZI` operator halimbawa ay isang shorthand para sa tensor product na $Z\otimes Z\otimes I$, na ibig sabihin ay sinusukat ang Z sa qubit 2 at Z sa qubit 1 nang sabay, at nakakakuha ng impormasyon tungkol sa ugnayan ng qubit 2 at qubit 1. Ang mga expectation value na katulad nito ay karaniwang isinusulat bilang $\langle Z_2 Z_1 \rangle$.

  Kung ang state na aming sinisiyasat ay ang three-qubit na GHZ state, ang pagsukat ng $\langle Z_2 Z_1 \rangle$ ay dapat maging 1.

</div>

## 2.2 I-optimize ang mga circuit at operator

Kapag nagpapatakbo ng mga circuit sa isang device, mahalaga na i-optimize ang set ng mga instruksyon na nilalaman ng circuit at bawasan ang kabuuang lalim (halos katumbas ng bilang ng mga instruksyon) ng circuit. Tinitiyak nito na makakakuha ka ng pinakamahusay na resulta sa pamamagitan ng pagbabawas ng epekto ng error at noise. Bukod dito, ang mga instruksyon ng circuit ay dapat sumunod sa [Instruction Set Architecture (ISA)](https://docs.quantum.ibm.com/guides/transpile#instruction-set-architecture) ng backend device at kailangang isaalang-alang ang mga basis gate at qubit connectivity ng device.

Ang sumusunod na code ay nagpo-instantiate ng isang tunay na device para pagpadahin ng isang job at bina-transform ang circuit at mga observable para tumugma sa ISA ng backend na iyon.
Kung hindi mo pa nase-save ang iyong mga kredensyal noon, sundin ang mga instruksyon [dito](https://docs.quantum.ibm.com/guides/setup-channel#iqp) para mag-authenticate gamit ang iyong API token.

```python
# Choose a real backend
service = QiskitRuntimeService(channel='ibm_quantum_platform',)
backend = service.least_busy(min_num_qubits=156)
# print backend details
print(
    f"Name: {backend.name}\n"
    f"Version: {backend.backend_version}\n"
    f"No. of qubits: {backend.num_qubits}\n"
    f"Processor type: {backend.processor_type}\n"
)
```

## 2.3 I-execute gamit ang quantum primitives

Ang mga quantum computer ay maaaring maglabas ng mga random na resulta, kaya karaniwang nangongolekta ka ng sample ng mga output sa pamamagitan ng paulit-ulit na pagpapatakbo ng circuit. Maaari mong i-estimate ang halaga ng observable gamit ang `Estimator` class. Ang `Estimator` ay isa sa dalawang [primitives](https://docs.quantum.ibm.com/guides/get-started-with-primitives); ang isa pa ay ang `Sampler`, na magagamit para kumuha ng datos mula sa isang quantum computer. Ang mga object na ito ay may `run()` na method na nagpapatupad ng piniling mga circuit, observable, at parameter (kung naaangkop), gamit ang isang [primitive unified bloc (PUB).](https://docs.quantum.ibm.com/guides/primitives#sampler)
Kapag pinapatakbo ang code na ito sa tunay na quantum hardware, isaalang-alang ang pag-apply ng mga [error mitigation at suppression technique](https://quantum.cloud.ibm.com/docs/en/guides/error-mitigation-and-suppression-techniques) para bawasan ang likas na noise ng quantum computer.

```python
# Construct the Estimator instance.
estimator = Estimator(mode=backend)
estimator.options.resilience_level = 1
estimator.options.default_shots = 5000
```

Mag-submit ng job gamit ang Estimator primitive.

```python
# One pub, with one circuit to run against six different observables.
job = estimator.run([(isa_circuit, mapped_observables)]) 

# Use the job ID to retrieve your job data later
print(f">>> Job ID: {job.job_id()}")
```

Pagkatapos ma-submit ang isang job, maaari kang maghintay hanggang matapos ang job sa loob ng iyong kasalukuyang python instance, o gamitin ang `job_id` para kunin ang datos sa susunod na pagkakataon. (Tingnan ang [seksyon sa pagkuha ng mga job](https://docs.quantum.ibm.com/guides/monitor-job#retrieve-job-results-at-a-later-time) para sa mga detalye.)

Pagkatapos makumpleto ang job, suriin ang output nito sa pamamagitan ng `result()` attribute ng job.

```python
# This is the result of the entire submission.  You submitted one Pub,
# so this contains one inner result (and some metadata of its own).
job_result = job.result()

# This is the result from our single pub, which had six observables,
# so contains information on all six.
pub_result = job.result()[0]
```

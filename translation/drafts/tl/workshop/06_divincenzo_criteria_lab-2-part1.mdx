**Limang Kriterya ni DiVincenzo para sa Quantum Computation**:

1. **Isang scalable na pisikal na sistema na may maayos na nakatukoy na mga qubit.**
2. **Kakayahang i-initialize ang mga qubit** sa isang simpleng fiducial state (hal. |00…0〉).
3. **Mahabang decoherence times** (ang coherence ng qubit ay mas matagal kaysa sa oras ng gate operation).
4. **Isang universal na set ng mga quantum gate** (kayang magsagawa ng arbitrary na unitary operations).
5. **Qubit-specific na kakayahan sa pagsukat** (basahin ang state ng bawat qubit).

*(Inilalarawan din ni DiVincenzo ang dalawang kriterya para sa quantum communication: ang kakayahang mag-interconvert ng stationary at "flying" na mga qubit, at maipapadala nang tapat ang mga flying qubit sa pagitan ng mga lokasyon. Isinama natin ang mga ito sa isang inirerekomendang aktibidad sa dulo ng notebook na ito.)*

## 1. Kriterya 1 – Scalable, Maayos na Nakatukoy na mga Qubit

**Kriterya 1:** *"Isang scalable na pisikal na sistema na may maayos na nakatukoy na mga qubit."* Ibig sabihin nito ay kailangan natin ng isang quantum hardware platform kung saan maaari tayong **dagdagan ang bilang ng mga qubit** at kontrolin pa rin ang mga ito nang mapagkakatiwalaan. Ang mga katangian ng bawat qubit (mga energy level, error rate, connectivity, atbp.) ay dapat na maunawaan nang maayos. Sa esensya, gusto nating bumuo ng mas malalaking circuit nang hindi nasisira ang sistema. Sa praktika, habang nagdadagdag tayo ng bilang ng qubit o circuit depth, nag-iipon ang mga error at decoherence, kaya ang pagpapakita ng *scalability* ay nangangahulugang pag-unawa rin kung paano nakakaapekto ang pagtaas ng laki sa performance.

**Layunin ng Demo:** Gamitin ang Qiskit para ipakita ang epekto ng pagpapalaki ng isang circuit (sa bilang ng qubit o gate depth) sa output fidelity. Mag-si-simulate tayo ng ideal kumpara sa may noise na sitwasyon para makita kung paano sumusuko sa decoherence at mga error ang mas malaking sistema o mas malalim na circuit.

Una, bumuo tayo ng maliit na entangled state (GHZ state) sa 3 qubit, pagkatapos ay mas malaki sa 5 qubit, bilang isang simpleng scaling test. Ang GHZ state ng *n* qubit ay $\frac{1}{\sqrt{2}}(|0...0\rangle + |1...1\rangle)$. Sa isang ideal na simulation, ang pagsukat ng n-qubit GHZ ay nagbibigay lamang ng dalawang resulta (lahat ng 0 o lahat ng 1) na may pantay na posibilidad. Ikukumpara natin ang **ideal na output** sa isang **may noise na output** habang pinapataas natin ang n o circuit depth.

```python
from qiskit import QuantumCircuit
from qiskit_aer import AerSimulator
from qiskit.visualization import plot_histogram
from qiskit.transpiler.preset_passmanagers import generate_preset_pass_manager
from qiskit_ibm_runtime import SamplerV2 as Sampler

# 3-qubit GHZ circuit
qc3 = QuantumCircuit(3, 3)
qc3.h(0)
qc3.cx(0, 1)
qc3.cx(1, 2)
qc3.measure([0, 1, 2], [0, 1, 2])

# 5-qubit GHZ circuit (scaling up the number of qubits)
qc5 = QuantumCircuit(5, 5)
qc5.h(0)
qc5.cx(0, range(1, 5))    # entangle qubit 0 with all others
qc5.measure(range(5), range(5))

# Transpile for a simulator backend
sim_backend = AerSimulator()
pm = generate_preset_pass_manager(backend=sim_backend, optimization_level=1)
isa_qc3 = pm.run(qc3)
isa_qc5 = pm.run(qc5)

# Run ideal simulations (no noise)
sampler = Sampler(mode=sim_backend)

job3 = sampler.run([isa_qc3], shots=1024)
result3 = job3.result()
counts3 = result3[0].data.c.get_counts()

job5 = sampler.run([isa_qc5], shots=1024)
result5 = job5.result()
counts5 = result5[0].data.c.get_counts()

print("3-qubit GHZ counts (ideal):", counts3)
plot_histogram(counts3, legend=['3-qubit ideal'], figsize=(6,4))
```

```text

```

**Inaasahang resulta (ideal na kaso):** Ang 3-qubit GHZ ay perpektong nagbibigay ng humigit-kumulang 50% `000` at 50% `111` sa mga count. Ang 5-qubit GHZ ay nagbibigay ng ~50% `00000` at 50% `11111`. Walang ibang bit-string ang lumalabas dahil ang state ay perpektong coherent at entangled. Dapat makakita ka ng dalawang mataas na bar sa histogram para sa bawat circuit na tumutugma sa lahat-ng-zero at lahat-ng-isa na mga resulta.

## 2. Kriterya 2 – Pag-initialize ng Qubit

**Kriterya 2:** *"Ang kakayahang i-initialize ang state ng mga qubit sa isang simpleng fiducial state, tulad ng |000…〉."* Lahat ng qubit ay dapat na mapagkakatiwalaang magsimula sa isang kilalang reference state (karaniwang ang ground state |0〉 para sa bawat qubit). Mahalaga ang initialization para magsimula ang mga algorithm sa isang malinis na slate. Sa praktika, sa mga IBM quantum device, ang bawat qubit ay awtomatikong nire-reset sa |0〉 sa simula ng bawat circuit execution. Nagbibigay din ang Qiskit ng mga instruksyon para i-reset ang mga qubit o maghanda ng custom na state sa panahon ng computation.

**Layunin ng Demo:** Ipakita kung paano i-initialize ang mga qubit sa Qiskit, mula sa simula at sa kalagitnaan ng circuit. Ipapakita natin ang paggamit ng instruksyong `reset` at mga paraan ng paghahanda ng state.

Ngayon, para sa mas pangkalahatang paghahanda ng state, pinapahintulutan ng Qiskit ang initialization sa arbitrary na state gamit ang paraang `initialize`. Halimbawa, ihanda natin ang isang qubit sa state na $|+\rangle = (|0\rangle+|1\rangle)/\sqrt{2}$, na isang superposition state, at isang pares ng qubit sa Bell state na $(|00\rangle+|11\rangle)/\sqrt{2}$:

```python
import numpy as np

# Initialize a single qubit in |+> state and measure in Z-basis
qc_plus = QuantumCircuit(1, 1)
state_plus = [1/np.sqrt(2), 1/np.sqrt(2)]   # amplitude for |0> and |1>
qc_plus.initialize(state_plus, 0)
qc_plus.measure(0, 0)

# Initialize two qubits in a Bell state manually
qc_bell = QuantumCircuit(2, 2)
bell_state = [1/np.sqrt(2), 0, 0, 1/np.sqrt(2)]  # amplitudes for |00>,|01>,|10>,|11>
qc_bell.initialize(bell_state, [0, 1])
qc_bell.measure([0, 1], [0, 1])

# Transpile and run the initialization circuits
isa_qc_plus = pm.run(qc_plus)
job_plus = sampler.run([isa_qc_plus], shots=1024)
result_plus = job_plus.result()
counts_plus = result_plus[0].data.c.get_counts()

print("Outcome of |+> state measured in Z-basis:", counts_plus)

isa_qc_bell = pm.run(qc_bell)
job_bell = sampler.run([isa_qc_bell], shots=1024)
result_bell = job_bell.result()
counts_bell = result_bell[0].data.c.get_counts()

print("Outcome of Bell state measured in Z-basis:", counts_bell)
```

```text
Outcome of |+> state measured in Z-basis: {'1': 499, '0': 525}
Outcome of Bell state measured in Z-basis: {'00': 508, '11': 516}
```

**Inaasahang mga resulta:** Ang single qubit na |+〉 state, kapag sinukat, ay magbibigay ng `0` at `1` na may humigit-kumulang 50% na posibilidad bawat isa. Ang pagsukat ng Bell state ay dapat magbigay ng humigit-kumulang 50% `00` at 50% `11`. Kung makikita mo ang mga ito, nagpapatunay ito na matagumpay ang ating initialization sa mga state na iyon.

**Initialization sa kalagitnaan ng circuit:** Ang `reset` ng Qiskit ay maaaring gamitin sa kalagitnaan ng isang circuit para muling i-initialize ang isang qubit sa |0〉 habang tumatakbo. Halimbawa, sa mga error correction code o iterative na algorithm, madalas na sinusukat ang isang qubit at pagkatapos ay rine-reset ito para magamit muli. Ang operasyong `reset` ay deterministiko; tinatanggal nito ang anumang kasalukuyang state at pinapalamig ang qubit pabalik sa ground state.

**Halimbawa sa device:** Sa hardware tulad ng **ibmq_brisbane** (127 qubit) o anumang IBM device, lahat ng qubit ay nagsisimula sa |0〉 bilang default kapag pinapatakbo ang isang job. Kung kailangan mo ng ibang panimulang state, mag-a-apply ka ng mga gate sa simula (gaya ng ginawa natin gamit ang X para makuha ang |1〉). Ang tuluy-tuloy na muling pag-initialize (para sa quantum error correction) ay isang aktibong paksa ng pananaliksik dahil mahirap itong gawin nang mabilis. Sa kabutihang palad, para sa pangunahing paggamit, available ang kakayahang magsimulang muli sa |0…0〉 at ipinakita natin kung paano makamit ang iba pang nais na panimulang state.

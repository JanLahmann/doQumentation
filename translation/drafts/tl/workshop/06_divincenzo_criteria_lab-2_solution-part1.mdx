## Panimula

Inilahad ng pisiko na si David DiVincenzo ang limang pangunahing kinakailangan para sa anumang pisikal na implementasyon ng isang quantum computer, kasama ang dalawang karagdagang kriterya para sa quantum communication. Sa notebook na ito, **mararanasan natin ang bawat DiVincenzo criterion sa pamamagitan ng mga praktikal na demonstrasyon gamit ang Qiskit**. Sa halip na sumabak nang malalim sa teorya, bawat seksyon ay maikling nagpapaliwanag ng isang kriterya at nagbibigay ng mga code exercise gamit ang Qiskit 2. Maaari kang magpatakbo ng mga circuit sa mga simulator at tunay na IBM Quantum device para **tuklasin ang bawat prinsipyo nang hands-on**.

**Limang Kriterya ni DiVincenzo para sa Quantum Computation**:

1. **Isang scalable na pisikal na sistema na may malinaw na katangiang mga qubit.**
2. **Kakayahang i-initialize ang mga qubit** sa isang simpleng fiducial state (hal. |00…0〉).
3. **Matagal na decoherence times** (ang coherence ng qubit ay mas matagal kaysa sa oras ng operasyon ng gate).
4. **Isang universal na hanay ng mga quantum gate** (kayang magsagawa ng arbitraryong unitary operation).
5. **Kakayahang sukatin ang bawat qubit nang hiwalay** (basahin ang estado ng bawat qubit).

*(Naglahad din si DiVincenzo ng dalawang kriterya para sa quantum communication: ang kakayahang mag-interconvert ng stationary at "flying" qubit, at ang tapat na pagpapadala ng mga flying qubit sa pagitan ng mga lokasyon. Isinama natin ang mga ito sa isang inirerekomendang aktibidad sa dulo ng notebook na ito.)*

Ang bawat sumusunod na seksyon ay katumbas ng isang kriterya. Gagamitin natin ang Qiskit para mailarawan ang konsepto gamit ang code at **mga interactive na eksperimento** na maaari mong subukan. Halimbawa, makikita natin kung paano nakakaapekto ang pagpapalaki ng bilang ng mga qubit at lalim ng circuit sa mga resulta (Criterion 1), kung paano i-reset at ihanda ang mga estado ng qubit (Criterion 2), kung paano sukatin ang mga qubit sa mga simulator kumpara sa tunay na device (Criterion 4), kung paano nagbubuod ang Qiskit ng mga universal gate (Criterion 3), at kung paano nakakaapekto ang limitadong coherence (T₁, T₂) sa mga kalkulasyon (Criterion 5). Sa pagtatapos, magkakaroon ka ng mas malalim na intuisyon kung ano ang ibig sabihin ng bawat DiVincenzo criterion sa praktis at kung paano pinapadali ng Qiskit ang pag-eksperimento sa mga ito.

## 1. Criterion 1 – Scalable, Well-Characterized Qubit

**Criterion 1:** *"Isang scalable na pisikal na sistema na may malinaw na katangiang mga qubit."* Ibig sabihin, kailangan natin ng isang quantum hardware platform kung saan maaari tayong **dagdagan ang bilang ng mga qubit** at kontrolin pa rin ang mga ito nang mapagkakatiwalaan. Ang mga katangian ng bawat qubit (antas ng enerhiya, rate ng error, koneksyon, atbp.) ay dapat na malinaw na naiintindihan. Sa esensya, gusto nating bumuo ng mas malalaking circuit nang hindi bumagsak ang sistema. Sa praktis, habang pinapalaki natin ang bilang ng qubit o lalim ng circuit, nag-iipon ang mga error at decoherence, kaya ang pagpapakita ng *scalability* ay nangangahulugang pag-unawa rin kung paano nakakaapekto ang pagpapalaki sa performance.

**Layunin ng Demo:** Gamitin ang Qiskit para ipakita ang epekto ng pagpapalaki ng isang circuit (sa bilang ng qubit o lalim ng gate) sa output fidelity. Mag-simulate tayo ng ideal kumpara sa maingay na sitwasyon para makita kung paano sumisira ang mas malaking sistema o mas malalim na circuit sa decoherence at mga error.

Una, bumuo tayo ng maliit na entangled state (GHZ state) sa 3 qubit, pagkatapos ay mas malaki sa 5 qubit, bilang isang simpleng scaling test. Ang GHZ state ng *n* qubit ay $\frac{1}{\sqrt{2}}(|0...0\rangle + |1...1\rangle)$. Sa isang ideal na simulation, ang pagsukat ng n-qubit GHZ ay nagbibigay lamang ng dalawang resulta (lahat 0 o lahat 1) na may pantay na posibilidad. Ikukumpara natin ang **ideal na output** sa isang **maingay na output** habang pinapalaki natin ang n o lalim ng circuit.

```python
from qiskit import QuantumCircuit
from qiskit_aer import AerSimulator
from qiskit.visualization import plot_histogram
from qiskit.transpiler.preset_passmanagers import generate_preset_pass_manager
from qiskit_ibm_runtime import SamplerV2 as Sampler

# 3-qubit GHZ circuit
qc3 = QuantumCircuit(3, 3)
qc3.h(0)
qc3.cx(0, 1)
qc3.cx(1, 2)
qc3.measure([0, 1, 2], [0, 1, 2])

# 5-qubit GHZ circuit (scaling up the number of qubits)
qc5 = QuantumCircuit(5, 5)
qc5.h(0)
qc5.cx(0, range(1, 5))    # entangle qubit 0 with all others
qc5.measure(range(5), range(5))

# Transpile for a simulator backend
sim_backend = AerSimulator()
pm = generate_preset_pass_manager(backend=sim_backend, optimization_level=1)
isa_qc3 = pm.run(qc3)
isa_qc5 = pm.run(qc5)

# Run ideal simulations (no noise)
sampler = Sampler(mode=sim_backend)

job3 = sampler.run([isa_qc3], shots=1024)
result3 = job3.result()
counts3 = result3[0].data.c.get_counts()

job5 = sampler.run([isa_qc5], shots=1024)
result5 = job5.result()
counts5 = result5[0].data.c.get_counts()

print("3-qubit GHZ counts (ideal):", counts3)
plot_histogram(counts3, legend=['3-qubit ideal'], figsize=(6,4))
```

```python
print("5-qubit GHZ counts (ideal):", counts5)
plot_histogram(counts5, legend=['5-qubit ideal'], figsize=(6,4))
```

**Inaasahang resulta (ideal na kaso):** Ang 3-qubit GHZ ay nagbibigay ng humigit-kumulang 50% `000` at 50% `111` sa mga bilang. Ang 5-qubit GHZ ay nagbibigay ng ~50% `00000` at 50% `11111`. Walang ibang bit-string ang lumalabas dahil ang estado ay perpektong coherent at entangled. Dapat kang makakita ng dalawang matataas na bar sa histogram para sa bawat circuit na katumbas ng lahat-sero at lahat-isa na mga resulta.

Susunod, tingnan natin kung ano ang mangyayari sa isang **maingay na kapaligiran**. Gagamit tayo ng noise model capabilities ng Qiskit Aer para tularan ang mga error ng isang tunay na device. Halimbawa, maaari tayong kumuha ng mga katangian ng isang IBM backend para lumikha ng noise model na kinabibilangan ng mga gate error, limitadong gate time, qubit relaxation (T₁), dephasing (T₂), at readout error. Dito, gagamit tayo ng **fake backend** na kumakatawan sa IBM Quantum Brisbane device para makabuo ng noise model, at patakbuhin muli ang mga GHZ circuit doon.

**Inaasahang resulta (maingay kumpara sa ideal):** Kapag may noise, maging sa simulation o sa tunay na device, ang GHZ state ay **hindi perpekto**. Makakakita ka ng mga karagdagang resulta bukod sa lahat-0 at lahat-1. Para sa 3 qubit, sa halip na 100% sa `000`/`111`, may ilang posibilidad na lumilipat sa ibang bitstring (hal. `001`, `010`, atbp.) dahil sa mga gate error o decoherence na nagbabago ng ilang qubit. Para sa 5 qubit, mas malinaw ang epekto; ang mas malaking circuit (mas maraming qubit at CNOT gate) ay nag-iipon ng mas maraming error, kaya ang lahat-0 at lahat-1 na tuktok ay mas mababa, at maraming ibang resulta ang lumalabas. Ang trend na ito ay nagpapakita ng hamon ng *scalability*: habang lumalaki tayo, mas mahirap na panatilihin ang mataas na fidelity nang walang error correction.

**Pananaw:** Ang isang scalable na quantum computer ay kailangang mapanatili ang mga quantum correlation habang lumalaki ang sistema. Ang aming mga halimbawa ay nagpapakita kung paano ang pagdaragdag ng bilang ng qubit/lalim ng gate ay nagdudulot ng pagbaba ng outcome fidelity kapag may noise. Ang mga natitirang kriterya ay tutugon sa pagpapanatiling maayos ng mga qubit na iyon (mababang error, maaaring i-initialize, atbp.) habang lumalaki tayo.
## 2. Criterion 2 – Initialization ng Qubit

**Criterion 2:** *"Ang kakayahang i-initialize ang estado ng mga qubit sa isang simpleng fiducial state, tulad ng |000…〉."* Lahat ng qubit ay dapat na maaasahang magsimula sa isang kilalang reference state (karaniwang ang ground state |0〉 para sa bawat qubit). Ang initialization ay mahalaga para ang mga algorithm ay magsimula nang malinis. Sa praktis, sa mga IBM quantum device, ang bawat qubit ay awtomatikong rina-reset sa |0〉 sa simula ng bawat circuit execution. Nagbibigay din ang Qiskit ng mga instruksyon para i-reset ang mga qubit o maghanda ng mga custom na estado sa loob ng isang kalkulasyon.

**Layunin ng Demo:** Ipakita kung paano i-initialize ang mga qubit sa Qiskit, maging sa simula at sa gitna ng circuit. Ipapakita natin ang paggamit ng `reset` na instruksyon at mga paraan ng paghahanda ng estado.

Dapat mong makita ang `10` (binary para sa qubit1=1, qubit0=0) na may 100% posibilidad mula sa simulation, ibig sabihin ang qubit 1 ay matagumpay na inihanda sa |1〉 at ang qubit 0 sa |0〉.

Ngayon, para sa mas pangkalahatang paghahanda ng estado, pinapayagan ng Qiskit ang initialization sa mga arbitraryong estado gamit ang `initialize` na pamamaraan. Halimbawa, ihanda natin ang isang qubit sa estado $|+\rangle = (|0\rangle+|1\rangle)/\sqrt{2}$, na isang superposition state, at isang pares ng qubit sa Bell state $(|00\rangle+|11\rangle)/\sqrt{2}$:

```python
import numpy as np

# Initialize a single qubit in |+> state and measure in Z-basis
qc_plus = QuantumCircuit(1, 1)
state_plus = [1/np.sqrt(2), 1/np.sqrt(2)]   # amplitude for |0> and |1>
qc_plus.initialize(state_plus, 0)
qc_plus.measure(0, 0)

# Initialize two qubits in a Bell state manually
qc_bell = QuantumCircuit(2, 2)
bell_state = [1/np.sqrt(2), 0, 0, 1/np.sqrt(2)]  # amplitudes for |00>,|01>,|10>,|11>
qc_bell.initialize(bell_state, [0, 1])
qc_bell.measure([0, 1], [0, 1])

# Transpile and run the initialization circuits
isa_qc_plus = pm.run(qc_plus)
job_plus = sampler.run([isa_qc_plus], shots=1024)
result_plus = job_plus.result()
counts_plus = result_plus[0].data.c.get_counts()

print("Outcome of |+> state measured in Z-basis:", counts_plus)

isa_qc_bell = pm.run(qc_bell)
job_bell = sampler.run([isa_qc_bell], shots=1024)
result_bell = job_bell.result()
counts_bell = result_bell[0].data.c.get_counts()

print("Outcome of Bell state measured in Z-basis:", counts_bell)
```

**Inaasahang mga resulta:** Ang iisang qubit na estado na |+〉, kapag sinukat, ay magbibigay ng `0` at `1` na may halos 50% posibilidad bawat isa. Ang pagsukat ng Bell state ay dapat magbigay ng halos 50% `00` at 50% `11`. Kung makikita mo ito, nagpapatunay ito na ang ating initialization sa mga estadong iyon ay matagumpay.

**Initialization sa gitna ng circuit:** Ang `reset` ng Qiskit ay maaaring gamitin sa gitna ng isang circuit para muling i-initialize ang isang qubit sa |0〉 habang tumatakbo. Halimbawa, sa mga error correction code o iterative na algorithm, madalas na sinusukat ang isang qubit at pagkatapos ay rina-reset ito para muling gamitin. Ang operasyong `reset` ay deterministiko; itinatatapon nito ang anumang umiiral na estado at pinapalamig ang qubit pabalik sa ground state.

**Halimbawa sa device:** Sa hardware tulad ng **ibmq_brisbane** (127 qubit) o anumang IBM device, lahat ng qubit ay nagsisimula sa |0〉 bilang default kapag pinatakbo ang isang job. Kung kailangan mo ng ibang simula ng estado, mag-a-apply ka ng mga gate sa simula (tulad ng ginawa natin sa X para makakuha ng |1〉). Ang tuloy-tuloy na muling initialization (para sa quantum error correction) ay isang aktibong paksa ng pananaliksik dahil mahirap gawin ito nang mabilis. Sa kabutihang palad, para sa pangunahing paggamit, ang kakayahang magsimulang muli sa |0…0〉 ay available at ipinakita natin kung paano makamit ang iba pang ninanais na simula ng estado.

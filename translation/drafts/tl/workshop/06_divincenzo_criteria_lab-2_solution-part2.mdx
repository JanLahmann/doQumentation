## 3. Kriterya 3 – Mahabang Coherence Time (Decoherence kumpara sa Gate Time)

**Kriterya 3:** *"Mahabang kaugnay na decoherence time, na mas mahaba kaysa sa oras ng gate operation."* Tinutugunan nito ang pangangailangan ng mga qubit na mapanatili ang kanilang quantum state nang sapat na matagal para maisagawa ang mga kinakailangang operasyon. Ang bawat qubit ay may **T₁ time** (energy relaxation time, gaano kabilis bumagsak ang |1〉 pabalik sa |0〉) at **T₂ time** (dephasing time, gaano kabilis mawala ang relative phase coherence). Para gumana ang isang quantum computer, ang mga time scale na ito ay dapat na higit na lampas sa tagal ng mga gate operation.

**Layunin ng Demo:** Suriin ang qubit coherence sa Qiskit sa pamamagitan ng pagpapakita kung paano nakakaapekto ang decoherence sa mga resulta ng circuit habang lumalaki ang haba ng pagpapatupad. Gagamitin natin ang isang fake backend na may kilalang T1/T2 times para i-simulate ang epektong ito.

Para **ipakita ang epekto ng limitadong coherence**, mag-si-simulate tayo ng T1 decay experiment. Ihahanda natin ang isang qubit sa estado |1〉, maghihintay ng ilang oras gamit ang isang `delay` na instruksyon, at pagkatapos ay susukat. Inaasahan nating bababa ang posibilidad ng pagsukat ng |1〉 habang lumalaki ang delay.

```python
# This part is done for you. We are creating a list of circuits,
# each with a different delay time.

time_delays_ns = [0, 50000, 100000, 150000, 200000, 250000, 300000]  # delay durations in ns

decay_expts = []
for delay in time_delays_ns:
    qc = QuantumCircuit(1, 1)
    qc.x(0)  # initialize qubit to |1>
    if delay > 0:
        qc.delay(delay, 0, unit='ns')  # wait 'delay' nanoseconds
    qc.measure(0, 0)
    decay_expts.append(qc)

decay_expts[1].draw('mpl') # Visualize one of the circuits
```

## 4. Kriterya 4 – Universal na Hanay ng Quantum Gates

**Kriterya 4:** *"Isang 'universal' na hanay ng quantum gates."* Ibig sabihin nito, ang ating hardware ay dapat na nagpapahintulot sa atin na isagawa ang *anumang* quantum computation sa pamamagitan ng pagsasama-sama ng isang limitadong hanay ng mga pangunahing gate. Sa classical computing, ang NAND ay universal; sa quantum, maraming pagpipilian ng universal gate sets (hal. \{H, T, CNOT\} o ang mga native gate ng isang partikular na makina). Ang mga IBM device, halimbawa, ay may hanay ng mga native operation tulad ng arbitrary single-qubit rotation at mga CNOT sa pagitan ng ilang qubit, na magkasama ay universal. Ang trabaho ng Qiskit ay kadalasang **mag-compile ng mga high-level gate papunta sa mga basis gate na ito**.

**Layunin ng Demo:** Ipakita ang gate universality sa pamamagitan ng pagpapakita kung paano dina-decompose ng Qiskit ang mga gate. Kukuha tayo ng isang hindi native gate (tulad ng isang 3-qubit Toffoli gate, CCX) at titingnan kung paano ito nahahati sa mga basis gate ng device. Pinapatunayan nito na ang ibinigay na gate set ay talagang *universal* – kaya nitong gawin ang mas kumplikadong operasyon.

Direktang ipinapakita nito kung paano nagreresulta ang limitadong coherence time sa mga computational error kung masyadong matagal ang pagpapatakbo ng circuit.

**Epekto sa mga algorithm:** Kung susubukan mo ang isang mas mahabang algorithm (na may maraming sunud-sunod na gate), ang kabuuang oras ng pagpapatupad ay maaaring malapit o lampasan ang T2, na nagiging sanhi ng pagkawala ng coherence ng estado bago matapos. ## 5. Kriterya 5 – Pagsukat ng Qubit

### Exercise 5: Mag-simulate ng Readout Error
Kumpletuhin ang code para mag-define ng simpleng readout error model kung saan ang bawat qubit ay may 2% na pagkakataon na masukatan nang mali (ang isang 0 ay mabasa bilang 1, o ang isang 1 ay mabasa bilang 0). Pagkatapos, patakbuhin ang measurement circuit gamit ang noise model na ito.

```python
from qiskit_aer.noise import NoiseModel, ReadoutError

# --- YOUR CODE HERE ---

# 1. Define a 2% readout error for each single qubit.
# [[P(0|0), P(1|0)], [P(0|1), P(1|1)]]
ro_error = ReadoutError([[0.98, 0.02], [0.02, 0.98]])

# 2. Create a new noise model
noise_model_ro = NoiseModel()

# 3. Add the readout error to all qubits in the noise model
noise_model_ro.add_all_qubit_readout_error(ro_error)

# --- END YOUR CODE ---

sim_backend.set_options(noise_model=noise_model_ro)
pm = generate_preset_pass_manager(backend=sim_backend, optimization_level=1)
isa_qc_measure = pm.run(qc_measure)

# Run the measurement circuit with readout noise
sampler = Sampler(mode=sim_backend)

job = sampler.run([isa_qc_measure], shots=1024)
result = job.result()
counts = result[0].data.c.get_counts()

print("Simulation with 2% readout error:", counts)
```

Ipapakita ng na-simulate na output ang ilang maling counts (tulad ng `11`, `00`, `10`) na katulad ng maaaring i-produce ng tunay na hardware, na nagpapakita ng epekto ng hindi perpektong pagsukat.

## Mga Kriterya sa Quantum Communication (Flying Qubits)

Naglista rin si DiVincenzo ng dalawang kriterya na tiyak para sa quantum communication, mahalagang kung nagtatayo ng isang networked quantum computer:

6. **Kakayahang mag-interconvert ng stationary at flying qubits.** (Hal., i-map ang isang qubit sa isang processor papunta sa isang photon na makakapaglakbay.)
7. **Kakayahang tapat na magpadala ng flying qubits sa pagitan ng mga lokasyon.** (Hal., magpadala ng isang photon qubit sa pamamagitan ng fiber nang hindi nawawala ang quantum information.)

Ito ay higit pa sa karaniwang paggamit ng Qiskit dahil pangunahin itong nakikitungo sa mga stationary qubit sa isang chip. Gayunpaman, maaari nating ilarawan ang *konsepto* ng mga kriterya na ito gamit ang isang simpleng halimbawa: **quantum teleportation**. Ipinapakita ng teleportation ang pag-convert ng estado ng isang stationary qubit sa impormasyon na dinadala ng isang entangled pair (ang "flying" na bahagi) at classical communication, na pagkatapos ay ginagamit para i-reconstruct ang estado sa isa pang stationary qubit sa ibang lugar.
### Inirerekomendang aktibidad: Gawin ang Qiskit in Classrooms *Quantum Teleportation* module

Ang Qiskit in Classrooms [Quantum Teleportation](https://quantum.cloud.ibm.com/learning/en/modules/computer-science/quantum-teleportation) module ni Dr. Katie McCormick ay gagabay sa iyo sa isa sa mga pinaka-kahanga-hangang protocol sa quantum information: quantum teleportation, kung saan isang quantum state (isang qubit) ay ipinapadala mula kay Alice papunta kay Bob gamit ang entanglement at dalawang classical bit lang. Matututunan mo ang buong teleportation procedure hakbang-hakbang — kung paano ihanda ang entangled Bell pair, magsagawa ng Bell-basis measurement sa panig ni Alice, magpadala ng mga classical na resulta, at mag-apply ng tamang quantum gate sa qubit ni Bob para perpektong mabawi ang orihinal na estado. Sa proseso, tutuklasin mo kung bakit ang pag-teleport ng impormasyon ng isang qubit ay hindi lumalabag sa no-cloning theorem o lumalagpas sa bilis ng liwanag. Sa pamamagitan ng mga hands-on na ehersisyo gamit ang IBM Quantum hardware o mga simulator, makakakuha ka ng praktikal na pag-unawa sa pagsukat, entanglement, at feed-forward control sa aksyon.

Sa pamamagitan ng pag-master ng quantum teleportation, mauunawaan mo kung paano i-encode, ipadala, at mabawi ang quantum information sa pagitan ng mga natatanging node — nagtatayo ng pundasyon para sa mga quantum network, repeater system, ligtas na communication scheme, at scalable modular quantum computing.
**Kung paano ito naiugnay sa kriterya 6 at 7:** Sa isang tunay na quantum network, ang shared entangled pair ay malilikha sa pamamagitan ng pamamahagi ng mga "flying" qubit (tulad ng mga photon) sa pagitan ng mga lokasyon ng Alice at Bob (Kriterya 7: tapat na pagpapadala). Ang teleportation protocol mismo ay nagsisilbing paraan para i-map ang stationary qubit state ni Alice papunta sa kanyang kalahati ng entangled pair, na epektibong 'nagpapadala' nito kay Bob (Kriterya 6: interconversion). Pinapahintulutan kami ng Qiskit na i-simulate nang perpekto ang lohika ng protocol, na nagbibigay ng conceptual model kung paano natutugunan ang mga kriterya na ito sa mga communication architecture.

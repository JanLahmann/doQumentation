Sa quantum mechanics, ang mga observable ay tumutugma sa mga pisikal na katangian na maaaring sukatin.
Kapag pinag-aaralan ang isang sistema ng mga spin, halimbawa, maaaring interesado ka sa pagsukat ng enerhiya ng sistema o sa pagkuha ng impormasyon tungkol sa pagkakahanay ng mga spin, tulad ng magnetization o ng mga kaugnayan sa pagitan ng mga spin.

Para sukatin ang isang $n$-qubit observable na $O$ sa isang quantum computer, kailangan mong ipahayag ito bilang kabuuan ng mga tensor product ng mga Pauli operator, ibig sabihin

$$
O = \sum_{k=1}^K \alpha_k P_k,~~ P_k \in \{I, X, Y, Z\}^{\otimes n},~~ \alpha_k \in \mathbb{R},
$$

kung saan

$$
I = \begin{pmatrix}
1 & 0 \\ 0 & 1
\end{pmatrix}
~~
X = \begin{pmatrix}
0 & 1 \\ 1 & 0
\end{pmatrix}
~~
Y = \begin{pmatrix}
0 & -i \\ i & 0
\end{pmatrix}
~~
Z = \begin{pmatrix}
1 & 0 \\ 0 & -1
\end{pmatrix}
$$

at ginagamit mo ang katotohanang ang observable ay Hermitian, ibig sabihin, $O^\dagger = O$. Kung ang $O$ ay hindi Hermitian, maaari pa rin itong i-decompose bilang kabuuan ng mga Pauli, ngunit ang coefficient na $\alpha_k$ ay magiging complex.

Sa maraming kaso, ang observable ay natural na tinutukoy sa representasyong ito matapos i-map ang sistema ng interes sa mga qubit.
Halimbawa, ang isang spin-1/2 na sistema ay maaaring i-map sa isang Ising Hamiltonian

$$
H = \sum_{\langle i, j\rangle} Z_i Z_j - \sum_{i=1}^n X_i,
$$

kung saan ang mga index na $\langle i, j\rangle$ ay tumatakbo sa mga nakikipag-ugnayan na spin at ang mga spin ay napapailalim sa transversal field sa $X$.
Ang subscript index ay nagpapahiwatig kung saang qubit kumikilos ang Pauli operator, ibig sabihin, ang $X_i$ ay nag-aaplay ng $X$ operator sa qubit $i$ at iniiwanan ang natitirang hindi nagbabago.

Sa Qiskit SDK, ang Hamiltonian na ito ay maaaring itayo gamit ang sumusunod na code.

```python
from qiskit.quantum_info import SparsePauliOp

# tukuyin ang bilang ng mga qubit
n = 12

# tukuyin ang mga indibidwal na Pauli term bilang ("Paulis", [mga index], coefficient)
interactions = [
    ("ZZ", [i, i + 1], 1) for i in range(n - 1)
]  # inaakala natin ang mga spin sa isang 1D linya
field = [("X", [i], -1) for i in range(n)]

# itayo ang operator
hamiltonian = SparsePauliOp.from_sparse_list(
    interactions + field, num_qubits=n
)
print(hamiltonian)
```

Kung gusto nating sukatin ang enerhiya, ang observable ay ang Hamiltonian mismo. Bilang alternatibo, maaari tayong
interesado sa pagsukat ng mga katangian ng sistema tulad ng average magnetization sa pamamagitan ng pagbibilang ng bilang ng mga spin
na nakahanay sa $Z$-direksyon gamit ang observable

$$
O = \frac{1}{n} \sum_{i=1} Z_i
$$

Para sa mga observable na hindi ibinibigay sa mga Pauli operator kundi sa matrix form, kailangan muna nating baguhin ang representasyon nito
sa Pauli basis para masuri ang mga ito sa isang quantum computer.
Palagi tayong makakahanap ng ganitong representasyon dahil ang mga Pauli matrix ay bumubuo ng basis para sa Hermitian $2^n \times 2^n$ na mga matrix.
Inaayos natin ang observable na $O$ bilang

$$
O = \sum_{P \in \{I, X, Y, Z\}^{\otimes n}} \mathrm{Tr}(O P) P,
$$

kung saan ang kabuuan ay tumatakbo sa lahat ng posibleng $n$-qubit Pauli term at ang $\mathrm{Tr}(\cdot)$ ay ang trace ng isang matrix, na gumaganap bilang inner product.
Maaari mong ipatupad ang decomposition na ito mula sa isang matrix patungong Pauli term gamit ang `SparsePauliOp.from_operator` na method, ganito:

```python
import numpy as np
from qiskit.quantum_info import SparsePauliOp

matrix = np.array(
    [[-1, 0, 0.5, -1], [0, 1, 1, 0.5], [0.5, 1, -1, 0], [-1, 0.5, 0, 1]]
)

observable = SparsePauliOp.from_operator(matrix)
print(observable)
```

```text
SparsePauliOp(['IZ', 'XI', 'YY'],
              coeffs=[-1. +0.j,  0.5+0.j,  1. -0.j])
```

Ibig sabihin nito, ang matrix ay maaaring isulat bilang mga Pauli term na $O = -Z_1 + 0.5 X_2 + Y_2 Y_1$.

<Admonition type="note">
  Tandaan na ang tensor product order ay namamap sa mga qubit bilang $q_n \otimes q_{n-1} \otimes \cdots \otimes q_1$.
</Admonition>

<Admonition type="note">
  Kung ang observable ay Hermitian (ibig sabihin, $O^\dagger = O$), ang mga Pauli coefficient ay mga tunay na numero.
  Gayunpaman, maaari rin nating i-decompose ang anumang iba pang complex matrix sa mga Pauli, kung papayagan natin ang mga complex-valued na coefficient.
</Admonition>

## Sukatin sa mga Pauli basis {#measure-in-pauli-bases}

Ang isang measurement ay nagpro-project ng estado ng qubit sa computational basis na $\{|0\rangle, |1\rangle\}$. Ibig sabihin nito, maaari ka lang sumukat ng mga observable na diagonal sa basis na ito, tulad ng mga Pauli na binubuo lamang ng $I$ at $Z$ na mga term.
Ang pagsukat ng mga arbitrary na Pauli term ay nangangailangan ng pagbabago ng basis para i-diagonalize ang mga ito. Para gawin ito, isagawa ang mga sumusunod na transformasyon,

$$
\begin{aligned}
    X &\rightarrow Z = H X H \\
    Y &\rightarrow Z = H S^\dagger Y S H,
\end{aligned}
$$

kung saan ang $H$ ay ang Hadamard gate at ang $S = \sqrt{Z}$ ay tinatawag minsan na phase gate.
Kung gumagamit ka ng [Estimator](https://docs.quantum.ibm.com/api/qiskit/1.4/qiskit.primitives.Estimator) para sa pagkalkula ng mga expectation value, ang mga basis transformation ay awtomatikong isinasagawa.

Nasa ibaba ang isang halimbawa na nagpapakita kung paano maghanda ng quantum circuit at mano-manong sukatin ang qubit 0 sa X basis,
ang qubit 1 sa Y basis, at ang qubit 2 sa Z basis.
Inilalapat natin ang mga transformasyong ipinakita sa nakaraang equation at nakukuha natin ang sumusunod na circuit:

```python
from qiskit.circuit import QuantumCircuit

# lumikha ng circuit, kung saan gusto nating sukatin
# ang q0 sa X basis, q1 sa Y basis, at q2 sa Z basis
circuit = QuantumCircuit(3)
circuit.ry(0.8, 0)
circuit.cx(0, 1)
circuit.cx(1, 2)
circuit.barrier()

# i-diagonalize ang X gamit ang Hadamard gate
circuit.h(0)

# i-diagonalize ang Y gamit ang Hadamard bilang S^\dagger
circuit.sdg(1)
circuit.h(1)

# ang Z basis ay ang default, walang kinakailangang aksyon dito

# sukatin ang lahat ng qubit
circuit.measure_all()
circuit.draw("mpl")
```

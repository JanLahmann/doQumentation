# I-set up ang iyong IBM Cloud account

Kung nakatanggap ka ng email na imbitasyon para sumali sa isang account, sundin ang mga hakbang sa [I-set up ang iyong IBM Cloud account - mga inimbitahang gumagamit](/guides/cloud-setup-invited) sa halip.

Maaari kang magpatakbo ng mga workload sa IBM&reg; quantum processing units (QPUs) sa pamamagitan ng pag-set up ng account sa IBM Cloud&reg;. Kakailanganin mo ring [i-install at i-set up ang Qiskit at Qiskit Runtime](/guides/install-qiskit#local).

Ang iyong user account ay naka-ugnay sa isa o higit pang [mga instance,](./instances) na tinutukoy ng _Cloud Resource Names_ (CRNs) na nagbibigay ng access sa mga serbisyo ng IBM Quantum. Bukod dito, isang natatanging API token (tinatawag din na key) ang itinalaga sa bawat account, na nagpapahintulot ng IBM Quantum na access mula sa Qiskit. Para sa mga tagubilin sa pagpili ng isang partikular na instance, tingnan ang [Kumonekta sa isang instance](./instances#connect-instance).

<span id="cloud"></span>
## I-set up para gamitin ang IBM Cloud {#set-up-to-use-ibm-cloud}

### Bago ka magsimula {#before-you-begin}

1. Kung wala ka pa, mag-set up ng [IBM Cloud account.](https://quantum.cloud.ibm.com/registration)
   <Admonition type="note">
   Maaari kang maging bahagi ng maraming IBM Cloud account. Maaari mong ma-access ang alinman sa iyong mga IBM Cloud account anumang oras mula sa account switcher sa header ng interface ng [IBM Quantum Platform](https://quantum.cloud.ibm.com).
   </Admonition>
1. Mag-log in sa [IBM Quantum Platform](https://quantum.cloud.ibm.com) gamit ang IBMid o Google account. Kung wala ka pa, gagabayan ka sa paglikha ng isa. (Ang mga login credential na ito ay hindi ang iyong mga IBM Cloud account credential.)
1. Siguraduhing ang tamang account at rehiyon ay napili sa account switcher sa header, tulad ng ipinapakita sa sumusunod na larawan.

    Ang rehiyon ang nagkokontrol kung saan pinapatakbo ang iyong mga trabaho at kung saan iniingatan ang data ng trabaho. Maaari mong ma-access ang alinmang rehiyon gamit ang parehong API key, ngunit makikita at ma-access mo lamang ang mga instance na nilikha sa rehiyong naka-log in ka.

    ![The IBM Quantum Platform header is shown.  The account switcher is to the right of the search bar.  The region switcher is to the right of the account switcher.](/docs/images/guides/cloud-setup/IQP-Header.svg 'IBM Quantum Platform header')

### 1. Lumikha ng instance {#1-create-an-instance}

Maaari kang lumikha ng instance mula sa pahina ng [Instances](https://quantum.cloud.ibm.com/instances) ng IBM Quantum Platform. Tingnan ang [Lumikha ng instance](/guides/instances#add-instance) para sa mga detalyadong tagubilin.

### 2. Opsyonal: I-save ang iyong mga access credential {#2-optional-save-your-access-credentials}

Kung gumagamit ka ng pinagkakatiwalaang Python na kapaligiran, tulad ng personal na laptop, inirerekomenda na gamitin ang `save_account()` na paraan para [i-save ang iyong mga credential nang lokal.](/guides/save-credentials)

<span id="cloud-save"></span>
### 3. Ikonekta ang Qiskit sa iyong service instance {#3-connect-qiskit-with-your-service-instance}

- [I-initialize ang serbisyo](/guides/initialize-account) sa isang **pinagkakatiwalaang Python na kapaligiran**
- [I-initialize ang serbisyo sa isang **hindi pinagkakatiwalaang kapaligiran**](/guides/cloud-setup-untrusted) (tulad ng pampublikong computer)

<Admonition type="note" title="Mga Tala">
   * Sundin ang [mga tagubiling ito](/guides/cloud-setup-rest-api) kung gusto mong kumonekta gamit ang REST API sa halip na Qiskit.
   * Kung kinakailangan, gamitin ang [impormasyong ito](/guides/quickstart-steps-org#firewall) para i-configure ang iyong firewall upang paganahin ang access sa mga IBM Quantum API endpoint.
</Admonition>

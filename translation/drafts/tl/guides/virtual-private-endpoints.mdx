# Gamitin ang mga virtual private endpoint para sa VPC para kumonekta nang pribado sa IBM Quantum Platform

Gamitin ang Virtual Private Endpoints (VPE) para sa VPC para kumonekta sa IBM Quantum&reg; mula sa iyong VPC network gamit ang IP address na iyong tinutukoy, na inilaan mula sa isang subnet sa loob ng iyong VPC.

Ang mga VPE ay mga virtual IP interface na nakatali sa isang endpoint gateway na nilikha batay sa bawat serbisyo. Ang endpoint gateway ay isang virtualized na function na nagpapalaki nang pahalang, ay redundant at lubos na available, at sumasaklaw sa lahat ng availability zone ng iyong VPC. Ang mga endpoint gateway ay nagbibigay-daan sa komunikasyon mula sa mga virtual server instance sa loob ng iyong VPC at IBM Cloud&reg; service sa pribadong backbone. Ang VPE para sa VPC ay nagbibigay sa iyo ng kontrol sa lahat ng pribadong address sa loob ng iyong cloud. Para sa karagdagang impormasyon, tingnan ang [About virtual private endpoint gateways](https://cloud.ibm.com/docs/vpc?topic=vpc-about-vpe).

Sa loob ng IBM Quantum Platform, lahat ng data ng customer ay ipinapadala sa pamamagitan ng pribadong network, anuman ang paraan ng pag-access — public endpoint man o VPE.

Ang pagkonekta sa IBM Quantum Platform sa pamamagitan ng mga public endpoint ay nagpapadala ng lahat ng request at response na data sa pamamagitan ng pampublikong internet. Para kumonekta sa IBM Quantum Platform gamit ang VPE, kailangan mong gamitin ang IBM Quantum Platform API o SDK. Ang IBM Quantum Platform ay maaari lamang ma-access sa pamamagitan ng pampublikong network.

## Bago ka magsimula {#before-you-begin}

Bago ka mag-target ng VPE para sa IBM Quantum Platform, kumpletuhin ang mga sumusunod na hakbang:

* [Gumawa ng Virtual Private Cloud](https://cloud.ibm.com/docs/vpc?topic=vpc-getting-started).
* [Mag-plano para sa network topology](https://cloud.ibm.com/docs/vpc?topic=vpc-planning-considerations) para kumonekta sa mga VPE.
* [Magtakda ng mga access control](https://cloud.ibm.com/docs/vpc?topic=vpc-configure-acls-sgs-endpoint-gateways) para sa iyong VPE.
* Unawain ang mga [limitasyon](https://cloud.ibm.com/docs/vpc?topic=vpc-limitations-vpe) ng pagkakaroon ng VPE.
* Unawain kung paano [tingnan ang mga detalye ng VPE](https://cloud.ibm.com/docs/vpc?topic=vpc-vpe-viewing-details-of-an-endpoint-gateway).

## Mag-set up ng VPE para sa IBM Quantum Platform {#set-up-a-vpe-for-ibm-quantum-platform}

May ilang paraan para gumawa ng VPE gateway. Kung gagamitin mo ang [CLI](https://cloud.ibm.com/docs/vpc?topic=vpc-vpc-reference&interface=cli#vpe-clis) o [API](https://cloud.ibm.com/apidocs/vpc/latest#create-endpoint-gateway), kailangan mong tukuyin ang [Cloud Resource Name (CRN)](https://cloud.ibm.com/docs/account?topic=account-crn) ng rehiyon kung saan mo gustong kumonekta sa IBM Quantum Platform. Suriin ang sumusunod na talahanayan para sa mga available na rehiyon at CRN.

| Rehiyon | Mga Plano   | Fully Qualified Domain Name (FDQN) | Cloud Resource Name (CRN) |
|-----------------|-----------------|-----------------|-----------------|
| `us-east` | `Standard` | `private.us-east.quantum.cloud.ibm.com` | `crn:v1:bluemix:public:quantum-computing:us-east:::endpoint:private.us-east.quantum.cloud.ibm.com` |
| `eu-de` | `Standard` | `private.eu-de.quantum.cloud.ibm.com` | `crn:v1:bluemix:public:quantum-computing:eu-de:::endpoint:private.eu-de.quantum.cloud.ibm.com` |

### Pag-configure ng endpoint gateway {#configuring-an-endpoint-gateway}

Para i-configure ang isang VPE gateway, sundin ang mga hakbang na ito:

1. Ilista ang mga available na serbisyo, kasama ang mga IBM Cloud infrastructure service na available (bilang default) para sa lahat ng VPC user.
1. [Gumawa ng endpoint gateway](https://cloud.ibm.com/docs/vpc?topic=vpc-ordering-endpoint-gateway) para sa IBM Quantum Platform na gusto mong maging pribadong available sa VPC.
1. [I-bind ang isang reserved na IP address](https://cloud.ibm.com/docs/vpc?topic=vpc-bind-unbind-reserved-ip) sa endpoint gateway.
1. [Tingnan ang mga nalikha na VPE gateway](https://cloud.ibm.com/docs/vpc?topic=vpc-vpe-viewing-details-of-an-endpoint-gateway) na nauugnay sa IBM Quantum Platform.

Ngayon ang iyong mga virtual server instance sa VPC ay maaaring ma-access ang iyong IBM Quantum Platform instance nang pribado.

## Gamitin ang iyong VPE para sa IBM Quantum Platform {#use-your-vpe-for-ibm-quantum-platform}

Pagkatapos gumawa ng endpoint gateway para sa IBM Quantum Platform, sundin ang mga hakbang na ito:

### Gamitin ang VPE kasama ang `qiskit-ibm-runtime` (Python SDK) {#use-the-vpe-with-qiskit-ibm-runtime-python-sdk}

<Admonition type="note">
Ang suporta sa VPE ay nangangailangan ng `qiskit-ibm-runtime` v0.24.0 o mas bago.
</Admonition>

Kapag nag-instantiate ng `QiskitRuntimeService`, tukuyin ang `private_endpoint=True`.

```python
service = QiskitRuntimeService(token="your-API_KEY", # Use the 44-character API_KEY you created and saved from the IBM Quantum Platform Home dashboard
                              instance="INSTANCE_CRN",
                              channel="ibm_cloud",
                              private_endpoint=True)
```

### Gamitin ang VPE kasama ang IBM Quantum API {#use-the-vpe-with-the-ibm-quantum-api}

Pagkatapos gumawa ng endpoint gateway para sa IBM Quantum Platform, gamitin ang FQDN ng service endpoint para sa target na rehiyon.

```bash
  curl -X POST https://private.us-east.quantum.cloud.ibm.com/api/v1/jobs -H "Authorization: Bearer $BEARER_TOKEN" -H "Service-CRN: $SERVICE_INSTANCE_CRN" -d '{
    "backend": "backend",
    "program_id": "sampler"
  }'
```

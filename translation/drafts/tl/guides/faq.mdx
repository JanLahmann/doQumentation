# Mga Madalas na Tanong (FAQ)

Para sa mga FAQ na partikular sa mga execution mode, tingnan ang gabay na [Qiskit Runtime execution modes FAQ](/guides/execution-modes-faq).

## Paano ko isi-cite ang Qiskit sa aking pananaliksik? {#how-do-i-cite-qiskit-in-my-research}

I-cite ang Qiskit gamit ang kasamang [BibTeX file](https://github.com/Qiskit/qiskit/blob/main/CITATION.bib).

## Paano ko isi-cite ang isang IBM Quantum computer sa aking pananaliksik? {#how-do-i-cite-an-ibm-quantum-computer-in-my-research}

Para sa mga research paper, hinihikayat namin ang mga may-akda na kilalanin ang IBM Quantum&reg; sa pamamagitan ng:

>    We acknowledge the use of IBM Quantum services for this work. The views expressed are those of the authors, and do not reflect the official policy or position of IBM or the IBM Quantum team.

Ang mga sanggunian sa papel ay dapat gumamit ng sumusunod na citation:

> IBM Quantum. https://<span>quantum.cloud.ibm.com</span>/, YYYY

Ang mga quantum processing unit sa papel ay dapat i-reference gamit ang kanilang natatanging pangalan (halimbawa, `ibmq_vigo`) at maaaring idagdag ang bersyon (halimbawa, `ibmq_vigo` v1.0.2). Hinihikayat din naming i-reference ang processor.
Halimbawa:

> In this paper we used `ibmq_vigo`, which is one of the IBM Quantum Canary processors.

Halimbawa ng pag-cite ng IBM Quantum program:

> IBM Quantum (2025). Estimator primitive (Version x.y.z) [computer software]. https://<span>quantum.cloud.ibm.com</span>/

## Paano ko isi-cite ang IBM Quantum Composer sa aking pananaliksik? {#how-do-i-cite-ibm-quantum-composer-in-my-research}

Halimbawa ng citation para sa IBM Quantum Composer:

> IBM Quantum Composer. 2025. url: https://<span>quantum.cloud.ibm.com/composer</span>

## Hindi ko nakikita ang isang bagay sa user interface, tulad ng mga gawain sa administrasyon. {#i-dont-see-something-in-the-user-interface-such-as-my-administration-tasks}

Subukang pumili ng ibang account mula sa account switcher sa header ng [IBM Quantum Platform](https://quantum.cloud.ibm.com) na interface.

## Gumagamit lang ako ng Open Plan, pero naglagay pa rin ako ng credit card. Sisisingilin ba ako? {#im-just-using-the-open-plan-but-i-had-to-put-in-my-credit-card-will-i-be-charged}

Hindi ka sisisingilin maliban kung gumawa ka ng Pay-As-You-Go [instance](/guides/instances) at ginamitin ito para magpadala ng mga workload sa QPUs. Tingnan ang [Manage cost](/guides/manage-cost) para sa mga tagubilin sa pagtatakda ng limitasyon sa gastos at para malaman ang iba pang paraan ng pamamahala ng gastos.

## Ano ang pagkakaiba ng API key at API token? {#whats-the-difference-between-an-api-key-and-an-api-token}

Wala itong pagkakaiba. Ang mga terminong API key at API token ay ginagamit nang palitan. Makikita mo ang parehong termino sa dokumentasyon dahil gumagamit ng iba't ibang termino ang iba't ibang interface.

<span id="bearer"></span>
## Ano ang pagkakaiba ng API token at bearer token? {#whats-the-difference-between-an-api-token-and-a-bearer-token}

Ang API token ay isang pangmatagalang kredensyal na ginagamit para [mag-authenticate sa Qiskit Runtime service.](/guides/hello-world#install-and-authenticate) Ang bearer token naman ay isang pansamantalang kredensyal na ginagamit para [mag-authenticate sa REST API sa hindi pinagkakatiwalaang kapaligiran.](/guides/cloud-setup-untrusted#authenticate-to-the-rest-api-in-an-untrusted-environment) Hindi maaaring gamitin ang bearer token para mag-authenticate sa Qiskit Runtime service.

## Kailangan ko bang pumili ng rehiyon na tinitirhan ko? {#do-i-have-to-choose-the-region-that-i-live-in}

Hindi. Maaari kang gumamit ng alinmang rehiyon. Tinutukoy ng rehiyon kung saan pinapatakbo ang mga trabaho at kung saan pinapanatili ang data. Bukod dito, makikita at magagamit mo lang ang mga instance na nilikha sa instance na naka-log in ka. Tandaan na ang us-east na rehiyon lang ang sumusuporta sa Open Plan.

## Maaari ba akong kumonekta sa pamamagitan ng proxy server? {#can-i-connect-through-a-proxy-server}

Oo, pero kailangan mong gumamit ng Qiskit Runtime v0.44.0 o mas bago.

## Ano ang patakaran sa pagpapanatili ng data? {#what-is-the-data-retention-policy}

Pinapanatili ng IBM ang iyong (hindi pa tinanggal na) data ng trabaho at paggamit sa loob ng tatlong taon habang patuloy mong ginagamit ang Qiskit Runtime service.

## Ano ang nangyari sa mga Python distribution tulad ng Qiskit Aer, Qiskit Nature, at Qiskit Experiments? {#what-happened-to-python-distributions-such-as-qiskit-aer-qiskit-nature-and-qiskit-experiments}

Mula nang ilabas ang Qiskit SDK v1.0, ang command na `pip install qiskit` ay mag-i-install **lamang** ng [Qiskit SDK](https://github.com/Qiskit/qiskit). Ibig sabihin, ang ilang module na ginamit mo noon, tulad ng `qiskit-aer` o `qiskit-nature`, ay kailangan nang i-install nang hiwalay.

Bago ilabas ang Qiskit SDK v1.0, ang Qiskit ay binubuo ng maraming iba't ibang Python distribution, tulad ng `qiskit-terra`, ang compiler core; `qiskit-aer`, ang high-performance simulator; ang orihinal na IBM Quantum&reg; provider; at ilang ngayon ay lipas na (obsolete) na package na nagbibigay ng mga partikular na eksperimental na algorithmic o experiment-running na feature.

Ang mga hiwalay na software package na ito ay pinagsama sa tinatawag na Qiskit *metapackage*, na nagpahintulot sa mga user na i-install ang lahat ng component na ito nang sabay-sabay sa pamamagitan ng `pip install qiskit`. Ang metapackage architecture na ito ay tinanggal nang ilabas ang Qiskit SDK v1.0, at ngayon ay kailangang i-install nang hiwalay ang bawat isa sa mga indibidwal na software distribution.

Kung gusto mong i-install ang mga package na dati ay kasama sa Qiskit metapackage, bisitahin ang [Qiskit ecosystem](https://www.ibm.com/quantum/ecosystem) para makahanap ng iba't ibang package na angkop sa iyong pangangailangan. Maaari mo ring basahin ang [v1.0 migration guide](/guides/metapackage-migration) para sa karagdagang impormasyon tungkol sa pagbabagong ito.

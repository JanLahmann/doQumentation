# Gamitin ang IBM Cloud Platform APIs para ma-access ang mga instance

Ang gabay na ito ay para sa sinumang gumagamit ng IBM Cloud Platform APIs para mag-provision, mag-update, o pamahalaan ang access sa mga instance. Ibinibigay nito ang mga detalye ng [mga parameter](#parameters) na maaari mong itakda kapag nagpo-provision ng instance, pati na rin ang hanay ng [mga extension field](#extensions) na ibinabalik kapag ina-access mo ang isang instance.

<Admonition type="note">
Tingnan ang dokumentasyon ng IBM Cloud&reg; ResourceController API para sa karagdagang impormasyon.

Ilang mga kaugnay na pahina:
- [Gumawa (mag-provision) ng bagong resource instance](https://cloud.ibm.com/apidocs/resource-controller/resource-controller#create-resource-instance)
- [I-update ang isang resource instance](https://cloud.ibm.com/apidocs/resource-controller/resource-controller#update-resource-instance)
- [Kunin ang isang resource instance](https://cloud.ibm.com/apidocs/resource-controller/resource-controller#get-resource-instance)
- [Kunin ang listahan ng lahat ng resource instance](https://cloud.ibm.com/apidocs/resource-controller/resource-controller#list-resource-instances)

    Kapag kinukuha ang listahan ng lahat ng resource instance, tandaan na maaari kang mag-filter para makita lang ang mga Qiskit Runtime instance gamit ang `resource_id = "b6049020-80f4-11eb-a0f7-e35ec9b4054f"`

</Admonition>

## `parameters` {#parameters}

Sa body kapag nagpo-provision o nag-a-update ng instance:

- **instance_limit_seconds** (string o null, `^[0-9]+$`): Ang limitasyon na itatakda sa instance. Para sa Open Plan: kung hindi tinukoy, ang default nito ay 10 minuto.
.
- **usage_allocation_seconds** (string o null, `^[0-9]+$`): Inilaan na oras para sa instance, ginagamit ng fair-share scheduler para matukoy ang priyoridad sa pila, batay sa paggamit ng lahat ng QPU. (Hindi ito naaangkop sa mga Pay-As-You-Go instance.)
- **backends** (array ng mga listahan ng backends (strings o null) o array na may "ANY"): ang allow-list ng mga pangalan ng backends na available sa instance na ito. Default: `["ANY"]`, ibig sabihin lahat ng backends na available sa planong ito. `[]` ibig sabihin walang backends.

## `extensions` {#extensions}

Sa response kapag nagpo-provision o kapag kinukuha ang mga detalye ng instance:

Mga karagdagang katangian ng instance na ibinibigay ng serbisyo at kinakatawan bilang mga key-value pair. Ang mga katangiang ito ay nananatili sa instance sa ilalim ng extensions property.

- **instance_limit_seconds** (integer o null, `>= 0`): Ang limitasyon sa oras na itatakda sa instance.
- **usage_allocation_seconds** (integer o null, `>= 0`): Inilaan na oras para sa instance, ginagamit ng fair-share scheduler para matukoy ang priyoridad sa pila, batay sa paggamit ng lahat ng QPU.
- **backends** (Kinakailangan; array ng mga listahan ng backends (strings) o array na may "ANY" (strings)): Ang allow-list ng mga pangalan ng backends na available sa instance na ito. Default: `["ANY"]`, ibig sabihin lahat ng backends ay available sa planong ito. `[]` ibig sabihin walang backends.

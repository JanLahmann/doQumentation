# Talahanayan ng mga feature ng OpenQASM 3 {#openqasm-3-feature-table}

Nasa ibaba ang listahan ng mga feature ng wikang `OpenQASM 3`.

Para sa mas detalyadong impormasyon tungkol sa mga kakayahang ito, tingnan ang [OpenQASM 3.X Live Specification](https://openqasm.com/).

Susi:

*   ❌ Hindi sinusuportahan
*   🟡 Bahagyang sinusuportahan
*   ✅ Sinusuportahan

Ang kahulugan ng kumpletong marka na "sinusuportahan" ay depende sa column:

* *Qiskit SDK*: Maaaring i-parse ang feature gamit ang [`qiskit.qasm3.loads`](https://docs.quantum.ibm.com/api/qiskit/qasm3) (sa pamamagitan ng `qiskit-qasm3-import` extension), marepresenta sa isang [`QuantumCircuit`,](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.QuantumCircuit)
  at ma-export sa OpenQASM 3 gamit ang [`qiskit.qasm3.dumps`.](https://docs.quantum.ibm.com/api/qiskit/qasm3#dumps)

* *IBM Qiskit Runtime*: Ang isang Circuit na naglalaman ng kaukulang Qiskit feature ay maaaring matagumpay na isagawa sa hardware sa pamamagitan ng IBM&reg; Qiskit Runtime.

Ang kahulugan ng "bahagyang sinusuportahan" ay karaniwang nakasalalay sa mga nakalinkang tala.

<Admonition type="note">
Ang pinakakaraniwang paraan ng pagsusumite ng mga Circuit sa IBM Qiskit Runtime ay ang paglikha ng Circuit sa Python-space interface ng Qiskit SDK.
Ang mga Circuit na nilikha at isinumite sa ganitong paraan ay hindi na kailangang i-load mula sa mga OpenQASM 3 file papunta sa Qiskit SDK.

Kung hindi mo direktang ginagamit ang OpenQASM 3, maaari kang ligtas na gumamit ng mga feature na sinusuportahan para sa representasyon sa Qiskit SDK, pag-export sa OpenQASM 3, at pagsusumite sa IBM Qiskit Runtime.
Kasama rito ang mga feature na hindi maaaring i-load ng Qiskit SDK mula sa OpenQASM 3.
</Admonition>

## Mga Tala {#notes}

<span id="1"></span>
1.  Ang mga feature ng OpenQASM 3 program na ito ay walang epekto sa pagpapatupad at tinatanggal ng Qiskit ang mga ito bilang bahagi ng pag-parse ng mga file. Ang mga file na gumagamit ng mga ito ay maaaring isumite ngunit wala silang epekto. Para sa mga `include` file, ang `stdgates.inc` ay kasalukuyang sinusuportahan bilang input sa Qiskit, at ang pagpapatupad sa backend ay palaging nangangailangan na ang mga Circuit ay na-compile sa backend Instruction Set Architecture (ISA), kung saan ang mga `include` file ay hindi na kaugnay.

<span id="2"></span>
2. Sinusuportahan ng Qiskit SDK ang pag-parse at pag-dump ng mga OpenQASM 3 file na may anumang `qubit` declaration. Para sa pagpapatupad sa hardware, tanging ang mga Circuit na tinukoy sa mga hardware qubit (halimbawa, `$0`) ang valid. Awtomatikong naglalagay ang Qiskit SDK ng OpenQASM 3 sa mga sinusuportahang hardware-qubit identifier kung ang Circuit ay na-transpile para sa isang backend na may impormasyon ng layout.

<span id="3"></span>
3. Ang mga variable declaration na may uri `bit` at `bit[n]` sa Qiskit SDK ay tumutugma sa mga `Clbit` at `ClassicalRegister` declaration.

<span id="4"></span>
4. Simula Hulyo 2025, kaya ng Qiskit SDK na kumatawan sa mga lokal na variable ng limitadong hanay ng mga uri, kumatawan sa maraming runtime operation sa mga object na ito, at sumusuporta sa pag-output ng mga ito sa OpenQASM 3. Gayunpaman, ang Qiskit SDK (sa pamamagitan ng `qiskit-qasm3-import` v0.6.0) ay hindi sumusuporta sa pag-parse ng mga OpenQASM 3 file na naglalaman ng mga variable declaration, at napaka-limitado ang suporta nito sa pag-parse ng mga variable expression. Sa pangkalahatan, karamihan sa kung ano ang maaaring katawanin ng Qiskit sa expression system nito ay maaaring isagawa sa angkop na dynamic circuits hardware, kahit na hindi pa maaaring ma-parse ng Qiskit SDK ang expression. Tingnan ang [dokumentasyon ng Qiskit para sa `qiskit.circuit.classical` module](https://docs.quantum.ibm.com/api/qiskit/circuit_classical) para sa pinakabagong impormasyon.

<span id="5"></span>
5. Kaya ng Qiskit SDK na katawanin ang register aliasing para sa parehong quantum at classical register, ngunit lubos na hindi inirerekomenda ang paggamit ng aliasing ng mga classical register. Karamihan sa mga expression sa mga classical register ay hindi gumagana sa mga alias, at ang mga aliased classical register ay hindi sinusuportahan para sa pagpapatupad sa hardware. Ang Qiskit OpenQASM 3 parser ay maaaring mag-resolve ng `let` alias statement na nagbibigkis sa resulta ng register concatenation.

<span id="6"></span>
6. Sinusuportahan ng Qiskit SDK ang mga explicit na delay sa pamamagitan ng `QuantumCircuit.delay`, at ang mga circuit box (`QuantumCircuit.box`) ay maaari ring magkaroon ng mga explicit na tagal. Ang mga tagal na ito ay maaaring magsama ng mga classical expression ng mga `stretch` variable. Ang Qiskit SDK (simula Hulyo 2025 sa pamamagitan ng `qiskit-qasm3-import` v0.6.0) ay hindi sumusuporta sa pag-parse ng mga declaration ng uri `duration` o uri `stretch` mula sa mga OpenQASM 3 file. Limitado ang suporta ng hardware para sa mga tagal na kinabibilangan ng `stretch`.

<span id="7"></span>
7. Ang mga Circuit ay dapat i-transpile sa backend ISA upang tumakbo sa IBM hardware. Dahil dito, ang mga custom na `gate` definition at mas mataas na antas na konstruksyon tulad ng mga gate modifier (tulad ng `inv @`) ay hindi valid para sa direktang pagpapatupad sa hardware, ngunit ang proseso ng `transpile` ay nireresulba ang mga ito sa mga valid na ISA circuit. Ang Qiskit SDK (simula Hulyo 2025, sa pamamagitan ng `qiskit-qasm3-import` v0.6.0) ay agad na susuriin ang mga gate modifier habang nini-parse, kaya hindi na ito makikita sa resultang `QuantumCircuit`, na posibleng may kasamang runtime cost.

<span id="8"></span>
8. Kaya ng Qiskit SDK na katawanin ang structured control flow at i-export ito sa OpenQASM 3. Ang mga statement na `continue` at `break` ay teknikal na makakatawan ng Qiskit, ngunit hindi well-supported kahit sa loob ng Qiskit SDK. Ang mga `for` loop sa Qiskit v2.1.0 ay hindi well-supported. Ang mga nested control flow (tulad ng isang `if` sa loob ng isa pang `if`, o isang `else if` statement) ay hindi eligible para sa pagpapatupad sa hardware.

<span id="9"></span>
9. Sinusuportahan ng Qiskit SDK ang pag-declare ng anumang sinusuportahang classical type bilang isang `input` variable sa Circuit. Ang mga variable na ito ay hindi kasalukuyang eligible para sa pagpapatupad sa hardware, at hindi maaaring i-load ng Qiskit OpenQASM 3 importer. Ang mga hindi nakatali na `Parameter` object na naroroon sa `QuantumCircuit` ay inie-export bilang mga `input float[64]` variable. Ang ilang partikular na runtime configuration option ay maaaring magpahintulot sa pagpapatupad ng mga ganitong Circuit sa ilang backend.

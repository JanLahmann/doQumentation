# Logging para sa Qiskit Runtime

Ang mga serbisyo ng IBM Cloud&reg;, tulad ng Qiskit Runtime, ay gumagawa ng mga platform log na magagamit mo para imbestigahan ang hindi normal na aktibidad at mahahalagang aksyon sa iyong account, at para malutas ang mga problema.

Magagamit mo ang [IBM Cloud Logs Routing,](https://cloud.ibm.com/docs/logs-router?topic=logs-router-about) na isang platform service, para i-route ang mga platform log sa iyong account patungo sa destinasyon na gusto mo sa pamamagitan ng pag-configure ng tenant na nagtatakda kung saan ipapadala ang mga platform log. Para sa karagdagang impormasyon, tingnan ang [About Logs Routing.](https://cloud.ibm.com/docs/logs-router?topic=logs-router-about)

Magagamit mo ang [IBM Cloud Logs](https://cloud.ibm.com/docs/cloud-logs) para i-visualize at makatanggap ng alerto sa mga platform log na ginawa sa iyong account at na-route ng IBM Cloud Logs Routing patungo sa isang IBM Cloud Logs instance.

## Mga lokasyon kung saan ginagawa ang mga platform log {#locations-where-platform-logs-are-generated}

Ang IBM Cloud Container Registry ay gumagawa ng mga platform log na ipinapakita sa iyong mga logging instance. Inililista ng sumusunod na talahanayan ang mga lokasyon kung saan pinagana ang awtomatikong koleksyon ng mga Container Registry service log.

| Lokasyon                   | Available ba ang mga platform services log? |
|----------------------------|---------------------------------------------|
| Washington, D.C. (us-east) |                    Oo                       |
| Frankfurt (eu-de)          |                    Oo                       |

## Mga platform log na ginagawa {#platform-logs-that-are-generated}

Ang IBM Qiskit Runtime ay gumagawa ng mga platform services log na ipinapakita sa iyong mga logging instance para sa mga sumusunod na partikular na kaso:

- Kapag nagbago ang estado ng isang job.

## Pagtingin sa mga log mula sa pahina ng Observability {#viewing-logs-from-the-observability-page}

Para sa impormasyon tungkol sa paglulunsad ng IBM Cloud Logs interface, tingnan ang **Launching the UI** sa [IBM Cloud Logs documentation.](https://cloud.ibm.com/docs/cloud-logs?topic=cloud-logs-instance-launch)

## Mga field ayon sa uri ng log {#fields-by-log-type}

Para sa impormasyon tungkol sa mga field na kasama sa bawat platform log, tingnan ang [Fields that are included in platform logs.](https://cloud.ibm.com/docs/logs-router?topic=logs-router-about-platform-logs#about-platform-logs-2)

Ang mga sumusunod na field ay kasama sa log record.

| Field           | Uri     | Paglalarawan                                                                              |
|-----------------|---------|-------------------------------------------------------------------------------------------|
| logSourceCRN    | Required | Tinutukoy ang account at flow log instance kung saan na-publish ang log.                 |
| saveServiceCopy | Required | Tinutukoy kung nag-iingat ang IBM ng kopya ng record para sa mga layuning operasyonal.   |
| message         | Required | Paglalarawan ng log na ginawa.                                                            |

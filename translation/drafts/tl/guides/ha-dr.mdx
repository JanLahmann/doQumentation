# Pag-unawa sa mataas na availability at disaster recovery para sa Qiskit Runtime {#understanding-high-availability-and-disaster-recovery-for-qiskit-runtime}

Ang high availability (HA) ay ang kakayahan ng isang serbisyo na manatiling gumagana at naa-access kahit may mga hindi inaasahang pagpalya. Ang disaster recovery naman ay ang proseso ng pagbabalik ng service instance sa isang gumaganang estado. Ang mga customer ng Qiskit Runtime ay hindi responsable sa pagpaplano o pagkilos para sa disaster recovery.

Ang Qiskit Runtime ay isang regional na serbisyo na tumutupad sa mga tinukoy na [Service Level Objectives (SLO)](https://cloud.ibm.com/docs/resiliency?topic=resiliency-slo) sa anumang bayad na plano. Para sa karagdagang impormasyon, tingnan ang [Service Level Agreement (SLA)](https://www.ibm.com/support/customer/csol/terms/?id=i126-9268&lc=en) para sa IBM Cloud&reg;. Para sa karagdagang impormasyon tungkol sa mga available na IBM Cloud region at data center para sa Qiskit Runtime, tingnan ang [Service and infrastructure availability by location.](https://cloud.ibm.com/docs/overview?topic=overview-services_region)

## Arkitektura ng high availability {#high-availability-architecture}

### Mga tampok ng high availability {#high-availability-features}

Ang serbisyong ito ay ini-provision sa tatlong zone sa isang multi-zone region na walang single point of failure. Ang mga API request ay ini-route sa pamamagitan ng load balancer papunta sa mga HA instance node, na bawat isa ay nasa iba't ibang zone.

Kung may HA instance node o zone na mabigo, ang serbisyo ay patuloy na nagpapatakbo ng mga API request sa pamamagitan ng mga natitirang HA instance node. Walang kapansin-pansing pagbaba ng kalidad ng serbisyo ang nagaganap sa panahon ng zonal failure.

Available ito sa mga sumusunod na region:

- Washington DC (us-east)
- Frankfurt (eu-de)

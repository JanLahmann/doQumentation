# Itakda ang mga limitasyon sa allocation ng instance

Ang isang account ay maaaring may nakalaan na oras sa Premium o Flex plan. Dahil ang bawat account ay maaaring magkaroon ng maraming instance sa isang plan, ang pagtatakda ng mga limitasyon sa allocation ay tumutulong sa iyo na kontrolin kung aling mga instance ang pinapayagang gamitin ang oras na iyon. Tandaan na kung ang allocation ay hindi itinakda bilang limitasyon, maaaring gumamit ang instance ng higit sa nakalaan nitong oras.

Ang oras na inilalaan mo sa mga instance ay ginagamit ng fair-share algorithm upang matukoy ang priyoridad sa queue, batay sa paggamit ng lahat ng QPU sa loob ng rolling time window na 28 araw (para sa mga gumagamit ng Premium Plan) o sa buong subscription period (para sa mga gumagamit ng Flex Plan).

Ang allocation ay itinakda sa paglikha ng instance. Gayunpaman, maaari mong tingnan at baguhin ang nakalaan na oras sa [pahina ng mga instance.](https://quantum.cloud.ibm.com/instances)

## Ano ang nangyayari kapag naabot ang limitasyon {#what-happens-when-the-limit-is-reached}

Maaari mong opsyonal na itakda ang time allocation bilang isang limitasyon para sa instance. Narito ang mga mangyayari kapag naabot ng instance ang limitasyon nito:

- Patuloy na tumatakbo ang mga aktibong workload.
- Nananatiling naka-queue ang mga nakabinbing job.
- Walang workload na kinakansela.
- Maaari pa ring mag-submit ng mga bagong workload ang mga collaborator, na mananatiling naka-queue hanggang sa magkaroon ng karagdagang oras.
- Para sa Flex Plan, hindi na makakapag-proseso ng mga karagdagang job ang instance maliban na lang kung naitaas na ang limitasyon o, para sa Premium Plan, pagkatapos magsimula ang susunod na usage window.

## Magtakda ng iba't ibang halaga para sa allocation limit at usage limit {#set-different-values-for-the-allocation-limit-and-usage-limit}

Kapag itinakda mo ang allocation limit para sa isang instance sa IBM Quantum Platform, ang parehong halaga ay itinakda bilang usage limit ng instance. Kung gusto mong makapag-submit at makapagpatakbo pa rin ng mga job ang mga gumagamit pagkatapos maabot ang allocation limit ng instance, maaari kang magtakda ng mas mataas na halaga para sa usage limit sa pamamagitan ng paggamit ng parameter na [usage_allocation_seconds](/guides/access-instances-platform-apis#parameters) sa [Update a resource instance](https://cloud.ibm.com/apidocs/resource-controller/resource-controller#update-resource-instance) API.

# Pamahalaan ang mga IBM Cloud user

Ipinapakita ng gabay na ito kung paano gamitin ang IBM Cloud&reg; para paganahin ang mga user na may IBM Cloud account at nagbibigay ng mga tagubilin para ma-access ng mga user ang kapaligiran.

Para pamahalaan ang mga ID provider user sa halip, sundin ang mga tagubilin sa paksa ng [Pamahalaan ang mga ID provider](/guides/manage-appid).

## Mga hakbang {#steps}

Sundin ang mga mataas na antas na hakbang na ito para pamahalaan ang mga IBM Cloud user:

1. Siguraduhing ang mga user na gusto mong imbitahan ay may IBM Cloud account.
2. Pumunta sa IAM at [imbitahan ang mga user](https://cloud.ibm.com/iam/users/invite_users)
   <Admonition type="note">
   Hindi maaaring pamahalaan ang mga user hanggang hindi nila tinatanggap ang imbitasyon at naka-log in nang kahit isang beses.
   </Admonition>
3. Opsyonal: Baguhin ang mga takdang proyekto at access group ng mga user mula sa [IAM.](https://cloud.ibm.com/iam/users)

<span id="user-org"></span>
## Daloy ng user {#user-flow}

1. Tinatanggap ng user ang imbitasyon, pagkatapos ay nag-log on sa pamamagitan ng [IBM Cloud portal](https://cloud.ibm.com/).
2. Gumagawa ang user ng API key sa pamamagitan ng pagpunta sa ([Manage → Access (IAM) → API keys](https://cloud.ibm.com/iam/apikeys)).
3. Para sa karagdagang impormasyon, maaaring tingnan ng mga user ang [I-install ang Qiskit.](/guides/install-qiskit)

<span id="steps-cloud-org"></span>
## Halimbawang sitwasyon {#example-scenario}

Nililikha ng halimbawang ito ang sumusunod na setup:

* May dalawang proyekto, `ml` at `finance`.
   * Ang proyektong `ml` ay dapat may access sa mga service instance na `QR-ml` at `QR-common`.
   * Ang proyektong `finance` ay dapat may access sa mga service instance na `QR-finance` at `QR-common`.
* May tatlong user:
   * Si Fatima ay dapat may access sa proyektong `ml`.
   * Si Ravi ay dapat may access sa proyektong `finance`.
   * Si Amyra ay dapat may access sa parehong proyekto.
* Gumagamit ito ng mga access group nang walang resource group.
* Ang mga user ay tinukoy sa IBM Cloud at ginagawa rin doon ang mga takdang proyekto.
* Dapat na makakagawa ang mga user ng delete ng mga trabaho.

Ang mga hakbang para maipatupad ang setup na ito ay:

2. Gumagawa ang Cloud administrator ng tatlong service instance: `QR-ml`, `QR finance`, at `QR-common`.
3. Gumagawa ang Cloud administrator ng custom rule na nagsasama ng aksyong `quantum-computing.job.delete`.
1. Gumagawa ang Cloud administrator ng dalawang access group:
   * Ang access group na `ml` ay maaaring mag-access ng `QR-ml` at `QR-common`.
   * Ang access group na `finance` ay maaaring mag-access ng `QR-finance` at `QR-common`.
1. Inimbitahan ng Cloud administrator ang mga cloud user sa naaangkop na proyekto. Partikular, inimbitahan at itinalaga nila ang mga user sa isang access group na nagsasama ng proyekto.
   * Si Fatima ay idinagdag sa access group na "ml".
   * Si Ravi ay idinagdag sa access group na "finance".
   * Si Amyra ay idinagdag sa parehong "ml" at "finance" na access group.
1. Maaaring mag-log in ang mga user sa pamamagitan ng IBM Cloud portal, gumawa ng mga API key, at magtrabaho sa mga service instance ng kanilang mga proyekto.

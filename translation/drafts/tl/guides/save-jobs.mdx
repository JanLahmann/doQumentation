Ang mga quantum workflow ay madalas na tumatagal bago matapos at maaaring tumaas sa maraming session. Ang pag-restart ng iyong Python kernel ay nangangahulugang mawawala ang lahat ng mga resulta na naka-imbak sa memorya. Para maiwasan ang pagkawala ng data, maaari kang mag-save ng mga resulta sa file at i-retrieve ang mga resulta ng mga nakaraang job mula sa IBM Quantum&reg; para matuloy ng iyong susunod na session kung saan ka natigil.

## I-retrieve ang mga resulta ng job mula sa IBM Quantum {#retrieve-job-results-from-ibm-quantum}

Awtomatikong iniimbak ng IBM Quantum ang mga resulta mula sa bawat job para mai-retrieve mo sa ibang pagkakataon. Gamitin ang feature na ito para ituloy ang mga quantum program pagkatapos ng kernel restart at suriin ang mga nakaraang resulta. Makukuha mo ang ID ng isang job nang programatiko sa pamamagitan ng `job_id` method nito, o maaari kang tingnan ang lahat ng iyong mga na-submit na job at ang kanilang mga ID sa [pahina ng Workloads](https://quantum.cloud.ibm.com/workloads).

Para mahanap ang isang job nang programatiko, gamitin ang [`QiskitRuntimeService.jobs`](https://docs.quantum.ibm.com/api/qiskit-ibm-runtime/qiskit-runtime-service#jobs) method. By default, ibinabalik nito ang mga pinakabagong na-submit na job, pero maaari ka ring mag-filter ng mga job ayon sa backend name, petsa ng paglikha, at iba pa. Hinahanap ng sumusunod na cell ang anumang mga job na na-submit sa nakalipas na tatlong buwan. Ang `created_after` na argument ay dapat na isang [`datetime.datetime`](https://docs.python.org/3.8/library/datetime.html#datetime.datetime) na object.

```python
import datetime
from qiskit_ibm_runtime import QiskitRuntimeService

three_months_ago = datetime.datetime.now() - datetime.timedelta(days=90)

service = QiskitRuntimeService()
jobs_in_last_three_months = service.jobs(created_after=three_months_ago)
jobs_in_last_three_months[:3]  # show first three jobs
```

Maaari ka ring pumili ayon sa backend, estado ng job, session, at iba pa. Para sa karagdagang impormasyon, tingnan ang [`QiskitRuntimeService.jobs`](https://docs.quantum.ibm.com/api/qiskit-ibm-runtime/qiskit-runtime-service#jobs) sa dokumentasyon ng API.

Kapag mayroon ka nang job ID, gamitin ang [`QiskitRuntimeService.job`](https://docs.quantum.ibm.com/api/qiskit-ibm-runtime/qiskit-runtime-service#job) method para i-retrieve ito.

```python
# Get ID of most recent successful job for demonstration.

successful_job = next(
    j for j in service.jobs(limit=1000) if j.status() == "DONE"

job_id = successful_job.job_id()
print(job_id)
```

## I-save ang mga resulta sa disk {#save-results-to-disk}

Maaari ka ring mag-save ng mga resulta sa disk. Para magawa ito, gamitin ang built-in na JSON library ng Python kasama ang mga encoder mula sa Qiskit Runtime.

```python
import json
from qiskit_ibm_runtime import RuntimeEncoder

with open("result.json", "w") as file:
    json.dump(retrieved_job.result(), file, cls=RuntimeEncoder)
```

Maaari mo ring i-load ang array na ito mula sa disk sa isang hiwalay na kernel.

```python
from qiskit_ibm_runtime import RuntimeDecoder

with open("result.json", "r") as file:
    result = json.load(file, cls=RuntimeDecoder)

result
```

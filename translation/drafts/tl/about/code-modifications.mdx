# Awtomatikong Pagbabago ng Code

Awtomatiko ang doQumentation sa pag-apply ng iilang pagbabago sa upstream na
Qiskit tutorial at gabay para masiguro ang maayos at interactive na karanasan.
Idinetalye ng pahinang ito ang bawat pagbabago para maintindihan mo kung ano
ang nagbago kumpara sa orihinal na dokumentasyon ng IBM Quantum.

## Kopya ng Notebook (Open in Colab / Binder / Code Engine)

Kapag na-click mo ang **Open in Colab**, **Open in JupyterLab**, o **Open in Code
Engine**, makukuha mo ang kopya ng orihinal na notebook na may mga dagdag na ito:

### 1. Setup notice cell (markdown)

May blockquote cell na ipinasok sa pinakataas na nagpapaliwanag na
nagdagdag ang doQumentation ng awtomatikong setup cell. May link ito pabalik sa pahinang ito.

### 2. Prerequisites cell (code)

May code cell na ipinasok pagkatapos ng notice na:

- **Nag-iinstall ng mga kinakailangang package** (`qiskit`, `qiskit-aer`, `qiskit-ibm-runtime`,
  `pylatexenc`, kasama ang mga tutorial-specific na package na natukoy sa pamamagitan ng import
  scanning). **Nilalaktawan** ang install kung naka-install na ang mga package
  (hal. sa Binder o Code Engine kung saan pre-installed na ang mga ito).
- **Nagbibigay ng commented-out credentials template** para sa IBM Quantum, para sa mga gusto
  mag-run sa totoong hardware at maaaring mag-uncomment at maglagay ng kanilang API key.

Sa Google Colab, awtomatikong tumatakbo ang cell na ito kapag binuksan ang notebook sa pamamagitan ng
`cell_execution_strategy: setup` metadata flag.

### 3. Pag-rewrite ng image path

Ang mga relative image path (`/docs/images/...`, `/learning/images/...`) ay ina-rewrite
para gumana nang tama sa standalone na notebook environment.

## Mga MDX page (in-browser rendering)

Ang mga tutorial na ipinapakita sa website na ito ay kino-convert mula sa upstream na `.ipynb`
na mga notebook o `.mdx` na mga file. Inilalapat ang mga sumusunod na transform:

- **Mga `pip install` line** ay idinaragdag sa mga Python code block na nag-iimport ng
  third-party na mga package, para ma-enable ang one-click na execution sa pamamagitan ng thebelab.
- **IBM Tutorial Survey section**: Idinaragdag ang isang tala na nagliliwanag na ang survey
  ay sa IBM Quantum at may link sa GitHub Issues ng doQumentation para sa
  site-specific na feedback.
- **Feedback widget**: Isang "Was this helpful?" widget ang idinaragdag sa ibaba
  ng bawat tutorial, tina-track sa pamamagitan ng privacy-friendly na Umami analytics.
- **Mga pag-aayos ng MDX syntax**: Awtomatikong inaayos ang mga curly brace, heading hierarchy, at JSX compatibility
  na mga isyu para sa Docusaurus rendering.
- **OpenInLabBanner**: Isang interactive na banner ang inilalagay sa ilalim ng titulo na may
  mga button para buksan ang notebook sa Colab, Binder, o Code Engine.

## Ano ang HINDI binabago

- Ang **mismong nilalaman ng tutorial** (mga paliwanag, logic ng code, mga output) ay hindi
  kailanman binabago.
- **Ang attribution sa orihinal na may-akda** ay pinapanatili sa pamamagitan ng frontmatter at ng NOTICE
  na file (Apache 2.0 / CC BY-SA 4.0 na mga lisensya).
- **Walang telemetry o tracking code** na inino-inject sa mga notebook. Ang analytics
  (Umami) ay tumatakbo lang sa website ng doQumentation, hindi sa mga exported na notebook.

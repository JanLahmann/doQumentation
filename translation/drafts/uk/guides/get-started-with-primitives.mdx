## Початок роботи з Estimator {#start-estimator}

{/*Verified the v2 examples 2/29/24 - updated 10/29/24*/}

### 1. Ініціалізація акаунту

Оскільки Qiskit Runtime Estimator — це керований сервіс, спочатку потрібно ініціалізувати свій акаунт. Після цього можна вибрати QPU, яку ти хочеш використовувати для обчислення очікуваного значення.

Виконай кроки з [теми встановлення та налаштування](install-qiskit), якщо акаунт ще не налаштовано.

```python
from qiskit_ibm_runtime import QiskitRuntimeService

service = QiskitRuntimeService()
backend = service.least_busy(
    operational=True, simulator=False, min_num_qubits=127
)

print(backend.name)
```

```text
ibm_torino
```

### 2. Створення Circuit та спостережуваної величини

Тобі потрібен щонайменше один Circuit і одна спостережувана величина як вхідні дані для примітиву Estimator.

```python
from qiskit.circuit.library import qaoa_ansatz
from qiskit.quantum_info import SparsePauliOp

entanglement = [tuple(edge) for edge in backend.coupling_map.get_edges()]
observable = SparsePauliOp.from_sparse_list(
    [("ZZ", [i, j], 0.5) for i, j in entanglement],
    num_qubits=backend.num_qubits,
)
circuit = qaoa_ansatz(observable, reps=2)
# the circuit is parametrized, so we will define the parameter values for execution
param_values = [0.1, 0.2, 0.3, 0.4]

print(f">>> Observable: {observable.paulis}")
```

Circuit і спостережувана величина мають бути перетворені так, щоб використовувати лише інструкції, підтримувані QPU (відомі як схеми *instruction set architecture (ISA)*). Для цього скористаємося Transpiler.

```python
from qiskit.transpiler import generate_preset_pass_manager

pm = generate_preset_pass_manager(optimization_level=1, backend=backend)
isa_circuit = pm.run(circuit)
isa_observable = observable.apply_layout(isa_circuit.layout)
print(f">>> Circuit ops (ISA): {isa_circuit.count_ops()}")
```

```text
>>> Circuit ops (ISA): OrderedDict([('rz', 3826), ('sx', 1601), ('cz', 968)])
```

### 3. Ініціалізація Qiskit Runtime Estimator

Під час ініціалізації Estimator використовуй параметр `mode`, щоб вказати режим роботи. Можливі значення — `batch`, `session` або об'єкти `backend` для режимів пакетного, сеансового та одиночного виконання відповідно. Для отримання додаткової інформації дивись [Введення у режими виконання Qiskit Runtime.](execution-modes)

```python
from qiskit_ibm_runtime import EstimatorV2 as Estimator

estimator = Estimator(mode=backend)
```

### 4. Виклик Estimator та отримання результатів

Далі виклич метод `run()`, щоб обчислити очікувані значення для вхідних схем і спостережуваних величин. Circuit, спостережувана величина та необов'язкові набори значень параметрів передаються у вигляді кортежів *primitive unified bloc* (PUB).

```python
job = estimator.run([(isa_circuit, isa_observable, param_values)])
print(f">>> Job ID: {job.job_id()}")
print(f">>> Job Status: {job.status()}")
```

result = job.result()

## Початок роботи з Sampler {#start-sampler}

### 1. Ініціалізація акаунту

Оскільки Qiskit Runtime Sampler — це керований сервіс, спочатку потрібно ініціалізувати свій акаунт. Після цього можна вибрати QPU, яку ти хочеш використовувати для обчислення очікуваного значення.

Виконай кроки з [теми встановлення та налаштування](install-qiskit), якщо акаунт ще не налаштовано.

```python
from qiskit_ibm_runtime import QiskitRuntimeService

service = QiskitRuntimeService()
backend = service.least_busy(
    operational=True, simulator=False, min_num_qubits=127
)
```

### 2. Створення Circuit

Тобі потрібен щонайменше один Circuit як вхідні дані для примітиву Sampler.

```python
import numpy as np
from qiskit.circuit.library import efficient_su2

circuit = efficient_su2(127, entanglement="linear")
circuit.measure_all()
# The circuit is parametrized, so we will define the parameter values for execution
param_values = np.random.rand(circuit.num_parameters)
```

Використай Transpiler, щоб отримати ISA Circuit.

```python
from qiskit.transpiler import generate_preset_pass_manager

pm = generate_preset_pass_manager(optimization_level=1, backend=backend)
isa_circuit = pm.run(circuit)
print(f">>> Circuit ops (ISA): {isa_circuit.count_ops()}")
```

```text
>>> Circuit ops (ISA): OrderedDict([('sx', 3089), ('rz', 3036), ('cz', 1092), ('measure', 127), ('barrier', 1)])
```

### 3. Ініціалізація Qiskit Runtime Sampler

Під час ініціалізації Sampler використовуй параметр `mode`, щоб вказати режим роботи. Можливі значення — `batch`, `session` або об'єкти `backend` для режимів пакетного, сеансового та одиночного виконання відповідно. Для отримання додаткової інформації дивись [Введення у режими виконання Qiskit Runtime.](execution-modes) Зверни увагу, що користувачі Open Plan не можуть надсилати задачі в режимі сеансу.

```python
from qiskit_ibm_runtime import SamplerV2 as Sampler

sampler = Sampler(mode=backend)
```

### 4. Виклик Sampler та отримання результатів

Далі виклич метод `run()`, щоб отримати результати. Circuit і необов'язкові набори значень параметрів передаються у вигляді кортежів *primitive unified bloc* (PUB).

```python
job = sampler.run([(isa_circuit, param_values)])
print(f">>> Job ID: {job.job_id()}")
print(f">>> Job Status: {job.status()}")
```

## Початок роботи з бекенд-примітивами {#backend}

На відміну від примітивів, специфічних для провайдера, бекенд-примітиви — це загальні реалізації, які можна використовувати з довільним
об'єктом `backend`, якщо він реалізує інтерфейс [`Backend`](https://docs.quantum.ibm.com/api/qiskit/qiskit.providers.Backend).

- Примітив Sampler можна запустити з будь-яким провайдером за допомогою [`qiskit.primitives.BackendSamplerV2`](https://docs.quantum.ibm.com/api/qiskit/qiskit.primitives.BackendSamplerV2).
- Примітив Estimator можна запустити з будь-яким провайдером за допомогою [`qiskit.primitives.BackendEstimatorV2`](../api/qiskit/qiskit.primitives.BackendEstimatorV2).

Деякі провайдери реалізують примітиви нативно. Дивись [сторінку Qiskit Ecosystem](https://qiskit.github.io/ecosystem#provider) для деталей.
### Приклад: BackendEstimator

```python
from qiskit.primitives import BackendEstimatorV2
from <some_qiskit_provider> import QiskitProvider

provider = QiskitProvider()
backend = provider.get_backend('backend_name')
estimator = BackendEstimatorV2(backend)
```

### Приклад: BackendSampler

```python
from qiskit.primitives import BackendSamplerV2
from <some_qiskit_provider> import QiskitProvider

provider = QiskitProvider()
backend = provider.get_backend('backend_name')
sampler = BackendSamplerV2(backend)
```
### Схожість та відмінності між бекенд-примітивами та Runtime-примітивами

- Вхідні та вихідні дані [`qiskit.primitives.BackendSamplerV2`](https://docs.quantum.ibm.com/api/qiskit/qiskit.primitives.BackendSamplerV2) і [`qiskit.primitives.BackendEstimatorV2`](../api/qiskit/qiskit.primitives.BackendEstimatorV2)
дотримуються того ж формату PUB, що й примітиви в Qiskit Runtime. Дивись [Вхідні та вихідні дані примітивів](primitive-input-output) для деталей.
Проте поля повернених метаданих можуть відрізнятися.

- Клас [`qiskit.primitives.BackendEstimatorV2`](https://docs.quantum.ibm.com/api/qiskit/qiskit.primitives.BackendEstimatorV2) не пропонує жодної вбудованої реалізації пом'якшення помилок вимірювань або гейтів, оскільки
бекенд-примітиви розроблені для локального запуску на машині користувача.

- Клас [`qiskit.primitives.BackendSamplerV2`](https://docs.quantum.ibm.com/api/qiskit/qiskit.primitives.BackendSamplerV2) вимагає бекенду, що підтримує параметр `memory`.

- Інтерфейси бекенд-примітивів надають власні `Options` для [`SamplerV2`](https://docs.quantum.ibm.com/api/qiskit/qiskit.primitives.BackendSamplerV2) і [`EstimatorV2`](https://docs.quantum.ibm.com/api/qiskit/qiskit.primitives.BackendEstimatorV2), які відрізняються від реалізацій Runtime.

# Події відстеження активності для Qiskit Runtime

Сервіси IBM Cloud&reg;, зокрема Qiskit Runtime, генерують події відстеження активності.

Події відстеження активності фіксують дії, що змінюють стан сервісу в IBM Cloud. Ти можеш використовувати ці події для розслідування підозрілої активності, критичних дій та дотримання регуляторних вимог аудиту.

Ти можеш використовувати IBM Cloud Activity Tracker Event Routing — платформний сервіс — для маршрутизації подій аудиту у своєму акаунті до потрібних місць призначення, налаштовуючи цілі й маршрути, що визначають, куди надсилаються події відстеження активності. Докладніше читай у [Про маршрутизацію подій IBM Cloud Activity Tracker.](https://cloud.ibm.com/docs/atracker?topic=atracker-about)

Ти можеш використовувати IBM Cloud Logs для візуалізації та сповіщень про події, що генеруються у твоєму акаунті та маршрутизуються IBM Cloud Activity Tracker Event Routing до екземпляра IBM Cloud Logs.

## Місця генерації подій відстеження активності {#locations-where-activity-tracking-events-are-generated}

Qiskit Runtime надсилає події відстеження активності в таких регіонах:

- Вашингтон ("us-east")
- Франкфурт ("eu-de")

## Перегляд подій відстеження активності для Qiskit Runtime {#viewing-activity-tracking-events-for-qiskit-runtime}

Ти можеш використовувати IBM Cloud Logs для візуалізації та сповіщень про події, що генеруються у твоєму акаунті та маршрутизуються IBM Cloud Activity Tracker Event Routing до екземпляра IBM Cloud Logs.

### Запуск IBM Cloud Logs зі сторінки Observability {#launching-ibm-cloud-logs-from-the-observability-page}

Інформацію про запуск інтерфейсу IBM Cloud Logs дивись у розділі [Launching the UI](https://cloud.ibm.com/docs/cloud-logs?topic=cloud-logs-instance-launch#instance-launch-cloud-ui) документації IBM Cloud Logs.

# Налаштування IBM Quantum Platform для організації

Дотримуйся цих кроків, щоб налаштувати IBM Quantum Platform.

<span id="iam-org"></span>
## 1. Налаштування параметрів IAM {#configure-iam-settings}

Спочатку налаштуй наступні параметри в обліковому записі IAM адміністратора. Щоб переглянути та налаштувати ці параметри, перейди до [Manage → IAM → Settings](https://cloud.ibm.com/iam/settings).

* **User list visibility** визначає, чи можуть користувачі бачити один одного, незалежно від призначення проєкту. Параметр `enabled` обмежує видимість користувачів. Тобто користувачі в твоєму обліковому записі не можуть бачити один одного, навіть якщо вони мають доступ до однакових ресурсів. Вибери відповідне значення для свого середовища. Докладніше дивись у розділі [Controlling user visibility](https://cloud.ibm.com/docs/account?topic=account-iam-user-setting).
* **API key creation** контролює, чи можуть користувачі створювати API-ключі. У IBM Quantum Platform зазвичай використовуються API-ключі. Якщо API-ключі використовуються, вибери `disabled`. Або можна надати конкретні дозволи кожному користувачу окремо.

<span id="crt-rsc-grp-org"></span>
## 2. (Необов'язково) Створення груп ресурсів {#create-resource-groups}

Пропусти цей крок, якщо ти використовуєш групи доступу, які безпосередньо посилаються на екземпляри сервісу IBM Quantum.

Якщо ти вирішив(-ла) використовувати групи ресурсів, перейди до [Manage → Account → Resource groups (in Account resources)](https://cloud.ibm.com/account/resource-groups) і натисни **Create**.

<span id="create-instance-org"></span>
## 3. Створення екземплярів сервісу IBM Quantum {#create-ibm-quantum-service-instances}

Якщо ти вже створив(-ла) екземпляри сервісу IBM Quantum, пропусти цей крок.

* Якщо ти використовуєш групи ресурсів, переконайся, що екземпляри сервісу створені у відповідній групі ресурсів.
* Ім'я екземпляра сервісу, наприклад `QR-ml`, потрібне для посилань на групи доступу.

Дотримуйся кроків у розділі [Set up to use IBM Cloud](/guides/cloud-setup#cloud) для отримання інструкцій.

<span id="create-group-org"></span>
## 4. Створення груп доступу для проєктів {#create-access-groups-for-projects}

Спочатку [створи власну роль](/guides/custom-roles#custom), яка дозволяє користувачам виконувати дії для роботи з екземплярами сервісу IBM Quantum.
Далі [створи групу доступу](/guides/access-groups) для кожного проєкту й надай цій групі мінімальний набір дозволів, необхідних для роботи з ресурсами проєкту.

<span id="firewall"></span>
## 6. Налаштування брандмауера {#configure-your-firewall}

Щоб налаштувати брандмауер для забезпечення доступу до кінцевих точок IBM Quantum API, додай такі URL-адреси до свого списку дозволених:

- Глобальні сервіси IBM Cloud:
    ```text
    cloud.ibm.com
    iam.cloud.ibm.com
    api.global-search-tagging.cloud.ibm.com
    globalcatalog.cloud.ibm.com
    resource-controller.cloud.ibm.com
    ```
- Сервіси IBM Quantum Qiskit Runtime — регіон us-east:
    ```text
    quantum.cloud.ibm.com
    s3.us-east.cloud-object-storage.appdomain.cloud
    ```
- Сервіси IBM Quantum Qiskit Runtime — регіон eu-de:
    ```text
    eu-de.quantum.cloud.ibm.com
    s3.direct.eu-de.cloud-object-storage.appdomain.cloud
    ```
- Сервіс попереднього перегляду IBM Quantum Qiskit Functions:
    ```text
    qiskit-serverless.quantum.ibm.com
    ```

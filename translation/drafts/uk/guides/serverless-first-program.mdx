## Приклад: віддалена транспіляція з Qiskit Serverless {#example-remote-transpilation-with-qiskit-serverless}

Почнемо з такого прикладу, що транспілює `circuit` для заданого `backend` і цільового `optimization_level`, поступово додаючи елементи для розгортання робочого навантаження у Qiskit Serverless.

Помісти наведений нижче блок коду у файл `./source_files/transpile_remote.py`. Це і є програма, яку потрібно завантажити до Qiskit Serverless.

```python
%%writefile ./source_files/transpile_remote.py

from qiskit.transpiler import generate_preset_pass_manager

def transpile_remote(circuit, optimization_level, backend):
    """Transpiles an abstract circuit into an ISA circuit for a given backend."""
    pass_manager = generate_preset_pass_manager(
        optimization_level=optimization_level,
		backend=backend
    )
    isa_circuit = pass_manager.run(circuit)
    return isa_circuit
```

## Налаштуй файли {#set-up-your-files}

Qiskit Serverless вимагає організувати `.py`-файли робочого навантаження в окремий каталог. Наступна структура є прикладом хорошої практики:

```text
serverless_program
├── program_uploader.ipynb
└── source_files
    ├── transpile_remote.py
    └── *.py
```

Serverless завантажує вміст `source_files` для віддаленого запуску. Після налаштування можна скоригувати `transpile_remote.py` для отримання вхідних даних і повернення результатів.

### Отримай аргументи програми {#get-program-arguments}

Початковий `transpile_remote.py` має три вхідні параметри: `circuits`, `backend_name` і `optimization_level`. Serverless наразі підтримує лише серіалізовані вхідні й вихідні дані. З цієї причини не можна передати `backend` безпосередньо — замість цього використовуй `backend_name` як рядок.

```python
%%writefile --append ./source_files/transpile_remote.py

from qiskit_serverless import get_arguments, save_result, distribute_task, get

# Get program arguments
arguments = get_arguments()
circuits = arguments.get("circuits")
backend_name = arguments.get("backend_name")
optimization_level = arguments.get("optimization_level")
```

```text
Appending to ./source_files/transpile_remote.py
```

На цьому етапі можна отримати бекенд через `QiskitRuntimeService` і додати існуючу програму з таким кодом. Наведений нижче код передбачає, що ти вже [зберіг свої облікові дані](/guides/cloud-setup).

```python
%%writefile --append ./source_files/transpile_remote.py

from qiskit_ibm_runtime import QiskitRuntimeService

service = QiskitRuntimeService()
backend = service.backend(backend_name)
```

```text
Appending to ./source_files/transpile_remote.py
```

Нарешті, можна запустити `transpile_remote()` для всіх переданих `circuits` і повернути `transpiled_circuits` як результат:

```python
%%writefile --append ./source_files/transpile_remote.py

results = [
    transpile_remote(circuit, 1, backend)
    for circuit in circuits
]

save_result({
    "transpiled_circuits": results
})
```


## Розгортання на IBM Quantum Platform {#deploy-to-ibm-quantum-platform}

У попередньому розділі була створена програма для віддаленого запуску. Блоки коду в цьому розділі завантажують цю програму до Qiskit Serverless.

Використовуй `qiskit-ibm-catalog` для автентифікації в `QiskitServerless` за допомогою свого API-ключа, який можна знайти на [дашборді IBM Quantum](https://quantum.cloud.ibm.com), і завантаж програму.

Можна використати `save_account()` для збереження облікових даних (див. розділ [Налаштування IBM Cloud](/guides/cloud-setup#cloud-save)). Зверни увагу, що це записує облікові дані в той самий файл, що й [`QiskitRuntimeService.save_account()`](https://docs.quantum.ibm.com/api/qiskit-ibm-runtime/qiskit-runtime-service#save_account).

```python
from qiskit_ibm_catalog import QiskitServerless, QiskitFunction

# Authenticate to the remote cluster and submit the pattern for remote execution
serverless = QiskitServerless()
```

Qiskit Serverless стискає вміст `working_dir` (у цьому випадку `source_files`) у `tar`-архів, який завантажується, а після цього очищається. `entrypoint` визначає головний виконуваний файл програми для Qiskit Serverless. Крім того, якщо програма має власні залежності `pip`, їх можна додати до масиву `dependencies`:

```python
transpile_remote_demo = QiskitFunction(
    title="transpile_remote_serverless",
    entrypoint="transpile_remote.py",
    working_dir="./source_files/",
)
```

```python
serverless.upload(transpile_remote_demo)
```

```text
QiskitFunction(transpile_remote_serverless)
```

Щоб перевірити успішність завантаження, використовуй `serverless.list()`:

```python
# Get program from serverless.list() that matches the title of the one we uploaded
```

```python
# This cell is hidden from users, it checks the program executes correctly
from time import sleep
from qiskit import QuantumCircuit

qc = QuantumCircuit(2)
transpile_remote_serverless = serverless.load("transpile_remote_serverless")
job = transpile_remote_serverless.run(
    circuits=[qc],
    backend="ibm_sherbrooke",
    optimization_level=1,
)
while True:
    sleep(5)
    status = job.status()
    if status not in ["QUEUED", "INITIALIZING", "RUNNING", "DONE"]:
        raise Exception(
            f"Unexpected job status: '{status}'\n"
            + "Here are the logs:\n"
            + job.logs()
        )
    if status == "DONE":
        break
```

# Встановлення Qiskit SDK і Qiskit Runtime з вихідного коду

Встановлення Qiskit SDK з вихідного коду дає доступ до поточної версії для розробників, замість використання версії з репозиторію Python Package Index (PyPI). Це дозволяє ефективніше переглядати та розширювати останню версію коду Qiskit.

1. Перейди до директорії свого проєкту та створи мінімальне середовище лише з Python.

## Встановлення компілятора Rust {#install-the-rust-compiler}

На твоїй системі має бути встановлений компілятор Rust для компіляції Qiskit. Щоб встановити компілятор Rust, скористайся кросплатформним інсталятором [`rustup`](https://rustup.rs/) або [іншим методом встановлення.](https://forge.rust-lang.org/infra/other-installation-methods.html)

## Встановлення Qiskit {#install-qiskit}

Виконай такі кроки для встановлення Qiskit:

1. Клонуй репозиторій Qiskit.

```bash
git clone https://github.com/Qiskit/qiskit.git
```

2. Перейди до директорії `qiskit`.

```bash
cd qiskit
```

3. (Необов'язково) Якщо хочеш запускати тести або перевірки стилю коду, встанови залежності для розробників.

```bash
pip install -r requirements-dev.txt
```

4. Встанови `qiskit`.

  * **Стандартне встановлення**:

    ```bash
    pip install .
    ```

  * **Режим редагування**: У цьому режимі не потрібно перевстановлювати Qiskit після змін у коді проєкту.

    ```bash
    pip install -e .
    ```

    У режимі редагування скомпільовані розширення збираються в _режимі налагодження_ без оптимізацій. Це впливає на швидкодію скомпільованого коду під час виконання. Щоб зібрати скомпільовані розширення з увімкненими оптимізаціями, виконай таку команду для перезбірки бінарного файлу в _режимі випуску_:

    ```bash
    python setup.py build_rust --release --inplace
    ```

    <Admonition type="note">
    Якщо ти працюєш над кодом Rust у Qiskit, потрібно перезбирати код розширення щоразу після локальних змін. У режимі редагування розширення Rust збирається лише під час виконання команди встановлення, тому локальні зміни в коді Rust не відображаються у встановленому пакеті, якщо не перезібрати розширення, повторно запустивши `build_rust` (з `--release` або без, залежно від того, чи потрібен режим випуску або налагодження).
    </Admonition>

## Встановлення Qiskit Runtime {#install-qiskit-runtime}

Виконай ці кроки, якщо хочеш встановити Qiskit Runtime:

1. Клонуй репозиторій Qiskit Runtime.

```bash
git clone https://github.com/Qiskit/qiskit-ibm-runtime.git
```

2. Перейди до директорії `qiskit-ibm-runtime`.

```bash
cd qiskit-ibm-runtime
```

3. Встанови `qiskit-runtime`. Рекомендуємо використовувати [віртуальне середовище](https://docs.python.org/3/library/venv.html), щоб не забруднювати глобальне встановлення Python.

  * **Стандартне встановлення**:

    ```bash
    pip install .
    ```

  * **Режим редагування**: У цьому режимі не потрібно перевстановлювати Qiskit після змін у коді проєкту.

    ```bash
    pip install -e .
    ```

    У режимі редагування скомпільовані розширення збираються в _режимі налагодження_ без оптимізацій.

4. (Необов'язково) Якщо хочеш запускати тести або перевірки стилю коду, встанови залежності для розробників. Рекомендуємо використовувати [віртуальне середовище](https://docs.python.org/3/library/venv.html), щоб не забруднювати глобальне встановлення Python.

```bash
pip install -e ".[dev]"
```

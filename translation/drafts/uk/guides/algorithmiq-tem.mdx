## Огляд {#overview}

Метод пом'якшення помилок за допомогою тензорних мереж (TEM) від Algorithmiq — це гібридний квантово-класичний алгоритм, розроблений для усунення шумових похибок повністю на класичному етапі постобробки. За допомогою TEM можна обчислювати математичні сподівання спостережуваних величин, нівелюючи неминучі спричинені шумом помилки квантового заліза з підвищеною точністю та економічністю — що робить його надзвичайно привабливим варіантом як для дослідників у галузі квантових обчислень, так і для фахівців-практиків.

Метод полягає у побудові тензорної мережі, що представляє обернений глобальний канал шуму, який впливає на стан квантового процесора, а потім у застосуванні цього відображення до інформаційно-повних результатів вимірювань, отриманих зі зашумленого стану, щоб отримати незміщені оцінювачі для спостережуваних величин.

Перевага TEM полягає в тому, що він використовує інформаційно-повні вимірювання, надаючи доступ до широкого набору пом'якшених математичних сподівань спостережуваних і маючи оптимальні витрати вибірки на квантовому залізі, як описано у Filippov et al. (2023), [arXiv:2307.11740](https://arxiv.org/abs/2307.11740), та Filippov et al. (2024), [arXiv:2403.13542](https://arxiv.org/abs/2403.13542). Витрати вимірювання — це кількість додаткових вимірювань, необхідних для ефективного пом'якшення помилок, що є критичним фактором у здійсненності квантових обчислень. Таким чином, TEM здатен забезпечити квантову перевагу в складних сценаріях: зокрема у квантовому хаосі, фізиці багатьох тіл, динаміці моделі Хаббарда та симуляціях хімії малих молекул.

Основні можливості та переваги TEM можна підсумувати так:

1.  **Оптимальні витрати вимірювання**: TEM є оптимальним відносно [теоретичних меж](https://journals.aps.org/prl/abstract/10.1103/PhysRevLett.131.210601), тобто жоден інший метод не може досягти менших витрат вимірювання. Інакше кажучи, TEM потребує мінімальної кількості додаткових вимірювань для пом'якшення помилок. Це своєю чергою означає, що TEM використовує мінімальний квантовий час виконання.
2.  **Економічність**: Оскільки TEM виконує пом'якшення помилок повністю на етапі постобробки, немає потреби додавати додаткові схеми до квантового комп'ютера — це не лише здешевлює обчислення, а й знижує ризик внесення додаткових помилок через недосконалість квантових пристроїв.
3.  **Оцінка кількох спостережуваних**: Завдяки інформаційно-повним вимірюванням TEM ефективно оцінює кілька спостережуваних на основі тих самих даних вимірювань із квантового комп'ютера.
4.  **Пом'якшення помилок зчитування**: Функція TEM Qiskit також включає [запатентований метод пом'якшення помилок зчитування](https://journals.aps.org/prresearch/abstract/10.1103/PhysRevResearch.5.033154), здатний суттєво зменшити похибки зчитування після короткого калібрувального запуску.
5.  **Точність**: TEM значно підвищує точність і надійність цифрових квантових симуляцій, роблячи квантові алгоритми більш чіткими та передбачуваними.
## Опис {#description}

Функція TEM дає змогу отримувати пом'якшені за помилками математичні сподівання для кількох спостережуваних на квантовій схемі з мінімальними витратами вибірки. Схема вимірюється за допомогою інформаційно-повної позитивно-значущої операторно-значущої міри (IC-POVM), а зібрані результати вимірювань обробляються на класичному комп'ютері. Це вимірювання використовується для застосування методів тензорних мереж і побудови мапи інверсії шуму. Функція застосовує відображення, що повністю обертає всю зашумлену схему за допомогою тензорних мереж для представлення зашумлених шарів.

![Схема TEM](/docs/images/guides/algorithmiq-tem/tem_scheme.svg "Оцінка спостережуваної O зі зменшенням помилок шляхом постобробки результатів вимірювань зашумленого квантового процесора. U і N позначають ідеальну квантову операцію та відповідну карту шуму, яка може бути загально нелокальною (і розширеною до сірих блоків). D — це тензор операторів, двоїстих до ефектів у вимірюванні IC. Модуль пом'якшення шуму M є тензорною мережею, що ефективно стягується від центру назовні. Перша ітерація стягування представлена пунктирною фіолетовою лінією, друга — штриховою лінією, третя — суцільною лінією.")

Після надсилання схем до функції вони транспілюються та оптимізуються для мінімізації кількості шарів із двокубітними гейтами (найбільш зашумленими гейтами на квантових пристроях). Шум, що впливає на шари, вивчається через [Qiskit Runtime](https://docs.quantum.ibm.com/api/qiskit-ibm-runtime/noise-learner-noise-learner) з використанням розрідженої моделі шуму Паулі–Ліндблада, як описано у E. van den Berg, Z. Minev, A. Kandala, K. Temme, Nat. Phys. (2023). [arXiv:2201.09866](https://arxiv.org/abs/2201.09866).

Модель шуму є точним описом шуму на пристрої, здатним фіксувати тонкі особливості, зокрема перехресні завади між кубітами. Однак шум на пристроях може коливатися й дрейфувати, а вивчений шум може бути неточним у момент, коли виконується оцінка. Це може призводити до неточних результатів.

## Приклад {#example}

Наступний фрагмент демонструє приклад використання TEM для обчислення математичних сподівань спостережуваної для простої квантової схеми.

```python
from qiskit import QuantumCircuit
from qiskit.quantum_info import SparsePauliOp

# Create a quantum circuit
qc = QuantumCircuit(3)
qc.u(0.4, 0.9, -0.3, 0)
qc.u(-0.4, 0.2, 1.3, 1)
qc.u(-1.2, -1.2, 0.3, 2)
for _ in range(2):
    qc.barrier()
    qc.cx(0, 1)
    qc.cx(2, 1)
    qc.barrier()
    qc.u(0.4, 0.9, -0.3, 0)
    qc.u(-0.4, 0.2, 1.3, 1)
    qc.u(-1.2, -1.2, 0.3, 2)

# Define the observables
observable = SparsePauliOp("IYX", 1.0)

# Define the execution options
pub = (qc, [observable])
options = {"default_precision": 0.02}

# Define backend to use. TEM will choose the least-busy device reported by IBM if not specified
backend_name = "ibm_torino"

job = tem.run(pubs=[pub], backend_name=backend_name, options=options)
```

Для перевірки статусу навантаження функції Qiskit використовуй API Qiskit Serverless:

```python
print(job.status())
```

Отримати результати можна так:


**Параметри**

Назва | Тип | Опис | Обов'язковий | Типове значення | Приклад
-- | -- | -- | -- | -- | --
`pubs` | Iterable[EstimatorPubLike] | Ітерований набір PUB-подібних (примітивний уніфікований блок) об'єктів, наприклад кортежів `(circuit, observables)` або `(circuit, observables, parameters, precision)`. Докладніше у розділі [Огляд PUBs](/guides/primitive-input-output#overview-of-pubs). Якщо передано схему не у форматі ISA, її буде транспільовано з оптимальними налаштуваннями. Якщо передано ISA-схему, транспіляції не відбудеться; у цьому випадку спостережувана має бути визначена для всього QPU. | Так | N/A | (circuit, observables)
`backend_name` | str | Назва Backend, до якого надсилається запит. | Ні | Якщо не вказано, буде використано найменш завантажений Backend. | "ibm_fez"
`options` | dict | Вхідні параметри. Докладніше у розділі `Options`. | Ні | Докладніше у розділі `Options`. | \{"max_bond_dimension": 100\\}

<Admonition type="caution">
  На поточний момент TEM має такі обмеження:

  - Параметризовані схеми не підтримуються. Аргумент parameters слід встановити в `None`, якщо вказано precision. Це обмеження буде знято в майбутніх версіях.
  - Підтримуються лише схеми без циклів. Це обмеження буде знято в майбутніх версіях.
  - Не підтримуються негунітарні гейти, зокрема reset, measure і всі форми потоку управління. Підтримка reset буде додана в наступних релізах.
</Admonition>
### Параметри {#options}

Словник, що містить розширені параметри для TEM. Словник може містити ключі, наведені в таблиці нижче. Якщо будь-який з параметрів не вказано, використовуватиметься типове значення з таблиці. Типові значення підходять для більшості типових випадків використання TEM.

Назва | Значення | Опис | Типове значення
-- | -- | -- | --
`tem_max_bond_dimension` | int | Максимальний розмір зв'язку для тензорних мереж. | 500 |
`tem_compression_cutoff` | float | Порогове значення для тензорних мереж. | 1e-16
`compute_shadows_bias_from_observable` | bool | Булевий прапорець, що визначає, чи слід адаптувати зміщення протоколу класичних тіней до спостережуваної PUB. Якщо False, використовуватиметься протокол класичних тіней (рівна ймовірність вимірювання Z, X, Y). | False
`shadows_bias` | np.ndarray | Зміщення для рандомізованого протоколу класичних тіней — одновимірний або двовимірний масив розміром 3 або форми (num_qubits, 3) відповідно. Порядок: ZXY | np.array([1 / 3, 1 / 3, 1 / 3])
`max_execution_time` | int або `None` | Максимальний час виконання на QPU у секундах. Якщо час виконання перевищить це значення, задачу буде скасовано. Якщо `None`, застосовуватиметься типове обмеження Qiskit Runtime. | `None`
`num_randomizations` | int | Кількість рандомізацій для вивчення шуму та twirling гейтів. | 32
`max_layers_to_learn` | int | Максимальна кількість унікальних шарів для вивчення. | 4
`mitigate_readout_error` | bool | Булевий прапорець, що визначає, чи слід виконувати пом'якшення помилок зчитування. | True
`num_readout_calibration_shots` | int | Кількість вимірювань для пом'якшення помилок зчитування. | 10000
`default_precision` | float | Типова точність для PUBs, для яких precision не вказано. | 0.02
`seed` | int або `None` | Задає сід генератора випадкових чисел для відтворюваності. Якщо `None`, сід не встановлюється. | None
## Вихідні дані {#outputs}

Об'єкт Qiskit [PrimitiveResults](https://docs.quantum.ibm.com/api/qiskit/qiskit.primitives.PrimitiveResult), що містить пом'якшений TEM результат. Результат для кожного PUB повертається у вигляді [PubResult](https://docs.quantum.ibm.com/api/qiskit/qiskit.primitives.PubResult) із такими полями:

Назва | Тип | Опис
-- | -- | --
data | DataBin | Об'єкт Qiskit [DataBin](https://docs.quantum.ibm.com/api/qiskit/qiskit.primitives.DataBin), що містить пом'якшену TEM спостережувану та її стандартну похибку. DataBin має такі поля: <ul><li>`evs`: Значення спостережуваної, пом'якшене TEM.</li><li>`stds`: Стандартна похибка пом'якшеної TEM спостережуваної.</li></ul>
metadata | dict | Словник з додатковими результатами. Словник містить такі ключі: <ul><li>`"evs_non_mitigated"`: Значення спостережуваної без пом'якшення помилок.</li><li>`"stds_non_mitigated"`: Стандартна похибка результату без пом'якшення помилок.</li><li>`"evs_mitigated_no_readout_mitigation"`: Значення спостережуваної з пом'якшенням помилок, але без пом'якшення помилок зчитування.</li><li>`"stds_mitigated_no_readout_mitigation"`: Стандартна похибка результату з пом'якшенням помилок, але без пом'якшення помилок зчитування.</li><li>`"evs_non_mitigated_with_readout_mitigation"`: Значення спостережуваної без пом'якшення помилок, але з пом'якшенням помилок зчитування.</li><li>`"stds_non_mitigated_with_readout_mitigation"`: Стандартна похибка результату без пом'якшення помилок, але з пом'якшенням помилок зчитування.</li></ul>

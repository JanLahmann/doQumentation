# Розуміння своїх обов'язків при використанні Qiskit Runtime {#understanding-your-responsibilities-when-using-qiskit-runtime}

Ознайомся з наступними розділами щодо конкретних обов'язків тебе та IBM&reg; при використанні Qiskit Runtime. Щодо загальних умов використання, дивись [Умови та повідомлення](https://cloud.ibm.com/docs/overview/terms-of-use?topic=overview-terms) для IBM Cloud&reg;.

## Управління інцидентами та операціями {#incident-and-operations-management}

Управління інцидентами та операціями включає такі завдання, як моніторинг, управління подіями, висока доступність, визначення проблем, відновлення та повне резервне копіювання і відновлення.

| Завдання | Обов'язки IBM | Твої обов'язки |
| --- | --- | --- |
|Управління справністю інфраструктури|<ul><li>Безперервно надавати нові функції, оновлення та виправлення помилок за потреби прозорим чином.</li><li>Надавати сповіщення про заплановане технічне обслуговування, бюлетені безпеки або незаплановані збої.</li></ul> | <ul><li>[Налаштуй уподобання для отримання електронних листів](https://cloud.ibm.com/user/notifications) про сповіщення сервісу.</li><li>Моніторь сторінку [стану IBM Cloud](https://cloud.ibm.com/status) для отримання загальних оголошень.</li></ul> |
|Моніторинг і сповіщення про справність інфраструктури| <ul><li>Усувати всі проблеми середовища інфраструктури.</li><li>Повідомляти клієнтів про відповідні інциденти.</li></ul> | <ul><li>Оцінювати вплив кожного зареєстрованого інциденту.</li><li>Звертатися до [підтримки IBM](/guides/support) за необхідності.</li></ul>|
|Підтримка угоди про рівень обслуговування IBM Cloud з високою доступністю| Експлуатувати Qiskit Runtime відповідно до [Угод про рівень обслуговування IBM Cloud Public](https://cloud.ibm.com/docs/overview?topic=overview-slas) (SLA). | Дотримуватися найкращих практик високої доступності. |

## Управління змінами {#change-management}

Управління змінами включає такі завдання, як розгортання, конфігурація, оновлення, встановлення патчів, зміни конфігурації та видалення.

| Завдання | Обов'язки IBM | Твої обов'язки |
| --- | --- | --- |
| Оновлення основних версій | Забезпечити доступність та інструменти для оновлень основних версій. | Перейти на підтримувану версію до закінчення підтримки старої версії. |
| Зміни версій API | Версіонування REST API. | Інтегруватися з потрібною версією REST API. |
| Оновлення, виправлення та нові функції | <ul><li>Надавати регулярні оновлення, виправлення помилок та нові функції, прозоро дотримуючись моделі безперервного постачання.</li><li>Публікувати сповіщення про зміни, що стосуються клієнтів.</li></ul> | <ul><li>Налаштуй уподобання для [отримання сповіщень електронною поштою](https://cloud.ibm.com/user/notifications) та моніторь сторінку [стану IBM Cloud](https://cloud.ibm.com/status) для отримання загальних оголошень.</li><li>Читай системно-згенеровані сповіщення та [оголошення платформи](https://quantum.cloud.ibm.com/announcements), і вживай відповідних заходів.</li></ul> |

## Управління ідентифікацією та доступом {#identity-and-access-management}

Управління ідентифікацією та доступом включає такі завдання, як автентифікація, авторизація, управління політиками контролю доступу, а також схвалення, надання та відкликання доступу.

| Завдання | Обов'язки IBM | Твої обов'язки |
| --- | --- | --- |
| Ідентифікація та контроль доступу | Надати можливість управляти та обмежувати доступ до ресурсів через IBM Cloud. | Надавати, відкликати та управляти доступом до ресурсів через [Управління ідентифікацією та доступом Cloud](/guides/access-groups) (IAM). |
| Автентифікація сервісу | Гарантувати, що лише автентифіковані користувачі мають доступ до твоїх сервісів та ресурсів Qiskit Runtime. | <ul><li>[Захисти свій API-ключ](/guides/cloud-setup-untrusted#initialize-the-service-in-an-untrusted-environment) від несанкціонованого доступу.</li><li>Ротуй та оновлюй свої [API-ключі](https://cloud.ibm.com/iam/apikeys) відповідно до вимог своєї політики безпеки.</li></ul> |
| Спостережуваність | Дозволити інтеграцію IBM Cloud Activity Tracker Event Routing для аудиту дій користувачів. | Налаштуй [IBM Cloud Activity Tracker Event Routing](https://cloud.ibm.com/docs/vmware-service?topic=vmware-service-at_events) або інші функції для відстеження активності користувачів. |

## Безпека та відповідність нормативним вимогам {#security-and-regulation-compliance}

Безпека та відповідність нормативним вимогам включає такі завдання, як впровадження засобів контролю безпеки та сертифікація відповідності.

| Завдання | Обов'язки IBM | Твої обов'язки |
| --- | --- | --- |
| Загальне | <ul><li>Підтримувати засоби контролю, відповідні різним галузевим стандартам відповідності.</li><li>Моніторити, ізолювати та відновлювати функціональність сервісу.</li><li>Моніторити та звітувати про справність інтерфейсів сервісу.</li></ul> | Переконатися, що персональні та регульовані дані не надаються. |
| Вразливості | Моніторити для виявлення проблем з вразливостями та відповідністю безпеки. | <ul><li>Бути в курсі можливих вразливостей і проблем безпеки через бюлетені безпеки, що надають потенційні дії з усунення.</li><li>Перевіряй [розділ безпеки репозиторію Qiskit на GitHub.](https://github.com/Qiskit/qiskit/security) Усі вразливості безпеки публікуються там.</li><li>Перевіряй [вебсайт стану IBM Cloud](https://cloud.ibm.com/status) для отримання оголошень і бюлетенів безпеки про ключові події, що стосуються платформи IBM Cloud, інфраструктури та основних сервісів.</li></ul> |
| Шифрування | Шифрувати дані під час передачі у будь-яких публічних мережах та в мережі приватного центру обробки даних хмарного сервісу. Для захисту даних у русі використовуються методи шифрування, такі як HTTPS та TLS. | За необхідності переконуватися, що твоє з'єднання зашифроване від кінця до кінця. |
| Безпека | <ul><li>Гарантувати безпеку даних на диску та даних у русі в межах своєї інфраструктури.</li><li>Надавати безпечний сервіс, що відповідає ключовим стандартам.</li></ul> | Обмежувати [доступ користувачів](/guides/access-groups) до відповідних ресурсів та груп ресурсів. |

## Аварійне відновлення {#disaster-recovery}

| Завдання | Обов'язки IBM | Твої обов'язки |
| --- | --- | --- |
| Загальне | Відновлювати або перебудовувати середовища та ресурси постачання в постраждалих регіонах. | <ul><li>Моніторити стани екземплярів та ресурсів.</li><li>Постачати нові [екземпляри](/guides/instances) сервісу Qiskit Runtime в альтернативному регіоні, якщо доступно.</li></ul> |
| Відновлення сервісів | Автоматично відновлювати та повертати можливості сервісу Qiskit Runtime до нормальної роботи. | Повторно подавати будь-які постраждалі завдання після відновлення. |
| Резервне копіювання даних конфігурації користувача | IBM не несе відповідальності за резервне копіювання даних конфігурації користувача. | Резервне копіювання будь-яких даних конфігурації за необхідності. |
| Резервне копіювання та відновлення робочих навантажень | IBM не несе відповідальності за резервне копіювання або відновлення даних робочих навантажень користувача. | Резервне копіювання та відновлення своїх робочих навантажень за необхідності. |

# Встановлення лімітів розподілу для інстансу

Обліковий запис може мати виділений час за планом Premium або Flex. Оскільки кожен обліковий запис може мати багато інстансів у межах плану, встановлення лімітів розподілу допомагає контролювати, які інстанси мають право використовувати цей час. Зверни увагу: якщо розподіл не встановлено як ліміт, інстанс може використати більше часу, ніж йому виділено.

Час, виділений інстансам, використовується алгоритмом справедливого розподілу для визначення пріоритету в черзі — на основі використання всіх QPU за ковзне вікно 28 днів (для користувачів Premium Plan) або за весь період підписки (для користувачів Flex Plan).

Розподіл встановлюється під час створення інстансу. Однак переглянути й змінити виділений час можна на [сторінці інстансів.](https://quantum.cloud.ibm.com/instances)

## Що відбувається, коли ліміт вичерпано {#what-happens-when-the-limit-is-reached}

За бажанням можна встановити виділений час як ліміт для інстансу. Коли інстанс досягає свого ліміту, відбувається таке:

- Активні робочі навантаження продовжують виконуватися.
- Завдання, що очікують, залишаються в черзі.
- Жодне робоче навантаження не скасовується.
- Учасники можуть надсилати нові робочі навантаження — вони залишатимуться в черзі, доки не з'явиться більше доступного часу.
- Для плану Flex інстанс не може обробляти нові завдання, доки ліміт не буде збільшено або (для плану Premium) до початку наступного вікна використання.

## Встановлення різних значень для ліміту розподілу та ліміту використання {#set-different-values-for-the-allocation-limit-and-usage-limit}

Коли ти встановлюєш ліміт розподілу для інстансу в IBM Quantum Platform, те саме значення автоматично встановлюється як ліміт використання інстансу. Якщо ти хочеш дозволити користувачам продовжувати надсилати й виконувати завдання після досягнення ліміту розподілу, можна встановити вище значення для ліміту використання за допомогою параметра [usage_allocation_seconds](/guides/access-instances-platform-apis#parameters) в API [Update a resource instance](https://cloud.ibm.com/apidocs/resource-controller/resource-controller#update-resource-instance).

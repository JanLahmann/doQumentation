# Використання Qiskit Code Assistant у локальному режимі

Дізнайся, як встановити, налаштувати та використовувати будь-яку з моделей Qiskit Code Assistant на своїй локальній машині.

<Admonition type="note" title="Примітки">
    - Qiskit Code Assistant перебуває у статусі попереднього випуску і може змінюватися.
    - Якщо ти маєш відгуки або хочеш зв'язатися з командою розробників, скористайся [каналом Qiskit Slack Workspace](https://qiskit.enterprise.slack.com/archives/C07LYA6PL83) або відповідними публічними репозиторіями GitHub.
</Admonition>

## Швидкий старт (рекомендовано) {#quick-start-recommended}

Найпростіший спосіб розпочати роботу з Qiskit Code Assistant у локальному режимі — скористатися скриптами автоматичного налаштування для розширення VS Code або JupyterLab. Ці скрипти автоматично встановлять [Ollama](https://ollama.com/) для запуску LLM, завантажать рекомендовану модель і налаштують розширення за тебе.

### Налаштування розширення VS Code {#vs-code-extension-setup}

Виконай таку команду в терміналі:

```bash
bash <(curl -fsSL https://raw.githubusercontent.com/Qiskit/qiskit-code-assistant-vscode/main/setup_local.sh)
```

Цей скрипт виконує такі кроки:
- Встановлює Ollama (якщо ще не встановлено)
- Завантажує та налаштовує рекомендовану модель Qiskit Code Assistant
- Налаштовує розширення VS Code для роботи з локальним розгортанням

### Налаштування розширення JupyterLab {#jupyterlab-extension-setup}

Виконай таку команду в терміналі:

```bash
bash <(curl -fsSL https://raw.githubusercontent.com/Qiskit/qiskit-code-assistant-jupyterlab/main/setup_local.sh)
```

Цей скрипт виконає таке:
- Встановить Ollama (якщо ще не встановлено)
- Завантажить та налаштує рекомендовану модель Qiskit Code Assistant
- Налаштує розширення JupyterLab для роботи з локальним розгортанням

<summary>Ручне розгортання моделей Qiskit Code Assistant локально через Ollama</summary>

Існує кілька способів розгорнути завантажену модель Qiskit Code Assistant і взаємодіяти з нею. У цьому посібнику демонструється використання [Ollama](https://ollama.com) наступним чином: або через [застосунок Ollama](#using-the-ollama-application) з використанням інтеграції Hugging Face Hub чи локальної моделі, або через [пакет `llama-cpp-python`](#use-llama-cpp-python).

### Використання застосунку Ollama {#using-the-ollama-application}

Застосунок Ollama надає просте рішення для локального запуску LLM. Він простий у використанні, з CLI, що робить весь процес налаштування, управління моделями та взаємодію досить зрозумілими. Ідеально підходить для швидкого експериментування та для користувачів, які хочуть мінімуму технічних деталей.

#### Встановлення Ollama {#install-ollama}

1. Завантаж [застосунок Ollama](https://ollama.com/download)
1. Встанови завантажений файл
1. Запусти встановлений застосунок Ollama

    <Admonition type="info">Застосунок працює успішно, коли значок Ollama з'являється в рядку меню робочого стола. Ти також можеш перевірити, чи запущено сервіс, перейшовши за адресою `http://localhost:11434/`.</Admonition>

1. Спробуй Ollama у терміналі та запусти моделі. Наприклад:

    ```
    ollama run hf.co/Qiskit/Qwen2.5-Coder-14B-Qiskit
    ```

#### Налаштування Ollama через інтеграцію Hugging Face Hub {#set-up-ollama-using-the-hugging-face-hub-integration}

[Інтеграція Ollama/Hugging Face Hub](https://huggingface.co/docs/hub/ollama) надає спосіб взаємодії з моделями, розміщеними на Hugging Face Hub, без необхідності створювати новий modelfile або вручну завантажувати файли GGUF чи safetensors. Стандартні файли `template` і `params` вже включені для моделі на Hugging Face Hub.

1. Переконайся, що застосунок Ollama запущено.

1. Перейди на сторінку потрібної моделі та скопіюй URL. Наприклад, https://huggingface.co/Qiskit/Qwen2.5-Coder-14B-Qiskit-GGUF.

1. У терміналі виконай команду:

    ```
    ollama run hf.co/Qiskit/Qwen2.5-Coder-14B-Qiskit
    ```

Ти можеш використовувати модель `hf.co/Qiskit/Qwen2.5-Coder-14B-Qiskit` або будь-яку з інших поточних рекомендованих офіційних моделей GGUF: `hf.co/Qiskit/mistral-small-3.2-24b-qiskit-GGUF` або `hf.co/Qiskit/granite-3.3-8b-qiskit-GGUF`.

#### Налаштування Ollama з вручну завантаженою GGUF-моделлю Qiskit Code Assistant {#set-up-ollama-with-a-manually-downloaded-qiskit-code-assistant-gguf-model}

Якщо ти вручну завантажив(-ла) GGUF-модель, наприклад https://huggingface.co/Qiskit/Qwen2.5-Coder-14B-Qiskit-GGUF, і хочеш експериментувати з різними шаблонами та параметрами, дотримуйся цих кроків, щоб завантажити її у свій локальний застосунок Ollama.

1. Створи `Modelfile`, ввівши такий вміст, і обов'язково заміни `<PATH-TO-GGUF-FILE>` на фактичний шлях до завантаженої моделі.

    ```
    FROM <PATH-TO-GGUF-FILE>
    TEMPLATE """{{ if .System }}
    System:
    {{ .System }}

    {{ end }}{{ if .Prompt }}Question:
    {{ .Prompt }}

    {{ end }}Answer:
    ```python{{ .Response }}
    """

    PARAMETER stop "Question:"
    PARAMETER stop "Answer:"
    PARAMETER stop "System:"
    PARAMETER stop "```"

    PARAMETER temperature 0
    PARAMETER top_k 1
    ```

1. Виконай таку команду, щоб створити власний екземпляр моделі на основі `Modelfile`.

    ```
    ollama create Qwen2.5-Coder-14B-Qiskit -f ./path-to-model-file
    ```

    <Admonition type="note">Цей процес може тривати деякий час, поки Ollama зчитує файл моделі, ініціалізує екземпляр моделі та налаштовує його відповідно до наданих специфікацій.</Admonition>

#### Запуск вручну завантаженої моделі Qiskit Code Assistant в Ollama {#run-the-qiskit-code-assistant-model-manually-downloaded-in-ollama}

Після налаштування моделі `Qwen2.5-Coder-14B-Qiskit` в Ollama виконай таку команду, щоб запустити модель і взаємодіяти з нею в терміналі (у режимі чату).

```
ollama run Qwen2.5-Coder-14B-Qiskit
```

Корисні команди:

- `ollama list` — список моделей на твоєму комп'ютері
- `ollama rm Qwen2.5-Coder-14B-Qiskit` — видалення моделі
- `ollama show Qwen2.5-Coder-14B-Qiskit` — відображення інформації про модель
- `ollama stop Qwen2.5-Coder-14B-Qiskit` — зупинка запущеної моделі
- `ollama ps` — список завантажених на даний момент моделей

<summary>Ручне розгортання моделей Qiskit Code Assistant локально через пакет `llama-cpp-python`</summary>

<span id="use-llama-cpp-python"></span>
Альтернативою застосунку Ollama є пакет `llama-cpp-python` — це прив'язка Python для `llama.cpp`. Він надає більше контролю та гнучкості для локального запуску GGUF-моделі та ідеально підходить для користувачів, які хочуть інтегрувати локальну модель у свої робочі процеси та Python-застосунки.

1. Встанови [`llama-cpp-python`](https://pypi.org/project/llama-cpp-python/)
2. Взаємодій із моделлю зі свого застосунку за допомогою `llama_cpp`. Наприклад:

```python
from llama_cpp import Llama

model_path = <PATH-TO-GGUF-FILE>

model = Llama(
        model_path,
        seed=17,
        n_ctx=10000,
        n_gpu_layers=37, # to offload in gpu, but put 0 if all in cpu
    )

input = 'Generate a quantum circuit with 2 qubits'
raw_pred = model(input)["choices"][0]["text"]
```

Ти також можеш додати до моделі параметри генерації тексту для налаштування виводу:

```python
generation_kwargs = {
        "max_tokens": 512,
        "echo": False, # Echo the prompt in the output
        "top_k": 1
    }

raw_pred = model(input, **generation_kwargs)["choices"][0]["text"]
```

### Використання бібліотеки `llama.cpp` {#use-the-llamacpp-library}

Ще одна альтернатива — використання `llama.cpp`, бібліотеки з відкритим кодом для виконання інференсу LLM на CPU з мінімальним налаштуванням.
Вона надає низькорівневий контроль над виконанням моделі і зазвичай запускається з командного рядка, вказуючи на локальний файл моделі GGUF.

Є кілька способів встановити `llama.cpp` на своїй машині:

- Встановити llama.cpp за допомогою [brew, nix або winget](https://github.com/ggml-org/llama.cpp/blob/master/docs/install.md)
- Запустити через Docker: ознайомся з [документацією Docker від команди `llama.cpp`](https://github.com/ggml-org/llama.cpp/blob/master/docs/docker.md)
- Завантажити готові бінарні файли зі [сторінки релізів](https://github.com/ggml-org/llama.cpp/releases)
- Зібрати з вихідного коду, [клонувавши репозиторій](https://github.com/ggml-org/llama.cpp/blob/master/docs/build.md)

Після встановлення ти можеш використовувати `llama.cpp` для взаємодії з GGUF-моделями в режимі розмови:
```bash
# Use a local model file
llama-cli -m my_model.gguf -cnv

# Or download and run a model directly from Hugging Face
llama-cli -hf Qiskit/Qwen2.5-Coder-14B-Qiskit-GGUF -cnv
```

Ти також можеш запустити сервер API, сумісний з OpenAI, для моделі таким чином:
```bash
llama-server -hf Qiskit/Qwen2.5-Coder-14B-Qiskit-GGUF
```

#### Розширені параметри {#advanced-parameters}
За допомогою програми `llama-cli` ти можеш керувати генерацією моделі за допомогою параметрів командного рядка. Наприклад, ти можеш задати початковий "системний" промпт за допомогою прапорця `-p/--prompt`. У режимі розмови (`-cnv`) цей початковий промпт виступає системним повідомленням. В іншому випадку ти можеш просто додати будь-яку потрібну інструкцію на початок свого тексту промпту. Ти також можеш налаштовувати параметри семплінгу — наприклад: температуру (`--temp`), top-k (`--top-k`), top-p (`--top-p`), штраф за повторення (`--repeat-penalty`) і зерно (`--seed`). Нижче наведено приклад виклику з використанням цих опцій:

```bash
llama-cli -hf Qiskit/Qwen2.5-Coder-14B-Qiskit-GGUF \
  -p "You are a friendly assistant." -cnv \
  --temp 0.7 \
  --top-k 50 \
  --top-p 0.95 \
  --repeat-penalty 1.1 \
  --seed 42
```

Щоб забезпечити правильну роботу наших моделей Qiskit, рекомендуємо використовувати системний промпт, наданий у наших репозиторіях HF GGUF: системний промпт для [mistral-small-3.2-24b-qiskit-GGUF](https://huggingface.co/Qiskit/mistral-small-3.2-24b-qiskit-GGUF/blob/main/system), [Qwen2.5-Coder-14B-Qiskit-GGUF](https://huggingface.co/Qiskit/Qwen2.5-Coder-14B-Qiskit-GGUF/blob/main/system), [granite-3.3-8b-qiskit-GGUF](https://huggingface.co/Qiskit/granite-3.3-8b-qiskit-GGUF/blob/main/system) і [granite-3.2-8b-qiskit-GGUF.](https://huggingface.co/Qiskit/granite-3.2-8b-qiskit-GGUF/blob/main/system)

### Підключення через розширення Qiskit Code Assistant для VS Code {#connect-with-the-qiskit-code-assistant-vs-code-extension}

За допомогою розширення Qiskit Code Assistant для VS Code ти можеш взаємодіяти з моделлю та виконувати доповнення коду під час написання. Це добре підходить для користувачів, яким потрібна допомога у написанні коду Qiskit для Python-застосунків.

1. Встанови [розширення Qiskit Code Assistant для VS Code](/guides/qiskit-code-assistant-vscode).
1. У VS Code перейди до **User Settings** і встав URL свого локального розгортання Ollama у поле **Qiskit Code Assistant: Url** (наприклад, `http://localhost:11434`).
1. Перезавантаж VS Code, перейшовши до **View > Command Palette...** і вибравши **Developer: Reload Window**.

Модель Qiskit Code Assistant, налаштована в Ollama, має з'явитися в рядку стану та буде готова до використання.

### Підключення через розширення Qiskit Code Assistant для JupyterLab {#connect-with-the-qiskit-code-assistant-jupyterlab-extension}

За допомогою розширення Qiskit Code Assistant для JupyterLab ти можеш взаємодіяти з моделлю та виконувати доповнення коду безпосередньо в Jupyter Notebook. Користувачі, які переважно працюють з Jupyter Notebooks, можуть скористатися цим розширенням для покращення свого досвіду написання коду Qiskit.

1. Встанови [розширення Qiskit Code Assistant для JupyterLab](/guides/qiskit-code-assistant-jupyterlab).
1. У JupyterLab перейди до **Settings Editor** і встав URL свого локального розгортання Ollama у поле **Qiskit Code Assistant Service API** (наприклад, `http://localhost:11434`).

Модель Qiskit Code Assistant, налаштована в Ollama, має з'явитися в рядку стану та буде готова до використання.

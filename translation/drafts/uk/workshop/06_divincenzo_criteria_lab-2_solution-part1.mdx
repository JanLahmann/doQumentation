## Вступ

Фізик Девід ДіВінченцо сформулював п'ять ключових вимог до будь-якої фізичної реалізації квантового комп'ютера, а також два додаткових критерії для квантового зв'язку. У цьому ноутбуці ми **дослідимо кожен критерій ДіВінченцо через практичні демонстрації в Qiskit**. Замість глибокого занурення в теорію кожен розділ коротко пояснює один критерій і надає вправи з кодом у Qiskit 2. Ти запустиш Circuit на симуляторах і реальних пристроях IBM Quantum, щоб **дослідити кожен принцип на практиці**.

**П'ять критеріїв ДіВінченцо для квантових обчислень**:

1. **Масштабована фізична система з добре охарактеризованими Qubit'ами.**
2. **Можливість ініціалізувати Qubit'и** до простого фідуціального стану (наприклад, |00…0〉).
3. **Тривалий час декогеренції** (час когерентності Qubit'а значно більший за час виконання Gate'у).
4. **Універсальний набір квантових Gate'ів** (здатність виконувати довільні унітарні операції).
5. **Можливість вимірювання кожного Qubit'а окремо** (зчитування стану кожного Qubit'а).

*(ДіВінченцо також описав два критерії для квантового зв'язку: здатність взаємно перетворювати нерухомі та «летючі» Qubit'и, а також достовірно передавати летючі Qubit'и між місцями розташування. Ми включаємо їх у рекомендоване завдання наприкінці цього ноутбука.)*

Кожен із наступних розділів відповідає одному критерію. Ми використаємо Qiskit, щоб проілюструвати концепцію за допомогою коду та **інтерактивних експериментів**, які ти можеш спробувати. Наприклад, ми побачимо, як збільшення кількості Qubit'ів і глибини Circuit'у впливає на результати (Критерій 1), як скидати та готувати стани Qubit'ів (Критерій 2), як вимірювати Qubit'и на симуляторах та реальних пристроях (Критерій 4), як Qiskit складає універсальні Gate'и (Критерій 3) і як скінченна когерентність (T₁, T₂) впливає на обчислення (Критерій 5). Наприкінці ти матимеш глибшу інтуїцію щодо того, що означає кожен критерій ДіВінченцо на практиці і як Qiskit дозволяє з ними експериментувати.

## 1. Критерій 1 – Масштабовані, добре охарактеризовані Qubit'и

**Критерій 1:** *«Масштабована фізична система з добре охарактеризованими Qubit'ами.»* Це означає, що нам потрібна квантова апаратна платформа, де ми можемо **збільшувати кількість Qubit'ів** і при цьому надійно ними керувати. Властивості кожного Qubit'а (рівні енергії, рівні похибок, з'єднання тощо) мають бути добре зрозумілі. По суті, ми хочемо будувати більші Circuit'и без збоїв системи. На практиці, зі збільшенням кількості Qubit'ів або глибини Circuit'у накопичуються похибки та декогеренція, тому демонстрація *масштабованості* також означає розуміння того, як збільшення розміру впливає на продуктивність.

**Мета демонстрації:** Використати Qiskit, щоб показати вплив масштабування Circuit'у (за кількістю Qubit'ів або глибиною Gate'ів) на точність результату. Ми симулюємо ідеальний та шумний сценарії, щоб побачити, як більша система або глибший Circuit'у піддаються декогеренції та похибкам.

Спочатку побудуємо малий заплутаний стан (стан GHZ) на 3 Qubit'ах, а потім більший — на 5 Qubit'ах, як простий тест масштабування. Стан GHZ з *n* Qubit'ів — це $\frac{1}{\sqrt{2}}(|0...0\rangle + |1...1\rangle)$. В ідеальній симуляції вимірювання n-Qubit GHZ дає лише два результати (всі 0 або всі 1) з рівною ймовірністю. Ми порівняємо **ідеальний результат** із **шумним результатом** зі збільшенням n або глибини Circuit'у.

```python
from qiskit import QuantumCircuit
from qiskit_aer import AerSimulator
from qiskit.visualization import plot_histogram
from qiskit.transpiler.preset_passmanagers import generate_preset_pass_manager
from qiskit_ibm_runtime import SamplerV2 as Sampler

# 3-qubit GHZ circuit
qc3 = QuantumCircuit(3, 3)
qc3.h(0)
qc3.cx(0, 1)
qc3.cx(1, 2)
qc3.measure([0, 1, 2], [0, 1, 2])

# 5-qubit GHZ circuit (scaling up the number of qubits)
qc5 = QuantumCircuit(5, 5)
qc5.h(0)
qc5.cx(0, range(1, 5))    # entangle qubit 0 with all others
qc5.measure(range(5), range(5))

# Transpile for a simulator backend
sim_backend = AerSimulator()
pm = generate_preset_pass_manager(backend=sim_backend, optimization_level=1)
isa_qc3 = pm.run(qc3)
isa_qc5 = pm.run(qc5)

# Run ideal simulations (no noise)
sampler = Sampler(mode=sim_backend)

job3 = sampler.run([isa_qc3], shots=1024)
result3 = job3.result()
counts3 = result3[0].data.c.get_counts()

job5 = sampler.run([isa_qc5], shots=1024)
result5 = job5.result()
counts5 = result5[0].data.c.get_counts()

print("3-qubit GHZ counts (ideal):", counts3)
plot_histogram(counts3, legend=['3-qubit ideal'], figsize=(6,4))
```

```python
print("5-qubit GHZ counts (ideal):", counts5)
plot_histogram(counts5, legend=['5-qubit ideal'], figsize=(6,4))
```

**Очікуваний результат (ідеальний випадок):** Ідеальний 3-Qubit GHZ дає приблизно 50% `000` і 50% `111` у результатах. 5-Qubit GHZ дає ~50% `00000` і 50% `11111`. Інші бітові рядки не з'являються, оскільки стан ідеально когерентний і заплутаний. На гістограмі для кожного Circuit'у ти маєш побачити два високих стовпчики, що відповідають результатам «всі нулі» та «всі одиниці».

Тепер подивимося, що відбувається в **шумному середовищі**. Ми використаємо можливості моделювання шуму Qiskit Aer, щоб імітувати похибки реального пристрою. Наприклад, можна взяти властивості IBM Backend'у для створення моделі шуму, яка включає похибки Gate'ів, скінченний час Gate'ів, релаксацію Qubit'а (T₁), дефазування (T₂) і похибки зчитування. Тут ми використаємо **фейковий Backend**, що представляє пристрій IBM Quantum Brisbane, для генерації моделі шуму та повторного запуску GHZ Circuit'ів через неї.

Решта критеріїв стосуватиметься того, як підтримувати Qubit'и в хорошому стані (мало похибок, придатні для ініціалізації тощо) при масштабуванні.
## 2. Критерій 2 – Ініціалізація Qubit'ів

**Критерій 2:** *«Можливість ініціалізувати стан Qubit'ів до простого фідуціального стану, наприклад |000…〉.»* Усі Qubit'и мають надійно починати роботу в відомому еталонному стані (зазвичай основному стані |0〉 для кожного Qubit'а). Ініціалізація є необхідною умовою для того, щоб алгоритми починалися з чистого аркуша. На практиці, на пристроях IBM Quantum кожен Qubit автоматично скидається до |0〉 на початку кожного виконання Circuit'у. Qiskit також надає інструкції для скидання Qubit'ів або підготовки власних станів під час обчислення.

**Мета демонстрації:** Показати, як ініціалізувати Qubit'и в Qiskit — як на початку, так і посередині Circuit'у. Ми продемонструємо використання інструкції `reset` і методів підготовки стану.
### Вправа 2: Підготовка конкретного стану
У блоці коду нижче доповни `QuantumCircuit`, щоб підготувати стан $|10\rangle$. Це означає, що Qubit 0 має бути в стані $|0\rangle$, а Qubit 1 — в стані $|1\rangle$. Використай відповідний Gate і інструкцію для досягнення цього.

```python
from qiskit import QuantumCircuit
from qiskit_aer import AerSimulator

# Create a circuit to initialize qubits to |10> and verify by measurement
qc_init = QuantumCircuit(2, 2)

# --- YOUR CODE HERE ---

# 1. Set qubit 1 to the |1> state
qc_init.x(1)

# 2. Explicitly reset qubit 0 to the |0> state
qc_init.reset(0)

# --- END YOUR CODE ---

qc_init.measure([0, 1], [0, 1])
qc_init.draw('mpl')
```

```python
# Run the circuit and check the outcome
sim_backend = AerSimulator()
pm = generate_preset_pass_manager(backend=sim_backend, optimization_level=1)
isa_qc_init = pm.run(qc_init)

sampler = Sampler(mode=sim_backend)

job = sampler.run([isa_qc_init], shots=1024)
result = job.result()
counts = result[0].data.c.get_counts()

print("Outcome of |10> state measured in Z-basis:", counts)
plot_histogram(counts)
```

З симуляції ти маєш побачити `10` (двійкове для qubit1=1, qubit0=0) зі 100% ймовірністю, що означає: Qubit 1 успішно підготовлено в |1〉, а Qubit 0 — в |0〉.

Тепер, для більш загальної підготовки стану, Qiskit дозволяє ініціалізацію довільних станів за допомогою методу `initialize`. Наприклад, підготуємо Qubit у стані $|+\rangle = (|0\rangle+|1\rangle)/\sqrt{2}$, який є станом суперпозиції, і пару Qubit'ів у стані Белла $(|00\rangle+|11\rangle)/\sqrt{2}$:

```python
import numpy as np

# Initialize a single qubit in |+> state and measure in Z-basis
qc_plus = QuantumCircuit(1, 1)
state_plus = [1/np.sqrt(2), 1/np.sqrt(2)]   # amplitude for |0> and |1>
qc_plus.initialize(state_plus, 0)
qc_plus.measure(0, 0)

# Initialize two qubits in a Bell state manually
qc_bell = QuantumCircuit(2, 2)
bell_state = [1/np.sqrt(2), 0, 0, 1/np.sqrt(2)]  # amplitudes for |00>,|01>,|10>,|11>
qc_bell.initialize(bell_state, [0, 1])
qc_bell.measure([0, 1], [0, 1])

# Transpile and run the initialization circuits
isa_qc_plus = pm.run(qc_plus)
job_plus = sampler.run([isa_qc_plus], shots=1024)
result_plus = job_plus.result()
counts_plus = result_plus[0].data.c.get_counts()

print("Outcome of |+> state measured in Z-basis:", counts_plus)

isa_qc_bell = pm.run(qc_bell)
job_bell = sampler.run([isa_qc_bell], shots=1024)
result_bell = job_bell.result()
counts_bell = result_bell[0].data.c.get_counts()

print("Outcome of Bell state measured in Z-basis:", counts_bell)
```

**Очікувані результати:** Однокубітний стан |+〉 при вимірюванні дасть `0` і `1` приблизно з 50% ймовірністю кожен. Вимірювання стану Белла має дати приблизно 50% `00` і 50% `11`. Якщо ти бачиш ці результати, це підтверджує, що ініціалізація до цих станів пройшла успішно.

**Ініціалізація посередині Circuit'у:** `reset` у Qiskit можна використовувати посередині Circuit'у для повторної ініціалізації Qubit'а до |0〉 на льоту. Наприклад, у кодах виправлення похибок або ітеративних алгоритмах часто вимірюють Qubit і потім скидають його для повторного використання. Операція `reset` є детермінованою; вона скидає будь-який існуючий стан і охолоджує Qubit до основного стану.

**Приклад пристрою:** На апаратному забезпеченні, такому як **ibmq_brisbane** (127 Qubit'ів) або будь-який пристрій IBM, усі Qubit'и за замовчуванням починають у |0〉 при запуску завдання. Якщо тобі потрібен інший початковий стан, ти застосуєш Gate'и на початку (як ми зробили з X, щоб отримати |1〉). Безперервна повторна ініціалізація (для квантового виправлення похибок) є активною темою досліджень, оскільки швидке її виконання є складним завданням. На щастя, для базового використання можливість починати заново з |0…0〉 доступна, і ми продемонстрували, як досягти інших бажаних початкових станів теж.

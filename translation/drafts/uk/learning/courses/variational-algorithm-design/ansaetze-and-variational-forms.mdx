# Ансаци та варіаційні форми {#ansaetze-and-variational-forms}

В основі всіх варіаційних алгоритмів лежить ключова ідея: аналіз відмінностей між станами, які зручно пов'язані через деяке добре визначене відображення (наприклад, неперервне, диференційовне) з набору параметрів або змінних — звідси і назва.

Спочатку ми розберемося, як будувати параметризовані схеми вручну. Ми використаємо ці схеми для визначення <DefinitionTooltip definition="Шари параметризованих гейтів, що повторюються певну кількість разів; параметри гейтів оптимізуються під час алгоритму для мінімізації функції витрат.">варіаційної форми</DefinitionTooltip>, що представляє колекцію параметризованих станів для дослідження варіаційним алгоритмом. Після цього ми побудуємо наш <DefinitionTooltip definition="Комбінація референсного оператора та варіаційної форми для опису простору пошуку, який ми досліджуємо.">ансац</DefinitionTooltip>, застосувавши варіаційну форму до нашого референсного стану.

Крім того, ми розглянемо компроміс між швидкістю та точністю під час дослідження цього простору пошуку.

## Параметризовані квантові схеми {#parameterized-quantum-circuits}

Варіаційні алгоритми працюють, досліджуючи та порівнюючи набір квантових станів $|\psi(\vec{\theta})\rangle$, які залежать від скінченного набору $k$ параметрів $\vec{\theta} = (\theta^0, \ldots, \theta^{k-1})$. Ці стани можна підготувати за допомогою параметризованої квантової схеми, де гейти визначені з регульованими параметрами. Можна створити таку параметризовану схему, не прив'язуючи конкретних кутів:

```python

from qiskit.circuit import QuantumCircuit, Parameter

qc = QuantumCircuit(3)
qc.rx(theta, 0)
qc.cx(0, 1)
qc.x(2)

qc.draw("mpl")
```

## Варіаційна форма та ансац {#variational-form-and-ansatz}

Щоб ітеративно оптимізувати від референсного стану $|\rho\rangle$ до цільового стану $|\psi(\vec\theta)\rangle$, потрібно визначити варіаційну форму $U_V(\vec{\theta})$, яка представляє колекцію параметризованих станів для дослідження варіаційним алгоритмом:

$$
\begin{aligned}
|0\rangle \xrightarrow{U_R} U_R|0\rangle

& = |\rho\rangle \xrightarrow{U_V(\vec{\theta})} U_A(\vec{\theta})|0\rangle \\[1mm]

& = U_V(\vec{\theta})U_R|0\rangle \\[1mm]

& = U_V(\vec{\theta})|\rho\rangle \\[1mm]

& = |\psi(\vec{\theta})\rangle \\[1mm]

\end{aligned}
$$

Зверни увагу: параметризований стан залежить як від референсного стану $|\rho\rangle$, що не залежить від жодних параметрів, так і від варіаційної форми $U_V(\vec{\theta})$, яка завжди залежить від параметрів. Комбінацію цих двох частин ми називаємо ансацом: $U_A(\vec\theta) := U_V(\vec\theta)U_R$.

Будуючи ансац для представлення колекції параметризованих станів, ми стикаємося з важливою проблемою: розмірністю. Система з $n$ кубітів (тобто простір Гільберта) містить величезну кількість різних квантових станів у просторі конфігурацій. Нам знадобилася б непрактично велика кількість параметрів, щоб повністю його дослідити. Кількісно розмірність дорівнює $D = 2^{2n}$. Крім того, часова складність пошукових алгоритмів зростає експоненційно з цією розмірністю — явище, що часто називають «прокляттям розмірності».

Щоб подолати цю проблему, прийнято накладати деякі розумні обмеження на варіаційну форму, щоб досліджувалися лише найрелевантніші стани. Пошук ефективного усіченого ансацу є активною областю досліджень, але ми розглянемо два поширені підходи.
## Евристичні ансаци та компроміси {#heuristic-ansaetze-and-trade-offs}

Якщо у тебе немає конкретної інформації про задачу, яка допомогла б обмежити розмірність, можна спробувати довільну сім'ю параметризованих схем із менш ніж $2^{2n}$ параметрами. Однак варто враховувати деякі компроміси:

- **Швидкість**: Скорочення простору пошуку дозволяє алгоритму працювати швидше.
- **Точність**: Скорочення простору може призвести до виключення справжнього розв'язку задачі та отримання субоптимальних результатів.
- **Шум**: Глибші схеми більше схильні до шуму, тому потрібно експериментувати зі зв'язністю ансацу, гейтами та їхньою точністю.

Між якістю (або навіть розв'язуваністю) і швидкістю існує фундаментальний компроміс: чим більше параметрів — тим вища ймовірність отримати точний результат, але тим довше виконуватиметься алгоритм.
### N-локальні схеми {#n-local-circuits}

Одним із найпоширеніших прикладів евристичних ансаців є [N-локальні схеми](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.library.NLocal) з кількох причин:

- **Ефективна реалізація**: N-локальний ансац, як правило, складається з простих локальних гейтів, які можна ефективно реалізувати на квантовому комп'ютері, використовуючи невелику кількість фізичних кубітів. Це спрощує побудову та оптимізацію квантових схем.
- **Врахування важливих кореляцій**: N-локальний ансац може вловлювати важливі кореляції між кубітами квантової системи навіть із невеликою кількістю гейтів. Це можливо тому, що локальні гейти діють на сусідні кубіти та створюють між ними заплутаність, яка важлива для моделювання складних квантових систем.

Ці схеми складаються з шарів обертань і заплутаності, що чергуються один або кілька разів:

- Кожен шар формується гейтами розміром не більше $N$, де $N$ має бути менше кількості кубітів.
- У шарі обертань гейти складаються один над одним. Можна використовувати стандартні операції обертання, наприклад [`RX`](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.library.RXGate) або [`CRZ`](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.library.CRZGate).
- У шарі заплутаності можна використовувати гейти на кшталт [гейтів Тоффолі](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.library.CCXGate) або [`CX`](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.library.CXGate) зі стратегією заплутаності.
- Обидва типи шарів можуть бути параметризованими або ні, але принаймні один із них повинен містити параметри. Інакше без хоча б одного параметра не було б жодних варіацій!
- За бажанням, наприкінці схеми додається додатковий шар обертань.

Наприклад, створимо п'ятикубітну схему [`NLocal`](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.library.NLocal) з блоками обертань, сформованими гейтами [`RX`](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.library.RXGate) та [`CRZ`](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.library.CRZGate), блоками заплутаності з [гейтів Тоффолі](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.library.CCXGate), що діють на кубіти $[0,1,2]$, $[0,2,3]$, $[4,2,1]$ та $[3,1,0]$, і $2$ повторюваннями кожного шару.

```python
from qiskit.circuit.library import NLocal, CCXGate, CRZGate, RXGate
from qiskit.circuit import Parameter

theta = Parameter("θ")
ansatz = NLocal(
    num_qubits=5,
    rotation_blocks=[RXGate(theta), CRZGate(theta)],
    entanglement_blocks=CCXGate(),
    entanglement=[[0, 1, 2], [0, 2, 3], [4, 2, 1], [3, 1, 0]],
    reps=2,
    insert_barriers=True,
)
ansatz.decompose().draw("mpl")
```

![Результат виконання попередньої комірки коду](/learning/images/courses/variational-algorithm-design/ansaetze-and-variational-forms/extracted-outputs/c28d7b53-fbd7-42f5-a619-5a6b0b44e9fc-0.avif)

У наведеному прикладі найбільшим гейтом є гейт Тоффолі, що діє на три кубіти, тому схема є $3$-локальною. Найпоширенішим типом $N$-локальних схем є [$2$-локальні](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.library.TwoLocal) схеми з однокубітними гейтами обертань та двокубітними гейтами заплутаності.

Створимо $2$-локальну схему за допомогою класу [`TwoLocal`](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.library.TwoLocal) з Qiskit. Синтаксис аналогічний до `NLocal`, але є деякі відмінності. Наприклад, більшість гейтів, таких як `RX`, `RZ` та `CNOT`, можна передавати як рядки, не імпортуючи гейти та не створюючи екземпляр `Parameter`.

```python
from qiskit.circuit.library import TwoLocal

ansatz = TwoLocal(
    num_qubits=5,
    rotation_blocks=["rx", "rz"],
    entanglement_blocks="cx",
    entanglement="linear",
    reps=2,
    insert_barriers=True,
)
ansatz.decompose().draw("mpl")
```

У цьому випадку ми використали лінійний розподіл заплутаності, де кожен кубіт заплутаний із наступним. Щоб дізнатися про інші стратегії, звернися до [документації `TwoLocal`](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.library.TwoLocal).
### Efficient SU2 {#efficient-su2}

[`efficient_su2`](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.library.EfficientSU2) — це апаратно-ефективна схема, що складається з шарів однокубітних операцій у просторі SU(2) та заплутаності `CX`. Це евристичний шаблон, який можна використовувати для підготовки пробних хвильових функцій у варіаційних квантових алгоритмах або як схему класифікації в машинному навчанні.

```python
from qiskit.circuit.library import efficient_su2

ansatz = efficient_su2(4, su2_gates=["rx", "y"], entanglement="linear", reps=1)
ansatz.decompose().draw("mpl")
```

## Проблемно-специфічні ансаци {#problem-specific-ansaetze}

Тоді як евристичні та апаратно-ефективні ансаци допомагають розв'язати задачу наївним способом, можна використовувати знання про конкретну задачу, щоб обмежити простір пошуку схем певним типом. Це дозволить виграти у швидкості без втрати точності пошуку.
### Оптимізація {#optimization}

У задачі максимального розрізу (max-cut) потрібно розбити вузли графа так, щоб максимізувати кількість ребер між вузлами різних груп. Бажаний розподіл для наведеного нижче графа очевидний: нульовий вузол зліва має бути відокремлений від решти вузлів справа.

```python
import rustworkx as rx
from rustworkx.visualization import mpl_draw

n = 4
G = rx.PyGraph()
G.add_nodes_from(range(n))
# The edge syntax is (start, end, weight)
edges = [(0, 1, 1.0), (0, 2, 1.0), (0, 3, 1.0), (1, 2, 1.0), (2, 3, 1.0)]
G.add_edges_from(edges)

mpl_draw(
    G, pos=rx.shell_layout(G), with_labels=True, edge_labels=str, node_color="#1192E8"
)
```

![Результат виконання попередньої комірки коду](/learning/images/courses/variational-algorithm-design/ansaetze-and-variational-forms/extracted-outputs/ef9a5e1d-f70a-4f59-9112-2d8f71b253ba-0.avif)

Щоб застосувати алгоритм QAOA до задачі max-cut, потрібен гамільтоніан Паулі, що кодує вартість так, щоб мінімальне очікуване значення оператора відповідало максимальній кількості ребер між вузлами двох різних груп.

Для цього простого прикладу оператор є лінійною комбінацією членів з операторами Z на вузлах, з'єднаних ребром (нагадуємо: нульовий кубіт знаходиться крайнім праворуч): $ZZII + IZZI + ZIIZ + IZIZ + IIZZ$. Після побудови оператора ансац для алгоритму QAOA можна легко створити за допомогою схеми `QAOAAnsatz` з бібліотеки схем Qiskit.

```python
# Pre-defined ansatz circuit, operator class and visualization tools
from qiskit.circuit.library import QAOAAnsatz
from qiskit.quantum_info import SparsePauliOp

# Problem to Hamiltonian operator
hamiltonian = SparsePauliOp.from_list(
    [("ZZII", 1), ("IZZI", 1), ("ZIIZ", 1), ("IZIZ", 1), ("IIZZ", 1)]
)
# QAOA ansatz circuit
ansatz = QAOAAnsatz(hamiltonian, reps=2)
# Draw
ansatz.decompose(reps=3).draw("mpl")
```

На попередньому зображенні ансац зображено в базових гейтах для наочності. Однак його можна представити на кількох рівнях декомпозиції, змінюючи аргумент `reps` або малюючи схему без методу decompose. ### Квантове машинне навчання {#quantum-machine-learning}

У машинному навчанні поширеним застосуванням є класифікація даних на дві або більше категорій. Це передбачає кодування точки даних у карту ознак, яка відображає класичні вектори ознак у квантовий простір Гільберта. Побудова квантових карт ознак на основі параметризованих квантових схем, які важко моделювати класично, є важливим кроком до отримання потенційної переваги над класичними підходами до машинного навчання та є активною областю поточних досліджень.

`zz_feature_map` можна використовувати для створення параметризованої схеми. Можна передати точки даних у карту ознак ($x$) та окрему варіаційну форму для передачі ваг як параметрів ($\theta$).

```python
from qiskit.circuit.library import zz_feature_map, TwoLocal

data = [0.1, 0.2]

zz_feature_map_reference = zz_feature_map(feature_dimension=2, reps=2)
zz_feature_map_reference = zz_feature_map_reference.assign_parameters(data)

variation_form = TwoLocal(2, ["ry", "rz"], "cz", reps=2)
vqc_ansatz = zz_feature_map_reference.compose(variation_form)
vqc_ansatz.decompose().draw("mpl")
```

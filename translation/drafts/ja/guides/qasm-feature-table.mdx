# OpenQASM 3 機能一覧表 {#openqasm-3-feature-table}

以下は `OpenQASM 3` 言語機能の一覧です。

これらの機能の詳細については、[OpenQASM 3.X ライブ仕様](https://openqasm.com/)をご参照ください。

凡例：

*   ❌ 未サポート
*   🟡 部分的サポート
*   ✅ サポート済み

「サポート済み」の意味は列によって異なります。

* *Qiskit SDK*: [`qiskit.qasm3.loads`](https://docs.quantum.ibm.com/api/qiskit/qasm3) によってパース可能（`qiskit-qasm3-import` 拡張を使用）、[`QuantumCircuit`](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.QuantumCircuit) で表現可能、かつ [`qiskit.qasm3.dumps`](https://docs.quantum.ibm.com/api/qiskit/qasm3#dumps) によって OpenQASM 3 としてエクスポート可能であることを意味します。

* *IBM Qiskit Runtime*: 対応する Qiskit 機能を含む回路が、IBM&reg; Qiskit Runtime を通じてハードウェア上で正常に実行できることを意味します。

「部分的サポート」の意味は、リンク先のノートによって異なります。

<Admonition type="note">
IBM Qiskit Runtime への回路送信で最も一般的な方法は、Qiskit SDK の Python インターフェースで回路を作成することです。
この方法で構築・送信される回路は、OpenQASM 3 ファイルから Qiskit SDK に読み込む必要はありません。

OpenQASM 3 を直接使用しない場合は、Qiskit SDK での表現、OpenQASM 3 へのエクスポート、IBM Qiskit Runtime への送信がサポートされている機能を安全にご利用いただけます。
これには、Qiskit SDK が OpenQASM 3 から読み込めない機能も含まれます。
</Admonition>

## 注記 {#notes}

<span id="1"></span>
1.  これらの OpenQASM 3 プログラム機能は実行に影響を与えないため、Qiskit はファイルのパース時にそれらを取り除きます。これらを使用したファイルは送信できますが、効果はありません。`include` ファイルについては、現在 `stdgates.inc` が Qiskit への入力としてサポートされており、バックエンド実行では常に回路がバックエンドの命令セットアーキテクチャ（ISA）にコンパイルされている必要があるため、`include` ファイルは無関係になります。

<span id="2"></span>
2. Qiskit SDK は、任意の ``qubit`` 宣言を含む OpenQASM 3 ファイルのパースおよびダンプをサポートしています。ハードウェア上での実行には、ハードウェア量子ビット（例：`$0`）で定義された回路のみが有効です。レイアウト情報を持つバックエンド向けにトランスパイルされた回路の場合、Qiskit SDK はサポートされているハードウェア量子ビット識別子を使用して OpenQASM 3 を自動的に出力します。

<span id="3"></span>
3. Qiskit SDK における `bit` 型および `bit[n]` 型の変数宣言は、`Clbit` および `ClassicalRegister` の宣言に対応します。

<span id="4"></span>
4. 2025年7月時点で、Qiskit SDK は限定された型のローカル変数を表現でき、これらのオブジェクトに対する多くのランタイム演算を表現でき、OpenQASM 3 への出力をサポートしています。ただし、Qiskit SDK（`qiskit-qasm3-import` v0.6.0 経由）は変数宣言を含む OpenQASM 3 ファイルのパースをサポートしておらず、変数式のパースサポートも非常に限定的です。一般に、Qiskit が式システムで表現できる大部分のものは、式が Qiskit SDK でパースできない場合でも、適切なダイナミック回路ハードウェア上で実行できます。最新情報については、[`qiskit.circuit.classical` モジュールの Qiskit ドキュメント](https://docs.quantum.ibm.com/api/qiskit/circuit_classical)をご参照ください。

<span id="5"></span>
5. Qiskit SDK は量子レジスタおよび古典レジスタの両方に対するレジスタエイリアシングを表現できますが、古典レジスタのエイリアシングの使用は強く推奨されません。古典レジスタに対するほとんどの式はエイリアスでは機能せず、エイリアスされた古典レジスタはハードウェア上での実行がサポートされていません。Qiskit の OpenQASM 3 パーサーは、レジスタ連結の結果を束縛する `let` エイリアス文を解決できます。

<span id="6"></span>
6. Qiskit SDK は `QuantumCircuit.delay` による明示的な遅延をサポートしており、回路ボックス（`QuantumCircuit.box`）にも明示的な持続時間を設定できます。これらの持続時間には `stretch` 変数の古典式を含めることができます。Qiskit SDK（2025年7月時点、`qiskit-qasm3-import` v0.6.0 経由）は、OpenQASM 3 ファイルからの `duration` 型または `stretch` 型の宣言のパースをサポートしていません。ハードウェアでは `stretch` を含む持続時間のサポートが限定的です。

<span id="7"></span>
7. IBM ハードウェア上で実行するには、回路をバックエンド ISA にトランスパイルする必要があります。これにより、カスタム `gate` 定義やゲート修飾子（`inv @` など）などの高水準構成要素は、そのままではハードウェア実行に有効ではありませんが、`transpile` プロセスによって有効な ISA 回路に解決されます。Qiskit SDK（2025年7月時点、`qiskit-qasm3-import` v0.6.0 経由）はパース時にゲート修飾子を積極的に評価するため、結果として得られる `QuantumCircuit` にはこれらが反映されず、ランタイムコストが発生する可能性があります。

<span id="8"></span>
8. Qiskit SDK は構造化制御フローを表現し、OpenQASM 3 にエクスポートできます。`continue` および `break` 文は技術的に Qiskit で表現できますが、Qiskit SDK 内でも十分にサポートされていません。Qiskit v2.1.0 の `for` ループも十分にサポートされていません。ネストされた制御フロー（例：`if` の中の `if` や `else if` 文）はハードウェア上での実行には対応していません。

<span id="9"></span>
9. Qiskit SDK は、サポートされている任意の古典型を回路の `input` 変数として宣言できます。このような変数は現在ハードウェア上での実行には対応しておらず、Qiskit の OpenQASM 3 インポーターでも読み込めません。`QuantumCircuit` に存在するバインドされていない `Parameter` オブジェクトは `input float[64]` 変数としてエクスポートされます。特定のランタイム設定オプションにより、一部のバックエンドでこのような回路の実行が可能になる場合があります。

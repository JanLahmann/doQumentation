## 概要 {#overview}

AlgorithmiqのテンソルネットワークError Mitigation（TEM）は、古典的な後処理段階で完全にノイズ軽減を行うように設計されたハイブリッド量子古典アルゴリズムです。TEMを使用することで、量子ハードウェア上で不可避的に発生するノイズ誘発エラーを軽減しながら、より高い精度とコスト効率でオブザーバブルの期待値を計算できます。これにより、量子研究者や産業実践者にとって非常に魅力的な選択肢となっています。

この手法は、量子プロセッサの状態に影響を与えるグローバルなノイズチャンネルの逆数を表すテンソルネットワークを構築し、そのマップをノイズのある状態から取得した情報完全測定結果に適用することで、オブザーバブルの不偏推定量を得ることで構成されます。

TEMの利点として、情報完全測定を活用することで多数の軽減された期待値へのアクセスが可能となり、量子ハードウェアでの測定オーバーヘッドが最適化されています。これはFilippov et al.（2023年）[arXiv:2307.11740](https://arxiv.org/abs/2307.11740) および Filippov et al.（2024年）[arXiv:2403.13542](https://arxiv.org/abs/2403.13542) に記載されています。測定オーバーヘッドとは、効率的な誤り軽減を行うために必要な追加測定の数を指し、量子計算の実現可能性における重要な要素です。そのため、TEMは量子カオス、多体物理学、ハバード動力学、小分子化学シミュレーションなどの分野の複雑なシナリオで量子アドバンテージを実現できる可能性を持っています。

TEMの主な特徴と利点を以下にまとめます：

1.  **最適な測定オーバーヘッド**: TEMは[理論的な限界](https://journals.aps.org/prl/abstract/10.1103/PhysRevLett.131.210601)に対して最適であり、どの手法もそれより小さい測定オーバーヘッドを達成できないことを意味します。言い換えれば、TEMは誤り軽減を行うために必要な追加測定の最小数を必要とします。これにより、TEMは最小限の量子ランタイムを使用します。
2.  **コスト効率**: TEMはノイズ軽減を後処理段階で完全に処理するため、量子コンピュータに追加の回路を追加する必要がなく、計算コストが削減されるだけでなく、量子デバイスの不完全性による追加エラーのリスクも軽減されます。
3.  **複数のオブザーバブルの推定**: 情報完全測定のおかげで、TEMは量子コンピュータからの同じ測定データを使用して複数のオブザーバブルを効率的に推定します。
4.  **測定誤り軽減**: TEM Qiskit Functionには、短いキャリブレーション実行の後に読み出しエラーを大幅に削減できる[独自の測定誤り軽減手法](https://journals.aps.org/prresearch/abstract/10.1103/PhysRevResearch.5.033154)も含まれています。
5.  **精度**: TEMはデジタル量子シミュレーションの精度と信頼性を大幅に向上させ、量子アルゴリズムをより正確で信頼性の高いものにします。
## 説明 {#description}

TEM Functionを使用すると、最小限のサンプリングオーバーヘッドで複数のオブザーバブルの誤り軽減された期待値を取得できます。回路は情報完全正演算子値測度（IC-POVM）で測定され、収集された測定結果が古典コンピュータ上で処理されます。この測定はテンソルネットワーク手法を実行し、ノイズ逆変換マップを構築するために使用されます。この関数は、テンソルネットワークを使用してノイズのある層を表し、ノイズのある回路全体を完全に反転させるマップを適用します。

![TEMの模式図](/docs/images/guides/algorithmiq-tem/tem_scheme.svg "ノイズのある量子プロセッサの測定結果の後処理によるオブザーバブルOの誤り軽減推定。UとNはそれぞれ理想的な量子演算と関連するノイズマップを表し、一般的に非局所的である（灰色のボックスに拡張される）。DはIC測定の効果に対して双対な演算子のテンソルを表す。ノイズ軽減モジュールMは中央から外に向かって効率的に縮約されるテンソルネットワークである。縮約の最初の反復は点線の紫の線で、2番目は破線で、3番目は実線で表される。")

回路がFunctionに送信されると、2量子ビットゲート（量子デバイス上でノイズが多いゲート）の層数を最小化するためにトランスパイルおよび最適化されます。層に影響するノイズは、E. van den Berg, Z. Minev, A. Kandala, K. Temme, Nat. Phys.（2023年）[arXiv:2201.09866](https://arxiv.org/abs/2201.09866) に記載されたスパースPauli-Lindbladノイズモデルを使用して、[Qiskit Runtime](https://docs.quantum.ibm.com/api/qiskit-ibm-runtime/noise-learner-noise-learner)を通じて学習されます。

ノイズモデルは量子ビットのクロストークを含む微妙な特徴を捉えることができる、デバイスのノイズの正確な記述です。ただし、デバイスのノイズは変動・ドリフトする場合があり、学習されたノイズが推定実行時点で正確でない場合があります。これにより不正確な結果が生じる可能性があります。

## 例 {#example}

以下のスニペットは、TEMを使用して簡単な量子回路でオブザーバブルの期待値を計算する例を示しています。

```python
from qiskit import QuantumCircuit
from qiskit.quantum_info import SparsePauliOp

# Create a quantum circuit
qc = QuantumCircuit(3)
qc.u(0.4, 0.9, -0.3, 0)
qc.u(-0.4, 0.2, 1.3, 1)
qc.u(-1.2, -1.2, 0.3, 2)
for _ in range(2):
    qc.barrier()
    qc.cx(0, 1)
    qc.cx(2, 1)
    qc.barrier()
    qc.u(0.4, 0.9, -0.3, 0)
    qc.u(-0.4, 0.2, 1.3, 1)
    qc.u(-1.2, -1.2, 0.3, 2)

# Define the observables
observable = SparsePauliOp("IYX", 1.0)

# Define the execution options
pub = (qc, [observable])
options = {"default_precision": 0.02}

# Define backend to use. TEM will choose the least-busy device reported by IBM if not specified
backend_name = "ibm_torino"

job = tem.run(pubs=[pub], backend_name=backend_name, options=options)
```

Qiskit Serverless APIを使用して、Qiskit Functionのワークロードステータスを確認します：

```python
print(job.status())
```


結果は以下のように取得できます：

```python

```

## 入力 {#inputs}

**パラメータ**

名前 | 型 | 説明 | 必須 | デフォルト | 例

`backend_name` | str | クエリを行うBackendの名前。| いいえ | 指定しない場合、最も負荷の少ないBackendが使用されます。 | "ibm_fez"
`options` | dict | 入力オプション。詳細は `Options` セクションを参照してください。 | いいえ | 詳細は `Options` セクションを参照してください。| \{"max_bond_dimension": 100\\}

<Admonition type="caution">
  TEMには現在、以下の制限があります：

  - パラメータ化された回路はサポートされていません。精度が指定されている場合、parametersの引数は `None` に設定する必要があります。この制限は将来のバージョンで解除される予定です。
  - ループのない回路のみがサポートされています。この制限は将来のバージョンで解除される予定です。
  - reset、measure、あらゆる形式の制御フローなど、ユニタリでないゲートはサポートされていません。resetのサポートは今後のリリースで追加される予定です。
</Admonition>
### オプション {#options}

TEMの高度なオプションを含む辞書です。辞書には以下の表のキーを含めることができます。いずれかのオプションが指定されていない場合、表に記載されているデフォルト値が使用されます。デフォルト値はTEMの一般的な使用に適しています。

名前 | 選択肢 | 説明 | デフォルト
-- | -- | -- | --
`tem_max_bond_dimension` | int | テンソルネットワークに使用する最大ボンド次元。 | 500 |
`tem_compression_cutoff` | float | テンソルネットワークに使用するカットオフ値。 | 1e-16
`compute_shadows_bias_from_observable` | bool | 古典シャドウ測定プロトコルのバイアスをPUBのオブザーバブルに合わせるかどうかを示すブールフラグ。Falseの場合、古典シャドウプロトコル（Z、X、Yを測定する等確率）が使用されます。| False
`shadows_bias` | np.ndarray | ランダム化された古典シャドウ測定プロトコルに使用するバイアス。サイズ3の1次元配列、またはそれぞれ形状(num_qubits, 3)の2次元配列。順序はZXY | np.array([1 / 3, 1 / 3, 1 / 3])
`max_execution_time` | int または `None` | QPU上の最大実行時間（秒）。ランタイムがこの値を超えた場合、ジョブはキャンセルされます。`None` の場合、Qiskit Runtimeが設定するデフォルト制限が適用されます。 | `None`
`num_randomizations` | int | ノイズ学習とゲートツワーリングに使用するランダム化の回数。 | 32
`max_layers_to_learn` | int | 学習する一意の層の最大数。 | 4
`mitigate_readout_error` | bool | 読み出し誤り軽減を実行するかどうかを示すブールフラグ。 | True
`num_readout_calibration_shots` | int | 読み出し誤り軽減に使用するショット数。 | 10000
`default_precision` | float | 精度が指定されていないPUBに使用するデフォルトの精度。 |0.02
`seed` | int または `None` | 再現性のための乱数生成器のシードを設定します。`None` の場合、シードを設定しません。 | None
## 出力 {#outputs}

TEM軽減された結果を含むQiskit [PrimitiveResults](https://docs.quantum.ibm.com/api/qiskit/qiskit.primitives.PrimitiveResult)。各PUBの結果は、以下のフィールドを含む [PubResult](https://docs.quantum.ibm.com/api/qiskit/qiskit.primitives.PubResult) として返されます：

名前 | 型 | 説明
-- | -- | --
data | DataBin | TEM軽減されたオブザーバブルとその標準誤差を含むQiskit [DataBin](https://docs.quantum.ibm.com/api/qiskit/qiskit.primitives.DataBin)。DataBinには以下のフィールドがあります: <ul><li>`evs`: TEM軽減されたオブザーバブルの値。</li><li>`stds`: TEM軽減されたオブザーバブルの標準誤差。</li></ul>
metadata | dict | 追加結果を含む辞書。辞書には以下のキーが含まれます: <ul><li>`"evs_non_mitigated"`: 誤り軽減なしのオブザーバブルの値。</li><li>`"stds_non_mitigated"`: 誤り軽減なしの結果の標準誤差。</li><li>`"evs_mitigated_no_readout_mitigation"`: 誤り軽減あり・読み出し誤り軽減なしのオブザーバブルの値。</li><li>`"stds_mitigated_no_readout_mitigation"`: 誤り軽減あり・読み出し誤り軽減なしの結果の標準誤差。</li><li>`"evs_non_mitigated_with_readout_mitigation"`: 誤り軽減なし・読み出し誤り軽減ありのオブザーバブルの値。</li><li>`"stds_non_mitigated_with_readout_mitigation"`: 誤り軽減なし・読み出し誤り軽減ありの結果の標準誤差。</li></ul>

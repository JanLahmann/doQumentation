# Qiskit Runtimeを使用する際の責任範囲について {#understanding-your-responsibilities-when-using-qiskit-runtime}

Qiskit Runtimeを使用する際のお客様およびIBM&reg;の具体的な責任範囲については、以下の各セクションをご確認ください。全体的な利用規約については、IBM Cloud&reg;の[利用規約と注意事項](https://cloud.ibm.com/docs/overview/terms-of-use?topic=overview-terms)をご参照ください。

## インシデントおよびオペレーション管理 {#incident-and-operations-management}

インシデントおよびオペレーション管理には、監視、イベント管理、高可用性、問題の特定、復旧、完全な状態のバックアップと復旧などのタスクが含まれます。

| タスク | IBMの責任 | お客様の責任 |
| --- | --- | --- |
|インフラストラクチャの健全性管理|<ul><li>新機能、アップデート、バグ修正を必要に応じて透明性を持って継続的に提供します。</li><li>計画されたメンテナンス、セキュリティ情報、または計画外の停止について通知します。</li></ul> | <ul><li>サービス通知に関するメールを受け取るよう[通知設定](https://cloud.ibm.com/user/notifications)を行ってください。</li><li>一般的なお知らせについては[IBM Cloudステータス](https://cloud.ibm.com/status)ページを監視してください。</li></ul> |
|インフラストラクチャの健全性監視と通知| <ul><li>インフラストラクチャ環境のすべての問題を修正します。</li><li>該当するインシデントについてお客様に通知します。</li></ul> | <ul><li>報告された各インシデントの影響を評価してください。</li><li>必要に応じて[IBMサポート](/guides/support)にお問い合わせください。</li></ul>|
|IBM Cloudの高可用性SLAの維持| [IBM Cloud パブリック サービス レベル アグリーメント](https://cloud.ibm.com/docs/overview?topic=overview-slas)（SLA）に基づいてQiskit Runtimeを運用します。 | 高可用性のベストプラクティスに従ってください。 |

## 変更管理 {#change-management}

変更管理には、デプロイ、設定、アップグレード、パッチ適用、設定変更、削除などのタスクが含まれます。

| タスク | IBMの責任 | お客様の責任 |
| --- | --- | --- |
| メジャーバージョンのアップグレード | メジャーバージョンのアップデートに必要な可用性とツールを提供します。 | 旧バージョンのサポート終了前に、サポート対象バージョンへ移行してください。 |
| APIバージョンの変更 | REST APIのバージョン管理を行います。 | 必要なバージョンのREST APIと統合してください。 |
| アップデート、修正、および新機能 | <ul><li>継続的デリバリーモデルに従い、定期的なアップデート、バグ修正、新機能を透明性を持って提供します。</li><li>お客様に影響を与える変更についての通知を掲載します。</li></ul> | <ul><li>[メール通知を受け取る](https://cloud.ibm.com/user/notifications)よう設定し、一般的なお知らせについては[IBM Cloudステータス](https://cloud.ibm.com/status)ページを監視してください。</li><li>システム生成の通知および[プラットフォームのお知らせ](https://quantum.cloud.ibm.com/announcements)を確認し、適切な対応を取ってください。</li></ul> |

## ID・アクセス管理 {#identity-and-access-management}

ID・アクセス管理には、認証、認可、アクセス制御ポリシーの管理、アクセスの承認・付与・取り消しなどのタスクが含まれます。

| タスク | IBMの責任 | お客様の責任 |
| --- | --- | --- |
| IDとアクセス制御 | IBM Cloudを通じてリソースへのアクセスを管理・制限する機能を提供します。 | [Cloud Identity and Access Management](/guides/access-groups)（IAM）を通じてリソースへのアクセスを付与・取り消し・管理してください。 |
| サービス認証 | 認証済みのユーザーのみがQiskit Runtimeのサービスおよびリソースにアクセスできるようにします。 | <ul><li>不正アクセスからの保護のため、[APIキーを安全に管理](/guides/cloud-setup-untrusted#initialize-the-service-in-an-untrusted-environment)してください。</li><li>セキュリティポリシーの要件に従って[APIキー](https://cloud.ibm.com/iam/apikeys)をローテーション・更新してください。</li></ul> |
| オブザーバビリティ | ユーザーのアクションを監査するためのIBM Cloud Activity Tracker Event Routingとの統合を可能にします。 | ユーザーアクティビティを追跡するために、[IBM Cloud Activity Tracker Event Routing](https://cloud.ibm.com/docs/vmware-service?topic=vmware-service-at_events)またはその他の機能を設定してください。 |

## セキュリティおよび規制準拠 {#security-and-regulation-compliance}

セキュリティおよび規制準拠には、セキュリティ管理の実装およびコンプライアンス認証などのタスクが含まれます。

| タスク | IBMの責任 | お客様の責任 |
| --- | --- | --- |
| 全般 | <ul><li>様々な業界コンプライアンス標準に見合った管理策を維持します。</li><li>サービス機能の監視、分離、および復旧を行います。</li><li>サービスインターフェースの健全性を監視・報告します。</li></ul> | 個人情報や規制対象データが提供されないようにしてください。 |
| 脆弱性 | 脆弱性およびセキュリティコンプライアンスの問題を検出するための監視を行います。 | <ul><li>潜在的な修正措置を提供するセキュリティ情報を通じて、脆弱性やセキュリティ問題について常に情報を収集してください。</li><li>[QiskitのGitHubリポジトリのセキュリティセクション](https://github.com/Qiskit/qiskit/security)を確認してください。すべてのセキュリティ上の脆弱性はそこに掲載されます。</li><li>[IBM Cloudステータスウェブサイト](https://cloud.ibm.com/status)を確認し、IBM Cloudプラットフォーム、インフラストラクチャ、および主要サービスに影響する重要なイベントに関するお知らせやセキュリティ情報を確認してください。</li></ul> |
| 暗号化 | 公衆ネットワーク上および Cloud Service のプライベートデータセンターネットワーク内で転送されるデータを暗号化します。HTTPSやTLSなどの暗号化方式を使用してデータの転送中の保護を行います。 | 必要に応じて、接続がエンドツーエンドで暗号化されていることを確認してください。 |
| セキュリティ | <ul><li>インフラストラクチャ内の保存データおよび転送中のデータのセキュリティを確保します。</li><li>主要な標準に準拠したセキュアなサービスを提供します。</li></ul> | 適切なリソースおよびリソースグループへの[ユーザーアクセス](/guides/access-groups)を制限してください。 |

## 災害復旧 {#disaster-recovery}

| タスク | IBMの責任 | お客様の責任 |
| --- | --- | --- |
| 全般 | 影響を受けたリージョンのプロビジョニング環境とリソースを復元または再構築します。 | <ul><li>インスタンスとリソースの状態を監視してください。</li><li>利用可能な場合は、別のリージョンに新しいQiskit Runtimeサービス[インスタンス](/guides/instances)をプロビジョニングしてください。</li></ul> |
| サービスの復旧 | Qiskit Runtimeサービスの機能を自動的に復旧・復元し、通常の運用状態に戻します。 | 復旧後、影響を受けたジョブを再送信してください。 |
| ユーザー設定データのバックアップ | IBMはユーザー設定データのバックアップについては責任を負いません。 | 必要に応じて設定データをバックアップしてください。 |
| ワークロードのバックアップと復旧 | IBMはユーザーのワークロードデータのバックアップまたは復旧については責任を負いません。 | 必要に応じてワークロードのバックアップと復旧を行ってください。 |

# ジョブの実行時間を最小化する {#minimize-job-run-time}

ジョブの処理と実行に費やす時間を制限する方法はいくつかあります：

- **必要なショット数だけを実行する**：ジョブにかかる量子時間（およびそのコスト）はショット数に比例してスケールします。そのため、必要なショット数だけを実行することでコストを管理できます。Estimator ジョブでは、精度を低くすると一般的にショット数が増え、実行時間も長くなります。

- **実行時間に制限を設ける**：各ジョブ、バッチ、またはセッションの実行時間を制限できます。詳細については、[Qiskit Runtime ワークロードの最大実行時間](max-execution-time)を参照してください。

- **エラー抑制とエラー軽減に必要な設定のみを使用する**：値を高く設定するとジョブの実行時間が長くなる場合があります。詳細については、[オプションの概要](./runtime-options-overview)、[エラー抑制の設定](./configure-error-suppression)、および[エラー軽減の設定](configure-error-mitigation)を参照してください。

- **同じ（パラメータ化された）回路を含む複数のジョブを実行している場合**で、PEA や PEC などのノイズモデルを必要とするエラー軽減手法を使用している場合は、`NoiseLearner` の使用を検討してください。このヘルパープログラムを使用すると、回路のノイズモデルを一度学習し、その後の Estimator クエリでモデルを再利用できます。ノイズモデルは一定時間後に古くなるため、ジョブ間に長い遅延がない場合（例えば、セッション内）にのみ実用的です。詳細については、[ノイズ学習ヘルパー](./noise-learning)を参照してください。

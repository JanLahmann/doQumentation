# `BackendV1`から`BackendV2`への移行 {#migrate-from-backendv1-to-backendv2}

Qiskitの[`BackendV1`](https://docs.quantum.ibm.com/api/qiskit/1.4/qiskit.providers.BackendV1)クラスは非推奨となり、サービスから削除される予定です。このマイグレーションガイドでは、[`BackendV1`](https://docs.quantum.ibm.com/api/qiskit/1.4/qiskit.providers.BackendV1)から[`BackendV2`](https://docs.quantum.ibm.com/api/qiskit/qiskit.providers.BackendV2)にアップグレードしたプロバイダーを使用する場合に必要な小さな調整について説明します。

<Admonition type="note">
 `qiskit_ibm_runtime`と`qiskit_aer`のみを使用している場合は、対応は不要です。`qiskit_ibm_runtime`パッケージは常に`BackendV2`を使用しており、`qiskit_aer`はバージョン0.13以降`BackendV2`を使用しています。
 </Admonition>

## `BackendV2`における高レベルの変更点 {#high-level-changes-in-backendv2}

Qiskitの[`Backend`](https://docs.quantum.ibm.com/api/qiskit/qiskit.providers.Backend)モデルは、Qiskit SDKに抽象化レイヤーを提供し、SDK内でコンピューターについて推論できるように設計されました。モデルの最初のバージョンは、[`BackendV1`](https://docs.quantum.ibm.com/api/qiskit/1.4/qiskit.providers.BackendV1)クラスとして導入されました。このクラスは、バックエンド情報を一連のデータコンテナ、すなわち[`BackendConfiguration`](https://docs.quantum.ibm.com/api/qiskit/1.4/qiskit.providers.models.BackendConfiguration)クラスと[`BackendProperties`](https://docs.quantum.ibm.com/api/qiskit/1.4/qiskit.providers.models.BackendProperties)クラスに格納していました。

[`BackendV2`](https://docs.quantum.ibm.com/api/qiskit/qiskit.providers.BackendV2)クラスは、ほとんどのバックエンドプロパティに対するユーザーアクセスを再定義し、ネイティブなQiskitデータ構造と連携し、よりフラットなアクセスパターンを持つようになりました。[`BackendV2`](https://docs.quantum.ibm.com/api/qiskit/qiskit.providers.BackendV2)モデルの核心は[`Target`](https://docs.quantum.ibm.com/api/qiskit/qiskit.transpiler.Target)クラスです。これはQPUを表現するもので、Qiskitが実行のために回路を最適化するために使用できるトランスパイルの制約条件が含まれています。

Qiskit SDKは[`BackendV2`](https://docs.quantum.ibm.com/api/qiskit/qiskit.providers.BackendV2)入力のみで動作するように更新されており、ほとんどのプロバイダーが[`BackendV1`](https://docs.quantum.ibm.com/api/qiskit/1.4/qiskit.providers.BackendV1)から[`BackendV2`](https://docs.quantum.ibm.com/api/qiskit/qiskit.providers.BackendV2)にアップグレードしています。既存のプロバイダーは可能な限り古いアクセス方法を非推奨にして円滑な移行を提供することが期待されていますが、最終的にはユーザーがコードを調整する必要があります。

[`BackendV2`](https://docs.quantum.ibm.com/api/qiskit/qiskit.providers.BackendV2)の原則は、バックエンドに関するほとんどの情報がその[`Target`](https://docs.quantum.ibm.com/api/qiskit/qiskit.transpiler.Target)オブジェクトに含まれており、バックエンドの属性が情報を返すために`BackendV2.target`属性を参照することが多いというものです。ただし、多くの場合、属性はターゲットが含むことのできる情報のサブセットのみを提供します。たとえば、`backend.coupling_map`は`BackendV2.target`属性でアクセス可能な[`Target`](https://docs.quantum.ibm.com/api/qiskit/qiskit.transpiler.Target)から構築された[`CouplingMap`](https://docs.quantum.ibm.com/api/qiskit/qiskit.transpiler.CouplingMap)を返します。しかし、ターゲットには2量子ビット以上で動作する命令（[`CouplingMap`](https://docs.quantum.ibm.com/api/qiskit/qiskit.transpiler.CouplingMap)では表現できない）が含まれていたり、量子ビットのサブセット（または2量子ビット命令の場合は2量子ビットリンク）でのみ動作する命令が含まれていたりする場合があり、`BackendV2.coupling_map`が返すフルのカップリングマップには詳細が示されないことがあります。したがって、ユースケースによっては、[`BackendV2`](https://docs.quantum.ibm.com/api/qiskit/qiskit.providers.BackendV2)での単純なアクセス以上に深く調べる必要があるかもしれません。

## `BackendV2`における具体的な変更点 {#specific-changes-in-backendv2}

ほとんどの属性には直接の置き換えがあり、移行作業が簡略化されています。インターフェース間で不一致が生じる唯一の点は[`CouplingMap`](https://docs.quantum.ibm.com/api/qiskit/qiskit.transpiler.CouplingMap)です。

以下は、[`BackendV1`](https://docs.quantum.ibm.com/api/qiskit/1.4/qiskit.providers.BackendV1)でのアクセスパターンの例と、[`BackendV2`](https://docs.quantum.ibm.com/api/qiskit/qiskit.providers.BackendV2)での新しい形式の対応表です。

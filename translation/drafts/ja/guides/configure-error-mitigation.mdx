エラー緩和技術により、ユーザーは実行時にデバイスノイズをモデル化することでCircuitのエラーを軽減できます。これは通常、モデル訓練に関連する量子前処理オーバーヘッドと、生成されたモデルを使用して生の結果のエラーを緩和するための古典後処理オーバーヘッドをもたらします。

Estimatorプリミティブは、[TREX](https://docs.quantum.ibm.com/api/qiskit-ibm-runtime/options-resilience-options-v2#measure_mitigation)、[ZNE](https://docs.quantum.ibm.com/api/qiskit-ibm-runtime/options-resilience-options-v2#zne)、[PEC](https://docs.quantum.ibm.com/api/qiskit-ibm-runtime/options-resilience-options-v2#pec)、[PEA](https://docs.quantum.ibm.com/api/qiskit-ibm-runtime/options-zne-options)を含む複数のエラー緩和技術をサポートしています。各技術の説明については[エラー緩和と抑制技術](error-mitigation-and-suppression-techniques)をご覧ください。プリミティブを使用する際は、個別のメソッドをオン/オフできます。詳細は[カスタムエラー設定](#advanced-error)セクションをご覧ください。

<Admonition type="note">

Samplerはエラー緩和をサポートしていませんが、[mthree](https://qiskit.github.io/qiskit-addon-mthree/)（matrix-free measurement mitigation）パッケージを使用してローカルでエラー緩和を実行できます。

</Admonition>

Estimatorは`resilience_level`もサポートしています。レジリエンスレベルは、エラーに対してどの程度の耐性を構築するかを指定します。レベルが高いほど精度の高い結果が生成されますが、処理時間が長くなります。レジリエンスレベルを使用して、プリミティブクエリにエラー緩和を適用する際のコスト/精度のトレードオフを設定できます。エラー緩和は、関連するCircuitのコレクションまたはアンサンブルからの出力を処理することで、結果のエラー（バイアス）を削減します。エラー削減の程度は、適用されるメソッドに依存します。レジリエンスレベルは、エラー緩和メソッドの詳細な選択を抽象化し、ユーザーがアプリケーションに適したコスト/精度のトレードオフについて判断できるようにします。

このため、各レベルは量子サンプリングオーバーヘッドが増加するメソッドに対応しており、さまざまな時間-精度のトレードオフを実験できます。以下の表は、各プリミティブで利用可能なレベルと対応するメソッドを示しています。

<Admonition type="info" title="注意">
エラー緩和はタスク固有であるため、分布のサンプリングか期待値の生成かによって、適用できる技術は異なります。
</Admonition>

<span id="resilience-table"></span>

Estimatorは以下のレジリエンスレベルをサポートしています。Samplerはレジリエンスレベルをサポートしていません。

| Resilience Level | Definition                                                                                            | Technique                                                                 |
|------------------|-------------------------------------------------------------------------------------------------------|---------------------------------------------------------------------------|
| 0                | No mitigation                                                                                         | None                                                                      |
| 1 [Default]      | Minimal mitigation costs: Mitigate error associated with readout errors                               | Twirled Readout Error eXtinction  (TREX) measurement twirling             |
| 2                | Medium mitigation costs. Typically reduces bias in estimators, but is not guaranteed to be zero-bias. | Level 1 + Zero Noise Extrapolation (ZNE) and gate twirling

<Admonition type="info" title="注意">

レジリエンスレベルは現在ベータ版であるため、サンプリングオーバーヘッドとソリューション品質はCircuitごとに異なります。新機能、高度なオプション、管理ツールはローリングベースでリリースされます。各レジリエンスレベルで特定のエラー緩和メソッドが適用されることは保証されていません。
</Admonition>

## レジリエンスレベルによるEstimatorの設定 {#configure-estimator-with-resilience-levels}

レジリエンスレベルを使用してエラー緩和技術を指定するか、[カスタムエラー設定](#advanced-error)で説明されているように個別のカスタム技術を設定できます。

<details>
<summary>レジリエンスレベル0</summary>

ユーザープログラムにエラー緩和は適用されません。

</details>

<details>
<summary>レジリエンスレベル1</summary>

レベル1は、Twirled Readout Error eXtinction（TREX）として知られるモデルフリー技術を適用して、**読み取りエラー緩和**と**測定トワーリング**を実行します。測定の直前にX Gateを通じてQubitをランダムに反転させることで、測定に関連するノイズチャネルを対角化し、測定エラーを削減します。対角ノイズチャネルからのスケーリング項は、ゼロ状態で初期化されたランダムなCircuitをベンチマークすることで学習されます。これにより、サービスは読み取りノイズに起因する期待値のバイアスを除去できます。このアプローチの詳細は[Model-free readout-error mitigation for quantum expectation values](https://arxiv.org/abs/2012.09738)で説明されています。

</details>

<details>
<summary>レジリエンスレベル2</summary>

レベル2は**レベル1に含まれるエラー緩和技術**を適用し、さらに**Gateトワーリング**と**ゼロノイズ外挿法（ZNE）**を使用します。ZNEは、異なるノイズ係数（増幅段階）で観測量の期待値を計算し、測定された期待値を使用してゼロノイズ限界での理想的な期待値を推論します（外挿段階）。このアプローチは期待値のエラーを削減する傾向がありますが、バイアスのない結果が得られることは保証されていません。

![This image shows a graph.  The x-axis is labeled Noise amplification factor.  The y-axis is labeled Expectation value.  An upward sloping line is labeled Mitigated value.  Points near the line are noise-amplified values.  There is a horizontal line just above the X-axis labeled Exact value.](/docs/images/guides/configure-error-mitigation/resilience-2.svg "Illustration of the ZNE method")

このメソッドのオーバーヘッドはノイズ係数の数に応じてスケーリングします。デフォルト設定では3つのノイズ係数で期待値をサンプリングするため、このレジリエンスレベルを使用すると約3倍のオーバーヘッドが発生します。

レベル2では、TREXメソッドが測定の直前にX Gateを通じてQubitをランダムに反転させ、X Gateが適用された場合は対応する測定ビットを反転させます。このアプローチの詳細は[Model-free readout-error mitigation for quantum expectation values](https://arxiv.org/abs/2012.09738)で説明されています。

</details>

### 例 {#example}

`EstimatorV2`インターフェースにより、ユーザーは観測量の期待値のエラーを削減するためのさまざまなエラー緩和メソッドをシームレスに使用できます。以下のコードは、`resilience_level 2`を設定するだけで、ゼロノイズ外挿と読み取りエラー緩和を使用します。

```python
from qiskit_ibm_runtime import QiskitRuntimeService
from qiskit_ibm_runtime import EstimatorV2 as Estimator

service = QiskitRuntimeService()
backend = service.least_busy(operational=True, simulator=False)

estimator = Estimator(backend, options={"resilience_level": 2})
```

## カスタムエラー設定 {#custom-error-settings}

動的デカップリング、GateおよびMeasurementトワーリング、測定エラー緩和、PEC、ZNEなど、個別のエラー緩和および抑制メソッドをオン/オフできます。各メソッドの説明については[エラー緩和と抑制技術](error-mitigation-and-suppression-techniques)をご覧ください。

<Admonition type="note" title="注意">
- すべてのオプションが両方のプリミティブで利用できるわけではありません。利用可能なオプションのリストについては[利用可能なオプション表](runtime-options-overview#options-table)をご覧ください。
- すべてのメソッドがすべてのタイプのCircuitで連携して動作するわけではありません。詳細は[機能互換性表](runtime-options-overview#options-compatibility-table)をご覧ください。
</Admonition>

    ```python
from qiskit_ibm_runtime import QiskitRuntimeService
from qiskit_ibm_runtime import EstimatorV2 as Estimator

service = QiskitRuntimeService()
backend = service.least_busy(operational=True, simulator=False)

estimator = Estimator(backend)
options = estimator.options
# Turn on gate twirling.
options.twirling.enable_gates = True
# Turn on measurement error mitigation.
options.resilience.measure_mitigation = True

print(f">>> gate twirling is turned on: {estimator.options.twirling.enable_gates}")
print(f">>> measurement error mitigation is turned on: {estimator.options.resilience.measure_mitigation}")
```

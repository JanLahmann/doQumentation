エラー緩和技術を使用すると、実行時にデバイスノイズをモデル化することで、Circuit のエラーを緩和できます。これにより、通常はモデルトレーニングに関連する量子前処理オーバーヘッドと、生成されたモデルを使用して生の結果のエラーを緩和するための古典後処理オーバーヘッドが発生します。

Estimator プリミティブは、[TREX](https://docs.quantum.ibm.com/api/qiskit-ibm-runtime/options-resilience-options-v2#measure_mitigation)、[ZNE](https://docs.quantum.ibm.com/api/qiskit-ibm-runtime/options-resilience-options-v2#zne)、[PEC](https://docs.quantum.ibm.com/api/qiskit-ibm-runtime/options-resilience-options-v2#pec)、[PEA](https://docs.quantum.ibm.com/api/qiskit-ibm-runtime/options-zne-options) を含む複数のエラー緩和技術をサポートしています。各技術の説明については[エラー緩和と抑制技術](error-mitigation-and-suppression-techniques)を参照してください。プリミティブを使用する際は、個々のメソッドをオン・オフに切り替えられます。詳細は[カスタムエラー設定](#advanced-error)セクションを参照してください。

<Admonition type="note">

Sampler はエラー緩和をサポートしていませんが、[mthree](https://qiskit.github.io/qiskit-addon-mthree/)（行列なし測定緩和）パッケージを使用してローカルでエラー緩和を実行できます。

</Admonition>

Estimator は `resilience_level` もサポートしています。レジリエンスレベルは、エラーに対してどの程度の耐性を構築するかを指定します。レベルが高いほど精度の高い結果が得られますが、処理時間が長くなります。レジリエンスレベルを使用して、プリミティブクエリにエラー緩和を適用する際のコスト/精度のトレードオフを設定できます。エラー緩和は、関連する Circuit の集合（アンサンブル）の出力を処理することで、結果のエラー（バイアス）を低減します。エラー低減の度合いは適用されるメソッドによって異なります。レジリエンスレベルは、エラー緩和メソッドの詳細な選択を抽象化し、ユーザーがアプリケーションに適したコスト/精度のトレードオフを検討できるようにします。

これを踏まえ、各レベルは量子サンプリングオーバーヘッドが増加するメソッドまたはメソッド群に対応しており、異なる時間・精度のトレードオフを試すことができます。次の表は、各プリミティブで利用可能なレベルと対応するメソッドを示しています。

<Admonition type="info" title="注意">
エラー緩和はタスク固有のため、適用できる技術は分布をサンプリングするか期待値を生成するかによって異なります。
</Admonition>

<span id="resilience-table"></span>

Estimator は以下のレジリエンスレベルをサポートしています。Sampler はレジリエンスレベルをサポートしていません。

| レジリエンスレベル | 定義                                                                                                   | 技術                                                                      |
|--------------------|--------------------------------------------------------------------------------------------------------|---------------------------------------------------------------------------|
| 0                  | 緩和なし                                                                                               | なし                                                                      |
| 1 [デフォルト]     | 最小限の緩和コスト：読み出しエラーに関連するエラーを緩和                                               | Twirled Readout Error eXtinction (TREX) 測定トワーリング                  |
| 2                  | 中程度の緩和コスト。推定器のバイアスを通常低減しますが、ゼロバイアスが保証されるわけではありません。    | レベル1 + Zero Noise Extrapolation (ZNE) とゲートトワーリング

<Admonition type="info" title="注意">

レジリエンスレベルは現在ベータ版のため、サンプリングオーバーヘッドおよびソリューション品質は Circuit ごとに異なる場合があります。新機能、高度なオプション、および管理ツールは随時リリースされる予定です。各レジリエンスレベルで特定のエラー緩和メソッドが適用されることは保証されません。
</Admonition>

## レジリエンスレベルを使用した Estimator の設定 {#configure-estimator-with-resilience-levels}

レジリエンスレベルを使用してエラー緩和技術を指定するか、[カスタムエラー設定](#advanced-error)で説明されているように個別にカスタム技術を設定できます。

<details>
<summary>レジリエンスレベル 0</summary>

ユーザープログラムにはエラー緩和は適用されません。

</details>

<details>
<summary>レジリエンスレベル 1</summary>

レベル1は、Twirled Readout Error eXtinction（TREX）と呼ばれるモデルフリー技術を適用することで、**読み出しエラー緩和**と**測定トワーリング**を適用します。測定直前に X ゲートを通じて Qubit をランダムにフリップすることで、測定に関連するノイズチャネルを対角化し、測定エラーを低減します。対角ノイズチャネルからの再スケーリング項は、ゼロ状態で初期化されたランダム Circuit のベンチマークによって学習されます。これにより、読み出しノイズから生じる期待値のバイアスをサービスが除去できるようになります。このアプローチの詳細は、[Model-free readout-error mitigation for quantum expectation values](https://arxiv.org/abs/2012.09738) で説明されています。

</details>

<details>
<summary>レジリエンスレベル 2</summary>

レベル2は**レベル1に含まれるエラー緩和技術**を適用し、さらに**ゲートトワーリング**と**Zero Noise Extrapolation メソッド（ZNE）**を使用します。ZNE は、異なるノイズ係数に対してオブザーバブルの期待値を計算し（増幅段階）、測定された期待値を使用してゼロノイズ極限での理想的な期待値を推定します（外挿段階）。このアプローチは期待値のエラーを低減する傾向がありますが、バイアスのない結果が保証されるわけではありません。

![この画像はグラフを示しています。X軸はノイズ増幅係数とラベル付けされています。Y軸は期待値とラベル付けされています。上向きの傾きの直線は緩和値とラベル付けされています。直線付近の点はノイズ増幅値です。X軸のすぐ上に水平線があり、正確な値とラベル付けされています。](/docs/images/guides/configure-error-mitigation/resilience-2.svg "ZNEメソッドの図解")

このメソッドのオーバーヘッドはノイズ係数の数に比例してスケールします。デフォルト設定では、3つのノイズ係数で期待値をサンプリングするため、このレジリエンスレベルを使用すると約3倍のオーバーヘッドが発生します。

レベル2では、TREXメソッドが測定直前にXゲートを通じてQubitをランダムにフリップし、Xゲートが適用された場合は対応する測定ビットもフリップします。このアプローチの詳細は、[Model-free readout-error mitigation for quantum expectation values](https://arxiv.org/abs/2012.09738) で説明されています。

</details>

### 例 {#example}

`EstimatorV2` インターフェースを使用すると、さまざまなエラー緩和メソッドをシームレスに活用して、オブザーバブルの期待値のエラーを低減できます。次のコードは、`resilience_level 2` を設定するだけで Zero Noise Extrapolation と読み出しエラー緩和を使用します。

```python
# Added by doQumentation — required packages for this notebook
```

```python
from qiskit_ibm_runtime import QiskitRuntimeService
from qiskit_ibm_runtime import EstimatorV2 as Estimator

service = QiskitRuntimeService()
backend = service.least_busy(operational=True, simulator=False)

# Setting options during primitive initialization
estimator = Estimator(backend, options={"resilience_level": 2})
```

## カスタムエラー設定 {#custom-error-settings}

ダイナミカルデカップリング、ゲートと測定のトワーリング、測定エラー緩和、PEC、ZNEなど、個々のエラー緩和および抑制メソッドをオン・オフに切り替えられます。各技術の説明については[エラー緩和と抑制技術](error-mitigation-and-suppression-techniques)を参照してください。

    ```python
from qiskit_ibm_runtime import QiskitRuntimeService
from qiskit_ibm_runtime import EstimatorV2 as Estimator

service = QiskitRuntimeService()
backend = service.least_busy(operational=True, simulator=False)

estimator = Estimator(backend)
options = estimator.options
# Turn on gate twirling.
options.twirling.enable_gates = True
# Turn on measurement error mitigation.
options.resilience.measure_mitigation = True

print(f">>> gate twirling is turned on: {estimator.options.twirling.enable_gates}")
```

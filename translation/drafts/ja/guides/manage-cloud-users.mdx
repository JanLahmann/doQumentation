# IBM Cloudユーザーの管理

このガイドでは、IBM Cloud&reg;を使用してIBM Cloudアカウントを持つユーザーを有効化する方法を説明し、ユーザーが環境にアクセスするための手順を提供します。

IDプロバイダーのユーザーを管理する場合は、代わりに[IDプロバイダーの管理](/guides/manage-appid)トピックの手順に従ってください。

## 手順 {#steps}

IBM Cloudユーザーを管理するには、以下の手順に従ってください：

1. 招待するユーザーがIBM Cloudアカウントを持っていることを確認します。
2. IAMにアクセスして[ユーザーを招待](https://cloud.ibm.com/iam/users/invite_users)します。
   <Admonition type="note">
   ユーザーは招待を承認し、少なくとも一度ログインするまで管理できません。
   </Admonition>
3. 任意：[IAM](https://cloud.ibm.com/iam/users)からユーザーのプロジェクト割り当てとアクセスグループを変更します。

<span id="user-org"></span>
## ユーザーフロー {#user-flow}

1. ユーザーは招待を承認し、[IBM Cloudポータル](https://cloud.ibm.com/)からログオンします。
2. ユーザーは([管理 → アクセス (IAM) → APIキー](https://cloud.ibm.com/iam/apikeys))にアクセスしてAPIキーを作成します。
3. 詳細については、ユーザーは[Qiskitのインストール](/guides/install-qiskit)を参照できます。

<span id="steps-cloud-org"></span>
## シナリオ例 {#example-scenario}

この例では、以下のセットアップを作成します：

* `ml`と`finance`の2つのプロジェクトがあります。
   * `ml`プロジェクトはサービスインスタンス`QR-ml`および`QR-common`にアクセスできる必要があります。
   * `finance`プロジェクトはサービスインスタンス`QR-finance`および`QR-common`にアクセスできる必要があります。
* 3人のユーザーがいます：
   * FatimaはMLプロジェクトにアクセスできる必要があります。
   * Raviはfinanceプロジェクトにアクセスできる必要があります。
   * Amyraは両方のプロジェクトにアクセスできる必要があります。
* リソースグループなしでアクセスグループを使用します。
* ユーザーはIBM Cloudで定義され、プロジェクトの割り当てもそこで行われます。
* ユーザーはジョブを削除できる必要があります。

このセットアップを実装する手順は以下のとおりです：

2. クラウド管理者は`QR-ml`、`QR finance`、`QR-common`の3つのサービスインスタンスを作成します。
3. クラウド管理者は`quantum-computing.job.delete`アクションを含むカスタムルールを作成します。
1. クラウド管理者は2つのアクセスグループを作成します：
   * `ml`アクセスグループは`QR-ml`および`QR-common`にアクセスできます。
   * `finance`アクセスグループは`QR-finance`および`QR-common`にアクセスできます。
1. クラウド管理者は適切なプロジェクトにクラウドユーザーを招待します。具体的には、プロジェクトを含むアクセスグループにユーザーを招待して割り当てます。
   * Fatimaは「ml」アクセスグループに追加されます。
   * Raviは「finance」アクセスグループに追加されます。
   * Amyraは「ml」と「finance」の両方のアクセスグループに追加されます。
1. ユーザーはIBM Cloudポータルからログインし、APIキーを作成して、プロジェクトのサービスインスタンスで作業できます。

Qiskitユーザーのより広いコミュニティがカスタムトランスパイルコードを開発・再利用しやすくするため、Qiskit SDKはプラグインインターフェースをサポートしています。これにより、サードパーティのPythonパッケージがQiskit経由でアクセス可能な拡張トランスパイル機能を提供することを宣言できます。

現在、サードパーティプラグインは以下の3つの方法で拡張トランスパイル機能を提供できます。

- [トランスパイラーステージプラグイン](https://docs.quantum.ibm.com/api/qiskit/transpiler_plugins)は、プリセットステージドパスマネージャーの[6つのステージ](transpiler-stages)（`init`、`layout`、`routing`、`translation`、`optimization`、`scheduling`）のいずれかに代わって使用できるパスマネージャーを提供します。
- [ユニタリー合成プラグイン](https://docs.quantum.ibm.com/api/qiskit/qiskit.transpiler.passes.synthesis.plugin.UnitarySynthesisPlugin)は、ユニタリーゲート合成の拡張機能を提供します。
- [高レベル合成プラグイン](https://docs.quantum.ibm.com/api/qiskit/qiskit.transpiler.passes.synthesis.plugin.HighLevelSynthesisPlugin)は、線形関数やClifford演算子などの「高レベルオブジェクト」を合成するための拡張機能を提供します。高レベルオブジェクトは、[Operation](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.Operation)クラスのサブクラスで表現されます。

このページの残りの部分では、利用可能なプラグインの一覧表示、新しいプラグインのインストール、およびその使用方法について説明します。

## 利用可能なプラグインの一覧と新規インストール {#list-available-plugins-and-install-new-ones}

Qiskitにはすでにトランスパイル用の組み込みプラグインがいくつか含まれています。さらにインストールするには、Pythonのパッケージマネージャーを使用します。たとえば、`pip install qiskit-toqm`を実行して[Qiskit TOQM](https://github.com/qiskit-toqm/qiskit-toqm)ルーティングステージプラグインをインストールできます。多数のサードパーティプラグインが[Qiskitエコシステム](https://qiskit.github.io/ecosystem/#transpiler_plugin)の一部となっています。

### 利用可能なトランスパイラーステージプラグインの一覧 {#list-available-transpiler-stage-plugins}

[list_stage_plugins](https://docs.quantum.ibm.com/api/qiskit/transpiler_plugins#qiskit.transpiler.preset_passmanagers.plugin.list_stage_plugins)関数を使用し、一覧表示したいステージの名前を引数として渡します。

from qiskit.transpiler.preset_passmanagers.plugin import list_stage_plugins

```

```

`qiskit-toqm`がインストールされている場合、`toqm`が`routing`プラグインの一覧に表示されます。

### 利用可能なユニタリー合成プラグインの一覧 {#list-available-unitary-synthesis-plugins}

[unitary_synthesis_plugin_names](https://docs.quantum.ibm.com/api/qiskit/qiskit.transpiler.passes.synthesis.plugin.unitary_synthesis_plugin_names)関数を使用します。

unitary_synthesis_plugin_names()

### 利用可能な高レベル合成プラグインの一覧 {#list-available-high-level-synthesis-plugins}

[high_level_synthesis_plugin_names](https://docs.quantum.ibm.com/api/qiskit/qiskit.transpiler.passes.synthesis.plugin.high_level_synthesis_plugin_names)関数を使用し、合成する「高レベルオブジェクト」の型名を引数として渡します。この名前は、合成対象のオブジェクト型を表す[Operation](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.Operation)クラスの[`name`](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.Operation#name)属性に対応します。

## プラグインの使用 {#use-a-plugin}

このセクションでは、トランスパイラープラグインの使用方法を説明します。コード例ではQiskitに付属のプラグインを使用していますが、サードパーティパッケージからインストールしたプラグインも同じ方法で使用できます。

### トランスパイラーステージプラグインの使用 {#use-a-transpiler-stage-plugin}

トランスパイラーステージプラグインを使用するには、[`generate_preset_pass_manager`](https://docs.quantum.ibm.com/api/qiskit/qiskit.transpiler.generate_preset_pass_manager#qiskit.transpiler.generate_preset_pass_manager)または[`transpile`](https://docs.quantum.ibm.com/api/qiskit/compiler#qiskit.compiler.transpile)の適切な引数にプラグイン名を指定します。引数名はトランスパイルステージ名に`_method`を付加した形式になります。たとえば、`lookahead`ルーティングプラグインを使用するには、`routing_method`引数に`lookahead`を指定します。

```python
from qiskit.transpiler import generate_preset_pass_manager
from qiskit_ibm_runtime import QiskitRuntimeService

service = QiskitRuntimeService()
backend = service.backend("ibm_fez")

pass_manager = generate_preset_pass_manager(
    optimization_level=3, backend=backend, routing_method="lookahead"
)
```

### ユニタリー合成プラグインの使用 {#use-a-unitary-synthesis-plugin}

ユニタリー合成プラグインを使用するには、[`generate_preset_pass_manager`](https://docs.quantum.ibm.com/api/qiskit/qiskit.transpiler.generate_preset_pass_manager#qiskit.transpiler.generate_preset_pass_manager)または[`transpile`](https://docs.quantum.ibm.com/api/qiskit/compiler#qiskit.compiler.transpile)の`unitary_synthesis_method`引数にプラグイン名を指定します。

```python
pass_manager = generate_preset_pass_manager(
    optimization_level=3,
    backend=backend,
    unitary_synthesis_method="sk",
    unitary_synthesis_plugin_config=dict(
        basis_gates=["cz", "id", "rz", "sx", "x"]
    ),
)
```

ユニタリー合成は、[`generate_preset_pass_manager`](https://docs.quantum.ibm.com/api/qiskit/qiskit.transpiler.generate_preset_pass_manager#qiskit.transpiler.generate_preset_pass_manager)が返す、または[`transpile`](https://docs.quantum.ibm.com/api/qiskit/compiler#qiskit.compiler.transpile)で使用されるステージドパスマネージャーの`init`、`translation`、`optimization`ステージで使用されます。これらのステージの説明については、[トランスパイラーステージ](transpiler-stages)を参照してください。

ユニタリー合成メソッドのオプションを渡すには、自由形式の辞書である`unitary_synthesis_plugin_config`引数を使用します。合成メソッドのドキュメントに、サポートされるオプションが記載されています。組み込みユニタリー合成プラグインのドキュメントへのリンクは[こちらのリスト](https://docs.quantum.ibm.com/api/qiskit/transpiler_synthesis_plugins#unitary-synthesis-plugins-2)を参照してください。

### 高レベル合成プラグインの使用 {#use-a-high-level-synthesis-plugin}

まず、さまざまな高レベルオブジェクトに使用するプラグイン名を格納するために[HLSConfig](https://docs.quantum.ibm.com/api/qiskit/qiskit.transpiler.passes.HLSConfig)を作成します。たとえば次のようにします。

```python
from qiskit.transpiler.passes import HLSConfig

hls_config = HLSConfig(clifford=["layers"], linear_function=["pmh"])
```

このコードセルは、[Clifford](https://docs.quantum.ibm.com/api/qiskit/qiskit.quantum_info.Clifford#clifford)オブジェクトの合成に`layers`プラグインを使用し、[LinearFunction](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.library.LinearFunction#linearfunction)オブジェクトの合成に`pmh`プラグインを使用する高レベル合成設定を作成します。キーワード引数の名前は、合成対象のオブジェクト型を表す[Operation](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.Operation)クラスの[`name`](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.Operation#name)属性に対応します。各高レベルオブジェクトについて、指定されたプラグインのリストがいずれかが成功するまで順番に試行されます（上記の例では、各リストにプラグインが1つだけ含まれています）。

プラグインを名前で指定する代わりに、`(name, options)`タプルを渡すこともできます。タプルの2番目の要素はプラグインのオプションを含む辞書です。合成メソッドのドキュメントに、サポートされるオプションが記載されています。組み込み高レベル合成プラグインのドキュメントへのリンクは[こちらのリスト](https://docs.quantum.ibm.com/api/qiskit/transpiler_synthesis_plugins#high-level-synthesis-plugins-2)を参照してください。

`HLSConfig`オブジェクトを作成したら、[`generate_preset_pass_manager`](https://docs.quantum.ibm.com/api/qiskit/qiskit.transpiler.generate_preset_pass_manager#qiskit.transpiler.generate_preset_pass_manager)または[`transpile`](https://docs.quantum.ibm.com/api/qiskit/compiler#qiskit.compiler.transpile)の`hls_config`引数として渡します。

```python
pass_manager = generate_preset_pass_manager(
    optimization_level=3, backend=backend, hls_config=hls_config
)
```

高レベル合成は、[`generate_preset_pass_manager`](https://docs.quantum.ibm.com/api/qiskit/qiskit.transpiler.generate_preset_pass_manager#qiskit.transpiler.generate_preset_pass_manager)が返す、または[`transpile`](https://docs.quantum.ibm.com/api/qiskit/compiler#qiskit.compiler.transpile)で使用されるステージドパスマネージャーの`init`、`translation`、`optimization`ステージで使用されます。これらのステージの説明については、[トランスパイラーステージ](transpiler-stages)を参照してください。

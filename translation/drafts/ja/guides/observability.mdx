# Qiskit Runtimeのアクティビティー追跡イベント {#activity-tracking-events-for-qiskit-runtime}

IBM Cloud&reg;のサービス（Qiskit Runtimeなど）は、アクティビティー追跡イベントを生成します。

アクティビティー追跡イベントは、IBM Cloud上のサービスの状態を変化させるアクティビティーを記録します。このイベントを使用して、異常なアクティビティーや重要なアクションを調査したり、規制上の監査要件に準拠したりすることができます。

IBM Cloud Activity Tracker Event Routing（プラットフォームサービス）を使用すると、ターゲットとルートを設定することで、アカウント内の監査イベントを任意の送信先にルーティングできます。詳細については、[IBM Cloud Activity Tracker Events Routingについて](https://cloud.ibm.com/docs/atracker?topic=atracker-about)をご参照ください。

IBM Cloud Logsを使用すると、アカウント内で生成され、IBM Cloud Activity Tracker Event RoutingによってIBM Cloud LogsインスタンスにルーティングされたイベントをIBM Cloud Logsインスタンスで可視化し、アラートを設定することができます。

## アクティビティー追跡イベントが生成される場所 {#locations-where-activity-tracking-events-are-generated}

Qiskit Runtimeは、以下のリージョンでアクティビティー追跡イベントを送信します。

- ワシントンD.C.（"us-east"）
- フランクフルト（"eu-de"）

## Qiskit Runtimeのアクティビティー追跡イベントの表示 {#viewing-activity-tracking-events-for-qiskit-runtime}

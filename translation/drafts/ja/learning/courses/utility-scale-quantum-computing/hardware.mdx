## 2. バックエンドとターゲット {#2-backend-and-target}

Qiskitは、量子デバイスに関する静的・動的な情報を取得するためのAPIを提供しています。デバイスとのインタフェースには`Backend`インスタンスを使用します。`Backend`インスタンスには`Target`インスタンスが含まれており、これはISA（命令セットアーキテクチャ）やその他の特性・制約といった重要な機能をまとめた抽象的なマシンモデルです。
これらのバックエンドインスタンスを使って、IBM Quantum&reg; Platformの[コンピュートリソース](https://quantum.cloud.ibm.com/computers)ページに表示されている情報の一部を取得してみましょう。まず、対象デバイスのバックエンドインスタンスを作成します。以下では、"ibm_kyoto"、"ibm_kawasaki"、またはビジー状態が最も少ないEagleマシンを選択します。QPUへのアクセス権は人によって異なりますので、バックエンド名を適宜変更してください。

```python
service = QiskitRuntimeService()
# backend = service.backend("ibm_kawasaki") # an Eagle, if you have access to ibm_kawasaki
backend = service.least_busy(
    operational=True, simulator=False, min_num_qubits=127
)  # Eagle
backend.name
```

```text
'ibm_strasbourg'
```

デバイスの基本的な（静的な）情報から確認していきましょう。

```python
print(
    f"""
{backend.name}, {backend.num_qubits} qubits
processor type = {backend.processor_type}
basis gates = {backend.basis_gates}
"""
)
```

```text
ibm_strasbourg, 127 qubits
processor type = {'family': 'Eagle', 'revision': 3}
basis gates = ['ecr', 'id', 'rz', 'sx', 'x']
```

### 3.2 ゲートエラーと読み出しエラー {#32-gate-and-readout-errors}

次に、ゲートエラーについて見ていきます。まず、`target`インスタンスのデータ構造を調べます。これは操作名をキーとする辞書です。

```python
target = backend.target
target.keys()
```

```text
dict_keys(['measure', 'id', 'sx', 'delay', 'x', 'for_loop', 'rz', 'if_else', 'ecr', 'reset', 'switch_case'])
```

値もまた辞書になっています。'sx'操作に対する値（辞書）のいくつかの項目を見てみましょう。

```python
for i, qq in enumerate(target["sx"]):
    if i >= 5:
        break
    print(i, qq, target["sx"][qq])
```


ご覧のとおり、読み出しエラーは2量子ビット操作のエラーより大きく、2量子ビット操作のエラーは1量子ビット操作のエラーより大きい傾向があります。

データ構造を理解したところで、'sx'ゲートと'ecr'ゲートの中央値エラーを計算してみましょう。こちらも[IBM Quantum Platform](https://quantum.cloud.ibm.com/)のデバイスページに表示されている値と比較してみてください。

```python
sx_errors = [inst_prop.error for inst_prop in target["sx"].values()]
f"Median SX error: {(statistics.median(sx_errors)):.3e}"
```

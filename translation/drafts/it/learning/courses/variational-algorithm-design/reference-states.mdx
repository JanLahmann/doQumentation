## Stato predefinito {#default-state}

Uno *stato di riferimento* indica il punto di partenza fisso e iniziale per il nostro problema. Per preparare uno stato di riferimento, dobbiamo applicare l'unitario non parametrizzato appropriato $U_R$ all'inizio del nostro Circuit quantistico, in modo che $|\rho\rangle = U_R |0\rangle$. Se hai un'ipotesi ragionata o un valore di partenza proveniente da una soluzione ottimale già nota, l'algoritmo variazionale convergerà probabilmente più in fretta se lo usi come punto di partenza.

Lo stato di riferimento più semplice possibile è lo stato predefinito, in cui si utilizza lo stato iniziale di un Circuit quantistico a $n$ Qubit: $|0\rangle^{\otimes n}$. Per lo stato predefinito, l'operatore unitario è $U_R \equiv I$. Grazie alla sua semplicità, lo stato predefinito è uno stato di riferimento valido e utilizzato in molti scenari.

## Stato di riferimento classico {#classical-reference-state}

Supponiamo di avere un sistema a tre Qubit e di voler partire dallo stato $|001\rangle$ invece dello stato predefinito $|000\rangle$. Questo è un esempio di stato di riferimento puramente classico: per costruirlo, è sufficiente applicare un [Gate X](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.library.XGate) al Qubit $0$ (seguendo l'ordinamento dei Qubit di Qiskit), poiché $|001\rangle = X_0 |000\rangle$.

In questo caso, l'operatore unitario è $U_R \equiv X_0$, che produce lo stato di riferimento $|\rho\rangle \equiv |001\rangle$.

## Stato di riferimento quantistico {#quantum-reference-state}

Supponiamo di voler partire da uno stato più complesso che coinvolga sovrapposizione e/o entanglement, come $\frac{1}{\sqrt{2}}(|100\rangle+|111\rangle)$.

Per ottenere questo stato a partire da $|000\rangle$, un approccio possibile è applicare un [Gate di Hadamard](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.library.HGate) al Qubit $0$ ($H_0$), un Gate [CNOT (CX)](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.library.CXGate) con il Qubit $0$ come qubit di controllo e il Qubit $1$ come qubit target ($CNOT_{01}$), e infine un Gate $X$ applicato al Qubit $2$ ($X_2$).

In questo scenario, l'operatore unitario è $U_{R} \equiv X_2CNOT_{01}H_0|000\rangle$ e lo stato di riferimento è $|\rho\rangle \equiv \frac{1}{\sqrt{2}}(|100\rangle+|111\rangle)$.

```python
qc = QuantumCircuit(3)
qc.h(0)
qc.cx(0, 1)
qc.x(2)

qc.draw("mpl")
```

## Costruire stati di riferimento con Circuit template {#constructing-reference-states-using-template-circuits}

Possiamo anche usare vari Circuit template, come [`TwoLocal`](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.library.TwoLocal), che permette di esprimere con facilità molteplici parametri regolabili ed entanglement. Tratteremo questi Circuit template in modo più dettagliato nella prossima lezione, ma possiamo usarli per i nostri stati di riferimento *a condizione di* fissare i parametri:

```python
from qiskit.circuit.library import TwoLocal
from math import pi

reference_circuit = TwoLocal(2, "rx", "cz", entanglement="linear", reps=1)
theta_list = [pi / 2, pi / 3, pi / 3, pi / 2]

reference_circuit = reference_circuit.assign_parameters(theta_list)

reference_circuit.decompose().draw("mpl")
```

## Stati di riferimento specifici per applicazione {#application-specific-reference-states}

### Machine learning quantistico {#quantum-machine-learning}

Nel contesto di un classificatore quantistico variazionale (VQC), i dati di addestramento vengono codificati in uno stato quantistico tramite un Circuit parametrizzato noto come *feature map*, in cui ogni valore di parametro rappresenta un punto del dataset di addestramento. La `zz_feature_map` è un tipo di Circuit parametrizzato che può essere utilizzato per passare i nostri punti dati ($x$) a questa feature map.

```python
from qiskit.circuit.library import zz_feature_map

data = [0.1, 0.2]

zz_feature_map_reference = zz_feature_map(feature_dimension=2, reps=2)
zz_feature_map_reference = zz_feature_map_reference.assign_parameters(data)
zz_feature_map_reference.decompose().draw("mpl")
```

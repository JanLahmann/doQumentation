# Installare il Qiskit SDK e Qiskit Runtime dal codice sorgente

Installare il Qiskit SDK dal codice sorgente ti permette di accedere alla versione di sviluppo corrente, invece di utilizzare la versione disponibile nel repository Python Package Index (PyPI). Questo ti consente di ispezionare ed estendere l'ultima versione del codice Qiskit in modo più efficiente.

## Creare e attivare un nuovo ambiente virtuale {#create-and-activate-a-new-virtual-environment}

1. Naviga nella directory del tuo progetto e crea un ambiente minimale con solo Python installato.

        <OperatingSystemTabs>
          <TabItem value="mac" label="macOS">
            ```shell
            python3 -m venv .venv
            ```
          </TabItem>

          <TabItem value="linux" label="Linux">
            ```shell
            python3 -m venv .venv
            ```
          </TabItem>

          <TabItem value="win" label="Windows">
            ```text
            python -m venv .venv
            ```
          </TabItem>
         </OperatingSystemTabs>

1. Attiva il nuovo ambiente.

         <OperatingSystemTabs>
           <TabItem value="mac" label="macOS">
             ```shell

             ```

           <TabItem value="linux" label="Linux">

## Installare il compilatore Rust {#install-the-rust-compiler}

Sul tuo sistema deve essere installato un compilatore Rust per compilare Qiskit. Per installarlo, usa il programma di installazione multipiattaforma [`rustup`](https://rustup.rs/) oppure un [altro metodo di installazione.](https://forge.rust-lang.org/infra/other-installation-methods.html)

## Installare Qiskit {#install-qiskit}

Segui questi passaggi per installare Qiskit:

1. Clona il repository di Qiskit.

```bash
git clone https://github.com/Qiskit/qiskit.git
```

2. Spostati nella directory `qiskit`.

```bash
cd qiskit
```

3. (Facoltativo) Se vuoi eseguire test o controlli di linting, installa i requisiti per sviluppatori.

```bash
pip install -r requirements-dev.txt
```

4. Installa `qiskit`.

  * **Installazione standard**:

    ```bash
    pip install .
    ```

  * **Modalità modificabile**: In questa modalità non è necessario reinstallare Qiskit ogni volta che vengono apportate modifiche al codice del progetto.

    ```bash
    pip install -e .
    ```

    In modalità modificabile, le estensioni compilate vengono costruite in _modalità debug_ senza ottimizzazioni. Questo influisce sulle prestazioni a runtime del codice compilato. Per costruire le estensioni compilate con le ottimizzazioni attive, esegui il comando seguente per ricompilare il binario in _modalità release_:

    ```bash
    python setup.py build_rust --release --inplace
    ```

    <Admonition type="note">
    Se stai lavorando sul codice Rust in Qiskit, devi ricompilare il codice dell'estensione ogni volta che apporti una modifica locale. In modalità modificabile, l'estensione Rust viene compilata solo quando viene eseguito il comando di installazione, quindi le modifiche locali al codice Rust non si riflettono nel pacchetto installato a meno che tu non ricompili l'estensione rieseguendo `build_rust` (con o senza `--release`, a seconda che tu voglia compilare in modalità release o debug).
    </Admonition>

## Installare Qiskit Runtime {#install-qiskit-runtime}

Segui questi passaggi se vuoi installare Qiskit Runtime:

1. Clona il repository di Qiskit Runtime.

```bash
git clone https://github.com/Qiskit/qiskit-ibm-runtime.git
```

2. Spostati nella directory `qiskit-ibm-runtime`.

```bash
cd qiskit-ibm-runtime
```

3. Installa `qiskit-runtime`. Ti consigliamo di usare un [ambiente virtuale](https://docs.python.org/3/library/venv.html) per evitare di inquinare l'installazione globale di Python.

  * **Installazione standard**:

    ```bash
    pip install .
    ```

  * **Modalità modificabile**: In questa modalità non è necessario reinstallare Qiskit ogni volta che vengono apportate modifiche al codice del progetto.

    ```bash
    pip install -e .
    ```

    In modalità modificabile, le estensioni compilate vengono costruite in _modalità debug_ senza ottimizzazioni.

4. (Facoltativo) Se vuoi eseguire test o controlli di linting, installa i requisiti per sviluppatori. Ti consigliamo di usare un [ambiente virtuale](https://docs.python.org/3/library/venv.html) per evitare di inquinare l'installazione globale di Python.

```bash
pip install -e ".[dev]"
```

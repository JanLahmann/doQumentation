# Eventi di monitoraggio attivitĂ  per Qiskit Runtime {#activity-tracking-events-for-qiskit-runtime}

I servizi IBM Cloud&reg;, come Qiskit Runtime, generano eventi di monitoraggio attivitĂ .

Gli eventi di monitoraggio attivitĂ  registrano le operazioni che modificano lo stato di un servizio in IBM Cloud. Puoi usare gli eventi per investigare attivitĂ  anomale e azioni critiche, e per soddisfare i requisiti di audit normativi.

Puoi usare IBM Cloud Activity Tracker Event Routing, un servizio di piattaforma, per instradare gli eventi di auditing nel tuo account verso destinazioni a tua scelta, configurando target e route che definiscono dove vengono inviati gli eventi di monitoraggio attivitĂ . Per ulteriori informazioni, consulta [Informazioni su IBM Cloud Activity Tracker Events Routing.](https://cloud.ibm.com/docs/atracker?topic=atracker-about)

Puoi usare IBM Cloud Logs per visualizzare e ricevere avvisi sugli eventi generati nel tuo account e instradati da IBM Cloud Activity Tracker Event Routing a un'istanza di IBM Cloud Logs.

## Posizioni in cui vengono generati gli eventi di monitoraggio attivitĂ  {#locations-where-activity-tracking-events-are-generated}

Qiskit Runtime invia eventi di monitoraggio attivitĂ  nelle seguenti regioni:

- Washington ("us-east")
- Francoforte ("eu-de")

## Visualizzazione degli eventi di monitoraggio attivitĂ  per Qiskit Runtime {#viewing-activity-tracking-events-for-qiskit-runtime}

Puoi usare IBM Cloud Logs per visualizzare e ricevere avvisi sugli eventi generati nel tuo account e instradati da IBM Cloud Activity Tracker Event Routing a un'istanza di IBM Cloud Logs.

# Logging per Qiskit Runtime {#logging-for-qiskit-runtime}

I servizi IBM Cloud&reg;, come Qiskit Runtime, generano log di piattaforma che puoi usare per analizzare attività anomale e azioni critiche nel tuo account, e per risolvere eventuali problemi.

Puoi utilizzare [IBM Cloud Logs Routing,](https://cloud.ibm.com/docs/logs-router?topic=logs-router-about) un servizio di piattaforma, per instradare i log di piattaforma del tuo account verso una destinazione a tua scelta, configurando un tenant che definisce dove vengono inviati i log. Per ulteriori informazioni, consulta [Informazioni su Logs Routing.](https://cloud.ibm.com/docs/logs-router?topic=logs-router-about)

Puoi usare [IBM Cloud Logs](https://cloud.ibm.com/docs/cloud-logs) per visualizzare i log di piattaforma generati nel tuo account e instradati da IBM Cloud Logs Routing verso un'istanza IBM Cloud Logs, nonché per configurare avvisi su di essi.

## Posizioni in cui vengono generati i log di piattaforma {#locations-where-platform-logs-are-generated}

IBM Cloud Container Registry genera log di piattaforma visualizzati nelle tue istanze di logging. La tabella seguente elenca le posizioni in cui è abilitata la raccolta automatica dei log del servizio Container Registry.

| Posizione                     | Log dei servizi di piattaforma disponibili? |
|-------------------------------|---------------------------------------------|
| Washington, D.C. (us-east)   |                    Sì                       |
| Francoforte (eu-de)           |                    Sì                       |

## Log di piattaforma generati {#platform-logs-that-are-generated}

IBM Qiskit Runtime genera log dei servizi di piattaforma visualizzati nelle tue istanze di logging nei seguenti casi specifici:

- Quando lo stato di un job cambia.

## Visualizzare i log dalla pagina Observability {#viewing-logs-from-the-observability-page}

Per informazioni su come avviare l'interfaccia IBM Cloud Logs, consulta **Launching the UI** nella [documentazione di IBM Cloud Logs.](https://cloud.ibm.com/docs/cloud-logs?topic=cloud-logs-instance-launch)

## Campi per tipo di log {#fields-by-log-type}

Per informazioni sui campi inclusi in ogni log di piattaforma, consulta [Campi inclusi nei log di piattaforma.](https://cloud.ibm.com/docs/logs-router?topic=logs-router-about-platform-logs#about-platform-logs-2)

I seguenti campi sono inclusi nel record di log.

| Campo           | Tipo       | Descrizione                                                                           |
|-----------------|------------|---------------------------------------------------------------------------------------|
| logSourceCRN    | Obbligatorio | Definisce l'account e l'istanza di flow log in cui il log viene pubblicato.          |
| saveServiceCopy | Obbligatorio | Definisce se IBM salva una copia del record per scopi operativi.                     |
| message         | Obbligatorio | Descrizione del log generato.                                                        |

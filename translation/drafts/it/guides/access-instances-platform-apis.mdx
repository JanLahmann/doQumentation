# Usa le API della piattaforma IBM Cloud per accedere alle istanze

Questa guida è rivolta a chi utilizza le API della piattaforma IBM Cloud per eseguire il provisioning, aggiornare o gestire l'accesso alle istanze. Fornisce dettagli sui [parametri](#parameters) che puoi impostare durante il provisioning di un'istanza, nonché l'insieme dei [campi di estensione](#extensions) restituiti quando accedi a un'istanza.

<Admonition type="note">
Consulta la documentazione dell'API ResourceController di IBM Cloud&reg; per ulteriori informazioni.

Alcune pagine pertinenti:
- [Crea (provisioning) una nuova istanza di risorsa](https://cloud.ibm.com/apidocs/resource-controller/resource-controller#create-resource-instance)
- [Aggiorna un'istanza di risorsa](https://cloud.ibm.com/apidocs/resource-controller/resource-controller#update-resource-instance)
- [Ottieni un'istanza di risorsa](https://cloud.ibm.com/apidocs/resource-controller/resource-controller#get-resource-instance)
- [Ottieni un elenco di tutte le istanze di risorsa](https://cloud.ibm.com/apidocs/resource-controller/resource-controller#list-resource-instances)

    Quando recuperi l'elenco di tutte le istanze di risorsa, tieni presente che puoi filtrare per ottenere solo le istanze Qiskit Runtime con `resource_id = "b6049020-80f4-11eb-a0f7-e35ec9b4054f"`

</Admonition>

## `parameters` {#parameters}

Nel corpo della richiesta durante il provisioning o l'aggiornamento di un'istanza:

- **instance_limit_seconds** (stringa o null, `^[0-9]+$`): Il limite da impostare sull'istanza. Per il piano Open: se omesso, il valore predefinito è 10 minuti.
.
- **usage_allocation_seconds** (stringa o null, `^[0-9]+$`): Tempo allocato per l'istanza, utilizzato dallo scheduler fair-share per determinare la priorità nella coda in base all'utilizzo di tutti i QPU. (Non si applica alle istanze Pay-As-You-Go.)
- **backends** (array di liste di Backend (stringhe o null) oppure array con "ANY"): l'elenco consentito dei nomi di Backend disponibili per questa istanza. Valore predefinito: `["ANY"]`, significa che tutti i Backend disponibili nel piano sono accessibili. `[]` significa nessun Backend.

## `extensions` {#extensions}

Nella risposta durante il provisioning o quando si recuperano i dettagli di un'istanza:

Proprietà aggiuntive dell'istanza fornite dal servizio, rappresentate come coppie chiave-valore. Queste proprietà persistono con l'istanza sotto la proprietà extensions.

- **instance_limit_seconds** (intero o null, `>= 0`): Il limite di tempo da impostare sull'istanza.
- **usage_allocation_seconds** (intero o null, `>= 0`): Tempo allocato per l'istanza, utilizzato dallo scheduler fair-share per determinare la priorità nella coda in base all'utilizzo di tutti i QPU.
- **backends** (Obbligatorio; array di liste di Backend (stringhe) oppure array con "ANY" (stringhe)): L'elenco consentito dei nomi di Backend disponibili per questa istanza. Valore predefinito: `["ANY"]`, significa che tutti i Backend del piano sono disponibili. `[]` significa nessun Backend.

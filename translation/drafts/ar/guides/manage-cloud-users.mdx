# إدارة مستخدمي IBM Cloud

يوضح هذا الدليل كيفية استخدام IBM Cloud&reg; لتفعيل المستخدمين الذين لديهم حسابات IBM Cloud، ويقدم تعليمات للمستخدمين للوصول إلى البيئة.

لإدارة مستخدمي موفر الهوية (ID provider) بدلاً من ذلك، اتبع التعليمات الموجودة في موضوع [إدارة موفري الهوية](/guides/manage-appid).

## الخطوات {#steps}

اتبع هذه الخطوات الرئيسية لإدارة مستخدمي IBM Cloud:

1. تأكد من أن المستخدمين الذين تريد دعوتهم لديهم حسابات IBM Cloud.
2. انتقل إلى IAM و[ادعُ المستخدمين](https://cloud.ibm.com/iam/users/invite_users)
   <Admonition type="note">
   لا يمكن إدارة المستخدمين حتى يقبلوا الدعوة ويسجلوا دخولهم مرة واحدة على الأقل.
   </Admonition>
3. اختياري: عدّل تعيينات المشاريع ومجموعات الوصول الخاصة بالمستخدمين من [IAM.](https://cloud.ibm.com/iam/users)

<span id="user-org"></span>
## مسار المستخدم {#user-flow}

1. يقبل المستخدم الدعوة، ثم يسجل دخوله عبر [بوابة IBM Cloud](https://cloud.ibm.com/).
2. ينشئ المستخدم مفتاح API بالذهاب إلى ([Manage → Access (IAM) → API keys](https://cloud.ibm.com/iam/apikeys)).
3. للاطلاع على مزيد من المعلومات، يمكن للمستخدمين مراجعة [تثبيت Qiskit.](/guides/install-qiskit)

<span id="steps-cloud-org"></span>
## مثال توضيحي {#example-scenario}

يُنشئ هذا المثال الإعداد التالي:

* يوجد مشروعان، `ml` و`finance`.
   * يجب أن يتمتع مشروع `ml` بالوصول إلى نسختَي الخدمة `QR-ml` و`QR-common`.
   * يجب أن يتمتع مشروع `finance` بالوصول إلى نسختَي الخدمة `QR-finance` و`QR-common`.
* يوجد ثلاثة مستخدمين:
   * يجب أن تتمتع فاطمة بالوصول إلى مشروع `ml`.
   * يجب أن يتمتع رافي بالوصول إلى مشروع `finance`.
   * يجب أن تتمتع أميرة بالوصول إلى كلا المشروعين.
* يستخدم مجموعات الوصول دون مجموعات الموارد.
* يُعرَّف المستخدمون في IBM Cloud وتُنجَز تعيينات المشاريع هناك أيضاً.
* يجب أن يتمكن المستخدمون من حذف الوظائف.

خطوات تطبيق هذا الإعداد هي:

2. يُنشئ مسؤول Cloud ثلاث نسخ من الخدمة: `QR-ml` و`QR finance` و`QR-common`.
3. يُنشئ مسؤول Cloud قاعدة مخصصة تتضمن الإجراء `quantum-computing.job.delete`.
1. يُنشئ مسؤول Cloud مجموعتَي وصول:
   * تستطيع مجموعة الوصول `ml` الوصول إلى `QR-ml` و`QR-common`.
   * تستطيع مجموعة الوصول `finance` الوصول إلى `QR-finance` و`QR-common`.
1. يدعو مسؤول Cloud مستخدمي السحابة إلى المشروع المناسب. تحديداً، يدعوهم ويعيّنهم إلى مجموعة وصول تشمل المشروع.
   * تُضاف فاطمة إلى مجموعة الوصول "ml".
   * يُضاف رافي إلى مجموعة الوصول "finance".
   * تُضاف أميرة إلى مجموعتَي الوصول "ml" و"finance" معاً.
1. يمكن للمستخدمين تسجيل الدخول عبر بوابة IBM Cloud، وإنشاء مفاتيح API، والعمل مع نسخ الخدمة الخاصة بمشاريعهم.

# أحداث تتبع النشاط لـ Qiskit Runtime

تُولّد خدمات IBM Cloud&reg;، مثل Qiskit Runtime، أحداث تتبع النشاط.

تُسجّل أحداث تتبع النشاط الأنشطة التي تُغيّر حالة الخدمة في IBM Cloud. يمكنك استخدام هذه الأحداث للتحقيق في الأنشطة غير الاعتيادية والإجراءات الحرجة، والامتثال لمتطلبات التدقيق التنظيمي.

يمكنك استخدام IBM Cloud Activity Tracker Event Routing، وهي خدمة منصة، لتوجيه أحداث التدقيق في حسابك إلى وجهات تختارها من خلال تكوين أهداف ومسارات تُحدد إلى أين تُرسَل أحداث تتبع النشاط. لمزيد من المعلومات، راجع [About IBM Cloud Activity Tracker Events Routing.](https://cloud.ibm.com/docs/atracker?topic=atracker-about)

يمكنك استخدام IBM Cloud Logs لتصوير الأحداث وإنشاء تنبيهات بشأنها، تلك الأحداث التي تُولَّد في حسابك ويُوجّهها IBM Cloud Activity Tracker Event Routing إلى نسخة IBM Cloud Logs.

## المواقع التي تُولَّد فيها أحداث تتبع النشاط {#locations-where-activity-tracking-events-are-generated}

يُرسل Qiskit Runtime أحداث تتبع النشاط في المناطق التالية:

- واشنطن ("us-east")
- فرانكفورت ("eu-de")

## عرض أحداث تتبع النشاط لـ Qiskit Runtime {#viewing-activity-tracking-events-for-qiskit-runtime}

يمكنك استخدام IBM Cloud Logs لتصوير الأحداث وإنشاء تنبيهات بشأنها، تلك الأحداث التي تُولَّد في حسابك ويُوجّهها IBM Cloud Activity Tracker Event Routing إلى نسخة IBM Cloud Logs.

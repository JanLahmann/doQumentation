في هذا الدرس، هنقلل عمق الـ circuit عن طريق قطع الـ gates البعيدة، وبكدا نتجنب الـ swap gates اللي كانت هتتضاف أثناء عملية التوجيه (routing).

دي الخطوات اللي هنتبعها في هذا [نمط Qiskit](https://quantum.cloud.ibm.com/docs/guides/intro-to-patterns):

- **الخطوة 1: تحويل المشكلة لـ circuits وعوامل كمية**:
    - تحويل الهاميلتونيان لـ circuit كمي.
- **الخطوة 2: التحسين للهاردوير المستهدف** [_بتستخدم إضافة الـ cutting_]:
    - <font color='#0F62FE'>قطع الـ circuit والـ observable.</font>
    - ترجمة الـ subexperiments للهاردوير.
- **الخطوة 3: التنفيذ على الهاردوير المستهدف**:
    - تشغيل الـ subexperiments اللي اتحصلنا عليها في الخطوة 2 باستخدام بدائي `Sampler`.
- **الخطوة 4: معالجة النتائج** [_بتستخدم إضافة الـ cutting_]:
    - <font color='#0F62FE'>دمج نتائج الخطوة 3 لإعادة بناء القيمة المتوقعة للـ observable المطلوب.</font>

## الخطوة 2: التحسين

### تحديد الـ backend

ممكن تحدد إما backend وهمي أو backend حقيقي من Qiskit Runtime.

```python
from qiskit_ibm_runtime.fake_provider import FakeManilaV2

backend = FakeManilaV2()
```

### ترجمة الـ circuit، ومشاهدة الـ swaps، وتسجيل العمق

بنختار تخطيط (layout) يحتاج لاثنين swap عشان ننفذ الـ gates بين الـ qubits 3 و0، واتنين تانيين عشان نرجع الـ qubits لمواضعها الأصلية.

```python
from qiskit.transpiler import generate_preset_pass_manager

pass_manager = generate_preset_pass_manager(
    optimization_level=1, backend=backend, initial_layout=[0, 1, 2, 3]
)

transpiled_qc = pass_manager.run(circuit)
print(f"Transpiled circuit depth: {transpiled_qc.depth(lambda x: len(x.qubits) >= 2)}")
```

```python
transpiled_qc.draw("mpl", scale=0.4, idle_wires=False, fold=-1)
```

### استبدال الـ gates البعيدة بـ `TwoQubitQPDGate`s عن طريق تحديد فهارسها

`cut_gates` هتستبدل الـ gates عند الفهارس المحددة بـ `TwoQubitQPDGate`s، وكمان هترجع قائمة من نسخ `QPDBasis` — واحدة لكل تحليل gate.

```python
from qiskit_addon_cutting import cut_gates

# Find the indices of the distant gates
cut_indices = [
    i
    for i, instruction in enumerate(circuit.data)
    if {circuit.find_bit(q)[0] for q in instruction.qubits} == {0, 3}
]

# Decompose distant CNOTs into TwoQubitQPDGate instances
qpd_circuit, bases = cut_gates(circuit, cut_indices)

qpd_circuit.draw("mpl", scale=0.8)
```

![Quantum circuit diagram](./_02_gate_cutting_to_reduce_circuit_depth_imgs/output_3.png)

### توليد الـ subexperiments للتشغيل على الـ backend

`generate_cutting_experiments` بتقبل circuit فيها نسخ `TwoQubitQPDGate` والـ observables كـ `PauliList`.

عشان نحاكي القيمة المتوقعة للـ circuit الكاملة، بيتم توليد عدد كبير من الـ subexperiments من التوزيع الاحتمالي المشترك للـ gates المحللة، وبعدين بتتنفذ على backend أو أكتر. عدد العينات المأخوذة من التوزيع بتتحكم فيه `num_samples`، ومعامل مدموج واحد بيتم إعطاؤه لكل عينة فريدة. لمزيد من المعلومات عن طريقة حساب المعاملات، ارجع للـ [مواد الشرح](../explanation/index.rst).

**ملاحظة:** الـ ``observables`` kwarg الخاص بـ `generate_cutting_experiments` من نوع `PauliList`. معاملات ومراحل حدود الـ observable بيتم تجاهلها أثناء تحليل المشكلة وتنفيذ الـ subexperiments. ممكن تتطبق من جديد أثناء إعادة بناء القيمة المتوقعة.

```python
import numpy as np
from qiskit_addon_cutting import generate_cutting_experiments

# Generate the subexperiments and sampling coefficients
subexperiments, coefficients = generate_cutting_experiments(
    circuits=qpd_circuit, observables=observable.paulis, num_samples=np.inf
)
```

### حساب تكلفة أخذ العينات للقطوع المختارة

هنا بنقطع تلات CNOT gates، وده بيدي تكلفة أخذ عينات بقيمة $9^3$.

لمزيد من المعلومات عن تكلفة أخذ العينات الناتجة عن circuit cutting، ارجع للـ [مواد الشرح](../explanation/index.rst).

```python
print(f"Sampling overhead: {np.prod([basis.overhead for basis in bases])}")
```

```text

```

### إثبات أن الـ QPD subexperiments هتبقى أقل عمقًا بعد قطع الـ gates البعيدة

هنا مثال على subexperiment مختار عشوائيًا من الـ QPD circuit. عمقه اتقلل بأكتر من النص. لازم نولد ونقيّم كتير من الـ subexperiments الاحتمالية دي عشان نعيد بناء القيمة المتوقعة للـ circuit الأعمق.

```python
# Transpile the decomposed circuit to the same layout
transpiled_qpd_circuit = pass_manager.run(subexperiments[100])

print(
    f"Original circuit depth after transpile: {transpiled_qc.depth(lambda x: len(x.qubits) >= 2)}"
)
print(
    f"QPD subexperiment depth after transpile: {transpiled_qpd_circuit.depth(lambda x: len(x.qubits) >= 2)}"
```

### تجهيز الـ subexperiments للـ backend

```python
# Transpile the subeperiments to the backend's instruction set architecture (ISA)
isa_subexperiments = pass_manager.run(subexperiments)
```

## الخطوة 3: التنفيذ

### تشغيل الـ subexperiments باستخدام بدائي Qiskit Runtime Sampler

```python
from qiskit_ibm_runtime import SamplerV2

# Set up the Qiskit Runtime Sampler primitive.  For a fake backend, this will use a local simulator.
sampler = SamplerV2(backend)

# Submit the subexperiments
job = sampler.run(isa_subexperiments)
```

```python
# Retrieve the results
results = job.result()
```

## الخطوة 4: معالجة النتائج

### إعادة بناء القيمة المتوقعة

إعادة بناء القيم المتوقعة لكل حد من حدود الـ observable ودمجها لإعادة بناء القيمة المتوقعة للـ observable الأصلي.

```python
from qiskit_addon_cutting import reconstruct_expectation_values

reconstructed_expval_terms = reconstruct_expectation_values(
    results,
    coefficients,
    observable.paulis,
)
# Reconstruct final expectation value
reconstructed_expval = np.dot(reconstructed_expval_terms, observable.coeffs)
```

### مقارنة القيمة المتوقعة المعاد بناؤها بالقيمة المتوقعة الدقيقة من الـ circuit والـ observable الأصليين

```python
from qiskit_aer.primitives import EstimatorV2

estimator = EstimatorV2()
exact_expval = estimator.run([(circuit, observable)]).result()[0].data.evs
print(f"Reconstructed expectation value: {np.real(np.round(reconstructed_expval, 8))}")
print(f"Exact expectation value: {np.round(exact_expval, 8)}")
print(f"Error in estimation: {np.real(np.round(reconstructed_expval-exact_expval, 8))}")
print(
    f"Relative error in estimation: {np.real(np.round((reconstructed_expval-exact_expval) / exact_expval, 8))}"
)
```

```text
Reconstructed expectation value: 0.44018555
Exact expectation value: 0.50497603
Error in estimation: -0.06479049
Relative error in estimation: -0.12830408
```

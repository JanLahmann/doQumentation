في هذا الـ notebook، هنشتغل على خطوات [نمط Qiskit](https://quantum.cloud.ibm.com/docs/guides/intro-to-patterns) ونستخدم **circuit cutting** لتقليل عدد الـ qubits في الـ circuit. هنقطع الـ gates عشان نقدر نعيد بناء القيمة المتوقعة لـ circuit بأربع qubits باستخدام تجارب بـ qubitين بس.

دي هي الخطوات اللي هناخدها:

- **الخطوة الأولى: تحويل المسألة لـ circuits وعمليات كمية**:
    - تحويل الهاميلتوني لـ circuit كمي.
- **الخطوة التانية: التحسين للهاردوير المستهدف** [_بيستخدم cutting addon_]:
    - <font color='#0F62FE'>قطع الـ circuit والـ observable.</font>
    - Transpile الـ subexperiments للهاردوير.
- **الخطوة التالتة: التنفيذ على الهاردوير المستهدف**:
    - تشغيل الـ subexperiments اللي اتحصلنا عليها في الخطوة التانية باستخدام primitive اسمه `Sampler`.
- **الخطوة الرابعة: المعالجة اللاحقة للنتائج** [_بيستخدم cutting addon_]:
    - <font color='#0F62FE'>دمج نتائج الخطوة التالتة لإعادة بناء القيمة المتوقعة للـ observable المطلوب.</font>

## الخطوة التانية: التحسين

### تقسيم الـ circuit والـ observable حسب توزيع محدد للـ qubits

كل label في `partition_labels` بيقابل الـ qubit في الـ `circuit` بنفس الترتيب. الـ qubits اللي عندها نفس الـ label هتتجمع مع بعض، والـ gates غير المحلية اللي بتمتد على أكتر من partition هيتعملها قطع.

**ملاحظة:** الـ ``observables`` kwarg في `partition_problem` نوعه `PauliList`. معاملات وأطوار حدود الـ observable بيتم تجاهلها خلال تحليل المسألة وتنفيذ الـ subexperiments. ممكن إعادة تطبيقها أثناء إعادة بناء القيمة المتوقعة.

```python
from qiskit_addon_cutting import partition_problem

partitioned_problem = partition_problem(
    circuit=qc, partition_labels="AABB", observables=observable.paulis
)
subcircuits = partitioned_problem.subcircuits
subobservables = partitioned_problem.subobservables
bases = partitioned_problem.bases
```

### إنشاء الـ subexperiments للتشغيل على الـ backend

`generate_cutting_experiments` بيقبل `circuits`/`observables` كـ dictionaries بتربط labels أقسام الـ qubits بالـ `subcircuit`/`subobservables` المقابلة.

لمحاكاة القيمة المتوقعة للـ circuit كاملة، بيتم إنشاء كتير من الـ subexperiments من التوزيع الشبه-احتمالي المشترك للـ gates المحللة وتنفيذها على backend أو أكتر. عدد العينات المأخوذة من التوزيع بيتحكم فيه `num_samples`، وبيتم إعطاء معامل مجمع واحد لكل عينة فريدة. لمزيد من المعلومات عن كيفية حساب المعاملات، راجع [المواد التفسيرية](../explanation/index.rst).

```python
from qiskit_addon_cutting import generate_cutting_experiments

subexperiments, coefficients = generate_cutting_experiments(
    circuits=subcircuits, observables=subobservables, num_samples=np.inf
)
```

### اختيار الـ backend

هنا بنستخدم backend وهمي، وده هيخلي Qiskit Runtime يشتغل في الوضع المحلي (يعني على محاكي محلي).

```python
from qiskit_ibm_runtime.fake_provider import FakeManilaV2

backend = FakeManilaV2()
```

### تجهيز الـ subexperiments للـ backend

لازم نعمل transpile للـ circuits باستخدام الـ backend كـ هدف قبل ما نبعتهم لـ Qiskit Runtime.

```python
from qiskit.transpiler import generate_preset_pass_manager

# Transpile the subexperiments to ISA circuits
pass_manager = generate_preset_pass_manager(optimization_level=1, backend=backend)
isa_subexperiments = {
    label: pass_manager.run(partition_subexpts)
    for label, partition_subexpts in subexperiments.items()
}
```

## الخطوة التالتة: التنفيذ

### تشغيل الـ subexperiments باستخدام Qiskit Runtime Sampler primitive

```python
from qiskit_ibm_runtime import SamplerV2, Batch

# Submit each partition's subexperiments to the Qiskit Runtime Sampler
# primitive, in a single batch so that the jobs will run back-to-back.
with Batch(backend=backend) as batch:
    sampler = SamplerV2(mode=batch)
    jobs = {
        label: sampler.run(subsystem_subexpts, shots=2**12)
        for label, subsystem_subexpts in isa_subexperiments.items()
    }
```

## الخطوة الرابعة: المعالجة اللاحقة

### إعادة بناء القيمة المتوقعة

إعادة بناء القيم المتوقعة لكل حد من حدود الـ observable ودمجها لإعادة بناء القيمة المتوقعة للـ observable الأصلي.

```python
from qiskit_addon_cutting import reconstruct_expectation_values

# Get expectation values for each observable term
reconstructed_expval_terms = reconstruct_expectation_values(
    results,
    coefficients,
    subobservables,
)

# Reconstruct final expectation value
reconstructed_expval = np.dot(reconstructed_expval_terms, observable.coeffs)
```

### مقارنة القيمة المتوقعة المعاد بناؤها بالقيمة المتوقعة الدقيقة من الـ circuit والـ observable الأصليين

```python
from qiskit_aer.primitives import EstimatorV2

estimator = EstimatorV2()
exact_expval = estimator.run([(qc, observable)]).result()[0].data.evs
print(f"Reconstructed expectation value: {np.real(np.round(reconstructed_expval, 8))}")
print(f"Exact expectation value: {np.round(exact_expval, 8)}")
print(f"Error in estimation: {np.real(np.round(reconstructed_expval-exact_expval, 8))}")
print(
    f"Relative error in estimation: {np.real(np.round((reconstructed_expval-exact_expval) / exact_expval, 8))}"
)
```

## الخطوة 2: التحسين

### إنشاء circuit جديدة مع وضع تعليمات `Move` في أماكن القطع المطلوبة

بناءً على الـ circuit اللي فوق، عايزين نحط wire cuts اتنين على خط الـ qubit الأوسط، عشان الـ circuit تنقسم لاتنين circuits كل واحدة فيها أربعة qubits. طريقة لعمل ده هي إننا نحط يدويًا تعليمات `Move` ثنائية الـ qubit اللي بتنقل الحالة من سلك qubit لتاني. تعليمة `Move` بمفهومها زي عملية reset على الـ qubit التاني متبوعة بـ gate نوعها SWAP. تأثير التعليمة دي هو نقل حالة الـ qubit الأول (المصدر) للـ qubit التاني (الوجهة)، مع التخلص من الحالة الواردة للـ qubit التاني. عشان ده يشتغل صح، لازم الـ qubit التاني (الوجهة) ما يكونش مشترك في أي تشابك كمي مع باقي النظام؛ وإلا عملية الـ reset هتخلي حالة باقي النظام تنهار جزئيًا.

هنا، بنبني circuit جديدة بـ qubit إضافي وعمليات الـ `Move` في مكانها. في المثال ده، قادرين نعيد استخدام qubit: الـ qubit المصدر في أول `Move` بيبقى هو الـ qubit الوجهة في عملية `Move` التانية.

ملاحظة: كبديل للتعامل المباشر مع تعليمات `Move`، ممكن تختار تضع علامات على wire cuts باستخدام تعليمة `CutWire` أحادية الـ qubit. دالة `cut_wires` موجودة لتحويل `CutWire` لتعليمات `Move` على qubits جديدة. لكن، عكس الطريقة اليدوية، الطريقة الأوتوماتيكية دي ما بتسمحش بإعادة استخدام أسلاك الـ qubit. شوف [دليل التطبيق](../how-tos/how_to_specify_cut_wires.ipynb) الخاص بـ `CutWire` للتفاصيل.

```python
from qiskit_addon_cutting.instructions import Move

qc_1 = QuantumCircuit(8)
for i in [*range(4), *range(5, 8)]:
    qc_1.rx(np.pi / 4, i)
qc_1.cx(0, 3)
qc_1.cx(1, 3)
qc_1.cx(2, 3)
qc_1.append(Move(), [3, 4])
qc_1.cx(4, 5)
qc_1.cx(4, 6)
qc_1.cx(4, 7)
qc_1.append(Move(), [4, 3])
qc_1.cx(0, 3)
qc_1.cx(1, 3)
qc_1.cx(2, 3)

qc_1.draw("mpl")
```

### توليد الـ subexperiments لتشغيلها على الـ backend

`generate_cutting_experiments` بتقبل وسيطات `circuits`/`observables` كـ dictionaries بتربط تسميات أقسام الـ qubit بالـ `subcircuit`/`subobservables` المقابلة لها.

لمحاكاة القيمة المتوقعة للـ circuit الكاملة، بيتم توليد كتير من الـ subexperiments من التوزيع الاحتمالي شبه الكمومي للـ gates المحللة ثم تنفيذها على واحد أو أكتر من الـ backends. عدد العينات المأخوذة من التوزيع بيتحكم فيه `num_samples`، وبيتم إعطاء معامل مدمج واحد لكل عينة فريدة. لمزيد من المعلومات عن طريقة حساب المعاملات، ارجع للـ [مواد الشرح](../explanation/index.rst).

```python
from qiskit_addon_cutting import generate_cutting_experiments

subexperiments, coefficients = generate_cutting_experiments(
    circuits=subcircuits, observables=subobservables, num_samples=np.inf
)
```

### اختيار الـ backend

هنا بنستخدم backend وهمي، وده هيخلي Qiskit Runtime يشتغل في الوضع المحلي (يعني على محاكي محلي).

```python
from qiskit_ibm_runtime.fake_provider import FakeManilaV2

backend = FakeManilaV2()
```

### تجهيز الـ subexperiments للـ backend

لازم نعمل transpile للـ circuits مع الـ backend كهدف قبل ما نبعتها لـ Qiskit Runtime.

```python
from qiskit.transpiler import generate_preset_pass_manager

# Transpile the subexperiments to ISA circuits
pass_manager = generate_preset_pass_manager(optimization_level=1, backend=backend)
isa_subexperiments = {
    label: pass_manager.run(partition_subexpts)
    for label, partition_subexpts in subexperiments.items()
}
```

## الخطوة 3: التنفيذ

### تشغيل الـ subexperiments باستخدام primitive اسمها Qiskit Runtime Sampler

```python
from qiskit_ibm_runtime import SamplerV2, Batch

# Submit each partition's subexperiments to the Qiskit Runtime Sampler
# primitive, in a single batch so that the jobs will run back-to-back.
with Batch(backend=backend) as batch:
    sampler = SamplerV2(mode=batch)
    jobs = {
        label: sampler.run(subsystem_subexpts, shots=2**12)
        for label, subsystem_subexpts in isa_subexperiments.items()
    }
```

## الخطوة 4: معالجة النتائج

### إعادة بناء القيمة المتوقعة

نعيد بناء القيم المتوقعة لكل حد من حدود الـ observable ثم ندمجها لإعادة بناء القيمة المتوقعة للـ observable الأصلي.

```python
from qiskit_addon_cutting import reconstruct_expectation_values

reconstructed_expval_terms = reconstruct_expectation_values(
    results,
    coefficients,
    subobservables,
)
reconstructed_expval = np.dot(reconstructed_expval_terms, observable.coeffs)
```

### مقارنة القيمة المتوقعة المُعاد بناؤها بالقيمة المتوقعة الدقيقة من الـ circuit والـ observable الأصليين

```python
from qiskit_aer.primitives import EstimatorV2

estimator = EstimatorV2()
exact_expval = estimator.run([(qc_0, observable)]).result()[0].data.evs
print(f"Reconstructed expectation value: {np.real(np.round(reconstructed_expval, 8))}")
print(f"Exact expectation value: {np.round(exact_expval, 8)}")
print(f"Error in estimation: {np.real(np.round(reconstructed_expval-exact_expval, 8))}")
print(
    f"Relative error in estimation: {np.real(np.round((reconstructed_expval-exact_expval) / exact_expval, 8))}"
)
```

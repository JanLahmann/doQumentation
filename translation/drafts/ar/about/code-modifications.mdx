# التعديلات التلقائية على الكود

doQumentation يطبّق تلقائيًا عددًا محدودًا من التعديلات على محتوى دروس
Qiskit والأدلة الإرشادية من المصدر الأصلي، وذلك لضمان تجربة سلسة وتفاعلية.
هذه الصفحة توثّق كل تعديل حتى تفهم بالضبط ما الذي تغيّر مقارنةً
بوثائق IBM Quantum الأصلية.

## نسخ دفاتر الملاحظات (فتح في Colab / Binder / Code Engine)

لما تضغط على **Open in Colab** أو **Open in JupyterLab** أو **Open in Code
Engine**، بتاخد نسخة من دفتر الملاحظات الأصلي مع هذه الإضافات:

### 1. خلية إشعار الإعداد (markdown)

يتم إدراج خلية blockquote في أعلى الصفحة تشرح إن doQumentation
أضاف خلية إعداد تلقائية. وتحتوي على رابط يرجع لهذه الصفحة.

### 2. خلية المتطلبات الأساسية (code)

يتم إدراج خلية كود بعد الإشعار تقوم بـ:

- **تثبيت الحزم المطلوبة** (`qiskit`، `qiskit-aer`، `qiskit-ibm-runtime`،
  `pylatexenc`، بالإضافة لأي حزم خاصة بالدرس يتم اكتشافها عبر فحص
  الاستيرادات). يتم **تخطي التثبيت** إذا كانت الحزم موجودة مسبقًا
  (مثلاً على Binder أو Code Engine حيث تكون مثبّتة مسبقًا).
- **توفير قالب بيانات اعتماد معلّق** لـ IBM Quantum، حتى يقدر المستخدمون
  اللي يريدون التشغيل على أجهزة حقيقية إلغاء التعليق وإدخال مفتاح API الخاص بهم.

على Google Colab، هذه الخلية تُشغَّل تلقائيًا عند فتح دفتر الملاحظات عبر
علامة البيانات الوصفية `cell_execution_strategy: setup`.

### 3. إعادة كتابة مسارات الصور

مسارات الصور النسبية (`/docs/images/...`، `/learning/images/...`) تُعاد كتابتها
لتعمل بشكل صحيح في بيئات دفاتر الملاحظات المستقلة.

## صفحات MDX (العرض في المتصفح)

الدروس المعروضة على هذا الموقع تُحوَّل من دفاتر ملاحظات `.ipynb` الأصلية
أو ملفات `.mdx`. يتم تطبيق التحويلات التالية:

- **أسطر `pip install`** تُضاف إلى كتل كود Python التي تستورد
  حزمًا خارجية، مما يتيح التنفيذ بنقرة واحدة عبر thebelab.
- **قسم استطلاع IBM Tutorial**: تُضاف ملاحظة توضّح إن الاستطلاع
  ينتمي لـ IBM Quantum وترتبط بمشكلات GitHub الخاصة بـ doQumentation للحصول على
  ملاحظات خاصة بالموقع.
- **أداة Feedback**: تُضاف أداة "هل كان هذا مفيدًا؟" في أسفل
  كل درس، يتم تتبّعها عبر تحليلات Umami الصديقة للخصوصية.
- **إصلاحات صيغة MDX**: يتم تصحيح مشكلات الأقواس المعقوصة وتسلسل العناوين
  وتوافق JSX تلقائيًا لعرض Docusaurus.
- **OpenInLabBanner**: يتم حقن لافتة تفاعلية أسفل العنوان مع
  أزرار لفتح دفتر الملاحظات في Colab أو Binder أو Code Engine.

## ما الذي لا يتم تعديله

- **محتوى الدرس نفسه** (الشروحات، منطق الكود، المخرجات) لا يتم
  تغييره أبدًا.
- **نسب المؤلف الأصلي** محفوظة عبر frontmatter وملف NOTICE
  (تراخيص Apache 2.0 / CC BY-SA 4.0).
- **لا يوجد كود قياس أداء أو تتبّع** يُحقن في دفاتر الملاحظات. التحليلات
  (Umami) تعمل فقط على موقع doQumentation، وليس في دفاتر الملاحظات المصدَّرة.

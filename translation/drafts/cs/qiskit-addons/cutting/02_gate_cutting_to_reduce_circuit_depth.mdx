V tomto tutoriálu snížíme hloubku Circuit řezáním vzdálených Gate, čímž se vyhneme swap gate, které by jinak byly zavedeny routingem.

Toto jsou kroky, které provedeme v rámci tohoto [Qiskit vzoru](https://quantum.cloud.ibm.com/docs/guides/intro-to-patterns):

- **Krok 1: Mapování problému na kvantové Circuit a operátory**:
    - Namapuj hamiltonián na kvantový Circuit.
- **Krok 2: Optimalizace pro cílový hardware** [_Používá cutting addon_]:
    - <font color='#0F62FE'>Rozřež Circuit a observabl.</font>
    - Transpiluj dílčí experimenty pro hardware.
- **Krok 3: Spuštění na cílovém hardware**:
    - Spusť dílčí experimenty získané v Kroku 2 pomocí primitivy `Sampler`.
- **Krok 4: Post-processing výsledků** [_Používá cutting addon_]:
    - <font color='#0F62FE'>Zkombinuj výsledky Kroku 3 a rekonstruuj očekávanou hodnotu sledovaného observablu.</font>

## Krok 2: Optimalizace {#krok-2-optimalizace}

### Zadání Backend {#zadání-backend}

Můžeš zadat buď falešný Backend, nebo hardwarový Backend z Qiskit Runtime.

```python
from qiskit_ibm_runtime.fake_provider import FakeManilaV2

backend = FakeManilaV2()
```

### Transpilování Circuit, vizualizace swapů a zaznamenání hloubky {#transpilování-circuit-vizualizace-swapů-a-zaznamenání-hloubky}

Volíme rozložení, které vyžaduje dva swapy pro provedení Gate mezi Qubit 3 a 0 a další dva swapy pro vrácení Qubit do jejich počátečních pozic.

```python
from qiskit.transpiler import generate_preset_pass_manager

pass_manager = generate_preset_pass_manager(
    optimization_level=1, backend=backend, initial_layout=[0, 1, 2, 3]
)

transpiled_qc = pass_manager.run(circuit)
print(f"Transpiled circuit depth: {transpiled_qc.depth(lambda x: len(x.qubits) >= 2)}")
```

### Nahrazení vzdálených Gate pomocí `TwoQubitQPDGate` zadáním jejich indexů {#nahrazení-vzdálených-gate-pomocí-twoqubitqpdgate-zadáním-jejich-indexů}

`cut_gates` nahradí Gate na zadaných indexech instancemi `TwoQubitQPDGate` a také vrátí seznam instancí `QPDBasis` – jednu pro každý rozklad Gate.

```python
from qiskit_addon_cutting import cut_gates

# Find the indices of the distant gates
cut_indices = [
    i
    for i, instruction in enumerate(circuit.data)
    if {circuit.find_bit(q)[0] for q in instruction.qubits} == {0, 3}
]

# Decompose distant CNOTs into TwoQubitQPDGate instances
qpd_circuit, bases = cut_gates(circuit, cut_indices)

qpd_circuit.draw("mpl", scale=0.8)
```

![Diagram kvantového Circuit](./output_3.png)

### Generování dílčích experimentů ke spuštění na Backend {#generování-dílčích-experimentů-ke-spuštění-na-backend}

`generate_cutting_experiments` přijímá Circuit obsahující instance `TwoQubitQPDGate` a observably jako `PauliList`.

Pro simulaci očekávané hodnoty Circuit v plné velikosti se z kvazirozděleníj pravděpodobnosti rozložených Gate generuje mnoho dílčích experimentů, které jsou pak spuštěny na jednom nebo více Backend. Počet vzorků odebraných z rozdělení je řízen parametrem `num_samples` a pro každý jedinečný vzorek je uveden jeden kombinovaný koeficient. Více informací o výpočtu koeficientů najdeš v [vysvětlujícím materiálu](../explanation/index.rst).

**Poznámka:** Argument ``observables`` funkce `generate_cutting_experiments` je typu `PauliList`. Koeficienty a fáze termů observablu jsou při rozkladu problému a provádění dílčích experimentů ignorovány. Mohou být znovu aplikovány při rekonstrukci očekávané hodnoty.

```python
import numpy as np
from qiskit_addon_cutting import generate_cutting_experiments

# Generate the subexperiments and sampling coefficients
subexperiments, coefficients = generate_cutting_experiments(
    circuits=qpd_circuit, observables=observable.paulis, num_samples=np.inf
)
```

### Ukázka, že dílčí experimenty QPD budou po rozřezání vzdálených Gate mělčí {#ukázka-že-dílčí-experimenty-qpd-budou-po-rozřezání-vzdálených-gate-mělčí}

Zde je příklad libovolně zvoleného dílčího experimentu vygenerovaného z QPD Circuit. Jeho hloubka byla snížena o více než polovinu. Aby bylo možné rekonstruovat očekávanou hodnotu hlubšího Circuit, musí být vygenerováno a vyhodnoceno mnoho takovýchto pravděpodobnostních dílčích experimentů.

# Transpile the decomposed circuit to the same layout

isa_subexperiments = pass_manager.run(subexperiments)

## Krok 3: Spuštění {#krok-3-spuštění}

### Spuštění dílčích experimentů pomocí primitivy Qiskit Runtime Sampler {#spuštění-dílčích-experimentů-pomocí-primitivy-qiskit-runtime-sampler}

```python
from qiskit_ibm_runtime import SamplerV2

# Set up the Qiskit Runtime Sampler primitive.  For a fake backend, this will use a local simulator.
sampler = SamplerV2(backend)

# Submit the subexperiments
job = sampler.run(isa_subexperiments)
```

```python
# Retrieve the results
results = job.result()
```

## Krok 4: Post-processing {#krok-4-post-processing}

### Rekonstrukce očekávané hodnoty {#rekonstrukce-očekávané-hodnoty}

Rekonstruuj očekávané hodnoty pro každý term observablu a zkombinuj je, aby vznikla očekávaná hodnota pro původní observabl.

```python
from qiskit_addon_cutting import reconstruct_expectation_values

reconstructed_expval_terms = reconstruct_expectation_values(
    results,
    coefficients,
    observable.paulis,
)
# Reconstruct final expectation value
reconstructed_expval = np.dot(reconstructed_expval_terms, observable.coeffs)
```

### Porovnání rekonstruované očekávané hodnoty s přesnou očekávanou hodnotou z původního Circuit a observablu {#porovnání-rekonstruované-očekávané-hodnoty-s-přesnou-očekávanou-hodnotou-z-původního-circuit-a-observablu}

```python
from qiskit_aer.primitives import EstimatorV2

estimator = EstimatorV2()
exact_expval = estimator.run([(circuit, observable)]).result()[0].data.evs
print(f"Reconstructed expectation value: {np.real(np.round(reconstructed_expval, 8))}")
print(f"Exact expectation value: {np.round(exact_expval, 8)}")
print(f"Error in estimation: {np.real(np.round(reconstructed_expval-exact_expval, 8))}")
print(
    f"Relative error in estimation: {np.real(np.round((reconstructed_expval-exact_expval) / exact_expval, 8))}"
)
```

```text
Reconstructed expectation value: 0.44018555
Exact expectation value: 0.50497603
Error in estimation: -0.06479049
Relative error in estimation: -0.12830408
```

## Testování relací neurčitosti {#testing-uncertainty-relations}

Výše uvedený test prokázal platnost relace neurčitosti pouze pro jediný výběr stavového vektoru $|\psi\rangle = |+\rangle_x$. Abychom se přesvědčili, že je to obecně v souladu s experimentem, měli bychom provést podobné výpočty pomocí Estimatoru pro mnoho různých stavových vektorů. Začneme tím, že otočíme náš stavový vektor pryč od osy $z$ pomocí Gate `RY`, čímž vytvoříme různé počáteční stavy s parametrem $\theta$.

```python
# Výpočet níže využívá přibližně 3–4 minuty času QPU.
# Krok 1: Mapování problému na kvantový Circuit

from qiskit.circuit import Parameter
import numpy as np

# Zadání pozorovatelných veličin
obs1 = SparsePauliOp("X")
obs2 = SparsePauliOp("Y")
obs3 = SparsePauliOp("Z")

# Definice registrů
qr = QuantumRegister(1, "q")
cr = ClassicalRegister(1, "c")
qc = QuantumCircuit(qr, cr)

# Otočení pryč od |0>
theta = Parameter("θ")
qc.ry(theta, 0)

params = np.linspace(0, 2, num=21)

# Krok 2: Transpilace Circuitu

pm = generate_preset_pass_manager(target=target, optimization_level=3)
qc_isa = pm.run(qc)
obs1_isa = obs1.apply_layout(layout=qc_isa.layout)
obs2_isa = obs2.apply_layout(layout=qc_isa.layout)
obs3_isa = obs3.apply_layout(layout=qc_isa.layout)

# Krok 3: Spuštění Circuitu na skutečném kvantovém počítači

with Batch(backend=backend) as batch:
    estimator = Estimator(mode=batch)
    pubs = [(qc_isa, [[obs1_isa], [obs2_isa], [obs3_isa]], [params])]
    job = estimator.run(pubs, precision=0.01)
    res = job.result()

batch.close()

# Spuštění úlohy na simulátoru Aer s modelem šumu ze skutečného Backend

# job = noisy_estimator.run([(qc, [[obs1], [obs2], [obs3]], [params])])
# res=job.result()
```

```python
# Krok 4: Postprocessing a klasická analýza.
xs = res[0].data.evs[0]
ys = abs(res[0].data.evs[1])
zs = res[0].data.evs[2]

# Výpočet neurčitostí

delx = []
delz = []
prodxz = []
for i in range(len(xs)):
    delx.append(abs((1 - xs[i] * xs[i])) ** 0.5)
    delz.append(abs((1 - zs[i] * zs[i])) ** 0.5)
    prodxz.append(delx[i] * delz[i])
```

```python
# Zde můžeme vykreslit výsledky z této simulace.
import matplotlib.pyplot as plt

plt.plot(params, delx, label=r"$\Delta$ X")
plt.plot(params, ys, label=r"$\langle$ Y $\rangle$")
plt.plot(params, delz, label=r"$\Delta$ Z")
plt.plot(params, prodxz, label=r"$\Delta$X $\Delta$Z")
plt.xlabel(r"$\theta$")
plt.ylabel("Expectation/Uncertainty Values")
plt.legend()
plt.show()
```

Všimni si, že červená křivka $(\Delta X \Delta Z)$ je vždy větší než oranžová křivka $\langle Y \rangle.$ Někdy součin neurčitostí klesne a přiblíží se limitu, jindy stoupne a od limitu se vzdálí, ale relaci neurčitosti vždy splňuje.

Tento test relace neurčitosti ovšem nemusí být nejlepší, protože naše mez $\langle Y \rangle$ je vždy velmi blízko nule. Použijme kvantový stav s větší projekcí na vlastní stavy $Y$. Konkrétně budeme stále otáčet $|0\rangle$ dolů od osy $z$ o různé úhly, ale tentokrát výsledný stav ještě otočíme kolem osy $z$ o určitý úhel, třeba $\pi/4$, a uvidíme, co se stane.

qc.ry(theta, 0)
qc.rz(pi / 4, 0)

Nyní vidíme, že mez neurčitosti $(\Delta X \Delta Z)$ je opravdu prověřována! Červená křivka se k oranžové přibližuje mnohem více než dříve. Ve skutečnosti by v nepřítomnosti šumu byla relace neurčitosti přesně saturována ($(\Delta X \Delta Z) = \langle Y \rangle$) v jednom bodě. V přítomnosti šumu a chyby čtení by nemělo překvapit, pokud běh občas vykáže $(\Delta X \Delta Z)$ velmi mírně větší než $\langle Y \rangle.$ Nejde o skutečné porušení principu neurčitosti — je to jen artefakt nenulové chyby.

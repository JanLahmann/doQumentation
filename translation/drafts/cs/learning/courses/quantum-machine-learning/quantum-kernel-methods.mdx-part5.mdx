### Krok 2: Optimalizace problému pro kvantové spuštění {#step-2-optimize-problem-for-quantum-execution}

Začneme výběrem nejméně vytíženého backendu a poté optimalizujeme náš Circuit pro spuštění na tomto backendu.

```python
# Import needed packages
from qiskit.transpiler.preset_passmanagers import generate_preset_pass_manager
from qiskit_ibm_runtime import QiskitRuntimeService

# Get the least busy backend
service = QiskitRuntimeService()
backend = service.least_busy(
    operational=True, simulator=False, min_num_qubits=fm.num_qubits
)
print(backend)
```


U malých úloh preset pass manager často spolehlivě vrátí stejný Circuit se stejnou hloubkou. Ale u velmi velkých a složitých Circuitů může pass manager pokaždé vrátit jiný transpilovaný Circuit. Je to proto, že používá heuristiky, a protože velmi velké Circuity mají komplikovaný prostor možných optimalizací. Často je užitečné transpilovat několikrát a vzít nejplošší Circuit. Tím vznikne pouze klasická režie a výsledky z kvantového počítače se mohou podstatně zlepšit.

Zde transpilujeme Circuit unitárního překryvu 20krát a podíváme se na hloubky získaných Circuitů.

```python
# Apply level 3 optimization to our overlap circuit
transpiled_qcs = []
transpiled_depths = []
transpiled_twoqubit_depths = []
for i in range(1, 20):
    pm = generate_preset_pass_manager(optimization_level=3, backend=backend)
    overlap_ibm = pm.run(overlap_circ)
    transpiled_qcs.append(overlap_ibm)
    transpiled_depths.append(overlap_ibm.decompose().depth())
    transpiled_twoqubit_depths.append(
        overlap_ibm.decompose().depth(lambda instr: len(instr.qubits) > 1)
    )

print("circuit depth = ", overlap_ibm.decompose().depth())
```

### Krok 3: Spuštění pomocí Qiskit Runtime Primitives {#step-3-execute-using-qiskit-runtime-primitives}

S tím, jak se přibližujeme k praktickému využití, simulátory přestanou být užitečné. Zde je uvedena pouze syntaxe pro skutečné kvantové počítače.

```python
# Run on ibm_osaka, 7-12-24, required 22 sec.

# Import our runtime primitive
from qiskit_ibm_runtime import SamplerV2 as Sampler

# Open a Runtime session:
session = Session(backend=backend)
num_shots = 10000
# Use sampler and get the counts

sampler = Sampler(mode=session)
options = sampler.options
options.dynamical_decoupling.enable = True
options.twirling.enable_gates = True
counts = (
    sampler.run([overlap_ibm], shots=num_shots).result()[0].data.meas.get_int_counts()
)

# Close session after done
session.close()
```

### Škálování a budoucí práce {#scaling-and-future-work}

Metoda kernel vyžaduje, abychom co nejpřesněji měřili stav $|0\rangle$. Chyby Gate a chyby čtení však způsobují nenulovou pravděpodobnost $p$, že libovolný Qubit bude chybně naměřen ve stavu $|1\rangle$. I se zjednodušujícím předpokladem, že pravděpodobnost $|0\rangle$ by měla být $100\,\%$, je pro mnoho příznaků zakódovaných na $N$ bitech pravděpodobnost správného naměření všech bitů jako $|0\rangle$ snížena na $(1-p)^N$. S rostoucím $N$ se tato metoda stává stále méně spolehlivou. Překonání této obtíže a škálování odhadu kernelu na stále více příznaků je oblastí současného výzkumu. Chceš-li se o tomto problému dozvědět více, podívej se na práci autorů [Thanasilp, Wang, Cerezo a Holmes.](https://www.nature.com/articles/s41467-024-49287-w) Doporučujeme ti prozkoumat, co lze s dnešními kvantovými počítači dělat, a zároveň se těšit na to, co bude možné v éře opravy chyb.

### Opakování {#review}

Výpočet kvantového kernelu zahrnuje:
* výpočet prvků kernel matice pomocí párů trénovacích datových bodů
* zakódování dat a jejich mapování prostřednictvím feature map
* optimalizaci Circuitu pro spuštění na skutečných kvantových počítačích / backendech

Kvantový kernel pak může být použit v klasických algoritmech strojového učení, jak je ukázáno v této lekci.

Při používání kvantových kernelů mej na paměti několik klíčových věcí:
* Je pravděpodobné, že datová sada bude profitovat z metod kvantového kernelu?
* Zkus různé feature mapy a různá schémata provázání.
* Je hloubka Circuitu přijatelná?
* Zkus spustit pass manager několikrát a použij Circuit s nejmenší hloubkou, jaký získáš.

Metody kvantového kernelu jsou potenciálně výkonné nástroje při správném souladu mezi datovými sadami s kvantově vhodnými příznaky a vhodnou kvantovou feature map. Pro lepší pochopení toho, kde jsou kvantové kernely pravděpodobně užitečné, doporučujeme přečíst [Liu, Arunachalam & Temme (2021)](https://www.nature.com/articles/s41567-021-01287-z).

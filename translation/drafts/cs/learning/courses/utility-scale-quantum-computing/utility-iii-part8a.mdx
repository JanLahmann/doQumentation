## 5. Tvůj cíl (shrnutí) {#your-goal-recap}

Sestav GHZ Circuit pro 20 nebo více qubitů tak, aby výsledek měření splňoval toto kritérium: věrnost tvého GHZ stavu > 0,5.

- Musíš použít zařízení Eagle (například `ibm_brisbane`) a nastavit počet měření (shots) na 40 000.
- GHZ Circuit bys měl(a) spustit pomocí funkce `execute_ghz_fidelity` a věrnost vypočítat funkcí `check_ghz_fidelity_from_jobs`.
Najdi největší GHZ Circuit (co do počtu qubitů), který kritérium splňuje. Napiš svůj kód níže a výsledek zobraz pomocí funkce `check_ghz_fidelity_from_jobs`.

Nyní implementujeme stejný GHZ postup jako v předchozí části, ale na zařízení Heron. Získáš tak zkušenost s rozložením a vlastnostmi procesorů Heron. Žádné nové strategie se nepředstavují.

*Přibližná doba QPU pro spuštění tohoto experimentu je 4 min 40 s.*

```python
BAD_READOUT_ERROR_THRESHOLD = 0.1
BAD_CZGATE_ERROR_THRESHOLD = 0.1
bad_readout_qubits = [
    q
    for q in range(backend.num_qubits)
    if backend.target["measure"][(q,)].error > BAD_READOUT_ERROR_THRESHOLD
]
bad_czgate_edges = [
    qpair
    for qpair in backend.target["cz"]
    if backend.target["cz"][qpair].error > BAD_CZGATE_ERROR_THRESHOLD
]
print("Bad readout qubits:", bad_readout_qubits)
print("Bad CZ gates:", bad_czgate_edges)
```

```text

Bad CZ gates: [(111, 112), (112, 111), (112, 113), (113, 112), (120, 121), (121, 120), (130, 131), (131, 130), (145, 146), (146, 145), (146, 147), (147, 146)]
```

```python
g = backend.coupling_map.graph.copy().to_undirected()
g.remove_edges_from(
    bad_czgate_edges
)  # remove edge first (otherwise might fail with a NoEdgeBetweenNodes error)
g.remove_nodes_from(bad_readout_qubits)
```

```python
qubit_color = []
```

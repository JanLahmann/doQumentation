### Přidání referenčního stavu {#add-reference-state}

V předchozím příkladu jsme nepoužili žádný referenční operátor $U_R$. Nyní se zamysleme nad tím, jak lze získat ideální vlastní stav $\frac{1}{\sqrt{2}}(|00\rangle + |11\rangle)$. Uvažuj následující Circuit.

```python
from qiskit import QuantumCircuit

ideal_qc = QuantumCircuit(2)
ideal_qc.h(0)
ideal_qc.cx(0, 1)

ideal_qc.draw("mpl")
```

![Výstup předchozí buňky kódu](/learning/images/courses/variational-algorithm-design/examples-and-applications/extracted-outputs/55b26ae7-ad26-4b54-b4f3-2004bb2db0c9-0.avif)

Můžeme rychle ověřit, že tento Circuit nám dá požadovaný stav.

```python
from qiskit.quantum_info import Statevector

Statevector(ideal_qc)
```

```text
Statevector([0.70710678+0.j, 0.        +0.j, 0.        +0.j,
             0.70710678+0.j],
            dims=(2, 2))
```

Nyní, když jsme viděli, jak vypadá Circuit připravující řešení, zdá se rozumné použít Hadamardovu Gate jako referenční Circuit, takže celý Ansatz bude mít podobu:

```python
reference = QuantumCircuit(2)
reference.h(0)
reference.cx(0, 1)
# Include barrier to separate reference from variational form
reference.barrier()

ref_ansatz = variational_form.decompose().compose(reference, front=True)

ref_ansatz.draw("mpl")
```

![Výstup předchozí buňky kódu](/learning/images/courses/variational-algorithm-design/examples-and-applications/extracted-outputs/14e04c7a-e81f-41e1-b0cf-790954581be9-0.avif)

Pro tento nový Circuit by ideálního řešení bylo dosaženo při nastavení všech parametrů na $0$, což potvrzuje, že volba referenčního Circuitu je rozumná.

Nyní porovnejme počet vyhodnocení účelové funkce, iterací optimalizátoru a dobu výpočtu s předchozím pokusem.

```python
import time

start_time = time.time()

ref_result = minimize(
    cost_func_vqe, x0, args=(ref_ansatz, observable_1, estimator), method="COBYLA"
)

end_time = time.time()
execution_time = end_time - start_time
```

Použití optimálních parametrů k výpočtu minimální vlastní hodnoty:

```python
experimental_min_eigenvalue_ref = cost_func_vqe(
    ref_result.x, ref_ansatz, observable_1, estimator
)
print(experimental_min_eigenvalue_ref)
```

```text
-5.999999996759607
```

```python
print("ADDED REFERENCE STATE:")
print(f"""Number of iterations: {ref_result.nfev}""")
print(f"""Time (s): {execution_time}""")
print(
    f"Percent error: {100*abs((experimental_min_eigenvalue_ref - solution_eigenvalue)/solution_eigenvalue):.2e}"

```

V závislosti na tvém konkrétním systému může, ale nemusí dojít ke zlepšení rychlosti nebo přesnosti v tomto velmi malém příkladu. Podstatné je, že začínat s fyzikálně motivovanými referenčními stavy je stále důležitější pro zlepšení rychlosti a přesnosti s rostoucí velikostí problémů.
### Změna počátečního bodu

Nyní, když jsme viděli efekt přidání referenčního stavu, se podíváme na to, co se stane, když zvolíme různé počáteční body $\vec{\theta_0}$. Konkrétně použijeme $\vec{\theta_0}=(0,0,0,0,6,0,0,0)$ a $\vec{\theta_0}=(6,6,6,6,6,6,6,6,6)$.

Pamatuj, že jak bylo probráno při představení referenčního stavu, ideální řešení by bylo nalezeno při nastavení všech parametrů na $0$, takže první počáteční bod by měl vést k menšímu počtu vyhodnocení.

```python
import time

start_time = time.time()

x0 = [0, 0, 0, 0, 6, 0, 0, 0]

x0_1_result = minimize(
    cost_func_vqe, x0, args=(raw_ansatz, observable_1, estimator), method="COBYLA"
)

end_time = time.time()
execution_time = end_time - start_time
```

```python
print("INITIAL POINT 1:")
print(f"""Number of iterations: {x0_1_result.nfev}""")
print(f"""Time (s): {execution_time}""")
```

```text
INITIAL POINT 1:
Number of iterations: 108
Time (s): 0.4492197036743164
```

Úprava počátečního bodu na $\vec{\theta_0}=(6,6,6,6,6,6,6,6,6)$:

import time

### Experimentování s různými optimalizátory

Optimalizátor lze upravit pomocí argumentu `method` funkce `minimize` z knihovny SciPy, přičemž další možnosti [najdeš zde](https://docs.scipy.org/doc/scipy/reference/generated/scipy.optimize.minimize.html). Původně jsme použili podmíněný minimalizátor (`COBYLA`). V tomto příkladu prozkoumáme použití nepodmíněného minimalizátoru (`BFGS`).

```python
import time

start_time = time.time()

result = minimize(
    cost_func_vqe, x0, args=(raw_ansatz, observable_1, estimator), method="BFGS"
)

end_time = time.time()
execution_time = end_time - start_time
```

```python
print("CHANGED TO BFGS OPTIMIZER:")
print(f"""Number of iterations: {result.nfev}""")
print(f"""Time (s): {execution_time}""")
```

```text
CHANGED TO BFGS OPTIMIZER:
Number of iterations: 117
Time (s): 0.31656408309936523
```

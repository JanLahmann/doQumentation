## Vlastní VQE {#custom-vqe}

Nejprve prozkoumáme, jak ručně sestavit instanci VQE pro nalezení nejnižší vlastní hodnoty $\hat{O}_1$. Využijeme přitom různé techniky, které jsme v průběhu tohoto kurzu probrali.

```python
def cost_func_vqe(params, ansatz, hamiltonian, estimator):
    """Return estimate of energy from estimator

    Parameters:
        params (ndarray): Array of ansatz parameters
        ansatz (QuantumCircuit): Parameterized ansatz circuit
        hamiltonian (SparsePauliOp): Operator representation of Hamiltonian
        estimator (Estimator): Estimator primitive instance

    Returns:
        float: Energy estimate
    """
    pub = (ansatz, hamiltonian, params)
    cost = estimator.run([pub]).result()[0].data.evs

    return cost
```

## Příklad VQD {#vqd-example}
### Krok 1: Mapování klasických vstupů na kvantový problém {#step-1-map-classical-inputs-to-a-quantum-problem}

Nyní místo hledání pouze nejnižší vlastní hodnoty našich pozorovatelných budeme hledat všechny $4$, (kde $k=4$).

Připomeň si, že účelové funkce VQD jsou:

$$
C_{l}(\vec{\theta}) :=
\langle \psi(\vec{\theta}) | \hat{H} | \psi(\vec{\theta})\rangle +
\sum_{j=0}^{l-1}\beta_j |\langle \psi(\vec{\theta})| \psi(\vec{\theta^j})\rangle  |^2
\quad \forall l\in\{1,\cdots,k\}=\{1,\cdots,4\}
$$

To je zvláště důležité, protože vektor $\vec{\beta}=(\beta_0,\cdots,\beta_{k-1})$ (v tomto případě $(\beta_0, \beta_1, \beta_2, \beta_3)$) musí být předán jako argument při definování objektu `VQD`.

Také v Qiskitově implementaci VQD se místo uvažování efektivních pozorovatelných popsaných v předchozím notebooku počítají věrnosti $|\langle \psi(\vec{\theta})| \psi(\vec{\theta^j})\rangle  |^2$ přímo pomocí algoritmu `ComputeUncompute`, který využívá primitivum `Sampler` k vzorkování pravděpodobnosti získání $|0\rangle$ pro Circuit
$U_A^\dagger(\vec{\theta})U_A(\vec{\theta^j})$. To funguje právě proto, že tato pravděpodobnost je

$$
\begin{aligned}

p_0

& = |\langle 0|U_A^\dagger(\vec{\theta})U_A(\vec{\theta^j})|0\rangle|^2 \\[1mm]

& = |\big(\langle 0|U_A^\dagger(\vec{\theta})\big)\big(U_A(\vec{\theta^j})|0\rangle\big)|^2 \\[1mm]

& = |\langle \psi(\vec{\theta}) |\psi(\vec{\theta^j}) \rangle|^2 \\[1mm]

\end{aligned}
$$

Budeme používat následující funkci pro výpočet penalizace překryvu. Všimni si, že toto je stále součástí mapování problému na kvantové Circuity. Jak však bylo probráno v předchozí lekci, tato funkce počítá překryv mezi aktuálním variačním Circuitem a optimalizovaným Circuitem z předchozího stavu s nižší energií/náklady. Nový generovaný Circuit musí být také transpilován pro spuštění na skutečném hardwaru. Tuto funkci jsme viděli dříve, použitou na simulátoru. Zde musíme již zohlednit transpilaci a související optimalizaci pro případ, kdy použijeme skutečný Backend — odtud řádky kolem `if realbackend == 1`. To trochu mísí krok 2, ale krok 2 výslovně zmíníme později.

```python
import numpy as np
from qiskit.transpiler.preset_passmanagers import generate_preset_pass_manager

def calculate_overlaps(
    ansatz, prev_circuits, parameters, sampler, realbackend, backend
):
    def create_fidelity_circuit(circuit_1, circuit_2):
        if len(circuit_1.clbits) > 0:
            circuit_1.remove_final_measurements()
        if len(circuit_2.clbits) > 0:
            circuit_2.remove_final_measurements()

        circuit = circuit_1.compose(circuit_2.inverse())
        circuit.measure_all()
        return circuit

    overlaps = []

    for prev_circuit in prev_circuits:
        fidelity_circuit = create_fidelity_circuit(ansatz, prev_circuit)
        if realbackend == 1:
            pm = generate_preset_pass_manager(backend=backend, optimization_level=3)
            fidelity_circuit = pm.run(fidelity_circuit)
        sampler_job = sampler.run([(fidelity_circuit, parameters)])
        meas_data = sampler_job.result()[0].data.meas

        counts_0 = meas_data.get_int_counts().get(0, 0)
        shots = meas_data.num_shots
        overlap = counts_0 / shots
        overlaps.append(overlap)

    return np.array(overlaps)
```

Nyní přidáme účelovou funkci VQD. Všimni si, že ve srovnání s předchozí lekcí máme nyní dva další argumenty (`realbackend` a `backend`), které nám pomáhají s transpilací při použití skutečných Backendů.

```python
def cost_func_vqd(
    parameters,
    ansatz,
    prev_states,
    step,
    betas,
    estimator,
    sampler,
    hamiltonian,
    realbackend,
    backend,
):
    estimator_job = estimator.run([(ansatz, hamiltonian, [parameters])])

    total_cost = 0

    if step > 1:
        overlaps = calculate_overlaps(
            ansatz, prev_states, parameters, sampler, realbackend, backend
        )
        total_cost = np.sum(
            [np.real(betas[state] * overlap) for state, overlap in enumerate(overlaps)]
        )

    estimator_result = estimator_job.result()[0]

    value = estimator_result.data.evs[0] + total_cost

    return value
```

from qiskit import QuantumCircuit

def cost_func_vqe(params, ansatz, hamiltonian, estimator):
    """Return estimate of energy from estimator

    Parameters:
        params (ndarray): Array of ansatz parameters
        ansatz (QuantumCircuit): Parameterized ansatz circuit

    pub = (ansatz, hamiltonian, params)

Když tedy požadujeme, aby byl excitovaný stav ortogonální k prvnímu nalezenému stavu, nevynucujeme ortogonalitu s pravým základním stavem, nýbrž jen s jeho přibližnou verzí (někdy velmi nepřesnou). Excitovaný stav tedy nebyl nucen být ortogonální k pravému základnímu stavu a naše odhady energií excitovaných stavů byly ve skutečnosti velmi blízké energii základního stavu.

Toto bude vždy představovat riziko při použití VQD. V principu to lze napravit zvýšením maximálního počtu iterací klasického optimalizátoru, snížením tolerance klasického optimalizátoru a případně také vyzkoušením jiného Ansatzu, pokud pravý základní stav opakovaně nenajdeme. Jak jsme viděli, může být také nutné upravit penalizace překryvu (betas). To je však v podstatě samostatný problém. Žádná penalizace za překryv tě neochrání před základním stavem, pokud jsi ještě nenašel dobrý odhad skutečného základního stavu pro Circuit překryvu.
## Optimalizace: Max-Cut {#optimization-max-cut}

Problém maximálního řezu (Max-Cut) je kombinatorický optimalizační problém, který spočívá v rozdělení vrcholů grafu do dvou disjunktních množin tak, aby byl maximalizován počet hran mezi těmito dvěma množinami. Formálněji, pro daný neorientovaný graf $G=(V,E)$, kde $V$ je množina vrcholů a $E$ je množina hran, se problém Max-Cut ptá na to, jak rozdělit vrcholy do dvou disjunktních podmnožin, $S$ a $T$, tak aby byl maximalizován počet hran, jejichž jeden koncový bod leží v $S$ a druhý v $T$.

Max-Cut můžeme použít k řešení různých problémů, jako je shlukování, návrh sítí a fázové přechody. Začneme vytvořením grafu problému:

```python
import rustworkx as rx
from rustworkx.visualization import mpl_draw

n = 4
G = rx.PyGraph()
G.add_nodes_from(range(n))
# The edge syntax is (start, end, weight)
edges = [(0, 1, 1.0), (0, 2, 1.0), (0, 3, 1.0), (1, 2, 1.0), (2, 3, 1.0)]
G.add_edges_from(edges)

mpl_draw(
    G, pos=rx.shell_layout(G), with_labels=True, edge_labels=str, node_color="#1192E8"
)
```

![Output of the previous code cell](/learning/images/courses/variational-algorithm-design/examples-and-applications/extracted-outputs/3eb71b79-a988-4807-aa88-7fb25a78a236-0.avif)

Tento problém lze vyjádřit jako binární optimalizační problém. Pro každý uzel $0 \leq i < n$, kde $n$ je počet uzlů grafu (v tomto případě $n=4$), budeme uvažovat binární proměnnou $x_i$. Tato proměnná bude mít hodnotu $1$, pokud uzel $i$ patří do jedné ze skupin, kterou označíme jako $1$, a $0$, pokud je v jiné skupině, kterou označíme jako $0$. Dále budeme označovat $w_{ij}$ (prvek $(i,j)$ matice sousednosti $w$) jako váhu hrany vedoucí z uzlu $i$ do uzlu $j$. Protože je graf neorientovaný, platí $w_{ij}=w_{ji}$. Pak můžeme náš problém formulovat jako maximalizaci následující účelové funkce:

$$
\begin{aligned}
C(\vec{x})
& =\sum_{i,j=0}^n w_{ij} x_i(1-x_j)\\[1mm]

& = \sum_{i,j=0}^n w_{ij} x_i - \sum_{i,j=0}^n w_{ij} x_ix_j\\[1mm]

& = \sum_{i,j=0}^n w_{ij} x_i - \sum_{i=0}^n \sum_{j=0}^i 2w_{ij} x_ix_j
\end{aligned}
$$

Abychom mohli tento problém řešit na kvantovém počítači, vyjádříme účelovou funkci jako střední hodnotu pozorovatelné veličiny. Pozorovatelné veličiny, které Qiskit nativně podporuje, se skládají z Pauliho operátorů, jejichž vlastní hodnoty jsou $1$ a $-1$ namísto $0$ a $1$. Proto provedeme následující změnu proměnné:

Kde $\vec{x}=(x_0,x_1,\cdots ,x_{n-1})$. K pohodlnému přístupu k vahám všech hran můžeme použít matici sousednosti $w$. Ta bude využita k odvození naší účelové funkce:

$$
z_i = 1-2x_i \rightarrow x_i = \frac{1-z_i}{2}
$$

Z toho plyne:

$$
\begin{array}{lcl} x_i=0 & \rightarrow & z_i=1 \\ x_i=1 & \rightarrow & z_i=-1.\end{array}
$$

Nová účelová funkce, kterou chceme maximalizovat, tedy je:

$$
\begin{aligned}
C(\vec{z})
& = \sum_{i,j=0}^n w_{ij} \bigg(\frac{1-z_i}{2}\bigg)\bigg(1-\frac{1-z_j}{2}\bigg)\\[1mm]

& = \sum_{i,j=0}^n \frac{w_{ij}}{4} - \sum_{i,j=0}^n \frac{w_{ij}}{4} z_iz_j\\[1mm]

& = \sum_{i=0}^n \sum_{j=0}^i \frac{w_{ij}}{2} -  \sum_{i=0}^n \sum_{j=0}^i \frac{w_{ij}}{2} z_iz_j
\end{aligned}
$$

Kvantový počítač má navíc přirozenou tendenci hledat minima (obvykle nejnižší energii) namísto maxim, proto místo maximalizace $C(\vec{z})$ budeme minimalizovat:

$$
-C(\vec{z}) =  \sum_{i=0}^n \sum_{j=0}^i \frac{w_{ij}}{2} z_iz_j -  \sum_{i=0}^n \sum_{j=0}^i \frac{w_{ij}}{2}
$$

Nyní, když máme účelovou funkci k minimalizaci, jejíž proměnné mohou nabývat hodnot $-1$ a $1$, můžeme provést následující analogii s Pauliho operátorem $Z$:

$$
z_i \equiv Z_i = \overbrace{I}^{n-1}\otimes ... \otimes \overbrace{Z}^{i} \otimes ... \otimes \overbrace{I}^{0}
$$

Jinými slovy, proměnná $z_i$ bude ekvivalentní Gate $Z$ působícímu na Qubit $i$. Dále platí:

$$
Z_i|x_{n-1}\cdots x_0\rangle = z_i|x_{n-1}\cdots x_0\rangle \rightarrow \langle x_{n-1}\cdots x_0 |Z_i|x_{n-1}\cdots x_0\rangle = z_i
$$

Pozorovatelná veličina, kterou budeme uvažovat, je tedy:

$$
\hat{H} = \sum_{i=0}^n \sum_{j=0}^i \frac{w_{ij}}{2} Z_iZ_j
$$

ke které bude následně třeba přičíst nezávislý člen:

$$
\texttt{offset} = - \sum_{i=0}^n \sum_{j=0}^i \frac{w_{ij}}{2}
$$
Operátor je lineární kombinací členů se Z operátory na uzlech spojených hranou (připomeňme, že 0. Qubit je zcela vpravo): $IIZZ + IZIZ + IZZI + ZIIZ + ZZII$. Jakmile je operátor sestrojen, ansatz pro algoritmus QAOA lze snadno sestavit pomocí Circuit `QAOAAnsatz` z knihovny Qiskit circuit library.

```python

    [("IIZZ", 1), ("IZIZ", 1), ("IZZI", 1), ("ZIIZ", 1), ("ZZII", 1)]

```python
def cost_func(params, ansatz, hamiltonian, estimator):
    """Return estimate of energy from estimator

    Parameters:
        params (ndarray): Array of ansatz parameters
        ansatz (QuantumCircuit): Parameterized ansatz circuit
        hamiltonian (SparsePauliOp): Operator representation of Hamiltonian
        estimator (Estimator): Estimator primitive instance

    Returns:
        float: Energy estimate
    """
    pub = (ansatz, hamiltonian, params)
    cost = estimator.run([pub]).result()[0].data.evs
    #    cost = estimator.run(ansatz, hamiltonian, parameter_values=params).result().values[0]
    return cost
```

K minimalizaci účelové funkce lze použít libovolný klasický optimalizátor. Na skutečném kvantovém systému si optimalizátory navržené pro nespojité krajiny účelové funkce obvykle vedou lépe. Zde používáme [rutinu COBYLA](https://docs.scipy.org/doc/scipy/reference/optimize.minimize-cobyla.html#optimize-minimize-cobyla) z SciPy prostřednictvím funkce minimize.

Protože iterativně provádíme velké množství volání Runtime, využíváme Session ke spuštění všech volání v rámci jediného bloku. Navíc u QAOA je řešení zakódováno ve výstupním rozdělení Circuit ansatz svázaného s optimálními parametry získanými minimalizací. Proto budeme potřebovat primitiv Sampler a vytvoříme jeho instanci se stejnou Session.
Spustíme naši minimalizační rutinu:

result = minimize(

#### Symetrická výměna klíčů pomocí RSA {#symmetric-key-exchange-with-rsa}
Jak jsme již probírali, <DefinitionTooltip definition="Asymetrické šifrování klíčů.">AKC</DefinitionTooltip> umožňuje dvěma stranám, které chtějí komunikovat, bezpečně ustanovit *sdílené tajemství*, jež lze využít například jako tajný klíč pro následné symetrické šifrování objemného prostého textu.

Uvažujme následující scénář. Alice a Bob chtějí používat <DefinitionTooltip definition="Symetrické šifrování klíčů.">SKC</DefinitionTooltip> k šifrování a dešifrování zpráv. Než však tento proces může začít, musí se dohodnout na společném tajném klíči. Jednou z možností je, že jedna strana — řekněme Alice — vygeneruje tajný klíč a poté ho bezpečně přenese Bobovi. Aby bylo toto předání bezpečné, Alice se rozhodne použít RSA jako <DefinitionTooltip definition="Mechanismus zapouzdření klíče. Zabezpečuje symetrický klíč pomocí asymetrického šifrování klíčů. Jednoduchý přístup spočívá v zašifrování náhodného symetrického klíče veřejným klíčem, avšak existují pokročilejší a bezpečnější KEM mechanismy, přičemž neustále vznikají nové.">mechanismus zapouzdření klíče</DefinitionTooltip> (KEM).

Celý proces zahrnuje následující kroky:

- Nejprve Alice vygeneruje náhodný symetrický klíč, který hodlá sdílet s Bobem.
- Poté Bob vygeneruje asymetrický pár klíčů a svůj veřejný klíč zpřístupní na vhodném kanálu.
- Následně Alice použije Bobův veřejný klíč k zašifrování symetrického klíče, čímž ho zapouzdří do šifrového textu.
- Alice pak šifrový text vysílá přes spolehlivý, ale ne nutně bezpečný kanál.
- Nakonec Bob obdrží šifrový text a dešifruje ho pomocí svého soukromého klíče. Tím získá přístup k symetrickému klíči vygenerovanému Alicí.

##### Příklad výměny klíčů s paddingem v Pythonu
Praktický postup využívající RSA pro neinteraktivní výměnu klíčů s paddingem je nyní ilustrován pomocí knihovny ``cryptography`` pro Python.

Importujeme potřebné moduly z knihovny ``cryptography`` pro Python. V případě potřeby lze tuto knihovnu nainstalovat příkazem ``pip install cryptography``.

Alice pak vygeneruje náhodný tajný klíč, který hodlá přenést Bobovi.

```python
# pip install cryptography
from cryptography.hazmat.primitives import serialization
from cryptography.hazmat.primitives.asymmetric import rsa, padding
from cryptography.fernet import Fernet
from cryptography.hazmat.primitives import hashes

symmetric_key = Fernet.generate_key()
print(f"\nSymmetric key generated by Alice: {symmetric_key}")
```

Pomocí modulu ``rsa`` z knihovny ``cryptography`` Bob vygeneruje pár klíčů a poté vysílá svůj veřejný klíč. Kdokoli může tento veřejný klíč zachytit a vyčíst z něj veřejná čísla $(e,n)$, která klíč tvoří.

```python
# Bob generates a 2048-bit RSA key pair
bob_private_key = rsa.generate_private_key(public_exponent=65537, key_size=2048)
bob_public_key = bob_private_key.public_key()
print(f"Public key broadcast by Bob: {bob_public_key}")
print(f"\nPublic numbers in Bobs' public key: {bob_public_key.public_numbers()}")
```

V tuto chvíli předpokládáme, že Alice obdržela veřejný klíč, který Bob vyslal. Alicin ``symmetric_key`` výše lze nyní zašifrovat pomocí Bobova veřejného klíče a vytvořit tak šifrový text. V reálném prostředí bude Alice také používat dodatečné metody <DefinitionTooltip definition="Přidání dat na začátek, do středu nebo na konec zprávy. Často se používá u blokových šifer k doplnění bloků s nedostatečnými daty na požadovanou velikost.">paddingu</DefinitionTooltip>, například [OAEP](https://en.wikipedia.org/w/index.php?title=Optimal_asymmetric_encryption_padding&oldid=1163481577), aby zajistila <DefinitionTooltip definition="Útočník, který má k dispozici pouze šifrový text a veškeré veřejně dostupné informace (jako je veřejný klíč u asymetrického šifrování), nedokáže získat ani dílčí informace o prostém textu.">sémantickou bezpečnost</DefinitionTooltip> své komunikace s Bobem.

```python
# Encryption
ciphertext = bob_public_key.encrypt(
    symmetric_key,
    padding.OAEP(
        mgf=padding.MGF1(algorithm=hashes.SHA256()),
        algorithm=hashes.SHA256(),
        label=None,
    ),
)

print("Ciphertext:", ciphertext)
```

Alice pak vysílá šifrový text přes otevřený kanál v důvěře, že ho bude moci dešifrovat jedině Bob s odpovídajícím soukromým klíčem. Předpokládáme, že Bob obdržel šifrový text a může ho nyní dešifrovat pomocí svého důvěrného soukromého klíče.

```python
# Bob decrypts ciphertext to access the symmetric key
decrypted_symmetric_key = bob_private_key.decrypt(
    ciphertext,
    padding.OAEP(
        mgf=padding.MGF1(algorithm=hashes.SHA256()),
        algorithm=hashes.SHA256(),
        label=None,
    ),
)

print("Decrypted key:", decrypted_symmetric_key)
assert decrypted_symmetric_key == symmetric_key
```

V tuto chvíli mají Alice i Bob přístup k tajnému symetrickému klíči, který mohou využít pro aplikace <DefinitionTooltip definition="Symetrické šifrování klíčů.">SKC</DefinitionTooltip>.
##### Simulace mechanismu zapouzdření klíče pomocí RSA v Pythonu
V následujícím postupu ilustrujeme použití RSA k simulaci mechanismu zapouzdření klíče (KEM), při němž je bezpečně vyměněna dostatečně dlouhá náhodná tajná zpráva, která je následně pomocí KDF převedena na sdílené tajemství odpovídající délky.

Alice a Bob opět chtějí neinteraktivně ustanovit sdílené tajemství a Alice je strana, která rozhoduje o tom, jaké tajemství použít.

Začneme importem potřebných knihoven pro Python.

Bob pak vygeneruje svůj RSA pár klíčů a vysílá svůj veřejný klíč.

```python
from cryptography.hazmat.primitives.asymmetric import rsa, padding
from cryptography.hazmat.primitives.kdf.hkdf import HKDF
from cryptography.hazmat.primitives import hashes
from os import urandom

# Bob's RSA key pair
private_key_Bob = rsa.generate_private_key(public_exponent=65537, key_size=2048)
public_key_Bob = private_key_Bob.public_key()

print("Bob's private and public keys created")
```

Alice nejprve vygeneruje dlouhé náhodné tajemství, z nějž bude nakonec odvozeno sdílené tajemství. V čistém [KEM](https://en.wikipedia.org/w/index.php?title=Key_encapsulation_mechanism&oldid=1110687965) bude tímto dlouhým tajemstvím náhodný prvek z algebraické struktury, na níž je kryptosystém postaven. V případě 2048-bitového RSA by to byl náhodný celý číslo modulo 2048-bitový RSA modul. Čistý KEM tedy nevyžaduje dodatečný padding, ale v tomto příkladu pouze simulujeme KEM pomocí RSA a knihovna `cryptography` vyžaduje použití paddingu při šifrování s RSA. Použijeme proto poněkud kratší dlouhé tajemství, které je nicméně podstatně delší než standardní 256-bitový klíč AES.

```python
Alice_long_secret = urandom(160)  # A 160 byte or 1280 bit random message
print("Alice's secret created")
```

Dále Alice zašifruje dlouhé tajemství pomocí Bobova veřejného klíče a zašifrované tajemství předá Bobovi.

```python
Alice_encrypted_secret = public_key_Bob.encrypt(
    Alice_long_secret,
    padding.OAEP(
        mgf=padding.MGF1(algorithm=hashes.SHA256()),
        algorithm=hashes.SHA256(),
        label=None,
    ),
)
print("Alice's secret encrypted")
```

Bob dešifruje zašifrované tajemství obdržené od Alice pomocí svého soukromého klíče.

```python
Bob_decrypted_secret = private_key_Bob.decrypt(
    Alice_encrypted_secret,
    padding.OAEP(
        mgf=padding.MGF1(algorithm=hashes.SHA256()),
        algorithm=hashes.SHA256(),
        label=None,
    ),
)

assert Alice_long_secret == Bob_decrypted_secret, "Secrets do not match!"

# if we get here they match
print("Secrets match")
```

Nakonec Alice i Bob samostatně aplikují dohodnutou [funkci odvození klíče](https://en.wikipedia.org/w/index.php?title=Key_derivation_function&oldid=1165650237) (KDF) na dlouhé tajemství, aby odvodili symetrický klíč.

Všimni si, že tento proces zahrnuje hashovací protokol a použití náhodné <DefinitionTooltip definition="V kontextu hashování hesel platí, že stejně jako sůl dodává jídlu jedinečnou chuť, přidání soli v tomto kontextu dělá naše heslo jedinečným, charakteristickým a těžko reprodukovatelným.">*soli*</DefinitionTooltip>, která zajišťuje jedinečnost a nepředvídatelnost sdíleného symetrického klíče pro případ, že by bylo dlouhé tajemství někdy znovu použito (nedoporučuje se). Samotná *sůl* však nemusí být tajná, a jakmile je náhodně vygenerována — v tomto příkladu Alicí — může být Bobovi předána spolu se zašifrovaným dlouhým tajemstvím.

Budeme tedy předpokládat, že Alice i Bob mají přístup ke stejné náhodné *soli*.

```python
def key_derivation_function(secret, salt):
    hkdf = HKDF(
        algorithm=hashes.SHA256(),
        length=32,  # Desired key length
        salt=salt,
        info=None,
        backend=None,
    )
    return hkdf.derive(secret)

common_salt = urandom(16)  # Random salt accessible to both Alice and Bob

symmetric_key_Alice = key_derivation_function(Alice_long_secret, common_salt)
symmetric_key_Bob = key_derivation_function(Bob_decrypted_secret, common_salt)

assert symmetric_key_Alice == symmetric_key_Bob, "Derived keys do not match!"
print(
    f"A symmetric key of length {len(symmetric_key_Alice)*8} bits was successfully derived by both Alice and Bob!"
)
```

#### Digitální podpisy s RSA {#digital-signatures-with-rsa}
Výše popsaný scénář důvěrné komunikace mezi Alice a Bobem nyní rozšíříme o ověřování pomocí [digitálního podpisu](https://en.wikipedia.org/w/index.php?title=Digital_signature&oldid=1161173191).

Stejně jako předtím Alice důvěrně pošle Bobovi zprávu obsahující symetrický klíč, ale tentokrát ji také digitálně podepíše. Bob tak po přijetí zprávy bude moci ověřit, že ji skutečně odeslala Alice a že obsah zprávy nebyl během přenosu pozměněn.

Obecněji řečeno, je žádoucí umožnit ověřování bez narušení důvěrnosti — aby mohl kdokoli se zájmem ověřit <DefinitionTooltip definition="Integrita je zachována tehdy, když je zpráva chráněna před neoprávněnými změnami.">integritu</DefinitionTooltip>, <DefinitionTooltip definition="Autenticita je důkaz toho, že zpráva pochází z tvrzeného zdroje.">autenticitu</DefinitionTooltip> a prokázat nepopiratelnost dané komunikace, i když tento subjekt nemá přístup ke skutečné zprávě v otevřeném textu.

Budeme uvažovat tento obecný scénář, který zahrnuje následující kroky:

- Nejprve Bob i Alice zveřejní své veřejné klíče přes otevřený kanál.
- Poté Alice zašifruje otevřený text Bobovým veřejným klíčem a vytvoří šifrový text.
- Dále Alice vytvoří hash šifrového textu pomocí *hašovací funkce* a výsledný zahašovaný šifrový text zašifruje svým soukromým klíčem. Tento zašifrovaný hash je *podpisem*.
- Pak Alice přenese přes otevřený kanál jak šifrový text, tak podpis.
- Bob pak pomocí Alicina veřejného klíče dešifruje podpis a získá zahašovaný šifrový text.
- Protože má Bob k dispozici také samotný šifrový text, použije stejnou hašovací funkci jako Alice a vytvoří druhou instanci zahašovaného šifrového textu. Pokud se shoduje s tím, který získal dešifrováním Alicina podpisu, je zpráva ověřena — přestože samotný šifrový text ještě nebyl dešifrován.
- Nakonec Bob, který zprávu ověřil, dešifruje šifrový text pomocí svého soukromého klíče.

Tento pracovní postup digitálního podpisu je ilustrován níže.

Opět importujeme několik užitečných modulů z knihovny ``cryptography``.
Stejně jako předtím chce Alice bezpečně poslat Bobovi symetrický klíč, ale tentokrát ho také chce digitálně podepsat. V tomto případě potřebujeme veřejné klíče jak pro Alice, tak pro Boba. Prvním krokem je tedy to, aby si oba — Alice i Bob — vytvořili vlastní pár klíčů pomocí RSA a zveřejnili svůj veřejný klíč světu.

```python
from cryptography.hazmat.primitives import hashes
from cryptography.hazmat.primitives.asymmetric import padding, rsa
from cryptography.hazmat.primitives.asymmetric.utils import Prehashed

# Generate keys for Bob
bob_private_key = rsa.generate_private_key(public_exponent=65537, key_size=2048)
bob_public_key = bob_private_key.public_key()

# Generate keys for Alice
alice_private_key = rsa.generate_private_key(public_exponent=65537, key_size=2048)
alice_public_key = alice_private_key.public_key()

print("Private and Public keys generated for Bob and Alice.")
```

V dalším kroku Alice stejně jako předtím pomocí Bobova veřejného klíče zašifruje symetrický klíč a připraví šifrový text.

```python
# Alice encrypts the message using Bob's public key
ciphertext = bob_public_key.encrypt(
    symmetric_key,
    padding.OAEP(
        mgf=padding.MGF1(algorithm=hashes.SHA256()),
        algorithm=hashes.SHA256(),
        label=None,
    ),
)

print("ciphertext of symmetric key: ", ciphertext)
```

V tuto chvíli Alice nechce pouze zveřejnit šifrový text, ale chce k němu přiložit digitální podpis, jímž Bobovi prokáže, že je odesílatelem zprávy. To se provede ve dvou krocích:

1. Vytvoř hash šifrového textu pomocí hašovacího algoritmu.
2. Zašifruj hash Aliciným soukromým klíčem — tím vznikne podpis.

```python
# Alice signs the ciphertext using her private key
digest = hashes.Hash(hashes.SHA256())
digest.update(ciphertext)
hash_to_sign = digest.finalize()

signature = alice_private_key.sign(
    hash_to_sign,
    padding.PSS(mgf=padding.MGF1(hashes.SHA256()), salt_length=padding.PSS.MAX_LENGTH),
    Prehashed(hashes.SHA256()),
)

print("signature: ", signature)
```

Alice pak zveřejní šifrový text a podpis přes síť, aby je Bob mohl zachytit.

```python
# Bob receives the ciphertext and signature
received_ciphertext = ciphertext
received_signature = signature

# Send signature and ciphertext here
print("Sending ciphertext and signature.....")
```

Na Bobově straně je prvním úkolem ověřit <DefinitionTooltip definition="Integrita je zachována tehdy, když je zpráva chráněna před neoprávněnými změnami.">integritu</DefinitionTooltip> a <DefinitionTooltip definition="Autenticita je důkaz toho, že zpráva pochází z tvrzeného zdroje.">autenticitu</DefinitionTooltip> přijatého šifrového textu. K tomu Bob vytvoří hash přijatého šifrového textu pomocí stejného hašovacího algoritmu, jaký použila Alice.

```python
# Bob creates a hash of the ciphertext using the same algorithm used by Alice
digest = hashes.Hash(hashes.SHA256())
digest.update(received_ciphertext)
hash_to_verify = digest.finalize()

print("hash to verify: ", hash_to_verify)
```

Poté Bob dešifruje přijatý podpis pomocí Alicina veřejného klíče. Protože Alice použila k vytvoření podpisu svůj soukromý klíč, Bob ho může dešifrovat pomocí jejího veřejného klíče. Dešifrovaný podpis není nic jiného než hash šifrového textu vytvořený na Alicině straně. Pokud se hash vytvořený Bobem shoduje s dešifrovaným podpisem, Bob ověřil, že přijatý šifrový text nebyl pozměněn a že to byla Alice, kdo šifrový text podepsal.

V níže uvedeném kódu v Pythonu jsou tyto operace sloučeny do užitečné pomocné funkce ``verify``, kterou poskytuje objekt přidružený k Alicinu veřejnému klíči.

```python
from cryptography.exceptions import InvalidSignature

def is_signature_valid(public_key, signature, data_hash):
    try:
        public_key.verify(
            signature,
            data_hash,
            padding.PSS(
                mgf=padding.MGF1(hashes.SHA256()), salt_length=padding.PSS.MAX_LENGTH
            ),
            Prehashed(hashes.SHA256()),
        )
        return True
    except InvalidSignature:
        return False

if is_signature_valid(alice_public_key, received_signature, hash_to_verify):
    print("The signature is valid.")
else:
    print("The signature is not valid.")
```

Poté, co Bob ověřil <DefinitionTooltip definition="Integrita je zachována tehdy, když je zpráva chráněna před neoprávněnými změnami.">integritu</DefinitionTooltip> a <DefinitionTooltip definition="Autenticita je důkaz toho, že zpráva pochází z tvrzeného zdroje.">autenticitu</DefinitionTooltip> přijatého šifrového textu, může ho dešifrovat pomocí svého soukromého klíče, protože Alice šifrový text vytvořila pomocí Bobova veřejného klíče.

```python
# Bob decrypts the message using his private key
decrypted_message = bob_private_key.decrypt(
    received_ciphertext,
    padding.OAEP(
        mgf=padding.MGF1(algorithm=hashes.SHA256()),
        algorithm=hashes.SHA256(),
        label=None,
    ),
)

print("Decrypted message:", decrypted_message.decode())
```

Ve výše popsaném scénáři digitálního podpisu může kdokoli — nejen Bob — ověřit, že Alice je odesílatelem šifrového textu, protože každý má přístup k Alicinu veřejnému klíči, šifrovému textu a digitálnímu podpisu. Navíc poté, co Alice odeslala šifrový text a podpis, nemůže popřít, že to udělala, protože podpis lze dešifrovat na smysluplný hash pouze pomocí jejího veřejného klíče. Tím se zajišťuje <DefinitionTooltip definition="Nepopiratelnost označuje schopnost prokázat, že konkrétní strana odeslala zprávu. V symetrickém klíčovém šifrování, kde se stejný klíč používá jak pro šifrování, tak pro dešifrování, nelze určit, která strana vytvořila konkrétní šifrový text. Naproti tomu asymetrické klíčové šifrování zajišťuje nepopiratelnost prostřednictvím digitálních podpisů.">nepopiratelnost</DefinitionTooltip>.

Díky umožnění bezpečné distribuce klíčů a podpoře nepopiratelnosti vytváří kryptografie s veřejným klíčem pevný základ pro bezpečnou digitální komunikaci.
## Prolomení RSA

Užitečnost a bezpečnost algoritmu <DefinitionTooltip definition="Kryptosystém s veřejným klíčem pojmenovaný po Ronu Rivestovi, Adi Shamirovi a Leonardu Adlemanovi. Vyvinut v roce 1977.">RSA</DefinitionTooltip> popsaného výše spočívá na dvou matematických předpokladech:

1. Nalezení <DefinitionTooltip definition="Pro celé číslo $a$ a modulus $m$ je zbytek po dělení ax/m roven 1. To platí pouze pro nesoudělná čísla.">modulárního multiplikativního inverzního prvku</DefinitionTooltip> $d$ při znalosti pouze $(e, n)$ je výpočetně neproveditelné.

2. V kontextu RSA se operace modulárního umocňování chová jako <DefinitionTooltip definition="Jednosměrná funkce, u níž je snadné vypočítat výstup ze vstupů, ale obtížné určit vstupy z výstupu.">jednosměrná trapdoor funkce</DefinitionTooltip>. Při použití pro šifrování vytváří nečitelný šifrový text a bez přístupu k soukromému klíči není proveditelné operaci invertovat a z šifrového textu získat otevřený text. S přístupem k soukromému klíči, který funguje jako trapdoor, však lze šifrový text snadno dešifrovat.

Nejznámější útok na algoritmus <DefinitionTooltip definition="Kryptosystém s veřejným klíčem pojmenovaný po Ronu Rivestovi, Adi Shamirovi a Leonardu Adlemanovi. Vyvinut v roce 1977.">RSA</DefinitionTooltip> se snaží narušit předpoklad 1 tím, že efektivně obnoví soukromý klíč $d$ pomocí faktorizace <DefinitionTooltip definition="Modulus je zbytek po dělení zadanou hodnotou. Modulo je operace, která to provádí.">modulu</DefinitionTooltip> $n$. Jak bude ukázáno níže, je snadné obnovit $d$, pokud má někdo přístup buď k <DefinitionTooltip definition="Prvočíselné faktory čísla n jsou ta prvočísla, jejichž součin dává n.">prvočíselným faktorům</DefinitionTooltip> $p$ a $q$ čísla $n$, nebo k Eulerově funkci <DefinitionTooltip definition="Řecký symbol fí, často používaný k označení Eulerovy funkce.">$φ$</DefinitionTooltip>$(n)$. Připomeňme, že $p$, $q$ a <DefinitionTooltip definition="Řecký symbol fí, často používaný k označení Eulerovy funkce.">$φ$</DefinitionTooltip>$(n)$ jsou během generování klíčů uchovávány v tajnosti a po vygenerování klíčů zahozeny. Třetí strana, která tuto informaci získá pomocí klasického nebo kvantového počítače, v podstatě odhalí soukromý klíč a RSA tím prolomí. Prvočíselná faktorizace je tedy klíčovým výpočetním primitivem nezbytným pro prolomení RSA.
### Klasické výpočty a RSA

Prvočíselná faktorizace velkých celých čísel vykazuje na klasických počítačích <DefinitionTooltip definition="Roste rychleji než polynom; například exponenciální funkce.">superpolynomiální</DefinitionTooltip> nebo <DefinitionTooltip definition="Škáluje pomaleji než exponenciálně. Například polynomiální nebo lineární škálování.">subexponenciální</DefinitionTooltip> škálování. Nejlepší známý klasický algoritmus pro faktorizaci celých čísel větších než $10^{100}$ je <DefinitionTooltip definition="General Number Field Sieve — efektivní klasický algoritmus pro faktorizaci velkých celých čísel.">*obecné síto číselného tělesa*</DefinitionTooltip> [(GNFS)](https://en.wikipedia.org/w/index.php?title=General_number_field_sieve&oldid=1151781536)

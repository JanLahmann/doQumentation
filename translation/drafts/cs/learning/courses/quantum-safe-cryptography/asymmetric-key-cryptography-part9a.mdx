### Bezpečnost DSA {#security-of-the-dsa}

Při klasickém výpočtu může být bezpečnost DSA ovlivněna několika faktory:

1. **Velikost klíče**: Jak je tomu vždy, velikost klíče poskytuje základní ochranu proti útokům hrubou silou. Moderní aplikace využívající DSA používají klíče o délce alespoň 2048 bitů.

2. **Kvalita $k$**: V DSA každý podpis vyžaduje jedinečnou, náhodnou a tajnou hodnotu $k$ (viz výše). Jedinečnost, entropie a utajení $k$ jsou klíčové — slabina v kterémkoli z těchto aspektů může vést k prozrazení soukromého klíče $x$. Generátory náhodných čísel používané k vytváření $k$ musí být samy o sobě bezpečné.

3. **Odolnost hashovací funkce**: Protože DSA používá hashovací funkci jako součást procesu generování a ověřování podpisu, slabá hashovací funkce by mohla ohrozit bezpečnost digitálního podpisu. Například použití SHA-1 s DSA je zastaralé a doporučuje se SHA-2 nebo vyšší.

Kromě toho by robustní implementace DSA měla chránit i před dalšími typy útoků zaměřených na kryptografii s asymetrickými klíči, jak bylo popsáno výše.

### Ověřená výměna klíčů pomocí Diffie-Hellman a DSA {#authenticated-key-exchange-with-diffie-hellman-and-dsa}

Moderní protokoly pro zabezpečení sítí, jako je [Transport Layer Security (TLS)](https://en.wikipedia.org/w/index.php?title=Transport_Layer_Security&oldid=1155601531) a mnohé další, kombinují funkce výměny klíčů a autentizace. V kontextu Diffie-Hellmana je autentizace potřebná jako ochrana proti útokům MITM. V následujících buňkách s kódem ilustrujeme příklad, kde Alice a Bob provádějí ověřenou výměnu klíčů kombinací protokolů Diffie-Hellman a DSA. K tomu použijeme Python knihovnu `cryptography`.

![Obrázek 2. Ověřená výměna klíčů pomocí Diffie-Hellman a DSA](/learning/images/courses/quantum-safe-cryptography/asymmetric-key-cryptography/auth_key_exchange.avif)

*Obrázek 2. Ověřená výměna klíčů pomocí Diffie-Hellman a DSA*

Nejprve se Alice a Bob dohodnou na sadě parametrů DH a vygenerují sadu párů veřejného a soukromého klíče DH.

```python
# import necessary library modules
from cryptography.hazmat.primitives import hashes
from cryptography.hazmat.primitives.asymmetric import dh, dsa

# Generate DH parameters
parameters = dh.generate_parameters(generator=2, key_size=2048)

# Generate Alice's DH private key and public key
alice_dh_private_key = parameters.generate_private_key()
alice_dh_public_key = alice_dh_private_key.public_key()

# Generate Bob's DH private key and public key
bob_dh_private_key = parameters.generate_private_key()
bob_dh_public_key = bob_dh_private_key.public_key()

print("Public and private keys generated for Bob and Alice")
```

Veřejné klíče DH jsou zveřejněny. Dále si Alice i Bob každý zvlášť vygenerují pár klíčů pro použití s DSA. Tyto klíče budou následně použity k podepsání vyměňovaných veřejných klíčů DH.

```python
# Generate DSA keys for Alice and Bob
alice_dsa_private_key = dsa.generate_private_key(key_size=2048)
alice_dsa_public_key = alice_dsa_private_key.public_key()
bob_dsa_private_key = dsa.generate_private_key(key_size=2048)
bob_dsa_public_key = bob_dsa_private_key.public_key()

print("Additional key pair generated for signing")
```

Alice podepíše svůj veřejný klíč DH pomocí svého soukromého klíče DSA.

```python
# Alice signs
alice_public_bytes = alice_dh_public_key.public_bytes(
    encoding=serialization.Encoding.PEM,
    format=serialization.PublicFormat.SubjectPublicKeyInfo,
)
alice_signature = alice_dsa_private_key.sign(alice_public_bytes, hashes.SHA256())

print("Alice signed public key")
```

Podobně Bob podepíše svůj veřejný klíč DH pomocí svého soukromého klíče DSA.

```python
bob_public_bytes = bob_dh_public_key.public_bytes(
    encoding=serialization.Encoding.PEM,
    format=serialization.PublicFormat.SubjectPublicKeyInfo,
)
bob_signature = bob_dsa_private_key.sign(bob_public_bytes, hashes.SHA256())

print("Bob signed public key")
```

Veřejné klíče DH a odpovídající podpisy jsou nyní zveřejněny oběma stranami. Po obdržení veřejného klíče a podpisu protistrany každá ze stran ověří platnost podpisu. Takto lze zabránit útoku MITM, protože například Alice ví, že veřejný klíč DH Boba byl skutečně podepsán Bobem, a naopak.

```python
# Alice and Bob verify each other's DH public keys using DSA public keys
# An InvalidSignature exception will occur if they are not valid
alice_dsa_public_key.verify(alice_signature, alice_public_bytes, hashes.SHA256())
bob_dsa_public_key.verify(bob_signature, bob_public_bytes, hashes.SHA256())

print("Signatures are valid")
```

Po ověření podpisů vygenerují Alice a Bob sdílené tajemství, čímž je výměna klíčů dokončena.

```python
# Perform key exchange
alice_shared_key = alice_dh_private_key.exchange(bob_dh_public_key)
bob_shared_key = bob_dh_private_key.exchange(alice_dh_public_key)

print("Shared secrets generated")
```

Volitelně mohou Alice a Bob pro vyšší bezpečnost použít specializovanou funkci pro odvozování klíčů, jako je *HKDF*, a pomocí technik *key stretching* z jejich sdíleného tajemství odvodit bezpečnější symetrický klíč.

```python
# Derive a shared symmetric key using key-stretching
def derive_key(shared_key):
    return HKDF(
        algorithm=hashes.SHA256(),
        length=32,
        salt=None,
        info=None,
    ).derive(shared_key)

alice_symmetric_key = derive_key(alice_shared_key)
bob_symmetric_key = derive_key(bob_shared_key)

assert alice_symmetric_key == bob_symmetric_key
print("Keys checked to be the same")
```

#### Generování RSA klíčů {#rsa-key-generation}

Pojďme si krok za krokem projít jednoduchý příklad RSA algoritmu s malými prvočísly.

Budeme potřebovat umět vypočítat <DefinitionTooltip definition="Největší společný dělitel – největší kladné celé číslo, které dělí dvě nebo více zadaných celých čísel.">největší společný dělitel</DefinitionTooltip> dvou celých čísel, protože ho budeme potřebovat k ověření, zda jsou dvě celá čísla <DefinitionTooltip definition="Dvě čísla, jejichž největší společný dělitel je 1. Například 3 a 13 jsou nesoudělná, protože je dělí pouze 1.">nesoudělná</DefinitionTooltip>.

Vysvětlíme jeden jednoduchý způsob výpočtu, ale pro větší celá čísla je mnohem efektivnější použít funkci `math.gcd` z Pythonu.


```python
import math

# Example function to compute the gcd (greatest common divisor)
def gcd(a, b):
    if b == 0:
        return a
    else:
        return gcd(b, a % b)

# let's calculate some examples using algorithm
n1 = gcd(50, 10)
n2 = gcd(99, 33)
n3 = gcd(59, 9)

# do the same with the python library call

m1 = math.gcd(50, 10)
m2 = math.gcd(99, 33)
m3 = math.gcd(59, 9)

# Confirm they are the same
assert n1 == m1
assert n2 == m2
assert n3 == m3

# They are - print out the values for explanation
print("gcd(50,10) =", m1)
print("gcd(99,33) =", m2)
print("gcd(59,9) =", m3)
```

První fází RSA pracovního postupu je generování klíčů. Začíná výběrem dvou prvočísel, která musí zůstat tajná pro entitu generující klíče.

```python
# Choosing two prime numbers and keep them secret
p = 13
q = 19
print("The secret prime numbers p and q are:", p, q)
```

Dále se vypočítá <DefinitionTooltip definition="Modulus je zbytek po dělení zadanou hodnotou. Modulo je operace, která toto provádí.">modulus</DefinitionTooltip> $n$, což je jednoduše součin dvou zvolených prvočísel. Tento modulus bude zveřejněn jako součást veřejného klíče.

```python
# Calculate n which is the modulus for both the public and private keys
n = p * q
print("modulus n (p*q)=", n)
```

Jako další se vypočítá Eulerova <DefinitionTooltip definition="Totient je číslo nebo čísla, která nejsou větší než dané číslo a jsou k němu nesoudělná, tedy nemají žádné společné dělitele.">totientová funkce</DefinitionTooltip> <DefinitionTooltip definition="Řecký symbol fí, často používaný k označení Eulerovy totientové funkce.">$\varphi(n)$</DefinitionTooltip>, která je potřebná pro operaci <DefinitionTooltip definition="Pro celé číslo $a$ a modulus $m$ je zbytek ax/m roven 1. To platí pouze pro nesoudělná čísla.">modulárního multiplikativního inverzu</DefinitionTooltip> používanou při určování klíčů v RSA. $phi$ je rovněž uchováváno v tajnosti a po vygenerování klíčů se obvykle zahodí.

```python
# Compute Euler's totient function, φ(n) and keep it secret
phi = (p - 1) * (q - 1)
print("The secret Euler's function (totient) [phi(n)]:", phi)
```

Nyní jsme připraveni vypočítat veřejný a soukromý klíč. V RSA je každý z nich specifikován dvojicí dvou celých čísel. Prvním prvkem každé dvojice je odlišné celé číslo a druhým prvkem je modulus $n$, který je společný pro oba klíče.

Prvním prvkem veřejného klíče může být libovolné celé číslo větší než 1, které je nesoudělné s $phi$. Dvě celá čísla jsou nesoudělná, pokud je jejich <DefinitionTooltip definition="Největší společný dělitel – největší kladné celé číslo, které dělí dvě nebo více zadaných celých čísel.">největší společný dělitel</DefinitionTooltip> roven 1. Proto použijeme funkci `math.gcd` k nalezení celého čísla $e$ nesoudělného s $phi$.

```python
# Choose an integer e such that e and φ(n) are coprime
e = 2
while e < phi:
    if math.gcd(e, phi) == 1:
        break
    else:
        e += 1
print("Public Key (e):", e)
```

Soukromý klíč vyžaduje celé číslo $d$, které je <DefinitionTooltip definition="Také známé jako reciproká hodnota – číslo, které při vynásobení daným číslem dá výsledek 1.">multiplikativním inverzem</DefinitionTooltip> $e$ modulo $phi$; jinými slovy, splňuje <DefinitionTooltip definition="Konzistence nebo ekvivalence, tj. chování stejným způsobem.">kongruenční vztah</DefinitionTooltip> $d*e\equiv 1 \pmod{\varphi(n)}$. Pro tuto jednoduchou ilustraci, kde pracujeme s malými celými čísly, můžeme jednoduše procházet kladná celá čísla a hledat vhodné $d$. V reálných situacích se k tomuto účelu používá výpočetně efektivní <DefinitionTooltip definition="Efektivní způsob výpočtu největšího společného dělitele dvou celých čísel. Vylepšuje původní Eukleidův algoritmus.">rozšířený Eukleidův algoritmus</DefinitionTooltip>.

```python
# Compute a value for d such that (d * e) % φ(n) = 1
d = 1
while True:
    if (d * e) % phi == 1:
        break
    else:
        d += 1
print("Private Key (d):", d)
```

Nyní vytvoříme dvojice $(e, n), (d, n)$, které tvoří veřejný a soukromý klíč. Veřejný klíč se poté zveřejní, zatímco soukromý klíč zůstane tajný.

```python
# Public and Private Key pair
public = (e, n)
private = (d, n)

print(f"The Public key is {public} and Private Key is {private}")
```

Šifrování a dešifrování v RSA používají [operaci modulárního umocňování](https://en.wikipedia.org/w/index.php?title=Modular_exponentiation&oldid=1160764843). Veřejný a soukromý klíč jsou navíc komplementární v tom smyslu, že kterýkoli z nich lze použít k zašifrování zprávy, kterou pak ten druhý dokáže dešifrovat.

Zde ilustrujeme případ, kdy se veřejný klíč $(e,n)$ používá pro šifrování a soukromý klíč $(d, n)$ pro dešifrování – pro každou operaci definujeme příslušnou funkci v Pythonu.

Poté zašifrujeme a dešifrujeme celočíselnou zprávu $msg$.

```python
# Encryption function
def encrypt(plain_text):
    return (plain_text**e) % n

# Decryption function
def decrypt(cipher_text):
    return (cipher_text**d) % n

# Simple message to encode
msg = 9

# encrypt then decrypt
enc_msg = encrypt(msg)
dec_msg = decrypt(enc_msg)

print("Original Message:", msg)
print("Encrypted Message:", enc_msg)
print("Decrypted Message:", dec_msg)
```

Přestože jsou malá celá čísla použitá výše užitečná pro snadné nastínění základních myšlenek RSA algoritmu, v reálných aplikacích RSA vyžaduje použití velmi velkých celých čísel. Například RSA s 2048 bity pracuje s <DefinitionTooltip definition="Modulus je zbytek po dělení zadanou hodnotou. Modulo je operace, která toto provádí.">modulem</DefinitionTooltip> $n$ o délce 2048 bitů, jehož desetinné ekvivalenty jsou přibližně 10$^616$. Tato opravdu obrovská čísla jsou nezbytná pro praktickou bezpečnost RSA.

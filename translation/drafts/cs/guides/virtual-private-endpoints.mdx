# Použití virtuálních privátních koncových bodů pro VPC pro soukromé připojení k IBM Quantum Platform

Použij Virtual Private Endpoints (VPE) pro VPC a připoj se k IBM Quantum&reg; ze své sítě VPC pomocí IP adresy, kterou sám určíš a která je přidělena z podsítě ve tvém VPC.

VPE jsou virtuální IP rozhraní vázaná na endpoint gateway vytvořenou pro každou službu zvlášť. Endpoint gateway je virtualizovaná funkce, která se horizontálně škáluje, je redundantní, vysoce dostupná a pokrývá všechny zóny dostupnosti tvého VPC. Endpoint gateways umožňují komunikaci z instancí virtuálních serverů v rámci tvého VPC a služby IBM Cloud&reg; přes privátní páteřní síť. VPE pro VPC ti umožňuje řídit veškeré privátní adresování v rámci tvého cloudu. Více informací najdeš v části [About virtual private endpoint gateways](https://cloud.ibm.com/docs/vpc?topic=vpc-about-vpe).

V rámci IBM Quantum Platform jsou veškerá zákaznická data přenášena přes privátní síť bez ohledu na to, zda je k nim přistupováno přes veřejný koncový bod nebo VPE.

Připojení k IBM Quantum Platform přes veřejné koncové body přenáší veškerá data požadavků a odpovědí přes veřejný internet. Pro připojení k IBM Quantum Platform pomocí VPE musíš použít IBM Quantum Platform API nebo SDK. K IBM Quantum Platform lze přistupovat pouze přes veřejnou síť.

## Nastavení VPE pro IBM Quantum Platform {#set-up-a-vpe-for-ibm-quantum-platform}

Existuje několik způsobů, jak vytvořit VPE gateway. Pokud používáš [CLI](https://cloud.ibm.com/docs/vpc?topic=vpc-vpc-reference&interface=cli#vpe-clis) nebo [API](https://cloud.ibm.com/apidocs/vpc/latest#create-endpoint-gateway), musíš zadat [Cloud Resource Name (CRN)](https://cloud.ibm.com/docs/account?topic=account-crn) regionu, ke kterému se chceš připojit v IBM Quantum Platform. Dostupné regiony a CRN najdeš v následující tabulce.

| Region | Plány   | Plně kvalifikovaný název domény (FDQN) | Cloud Resource Name (CRN) |
|-----------------|-----------------|-----------------|-----------------|
| `us-east` | `Standard` | `private.us-east.quantum.cloud.ibm.com` | `crn:v1:bluemix:public:quantum-computing:us-east:::endpoint:private.us-east.quantum.cloud.ibm.com` |
| `eu-de` | `Standard` | `private.eu-de.quantum.cloud.ibm.com` | `crn:v1:bluemix:public:quantum-computing:eu-de:::endpoint:private.eu-de.quantum.cloud.ibm.com` |

### Konfigurace endpoint gateway {#configuring-an-endpoint-gateway}

Při konfiguraci VPE gateway postupuj podle těchto kroků:

1. Zobraz seznam dostupných služeb, včetně infrastrukturních služeb IBM Cloud dostupných (ve výchozím nastavení) pro všechny uživatele VPC.
1. [Vytvoř endpoint gateway](https://cloud.ibm.com/docs/vpc?topic=vpc-ordering-endpoint-gateway) pro IBM Quantum Platform, která má být soukromě dostupná pro VPC.
1. [Přiřaď rezervovanou IP adresu](https://cloud.ibm.com/docs/vpc?topic=vpc-bind-unbind-reserved-ip) k endpoint gateway.
1. [Zobraz vytvořené VPE gateways](https://cloud.ibm.com/docs/vpc?topic=vpc-vpe-viewing-details-of-an-endpoint-gateway) přidružené k IBM Quantum Platform.

Nyní mohou instance virtuálních serverů v tvém VPC přistupovat k instanci IBM Quantum Platform soukromě.

## Použití VPE pro IBM Quantum Platform {#use-your-vpe-for-ibm-quantum-platform}

Po vytvoření endpoint gateway pro IBM Quantum Platform postupuj podle těchto kroků:

### Použití VPE s `qiskit-ibm-runtime` (Python SDK) {#use-the-vpe-with-qiskit-ibm-runtime-python-sdk}

<Admonition type="note">
Podpora VPE vyžaduje `qiskit-ibm-runtime` v0.24.0 nebo novější.
</Admonition>

Při vytváření instance `QiskitRuntimeService` zadej `private_endpoint=True`.

```python
service = QiskitRuntimeService(token="your-API_KEY", # Use the 44-character API_KEY you created and saved from the IBM Quantum Platform Home dashboard
                              instance="INSTANCE_CRN",
                              channel="ibm_cloud",
                              private_endpoint=True)
```

### Použití VPE s IBM Quantum API {#use-the-vpe-with-the-ibm-quantum-api}

Po vytvoření endpoint gateway pro IBM Quantum Platform použij FQDN koncového bodu služby pro cílový region.

```bash
  curl -X POST https://private.us-east.quantum.cloud.ibm.com/api/v1/jobs -H "Authorization: Bearer $BEARER_TOKEN" -H "Service-CRN: $SERVICE_INSTANCE_CRN" -d '{
    "backend": "backend",
    "program_id": "sampler"
  }'
```

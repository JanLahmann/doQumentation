## qiskit.tools {#qiskittools}

Modul `qiskit.tools` byl v Qiskit 1.0 odstraněn. Většina jeho funkcionality byla
buď nahrazena podobnou funkcionalitou v jiných balíčcích, nebo odstraněna bez náhrady.
Hlavní výjimkou je funkce `qiskit.tools.parallel_map()`, která byla přesunuta do
modulu [`qiskit.utils`](https://docs.quantum.ibm.com/api/qiskit/utils#parallel_map). Místo toho ji můžeš používat
z tohoto nového umístění. Například:

Pokud jsi dříve používal/a:

```python
# Previous
from qiskit.tools import parallel_map

parallel_map(func, input)

# Current
from qiskit.utils import parallel_map

parallel_map(func, input)
```

<span id="jupyter"></span>
### jupyter {#jupyter}

Submodul `qiskit.tools.jupyter` byl odstraněn, protože jeho funkcionalita
je svázána se starším balíčkem `qiskit-ibmq-provider`, který již není
podporován. Také podporoval pouze `BackendV1`, nikoli novější
rozhraní [`BackendV2`](https://docs.quantum.ibm.com/api/qiskit/qiskit.providers.BackendV2).

<span id="monitor"></span>
### monitor {#monitor}

Submodul `qiskit.tools.monitor` byl odstraněn, protože byl svázán se
starším balíčkem `qiskit-ibmq-provider`, který již není podporován (také
podporoval pouze rozhraní `BackendV1`, nikoli novější
rozhraní [`BackendV2`](https://docs.quantum.ibm.com/api/qiskit/qiskit.providers.BackendV2)). Pro tuto funkcionalitu
není k dispozici žádná náhrada.

<span id="visualization"></span>
### visualization {#visualization}

Submodul `qiskit.tools.visualization` byl odstraněn. Tento modul byl
starším přesměrováním z původního umístění vizualizačního modulu Qiskit, který byl
přesunut do [`qiskit.visualization`](https://docs.quantum.ibm.com/api/qiskit/visualization) v Qiskit
0.8.0. Pokud tuto cestu stále používáš, aktualizuj své importy z
`qiskit.tools.visualization` na
[`qiskit.visualization`](https://docs.quantum.ibm.com/api/qiskit/visualization).

```python
# Previous
from qiskit.tools.visualization import plot_histogram

plot_histogram(counts)

# Current
from qiskit.visualization import plot_histogram

plot_histogram(counts)
```

<span id="events"></span>
### events {#events}

Modul `qiskit.tools.events` a utilita `progressbar()`, kterou zpřístupňoval,
byly odstraněny. Funkcionalita tohoto modulu nebyla příliš využívána a je
lépe pokryta specializovanými balíčky, jako je
[tqdm](https://github.com/tqdm/tqdm).

## qiskit.transpiler {#qiskittranspiler}

<span id="synthesis"></span>
### synthesis {#synthesis}

Položky v modulu `qiskit.transpiler.synthesis` byly přesunuty na nová umístění:

Odstraněno    | Alternativa
-----------|------------
`qiskit.transpiler.synthesis.aqc` (kromě [`AQCSynthesisPlugin`](https://docs.quantum.ibm.com/api/qiskit/qiskit.transpiler.passes.synthesis.aqc_plugin.AQCSynthesisPlugin)) | [`qiskit.synthesis.unitary.aqc`](https://docs.quantum.ibm.com/api/qiskit/qiskit.synthesis.unitary.aqc)
`qiskit.transpiler.synthesis.graysynth` | [`qiskit.synthesis.synth_cnot_phase_aam`](https://docs.quantum.ibm.com/api/qiskit/synthesis#synth_cnot_phase_aam)
`qiskit.transpiler.synthesis.cnot_synth` | [`qiskit.synthesis.synth_cnot_count_full_pmh`](https://docs.quantum.ibm.com/api/qiskit/synthesis#synth_cnot_count_full_pmh)

<span id="transpiler.passes"></span>
### passes {#passes}

Transpiler pass `NoiseAdaptiveLayout` byl nahrazen
[`VF2Layout`](https://docs.quantum.ibm.com/api/qiskit/qiskit.transpiler.passes.VF2Layout) a
[`VF2PostLayout`](https://docs.quantum.ibm.com/api/qiskit/qiskit.transpiler.passes.VF2PostLayout),
které nastavují rozvržení na základě hlášených charakteristik šumu
backendu. Jak tento pass, tak odpovídající plugin fáze rozvržení `"noise_adaptive"`
byly z Qiskitu odstraněny.

Transpiler pass `CrosstalkAdaptiveSchedule` byl odstraněn ze
zdrojového kódu. Tento pass již nebylo možné používat, protože jeho vnitřní operace
závisela na vlastních vlastnostech nastavených v payloadu `BackendProperties`
instance `BackendV1`. Protože žádné backendy tato pole nenastavují, byl pass
odstraněn.

<span id="passmanager"></span>
### passmanager {#passmanager}

Metody `append` tříd
[`ConditionalController`](https://docs.quantum.ibm.com/api/qiskit/qiskit.passmanager.ConditionalController),
[`FlowControllerLinear`](https://docs.quantum.ibm.com/api/qiskit/qiskit.passmanager.FlowControllerLinear) a
[`DoWhileController`](https://docs.quantum.ibm.com/api/qiskit/qiskit.passmanager.DoWhileController)
byly odstraněny. Místo toho musí být všechny úlohy poskytnuty při vytváření
objektů kontroléru.

<span id="qiskit.utils"></span>
## qiskit.utils {#qiskitutils}

Následující nástroje v `qiskit.utils` byly odstraněny bez náhrady:
  * `qiskit.utils.arithmetic`
  * `qiskit.utils.circuit_utils`
  * `qiskit.utils.entangler_map`
  * `qiskit.utils.name_unnamed_args`

Tyto funkce byly používány výhradně v modulech `qiskit.algorithms` a
`qiskit.opflow`, které byly také odstraněny.

<span id="qiskit.visualization"></span>
## qiskit.visualization {#qiskitvisualization}

Modul `qiskit.visualization.qcstyle` byl odstraněn. Jako přímou náhradu použij
`qiskit.visualization.circuit.qcstyle`.

# Použití Qiskit Code Assistant v lokálním režimu

Zjisti, jak nainstalovat, nakonfigurovat a používat libovolný model Qiskit Code Assistant na svém lokálním počítači.

<Admonition type="note" title="Poznámky">
    - Qiskit Code Assistant je ve stavu náhledového vydání a může se změnit.
    - Pokud máš zpětnou vazbu nebo chceš kontaktovat vývojářský tým, použij [kanál Qiskit Slack Workspace](https://qiskit.enterprise.slack.com/archives/C07LYA6PL83) nebo příslušné veřejné repozitáře na GitHubu.
</Admonition>

## Rychlý start (doporučeno) {#rychlý-start-doporučeno}

Nejjednodušší způsob, jak začít s Qiskit Code Assistant v lokálním režimu, je použít automatizované instalační skripty pro rozšíření VS Code nebo JupyterLab. Tyto skripty automaticky nainstalují [Ollama](https://ollama.com/) pro spuštění LLM, stáhnou doporučený model a nakonfigurují za tebe rozšíření.

### Nastavení rozšíření VS Code {#nastavení-rozšíření-vs-code}

Spusť v terminálu následující příkaz:

```bash
bash <(curl -fsSL https://raw.githubusercontent.com/Qiskit/qiskit-code-assistant-vscode/main/setup_local.sh)
```

Tento skript provede následující kroky:
- Nainstaluje Ollama (pokud ještě není nainstalováno)
- Stáhne a nakonfiguruje doporučený model Qiskit Code Assistant
- Nastaví rozšíření VS Code pro práci s tvým lokálním nasazením

### Nastavení rozšíření JupyterLab {#nastavení-rozšíření-jupyterlab}

Spusť v terminálu následující příkaz:

```bash
bash <(curl -fsSL https://raw.githubusercontent.com/Qiskit/qiskit-code-assistant-jupyterlab/main/setup_local.sh)
```

Tento skript:
- Nainstaluje Ollama (pokud ještě není nainstalováno)
- Stáhne a nakonfiguruje doporučený model Qiskit Code Assistant
- Nastaví rozšíření JupyterLab pro práci s tvým lokálním nasazením

<summary>Ruční nasazení modelů Qiskit Code Assistant lokálně přes Ollama</summary>

Existuje více způsobů, jak nasadit stažený model Qiskit Code Assistant a pracovat s ním. Tato příručka ukazuje použití [Ollama](https://ollama.com) takto: buď pomocí [aplikace Ollama](#using-the-ollama-application) prostřednictvím integrace Hugging Face Hub nebo lokálního modelu, nebo pomocí [balíčku `llama-cpp-python`](#use-llama-cpp-python).

### Použití aplikace Ollama {#using-the-ollama-application}

Aplikace Ollama poskytuje jednoduché řešení pro spouštění LLM lokálně. Je snadná na použití – díky CLI je celý proces nastavení, správa modelů a interakce poměrně přímočará. Je ideální pro rychlé experimenty a pro uživatele, kteří chtějí řešit méně technických detailů.

#### Instalace Ollama {#instalace-ollama}

1. Stáhni [aplikaci Ollama](https://ollama.com/download)
1. Nainstaluj stažený soubor
1. Spusť nainstalovanou aplikaci Ollama

    <Admonition type="info">Aplikace běží úspěšně, když se ikona Ollama zobrazí v panelu nabídek na ploše. Spuštění služby můžeš také ověřit přechodem na `http://localhost:11434/`.</Admonition>

1. Vyzkoušej Ollama v terminálu a začni spouštět modely. Například:

    ```
    ollama run hf.co/Qiskit/Qwen2.5-Coder-14B-Qiskit
    ```

#### Nastavení Ollama pomocí integrace Hugging Face Hub {#nastavení-ollama-pomocí-integrace-hugging-face-hub}

[Integrace Ollama/Hugging Face Hub](https://huggingface.co/docs/hub/ollama) umožňuje pracovat s modely hostovanými na Hugging Face Hub bez nutnosti vytvářet nový modelfile nebo ručně stahovat soubory GGUF nebo safetensors. Výchozí soubory `template` a `params` jsou pro model na Hugging Face Hub již součástí balení.

1. Ujisti se, že aplikace Ollama běží.

1. Přejdi na stránku požadovaného modelu a zkopíruj URL. Například https://huggingface.co/Qiskit/Qwen2.5-Coder-14B-Qiskit-GGUF.

1. V terminálu spusť příkaz:

    ```
    ollama run hf.co/Qiskit/Qwen2.5-Coder-14B-Qiskit
    ```

Můžeš použít model `hf.co/Qiskit/Qwen2.5-Coder-14B-Qiskit` nebo jakýkoli jiný aktuálně doporučený oficiální model GGUF: `hf.co/Qiskit/mistral-small-3.2-24b-qiskit-GGUF` nebo `hf.co/Qiskit/granite-3.3-8b-qiskit-GGUF`.

#### Nastavení Ollama s ručně staženým modelem GGUF pro Qiskit Code Assistant {#nastavení-ollama-s-ručně-staženým-modelem-gguf}

Pokud jsi ručně stáhl(a) model GGUF, například https://huggingface.co/Qiskit/Qwen2.5-Coder-14B-Qiskit-GGUF, a chceš experimentovat s různými šablonami a parametry, můžeš jej načíst do lokální aplikace Ollama podle těchto kroků.

1. Vytvoř soubor `Modelfile` s následujícím obsahem a nezapomeň nahradit `<PATH-TO-GGUF-FILE>` skutečnou cestou ke staženému modelu.

    ```
    FROM <PATH-TO-GGUF-FILE>
    TEMPLATE """{{ if .System }}
    System:
    {{ .System }}

    {{ end }}{{ if .Prompt }}Question:
    {{ .Prompt }}

    {{ end }}Answer:
    ```python{{ .Response }}
    """

    PARAMETER stop "Question:"
    PARAMETER stop "Answer:"
    PARAMETER stop "System:"
    PARAMETER stop "```"

    PARAMETER temperature 0
    PARAMETER top_k 1
    ```

1. Spusť následující příkaz pro vytvoření vlastní instance modelu na základě souboru `Modelfile`.

    ```
    ollama create Qwen2.5-Coder-14B-Qiskit -f ./path-to-model-file
    ```

    <Admonition type="note">Tento proces může chvíli trvat, než Ollama přečte soubor modelu, inicializuje instanci modelu a nakonfiguruje ji podle zadaných specifikací.</Admonition>

#### Spuštění ručně staženého modelu Qiskit Code Assistant v Ollama {#spuštění-modelu-qiskit-code-assistant-v-ollama}

Jakmile je model `Qwen2.5-Coder-14B-Qiskit` nastaven v Ollama, spusť jej následujícím příkazem a interaguj s ním v terminálu (v režimu chatu).

```
ollama run Qwen2.5-Coder-14B-Qiskit
```

Užitečné příkazy:

- `ollama list` – Zobrazí seznam modelů na tvém počítači
- `ollama rm Qwen2.5-Coder-14B-Qiskit` – Smaže model
- `ollama show Qwen2.5-Coder-14B-Qiskit` – Zobrazí informace o modelu
- `ollama stop Qwen2.5-Coder-14B-Qiskit` – Zastaví aktuálně běžící model
- `ollama ps` – Zobrazí seznam aktuálně načtených modelů

<summary>Ruční nasazení modelů Qiskit Code Assistant lokálně přes balíček `llama-cpp-python`</summary>

<span id="use-llama-cpp-python"></span>
Alternativou k aplikaci Ollama je balíček `llama-cpp-python`, což je Python binding pro `llama.cpp`. Poskytuje větší kontrolu a flexibilitu při lokálním spouštění modelu GGUF a je ideální pro uživatele, kteří chtějí integrovat lokální model do svých pracovních postupů a Python aplikací.

1. Nainstaluj [`llama-cpp-python`](https://pypi.org/project/llama-cpp-python/)
2. Interaguj s modelem ze své aplikace pomocí `llama_cpp`. Například:

```python
from llama_cpp import Llama

model_path = <PATH-TO-GGUF-FILE>

model = Llama(
        model_path,
        seed=17,
        n_ctx=10000,
        n_gpu_layers=37, # to offload in gpu, but put 0 if all in cpu
    )

input = 'Generate a quantum circuit with 2 qubits'
raw_pred = model(input)["choices"][0]["text"]
```

Můžeš také přidat parametry generování textu do modelu pro přizpůsobení inference:

```python
generation_kwargs = {
        "max_tokens": 512,
        "echo": False, # Echo the prompt in the output
        "top_k": 1
    }

raw_pred = model(input, **generation_kwargs)["choices"][0]["text"]
```

</details>

### Použití knihovny `llama.cpp` {#použití-knihovny-llama-cpp}

Další alternativou je použití `llama.cpp`, open-source knihovny pro provádění inference LLM na CPU s minimálním nastavením.
Poskytuje nízkoúrovňovou kontrolu nad spouštěním modelu a obvykle se spouští z příkazové řádky s odkazem na lokální soubor modelu GGUF.

Existuje několik způsobů instalace `llama.cpp` na tvůj počítač:

- Nainstaluj llama.cpp pomocí [brew, nix nebo winget](https://github.com/ggml-org/llama.cpp/blob/master/docs/install.md)
- Spusť přes Docker: Viz [dokumentaci Dockeru od týmu `llama.cpp`](https://github.com/ggml-org/llama.cpp/blob/master/docs/docker.md)
- Stáhni předkompilované binární soubory ze [stránky vydání](https://github.com/ggml-org/llama.cpp/releases)
- Sestav ze zdrojového kódu [klonováním tohoto repozitáře](https://github.com/ggml-org/llama.cpp/blob/master/docs/build.md)

Po instalaci můžeš použít `llama.cpp` k interakci s modely GGUF v konverzačním režimu takto:
```bash
# Use a local model file
llama-cli -m my_model.gguf -cnv

# Or download and run a model directly from Hugging Face
llama-cli -hf Qiskit/Qwen2.5-Coder-14B-Qiskit-GGUF -cnv
```

Můžeš také spustit server s API kompatibilním s OpenAI pro model tímto způsobem:
```bash
llama-server -hf Qiskit/Qwen2.5-Coder-14B-Qiskit-GGUF
```

#### Pokročilé parametry {#pokročilé-parametry}
Pomocí programu `llama-cli` můžeš ovládat generování modelu prostřednictvím voleb příkazové řádky. Například můžeš zadat počáteční „systémový" prompt pomocí příznaku `-p/--prompt`. V konverzačním režimu (`-cnv`) tento počáteční prompt slouží jako systémová zpráva. Jinak můžeš jednoduše přidat libovolnou požadovanou instrukci na začátek textu promptu. Můžeš také upravovat parametry vzorkování – například: teplotu (`--temp`), top-k (`--top-k`), top-p (`--top-p`), penalizaci opakování (`--repeat-penalty`) a seed (`--seed`). Níže je příklad volání s těmito volbami:

```bash
llama-cli -hf Qiskit/Qwen2.5-Coder-14B-Qiskit-GGUF \
  -p "You are a friendly assistant." -cnv \
  --temp 0.7 \
  --top-k 50 \
  --top-p 0.95 \
  --repeat-penalty 1.1 \
  --seed 42
```

Pro zajištění správné funkčnosti našich modelů Qiskit doporučujeme použít systémový prompt uvedený v našich repozitářích HF GGUF: systémový prompt pro [mistral-small-3.2-24b-qiskit-GGUF](https://huggingface.co/Qiskit/mistral-small-3.2-24b-qiskit-GGUF/blob/main/system), [Qwen2.5-Coder-14B-Qiskit-GGUF](https://huggingface.co/Qiskit/Qwen2.5-Coder-14B-Qiskit-GGUF/blob/main/system), [granite-3.3-8b-qiskit-GGUF](https://huggingface.co/Qiskit/granite-3.3-8b-qiskit-GGUF/blob/main/system) a [granite-3.2-8b-qiskit-GGUF.](https://huggingface.co/Qiskit/granite-3.2-8b-qiskit-GGUF/blob/main/system)

<summary>Ruční připojení rozšíření k lokálnímu nasazení</summary>

Pro odesílání promptů lokálně nasazenému modelu Qiskit Code Assistant použij rozšíření VS Code a JupyterLab. Jakmile máš aplikaci Ollama [nastavenou s modelem](#using-the-ollama-application), můžeš nakonfigurovat rozšíření pro připojení k lokální službě.

### Připojení pomocí rozšíření Qiskit Code Assistant pro VS Code {#připojení-pomocí-rozšíření-qiskit-code-assistant-pro-vs-code}

Pomocí rozšíření Qiskit Code Assistant pro VS Code můžeš interagovat s modelem a provádět doplňování kódu při jeho psaní. To se dobře hodí pro uživatele, kteří hledají pomoc při psaní kódu Qiskit pro své Python aplikace.

1. Nainstaluj [rozšíření Qiskit Code Assistant pro VS Code](/guides/qiskit-code-assistant-vscode).
1. Ve VS Code přejdi do **Uživatelského nastavení** a nastav **Qiskit Code Assistant: Url** na URL tvého lokálního nasazení Ollama (například `http://localhost:11434`).
1. Znovu načti VS Code přechodem na **Zobrazení > Paleta příkazů...** a výběrem **Developer: Reload Window**.

Model Qiskit Code Assistant nakonfigurovaný v Ollama by se měl zobrazit ve stavovém řádku a je připraven k použití.

### Připojení pomocí rozšíření Qiskit Code Assistant pro JupyterLab {#připojení-pomocí-rozšíření-qiskit-code-assistant-pro-jupyterlab}

Pomocí rozšíření Qiskit Code Assistant pro JupyterLab můžeš interagovat s modelem a provádět doplňování kódu přímo v Jupyter Notebooku. Uživatelé, kteří převážně pracují s Jupyter Notebooky, mohou využít toto rozšíření k dalšímu vylepšení svého zážitku při psaní kódu Qiskit.

1. Nainstaluj [rozšíření Qiskit Code Assistant pro JupyterLab](/guides/qiskit-code-assistant-jupyterlab).
1. V JupyterLab přejdi do **Editoru nastavení** a nastav **Qiskit Code Assistant Service API** na URL tvého lokálního nasazení Ollama (například `http://localhost:11434`).

Model Qiskit Code Assistant nakonfigurovaný v Ollama by se měl zobrazit ve stavovém řádku a je připraven k použití.

</details>

### builder.build {#builderbuild}

Následující argumenty funkce `qiskit.pulse.builder.build` byly odstraněny
bez náhrady.

* `default_transpiler_settings`
* `default_circuit_scheduler_settings`

Byly odstraněny také tyto funkce:

* `qiskit.pulse.builder.active_transpiler_settings`
* `qiskit.pulse.builder.active_circuit_scheduler_settings`
* `qiskit.pulse.builder.transpiler_settings`
* `qiskit.pulse.builder.circuit_scheduler_settings`

Důvodem je to, že do kontextu builderu již není možné vkládat objekty Circuit
(viz [Vkládání operací Circuit Gate](#injecting-circuit-gate-operations));
tato nastavení sloužila k převodu vložených objektů do podoby pulsů.

<span id="qiskit.qasm"></span>
## qiskit.qasm {#qiskitqasm}

Starší modul parseru OpenQASM 2, který byl dříve součástí `qiskit.qasm`,
byl nahrazen modulem [`qiskit.qasm2`](https://docs.quantum.ibm.com/api/qiskit/qasm2), jenž
poskytuje rychlejší a přesnější parser pro OpenQASM 2. Vysokoúrovňové
metody třídy `QuantumCircuit`
[`from_qasm_file()`](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.QuantumCircuit#from_qasm_file)
a
[`from_qasm_str()`](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.QuantumCircuit#from_qasm_str)
zůstávají stejné, ale interně budou používat nový parser. Veřejné rozhraní
modulu `qasm2` však není totožné. Zatímco modul `qiskit.qasm` poskytoval
rozhraní k abstraktnímu syntaktickému stromu vráceného knihovnou parseru `ply`,
modul `qiskit.qasm2` AST ani žádné nižší implementační detaily parseru
nezveřejňuje. Místo toho přijímá vstup ve formátu OpenQASM 2 a na výstupu
vrací objekt [`QuantumCircuit`](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.QuantumCircuit).

Pokud jsi například dříve spouštěl něco jako toto:

```python
import qiskit.qasm
from qiskit.converters import ast_to_dag, dag_to_circuit

ast = qiskit.qasm.Qasm(filename="myfile.qasm").parse()
dag = ast_to_dag(ast)
qasm_circ = dag_to_circuit(dag)
```

Nahraď to následujícím:

```python
import qiskit.qasm2

qasm_circ = qiskit.qasm2.load("myfile.qasm")
```

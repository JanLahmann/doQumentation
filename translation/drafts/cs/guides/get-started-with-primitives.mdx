## Začínáme s Estimatorem {#začínáme-s-estimatorem}

{/*Verified the v2 examples 2/29/24 - updated 10/29/24*/}

### 1. Inicializace účtu {#1-inicializace-účtu}

Protože Qiskit Runtime Estimator je spravovaná služba, musíš nejprve inicializovat svůj účet. Poté můžeš vybrat QPU, které chceš použít k výpočtu střední hodnoty.

Pokud ještě nemáš účet, postupuj podle kroků v tématu [Instalace a nastavení](install-qiskit).

```python
from qiskit_ibm_runtime import QiskitRuntimeService

service = QiskitRuntimeService()
backend = service.least_busy(
    operational=True, simulator=False, min_num_qubits=127
)

print(backend.name)
```

```text
ibm_torino
```

### 2. Vytvoření Circuit a observovatelné veličiny {#2-vytvoření-circuit-a-observovatelné-veličiny}

Jako vstup pro primitiv Estimator potřebuješ alespoň jeden Circuit a jednu observovatelnou veličinu.

```python
from qiskit.circuit.library import qaoa_ansatz
from qiskit.quantum_info import SparsePauliOp

entanglement = [tuple(edge) for edge in backend.coupling_map.get_edges()]
observable = SparsePauliOp.from_sparse_list(
    [("ZZ", [i, j], 0.5) for i, j in entanglement],
    num_qubits=backend.num_qubits,
)
circuit = qaoa_ansatz(observable, reps=2)
# the circuit is parametrized, so we will define the parameter values for execution
param_values = [0.1, 0.2, 0.3, 0.4]

print(f">>> Observable: {observable.paulis}")
```

Circuit a observovatelná veličina musí být transformovány tak, aby používaly pouze instrukce podporované QPU (označované jako *instrukční sada architektury (ISA)* Circuit). K tomu použijeme Transpiler.

```python
from qiskit.transpiler import generate_preset_pass_manager

pm = generate_preset_pass_manager(optimization_level=1, backend=backend)
isa_circuit = pm.run(circuit)
isa_observable = observable.apply_layout(isa_circuit.layout)
print(f">>> Circuit ops (ISA): {isa_circuit.count_ops()}")
```

```text
>>> Circuit ops (ISA): OrderedDict([('rz', 3826), ('sx', 1601), ('cz', 968)])
```

### 3. Inicializace Qiskit Runtime Estimatoru {#3-inicializace-qiskit-runtime-estimatoru}

Při inicializaci Estimatoru použij parametr `mode` k určení režimu, ve kterém má běžet. Možné hodnoty jsou objekty `batch`, `session` nebo `backend` pro dávkový, Session a úlohový režim spouštění. Více informací najdeš v tématu [Úvod do režimů spouštění Qiskit Runtime.](execution-modes)

```python
from qiskit_ibm_runtime import EstimatorV2 as Estimator

estimator = Estimator(mode=backend)
```

### 4. Volání Estimatoru a získání výsledků {#4-volání-estimatoru-a-získání-výsledků}

Dále zavolej metodu `run()` k výpočtu středních hodnot pro vstupní Circuit a observovatelné veličiny. Circuit, observovatelná veličina a volitelné sady hodnot parametrů se zadávají jako n-tice *primitive unified bloc* (PUB).

```python
job = estimator.run([(isa_circuit, isa_observable, param_values)])
print(f">>> Job ID: {job.job_id()}")
print(f">>> Job Status: {job.status()}")
```



```python
result = job.result()
print(f">>> {result}")

```

## Začínáme se Samplerem {#začínáme-se-samplerem}

### 1. Inicializace účtu {#1-inicializace-účtu-sampler}

Protože Qiskit Runtime Sampler je spravovaná služba, musíš nejprve inicializovat svůj účet. Poté můžeš vybrat QPU, které chceš použít k výpočtu střední hodnoty.

Pokud ještě nemáš nastavený účet, postupuj podle kroků v tématu [Instalace a nastavení](install-qiskit).

```python
from qiskit_ibm_runtime import QiskitRuntimeService

service = QiskitRuntimeService()
backend = service.least_busy(
    operational=True, simulator=False, min_num_qubits=127
)
```

### 2. Vytvoření Circuit {#2-vytvoření-circuit}

Jako vstup pro primitiv Sampler potřebuješ alespoň jeden Circuit.

```python
import numpy as np
from qiskit.circuit.library import efficient_su2

circuit = efficient_su2(127, entanglement="linear")
circuit.measure_all()
# The circuit is parametrized, so we will define the parameter values for execution
param_values = np.random.rand(circuit.num_parameters)
```

Použij Transpiler k získání ISA Circuit.

```python
from qiskit.transpiler import generate_preset_pass_manager

pm = generate_preset_pass_manager(optimization_level=1, backend=backend)
isa_circuit = pm.run(circuit)
print(f">>> Circuit ops (ISA): {isa_circuit.count_ops()}")
```

```text
>>> Circuit ops (ISA): OrderedDict([('sx', 3089), ('rz', 3036), ('cz', 1092), ('measure', 127), ('barrier', 1)])
```

### 3. Inicializace Qiskit Runtime Sampleru {#3-inicializace-qiskit-runtime-sampleru}

Při inicializaci Sampleru použij parametr `mode` k určení režimu, ve kterém má běžet. Možné hodnoty jsou objekty `batch`, `session` nebo `backend` pro dávkový, Session a úlohový režim spouštění. Více informací najdeš v tématu [Úvod do režimů spouštění Qiskit Runtime.](execution-modes) Uživatelé Open Plan nemohou odesílat Session úlohy.

```python
from qiskit_ibm_runtime import SamplerV2 as Sampler

sampler = Sampler(mode=backend)
```

### 4. Volání Sampleru a získání výsledků {#4-volání-sampleru-a-získání-výsledků}

Dále zavolej metodu `run()` k vygenerování výstupu. Circuit a volitelné sady hodnot parametrů se zadávají jako n-tice *primitive unified bloc* (PUB).

```python
job = sampler.run([(isa_circuit, param_values)])
print(f">>> Job ID: {job.job_id()}")
print(f">>> Job Status: {job.status()}")
```

```text

>>> Job Status: QUEUED
```

```python
result = job.result()
```

## Začínáme s backendovými primitivy {#začínáme-s-backendovými-primitivy}

Na rozdíl od primitiv specifických pro poskytovatele jsou backendové primitivy obecné implementace, které lze použít s libovolným
objektem `backend`, pokud implementuje rozhraní [`Backend`](https://docs.quantum.ibm.com/api/qiskit/qiskit.providers.Backend).

- Primitiv Sampler lze spustit u libovolného poskytovatele pomocí [`qiskit.primitives.BackendSamplerV2`](https://docs.quantum.ibm.com/api/qiskit/qiskit.primitives.BackendSamplerV2).
- Primitiv Estimator lze spustit u libovolného poskytovatele pomocí [`qiskit.primitives.BackendEstimatorV2`](../api/qiskit/qiskit.primitives.BackendEstimatorV2).

Někteří poskytovatelé implementují primitivy nativně. Podrobnosti najdeš na [stránce ekosystému Qiskit](https://qiskit.github.io/ecosystem#provider).
### Příklad: BackendEstimator {#příklad-backendestimator}

```python
from qiskit.primitives import BackendEstimatorV2
from <some_qiskit_provider> import QiskitProvider

provider = QiskitProvider()
backend = provider.get_backend('backend_name')
estimator = BackendEstimatorV2(backend)
```

### Příklad: BackendSampler {#příklad-backendsampler}

```python
from qiskit.primitives import BackendSamplerV2
from <some_qiskit_provider> import QiskitProvider

provider = QiskitProvider()
backend = provider.get_backend('backend_name')
sampler = BackendSamplerV2(backend)
```
### Podobnosti a rozdíly mezi backendovými a Runtime primitivy {#podobnosti-a-rozdíly-mezi-backendovými-a-runtime-primitivy}

- Vstupy a výstupy [`qiskit.primitives.BackendSamplerV2`](https://docs.quantum.ibm.com/api/qiskit/qiskit.primitives.BackendSamplerV2) a [`qiskit.primitives.BackendEstimatorV2`](../api/qiskit/qiskit.primitives.BackendEstimatorV2)
dodržují stejný formát PUB jako primitivy v Qiskit Runtime. Podrobnosti najdeš v tématu [Vstupy a výstupy primitiv](primitive-input-output).
Mohou však existovat rozdíly v polích vrácených metadat.

- Třída [`qiskit.primitives.BackendEstimatorV2`](https://docs.quantum.ibm.com/api/qiskit/qiskit.primitives.BackendEstimatorV2) nenabízí žádné implementace zmírňování chyb měření ani Gate bez dalšího nastavení, protože
backendové primitivy jsou navrženy ke spuštění lokálně na uživatelském počítači.

- Třída [`qiskit.primitives.BackendSamplerV2`](https://docs.quantum.ibm.com/api/qiskit/qiskit.primitives.BackendSamplerV2) vyžaduje backend, který podporuje možnost `memory`.

- Rozhraní backendových primitiv zpřístupňují vlastní `Options` pro [`SamplerV2`](https://docs.quantum.ibm.com/api/qiskit/qiskit.primitives.BackendSamplerV2) a [`EstimatorV2`](https://docs.quantum.ibm.com/api/qiskit/qiskit.primitives.BackendEstimatorV2), které se liší od Runtime implementací.

## Začínáme s nástroji {#zaciname-s-nastroji}

Balíček `qiskit-addon-utils` obsahuje několik modulů, včetně modulu pro generování problémů při simulaci kvantových systémů, barvení grafů pro efektivnější umísťování Gate operací v kvantovém Circuit, a rozdělování obvodů (slicing), které může pomoci s [operator backpropagation](/guides/qiskit-addons-obp). Následující části shrnují každý modul. Užitečné informace obsahuje také [dokumentace API](https://docs.quantum.ibm.com/api/qiskit-addon-utils) tohoto balíčku.

### Generování problémů {#generovani-problemu}

Obsah modulu [`qiskit_addon_utils.problem_generators`](../api/qiskit-addon-utils/problem-generators) zahrnuje:

- Funkci [`generate_xyz_hamiltonian()`](../api/qiskit-addon-utils/problem-generators#generate_xyz_hamiltonian), která generuje reprezentaci Isingova XYZ modelu typu `SparsePauliOp` respektující konektivitu:

$$H = \sum_{(j,k)\in E} \left(J_x X_jX_k + J_yY_jY_k + J_zZ_jZ_k\right) + \sum_{j\in V} \left(h_x X_j + h_y Y_j + h_z Z_j\right) $$
- Funkci [`generate_time_evolution_circuit()`](../api/qiskit-addon-utils/problem-generators#generate_time_evolution_circuit), která sestavuje Circuit modelující časový vývoj daného operátoru.
- Tři různé objekty [`PauliOrderStrategy`](../api/qiskit-addon-utils/problem-generators#pauliorderstrategy) pro výčet různých uspořádání Pauliho řetězců. To je nejužitečnější při použití společně s barvením grafů a lze je použít jako argumenty ve funkcích `generate_xyz_hamiltonian()` i `generate_time_evolution_circuit()`.

### Barvení grafů {#barveni-grafu}

Modul [`qiskit_addon_utils.coloring`](../api/qiskit-addon-utils/coloring) slouží k barvení hran v coupling map a k využití tohoto barvení pro efektivnější umísťování Gate operací v kvantovém Circuit. Účelem takto obarvené coupling map je najít sadu barev hran tak, aby žádné dvě hrany stejné barvy nesdílely společný uzel. Pro QPU to znamená, že Gate operace na hranách stejné barvy (spojení Qubitů) lze spustit současně a Circuit se provede rychleji.

Jako rychlý příklad můžeš použít funkci [`auto_color_edges()`](../api/qiskit-addon-utils/coloring#auto_color_edges) ke generování barvení hran pro naivní Circuit provádějící `CZGate` na každém spojení Qubitů. Níže uvedená ukázka kódu používá coupling map Backendu `FakeSherbrooke`, vytvoří tento naivní Circuit, a poté použije funkci `auto_color_edges()` k vytvoření ekvivalentního efektivnějšího Circuit.

```python
from qiskit_ibm_runtime.fake_provider import FakeSherbrooke
from qiskit import QuantumCircuit
from qiskit_addon_utils.coloring import auto_color_edges
from qiskit_addon_utils.slicing import combine_slices, slice_by_depth
from collections import defaultdict

backend = FakeSherbrooke()
coupling_map = backend.coupling_map

# Create naive circuit
circuit = QuantumCircuit(backend.num_qubits)
for edge in coupling_map.graph.edge_list():
    circuit.cz(edge[0], edge[1])

# Color the edges of the coupling map
coloring = auto_color_edges(coupling_map)
circuit_with_coloring = QuantumCircuit(backend.num_qubits)

# Make a reverse coloring dict in order to make the circuit
color_to_edge = defaultdict(list)
for edge, color in coloring.items():
    color_to_edge[color].append(edge)

# Place edges in order of color
for edges in color_to_edge.values():
    for edge in edges:
        circuit_with_coloring.cz(edge[0], edge[1])

print(f"The circuit without using edge coloring has depth: {circuit.depth()}")
print(
    f"The circuit using edge coloring has depth: {circuit_with_coloring.depth()}"
)
```

```text
The circuit without using edge coloring has depth: 37
The circuit using edge coloring has depth: 3
```

### Rozdělování obvodů {#rozdelovani-obvodu}

Modul [`qiskit-addon-utils.slicing`](../api/qiskit-addon-utils/slicing) obsahuje funkce a průchody Transpileru pro práci s vytvářením „řezů" (slices) obvodů – časových oddílů [`QuantumCircuit`](../api/qiskit/qiskit.circuit.QuantumCircuit) přesahujících přes všechny Qubitů. Tyto řezy se primárně používají pro [operator backpropagation](/guides/qiskit-addons-obp-get-started). Čtyři hlavní způsoby, jak lze Circuit rozdělit, jsou podle typu Gate, hloubky, barvení nebo instrukcí [`Barrier`](../api/qiskit/circuit#barrier). Výstup těchto funkcí pro rozdělování vrací seznam objektů [`QuantumCircuit`](../api/qiskit/qiskit.circuit.QuantumCircuit). Rozdělené Circuit lze také znovu sloučit pomocí funkce [`combine_slices()`](../api/qiskit-addon-utils/slicing#combine_slices). Více informací najdeš v [referenci API](../api/qiskit-addon-utils/slicing) modulu.

Níže je několik příkladů, jak vytvořit tyto řezy za použití následujícího Circuit:

```python
import numpy as np
from qiskit import QuantumCircuit

num_qubits = 9
qc = QuantumCircuit(num_qubits)
qc.ry(np.pi / 4, range(num_qubits))
qubits_1 = [i for i in range(num_qubits) if i % 2 == 0]
qubits_2 = [i for i in range(num_qubits) if i % 2 == 1]
qc.cx(qubits_1[:-1], qubits_2)
qc.cx(qubits_2, qubits_1[1:])
qc.cx(qubits_1[-1], qubits_1[0])
qc.rx(np.pi / 4, range(num_qubits))
qc.rz(np.pi / 4, range(num_qubits))
qc.draw("mpl", scale=0.6)
```

Pokud je tvůj pracovní postup navržen tak, aby využíval fyzickou konektivitu Qubitů na QPU, na kterém bude spuštěn, můžeš vytvořit řezy na základě barvení hran. Ukázka kódu níže přiřadí třibarvení hranám Circuit a rozdělí Circuit podle tohoto barvení hran. (Poznámka: toto ovlivňuje pouze nelokální Gate operace. Jednoqubitové Gate operace budou rozděleny podle typu Gate.)

```python
from qiskit_addon_utils.slicing import slice_by_coloring

# Assign a color to each set of connected qubits
coloring = {}
for i in range(num_qubits - 1):
    coloring[(i, i + 1)] = i % 3
coloring[(num_qubits - 1, 0)] = 2

# Create a circuit with operations added in order of color
qc = QuantumCircuit(num_qubits)
qc.ry(np.pi / 4, range(num_qubits))
edges = [
    edge for color in range(3) for edge in coloring if coloring[edge] == color
]
for edge in edges:
    qc.cx(edge[0], edge[1])
qc.rx(np.pi / 4, range(num_qubits))
qc.rz(np.pi / 4, range(num_qubits))

# Create slices by edge color
slices = slice_by_coloring(qc, coloring=coloring)

# Recombine slices in order to visualize the partitions together
combined_slices = combine_slices(slices, include_barriers=True)
combined_slices.draw("mpl", scale=0.6)
```

Pokud máš vlastní strategii rozdělování, můžeš místo toho umístit Barrier do Circuit k označení míst, kde má být rozdělen, a použít funkci [`slice_by_barriers`](../api/qiskit-addon-utils/slicing#slice_by_barriers).

```python
qc = QuantumCircuit(num_qubits)
qc.ry(np.pi / 4, range(num_qubits))
qc.barrier()
qubits_1 = [i for i in range(num_qubits) if i % 2 == 0]
qubits_2 = [i for i in range(num_qubits) if i % 2 == 1]
qc.cx(qubits_1[:-1], qubits_2)
qc.cx(qubits_2, qubits_1[1:])
qc.cx(qubits_1[-1], qubits_1[0])
qc.barrier()
qc.rx(np.pi / 4, range(num_qubits))
qc.rz(np.pi / 4, range(num_qubits))
qc.draw("mpl", scale=0.6)
```

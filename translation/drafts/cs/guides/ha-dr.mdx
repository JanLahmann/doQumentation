# Pochopení vysoké dostupnosti a obnovy po havárii pro Qiskit Runtime

Vysoká dostupnost (HA) je schopnost služby zůstat v provozu a být přístupná i při neočekávaných selháních. Obnova po havárii je proces obnovení instance služby do funkčního stavu. Zákazníci Qiskit Runtime nejsou zodpovědní za plánování ani provádění obnovy po havárii.

Qiskit Runtime je regionální služba, která splňuje definované [Cíle úrovně služeb (SLO)](https://cloud.ibm.com/docs/resiliency?topic=resiliency-slo) u jakéhokoli placeného tarifu. Další informace najdeš ve [Smlouvě o úrovni služeb (SLA)](https://www.ibm.com/support/customer/csol/terms/?id=i126-9268&lc=en) pro IBM Cloud&reg;. Více informací o dostupných regionech a datových centrech IBM Cloud pro Qiskit Runtime najdeš v části [Dostupnost služeb a infrastruktury podle umístění.](https://cloud.ibm.com/docs/overview?topic=overview-services_region)

## Architektura vysoké dostupnosti {#high-availability-architecture}

### Funkce vysoké dostupnosti {#high-availability-features}

Tato služba je nasazena ve třech zónách v regionu s více zónami bez jediného bodu selhání. Požadavky API jsou směrovány přes nástroj pro vyvažování zátěže na uzly instance HA, přičemž každý uzel se nachází v jiné zóně.

Pokud uzel instance HA nebo zóna selže, služba nadále zpracovává požadavky API prostřednictvím zbývajících uzlů instance HA. Při zonálním selhání nedochází k žádnému znatelné snížení kvality služby.

Je dostupná v následujících regionech:

- Washington DC (us-east)
- Frankfurt (eu-de)

# Použití IBM Cloud Platform API pro přístup k instancím

Tento průvodce je určen pro každého, kdo používá IBM Cloud Platform API ke zřizování, aktualizaci nebo správě přístupu k instancím. Obsahuje podrobnosti o [parametrech](#parameters), které můžeš nastavit při zřizování instance, a také o sadě [rozšiřujících polí](#extensions) vrácených při přístupu k instanci.

<Admonition type="note">
Další informace najdeš v dokumentaci IBM Cloud&reg; ResourceController API.

Některé relevantní stránky:
- [Vytvoření (zřízení) nové instance prostředku](https://cloud.ibm.com/apidocs/resource-controller/resource-controller#create-resource-instance)
- [Aktualizace instance prostředku](https://cloud.ibm.com/apidocs/resource-controller/resource-controller#update-resource-instance)
- [Získání instance prostředku](https://cloud.ibm.com/apidocs/resource-controller/resource-controller#get-resource-instance)
- [Získání seznamu všech instancí prostředků](https://cloud.ibm.com/apidocs/resource-controller/resource-controller#list-resource-instances)  
    
    Při získávání seznamu všech instancí prostředků si všimni, že můžeš filtrovat pouze instance Qiskit Runtime pomocí `resource_id = "b6049020-80f4-11eb-a0f7-e35ec9b4054f"`

</Admonition>

## `parameters`

V těle požadavku při zřizování nebo aktualizaci instance:

- **instance_limit_seconds** (řetězec nebo null, `^[0-9]+$`): Limit nastavený pro instanci. Pro plán Open Plan: pokud je vynecháno, výchozí hodnota je 10 minut.
.
- **usage_allocation_seconds** (řetězec nebo null, `^[0-9]+$`): Přidělený čas pro instanci, který využívá plánovač fair-share k určení priority ve frontě na základě využití všech QPU. (Neplatí pro instance s průběžnými platbami.)
- **backends** (pole seznamů Backend (řetězce nebo null) nebo pole s hodnotou "ANY"): seznam povolených názvů Backend, které jsou pro tuto instanci dostupné. Výchozí hodnota: `["ANY"]` znamená, že jsou dostupné všechny Backend v rámci tohoto plánu. `[]` znamená žádné Backend.

## `extensions`

V odpovědi při zřizování nebo při získávání podrobností o instanci:

Další vlastnosti instance poskytnuté službou, které jsou reprezentovány jako páry klíč–hodnota. Tyto vlastnosti jsou s instancí uchovávány pod vlastností extensions.

- **instance_limit_seconds** (celé číslo nebo null, `>= 0`): Časový limit nastavený pro instanci.
- **usage_allocation_seconds** (celé číslo nebo null, `>= 0`): Přidělený čas pro instanci, který využívá plánovač fair-share k určení priority ve frontě na základě využití všech QPU.
- **backends** (povinné; pole seznamů Backend (řetězce) nebo pole s hodnotou "ANY" (řetězce)): seznam povolených názvů Backend, které jsou pro tuto instanci dostupné. Výchozí hodnota: `["ANY"]` znamená, že jsou dostupné všechny Backend v rámci tohoto plánu. `[]` znamená žádné Backend.

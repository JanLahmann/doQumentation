# Registro de eventos para Qiskit Runtime

Los servicios de IBM Cloud&reg;, como Qiskit Runtime, generan registros de plataforma que puedes usar para investigar actividad anómala y acciones críticas en tu cuenta, y para solucionar problemas.

Puedes usar [IBM Cloud Logs Routing,](https://cloud.ibm.com/docs/logs-router?topic=logs-router-about) que es un servicio de plataforma, para enrutar los registros de plataforma de tu cuenta hacia el destino que elijas configurando un tenant que define a dónde se envían los registros de plataforma. Para más información, consulta [Acerca de Logs Routing.](https://cloud.ibm.com/docs/logs-router?topic=logs-router-about)

Puedes usar [IBM Cloud Logs](https://cloud.ibm.com/docs/cloud-logs) para visualizar y recibir alertas sobre los registros de plataforma que se generan en tu cuenta y que IBM Cloud Logs Routing enruta hacia una instancia de IBM Cloud Logs.

## Ubicaciones donde se generan los registros de plataforma {#locations-where-platform-logs-are-generated}

IBM Cloud Container Registry genera registros de plataforma que se muestran en tus instancias de registro. La siguiente tabla lista las ubicaciones donde está habilitada la recopilación automática de registros del servicio Container Registry.

| Ubicación                  | ¿Hay registros de servicios de plataforma disponibles? |
|----------------------------|--------------------------------------------------------|
| Washington, D.C. (us-east) |                          Sí                           |
| Frankfurt (eu-de)          |                          Sí                           |

## Registros de plataforma que se generan {#platform-logs-that-are-generated}

IBM Qiskit Runtime genera registros de servicios de plataforma que se muestran en tus instancias de registro para los siguientes casos específicos:

- Cuando el estado de un job cambia.

## Ver registros desde la página de Observabilidad {#viewing-logs-from-the-observability-page}

Para información sobre cómo abrir la interfaz de IBM Cloud Logs, consulta **Launching the UI** en la [documentación de IBM Cloud Logs.](https://cloud.ibm.com/docs/cloud-logs?topic=cloud-logs-instance-launch)

## Campos por tipo de registro {#fields-by-log-type}

Para información sobre los campos incluidos en cada registro de plataforma, consulta [Fields that are included in platform logs.](https://cloud.ibm.com/docs/logs-router?topic=logs-router-about-platform-logs#about-platform-logs-2)

Los siguientes campos están incluidos en el registro de log.

| Campo           | Tipo       | Descripción                                                                          |
|-----------------|------------|--------------------------------------------------------------------------------------|
| logSourceCRN    | Requerido  | Define la cuenta y la instancia de log de flujo donde se publica el registro.        |
| saveServiceCopy | Requerido  | Define si IBM guarda una copia del registro para fines operativos.                   |
| message         | Requerido  | Descripción del log que se genera.                                                   |

Los flujos de trabajo cuánticos suelen tardar bastante en completarse y pueden extenderse a lo largo de muchas sesiones. Si reinicias el kernel de Python, perderás cualquier resultado almacenado en memoria. Para evitar la pérdida de datos, puedes guardar los resultados en un archivo y recuperar los resultados de trabajos anteriores desde IBM Quantum&reg;, de modo que tu próxima sesión pueda retomar donde la dejaste.
## Recuperar resultados de trabajos desde IBM Quantum {#retrieve-job-results-from-ibm-quantum}

IBM Quantum almacena automáticamente los resultados de cada trabajo para que puedas recuperarlos en una fecha posterior. Usa esta función para continuar programas cuánticos entre reinicios del kernel y revisar resultados pasados. Puedes obtener el ID de un trabajo de forma programática a través de su método `job_id`, o puedes ver todos tus trabajos enviados y sus IDs en la [página de Cargas de trabajo](https://quantum.cloud.ibm.com/workloads).

Para encontrar un trabajo de forma programática, usa el método [`QiskitRuntimeService.jobs`](https://docs.quantum.ibm.com/api/qiskit-ibm-runtime/qiskit-runtime-service#jobs). Por defecto, esto devuelve los trabajos enviados más recientemente, pero también puedes filtrar trabajos por nombre de backend, fecha de creación y más. La siguiente celda encuentra todos los trabajos enviados en los últimos tres meses. El argumento `created_after` debe ser un objeto [`datetime.datetime`](https://docs.python.org/3.8/library/datetime.html#datetime.datetime).


```python
import datetime
from qiskit_ibm_runtime import QiskitRuntimeService

three_months_ago = datetime.datetime.now() - datetime.timedelta(days=90)

service = QiskitRuntimeService()
jobs_in_last_three_months = service.jobs(created_after=three_months_ago)
jobs_in_last_three_months[:3]  # show first three jobs
```

También puedes filtrar por backend, estado del trabajo, sesión y más. Para más información, consulta [`QiskitRuntimeService.jobs`](https://docs.quantum.ibm.com/api/qiskit-ibm-runtime/qiskit-runtime-service#jobs) en la documentación de la API.

Una vez que tengas el ID del trabajo, usa el método [`QiskitRuntimeService.job`](https://docs.quantum.ibm.com/api/qiskit-ibm-runtime/qiskit-runtime-service#job) para recuperarlo.

```python
# Get ID of most recent successful job for demonstration.

job_id = successful_job.job_id()
print(job_id)
```

## Guardar resultados en disco {#save-results-to-disk}

También puede que quieras guardar los resultados en disco. Para ello, usa la biblioteca JSON integrada de Python con los codificadores de Qiskit Runtime.

```python
import json
from qiskit_ibm_runtime import RuntimeEncoder

with open("result.json", "w") as file:
    json.dump(retrieved_job.result(), file, cls=RuntimeEncoder)
```

Luego puedes cargar este array desde disco en un kernel separado.

```python
from qiskit_ibm_runtime import RuntimeDecoder

with open("result.json", "r") as file:
    result = json.load(file, cls=RuntimeDecoder)

result
```

Las fórmulas multiproducto (MPF, por sus siglas en inglés) se pueden usar para simular con mayor precisión la dinámica de un sistema cuántico, a costa de un mayor número de ejecuciones de circuitos. Esta es una técnica de postprocesamiento que mitiga el error de los valores esperados para estados que han evolucionado en el tiempo.

Se utiliza computación clásica para resolver un sistema de ecuaciones lineales que proporciona coeficientes a una combinación ponderada de varias ejecuciones de circuitos. El uso de esta combinación ponderada puede reducir el error asociado con la simulación de la evolución temporal, dada una buena selección de pasos de Trotter. La herramienta MPF recibe una selección de datos --- incluyendo el número de pasos de Trotter y el orden de la aproximación de Trotter --- para preparar y resolver (o aproximar la solución de) el sistema de ecuaciones lineales asociado, que luego puedes usar para postprocesar las mediciones de los valores esperados de un estado con evolución temporal.
## Instalar el paquete MPF {#install-the-mpf-package}

Hay dos formas de instalar el paquete MPF: a través de PyPI y compilando desde el código fuente. Se recomienda instalarlo en un [entorno virtual](https://docs.python.org/3.10/tutorial/venv.html) para garantizar la separación entre las dependencias de los paquetes.

### Instalar desde PyPI {#install-from-pypi}

La forma más sencilla de instalar el paquete `qiskit-addon-mpf` es a través de PyPI.
```bash
pip install qiskit-addon-mpf
```

### Compilar desde el código fuente {#build-from-source}
Los usuarios que deseen desarrollar en el repositorio o que quieran instalarlo manualmente pueden hacerlo clonando primero el repositorio:
```bash
git clone git@github.com:Qiskit/qiskit-addon-mpf.git
```

e instalar el paquete mediante `pip`. El repositorio también contiene varias dependencias opcionales que habilitan ciertas funcionalidades.

Ajusta las opciones según tus necesidades.

## Fundamentos teóricos {#theoretical-background}

Las MPF pueden reducir el error de aproximación de Trotter asociado con la simulación de la dinámica de sistemas cuánticos mediante una combinación ponderada de varias ejecuciones de circuitos. Esta suma ponderada se define como:

$$ \mu(t) = \sum_j x_j\rho_j^{k_j}\left(\frac{t}{k_j}\right) + \text{some remaining Trotter error}, $$

donde $x_j$ son los coeficientes de ponderación, $\rho_j^{k_j}$ es la matriz de densidad que corresponde al estado puro obtenido al evolucionar el estado inicial mediante una fórmula producto $S^{k_j}$ que aproxima el operador de evolución temporal usando $k_j$ pasos de Trotter, y $j$ indexa cada fórmula producto usada en la suma.

![Varios circuitos con un número variable de pasos de Trotter se usan para calcular el observable objetivo](/docs/images/guides/qiskit-addons/mpf-diagram.avif)

En general, sin embargo, el objetivo de simular la dinámica cuántica es medir algún observable $\mathcal{O}(t)$, que es una función del tiempo. Al usar MPF, se ejecutan múltiples circuitos --- cada uno usando $k_j$ pasos de Trotter --- para obtener varias mediciones del observable objetivo $\mathcal{O}_{k_j}(t)$. La medición del observable objetivo se obtiene entonces calculando:

$$ \langle \mathcal{O}(t) \rangle = \sum_j x_j(t) \langle \mathcal{O}_{k_j}(t) \rangle. $$

En esencia, puedes reducir el error de Trotter general aproximando el operador de evolución temporal mediante varias fórmulas producto con un número variable de pasos de Trotter en lugar de una única fórmula producto. Construyes un circuito para cada término de la suma ponderada, que evoluciona el sistema de acuerdo con cada uno de los $k_j$ pasos de Trotter. Cada circuito se ejecuta entonces por separado en una QPU para reconstruir los resultados en un paso de postprocesamiento. La utilidad de esta técnica puede verse desde dos perspectivas:
1. Para un número fijo de pasos de Trotter que ejecutas, puedes obtener resultados con un error de Trotter menor en total.
2. Para un número de pasos de Trotter que resulta en circuitos profundos, puedes usar MPF para encontrar varios circuitos de menor profundidad que ejecutar, lo que produce un error de aproximación de Trotter similar.

## Determinar los coeficientes MPF {#determine-mpf-coefficients}

La funcionalidad central del paquete `qiskit-addon-mpf` reside en determinar los coeficientes MPF $x_j(t)$ (que pueden depender del tiempo). El proceso para obtener cada $x_j(t)$ implica resolver un sistema de ecuaciones lineales $Ax=b$ donde $x$ es el vector de coeficientes a determinar, $A$ es una matriz que depende de cada uno de los $k_j$ y de la fórmula producto usada $S$ (es decir, el orden de aproximación y el número de pasos de Trotter), y $b$ es un vector de restricciones. Este sistema de ecuaciones puede resolverse de forma exacta o con un modelo aproximado que minimiza la norma-1 de los coeficientes. Además, la elección de cada $k_j$ es un proceso heurístico, pero puede acotarse mediante las siguientes restricciones:

1. El valor **más grande** de $k_j$ está acotado por la mayor profundidad de circuito que se puede ejecutar de forma confiable
2. El valor **más pequeño** de $k_j$ debe satisfacer $dt = t/k_j < 1$, ya que es donde el error de Trotter se comporta de manera más predecible
3. Ninguno de los coeficientes $x_j$ debe estar cerca de $0$, ya que esto implicaría que no contribuyen mucho a la MPF
4. De igual forma, el coeficiente asociado al valor más grande de $k_j$ no debe dominar, ya que esto implicaría que estás usando una única fórmula producto
5. Por último, la norma de los coeficientes $x_j$ obtenidos debe ser pequeña, ya que esto indica una MPF bien condicionada [1](#references)

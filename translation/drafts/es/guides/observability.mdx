# Eventos de seguimiento de actividad para Qiskit Runtime

Los servicios de IBM Cloud&reg;, como Qiskit Runtime, generan eventos de seguimiento de actividad.

Los eventos de seguimiento de actividad informan sobre las acciones que cambian el estado de un servicio en IBM Cloud. Puedes usar los eventos para investigar actividades anómalas y acciones críticas, y para cumplir con los requisitos de auditoría regulatoria.

Puedes usar IBM Cloud Activity Tracker Event Routing, un servicio de plataforma, para enrutar eventos de auditoría en tu cuenta hacia los destinos que elijas configurando destinos y rutas que definen adónde se envían los eventos de seguimiento de actividad. Para más información, consulta [Acerca de IBM Cloud Activity Tracker Events Routing.](https://cloud.ibm.com/docs/atracker?topic=atracker-about)

Puedes usar IBM Cloud Logs para visualizar y recibir alertas sobre eventos que se generan en tu cuenta y que IBM Cloud Activity Tracker Event Routing enruta hacia una instancia de IBM Cloud Logs.

## Ubicaciones donde se generan los eventos de seguimiento de actividad {#locations-where-activity-tracking-events-are-generated}

Qiskit Runtime envía eventos de seguimiento de actividad en las siguientes regiones:

- Washington ("us-east")
- Frankfurt ("eu-de")

## Ver eventos de seguimiento de actividad para Qiskit Runtime {#viewing-activity-tracking-events-for-qiskit-runtime}

Puedes usar IBM Cloud Logs para visualizar y recibir alertas sobre eventos que se generan en tu cuenta y que IBM Cloud Activity Tracker Event Routing enruta hacia una instancia de IBM Cloud Logs.

### Acceder a IBM Cloud Logs desde la página de Observabilidad {#launching-ibm-cloud-logs-from-the-observability-page}

Para obtener información sobre cómo acceder a la interfaz de IBM Cloud Logs, consulta [Iniciar la interfaz de usuario](https://cloud.ibm.com/docs/cloud-logs?topic=cloud-logs-instance-launch#instance-launch-cloud-ui) en la documentación de IBM Cloud Logs.

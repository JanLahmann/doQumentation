## Ejemplo: transpilación remota con Qiskit Serverless {#example-remote-transpilation-with-qiskit-serverless}

Comienza con el siguiente ejemplo que transpila un `circuit` contra un `backend` dado y un `optimization_level` objetivo, y agrega gradualmente más elementos para desplegar tu carga de trabajo en Qiskit Serverless.

Coloca el siguiente bloque de código en el archivo `./source_files/transpile_remote.py`. Este archivo es el programa que se subirá a Qiskit Serverless.



```python
%%writefile ./source_files/transpile_remote.py

from qiskit.transpiler import generate_preset_pass_manager

def transpile_remote(circuit, optimization_level, backend):
    """Transpiles an abstract circuit into an ISA circuit for a given backend."""
    pass_manager = generate_preset_pass_manager(
        optimization_level=optimization_level,
		backend=backend
    )
    isa_circuit = pass_manager.run(circuit)
    return isa_circuit
```

## Configura tus archivos {#set-up-your-files}

Qiskit Serverless requiere organizar los archivos `.py` de tu carga de trabajo en un directorio dedicado. La siguiente estructura es un ejemplo de buenas prácticas:

```text
serverless_program
├── program_uploader.ipynb
└── source_files
    ├── transpile_remote.py
    └── *.py
```

Serverless sube el contenido de `source_files` para ejecutarlo de forma remota. Una vez configurados, puedes ajustar `transpile_remote.py` para obtener entradas y devolver salidas.

### Obtén los argumentos del programa {#get-program-arguments}

Tu archivo inicial `transpile_remote.py` tiene tres entradas: `circuits`, `backend_name` y `optimization_level`. Serverless actualmente solo acepta entradas y salidas serializables. Por esta razón, no puedes pasar `backend` directamente, así que usa `backend_name` como cadena de texto en su lugar.

```python
%%writefile --append ./source_files/transpile_remote.py

from qiskit_serverless import get_arguments, save_result, distribute_task, get

# Get program arguments
arguments = get_arguments()
circuits = arguments.get("circuits")
backend_name = arguments.get("backend_name")
optimization_level = arguments.get("optimization_level")
```

```text
Appending to ./source_files/transpile_remote.py
```

En este punto, puedes obtener tu backend con `QiskitRuntimeService` y agregar tu programa existente con el siguiente código. El código a continuación requiere que ya hayas [guardado tus credenciales](/guides/cloud-setup).

```python
%%writefile --append ./source_files/transpile_remote.py

from qiskit_ibm_runtime import QiskitRuntimeService

service = QiskitRuntimeService()
backend = service.backend(backend_name)
```

```text
Appending to ./source_files/transpile_remote.py
```

Por último, puedes ejecutar `transpile_remote()` sobre todos los `circuits` pasados como entrada y devolver los `transpiled_circuits` como resultado:

```python
%%writefile --append ./source_files/transpile_remote.py

results = [
    transpile_remote(circuit, 1, backend)
    for circuit in circuits
]

save_result({
    "transpiled_circuits": results
})
```

```text
Appending to ./source_files/transpile_remote.py
```

## Despliega en IBM Quantum Platform {#deploy-to-ibm-quantum-platform}

La sección anterior creó un programa para ejecutarse de forma remota. Los bloques de código de esta sección suben ese programa a Qiskit Serverless.

Usa `qiskit-ibm-catalog` para autenticarte en `QiskitServerless` con tu clave de API, que puedes encontrar en el [panel de IBM Quantum](https://quantum.cloud.ibm.com), y sube el programa.

Puedes usar `save_account()` para guardar tus credenciales (consulta la sección [Configuración para usar IBM Cloud](/guides/cloud-setup#cloud-save)). Ten en cuenta que esto escribe tus credenciales en el mismo archivo que [`QiskitRuntimeService.save_account()`](https://docs.quantum.ibm.com/api/qiskit-ibm-runtime/qiskit-runtime-service#save_account).

```python
from qiskit_ibm_catalog import QiskitServerless, QiskitFunction

# Authenticate to the remote cluster and submit the pattern for remote execution
serverless = QiskitServerless()
```

Qiskit Serverless comprime el contenido de `working_dir` (en este caso, `source_files`) en un archivo `tar`, que se sube y se limpia después. El `entrypoint` identifica el ejecutable principal del programa que Qiskit Serverless debe ejecutar. Además, si tu programa tiene dependencias `pip` personalizadas, puedes agregarlas a un array `dependencies`:

```python
transpile_remote_demo = QiskitFunction(
    title="transpile_remote_serverless",
    entrypoint="transpile_remote.py",
    working_dir="./source_files/",
)
```

```python
serverless.upload(transpile_remote_demo)
```

```text
QiskitFunction(transpile_remote_serverless)
```

Para verificar si se subió correctamente, usa `serverless.list()`:

```python
# Get program from serverless.list() that matches the title of the one we uploaded
next(
    program
    for program in serverless.list()
    if program.title == "transpile_remote_serverless"
)
```

```python
# This cell is hidden from users, it checks the program executes correctly
from time import sleep
from qiskit import QuantumCircuit

qc = QuantumCircuit(2)
transpile_remote_serverless = serverless.load("transpile_remote_serverless")
job = transpile_remote_serverless.run(
    circuits=[qc],
    backend="ibm_sherbrooke",
    optimization_level=1,
)
while True:
    sleep(5)
    status = job.status()
    if status not in ["QUEUED", "INITIALIZING", "RUNNING", "DONE"]:
        raise Exception(
            f"Unexpected job status: '{status}'\n"
            + "Here are the logs:\n"
            + job.logs()
        )
    if status == "DONE":
        break
```

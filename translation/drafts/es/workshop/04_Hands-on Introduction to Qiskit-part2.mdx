## 2.1 Mapear el problema a un formato nativo cuántico

En un programa cuántico, los *circuitos cuánticos* son el formato nativo para representar instrucciones cuánticas, y los *operadores* representan los observables que se van a medir. Al crear un circuito, normalmente crearás un nuevo objeto [`QuantumCircuit`](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.QuantumCircuit#quantumcircuit-class) y luego añadirás instrucciones a él en secuencia.

La siguiente celda de código crea un circuito que produce el estado GHZ, que es un estado en el que tres qubits están completamente entrelazados entre sí.

<div class="alert alert-info">

  El SDK de Qiskit utiliza la numeración de bits LSb 0, donde el $n^{th}$ dígito tiene el valor $1 \ll n$ o $2^n$. Para más detalles, consulta el tema [Orden de bits en el SDK de Qiskit](https://docs.quantum.ibm.com/guides/bit-ordering).

</div>

```python
# Create a GHZ state circuit

qc = QuantumCircuit(3)
qc.h(0)
qc.cx(0,1)
qc.cx(0,2)
# Draw the circuit
qc.draw("mpl")
```

![Quantum circuit diagram](./_04_Hands-on Introduction to Qiskit_imgs/output_15.png)

Consulta [`QuantumCircuit`](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.QuantumCircuit#quantumcircuit-class) en la documentación para ver todas las operaciones disponibles.

Al crear circuitos cuánticos, también debes considerar qué tipo de datos quieres que se devuelvan tras la ejecución. Qiskit ofrece dos formas de devolver datos: puedes obtener una distribución de probabilidad para un conjunto de qubits que elijas medir, o puedes obtener el valor esperado de un observable. Prepara tu carga de trabajo para medir tu circuito de una de estas dos formas con los [primitivos de Qiskit](https://docs.quantum.ibm.com/guides/get-started-with-primitives) (explicados en detalle en el Paso 3).

Este ejemplo mide valores esperados utilizando el submódulo `qiskit.quantum_info`, que se especifica mediante operadores (objetos matemáticos utilizados para representar una acción o proceso que cambia un estado cuántico). La siguiente celda de código crea seis operadores de Pauli de tres qubits: `ZZZ`, `ZZX`, `ZII`, `XXI`, `ZZI` e `III`.

```python
# Set up six different observables.

observables_labels = ["ZZZ", "ZZX", "ZII", "XXI", "ZZI", "III"]

observables = [SparsePauliOp(label) for label in observables_labels]
print(observables)
```

```text
[SparsePauliOp(['ZZZ'],
              coeffs=[1.+0.j]), SparsePauliOp(['ZZX'],
              coeffs=[1.+0.j]), SparsePauliOp(['ZII'],
              coeffs=[1.+0.j]), SparsePauliOp(['XXI'],
              coeffs=[1.+0.j]), SparsePauliOp(['ZZI'],
              coeffs=[1.+0.j]), SparsePauliOp(['III'],
              coeffs=[1.+0.j])]
```

<div class="alert alert-info">

  Aquí, algo como el operador `ZZI` es una abreviatura del producto tensorial $Z\otimes Z\otimes I$, que significa medir Z en el qubit 2 y Z en el qubit 1 juntos, y obtener información sobre la correlación entre el qubit 2 y el qubit 1. Los valores esperados como este también se escriben típicamente como $\langle Z_2 Z_1 \rangle$.

  Si el estado que observamos es el estado GHZ de tres qubits, entonces la medición de $\langle Z_2 Z_1 \rangle$ debería ser 1.

</div>

## 2.2 Optimizar los circuitos y operadores

Al ejecutar circuitos en un dispositivo, es importante optimizar el conjunto de instrucciones que contiene el circuito y minimizar la profundidad total (aproximadamente el número de instrucciones) del circuito. Esto garantiza que obtengas los mejores resultados posibles al reducir los efectos del error y el ruido. Además, las instrucciones del circuito deben ajustarse a la [Arquitectura del Conjunto de Instrucciones (ISA)](https://docs.quantum.ibm.com/guides/transpile#instruction-set-architecture) de un dispositivo backend y deben tener en cuenta las puertas base y la conectividad de qubits del dispositivo.

El siguiente código instancia un dispositivo real al que enviar un trabajo y transforma el circuito y los observables para que coincidan con la ISA de ese backend.
Si no has guardado previamente tus credenciales, sigue las instrucciones [aquí](https://docs.quantum.ibm.com/guides/setup-channel#iqp) para autenticarte con tu token de API.

```python
# Choose a real backend
service = QiskitRuntimeService(channel='ibm_quantum_platform',)
backend = service.least_busy(min_num_qubits=156)
# print backend details
print(
    f"Name: {backend.name}\n"
    f"Version: {backend.backend_version}\n"
    f"No. of qubits: {backend.num_qubits}\n"
    f"Processor type: {backend.processor_type}\n"
)
```

## 2.3 Ejecutar usando los primitivos cuánticos

Las computadoras cuánticas pueden producir resultados aleatorios, por lo que normalmente se recopila una muestra de las salidas ejecutando el circuito muchas veces. Puedes estimar el valor del observable utilizando la clase `Estimator`. `Estimator` es uno de los dos [primitivos](https://docs.quantum.ibm.com/guides/get-started-with-primitives); el otro es `Sampler`, que se puede usar para obtener datos de una computadora cuántica. Estos objetos poseen un método `run()` que ejecuta la selección de circuitos, observables y parámetros (si aplica), usando un [bloque unificado de primitivos (PUB).](https://docs.quantum.ibm.com/guides/primitives#sampler)
Al ejecutar este código en hardware cuántico real, considera aplicar [técnicas de mitigación y supresión de errores](https://quantum.cloud.ibm.com/docs/en/guides/error-mitigation-and-suppression-techniques) para reducir el ruido intrínseco de la computadora cuántica.

```python
# Construct the Estimator instance.
estimator = Estimator(mode=backend)
estimator.options.resilience_level = 1
estimator.options.default_shots = 5000
```

Envía un trabajo usando el primitivo Estimator.

```python
# One pub, with one circuit to run against six different observables.
job = estimator.run([(isa_circuit, mapped_observables)]) 

# Use the job ID to retrieve your job data later
print(f">>> Job ID: {job.job_id()}")
```

Después de enviar un trabajo, puedes esperar hasta que el trabajo se complete en tu instancia actual de Python, o usar el `job_id` para recuperar los datos en un momento posterior. (Consulta la [sección sobre recuperación de trabajos](https://docs.quantum.ibm.com/guides/monitor-job#retrieve-job-results-at-a-later-time) para más detalles.)

Una vez que el trabajo se completa, examina su salida a través del atributo `result()` del trabajo.

```python
# This is the result of the entire submission.  You submitted one Pub,
# so this contains one inner result (and some metadata of its own).
job_result = job.result()

# This is the result from our single pub, which had six observables,
# so contains information on all six.
pub_result = job.result()[0]
```

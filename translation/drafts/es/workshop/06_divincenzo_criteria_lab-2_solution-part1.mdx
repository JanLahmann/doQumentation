## Introducción

El físico David DiVincenzo esbozó cinco requisitos clave para cualquier implementación física de una computadora cuántica, más dos criterios adicionales para la comunicación cuántica. En este notebook, **experimentaremos cada criterio de DiVincenzo a través de demostraciones prácticas con Qiskit**. En lugar de profundizar en la teoría, cada sección explica brevemente un criterio y luego proporciona ejercicios de código usando Qiskit 2. Podrás ejecutar circuitos en simuladores y dispositivos IBM Quantum reales para **explorar cada principio de forma práctica**.

**Los Cinco Criterios de DiVincenzo para la Computación Cuántica**:

1. **Un sistema físico escalable con qubits bien caracterizados.**
2. **Capacidad de inicializar qubits** a un estado fiducial simple (p. ej. |00…0〉).
3. **Tiempos de decoherencia largos** (coherencia del qubit mucho mayor que el tiempo de operación de la compuerta).
4. **Un conjunto universal de compuertas cuánticas** (capaz de realizar operaciones unitarias arbitrarias).
5. **Capacidad de medición específica por qubit** (leer el estado de cada qubit).

*(DiVincenzo también describió dos criterios para la comunicación cuántica: la capacidad de interconvertir qubits estacionarios y "voladores", y de transmitir fielmente qubits voladores entre ubicaciones. Los incluimos en una actividad recomendada al final de este notebook.)*

Cada una de las secciones siguientes corresponde a un criterio. Usaremos Qiskit para ilustrar el concepto con código y **experimentos interactivos** que puedes probar. Por ejemplo, veremos cómo escalar el número de qubits y la profundidad del circuito afecta los resultados (Criterio 1), cómo resetear y preparar estados de qubits (Criterio 2), cómo medir qubits en simuladores frente a dispositivos reales (Criterio 4), cómo Qiskit compone compuertas universales (Criterio 3), y cómo la coherencia finita (T₁, T₂) impacta los cómputos (Criterio 5). Al final, tendrás una intuición más profunda de lo que significa cada criterio de DiVincenzo en la práctica y cómo Qiskit permite experimentar con ellos.

## 1. Criterio 1 – Qubits Escalables y Bien Caracterizados

**Criterio 1:** *"Un sistema físico escalable con qubits bien caracterizados."* Esto significa que necesitamos una plataforma de hardware cuántico donde podamos **aumentar el número de qubits** y aún controlarlos de manera confiable. Las propiedades de cada qubit (niveles de energía, tasas de error, conectividad, etc.) deben estar bien entendidas. Esencialmente, queremos construir circuitos más grandes sin que el sistema se desmorone. En la práctica, a medida que escalamos el conteo de qubits o la profundidad del circuito, los errores y la decoherencia se acumulan, por lo que demostrar *escalabilidad* también implica comprender cómo el aumento de tamaño afecta el rendimiento.

**Objetivo de la Demo:** Usar Qiskit para mostrar el efecto de escalar un circuito (en conteo de qubits o profundidad de compuerta) en la fidelidad de la salida. Simularemos un escenario ideal frente a uno ruidoso para ver cómo un sistema más grande o un circuito más profundo sucumbe a la decoherencia y los errores.

Primero, construyamos un pequeño estado entrelazado (estado GHZ) en 3 qubits, luego uno más grande en 5 qubits, como una prueba de escalado simple. Un estado GHZ de *n* qubits es $\frac{1}{\sqrt{2}}(|0...0\rangle + |1...1\rangle)$. En una simulación ideal, medir un GHZ de n qubits produce solo dos resultados (todos 0s o todos 1s) con igual probabilidad. Compararemos la **salida ideal** con una **salida ruidosa** a medida que aumentamos n o la profundidad del circuito.

```python
from qiskit import QuantumCircuit
from qiskit_aer import AerSimulator
from qiskit.visualization import plot_histogram
from qiskit.transpiler.preset_passmanagers import generate_preset_pass_manager
from qiskit_ibm_runtime import SamplerV2 as Sampler

# 3-qubit GHZ circuit
qc3 = QuantumCircuit(3, 3)
qc3.h(0)
qc3.cx(0, 1)
qc3.cx(1, 2)
qc3.measure([0, 1, 2], [0, 1, 2])

# 5-qubit GHZ circuit (scaling up the number of qubits)
qc5 = QuantumCircuit(5, 5)
qc5.h(0)
qc5.cx(0, range(1, 5))    # entangle qubit 0 with all others
qc5.measure(range(5), range(5))

# Transpile for a simulator backend
sim_backend = AerSimulator()
pm = generate_preset_pass_manager(backend=sim_backend, optimization_level=1)
isa_qc3 = pm.run(qc3)
isa_qc5 = pm.run(qc5)

# Run ideal simulations (no noise)
sampler = Sampler(mode=sim_backend)

job3 = sampler.run([isa_qc3], shots=1024)
result3 = job3.result()
counts3 = result3[0].data.c.get_counts()

job5 = sampler.run([isa_qc5], shots=1024)
result5 = job5.result()
counts5 = result5[0].data.c.get_counts()

print("3-qubit GHZ counts (ideal):", counts3)
plot_histogram(counts3, legend=['3-qubit ideal'], figsize=(6,4))
```

```python
print("5-qubit GHZ counts (ideal):", counts5)
plot_histogram(counts5, legend=['5-qubit ideal'], figsize=(6,4))
```

**Resultado esperado (caso ideal):** El GHZ de 3 qubits idealmente produce aproximadamente 50% `000` y 50% `111` en los conteos. El GHZ de 5 qubits produce ~50% `00000` y 50% `11111`. No aparecen otras cadenas de bits porque el estado es idealmente completamente coherente y entrelazado. Deberías ver dos barras altas en el histograma para cada circuito correspondientes a los resultados de todos ceros y todos unos.

A continuación, veamos qué sucede en un **entorno ruidoso**. Usaremos las capacidades del modelo de ruido de Qiskit Aer para imitar los errores de un dispositivo real. Por ejemplo, podemos tomar las propiedades de un backend de IBM para crear un modelo de ruido que incluya errores de compuerta, tiempos de compuerta finitos, relajación de qubits (T₁), desfase (T₂) y errores de lectura. Aquí, usaremos un **backend falso** que representa el dispositivo IBM Quantum Brisbane para generar un modelo de ruido, y re-ejecutaremos los circuitos GHZ a través de él.

## 2. Criterio 2 – Inicialización de Qubits

**Criterio 2:** *"La capacidad de inicializar el estado de los qubits a un estado fiducial simple, como |000…〉."* Todos los qubits deben comenzar de manera confiable en un estado de referencia conocido (típicamente el estado base |0〉 para cada qubit). La inicialización es esencial para que los algoritmos comiencen con un estado limpio. En la práctica, en los dispositivos cuánticos de IBM cada qubit se resetea automáticamente a |0〉 al inicio de cada ejecución de circuito. Qiskit también proporciona instrucciones para resetear qubits o preparar estados personalizados durante un cómputo.

**Objetivo de la Demo:** Mostrar cómo inicializar qubits en Qiskit, tanto al inicio como en mitad del circuito. Demostraremos el uso de la instrucción `reset` y los métodos de preparación de estados.
### Ejercicio 2: Preparar un Estado Específico
En el bloque de código a continuación, completa el `QuantumCircuit` para preparar el estado $|10\rangle$. Esto significa que el qubit 0 debe estar en el estado $|0\rangle$ y el qubit 1 debe estar en el estado $|1\rangle$. Usa la compuerta e instrucción apropiadas para lograr esto.

```python
from qiskit import QuantumCircuit
from qiskit_aer import AerSimulator

# Create a circuit to initialize qubits to |10> and verify by measurement
qc_init = QuantumCircuit(2, 2)

# --- YOUR CODE HERE ---

# 1. Set qubit 1 to the |1> state
qc_init.x(1)

# 2. Explicitly reset qubit 0 to the |0> state
qc_init.reset(0)

# --- END YOUR CODE ---

qc_init.measure([0, 1], [0, 1])
qc_init.draw('mpl')
```

```python
# Run the circuit and check the outcome
sim_backend = AerSimulator()
pm = generate_preset_pass_manager(backend=sim_backend, optimization_level=1)
isa_qc_init = pm.run(qc_init)

sampler = Sampler(mode=sim_backend)

job = sampler.run([isa_qc_init], shots=1024)
result = job.result()
counts = result[0].data.c.get_counts()

print("Outcome of |10> state measured in Z-basis:", counts)
plot_histogram(counts)
```

Deberías ver `10` (binario para qubit1=1, qubit0=0) con 100% de probabilidad en la simulación, lo que significa que el qubit 1 fue preparado exitosamente en |1〉 y el qubit 0 en |0〉.

Ahora, para una preparación de estado más general, Qiskit permite la inicialización a estados arbitrarios usando el método `initialize`. Por ejemplo, preparemos un qubit en el estado $|+\rangle = (|0\rangle+|1\rangle)/\sqrt{2}$, que es un estado de superposición, y un par de qubits en el estado de Bell $(|00\rangle+|11\rangle)/\sqrt{2}$:

```python
import numpy as np

# Initialize a single qubit in |+> state and measure in Z-basis
qc_plus = QuantumCircuit(1, 1)
state_plus = [1/np.sqrt(2), 1/np.sqrt(2)]   # amplitude for |0> and |1>
qc_plus.initialize(state_plus, 0)
qc_plus.measure(0, 0)

# Initialize two qubits in a Bell state manually
qc_bell = QuantumCircuit(2, 2)
bell_state = [1/np.sqrt(2), 0, 0, 1/np.sqrt(2)]  # amplitudes for |00>,|01>,|10>,|11>
qc_bell.initialize(bell_state, [0, 1])
qc_bell.measure([0, 1], [0, 1])

# Transpile and run the initialization circuits
isa_qc_plus = pm.run(qc_plus)
job_plus = sampler.run([isa_qc_plus], shots=1024)
result_plus = job_plus.result()
counts_plus = result_plus[0].data.c.get_counts()

print("Outcome of |+> state measured in Z-basis:", counts_plus)

isa_qc_bell = pm.run(qc_bell)
job_bell = sampler.run([isa_qc_bell], shots=1024)
result_bell = job_bell.result()
counts_bell = result_bell[0].data.c.get_counts()

print("Outcome of Bell state measured in Z-basis:", counts_bell)
```

**Resultados esperados:** El estado de un solo qubit |+〉, cuando se mide, producirá `0` y `1` con aproximadamente 50% de probabilidad cada uno. La medición del estado de Bell debería dar aproximadamente 50% `00` y 50% `11`. Si ves esto, confirma que nuestra inicialización a esos estados fue exitosa.

**Inicialización en mitad del circuito:** El `reset` de Qiskit se puede usar en medio de un circuito para reinicializar un qubit a |0〉 sobre la marcha. Por ejemplo, en códigos de corrección de errores o algoritmos iterativos, a menudo se mide un qubit y luego se resetea para reutilizarlo. La operación `reset` es determinista; descarta cualquier estado existente y enfría el qubit al estado base.

**Ejemplo en dispositivo:** En hardware como **ibmq_brisbane** (127 qubits) o cualquier dispositivo IBM, todos los qubits comienzan en |0〉 por defecto cuando se ejecuta un trabajo. Si necesitaras un estado inicial diferente, aplicarías compuertas al principio (como hicimos con X para obtener |1〉). La re-inicialización continua (para la corrección de errores cuánticos) es un tema de investigación activo porque hacerlo rápidamente es un desafío. Afortunadamente, para uso básico, la capacidad de comenzar de cero en |0…0〉 está disponible y demostramos cómo lograr otros estados iniciales deseados también.

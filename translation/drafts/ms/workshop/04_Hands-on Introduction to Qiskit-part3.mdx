## 2.4 Analisis Keputusan {#analyze-the-results}

Langkah analisis ni biasanya tempat kau boleh buat post-processing pada keputusan, contohnya guna teknik pengurangan ralat pengukuran atau zero noise extrapolation (ZNE). Kau mungkin nak hantar keputusan ni ke dalam workflow lain untuk analisis lanjut, atau sediakan plot untuk nilai-nilai dan data penting. Secara umumnya, langkah ni memang spesifik ikut masalah kau. Untuk contoh ni, plot setiap expectation value yang diukur untuk Circuit kita.

Expectation value dan sisihan piawai untuk observable yang kau dah tentukan kepada Estimator boleh diakses melalui atribut `PubResult.data.evs` dan `PubResult.data.stds` dalam hasil kerja. Untuk maklumat lanjut, tengok [Get started with Sampler.](https://docs.quantum.ibm.com/guides/get-started-with-primitives#get-started-with-sampler)

## 2.5 Skala ke Bilangan Qubit yang Lebih Besar {#scale-to-large-numbers-of-qubits}

Dalam pengkomputeran kuantum, kerja berskala utiliti sangat penting untuk membuat kemajuan dalam bidang ni. Kerja sebegini memerlukan pengiraan dilakukan pada skala yang jauh lebih besar; bekerja dengan Circuit yang mungkin guna lebih dari 100 Qubit dan lebih dari 1000 Gate. Contoh ni mengambil langkah kecil ke arah tu dengan menskalakan masalah GHZ ke $n=10$ Qubit. Ia guna workflow Qiskit patterns dan berakhir dengan mengukur expectation value $\langle Z_0 Z_i \rangle $.

### Langkah 1. Peta masalah {#step-1-map-the-problem}

Tulis fungsi yang pulangkan `QuantumCircuit` yang sediakan keadaan GHZ $n$-Qubit (pada dasarnya keadaan Bell yang dilanjutkan), kemudian guna fungsi tu untuk sediakan keadaan GHZ 10-Qubit dan kumpulkan observable yang nak diukur.

```python
def get_qc_for_n_qubit_GHZ_state(n: int) -> QuantumCircuit:

    qc = QuantumCircuit(n) 
    qc.h(0)
    for i in range(n-1):
        qc.cx(i, i+1)
    return qc
n = 10
qc_n_GHZ = get_qc_for_n_qubit_GHZ_state(n)
qc_n_GHZ.draw("mpl")
```

### Langkah 4. Post-proses keputusan {#step-4-post-process-results}

Untuk fahami dengan lebih baik kelakuan keadaan kuantum yang terbelit pada perkakasan sebenar, kita analisis korelasi berpasangan antara Qubit dalam asas Z. Khususnya, kita tengok expectation value ⟨Z₀Zᵢ⟩, yang mengukur seberapa kuat Qubit 0 berkorelasi dengan setiap Qubit lain i. Khususnya kita akan plot: 
$$
\langle Z_i Z_0 \rangle / \langle Z_1 Z_0 \rangle 
$$
<div class="alert alert-success">

Nilai $\langle Z_i Z_0 \rangle / \langle Z_1 Z_0 \rangle $ mana yang kau jangka akan nampak dalam plot?

Pilihan:

a) Menurun apabila kita naikkan $i$

b) Malar pada 1

c) Sisihan kecil di sekitar 1

d) Bertukar-tukar antara 1 dan 0 untuk nilai $i$ ganjil dan genap

</div>

```python
data = list(range(1, len(operators) + 1))  # Distance between the Z operators
result = job.result()[0]
values = result.data.evs  # Expectation value at each Z operator.
values = [
    v / values[0] for v in values
]  # Normalize the expectation values to evaluate how they decay with distance.

plt.plot(data, values, marker="o", label=f"{n}-qubit GHZ state")
plt.xlabel("Distance between qubits $i$")
plt.ylabel(r"$\langle Z_i Z_0 \rangle / \langle Z_1 Z_0 \rangle $")
plt.legend()
plt.show()
```

![Plot output](./_04_Hands-on Introduction to Qiskit_imgs/output_21.png)

Dalam plot ni kita perasan yang $\langle Z_0 Z_i \rangle$ turun naik di sekitar nilai 1, walaupun dalam simulasi ideal semua $\langle Z_0 Z_i \rangle$ sepatutnya bernilai 1.

Seperti yang kau nampak, keputusan eksperimen 10 Qubit ni memang bagus tapi masih ada sikit ralat. Salah satu cara untuk baiki keputusan adalah dengan laksanakan keadaan GHZ dengan lebih cekap.

Biasanya orang laksanakan keadaan GHZ dengan urutan Gate CNOT berbentuk tangga. Tapi kau boleh laksanakan keadaan GHZ dengan lebih cekap, mengurangkan kedalaman 2-Qubit dari `n` ke `n/2` atau kurang.
<div class="alert alert-info">

Satu metrik penting untuk nilai seberapa tepat keputusan yang akan diperolehi, atau seberapa sikit bunyi yang Circuit akan ada ialah kedalaman Gate 2-Qubit. Ini sebab kadar ralat untuk Gate 2-Qubit (~10 kali lebih tinggi dari Gate Qubit tunggal) mendominasi ralat keseluruhan Circuit. Guna kod berikut untuk dapatkan kedalaman Gate 2-Qubit sesebuah Circuit.

```
qc.depth(lambda x: x.operation.num_qubits == 2)
```

</div>

```python
def better_ghz(n):
    "fan out"
    s = int(n / 2)
    qc = QuantumCircuit(n)
    qc.h(s)
    for m in range(s, 0, -1):
        qc.cx(m, m - 1)
        if not (n % 2 == 0 and m == s): 
            qc.cx(n - m - 1, n - m)
    return qc

better_ghz(n).draw("mpl")
```

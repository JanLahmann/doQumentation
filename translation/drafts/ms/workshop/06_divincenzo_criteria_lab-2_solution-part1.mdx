## 1. Kriteria 1 – Qubit yang Boleh Diskala dan Berkarakter Baik {#criterion-1-scalable-well-characterized-qubits}

**Kriteria 1:** *"Sistem fizikal yang boleh diskala dengan Qubit yang berkarakter baik."* Ini bermakna kita perlukan platform perkakasan kuantum yang boleh kita **tingkatkan bilangan Qubit** dan masih mengawalnya dengan boleh dipercayai. Sifat setiap Qubit (paras tenaga, kadar ralat, ketersambungan, dsb.) sepatutnya difahami dengan baik. Pada asasnya, kita nak bina Circuit yang lebih besar tanpa sistem itu runtuh. Dalam praktik, apabila kita tingkatkan bilangan Qubit atau kedalaman Circuit, ralat dan dekoherensi terkumpul, jadi menunjukkan *skalabiliti* juga bermakna memahami macam mana peningkatan saiz mempengaruhi prestasi.

**Matlamat Demo:** Guna Qiskit untuk menunjukkan kesan peningkatan skala Circuit (dalam bilangan Qubit atau kedalaman Gate) ke atas ketepatan output. Kita akan simulasikan senario ideal berbanding berisik untuk tengok macam mana sistem yang lebih besar atau Circuit yang lebih dalam tewas kepada dekoherensi dan ralat.

Pertama, jom bina keadaan terbelit kecil (keadaan GHZ) pada 3 Qubit, kemudian yang lebih besar pada 5 Qubit, sebagai ujian skala yang mudah. Keadaan GHZ bagi *n* Qubit ialah $\frac{1}{\sqrt{2}}(|0...0\rangle + |1...1\rangle)$. Dalam simulasi ideal, mengukur GHZ n-Qubit hanya menghasilkan dua keputusan (semua 0 atau semua 1) dengan kebarangkalian yang sama. Kita akan bandingkan **output ideal** dengan **output berisik** apabila kita tingkatkan n atau kedalaman Circuit.

```python
from qiskit import QuantumCircuit
from qiskit_aer import AerSimulator
from qiskit.visualization import plot_histogram
from qiskit.transpiler.preset_passmanagers import generate_preset_pass_manager
from qiskit_ibm_runtime import SamplerV2 as Sampler

# 3-qubit GHZ circuit
qc3 = QuantumCircuit(3, 3)
qc3.h(0)
qc3.cx(0, 1)
qc3.cx(1, 2)
qc3.measure([0, 1, 2], [0, 1, 2])

# 5-qubit GHZ circuit (scaling up the number of qubits)
qc5 = QuantumCircuit(5, 5)
qc5.h(0)
qc5.cx(0, range(1, 5))    # entangle qubit 0 with all others
qc5.measure(range(5), range(5))

# Transpile for a simulator backend
sim_backend = AerSimulator()
pm = generate_preset_pass_manager(backend=sim_backend, optimization_level=1)
isa_qc3 = pm.run(qc3)
isa_qc5 = pm.run(qc5)

# Run ideal simulations (no noise)
sampler = Sampler(mode=sim_backend)

job3 = sampler.run([isa_qc3], shots=1024)
result3 = job3.result()
counts3 = result3[0].data.c.get_counts()

job5 = sampler.run([isa_qc5], shots=1024)
result5 = job5.result()
counts5 = result5[0].data.c.get_counts()

print("3-qubit GHZ counts (ideal):", counts3)
plot_histogram(counts3, legend=['3-qubit ideal'], figsize=(6,4))
```

```python
print("5-qubit GHZ counts (ideal):", counts5)
plot_histogram(counts5, legend=['5-qubit ideal'], figsize=(6,4))
```

**Keputusan yang dijangka (kes ideal):** GHZ 3-Qubit secara ideal menghasilkan kira-kira 50% `000` dan 50% `111` dalam kiraan. GHZ 5-Qubit menghasilkan ~50% `00000` dan 50% `11111`. Tiada rentetan bit lain muncul kerana keadaan itu secara ideal sepenuhnya koheren dan terbelit. Kau patut nampak dua bar tinggi pada histogram untuk setiap Circuit yang sepadan dengan keputusan semua-sifar dan semua-satu.

Seterusnya, jom tengok apa yang berlaku dalam **persekitaran berisik**. Kita akan guna keupayaan model bunyi Qiskit Aer untuk meniru ralat peranti sebenar. Sebagai contoh, kita boleh ambil sifat Backend IBM untuk mencipta model bunyi yang merangkumi ralat Gate, masa Gate terhad, kelonggaran Qubit (T₁), defasaan (T₂), dan ralat pembacaan. Di sini, kita akan guna **fake backend** yang mewakili peranti IBM Quantum Brisbane untuk menjana model bunyi, dan jalankan semula Circuit GHZ melaluinya.
### Latihan 1a: Simulasi dengan Bunyi {#exercise-1a-simulate-with-noise}
Lengkapkan kod di bawah untuk mensimulasikan Circuit GHZ pada simulator berisik berdasarkan Backend `FakeBrisbane`. Ini akan tunjukkan kepada kau macam mana prestasi merosot apabila sistem diskala dalam persekitaran bunyi yang realistik.

```python
from qiskit_ibm_runtime.fake_provider import FakeBrisbane

# We will reuse the ideal circuits qc3 and qc5 and their results from the previous cell.

# 1. Create a fake backend for IBM Quantum Brisbane
brisbane_backend = FakeBrisbane()

# 2. Create a noisy AerSimulator from the fake backend's properties
noisy_sim = AerSimulator.from_backend(brisbane_backend)

# 3. Transpile the circuits for the noisy simulator
pm = generate_preset_pass_manager(backend=noisy_sim, optimization_level=1)

isa_qc3_noisy = pm.run(qc3)

isa_qc5_noisy = pm.run(qc5)

# 4. Run the noisy simulations using the Sampler and get the counts
sampler = Sampler(mode=noisy_sim)

job3 = sampler.run([isa_qc3_noisy], shots=1024)

result3_noisy = job3.result()

counts3_noisy = result3_noisy[0].data.meas.get_counts()

job5 = sampler.run([isa_qc5_noisy], shots=1024)

result5_noisy = job5.result()

counts5_noisy = result5_noisy[0].data.meas.get_counts()

# --- END YOUR CODE ---

# This part is done for you to visualize the results.
print("3-qubit GHZ counts (noisy):", counts3_noisy)
plot_histogram(counts3_noisy, legend=['3-qubit noisy'], figsize=(6,4))
```

**Keputusan yang dijangka (berisik berbanding ideal):** Dengan bunyi, sama ada disimulasikan atau pada peranti sebenar, keadaan GHZ adalah **kurang sempurna**. Kau akan nampak keputusan tambahan selain semua-0 dan semua-1. Untuk 3 Qubit, daripada 100% dalam `000`/`111`, sebahagian kebarangkalian bocor ke rentetan bit lain (cth. `001`, `010`, dsb.) akibat ralat Gate atau dekoherensi yang membalikkan sebahagian Qubit. Untuk 5 Qubit, kesannya lebih ketara; Circuit yang lebih besar (lebih banyak Qubit dan Gate CNOT) terkumpul lebih banyak ralat, jadi puncak semua-0 dan semua-1 lebih rendah, dan banyak keputusan lain muncul. Tren ini menggambarkan cabaran *skalabiliti*: apabila kita tingkatkan skala, mengekalkan ketepatan tinggi menjadi lebih sukar tanpa pembetulan ralat.

**Pandangan:** Komputer kuantum yang boleh diskala perlu mengekalkan korelasi kuantum apabila sistem berkembang. Contoh-contoh kita menunjukkan macam mana peningkatan bilangan Qubit/kedalaman Gate menyebabkan ketepatan output merosot apabila bunyi hadir. Kriteria-kriteria yang selebihnya akan berkaitan dengan memastikan Qubit itu berkelakuan baik (ralat rendah, boleh dimulakan, dsb.) apabila kita tingkatkan skala.

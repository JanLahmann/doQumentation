## 2.1 Petakan masalah ke format natif kuantum {#map-the-problem-to-a-quantum-native-format}

Dalam program kuantum, *quantum circuits* adalah format natif untuk mewakili arahan kuantum, dan *operators* pula mewakili observable yang nak diukur. Bila buat Circuit, kau biasanya akan buat objek [`QuantumCircuit`](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.QuantumCircuit#quantumcircuit-class) baru, lepas tu tambah arahan satu per satu ikut urutan.

Kod berikut buat Circuit yang menghasilkan keadaan GHZ, iaitu keadaan di mana tiga Qubit saling terikat penuh antara satu sama lain.

<div class="alert alert-info">

  Qiskit SDK guna penomboran bit LSb 0 di mana digit ke-$n$ mempunyai nilai $1 \ll n$ atau $2^n$. Untuk maklumat lanjut, tengok topik [Bit-ordering in the Qiskit SDK](https://docs.quantum.ibm.com/guides/bit-ordering).

</div>

```python
# Create a GHZ state circuit

qc = QuantumCircuit(3)
qc.h(0)
qc.cx(0,1)
qc.cx(0,2)
# Draw the circuit
qc.draw("mpl")
```

![Quantum circuit diagram](./_04_Hands-on Introduction to Qiskit_imgs/output_15.png)

Tengok [`QuantumCircuit`](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.QuantumCircuit#quantumcircuit-class) dalam dokumentasi untuk semua operasi yang tersedia.

Bila buat quantum circuits, kau kena fikirkan juga jenis data apa yang nak dikembalikan selepas pelaksanaan. Qiskit menyediakan dua cara untuk kembalikan data: kau boleh dapat taburan kebarangkalian untuk set Qubit yang kau pilih untuk diukur, atau kau boleh dapat nilai jangkaan (expectation value) bagi sesuatu observable. Sediakan beban kerja kau untuk mengukur Circuit dalam salah satu daripada dua cara ni dengan [Qiskit primitives](https://docs.quantum.ibm.com/guides/get-started-with-primitives) (dijelaskan dengan terperinci dalam Langkah 3).

Contoh ni mengukur expectation values dengan menggunakan submodul `qiskit.quantum_info`, yang ditentukan menggunakan operators (objek matematik yang digunakan untuk mewakili tindakan atau proses yang mengubah keadaan kuantum). Kod berikut mencipta enam operator Pauli tiga Qubit: `ZZZ`, `ZZX`, `ZII`, `XXI`, `ZZI` dan `III`.

```python
# Set up six different observables.

observables_labels = ["ZZZ", "ZZX", "ZII", "XXI", "ZZI", "III"]

observables = [SparsePauliOp(label) for label in observables_labels]
print(observables)
```

```text
[SparsePauliOp(['ZZZ'],
              coeffs=[1.+0.j]), SparsePauliOp(['ZZX'],
              coeffs=[1.+0.j]), SparsePauliOp(['ZII'],
              coeffs=[1.+0.j]), SparsePauliOp(['XXI'],
              coeffs=[1.+0.j]), SparsePauliOp(['ZZI'],
              coeffs=[1.+0.j]), SparsePauliOp(['III'],
              coeffs=[1.+0.j])]
```

<div class="alert alert-info">

  Di sini, operator seperti `ZZI` adalah singkatan untuk hasil darab tensor $Z\otimes Z\otimes I$, yang bermaksud mengukur Z pada Qubit 2 dan Z pada Qubit 1 secara serentak, dan mendapatkan maklumat tentang korelasi antara Qubit 2 dan Qubit 1. Expectation values seperti ni biasanya ditulis juga sebagai $\langle Z_2 Z_1 \rangle$.

  Kalau keadaan yang kita perhatikan adalah keadaan GHZ tiga Qubit, maka pengukuran $\langle Z_2 Z_1 \rangle$ sepatutnya bernilai 1.

</div>

## 2.2 Optimumkan circuits dan operators {#optimize-the-circuits-and-operators}

Bila laksanakan circuits pada peranti, penting untuk optimumkan set arahan yang terkandung dalam Circuit dan minimumkan kedalaman keseluruhan (lebih kurang bilangan arahan) Circuit tu. Ini memastikan kau dapat hasil terbaik dengan mengurangkan kesan ralat dan hingar. Selain tu, arahan dalam Circuit mesti mematuhi [Instruction Set Architecture (ISA)](https://docs.quantum.ibm.com/guides/transpile#instruction-set-architecture) peranti Backend dan kena ambil kira gate asas serta sambungan Qubit peranti tersebut.

Kod berikut memulakan peranti sebenar untuk hantar kerja ke sana dan mengubah Circuit serta operators untuk sepadan dengan ISA Backend tu.
Kalau kau belum simpan kelayakan kau sebelum ni, ikut arahan [di sini](https://docs.quantum.ibm.com/guides/setup-channel#iqp) untuk mengesahkan dengan token API kau.

```python
# Choose a real backend
service = QiskitRuntimeService(channel='ibm_quantum_platform',)

    f"No. of qubits: {backend.num_qubits}\n"
    f"Processor type: {backend.processor_type}\n"
)
```

## 2.3 Laksanakan menggunakan quantum primitives {#execute-using-the-quantum-primitives}

Komputer kuantum boleh menghasilkan keputusan rawak, jadi kau biasanya kumpul sampel output dengan menjalankan Circuit banyak kali. Kau boleh anggarkan nilai observable menggunakan kelas `Estimator`. `Estimator` adalah salah satu daripada dua [primitives](https://docs.quantum.ibm.com/guides/get-started-with-primitives); yang satu lagi ialah `Sampler`, yang boleh digunakan untuk dapat data daripada komputer kuantum. Objek-objek ni ada kaedah `run()` yang melaksanakan pilihan circuits, observables, dan parameter (jika berkenaan), menggunakan [primitive unified bloc (PUB).](https://docs.quantum.ibm.com/guides/primitives#sampler)
Bila jalankan kod ni pada perkakasan kuantum sebenar, pertimbangkan untuk gunakan [teknik pengurangan dan penindasan ralat](https://quantum.cloud.ibm.com/docs/en/guides/error-mitigation-and-suppression-techniques) bagi mengurangkan hingar intrinsik komputer kuantum.

```python
# Construct the Estimator instance.
estimator = Estimator(mode=backend)
estimator.options.resilience_level = 1
estimator.options.default_shots = 5000
```

Hantar kerja menggunakan Estimator primitive.

```python
# One pub, with one circuit to run against six different observables.
job = estimator.run([(isa_circuit, mapped_observables)]) 

# Use the job ID to retrieve your job data later
print(f">>> Job ID: {job.job_id()}")
```

(Tengok [bahagian tentang mendapatkan semula kerja](https://docs.quantum.ibm.com/guides/monitor-job#retrieve-job-results-at-a-later-time) untuk butiran lanjut.)

Selepas kerja selesai, periksa outputnya melalui atribut `result()` kerja tersebut.

```python
# This is the result of the entire submission.  You submitted one Pub,
# so this contains one inner result (and some metadata of its own).
job_result = job.result()

# This is the result from our single pub, which had six observables,
# so contains information on all six.
pub_result = job.result()[0]
```

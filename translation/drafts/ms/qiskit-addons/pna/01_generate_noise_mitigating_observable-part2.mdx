#### Kaitkan kotak litar dengan hingar yang dipelajari {#associate-circuit-boxes-with-learned-noise}

Di sini, kita cipta satu pemetaan antara ID rujukan ``InjectNoise`` bagi setiap kotak dengan model hingar yang dipelajari (`PauliLindbladMap`) yang mempengaruhi get-get penjalin dalam kotak tersebut.

```python
from samplomatic.annotations import InjectNoise
from samplomatic.utils import get_annotation

# map inject noise refs to pauli lindblad maps
refs_to_noise_models = {}
for instruction, result in zip(unique_2q_layers_and_meas, noise_learner_result, strict=False):
    if inject_noise_annot := get_annotation(instruction.operation, InjectNoise):
        refs_to_noise_models[inject_noise_annot.ref] = result.to_pauli_lindblad_map()
```

#### Sebarkan observable melalui anti-hingar yang dipelajari untuk mendapatkan observable penyerap hingar {#propagate-the-observable-through-the-learned-anti-noise-to-get-a-noise-mitigating-observable}

Seperti yang dibincangkan di atas, ini dilakukan dalam dua langkah. Pertama, kita sebarkan penjana anti-hingar ke penghujung litar. Selepas itu, kita sebarkan observable melalui penjana yang telah berkembang itu. Proses ini diulang untuk setiap penjana anti-hingar dalam litar. Dalam pelaksanaan ini, setiap penjana dalam lapisan tertentu disebarkan ke penghujung litar secara selari. Selain itu, pemprosesan berbilang Python digunakan untuk menjalankan penyebaran ke hadapan bagi anti-hingar serta penyebaran ke belakang bagi observable secara selari. Ini menghalang penumpukan penjana yang telah berkembang dalam ingatan dan juga memaksimumkan sumber pengiraan.

Semasa menjalankan PNA, anda sentiasa perlu menyediakan litar bermasalah dan observable. Jika litar bermasalah anda adalah litar berbentuk kotak dengan anotasi `InjectNoise`, anda perlu menyediakan pemetaan yang kita cipta dalam langkah di atas. Seseorang boleh juga menghantar litar tanpa kotak yang mengandungi arahan ``PauliLindbladError`` daripada ``qiskit-aer``. Dalam kes itu, ``refs_to_noise_models`` tidak perlu disediakan. Selain input utama, pengguna perlu mempertimbangkan perkara berikut:

- `max_err_terms`: Bilangan sebutan yang perlu disimpan dalam setiap penjana anti-hingar semasa ia disebarkan ke hadapan. Nilai yang lebih besar biasanya meningkatkan ketepatan, tetapi tingkah laku ini tidak dijamin bersifat monoton.
- `max_obs_terms`: Bilangan sebutan yang perlu disimpan dalam observable penyerap hingar, $\tilde{O}$, semasa ia disebarkan ke belakang melalui anti-hingar yang telah berkembang. Nilai yang lebih besar biasanya meningkatkan ketepatan, tetapi ini tidak dijamin berlaku secara monoton.
- `num_processes`: Bilangan teras yang didedikasikan kepada proses ini. Ingat, penjana disebarkan ke hadapan dan digunakan pada observable secara selari.
- `search_step`: Langkah penyebaran ke belakang menggunakan kaedah tamak untuk mengkonjugat dua pengoperasi dalam asas Pauli secara anggaran. Kaedah ini boleh dipercepatkan dengan menambah ``search_step``. Lihat [dokumentasi pauli-prop](https://qiskit.github.io/pauli-prop/) untuk maklumat lanjut.
- `num_to_measure`: Walaupun pemboleh ubah ini bukan input kepada ``generate_noise_mitigating_observable``, kita gunakannya untuk mengawal berapa banyak sebutan daripada $\tilde{O}$ yang kita sebenarnya mahu ukur. Di sini kita hanya akan mengukur 30 sebutan teratas, yang merupakan sebutan asal dalam observable kita. Sebutan-sebutan ini kini telah diskalakan semula supaya mengukurnya memberi kesan mitigasi hingar get yang dipelajari. Walaupun kita hanya mengukur 30 sebutan daripada $\tilde{O}$, ia masih kerap berguna untuk membiarkannya berkembang besar, kerana itu meningkatkan ketepatan faktor penskalaan sebutan-sebutan terkemuka.

```python
from qiskit_addon_pna import generate_noise_mitigating_observable

# PNA parameters
num_processes = 8
max_err_terms = 10_000
max_obs_terms = 10_000
num_to_measure = num_qubits

obs_tilde_isa = generate_noise_mitigating_observable(
    boxed_circuit,
    isa_observable,
    refs_to_noise_models,
    max_err_terms=max_err_terms,
    max_obs_terms=max_obs_terms,
    num_processes=num_processes,
    print_progress=True,
    search_step=8,
)
p_2_v = {p: v for v, p in enumerate(layout)}
obs_tilde_virtual = SparsePauliOp.from_sparse_list(
    [
        (pstr, [p_2_v[p] for p in p_qubits], coeff)
        for (pstr, p_qubits, coeff) in obs_tilde_isa.to_sparse_list()
    ],
    num_qubits=num_qubits,
)
obs_tilde_virtual = obs_tilde_virtual[np.argsort(np.abs(obs_tilde_virtual.coeffs))[::-1]][
    :num_to_measure
]
```

#### Tukar asas pengukuran kepada bentuk kanonik {#transform-the-measurement-bases-to-canonical-form}

Seterusnya, kita akan mencari set asas minimum untuk diukur supaya kita boleh meliputi setiap sebutan Pauli dalam observable yang diukur (***banyak observable boleh diukur serentak jika ia bertukar antara satu sama lain secara Qubit-demi-Qubit***). Memandangkan kita hanya mengukur sebutan dalam observable asal kita, yang merupakan jumlah semua Pauli `Z` tunggal, hanya satu asas yang diperlukan -- asas semua-`Z`.

Selain mencari set asas pengukuran Pauli, kita perlu memetakan sebutan Pauli ini ke bentuk kanonik yang dijangkakan oleh primitif ``Executor``. Untuk maklumat lanjut tentang penataan Qubit kanonik, lawati [dokumentasi samplomatic](https://qiskit.github.io/samplomatic/guides/samplex_io.html#qubit-ordering-convention).

```python
from qiskit_addon_utils.exp_vals.measurement_bases import get_measurement_bases

meas_box = boxed_circuit.data[-1]
canonical_qubits = [
    idx for idx, qubit in enumerate(boxed_circuit.qubits) if qubit in meas_box.qubits
]
c_2_p = {c: p for c, p in enumerate(canonical_qubits)}  # canonical -> physical
p_2_v = {p: v for v, p in enumerate(layout)}  # physical -> virtual
c_2_v = {c: p_2_v[p] for c, p in c_2_p.items()}  # canonical -> virtual
meas_bases, bases_reverser = get_measurement_bases(obs_tilde_virtual)
meas_bases_canonical = [
    np.array([base[c_2_v[c]] for c in range(num_qubits)], dtype=np.uint8) for base in meas_bases
]
```

#### Tentukan cara pensampelan dalam ``QuantumProgram`` {#specify-how-to-sample-in-the-quantumprogram}

``QuantumProgram`` adalah tempat kita menentukan cara eksperimen akan disampel:

- ``template_circuit``: Circuit yang mengandungi semua get yang diperlukan untuk melaksanakan semua rawak yang dikehendaki (daripada rawak pengelintiran, parameter, dan sebagainya).
- ``samplex``: Objek yang menakrifkan taburan kebarangkalian bagi semua rawak Circuit yang mungkin untuk disampel.
- ``samplex_arguments``: Pengikatan yang diperlukan untuk mentakrifkan samplex sepenuhnya
    - ``basis_changes``: Di sinilah kita menentukan set asas untuk diukur yang akan meliputi semua sebutan Pauli dalam observable yang diukur.
    - ``noise_scales.ref``: Kita tetapkan skala setiap lapisan hingar kepada `0.0` untuk mengelak sebarang hingar tambahan daripada disuntik ke dalam sampel kita
    - ``pauli_lindblad_maps``: Diperlukan jika ``noise_scales`` dihantar. Ini hanya memetakan lapisan hingar kepada model hingar yang berkaitan.
- ``shape``: Tuple bentuk untuk melanjutkan bentuk tersirat yang ditakrifkan oleh ``samplex_arguments``. Paksi bukan remeh yang diperkenalkan oleh lanjutan ini menghitung rawak.

```python
from qiskit_ibm_runtime import QuantumProgram

# Control the # of shots during execution
shots_per_randomization_exec = 64
num_randomizations_exec = 6144

# Zero out the noise to prevent noise from being injected during execution.
# We only added InjectNoise annotations so PNA could associate the noise
# to layers in the circuit
samplex_inputs = {f"noise_scales.{ref}": 0.0 for ref in refs_to_noise_models}
samplex_inputs |= {"pauli_lindblad_maps": refs_to_noise_models}

# Specify the bases to measure
bases_broadcastable = np.expand_dims(np.array(meas_bases_canonical), axis=1)
samplex_inputs |= {"basis_changes": {"basis0": bases_broadcastable}}

# Convert samplex_inputs into a dict to pass to QuantumProgram
samplex_arguments = samplex.inputs().make_broadcastable().bind(**samplex_inputs)

# Instantiate the QuantumProgram with the specified parameters
program = QuantumProgram(shots=shots_per_randomization_exec)
program.append(
    circuit=template_circuit,
    samplex=samplex,
    samplex_arguments=samplex_arguments,
    shape=(num_randomizations_exec),
)
```

#### Sampel Circuit menggunakan prototaip primitif ``Executor`` {#sample-the-circuit-using-the-executor-primitive-prototype}

Kini setelah kita menakrifkan ``QuantumProgram`` kita, melaksanakan eksperimen adalah mudah. Kita hanya perlu menginstansi objek ``Executor``, menyediakan Backend kepadanya, dan menjalankan program.

```python
from qiskit_ibm_runtime import Executor

# Execute (sample) the circuit
executor = Executor(backend)
job_exec = executor.run(program)
exec_results = job_exec.result()
```

#### Proses selepas sampel untuk mengira nilai jangkaan termitigasi ralat {#post-process-the-samples-to-calculate-an-error-mitigated-expectation-value}

Untuk mengira nilai jangkaan termitigasi ralat, kita akan:

- Kira faktor penskalaan TREX berdasarkan hingar yang dipelajari yang mempengaruhi pengukuran
- Jana topeng untuk menyimpan hanya sampel yang terpilih selepas pemilihan
- Gunakan fungsi ``executor_expectation_values`` daripada ``qiskit-addon-utils`` untuk menggabungkan semua data menjadi nilai jangkaan termitigasi ralat.

```python
from qiskit_addon_utils.exp_vals.expectation_values import executor_expectation_values
from qiskit_addon_utils.noise_management import trex_factors
from qiskit_addon_utils.noise_management.post_selection import PostSelector

# Computing the TREX factors
measurement_noise_map = noise_learner_result[2].to_pauli_lindblad_map()
trex_rescale_factors = trex_factors(measurement_noise_map, bases_reverser)

# Post-select the results
post_selector = PostSelector.from_circuit(
    circuit=template_circuit, coupling_map=backend.coupling_map
)

# Compute the ps mask for filtering results
mask = post_selector.compute_mask(exec_results[0], strategy="edge")

# Compute expvals using post selected results
results = executor_expectation_values(
    exec_results[0]["meas"],
    bases_reverser,
    meas_basis_axis=0,
    avg_axis=1,
    measurement_flips=exec_results[0]["measurement_flips.meas"],
    pauli_signs=exec_results[0].get("pauli_signs", None),
    postselect_mask=mask,
    rescale_factors=trex_rescale_factors,
)
```

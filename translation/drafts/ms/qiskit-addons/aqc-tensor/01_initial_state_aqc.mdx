Dalam notebook ini, kita akan lalui langkah-langkah sebuah [pola Qiskit](https://quantum.cloud.ibm.com/docs/guides/intro-to-patterns) sambil menggunakan **penggabungan kuantum anggaran dengan rangkaian tensor (AQC-Tensor)** untuk mencapai kedalaman Circuit yang lebih rendah daripada yang biasanya diperlukan untuk melakukan evolusi Trotter.

Ini adalah langkah-langkah yang akan kita ambil:

- **Langkah 1: Peta ke masalah kuantum**
    - Mulakan Hamiltonian dan boleh diperhatikan (observable) masalah kita
    - <font color='#0F62FE'>Jana keadaan rangkaian tensor sasaran untuk bahagian awal Circuit</font>
    - <font color='#0F62FE'>Jana Circuit kedalaman rendah yang menghampiri bahagian yang dimampatkan</font>
    - <font color='#0F62FE'>Jana ansatz umum daripada Circuit tersebut</font>
    - <font color='#0F62FE'>Optimumkan parameter untuk membawa ansatz sedekat mungkin ke sasaran</font>
    - <font color='#0F62FE'>Tambah langkah Trotter seterusnya kepada ansatz yang dioptimumkan</font>
- **Langkah 2: Optimumkan untuk perkakasan sasaran**
    - Transpile Circuit untuk perkakasan
- **Langkah 3: Jalankan eksperimen**
    - Guna backend palsu untuk kesederhanaan
- **Langkah 4: Bina semula keputusan**
    - T/B; sebaliknya, kita hanya keluarkan observable yang diukur
## Langkah 1: Peta ke Circuit kuantum dan operator {#step-1-map-to-quantum-circuit-and-operator}

### Sediakan Hamiltonian model dan observable {#set-up-a-model-hamiltonian-and-observable}

Dalam notebook ini, kita guna model Ising pada bulatan 10 tapak:
$$
\hat{\mathcal{H}}_{\text{Ising}} = \sum_{i=1}^{10} J_{i,(i+1)} Z_i Z_{(i+1)} + h_i X_i \, ,
$$
di mana syarat sempadan berkala bermaksud bahawa untuk $i=10$ kita dapat $i+1=11\rightarrow1$, $J$ ialah kekuatan gandingan antara dua tapak dan $h$ ialah medan magnet luaran.

```python
# Added by doQumentation — required packages for this notebook

```

```python
from qiskit.transpiler import CouplingMap
from qiskit_addon_utils.problem_generators import generate_xyz_hamiltonian

# Generate some coupling map to use for this example
coupling_map = CouplingMap.from_heavy_hex(3, bidirectional=False)

# Choose a 10-qubit circle on this coupling map
reduced_coupling_map = coupling_map.reduce([0, 13, 1, 14, 10, 16, 4, 15, 3, 9])

# Get a qubit operator describing the Ising field model
hamiltonian = generate_xyz_hamiltonian(
    reduced_coupling_map,
    coupling_constants=(0.0, 0.0, 1.0),
    ext_magnetic_field=(0.4, 0.0, 0.0),
)
```

Observable yang akan kita ukur ialah jumlah pemagnetan.

```python
from qiskit.quantum_info import SparsePauliOp

L = reduced_coupling_map.size()
observable = SparsePauliOp.from_sparse_list([("Z", [i], 1 / L / 2) for i in range(L)], num_qubits=L)
```

### Tentukan berapa banyak evolusi masa yang perlu disimulasikan secara klasik {#determine-how-much-of-the-time-evolution-to-simulate-classically}

Matlamat keseluruhan kita ialah untuk mensimulasikan evolusi masa model Hamiltonian di atas. Kita lakukan ini dengan evolusi Trotter, yang kita bahagikan kepada dua bahagian:

1. Bahagian awal yang boleh disimulasikan dengan keadaan produk matriks (MPS). Kita akan "kompil" bahagian ini menggunakan AQC seperti yang dibentangkan dalam https://arxiv.org/abs/2301.08609.
2. Bahagian Circuit seterusnya yang akan dijalankan pada perkakasan.
Mari rancang untuk guna AQC-Tensor bagi memampatkan Circuit evolusi masa kita sehingga masa $t=4$, kemudian berkembang menggunakan langkah Trotter biasa sehingga $t=5$.
### Jana Circuit sebelum dan selepas pemisahan {#generate-circuits-before-and-after-split}

Sekarang kita dah pilih untuk membahagi pada $t=4$, kita akan jana dua Circuit:

1. Circuit "sasaran" untuk bahagian AQC evolusi, dari $t_i=0$ hingga $t_f=4$. Kerana ini disimulasikan oleh simulator rangkaian tensor, bilangan lapisan hanya mempengaruhi masa pelaksanaan dengan faktor malar, jadi kita boleh guna bilangan lapisan yang banyak untuk meminimumkan ralat Trotter.

```python
from qiskit.synthesis import SuzukiTrotter
from qiskit_addon_utils.problem_generators import generate_time_evolution_circuit

aqc_evolution_time = 4.0
aqc_target_num_trotter_steps = 45

aqc_target_circuit = generate_time_evolution_circuit(
    hamiltonian,
    synthesis=SuzukiTrotter(reps=aqc_target_num_trotter_steps),
    time=aqc_evolution_time,
)
```

2. Circuit evolusi seterusnya, yang berkembang dari $t_i=4$ hingga $t_f=5$. Kerana ini akan dijalankan pada perkakasan kuantum, adalah lebih baik guna bilangan lapisan Trotter yang sesedikit mungkin.

```python
subsequent_evolution_time = 1.0
subsequent_num_trotter_steps = 5

subsequent_circuit = generate_time_evolution_circuit(
    hamiltonian,
    synthesis=SuzukiTrotter(reps=subsequent_num_trotter_steps),
    time=subsequent_evolution_time,
)
```

Untuk perbandingan kemudian, jom kita jana Circuit ketiga: satu yang berkembang untuk `aqc_evolution_time` tetapi mempunyai masa evolusi per langkah Trotter yang sama dengan Circuit seterusnya. Ini adalah Circuit yang akan kita gunakan kalau kita tak guna bilangan langkah Trotter yang banyak untuk Circuit sasaran. Kita akan rujuk ini sebagai _Circuit perbandingan_.

```python
aqc_comparison_num_trotter_steps = int(
    subsequent_num_trotter_steps / subsequent_evolution_time * aqc_evolution_time
)
aqc_comparison_num_trotter_steps
```


```python
comparison_circuit = generate_time_evolution_circuit(
    hamiltonian,
    synthesis=SuzukiTrotter(reps=aqc_comparison_num_trotter_steps),
    time=aqc_evolution_time,
)
```

### Jana ansatz dan parameter awal daripada Circuit Trotter dengan lebih sedikit langkah {#generate-an-ansatz-and-initial-parameters-from-a-trotter-circuit-with-fewer-steps}

Pertama, kita bina Circuit "bagus" yang mempunyai masa evolusi yang sama dengan Circuit sasaran, tetapi dengan lebih sedikit langkah Trotter (dan dengan itu lebih sedikit lapisan).

Kemudian kita hantar Circuit "bagus" ini ke fungsi `generate_ansatz_from_circuit` AQC-Tensor. Fungsi ini menganalisis ketersambungan dua Qubit Circuit dan mengembalikan dua perkara:
1. Circuit ansatz umum berparameter dengan ketersambungan dua Qubit yang sama seperti Circuit input; dan,
2. parameter yang, apabila dimasukkan ke dalam ansatz, menghasilkan Circuit input (bagus).

Tak lama lagi kita akan ambil parameter-parameter ini dan laraskan secara berulang untuk membawa Circuit ansatz sedekat mungkin ke MPS sasaran.

```python
from qiskit_addon_aqc_tensor import generate_ansatz_from_circuit

aqc_ansatz_num_trotter_steps = 5

aqc_good_circuit = generate_time_evolution_circuit(
    hamiltonian,
    synthesis=SuzukiTrotter(reps=aqc_ansatz_num_trotter_steps),
    time=aqc_evolution_time,
)

aqc_ansatz, aqc_initial_parameters = generate_ansatz_from_circuit(
    aqc_good_circuit, qubits_initially_zero=True
)
aqc_ansatz.draw("mpl", fold=-1)
```

### Pilih tetapan untuk simulasi rangkaian tensor {#choose-settings-for-tensor-network-simulation}

Di sini, kita guna simulator rangkaian tensor berdasarkan [quimb](http://quimb.readthedocs.io/). Dalam contoh ini, kita guna simulator keadaan produk matriks (MPS) quimb, dan kita guna [JAX](https://docs.jax.dev/en/latest/) untuk pembezaan automatik. Tengok [dokumentasi API](../stubs/qiskit_addon_aqc_tensor.simulation.quimb.QuimbSimulator.rst) untuk maklumat lanjut tentang cara guna simulator quimb.

```python
from functools import partial

import quimb.tensor

from qiskit_addon_aqc_tensor.simulation.quimb import QuimbSimulator

simulator_settings = QuimbSimulator(
    partial(quimb.tensor.CircuitMPS, max_bond=100, cutoff=1e-8),
    autodiff_backend="jax",
)
```

### Bina representasi keadaan produk matriks bagi keadaan sasaran AQC {#construct-matrix-product-state-representation-of-the-aqc-target-state}

Seterusnya, kita bina representasi produk matriks bagi keadaan yang hendak dihampirkan oleh AQC.

```python
from qiskit_addon_aqc_tensor.simulation import tensornetwork_from_circuit

aqc_target_mps = tensornetwork_from_circuit(aqc_target_circuit, simulator_settings)
```

Perhatikan bahawa kerana kita memilih bilangan langkah Trotter yang banyak untuk keadaan sasaran, ia sebenarnya mempunyai ralat Trotter yang lebih kecil daripada Circuit perbandingan. Kita boleh kira fideliti ($| \langle \psi_1 | \psi_2 \rangle |^2$) keadaan yang disediakan oleh Circuit perbandingan berbanding keadaan sasaran:

```python
from qiskit_addon_aqc_tensor.simulation import compute_overlap

comparison_mps = tensornetwork_from_circuit(comparison_circuit, simulator_settings)
comparison_fidelity = abs(compute_overlap(comparison_mps, aqc_target_mps)) ** 2
comparison_fidelity
```

### Optimumkan parameter ansatz menggunakan pengiraan MPS {#optimize-the-parameters-of-the-ansatz-using-mps-calculations}

Di sini, kita minimumkan fungsi kos yang paling mudah, `MaximizeStateFidelity`, dengan menggunakan pengoptimum L-BFGS daripada scipy.

Kita pilih titik berhenti untuk fideliti supaya ia akan melebihi apa yang Circuit perbandingan mampu capai tanpa menggunakan AQC. Bila ini dicapai, Circuit yang dimampatkan mempunyai ralat Trotter yang lebih kecil _dan_ kedalaman yang lebih rendah daripada Circuit asal. Dengan lebih banyak masa pemprosesan, langkah pengoptimuman selanjutnya boleh dilakukan untuk meningkatkan fideliti lagi.

```python
from scipy.optimize import OptimizeResult, minimize

from qiskit_addon_aqc_tensor.objective import MaximizeStateFidelity

objective = MaximizeStateFidelity(aqc_target_mps, aqc_ansatz, simulator_settings)

stopping_point = 1 - comparison_fidelity

def callback(intermediate_result: OptimizeResult):
    print(f"Intermediate result: Fidelity {1 - intermediate_result.fun:.8}")
    if intermediate_result.fun < stopping_point:
        # Good enough for now
        raise StopIteration

result = minimize(
    objective.loss_function,
    aqc_initial_parameters,
    method="L-BFGS-B",
    jac=True,
    options={"maxiter": 100},
    callback=callback,
)
if result.status not in (
    0,
    1,
    99,
):  # 0 => success; 1 => max iterations reached; 99 => early termination via StopIteration
    raise RuntimeError(f"Optimization failed: {result.message} (status={result.status})")

print(f"Done after {result.nit} iterations.")
aqc_final_parameters = result.x
```

## Langkah 2: Transpile untuk pelaksanaan pada perkakasan sasaran {#step-2-transpile-for-execution-on-target-hardware}

Dalam Langkah 2 sebuah [pola Qiskit](https://quantum.cloud.ibm.com/docs/guides/intro-to-patterns), kita Transpile Circuit ini dan mana-mana observable yang diingini untuk pelaksanaan pada peranti sasaran. Di sini kita guna backend palsu yang disediakan oleh `qiskit-ibm-runtime`.

```python
from qiskit import transpile
from qiskit_ibm_runtime.fake_provider import FakeMelbourneV2

backend = FakeMelbourneV2()

isa_circuit = transpile(final_circuit, backend)
isa_observable = observable.apply_layout(isa_circuit.layout)
```

Circuit ISA yang dihasilkan kemudiannya boleh dihantar untuk pelaksanaan pada Backend (langkah 3 sebuah [pola Qiskit](https://quantum.cloud.ibm.com/docs/guides/intro-to-patterns)).
## Langkah 3: Jalankan pada perkakasan kuantum {#step-3-execute-on-quantum-hardware}

```python
from qiskit_ibm_runtime import EstimatorV2 as Estimator

estimator = Estimator(backend)
job = estimator.run([(isa_circuit, isa_observable)])
pub_result = job.result()[0]
```

# Guna Qiskit Code Assistant dalam mod tempatan

Pelajari cara memasang, mengkonfigurasi, dan menggunakan mana-mana model Qiskit Code Assistant pada mesin tempatan anda.

<Admonition type="note" title="Nota">
    - Qiskit Code Assistant sedang dalam status keluaran pratonton dan boleh berubah.
    - Jika anda ada maklum balas atau ingin menghubungi pasukan pembangun, gunakan [saluran Qiskit Slack Workspace](https://qiskit.enterprise.slack.com/archives/C07LYA6PL83) atau repositori GitHub awam yang berkaitan.
</Admonition>

## Mula pantas (disyorkan) {#quick-start-recommended}

Cara paling mudah untuk bermula dengan Qiskit Code Assistant dalam mod tempatan ialah menggunakan skrip persediaan automatik untuk sambungan VS Code atau JupyterLab. Skrip ini akan memasang [Ollama](https://ollama.com/) secara automatik untuk menjalankan LLM, memuat turun model yang disyorkan, dan mengkonfigurasi sambungan untuk anda.

### Persediaan sambungan VS Code {#vs-code-extension-setup}

Jalankan arahan berikut dalam terminal anda:

```bash
bash <(curl -fsSL https://raw.githubusercontent.com/Qiskit/qiskit-code-assistant-vscode/main/setup_local.sh)
```

Skrip ini melakukan langkah-langkah berikut:
- Pasang Ollama (jika belum dipasang)
- Muat turun dan konfigurasikan model Qiskit Code Assistant yang disyorkan
- Sediakan sambungan VS Code untuk berfungsi dengan penggunaan tempatan anda

### Persediaan sambungan JupyterLab {#jupyterlab-extension-setup}

Jalankan arahan berikut dalam terminal anda:

```bash
bash <(curl -fsSL https://raw.githubusercontent.com/Qiskit/qiskit-code-assistant-jupyterlab/main/setup_local.sh)
```

Skrip ini akan:
- Pasang Ollama (jika belum dipasang)
- Muat turun dan konfigurasikan model Qiskit Code Assistant yang disyorkan
- Sediakan sambungan JupyterLab untuk berfungsi dengan penggunaan tempatan anda

### Menggunakan aplikasi Ollama {#using-the-ollama-application}

Aplikasi Ollama menyediakan penyelesaian mudah untuk menjalankan LLM secara tempatan. Ia mudah digunakan, dengan CLI yang menjadikan keseluruhan proses persediaan, pengurusan model, dan interaksi agak mudah. Ia sesuai untuk eksperimen pantas dan bagi pengguna yang mahukan lebih sedikit butiran teknikal untuk diuruskan.

#### Pasang Ollama {#install-ollama}

1. Muat turun [aplikasi Ollama](https://ollama.com/download)
1. Pasang fail yang dimuat turun
1. Lancarkan aplikasi Ollama yang dipasang

    <Admonition type="info">Aplikasi berjalan dengan berjaya apabila ikon Ollama muncul di bar menu desktop. Anda juga boleh mengesahkan perkhidmatan sedang berjalan dengan pergi ke `http://localhost:11434/`.</Admonition>

1. Cuba Ollama dalam terminal anda dan mula menjalankan model. Contohnya:

    ```
    ollama run hf.co/Qiskit/Qwen2.5-Coder-14B-Qiskit
    ```

#### Sediakan Ollama menggunakan integrasi Hugging Face Hub {#set-up-ollama-using-the-hugging-face-hub-integration}

[Integrasi Ollama/Hugging Face Hub](https://huggingface.co/docs/hub/ollama) menyediakan cara untuk berinteraksi dengan model yang dihoskan di Hugging Face Hub tanpa perlu membuat modelfile baru atau memuat turun fail GGUF atau safetensors secara manual. Fail `template` dan `params` lalai sudah disertakan untuk model di Hugging Face Hub.

1. Pastikan aplikasi Ollama sedang berjalan.

1. Pergi ke halaman model yang dikehendaki, dan salin URL tersebut. Contohnya, https://huggingface.co/Qiskit/Qwen2.5-Coder-14B-Qiskit-GGUF.

1. Dari terminal anda, jalankan arahan:

    ```
    ollama run hf.co/Qiskit/Qwen2.5-Coder-14B-Qiskit
    ```

Anda boleh menggunakan model `hf.co/Qiskit/Qwen2.5-Coder-14B-Qiskit` atau mana-mana model GGUF rasmi yang disyorkan lain `hf.co/Qiskit/mistral-small-3.2-24b-qiskit-GGUF` atau `hf.co/Qiskit/granite-3.3-8b-qiskit-GGUF`.

#### Sediakan Ollama dengan model GGUF Qiskit Code Assistant yang dimuat turun secara manual {#set-up-ollama-with-a-manually-downloaded-qiskit-code-assistant-gguf-model}

Jika anda telah memuat turun model GGUF secara manual seperti https://huggingface.co/Qiskit/Qwen2.5-Coder-14B-Qiskit-GGUF dan ingin bereksperimen dengan templat dan parameter yang berbeza, anda boleh mengikuti langkah-langkah ini untuk memuatnya ke dalam aplikasi Ollama tempatan anda.

1. Buat `Modelfile` dengan memasukkan kandungan berikut dan pastikan anda mengemas kini `<PATH-TO-GGUF-FILE>` kepada laluan sebenar model yang dimuat turun anda.

    ```
    FROM <PATH-TO-GGUF-FILE>
    TEMPLATE """{{ if .System }}
    System:
    {{ .System }}

    {{ end }}{{ if .Prompt }}Question:
    {{ .Prompt }}

    {{ end }}Answer:
    ```python{{ .Response }}
    """

    PARAMETER stop "Question:"
    PARAMETER stop "Answer:"
    PARAMETER stop "System:"
    PARAMETER stop "```"

    PARAMETER temperature 0
    PARAMETER top_k 1
    ```

1. Jalankan arahan berikut untuk mencipta contoh model tersuai berdasarkan `Modelfile`.

    ```
    ollama create Qwen2.5-Coder-14B-Qiskit -f ./path-to-model-file
    ```

    <Admonition type="note">Proses ini mungkin mengambil sedikit masa untuk Ollama membaca fail model, memulakan contoh model, dan mengkonfigurasinya mengikut spesifikasi yang diberikan.</Admonition>

#### Jalankan model Qiskit Code Assistant yang dimuat turun secara manual dalam Ollama {#run-the-qiskit-code-assistant-model-manually-downloaded-in-ollama}

Selepas model `Qwen2.5-Coder-14B-Qiskit` telah disediakan dalam Ollama, jalankan arahan berikut untuk melancarkan model dan berinteraksi dengannya dalam terminal (dalam mod chat).

```
ollama run Qwen2.5-Coder-14B-Qiskit
```

Beberapa arahan berguna:

- `ollama list` - Senaraikan model pada komputer anda
- `ollama rm Qwen2.5-Coder-14B-Qiskit` - Padam model
- `ollama show Qwen2.5-Coder-14B-Qiskit` - Tunjukkan maklumat model
- `ollama stop Qwen2.5-Coder-14B-Qiskit` - Hentikan model yang sedang berjalan
- `ollama ps` - Senaraikan model yang sedang dimuatkan

<summary>Gunakan model Qiskit Code Assistant secara tempatan melalui pakej `llama-cpp-python` secara manual</summary>

<span id="use-llama-cpp-python"></span>
Alternatif kepada aplikasi Ollama ialah pakej `llama-cpp-python`, iaitu binding Python untuk `llama.cpp`. Ia memberi anda lebih kawalan dan fleksibiliti untuk menjalankan model GGUF secara tempatan, dan sesuai untuk pengguna yang ingin mengintegrasikan model tempatan dalam aliran kerja dan aplikasi Python mereka.

1. Pasang [`llama-cpp-python`](https://pypi.org/project/llama-cpp-python/)
2. Berinteraksi dengan model dari dalam aplikasi anda menggunakan `llama_cpp`. Contohnya:

```python
from llama_cpp import Llama

model_path = <PATH-TO-GGUF-FILE>

model = Llama(
        model_path,
        seed=17,
        n_ctx=10000,
        n_gpu_layers=37, # to offload in gpu, but put 0 if all in cpu
    )

input = 'Generate a quantum circuit with 2 qubits'
raw_pred = model(input)["choices"][0]["text"]
```

Anda juga boleh menambah parameter penjanaan teks pada model untuk menyesuaikan inferens:

```python
generation_kwargs = {
        "max_tokens": 512,
        "echo": False, # Echo the prompt in the output
        "top_k": 1
    }

raw_pred = model(input, **generation_kwargs)["choices"][0]["text"]
```

</details>

### Guna pustaka `llama.cpp` {#use-the-llama-cpp-library}

Alternatif lain ialah menggunakan `llama.cpp`, pustaka sumber terbuka untuk menjalankan inferens LLM pada CPU dengan persediaan minimum.
Ia menyediakan kawalan peringkat rendah ke atas pelaksanaan model dan biasanya dijalankan dari baris arahan, menunjuk ke fail model GGUF tempatan.

Terdapat beberapa cara untuk memasang `llama.cpp` pada mesin anda:

- Pasang llama.cpp menggunakan [brew, nix, atau winget](https://github.com/ggml-org/llama.cpp/blob/master/docs/install.md)
- Jalankan dengan Docker: Lihat [dokumentasi Docker oleh pasukan `llama.cpp`](https://github.com/ggml-org/llama.cpp/blob/master/docs/docker.md)
- Muat turun binari yang telah dibina dari [halaman keluaran](https://github.com/ggml-org/llama.cpp/releases)
- Bina dari sumber dengan [mengklon repositori ini](https://github.com/ggml-org/llama.cpp/blob/master/docs/build.md)

Setelah dipasang, anda boleh menggunakan `llama.cpp` untuk berinteraksi dengan model GGUF dalam mod perbualan seperti berikut:
```bash
# Use a local model file
llama-cli -m my_model.gguf -cnv

# Or download and run a model directly from Hugging Face
llama-cli -hf Qiskit/Qwen2.5-Coder-14B-Qiskit-GGUF -cnv
```

Anda juga boleh melancarkan pelayan API yang serasi dengan OpenAI untuk model dengan cara berikut:
```bash
llama-server -hf Qiskit/Qwen2.5-Coder-14B-Qiskit-GGUF
```

#### Parameter lanjutan {#advanced-parameters}
Dengan program `llama-cli`, anda boleh mengawal penjanaan model menggunakan pilihan baris arahan. Contohnya, anda boleh memberikan arahan sistem "system" awal menggunakan bendera `-p/--prompt`. Dalam mod perbualan (`-cnv`), arahan awal ini bertindak sebagai mesej sistem. Jika tidak, anda boleh menambah mana-mana arahan yang dikehendaki ke teks arahan anda. Anda juga boleh melaraskan parameter pensampelan - contohnya: suhu (`--temp`), top-k (`--top-k`), top-p (`--top-p`), penalti pengulangan (`--repeat-penalty`), dan benih yang digunakan (`--seed`). Berikut adalah contoh pemanggilan menggunakan pilihan-pilihan ini:

```bash
llama-cli -hf Qiskit/Qwen2.5-Coder-14B-Qiskit-GGUF \
  -p "You are a friendly assistant." -cnv \
  --temp 0.7 \
  --top-k 50 \
  --top-p 0.95 \
  --repeat-penalty 1.1 \
  --seed 42
```

Untuk memastikan fungsi yang betul bagi model Qiskit kami, kami mengesyorkan penggunaan arahan sistem yang disediakan dalam repositori HF GGUF kami: arahan sistem untuk [mistral-small-3.2-24b-qiskit-GGUF](https://huggingface.co/Qiskit/mistral-small-3.2-24b-qiskit-GGUF/blob/main/system), [Qwen2.5-Coder-14B-Qiskit-GGUF](https://huggingface.co/Qiskit/Qwen2.5-Coder-14B-Qiskit-GGUF/blob/main/system), [granite-3.3-8b-qiskit-GGUF](https://huggingface.co/Qiskit/granite-3.3-8b-qiskit-GGUF/blob/main/system), dan [granite-3.2-8b-qiskit-GGUF.](https://huggingface.co/Qiskit/granite-3.2-8b-qiskit-GGUF/blob/main/system)

<summary>Sambungkan sambungan ke penggunaan tempatan secara manual</summary>

Gunakan sambungan VS Code dan sambungan JupyterLab untuk Qiskit Code Assistant bagi menghantar arahan kepada model Qiskit Code Assistant yang digunakan secara tempatan. Setelah anda mempunyai aplikasi Ollama yang [disediakan dengan model](#using-the-ollama-application), anda boleh mengkonfigurasi sambungan untuk menyambung ke perkhidmatan tempatan.

### Sambungkan dengan sambungan VS Code Qiskit Code Assistant {#connect-with-the-qiskit-code-assistant-vs-code-extension}

Dengan sambungan VS Code Qiskit Code Assistant, anda boleh berinteraksi dengan model dan melakukan penyiapan kod semasa menulis kod anda. Ini boleh berfungsi dengan baik untuk pengguna yang mencari bantuan menulis kod Qiskit untuk aplikasi Python mereka.

1. Pasang [sambungan VS Code Qiskit Code Assistant](/guides/qiskit-code-assistant-vscode).
1. Dalam VS Code, pergi ke **User Settings** dan tetapkan **Qiskit Code Assistant: Url** kepada URL penggunaan Ollama tempatan anda (contohnya, `http://localhost:11434`).
1. Muat semula VS Code dengan pergi ke **View > Command Palette...** dan pilih **Developer: Reload Window**.

Model Qiskit Code Assistant yang dikonfigurasi dalam Ollama akan muncul dalam bar status dan sedia untuk digunakan.

### Sambungkan dengan sambungan JupyterLab Qiskit Code Assistant {#connect-with-the-qiskit-code-assistant-jupyterlab-extension}

Dengan sambungan JupyterLab Qiskit Code Assistant, anda boleh berinteraksi dengan model dan melakukan penyiapan kod terus dalam Jupyter Notebook anda. Pengguna yang kebanyakannya bekerja dengan Jupyter Notebook boleh memanfaatkan sambungan ini untuk meningkatkan lagi pengalaman mereka menulis kod Qiskit.

1. Pasang [sambungan JupyterLab Qiskit Code Assistant](/guides/qiskit-code-assistant-jupyterlab).
1. Dalam JupyterLab, pergi ke **Settings Editor** dan tetapkan **Qiskit Code Assistant Service API** kepada URL penggunaan Ollama tempatan anda (contohnya, `http://localhost:11434`).

Model Qiskit Code Assistant yang dikonfigurasi dalam Ollama akan muncul dalam bar status dan sedia untuk digunakan.

</details>

# Sediakan untuk menggunakan IBM Quantum Platform dengan REST API

Anda boleh mengakses pemproses kuantum dengan REST API, membolehkan anda bekerja dengan QPU menggunakan mana-mana bahasa pengaturcaraan atau rangka kerja.

### 1. Dapatkan akses {#1-get-access}

1. Jika anda belum mempunyai akaun pengguna, dapatkan satu di [halaman log masuk IBM Quantum.](https://quantum.cloud.ibm.com/login)
1. Cipta kunci API (juga dipanggil token) di [papan pemuka.](https://quantum.cloud.ibm.com) Perlu diambil perhatian bahawa kunci API yang sama boleh digunakan untuk mana-mana rantau.
1. Jana token pembawa (bearer token) IBM Cloud Identity and Access Management (IAM). Ini ialah token jangka pendek yang digunakan untuk mengesahkan permintaan ke REST API. Untuk menjana satu, panggil [API Perkhidmatan Identiti IAM](https://cloud.ibm.com/apidocs/iam-identity-token-api#create-api-key) seperti yang ditunjukkan dalam permintaan contoh berikut:
<Tabs>

   <TabItem value="Curl" label="Curl">
   ```bash
   curl -X POST 'https://iam.cloud.ibm.com/identity/token' \
   -H 'Content-Type: application/x-www-form-urlencoded' \
   -d 'grant_type=urn:ibm:params:oauth:grant-type:apikey&apikey=MY_APIKEY'
   ```
   Respons yang Dijangka
   ```text
   {
      "access_token": "eyJhbGciOiJIUz......sgrKIi8hdFs",
      "refresh_token": "SPrXw5tBE3......KBQ+luWQVY=",
      "token_type": "Bearer",
      "expires_in": 3600,
      "expiration": 1473188353
   }
   ```
   </TabItem>

   <TabItem value="Python" label="Python">
   ```python
   # Use 'service' to invoke operations.
   import requests
   import json

   url = 'https://iam.cloud.ibm.com/identity/token'
   api_key = 'MY_APIKEY'
   headers = {
    'Content-Type': 'application/x-www-form-urlendcoded',
   }
   data = f'grant_type=urn:ibm:params:oauth:grant-type:apikey&apikey={api_key}'
   response = requests.post(url, headers=headers, data=data)

   # Bearer token to authorize requests to the REST API
   bearer_token = response.json()['access_token']
   ```
   </TabItem>
</Tabs>

### 2. Pilih kaedah pengesahan {#2-choose-an-authentication-method}

Pilih kaedah pengesahan yang sesuai, bergantung pada persekitaran kerja anda:

* [Cipta pemboleh ubah persekitaran untuk kunci API anda (persekitaran Python yang dipercayai)](#environment_variable)
* [Guna kunci API anda secara langsung (persekitaran yang tidak dipercayai)](/guides/cloud-setup-untrusted#rest-untrusted)

<span id="environment_variable"></span>
#### Cipta pemboleh ubah persekitaran (persekitaran yang dipercayai) {#create-an-environment-variable-trusted-environment}

1. Untuk menetapkan pemboleh ubah persekitaran IQP_API_TOKEN dalam sistem anda, anda boleh menambah baris berikut ke profil shell anda (contohnya, .bashrc atau .zshrc) atau dengan menetapkannya terus dalam terminal anda:

    ```shell
    export IQP_API_TOKEN=<your-API_KEY> # Use the 44-character API_KEY you created and saved from the IBM Quantum Platform Home dashboard
    ```
    Apabila anda memanggil pemboleh ubah persekitaran dalam kod anda, sertakan `import os`, seperti dalam contoh ini:

    ```python
    import os
    api_token = os.environ['IQP_API_TOKEN']
    ```
    Perlu diambil perhatian bahawa apabila mencipta pemboleh ubah persekitaran, kunci API anda masih disimpan secara tempatan dalam teks biasa, dan perlu dilindungi.

1. Sahkan permintaan ke Qiskit Runtime REST API dengan menyertakan CRN dan token pembawa dalam pengepala permintaan.
```bash
curl -X 'GET' \
   'https://quantum.cloud.ibm.com/api/v1/usage' \
   '-H accept: application/json' \
   '-H authorization: Bearer <BEARER_TOKEN>' \
   '-H Service-CRN: <INSTANCE_CRN>'
```

Aliran kerja kuantum sering mengambil masa yang lama untuk diselesaikan dan boleh berjalan merentas banyak Session. Memulakan semula kernel Python bermakna anda akan kehilangan sebarang keputusan yang tersimpan dalam ingatan. Untuk mengelakkan kehilangan data, anda boleh menyimpan keputusan ke fail dan mengambil semula keputusan kerja lampau dari IBM Quantum&reg; supaya Session seterusnya boleh meneruskan dari mana anda berhenti.
## Ambil semula keputusan kerja dari IBM Quantum {#retrieve-job-results-from-ibm-quantum}

IBM Quantum secara automatik menyimpan keputusan dari setiap kerja untuk anda ambil semula pada tarikh kemudian. Gunakan ciri ini untuk meneruskan program kuantum merentas but semula kernel dan menyemak keputusan lampau. Anda boleh mendapatkan ID kerja secara aturcara melalui kaedah `job_id`-nya, atau anda boleh melihat semua kerja yang telah dihantar dan ID mereka pada [halaman Beban Kerja](https://quantum.cloud.ibm.com/workloads).

Untuk mencari kerja secara aturcara, gunakan kaedah [`QiskitRuntimeService.jobs`](https://docs.quantum.ibm.com/api/qiskit-ibm-runtime/qiskit-runtime-service#jobs). Secara lalai, ini mengembalikan kerja yang paling baru dihantar, tetapi anda juga boleh menapis kerja mengikut nama Backend, tarikh penciptaan, dan banyak lagi. Sel berikut mencari sebarang kerja yang dihantar dalam tiga bulan terakhir. Argumen `created_after` mestilah objek [`datetime.datetime`](https://docs.python.org/3.8/library/datetime.html#datetime.datetime).

```python
import datetime
from qiskit_ibm_runtime import QiskitRuntimeService

three_months_ago = datetime.datetime.now() - datetime.timedelta(days=90)

service = QiskitRuntimeService()
jobs_in_last_three_months = service.jobs(created_after=three_months_ago)
jobs_in_last_three_months[:3]  # show first three jobs
```

Anda juga boleh memilih mengikut Backend, status kerja, Session, dan banyak lagi. Untuk maklumat lanjut, lihat [`QiskitRuntimeService.jobs`](https://docs.quantum.ibm.com/api/qiskit-ibm-runtime/qiskit-runtime-service#jobs) dalam dokumentasi API.

Setelah anda mempunyai ID kerja, gunakan kaedah [`QiskitRuntimeService.job`](https://docs.quantum.ibm.com/api/qiskit-ibm-runtime/qiskit-runtime-service#job) untuk mengambilnya semula.

## Simpan keputusan ke cakera {#save-results-to-disk}

Anda mungkin juga mahu menyimpan keputusan ke cakera. Untuk melakukan ini, gunakan pustaka JSON terbina dalam Python dengan pengekod dari Qiskit Runtime.

```python
import json
from qiskit_ibm_runtime import RuntimeEncoder

with open("result.json", "w") as file:
    json.dump(retrieved_job.result(), file, cls=RuntimeEncoder)
```

Anda kemudian boleh memuatkan tatasusunan ini dari cakera dalam kernel yang berasingan.

```python
from qiskit_ibm_runtime import RuntimeDecoder

with open("result.json", "r") as file:
    result = json.load(file, cls=RuntimeDecoder)

result
```

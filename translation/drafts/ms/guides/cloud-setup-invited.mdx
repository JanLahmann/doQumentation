# Sediakan akaun IBM Cloud anda - pengguna yang dijemput

Jika anda tidak mempunyai jemputan daripada pentadbir untuk menyertai akaun yang telah disediakan oleh pentadbir, hubungi pentadbir anda. Jika anda perlu menyediakan akaun sendiri, ikuti langkah dalam [Sediakan akaun IBM Cloud anda](/guides/cloud-setup).

Anda boleh menjalankan beban kerja pada unit pemprosesan kuantum (QPU) IBM&reg; dengan menyediakan akaun di IBM Cloud&reg;. Anda juga perlu [memasang dan menyediakan Qiskit dan Qiskit Runtime](/guides/install-qiskit#local).

Akaun pengguna anda dikaitkan dengan satu atau lebih [instans](./instances), yang dikenal pasti oleh _Cloud Resource Names_ (CRN) yang memberikan akses ke perkhidmatan IBM Quantum. Selain itu, token API unik (juga dipanggil kunci) diberikan kepada setiap akaun, membolehkan akses IBM Quantum dari Qiskit. Untuk arahan memilih instans tertentu, lihat [Sambung ke instans](./instances#connect-instance).

    <Admonition type="note">
      [Papan pemuka IBM Quantum Platform](https://quantum.cloud.ibm.com) dan [halaman Instans](https://quantum.cloud.ibm.com/instances) menyenaraikan instans yang boleh anda akses.
    </Admonition>

<span id="cloud"></span>
## Sediakan untuk menggunakan IBM Cloud {#set-up-to-use-ibm-cloud}

### Sebelum anda mula {#before-you-begin}

1. Terima jemputan e-mel.
1. Log masuk dengan akaun IBMid atau Google. Perlu diambil perhatian bahawa profil yang anda gunakan untuk log masuk ke IBM Cloud atau IBM Quantum Platform dalam langkah ini bukan akaun IBM Cloud anda. Selepas log masuk, sahkan maklumat anda dan terima jemputan untuk menyertai akaun.
1. Navigasi kembali ke [IBM Quantum Platform.](https://quantum.cloud.ibm.com)
1. Pastikan akaun dan rantau yang betul dipilih dalam penukar akaun di pengepala, seperti yang ditunjukkan dalam imej berikut.
    * Pilih akaun yang dijemput dalam e-mel.
    * Rantau mengawal di mana kerja anda dijalankan dan di mana data kerja disimpan. Anda boleh mengakses mana-mana rantau menggunakan kunci API yang sama, tetapi anda hanya boleh melihat dan mengakses instans yang dicipta dalam rantau yang anda log masuk.

    ![Pengepala IBM Quantum Platform ditunjukkan. Penukar akaun berada di sebelah kanan bar carian. Penukar rantau berada di sebelah kanan penukar akaun.](/docs/images/guides/cloud-setup/IQP-Header.svg 'Pengepala IBM Quantum Platform')

### 2. Pilihan: Simpan kelayakan akses anda {#2-optional-save-your-access-credentials}

Jika anda menggunakan persekitaran Python yang dipercayai, seperti komputer riba peribadi, adalah disyorkan agar anda menggunakan kaedah `save_account()` untuk [menyimpan kelayakan anda secara tempatan.](/guides/save-credentials)

<span id="cloud-save"></span>
### 3. Sambungkan Qiskit dengan instans perkhidmatan anda {#3-connect-qiskit-with-your-service-instance}

- [Mulakan perkhidmatan](/guides/initialize-account) dalam **persekitaran Python yang dipercayai**
- [Mulakan perkhidmatan dalam **persekitaran yang tidak dipercayai**](/guides/cloud-setup-untrusted) (seperti komputer awam)

<Admonition type="note" title="Nota">
   * Ikuti [arahan ini](/guides/cloud-setup-rest-api) jika anda ingin menyambung menggunakan REST API dan bukannya menggunakan Qiskit.
   * Jika perlu, gunakan [maklumat ini](/guides/quickstart-steps-org#firewall) untuk mengkonfigurasi tembok api anda bagi membolehkan akses ke titik akhir API IBM Quantum.
</Admonition>

## Clifford operations และการเข้ารหัส {#clifford-operations-and-encodings}

ต่อไป เราจะพูดถึงโดยย่อว่า qubits สามารถเข้ารหัสโดยใช้ stabilizer codes ได้อย่างไร แต่ก่อนอื่นเราต้องแนะนำ *Clifford operations* ก่อน

### Clifford operations {#clifford-operations}

Clifford operations คือ unitary operations บน qubits จำนวนเท่าใดก็ได้ ที่สามารถ implement ได้โดย quantum circuits ที่มี gates ชุดจำกัด:
- Hadamard gates
- $S$ gates
- CNOT gates

สังเกตว่า $T$ gates ไม่รวมอยู่ด้วย รวมถึง Toffoli gates และ Fredkin gates
ไม่เพียงแต่ gates เหล่านั้นจะไม่อยู่ในรายการ แต่จริงๆ แล้วมันเป็นไปไม่ได้ที่จะ implement gates เหล่านั้นโดยใช้ gates ที่ระบุไว้ที่นี่ มันไม่ใช่ Clifford operations
ในทางกลับกัน Pauli operations เป็น Clifford operations เพราะสามารถ implement ได้ด้วยลำดับของ Hadamard และ $S$ gates

นั่นเป็นวิธีง่ายๆ ในการนิยาม Clifford operations แต่ไม่ได้อธิบายว่าทำไมถึงนิยามแบบนี้หรืออะไรที่พิเศษเกี่ยวกับการรวบรวม gates ชุดนี้โดยเฉพาะ
เหตุผล *ที่แท้จริง* ที่ Clifford operations ถูกนิยามแบบนี้คือ นอกเหนือจาก global phase factors แล้ว Clifford operations คือ unitary operations ที่แปลง Pauli operations เป็น Pauli operations เสมอโดย conjugation
พูดให้ชัดเจนขึ้น $n$-qubit unitary operation $U$ เทียบเท่ากับ Clifford operation นอกเหนือจาก phase factor ก็ต่อเมื่อ สำหรับ *ทุก* $n$-qubit Pauli operation $P$ เรามี

$$
U P U^{\dagger} = \pm Q
$$

สำหรับ $n$-qubit Pauli operation $Q$ บางตัว

(สังเกตว่าเป็นไปไม่ได้ที่จะมี $U P U^{\dagger} = \alpha Q$ สำหรับ $\alpha\notin\{+1,-1\}$ เมื่อ $U$ เป็น unitary และ $P$ และ $Q$ เป็น Pauli operations
สิ่งนี้ตามมาจากข้อเท็จจริงที่ว่า matrix ทางด้านซ้ายของสมการนั้นเป็นทั้ง unitary และ Hermitian และ $+1$ กับ $-1$ เป็นตัวเลือกเดียวสำหรับ $\alpha$ ที่ทำให้ด้านขวาเป็น unitary และ Hermitian ได้ด้วย)

การตรวจสอบคุณสมบัติ conjugation ที่กล่าวมานั้นตรงไปตรงมาเมื่อ $U$ เป็น Hadamard, $S,$ หรือ CNOT gate
โดยเฉพาะอย่างยิ่ง สำหรับ Hadamard gates ทำได้ง่าย

$$
H X H^{\dagger} = Z, \qquad
H Y H^{\dagger} = -Y, \qquad
H Z H^{\dagger} = X,
$$

และ $S$ gates,

$$
S X S^{\dagger} = Y, \qquad
S Y S^{\dagger} = -X, \qquad
S Z S^{\dagger} = Z.
$$

สำหรับ CNOT gates มี 15 non-identity Pauli operations บน two qubits ที่ต้องตรวจสอบ
แน่นอนว่าสามารถตรวจสอบทีละตัว — แต่ความสัมพันธ์ระหว่าง CNOT gates กับ $X$ และ $Z$ gates ที่ระบุไว้ (ในรูป circuit) ในบทเรียนที่แล้ว พร้อมกับกฎการคูณของ Pauli matrices เสนอทางลัดไปสู่ข้อสรุปเดียวกัน

เมื่อเรารู้ว่าคุณสมบัติ conjugation นี้เป็นจริงสำหรับ Hadamard, $S,$ และ CNOT gates เราสรุปได้ทันทีว่ามันเป็นจริงสำหรับ *circuits* ที่ประกอบด้วย gates เหล่านี้ — นั่นก็คือ Clifford operations ทั้งหมด

การพิสูจน์ว่าความสัมพันธ์ทำงานในทิศทางตรงข้ามนั้นยากกว่า ซึ่งก็คือถ้า unitary operation $U$ ที่กำหนดตอบสนองคุณสมบัติ conjugation สำหรับ Pauli operations แล้วต้องสามารถ implement มันได้ (นอกเหนือจาก global phase) โดยใช้เพียง Hadamard, $S,$ และ CNOT gates
สิ่งนี้จะไม่อธิบายในบทเรียนนี้ แต่เป็นความจริง

Clifford operations ไม่ใช่ universal สำหรับ quantum computation; ต่างจาก universal sets ของ quantum gates การประมาณ unitary operations โดยพลการในระดับความแม่นยำที่ต้องการด้วย Clifford operations นั้นเป็นไปไม่ได้
จริงๆ แล้ว สำหรับค่า $n$ ที่กำหนด มี $n$-qubit Clifford operations *จำนวนจำกัด* เท่านั้น (นอกเหนือจาก phase factors)
การดำเนินการ Clifford operations บน standard basis states ตามด้วยการวัด standard basis ก็ไม่สามารถทำการคำนวณที่อยู่นอกเหนือขอบเขตของ classical algorithms ได้ — เพราะเราสามารถ *จำลอง* การคำนวณในรูปแบบนี้ได้อย่างมีประสิทธิภาพด้วย classical computation
ข้อเท็จจริงนี้รู้จักกันในชื่อ *Gottesman-Knill theorem*

### Encoders สำหรับ stabilizer codes {#encoders-for-stabilizer-codes}

Stabilizer code กำหนด code space ที่มีมิติหนึ่ง และเรามีอิสระที่จะใช้ code space นั้นตามที่ต้องการ —
ไม่มีอะไรบังคับให้เราเข้ารหัส qubits ลงใน code space นั้นในแบบใดแบบหนึ่งโดยเฉพาะ
อย่างไรก็ตาม เป็นไปได้เสมอที่จะใช้ *Clifford operation* เป็น encoder ถ้าเราเลือกทำเช่นนั้น
พูดให้ชัดเจนขึ้น สำหรับ stabilizer code ใดๆ ที่อนุญาตให้เข้ารหัส $m$ qubits ลงใน $n$ qubits มี $n$-qubit Clifford operation $U$ ที่สำหรับ $m$-qubit quantum state vector $\vert\phi\rangle$ ใดๆ เรามีว่า

$$
\vert\psi\rangle = U \bigl(\vert 0^{n-m} \rangle \otimes \vert \phi\rangle\bigr)
$$

คือ quantum state vector ใน code space ของ code เราที่เราอาจตีความเป็นการเข้ารหัสของ $\vert\phi\rangle$

สิ่งนี้ดีเพราะ Clifford operations ค่อนข้างเรียบง่าย เมื่อเทียบกับ unitary operations โดยพลการ และมีวิธีการ optimize การ implement โดยใช้เทคนิคที่คล้ายกับที่พบในการพิสูจน์ Gottesman-Knill theorem
ดังนั้น circuits สำหรับการเข้ารหัส states โดยใช้ stabilizer codes จึงไม่จำเป็นต้องใหญ่มากนัก
โดยเฉพาะอย่างยิ่ง เป็นไปได้เสมอที่จะดำเนินการเข้ารหัสสำหรับ $n$-qubit stabilizer code โดยใช้ Clifford operation ที่ต้องการ $O(n^2/\log(n))$ gates
เนื่องจาก *ทุก* Clifford operation บน $n$ qubits สามารถ implement ได้ด้วย circuit ขนาดนี้

ตัวอย่างเช่น นี่คือ encoder สำหรับ 7-qubit Steane code
มันเป็น Clifford operation จริงๆ และปรากฏว่าตัวนี้ไม่ต้องการ $S$ gates ด้วยซ้ำ

## การตรวจจับข้อผิดพลาด {#detecting-errors}

สำหรับ $n$-qubit stabilizer code ที่อธิบายด้วย stabilizer generators $P_1,\ldots, P_r$ การตรวจจับข้อผิดพลาดทำงานในแบบต่อไปนี้

ในการตรวจจับข้อผิดพลาด stabilizer generators ทั้งหมดจะถูกวัดในฐานะ observables
มี $r$ stabilizer generators ดังนั้นจึงมี $r$ ผลการวัด แต่ละตัวเป็น $+1$ หรือ $-1$ (หรือค่า binary ถ้าเลือกใช้ $0$ แทน $+1$ และ $1$ แทน $-1$ ตามลำดับ)
เราตีความผลลัพธ์ $r$ ตัวรวมกัน เป็น vector หรือ string เรียกว่า *syndrome*
Syndrome $(+1,\ldots,+1)$ บ่งชี้ว่าไม่มีข้อผิดพลาดถูกตรวจพบ ในขณะที่ $-1$ อย่างน้อยหนึ่งตัวใน syndrome บ่งชี้ว่าตรวจพบข้อผิดพลาด

สมมติโดยเฉพาะว่า $E$ เป็น $n$-qubit Pauli operation ที่แทนข้อผิดพลาดสมมุติ
(เราพิจารณาเฉพาะ Pauli operations เป็นข้อผิดพลาด เนื่องจากการแยกส่วนของข้อผิดพลาดทำงานในแบบเดียวกันสำหรับ stabilizer codes โดยพลการเช่นเดียวกับ 9-qubit Shor code)
มีสามกรณีที่กำหนดว่า $E$ ถูกตรวจพบว่าเป็นข้อผิดพลาดหรือไม่

### กรณีการตรวจจับข้อผิดพลาด {#error-detection-cases}

1. การดำเนินการ $E$ เป็น proportional กับ element ใน stabilizer

   $$
   E = \pm Q \; \text{for some}\; Q \in \langle P_1,\ldots,P_r\rangle
   $$

   ในกรณีนี้ $E$ ต้องสับเปลี่ยนได้กับ stabilizer generator ทุกตัว ดังนั้นเราได้ syndrome $(+1,\ldots,+1)$ หมายความว่า $E$ ไม่ถูกตรวจพบว่าเป็นข้อผิดพลาด

2. การดำเนินการ $E$ ไม่ได้ proportional กับ element ใน stabilizer แต่กลับสับเปลี่ยนได้กับ stabilizer generator ทุกตัว

   $$
   E\neq \pm Q\; \text{for} \; Q \in \langle P_1,\ldots,P_r\rangle, \;\text{but}\;
   E P_k = P_k E \;\text{for every}\; k\in\{1,\ldots,r\}
   $$

   นี่คือข้อผิดพลาดที่เปลี่ยนเวกเตอร์ใน code space ในแบบที่ไม่ใช่ trivial อย่างไรก็ตาม เนื่องจาก $E$ สับเปลี่ยนได้กับ stabilizer generator ทุกตัว syndrome จึงเป็น $(+1,\ldots,+1)$ ดังนั้น $E$ จึงไม่ถูก code ตรวจพบ

3. การดำเนินการ $E$ anti-commutes กับ stabilizer generator อย่างน้อยหนึ่งตัว

   $$
   P_k E = -E P_k \; \text{for at least one} \; k\in\{1,\ldots,r\}
   $$

   Syndrome ต่างจาก $(+1,\ldots,+1)$ ดังนั้นข้อผิดพลาด $E$ จึงถูก code ตรวจพบ

ในกรณีแรก ข้อผิดพลาด $E$ ไม่น่าเป็นห่วง เพราะการดำเนินการนี้ไม่ทำอะไรกับเวกเตอร์ใน code space นอกจากอาจใส่ global phase ที่ไม่เกี่ยวข้อง: $E \ket{\psi} = \pm\ket{\psi}$ สำหรับทุก encoded state $\ket{\psi}$ โดยแก่นแท้แล้ว นี่ไม่ใช่ข้อผิดพลาดจริงๆ — การกระทำที่ไม่ trivial ใดๆ ของ $E$ เกิดขึ้นนอก code space — จึงเป็นเรื่องดีที่ $E$ ไม่ถูกตรวจพบว่าเป็นข้อผิดพลาด เพราะไม่ต้องทำอะไร

กรณีที่สอง พูดในแง่ intuitive คือกรณีที่ไม่ดี
มันคือ *anti-commutation* ของข้อผิดพลาดกับ stabilizer generator ที่ทำให้เกิด $-1$ ใน syndrome บ่งสัญญาณข้อผิดพลาด แต่สิ่งนั้นไม่เกิดขึ้นในกรณีนี้
ดังนั้น เรามีข้อผิดพลาด $E$ ที่เปลี่ยนเวกเตอร์ใน code space ในแบบที่ไม่ใช่ trivial แต่ code ไม่ตรวจพบ
ตัวอย่างเช่น สำหรับ 3-bit repetition code การดำเนินการ $E = X\otimes X\otimes X$ อยู่ในหมวดหมู่นี้

ข้อเท็จจริงที่ว่าข้อผิดพลาด $E$ ดังกล่าวต้องเปลี่ยนเวกเตอร์บางส่วนใน code space ในแบบที่ไม่ trivial สามารถโต้แย้งได้ดังต่อไปนี้
จากสมมติฐานที่ว่า $E$ สับเปลี่ยนได้กับ $P_1,\ldots,P_r$ แต่ไม่ proportional กับ stabilizer element เราสรุปได้ว่าเราจะได้ stabilizer code ใหม่ที่ถูกต้องโดยรวม $E$ เป็น stabilizer generator ร่วมกับ $P_1,\ldots,P_r$
อย่างไรก็ตาม code space สำหรับ code ใหม่นี้มีมิติเพียงครึ่งหนึ่งของ code space เดิม ซึ่งทำให้เราสรุปได้ว่า action ของ $E$ บน code space เดิมไม่สามารถ proportional กับ identity operation ได้

สำหรับกรณีที่สามในสามกรณี ซึ่งคือข้อผิดพลาด $E$ anti-commutes กับ stabilizer generator อย่างน้อยหนึ่งตัว
syndrome มี $-1$ อย่างน้อยหนึ่งตัว ซึ่งบ่งชี้ว่ามีบางอย่างผิดปกติ
อย่างที่เราได้พูดถึงแล้ว syndrome จะไม่ระบุ $E$ อย่างเจาะจงโดยทั่วไป ดังนั้นจึงยังจำเป็นต้องเลือก correction operation สำหรับแต่ละ syndrome ซึ่งอาจหรืออาจไม่แก้ไขข้อผิดพลาด $E$
เราจะพูดถึงขั้นตอนนี้ในไม่ช้า ในส่วนสุดท้ายของบทเรียน

### ระยะของ stabilizer code {#distance-of-a-stabilizer-code}

ในฐานะคำศัพท์ เมื่อเราอ้างถึง *ระยะ* (distance) ของ stabilizer code เราหมายถึง *น้ำหนักต่ำสุด* (minimum weight) ของ Pauli operation $E$ ที่อยู่ในหมวดหมู่ที่สองข้างต้น — หมายความว่ามันเปลี่ยน code space ในแบบที่ไม่ใช่ trivial แต่ code ไม่ตรวจพบ
เมื่อกล่าวว่า stabilizer code เป็น $[[n,m,d]]$ stabilizer code โดยใช้วงเล็บเหลี่ยมคู่ หมายความดังต่อไปนี้:

1. การเข้ารหัสมีความยาว $n$ qubits
2. code อนุญาตให้เข้ารหัส $m$ qubits และ
3. ระยะของ code คือ $d$

ตัวอย่างเช่น พิจารณา 7-qubit Steane code
นี่คือ stabilizer generators สำหรับ code นี้:

$$
\begin{array}{ccccccc}
Z & Z & Z & Z & \mathbb{I} & \mathbb{I} & \mathbb{I} \\[1mm]
Z & Z & \mathbb{I} & \mathbb{I} & Z & Z & \mathbb{I} \\[1mm]
Z & \mathbb{I} & Z & \mathbb{I} & Z & \mathbb{I} & Z \\[1mm]
X & X & X & X & \mathbb{I} & \mathbb{I} & \mathbb{I} \\[1mm]
X & X & \mathbb{I} & \mathbb{I} & X & X & \mathbb{I} \\[1mm]
X & \mathbb{I} & X & \mathbb{I} & X & \mathbb{I} & X
\end{array}
$$

code นี้มีระยะ 3 และเราสามารถโต้แย้งได้ดังต่อไปนี้

ก่อนอื่น พิจารณา Pauli operation $E$ ใดๆ ที่มีน้ำหนักไม่เกิน 2 และสมมติว่า operation นี้สับเปลี่ยนได้กับ stabilizer generators ทั้งหกตัว
เราจะสรุปว่า $E$ ต้องเป็น identity operation ซึ่ง (อย่างเสมอ) เป็น element ของ stabilizer
สิ่งนี้จะแสดงว่าระยะของ code มากกว่า 2 อย่างเคร่งครัด
สมมติโดยเฉพาะว่า $E$ มีรูปแบบ

$$
E = P \otimes Q \otimes \mathbb{I} \otimes \mathbb{I} \otimes \mathbb{I} \otimes \mathbb{I} \otimes \mathbb{I}
$$

โดยที่ $P$ และ $Q$ เป็น Pauli matrices ที่อาจไม่ใช่ identity
นี่เป็นเพียงหนึ่งกรณี และจำเป็นต้องทำซ้ำการโต้แย้งที่ตามมาสำหรับตำแหน่งที่เป็นไปได้ทั้งหมดอื่นๆ สำหรับ Pauli matrices ที่ไม่ใช่ identity ใน tensor factors ของ $E$ แต่การโต้แย้งนั้นเหมือนกันโดยพื้นฐานสำหรับตำแหน่งที่เป็นไปได้ทั้งหมด

การดำเนินการ $E$ สับเปลี่ยนได้กับ stabilizer generators ทั้งหกตัว ดังนั้นมันสับเปลี่ยนได้กับสองตัวนี้โดยเฉพาะ:

$$
\begin{gathered}
Z \otimes \mathbb{I} \otimes Z \otimes \mathbb{I} \otimes Z \otimes \mathbb{I} \otimes Z\\[1mm]
X \otimes \mathbb{I} \otimes X \otimes \mathbb{I} \otimes X \otimes \mathbb{I} \otimes X
\end{gathered}
$$

Tensor factor $Q$ ใน error $E$ ของเราตรงกับ identity matrix ในทั้งสอง stabilizer generators นี้ (นั่นเป็นเหตุผลที่เลือกมัน)
เนื่องจากเรามี identity matrices ใน 5 ตำแหน่งขวาสุดของ $E$ เราสรุปได้ว่า $P$ ต้องสับเปลี่ยนได้กับ $X$ และ $Z$ มิฉะนั้น $E$ จะ anti-commute กับ generator ตัวใดตัวหนึ่งในสอง
อย่างไรก็ตาม Pauli matrix เดียวที่สับเปลี่ยนได้กับทั้ง $X$ และ $Z$ คือ identity matrix ดังนั้น $P = \mathbb{I}$

ตอนนี้ที่เรารู้สิ่งนี้แล้ว เราสามารถเลือก stabilizer generators สองตัวเพิ่มเติมที่มี $X$ และ $Z$ ในตำแหน่งที่สองจากซ้าย และเราสรุปได้ในลักษณะเดียวกัน: $Q = \mathbb{I}$
ดังนั้น $E$ จึงเป็น identity operation

ดังนั้น ไม่มีทางที่ข้อผิดพลาดที่มีน้ำหนักไม่เกิน 2 จะไม่ถูก code ตรวจพบ ยกเว้น identity operation (ซึ่งอยู่ใน stabilizer จึงไม่ใช่ข้อผิดพลาดจริงๆ)
ในทางกลับกัน มี weight 3 Pauli operations ที่สับเปลี่ยนได้กับ stabilizer generators ทั้งหกตัว แต่ไม่ proportional กับ stabilizer elements เช่น $\mathbb{I}\otimes\mathbb{I}\otimes\mathbb{I}\otimes\mathbb{I}\otimes X\otimes X\otimes X$
และ $\mathbb{I}\otimes\mathbb{I}\otimes\mathbb{I}\otimes\mathbb{I}\otimes Z\otimes Z\otimes Z$
สิ่งนี้แสดงว่า code นี้มีระยะ 3 ตามที่อ้างไว้

## การแก้ไขข้อผิดพลาด {#correcting-errors}

หัวข้อสุดท้ายของบทเรียนนี้คือการ *แก้ไข* ข้อผิดพลาดสำหรับ stabilizer codes
อย่างเคยสมมติว่าเรามี stabilizer code ที่กำหนดด้วย stabilizer generators $P_1, \ldots, P_r$ บน n qubits

$n$-qubit Pauli operations ในฐานะข้อผิดพลาดที่อาจส่งผลต่อ states ที่เข้ารหัสโดยใช้ code นี้ ถูกแบ่งเป็น collections ขนาดเท่ากันตาม syndrome ที่มันทำให้เกิดขึ้น
มี $2^r$ syndromes ที่แตกต่างกันและ $4^n$ Pauli operations หมายความว่ามี $4^n/2^r$ Pauli operations ที่ทำให้เกิดแต่ละ syndrome
ข้อผิดพลาดใดๆ ตัวหนึ่งเหล่านี้อาจรับผิดชอบต่อ syndrome ที่สอดคล้องกัน

อย่างไรก็ตาม ในบรรดา $4^n/2^r$ Pauli operations ที่ทำให้เกิดแต่ละ syndrome มีบางตัวที่ควรพิจารณาว่าเทียบเท่ากัน
โดยเฉพาะอย่างยิ่ง ถ้า product ของ Pauli operations สองตัว proportional กับ stabilizer element แล้วทั้งสองตัวนั้นเทียบเท่ากันโดยพื้นฐานในฐานะข้อผิดพลาด

อีกวิธีในการพูดสิ่งนี้คือถ้าเราใช้ correction operation $C$ เพื่อพยายามแก้ไขข้อผิดพลาด $E$ แล้วการแก้ไขนี้สำเร็จตราบใดที่ composition $CE$ proportional กับ stabilizer element
เนื่องจากมี $2^r$ elements ใน stabilizer จึงตามมาว่า correction operation $C$ แต่ละตัวแก้ไข Pauli errors จำนวน $2^r$ ตัวที่แตกต่างกัน
ซึ่งเหลือ $4^{n-r}$ classes ที่ไม่เทียบเท่ากันของ Pauli operations ที่พิจารณาเป็นข้อผิดพลาด ซึ่งสอดคล้องกับแต่ละ syndrome ที่เป็นไปได้

หมายความว่า ยกเว้นกรณีที่ $n=r$ (ซึ่งในกรณีนั้นเรามี code space หนึ่งมิติ trivial) เราไม่สามารถแก้ไขทุกข้อผิดพลาดที่ตรวจพบโดย stabilizer code ได้
สิ่งที่เราต้องทำแทนคือเลือก correction operation เพียง *หนึ่ง* ตัวสำหรับแต่ละ syndrome โดยหวังว่าจะแก้ไขเพียง class เดียวของข้อผิดพลาดที่เทียบเท่ากันซึ่งทำให้เกิด syndrome นั้น

กลยุทธ์หนึ่งที่เป็นธรรมชาติในการเลือก correction operation สำหรับแต่ละ syndrome คือการเลือก Pauli operation ที่มี *น้ำหนักต่ำสุด* ที่ในฐานะข้อผิดพลาดทำให้เกิด syndrome นั้น
อาจมีหลาย operations ที่มีน้ำหนักต่ำสุดเท่ากันสำหรับ syndrome ที่กำหนด ในกรณีนั้นสามารถเลือกตัวใดตัวหนึ่งได้
แนวคิดคือ Pauli operations ที่มีน้ำหนักต่ำกว่าแทนคำอธิบายที่เป็นไปได้มากกว่าสำหรับ syndrome ที่วัดได้
สิ่งนี้อาจไม่เป็นจริงสำหรับ noise models บางอย่าง และกลยุทธ์ทางเลือกหนึ่งคือการคำนวณข้อผิดพลาดที่ *น่าจะเป็น* มากที่สุดที่ทำให้เกิด syndrome ที่กำหนด ตาม noise model ที่เลือก
อย่างไรก็ตาม สำหรับบทเรียนนี้ เราจะทำให้เรียบง่ายและพิจารณาเฉพาะการแก้ไขน้ำหนักต่ำสุดเท่านั้น

สำหรับ stabilizer code ที่มีระยะ $d$ กลยุทธ์การเลือก correction operation ให้เป็น Pauli operation น้ำหนักต่ำสุดที่สอดคล้องกับ syndrome ที่วัดได้นี้ รับประกันเสมอว่าสามารถแก้ไขข้อผิดพลาดที่มีน้ำหนักน้อยกว่าครึ่งหนึ่งของ $d$ อย่างเคร่งครัด หรือพูดอีกอย่างคือน้ำหนักไม่เกิน $(d-1)/2$
สิ่งนี้แสดงให้เห็น เช่น ว่า 7-qubit Steane code สามารถแก้ไข Pauli error ที่มีน้ำหนักหนึ่งได้ และจากการแยกส่วนของข้อผิดพลาด หมายความว่า Steane code สามารถแก้ไขข้อผิดพลาดโดยพลการบน qubit หนึ่งตัวได้

เพื่อดูว่าสิ่งนี้ทำงานอย่างไร พิจารณา diagram ด้านล่าง
วงกลมทางซ้ายแทน Pauli operations ทั้งหมดที่ทำให้เกิด syndrome $(+1,\ldots,+1)$ ซึ่งเป็น syndrome ที่บ่งชี้ว่าไม่มีข้อผิดพลาดและทุกอย่างปกติ
ในบรรดา operations เหล่านี้ เรามี elements ของ stabilizer (หรือ operations ที่ *proportional* กับ elements ของ stabilizer ให้แม่นยำยิ่งขึ้น) และยังมีข้อผิดพลาดที่ไม่ trivial ที่เปลี่ยน code space ในบางแง่แต่ code ไม่ตรวจพบ
ตามนิยามของระยะ Pauli operation ทุกตัวในหมวดหมู่นี้ต้องมีน้ำหนักอย่างน้อย $d$ เพราะ $d$ ถูกนิยามเป็นน้ำหนักต่ำสุดของ operations เหล่านี้

วงกลมทางขวาแทน Pauli operations ที่ทำให้เกิด syndrome ต่าง $s\neq(+1,\ldots,+1)$ รวมถึงข้อผิดพลาด $E$ ที่มีน้ำหนักน้อยกว่า $d/2$ อย่างเคร่งครัดที่เราจะพิจารณา

![Lowest-weight error correction diagram](/learning/images/courses/foundations-of-quantum-error-correction/stabilizer-formalism/lowest-weight-correction.svg)

Correction operation $C$ ที่เลือกสำหรับ syndrome $s$ คือ Pauli operation น้ำหนักต่ำสุดใน collection ที่แทนด้วยวงกลมทางขวาใน diagram (หรือตัวใดตัวหนึ่งในกรณีที่เสมอกัน)
ดังนั้น อาจเป็นไปได้ว่า $C = E$ แต่ไม่จำเป็นเสมอไป
อย่างไรก็ตาม สิ่งที่เราพูดได้อย่างแน่นอนคือ $C$ ไม่สามารถมีน้ำหนักมากกว่าน้ำหนักของ $E$ เพราะ $C$ มีน้ำหนักต่ำสุดใน operations ใน collection นี้ — ดังนั้น $C$ จึงมีน้ำหนักน้อยกว่า $d/2$ อย่างเคร่งครัด

ตอนนี้พิจารณาสิ่งที่เกิดขึ้นเมื่อ correction operation $C$ ถูกนำไปใช้กับ state ที่เราได้รับหลังจากที่ข้อผิดพลาด $E$ เกิดขึ้น
สมมติว่า encoding เดิมเป็น $\vert\psi\rangle$ เราเหลือ $CE\vert\psi\rangle$
เป้าหมายของเราคือแสดงว่า $CE$ proportional กับ element ใน stabilizer หมายความว่าการแก้ไขสำเร็จและ (นอกเหนือจาก global phase) เราเหลือ encoded state เดิม $\vert\psi\rangle$

ประการแรก เนื่องจาก $E$ และ $C$ ทำให้เกิด syndrome เดียวกัน composition $CE$ ต้องสับเปลี่ยนได้กับ stabilizer generator ทุกตัว
โดยเฉพาะอย่างยิ่ง ถ้า $P_k$ เป็น stabilizer generator ตัวใดตัวหนึ่ง เราต้องมี

$$
P_k E = \alpha E P_k
\quad\text{and}\quad
P_k C = \alpha C P_k
$$

สำหรับค่า $\alpha\in\{+1,-1\}$ เดียวกัน เพราะนี่คือรายการที่ $k$ ใน syndrome $s$ ที่ทั้ง $C$ และ $E$ สร้างขึ้น
ดังนั้น เรามี

$$
P_k (CE) = \alpha C P_k E = \alpha^2 (CE) P_k = (CE) P_k,
$$

ดังนั้น $P_k$ สับเปลี่ยนได้กับ $CE$
เราจึงแสดงให้เห็นว่า $CE$ อยู่ในวงกลมทางซ้ายใน diagram เพราะมันสร้าง syndrome $(+1,\ldots,+1)$

ประการที่สอง composition $CE$ ต้องมีน้ำหนักไม่เกินผลรวมของน้ำหนักของ $C$ และ $E$ — ซึ่งตามมาจากการคิดสักครู่เกี่ยวกับ products ของ Pauli operations — ดังนั้นน้ำหนักของ $CE$ น้อยกว่า $d$ อย่างเคร่งครัด
สิ่งนี้หมายความว่า $CE$ proportional กับ element ใน stabilizer ของ code เรา ซึ่งเป็นสิ่งที่เราต้องการแสดง
ด้วยการเลือก correction operations ให้เป็นตัวแทนน้ำหนักต่ำสุดของเซต errors ที่สร้าง syndrome แต่ละตัว เราจึงรับประกันได้ว่าจะแก้ไข Pauli errors ที่มีน้ำหนักน้อยกว่าครึ่งหนึ่งของระยะ code

อย่างไรก็ตาม มีปัญหาหนึ่ง
สำหรับ stabilizer codes โดยทั่วไป การคำนวณ Pauli operation น้ำหนักต่ำสุดที่ทำให้เกิด syndrome ที่กำหนดเป็นปัญหาที่ยากในเชิงคำนวณ
(จริงๆ แล้ว สิ่งนี้เป็นจริงแม้แต่สำหรับ classical codes ซึ่งในบริบทนี้เราสามารถคิดว่าเป็น stabilizer codes ที่มีเฉพาะ $\mathbb{I}$ และ $Z$ matrices ปรากฏเป็น tensor factors ใน stabilizer generators)
ดังนั้น ต่างจากขั้นตอนการเข้ารหัส Clifford operations จะไม่มาช่วยคราวนี้

ทางออกคือการเลือก specific codes ที่สามารถคำนวณการแก้ไขที่ดีได้อย่างมีประสิทธิภาพ ซึ่งไม่มีสูตรง่ายๆ
พูดง่ายๆ คือการออกแบบ stabilizer codes ที่สามารถคำนวณ correction operations ที่ดีได้อย่างมีประสิทธิภาพเป็นส่วนหนึ่งของศิลปะในการออกแบบ quantum code
เราจะเห็นศิลปะนี้ในบทเรียนถัดไป

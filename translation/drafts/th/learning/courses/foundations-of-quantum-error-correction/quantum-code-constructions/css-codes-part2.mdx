## การตรวจจับและแก้ไขข้อผิดพลาด {#error-detection-and-correction}

เกี่ยวกับการตรวจจับและแก้ไขข้อผิดพลาด CSS codes โดยทั่วไปมีลักษณะคล้ายกับ 9-qubit Shor code กล่าวคือ $X$ และ $Z$ errors สามารถตรวจจับและแก้ไขได้อย่างอิสระโดยสมบูรณ์ โดยที่ $Z$ stabilizer generators อธิบายโค้ดที่ป้องกัน bit flips และ $X$ stabilizer generators อธิบายโค้ดที่ป้องกัน phase flips อย่างอิสระ
สิ่งนี้ทำงานได้เพราะ $Z$ stabilizer generators commute กับ $Z$ errors อย่างจำเป็น รวมถึง $Z$ operations ที่นำไปใช้เป็น corrections ดังนั้นพวกมันมองไม่เห็นทั้งสองอย่างโดยสมบูรณ์ และเช่นเดียวกันสำหรับ $X$ stabilizer generators, errors และ corrections

เป็นตัวอย่าง มาพิจารณา 7-qubit Steane code

$$
\begin{array}{ccccccc}
Z & Z & Z & Z & \mathbb{I} & \mathbb{I} & \mathbb{I} \\[1mm]
Z & Z & \mathbb{I} & \mathbb{I} & Z & Z & \mathbb{I} \\[1mm]
Z & \mathbb{I} & Z & \mathbb{I} & Z & \mathbb{I} & Z \\[1mm]
X & X & X & X & \mathbb{I} & \mathbb{I} & \mathbb{I} \\[1mm]
X & X & \mathbb{I} & \mathbb{I} & X & X & \mathbb{I} \\[1mm]
X & \mathbb{I} & X & \mathbb{I} & X & \mathbb{I} & X
\end{array}
$$

แนวคิดพื้นฐานสำหรับโค้ดนี้ชัดเจนแล้วในตอนนี้: มันเป็น $[7,4,3]$-Hamming code สำหรับ bit-flip errors และ $[7,4,3]$-Hamming code สำหรับ phase-flip errors
ข้อเท็จจริงที่ $X$ และ $Z$ stabilizer generators commute กันอาจเป็นความโชคดี เพราะนี่จะไม่เป็น valid stabilizer code หากไม่ได้เป็นเช่นนั้น
แต่อันที่จริง มีตัวอย่างที่รู้จักมากมายของ classical linear codes ที่ให้ valid stabilizer code เมื่อใช้ในลักษณะที่คล้ายกัน

โดยทั่วไป สมมติว่ามี CSS code ที่ $Z$ stabilizer generators อนุญาตให้แก้ไข bit-flip errors สูงสุด $j$ อย่าง และ $X$ stabilizer generators อนุญาตให้แก้ไข phase-flip errors สูงสุด $k$ อย่าง
ตัวอย่างเช่น $j = 1$ และ $k = 1$ สำหรับ Steane code เนื่องจาก $[7,4,3]$-Hamming code สามารถแก้ไข bit flip หนึ่งอย่างได้
จากนั้นตามมาว่า จาก discretization of errors CSS code นี้สามารถแก้ไข *ข้อผิดพลาดใด ๆ* บน qubits จำนวนหนึ่งสูงสุดถึง *minimum* ของ $j$ และ $k$
เพราะเมื่อวัด syndrome ข้อผิดพลาดใด ๆ บน qubit จำนวนนั้นจะ collapse อย่าง probabilistic เป็น combination บางอย่างของ $X$ errors, $Z$ errors หรือทั้งสอง — แล้ว $X$ errors และ $Z$ errors จะถูกตรวจจับและแก้ไขอย่างอิสระ

สรุปได้ว่า ถ้ามี classical linear codes สองตัว (หรือสำเนาสองตัวของ classical linear code เดียวกัน) ที่เข้ากันได้ ในแง่ที่กำหนด $X$ และ $Z$ stabilizer generators ที่ commute กัน CSS code ที่ได้จากการรวมพวกมันจะรับทอดคุณสมบัติการแก้ไขข้อผิดพลาดของโค้ดทั้งสองนั้น ในแง่ที่เพิ่งอธิบาย

สังเกตว่ามีราคาที่ต้องจ่ายด้วย กล่าวคือเราไม่สามารถ *encode* qubits ได้มากเท่ากับที่เราสามารถ encode bits ด้วย classical codes สองตัว
เพราะจำนวนรวมของ stabilizer generators สำหรับ CSS code คือ *ผลรวม* ของจำนวน parity checks สำหรับ classical linear codes สองตัว และ stabilizer generator แต่ละตัวลด dimension ของ code space ลงครึ่งหนึ่ง
ตัวอย่างเช่น $[7,4,3]$-Hamming code อนุญาตให้ encode classical bits สี่อย่าง เพราะมีเพียง parity check strings สามตัวสำหรับโค้ดนี้ ในขณะที่ 7-qubit Steane code encode เพียงหนึ่ง qubit เพราะมี stabilizer generators หกตัว

## Code spaces ของ CSS codes {#code-spaces-of-css-codes}

สิ่งสุดท้ายที่เราจะทำในการอภิปราย CSS codes นี้คือพิจารณา *code spaces* ของโค้ดเหล่านี้
สิ่งนี้จะให้โอกาสตรวจสอบความสัมพันธ์ที่ต้องมีระหว่าง classical linear codes สองตัวเพื่อให้เข้ากันได้ ในแง่ที่สามารถรวมเข้าด้วยกันเพื่อเป็น CSS code

พิจารณา CSS code ใด ๆ บน $n$ qubits และให้ $z_1, \ldots, z_s \in \Sigma^n$ เป็น $n$-bit parity check strings ที่สอดคล้องกับ $Z$ stabilizer generators ของโค้ดนี้
หมายความว่า classical linear code ที่อธิบายด้วยเฉพาะ $Z$ stabilizer generators ซึ่งเราจะตั้งชื่อว่า $\mathcal{C}_Z$ มีรูปแบบต่อไปนี้

$$
\mathcal{C}_Z = \bigl\{ u \in \Sigma^n \,:\, u \cdot z_1 = \cdots = u \cdot z_s = 0 \bigr\}
$$

พูดเป็นคำพูด classical linear code $\mathcal{C}_Z$ มีทุก string ที่ binary dot product กับทุก parity check strings $z_1, \ldots, z_s$ เป็นศูนย์

ในทิศทางเดียวกัน ให้นำ $x_1,\ldots,x_t\in\Sigma^n$ เป็น $n$-bit parity check strings ที่สอดคล้องกับ $X$ stabilizer generators ของโค้ดของเรา
ดังนั้น classical linear code ที่สอดคล้องกับ $X$ stabilizer generators มีรูปแบบนี้

$$
\mathcal{C}_X = \bigl\{ u \in \Sigma^n \,:\, u \cdot x_1 = \cdots = u \cdot x_t = 0 \bigr\}
$$

$X$ stabilizer generators เพียงอย่างเดียวจึงอธิบายโค้ดที่คล้ายกับโค้ดนี้ แต่ใน
$\{\vert {+}\rangle,\vert {-}\rangle\}$ basis แทน standard basis

ตอนนี้เราจะแนะนำ classical linear codes ใหม่สองตัวที่ได้มาจากทางเลือกเดียวกันของ strings $z_1,\ldots,z_s$ และ $x_1,\ldots,x_t$ แต่เราถือว่า strings เหล่านี้เป็น *generators* แทนที่จะเป็น parity check strings
โดยเฉพาะ เราได้โค้ดสองตัวนี้

$$
\begin{aligned}
\mathcal{D}_Z & =
\bigl\{ \alpha_1 z_1 \oplus \cdots \oplus
\alpha_s z_s \,:\, \alpha_1,\ldots,\alpha_s \in \{0,1\}
\bigr\}\\[1mm]
\mathcal{D}_X & =
\bigl\{ \alpha_1 x_1 \oplus \cdots \oplus
\alpha_t x_t \,:\, \alpha_1,\ldots,\alpha_t \in \{0,1\}
\bigr\}
\end{aligned}
$$

สิ่งเหล่านี้เรียกว่า *dual codes* ของโค้ดที่นิยามก่อนหน้า:
$\mathcal{D}_Z$ คือ dual code ของ $\mathcal{C}_Z$ และ $\mathcal{D}_X$ คือ dual code ของ $\mathcal{C}_X$
อาจไม่ชัดเจนในจุดนี้ว่า dual codes เหล่านี้เกี่ยวข้องอย่างไร แต่มันเกี่ยวข้องอย่างมากด้วยหลายเหตุผล รวมถึงสองเหตุผลที่อธิบายในย่อหน้าต่อไปนี้

ประการแรก เงื่อนไขที่ต้องมีสำหรับ classical linear codes $\mathcal{C}_Z$ และ $\mathcal{C}_X$ สองตัวเพื่อให้เข้ากันได้ ในแง่ที่สามารถจับคู่กันเพื่อเป็น CSS code สามารถอธิบายได้ในคำพูดง่าย ๆ โดยอ้างถึง dual codes
โดยเฉพาะ ต้องเป็นว่า $\mathcal{D}_Z\subseteq\mathcal{C}_X$ หรือเทียบเท่าคือ $\mathcal{D}_X\subseteq\mathcal{C}_Z$
พูดเป็นคำพูด dual code $\mathcal{D}_Z$ รวม strings ที่สอดคล้องกับ $Z$ stabilizer generators และการที่พวกมันอยู่ใน $\mathcal{C}_X$ เทียบเท่ากับ binary dot product ของแต่ละ string เหล่านี้กับตัวที่สอดคล้องกับ $X$ stabilizer generators เป็นศูนย์
สิ่งนั้น ในทางกลับกัน เทียบเท่ากับ $Z$ stabilizer generator แต่ละตัว commute กับ $X$ stabilizer generator แต่ละตัว
หรือโดยสลับบทบาทของ $X$ และ $Z$ stabilizer generators และเริ่มจากการบรรจุ $\mathcal{D}_X\subseteq\mathcal{C}_Z$ เราสามารถสรุปผลเดียวกันได้

ประการที่สอง โดยอ้างถึง dual codes เราสามารถอธิบาย code spaces ของ CSS code ที่กำหนดได้ง่าย
โดยเฉพาะ code space ถูก span ด้วย vectors ของรูปแบบต่อไปนี้

$$
\vert u \oplus \mathcal{D}_X\rangle
= \frac{1}{\sqrt{2^t}} \sum_{v\in\mathcal{D}_X} \vert u \oplus v\rangle \qquad \text{(for all $u\in\mathcal{C}_Z$)}
$$

พูดเป็นคำพูด vectors เหล่านี้คือ uniform superpositions บน strings ใน dual code $\mathcal{D}_X$ ของโค้ดที่สอดคล้องกับ $X$ stabilizer generators ที่ถูก shift ด้วย (กล่าวคือ bitwise XORed กับ) strings ใน code $\mathcal{C}_Z$ ที่สอดคล้องกับ $Z$ stabilizer generators
เพื่อให้ชัดเจน ทางเลือกต่าง ๆ สำหรับ shift — แทนด้วย string $u$ ใน expression นี้ — อาจให้ vector เดียวกัน
ดังนั้น states เหล่านี้ไม่ได้แตกต่างกันทั้งหมด แต่รวมกันแล้ว span code space ทั้งหมด

นี่คือคำอธิบายเชิงสัญชาตญาณว่าทำไม vectors ดังกล่าวถึงทั้งอยู่ใน code space และ span มัน
พิจารณา $n$-qubit standard basis state $\vert u\rangle$ สำหรับ $n$-bit string $u$ ใด ๆ และสมมติว่าเรา *project* state นี้ลงบน code space
กล่าวคือ ให้ $\Pi$ แทน projection ลงบน code space ของ CSS code ของเรา พิจารณา vector $\Pi\vert u\rangle$
มีสองกรณี:

- กรณีที่ 1: $u\in\mathcal{C}_Z$
  สิ่งนี้หมายความว่า $Z$ stabilizer generator แต่ละตัวของ CSS code ของเราทำงาน trivially บน $\vert u\rangle$
  ในทางกลับกัน $X$ stabilizer generators แต่ละตัว flip bits บางส่วนของ $\vert u\rangle$
  โดยเฉพาะ สำหรับ generator $v$ ของ $\mathcal{D}_X$ แต่ละตัว $X$ stabilizer generator ที่สอดคล้องกับ $v$ แปลง $\vert u\rangle$ เป็น $\vert u\oplus v\rangle$
  โดยระบุ projection $\Pi$ เป็น *ค่าเฉลี่ย* บน elements ของ stabilizer (ดังที่เราเห็นในบทก่อนหน้า) เราได้สูตรนี้:

  $$
  \Pi \vert u \rangle = \frac{1}{2^t} \sum_{v\in\mathcal{D}_{X}} \vert u \oplus v\rangle = \frac{1}{\sqrt{2^t}} \vert u \oplus \mathcal{D}_X\rangle.
  $$

- กรณีที่ 2: $u\notin\mathcal{C}_Z$ สิ่งนี้หมายความว่า parity checks อย่างน้อยหนึ่งอันที่สอดคล้องกับ $Z$ stabilizer generators ล้มเหลว ซึ่งก็คือ $\vert u\rangle$ ต้องเป็น $-1$ eigenvector ของ $Z$ stabilizer generators อย่างน้อยหนึ่งตัว code space ของ CSS code คือ intersection ของ $+1$ eigenspaces ของ stabilizer generators ดังนั้น ในฐานะ $-1$ eigenvector ของ $Z$ stabilizer generators อย่างน้อยหนึ่งตัว $\vert u\rangle$ จึง orthogonal กับ code space:

  $$
  \Pi\vert u\rangle = 0.
  $$

และตอนนี้ เมื่อเราวนซ้ำบน $n$-bit strings $u$ ทั้งหมด ทิ้งตัวที่ $\Pi\vert u\rangle = 0$ และ normalize ตัวที่เหลือ เราได้ vectors ที่อธิบายก่อนหน้า ซึ่งแสดงว่าพวกมัน span code space

เรายังสามารถใช้ความสมมาตรระหว่าง $X$ และ $Z$ stabilizer generators เพื่ออธิบาย code space ในทิศทางที่คล้ายกันแต่แตกต่าง
โดยเฉพาะ มันคือ space ที่ span ด้วย vectors ที่มีรูปแบบต่อไปนี้

$$
H^{\otimes n} \vert u \oplus \mathcal{D}_Z\rangle
= \frac{1}{\sqrt{2^s}} \sum_{v\in\mathcal{D}_Z}
H^{\otimes n}\vert u \oplus v\rangle \qquad \text{(for $u\in\mathcal{C}_X$)}
$$

โดยพื้นฐาน $X$ และ $Z$ ถูกสลับในทุก instance ที่ปรากฏ — แต่เราต้องสลับ standard basis สำหรับ $\{\vert+\rangle,\vert-\rangle\}$ basis ด้วย ซึ่งเป็นเหตุผลที่ Hadamard operations รวมอยู่

เป็นตัวอย่าง มาพิจารณา 7-qubit Steane code
parity check strings สำหรับทั้ง $X$ และ $Z$ stabilizer generators เหมือนกัน:
$1111000,$ $1100110,$ และ $1010101$
ดังนั้น codes $\mathcal{C}_X$ และ $\mathcal{C}_Z$ เหมือนกัน ทั้งสองเท่ากับ $[7,4,3]$-Hamming code

$$
\mathcal{C}_X = \mathcal{C}_Z =
\{0000000, 0000111, 0011001, 0011110, 0101010, 0101101, 0110011, 0110100, 1001011, 1001100, 1010010, 1010101, 1100001, 1100110, 1111000, 1111111\}
$$

ดังนั้น dual codes $\mathcal{D}_X$ และ $\mathcal{D}_Z$ ก็เหมือนกันด้วย
เรามี generators สามตัว ดังนั้นเราได้ strings แปดตัว

$$
\mathcal{D}_X = \mathcal{D}_Z =
\{0000000, 0011110, 0101101, 0110011, 1001011, 1010101, 1100110, 1111000\}
$$

strings เหล่านี้อยู่ใน $[7,4,3]$-Hamming code ทั้งหมด ดังนั้นเงื่อนไข CSS จึงตอบสนอง: $\mathcal{D}_Z \subseteq \mathcal{C}_X$ หรือเทียบเท่าคือ $\mathcal{D}_X \subseteq \mathcal{C}_Z$

เนื่องจาก $\mathcal{D}_X$ มีครึ่งหนึ่งของ strings ทั้งหมดใน $\mathcal{C}_Z$ จึงมีเพียง vectors $\vert u\oplus \mathcal{D}_X\rangle$ สองตัวที่แตกต่างที่ได้จากการเลือก $u\in\mathcal{C}_Z$
สิ่งนี้คาดหวังได้ เพราะ 7-qubit Steane code มี code space สองมิติ
เราสามารถใช้สอง states ที่เราได้ในลักษณะนี้เพื่อ encode logical state $\vert 0\rangle$ และ $\vert 1\rangle$ ดังนี้

$$
\begin{aligned}
\vert 0\rangle & \mapsto
\frac{
  \vert 0000000\rangle + \vert 0011110\rangle + \vert 0101101\rangle + \vert 0110011\rangle + \vert 1001011\rangle + \vert 1010101\rangle + \vert 1100110\rangle + \vert 1111000\rangle
}{\sqrt{8}}\\[4mm]
\vert 1\rangle & \mapsto
\frac{
  \vert 0000111\rangle + \vert 0011001\rangle + \vert 0101010\rangle + \vert 0110100\rangle + \vert 1001100\rangle + \vert 1010010\rangle + \vert 1100001\rangle + \vert 1111111\rangle
}{\sqrt{8}}
\end{aligned}
$$

ตามปกติ ทางเลือกนี้ไม่ได้บังคับ เราเป็นอิสระในการใช้ code space เพื่อ encode qubits อย่างไรก็ได้ตามที่เลือก
อย่างไรก็ตาม encoding นี้สอดคล้องกับตัวอย่างของ encoding circuit สำหรับ 7-qubit Steane code ในบทก่อนหน้า

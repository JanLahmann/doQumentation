## Cost functions

โดยทั่วไป cost function ใช้เพื่ออธิบายเป้าหมายของปัญหาและบอกว่า trial state ทำได้ดีแค่ไหนเทียบกับเป้าหมายนั้น นิยามนี้สามารถนำไปใช้กับตัวอย่างต่างๆ ในเคมี, machine learning, การเงิน, optimization เป็นต้น

มาพิจารณาตัวอย่างง่ายๆ ของการหา ground state ของระบบ เป้าหมายของเราคือการ minimize expectation value ของ observable ที่แสดงพลังงาน (Hamiltonian $\hat{\mathcal{H}}$):

$$
\min_{\vec\theta} \langle\psi(\vec\theta)|\hat{\mathcal{H}}|\psi(\vec\theta)\rangle
$$

เราสามารถใช้ `Estimator` เพื่อประเมิน expectation value และส่งค่านี้ให้ optimizer เพื่อ minimize ถ้า optimization สำเร็จ จะคืนชุดค่า optimal parameter $\vec\theta^*$ ซึ่งเราจะสามารถสร้าง proposed solution state $|\psi(\vec\theta^*)\rangle$ และคำนวณ observed expectation value เป็น $C(\vec\theta^*)$

สังเกตว่าเราจะ minimize cost function ได้เฉพาะสำหรับ state ที่เราพิจารณาเท่านั้น ซึ่งนำไปสู่สองความเป็นไปได้:

- **Ansatz ของเราไม่กำหนด solution state ทั่ว search space**: ถ้าเป็นเช่นนี้ optimizer ของเราจะไม่มีวันหาคำตอบได้ และเราต้องทดลอง ansatz อื่นที่อาจแสดง search space ได้แม่นยำกว่า
- **Optimizer ของเราไม่สามารถหา solution ที่ valid นี้ได้**: Optimization สามารถกำหนดแบบ global และ local เราจะสำรวจความหมายของสิ่งนี้ในส่วนถัดไป

โดยรวมแล้ว เราจะดำเนินการ classical optimization loop แต่ขึ้นกับการประเมิน cost function กับคอมพิวเตอร์ควอนตัม จากมุมมองนี้ อาจคิด optimization เป็นความพยายาม classical ล้วนๆ ที่เราเรียก <DefinitionTooltip definition="A hypothetical device or software component that performs a specific function, but the internal workings of which are unknown. A user is only aware of the inputs and outputs of the black box, and has no knowledge of how the black box processes the inputs to produce the outputs.">black-box quantum oracle</DefinitionTooltip> ทุกครั้งที่ optimizer ต้องการประเมิน cost function

```python
def cost_func_vqe(params, circuit, hamiltonian, estimator):
    """Return estimate of energy from estimator

    Parameters:
        params (ndarray): Array of ansatz parameters
        ansatz (QuantumCircuit): Parameterized ansatz circuit
        hamiltonian (SparsePauliOp): Operator representation of Hamiltonian
        estimator (Estimator): Estimator primitive instance

    Returns:
        float: Energy estimate
    """
    pub = (circuit, hamiltonian, params)
    cost = estimator.run([pub]).result()[0].data.evs
    return cost
```

### ตัวอย่างการ mapping ไปยังระบบที่ไม่ใช่ physical

ปัญหา maximum cut (Max-Cut) เป็น combinatorial optimization problem ที่เกี่ยวข้องกับการแบ่ง vertex ของกราฟออกเป็นสองชุดที่ไม่มีจุดร่วมกัน โดยให้จำนวน edge ระหว่างสองชุดมีค่าสูงสุด พูดอย่างเป็นทางการ กำหนดกราฟ undirected $G=(V,E)$ โดย $V$ คือชุดของ vertex และ $E$ คือชุดของ edge ปัญหา Max-Cut ต้องการแบ่ง vertex ออกเป็นสองกลุ่มที่ไม่มีจุดร่วมกัน $S$ และ $T$ โดยให้จำนวน edge ที่มี endpoint หนึ่งอยู่ใน $S$ และอีกอันอยู่ใน $T$ มีค่าสูงสุด

เราสามารถนำ Max-Cut ไปใช้แก้ปัญหาหลายประเภท รวมถึง: การจัดกลุ่ม, network design, phase transition เป็นต้น เริ่มต้นด้วยการสร้าง problem graph:

ปัญหานี้สามารถแสดงเป็น binary optimization problem สำหรับ node แต่ละตัว $0 \leq i < n$ โดย $n$ คือจำนวน node ของกราฟ (ในกรณีนี้ $n=4$) เราจะพิจารณา binary variable $x_i$ ตัวแปรนี้จะมีค่า $1$ ถ้า node $i$ อยู่ในกลุ่มที่เราเรียกว่า $1$ และ $0$ ถ้าอยู่ในอีกกลุ่มที่เรียกว่า $0$ เราจะแทน $w_{ij}$ (element $(i,j)$ ของ adjacency matrix $w$) เป็นน้ำหนักของ edge จาก node $i$ ไปยัง node $j$ เนื่องจากกราฟเป็น undirected จึงได้ $w_{ij}=w_{ji}$ จากนั้นเราสามารถกำหนดปัญหาของเราเป็นการ maximize cost function ต่อไปนี้:

$$
\begin{aligned}
C(\vec{x})
& =\sum_{i,j=0}^n w_{ij} x_i(1-x_j)\\[1mm]

& = \sum_{i,j=0}^n w_{ij} x_i - \sum_{i,j=0}^n w_{ij} x_ix_j\\[1mm]

& = \sum_{i,j=0}^n w_{ij} x_i - \sum_{i=0}^n \sum_{j=0}^i 2w_{ij} x_ix_j
\end{aligned}
$$

เพื่อแก้ปัญหานี้ด้วยคอมพิวเตอร์ควอนตัม เราจะแสดง cost function เป็น expected value ของ observable อย่างไรก็ตาม observable ที่ Qiskit รองรับโดยธรรมชาติประกอบด้วย Pauli operator ที่มี eigenvalue $1$ และ $-1$ แทนที่จะเป็น $0$ และ $1$ นั่นคือเหตุผลที่เราจะทำการเปลี่ยนตัวแปรต่อไปนี้:

โดยที่ $\vec{x}=(x_0,x_1,\cdots ,x_{n-1})$ เราสามารถใช้ adjacency matrix $w$ เพื่อเข้าถึงน้ำหนักของ edge ทั้งหมดได้สะดวก สิ่งนี้จะนำไปสู่ cost function ของเรา:

$$
z_i = 1-2x_i \rightarrow x_i = \frac{1-z_i}{2}
$$

ซึ่งหมายความว่า:

$$
\begin{array}{lcl} x_i=0 & \rightarrow & z_i=1 \\ x_i=1 & \rightarrow & z_i=-1.\end{array}
$$

ดังนั้น cost function ใหม่ที่เราต้องการ maximize คือ:

$$
\begin{aligned}
C(\vec{z})
& = \sum_{i,j=0}^n w_{ij} \bigg(\frac{1-z_i}{2}\bigg)\bigg(1-\frac{1-z_j}{2}\bigg)\\[1mm]

& = \sum_{i,j=0}^n \frac{w_{ij}}{4} - \sum_{i,j=0}^n \frac{w_{ij}}{4} z_iz_j\\[1mm]

& = \sum_{i=0}^n \sum_{j=0}^i \frac{w_{ij}}{2} -  \sum_{i=0}^n \sum_{j=0}^i \frac{w_{ij}}{2} z_iz_j
\end{aligned}
$$

นอกจากนี้ แนวโน้มตามธรรมชาติของคอมพิวเตอร์ควอนตัมคือการหา minima (มักเป็น lowest energy) แทนที่จะเป็น maxima ดังนั้นแทนที่จะ maximize $C(\vec{z})$ เราจะ minimize:

$$
-C(\vec{z}) =  \sum_{i=0}^n \sum_{j=0}^i \frac{w_{ij}}{2} z_iz_j -  \sum_{i=0}^n \sum_{j=0}^i \frac{w_{ij}}{2}
$$

เมื่อเรามี cost function ที่ต้องการ minimize ซึ่งมีตัวแปรที่มีค่าได้ $-1$ และ $1$ เราสามารถทำ analogy ต่อไปนี้กับ Pauli $Z$:

$$
z_i \equiv Z_i = \overbrace{I}^{n-1}\otimes ... \otimes \overbrace{Z}^{i} \otimes ... \otimes \overbrace{I}^{0}
$$

กล่าวอีกนัยหนึ่ง ตัวแปร $z_i$ จะเทียบเท่ากับ $Z$ gate ที่กระทำกับ Qubit $i$ นอกจากนี้:

$$
Z_i|x_{n-1}\cdots x_0\rangle = z_i|x_{n-1}\cdots x_0\rangle \rightarrow \langle x_{n-1}\cdots x_0 |Z_i|x_{n-1}\cdots x_0\rangle = z_i
$$

จากนั้น observable ที่เราจะพิจารณาคือ:

$$
\hat{H} = \sum_{i=0}^n \sum_{j=0}^i \frac{w_{ij}}{2} Z_iZ_j
$$

ซึ่งเราต้องเพิ่ม independent term ทีหลัง:

$$
\texttt{offset} = - \sum_{i=0}^n \sum_{j=0}^i \frac{w_{ij}}{2}
$$
Operator เป็น linear combination ของ term ที่มี Z operator บน node ที่เชื่อมต่อกันด้วย edge (recall ว่า Qubit ที่ 0 อยู่ขวาสุด): $IIZZ + IZIZ + IZZI + ZIIZ + ZZII$ เมื่อสร้าง operator แล้ว ansatz สำหรับอัลกอริทึม QAOA ก็สร้างได้ง่ายโดยใช้ `QAOAAnsatz` circuit จาก Qiskit circuit library


เมื่อ Runtime Estimator รับ Hamiltonian และ parameterized ansatz โดยตรงและคืนพลังงานที่จำเป็น cost function สำหรับ QAOA instance จึงค่อนข้างง่าย:

```python
def cost_func(params, ansatz, hamiltonian, estimator):
    """Return estimate of energy from estimator

    Parameters:
        params (ndarray): Array of ansatz parameters
        ansatz (QuantumCircuit): Parameterized ansatz circuit
        hamiltonian (SparsePauliOp): Operator representation of Hamiltonian
        estimator (Estimator): Estimator primitive instance

    Returns:
        float: Energy estimate
    """
    pub = (ansatz, hamiltonian, params)
    cost = estimator.run([pub]).result()[0].data.evs
    #    cost = estimator.run(ansatz, hamiltonian, parameter_values=params).result().values[0]
    return cost
```

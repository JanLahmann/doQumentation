## อัลกอริทึม Deutsch {#deutschs-algorithm}

แม้ว่า quantum parallelism เพียงอย่างเดียวจะไม่ได้ให้ข้อได้เปรียบเหนือคอมพิวเตอร์คลาสสิก แต่เราสามารถจับคู่มันกับปรากฏการณ์ควอนตัมอีกอย่างหนึ่งคือ interference เพื่อให้ได้ความเร็วที่เพิ่มขึ้น อัลกอริทึมที่ปัจจุบันรู้จักกันในชื่อ "อัลกอริทึม Deutsch" คือตัวอย่างแรกของอัลกอริทึมที่บรรลุสิ่งนี้

### ปัญหา {#the-problem}

ปัญหาคือ:

กำหนด input bit $x = \{0,1\}$ และ input function $f(x) = \{0,1\}$ ให้ระบุว่าฟังก์ชันนั้น*balanced* หรือ *constant* นั่นคือ ถ้า balanced แสดงว่า output ของฟังก์ชันเป็น 0 ครึ่งหนึ่งของเวลาและเป็น 1 อีกครึ่งหนึ่ง ถ้า constant แสดงว่า output ของฟังก์ชันเป็น 0 หรือ 1 เสมอ ลองนึกถึงตารางฟังก์ชันสี่แบบที่เป็นไปได้ซึ่งรับ bit เดียวไปยัง bit เดียว:

| $x$    |  $f_1(x)$ | $f_2(x)$ | $f_3(x)$ | $f_4(x)$ |
| ----------- | ------- |-------| -------- | ------- |
| 0  | 0   | 0  | 1   | 1 |
| 1  | 0   | 1  | 0 | 1 |

ฟังก์ชันแรกและสุดท้าย $f_1(x)$ และ $f_4(x)$ เป็น constant ส่วนสองฟังก์ชันกลาง $f_2(x)$ และ $f_3(x)$ เป็น balanced
### อัลกอริทึม {#the-algorithm}

วิธีที่ Deutsch เข้าถึงปัญหานี้คือผ่าน "query model" ใน query model input function ($f_i(x)$ ข้างต้น) อยู่ใน "black box" — เราไม่มีสิทธิ์เข้าถึงเนื้อหาโดยตรง แต่เราสามารถ query black box และมันจะให้ output ของฟังก์ชันแก่เรา บางครั้งเราบอกว่า "oracle" ให้ข้อมูลนี้ ดู [บทที่ 1: Quantum Query Algorithms](/learning/courses/fundamentals-of-quantum-algorithms/quantum-query-algorithms/introduction) ของหลักสูตร Fundamentals of Quantum Algorithms สำหรับข้อมูลเพิ่มเติมเกี่ยวกับ query model

เพื่อพิจารณาว่าอัลกอริทึมควอนตัมมีประสิทธิภาพมากกว่าอัลกอริทึมคลาสสิกใน query model หรือไม่ เราสามารถเปรียบเทียบจำนวน query ที่เราต้องทำกับ black box ในแต่ละกรณี ในกรณีคลาสสิก เพื่อทราบว่าฟังก์ชันใน black box เป็น balanced หรือ constant เราจะต้อง query กล่องสองครั้งเพื่อได้ทั้ง $f(0)$ และ $f(1)$

ในอัลกอริทึมควอนตัมของ Deutsch เขาพบวิธีที่จะได้ข้อมูลด้วยการ query เพียงครั้งเดียว! เขาทำการปรับเปลี่ยนหนึ่งอย่างใน Circuit "quantum parallelism" ข้างต้น โดยเตรียม superposition state บน*ทั้ง*สอง Qubit แทนที่จะเตรียมเฉพาะ Qubit 0 จากนั้น output ทั้งสองของฟังก์ชัน $f(0)$ และ $f(1)$ จะ interfere เพื่อคืนค่า 0 หากทั้งคู่เป็น 0 หรือทั้งคู่เป็น 1 (ฟังก์ชันเป็น constant) และคืนค่า 1 หากต่างกัน (ฟังก์ชันเป็น balanced) ด้วยวิธีนี้ Deutsch สามารถแยกแยะระหว่างฟังก์ชัน constant และ balanced ด้วยการ query เพียงครั้งเดียว

$$
|\pi_2\rangle = \begin{cases}
\pm[\frac{|0\rangle-|1\rangle}{\sqrt{2}}][\frac{|0\rangle+|1\rangle}{\sqrt{2}}] & \text{if} & f(0) = f(1) \\
\pm[\frac{|0\rangle-|1\rangle}{\sqrt{2}}][\frac{|0\rangle-|1\rangle}{\sqrt{2}}] &\text{if} & f(0) \neq f(1) \\
\end{cases}
$$

$$
|\pi_3\rangle = \begin{cases}
\pm[\frac{|0\rangle-|1\rangle}{\sqrt{2}}]|0\rangle & \text{if} & f(0) = f(1) \\
\pm[\frac{|0\rangle-|1\rangle}{\sqrt{2}}]|1\rangle &\text{if} & f(0) \neq f(1) \\
\end{cases}
$$

</details>

เมื่อดูคำตอบสำหรับคำถามข้างต้น สังเกตว่ามีบางอย่างที่น่าแปลกใจเกิดขึ้น แม้ว่า $U_f$ จะไม่ได้ทำอะไรกับ state ของ Qubit 0 โดยตรง แต่เนื่องจากมันเปลี่ยน Qubit 1 ตาม state ของ Qubit 0 อาจเกิดขึ้นได้ว่านี่ทำให้เกิด phase shift ใน Qubit 0 ปรากฏการณ์นี้เรียกว่า "phase-kickback" และได้รับการอธิบายในรายละเอียดเพิ่มเติมใน [บทที่ 1: Quantum Query Algorithms](/learning/courses/fundamentals-of-quantum-algorithms/quantum-query-algorithms/introduction) ของหลักสูตร Fundamentals of Quantum Algorithms

## อัลกอริทึม Deutsch-Jozsa {#the-deutsch-jozsa-algorithm}

อัลกอริทึม Deutsch เป็นก้าวแรกที่สำคัญในการแสดงให้เห็นว่าคอมพิวเตอร์ควอนตัมอาจมีประสิทธิภาพมากกว่าคอมพิวเตอร์คลาสสิก แต่เป็นเพียงการพัฒนาที่ไม่มากนัก: ต้องใช้เพียง query เดียว เทียบกับสองครั้งในกรณีคลาสสิก ในปี 1992 Deutsch และเพื่อนร่วมงาน Richard Jozsa ได้ขยายอัลกอริทึมสอง Qubit ดั้งเดิมให้รองรับ Qubit มากขึ้น ปัญหายังคงเหมือนเดิม: ระบุว่าฟังก์ชันเป็น*balanced* หรือ *constant* แต่คราวนี้ฟังก์ชันรับ $n$ bits ไปยัง bit เดียว ไม่ว่าฟังก์ชันจะคืนค่า 0 และ 1 เท่ากัน (เป็น*balanced*) หรือฟังก์ชันคืนค่า 1 หรือ 0 เสมอ (เป็น*constant*)

นี่คือ Circuit diagram ของอัลกอริทึม:

![DJ_algo.png](/learning/images/modules/computer-science/deutsch-jozsa/DJ_algo.avif)

อัลกอริทึมนี้ทำงานในลักษณะเดียวกับอัลกอริทึม Deutsch: phase-kickback ช่วยให้อ่าน state ของ Qubit 0 เพื่อระบุว่าฟังก์ชันเป็น constant หรือ balanced ซึ่งยากกว่าในกรณีอัลกอริทึม Deutsch สอง Qubit เล็กน้อย เนื่องจาก state จะรวม sums เหนือ $n$ Qubit ดังนั้นการหา state เหล่านั้นจึงเหลือเป็นแบบฝึกหัดเพิ่มเติมสำหรับคุณในตอนท้ายของโมดูล อัลกอริทึมจะคืนค่า bitstring ของเลข 0 ทั้งหมดหากฟังก์ชัน constant และคืน bitstring ที่มีอย่างน้อยหนึ่ง 1 หากฟังก์ชัน balanced

เพื่อดูว่าอัลกอริทึมทำงานใน Qiskit อย่างไร ก่อนอื่นเราต้องสร้าง oracle ของเรา: ฟังก์ชันสุ่มที่รับประกันว่าเป็น constant หรือ balanced โค้ดด้านล่างจะสร้างฟังก์ชัน balanced 50% ของเวลาและฟังก์ชัน constant 50% ของเวลา ไม่ต้องกังวลหากคุณไม่เข้าใจโค้ดทั้งหมด — มันซับซ้อนและไม่จำเป็นต้องใช้สำหรับความเข้าใจอัลกอริทึมควอนตัมของเรา

```python
from qiskit import QuantumCircuit
import numpy as np

def dj_function(num_qubits):
    """
    Create a random Deutsch-Jozsa function.
    """

    qc_dj = QuantumCircuit(num_qubits + 1)
    if np.random.randint(0, 2):
        # Flip output qubits with 50% chance
        qc_dj.x(num_qubits)
    if np.random.randint(0, 2):
        # return constant circuit with 50% chance.
        return qc_dj

    # If the "if" statement above was "TRUE" then we've returned the constant
    # function and the function is complete. If not, we proceed in creating our
    # balanced function. Everything below is to produce the balanced function:

    # select half of all possible states at random:
    on_states = np.random.choice(
        range(2**num_qubits),  # numbers to sample from
        2**num_qubits // 2,  # number of samples
        replace=False,  # makes sure states are only sampled once
    )

    def add_cx(qc_dj, bit_string):
        for qubit, bit in enumerate(reversed(bit_string)):
            if bit == "1":
                qc_dj.x(qubit)
        return qc_dj

    for state in on_states:
        # qc_dj.barrier()  # Barriers are added to help visualize how the functions are created. They can safely be removed.
        qc_dj = add_cx(qc_dj, f"{state:0b}")
        qc_dj.mcx(list(range(num_qubits)), num_qubits)
        qc_dj = add_cx(qc_dj, f"{state:0b}")

    # qc_dj.barrier()

    return qc_dj

n = 3  # number of input qubits

oracle = dj_function(n)

display(oracle.draw("mpl"))
```

นี่คือฟังก์ชัน oracle ซึ่งเป็นทั้ง balanced หรือ constant คุณมองดู Circuit แล้วบอกได้ไหมว่า output บน Qubit สุดท้ายขึ้นอยู่กับค่าที่ใส่ใน $n$ Qubit แรกหรือไม่? ถ้า output ของ Qubit สุดท้ายขึ้นอยู่กับ $n$ Qubit แรก คุณบอกได้ไหมว่า output ที่ขึ้นอยู่กับนั้นเป็น balanced หรือไม่?

เราสามารถบอกได้ว่าฟังก์ชันเป็น balanced หรือ constant โดยดู Circuit ข้างต้น แต่จำไว้ว่าเพื่อจุดประสงค์ของปัญหานี้ เราคิดว่าฟังก์ชันนี้เป็น "black box" เราไม่สามารถแอบดู Circuit diagram ได้ แต่เราต้อง query กล่อง

เพื่อ query กล่อง เราใช้อัลกอริทึม Deutsch-Jozsa และระบุว่าฟังก์ชันเป็น constant หรือ balanced:

```python
blackbox = oracle.to_gate()
blackbox.label = "$U_f$"

qc_dj = QuantumCircuit(n + 1, n)
qc_dj.x(n)
qc_dj.h(range(n + 1))
qc_dj.barrier()
qc_dj.compose(blackbox, inplace=True)
qc_dj.barrier()
qc_dj.h(range(n))
qc_dj.measure(range(n), range(n))

qc_dj.decompose().decompose()

qc_dj.draw("mpl")
```

__เฉลย:__

มี $2^n$ bitstring ที่เป็นไปได้ที่ต้องตรวจสอบ และในกรณีที่แย่ที่สุด คุณจะต้องทดสอบ $2^n/2+1$ ของสิ่งเหล่านี้ ตัวอย่างเช่น ถ้าฟังก์ชันเป็น constant และคุณยังคงวัดค่า "1" เป็น output ของฟังก์ชัน คุณจะไม่แน่ใจว่ามันเป็น constant จริงๆ จนกว่าจะตรวจสอบผลลัพธ์มากกว่าครึ่ง ก่อนนั้น คุณอาจโชคร้ายมากที่ยังคงวัดค่า "1" บนฟังก์ชัน balanced ก็ได้ เหมือนกับการโยนเหรียญซ้ำๆ แล้วออกหัวทุกครั้ง มันไม่น่าจะเกิดขึ้น แต่ก็ไม่ใช่ไม่ไปเป็นไปได้

</details>

<details>
<summary>

คำตอบข้างต้นของคุณจะเปลี่ยนอย่างไรหากคุณแค่ต้องวัดจนกว่าผลลัพธ์หนึ่ง (balanced หรือ constant) มีแนวโน้มมากกว่าอีกอันหนึ่ง? ในกรณีนี้ต้องใช้กี่ query?

</summary>

__เฉลย:__

ในกรณีนี้ คุณวัดสองครั้งก็ได้ ถ้าสองการวัดต่างกัน คุณรู้ว่าฟังก์ชันเป็น balanced ถ้าสองการวัดเหมือนกัน มันอาจเป็น balanced หรือ constant ก็ได้ ความน่าจะเป็นที่เป็น balanced ด้วยชุดการวัดนี้คือ: $\frac{1}{2}\frac{2^n /2 - 1}{2^n-1}$ ซึ่งน้อยกว่า 1/2 ดังนั้นมีแนวโน้มมากกว่าที่ฟังก์ชันจะ constant ในกรณีนี้

</details>

ดังนั้น อัลกอริทึม Deutsch-Jozsa แสดงให้เห็น exponential speedup เหนืออัลกอริทึมคลาสสิก*แบบ deterministic* (อัลกอริทึมที่คืนคำตอบด้วยความแน่นอน 100%) แต่ไม่มี speedup ที่สำคัญเหนืออัลกอริทึม*probabilistic* (อัลกอริทึมที่คืนผลลัพธ์ที่*น่าจะ*เป็นคำตอบที่ถูกต้อง)
### ปัญหา Bernstein-Vazirani {#the-bernstein---vazirani-problem}

ในปี 1997 Ethan Bernstein และ Umesh Vazirani ใช้อัลกอริทึม Deutsch-Jozsa เพื่อแก้ปัญหาที่เฉพาะเจาะจงและจำกัดกว่าเมื่อเทียบกับปัญหา Deutsch-Jozsa แทนที่จะพยายามแยกแยะระหว่างฟังก์ชันสองประเภทที่แตกต่างกันดังในกรณี D-J Bernstein และ Vazirani ใช้อัลกอริทึม Deutsch-Jozsa เพื่อเรียนรู้ string จริงๆ ที่ encode ไว้ในฟังก์ชัน นี่คือปัญหา:

ฟังก์ชัน $f:\{0,1\}^n \rightarrow \{0,1\}$ ยังคงรับ $n$-bit string และ output bit เดียว แต่ตอนนี้แทนที่จะ promise ว่าฟังก์ชันเป็น balanced หรือ constant เราถูก promise ว่าฟังก์ชันคือ dot product ระหว่าง input string $x$ และ secret $n$-bit string $s$ บวก modulo 2 (dot product modulo 2 นี้เรียกว่า "binary dot product") ปัญหาคือหาว่า secret $n$-bit string คืออะไร

เขียนอีกแบบหนึ่ง เราได้รับ black-box function $f: {0,1}^n \rightarrow {0,1}$ ที่ satisfy $f(x) = s \cdot x$ สำหรับ string $s$ บางตัว และเราต้องการเรียนรู้ string $s$

มาดูว่าอัลกอริทึม D-J แก้ปัญหานี้อย่างไร:

1. ขั้นแรก Hadamard gate ถูกใช้กับ $n$ input Qubit และ NOT gate บวก Hadamard ถูกใช้กับ output Qubit ทำให้ state เป็น:

$$
|\Psi\rangle = |-\rangle_{n} \otimes |+\rangle_{n-1} \otimes |+\rangle_{n-2} \otimes ... \otimes |+\rangle_0
$$

   State ของ Qubit 1 ถึง $n$ สามารถเขียนได้ง่ายกว่าเป็น sum เหนือ $n$-qubit basis states ทั้งหมด $|00...00\rangle, |00...01\rangle, |000...11\rangle, ..., |111...11\rangle$ เราเรียกชุดของ basis states เหล่านี้ว่า $\Sigma^n$ (ดู [Fundamentals of Quantum Algorithms](/learning/courses/fundamentals-of-quantum-algorithms/quantum-query-algorithms/deutsch-jozsa-algorithm) สำหรับรายละเอียดเพิ่มเติม)

$$
|\Psi\rangle = |-\rangle \otimes \frac{1}{\sqrt{2^n}}\sum\limits_{x \in \Sigma^n}{|x\rangle}
$$

2. ถัดไป Gate $U_f$ ถูกใช้กับ Qubit ต่างๆ Gate นี้จะรับ $n$ Qubit แรกเป็น input (ซึ่งตอนนี้อยู่ใน equal superposition ของ $n$-bit string ที่เป็นไปได้ทั้งหมด) และใช้ฟังก์ชัน $f(x)=s \cdot x$ กับ output Qubit ทำให้ Qubit นี้อยู่ใน state: $ |- \oplus f(x)\rangle$ ต้องขอบคุณกลไก phase kickback state ของ Qubit นี้ยังคงไม่เปลี่ยนแปลง แต่บางพจน์ใน input Qubit state ได้รับ minus sign:

$$
|\Psi\rangle = |-\rangle \otimes \frac{1}{\sqrt{2^n}}\sum\limits_{x \in \Sigma^n}{(-1)^{f(x)}|x\rangle}
$$

3. ตอนนี้ชุด Hadamard ถัดไปถูกใช้กับ Qubit 0 ถึง $n-1$ การติดตาม minus sign ในกรณีนี้อาจยุ่งยาก เป็นประโยชน์ที่จะรู้ว่าการใช้ layer ของ Hadamards กับ $n$ Qubit ใน standard basis state $|x\rangle$ สามารถเขียนได้เป็น:

$$
H^{\otimes n} |x\rangle = \frac{1}{\sqrt{2^n}}\sum\limits_{y \in \Sigma^n}{(-1)^{x \cdot y}|y\rangle}
$$

ดังนั้น state กลายเป็น:

$$
|\Psi\rangle = |-\rangle \otimes \frac{1}{2^n}\sum\limits_{x \in \Sigma^n}\sum\limits_{y \in \Sigma^n}{(-1)^{(s \cdot x) + (x \cdot y)}|y\rangle}
$$

4. ขั้นตอนถัดไปคือวัด $n$ bits แรก แต่เราจะวัดอะไรได้? ปรากฎว่า state ข้างต้น simplify เป็น: $|\Psi\rangle = |-\rangle \otimes |s\rangle$ แต่นั่นไม่ชัดเจนเลย ถ้าต้องการติดตามคณิตศาสตร์ ดูหลักสูตร [Fundamentals of Quantum Algorithms](/learning/courses/fundamentals-of-quantum-algorithms/quantum-query-algorithms/deutsch-jozsa-algorithm#the-bernstein-vazirani-problem) ของ John Watrous ประเด็นคือกลไก phase kickback ทำให้ input Qubit อยู่ใน state $|s\rangle$ ดังนั้น เพื่อหาว่า secret string $s$ คืออะไร คุณเพียงแค่วัด Qubit!

### ตรวจสอบความเข้าใจ {#check-your-understanding-4}
อ่านคำถามด้านล่าง คิดคำตอบของคุณ แล้วคลิกสามเหลี่ยมเพื่อดูเฉลย

<details>
<summary>

ยืนยันว่า state จากขั้นตอนที่ 3 ข้างต้นเป็น state $|s\rangle$ จริงๆ สำหรับกรณีพิเศษของ $n=1$

</summary>

__เฉลย:__

เมื่อเขียน summation ทั้งสองออกมาอย่างชัดเจน คุณควรได้ state ที่มีสี่พจน์ (ขอละเว้น output state $|-\rangle$ สำหรับสิ่งนี้):

$$
|\Psi\rangle = \frac{1}{2}[|0\rangle + (-1)^s |0\rangle + |1\rangle + (-1)^{(s+1)} |1\rangle]
$$

ถ้า $s=0$ สองพจน์แรก add constructively และสองพจน์สุดท้าย cancel เหลือ $|\Psi\rangle = |0\rangle$ ถ้า $s=1$ สองพจน์สุดท้าย add constructively และสองพจน์แรก cancel เหลือ $|\Psi\rangle = |1\rangle$ ดังนั้นในทั้งสองกรณี $|\Psi\rangle = |s\rangle$ หวังว่ากรณีง่ายที่สุดนี้จะให้คุณเข้าใจว่ากรณีทั่วไปที่มี $n$ Qubit ทำงานอย่างไร: พจน์ทั้งหมดที่ไม่ใช่ $|s\rangle$ จะ interfere ออกไป เหลือแค่ state $|s\rangle$

</details>

<details>
<summary>
อัลกอริทึมเดียวกันแก้ได้ทั้งปัญหา Bernstein-Vazirani และ Deutsch-Jozsa ได้อย่างไร? เพื่อทำความเข้าใจสิ่งนี้ ลองคิดถึงฟังก์ชัน Bernstein-Vazirani ซึ่งมีรูปแบบ $f(x) = s \cdot x$ ฟังก์ชันเหล่านี้ยังเป็นฟังก์ชัน Deutsch-Jozsa ด้วยไหม? นั่นคือ ระบุว่าฟังก์ชันในรูปแบบนี้ satisfy promise ของปัญหา Deutsch-Jozsa หรือไม่: ว่าเป็น*constant* หรือ *balanced* สิ่งนี้ช่วยให้เราเข้าใจว่าอัลกอริทึมเดียวกันแก้สองปัญหาที่แตกต่างกันได้อย่างไร?

</summary>

__เฉลย:__

ทุกฟังก์ชัน Bernstein-Vazirani ในรูปแบบ $f(x) = s \cdot x$ ยังคง satisfy promise ของปัญหา Deutsch-Jozsa: ถ้า s=00...00 ฟังก์ชันเป็น constant (คืนค่า 0 เสมอสำหรับทุก string x) ถ้า s เป็น string อื่นใด ฟังก์ชันเป็น balanced ดังนั้น การใช้อัลกอริทึม Deutsch-Jozsa กับฟังก์ชันเหล่านี้จะแก้ทั้งสองปัญหาพร้อมกัน! มันคืน string และถ้า string นั้นเป็น 00...00 เราก็รู้ว่ามันเป็น constant ถ้ามีอย่างน้อยหนึ่ง "1" ใน string เราก็รู้ว่ามันเป็น balanced

</details>

เราสามารถยืนยันได้ด้วยว่าอัลกอริทึมนี้แก้ปัญหา Bernstein-Vazirani ได้สำเร็จโดยทดสอบจริง ขั้นแรก เราสร้างฟังก์ชัน B-V ที่อยู่ใน black box:

```python
# Step 1: Map the problem

def bv_function(s):
    """
    Create a Bernstein-Vazirani function from a string of 1s and 0s.
    """
    qc = QuantumCircuit(len(s) + 1)
    for index, bit in enumerate(reversed(s)):
        if bit == "1":
            qc.cx(index, len(s))
    return qc

display(bv_function("1000").draw("mpl"))
```

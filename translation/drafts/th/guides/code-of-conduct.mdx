# จรรยาบรรณ {#code-of-conduct}

## คำมั่นสัญญาของเรา {#our-pledge}

ชุมชน Qiskit ยึดมั่นในค่านิยมของการปฏิบัติต่อทุกคนด้วยความเคารพและให้เกียรติ
เพื่อส่งเสริมสภาพแวดล้อมที่เปิดกว้างและเป็นมิตร
ผู้เข้าร่วมทุกคน ทั้งผู้เข้าร่วมงาน วิทยากร ผู้สนับสนุน อาสาสมัคร ผู้มีส่วนร่วมออนไลน์ และพนักงาน IBM
ต้องแสดงความสุภาพต่อกันและต่อชุมชนของเรา โดยสร้างประสบการณ์ที่ปราศจากการคุกคาม
ไม่ว่าจะด้วยอายุ รูปลักษณ์ภายนอก ความพิการ เชื้อชาติ อัตลักษณ์และการแสดงออกทางเพศ
รูปร่างร่างกาย ระดับประสบการณ์ สัญชาติ เผ่าพันธุ์ ศาสนา วรรณะ หรือรสนิยมและอัตลักษณ์ทางเพศ
พฤติกรรมที่คาดหวังนี้ใช้กับการมีส่วนร่วมทั้งออนไลน์และออฟไลน์ภายในชุมชน Qiskit

## ขอบเขต {#scope}

จรรยาบรรณนี้มีวัตถุประสงค์เพื่อกำหนดและบังคับใช้ค่านิยมและพฤติกรรม
ของผู้มีส่วนร่วมและผู้เข้าร่วมในชุมชน open source ของ Qiskit
จรรยาบรรณนี้ใช้บังคับทั้งในพื้นที่ของโปรเจกต์และในพื้นที่สาธารณะ
เมื่อบุคคลนั้นมีส่วนร่วมกับชุมชน open source ของ Qiskit เช่น
การเข้าร่วมงาน Qiskit การมีส่วนร่วมในโปรเจกต์ออนไลน์ การแสดงความคิดเห็นบน Slack
หรือการเป็นตัวแทนโปรเจกต์หรือชุมชน รวมถึงการใช้อีเมลทางการของโปรเจกต์
การโพสต์ผ่านบัญชีโซเชียลมีเดียทางการ หรือการทำหน้าที่เป็นตัวแทนที่ได้รับแต่งตั้ง
ในงานออนไลน์หรือออฟไลน์
การเป็นตัวแทนของโปรเจกต์อาจถูกกำหนดและชี้แจงเพิ่มเติมโดย project maintainers

## มาตรฐานของเรา {#our-standards}

ตัวอย่างพฤติกรรมที่ช่วยสร้างสภาพแวดล้อมเชิงบวก ได้แก่:

- การใช้ภาษาที่เป็นมิตรและครอบคลุมทุกคน
- เคารพต่อมุมมอง ประสบการณ์ และวัฒนธรรมที่แตกต่างกัน
- รับฟังคำติชมเชิงสร้างสรรค์อย่างเต็มใจ
- มุ่งเน้นสิ่งที่ดีที่สุดสำหรับชุมชน
- แสดงความเห็นอกเห็นใจต่อสมาชิกคนอื่นในชุมชน
- ใส่ใจต่อสภาพแวดล้อมรอบข้างและเพื่อนผู้เข้าร่วม และรับฟังคนอื่น
- ให้คุณค่ากับการมีส่วนร่วมของทุกคน
- ร่วมมือก่อนที่จะขัดแย้ง
- ชี้ให้เห็นคำพูดหรือมุกตลกที่แฝงการเหยียดเชื้อชาติ เพศ วรรณะ หรืออคติโดยไม่ตั้งใจ เมื่อเกิดขึ้น

ตัวอย่างพฤติกรรมที่ยอมรับไม่ได้ แม้จะนำเสนอในรูปแบบ "ล้อเล่น" หรือ "ขำขัน" ได้แก่:

- การใช้ภาษาหรือภาพที่มีนัยทางเพศ และการสัมผัสทางกาย ความสนใจ หรือการเข้าหาที่ไม่พึงประสงค์
- การ troll ความคิดเห็นที่ดูถูกหรือเหยียดหยาม และการโจมตีส่วนตัวหรือทางการเมือง
- การคุกคามทั้งในที่สาธารณะและส่วนตัว รวมถึงภาษาที่ดูถูกหรือทำให้เสื่อมเสีย
- เผยแพร่ข้อมูลส่วนตัวของผู้อื่น เช่น ที่อยู่กายหรืออิเล็กทรอนิกส์ โดยไม่ได้รับอนุญาตอย่างชัดแจ้ง รวมถึงการ "เปิดเผย" ข้อมูลใดก็ตามเกี่ยวกับตัวตนของบุคคลโดยไม่ได้รับความยินยอม
- "Doxxing" การเผยแพร่ภาพหน้าจอหรือข้อความ โดยเฉพาะจาก Slack ส่วนตัว การแชทส่วนตัว หรืองานสาธารณะ โดยไม่ได้รับความยินยอมอย่างชัดแจ้งจากผู้ถูกอ้างถึงทุกคน
- การทำ spam เช่น การส่งข้อความที่ไม่ได้รับการร้องขอ ผลลัพธ์จาก LLMs (Large Language Models) โฆษณา หรือเนื้อหาเชิงส่งเสริมการขายซ้ำ ๆ ไปยังสมาชิกชุมชนโดยไม่ได้รับอนุญาตจาก IBM ก่อน
- พฤติกรรมอื่น ๆ ที่อาจถือว่าไม่เหมาะสมในสภาพแวดล้อมเชิงวิชาชีพ

## ความรับผิดชอบและการบังคับใช้ {#responsibilities-and-enforcement}

ชุมชน Qiskit ทั้งหมดมีความรับผิดชอบในการรักษาเงื่อนไขของจรรยาบรรณ
ในงานและพื้นที่ของชุมชน Qiskit และรายงานการละเมิดหากพบเห็น
ทีม Qiskit ภายใน IBM เป็นผู้รับผิดชอบสูงสุดในการชี้แจงมาตรฐานของพฤติกรรมที่ยอมรับได้
และการบังคับใช้ และคาดหวังให้ดำเนินการแก้ไขที่เหมาะสมและยุติธรรม
ต่อกรณีพฤติกรรมที่ยอมรับไม่ได้

หากผู้เข้าร่วมหรือผู้มีส่วนร่วมแสดงพฤติกรรมเชิงลบหรือเป็นอันตราย
IBM จะดำเนินการตามที่เห็นสมควร ซึ่งรวมถึงแต่ไม่จำกัดเพียง
การออกคำเตือน การขับออกจากงานโดยไม่คืนเงิน การลบความคิดเห็น
การแบนถาวรจากงานในอนาคตหรือชุมชนออนไลน์ หรือการแจ้งเจ้าหน้าที่ท้องถิ่น
IBM มีสิทธิ์และความรับผิดชอบในการลบ แก้ไข หรือปฏิเสธ
ความคิดเห็น commits โค้ด wiki edits issues และการมีส่วนร่วมอื่น ๆ
ที่ไม่สอดคล้องกับจรรยาบรรณนี้ หรือจะแบนผู้มีส่วนร่วมหรือผู้เข้าร่วมชั่วคราวหรือถาวร
สำหรับพฤติกรรมอื่น ๆ ที่ถือว่าไม่เหมาะสม คุกคาม ไม่เหมาะสม หรือเป็นอันตราย

ถ้าพบการละเมิดจรรยาบรรณ:

1. ถ้ารู้สึกสบายใจพอ ให้แจ้งให้บุคคลนั้นทราบว่าการกระทำของเขาไม่เหมาะสม
   และขอให้หยุด และ/หรือแก้ไขหรือลบข้อความของตน
2. ถ้าบุคคลนั้นไม่หยุดพฤติกรรมทันทีหรือไม่แก้ไขปัญหา
   หรือถ้ารู้สึกไม่สบายใจที่จะพูด ให้แจ้ง moderator และถ้าเหมาะสม
   ให้กรอก[แบบฟอร์มรายงานการละเมิดจรรยาบรรณ](https://airtable.com/shrl5mEF4Eun1aIDm) แบบไม่ระบุตัวตน
3. ชุมชน Qiskit จะเปิดการสอบสวนเมื่อได้รับแบบฟอร์มของคุณ
   เมื่อรายงาน โปรดรวมรายละเอียดที่เกี่ยวข้อง ลิงก์ ภาพหน้าจอ บริบท
   หรือข้อมูลอื่น ๆ ที่อาจช่วยให้เข้าใจและแก้ไขสถานการณ์ได้ดีขึ้น
4. ถ้าการละเมิดจรรยาบรรณเกิดขึ้นในงานและต้องการการตอบสนองทันที
   หรือมีความกังวลเกี่ยวกับบุคคลที่จะเข้าร่วมงานที่กำลังจะมาถึง
   ให้ติดต่อจุดติดต่อ on-call สำหรับจรรยาบรรณของงานนั้น
   ซึ่งระบุไว้ในเอกสารจรรยาบรรณเฉพาะของงาน ถ้ารู้สึกไม่สบายใจ
   ที่จะพูดกับจุดติดต่อด้วยตนเอง ให้กรอกแบบฟอร์มรายงานการละเมิดจรรยาบรรณ
   และระบุรายละเอียดของงาน เพื่อให้คณะกรรมการบังคับใช้จรรยาบรรณ
   ติดต่อจุดติดต่อ on-call ของงานได้
5. ถ้าพนักงาน IBM พบเห็นการละเมิดจรรยาบรรณในเวลาใดก็ตาม เช่น
   ในงาน ใน Slack channel หรือ open source forums
   เป็นความรับผิดชอบของพวกเขาที่จะต้องยื่นรายงานการละเมิดจรรยาบรรณ

จรรยาบรรณนี้ไม่ได้แทนที่นโยบายองค์กร IBM ที่มีอยู่แล้ว เช่น
IBM Business Conduct Guidelines และ IBM Business Partner Code of Conduct

# ตั้งค่าบัญชี IBM Cloud ของคุณ {#set-up-your-ibm-cloud-account}

ถ้าได้รับอีเมลเชิญให้เข้าร่วมบัญชี ให้ทำตามขั้นตอนใน [ตั้งค่าบัญชี IBM Cloud ของคุณ - ผู้ใช้ที่ได้รับเชิญ](/guides/cloud-setup-invited) แทน

คุณรัน workload บน IBM&reg; quantum processing units (QPUs) ได้โดยการตั้งค่าบัญชีบน IBM Cloud&reg; และยังต้อง [ติดตั้งและตั้งค่า Qiskit และ Qiskit Runtime](/guides/install-qiskit#local) ด้วย

บัญชีผู้ใช้ของคุณเชื่อมต่อกับ [instance](./instances) อย่างน้อยหนึ่งรายการ ซึ่งระบุด้วย _Cloud Resource Names_ (CRNs) เพื่อเข้าถึงบริการ IBM Quantum นอกจากนี้ ยังมี API token เฉพาะ (เรียกอีกชื่อว่า key) ที่กำหนดให้กับแต่ละบัญชี เพื่อให้เข้าถึง IBM Quantum จาก Qiskit ได้ สำหรับคำแนะนำในการเลือก instance เฉพาะ ดูที่ [เชื่อมต่อกับ instance](./instances#connect-instance)

<span id="cloud"></span>
## ตั้งค่าเพื่อใช้งาน IBM Cloud {#set-up-to-use-ibm-cloud}

### ก่อนเริ่มต้น {#before-you-begin}

1. ถ้ายังไม่มี ให้ตั้งค่า [บัญชี IBM Cloud](https://quantum.cloud.ibm.com/registration)
   <Admonition type="note">
   คุณสามารถเป็นสมาชิกของหลายบัญชี IBM Cloud พร้อมกันได้ สามารถเข้าถึงบัญชี IBM Cloud ใดก็ได้ตลอดเวลาผ่าน account switcher ในส่วน header ของ [IBM Quantum Platform](https://quantum.cloud.ibm.com)
   </Admonition>
1. ล็อกอินเข้า [IBM Quantum Platform](https://quantum.cloud.ibm.com) ด้วย IBMid หรือบัญชี Google ถ้ายังไม่มี ระบบจะพาผ่านขั้นตอนการสร้าง (ข้อมูลล็อกอินเหล่านี้ไม่ใช่ข้อมูลบัญชี IBM Cloud)
1. ตรวจสอบให้แน่ใจว่าเลือกบัญชีและภูมิภาคที่ถูกต้องใน account switcher ในส่วน header ดังที่แสดงในภาพต่อไปนี้

    ภูมิภาคควบคุมว่างานของคุณจะรันที่ไหนและข้อมูลงานจะถูกเก็บไว้ที่ใด คุณเข้าถึงภูมิภาคใดก็ได้โดยใช้ API key เดียวกัน แต่จะเห็นและเข้าถึงได้เฉพาะ instance ที่สร้างขึ้นในภูมิภาคที่คุณล็อกอินอยู่เท่านั้น

    ![แสดง header ของ IBM Quantum Platform ตัวสลับบัญชีอยู่ทางขวาของแถบค้นหา ตัวสลับภูมิภาคอยู่ทางขวาของตัวสลับบัญชี](/docs/images/guides/cloud-setup/IQP-Header.svg 'IBM Quantum Platform header')

### 1. สร้าง instance {#1-create-an-instance}

คุณสร้าง instance ได้จากหน้า [Instances](https://quantum.cloud.ibm.com/instances) ของ IBM Quantum Platform ดูคำแนะนำโดยละเอียดได้ที่ [สร้าง instance](/guides/instances#add-instance)

### 2. ทางเลือก: บันทึกข้อมูลการเข้าถึง {#2-optional-save-your-access-credentials}

ถ้าใช้งาน Python environment ที่น่าเชื่อถือ เช่น แล็ปท็อปส่วนตัว แนะนำให้ใช้เมธอด `save_account()` เพื่อ [บันทึกข้อมูลล็อกอินไว้ในเครื่อง](/guides/save-credentials)

<span id="cloud-save"></span>
### 3. เชื่อมต่อ Qiskit กับ service instance ของคุณ {#3-connect-qiskit-with-your-service-instance}

- [เริ่มต้นใช้งาน service](/guides/initialize-account) ใน **Python environment ที่น่าเชื่อถือ**
- [เริ่มต้นใช้งาน service ใน **environment ที่ไม่น่าเชื่อถือ**](/guides/cloud-setup-untrusted) (เช่น คอมพิวเตอร์สาธารณะ)

<Admonition type="note" title="หมายเหตุ">
   * ทำตาม [คำแนะนำเหล่านี้](/guides/cloud-setup-rest-api) ถ้าต้องการเชื่อมต่อผ่าน REST API แทน Qiskit
   * ถ้าจำเป็น ใช้ [ข้อมูลนี้](/guides/quickstart-steps-org#firewall) เพื่อกำหนดค่า firewall ให้เปิดการเข้าถึง IBM Quantum API endpoints
</Admonition>

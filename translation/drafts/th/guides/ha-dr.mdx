# ทำความเข้าใจ high availability และ disaster recovery สำหรับ Qiskit Runtime

High availability (HA) คือความสามารถของบริการในการทำงานต่อเนื่องและเข้าถึงได้แม้จะเกิดความล้มเหลวที่ไม่คาดคิด ส่วน disaster recovery คือกระบวนการกู้คืน service instance ให้กลับมาทำงานได้ตามปกติ ผู้ใช้ Qiskit Runtime ไม่มีหน้าที่รับผิดชอบในการวางแผนหรือดำเนินการด้าน disaster recovery

Qiskit Runtime เป็น regional service ที่ปฏิบัติตาม [Service Level Objectives (SLO)](https://cloud.ibm.com/docs/resiliency?topic=resiliency-slo) ที่กำหนดไว้สำหรับทุกแพ็กเกจแบบชำระเงิน ดูข้อมูลเพิ่มเติมได้ที่ [Service Level Agreement (SLA)](https://www.ibm.com/support/customer/csol/terms/?id=i126-9268&lc=en) สำหรับ IBM Cloud&reg; และสำหรับข้อมูลเกี่ยวกับ IBM Cloud region และ data center ที่รองรับ Qiskit Runtime ดูได้ที่ [Service and infrastructure availability by location](https://cloud.ibm.com/docs/overview?topic=overview-services_region)

## สถาปัตยกรรม High Availability {#high-availability-architecture}

### ฟีเจอร์ High Availability {#high-availability-features}

บริการนี้ถูก provision กระจายไปยังสาม zone ในพื้นที่ multi-zone region โดยไม่มีจุดล้มเหลวเพียงจุดเดียว (single point of failure) คำขอ API จะถูกส่งผ่าน load balancer ไปยัง HA instance node แต่ละตัว ซึ่งอยู่ในคนละ zone กัน

หาก HA instance node หรือ zone ใดล้มเหลว บริการจะยังคงรองรับคำขอ API ผ่าน HA instance node ที่เหลือต่อไปได้ ไม่มีการลดคุณภาพบริการที่เห็นได้ชัดเมื่อเกิด zonal failure

บริการพร้อมใช้งานในภูมิภาคต่อไปนี้:

- Washington DC (us-east)
- Frankfurt (eu-de)

# ตั้งค่าขีดจำกัดการจัดสรรของ instance {#set-instance-allocation-limits}

บัญชีอาจมีเวลาที่จัดสรรไว้สำหรับแผน Premium หรือ Flex เนื่องจากแต่ละบัญชีสามารถมี instance หลายรายการในแผนเดียว การตั้งขีดจำกัดการจัดสรรจึงช่วยให้ควบคุมได้ว่า instance ใดบ้างที่อนุญาตให้ใช้เวลานั้น โปรดทราบว่าหากไม่ได้ตั้งการจัดสรรเป็นขีดจำกัด instance อาจใช้เวลาเกินกว่าที่จัดสรรไว้ได้

เวลาที่จัดสรรให้ instance จะถูกใช้โดยอัลกอริทึม fair-share เพื่อกำหนดลำดับความสำคัญในคิว โดยอิงจากการใช้งาน QPU ทั้งหมดในช่วงเวลา 28 วัน (สำหรับผู้ใช้แผน Premium) หรือตลอดระยะเวลาสมัครสมาชิก (สำหรับผู้ใช้แผน Flex)

การจัดสรรจะถูกกำหนดตอนสร้าง instance อย่างไรก็ตาม สามารถดูและเปลี่ยนแปลงเวลาที่จัดสรรได้ที่[หน้า instances](https://quantum.cloud.ibm.com/instances)

## สิ่งที่เกิดขึ้นเมื่อถึงขีดจำกัด {#what-happens-when-the-limit-is-reached}

สามารถตั้งค่าการจัดสรรเวลาเป็นขีดจำกัดสำหรับ instance ได้ตามต้องการ สิ่งต่อไปนี้จะเกิดขึ้นเมื่อ instance ถึงขีดจำกัด:

- workload ที่กำลังทำงานอยู่ยังคงทำงานต่อไป
- job ที่รอดำเนินการยังคงอยู่ในคิว
- ไม่มี workload ใดถูกยกเลิก
- ผู้ร่วมใช้งานสามารถส่ง workload ใหม่ได้ ซึ่งจะยังคงอยู่ในคิวจนกว่าจะมีเวลาเพิ่มเติม
- สำหรับแผน Flex instance จะไม่สามารถประมวลผล job เพิ่มเติมได้จนกว่าขีดจำกัดจะถูกเพิ่มขึ้น หรือสำหรับแผน Premium หลังจากเริ่มช่วงเวลาการใช้งานถัดไป

## ตั้งค่าที่ต่างกันสำหรับขีดจำกัดการจัดสรรและขีดจำกัดการใช้งาน {#set-different-values-for-the-allocation-limit-and-usage-limit}

เมื่อตั้งขีดจำกัดการจัดสรรสำหรับ instance ใน IBM Quantum Platform ค่าเดียวกันจะถูกกำหนดเป็นขีดจำกัดการใช้งานของ instance หากต้องการให้ผู้ใช้สามารถส่งและรัน job ต่อได้หลังจากถึงขีดจำกัดการจัดสรรของ instance สามารถตั้งค่าขีดจำกัดการใช้งานให้สูงขึ้นได้โดยใช้พารามิเตอร์ [usage_allocation_seconds](/guides/access-instances-platform-apis#parameters) ใน API [Update a resource instance](https://cloud.ibm.com/apidocs/resource-controller/resource-controller#update-resource-instance)

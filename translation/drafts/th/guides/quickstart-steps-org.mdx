# กำหนดค่า IBM Quantum Platform สำหรับองค์กร

ทำตามขั้นตอนเหล่านี้เพื่อตั้งค่า IBM Quantum Platform

<span id="iam-org"></span>
## 1. กำหนดค่า IAM {#1-configure-iam-settings}

ขั้นแรก ให้กำหนดค่าต่อไปนี้ในบัญชี IAM ของผู้ดูแลระบบ หากต้องการตรวจสอบและกำหนดค่าเหล่านี้ ให้ไปที่ [Manage → IAM → Settings](https://cloud.ibm.com/iam/settings)

* **User list visibility** กำหนดว่าผู้ใช้สามารถมองเห็นกันและกันได้หรือไม่ โดยไม่คำนึงถึงการมอบหมายโปรเจกต์ การตั้งค่า `enabled` จะจำกัดการมองเห็นผู้ใช้ กล่าวคือ ผู้ใช้ในบัญชีของคุณไม่สามารถมองเห็นกันได้ แม้ว่าจะเข้าถึง resource เดียวกันก็ตาม เลือกค่าที่เหมาะสมกับสภาพแวดล้อมของคุณ ดู [Controlling user visibility](https://cloud.ibm.com/docs/account?topic=account-iam-user-setting) สำหรับข้อมูลเพิ่มเติม
* **API key creation** ควบคุมว่าผู้ใช้สามารถสร้าง API key ได้หรือไม่ ใน IBM Quantum Platform มักใช้ API key หากมีการใช้ API key ให้เลือก `disabled` หรือจะให้สิทธิ์เฉพาะกับผู้ใช้แต่ละคนก็ได้

<span id="crt-rsc-grp-org"></span>
## 2. (ไม่บังคับ) สร้าง resource group {#2-optional-create-resource-groups}

ข้ามขั้นตอนนี้หากคุณใช้ access group ที่อ้างอิง IBM Quantum service instance โดยตรง

หากเลือกใช้ resource group ให้ไปที่ [Manage → Account → Resource groups (in Account resources)](https://cloud.ibm.com/account/resource-groups) แล้วคลิก **Create**

<span id="create-instance-org"></span>
## 3. สร้าง IBM Quantum service instance {#3-create-ibm-quantum-service-instances}

หากสร้าง IBM Quantum service instance ไว้แล้ว ให้ข้ามขั้นตอนนี้

* หากใช้ resource group ให้แน่ใจว่าสร้าง service instance บน resource group ที่ถูกต้อง
* ชื่อ service instance เช่น `QR-ml` จำเป็นสำหรับการอ้างอิง access group

ทำตามขั้นตอนใน [Set up to use IBM Cloud](/guides/cloud-setup#cloud) สำหรับคำแนะนำ

<span id="create-group-org"></span>
## 4. สร้าง access group สำหรับโปรเจกต์ {#4-create-access-groups-for-projects}

ขั้นแรก [สร้าง custom role](/guides/custom-roles#custom) ที่อนุญาตให้ผู้ใช้ดำเนินการเพื่อใช้งาน IBM Quantum service instance
จากนั้น [สร้าง access group](/guides/access-groups) สำหรับแต่ละโปรเจกต์ และกำหนดสิทธิ์ขั้นต่ำที่จำเป็นต้องใช้งาน resource ของโปรเจกต์ให้กับกลุ่มนั้น

<span id="setup-idp-org"></span>
## 5. ตั้งค่า ID provider และมอบหมายผู้ใช้ {#5-set-up-your-id-provider-and-assign-users}

ทำตามขั้นตอนในหัวข้อที่เหมาะสม ขึ้นอยู่กับ ID provider ที่เลือกใช้:

* [ใช้ IBM Cloud&reg; เป็น ID provider สำหรับผู้ใช้ที่มีบัญชี IBM Cloud](/guides/manage-cloud-users)
* [ใช้ IBM Cloud เป็น ID provider สำหรับผู้ใช้ที่ไม่มีบัญชี IBM Cloud](/guides/manage-appid)
* [ใช้ ID provider อื่นนอกจาก IBM Cloud สำหรับผู้ใช้ที่ไม่มีบัญชี IBM Cloud](/guides/manage-appid)
* [เชิญผู้ใช้และจัดการการเข้าถึงของพวกเขา](/guides/invite-and-manage-users)

<span id="firewall"></span>
## 6. กำหนดค่า firewall {#6-configure-your-firewall}

ในการกำหนดค่า firewall เพื่อเปิดใช้การเข้าถึง IBM Quantum API endpoint ให้เพิ่ม URL ต่อไปนี้ใน allowlist:

- IBM Cloud global services:
    ```text
    cloud.ibm.com
    iam.cloud.ibm.com
    api.global-search-tagging.cloud.ibm.com
    globalcatalog.cloud.ibm.com
    resource-controller.cloud.ibm.com
    ```
- IBM Quantum Qiskit Runtime services - ภูมิภาค us-east:
    ```text
    quantum.cloud.ibm.com
    s3.us-east.cloud-object-storage.appdomain.cloud
    ```
- IBM Quantum Qiskit Runtime services - ภูมิภาค eu-de:
    ```text
    eu-de.quantum.cloud.ibm.com
    s3.direct.eu-de.cloud-object-storage.appdomain.cloud
    ```
- IBM Quantum Qiskit Functions preview service:
    ```text
    qiskit-serverless.quantum.ibm.com
    ```

# ใช้ IBM Cloud Platform APIs เพื่อเข้าถึง instances

คู่มือนี้สำหรับทุกคนที่ใช้ IBM Cloud Platform APIs เพื่อเปิดใช้งาน อัปเดต หรือจัดการการเข้าถึง instances โดยให้รายละเอียดเกี่ยวกับ [พารามิเตอร์](#parameters) ที่สามารถตั้งค่าได้เมื่อเปิดใช้งาน instance รวมถึงชุด [extension fields](#extensions) ที่ส่งกลับมาเมื่อเข้าถึง instance

<Admonition type="note">
ดูเอกสาร IBM Cloud&reg; ResourceController API สำหรับข้อมูลเพิ่มเติม

หน้าที่เกี่ยวข้อง:
- [สร้าง (เปิดใช้งาน) resource instance ใหม่](https://cloud.ibm.com/apidocs/resource-controller/resource-controller#create-resource-instance)
- [อัปเดต resource instance](https://cloud.ibm.com/apidocs/resource-controller/resource-controller#update-resource-instance)
- [ดู resource instance](https://cloud.ibm.com/apidocs/resource-controller/resource-controller#get-resource-instance)
- [ดูรายการ resource instances ทั้งหมด](https://cloud.ibm.com/apidocs/resource-controller/resource-controller#list-resource-instances)

    เมื่อดูรายการ resource instances ทั้งหมด สามารถกรองให้แสดงเฉพาะ Qiskit Runtime instances ได้ด้วย `resource_id = "b6049020-80f4-11eb-a0f7-e35ec9b4054f"`

</Admonition>

## `parameters` {#parameters}

ใน body เมื่อเปิดใช้งานหรืออัปเดต instance:

- **instance_limit_seconds** (string หรือ null, `^[0-9]+$`): ขีดจำกัดที่ตั้งไว้บน instance สำหรับ Open Plan: ถ้าไม่ระบุ ค่าเริ่มต้นคือ 10 นาที
.
- **usage_allocation_seconds** (string หรือ null, `^[0-9]+$`): เวลาที่จัดสรรให้ instance ซึ่งใช้โดย fair-share scheduler เพื่อกำหนดลำดับความสำคัญในคิวตามการใช้งาน QPU ทั้งหมด (ไม่นำไปใช้กับ Pay-As-You-Go instances)
- **backends** (array ของรายการ backends (strings หรือ null) หรือ array ที่มี "ANY"): รายการชื่อ backends ที่ได้รับอนุญาตสำหรับ instance นี้ ค่าเริ่มต้น: `["ANY"]` หมายถึง backends ทั้งหมดที่มีในแผนนี้ `[]` หมายถึงไม่มี backends

## `extensions` {#extensions}

ในการตอบกลับเมื่อเปิดใช้งานหรือเมื่อดูรายละเอียดของ instance:

คุณสมบัติเพิ่มเติมของ instance ที่ให้บริการโดย service ซึ่งแสดงเป็นคู่ key-value คุณสมบัติเหล่านี้จะคงอยู่กับ instance ภายใต้ extensions property

- **instance_limit_seconds** (integer หรือ null, `>= 0`): ขีดจำกัดเวลาที่ตั้งไว้บน instance
- **usage_allocation_seconds** (integer หรือ null, `>= 0`): เวลาที่จัดสรรให้ instance ซึ่งใช้โดย fair-share scheduler เพื่อกำหนดลำดับความสำคัญในคิวตามการใช้งาน QPU ทั้งหมด
- **backends** (Required; array ของรายการ backends (strings) หรือ array ที่มี "ANY" (strings)): รายการชื่อ backends ที่ได้รับอนุญาตสำหรับ instance นี้ ค่าเริ่มต้น: `["ANY"]` หมายถึง backends ทั้งหมดพร้อมใช้งานในแผนนี้ `[]` หมายถึงไม่มี backends

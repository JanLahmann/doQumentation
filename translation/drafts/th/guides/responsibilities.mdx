# ทำความเข้าใจความรับผิดชอบของคุณในการใช้ Qiskit Runtime

ตรวจสอบหัวข้อต่อไปนี้เพื่อดูความรับผิดชอบเฉพาะของคุณและของ IBM&reg; เมื่อคุณใช้ Qiskit Runtime สำหรับข้อกำหนดการใช้งานโดยรวม ดู [Terms and Notices](https://cloud.ibm.com/docs/overview/terms-of-use?topic=overview-terms) สำหรับ IBM Cloud&reg;

## การจัดการเหตุการณ์และการดำเนินงาน {#incident-and-operations-management}

การจัดการเหตุการณ์และการดำเนินงานครอบคลุมงานต่าง ๆ เช่น การตรวจสอบ การจัดการเหตุการณ์ ความพร้อมใช้งานสูง การระบุปัญหา การกู้คืน และการสำรองและกู้คืนสถานะเต็มรูปแบบ

| งาน | ความรับผิดชอบของ IBM | ความรับผิดชอบของคุณ |
| --- | --- | --- |
|การจัดการสุขภาพโครงสร้างพื้นฐาน|<ul><li>ส่งมอบฟีเจอร์ใหม่ อัปเดต และแก้ไขบั๊กอย่างต่อเนื่องตามความจำเป็นอย่างโปร่งใส</li><li>แจ้งเตือนเกี่ยวกับการบำรุงรักษาตามแผน ประกาศด้านความปลอดภัย หรือการหยุดทำงานที่ไม่ได้วางแผน</li></ul> | <ul><li>[ตั้งค่าการรับอีเมล](https://cloud.ibm.com/user/notifications) เกี่ยวกับการแจ้งเตือนบริการ</li><li>ติดตามหน้า [IBM Cloud status](https://cloud.ibm.com/status) สำหรับประกาศทั่วไป</li></ul> |
|การตรวจสอบสุขภาพโครงสร้างพื้นฐานและการแจ้งเตือน| <ul><li>แก้ไขปัญหาสภาพแวดล้อมโครงสร้างพื้นฐานทั้งหมด</li><li>แจ้งลูกค้าเกี่ยวกับเหตุการณ์ที่เกี่ยวข้อง</li></ul> | <ul><li>ประเมินผลกระทบของแต่ละเหตุการณ์ที่รายงาน</li><li>ติดต่อ [IBM Support](/guides/support) ตามความจำเป็น</li></ul>|
|รักษา IBM Cloud SLA ด้านความพร้อมใช้งานสูง| ดำเนินการ Qiskit Runtime ตาม [IBM Cloud Public Service Level Agreements](https://cloud.ibm.com/docs/overview?topic=overview-slas) (SLAs) | ปฏิบัติตามแนวทางปฏิบัติที่ดีที่สุดด้านความพร้อมใช้งานสูง |

## การจัดการการเปลี่ยนแปลง {#change-management}

การจัดการการเปลี่ยนแปลงครอบคลุมงานต่าง ๆ เช่น การนำใช้งาน การกำหนดค่า การอัปเกรด การแพตช์ การเปลี่ยนแปลงการกำหนดค่า และการลบ

| งาน | ความรับผิดชอบของ IBM | ความรับผิดชอบของคุณ |
| --- | --- | --- |
| การอัปเกรดเวอร์ชันหลัก | ให้ความพร้อมและเครื่องมือสำหรับการอัปเดตเวอร์ชันหลัก | ย้ายไปยังเวอร์ชันที่รองรับก่อนสิ้นสุดการสนับสนุนสำหรับเวอร์ชันเก่า |
| การเปลี่ยนแปลงเวอร์ชัน API | กำหนดเวอร์ชัน REST APIs | ผสานรวมกับเวอร์ชัน REST APIs ที่ต้องการ |
| อัปเดต แก้ไข และฟีเจอร์ใหม่ | <ul><li>ให้การอัปเดตสม่ำเสมอ แก้ไขบั๊ก และฟีเจอร์ใหม่ อย่างโปร่งใสตามโมเดลการส่งมอบต่อเนื่อง</li><li>โพสต์การแจ้งเตือนสำหรับการเปลี่ยนแปลงที่ส่งผลต่อลูกค้า</li></ul> | <ul><li>ตั้งค่าการ[รับการแจ้งเตือนทางอีเมล](https://cloud.ibm.com/user/notifications) และติดตามหน้า [IBM Cloud status](https://cloud.ibm.com/status) สำหรับประกาศทั่วไป</li><li>อ่านการแจ้งเตือนที่ระบบสร้างขึ้นและ[ประกาศแพลตฟอร์ม](https://quantum.cloud.ibm.com/announcements) แล้วดำเนินการที่เหมาะสม</li></ul> |

## การจัดการข้อมูลระบุตัวตนและการเข้าถึง {#identity-and-access-management}

การจัดการข้อมูลระบุตัวตนและการเข้าถึงครอบคลุมงานต่าง ๆ เช่น การยืนยันตัวตน การอนุญาต การจัดการนโยบายการควบคุมการเข้าถึง และการอนุมัติ มอบ และเพิกถอนการเข้าถึง

| งาน | ความรับผิดชอบของ IBM | ความรับผิดชอบของคุณ |
| --- | --- | --- |
| การควบคุมข้อมูลระบุตัวตนและการเข้าถึง | ให้ความสามารถในการจัดการและจำกัดการเข้าถึงทรัพยากรผ่าน IBM Cloud | มอบ เพิกถอน และจัดการการเข้าถึงทรัพยากรผ่าน [Cloud Identity and Access Management](/guides/access-groups) (IAM) |
| การยืนยันตัวตนบริการ | ตรวจสอบว่ามีเพียงผู้ใช้ที่ผ่านการยืนยันตัวตนเท่านั้นที่มีสิทธิ์เข้าถึงบริการและทรัพยากร Qiskit Runtime ของคุณ | <ul><li>[ปกป้อง API key ของคุณ](/guides/cloud-setup-untrusted#initialize-the-service-in-an-untrusted-environment) จากการเข้าถึงโดยไม่ได้รับอนุญาต</li><li>หมุนเวียนและอัปเดต [API keys](https://cloud.ibm.com/iam/apikeys) ของคุณตามข้อกำหนดนโยบายความปลอดภัย</li></ul> |
| การสังเกตการณ์ | อนุญาตการผสานรวม IBM Cloud Activity Tracker Event Routing เพื่อตรวจสอบการกระทำของผู้ใช้ | ตั้งค่า [IBM Cloud Activity Tracker Event Routing](https://cloud.ibm.com/docs/vmware-service?topic=vmware-service-at_events) หรือฟังก์ชันอื่น ๆ เพื่อติดตามกิจกรรมของผู้ใช้ |

## ความปลอดภัยและการปฏิบัติตามข้อบังคับ {#security-and-regulation-compliance}

ความปลอดภัยและการปฏิบัติตามข้อบังคับครอบคลุมงานต่าง ๆ เช่น การดำเนินการควบคุมความปลอดภัยและการรับรองการปฏิบัติตามข้อกำหนด

| งาน | ความรับผิดชอบของ IBM | ความรับผิดชอบของคุณ |
| --- | --- | --- |
| ทั่วไป | <ul><li>รักษาการควบคุมที่สอดคล้องกับมาตรฐานการปฏิบัติตามข้อกำหนดในอุตสาหกรรมต่าง ๆ</li><li>ตรวจสอบ แยก และกู้คืนการทำงานของบริการ</li><li>ตรวจสอบและรายงานสุขภาพของอินเทอร์เฟซบริการ</li></ul> | ตรวจสอบว่าไม่มีการให้ข้อมูลส่วนบุคคลหรือข้อมูลที่อยู่ภายใต้การควบคุม |
| ช่องโหว่ | ตรวจสอบเพื่อตรวจจับปัญหาช่องโหว่และการปฏิบัติตามข้อกำหนดด้านความปลอดภัย | <ul><li>รับทราบข้อมูลเกี่ยวกับช่องโหว่และปัญหาด้านความปลอดภัยที่อาจเกิดขึ้นผ่านประกาศด้านความปลอดภัยที่ให้แนวทางการแก้ไขที่เป็นไปได้</li><li>ตรวจสอบ[ส่วนความปลอดภัยของ Qiskit GitHub repository](https://github.com/Qiskit/qiskit/security) ช่องโหว่ด้านความปลอดภัยทั้งหมดจะถูกโพสต์ที่นั่น</li><li>ตรวจสอบ[เว็บไซต์ IBM Cloud status](https://cloud.ibm.com/status) เพื่อค้นหาประกาศและประกาศด้านความปลอดภัยเกี่ยวกับเหตุการณ์สำคัญที่ส่งผลต่อแพลตฟอร์ม IBM Cloud โครงสร้างพื้นฐาน และบริการหลัก</li></ul> |
| การเข้ารหัส | เข้ารหัสข้อมูลเมื่อส่งผ่านเครือข่ายสาธารณะและภายในเครือข่ายศูนย์ข้อมูลส่วนตัวของ Cloud Service วิธีการเข้ารหัส เช่น HTTPS และ TLS ใช้เพื่อปกป้องข้อมูลในขณะเคลื่อนย้าย | ตรวจสอบตามความจำเป็นว่าการเชื่อมต่อของคุณมีการเข้ารหัสแบบ end-to-end |
| ความปลอดภัย | <ul><li>ตรวจสอบความปลอดภัยของข้อมูลบนดิสก์และข้อมูลในขณะเคลื่อนย้ายภายในโครงสร้างพื้นฐานของตน</li><li>ให้บริการที่ปลอดภัยซึ่งสอดคล้องกับมาตรฐานสำคัญ</li></ul> | จำกัด[การเข้าถึงของผู้ใช้](/guides/access-groups) ไปยังทรัพยากรและกลุ่มทรัพยากรที่เหมาะสม |

## การกู้คืนระบบจากภัยพิบัติ {#disaster-recovery}

| งาน | ความรับผิดชอบของ IBM | ความรับผิดชอบของคุณ |
| --- | --- | --- |
| ทั่วไป | กู้คืนหรือสร้างใหม่สภาพแวดล้อมการจัดสรรและทรัพยากรในพื้นที่ที่ได้รับผลกระทบ | <ul><li>ตรวจสอบสถานะของ instance และทรัพยากร</li><li>จัดสรร[อินสแตนซ์](/guides/instances) บริการ Qiskit Runtime ใหม่ในพื้นที่สำรอง หากมี</li></ul> |
| การกู้คืนบริการ | กู้คืนและฟื้นฟูความสามารถของบริการ Qiskit Runtime สู่การดำเนินงานปกติโดยอัตโนมัติ | ส่ง job ที่ได้รับผลกระทบอีกครั้งหลังจากการกู้คืน |
| การสำรองข้อมูลการกำหนดค่าของผู้ใช้ | IBM ไม่รับผิดชอบต่อการสำรองข้อมูลการกำหนดค่าของผู้ใช้ | สำรองข้อมูลการกำหนดค่าตามความจำเป็น |
| การสำรองและกู้คืน workload | IBM ไม่รับผิดชอบต่อการสำรองหรือกู้คืนข้อมูล workload ของผู้ใช้ | สำรองและกู้คืน workload ของคุณตามความจำเป็น |

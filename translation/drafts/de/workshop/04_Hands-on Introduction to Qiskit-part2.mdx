## 2.1 Map the problem to a quantum-native format

In einem Quantenprogramm sind *Quantenschaltkreise* das native Format zur Darstellung von Quantenanweisungen, und *Operatoren* repräsentieren die zu messenden Observablen. Wenn du einen Circuit erstellst, legst du normalerweise ein neues [`QuantumCircuit`](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.QuantumCircuit#quantumcircuit-class)-Objekt an und fügst ihm dann der Reihe nach Anweisungen hinzu.

Die folgende Code-Zelle erstellt einen Circuit, der den GHZ-Zustand erzeugt – einen Zustand, in dem drei Qubits vollständig miteinander verschränkt sind.

<div class="alert alert-info">

  Das Qiskit SDK verwendet die LSb-0-Bitnummerierung, bei der die $n^{th}$ Stelle den Wert $1 \ll n$ oder $2^n$ hat. Weitere Details findest du im Thema [Bit-ordering in the Qiskit SDK](https://docs.quantum.ibm.com/guides/bit-ordering).

</div>

```python
# Create a GHZ state circuit

qc = QuantumCircuit(3)
qc.h(0)
qc.cx(0,1)
qc.cx(0,2)
# Draw the circuit
qc.draw("mpl")
```

![Quantum circuit diagram](./_04_Hands-on Introduction to Qiskit_imgs/output_15.png)

Alle verfügbaren Operationen findest du in der Dokumentation unter [`QuantumCircuit`](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.QuantumCircuit#quantumcircuit-class).

Beim Erstellen von Quantenschaltkreisen musst du auch überlegen, welche Art von Daten nach der Ausführung zurückgegeben werden sollen. Qiskit bietet zwei Möglichkeiten zur Rückgabe von Daten: Du kannst eine Wahrscheinlichkeitsverteilung für eine Menge von Qubits erhalten, die du zu messen wählst, oder du kannst den Erwartungswert einer Observablen erhalten. Bereite deine Arbeitslast so vor, dass dein Circuit auf eine dieser beiden Arten mit [Qiskit-Primitiven](https://docs.quantum.ibm.com/guides/get-started-with-primitives) gemessen wird (im Detail in Schritt 3 erläutert).

Dieses Beispiel misst Erwartungswerte mithilfe des Untermoduls `qiskit.quantum_info`, das durch Operatoren (mathematische Objekte, die eine Aktion oder einen Prozess darstellen, der einen Quantenzustand verändert) angegeben wird. Die folgende Code-Zelle erstellt sechs Drei-Qubit-Pauli-Operatoren: `ZZZ`, `ZZX`, `ZII`, `XXI`, `ZZI` und `III`.

```python
# Set up six different observables.

observables_labels = ["ZZZ", "ZZX", "ZII", "XXI", "ZZI", "III"]

observables = [SparsePauliOp(label) for label in observables_labels]
print(observables)
```

```text
[SparsePauliOp(['ZZZ'],
              coeffs=[1.+0.j]), SparsePauliOp(['ZZX'],
              coeffs=[1.+0.j]), SparsePauliOp(['ZII'],
              coeffs=[1.+0.j]), SparsePauliOp(['XXI'],
              coeffs=[1.+0.j]), SparsePauliOp(['ZZI'],
              coeffs=[1.+0.j]), SparsePauliOp(['III'],
              coeffs=[1.+0.j])]
```

<div class="alert alert-info">

  Hier ist zum Beispiel der Operator `ZZI` eine Kurzschreibweise für das Tensorprodukt $Z\otimes Z\otimes I$, was bedeutet, dass Z auf Qubit 2 und Z auf Qubit 1 gemeinsam gemessen wird und Informationen über die Korrelation zwischen Qubit 2 und Qubit 1 gewonnen werden. Erwartungswerte wie dieser werden üblicherweise auch als $\langle Z_2 Z_1 \rangle$ geschrieben.

  Wenn der beobachtete Zustand der Drei-Qubit-GHZ-Zustand ist, sollte die Messung von $\langle Z_2 Z_1 \rangle$ gleich 1 sein.

</div>

## 2.2 Optimize the circuits and operators

Bei der Ausführung von Circuits auf einem Gerät ist es wichtig, die im Circuit enthaltenen Anweisungen zu optimieren und die Gesamttiefe (grob die Anzahl der Anweisungen) des Circuits zu minimieren. Dies stellt sicher, dass du die bestmöglichen Ergebnisse erzielst, indem die Auswirkungen von Fehlern und Rauschen reduziert werden. Außerdem müssen die Anweisungen des Circuits der [Instruction Set Architecture (ISA)](https://docs.quantum.ibm.com/guides/transpile#instruction-set-architecture) eines Backend-Geräts entsprechen und die Basis-Gates sowie die Qubit-Konnektivität des Geräts berücksichtigen.

Der folgende Code instanziiert ein echtes Gerät, an das ein Job übermittelt werden soll, und transformiert den Circuit und die Observablen so, dass sie der ISA dieses Backends entsprechen.
Wenn du deine Zugangsdaten noch nicht gespeichert hast, folge den Anweisungen [hier](https://docs.quantum.ibm.com/guides/setup-channel#iqp), um dich mit deinem API-Token zu authentifizieren.

```python
# Choose a real backend
service = QiskitRuntimeService(channel='ibm_quantum_platform',)

    f"Version: {backend.backend_version}\n"
    f"No. of qubits: {backend.num_qubits}\n"
    f"Processor type: {backend.processor_type}\n"
)
```

## 2.3 Execute using the quantum primitives

Quantencomputer können zufällige Ergebnisse liefern, daher sammelst du normalerweise eine Stichprobe der Ausgaben, indem du den Circuit viele Male ausführst. Du kannst den Wert der Observablen mithilfe der Klasse `Estimator` schätzen. `Estimator` ist eine von zwei [Primitiven](https://docs.quantum.ibm.com/guides/get-started-with-primitives); die andere ist `Sampler`, mit der Daten von einem Quantencomputer abgerufen werden können. Diese Objekte besitzen eine `run()`-Methode, die die Auswahl an Circuits, Observablen und Parametern (falls zutreffend) mithilfe eines [primitive unified bloc (PUB)](https://docs.quantum.ibm.com/guides/primitives#sampler) ausführt.
Wenn du diesen Code auf echter Quantenhardware ausführst, solltest du [Fehlerminderungs- und Fehlerunterdrückungstechniken](https://quantum.cloud.ibm.com/docs/en/guides/error-mitigation-and-suppression-techniques) anwenden, um das intrinsische Rauschen des Quantencomputers zu reduzieren.

```python
# Construct the Estimator instance.
estimator = Estimator(mode=backend)
estimator.options.resilience_level = 1
estimator.options.default_shots = 5000
```

Übermittle einen Job mithilfe der Estimator-Primitive.

```python
# One pub, with one circuit to run against six different observables.
job = estimator.run([(isa_circuit, mapped_observables)]) 

# Use the job ID to retrieve your job data later
print(f">>> Job ID: {job.job_id()}")
```

Nachdem ein Job übermittelt wurde, kannst du entweder warten, bis der Job in deiner aktuellen Python-Instanz abgeschlossen ist, oder die `job_id` verwenden, um die Daten zu einem späteren Zeitpunkt abzurufen. (Weitere Informationen findest du im [Abschnitt zum Abrufen von Jobs](https://docs.quantum.ibm.com/guides/monitor-job#retrieve-job-results-at-a-later-time).)

Nachdem der Job abgeschlossen ist, kannst du seine Ausgabe über das Attribut `result()` des Jobs untersuchen.

```python
# This is the result of the entire submission.  You submitted one Pub,
# so this contains one inner result (and some metadata of its own).
job_result = job.result()

# This is the result from our single pub, which had six observables,
# so contains information on all six.
pub_result = job.result()[0]
```

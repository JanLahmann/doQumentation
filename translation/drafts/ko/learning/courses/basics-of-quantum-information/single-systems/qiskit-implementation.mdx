## Python에서의 벡터와 행렬 {#vectors-and-matrices-in-python}

Qiskit은 Python 프로그래밍 언어를 사용하므로, Qiskit 자체를 설명하기 전에 Python에서의 행렬 및 벡터 연산에 대해 간략히 살펴보는 것이 도움이 됩니다.

Python에서는 다양한 수치 및 과학 계산 기능을 제공하는 `NumPy` 라이브러리의 `array` 클래스를 사용하여 행렬 및 벡터 연산을 수행할 수 있습니다.
다음 코드는 이 라이브러리를 불러오고, Qubit 상태 벡터 $\vert 0\rangle$과 $\vert 1\rangle$에 해당하는 두 열벡터 `ket0`과 `ket1`을 정의한 후, 두 벡터의 평균을 출력합니다.

```python
import numpy as np

ket0 = np.array([[1], [0]])
ket1 = np.array([[0], [1]])

print(ket0 / 2 + ket1 / 2)
```

```text
[[0.5]
 [0.5]]
```

`array`를 사용하여 연산을 나타내는 행렬도 만들 수 있습니다.

```python
M1 = np.array([[1, 1], [0, 0]])
M2 = np.array([[1, 0], [0, 1]])
M = M1 / 2 + M2 / 2
print(M)
```

```text
[[1.  0.5]
 [0.  0.5]]
```

이 과정의 각 강의에 포함된 모든 코드는 순서대로 실행되어야 합니다.
따라서 여기서는 `NumPy`를 다시 임포트할 필요가 없습니다. 이미 임포트했기 때문입니다.

행렬 곱셈(행렬-벡터 곱셈 포함)은 `NumPy`의 `matmul` 함수를 사용하여 수행할 수 있습니다.

```python
print(np.matmul(M1, ket1))
print(np.matmul(M1, M2))
print(np.matmul(M, M))
```

```text
[[1]
 [0]]
[[1 1]
 [0 0]]
[[1.   0.75]
 [0.   0.25]]
```

이 출력 형식은 시각적으로 다소 아쉬운 부분이 있습니다.
더 보기 좋은 출력이 필요한 경우, `qiskit.visualization` 모듈의 `array_to_latex` 함수를 사용하는 방법이 있습니다.
다음 코드에서는 Python의 범용 `display` 함수를 사용하고 있습니다.
이와 대조적으로, `print`의 구체적인 동작은 출력 대상에 따라 달라질 수 있습니다. 예를 들어 `NumPy`로 정의된 배열의 경우가 그러합니다.

```python
from qiskit.visualization import array_to_latex

display(array_to_latex(np.matmul(M1, ket1)))
display(array_to_latex(np.matmul(M1, M2)))
display(array_to_latex(np.matmul(M, M)))
```

$$

\begin{bmatrix}
1  \\
 0  \\
 \end{bmatrix}
$$

$$

\begin{bmatrix}
1 & 1  \\
 0 & 0  \\
 \end{bmatrix}
$$

$$

\begin{bmatrix}
1 & \frac{3}{4}  \\
 0 & \frac{1}{4}  \\
 \end{bmatrix}
$$

## 상태, 측정, 연산 {#states-measurements-and-operations}

Qiskit에는 상태, 측정, 연산을 생성하고 조작할 수 있는 여러 클래스가 포함되어 있습니다. 따라서 양자 상태, 측정, 연산을 시뮬레이션하는 데 필요한 모든 것을 Python으로 직접 프로그래밍할 필요가 없습니다.
시작에 도움이 될 몇 가지 예제를 아래에 소개합니다.

### 상태 벡터 정의 및 표시 {#define-and-display-state-vectors}

Qiskit의 `Statevector` 클래스는 양자 상태 벡터를 정의하고 조작하는 기능을 제공합니다.
다음 코드에서는 `Statevector` 클래스를 임포트하고 몇 가지 벡터를 정의합니다.
(제곱근 계산을 위해 `NumPy` 라이브러리의 `sqrt` 함수도 임포트합니다.
위에서 이미 `NumPy`를 임포트했으므로 `np.sqrt`로 호출할 수도 있습니다. 이는 이 특정 함수만 별도로 임포트하여 사용하는 다른 방식일 뿐입니다.)

```python
from qiskit.quantum_info import Statevector
from numpy import sqrt

u = Statevector([1 / sqrt(2), 1 / sqrt(2)])
v = Statevector([(1 + 2.0j) / 3, -2 / 3])
w = Statevector([1 / 3, 2 / 3])
```

`Statevector` 클래스에는 상태 벡터를 다양한 방식으로 표시하는 `draw` 메서드가 포함되어 있습니다. 일반 텍스트를 위한 `text`, 렌더링된 LaTeX를 위한 `latex`, 문서에 붙여넣기 편리한 LaTeX 코드를 위한 `latex_source` 등이 있습니다.
(LaTeX 코드를 가장 잘 표시하려면 `display` 대신 `print`를 사용하세요.)

```python
display(u.draw("text"))
display(u.draw("latex"))
print(u.draw("latex_source"))
```

```text
[0.70710678+0.j,0.70710678+0.j]
```

$$\frac{\sqrt{2}}{2} |0\rangle+\frac{\sqrt{2}}{2} |1\rangle$$

```text
\frac{\sqrt{2}}{2} |0\rangle+\frac{\sqrt{2}}{2} |1\rangle
```

`Statevector` 클래스에는 주어진 벡터가 유효한 양자 상태 벡터인지(즉, 유클리드 노름이 1인지) 확인하는 `is_valid` 메서드도 있습니다.

```python
display(u.is_valid())
display(w.is_valid())
```

```text
True
```

```text
False
```

### `Statevector`를 이용한 측정 시뮬레이션 {#simulating-measurements-using-statevector}

다음으로, `Statevector` 클래스의 `measure` 메서드를 사용하여 Qiskit에서 양자 상태의 측정을 시뮬레이션하는 방법을 살펴보겠습니다.
앞서 정의한 Qubit 상태 벡터 `v`를 그대로 사용합니다.

```python
display(v.draw("latex"))
```

$$(\frac{1}{3} + \frac{2 i}{3}) |0\rangle- \frac{2}{3} |1\rangle$$

`measure` 메서드를 실행하면 표준 기저 측정이 시뮬레이션됩니다.
이 메서드는 측정 결과와 측정 후 시스템의 새로운 양자 상태 벡터를 반환합니다.
(여기서는 내장 표현식이 포함된 형식화된 출력을 위해 `f` 접두사가 붙은 Python의 `print` 함수를 사용합니다.)

```python
outcome, state = v.measure()
print(f"Measured: {outcome}\nPost-measurement state:")
display(state.draw("latex"))
```

```text
Measured: 1
Post-measurement state:
```

$$- |1\rangle$$

측정 결과는 확률적이므로, 이 메서드는 여러 번 실행할 때 다른 결과를 반환할 수 있습니다.
위에서 정의한 벡터 `v`의 특정 예제에서, `measure` 메서드는 측정 후 양자 상태 벡터를 다음과 같이 정의합니다.

$$
\biggl(\frac{1 + 2i}{\sqrt{5}}\biggr) \vert 0\rangle
$$

($\vert 0\rangle$ 대신) 또는

$$
- \vert 1\rangle
$$

($\vert 1\rangle$ 대신)으로, 측정 결과에 따라 달라집니다.
두 경우 모두, $\vert 0\rangle$과 $\vert 1\rangle$의 대안들은 사실상 이 상태 벡터들과 동등합니다. 이들은 *전역 위상까지 동등(equivalent up to a global phase)*하다고 하며, 하나가 다른 하나에 단위 원 위의 복소수를 곱한 것과 같기 때문입니다.
이 문제는 [Quantum circuits](/learning/courses/basics-of-quantum-information/quantum-circuits/introduction) 강의에서 더 자세히 다루며, 지금은 무시해도 됩니다.

`Statevector`는 `measure` 메서드가 유효하지 않은 양자 상태 벡터에 적용될 경우 오류를 발생시킵니다.

`Statevector`에는 또한 `sample_counts` 메서드가 있어, 매번 상태의 새로운 복사본으로 시작하여 시스템에 대한 임의 횟수의 측정을 시뮬레이션할 수 있습니다. 예를 들어, 다음 코드는 벡터 `v`를 $1000$번 측정한 결과를 보여줍니다. (높은 확률로) 결과 $0$은 $9$번 중 약 $5$번(즉, $1000$번 시도 중 약 $556$번), 결과 $1$은 $9$번 중 약 $4$번(즉, $1000$번 중 약 $444$번) 나타납니다.
또한 아래 코드는 결과를 시각화하기 위해 `qiskit.visualization` 모듈의 `plot_histogram` 함수를 사용하는 방법도 보여줍니다.

```python
from qiskit.visualization import plot_histogram

statistics = v.sample_counts(1000)
plot_histogram(statistics)
```

![Output of the previous code cell](/learning/images/courses/basics-of-quantum-information/single-systems/qiskit-implementation/extracted-outputs/fde6ccf9-0.avif)

$1000$ 대신 다른 샘플 수를 사용하여 이 코드를 여러 번 직접 실행해 보면, 시도 횟수가 각 결과의 출현 횟수에 어떤 영향을 미치는지 직관적으로 이해하는 데 도움이 됩니다.
샘플 수가 많아질수록, 각 경우의 샘플 비율은 해당 확률에 점점 더 가까워질 가능성이 높습니다.
이 현상은 확률론에서 일반적으로 *큰 수의 법칙(law of large numbers)*으로 알려져 있습니다.

### `Operator`와 `Statevector`로 연산 수행하기 {#perform-operations-with-operator-and-statevector}

유니터리 연산은 다음 예제와 같이 `Operator` 클래스를 사용하여 Qiskit에서 정의할 수 있습니다.
이 클래스에는 `Statevector`와 유사한 인자를 가진 `draw` 메서드가 포함되어 있습니다.
`latex` 옵션은 `array_from_latex`와 동일한 결과를 생성합니다.

```python
from qiskit.quantum_info import Operator

Y = Operator([[0, -1.0j], [1.0j, 0]])
H = Operator([[1 / sqrt(2), 1 / sqrt(2)], [1 / sqrt(2), -1 / sqrt(2)]])
S = Operator([[1, 0], [0, 1.0j]])
T = Operator([[1, 0], [0, (1 + 1.0j) / sqrt(2)]])

display(T.draw("latex"))
```

$$

\begin{bmatrix}
1 & 0  \\
 0 & \frac{\sqrt{2}}{2} + \frac{\sqrt{2} i}{2}  \\
 \end{bmatrix}
$$

`evolve` 메서드를 사용하여 상태 벡터에 유니터리 연산을 적용할 수 있습니다.

```python
v = Statevector([1, 0])

v = v.evolve(H)
v = v.evolve(T)
v = v.evolve(H)
v = v.evolve(S)
v = v.evolve(Y)

display(v.draw("latex"))
```

$$(0.1464466094 - 0.3535533906 i) |0\rangle+(-0.3535533906 + 0.8535533906 i) |1\rangle$$

### Quantum Circuit 미리보기 {#a-preview-of-quantum-circuits}

Quantum Circuit은 이 과정의 세 번째 강의인 [Quantum circuits](/learning/courses/basics-of-quantum-information/quantum-circuits/introduction) 강의에서 공식적으로 소개됩니다. 그러나 Qiskit의 `QuantumCircuit` 클래스를 사용하여 Qubit 유니터리 연산을 합성하는 실험을 미리 해볼 수 있습니다.
특히, 단일 Qubit에 수행되는 유니터리 연산의 단순한 시퀀스인 Quantum Circuit을 다음과 같이 정의할 수 있습니다.

```python
from qiskit import QuantumCircuit

circuit = QuantumCircuit(1)

circuit.h(0)
circuit.t(0)
circuit.h(0)
circuit.s(0)
circuit.y(0)

display(circuit.draw(output="mpl"))
```

![Output of the previous code cell](/learning/images/courses/basics-of-quantum-information/single-systems/qiskit-implementation/extracted-outputs/68490946-0.avif)

여기서는 `QuantumCircuit` 클래스의 `draw` 메서드를 `mpl` 렌더러(Python 시각화 라이브러리인 `Matplotlib`의 약자)와 함께 사용합니다.
이 과정에서 Quantum Circuit에는 이 렌더러만 사용하겠지만, 텍스트 기반 및 LaTeX 기반 렌더러 등 다른 옵션도 있습니다.

연산은 다이어그램에서 왼쪽에서 오른쪽으로 순서대로 적용됩니다.
이 Circuit에 해당하는 유니터리 행렬을 얻는 편리한 방법은 `Operator` 클래스의 `from_circuit` 메서드를 사용하는 것입니다.

```python
display(Operator.from_circuit(circuit).draw("latex"))
```

$$

\begin{bmatrix}
0.1464466094 - 0.3535533906 i & 0.8535533906 + 0.3535533906 i  \\
 -0.3535533906 + 0.8535533906 i & 0.3535533906 + 0.1464466094 i  \\
 \end{bmatrix}
$$

시작 양자 상태 벡터를 초기화한 후, Circuit에서 설명하는 연산 시퀀스에 따라 해당 상태를 발전시킬 수도 있습니다.

```python
ket0 = Statevector([1, 0])
v = ket0.evolve(circuit)
display(v.draw("latex"))
```

$$(0.1464466094 - 0.3535533906 i) |0\rangle+(-0.3535533906 + 0.8535533906 i) |1\rangle$$

다음 코드는 위 Circuit에서 얻은 상태를 표준 기저 측정으로 4000번 측정하는 실험을 시뮬레이션합니다(매번 상태의 새로운 복사본 사용).

```python
statistics = v.sample_counts(4000)
display(plot_histogram(statistics))
```

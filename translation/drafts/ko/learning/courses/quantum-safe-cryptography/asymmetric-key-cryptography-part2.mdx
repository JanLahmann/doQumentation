## RSA

[RSA (Rivest-Shamir-Adleman)](https://dl.acm.org/doi/10.1145/359340.359342) 알고리즘은 최초의 공개 키 암호 시스템 중 하나로, 안전한 데이터 전송에 널리 사용됩니다. RSA는 단일 프레임워크 내에서 암호화, 복호화, 디지털 서명, 키 교환에 필요한 연산을 모두 제공하는 다목적 암호 시스템입니다.

이 섹션에서는 간단한 예제를 통해 RSA를 이용한 비대칭 키 암호화(AKC)를 설명합니다.

안전한 통신을 원하는 두 당사자, 앨리스(Alice)와 밥(Bob)이라는 표준 시나리오를 사용하겠습니다.

### RSA 알고리즘

기본 RSA 알고리즘은 키 생성, 키 배포, 암호화, 복호화의 네 가지 연산으로 구성됩니다:
1. **키 생성:**

   공개 키와 개인 키는 소수(prime number)와 관련된 수학적 원리에 따라 생성됩니다. 소수를 계산하는 것은 쉽지만, 역방향 연산은 어렵습니다.

   여기서는 다음과 같이 표기합니다:
    - 공개 키: $(e, n)$
    - 개인 키: $(d, n)$

   $n$은 공개 키와 개인 키 모두에 공통으로 사용되며, 모듈러스(modulus)라고 합니다. 이후에 사용해야 하므로 기억해 두세요.

<details>
    <summary>
   수학적 세부 사항
   </summary>

   - 서로 다른 두 소수 $p$와 $q$를 선택합니다.
     - 보안을 위해 무작위로 선택합니다.
     - 인수 분해를 더 어렵게 만들기 위해 크기는 비슷하되 자릿수는 몇 자리 차이가 나야 합니다.
     - 소수는 <DefinitionTooltip definition="An algorithm to determine whether a given number is prime. This is relatively easy computationally compared to factorization.">소수 판정법(primality test)</DefinitionTooltip>을 사용하여 효율적으로 선택할 수 있습니다.
   - $n = p*q$를 계산합니다.
     - $n$은 공개 키와 개인 키 모두의 모듈러스입니다.
   - <DefinitionTooltip definition="The totient is a number, or numbers, that are not greater than the given number, and are relatively prime to it, having no factors in common.">토티엔트(totient)</DefinitionTooltip> <DefinitionTooltip definition="The greek symbol, phi, often used to indicate Euler's totient.">$φ$</DefinitionTooltip>$(n) = (p-1)*(q-1)$을 계산합니다.
     - 토티엔트는 비밀로 유지되어야 하며, 일반적으로 키 생성 후 폐기됩니다.
   - $1 < e < $<DefinitionTooltip definition="The greek symbol, phi, often used to indicate Euler's totient.">$φ$</DefinitionTooltip>$(n)$이고 <DefinitionTooltip definition="Greatest Common Divisor - the largest positive whole number that can divide two or more of the given whole numbers">$gcd$</DefinitionTooltip>$(e, $<DefinitionTooltip definition="The greek symbol, phi, often used to indicate Euler's totient.">$φ$</DefinitionTooltip>$(n)) = 1$을 만족하는 정수 $e$를 선택합니다.
     - 즉, $e$와 <DefinitionTooltip definition="The greek symbol, phi, often used to indicate Euler's totient.">$φ$</DefinitionTooltip>$(n)$는 <DefinitionTooltip definition="Two numbers that have a greatest common divisor of 1. For example, 3 and 13 are coprime because only 1 can divide both of them.">서로소(coprime)</DefinitionTooltip>여야 합니다.
     - 이 수 $e$는 공개 키 <DefinitionTooltip definition="The number of times a number is multiplied by itself.">지수(exponent)</DefinitionTooltip>를 구성하며, 계산 효율성을 위해 일반적으로 작은 수로 선택합니다.
     - 소수 $65537 = 2^{16} + 1$이 자주 사용됩니다.
     - <DefinitionTooltip definition="Consistency, or equivalence ie. all behaving in the same way.">합동 관계(congruence relation)</DefinitionTooltip> $d*e ≡ 1 ($<DefinitionTooltip definition="The modulus is the remainder left over after dividing by the value specified. modulo is the operation of doing this.">$mod$</DefinitionTooltip><DefinitionTooltip definition="The greek symbol, phi, often used to indicate Euler's totient.">$φ$</DefinitionTooltip>$(n))$을 만족하는 $d$를 계산합니다.
       - 즉, $d$는 <DefinitionTooltip definition="The greek symbol, phi, often used to indicate Euler's totient.">$φ$</DefinitionTooltip>$(n)$에 대한 $e$의 <DefinitionTooltip definition="Also known as reciprocal, a number, which when multiplied by the number given, results in a value of 1.">곱셈 역원(multiplicative inverse)</DefinitionTooltip>입니다.
       - 이는 <DefinitionTooltip definition="An efficient way of calculating the greatest common divisor of two whole numbers. It improves on the original Euclidean algorithm.">확장 유클리드 알고리즘(extended Euclidean algorithm)</DefinitionTooltip>을 사용하면 더 효율적으로 계산할 수 있습니다.
       - 이 수 $d$가 개인 키 <DefinitionTooltip definition="The number of times a number is multiplied by itself.">지수(exponent)</DefinitionTooltip>입니다.
   - 공개 키는 $(e, n)$으로, 개인 키는 $(d, n)$으로 구성됩니다.

   </details>
2. **키 배포:**

   - 공개 키 $(e, n)$은 메시지를 보내고자 하는 사람들에게 공개됩니다.
   - 개인 키 $(d, n)$은 비밀로 유지됩니다.
3. **암호화:**

   - 앨리스는 밥에게 메시지 $M$을 보내고자 합니다. 여기서는 간단한 정수를 예로 사용합니다.
   - 앨리스는 밥의 공개 키 $(e, n)$을 사용하여 메시지를 암호문 $C$로 암호화합니다.

   <details>

   <summary>
   수학적 세부 사항
   </summary>

   - $M$은 $0 ≤ M < n$을 만족하는 <DefinitionTooltip definition="We consider an integer in this example as the simplest case. Any message can be converted into an integer based representation.">정수(integer)</DefinitionTooltip>입니다.
   - $C ≡ M^e (mod n)$, 여기서 $C$가 암호문입니다.

   </details>
4. **복호화:**

   - 밥은 암호문 $C$를 수신합니다.
   - 밥은 자신의 개인 키 $(d, n)$을 사용하여 암호문을 원래 메시지 $M$으로 복호화합니다.

   <details>
   <summary>
   수학적 세부 사항
   </summary>

   - $M ≡ C^d (mod n)$.

   </details>
이것이 RSA의 기본 개요입니다. 실제로는 동일한 평문이 서로 다른 암호문을 생성하도록 암호화 전에 평문 $M$에 [더 정교한 패딩 기법](https://en.wikipedia.org/w/index.php?title=Optimal_asymmetric_encryption_padding&oldid=1163481577)이 적용됩니다. 이는 RSA의 단순 구현에 대한 다양한 공격을 방지하고 <DefinitionTooltip definition="With only the ciphertext, and any relevant public information (such as public key for asymmetric key cryptography), and adversary cannot retrieve even partial information from the plaintext.">의미론적 보안(semantic security)</DefinitionTooltip>을 가능하게 합니다.
### Python으로 RSA 구현 예제

다음 코드 셀에서는 작은 정수를 사용하여 RSA 알고리즘의 간단한 예제를 보여주고, Python 라이브러리를 사용한 실용적인 키 배포 및 디지털 서명 응용을 시연합니다.

<Admonition type="note">
참고: 이 섹션에서는 Python 코드의 일부로 수학 계산을 자세히 보여줍니다.
</Admonition>

#### RSA 키 생성

작은 소수를 사용하는 RSA 알고리즘의 간단한 예를 단계별로 살펴보겠습니다.

두 정수의 <DefinitionTooltip definition="Greatest Common Divisor - the largest positive whole number that can divide two or more of the given whole numbers.">최대공약수(greatest common divisor)</DefinitionTooltip>를 계산할 수 있어야 합니다. 이는 두 정수가 <DefinitionTooltip definition="Two numbers that have a greatest common divisor of 1. For example, 3 and 13 are coprime because only 1 can divide both of them.">서로소(coprime)</DefinitionTooltip>인지 검사하는 데 필요합니다.

간단한 계산 방법 하나를 설명하겠지만, 큰 정수를 다룰 때는 Python의 `math.gcd` 함수를 사용하는 것이 훨씬 효율적입니다.

```python
# Added by doQumentation — required packages for this notebook
!pip install -q cryptography
```

```python
import math

# Example function to compute the gcd (greatest common divisor)
def gcd(a, b):
    if b == 0:
        return a
    else:
        return gcd(b, a % b)

# let's calculate some examples using algorithm
n1 = gcd(50, 10)
n2 = gcd(99, 33)
n3 = gcd(59, 9)

# do the same with the python library call

m1 = math.gcd(50, 10)
m2 = math.gcd(99, 33)
m3 = math.gcd(59, 9)

# Confirm they are the same
assert n1 == m1
assert n2 == m2
assert n3 == m3

# They are - print out the values for explanation
print("gcd(50,10) =", m1)
print("gcd(99,33) =", m2)
print("gcd(59,9) =", m3)
```

RSA 워크플로의 첫 번째 단계는 키 생성입니다. 이는 두 개의 소수를 선택하는 것으로 시작하며, 이 소수들은 키를 생성하는 주체가 비밀로 유지해야 합니다.

```python
# Choosing two prime numbers and keep them secret
p = 13
q = 19
print("The secret prime numbers p and q are:", p, q)
```

다음으로, 선택한 두 소수의 곱인 <DefinitionTooltip definition="The modulus is the remainder left over after dividing by the value specified. modulo is the operation of doing this.">모듈러스(modulus)</DefinitionTooltip> $n$을 계산합니다. 이 모듈러스는 공개 키의 일부로 공개됩니다.

```python
# Calculate n which is the modulus for both the public and private keys
n = p * q
print("modulus n (p*q)=", n)
```

다음으로 오일러 <DefinitionTooltip definition="The totient is a number, or numbers, that are not greater than the given number, and are relatively prime to it, having no factors in common.">토티엔트(totient)</DefinitionTooltip> 함수 <DefinitionTooltip definition="The greek symbol, phi, often used to indicate Euler's totient.">$\varphi(n)$</DefinitionTooltip>를 계산합니다. 이는 RSA에서 키를 결정하는 데 사용되는 <DefinitionTooltip definition="Given an integer $a$, and modulus $m$, the remainder from ax/m is 1. This is only the case for coprimes.">모듈러 곱셈 역원(modular multiplicative inverse)</DefinitionTooltip> 연산에 필요합니다. $phi$ 역시 비밀로 유지되며, 일반적으로 키 생성 후 폐기됩니다.

```python
# Compute Euler's totient function, φ(n) and keep it secret
phi = (p - 1) * (q - 1)
print("The secret Euler's function (totient) [phi(n)]:", phi)
```

이제 공개 키와 개인 키를 계산할 준비가 되었습니다. RSA에서 각 키는 두 정수의 튜플로 지정됩니다. 각 튜플의 첫 번째 항목은 고유한 정수이며, 두 번째 항목은 두 키에 공통으로 사용되는 모듈러스 $n$입니다.

공개 키의 첫 번째 항목은 1보다 크고 $phi$와 서로소인 임의의 정수가 될 수 있습니다. 두 정수의 <DefinitionTooltip definition="Greatest Common Divisor - the largest positive whole number that can divide two or more of the given whole numbers.">최대공약수(greatest common divisor)</DefinitionTooltip>가 1이면 서로소입니다. 따라서 `math.gcd` 함수를 사용하여 $phi$와 서로소인 정수 $e$를 찾습니다.

```python
# Choose an integer e such that e and φ(n) are coprime
e = 2
while e < phi:
    if math.gcd(e, phi) == 1:
        break
    else:
        e += 1
print("Public Key (e):", e)
```

개인 키에는 정수 $d$가 필요한데, 이는 $phi$에 대한 $e$의 <DefinitionTooltip definition="Also known as reciprocal, a number, which when multiplied by the number given, results in a value of 1.">곱셈 역원(multiplicative inverse)</DefinitionTooltip>입니다. 즉, <DefinitionTooltip definition="Consistency, or equivalence ie. all behaving in the same way.">합동 관계(congruency relation)</DefinitionTooltip> $d*e\equiv 1 \pmod{\varphi(n)}$을 만족해야 합니다. 소수가 작은 이 간단한 예제에서는 양의 정수를 순회하여 적절한 $d$를 찾을 수 있습니다. 실제 환경에서는 계산 효율이 높은 <DefinitionTooltip definition="An efficient way of calculating the greatest common divisor of two whole numbers. It improves on the original Euclidean algorithm.">확장 유클리드 알고리즘(extended Euclidean algorithm)</DefinitionTooltip>이 이 목적에 사용됩니다.

```python
# Compute a value for d such that (d * e) % φ(n) = 1
d = 1
while True:
    if (d * e) % phi == 1:
        break
    else:
        d += 1
print("Private Key (d):", d)
```

이제 튜플 $(e, n), (d, n)$을 구성합니다. 이것이 각각 공개 키와 개인 키가 됩니다. 공개 키는 공개되고, 개인 키는 비밀로 유지됩니다.

```python
# Public and Private Key pair
public = (e, n)
private = (d, n)

print(f"The Public key is {public} and Private Key is {private}")
```

RSA에서의 암호화와 복호화는 [모듈러 지수 연산(modular exponentiation operation)](https://en.wikipedia.org/w/index.php?title=Modular_exponentiation&oldid=1160764843)을 사용합니다. 또한 공개 키와 개인 키는 상호 보완적이어서, 어느 쪽으로 암호화한 메시지도 나머지 키로 복호화할 수 있습니다.

여기서는 공개 키 $(e,n)$으로 암호화하고 개인 키 $(d, n)$으로 복호화하는 경우를 각 연산에 대한 Python 함수를 정의하여 예시합니다.

그런 다음 정수 메시지 $msg$를 암호화하고 복호화합니다.

```python
# Encryption function
def encrypt(plain_text):
    return (plain_text**e) % n

# Decryption function
def decrypt(cipher_text):
    return (cipher_text**d) % n

# Simple message to encode
msg = 9

# encrypt then decrypt
enc_msg = encrypt(msg)
dec_msg = decrypt(enc_msg)

print("Original Message:", msg)
print("Encrypted Message:", enc_msg)
print("Decrypted Message:", dec_msg)
```

위에서 사용한 작은 정수들은 RSA 알고리즘의 핵심 아이디어를 쉽게 설명하는 데 유용하지만, 실제 애플리케이션에서 RSA는 매우 큰 정수를 사용해야 합니다. 예를 들어, 2048비트 RSA는 2048비트 길이의 <DefinitionTooltip definition="The modulus is the remainder left over after dividing by the value specified. modulo is the operation of doing this.">모듈러스(modulus)</DefinitionTooltip> $n$을 사용하며, 이에 해당하는 십진수 정수는 약 10$^616$ 수준입니다. 이처럼 거대한 숫자들이 RSA의 실용적인 보안을 위해 반드시 필요합니다.

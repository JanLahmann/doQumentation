## Diffie-Hellman 키 교환과 디지털 서명 알고리즘

이전 섹션에서는 소인수분해의 계산적 어려움에 기반한 RSA 암호 시스템을 다루었습니다. 여기서는 두 가지 널리 사용되는 비대칭 키 암호화 프로토콜인 [*Diffie-Hellman* 키 교환](https://en.wikipedia.org/w/index.php?title=Diffie%E2%80%93Hellman_key_exchange&oldid=1163912552)(DH)과 [*디지털 서명 알고리즘*](https://en.wikipedia.org/w/index.php?title=Digital_Signature_Algorithm&oldid=1148950522)(DSA)을 살펴보겠습니다. 두 프로토콜 모두 이산 로그 문제(DLP)라는 서로 다른 수학적 문제에 기반합니다.
### 이산 로그 문제

다음 방정식에서 $e$, $M$, $c$만 주어졌을 때 $a$를 구해야 합니다.

$a^e$ $mod$ $M = c$

모듈러 산술을 사용하기 때문에 고전 컴퓨터로는 이를 풀기 어렵다고 알려져 있으며, 따라서 암호화 알고리즘의 좋은 수학적 기반이 됩니다.

이것이 바로 [이산 로그 문제](https://en.wikipedia.org/w/index.php?title=Discrete_logarithm&oldid=1161792952)(DLP)로 알려진 것입니다.

### 이산 로그 문제의 수학적 세부 사항

DLP는 일반적으로 <DefinitionTooltip definition="Group where all elements are generated by repeatedly applying a single element, known as a generator. This process creates a cycle of elements that closes upon itself.">순환 군</DefinitionTooltip>의 맥락에서 다음과 같이 정의됩니다.

군 원소 $g \in G$에 의해 생성되는 [순환 군](https://en.wikipedia.org/w/index.php?title=Cyclic_group&oldid=1150017831) $G$를 고려하고, 임의의 원소 $h \in G$가 주어졌을 때 $h = g^{k}$를 만족하는 정수 $k$를 구합니다.

여기서 정수 $k \equiv log_{g}h$가 이산 로그입니다. $G$의 순환 성질은 모든 $h$에 대해 유효한 정수 $k$가 존재함을 보장합니다.

암호학에서는 소수 $p$를 법으로 하는 정수의 <DefinitionTooltip definition="The set of integers modulo a prime number, where the group operation is modular multiplication. Elements in the multiplicative group are chosen in such a way that they are coprime to the modulus, ensuring that each element has a unique multiplicative inverse within the group.">곱셈군</DefinitionTooltip>, 즉 $(\mathbb{Z}_p)^{\times}$로 표기되는 군에서의 DLP가 유용합니다. $(\mathbb{Z}_p)^{\times}$의 원소는 $p$와 서로소인 정수를 법 $p$로 나타낸 합동류입니다.

예를 들면:

$(\mathbb{Z}_5)^{\times} = \{[1],[2],[3],[4]\}~\mathrm{and}~(\mathbb{Z}_7)^{\times} = \{[1],[2],[3],[4],[5],[6]\} $

이 군들에서의 곱셈 연산 $(\times)$은 단순히 정수 곱셈 후 $p$로 나눈 나머지를 취하는 것이며, 정수 $k$에 의한 거듭제곱은 $k$번 반복 곱셈 후 $p$로 나눈 나머지를 취하는 것입니다.

$(\mathbb{Z}_7)^{\times}$에서 DLP의 예시를 살펴보겠습니다.

이 곱셈군은 두 개의 생성원 ${[3],[5]}$(원시근이라고도 함)를 가집니다. 생성원으로 $[5]$를 사용하겠습니다. 즉, 5의 연속적인 정수 거듭제곱을 사용하여 $(\mathbb{Z}_7)^{\times}$의 모든 원소를 생성합니다.

```python
#Generate elements of (Z_7)^{x} using generator [5].
g = 5
p = 7
print(f"Using generator {g}")
for k in range(3*p):
    print(f"{g}**{k} mod {p} ≡ {(g**k)%7}")
```

mod 7 산술에서 5를 연속적인 정수 거듭제곱으로 올리면 $(\mathbb{Z}_7)^{\times}$의 모든 원소가 정확히 한 번씩 나타나고, 그 후 주기 $p-1 =6$으로 무한 반복됨을 알 수 있습니다.

따라서 생성원 [5]를 사용한 $(\mathbb{Z}_7)^{\times}$에서의 DLP는 다음과 같습니다:

$ \mathrm{Given}~h \in (\mathbb{Z}_7)^{\times} \mathrm{, find~} k \mathrm{~such~that~} 5^{k} \equiv h~(mod~7)$.

위 Python 셀 출력에서 다음을 알 수 있습니다:

$h = 2 \implies k=4 \mathrm{~or~}  4 = log_5(2) (mod~7)$

$h = 6 \implies k=3 \mathrm{~or~}  3 = log_5(6) (mod~7)$

일반 실수 산술에서 거듭제곱은 <DefinitionTooltip definition="A behavior or relationship that consistently moves in a single direction without reversals; for example in a monotonically increasing function, values consistently increase.">단조</DefinitionTooltip> 함수이며, 임의의 밑수에 대한 로그 계산은 계산적으로 쉽습니다. 반면, 위의 간단한 $(\mathbb{Z}_7)^{\times}$ 예시에서 볼 수 있듯이, 모듈러 거듭제곱은 비단조적이며, 주기 $p-1$을 가지기는 하지만 그 외에는 매우 복잡합니다. 따라서 그 역연산인 이산 로그는 고전 컴퓨터에서 큰 $p$에 대해 비효율적입니다.

이 관찰이 다음 섹션에서 다루는 Diffie-Hellman(DH) 키 교환과 디지털 서명 알고리즘(DSA) 모두의 기반이 됩니다.

DLP는 다음과 같이 순환 부분군으로 확장될 수 있습니다:

-   위에서 정의한 $(\mathbb{Z}_p)^{\times}$와 소수 위수 $r$을 갖는 원소 $g \in (\mathbb{Z}_p)^{\times}$, 즉 $g^r \equiv 1 (~mod~p)$를 고려합니다.
-   $g$의 정수 거듭제곱의 집합: $\{g^k~(mod~p) | 1 \le k \le r\} = \langle g \rangle$은 군 위수 $r$을 갖는 $(\mathbb{Z}_p)^{\times}$의 순환 부분군입니다.
-   $h \in \\langle g \rangle$를 선택하고 $ g^a~(mod~p) = h$를 만족하는 $1 \le a \le r$을 구함으로써 $\langle g \rangle$에 대한 DLP를 정의할 수 있습니다.
### Diffie-Hellman 키 교환

1976년, [Whitfield Diffie와 Martin Hellman은 안전하지 않은 통신 채널을 통해 공유 비밀 키를 생성할 수 있는 키 교환 프로토콜을 제안했습니다](https://doi.org/10.1109/TIT.1976.1055638). 이 비밀 키는 이를 공유하는 당사자들이 대칭 암호화에 사용할 수 있습니다. 이 알고리즘은 DLP에 의존합니다.

*Figure 1. Diffie-Hellman 키 교환*

<details>

<summary>
Diffie-Hellman 키 교환의 수학적 세부 사항
</summary>

통신하는 두 당사자를 Alice와 Bob이라 할 때, 프로토콜은 다음과 같이 동작합니다:

- 먼저, Alice와 Bob은 큰 소수 $p$와 원시근 또는 생성원 $a$에 합의합니다.
- 그런 다음, Alice는 $1 \le k_A \le p-2$를 만족하는 비밀 지수 $k_A$를 무작위로 선택하고 $h_A = a^{k_A}~(mod~p)$를 계산합니다. $k_A, h_A$는 각각 Alice의 개인 키와 공개 키입니다.
- 다음으로, Bob은 $1 \le k_B \le p-2$를 만족하는 비밀 지수 $k_B$를 무작위로 선택하고 $h_B = a^{k_B}~(mod~p)$를 계산합니다. $k_B, h_B$는 각각 Bob의 개인 키와 공개 키입니다.
- 그런 다음, Alice는 Bob에게 $h_A$를, Bob은 Alice에게 $h_B$를 신뢰할 수 있지만 반드시 안전할 필요는 없는 채널을 통해 전송합니다.
- 그런 다음, Alice는 수신한 $h_B$를 사용하여 공유 비밀 키 $ \kappa = h_B^{k_A}~(mod~p)$를 계산합니다.
- 마지막으로, Bob은 수신한 $h_A$를 사용하여 공유 비밀 키 $ \kappa = h_A^{k_B}~(mod~p)$를 계산합니다.

이 프로토콜을 통해,

- Alice와 Bob은 $h_B^{k_A}~(mod~p) = (a^{k_B})^{k_A}~(mod~p) = a^{k_A k_B}~(mod~p) = (a^{k_A})^{k_B}~(mod~p) = h_A^{k_B}~(mod~p)$이기 때문에 동일한 비밀 키 $\kappa$를 갖게 됩니다.
- $h_A$ 또는 $h_B$를 가로챈 제3자는 $k_B$ 또는 $k_A$에 각각 접근할 수 없으므로 비밀 키 $\kappa$를 구성할 수 없습니다.
- 공개 정보 $a$, $p$, $h_A$, $h_B$를 사용하여 $k_A$ 또는 $k_B$를 추출하는 것은 $(\mathbb{Z}_p)^{\times}$에서 DLP를 푸는 것과 동일하므로 계산적으로 어렵습니다.

</details>
#### Python으로 구현한 Diffie-Hellman 프로토콜 예시

작은 소수를 사용하여 Python으로 DH 프로토콜의 간단한 예시를 살펴보겠습니다:

<Admonition type="note">
이 섹션에서는 Python 코드의 일부로 수학적 계산을 자세히 보여드리겠습니다.
</Admonition>

**1단계:** Alice와 Bob이 소수 $p$와 원시근 $a$에 합의합니다. $p=11, a=7$로 선택하겠습니다.

```python
# Step 1: Choose a prime `p` and a primitive root `a`
p = 11
a = 7

print(f"prime: {p}")
print(f"primitive root: {a}")
```

**2, 3단계:** Alice는 비밀 지수 $k_A$를 선택하고 $h_A = a^{k_A}~(mod~p)$를 계산합니다. 마찬가지로, Bob은 비밀 지수 $k_B$를 선택하고 $h_B = a^{k_B}~(mod~p)$를 계산합니다.

```python
k_A = 4  # Alice's private key
h_A = (a ** (k_A)) % p  # Alice's public key

print(f"Alice's private key is {k_A} and public key is {h_A}")

k_B = 8  # Bob's private key
h_B = (a ** (k_B)) % p  # Bob's public key

print(f"Bob's private key is {k_B} and public key is {h_B}")
```

**4단계:** 두 당사자가 공개 키 $h_A$와 $h_B$를 공개합니다.

**5, 6단계:** 각 당사자는 자신의 개인 키와 상대방의 공개 키를 결합하여 공유 비밀 키를 생성합니다.

```python
secret_key_alice = h_B**k_A % p
secret_key_bob = h_A**k_B % p
assert secret_key_alice == secret_key_bob
print(f"The shared secret key is: {secret_key_bob}")
```

Alice와 Bob은 이제 공유 비밀 키를 대칭 암호화에 사용할 수 있습니다.
#### Diffie-Hellman 키 교환의 보안

앞서 언급했듯이, DH의 보안은 큰 소수 $p$를 사용한 DLP 풀기의 계산적 어려움에 기반합니다. 일반적인 응용에서 NIST는 DH 키 교환에 2048비트 또는 3072비트 소수 정수를 권장하며, 이는 고전 컴퓨터로 DLP를 풀려는 시도에 대해 충분히 안전한 것으로 간주됩니다.

**중간자(MITM) 공격**: DH는 공유 비밀이 한 당사자의 개인 키와 상대방의 공개 키를 결합하는 대화형 방식이기 때문에 소위 *중간자(MITM)* 공격에 취약합니다.

<details>

<summary>
DH 보안과 MITM 공격의 수학적 세부 사항
</summary>

이 시나리오에서, Eve라는 제3자가 전송 중인 공개 키 $h_A, h_B$를 가로채고 $h_A$와 $h_B$ 각각을 자신의 공개 키 $h_E$로 대체한 후 각각 Bob과 Alice에게 전달합니다.

그러면 Alice는 Bob의 공개 키를 사용하고 있다고 생각하면서 $h_B$ 대신 $h_E$를 사용하여 공유 비밀을 생성하게 됩니다. 마찬가지로, Bob은 Alice의 공개 키를 사용하고 있다고 생각하면서 $h_A$ 대신 $h_E$를 사용하여 공유 비밀을 생성하게 됩니다.

$h_E$가 Alice(Bob)의 공유 비밀 생성에 사용되었기 때문에, Alice(Bob)가 암호화한 평문을 Eve가 복호화할 수 있습니다.

따라서 DH 키 교환은 일반적으로 각 당사자가 공유 비밀 생성에 인증된 공개 키를 사용하도록 보장하기 위해 디지털 서명 알고리즘과 함께 사용됩니다.

</details>
### 디지털 서명 알고리즘(DSA)

[RSA](https://en.wikipedia.org/w/index.php?title=RSA_(cryptosystem)&oldid=1163962255)와 같은 일반적인 암호 시스템이 디지털 서명 기능을 제공하기는 하지만, 1994년 NIST는 모듈러 거듭제곱과 이산 로그 문제에 기반한 전문화된 서명 방식을 디지털 서명의 연방 표준으로 채택했습니다. [*디지털 서명 알고리즘*](https://en.wikipedia.org/w/index.php?title=Digital_Signature_Algorithm&oldid=1148950522)(DSA)으로 알려진 이 방식은 네 가지 뚜렷한 단계로 구성됩니다:
1. **키 생성**:

   DSA 키는 다음으로부터 생성됩니다:
     - 특정 규칙을 충족하는 2개의 소수
       -   $p$ - 일반적으로 256비트 (이 길이를 $N$이라 부릅니다)
       -   $q$ - 일반적으로 3072비트 (이 길이를 $L$이라 부릅니다)
     - 길이 $L$의 문자열을 $N$으로 변환하는 암호화 해시 함수
     - 추가 매개변수 $g$ (아래 세부 사항 참조)

   이로부터 임의의 수 $x$를 개인 키로 선택하고, 공개 키 $y$를 계산할 수 있습니다.

   <details>
   <summary>
   키 생성의 수학적 세부 사항
   </summary>

   - **매개변수 생성:** 수학적으로, DSA는 소수 위수 q < p를 갖는 원소 $g$에 의해 생성되는 $(\mathbb{Z}_p)^{\times}$의 순환 부분군을 다룹니다. 따라서 DSA의 첫 번째 단계는 필요한 수학적 구조를 설정하기 위해 두 소수 p, q를 선택하는 것입니다.

     - $N$비트 소수 $q$를 선택합니다.
     - $p-1$이 $q$의 배수가 되는 $L$비트 소수 $p$를 선택합니다. $p$의 선택은 군 $(\mathbb{Z}_p)^{\times}$를 결정합니다.
     - 정수 $1 < h < p-1$을 무작위로 선택하고 $g = h^{(p-1)/q}~mod~p$를 계산합니다. 드물게 $g=1$이 되면 다른 h를 시도합니다.
     - $g^q~mod~p = 1$을 확인합니다. 따라서 g는 소수 위수 q를 갖는 $(\mathbb{Z}_p)^{\times}$의 순환 부분군 $\langle g \rangle$의 생성원입니다.

     매개변수 $(q, p, g)$는 알고리즘의 인스턴스를 지정하며 공개 정보입니다. 실제 응용에서는 일반적으로 $ N \sim 256$이고 $L \sim 3072$입니다.

     또한 이 프로토콜은 이진 $L$비트 문자열을 $N$비트 문자열로 매핑하는 암호화 해시 함수 $H : \{0,1\}^L \rightarrow \{0, 1\}^N$(예: SHA-256)이 필요합니다.

   - **키 쌍 생성:** 서명자는 자신의 측에서 키 쌍을 생성해야 합니다.

     - 임의의 비밀 정수 $ x \in \{1,2...q-1\}$을 선택합니다. $x$가 개인 키입니다.
     - $ y = g^{x}~mod~p$를 생성합니다. $y$가 서명자의 공개 키입니다.

   </details>
2. **키 배포:**

   서명을 검증하고자 하는 모든 사람과 다음 정보를 공유합니다.
   - 알고리즘을 설명하는 사용된 매개변수 $(p,q,g)$
   - 해시 함수 $H$
   - 공개 키 $y$
1. **서명:**

   - 서명자는 이제 메시지 $m$에 서명할 수 있습니다. 결과 서명은 $(r,s)$입니다.
   - 메시지와 서명 모두 수신자에게 전송됩니다.

   <details>
   <summary>
   메시지 서명의 수학적 세부 사항
   </summary>

   서명자는 다음과 같이 메시지 $m$에 서명합니다:

   - $\{1,2...q-1\}$에서 비밀 정수 k를 무작위로 선택합니다.
   - $r = (g^k~mod~p)~mod~q$를 계산합니다. 드물게 $r=0$이 되면 다른 무작위 k를 시도합니다.
   - $s = (k^{-1} (H(m) + xr))~mod~q$를 계산합니다. 드물게 $s=0$이 되면 다른 무작위 k를 시도합니다.
   - 서명은 튜플 $(r, s)$입니다.
   - 서명자는 메시지 $m$과 서명 $(r,s)$를 검증자에게 전송합니다.

   $r, s$ 모두 정수이므로, 법 $q$에서 서명 크기는 더 긴 $L$비트가 아닌 $N$비트 수준입니다.

   </details>
2. **검증:**

   수신자는 이제 발신자의 진위를 확인하고자 합니다.
   수신자는 공개 정보 $(q, p, g, H, y, (r,s), m)$에 접근할 수 있습니다.
   수신자는 발신자가 개인 키 $x$에 접근했음을 증명하는 계산을 실행할 수 있습니다.

   수신자는 서명자가 개인 키 $x$에 접근할 수 있는 사람인지 확인하고자 합니다.

   <details>
   <summary>
   DSA 검증 방식의 수학적 세부 사항
   </summary>

   - $0 \lt r \lt q$이고 $0 \lt s \lt q$임을 확인합니다. 즉, $r, s$가 유효한 법 $q$ 정수인지 확인합니다.
   - $w = s^{-1}~mod~q$를 계산합니다.
   - $u_1 = H(m)w~mod~q$를 계산합니다.
   - $u_2 = rw~mod~q$를 계산합니다.
   - $v = (g^{u_1}y^{u_2}~mod~p)~mod~q$를 계산합니다.
   - $v = r$이면 서명이 검증됩니다.

   정당한 서명에 대해 DSA 알고리즘의 정확성은 다음과 같이 쉽게 증명됩니다:

   - 서명자 측에서: $s = (k^{-1} (H(m) + xr))~mod~q \implies k = ((H(m) + xr)s^{-1})~mod~q $
   - 후자 등식의 우변을 고려하면, 검증자는 $s, q, m, H$가 공개 정보이므로 $s^{-1}, H(m)$을 계산할 수 있습니다.
   - 따라서 검증자는 $w = s^{-1}~mod~q$와 $u_1 = H(m)w~mod~q = H(m)s^{-1}~mod~q$를 계산합니다.
   - 검증자는 $r$도 공개이므로 $u_2 = rw~mod~q = rs^{-1}~mod~q$도 계산합니다.
   - $k = (u_1 + xu_2)~mod~q$임에 주목합니다.
   - 그러나 검증자는 $x$가 서명자의 개인 키이므로 이에 접근할 수 없고, $k$ 자체를 직접 계산할 수 없습니다.  - 대신 이 방식은 검증자가 $k$를 계산할 수 없더라도, 정당한 서명자의 경우 검증자가 서명자의 공개 키 $y = g^{x}~mod~p$의 도움으로 $(g^k~mod~p)~mod~q = r$을 재계산할 수 있어야 한다는 사실에 의존합니다.
   - 따라서 검증자는 $v = (g^{u_1}y^{u_2}~mod~p)~mod~q = (g^{u_1}g^{xu_2}~mod~p)mod~q = (g^{u_1+xu_2}~mod~p)mod~q = (g^k~mod~p)mod~q$를 계산합니다.
   - 마지막 등식은 $g$가 위수 $q$를 가지기 때문에 성립하며, 유효한 서명에 대해 $v = r$임을 확인합니다.

   </details>

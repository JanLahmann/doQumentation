### DSA의 보안 {#security-of-the-dsa}

고전 컴퓨팅 환경에서 DSA의 보안은 여러 요인에 의해 영향을 받을 수 있습니다.

1. **키 크기**: 언제나 그렇듯, 키 크기는 무차별 대입 공격에 대한 기본적인 보호를 제공합니다. DSA를 사용하는 현대 애플리케이션은 최소 2048비트의 키 크기를 사용합니다.

2. **$k$의 품질**: DSA에서 각 서명에는 고유하고 무작위이며 비밀스러운 $k$ 값이 필요합니다(위 내용 참조). $k$의 고유성, 엔트로피, 기밀성은 매우 중요하며, 이 중 어느 하나라도 취약하면 개인 키 $x$가 노출될 수 있습니다. $k$ 생성에 사용되는 난수 생성기 자체도 안전해야 합니다.

3. **해시 함수의 강도**: DSA는 서명 생성 및 검증 과정에서 해시 함수를 사용하므로, 취약한 해시 함수는 디지털 서명의 보안을 훼손할 수 있습니다. 예를 들어, DSA와 함께 SHA-1을 사용하는 것은 더 이상 권장되지 않으며, SHA-2 이상의 사용이 권고됩니다.

이 외에도 견고한 DSA 구현은 앞서 설명한 비대칭 키 암호화를 겨냥한 다른 유형의 공격에도 대비해야 합니다.

### Diffie-Hellman과 DSA를 활용한 인증된 키 교환 {#authenticated-key-exchange-with-diffie-hellman-and-dsa}

[전송 계층 보안(TLS)](https://en.wikipedia.org/w/index.php?title=Transport_Layer_Security&oldid=1155601531)과 같은 현대 네트워크 보안 프로토콜은 키 교환과 인증 기능을 결합하는 방식을 사용합니다. Diffie-Hellman의 맥락에서 인증은 중간자(MITM) 공격을 막기 위해 필요합니다. 아래 코드 셀에서는 Alice와 Bob이 Diffie-Hellman과 DSA 프로토콜을 결합하여 인증된 키 교환을 수행하는 예시를 보여줍니다. 이를 위해 `cryptography` Python 라이브러리를 사용합니다.

![그림 2. Diffie-Hellman과 DSA를 활용한 인증된 키 교환](/learning/images/courses/quantum-safe-cryptography/asymmetric-key-cryptography/auth_key_exchange.avif)

*그림 2. Diffie-Hellman과 DSA를 활용한 인증된 키 교환*

먼저, Alice와 Bob은 DH 파라미터 집합에 합의하고 각각 DH 공개 키-개인 키 쌍을 생성합니다.

```python
# import necessary library modules
from cryptography.hazmat.primitives import hashes
from cryptography.hazmat.primitives.asymmetric import dh, dsa

# Generate DH parameters
parameters = dh.generate_parameters(generator=2, key_size=2048)

# Generate Alice's DH private key and public key
alice_dh_private_key = parameters.generate_private_key()
alice_dh_public_key = alice_dh_private_key.public_key()

# Generate Bob's DH private key and public key
bob_dh_private_key = parameters.generate_private_key()
bob_dh_public_key = bob_dh_private_key.public_key()

print("Public and private keys generated for Bob and Alice")
```

DH 공개 키는 공개적으로 배포됩니다. 다음으로, Alice와 Bob은 각각 DSA에 사용할 별도의 키 쌍을 생성합니다. 이 키들은 교환할 DH 공개 키에 서명하는 데 사용됩니다.

```python
# Generate DSA keys for Alice and Bob
alice_dsa_private_key = dsa.generate_private_key(key_size=2048)
alice_dsa_public_key = alice_dsa_private_key.public_key()
bob_dsa_private_key = dsa.generate_private_key(key_size=2048)
bob_dsa_public_key = bob_dsa_private_key.public_key()

print("Additional key pair generated for signing")
```

Alice는 자신의 DSA 개인 키를 사용하여 자신의 DH 공개 키에 서명합니다.

```python
# Alice signs
alice_public_bytes = alice_dh_public_key.public_bytes(
    encoding=serialization.Encoding.PEM,
    format=serialization.PublicFormat.SubjectPublicKeyInfo,
)
alice_signature = alice_dsa_private_key.sign(alice_public_bytes, hashes.SHA256())

print("Alice signed public key")
```

마찬가지로, Bob도 자신의 DSA 개인 키를 사용하여 자신의 DH 공개 키에 서명합니다.

```python
bob_public_bytes = bob_dh_public_key.public_bytes(
    encoding=serialization.Encoding.PEM,
    format=serialization.PublicFormat.SubjectPublicKeyInfo,
)
bob_signature = bob_dsa_private_key.sign(bob_public_bytes, hashes.SHA256())

print("Bob signed public key")
```

이제 Alice와 Bob은 각자의 DH 공개 키와 해당 서명을 공개적으로 배포합니다. 상대방의 공개 키와 서명을 받은 각 당사자는 서명의 유효성을 검증합니다. 이를 통해 MITM 공격을 방지할 수 있는데, 예를 들어 Alice는 Bob의 DH 공개 키가 실제로 Bob에 의해 서명되었음을 알 수 있고, 그 반대도 마찬가지입니다.

```python
# Alice and Bob verify each other's DH public keys using DSA public keys
# An InvalidSignature exception will occur if they are not valid
alice_dsa_public_key.verify(alice_signature, alice_public_bytes, hashes.SHA256())
bob_dsa_public_key.verify(bob_signature, bob_public_bytes, hashes.SHA256())

print("Signatures are valid")
```

서명 검증 후, Alice와 Bob은 공유 비밀을 생성하여 키 교환을 완료합니다.

```python
# Perform key exchange
alice_shared_key = alice_dh_private_key.exchange(bob_dh_public_key)
bob_shared_key = bob_dh_private_key.exchange(alice_dh_public_key)

print("Shared secrets generated")
```

선택적으로, 추가 보안을 위해 Alice와 Bob은 *키 스트레칭* 기법을 사용하는 *HKDF*와 같은 특수 키 유도 함수를 통해 공유 비밀로부터 보다 안전한 대칭 키를 생성할 수 있습니다.

```python
# Derive a shared symmetric key using key-stretching
def derive_key(shared_key):
    return HKDF(
        algorithm=hashes.SHA256(),
        length=32,
        salt=None,
        info=None,
    ).derive(shared_key)

alice_symmetric_key = derive_key(alice_shared_key)
bob_symmetric_key = derive_key(bob_shared_key)

assert alice_symmetric_key == bob_symmetric_key
print("Keys checked to be the same")
```

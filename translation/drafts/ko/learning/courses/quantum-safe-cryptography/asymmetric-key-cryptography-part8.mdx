## 타원 곡선 암호학

[*타원 곡선 암호학(ECC)*](https://en.wikipedia.org/w/index.php?title=Elliptic-curve_cryptography&oldid=1160709993)은 [*타원 곡선*](https://en.wikipedia.org/w/index.php?title=Elliptic_curve&oldid=1164567664)의 수학에 기반하며, 비대칭 키 암호학에 강력한 접근 방식을 제공합니다. ECC는 RSA와 같은 방식과 유사한 수준의 보안을 더 짧은 키로 제공하는 것으로 알려져 있어 효율성이 높으며, [더 작은 키 크기로 인한 저장 공간과 연산 비용 절감이 매우 유리한](https://doi.org/10.1109/ACCESS.2018.2881444) 임베디드 시스템이나 모바일 기기와 같이 자원이 제한된 시스템에 특히 적합합니다.

### 타원 곡선 암호학의 기본 원리

타원 곡선은 일반적으로 $y^2 = x^3 + ax + b$ 형태이며, 몇 가지 흥미로운 성질을 가집니다.
* 수평 대칭을 이룹니다. 즉, 곡선 위의 임의의 점 $(x,y)$에 대해 그 반사점 $(x,-y)$ 역시 곡선 위에 있습니다.
* 수직이 아닌 직선은 곡선과 최대 세 점에서만 교차합니다.

![그림 1. 타원 곡선에서의 덧셈 연산과 점 배가 연산. 면 1은 <b>P + Q = -R</b>을 정의합니다. 면 2는 점 배가 <b>2Q = -P</b>를 나타냅니다. 면 3은 점의 덧셈 역원이 x축에 대한 반사, 즉 <b>P = -Q</b>임을 정의합니다.](/learning/images/courses/quantum-safe-cryptography/asymmetric-key-cryptography/ECCfig.avif)

*그림 1. 타원 곡선에서의 덧셈 연산과 점 배가 연산. 면 1은 <b>P + Q = -R</b>을 정의합니다. 면 2는 점 배가 <b>2Q = -P</b>를 나타냅니다. 면 3은 점의 덧셈 역원이 x축에 대한 반사, 즉 <b>P = -Q</b>임을 정의합니다.*

타원 곡선 암호학은 곡선 위의 점들에 일련의 산술 연산을 적용하는 방식을 활용합니다. 각 연산은 곡선 위의 새로운 점으로 이동하는 것과 같습니다. 이 과정은 한 방향으로는 따라가기 쉽고, 더 짧은 키 크기 덕분에 RSA 같은 다른 알고리즘보다 효율적이지만, 이를 역으로 되돌리는 것은 매우 어렵습니다. 바로 이 점이 암호학에 활용되는 이유입니다.

대수체(algebraic field) $K$ 위에서 정의된 [타원 곡선](https://en.wikipedia.org/w/index.php?title=Elliptic_curve&oldid=1164567664)은 일반적으로 $y^2 = x^3 + ax + b$ 형태의 수학 방정식으로 표현됩니다. 계수 $a, b \in K$이며, 꺾임이나 자기 교차가 없는 조건 하에서 점 $(x, y) \in K \otimes K$를 기술합니다.

타원 곡선의 성질을 이용하면 곡선 위의 점들에 대해 "덧셈"과 "스칼라 곱셈" 연산을 정의할 수 있습니다.

**덧셈**: $P$와 $Q$가 타원 곡선 위의 두 점일 때, $P + Q$는 다음과 같이 정의된 고유한 세 번째 점입니다. $P$와 $Q$를 지나는 직선을 그어 곡선과 다시 만나는 세 번째 점 $R$을 찾습니다. 그런 다음 $P + Q = -R$로 정의하는데, 이는 $R$을 $x$축에 대해 반사한 점입니다(아래 그림 참조). $P, Q$를 잇는 직선이 유한한 $(x, y)$에서 곡선과 교차하지 않는 경우, 무한원점 $\mathbf{O}$에서 교차한다고 말합니다.

타원 곡선 덧셈은 무한원점을 덧셈 항등원으로 하는 대수적 **군(group)** 성질을 만족합니다.

**배가와 스칼라 곱셈:** 스칼라 $2$에 의한 곱셈에 해당하는 점 배가(point doubling) 연산은 덧셈 연산에서 $P = Q$로 놓으면 얻을 수 있으며, 기하학적으로는 $P$에서의 접선을 이용합니다. 정수 $n$에 의한 일반적인 스칼라 곱셈 $nP = P + P + ...~n$번은 반복적인 배가와 덧셈으로 구할 수 있습니다.

### 타원 곡선 이산 로그 문제

타원 곡선 이산 로그 문제(ECDLP)는 앞서 다룬 이산 로그 문제와 여러 면에서 유사하며, 인수를 찾는 어려움을 기반으로 합니다.

[타원 곡선에서의 스칼라 곱셈](https://en.wikipedia.org/w/index.php?title=Elliptic_curve_point_multiplication&oldid=1145917191)을 이용하면 [타원 곡선 이산 로그 문제](https://en.wikipedia.org/w/index.php?title=Elliptic-curve_cryptography&oldid=1160709993)를 다음과 같이 정의할 수 있습니다.

타원 곡선 위의 점 $P, Q$가 $Q = nP$를 만족할 때, $n$을 구하라.

이 문제는 $n$이 충분히 클 경우 고전 컴퓨터에서 풀기 어렵다고 알려져 있으며, 암호학적 보안의 기반을 제공합니다.

<details>
<summary>
ECDLP의 수학적 설명
</summary>

타원 곡선 암호학은 주로 특정 대수적 **유한체(finite field)** 위에서 정식화된 ECDLP를 기반으로 합니다. 1999년 NIST는 ECC에서 사용하기 위한 열 가지 유한체를 권장했습니다.

 - 크기가 $\{192, 224, 256, 384, 521\}$ 비트인 소수 $p$에 대한 다섯 개의 **소수체(prime field)** $\mathbb {F} _{p}$.
 - $n \in \{163, 233, 283, 409, 571\}$에 대한 다섯 개의 **이진체(binary field)** $\mathbb {F} _{2^{n}}$.

위 설정을 바탕으로, **소수체**의 경우 ECC 기반 비대칭 키 암호 시스템은 다음과 같이 명시할 수 있습니다.

- NIST 권장 값 목록에서 $\mathbb {F} _{p}$를 지정하기 위한 $p$를 선택합니다.
- 타원 곡선의 매개변수 $a, b$를 선택합니다.
- 곡선 위에서 차수 $n$인 순환 부분군을 "생성"하는 **기저점(base point)** $G$를 선택합니다. 즉, $nG = \mathbf{O}$를 만족하는 최솟값 $n$을 구합니다.
- **보인수(cofactor)** $h = |E(\mathbb {F} _{p})|/n$을 계산합니다. 여기서 $|E(\mathbb {F} _{p})|$는 곡선 위의 점의 수이며, $h$는 일반적으로 작은 정수입니다.
- **도메인 매개변수** $(p, a, b, G, n, h)$를 이용해 다음과 같이 비대칭 키를 지정합니다.

    - 개인 키는 소수 $p$의 비트 수만큼 무작위로 선택된 정수 $d$입니다. 비밀로 유지해야 합니다.
    - 공개 키는 기저점 $G$를 개인 키 $d$로 "곱한" 결과이며, 일반적으로 $Q = dG$로 표기합니다.

$d$를 복원하는 것은 ECDLP를 푸는 것과 동일하며, $d$가 충분히 클 경우 풀기 어렵다고 가정합니다. 따라서 ECDLP는 앞서 다룬 $(\mathbb{Z}_p)^{\times}$ 위에서 정의된 Diffie-Hellman 및 DSA 방식과 직접적인 유사성 속에서, 키 교환 및 디지털 서명 방식의 기반이 됩니다.

</details>

### ECC를 이용한 키 교환

ECC의 주요 활용 중 하나는 [타원 곡선 Diffie-Hellman](https://en.wikipedia.org/w/index.php?title=Elliptic-curve_Diffie%E2%80%93Hellman&oldid=1160050860)(ECDH)으로 알려진 키 교환 프로토콜입니다. ECDH에서는 각 당사자가 개인-공개 키 쌍을 생성한 뒤 공개 키를 교환합니다. 그런 다음 각 당사자는 자신의 개인 키와 상대방의 공개 키를 사용하여 공유 비밀(shared secret)을 계산하며, 이를 대칭 암호화의 키로 활용할 수 있습니다.

타원 곡선 위에서 점 덧셈을 수행하고 개인 키로부터 공개 키를 도출하는 것은 비교적 쉽지만, 그 반대 과정인 공개 키로부터 개인 키를 도출하는 것은 계산적으로 불가능합니다. 이러한 "단방향" 특성이 ECDH 키 교환의 보안을 제공합니다.

아래에서는 Python과 `cryptography` 라이브러리를 사용하여 ECDH 키 교환을 수행하는 예시를 살펴봅니다.

```python
from cryptography.hazmat.primitives.asymmetric import ec
from cryptography.hazmat.primitives import hashes

# Each party generates a private key
private_key1 = ec.generate_private_key(ec.SECP384R1())
private_key2 = ec.generate_private_key(ec.SECP384R1())

# They exchange public keys
public_key1 = private_key1.public_key()
public_key2 = private_key2.public_key()

# Each party uses their own private key and the other party's public key
# to derive the shared secret
shared_key1 = private_key1.exchange(ec.ECDH(), public_key2)
shared_key2 = private_key2.exchange(ec.ECDH(), public_key1)

# The shared secrets are the same
assert shared_key1 == shared_key2
print("Keys checked to be the same")
```

### ECC를 이용한 디지털 서명

ECC는 [타원 곡선 디지털 서명 알고리즘](https://en.wikipedia.org/w/index.php?title=Elliptic_Curve_Digital_Signature_Algorithm&oldid=1162097661)(ECDSA)을 사용하여 디지털 서명을 생성하는 데에도 활용됩니다. ECDSA에서 서명자는 자신의 개인 키로 서명을 생성하고, 다른 사람들은 서명자의 공개 키로 서명을 검증할 수 있습니다. ECDH와 마찬가지로 ECDSA의 보안도 ECDLP에 의존합니다. 서명자의 개인 키에 접근하지 않고 서명을 위조하는 것은 계산적으로 불가능합니다.

아래는 Python과 `cryptography`를 사용한 간단한 서명 및 검증 트랜잭션 예시입니다.

```python
from cryptography.hazmat.primitives import hashes
from cryptography.hazmat.primitives.asymmetric import ec

# Generate a private key for use in the signature
private_key = ec.generate_private_key(ec.SECP384R1())

message = b"A message to be signed"

# Sign the message
signature = private_key.sign(message, ec.ECDSA(hashes.SHA256()))

# Anyone can verify the signature with the public key
public_key = private_key.public_key()

def check_ecdsa_signature(public_key, signature, message):
    try:
        public_key.verify(signature, message, ec.ECDSA(hashes.SHA256()))
        return True
    except InvalidSignature:
        return False

if check_ecdsa_signature(public_key, signature, message):
    print("The signature is valid.")
else:
    print("The signature is invalid.")
```

위 코드에서 서명 후 메시지를 수정하면 검증이 실패하므로, 메시지의 진위성과 무결성을 보장할 수 있습니다.

### ECDH와 ECDSA 공격

정수 이산 로그 문제와 유사하게, ECDLP는 고전 컴퓨터에서는 풀기 어렵지만 양자 컴퓨터에서는 다시 한번 Shor의 알고리즘을 통해 효율적으로 풀 수 있습니다. [ECDLP 사례에 대한 Shor 알고리즘의 일반화](https://doi.org/10.1007/978-3-030-89432-0_8)와 관련된 논의는 최근 문헌을 참고하시기 바랍니다.

## 요약

이 단원에서는 먼저 비대칭 키 암호학(AKC)의 주요 특성을 살펴보고, 비대칭 암호 시스템을 뒷받침하는 기본 보안 개념을 논의했습니다. 특히 AKC의 두 가지 주요 응용인 키 교환과 디지털 서명을 소개했으며, 두 가지 모두 현대 인터넷 기반 통신의 필수 요소입니다.

그런 다음 RSA 암호 시스템을 살펴봤습니다. RSA는 1970년대 이후 단순하고 다용도적인 프레임워크 안에서 키 교환과 디지털 서명을 가능하게 함으로써 현대 디지털 통신 보안에 큰 가치를 입증해 왔습니다. 고전 컴퓨팅에 대한 RSA의 암호학적 보안은 큰 정수의 인수분해 어려움에 기반하며, 실제 응용에서 사용되는 정수가 인수분해에 충분히 저항하기 위해 2048비트 범위의 키 크기를 요구합니다.

이어서 Diffie-Hellman(DH) 키 교환과 디지털 서명 알고리즘(DSA)을 살펴봤습니다. 이 방식들의 보안은 정수 이산 로그(DLP) 문제에 기반하며, 개인 키는 공개 키가 주어졌을 때 계산적으로 추출하기 어렵고 고전 컴퓨터에서는 다항 시간 내 풀이가 존재하지 않습니다. 무작위하고 고유한 키를 사용하면 공격에 대한 추가적인 보안이 확보됩니다. 정수 유한체와 타원 곡선 기반의 DH 및 DSA 프로토콜은 현재 [*TLS*](https://en.wikipedia.org/w/index.php?title=Transport_Layer_Security&oldid=1155601531), [*SSH*](https://en.wikipedia.org/w/index.php?title=Secure_Shell&oldid=1158549718) 등 다양한 현대 디지털 통신 프로토콜에서 광범위하게 활용되고 있습니다.

마지막으로 타원 곡선 암호학을 살펴봤습니다. ECC는 효율적인 키 크기와 강력한 보안 특성을 갖추고 있어, 현재 키 교환부터 디지털 서명까지 다양한 암호학적 응용에 탁월한 선택으로 자리 잡고 있습니다.

이러한 알고리즘들은 모두 양자 알고리즘의 공격에 노출되어 있습니다. 이들의 설계 전제가 되었던 수학적 문제에 대한 풀이가 양자 알고리즘으로 개발될 수 있기 때문입니다. 그 대표적인 예가 Shor의 알고리즘입니다. 따라서 이 알고리즘들은 양자 공격에 더 강인하면서도 고전 알고리즘에 대해서도 안전한, 양자 안전 비대칭 암호 시스템으로 대체되어야 합니다. 이 알고리즘들이 의존하는 수학적 문제는 양자 컴퓨터에 의해 효율적으로 풀릴 수 있으므로, 양자 공격은 물론 고전적 보안도 유지할 수 있는 양자 안전 비대칭 암호 시스템의 개발이 필요합니다.

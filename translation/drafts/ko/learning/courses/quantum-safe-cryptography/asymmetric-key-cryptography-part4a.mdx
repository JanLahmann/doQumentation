## Diffie-Hellman 키 교환과 디지털 서명 알고리즘

이전 절에서는 소인수분해의 계산적 어려움에 기반한 RSA 암호 시스템을 살펴보았습니다. 이 절에서는 두 가지 널리 사용되는 비대칭 키 암호화 프로토콜인 [*Diffie-Hellman* 키 교환](https://en.wikipedia.org/w/index.php?title=Diffie%E2%80%93Hellman_key_exchange&oldid=1163912552)(DH)과 [*디지털 서명 알고리즘*](https://en.wikipedia.org/w/index.php?title=Digital_Signature_Algorithm&oldid=1148950522)(DSA)을 살펴봅니다. 두 프로토콜은 모두 이산 로그 문제(DLP)라는 다른 수학적 문제를 기반으로 합니다.
### 이산 로그 문제

다음 방정식에서 $e$, $M$, $c$만 주어졌을 때 $a$를 구해야 합니다.

$a^e$ $mod$ $M = c$

모듈로 산술을 사용하기 때문에 고전 컴퓨터로는 이를 풀기가 어렵다고 알려져 있으며, 따라서 암호화 알고리즘의 좋은 수학적 기반이 됩니다.

이것이 바로 [이산 로그 문제](https://en.wikipedia.org/w/index.php?title=Discrete_logarithm&oldid=1161792952)(DLP)입니다.

### 이산 로그 문제의 수학적 세부 사항

DLP는 일반적으로 <DefinitionTooltip definition="Group where all elements are generated by repeatedly applying a single element, known as a generator. This process creates a cycle of elements that closes upon itself.">순환군</DefinitionTooltip>의 맥락에서 다음과 같이 정의됩니다.

군 원소 $g \in G$로 생성되는 [순환군](https://en.wikipedia.org/w/index.php?title=Cyclic_group&oldid=1150017831) $G$를 생각합시다. 임의의 원소 $h \in G$가 주어졌을 때 $h = g^{k}$를 만족하는 정수 $k$를 구하십시오.

여기서 정수 $k \equiv log_{g}h$가 이산 로그입니다. $G$의 순환 성질에 의해 모든 $h$에 대해 유효한 정수 $k$가 존재합니다.

암호학에서는 소수 $p$에 대한 정수의 <DefinitionTooltip definition="The set of integers modulo a prime number, where the group operation is modular multiplication. Elements in the multiplicative group are chosen in such a way that they are coprime to the modulus, ensuring that each element has a unique multiplicative inverse within the group.">곱셈군</DefinitionTooltip> $(\mathbb{Z}_p)^{\times}$ 위의 DLP가 유용합니다. $(\mathbb{Z}_p)^{\times}$의 원소는 $p$에 대해 서로소인 정수로 표지된 잉여류입니다.

예를 들면 다음과 같습니다.

$(\mathbb{Z}_5)^{\times} = \{[1],[2],[3],[4]\}~\mathrm{and}~(\mathbb{Z}_7)^{\times} = \{[1],[2],[3],[4],[5],[6]\} $

이 군들에서의 곱셈 $(\times)$ 연산은 단순히 일반 정수 곱셈 후 $p$에 대한 나머지 연산이며, 정수 $k$에 의한 거듭제곱은 $k$번 반복 곱셈 후 $p$에 대한 나머지 연산입니다.

$(\mathbb{Z}_7)^{\times}$ 위의 DLP 사례를 살펴보겠습니다.

이 곱셈군에는 원시근이라고도 불리는 두 개의 생성원 ${[3],[5]}$가 있습니다. 생성원으로 $[5]$를 사용합니다. 즉, 5의 연속적인 정수 거듭제곱을 이용해 $(\mathbb{Z}_7)^{\times}$의 모든 원소를 생성합니다.

```python
#Generate elements of (Z_7)^{x} using generator [5].
g = 5
p = 7
print(f"Using generator {g}")
for k in range(3*p):
    print(f"{g}**{k} mod {p} ≡ {(g**k)%7}")
```

모듈로 7 산술에서 5를 연속적인 정수 거듭제곱으로 올리면 주기 $p-1 = 6$으로 순환이 무한히 반복되기 전에 $(\mathbb{Z}_7)^{\times}$의 모든 원소가 정확히 한 번씩 나타나는 것을 알 수 있습니다.

따라서 생성원 [5]를 사용한 $(\mathbb{Z}_7)^{\times}$ 위의 DLP는 다음과 같습니다.

$ \mathrm{Given}~h \in (\mathbb{Z}_7)^{\times} \mathrm{, find~} k \mathrm{~such~that~} 5^{k} \equiv h~(mod~7)$.

위의 Python 셀 출력에서 다음을 확인할 수 있습니다.

$h = 2 \implies k=4 \mathrm{~or~}  4 = log_5(2) (mod~7)$

$h = 6 \implies k=3 \mathrm{~or~}  3 = log_5(6) (mod~7)$

일반 실수 산술에서 거듭제곱은 <DefinitionTooltip definition="A behavior or relationship that consistently moves in a single direction without reversals; for example in a monotonically increasing function, values consistently increase.">단조</DefinitionTooltip> 함수이며 어떤 밑에 대해서도 로그를 계산하는 것은 쉽습니다. 반면에 위의 간단한 $(\mathbb{Z}_7)^{\times}$ 예제에서 알 수 있듯이, 모듈러 거듭제곱은 비단조적이며 주기 $p-1$로 주기적이기는 하지만 그 외에는 매우 불규칙합니다. 따라서 그 역함수인 이산 로그를 계산하는 것은 대형 소수 $p$에 대해 고전 컴퓨터에서 비효율적입니다.

이 관찰이 다음 절에서 설명하는 Diffie-Hellman(DH) 키 교환과 디지털 서명 알고리즘(DSA) 모두의 기반이 됩니다.

DLP는 다음과 같이 순환 부분군으로 확장할 수 있습니다.

-   위에서 정의한 $(\mathbb{Z}_p)^{\times}$와 소수 위수 $r$을 가지는 원소 $g \in (\mathbb{Z}_p)^{\times}$, 즉 $g^r \equiv 1 (~mod~p)$를 만족하는 원소를 생각합니다.
-   $g$의 정수 거듭제곱의 집합: $\{g^k~(mod~p) | 1 \le k \le r\} = \langle g \rangle$은 군의 위수가 $r$인 $(\mathbb{Z}_p)^{\times}$의 순환 부분군입니다.
-   $h \in \\langle g \rangle$을 선택하고 $g^a~(mod~p) = h$를 만족하는 $1 \le a \le r$을 구함으로써 $\langle g \rangle$ 위의 DLP를 정의할 수 있습니다.
### Diffie-Hellman 키 교환

1976년, [Whitfield Diffie와 Martin Hellman은 안전하지 않은 통신 채널을 통해 공유 비밀 키를 생성하기 위한 키 교환 프로토콜을 제안했습니다](https://doi.org/10.1109/TIT.1976.1055638). 생성된 비밀 키는 이를 공유하는 당사자들이 대칭 암호화에 사용할 수 있습니다. 이 알고리즘은 DLP에 의존합니다.

*그림 1. Diffie-Hellman 키 교환*

<details>

<summary>
Diffie-Hellman 키 교환의 수학적 세부 사항
</summary>

통신하는 두 당사자를 Alice와 Bob이라 하면, 프로토콜은 다음과 같이 동작합니다.

- 먼저, Alice와 Bob은 큰 소수 $p$와 원시근(생성원) $a$에 합의합니다.
- 그런 다음, Alice는 $1 \le k_A \le p-2$를 만족하는 비밀 지수 $k_A$를 무작위로 선택하고 $h_A = a^{k_A}~(mod~p)$를 계산합니다. $k_A$와 $h_A$는 각각 Alice의 개인 키와 공개 키입니다.
- 다음으로, Bob은 $1 \le k_B \le p-2$를 만족하는 비밀 지수 $k_B$를 무작위로 선택하고 $h_B = a^{k_B}~(mod~p)$를 계산합니다. $k_B$와 $h_B$는 각각 Bob의 개인 키와 공개 키입니다.
- 그런 다음, Alice는 Bob에게 $h_A$를, Bob은 Alice에게 $h_B$를 신뢰할 수 있지만 반드시 안전하지는 않은 채널을 통해 전송합니다.
- 이어서 Alice는 받은 $h_B$를 사용하여 공유 비밀 키 $\kappa = h_B^{k_A}~(mod~p)$를 계산합니다.
- 마지막으로, Bob은 받은 $h_A$를 사용하여 공유 비밀 키 $\kappa = h_A^{k_B}~(mod~p)$를 계산합니다.

이 프로토콜을 통해,

- Alice와 Bob은 $h_B^{k_A}~(mod~p) = (a^{k_B})^{k_A}~(mod~p) = a^{k_A k_B}~(mod~p) = (a^{k_A})^{k_B}~(mod~p) = h_A^{k_B}~(mod~p)$이므로 동일한 비밀 키 $\kappa$를 갖게 됩니다.
- $h_A$ 또는 $h_B$를 가로챈 제3자는 $k_B$ 또는 $k_A$에 접근할 수 없기 때문에 비밀 키 $\kappa$를 생성할 수 없습니다.
- 공개 정보 $a$, $p$, $h_A$, $h_B$를 이용하여 $k_A$ 또는 $k_B$를 추출하는 것은 $(\mathbb{Z}_p)^{\times}$ 위의 DLP를 푸는 것과 동일하므로 계산적으로 어렵습니다.

</details>
#### Python으로 보는 Diffie-Hellman 프로토콜 예시

작은 소수를 사용하여 Python에서 DH 프로토콜의 간단한 예를 살펴보겠습니다.

<Admonition type="note">
이 절에서는 Python 코드의 일부로 수학 계산을 자세히 보여드립니다.
</Admonition>

**1단계:** Alice와 Bob은 소수 $p$와 원시근 $a$에 합의합니다. $p=11, a=7$로 선택하겠습니다.

```python
# Step 1: Choose a prime `p` and a primitive root `a`
p = 11
a = 7

print(f"prime: {p}")
print(f"primitive root: {a}")
```

**2, 3단계:** Alice는 비밀 지수 $k_A$를 선택하고 $h_A = a^{k_A}~(mod~p)$를 계산합니다. 마찬가지로, Bob은 비밀 지수 $k_B$를 선택하고 $h_B = a^{k_B}~(mod~p)$를 계산합니다.

```python
k_A = 4  # Alice's private key
h_A = (a ** (k_A)) % p  # Alice's public key

print(f"Alice's private key is {k_A} and public key is {h_A}")

k_B = 8  # Bob's private key
h_B = (a ** (k_B)) % p  # Bob's public key

print(f"Bob's private key is {k_B} and public key is {h_B}")
```

**4단계:** 두 당사자는 각자의 공개 키 $h_A$와 $h_B$를 공개합니다.

**5, 6단계:** 각 당사자는 자신의 개인 키와 상대방의 공개 키를 결합하여 공유 비밀 키를 생성합니다.

```python
secret_key_alice = h_B**k_A % p
secret_key_bob = h_A**k_B % p
assert secret_key_alice == secret_key_bob
print(f"The shared secret key is: {secret_key_bob}")
```

이제 Alice와 Bob은 공유 비밀 키를 대칭 암호화에 사용할 수 있습니다.
#### Diffie-Hellman 키 교환의 보안

앞서 언급한 바와 같이, DH의 보안은 큰 소수 $p$를 사용하여 DLP를 푸는 것이 계산적으로 어렵다는 전제에 기반합니다. 일반적인 응용에서 NIST는 고전 컴퓨터를 사용한 DLP 풀이 시도에 대해 충분히 안전한 것으로 간주되는 2048비트 또는 3072비트의 소수 정수를 DH 키 교환에 권장합니다.

**중간자(MITM) 공격**: DH는 공유 비밀이 한 당사자의 개인 키와 상대방의 공개 키를 결합하는 대화형 방식에 의존하므로, 이른바 *중간자(MITM, Man-in-the-middle)* 공격에 취약합니다.

<details>

<summary>
DH 보안 및 MITM 공격의 수학적 세부 사항
</summary>

이 시나리오에서 제3자—예를 들어 Eve—는 전송 중 공개 키 $h_A$, $h_B$를 가로채고, 각각 Alice와 Bob에게 전달하기 전에 자신의 공개 키 $h_E$로 대체합니다.

그러면 Alice는 Bob의 공개 키를 사용한다고 생각하면서 $h_B$ 대신 $h_E$를 사용하여 공유 비밀을 생성합니다. 마찬가지로 Bob은 Alice의 공개 키를 사용한다고 생각하면서 $h_A$ 대신 $h_E$를 사용하여 공유 비밀을 생성합니다.

$h_E$가 Alice(Bob)의 공유 비밀을 생성하는 데 사용되었기 때문에, Alice(Bob)가 암호화한 평문을 Eve가 복호화할 수 있습니다.

따라서 DH 키 교환은 일반적으로 각 당사자가 인증된 공개 키를 사용하여 공유 비밀을 생성하도록 보장하기 위해 디지털 서명 알고리즘과 함께 사용됩니다.

</details>

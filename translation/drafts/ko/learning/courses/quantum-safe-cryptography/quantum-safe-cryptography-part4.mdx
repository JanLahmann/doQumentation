## Module-LWE와 CRYSTALS 스위트

위에서 간략히 소개하고 임의의 격자(lattice)에 대해 일반적으로 성립하는 LWE(Learning with Errors) 문제는, 결과로 생성되는 암호 시스템의 효율성을 높이기 위해 소위 [**Ring-LWE**](https://en.wikipedia.org/w/index.php?title=Ring_learning_with_errors&oldid=1148731729) 프레임워크 안에서 대수적 *환(ring)* 으로 확장되었습니다. 그러나 Ring-LWE의 추가적인 대수 구조는 잠재적으로 악용될 수 있으며, 현재까지 알려진 악용 사례는 없지만 그 가능성은 남아 있습니다.

NIST의 양자 내성 암호 표준화 과정에서 선정된 4개의 최종 후보 중 두 가지 — [**CRYSTALS-Kyber**](https://pq-crystals.org/kyber/index.shtml) 키 캡슐화 메커니즘(KEM)과 [**CRYSTALS-Dilithium**](https://pq-crystals.org/dilithium/index.shtml) 디지털 서명 프로토콜 — 는 <DefinitionTooltip definition="주어진 모듈의 모든 부분 모듈의 집합입니다. 부분 모듈은 동일한 연산 아래에서 그 자체로 모듈을 형성하는 모듈의 부분 집합입니다. 모든 부분 모듈의 집합은 격자를 형성하며, 이는 임의의 두 원소가 유일한 최소 상한과 최대 하한을 갖는 부분 순서 집합입니다.">*모듈 격자(module lattice)*</DefinitionTooltip>와 관련된 [**Module-LWE**](https://eprint.iacr.org/2012/090)를 기반으로 합니다.

수학적 세부 사항은 해당 주제를 다루는 전문 강좌를 참고하시기 바라며, 여기서는 Module-LWE가 LWE와 Ring-LWE 사이의 절충안으로 여겨진다는 점만 언급합니다. Module-LWE는 LWE보다 높은 효율성을 제공하면서(Ring-LWE보다는 낮지만), 문제의 대수 구조에 덜 의존하기 때문에 Ring-LWE보다 높은 수준의 보안 보장을 제공합니다.

### 키 캡슐화 메커니즘과 CRYSTALS-Kyber
전통적인 비대칭 키 암호 시스템은 *키 교환* 및 *디지털 서명* 기능에 가장 많이 활용되므로, NIST 표준화 과정에서는 이 두 가지 기능에 대한 양자 내성 대안을 개발하고자 했습니다. [**CRYSTALS-Kyber**](https://pq-crystals.org/kyber/index.shtml) 프로토콜은 RSA와 같은 범용 암호화 방식이 아닌, 전용 [**키 캡슐화 메커니즘**](https://en.wikipedia.org/w/index.php?title=Key_encapsulation_mechanism&oldid=1110687965)(KEM)으로 설계되었습니다. 두 당사자 간에 공유 비밀을 설정하기 위해 Kyber를 사용하는 워크플로는 아래와 같습니다.

#### liboqs를 사용한 KEM 예제 (로컬 설정 필요)

<Admonition type="note">
이 예제는 추가적인 로컬 설치가 필요하며 현재 이 사이트에서는 실행할 수 없습니다. 아래의 출력 결과를 검토하거나, 이 예제를 직접 실행하고 싶다면 해당 셀을 적합한 로컬 환경에 복사하시기 바랍니다.**
</Admonition>

여기서는 양자 내성 암호의 프로토타이핑 및 실험을 위한 라이브러리를 제공하는 `liboqs` 오픈 소스 프로젝트가 제공하는 알고리즘을 활용합니다. 설정 및 테스트 방법은 아래 링크를 참고하고, 로컬 Python 환경에서 이 예제를 실행해 보세요.

- **Python 라이브러리:** liboqs에 대한 Python 언어 바인딩을 제공합니다 - https://github.com/open-quantum-safe/liboqs-Python
- **liboqs 구현체:** Python 라이브러리에 필요하며, 실제 알고리즘 구현을 제공하므로 빌드가 필요합니다 - https://github.com/open-quantum-safe/liboqs

Alice는 양자 내성 암호화 방식을 사용해 공유 비밀을 생성하고 Bob에게 안전하게 전달하고자 합니다. 이를 위해 liboqs가 제공하는 `Kyber512` KEM 알고리즘을 사용합니다.

먼저 관련 Python 모듈을 임포트합니다:

```python
import warnings
import oqs
from pprint import pprint

warnings.filterwarnings("ignore")
```

`liboqs`는 다양한 KEM 구현을 제공하며, 사용 가능한 알고리즘 목록은 다음과 같이 쉽게 확인할 수 있습니다.

```python
kems = oqs.get_enabled_kem_mechanisms()
pprint(kems, compact=True)
```

목록에서 `Kyber512`를 확인할 수 있으며, 이 예제에서 이를 사용합니다. Kyber를 이용한 키 교환은 다음 세 단계로 이루어집니다.
1. 키 생성
2. 캡슐화(Encapsulation)
3. 탈캡슐화(Decapsulation)

키 생성 단계에서 수신자인 Bob은 `Kyber512`를 사용해 비대칭 키 쌍을 생성하고 공개 키를 공개합니다.

```python
kemalg = "Kyber512"
bob = oqs.KeyEncapsulation(kemalg)
bob_public_key = bob.generate_keypair()
```

다음으로 **캡슐화** 단계에서, Bob의 공개 키를 수신한 Alice는 `Kyber512`와 Bob의 공개 키를 사용해 공유 비밀과 그에 대응하는 암호문을 생성합니다. 공유 비밀을 *캡슐화*한 암호문은 이후 Bob에게 전송됩니다.

```python
alice = oqs.KeyEncapsulation(kemalg)
ciphertext, shared_secret_alice = alice.encap_secret(bob_public_key)
```

마지막으로 **탈캡슐화** 단계에서, Bob은 자신의 개인 키를 사용해 암호문으로부터 공유 비밀을 복원합니다.

```python
shared_secret_bob = bob.decap_secret(ciphertext)
print("\nShared secretes coincide:", shared_secret_bob == shared_secret_alice)
```

RSA를 사용한 키 교환과 달리, 여기서는 불필요한 패딩이나 키 유도 함수가 필요하지 않습니다. Kyber가 KEM으로 설계되었기 때문에 매우 간단한 사용자 인터페이스를 제공하면서도 키 교환에 있어 최고 수준의 [**IND-CCA2**](https://en.wikipedia.org/w/index.php?title=Ciphertext_indistinguishability&oldid=1130147047) 보안을 달성합니다.

### 격자 기반 암호의 IND-CPA 및 IND-CCA 보안
대칭 키 암호에 관한 강좌에서 우리는 **의미론적 보안(semantic security)** 또는 **IND-CPA** 보안 개념을 소개했습니다. 이는 [**선택 평문 공격 하의 구별 불가능성**](https://en.wikipedia.org/w/index.php?title=Ciphertext_indistinguishability&oldid=1130147047)을 의미합니다.

AES와 같은 대칭 암호나 RSA와 같은 비대칭 암호 등 전통적인 암호 시스템은 결정론적 함수를 사용해 암호화 연산을 수행합니다. 즉, 주어진 평문과 암호화 키의 조합은 항상 동일한 암호문을 생성합니다. 이러한 결정론적 암호 시스템은 *선택 평문 공격*에 취약한데, 공격자가 결정론적 암호화 함수에 임의의 평문을 암호화하도록 요청해 정보를 추출할 수 있기 때문입니다.

이러한 맥락에서 IND-CPA 보안을 달성하려면, 암호화 시 *초기화 벡터* 또는 *패딩*을 통해 **추가적인 무작위성**을 도입해야 합니다. 예를 들어 AES는 무작위 초기화 벡터를 사용하는 [CBC(Cipher Block Chaining) 또는 GCM(Galois/Counter Mode) 운용 모드](https://en.wikipedia.org/w/index.php?title=Block_cipher_mode_of_operation&oldid=1154901199#cite_note-23)에서만 IND-CPA 보안이 보장됩니다. 마찬가지로 RSA에서는 IND-CPA 보안을 위해 [OAEP 패딩](https://en.wikipedia.org/w/index.php?title=Optimal_asymmetric_encryption_padding&oldid=1163481577)이 필요합니다.

이와 달리, 격자 기반 암호화 방식은 문제 정의 자체에 의해 본질적으로 무작위성을 내포합니다. 특히 위에서 설명한 LWE 기반 암호화 방식에는 두 가지 별개의 무작위 요소가 있습니다.
- **(1)** 분포 $\chi$에서 추출된 오류(또는 노이즈) $\varepsilon$
- **(2)** 메시지의 각 비트를 암호화할 때 사용되는 무작위 이진 벡터 $\mathbf{r} \in \{0,1\}^N$

오류 $\varepsilon$는 공개 키의 보안에 기여하여 비밀 키 $\mathbf{s}$를 추론하기 어렵게 만듭니다. 반면 무작위 이진 벡터 $\mathbf{r}$은 동일한 평문 비트를 반복 암호화할 때 비결정론적으로 만드는 데 필요한 본질적인 무작위성을 제공합니다. 따라서 LWE 기반 방식은 패딩과 같은 외부 메커니즘 없이도 IND-CPA 보안을 갖춘 것으로 간주됩니다.

현대 암호 시스템은 **선택 암호문 공격 하의 구별 불가능성**을 의미하는 소위 [**IND-CCA**](https://en.wikipedia.org/w/index.php?title=Ciphertext_indistinguishability&oldid=1130147047) 보안을 달성하는 것을 목표로 합니다. 이 설정에서 공격자는 정보를 추출해 이후 암호 시스템을 해독하려는 목적으로, 자신이 선택한 비자명한(non-trivial) 암호문 집합에 대한 복호화 결과를 얻을 수 있습니다. 이러한 능력이 있더라도 공격자가 암호화된 메시지를 구별할 때 무작위 추측보다 나은 결과를 낼 수 없다면 해당 방식은 IND-CCA 보안을 갖습니다. IND-CCA는 IND-CPA보다 강한 보안 개념이며 IND-CPA를 포함합니다.

**Kyber**와 같은 양자 내성 KEM은 IND-CCA 보안을 갖도록 설계되었습니다. 이는 두 단계를 통해 달성됩니다.
1. IND-CPA 보안을 갖는 공개 키 암호화(PKE) 방식을 정의합니다. Kyber의 경우 이 PKE는 **Module-LWE**에 기반합니다.
2. [Fujisaki-Okamoto 변환](https://doi.org/10.1007/s00145-011-9114-1)(FO)의 변형을 적용해 CCA 보안을 갖는 KEM을 얻습니다. FO 변환은 IND-CPA 보안을 갖는 암호화 방식을 IND-CCA 보안을 갖는 방식으로 변환하는 일반적인 방법입니다. 자세한 내용은 [원본 논문](https://doi.org/10.1007/s00145-011-9114-1)을 참고하시기 바랍니다.

**Kyber**와 **Dilithium**의 보안 특성과 다양한 프로그래밍 언어의 참조 구현에 대한 자세한 정보는 [CRYSTALS 스위트 문서](https://pq-crystals.org/)를 참고하시기 바랍니다.

## 하이브리드 암호

하이브리드 암호란 양자 내성 공개 키 알고리즘을 RSA 또는 타원 곡선과 같은 전통적인 공개 키 알고리즘과 결합하여, 결과적으로 기존의 전통적인 암호보다 보안 수준이 낮아지지 않도록 하는 방식을 말합니다.

이 방식은 다음 두 가지 문제를 해결하는 데 도움이 됩니다.

1. 기존 표준에서 특정 알고리즘의 사용을 의무화하거나, 해당 알고리즘이 어떤 방식으로 승인되도록 요구할 수 있습니다. 양자 내성을 추가하기 위해, 먼저 일반 알고리즘으로 암호화한 뒤 양자 내성 알고리즘으로 추가 보호함으로써 여전히 요구되는 표준을 충족할 수 있습니다.

2. 양자 알고리즘은 새로운 분야이므로, 실제로 안전하고 올바른지 확인하기 위한 추가 분석이 필요합니다. 예를 들어 NIST가 초기에 후보로 선정한 알고리즘 중 하나가 이후 해독되어 안전하지 않음이 증명된 사례가 있습니다. 이러한 위험을 완화하기 위해, 검증되고 잘 검토된 표준 알고리즘(양자 공격을 제외하면 안전한!)과 양자 내성 알고리즘을 함께 사용하여 암호화할 수 있습니다.

## 요약
비대칭 키 암호 시스템의 양자 내성 암호(QSC)로의 전환은, NP-중간(NP-intermediate) 문제의 고전적 난이도에 기반한 전통적인 비대칭 키 알고리즘을 해독할 수 있는 양자 컴퓨터의 도래가 예상되면서 필요해졌습니다. QSC는 양자 컴퓨터로도 일반적으로 효율적으로 풀 수 없는 NP-난해(NP-hard) 문제에 보안이 기반하기 때문에 양자 공격에 강인한 것으로 추정됩니다.

이러한 맥락에서, LWE(Learning with Errors) 문제와 같이 수학적 격자 이론에 뿌리를 둔 어려운 문제들이 QSC 표준화의 유력한 후보로 부상했습니다. 특히 모듈 격자에 기반한 CRYSTALS-Kyber 및 CRYSTALS-Dilithium 알고리즘은 RSA, 타원 곡선, Diffie-Hellman, DSA 등 널리 사용되는 비대칭 키 프로토콜을 대체할 단기적 대안으로 유망한 위치에 있습니다.

양자 내성 암호라는 복잡한 환경을 헤쳐 나가는 과정에서, 이러한 혁신적인 솔루션을 수용하는 것은 양자 시대에 우리의 디지털 상호작용의 무결성과 기밀성을 유지하는 데 핵심적인 역할을 할 것입니다.

### $ZZ$ 특성 맵

$ZZ$ 특성 맵(ZZFM)은 ZFM을 확장한 것으로, 2-Qubit 얽힘 게이트, 구체적으로는 $ZZ$-회전 게이트 $R_{ZZ}(\theta)$를 추가합니다. ZZFM은 ZFM과 달리 고전 컴퓨터에서 일반적으로 계산하기 어려울 것으로 추정됩니다.

$R_{ZZ}(\theta)$는 $ZZ$-상호작용을 구현하며, $\theta = \textstyle{\frac{1}{2}}\pi$일 때 최대로 얽힘 상태를 만듭니다. $R_{ZZ}(\theta)$는 2-Qubit에 작용하는 일련의 게이트로 분해할 수 있으며, 아래 Qiskit 코드는 [RZZ 게이트](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.library.RZZGate)와 `QuantumCircuit` 클래스 메서드 ```decompose```를 사용하여 이를 보여줍니다. 여기서는 데이터 벡터 $\vec{x}$의 단일 특성 $\vec{x}_k=\pi$를 인코딩합니다.

```python
qc = QuantumCircuit(2)
qc.rzz(pi, 0, 1)
qc.draw("mpl", scale=1)
```

![이전 코드 셀의 출력](/learning/images/courses/quantum-machine-learning/data-encoding/extracted-outputs/6c312a5f-91a5-499c-a391-efc73cd0e4e1-0.avif)

흔히 볼 수 있듯이, `.decompose()`를 사용하여 구성 게이트를 모두 확인하기 전까지는 단일 게이트 단위로 표현됩니다.

```python
qc.decompose().draw("mpl", scale=1)
```

![이전 코드 셀의 출력](/learning/images/courses/quantum-machine-learning/data-encoding/extracted-outputs/92062646-78f2-4dd6-82ad-7a543cb6a566-0.avif)

데이터는 두 번째 Qubit에서 위상 회전 $P(\theta) = e^{i\theta/2}R_Z(\theta)$으로 매핑됩니다. $R_{ZZ}(\theta)$ 게이트는 인코딩된 특성 값에 의해 결정되는 얽힘 정도로 두 Qubit을 얽히게 합니다.

완전한 ZZFM Circuit은 ZFM과 마찬가지로 하다마르 게이트와 위상 게이트로 구성되며, 그 뒤에 위에서 설명한 얽힘 연산이 이어집니다. ZZFM Circuit의 단일 반복은 다음과 같습니다:

$$
\mathscr{U}_{\text{ZZFM}}(\vec{x}) = U_{ZZ}(\vec{x})\big(P(\vec{x}_1)\otimes\ldots P(\vec{x}_k)\otimes\ldots P(\vec{x}_N)H^{\otimes N}\big)=U_{ZZ}(\vec{x})\left(\bigotimes_{k = 1}^N P(\vec{x}_k)\right)H^{\otimes N},
$$

여기서 $U_{ZZ}(\vec{x})$는 얽힘 방식에 따라 구성된 ZZ-게이트 레이어를 포함합니다. 여러 얽힘 방식은 아래 코드 블록에 나와 있습니다. $U_{ZZ}(\vec{x})$의 구조에는 얽히는 Qubit들의 데이터 특성을 결합하는 함수도 포함되어 있습니다. Qubit $p$와 $q$에 $R_{ZZ}$ 게이트가 적용된다고 가정해 봅시다. 위상 레이어에서 이 Qubit들은 각각 $\vec{x}_p$와 $\vec{x}_q$를 인코딩하는 위상 게이트를 갖습니다. $R_{ZZ,q,p}(\theta_{q,p})$의 인수 $\theta_{q,p}$는 단순히 이 특성 중 하나가 아니라, 흔히 $\phi$로 표기되는 함수(방위각과 혼동하지 않도록 주의)로 결정됩니다:
$$
\theta_{q,p} \rightarrow \phi(\vec{x}_q, \vec{x}_p) = 2(\pi-\vec{x}_q)(\pi-\vec{x}_p).
$$
아래 여러 예제에서 이를 확인할 수 있습니다. 다중 반복으로의 확장은 `z_feature_map`의 경우와 동일합니다:
$$
\mathscr{U}^{(r)}_{\text{ZZFM}}\left(\vec{x}\right)=\prod_{s=1}^{r}\left[U_{ZZ}(\vec{x})\left(\bigotimes_{k = 1}^N P(\vec{x}_k)\right)H^{\otimes N}\right].
$$
연산자가 복잡해졌으므로, 먼저 다음 코드를 사용하여 2-Qubit ZZFM과 1회 반복으로 데이터 벡터 $\vec{x} = (x_0, x_1)$을 인코딩해 봅시다:

```python
from qiskit.circuit.library import zz_feature_map

feature_dim = 2
zzfeature_map = zz_feature_map(
    feature_dimension=feature_dim, entanglement="linear", reps=1
)
zzfeature_map.decompose(reps=1).draw("mpl", scale=1)
```

![이전 코드 셀의 출력](/learning/images/courses/quantum-machine-learning/data-encoding/extracted-outputs/1ec2f2e1-b665-4dde-a223-35489f17c695-0.avif)

Qiskit에서는 기본적으로 특성 $(\vec{x}_1, \vec{x}_2)$이 매핑 함수 $\theta_{1,2} = \phi(\vec{x}_1, \vec{x}_2) = 2(\pi-\vec{x}_1)(\pi-\vec{x}_2)$에 의해 $R_{ZZ}(\theta)$로 함께 매핑됩니다. Qiskit은 사용자가 전처리 단계로서 함수 $\phi$(또는 $R_{ZZ}$ 게이트로 연결되는 Qubit 쌍의 집합 $S$에 대한 $\phi_S$)를 사용자 정의할 수 있도록 허용합니다.

4차원 데이터 벡터 $\vec{x} = (\vec{x}_1, \vec{x}_2, \vec{x}_3, \vec{x}_4)$로 이동하여 1회 반복의 4-Qubit ZZFM에 매핑하면, 다양한 Qubit 쌍에 대한 매핑 $\phi$를 확인하기 시작할 수 있습니다. 또한 "linear" 얽힘의 의미도 확인할 수 있습니다:

```python
feature_dim = 4
zzfeature_map = zz_feature_map(
    feature_dimension=feature_dim, entanglement="linear", reps=1
)
zzfeature_map.decompose().draw("mpl", scale=1)
```

선형 얽힘 방식에서는 이 Circuit의 인접한(번호 순서대로) Qubit 쌍이 얽힙니다. Qiskit에는 `circular` 및 `full`을 포함한 다른 내장 얽힘 방식도 있습니다.
### 파울리 특성 맵

파울리 특성 맵(PFM)은 ZFM과 ZZFM을 임의의 파울리 게이트를 사용하도록 일반화한 것입니다. 파울리 특성 맵은 앞의 두 특성 맵과 매우 유사한 형태를 취합니다. 벡터 $\vec{x}$의 $N$개 특성을 $r$회 반복 인코딩하면,

$$
\mathscr{U}_{\text{PFM}}(\vec{x}) = \prod_{s=1}^{r} U(\vec{x}) H^{\otimes n}.
$$

PFM에서 $U(\vec{x})$는 파울리 전개 유니터리 연산자로 일반화됩니다. 여기서는 지금까지 살펴본 특성 맵의 더 일반화된 형태를 제시합니다:

$$
U(\vec{x}) = \exp\left(i \sum_{S \in\mathcal{I}} \phi_S(\vec{x}) \prod_{i \in S} \sigma_i \right),
$$

여기서 $\sigma_i$는 파울리 연산자로 $\sigma_i \in {I,X,Y,Z}$입니다. $\mathcal{I}$는 단일 Qubit 게이트에 작용하는 Qubit 집합을 포함하여, 특성 맵에 의해 결정되는 모든 Qubit 연결성의 집합입니다. 즉, Qubit 0에 위상 게이트가 작용하고 Qubit 2와 3에 $R_{ZZ}$ 게이트가 작용하는 특성 맵의 경우, 집합 $\mathcal{I}$는 $\{\{0\},\{2,3\}\}$을 포함합니다. $S$는 해당 집합의 모든 원소를 순회합니다. 이전 특성 맵에서 함수 $\phi_S(\vec{x})$는 단일 Qubit 게이트 또는 2-Qubit 게이트 중 하나에만 관여했습니다. 여기서는 이를 일반적으로 정의합니다:
$$
\phi_S(\vec{x})=
    \begin{cases}
      x_i & \text{if } S= \{i\} \text{ (single-qubit)}\\
      \prod_{j\in{S}}(\pi-x_j) & \text{if } |S|\ge2 \text{ (multi-qubit)}\\
    \end{cases}
$$

문서에 대해서는 [Qiskit `Pauli feature map` 클래스 문서](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.library.PauliFeatureMap))를 참조하세요. ZZFM에서 연산자 $\sigma_i$는 $Z_i$로 제한됩니다.

위의 유니터리를 이해하는 한 가지 방법은 물리 시스템의 전파자와의 유사성을 통해서입니다. 위의 유니터리는 이징 모델과 유사한 해밀토니안 $\mathcal{H}$에 대한 유니터리 발전 연산자 $\exp(it\mathcal{H})$이며, 여기서 시간 매개변수 $t$가 진화를 구동하는 데이터 값으로 대체됩니다. 이 유니터리 연산자의 전개가 PFM Circuit을 구성합니다. $S$의 얽힘 연결성은 스핀 격자에서의 이징 결합으로 해석될 수 있습니다.
파울리 $Y$ 및 $XX$ 연산자가 이징형 상호작용을 나타내는 예를 살펴보겠습니다. Qiskit은 단일 및 $n$-Qubit 게이트를 선택하여 PFM을 인스턴스화할 수 있는 `pauli_feature_map` 클래스를 제공하며, 이 예에서는 파울리 문자열 `'Y'`와 `'XX'`로 전달합니다. 일반적으로 $n$은 단일 및 2-Qubit 상호작용에 대해 각각 1 또는 2입니다. 얽힘 방식은 "linear"로, Quantum Circuit에서 인접한 Qubit들만 연결됩니다. 이는 양자 컴퓨터 자체의 인접한 Qubit에 해당하지 않을 수 있는데, 이 Quantum Circuit은 추상화 레이어이기 때문입니다.

```python
from qiskit.circuit.library import pauli_feature_map

feature_dim = 3
pfmap = pauli_feature_map(
    feature_dimension=feature_dim, entanglement="linear", reps=1, paulis=["Y", "XX"]
)

pfmap.decompose().draw("mpl", scale=1.5)
```

![이전 코드 셀의 출력](/learning/images/courses/quantum-machine-learning/data-encoding/extracted-outputs/5ba5df82-83c1-428c-a269-baea5f75c3dc-0.avif)

Qiskit은 파울리 특성 맵에서 파울리 회전의 크기를 제어하는 매개변수 $\alpha$를 제공합니다.

$$
U(\bar{x}) = \exp\left(i \alpha \sum_{S\subseteq[n]} \phi_S(\bar{x}) \prod_{i \in S} \sigma_i \right)
$$

$\alpha$의 기본값은 $2$입니다. 예를 들어 $[0,4]$ 구간에서 이 값을 최적화하면 양자 커널을 데이터에 더 잘 맞출 수 있습니다.

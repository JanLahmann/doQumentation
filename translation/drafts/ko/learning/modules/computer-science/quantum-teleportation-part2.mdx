## 실험 1: 기본 텔레포테이션

IBM Quantum은 "Qiskit 패턴"이라는 프레임워크를 사용하여 양자 컴퓨팅 문제를 다루도록 권장합니다. 이 프레임워크는 다음 단계로 구성됩니다.
- 1단계: 문제를 양자 Circuit에 매핑하기
- 2단계: 실제 양자 하드웨어에서 실행할 수 있도록 Circuit 최적화하기
- 3단계: Runtime Primitive를 사용하여 IBM 양자 컴퓨터에서 작업 실행하기
- 4단계: 결과 후처리하기

### 1단계: 문제를 양자 Circuit에 매핑하기

위에서 수행한 모든 수학적 계산이 바로 1단계에 해당합니다. 이제 이를 실제로 구현하여 Qiskit으로 양자 Circuit을 만들어 보겠습니다! 세 개의 Qubit으로 양자 Circuit을 생성하고, Alice와 Bob의 두 Qubit을 얽힘 상태로 만드는 것부터 시작합니다. 이 두 Qubit을 1번과 2번 Qubit으로 지정하고, 0번 Qubit은 비밀 상태를 위해 예약해 두겠습니다.

```python
# Step 1: Map your problem to a quantum circuit

# Import some general packages
from qiskit import ClassicalRegister, QuantumCircuit, QuantumRegister
import numpy as np

# Define registers
secret = QuantumRegister(1, "Q")
Alice = QuantumRegister(1, "A")
Bob = QuantumRegister(1, "B")

cr = ClassicalRegister(3, "c")

qc = QuantumCircuit(secret, Alice, Bob, cr)

# We entangle Alice's and Bob's qubits as in our work above. We apply a Hadamard gate and then a CNOT gate.
# Note that the second argument in the CNOT gate is the target.
qc.h(Alice)
qc.cx(Alice, Bob)

# Inserting a barrier changes nothing about the logic. It just allows us to force gates to be positioned in "layers".
qc.barrier()

# Now we will use random variables to create the secret state. Don't worry about the "u" gate and the details.
np.random.seed(42)  # fixing seed for repeatability
theta = np.random.uniform(0.0, 1.0) * np.pi  # from 0 to pi
varphi = np.random.uniform(0.0, 2.0) * np.pi  # from 0 to 2*pi

# Assign the secret state to the qubit on the other side of Alice's (qubit 0), labeled Q
qc.u(theta, varphi, 0.0, secret)
qc.barrier()

# Now entangle Q and Alice's qubits as in the discussion above.
qc.cx(secret, Alice)
qc.h(secret)
qc.barrier()

# Now Alice measures her qubits, and stores the outcomes in the "classical registers" cr[]
qc.measure(Alice, cr[1])
qc.measure(secret, cr[0])

# Now we insert some conditional logic. If Alice measures Q in a "1" we need a Z gate, and if Alice measures A in a "1" we need an X gate (see the table).
with qc.if_test((cr[1], 1)):
    qc.x(Bob)
with qc.if_test((cr[0], 1)):
    qc.z(Bob)

qc.draw(output="mpl")
```

Alice의 상태를 Bob에게 텔레포트하기 위해 필요한 작업은 이것이 전부입니다. 그런데 양자 상태 $\alpha_0 |0\rangle+\alpha_1|1\rangle$을 측정하면 $|0\rangle$ 또는 $|1\rangle$ 중 하나만 얻게 된다는 점을 기억해야 합니다. 따라서 이 모든 과정이 끝난 후 Bob은 분명히 Alice의 비밀 상태를 갖게 되지만, 측정만으로는 이를 쉽게 검증할 수 없습니다. 올바르게 수행했는지 측정을 통해 확인하려면 한 가지 트릭이 필요합니다. Alice의 비밀 상태를 준비하는 데 사용한 "U"(유니터리) 연산자의 역연산을 Circuit 마지막에 적용하면 됩니다. U가 Alice의 $|0\rangle$ 상태를 $\alpha_0 |0\rangle+\alpha_1|1\rangle$로 매핑했다면, U의 역연산은 Bob의 $\alpha_0 |0\rangle+\alpha_1|1\rangle$을 다시 $|0\rangle$로 되돌립니다. 물론 이 마지막 단계는 양자 정보를 이동시키는 것이 목표라면 반드시 필요한 과정은 아닙니다. 이는 오로지 우리가 올바르게 수행했는지 확인하기 위한 단계입니다.

```python
# Add the inverse of U and measure Bob's qubit.
qc.barrier()

qc.u(theta, varphi, 0.0, Bob).inverse()  # inverse of u(theta,varphi,0.0)
qc.measure(Bob, cr[2])  # add measurement gate

qc.draw(output="mpl")
```

따라서 올바르게 수행했다면, Bob의 Qubit 측정 결과는 $|0\rangle$ 상태여야 합니다. 물론 이 측정은 확률적입니다. Bob의 Qubit이 $|1\rangle$ 상태로 측정될 확률이 조금이라도 있다면, 단일 측정에서 $|1\rangle$이 나올 수 있습니다. 따라서 $|0\rangle$의 확률이 상당히 높다는 것을 확신하려면 많은 횟수의 측정이 필요합니다.

### 2단계: 양자 실행을 위한 문제 최적화

이 단계에서는 수행하려는 연산을 특정 양자 컴퓨터의 기능에 맞게 표현하고, 문제를 양자 컴퓨터의 레이아웃에 매핑합니다.

IBM 양자 컴퓨터와 통신하는 데 필요한 여러 패키지를 불러오는 것으로 시작합니다. 또한 실행할 Backend를 선택해야 합니다. 가장 여유 있는 Backend를 자동으로 선택하거나, 속성을 이미 알고 있는 특정 Backend를 직접 지정할 수 있습니다.

아래 코드에는 처음 사용 시 자격 증명을 저장하는 코드가 포함되어 있습니다. 자격 증명을 환경에 저장한 후에는 노트북에서 해당 정보를 반드시 삭제하여, 노트북을 공유할 때 자격 증명이 실수로 노출되지 않도록 주의하세요. 자세한 안내는 [IBM Cloud 계정 설정](/guides/initialize-account) 및 [신뢰할 수 없는 환경에서 서비스 초기화](/guides/cloud-setup-untrusted)를 참고하세요.

이제 양자 Circuit을 "트랜스파일"해야 합니다. 이 과정은 여러 하위 단계를 포함하는 흥미로운 주제입니다. 하위 단계의 예를 들자면, 모든 양자 컴퓨터가 Qiskit의 모든 논리 Gate를 직접 구현할 수 있는 것은 아닙니다. Circuit의 Gate를 해당 양자 컴퓨터가 구현할 수 있는 Gate로 다시 표현해야 합니다. 이 과정과 그 밖의 단계들을 사전 설정된 패스 매니저(pass manager)를 사용하여 수행할 수 있습니다. ```optimization = 3```(최고 수준의 최적화)으로 설정하면 추상적인 양자 Circuit에서 양자 컴퓨터에 전달되는 명령으로의 매핑이 전처리 단계에서 가능한 한 효율적으로 이루어집니다.

```python
# Step 2: Transpile
from qiskit.transpiler.preset_passmanagers import generate_preset_pass_manager

target = backend.target
pm = generate_preset_pass_manager(target=target, optimization_level=3)
qc_isa = pm.run(qc)
```

Sampler는 양자 Circuit에서 발생할 수 있는 상태를 샘플링하고, 어떤 상태가 어떤 확률로 측정될 수 있는지 통계를 수집하도록 설계된 Primitive입니다. 여기서는 Qiskit Runtime Sampler를 불러옵니다.

```python
# Load the Runtime primitive and session
from qiskit_ibm_runtime import SamplerV2 as Sampler

sampler = Sampler(mode=backend)
```

양자 컴퓨터에서의 모든 계산이 클래식 컴퓨터로 합리적으로 시뮬레이션 가능한 것은 아닙니다. 이 간단한 텔레포테이션은 시뮬레이션이 가능하지만, 정보를 한 곳에서 다른 곳으로 저장할 수 있다는 것은 전혀 놀라운 일이 아닙니다. 실제 IBM 양자 컴퓨터를 사용하여 이 계산을 수행할 것을 강력히 권장합니다. 다만 무료 월간 사용량을 모두 소진했거나, 수업 시간 중에 완료해야 하는데 대기열을 기다릴 수 없는 경우에는 시뮬레이터를 사용하여 이 모듈을 완료할 수 있습니다. 이를 위해 아래 셀을 실행하고 "실행" 단계의 해당 줄 주석을 해제하세요.

```python
# Load the backend sampler
from qiskit.primitives import BackendSamplerV2

# Load the Aer simulator and generate a noise model based on the currently-selected backend.
from qiskit_aer import AerSimulator
from qiskit_aer.noise import NoiseModel

noise_model = NoiseModel.from_backend(backend)

# Define a simulator using Aer, and use it in Sampler.
backend_sim = AerSimulator(noise_model=noise_model)
sampler_sim = BackendSamplerV2(backend=backend_sim)

# Alternatively, load a fake backend with generic properties and define a simulator.
# backend_gen = GenericBackendV2(num_qubits=18)
# sampler_gen = BackendSamplerV2(backend=backend_gen)
```

### 3단계: 실행

Sampler를 사용하여 Circuit을 인수로 전달하고 작업을 실행합니다.

```python
job = sampler.run([qc_isa])
# job = sampler_sim.run([qc_isa])
res = job.result()
counts = res[0].data.c.get_counts()
```

__정답:__

$|000\rangle,$ $|001\rangle,$ $|010\rangle,$ $|011\rangle$ 상태는 모두 텔레포테이션 성공과 일치합니다. 비밀 상태의 초기 준비를 되돌리는 Gate를 추가했기 때문입니다. 비밀 상태가 Bob의 Qubit으로 성공적으로 텔레포트되었다면, 추가된 Gate가 Bob의 Qubit을 $|0\rangle$ 상태로 되돌려야 합니다. 따라서 위 결과 중 Bob의 Qubit(0번 Qubit으로, 고전 레지스터의 0번 구성 요소에 측정되며, 가장 오른쪽에 위치)이 $|0\rangle$ 상태인 모든 결과가 성공을 나타냅니다.

</details>

이 그래프는 세 Qubit의 모든 측정 결과를 5,000회 시도(샷)에 걸쳐 보여줍니다. Alice는 Qubit A와 Q에서 모든 가능한 상태를 동등한 확률로 측정한다고 앞서 언급했습니다. Circuit에서 Qubit 0~2를 각각 Q, A, B에 할당했습니다. little-endian 표기법에서 Bob의 Qubit은 가장 왼쪽에 위치합니다. 따라서 왼쪽의 네 막대는 Bob의 Qubit이 $|0\rangle$인 경우에 해당하며, 나머지 두 Qubit은 거의 균등한 확률로 모든 조합을 나타냅니다. 측정의 거의 대부분(보통 약 95%)에서 Bob의 Qubit이 $|0\rangle$ 상태로 나타나 설정이 성공적임을 알 수 있습니다! 소수의 샷(약 5%)에서 Bob의 Qubit이 $|1\rangle$ 상태로 나타납니다. 이는 논리적으로 발생해서는 안 되는 결과입니다. 그러나 현대의 모든 양자 컴퓨터는 클래식 컴퓨터보다 훨씬 많은 노이즈와 오류를 겪으며, 양자 오류 정정은 아직 발전 중인 분야입니다.

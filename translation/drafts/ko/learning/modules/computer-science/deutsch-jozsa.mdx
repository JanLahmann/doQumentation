## 소개 {#intro}

1980년대 초, 양자 물리학자들과 컴퓨터 과학자들은 양자역학을 활용하면 고전 컴퓨터보다 훨씬 강력한 연산이 가능할 것이라는 막연한 직관을 갖고 있었습니다. 그 근거는 이러했습니다: 고전 컴퓨터로 양자 시스템을 시뮬레이션하기는 어렵지만, *양자* 컴퓨터라면 더 효율적으로 처리할 수 있을 것이라고 생각했습니다. 그리고 양자 컴퓨터가 양자 시스템을 더 효율적으로 시뮬레이션할 수 있다면, 고전 컴퓨터보다 더 효율적으로 수행할 수 있는 다른 작업들도 있을 것이라고 보았습니다.

이 논리는 타당했지만, 세부 내용은 아직 규명되지 않은 상태였습니다. 그러다 1985년 David Deutsch가 최초의 "범용 양자 컴퓨터"를 기술하면서 연구가 본격화되었습니다. 같은 논문에서 그는 양자 컴퓨터가 고전 컴퓨터보다 더 효율적으로 풀 수 있는 첫 번째 예시 문제를 제시했습니다. 이 최초의 간단한 예시는 현재 "Deutsch 알고리즘"으로 알려져 있습니다. Deutsch 알고리즘의 성능 개선은 미미했지만, 몇 년 후 Deutsch는 Richard Jozsa와 협력하여 고전 컴퓨터와 양자 컴퓨터 간의 격차를 더욱 벌려놓았습니다.

이 알고리즘들, 즉 Deutsch 알고리즘과 Deutsch-Jozsa 확장 알고리즘은 실용적인 용도는 크지 않지만, 몇 가지 중요한 이유로 여전히 의미가 있습니다:

1. 역사적으로 이 알고리즘들은 고전 알고리즘을 능가한 최초의 양자 알고리즘 중 하나였습니다. 이를 이해하면 양자 컴퓨팅에 대한 연구 커뮤니티의 사고가 어떻게 발전해 왔는지 파악하는 데 도움이 됩니다.
2. 이 알고리즘들은 놀랍도록 미묘한 질문에 대한 답의 일부를 이해하는 데 도움이 됩니다: 양자 컴퓨팅의 힘은 어디서 나오는가? 양자 컴퓨터를 거대하고 지수적으로 확장되는 병렬 프로세서에 비유하는 경우가 있지만, 이는 정확하지 않습니다. 이 질문에 대한 답의 일부는 이른바 "양자 병렬성"에 있지만, 단 한 번의 실행에서 가능한 한 많은 정보를 추출하는 것은 섬세한 기술이 필요합니다. Deutsch 알고리즘과 Deutsch-Jozsa 알고리즘은 이것이 어떻게 가능한지 보여줍니다.

이 모듈에서는 Deutsch 알고리즘, Deutsch-Jozsa 알고리즘, 그리고 이 알고리즘들이 양자 컴퓨팅의 힘에 대해 무엇을 가르쳐 주는지 배웁니다.
## 양자 병렬성과 그 한계 {#quantum-parallelism-and-its-limits}

양자 컴퓨팅의 힘 중 일부는 "양자 병렬성"에서 비롯됩니다. 양자 병렬성이란 Qubit 입력 상태가 고전적으로 허용된 여러 상태의 중첩에 있을 수 있기 때문에, 동시에 여러 입력에 대해 연산을 수행할 수 있는 능력을 말합니다. 그러나 양자 Circuit이 여러 입력 상태를 한 번에 평가할 수 있다 하더라도, 그 모든 정보를 한 번에 추출하는 것은 불가능합니다.

이것이 무슨 의미인지 살펴보겠습니다. 비트 $x$와 그 비트에 적용되는 함수 $f(x)$가 있다고 가정해 봅시다. 단일 비트를 또 다른 단일 비트로 대응시키는 이진 함수는 네 가지가 가능합니다:

| $x$    |  $f_1(x)$ | $f_2(x)$ | $f_3(x)$ | $f_4(x)$ |
| ----------- | ------- |-------| -------- | ------- |
| 0  | 0   | 0  | 1   | 1 |
| 1  | 0   | 1  | 0 | 1 |

우리는 $f(x)$가 이 네 함수(1~4) 중 어느 것인지 알고 싶습니다. 고전적으로는 함수를 두 번 실행해야 합니다 — $x=0$일 때 한 번, $x=1$일 때 한 번. 하지만 양자 Circuit으로 더 잘할 수 있는지 살펴보겠습니다. 다음 Gate를 통해 함수에 대한 정보를 얻을 수 있습니다:

![quantum_parallelism](/learning/images/modules/computer-science/deutsch-jozsa/quantum-parallelism.avif)

여기서 $U_f$ Gate는 $f(x)$를 계산하는데, $x$는 Qubit 0의 상태이며 그 결과를 Qubit 1에 적용합니다. 따라서 $|y\rangle = |0\rangle$일 때, 결과 상태 $|x\rangle|y\oplus f(x)\rangle$은 단순히 $|x\rangle|f(x)\rangle$이 됩니다. 이 상태에는 함수 $f(x)$를 알기 위해 필요한 모든 정보가 담겨 있습니다: Qubit 0은 $x$가 무엇인지, Qubit 1은 $f(x)$가 무엇인지 알려줍니다. 따라서 $|x\rangle = \frac{1}{\sqrt{2}}(|0\rangle+|1\rangle)$으로 초기화하면, 두 Qubit의 최종 상태는 다음과 같습니다: $|y\rangle|x\rangle = \frac{1}{\sqrt{2}}(|f(0)\rangle|0\rangle+|f(1)\rangle|1\rangle)$. 그런데 이 정보에 어떻게 접근할 수 있을까요?

```python
# Step 1: Map

from qiskit import QuantumCircuit

qc = QuantumCircuit(2)

def twobit_function(case: int):
    """
    Generate a valid two-bit function as a `QuantumCircuit`.
    """
    if case not in [1, 2, 3, 4]:
        raise ValueError("`case` must be 1, 2, 3, or 4.")

    f = QuantumCircuit(2)
    if case in [2, 3]:
        f.cx(0, 1)
    if case in [3, 4]:
        f.x(1)
    return f

# first, convert oracle circuit (above) to a single gate for drawing purposes. otherwise, the circuit is too large to display
# blackbox = twobit_function(2).to_gate()  # you may edit the number inside "twobit_function()" to select among the four valid functions
# blackbox.label = "$U_f$"

qc.h(0)
qc.barrier()
qc.compose(twobit_function(2), inplace=True)
qc.measure_all()

qc.draw("mpl")
```

from qiskit import QuantumCircuit
import numpy as np

def dj_function(num_qubits):
    """
    Create a random Deutsch-Jozsa function.
    """

    qc_dj = QuantumCircuit(num_qubits + 1)
    if np.random.randint(0, 2):
        # Flip output qubits with 50% chance
        qc_dj.x(num_qubits)
    if np.random.randint(0, 2):
        # return constant circuit with 50% chance.
        return qc_dj

    # If the "if" statement above was "TRUE" then we've returned the constant
    # function and the function is complete. If not, we proceed in creating our
    # balanced function. Everything below is to produce the balanced function:

    # select half of all possible states at random:
    on_states = np.random.choice(
        range(2**num_qubits),  # numbers to sample from
        2**num_qubits // 2,  # number of samples
        replace=False,  # makes sure states are only sampled once
    )

    def add_cx(qc_dj, bit_string):
        for qubit, bit in enumerate(reversed(bit_string)):
            if bit == "1":
                qc_dj.x(qubit)
        return qc_dj

    for state in on_states:
        # qc_dj.barrier()  # Barriers are added to help visualize how the functions are created. They can safely be removed.
        qc_dj = add_cx(qc_dj, f"{state:0b}")
        qc_dj.mcx(list(range(num_qubits)), num_qubits)
        qc_dj = add_cx(qc_dj, f"{state:0b}")

    # qc_dj.barrier()

    return qc_dj

n = 3  # number of input qubits

oracle = dj_function(n)

display(oracle.draw("mpl"))

## 결론

이러한 간단한 예제들을 살펴봄으로써, 양자 컴퓨터가 중첩, 얽힘, 간섭을 활용하여 고전 컴퓨터보다 강력한 능력을 발휘하는 방법에 대해 더 나은 직관을 얻으셨기를 바랍니다.

Deutsch-Jozsa 알고리즘은 고전 알고리즘 대비 처음으로 속도 향상을 입증했다는 점에서 역사적으로 매우 중요하지만, 이는 다항식적 속도 향상에 불과했습니다. Deutsch-Jozsa 알고리즘은 이야기의 시작일 뿐입니다.

Bernstein과 Vazirani는 이 알고리즘을 사용하여 자신들의 문제를 해결한 후, 이를 기반으로 *재귀적 푸리에 샘플링 문제(recursive Fourier sampling problem)*라 불리는 더 복잡한 재귀적 문제를 제시했습니다. 그들의 풀이는 고전 알고리즘 대비 초다항식적(super-polynomial) 속도 향상을 제공했습니다. 그리고 Bernstein과 Vazirani보다도 앞서, Peter Shor는 이미 양자 컴퓨터가 어떤 고전 알고리즘보다도 지수적으로 빠르게 큰 수를 인수분해할 수 있게 하는 유명한 알고리즘을 개발했습니다. 이러한 결과들은 미래 양자 컴퓨터의 흥미로운 가능성을 집합적으로 보여 주었고, 물리학자들과 엔지니어들이 이 미래를 현실로 만들기 위해 박차를 가하도록 이끌었습니다.

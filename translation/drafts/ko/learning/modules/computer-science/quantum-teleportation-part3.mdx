## 실험 2: 프로세서를 가로질러 텔레포테이션하기 {#experiment-2-teleporting-across-a-processor}

양자 텔레포테이션에서 가장 흥미로운 점은, 고전적 통신을 통한 추가 Gate 적용은 즉각적이지 않지만, 양자 상태 자체는 멀리 떨어진 거리까지 즉시 텔레포테이션될 수 있다는 것입니다. 이미 언급한 바와 같이, 프로세서에서 Qubit을 분리해 이동시킬 수는 없습니다. 하지만 한 Qubit에서 다른 Qubit으로 정보를 이동시켜, 텔레포테이션에 관여하는 Qubit들이 프로세서의 반대편 끝에 위치하도록 만들 수 있습니다. 위에서 수행한 단계를 반복하되, 이번에는 프로세서 전체를 아우를 수 있을 만큼 충분한 Qubit을 포함하는 더 큰 Circuit을 만들겠습니다.

### Step 1: 문제를 양자 Circuit으로 매핑하기 {#step-1-map-your-problem-to-a-quantum-circuit-2}

이번에는 Alice와 Bob에 해당하는 Qubit이 변경됩니다. 따라서 특정 Qubit에 "A" 또는 "B"라는 이름을 붙이지 않고, Qubit에 번호를 매기고 변수를 사용해 Alice와 Bob의 정보가 현재 어느 Qubit에 있는지를 나타냅니다. SWAP Gate를 제외한 다른 모든 단계는 앞에서 설명한 것과 동일합니다.

```python
# Step 1: Map

# Define registers
qr = QuantumRegister(13, "q")

qc = QuantumCircuit(qr, cr)

# Define registers
secret = QuantumRegister(1, "Q")
ebitsa = QuantumRegister(6, "A")
ebitsb = QuantumRegister(6, "B")
# q = ClassicalRegister(1, "q meas")
# a = ClassicalRegister(1, "a")
# b = ClassicalRegister(1, "b")
cr = ClassicalRegister(3, "c")
qc = QuantumCircuit(secret, ebitsa, ebitsb, cr)

# We'll start Alice in the middle of the circuit, then move information outward in both directions.
Alice = 5
Bob = 0
qc.h(ebitsa[Alice])
qc.cx(ebitsa[Alice], ebitsb[Bob])

# Starting with Bob and Alice in the center, we swap their information onto adjacent qubits, until the information is on distant qubits.

for n in range(Alice):
    qc.swap(ebitsb[Bob], ebitsb[Bob + 1])
    qc.swap(ebitsa[Alice], ebitsa[Alice - 1])
    Alice = Alice - 1
    Bob = Bob + 1

qc.barrier()

# Create a random state for Alice (qubit zero)
np.random.seed(42)  # fixing seed for repeatability
# theta = np.random.uniform(0.0, 1.0) * np.pi    #from 0 to pi
theta = 0.3
varphi = np.random.uniform(0.0, 2.0) * np.pi  # from 0 to 2*pi

qc.u(theta, varphi, 0.0, secret)

# Entangle Alice's two qubits
qc.cx(secret, ebitsa[Alice])
qc.h(secret)

qc.barrier()

# Make measurements of Alice's qubits and store the results in the classical register.
qc.measure(ebitsa[Alice], cr[1])
qc.measure(secret, cr[0])

# Send instructions to Bob's qubits based on the outcome of Alice's measurements.
with qc.if_test((cr[1], 1)):
    qc.x(ebitsb[Bob])
with qc.if_test((cr[0], 1)):
    qc.z(ebitsb[Bob])

qc.barrier()

# Invert the preparation we did for Carl's qubit so we can check whether we did this correctly.
qc.u(theta, varphi, 0.0, ebitsb[Bob]).inverse()  # inverse of u(theta,varphi,0.0)
qc.measure(ebitsb[Bob], cr[2])  # add measurement gate

qc.draw("mpl")
```

Circuit 다이어그램에서 논리적 단계가 동일함을 확인할 수 있습니다. 유일한 차이점은, SWAP Gate를 사용해 Alice의 Qubit 상태를 Qubit 6($A_5$)에서 Qubit 1($A_0$)로 옮겨 Q 바로 옆에 위치시켰다는 것입니다. 또한 SWAP Gate를 사용해 Bob의 초기 상태를 Qubit 7($B_0$)에서 Qubit 12($B_5$)로 이동시켰습니다. Qubit 12의 상태는 멀리 떨어진 Qubit 0과 1에서 측정이 이루어지기 전까지는 Q의 비밀 상태와 관련이 없으며, 조건부 $X$ 및 $Z$ Gate가 적용된 이후에야 비로소 비밀 상태와 동일해집니다.

### Step 2: Circuit 최적화하기 {#step-2-optimize-your-circuit}

일반적으로 pass manager를 사용해 Circuit을 트랜스파일하고 최적화할 때는 ```optimization_level = 3```으로 설정하는 것이 좋습니다. Circuit을 최대한 효율적으로 만들기 위해서입니다. 그런데 이 경우에는, Qubit 6과 7의 상태를 Qubit 1과 12로 이동시킬 계산상의 이유가 없습니다. 그것은 단순히 거리를 두고 텔레포테이션을 시연하기 위해 수행한 작업입니다. pass manager에 Circuit 최적화를 요청하면, 이 SWAP Gate들에는 논리적 이유가 없다고 판단해 제거하고, Gate 연산을 인접한 Qubit에서 직접 수행하게 됩니다. 따라서 이 특별한 경우에는 ```optimization_level = 0```을 사용합니다.

```python
# Step 2: Transpile
from qiskit.transpiler.preset_passmanagers import generate_preset_pass_manager

target = backend.target
pmzero = generate_preset_pass_manager(target=target, optimization_level=0)

qc_isa_zero = pmzero.run(qc)

print(qc_isa_zero.depth())
```

### Step 4: 고전적 후처리 {#step-4-classical-post-processing}

여기서도 Alice의 Qubit에 대한 가능한 결과들의 확률이 상당히 균등하게 분포되어 있음을 확인할 수 있습니다. 비밀 코드를 역변환한 후 Bob의 Qubit이 $|0\rangle$로 측정되는 경향이 강하게 나타나는데, 이는 비밀 상태를 프로세서의 Q에서 Bob(Qubit 0에서 12)까지 올바르게 텔레포테이션했을 확률이 높다는 것을 의미합니다. 다만, Bob에 대해 $|0\rangle$이 *측정되지 않을* 확률이 이전보다 더 높아졌다는 점에 주목하세요. 이것은 양자 컴퓨팅에서 중요한 교훈입니다. SWAP Gate와 같은 다중 Qubit Gate를 많이 사용할수록, 더 많은 노이즈와 오류가 발생합니다.

### 핵심 개념 {#critical-concepts}

- Qubit은 얽힘(entanglement) 상태가 될 수 있으며, 이는 한 Qubit의 측정이 다른 Qubit의 상태에 영향을 미치거나 결정할 수 있다는 것을 의미합니다.
- 얽힘은 고전적 상관관계와 다릅니다. 예를 들어, Qubit A와 B는 $\alpha_0|00\rangle+\alpha_1|11\rangle$과 같은 상태의 중첩에 있을 수 있습니다. A 또는 B의 상태는 자연에 의해 아직 결정되지 않았지만, A와 B는 항상 같은 상태에 있다고 보장될 수 있습니다.
- 얽힘과 측정의 조합을 통해 상태(정보를 저장할 수 있는)를 한 Qubit에서 다른 Qubit으로 전송할 수 있습니다. 이 전송은 먼 거리에 걸쳐 이루어질 수 있으며, 이를 양자 텔레포테이션이라고 합니다.
- 양자 텔레포테이션은 확률적인 양자 측정에 의존합니다. 따라서 텔레포테이션된 상태를 조정하기 위해 고전적 통신이 필요할 수 있습니다. 이로 인해 양자 텔레포테이션은 빛보다 빠르게 정보를 이동시킬 수 없습니다. 양자 텔레포테이션은 상대성 이론이나 인과율을 위반하지 않습니다.
- 현대의 양자 컴퓨터는 고전 컴퓨터보다 노이즈와 오류에 더 취약합니다. 몇 퍼센트의 오류를 예상해야 합니다.
- 순서대로 Gate를 더 많이 추가할수록(특히 2-Qubit Gate), 더 많은 오류와 노이즈가 예상됩니다.

### 참/거짓 문제 {#truefalse-questions}

1. 참/거짓 양자 텔레포테이션은 빛보다 빠르게 정보를 전송하는 데 사용될 수 있다.
2. 참/거짓 현대의 실험 증거는 양자 상태의 붕괴가 빛보다 빠르게 전파된다는 것을 시사한다.
3. 참/거짓 Qiskit에서 Qubit은 $|q_3,q_2,q_1, q_0\rangle$과 같이 가장 낮은 번호의 Qubit이 오른쪽에 오도록 상태 내에서 정렬된다.

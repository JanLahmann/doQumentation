## 실험 2: 프로세서 전체에 걸쳐 텔레포테이션하기

양자 텔레포테이션에서 가장 흥미로운 점은, 양자 상태를 먼 거리에서도 즉시 텔레포테이션할 수 있다는 것입니다 (단, 추가 Gate에 대한 고전 통신은 즉시 이루어지지 않습니다). 이미 언급했듯이, 프로세서에서 Qubit을 분리해 이동시킬 수는 없습니다. 그러나 한 Qubit에서 다른 Qubit으로 정보를 옮겨, 텔레포테이션에 관여하는 Qubit들이 프로세서의 양쪽 끝에 위치하도록 만들 수 있습니다. 위에서 수행한 단계를 반복하되, 이번에는 프로세서 전체를 가로지를 수 있을 만큼 충분한 Qubit을 포함하는 더 큰 Circuit을 구성해 보겠습니다.

### 1단계: 문제를 양자 Circuit으로 매핑하기

이번에는 Alice와 Bob에 해당하는 Qubit이 달라집니다. 따라서 특정 Qubit을 "A"나 "B"라고 명명하는 대신, Qubit에 번호를 매기고 Alice와 Bob의 정보가 현재 위치한 Qubit을 변수로 나타냅니다. 스왑 Gate를 제외한 다른 모든 단계는 앞서 설명한 것과 동일합니다.

```python
# Step 1: Map

# Define registers
qr = QuantumRegister(13, "q")

qc = QuantumCircuit(qr, cr)

# Define registers
secret = QuantumRegister(1, "Q")
ebitsa = QuantumRegister(6, "A")
ebitsb = QuantumRegister(6, "B")
# q = ClassicalRegister(1, "q meas")
# a = ClassicalRegister(1, "a")
# b = ClassicalRegister(1, "b")
cr = ClassicalRegister(3, "c")
qc = QuantumCircuit(secret, ebitsa, ebitsb, cr)

# We'll start Alice in the middle of the circuit, then move information outward in both directions.
Alice = 5
Bob = 0
qc.h(ebitsa[Alice])
qc.cx(ebitsa[Alice], ebitsb[Bob])

# Starting with Bob and Alice in the center, we swap their information onto adjacent qubits, until the information is on distant qubits.

for n in range(Alice):
    qc.swap(ebitsb[Bob], ebitsb[Bob + 1])
    qc.swap(ebitsa[Alice], ebitsa[Alice - 1])
    Alice = Alice - 1
    Bob = Bob + 1

qc.barrier()

# Create a random state for Alice (qubit zero)
np.random.seed(42)  # fixing seed for repeatability
# theta = np.random.uniform(0.0, 1.0) * np.pi    #from 0 to pi
theta = 0.3
varphi = np.random.uniform(0.0, 2.0) * np.pi  # from 0 to 2*pi

qc.u(theta, varphi, 0.0, secret)

# Entangle Alice's two qubits
qc.cx(secret, ebitsa[Alice])
qc.h(secret)

qc.barrier()

# Make measurements of Alice's qubits and store the results in the classical register.
qc.measure(ebitsa[Alice], cr[1])
qc.measure(secret, cr[0])

# Send instructions to Bob's qubits based on the outcome of Alice's measurements.
with qc.if_test((cr[1], 1)):
    qc.x(ebitsb[Bob])
with qc.if_test((cr[0], 1)):
    qc.z(ebitsb[Bob])

qc.barrier()

# Invert the preparation we did for Carl's qubit so we can check whether we did this correctly.
qc.u(theta, varphi, 0.0, ebitsb[Bob]).inverse()  # inverse of u(theta,varphi,0.0)
qc.measure(ebitsb[Bob], cr[2])  # add measurement gate

qc.draw("mpl")
```

Circuit 다이어그램에서 확인할 수 있듯이, 논리적 단계는 동일합니다. 차이점은 스왑 Gate를 사용하여 Alice Qubit의 상태를 Qubit 6($A_5$)에서 Qubit 1($A_0$)로 Q 바로 옆까지 이동시켰다는 것입니다. 또한 스왑 Gate를 사용하여 Bob의 초기 상태를 Qubit 7($B_0$)에서 Qubit 12($B_5$)로 이동시켰습니다. Qubit 12의 상태는 Qubit 0과 1에서 멀리 떨어진 측정이 이루어지기 전까지는 Q의 비밀 상태와 관련이 없으며, 조건부 $X$ Gate와 $Z$ Gate가 적용된 후에야 비로소 비밀 상태와 동일해진다는 점에 주목하세요.

### 2단계: Circuit 최적화하기

일반적으로 pass manager를 사용하여 Circuit을 Transpile하고 최적화할 때는 ```optimization_level = 3```으로 설정하는 것이 좋습니다. Circuit을 최대한 효율적으로 만들고 싶기 때문입니다. 그러나 이 경우에는 Qubit 6과 7의 상태를 Qubit 1과 12로 이동시키는 데 계산적인 이유가 없습니다. 이는 단지 거리에 걸친 텔레포테이션을 시연하기 위해 수행한 것입니다. pass manager에게 Circuit 최적화를 요청하면, 이 스왑 Gate들에 논리적 이유가 없음을 파악하고 이를 제거한 뒤 인접한 Qubit에서 Gate 연산을 수행하게 됩니다. 따라서 이 특별한 경우에는 ```optimization_level = 0```을 사용합니다.

```python
# Step 2: Transpile
from qiskit.transpiler.preset_passmanagers import generate_preset_pass_manager

target = backend.target
pmzero = generate_preset_pass_manager(target=target, optimization_level=0)

qc_isa_zero = pmzero.run(qc)

print(qc_isa_zero.depth())
```

### 4단계: 고전적 후처리

여기서도 Alice Qubit의 가능한 결과 확률이 상당히 균등하게 분포되어 있음을 확인할 수 있습니다. 비밀 코드를 역변환한 후 Bob의 Qubit이 $|0\rangle$로 측정되는 강한 선호도가 나타나는데, 이는 Q에서 Bob(Qubit 0에서 12)으로 프로세서 전체에 걸쳐 비밀 상태를 성공적으로 텔레포테이션했을 가능성이 높다는 것을 의미합니다. 단, Bob의 Qubit에서 $|0\rangle$이 *측정되지 않을* 확률이 이전보다 다소 높아졌음을 주목해야 합니다. 이것은 양자 컴퓨팅에서 중요한 교훈입니다. Gate가 많아질수록, 특히 스왑 Gate와 같은 다중 Qubit Gate가 많아질수록, 노이즈와 오류가 증가합니다.

## Deutsch-Jozsa 알고리즘

Deutsch의 알고리즘은 양자 컴퓨터가 고전 컴퓨터보다 더 효율적일 수 있음을 보여준 중요한 첫걸음이었지만, 그 개선 폭은 크지 않았습니다. 고전적인 경우에는 두 번의 쿼리가 필요한 반면, 단 한 번의 쿼리만으로 충분했기 때문입니다. 1992년에 Deutsch는 동료 Richard Jozsa와 함께 원래의 2-Qubit 알고리즘을 더 많은 Qubit으로 확장했습니다. 문제의 핵심은 동일합니다. 함수가 *균형(balanced)* 인지 *상수(constant)* 인지 판별하는 것입니다. 하지만 이번에는 함수가 $n$개의 비트에서 단일 비트로 대응됩니다. 함수가 0과 1을 같은 횟수만큼 반환하면 *균형*, 항상 1 또는 항상 0만 반환하면 *상수*입니다.

다음은 이 알고리즘의 Circuit 다이어그램입니다:

![DJ_algo.png](/learning/images/modules/computer-science/deutsch-jozsa/DJ_algo.avif)

이 알고리즘은 Deutsch의 알고리즘과 동일한 방식으로 동작합니다. 위상 킥백(phase-kickback)을 통해 Qubit 0의 상태를 읽어 함수가 상수인지 균형인지 판별합니다. 2-Qubit Deutsch 알고리즘에 비해 조금 더 복잡한데, 상태 표현에 $n$개의 Qubit에 대한 합산이 포함되기 때문입니다. 따라서 중간 상태를 직접 계산하는 작업은 모듈 끝부분의 선택적 연습 문제로 남겨 두겠습니다. 이 알고리즘은 함수가 상수이면 모두 0으로 이루어진 비트열을, 함수가 균형이면 최소 하나의 1을 포함하는 비트열을 반환합니다.

Qiskit에서 알고리즘이 어떻게 동작하는지 확인하려면 먼저 오라클(oracle)을 생성해야 합니다. 오라클은 상수 또는 균형이 보장된 무작위 함수입니다. 아래 코드는 50%의 확률로 균형 함수를, 50%의 확률로 상수 함수를 생성합니다. 코드를 완전히 이해하지 못해도 괜찮습니다 — 코드가 다소 복잡하지만 양자 알고리즘 자체를 이해하는 데 꼭 필요하지는 않습니다.

```python
from qiskit import QuantumCircuit
import numpy as np

def dj_function(num_qubits):
    """
    Create a random Deutsch-Jozsa function.
    """

    qc_dj = QuantumCircuit(num_qubits + 1)
    if np.random.randint(0, 2):
        # Flip output qubits with 50% chance
        qc_dj.x(num_qubits)
    if np.random.randint(0, 2):
        # return constant circuit with 50% chance.
        return qc_dj

    # If the "if" statement above was "TRUE" then we've returned the constant
    # function and the function is complete. If not, we proceed in creating our
    # balanced function. Everything below is to produce the balanced function:

    # select half of all possible states at random:
    on_states = np.random.choice(
        range(2**num_qubits),  # numbers to sample from
        2**num_qubits // 2,  # number of samples
        replace=False,  # makes sure states are only sampled once
    )

    def add_cx(qc_dj, bit_string):
        for qubit, bit in enumerate(reversed(bit_string)):
            if bit == "1":
                qc_dj.x(qubit)
        return qc_dj

    for state in on_states:
        # qc_dj.barrier()  # Barriers are added to help visualize how the functions are created. They can safely be removed.
        qc_dj = add_cx(qc_dj, f"{state:0b}")
        qc_dj.mcx(list(range(num_qubits)), num_qubits)
        qc_dj = add_cx(qc_dj, f"{state:0b}")

    # qc_dj.barrier()

    return qc_dj

n = 3  # number of input qubits

oracle = dj_function(n)

display(oracle.draw("mpl"))
```

이것이 오라클 함수로, 균형 또는 상수 함수 중 하나입니다. Circuit을 보고 마지막 Qubit의 출력이 앞의 $n$개 Qubit 입력값에 의존하는지 알 수 있나요? 만약 의존한다면, 그 출력이 균형인지 아닌지도 판단할 수 있나요?

위 Circuit을 보면 함수가 균형인지 상수인지 알 수 있지만, 이 문제의 설정상 우리는 이 함수를 "블랙박스"로 취급한다는 점을 기억하세요. 박스 내부의 Circuit 다이어그램을 들여다볼 수는 없습니다. 대신 박스에 쿼리를 해야 합니다.

박스에 쿼리하기 위해 Deutsch-Jozsa 알고리즘을 사용하여 함수가 상수인지 균형인지 판별합니다:

```python
blackbox = oracle.to_gate()
blackbox.label = "$U_f$"

qc_dj = QuantumCircuit(n + 1, n)
qc_dj.x(n)
qc_dj.h(range(n + 1))
qc_dj.barrier()
qc_dj.compose(blackbox, inplace=True)
qc_dj.barrier()
qc_dj.h(range(n))
qc_dj.measure(range(n), range(n))

qc_dj.decompose().decompose()

qc_dj.draw("mpl")
```

## 양자 병렬성과 그 한계 {#quantum-parallelism-and-its-limits}

양자 컴퓨팅의 힘 중 일부는 "양자 병렬성"에서 비롯됩니다. 양자 병렬성이란 Qubit 입력 상태가 고전적으로 허용된 여러 상태의 중첩에 있을 수 있기 때문에, 동시에 여러 입력에 대해 연산을 수행할 수 있는 능력을 말합니다. 그러나 양자 Circuit이 여러 입력 상태를 한 번에 평가할 수 있다 하더라도, 그 모든 정보를 한 번에 추출하는 것은 불가능합니다.

이것이 무슨 의미인지 살펴보겠습니다. 비트 $x$와 그 비트에 적용되는 함수 $f(x)$가 있다고 가정해 봅시다. 단일 비트를 또 다른 단일 비트로 대응시키는 이진 함수는 네 가지가 가능합니다:

| $x$    |  $f_1(x)$ | $f_2(x)$ | $f_3(x)$ | $f_4(x)$ |
| ----------- | ------- |-------| -------- | ------- |
| 0  | 0   | 0  | 1   | 1 |
| 1  | 0   | 1  | 0 | 1 |

우리는 $f(x)$가 이 네 함수(1~4) 중 어느 것인지 알고 싶습니다. 고전적으로는 함수를 두 번 실행해야 합니다 — $x=0$일 때 한 번, $x=1$일 때 한 번. 하지만 양자 Circuit으로 더 잘할 수 있는지 살펴보겠습니다. 다음 Gate를 통해 함수에 대한 정보를 얻을 수 있습니다:

![quantum_parallelism](/learning/images/modules/computer-science/deutsch-jozsa/quantum-parallelism.avif)

여기서 $U_f$ Gate는 $f(x)$를 계산하는데, $x$는 Qubit 0의 상태이며 그 결과를 Qubit 1에 적용합니다. 따라서 $|y\rangle = |0\rangle$일 때, 결과 상태 $|x\rangle|y\oplus f(x)\rangle$은 단순히 $|x\rangle|f(x)\rangle$이 됩니다. 이 상태에는 함수 $f(x)$를 알기 위해 필요한 모든 정보가 담겨 있습니다: Qubit 0은 $x$가 무엇인지, Qubit 1은 $f(x)$가 무엇인지 알려줍니다. 따라서 $|x\rangle = \frac{1}{\sqrt{2}}(|0\rangle+|1\rangle)$으로 초기화하면, 두 Qubit의 최종 상태는 다음과 같습니다: $|y\rangle|x\rangle = \frac{1}{\sqrt{2}}(|f(0)\rangle|0\rangle+|f(1)\rangle|1\rangle)$. 그런데 이 정보에 어떻게 접근할 수 있을까요?

### 2.1. Qiskit으로 직접 해보기:

Qiskit을 사용하여 위의 네 가지 함수 중 하나를 무작위로 선택하고 Circuit을 실행하겠습니다. 그런 다음 여러분의 과제는 양자 Circuit의 측정 결과를 활용하여 최소한의 실행 횟수로 함수를 파악하는 것입니다.

이 첫 번째 실험과 이 모듈 전반에 걸쳐, "Qiskit 패턴"이라고 알려진 양자 컴퓨팅 프레임워크를 사용합니다. 이 프레임워크는 워크플로를 다음 단계로 구분합니다:

- 1단계: 고전적 입력을 양자 문제로 매핑하기
- 2단계: 양자 실행을 위해 문제 최적화하기
- 3단계: Qiskit Runtime Primitives를 사용하여 실행하기
- 4단계: 후처리 및 고전적 분석

Qiskit Runtime Primitives를 포함한 필요한 패키지를 먼저 불러오겠습니다. 또한 사용 가능한 양자 컴퓨터 중 가장 여유 있는 것을 선택하겠습니다.

아래 코드에는 처음 사용 시 자격 증명을 저장하는 코드가 있습니다. 환경에 저장한 후에는 노트북에서 해당 정보를 반드시 삭제하여 노트북을 공유할 때 자격 증명이 실수로 노출되지 않도록 하세요. 자세한 안내는 [IBM Cloud 계정 설정](/guides/initialize-account) 및 [신뢰할 수 없는 환경에서 서비스 초기화하기](/guides/cloud-setup-untrusted)를 참조하세요.

필요한 패키지를 불러왔으니, 이제 Qiskit 패턴 워크플로를 진행할 수 있습니다. 아래 매핑 단계에서는 먼저 단일 비트를 다른 단일 비트로 대응시키는 네 가지 가능한 함수 중 하나를 선택하는 함수를 만듭니다.

```python
# Step 1: Map

from qiskit import QuantumCircuit

qc = QuantumCircuit(2)

def twobit_function(case: int):
    """
    Generate a valid two-bit function as a `QuantumCircuit`.
    """
    if case not in [1, 2, 3, 4]:
        raise ValueError("`case` must be 1, 2, 3, or 4.")

    f = QuantumCircuit(2)
    if case in [2, 3]:
        f.cx(0, 1)
    if case in [3, 4]:
        f.x(1)
    return f

# first, convert oracle circuit (above) to a single gate for drawing purposes. otherwise, the circuit is too large to display
# blackbox = twobit_function(2).to_gate()  # you may edit the number inside "twobit_function()" to select among the four valid functions
# blackbox.label = "$U_f$"

qc.h(0)
qc.barrier()
qc.compose(twobit_function(2), inplace=True)
qc.measure_all()

qc.draw("mpl")
```

위 Circuit에서 Hadamard Gate "H"는 처음에 $|0\rangle$ 상태에 있는 Qubit 0을 중첩 상태 $\frac{1}{\sqrt{2}}(|0\rangle+|1\rangle)$으로 변환합니다. 그런 다음 $U_f$가 함수 $f(x)$를 평가하고 그 결과를 Qubit 1에 적용합니다.

다음으로 양자 컴퓨터에서 실행하기 위해 Circuit을 최적화하고 트랜스파일해야 합니다:

```python
# Step 2: Transpile
from qiskit.transpiler.preset_passmanagers import generate_preset_pass_manager

target = backend.target
pm = generate_preset_pass_manager(target=target, optimization_level=3)

qc_isa = pm.run(qc)
```

마지막으로, 트랜스파일된 Circuit을 양자 컴퓨터에서 실행하고 결과를 시각화합니다:

```python
# Step 3: Run the job on a real quantum computer

job = sampler.run([qc_isa], shots=1)
# job = sampler_sim.run([qc_isa],shots=1) # uncomment this line to run on simulator instead
res = job.result()
counts = res[0].data.meas.get_counts()
```

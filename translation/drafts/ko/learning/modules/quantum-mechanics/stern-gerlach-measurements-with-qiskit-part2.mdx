### 1단계: 고전적 입력을 양자 문제로 변환

여기서 고전적 입력은 Stern-Gerlach 장치에서 측정 전 스핀의 방향입니다. 측정 전 양자 상태의 정확한 성질에 대해서는 너무 깊이 걱정하지 않아도 됩니다. 그 내용은 벨 정리(Bell's theorem)를 다루는 별도의 Qiskit Classrooms 모듈의 주제입니다.

IBM 양자 컴퓨터는 $z$ 축을 따라 상태를 측정한다는 점에 유의하세요. 따라서 이 첫 번째 실험은 자기장 기울기가 $z$ 방향인 Stern-Gerlach 실험과 매우 유사합니다. 다른 방향으로 측정하는 방법은 이후에 살펴보겠습니다.

먼저 스핀 상태의 유사체, 즉 $|\uparrow\rangle$와 $|\downarrow\rangle$, 또는 동등하게 $|0\rangle$와 $|1\rangle$의 어떤 혼합을 만들어 보겠습니다. 아래에 초기 값을 제안해 두었습니다. 다른 값이나 무작위 값을 자유롭게 사용해 보세요.

```python
import random
from numpy import pi
import numpy as np

# Use these lines to choose your own arbitrary state vector and normalize it.
# a = 2
# b = (1+1j)
# norm = np.sqrt(a*np.conjugate(a)+b*np.conjugate(b))
# a = a/norm
# b = b/norm
# print(a,b)

# Use these lines if you would rather look at at random spin orientations.
a = random.random()
b = random.random()
norm = np.sqrt(a * np.conjugate(a) + b * np.conjugate(b))
a = a / norm
b = b / norm
print(a, b)
```

```text
0.7032089086145691 0.7109832845047109
```

이제 위의 각도를 양자 Circuit의 파라미터로 사용합니다. 한 번에 하나의 입자만 고려하므로 Circuit에는 Qubit 하나만 사용하고, 고전 레지스터도 하나만 필요합니다.

```python
from qiskit.circuit import QuantumRegister, ClassicalRegister, QuantumCircuit, Parameter

# Define registers
qr = QuantumRegister(1, "q")
cr = ClassicalRegister(1, "c")
qc = QuantumCircuit(qr, cr)

# Initialize the quantum state
qc.initialize([a, b])
qc.measure(0, 0)
qc.draw("mpl")
```

### 2단계: 양자 실행을 위한 문제 최적화

실제 양자 컴퓨터에서 실험을 실행하려면 Qiskit Runtime Service를 로드하고 양자 컴퓨터(또는 "Backend")를 선택해야 합니다. 아래에서는 사용 가능한 양자 컴퓨터 중 가장 덜 바쁜 것을 선택합니다.

아래에는 처음 사용 시 자격 증명을 저장하는 코드가 있습니다. 자격 증명을 환경에 저장한 후에는 해당 정보를 노트북에서 반드시 삭제하여, 노트북을 공유할 때 자격 증명이 실수로 유출되지 않도록 하세요. 자세한 안내는 [IBM Cloud 계정 설정](/guides/cloud-setup) 및 [신뢰할 수 없는 환경에서 서비스 초기화](/guides/cloud-setup-untrusted)를 참조하세요.

```python
# Load the Runtime primitive and session

# Use the least busy backend, specify options as needed
```


이제 Circuit을 Transpile해야 합니다. 즉, Circuit을 선택한 양자 컴퓨터에서 사용 가능한 기본 Gate에 매핑하고, 해당 양자 컴퓨터에서 실행하기 위해 Circuit을 최적화해야 합니다.

```python
# Transpile the circuit and optimize for running on the quantum computer selected
from qiskit.transpiler.preset_passmanagers import generate_preset_pass_manager

pm = generate_preset_pass_manager(optimization_level=3, backend=backend)
qc_ibm = pm.run(qc)
```

### 3단계: Qiskit Runtime Primitives를 사용한 실행

이제 실제 양자 컴퓨터에서 실험을 실행합니다. 필요한 모든 구문은 아래 코드 블록에 있습니다. 실제 양자 컴퓨터에 할당된 시간을 모두 사용했거나 인터넷에 연결되어 있지 않은 경우, 다음 코드 블록의 주석을 해제하면 로컬 시뮬레이터에서 코드를 실행할 수 있습니다.

```python
# Specify that we want to use only a single shot, to represent a single measurement of a spin in a SG device.
num_shots = 1

# Evaluate the problem using a QPU via Qiskit IBM Runtime
# The best practice is to use a session as shown below. This is available to Premium Plan, Flex Plan, and On-Prem (IBM Quantum Platform API) Plan users.
with Session(backend=backend) as session:
    sampler = Sampler(mode=session)
    dist = sampler.run([qc_ibm], shots=num_shots).result()
session.close()
counts = dist[0].data.c.get_counts()
```

측정을 한 번 수행했고 "0"을 얻었습니다. 이는 우리에게 전혀 놀라운 일이 아닙니다. 우리는 Qubit으로 구성된 양자 컴퓨터를 사용하고 있으며, 고전적인 비트가 0 또는 1을 반환하는 것에 매우 익숙합니다. 하지만 기억하세요: 이것은 자기 모멘트를 가진 입자로 수행된 실험의 양자 컴퓨팅 유사체입니다. 만약 우리가 0과 +1 사이의 고른 분포를 기대했다면, 첫 번째 측정에서 한쪽 극값을 얻은 것이 실제로 놀라웠을 것입니다. Stern-Gerlach 실험에서 결과가 이처럼 양자화되어 나타난 것이 자연에 대한 더 깊은 이해로 이어졌고, 결국 양자 컴퓨터를 구축하는 데 도움이 되었습니다.

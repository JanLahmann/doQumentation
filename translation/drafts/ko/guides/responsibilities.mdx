# Qiskit Runtime 사용 시 귀하의 책임 이해하기 {#understanding-your-responsibilities-when-using-qiskit-runtime}

Qiskit Runtime 사용 시 귀하와 IBM&reg;의 구체적인 책임에 대해 다음 섹션을 검토하세요. 전반적인 이용 약관은 IBM Cloud&reg;의 [약관 및 공지사항](https://cloud.ibm.com/docs/overview/terms-of-use?topic=overview-terms)을 참조하세요.

## 인시던트 및 운영 관리 {#incident-and-operations-management}

인시던트 및 운영 관리에는 모니터링, 이벤트 관리, 고가용성, 문제 파악, 복구, 전체 상태 백업 및 복구와 같은 작업이 포함됩니다.

| 작업 | IBM의 책임 | 귀하의 책임 |
| --- | --- | --- |
|인프라 상태 관리|<ul><li>필요에 따라 투명한 방식으로 새로운 기능, 업데이트 및 버그 수정을 지속적으로 제공합니다.</li><li>계획된 유지보수, 보안 공지 또는 예기치 않은 중단에 대한 알림을 제공합니다.</li></ul> | <ul><li>서비스 알림에 대한 이메일을 받기 위해 [기본 설정을 지정하세요](https://cloud.ibm.com/user/notifications).</li><li>일반 공지사항을 위해 [IBM Cloud 상태](https://cloud.ibm.com/status) 페이지를 모니터링하세요.</li></ul> |
|인프라 상태 모니터링 및 알림| <ul><li>모든 인프라 환경 문제를 해결합니다.</li><li> 해당 인시던트를 고객에게 알립니다.</li></ul> | <ul><li> 보고된 각 인시던트의 영향을 파악하세요.</li><li>필요에 따라 [IBM 지원](/guides/support)에 문의하세요.</li></ul>|
|IBM Cloud 고가용성 SLA 유지| [IBM Cloud 공공 서비스 수준 계약](https://cloud.ibm.com/docs/overview?topic=overview-slas)(SLA)에 따라 Qiskit Runtime을 운영합니다. | 고가용성 모범 사례를 따르세요. |

## 변경 관리 {#change-management}

변경 관리에는 배포, 구성, 업그레이드, 패치, 구성 변경 및 삭제와 같은 작업이 포함됩니다.

| 작업 | IBM의 책임 | 귀하의 책임 |
| --- | --- | --- |
| 주요 버전 업그레이드 | 주요 버전 업데이트를 위한 가용성 및 도구를 제공합니다. | 이전 버전의 지원 종료 전에 지원되는 버전으로 마이그레이션하세요. |
| API 버전 변경 | REST API의 버전을 관리합니다. | 원하는 버전의 REST API와 통합하세요. |
| 업데이트, 수정 및 새로운 기능 | <ul><li>지속적인 제공 모델을 투명하게 따르며 정기적인 업데이트, 버그 수정 및 새로운 기능을 제공합니다.</li><li>고객에게 영향을 미치는 변경 사항에 대한 알림을 게시합니다.</li></ul> | <ul><li>[이메일 알림을 받기 위한 기본 설정을 지정하고](https://cloud.ibm.com/user/notifications) 일반 공지사항을 위해 [IBM Cloud 상태](https://cloud.ibm.com/status) 페이지를 모니터링하세요.</li><li>시스템 생성 알림 및 [플랫폼 공지사항](https://quantum.cloud.ibm.com/announcements)을 읽고 적절한 조치를 취하세요.</li></ul> |

## ID 및 접근 관리 {#identity-and-access-management}

ID 및 접근 관리에는 인증, 권한 부여, 접근 제어 정책 관리, 접근 승인·부여·취소와 같은 작업이 포함됩니다.

| 작업 | IBM의 책임 | 귀하의 책임 |
| --- | --- | --- |
| ID 및 접근 제어 | IBM Cloud를 통해 리소스에 대한 접근을 관리하고 제한하는 기능을 제공합니다. | [Cloud ID 및 접근 관리](/guides/access-groups)(IAM)를 통해 리소스에 대한 접근을 부여, 취소 및 관리하세요. |
| 서비스 인증 | 인증된 사용자만 귀하의 Qiskit Runtime 서비스 및 리소스에 접근할 수 있도록 보장합니다. | <ul><li>무단 접근으로부터 [API 키를 보호하세요](/guides/cloud-setup-untrusted#initialize-the-service-in-an-untrusted-environment).</li><li>보안 정책 요구 사항에 따라 [API 키](https://cloud.ibm.com/iam/apikeys)를 교체하고 업데이트하세요.</li></ul> |
| 관찰 가능성 | 사용자 작업을 감사하기 위해 IBM Cloud 활동 추적기 이벤트 라우팅 통합을 허용합니다. | 사용자 활동을 추적하기 위해 [IBM Cloud 활동 추적기 이벤트 라우팅](https://cloud.ibm.com/docs/vmware-service?topic=vmware-service-at_events) 또는 기타 기능을 설정하세요. |

## 보안 및 규정 준수 {#security-and-regulation-compliance}

보안 및 규정 준수에는 보안 제어 구현 및 준수 인증과 같은 작업이 포함됩니다.

| 작업 | IBM의 책임 | 귀하의 책임 |
| --- | --- | --- |
| 일반 | <ul><li>다양한 산업 규정 준수 표준에 맞는 제어를 유지합니다.</li><li>서비스 기능을 모니터링, 격리 및 복구합니다.</li><li>서비스 인터페이스의 상태를 모니터링하고 보고합니다.</li></ul> | 개인 데이터 및 규제 데이터가 제공되지 않도록 하세요. |
| 취약점 | 취약점 및 보안 준수 문제를 감지하기 위해 모니터링합니다. | <ul><li>잠재적인 수정 조치를 제공하는 보안 공지를 통해 가능한 취약점 및 보안 문제에 대한 정보를 유지하세요.</li><li>[Qiskit GitHub 저장소의 보안 섹션](https://github.com/Qiskit/qiskit/security)을 확인하세요. 모든 보안 취약점이 해당 위치에 게시됩니다.</li><li>IBM Cloud 플랫폼, 인프라 및 주요 서비스에 영향을 미치는 주요 이벤트에 대한 공지 및 보안 공지를 확인하려면 [IBM Cloud 상태 웹사이트](https://cloud.ibm.com/status)를 확인하세요.</li></ul> |
| 암호화 | 공공 네트워크 및 Cloud 서비스의 사설 데이터 센터 네트워크 내에서 전송되는 데이터를 암호화합니다. 전송 중인 데이터를 보호하기 위해 HTTPS 및 TLS와 같은 암호화 방법이 사용됩니다. | 필요에 따라 연결이 종단 간 암호화되도록 하세요. |
| 보안 | <ul><li>인프라 내에서 디스크의 데이터 및 전송 중인 데이터의 보안을 보장합니다.</li><li>주요 표준을 준수하는 안전한 서비스를 제공합니다.</li></ul> | 적절한 리소스 및 리소스 그룹에 대한 [사용자 접근](/guides/access-groups)을 제한하세요. |

## 재해 복구 {#disaster-recovery}

| 작업 | IBM의 책임 | 귀하의 책임 |
| --- | --- | --- |
| 일반 | 영향을 받는 지역의 프로비저닝 환경 및 리소스를 복원하거나 재구축합니다. | <ul><li>인스턴스 및 리소스 상태를 모니터링하세요.</li><li>가능한 경우 대체 지역에 새로운 Qiskit Runtime 서비스 [인스턴스](/guides/instances)를 프로비저닝하세요.</li></ul> |
| 서비스 복구 | Qiskit Runtime 서비스 기능을 정상 운영으로 자동으로 복구하고 복원합니다. | 복구 후 영향을 받은 모든 작업을 다시 제출하세요. |
| 사용자 구성 데이터 백업 | IBM은 사용자 구성 데이터 백업에 대한 책임이 없습니다. | 필요에 따라 구성 데이터를 백업하세요. |
| 워크로드 백업 및 복구 | IBM은 사용자 워크로드 데이터의 백업 또는 복구에 대한 책임이 없습니다. | 필요에 따라 워크로드를 백업하고 복구하세요. |

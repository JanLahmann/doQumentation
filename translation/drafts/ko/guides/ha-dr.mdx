# Qiskit Runtime의 고가용성 및 재해 복구 이해 {#understanding-high-availability-and-disaster-recovery-for-qiskit-runtime}

고가용성(HA)은 예기치 않은 장애가 발생하더라도 서비스가 계속 운영되고 액세스 가능한 상태를 유지하는 능력입니다. 재해 복구는 서비스 인스턴스를 정상 작동 상태로 복원하는 프로세스입니다. Qiskit Runtime 고객은 재해 복구 계획이나 조치에 대한 책임이 없습니다.

Qiskit Runtime은 유료 플랜에서 정의된 [서비스 수준 목표(SLO)](https://cloud.ibm.com/docs/resiliency?topic=resiliency-slo)를 충족하는 지역 서비스입니다. 자세한 내용은 IBM Cloud&reg;의 [서비스 수준 계약(SLA)](https://www.ibm.com/support/customer/csol/terms/?id=i126-9268&lc=en)를 참조하세요. Qiskit Runtime에 사용 가능한 IBM Cloud 지역 및 데이터 센터에 대한 자세한 내용은 [위치별 서비스 및 인프라 가용성](https://cloud.ibm.com/docs/overview?topic=overview-services_region)을 참조하세요.

## 고가용성 아키텍처 {#high-availability-architecture}

### 고가용성 기능 {#high-availability-features}

이 서비스는 단일 장애 지점 없이 다중 영역 지역의 세 개 영역에 걸쳐 프로비저닝됩니다. API 요청은 로드 밸런서를 통해 각각 다른 영역에 있는 HA 인스턴스 노드로 라우팅됩니다.

HA 인스턴스 노드 또는 영역에 장애가 발생하면, 서비스는 남아 있는 HA 인스턴스 노드를 통해 API 요청을 계속 처리합니다. 영역 장애 발생 시 서비스 성능 저하는 눈에 띄지 않습니다.

다음 지역에서 사용할 수 있습니다:

- 워싱턴 DC (us-east)
- 프랑크푸르트 (eu-de)

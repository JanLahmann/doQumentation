이 가이드는 이징 모델의 시간 진화를 예시로 사용하여 `qiskit-addon-mpf` 패키지를 사용하는 방법을 설명합니다. 이 패키지를 사용하면 관측값 측정에서 더 낮은 Trotter 오류를 달성할 수 있는 다중 곱 공식(MPF)을 구축할 수 있습니다. 제공된 도구를 통해 선택한 MPF의 가중치를 결정할 수 있으며, 이를 사용하여 서로 다른 수의 Trotter 스텝을 가진 여러 시간 진화 Circuit에서 추정된 기댓값을 재결합할 수 있습니다.

먼저 10개의 사이트를 가진 이징 모델의 해밀토니안을 고려합니다:

$$ H_{\text{Ising}} = \sum_{i=1}^9 J_{i,(i+1)}Z_iZ_{i+1} + \sum_{i=1}^{10} h_i X_i $$

여기서 $J_{i,(i+1)}$은 결합 강도이고 $h_i$는 외부 자기장 강도입니다. 문제를 설정하기 위해 측정할 관측값은 시스템의 총 자화량입니다.

$$ \langle M \rangle = \sum_{i=1}^{10} \langle Z_i \rangle. $$

아래 코드 스니펫은 `qiskit-addon-utils` 패키지를 사용하여 이징 체인의 해밀토니안을 준비하고 측정할 관측값을 정의합니다.

```python
from qiskit.transpiler import CouplingMap
from qiskit.synthesis import SuzukiTrotter
from qiskit.quantum_info import SparsePauliOp
from qiskit.primitives import StatevectorEstimator
from qiskit.providers.fake_provider import GenericBackendV2
from qiskit.transpiler import generate_preset_pass_manager
from qiskit_addon_utils.problem_generators import (
    generate_xyz_hamiltonian,
    generate_time_evolution_circuit,
)
from qiskit_addon_mpf.costs import (
    setup_exact_problem,
    setup_sum_of_squares_problem,
)
from qiskit_addon_mpf.static import setup_static_lse

from scipy.linalg import expm
import numpy as np

# Generate some coupling map to use for this example
coupling_map = CouplingMap.from_line(10, bidirectional=False)

# Get a qubit operator describing the Ising field model
hamiltonian = generate_xyz_hamiltonian(
    coupling_map,
    coupling_constants=(0.0, 0.0, 1.0),
    ext_magnetic_field=(0.4, 0.0, 0.0),
)
print(f"Hamiltonian:\n {hamiltonian}\n")

L = coupling_map.size()
observable = SparsePauliOp.from_sparse_list(
    [("Z", [i], 1 / L / 2) for i in range(L)], num_qubits=L
)
print(f"Observable:\n {observable}")
```

다음으로 MPF를 준비합니다. 첫 번째로 선택해야 할 사항은 계수가 정적(시간 독립적)인지 동적인지 여부입니다. 이 튜토리얼에서는 정적 MPF를 사용합니다. 다음으로 선택해야 할 사항은 $k_j$ 값의 집합입니다. 이는 MPF의 항 수와 각 항이 시간 진화를 시뮬레이션하는 데 사용하는 Trotter 스텝 수를 결정합니다. $k_j$ 값의 선택은 휴리스틱하므로 "좋은" $k_j$ 값의 집합을 직접 구해야 합니다. 좋은 값 집합을 찾는 지침은 [시작하기 페이지](/guides/qiskit-addons-mpf#determine-mpf-coefficients) 끝부분에서 읽을 수 있습니다.

그런 다음 $k_j$ 값이 결정되면 풀어야 할 연립 방정식 $Ax=b$를 준비할 수 있습니다. 행렬 $A$도 사용할 곱 공식에 의해 결정됩니다. 여기서 선택 사항은 이 예제에서 $2$로 설정된 *차수*와 곱 공식이 *대칭*이어야 하는지 여부입니다. 이 예제에서는 `True`로 설정됩니다. 아래 코드 스니펫은 시스템을 진화시킬 총 시간, 사용할 $k_j$ 값, [`qiskit_addon_mpf.static.setup_static_lse`](../api/qiskit-addon-mpf/static) 메서드를 사용하여 풀 방정식 집합을 선택합니다.

```python
time = 8.0
trotter_steps = (8, 12, 19)

lse = setup_static_lse(trotter_steps, order=2, symmetric=True)
print(lse)
```

```text

       [1.56250000e-02, 6.94444444e-03, 2.77008310e-03],

```

선형 연립 방정식이 인스턴스화되면 정확하게 풀거나 최소 제곱합(또는 동적 계수의 경우 프로베니우스 노름; 자세한 내용은 [API 참조](../api/qiskit-addon-mpf/costs) 참조)을 사용하는 근사 모델을 통해 풀 수 있습니다. 근사 모델을 사용하는 선택은 일반적으로 선택된 $k_j$ 값 집합에 대한 계수의 노름이 너무 높다고 판단되고 다른 $k_j$ 값 집합을 선택할 수 없을 때 발생합니다. 이 가이드는 결과를 비교하기 위해 두 가지 모두를 설명합니다.

```python
model_exact, coeffs_exact = setup_exact_problem(lse)
model_approx, coeffs_approx = setup_sum_of_squares_problem(
    lse, max_l1_norm=3.0
)
model_exact.solve()
model_approx.solve()
print(f"Exact solution: {coeffs_exact.value}")
print(f"Approximate solution: {coeffs_approx.value}")
```

```text
Exact solution: [ 0.17239057 -1.19447005  2.02207947]
Approximate solution: [-0.40454257  0.57553173  0.8290123 ]
```

<Admonition type="note">

`LSE` 객체는 연립 방정식을 정확하게 풀 수 있는 `LSE.solve()` 메서드도 가지고 있습니다. 이 가이드에서 `setup_exact_problem()`을 사용하는 이유는 다른 근사 방법이 제공하는 인터페이스를 설명하기 위함입니다.

## Trotter Circuit 설정 및 실행 {#set-up-and-execute-trotter-circuits}

계수 $x_j$를 구했으므로 마지막 단계는 MPF의 차수와 선택된 스텝 집합 $k_j$에 대한 시간 진화 Circuit을 생성하는 것입니다. `qiskit-addon-utils` 패키지를 사용하면 이 과정을 가속화할 수 있습니다.

```python
circuits = []
for k in trotter_steps:
    circ = generate_time_evolution_circuit(
        hamiltonian,
        synthesis=SuzukiTrotter(order=2, reps=k),
        time=time,
    )
    circuits.append(circ)
```

이 Circuit들이 구성되면 QPU를 사용하여 Transpiler 처리하고 실행할 수 있습니다. 이 예제에서는 Trotter 오류가 어떻게 감소하는지 시연하기 위해 노이즈 없는 시뮬레이터 중 하나를 사용합니다.

```python
backend = GenericBackendV2(num_qubits=10)
transpiler = generate_preset_pass_manager(
    optimization_level=2, backend=backend
)

transpiled_circuits = [transpiler.run(circ) for circ in circuits]

estimator = StatevectorEstimator()
job = estimator.run([(circ, observable) for circ in transpiled_circuits])
result = job.result()

mpf_evs = [res.data.evs for res in result]
print(mpf_evs)
```

## 결과 재구성 {#reconstruct-results}

Circuit이 실행되었으므로 결과를 재구성하는 것은 매우 간단합니다. [MPF 개요](./qiskit-addons-mpf#theoretical-background) 페이지에서 언급했듯이 관측값은 가중 합산을 통해 재구성됩니다.

$$ \langle M \rangle = \sum_j x_j \langle M_j \rangle. $$

여기서 $x_j$는 우리가 찾은 계수이고 $\langle M_j \rangle$은 각 Circuit에 대한 관측값 $\sum_i \langle Z_i \rangle$의 추정값입니다. 그런 다음 `scipy.linalg` 패키지를 사용하여 얻은 결과를 정확한 값과 비교할 수 있습니다.

```python
exp_H = expm(-1j * time * hamiltonian.to_matrix())
initial_state = np.zeros(exp_H.shape[0])
initial_state[0] = 1.0

time_evolved_state = exp_H @ initial_state
exact_obs = (
    time_evolved_state.conj() @ observable.to_matrix() @ time_evolved_state
)

# Print out the different observable measurements
print(f"Exact value: {exact_obs.real}")
print(f"PF with 19 steps: {mpf_evs[-1]}")
print(f"MPF using exact solution: {mpf_evs @ coeffs_exact.value}")
print(f"MPF using approximate solution: {mpf_evs @ coeffs_approx.value}")
```

```text
Exact value: 0.4006024248789992
PF with 19 steps: 0.3864990619977402
MPF using exact solution: 0.3954847855979902
MPF using approximate solution: 0.4299121425348959
```

여기서 MPF가 $k_j=19$인 단일 PF만으로 얻은 것에 비해 Trotter 오류를 감소시켰음을 확인할 수 있습니다. 그러나 근사 모델은 정확한 모델보다 더 나쁜 기댓값을 생성했습니다. 이는 근사 모델에서 엄격한 수렴 기준을 사용하고 "좋은" $k_j$ 값 집합을 찾는 것의 중요성을 보여줍니다.

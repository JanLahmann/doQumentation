# 소스에서 Qiskit SDK 및 Qiskit Runtime 설치하기 {#install-the-qiskit-sdk-and-qiskit-runtime-from-source}

소스에서 Qiskit SDK를 설치하면 Python Package Index(PyPI) 저장소의 버전 대신 현재 개발 버전에 접근할 수 있습니다. 이를 통해 최신 버전의 Qiskit 코드를 더 효율적으로 검사하고 확장할 수 있습니다.

## 새 가상 환경 생성 및 활성화 {#create-and-activate-a-new-virtual-environment}

1. 프로젝트 디렉터리로 이동하여 Python만 설치된 최소한의 환경을 생성합니다.

        <OperatingSystemTabs>
          <TabItem value="mac" label="macOS">
            ```shell
            python3 -m venv .venv
            ```
          </TabItem>

          <TabItem value="linux" label="Linux">
            ```shell
            python3 -m venv .venv
            ```
          </TabItem>

          <TabItem value="win" label="Windows">
            ```text
            python -m venv .venv
            ```
          </TabItem>
         </OperatingSystemTabs>

1. 새 환경을 활성화합니다.

         <OperatingSystemTabs>
           <TabItem value="mac" label="macOS">
             ```shell
             source .venv/bin/activate
             ```
           </TabItem>

           <TabItem value="linux" label="Linux">
             ```shell
             source .venv/bin/activate
             ```
           </TabItem>

           <TabItem value="win" label="Windows">
             ```text
             .venv\Scripts\Activate.ps1
             ```
           </TabItem>
         </OperatingSystemTabs>

## Rust 컴파일러 설치 {#install-the-rust-compiler}

Qiskit을 컴파일하려면 시스템에 Rust 컴파일러가 설치되어 있어야 합니다. Rust 컴파일러를 설치하려면 크로스 플랫폼 Rust 설치 도구인 [`rustup`](https://rustup.rs/) 또는 [다른 설치 방법](https://forge.rust-lang.org/infra/other-installation-methods.html)을 사용하세요.

## Qiskit 설치 {#install-qiskit}

다음 단계에 따라 Qiskit을 설치합니다.

1. Qiskit 저장소를 클론합니다.

```bash
git clone https://github.com/Qiskit/qiskit.git
```

2. `qiskit` 디렉터리로 이동합니다.

```bash
cd qiskit
```

3. (선택 사항) 테스트 또는 린팅 검사를 실행하려면 개발자 요구 사항을 설치합니다.

```bash
pip install -r requirements-dev.txt
```

4. `qiskit`을 설치합니다.

  * **표준 설치**:

    ```bash
    pip install .
    ```

  * **편집 가능 모드**: 이 모드에서는 프로젝트에 코드 변경이 있어도 Qiskit을 다시 설치할 필요가 없습니다.

    ```bash
    pip install -e .
    ```

    편집 가능 모드에서는 컴파일된 확장이 최적화 없이 _디버그 모드_로 빌드됩니다. 이는 컴파일된 코드의 런타임 성능에 영향을 줍니다. 최적화를 활성화하여 컴파일된 확장을 빌드하려면 다음 명령을 실행하여 바이너리를 _릴리스 모드_로 다시 빌드하세요.

    ```bash
    python setup.py build_rust --release --inplace
    ```

    <Admonition type="note">
    Qiskit에서 Rust 코드 작업을 하는 경우, 로컬에서 변경할 때마다 확장 코드를 다시 빌드해야 합니다. 편집 가능 모드에서는 설치 명령이 실행될 때만 Rust 확장이 빌드되므로, `build_rust`를 다시 실행하여 확장을 재빌드하지 않으면 Rust 코드에 대한 로컬 변경 사항이 설치된 패키지에 반영되지 않습니다(릴리스 모드 또는 디버그 모드 빌드 여부에 따라 `--release` 옵션 사용 여부를 결정하세요).
    </Admonition>

## Qiskit Runtime 설치 {#install-qiskit-runtime}

Qiskit Runtime을 설치하려면 다음 단계를 따릅니다.

1. Qiskit Runtime 저장소를 클론합니다.

```bash
git clone https://github.com/Qiskit/qiskit-ibm-runtime.git
```

2. `qiskit-ibm-runtime` 디렉터리로 이동합니다.

```bash
cd qiskit-ibm-runtime
```

3. `qiskit-runtime`을 설치합니다. 전역 Python 설치를 오염시키지 않으려면 [가상 환경](https://docs.python.org/3/library/venv.html)을 사용하는 것을 권장합니다.

  * **표준 설치**:

    ```bash
    pip install .
    ```

  * **편집 가능 모드**: 이 모드에서는 프로젝트에 코드 변경이 있어도 Qiskit을 다시 설치할 필요가 없습니다.

    ```bash
    pip install -e .
    ```

    편집 가능 모드에서는 컴파일된 확장이 최적화 없이 _디버그 모드_로 빌드됩니다.

4. (선택 사항) 테스트 또는 린팅 검사를 실행하려면 개발자 요구 사항을 설치합니다. 전역 Python 설치를 오염시키지 않으려면 [가상 환경](https://docs.python.org/3/library/venv.html)을 사용하는 것을 권장합니다.

```bash
pip install -e ".[dev]"
```

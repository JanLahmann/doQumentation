# IBM Cloud 사용자 관리 {#manage-ibm-cloud-users}

이 가이드는 IBM Cloud&reg;를 사용하여 IBM Cloud 계정을 가진 사용자를 활성화하는 방법을 설명하며, 사용자가 환경에 접근하기 위한 지침을 제공합니다.

ID 공급자 사용자를 대신 관리하려면 [ID 공급자 관리](/guides/manage-appid) 항목의 지침을 따르세요.

## 단계 {#steps}

IBM Cloud 사용자를 관리하려면 다음 상위 수준 단계를 따르세요:

1. 초대하려는 사용자가 IBM Cloud 계정을 보유하고 있는지 확인합니다.
2. IAM으로 이동하여 [사용자를 초대](https://cloud.ibm.com/iam/users/invite_users)합니다.
   <Admonition type="note">
   사용자는 초대를 수락하고 최소 한 번 로그인할 때까지 관리할 수 없습니다.
   </Admonition>
3. 선택 사항: [IAM](https://cloud.ibm.com/iam/users)에서 사용자의 프로젝트 할당 및 액세스 그룹을 수정합니다.

<span id="user-org"></span>
## 사용자 흐름 {#user-flow}

1. 사용자가 초대를 수락한 후 [IBM Cloud 포털](https://cloud.ibm.com/)을 통해 로그온합니다.
2. 사용자가 ([관리 → 액세스(IAM) → API 키](https://cloud.ibm.com/iam/apikeys))로 이동하여 API 키를 생성합니다.
3. 추가 정보는 사용자가 [Qiskit 설치](/guides/install-qiskit)를 검토할 수 있습니다.

<span id="steps-cloud-org"></span>
## 예시 시나리오 {#example-scenario}

이 예시는 다음 설정을 생성합니다:

* 두 개의 프로젝트 `ml`과 `finance`가 있습니다.
   * `ml` 프로젝트는 서비스 인스턴스 `QR-ml`과 `QR-common`에 접근할 수 있어야 합니다.
   * `finance` 프로젝트는 서비스 인스턴스 `QR-finance`와 `QR-common`에 접근할 수 있어야 합니다.
* 세 명의 사용자가 있습니다:
   * Fatima는 `ml` 프로젝트에 접근할 수 있어야 합니다.
   * Ravi는 `finance` 프로젝트에 접근할 수 있어야 합니다.
   * Amyra는 두 프로젝트 모두에 접근할 수 있어야 합니다.
* 리소스 그룹 없이 액세스 그룹을 사용합니다.
* 사용자는 IBM Cloud에서 정의되며 프로젝트 할당도 해당 환경에서 이루어집니다.
* 사용자는 작업을 삭제할 수 있어야 합니다.

이 설정을 구현하는 단계는 다음과 같습니다:

2. Cloud 관리자가 세 개의 서비스 인스턴스를 생성합니다: `QR-ml`, `QR finance`, `QR-common`.
3. Cloud 관리자가 `quantum-computing.job.delete` 액션을 포함하는 사용자 정의 규칙을 생성합니다.
1. Cloud 관리자가 두 개의 액세스 그룹을 생성합니다:
   * `ml` 액세스 그룹은 `QR-ml`과 `QR-common`에 접근할 수 있습니다.
   * `finance` 액세스 그룹은 `QR-finance`와 `QR-common`에 접근할 수 있습니다.
1. Cloud 관리자가 적절한 프로젝트에 클라우드 사용자를 초대합니다. 구체적으로, 프로젝트를 포함하는 액세스 그룹에 사용자를 초대하고 할당합니다.
   * Fatima는 "ml" 액세스 그룹에 추가됩니다.
   * Ravi는 "finance" 액세스 그룹에 추가됩니다.
   * Amyra는 "ml"과 "finance" 액세스 그룹 모두에 추가됩니다.
1. 사용자는 IBM Cloud 포털을 통해 로그인하고, API 키를 생성하며, 프로젝트의 서비스 인스턴스로 작업할 수 있습니다.

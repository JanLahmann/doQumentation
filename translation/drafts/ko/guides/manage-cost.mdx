# 비용 관리 {#manage-cost}

이 가이드는 종량제 플랜 인스턴스에 비용 한도를 설정하는 방법을 설명합니다. 또한 [워크로드의 양자 시간 최소화](/guides/minimize-time) 및 워크로드에 [최대 실행 시간](/guides/max-execution-time) 설정을 통해 비용을 관리할 수 있습니다.

_비용 한도_는 인스턴스가 생성된 이후 해당 인스턴스로 실행된 모든 워크로드의 총 비용을 의미하며, 항상 총 비용보다 크거나 같습니다. 인스턴스가 지정된 총 초 수에 도달하면 더 이상 워크로드를 실행할 수 없으며 추가 비용도 발생하지 않습니다.

    <Admonition type="note">
      비용 한도는 항상 미국 달러(USD)로 지정된 후 런타임 초로 환산됩니다. 그러나 월별 청구 목적으로는 IBM Cloud&reg; 계정에 지정된 현지 통화로 청구됩니다. 환율은 변동될 수 있으므로, X 런타임 초에 대한 비용이 처음 USD로 계산될 때와 실제로 현지 통화로 청구될 때 다를 수 있습니다. 따라서 현지 통화가 USD가 아닌 경우, 이 필드에 지정한 초 수에 대해 청구되는 총 금액은 지정한 달러 금액과 다를 수 있습니다.
    </Admonition>

<span id="cost-limit"></span>
## 비용 한도 설정 또는 변경 {#set-or-change-a-cost-limit}

지출 한도를 설정하거나 변경하는 방법은 여러 가지가 있습니다:

* **IBM Quantum Platform**: [인스턴스](https://quantum.cloud.ibm.com/instances) 페이지를 엽니다. 한도를 설정하려는 인스턴스를 찾아 행 끝의 작업 메뉴를 클릭한 다음 **세부 정보 편집**을 클릭합니다. 총 비용 한도 또는 사용량 한도를 설정한 후 **변경 사항 저장**을 클릭합니다.
   <Admonition type="note">
   종량제 플랜에서 새 인스턴스를 생성할 때도 비용 또는 사용량 한도를 설정할 수 있습니다. 종량제 타일을 선택하면 "총 비용 한도 (선택 사항)"라는 레이블이 있는 필드가 나타납니다. 이 필드에 총 비용 한도를 입력할 수 있습니다.
   </Admonition>

* **API**: [인스턴스 한도를 초 단위로 설정합니다.](https://docs.quantum.ibm.com/api/qiskit-runtime-rest/tags/instances#tags__instances__operations__replace_instance_data)

   ```api
   PUT /v1/instances/configuration "instance_limit":<seconds>
   ```

* **IBM Cloud CLI**: CLI 사용 지침은 [IBM Cloud CLI 시작하기](https://cloud.ibm.com/docs/cli?topic=cli-getting-started)를 참조하세요.

   ```cli
      ibmcloud resource service-instance-update <instance_CRN>
      -p '{"instance_limit_seconds":"<seconds>"}'
   ```

## 비용 한도 확인 {#view-the-cost-limit}

현재 비용 한도를 확인하는 방법은 여러 가지가 있습니다:

* **IBM Quantum Platform**: 최대 비용 한도는 [인스턴스](https://quantum.cloud.ibm.com/instances) 표의 종량제 탭에 표시됩니다.

* **API**:

   ```api
   GET /v1/instances/configuration
   ```
   [REST API 문서](https://docs.quantum.ibm.com/api/qiskit-runtime-rest/tags/instances)를 참조하세요.

* **IBM Cloud CLI**: CLI 사용 지침은 [IBM Cloud CLI 시작하기](https://cloud.ibm.com/docs/cli?topic=cli-getting-started)를 참조하세요.

   ```cli
   ibmcloud resource service-instance-update <instance_CRN> -p
   '{"instance_limit_seconds":null}'
   ```

## 비용 한도 제거 {#remove-a-cost-limit}

비용 한도를 제거하는 방법은 여러 가지가 있습니다:

* **IBM Quantum Platform**: [인스턴스](https://quantum.cloud.ibm.com/instances) 표를 엽니다. 업데이트하려는 인스턴스를 찾아 행 끝의 오버플로 메뉴를 클릭한 다음 **세부 정보 편집**을 클릭합니다. 총 비용 한도 또는 사용량을 삭제한 후 **변경 사항 저장**을 클릭합니다.

* **API**: [인스턴스 한도를 `null`로 설정합니다.](https://docs.quantum.ibm.com/api/qiskit-runtime-rest/tags/instances)

   ```api
   PUT /v1/instances/configuration "instance_limit":null
   ```

* **IBM Cloud CLI**: CLI 사용 지침은 [IBM Cloud CLI 시작하기](https://cloud.ibm.com/docs/cli?topic=cli-getting-started)를 참조하세요.

   ```cli
   ibmcloud resource service-instance-update <instance_CRN> -p
   '{"instance_limit_seconds":null}'
   ```
<Admonition type="note">
관리자와 소유자만 인스턴스를 편집할 수 있습니다. 액세스 권한 부여에 대한 정보는 [액세스 정책 및 액세스 그룹 생성](/guides/access-groups) 가이드를 참조하세요.
</Admonition>

### 비용 한도에 도달했을 때 발생하는 일 {#what-happens-when-the-cost-limit-is-reached}

인스턴스의 비용 한도에 도달하면 현재 실행 중인 작업이 중지됩니다. 해당 작업의 상태는 "너무 오래 실행됨"이라는 이유로 취소됨으로 설정됩니다. 사용 가능한 부분적인 결과는 유지됩니다.

비용 한도가 증가될 때까지 이 인스턴스를 사용하여 추가 작업을 제출할 수 없습니다.

## 청구 금액 확인 방법 {#how-to-see-what-youre-being-charged}

리소스 요금에 대한 세부 정보를 제공하는 월별 청구서를 받게 됩니다. [IBM Cloud 청구 및 사용량](https://cloud.ibm.com/billing) 페이지에서 언제든지 지출 금액을 확인할 수 있습니다.

또한 언제든지 인스턴스별 또는 작업별 비용을 확인할 수 있습니다.

### 사용량 확인 {#view-usage}

[분석](https://quantum.cloud.ibm.com/analytics) 페이지에서 인스턴스의 사용량을 확인하세요.

### 인스턴스 비용 확인 {#view-instance-cost}

현재 청구 주기 동안 인스턴스에 청구된 금액을 확인하려면 [인스턴스](https://quantum.cloud.ibm.com/instances) 페이지에서 인스턴스를 클릭하여 세부 정보 페이지를 엽니다.

비용과 관련된 필드는 다음과 같습니다:

- **청구 주기 사용량**: 현재 청구 주기 동안 이 인스턴스가 사용한 _양자 시간_ 양입니다. 양자 시간은 QPU가 사용자 요청을 처리하는 데 전념하는 시간입니다.
- **청구 주기 비용**: 현재 청구 주기 동안 작업을 실행한 총 비용입니다.
- **총 사용량**: 인스턴스가 생성된 이후 사용한 양자 시간 양입니다.
- **총 비용**: 인스턴스가 생성된 이후 이 인스턴스에서 작업을 실행한 총 비용입니다. 관리자만 이 값을 설정할 수 있습니다.

[청구 및 사용량 페이지](https://cloud.ibm.com/billing)에서 청구 주기를 확인할 수 있습니다.

### 작업 비용 확인 {#view-job-cost}

인스턴스와 연결된 각 작업에 청구된 금액을 확인하려면 [인스턴스](https://quantum.cloud.ibm.com/instances) 페이지에서 인스턴스를 클릭하여 세부 정보 페이지를 엽니다. 그런 다음 왼쪽에서 작업을 클릭합니다.

비용과 관련된 열은 다음과 같습니다:

- **사용량**: 이 작업이 사용한 양자 시간 양입니다. 양자 시간은 QPU가 사용자 요청을 처리하는 데 전념하는 시간입니다.
- **비용**: 이 작업을 실행한 총 비용입니다.

## 지출 알림 설정 {#set-up-spending-notifications}

계정 또는 특정 서비스가 설정한 특정 지출 임계값에 도달할 때 알림을 받도록 지출 알림을 설정할 수 있습니다. 자세한 내용은 [IBM Cloud 계정 유형](https://cloud.ibm.com/docs/account?topic=account-accounts) 페이지를 참조하세요. IBM Cloud 지출 알림은 여러 가지 이유로 다른 비용 관리 방법과 함께 사용해야 합니다:

*   알림은 비용이 지정된 한도를 초과한 *후에만* 트리거됩니다.
*   비용은 시간별로 청구 시스템에 제출됩니다. 따라서 작업 제출과 지출 알림 발송 사이에 긴 지연이 발생할 수 있습니다.
*   청구 시스템이 청구서 발행 시스템에 정보를 전달하는 데 며칠이 걸릴 수 있어 알림이 추가로 지연될 수 있습니다. IBM Cloud 청구 시스템의 작동 방식에 대한 자세한 내용은 [지출 알림 설정](https://cloud.ibm.com/docs/billing-usage?topic=billing-usage-spending)을 참조하세요.

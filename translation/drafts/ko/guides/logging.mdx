# Qiskit Runtime 로깅 {#logging-for-qiskit-runtime}

Qiskit Runtime과 같은 IBM Cloud&reg; 서비스는 플랫폼 로그를 생성합니다. 이 로그를 사용하여 계정의 비정상적인 활동 및 중요한 작업을 조사하고 문제를 해결할 수 있습니다.

[IBM Cloud Logs Routing](https://cloud.ibm.com/docs/logs-router?topic=logs-router-about)은 플랫폼 서비스로, 플랫폼 로그를 전송할 위치를 정의하는 테넌트를 구성하여 계정의 플랫폼 로그를 원하는 대상으로 라우팅하는 데 사용할 수 있습니다. 자세한 내용은 [Logs Routing 정보](https://cloud.ibm.com/docs/logs-router?topic=logs-router-about)를 참조하세요.

[IBM Cloud Logs](https://cloud.ibm.com/docs/cloud-logs)를 사용하면 계정에서 생성되고 IBM Cloud Logs Routing을 통해 IBM Cloud Logs 인스턴스로 라우팅된 플랫폼 로그를 시각화하고 알림을 설정할 수 있습니다.

## 플랫폼 로그가 생성되는 위치 {#locations-where-platform-logs-are-generated}

IBM Cloud Container Registry는 로깅 인스턴스에 표시되는 플랫폼 로그를 생성합니다. 다음 표에는 Container Registry 서비스 로그의 자동 수집이 활성화된 위치가 나열되어 있습니다.

| 위치                        | 플랫폼 서비스 로그 사용 가능 여부 |
|-----------------------------|----------------------------------|
| 워싱턴 D.C. (us-east)      | 예                               |
| 프랑크푸르트 (eu-de)        | 예                               |

## 로그 유형별 필드 {#fields-by-log-type}

모든 플랫폼 로그에 포함된 필드에 대한 정보는 [플랫폼 로그에 포함된 필드](https://cloud.ibm.com/docs/logs-router?topic=logs-router-about-platform-logs#about-platform-logs-2)를 참조하세요.

다음 필드가 로그 레코드에 포함됩니다.

| 필드            | 유형     | 설명                                                              |
|-----------------|----------|------------------------------------------------------------------|
| logSourceCRN    | 필수     | 로그가 게시되는 계정 및 플로우 로그 인스턴스를 정의합니다.       |
| saveServiceCopy | 필수     | IBM이 운영 목적으로 레코드 사본을 저장하는지 여부를 정의합니다.  |
| message         | 필수     | 생성된 로그에 대한 설명입니다.                                    |

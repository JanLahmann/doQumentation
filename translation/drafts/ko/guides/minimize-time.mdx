# 작업 실행 시간 최소화 {#minimize-job-run-time}

작업을 처리하고 실행하는 데 소요되는 시간을 제한하는 방법은 여러 가지가 있습니다.

- 필요한 만큼만 샷을 실행하세요: 작업에 소요되는 양자 시간(및 그에 따른 비용)은 샷 수에 비례하여 증가합니다. 따라서 필요한 만큼만 샷을 실행하여 비용을 관리할 수 있습니다. Estimator 작업의 경우, 정밀도가 낮을수록 일반적으로 더 많은 샷이 필요하므로 실행 시간이 길어집니다.

- 실행 시간에 제한을 설정하세요: 각 작업, 배치 또는 세션이 실행되는 시간을 제한할 수 있습니다. 자세한 내용은 [Qiskit Runtime 워크로드의 최대 실행 시간](max-execution-time)을 참조하세요.

- 오류 억제 및 오류 완화에 필요한 설정만 사용하세요. 값이 높을수록 작업 실행 시간이 길어질 수 있습니다. 자세한 내용은 [옵션 소개](./runtime-options-overview), [오류 억제 구성](./configure-error-suppression), [오류 완화 구성](configure-error-mitigation)을 참조하세요.

- 동일한(주로 매개변수화된) Circuit이 포함된 여러 작업을 실행하고 있으며, PEA 및 PEC와 같이 노이즈 모델이 필요한 오류 완화 방법을 사용하는 경우 `NoiseLearner` 사용을 고려하세요. 이 헬퍼 프로그램을 사용하면 Circuit의 노이즈 모델을 한 번 학습하고 이후의 Estimator 쿼리에서 해당 모델을 재사용할 수 있습니다. 노이즈 모델은 일정 시간이 지나면 유효하지 않게 되므로, 작업 사이에 긴 지연이 없는 경우에만(예: 세션 내에서) 실용적입니다. 자세한 내용은 [노이즈 학습 헬퍼
](./noise-learning)를 참조하세요.

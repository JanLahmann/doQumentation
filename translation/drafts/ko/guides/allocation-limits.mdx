# 인스턴스 할당 한도 설정 {#set-instance-allocation-limits}

계정에는 Premium 플랜 또는 Flex 플랜에서 할당된 시간이 있을 수 있습니다. 각 계정은 하나의 플랜에 여러 인스턴스를 가질 수 있으므로, 할당 한도를 설정하면 어떤 인스턴스가 해당 시간을 사용할 수 있는지 제어할 수 있습니다. 할당량이 한도로 설정되지 않은 경우, 인스턴스는 할당된 시간을 초과하여 사용할 수 있다는 점에 유의하세요.

인스턴스에 할당한 시간은 공정 배분 알고리즘에서 우선순위를 결정하는 데 사용됩니다. 이 우선순위는 28일 롤링 시간 창(Premium 플랜 사용자의 경우) 또는 구독 기간(Flex 플랜 사용자의 경우) 동안 모든 QPU의 사용량을 기반으로 산정됩니다.

할당량은 인스턴스를 생성할 때 설정됩니다. 그러나 [인스턴스 페이지](https://quantum.cloud.ibm.com/instances)에서 할당된 시간을 조회하고 변경할 수 있습니다.

## 한도에 도달했을 때 발생하는 일 {#what-happens-when-the-limit-is-reached}

시간 할당량을 인스턴스의 한도로 선택적으로 설정할 수 있습니다. 인스턴스가 한도에 도달하면 다음과 같은 일이 발생합니다:

- 활성 워크로드는 계속 실행됩니다.
- 대기 중인 작업은 큐에 남아 있습니다.
- 취소되는 워크로드는 없습니다.
- 협업자는 새 워크로드를 제출할 수 있으며, 더 많은 시간이 확보될 때까지 큐에 대기합니다.
- Flex 플랜의 경우, 한도가 늘어나지 않으면 인스턴스는 더 이상 작업을 처리할 수 없습니다. Premium 플랜의 경우, 다음 사용 창이 시작되면 처리가 재개됩니다.

## 할당 한도와 사용 한도에 서로 다른 값 설정 {#set-different-values-for-the-allocation-limit-and-usage-limit}

IBM Quantum Platform에서 인스턴스의 할당 한도를 설정하면, 동일한 값이 인스턴스의 사용 한도로도 설정됩니다. 인스턴스의 할당 한도에 도달한 이후에도 사용자가 작업을 계속 제출하고 실행할 수 있도록 하려면, [Update a resource instance](https://cloud.ibm.com/apidocs/resource-controller/resource-controller#update-resource-instance) API의 [usage_allocation_seconds](/guides/access-instances-platform-apis#parameters) 파라미터를 사용하여 사용 한도에 더 높은 값을 설정할 수 있습니다.

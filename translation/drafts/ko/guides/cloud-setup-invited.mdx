# IBM Cloud 계정 설정 - 초대된 사용자 {#set-up-your-ibm-cloud-account---invited-users}

관리자가 설정한 계정에 참여하도록 관리자로부터 초대를 받지 못한 경우, 관리자에게 문의하세요. 자신의 계정을 직접 설정해야 한다면 [IBM Cloud 계정 설정](/guides/cloud-setup) 문서의 단계를 따라 주세요.

IBM&reg; 양자 처리 장치(QPU)에서 워크로드를 실행하려면 IBM Cloud&reg;에서 계정을 설정해야 합니다. 또한 [Qiskit 및 Qiskit Runtime을 설치하고 설정](/guides/install-qiskit#local)해야 합니다.

사용자 계정은 하나 이상의 [인스턴스](./instances)와 연결되며, 이는 IBM Quantum 서비스에 대한 접근 권한을 부여하는 _클라우드 리소스 이름_(CRN)으로 식별됩니다. 또한 각 계정에는 고유한 API 토큰(키라고도 함)이 할당되어 Qiskit에서 IBM Quantum에 접근할 수 있습니다. 특정 인스턴스를 선택하는 방법은 [인스턴스에 연결](./instances#connect-instance)을 참고하세요.

    <Admonition type="note">
      [IBM Quantum Platform 대시보드](https://quantum.cloud.ibm.com) 및 [인스턴스 페이지](https://quantum.cloud.ibm.com/instances)에서 접근 가능한 인스턴스 목록을 확인할 수 있습니다.
    </Admonition>

<span id="cloud"></span>
## IBM Cloud 사용 설정 {#set-up-to-use-ibm-cloud}

### 시작하기 전에 {#before-you-begin}

1. 이메일 초대를 수락합니다.
1. IBMid 또는 Google 계정으로 로그인합니다. 이 단계에서 IBM Cloud 또는 IBM Quantum Platform에 로그인할 때 사용하는 프로필은 IBM Cloud 계정이 아닙니다. 로그인 후 정보를 확인하고 계정 참여 초대를 수락합니다.
1. [IBM Quantum Platform](https://quantum.cloud.ibm.com)으로 돌아갑니다.
1. 다음 이미지와 같이 헤더의 계정 전환기에서 올바른 계정과 지역이 선택되어 있는지 확인합니다.
    * 이메일에서 초대받은 계정을 선택합니다.
    * 지역은 작업이 실행되는 위치와 작업 데이터가 보관되는 위치를 결정합니다. 동일한 API 키로 두 지역 모두에 접근할 수 있지만, 현재 로그인된 지역에서 생성된 인스턴스만 확인하고 접근할 수 있습니다.

    ![IBM Quantum Platform 헤더가 표시됩니다. 계정 전환기는 검색창 오른쪽에 있습니다. 지역 전환기는 계정 전환기 오른쪽에 있습니다.](/docs/images/guides/cloud-setup/IQP-Header.svg 'IBM Quantum Platform header')

### 2. 선택 사항: 접근 자격 증명 저장 {#2-optional-save-your-access-credentials}

개인 노트북과 같이 신뢰할 수 있는 Python 환경을 사용하는 경우, `save_account()` 메서드를 사용하여 [자격 증명을 로컬에 저장](/guides/save-credentials)하는 것을 권장합니다.

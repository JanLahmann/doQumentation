<details>
<summary>

Jaki jest stan $|\pi_1\rangle$?

</summary>

__Odpowiedź:__

Zastosowanie Hadamarda przekształca stan $|0\rangle$ w $\frac{1}{\sqrt{2}}(|0\rangle+|1\rangle)$, a stan $|1\rangle$ w $\frac{1}{\sqrt{2}}(|0\rangle-|1\rangle)$. Pełny stan przyjmuje więc postać: $|\pi_1\rangle = [\frac{|0\rangle-|1\rangle}{\sqrt{2}}][\frac{|0\rangle+|1\rangle}{\sqrt{2}}]$

</details>

<details>
<summary>

Jaki jest stan $|\pi_2\rangle$?

</summary>

__Odpowiedź:__

Zanim zastosujemy $U_f$, przypomnijmy, co robi ta operacja. Zmienia ona stan Qubitu 1 na podstawie stanu Qubitu 0. Warto zatem wyciągnąć stan Qubitu 0 przed nawias: $|\pi_1\rangle = \frac{1}{2} (|0\rangle-|1\rangle)|0\rangle+\frac{1}{2}(|0\rangle-|1\rangle)|1\rangle$. Następnie, jeśli $f(0)=f(1)$, oba składniki przekształcają się w ten sam sposób i względny znak między nimi pozostaje dodatni; jeśli natomiast $f(0)\neq f(1)$, drugi składnik zyskuje znak minus względem pierwszego, zmieniając stan Qubitu 0 z $\frac{1}{\sqrt{2}}(|0\rangle+|1\rangle)$ na $\frac{1}{\sqrt{2}}(|0\rangle-|1\rangle)$. Zatem:
$$
|\pi_2\rangle = \begin{cases}
\pm[\frac{|0\rangle-|1\rangle}{\sqrt{2}}][\frac{|0\rangle+|1\rangle}{\sqrt{2}}] & \text{if} & f(0) = f(1) \\
\pm[\frac{|0\rangle-|1\rangle}{\sqrt{2}}][\frac{|0\rangle-|1\rangle}{\sqrt{2}}] &\text{if} & f(0) \neq f(1) \\
\end{cases}
$$

</details>

<details>
<summary>

Jaki jest stan $|\pi_3\rangle$?

</summary>

__Odpowiedź:__

Teraz stan Qubitu 0 wynosi $\frac{1}{\sqrt{2}}(|0\rangle+|1\rangle)$ albo $\frac{1}{\sqrt{2}}(|0\rangle-|1\rangle)$, w zależności od funkcji. Zastosowanie Hadamarda da odpowiednio $|0\rangle$ lub $|1\rangle$.

$$
|\pi_3\rangle = \begin{cases}
\pm[\frac{|0\rangle-|1\rangle}{\sqrt{2}}]|0\rangle & \text{if} & f(0) = f(1) \\
\pm[\frac{|0\rangle-|1\rangle}{\sqrt{2}}]|1\rangle &\text{if} & f(0) \neq f(1) \\
\end{cases}
$$

</details>

Analizując swoje odpowiedzi do powyższych pytań, zwróć uwagę na pewne zaskakujące zjawisko. Choć $U_f$ nie działa bezpośrednio na stan Qubitu 0, to ponieważ zmienia Qubit 1 na podstawie stanu Qubitu 0, może dochodzić do przesunięcia fazowego Qubitu 0. Zjawisko to nosi nazwę „phase-kickback" i jest szczegółowo omówione w [Lekcji 1: Algorytmy kwantowe z zapytaniami](/learning/courses/fundamentals-of-quantum-algorithms/quantum-query-algorithms/introduction) kursu Fundamentals of Quantum Algorithms.

## Algorytm Deutscha-Jozsy {#the-deutsch-jozsa-algorithm}

Algorytm Deutscha był ważnym pierwszym krokiem w demonstracji, że komputer kwantowy może być wydajniejszy od klasycznego, lecz stanowił jedynie skromne usprawnienie: wymagał tylko jednego zapytania, podczas gdy klasycznie potrzeba ich dwóch. W 1992 roku Deutsch i jego współpracownik Richard Jozsa rozszerzyli pierwotny algorytm dwuqubitowy na większą liczbę Qubitów. Problem pozostał ten sam: ustalić, czy funkcja jest *zrównoważona* (*balanced*), czy *stała* (*constant*). Tym razem jednak funkcja odwzorowuje $n$ bitów na jeden bit. Albo funkcja zwraca 0 i 1 po równo (jest *balanced*), albo zawsze zwraca 1 lub zawsze 0 (jest *constant*).

Poniżej przedstawiono schemat Circuit tego algorytmu:

![DJ_algo.png](/learning/images/modules/computer-science/deutsch-jozsa/DJ_algo.avif)

Algorytm działa tak samo jak algorytm Deutscha: phase-kickback pozwala odczytać stan Qubitu 0, aby stwierdzić, czy funkcja jest stała, czy zrównoważona. Zrozumienie tego mechanizmu jest nieco trudniejsze niż w dwuqubitowym przypadku algorytmu Deutscha, ponieważ stany obejmują sumy po $n$ Qubitach — wyprowadzenie tych stanów pozostawiono jako opcjonalne ćwiczenie na końcu modułu. Algorytm zwróci ciąg bitów złożony z samych zer, jeśli funkcja jest stała, lub ciąg zawierający co najmniej jedną jedynkę, jeśli funkcja jest zrównoważona.

Aby zobaczyć, jak algorytm działa w Qiskit, musimy najpierw wygenerować naszą wyrocznię: losową funkcję, która jest gwarantowanie albo stała, albo zrównoważona. Poniższy kod generuje funkcję zrównoważoną z prawdopodobieństwem 50% i funkcję stałą z prawdopodobieństwem 50%. Nie przejmuj się, jeśli nie rozumiesz wszystkich szczegółów kodu — jest on skomplikowany i nie jest niezbędny do zrozumienia algorytmu kwantowego.

```python
from qiskit import QuantumCircuit
import numpy as np

def dj_function(num_qubits):
    """
    Create a random Deutsch-Jozsa function.
    """

    qc_dj = QuantumCircuit(num_qubits + 1)
    if np.random.randint(0, 2):
        # Flip output qubits with 50% chance
        qc_dj.x(num_qubits)
    if np.random.randint(0, 2):
        # return constant circuit with 50% chance.
        return qc_dj

    # If the "if" statement above was "TRUE" then we've returned the constant
    # function and the function is complete. If not, we proceed in creating our
    # balanced function. Everything below is to produce the balanced function:

    # select half of all possible states at random:
    on_states = np.random.choice(
        range(2**num_qubits),  # numbers to sample from
        2**num_qubits // 2,  # number of samples
        replace=False,  # makes sure states are only sampled once
    )

    def add_cx(qc_dj, bit_string):
        for qubit, bit in enumerate(reversed(bit_string)):
            if bit == "1":
                qc_dj.x(qubit)
        return qc_dj

    for state in on_states:
        # qc_dj.barrier()  # Barriers are added to help visualize how the functions are created. They can safely be removed.
        qc_dj = add_cx(qc_dj, f"{state:0b}")
        qc_dj.mcx(list(range(num_qubits)), num_qubits)
        qc_dj = add_cx(qc_dj, f"{state:0b}")

    # qc_dj.barrier()

    return qc_dj

n = 3  # number of input qubits

oracle = dj_function(n)

display(oracle.draw("mpl"))
```

To jest funkcja wyroczni — albo zrównoważona, albo stała. Czy patrząc na ten Circuit, jesteś w stanie ocenić, czy wynik na ostatnim Qubicie zależy od wartości podanych dla pierwszych $n$ Qubitów? Jeśli wynik na ostatnim Qubicie zależy od pierwszych $n$ Qubitów, czy możesz stwierdzić, czy ta zależność jest zrównoważona, czy nie?

Możemy określić, czy funkcja jest zrównoważona, czy stała, patrząc na powyższy Circuit, ale pamiętaj — na potrzeby tego problemu traktujemy tę funkcję jako „czarną skrzynkę". Nie możemy zaglądać do środka, aby obejrzeć schemat Circuit. Zamiast tego musimy odpytać skrzynkę.

Aby odpytać skrzynkę, używamy algorytmu Deutscha-Jozsy i ustalamy, czy funkcja jest stała, czy zrównoważona:

```python
blackbox = oracle.to_gate()
blackbox.label = "$U_f$"

qc_dj = QuantumCircuit(n + 1, n)
qc_dj.x(n)
qc_dj.h(range(n + 1))
qc_dj.barrier()
qc_dj.compose(blackbox, inplace=True)
qc_dj.barrier()
qc_dj.h(range(n))
qc_dj.measure(range(n), range(n))

qc_dj.decompose().decompose()

qc_dj.draw("mpl")
```

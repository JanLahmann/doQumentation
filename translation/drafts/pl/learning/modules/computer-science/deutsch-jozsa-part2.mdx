## Równoległość kwantowa i jej ograniczenia

Część mocy obliczeń kwantowych wynika z "równoległości kwantowej", która zasadniczo polega na zdolności wykonywania operacji na wielu wejściach jednocześnie, ponieważ stany wejściowe Qubitów mogą znajdować się w superpozycji wielu stanów dozwolonych klasycznie. JEDNAK, choć obwód kwantowy może być w stanie ewaluować wiele stanów wejściowych naraz, wydobycie wszystkich tych informacji za jednym razem jest niemożliwe.

Aby zobaczyć, co mam tu na myśli, załóżmy, że mamy bit $x$ oraz pewną funkcję zastosowaną do tego bitu, $f(x)$. Istnieją cztery możliwe funkcje binarne przekształcające pojedynczy bit w inny pojedynczy bit:

| $x$    |  $f_1(x)$ | $f_2(x)$ | $f_3(x)$ | $f_4(x)$ |
| ----------- | ------- |-------| -------- | ------- |
| 0  | 0   | 0  | 1   | 1 |
| 1  | 0   | 1  | 0 | 1 |

Chcielibyśmy się dowiedzieć, którą z tych funkcji (1-4) jest nasze $f(x)$. Klasycznie musielibyśmy uruchomić funkcję dwa razy — raz dla $x=0$, raz dla $x=1$. Ale zobaczmy, czy możemy zrobić to lepiej za pomocą obwodu kwantowego. Możemy poznać funkcję dzięki następującej bramce:

![quantum_parallelism](/learning/images/modules/computer-science/deutsch-jozsa/quantum-parallelism.avif)

Tutaj bramka $U_f$ oblicza $f(x)$, gdzie $x$ jest stanem qubita 0, i nakłada to na qubit 1. Tak więc wynikowy stan, $|x\rangle|y\oplus f(x)\rangle$, po prostu staje się $|x\rangle|f(x)\rangle$, gdy $|y\rangle = |0\rangle$. Zawiera on wszystkie informacje, których potrzebujemy, aby poznać funkcję $f(x)$: qubit 0 mówi nam, czym jest $x$, a qubit 1 mówi nam, czym jest $f(x)$. Jeśli więc zainicjujemy $|x\rangle = \frac{1}{\sqrt{2}}(|0\rangle+|1\rangle)$, to końcowy stan obu qubitów będzie wynosił: $|y\rangle|x\rangle = \frac{1}{\sqrt{2}}(|f(0)\rangle|0\rangle+|f(1)\rangle|1\rangle)$. Ale jak uzyskać dostęp do tych informacji?

### 2.1. Wypróbuj to w Qiskit:

Korzystając z Qiskit, losowo wybierzemy jedną z czterech możliwych funkcji powyżej i uruchomimy obwód. Twoim zadaniem będzie wykorzystanie pomiarów obwodu kwantowego do poznania funkcji w jak najmniejszej liczbie uruchomień.

W tym pierwszym eksperymencie i przez cały moduł będziemy używać frameworka do obliczeń kwantowych zwanego "wzorcami Qiskit" (Qiskit patterns), który dzieli przepływy pracy na następujące kroki:

- Krok 1: Mapowanie klasycznych wejść na problem kwantowy
- Krok 2: Optymalizacja problemu pod kątem wykonania kwantowego
- Krok 3: Wykonanie przy użyciu prymitywów Qiskit Runtime
- Krok 4: Przetwarzanie końcowe i analiza klasyczna

Zacznijmy od załadowania niezbędnych pakietów, w tym prymitywów Qiskit Runtime. Wybierzemy też najmniej obciążony dostępny dla nas komputer kwantowy.

Teraz, gdy załadowaliśmy niezbędne pakiety, możemy przejść do przepływu pracy wzorców Qiskit. W kroku mapowania poniżej najpierw tworzymy funkcję, która wybiera spośród czterech możliwych funkcji przekształcających pojedynczy bit w inny pojedynczy bit.

```python
# Step 1: Map

from qiskit import QuantumCircuit

qc = QuantumCircuit(2)

def twobit_function(case: int):
    """
    Generate a valid two-bit function as a `QuantumCircuit`.
    """
    if case not in [1, 2, 3, 4]:
        raise ValueError("`case` must be 1, 2, 3, or 4.")

    f = QuantumCircuit(2)
    if case in [2, 3]:
        f.cx(0, 1)
    if case in [3, 4]:
        f.x(1)
    return f

# first, convert oracle circuit (above) to a single gate for drawing purposes. otherwise, the circuit is too large to display
# blackbox = twobit_function(2).to_gate()  # you may edit the number inside "twobit_function()" to select among the four valid functions
# blackbox.label = "$U_f$"

qc.h(0)
qc.barrier()
qc.compose(twobit_function(2), inplace=True)
qc.measure_all()

qc.draw("mpl")
```


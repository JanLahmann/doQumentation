Aby zastosować algorytm Grovera, musieliśmy zbudować operator Grovera, co oznacza, że musimy być w stanie odwrócić fazę stanów spełniających nasze kryteria rozwiązania. Nasuwa się pytanie: jeśli znamy zbiór rozwiązań tak dobrze, że możemy zaprojektować Circuit kwantowy oznaczający każde z nich, to czego właściwie szukamy?! Odpowiedź jest trójczęściowa i będziemy do niej wracać przez cały tutorial:

**(1) Tego rodzaju algorytmy zapytań często angażują dwie strony**: jedną, która dysponuje wyrocznią definiującą kryteria rozwiązania, i drugą, która próbuje odgadnąć lub znaleźć stan rozwiązania. Fakt, że jedna osoba może zbudować wyrocznię, nie znosi potrzeby przeszukiwania.

**(2) Istnieją problemy, dla których łatwiej jest określić kryterium rozwiązania niż samo rozwiązanie znaleźć.** Najlepiej znany przykład to identyfikowanie pierwszych czynników dużych liczb. Algorytm Grovera nie jest szczególnie efektywny w rozwiązaniu tego konkretnego problemu; do faktoryzacji pierwszej używalibyśmy algorytmu Shora. To tylko przykład pokazujący, że znajomość kryterium dotyczącego zachowania stanu nie jest równoznaczna ze znajomością samego stanu.

**(3) Algorytm Grovera nie zwraca wyłącznie danych klasycznych.** Rzeczywiście, jeśli dokonamy pomiaru stanu końcowego po $t$ powtórzeniach operatora Grovera, najprawdopodobniej otrzymamy informacje klasyczne identyfikujące stan rozwiązania. Ale co, jeśli nie chcemy informacji klasycznych; co, jeśli chcemy stanu rozwiązania przygotowanego na komputerze kwantowym do dalszego użycia w innym algorytmie? Algorytm Grovera faktycznie produkuje stan kwantowy, w którym rozwiązania mają silnie wzmocnione amplitudy. Możesz więc spodziewać się, że algorytm Grovera będzie pojawiał się jako podprogram w bardziej złożonych algorytmach kwantowych.

Mając to na uwadze, przejdźmy przez kilka przykładów. Zaczniemy od przykładu, w którym stan rozwiązania jest wyraźnie określony, abyśmy mogli śledzić logikę algorytmu, a następnie przejdziemy do przykładów, w których użyteczność algorytmu Grovera staje się bardziej oczywista.

## Krok 1: Odwzorowanie klasycznych danych wejściowych na problem kwantowy {#step-1-map-classical-inputs-to-a-quantum-problem}

Potrzebujemy bramki zapytania fazowego, która przypisuje ogólną fazę (-1) stanom będącym rozwiązaniami i pozostawia niezmienione stany niebędące rozwiązaniami. Inaczej mówiąc, algorytm Grovera wymaga wyroczni wskazującej jeden lub więcej oznaczonych stanów bazowych przestrzeni obliczeniowej, gdzie „oznaczony" oznacza stan z fazą równą -1. Realizuje się to za pomocą bramki controlled-Z lub jej uogólnienia z wieloma bitami kontrolnymi na $N$ Qubitach. Aby zobaczyć, jak to działa, rozważ konkretny przykład ciągu bitów `{110}`. Chcemy Circuit, który działa na stan $|\psi\rangle = |q_2,q_1,q_0\rangle$ i stosuje fazę, gdy $|\psi\rangle = |011\rangle$ (gdzie odwróciliśmy kolejność ciągu binarnego ze względu na notację w Qiskit, która umieszcza Qubit o najmniejszym znaczeniu (często 0) po prawej stronie).

Chcemy więc Circuit $Z_f$ realizującego

$$
Z_f|\psi\rangle = \begin{cases} -|\psi\rangle \qquad \text{if} \qquad |\psi\rangle = |011\rangle \\ |\psi\rangle \qquad \text{if} \qquad |\psi\rangle \neq |011\rangle\end{cases}
$$

Możemy użyć bramki z wieloma bitami kontrolnymi i wieloma celami (`MCMTGate`), aby zastosować bramkę Z kontrolowaną przez wszystkie Qubity (odwrócenie fazy, gdy wszystkie Qubity są w stanie $|1\rangle$). Oczywiście niektóre Qubity w naszym pożądanym stanie mogą być w stanie $|0\rangle$. Dlatego dla tych Qubitów musimy najpierw zastosować bramkę X, następnie wykonać bramkę Z z wielokrotnym sterowaniem, a następnie zastosować kolejną bramkę X, aby cofnąć naszą zmianę. `MCMTGate` wygląda następująco:

```python
mcmt_ex = QuantumCircuit(3)
mcmt_ex.compose(MCMTGate(ZGate(), 3 - 1, 1), inplace=True)
mcmt_ex.draw(output="mpl", style="iqp")
```

Zauważ, że wiele Qubitów może brać udział w procesie sterowania (tutaj trzy Qubity), ale żaden pojedynczy Qubit nie jest oznaczony jako cel. Wynika to z faktu, że cały stan otrzymuje ogólny znak „-" (odwrócenie fazy); Gate oddziałuje na wszystkie Qubity równoważnie. Różni się to od wielu innych bramek wielokubitowych, jak bramka `CX`, która ma jeden Qubit sterujący i jeden Qubit docelowy.

W poniższym kodzie definiujemy bramkę zapytania fazowego (czyli wyrocznię), która robi to, co opisaliśmy powyżej: oznacza jeden lub więcej wejściowych stanów bazowych zdefiniowanych poprzez reprezentację ciągu bitów. Bramka MCMT jest używana do implementacji bramki Z z wielokrotnym sterowaniem.

```python
def grover_oracle(marked_states):
    """Build a Grover oracle for multiple marked states

    Here we assume all input marked states have the same number of bits

    Parameters:
        marked_states (str or list): Marked states of oracle

    Returns:
        QuantumCircuit: Quantum circuit representing Grover oracle
    """
    if not isinstance(marked_states, list):
        marked_states = [marked_states]
    # Compute the number of qubits in circuit
    num_qubits = len(marked_states[0])

    qc = QuantumCircuit(num_qubits)
    # Mark each target state in the input list
    for target in marked_states:
        # Flip target bitstring to match Qiskit bit-ordering
        rev_target = target[::-1]
        # Find the indices of all the '0' elements in bitstring
        zero_inds = [
            ind for ind in range(num_qubits) if rev_target.startswith("0", ind)
        ]
        # Add a multi-controlled Z-gate with pre- and post-applied X-gates (open-controls)
        # where the target bitstring has a '0' entry
        qc.x(zero_inds)
        qc.compose(MCMTGate(ZGate(), num_qubits - 1, 1), inplace=True)
        qc.x(zero_inds)
    return qc
```

Teraz wybieramy konkretny stan „oznaczony" jako nasz cel i stosujemy właśnie zdefiniowaną funkcję. Zobaczmy, jaki Circuit został utworzony.

```python
marked_states = ["1110"]
oracle = grover_oracle(marked_states)
oracle.draw(output="mpl", style="iqp")
```

Jeśli Qubity 1–3 są w stanie $|1\rangle$, a Qubit 0 jest początkowo w stanie $|0\rangle$, pierwsza bramka X odwróci Qubit 0 do stanu $|1\rangle$ i wszystkie Qubity będą w stanie $|1\rangle.$ Oznacza to, że bramka MCMT zastosuje ogólną zmianę znaku lub odwrócenie fazy zgodnie z oczekiwaniami. W każdym innym przypadku albo Qubity 1–3 są w stanie $|0\rangle$, albo Qubit 0 jest przerzucany do stanu $|0\rangle$ i odwrócenie fazy nie zostanie zastosowane. Widzimy, że ten Circuit rzeczywiście oznacza nasz pożądany stan $|0111\rangle,$ czyli ciąg bitów `{1110}`.

Pełny operator Grovera składa się z bramki zapytania fazowego (wyroczni), warstw Hadamarda oraz operatora $Z_\text{OR}$. Możemy użyć wbudowanej funkcji `grover_operator`, aby zbudować go z wyroczni zdefiniowanej powyżej.

```python
grover_op = grover_operator(oracle)
grover_op.decompose(reps=0).draw(output="mpl", style="iqp")
```

![Output of the previous code cell](/learning/images/modules/computer-science/grovers/extracted-outputs/9426f7a5-0.avif)

Jak argumentowaliśmy powyżej, może być konieczne wielokrotne zastosowanie operatora Grovera. Optymalną liczbę iteracji $t$ maksymalizującą amplitudę stanu docelowego przy braku szumów można uzyskać z następującego wyrażenia:
$$
(2t+1)\theta = (2t+1)\sin^{-1}\left( \sqrt{\frac{|A_1|}{N}}\right) \approx (2t+1)\sqrt{\frac{|A_1|}{N}} \approx \frac{\pi}{2}\\
t\approx \frac{\pi}{4} \sqrt{\frac{N}{|A_1|}}-\frac{1}{2}
$$
Tutaj $A_1$ to liczba rozwiązań lub stanów docelowych. Na nowoczesnych zaszumionych komputerach kwantowych eksperymentalnie optymalna liczba iteracji może być różna — tutaj jednak obliczamy i używamy tej teoretycznej, optymalnej liczby przy $A_1=1$.

```python
optimal_num_iterations = math.floor(
    math.pi / (4 * math.asin(math.sqrt(len(marked_states) / 2**grover_op.num_qubits)))
)
print(optimal_num_iterations)
```

```text
3
```

Zbudujmy teraz Circuit zawierający początkowe bramki Hadamarda tworzące superpozycję wszystkich możliwych stanów i zastosujmy operator Grovera optymalną liczbę razy.

```python
qc = QuantumCircuit(grover_op.num_qubits)
# Create even superposition of all basis states
qc.h(range(grover_op.num_qubits))
# Apply Grover operator the optimal number of times
qc.compose(grover_op.power(optimal_num_iterations), inplace=True)
# Measure all qubits
qc.measure_all()
qc.draw(output="mpl", style="iqp")
```


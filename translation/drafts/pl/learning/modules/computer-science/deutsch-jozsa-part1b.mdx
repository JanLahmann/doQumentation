## Równoległość kwantowa i jej ograniczenia

Część mocy obliczeń kwantowych wynika z „równoległości kwantowej" — jest to w istocie zdolność do wykonywania operacji na wielu wejściach jednocześnie, ponieważ stany wejściowe Qubitów mogą znajdować się w superpozycji wielu stanów dopuszczalnych klasycznie. JEDNAK, choć kwantowy Circuit może być w stanie oceniać wiele stanów wejściowych naraz, wyodrębnienie wszystkich tych informacji w jednym podejściu jest niemożliwe.

Żeby pokazać, co mam na myśli, przyjmijmy, że mamy bit $x$ oraz pewną funkcję zastosowaną do tego bitu, $f(x)$. Istnieją cztery możliwe funkcje binarne odwzorowujące jeden bit na jeden bit:

| $x$    |  $f_1(x)$ | $f_2(x)$ | $f_3(x)$ | $f_4(x)$ |
| ----------- | ------- |-------| -------- | ------- |
| 0  | 0   | 0  | 1   | 1 |
| 1  | 0   | 1  | 0 | 1 |

Chcielibyśmy dowiedzieć się, którą z tych funkcji (1–4) jest nasze $f(x)$. Klasycznie musielibyśmy uruchomić funkcję dwa razy — raz dla $x=0$, raz dla $x=1$. Zobaczmy jednak, czy jesteśmy w stanie zrobić to lepiej za pomocą kwantowego Circuit. O funkcji możemy się dowiedzieć, korzystając z następującego Gate'a:

![quantum_parallelism](/learning/images/modules/computer-science/deutsch-jozsa/quantum-parallelism.avif)

Tutaj Gate $U_f$ oblicza $f(x)$, gdzie $x$ to stan Qubitu 0, i stosuje wynik do Qubitu 1. Tak więc wynikowy stan $|x\rangle|y\oplus f(x)\rangle$ staje się po prostu $|x\rangle|f(x)\rangle$, gdy $|y\rangle = |0\rangle$. Zawiera to wszystkie informacje potrzebne do poznania funkcji $f(x)$: Qubit 0 mówi nam, jakie jest $x$, a Qubit 1 mówi nam, jakie jest $f(x)$. Jeśli więc zainicjujemy $|x\rangle = \frac{1}{\sqrt{2}}(|0\rangle+|1\rangle)$, to końcowy stan obu Qubitów będzie wynosił: $|y\rangle|x\rangle = \frac{1}{\sqrt{2}}(|f(0)\rangle|0\rangle+|f(1)\rangle|1\rangle)$. Ale jak uzyskać dostęp do tych informacji?

### 2.1. Wypróbuj to w Qiskit:

Korzystając z Qiskit, losowo wybierzemy jedną z czterech możliwych funkcji powyżej i uruchomimy Circuit. Twoim zadaniem jest wykorzystanie pomiarów kwantowego Circuit do poznania funkcji przy jak najmniejszej liczbie uruchomień.

W tym pierwszym eksperymencie i przez cały moduł będziemy używać frameworka do obliczeń kwantowych zwanego „Qiskit patterns", który dzieli przepływy pracy na następujące kroki:

- Krok 1: Odwzoruj klasyczne dane wejściowe na problem kwantowy
- Krok 2: Zoptymalizuj problem pod kątem wykonania kwantowego
- Krok 3: Wykonaj, używając Qiskit Runtime Primitives
- Krok 4: Post-processing i klasyczna analiza

Zacznijmy od załadowania niezbędnych pakietów, w tym Qiskit Runtime primitives. Wybierzemy również najmniej obciążony dostępny komputer kwantowy.

Teraz, gdy załadowaliśmy niezbędne pakiety, możemy przystąpić do przepływu pracy Qiskit patterns. W kroku odwzorowania poniżej najpierw tworzymy funkcję, która wybiera spośród czterech możliwych funkcji odwzorowujących jeden bit na jeden bit.

```python
# Step 1: Map

from qiskit import QuantumCircuit

qc = QuantumCircuit(2)

def twobit_function(case: int):
    """
    Generate a valid two-bit function as a `QuantumCircuit`.
    """
    if case not in [1, 2, 3, 4]:
        raise ValueError("`case` must be 1, 2, 3, or 4.")

    f = QuantumCircuit(2)
    if case in [2, 3]:
        f.cx(0, 1)
    if case in [3, 4]:
        f.x(1)
    return f

# first, convert oracle circuit (above) to a single gate for drawing purposes. otherwise, the circuit is too large to display
# blackbox = twobit_function(2).to_gate()  # you may edit the number inside "twobit_function()" to select among the four valid functions
# blackbox.label = "$U_f$"

qc.h(0)
qc.barrier()
qc.compose(twobit_function(2), inplace=True)
qc.measure_all()

qc.draw("mpl")
```

Następnie musimy zoptymalizować i wykonać transpilację Circuit, aby uruchomić go na komputerze kwantowym:

```python
# Step 2: Transpile
from qiskit.transpiler.preset_passmanagers import generate_preset_pass_manager

target = backend.target
pm = generate_preset_pass_manager(target=target, optimization_level=3)

qc_isa = pm.run(qc)
```

Na koniec wykonujemy nasz transpilowany Circuit na komputerze kwantowym i wizualizujemy wyniki:

```python
# Step 3: Run the job on a real quantum computer

job = sampler.run([qc_isa], shots=1)
# job = sampler_sim.run([qc_isa],shots=1) # uncomment this line to run on simulator instead
```

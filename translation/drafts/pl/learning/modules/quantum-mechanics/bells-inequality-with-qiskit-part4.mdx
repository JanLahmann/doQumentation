```python
# Zaczniemy od zaimportowania modułu qiskit oraz modułu wizualizacji, abyśmy mogli narysować histogram wyników.
```

Pamiętaj, że 1/3 czasu oś Rihanny będzie oddalona o $2\pi/3$ radianów od osi Lucasa, 1/3 czasu — o $4\pi/3$ radianów od osi Lucasa, a 1/3 czasu oboje wybiorą tę samą oś. Musimy więc zbudować 3 Circuity kwantowe dla tych 3 przypadków i zsumować wyniki. Pierwszy z nich omówimy szczegółowo; dwa pozostałe po prostu podamy.

```python
# Zaczynamy od zadeklarowania pierwszego Circuitu kwantowego z dwoma Qubitami (pierwsze „2") i dwoma klasycznymi bitami do przechowywania wyników (drugie „2")
# Definiowanie rejestrów
from qiskit import ClassicalRegister, QuantumRegister

qr = QuantumRegister(2, "q")
cr = ClassicalRegister(2, "c")
qc1 = QuantumCircuit(qr, cr)

# Z naszej analizy wiemy, że potrzebujemy Gate X działającego na każdym z Qubitów (L i R)
qc1.x([0, 1])
# Potrzebujemy Gate Hadamarda działającego na Qubicie Lucasa, który nazywamy Qubitem 0.
qc1.h(0)
# Gate controlled-NOT używa Qubitu 0 (Lucasa) jako sterującego i Qubitu 1 (Rihanny) jako docelowego.
qc1.cx(0, 1)
# Gate rotacji działa na Qubicie 1 (Rihanny) z argumentem -2 pi/3
qc1.ry(-2 * pi / 3, 1)
# Na koniec mierzymy wszystkie Qubity w Circuit, aby uzyskać prawdopodobieństwa pomiarów, i przechowujemy wyniki w klasycznych bitach.
qc1.measure([0, 1], [0, 1])
# Teraz możemy narysować pierwszy z trzech Circuitów, które sprawdzą dla nas nierówność Bella.
qc1.draw(output="mpl")
```

Poniższy kod szybko konstruuje wszystkie trzy Circuity w bardziej zwartej formie. Zwróć uwagę, że jedyna różnica między tymi trzema Circuitami polega na tym, o ile obracamy dwa Qubity wokół osi $y$.

```python
qcs = [QuantumCircuit(2, 2), QuantumCircuit(2, 2), QuantumCircuit(2, 2)]
for i in range(0, len(qcs)):
    qcs[i].x([0, 1])
    qcs[i].h(0)
    qcs[i].cx(0, 1)

qcs[0].ry(-2 * pi / 3, 1)
qcs[1].ry(-4 * pi / 3, 1)
qcs[2].ry(-2 * pi / 3, 1)
qcs[2].ry(-4 * pi / 3, 1)

for i in range(0, len(qcs)):
    qcs[i].barrier()
    qcs[i].measure([0, 1], [0, 1])

counts_list = [None] * len(qcs)
```


Teraz skorzystamy z prymitywu Qiskit o nazwie StatevectorSampler. Sampler to prymityw zaprojektowany do próbkowania wszystkich możliwych stanów układu i zwracania prawdopodobieństw (lub w niektórych przypadkach quasiprawdopodobieństw) uzyskania każdego ze stanów. Możemy określić liczbę „strzałów" (shots) i sprawdzić „liczniki" (counts) dla każdego stanu.

```python
from qiskit.primitives import StatevectorSampler

sampler = StatevectorSampler()

# Uruchamiamy zadanie, które zwróci strzały dla wszystkich 100 zestawów wartości parametrów.
for i in range(0, len(qcs)):
    pub = qcs[i]
    job = sampler.run([pub], shots=10000)
    # Wyodrębniamy wynik dla 0. PUB-a (w tym przykładzie jest tylko jeden PUB).
    result = job.result()
    data_pub = result[0].data
    counts = data_pub.c.get_counts()
    counts_list[i] = counts
#    plot_histogram(counts)
```

Jeśli przyjrzymy się liczniom z każdego Circuitu, zobaczymy, że dwa z nich były praktycznie identyczne, a trzeci wyraźnie się różnił.

```python
plot_histogram(counts_list)
```

Stwórzmy listę możliwych wyników i zsumujmy wszystkie liczniki każdego stanu ze wszystkich trzech Circuitów, aby uzyskać ogólne prawdopodobieństwa.

```python
outcomes = ("00", "01", "10", "11")

# Zamieniamy wartości „None" na 0, aby móc sumować.

for i in range(0, len(qcs)):
    for j in range(0, len(outcomes)):
        if counts_list[i].get(outcomes[j]) is None:
            counts_list[i].update({outcomes[j]: 0})

# Tworzymy słownik przechowujący wszystkie wyniki i sumujący ich wystąpienia w każdym z Circuitów.

total_counts = {}
for i in range(0, len(outcomes)):
    total_counts[outcomes[i]] = sum(
        counts_list[j].get(outcomes[i]) for j in range(0, len(qcs))
    )
```

Teraz możemy wydrukować łączne liczniki dla każdego wyniku i narysować histogram.

```python
print(total_counts)
plot_histogram(total_counts)
```

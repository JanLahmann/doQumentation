## 1. Wprowadzenie {#1-wprowadzenie}

Ta lekcja omawia kilka aspektów optymalizacji obwodów w obliczeniach kwantowych. Zobaczymy wartość optymalizacji obwodów, korzystając z wbudowanych ustawień optymalizacyjnych Qiskit. Następnie zagłębimy się nieco i zobaczymy, co możesz zrobić jako ekspert w swojej dziedzinie, aby budować obwody w inteligentny sposób. Na koniec przyjrzymy się bliżej temu, co dzieje się podczas transpilacji, co pomaga nam optymalizować obwody.
## 2. Optymalizacja obwodów ma znaczenie {#2-optymalizacja-obwodow-ma-znaczenie}

Na początek porównamy wyniki uruchomienia obwodów przygotowujących 5-qubitowy stan GHZ ($\frac{1}{\sqrt{2}} \left( |00000\rangle + |11111\rangle \right)$) z optymalizacją i bez niej.

```python
from qiskit.circuit import QuantumCircuit
from qiskit.transpiler.preset_passmanagers import generate_preset_pass_manager
from qiskit.primitives import BackendSamplerV2 as Sampler
```

```python
from qiskit_ibm_runtime.fake_provider import FakeBrisbane

backend = FakeBrisbane()
```

Na początek użyjemy obwodu GHZ syntetyzowanego w naiwny sposób, jak poniżej.

```python
num_qubits = 5

ghz_circ = QuantumCircuit(num_qubits)
ghz_circ.h(0)
[ghz_circ.cx(0, i) for i in range(1, num_qubits)]
ghz_circ.measure_all()
ghz_circ.draw("mpl")
```

### 2.1 Poziom optymalizacji {#21-poziom-optymalizacji}
Dostępne są 4 poziomy `optimization_level` od 0 do 3. Im wyższy poziom optymalizacji, tym więcej wysiłku obliczeniowego poświęca się na optymalizację obwodu. Poziom 0 nie wykonuje żadnej optymalizacji i robi tylko minimum konieczne do uruchomienia obwodu na wybranym backendzie. Poziom 3 wkłada największy wysiłek (i zazwyczaj czas działania) w optymalizację obwodu. Poziom 1 jest domyślnym poziomem optymalizacji.
Transpilujemy obwód bez optymalizacji (`optimization_level=0`) i z optymalizacją (`optimization_level=2`).
Widzimy dużą różnicę w długości obwodów po transpilacji.

```python
pm0 = generate_preset_pass_manager(
    optimization_level=0, backend=backend, seed_transpiler=777
)
pm2 = generate_preset_pass_manager(
    optimization_level=2, backend=backend, seed_transpiler=777
)
circ0 = pm0.run(ghz_circ)
circ2 = pm2.run(ghz_circ)
print("optimization_level=0:")
display(circ0.draw("mpl", idle_wires=False, fold=-1))
print("optimization_level=2:")
display(circ2.draw("mpl", idle_wires=False, fold=-1))
```

Uruchom na fałszywym backendzie (symulacja z szumem). Zobacz Dodatek 1, aby dowiedzieć się, jak uruchomić na prawdziwym backendzie.

```python
# run the circuits on the fake backend (noisy simulator)
sampler = Sampler(backend=backend)
job = sampler.run([circ0, circ2], shots=10000)
print(f"Job ID: {job.job_id()}")
```


```python
# get results
result = job.result()
unoptimized_result = result[0].data.meas.get_counts()
optimized_result = result[1].data.meas.get_counts()
```

```python
from qiskit.visualization import plot_histogram

# plot
sim_result = {"0" * 5: 0.5, "1" * 5: 0.5}
plot_histogram(
    [result for result in [sim_result, unoptimized_result, optimized_result]],
    bar_labels=False,
    legend=[
        "ideal",
        "no optimization",
        "with optimization",
    ],
)
```

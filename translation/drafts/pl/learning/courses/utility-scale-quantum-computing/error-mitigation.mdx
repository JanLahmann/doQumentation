## 2. Zaszumiona symulacja bez mitygacji błędów {#2-noisy-simulation-without-error-mitigation}

Qiskit Aer to klasyczny symulator do obliczeń kwantowych. Potrafi symulować nie tylko idealne wykonanie, ale także zaszumione wykonanie obwodów kwantowych. Ten notatnik pokazuje, jak uruchomić noisy simulation z użyciem Qiskit Aer:

1. Zbudować noise model
2. Zbudować zaszumiony sampler (symulator) z noise model
3. Uruchomić obwód kwantowy na zaszumionym samplerze

```
noise_model = NoiseModel()
...
noisy_sampler = Sampler(options={"backend_options": {"noise_model": noise_model}})
job = noisy_sampler.run([circuit])
```

### 2.2 Budowa noise model {#22-build-a-noise-model}

Aby przeprowadzić noisy simulation, musimy określić `NoiseModel`. W tej sekcji pokazujemy, jak zbudować `NoiseModel`.
Najpierw musimy zdefiniować błędy kwantowe (lub błędy odczytu) do dodania do noise model.

```python
from qiskit_aer.noise.errors import (
    coherent_unitary_error,
    amplitude_damping_error,
    ReadoutError,
)
from qiskit.circuit.library import RXGate

# Błąd koherentny (unitary error): nadmiarowa rotacja X-rotation
# https://qiskit.github.io/qiskit-aer/stubs/qiskit_aer.noise.coherent_unitary_error.html#qiskit_aer.noise.coherent_unitary_error
OVER_ROTATION_ANGLE = 0.05
coherent_error = coherent_unitary_error(RXGate(OVER_ROTATION_ANGLE).to_matrix())

# Incoherent error: błąd tłumienia amplitudy (amplitude damping)
# https://qiskit.github.io/qiskit-aer/stubs/qiskit_aer.noise.amplitude_damping_error.html#qiskit_aer.noise.amplitude_damping_error
AMPLITUDE_DAMPING_PARAM = 0.02  # w [0, 1] (0: brak błędu)
incoherent_error = amplitude_damping_error(AMPLITUDE_DAMPING_PARAM)

# Błąd odczytu (błąd pomiaru): readout error
# https://qiskit.github.io/qiskit-aer/stubs/qiskit_aer.noise.ReadoutError.html#qiskit_aer.noise.ReadoutError
PREP0_MEAS1 = 0.03  # P(1|0): prawdopodobieństwo przygotowania 0 i pomiaru 1
PREP1_MEAS0 = 0.08  # P(0|1): prawdopodobieństwo przygotowania 1 i pomiaru 0
readout_error = ReadoutError(
    [[1 - PREP0_MEAS1, PREP0_MEAS1], [PREP1_MEAS0, 1 - PREP1_MEAS0]]
)
```

```python
from qiskit_aer.noise import NoiseModel

noise_model = NoiseModel()
noise_model.add_quantum_error(coherent_error.compose(incoherent_error), "x", (0,))
noise_model.add_readout_error(readout_error, (0,))
```

## 3. Rzeczywiste obliczenia kwantowe z mitygacją błędów {#3-real-quantum-computation-with-error-mitigation}

Rozważamy 6-kubitowe obwody Trotter do symulacji ewolucji czasowej jednowymiarowego modelu Ising i sprawdzamy, jak błąd skaluje się względem liczby kroków czasowych.

```python
NUM_QUBITS = 6
NUM_TIME_STEPS = list(range(8))
RX_ANGLE = 0.1
RZZ_ANGLE = 0.1
```

### 3.1 Budowanie obwodów {#31-build-circuits}

```python
# Buduj obwody z różną liczbą kroków czasowych
circuits = []
for n_steps in NUM_TIME_STEPS:
    circ = QuantumCircuit(NUM_QUBITS)
    for i in range(n_steps):
        # warstwa rx
        for q in range(NUM_QUBITS):
            circ.rx(RX_ANGLE, q)
        # 1. warstwa rzz
        for q in range(1, NUM_QUBITS - 1, 2):
            circ.rzz(RZZ_ANGLE, q, q + 1)
        # 2. warstwa rzz
        for q in range(0, NUM_QUBITS - 1, 2):
            circ.rzz(RZZ_ANGLE, q, q + 1)
    circ.barrier()  # aby nie optymalizować obwodu
    # Etap odwracający (uncompute)
    for i in range(n_steps):
        for q in range(0, NUM_QUBITS - 1, 2):
            circ.rzz(-RZZ_ANGLE, q, q + 1)
        for q in range(1, NUM_QUBITS - 1, 2):
            circ.rzz(-RZZ_ANGLE, q, q + 1)
        for q in range(NUM_QUBITS):
            circ.rx(-RX_ANGLE, q)
    circuits.append(circ)
```

Aby z góry znać idealny wynik, używamy obwodów typu compute-uncompute, które składają się z pierwszego etapu, w którym stosowany jest oryginalny obwód $U$, oraz drugiego etapu, w którym jest on odwracany $U^\dagger$.
Zauważ, że idealny wynik takich obwodów będzie trywialnie stanem wejściowym $|000000\rangle$, który ma trywialne wartości oczekiwane dla dowolnych observable Pauliego, na przykład $\langle IIIIIZ \rangle = 1$.

### 3.3 Wykonanie przy użyciu Estimator (z różnymi poziomami resilience) {#33-execute-using-estimator-with-different-resilience-levels}

Ustawienie poziomu resilience (`estimator.options.resilience_level`) jest najłatwiejszym sposobem zastosowania mitygacji błędów przy użyciu Qiskit Estimator. Estimator obsługuje następujące poziomy resilience (stan na 2024/06/28). Więcej szczegółów znajdziesz w przewodniku [Configure error mitigation](/guides/configure-error-mitigation).

![image.png](/learning/images/courses/utility-scale-quantum-computing/error-mitigation/res_level.avif)

```python
from qiskit_ibm_runtime import Batch
from qiskit_ibm_runtime import EstimatorV2 as Estimator

jobs = []
job_ids = []
with Batch(backend=backend):
    for resilience_level in [0, 1, 2]:
        estimator = Estimator()
        estimator.options.resilience_level = resilience_level
        job = estimator.run(
            [(circ, obs.apply_layout(circ.layout)) for circ in isa_circuits]
        )
        job_ids.append(job.job_id())
        print(f"Job ID (rl={resilience_level}): {job.job_id()}")
        jobs.append(job)
```

## 4. (Opcjonalnie) Dostosuj opcje mitygacji błędów {#4-optional-customize-error-mitigation-options}
Możemy dostosować stosowanie technik mitygacji błędów za pomocą opcji, jak pokazano poniżej.

```python
# TREX
estimator.options.twirling.enable_measure = True
estimator.options.twirling.num_randomizations = "auto"
estimator.options.twirling.shots_per_randomization = "auto"

# Gate twirling
estimator.options.twirling.enable_gates = True
# ZNE
estimator.options.resilience.zne_mitigation = True
estimator.options.resilience.zne.noise_factors = [1, 3, 5]
estimator.options.resilience.zne.extrapolator = ("exponential", "linear")

# Dynamical decoupling
estimator.options.dynamical_decoupling.enable = True  # Domyślnie: False
estimator.options.dynamical_decoupling.sequence_type = "XX"

# Inne opcje
estimator.options.default_shots = 10_000
```

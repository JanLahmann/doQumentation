### Czym jest propagowana absorpcja szumu (PNA)? {#czym-jest-propagowana-absorpcja-szumu-pna}

***Technika mitygacji błędów bramek polegająca na propagowaniu obserwabli przez odwrotny kanał szumu wpływający na bramki 2-qubitowe, co daje obserwablę mitygującą szum.***
Bramki 2Q w eksperymencie, który chcemy uruchomić, będą dotknięte znacznym szumem.
![Zaszumiony eksperyment](../images/noisy_expt.png)
Jeśli nauczymy się modelu szumu, możemy zastosować jego odwrotność i wyeliminować szum.
![Eksperyment z mitygacją szumu](../images/noise_mitigated_expt.png)
Zamiast implementować odwrotny kanał szumu poprzez jego próbkowanie na QPU jak w PEC, możemy zaimplementować go klasycznie w mierzonej obserwabli za pomocą propagacji Pauliego. Daje to bardziej złożoną obserwablę, której pomiar ma efekt mitygowania nauczonych błędów bramek.
![Przegląd PNA](../images/pna_overview.png)
### Wygenerowanie lustrzanego obwodu Trottera i obserwabli {#wygenerowanie-lustrzanego-obwodu-trottera-i-obserwabli}

W tym eksperymencie zbadamy dynamikę czasową modelu Isinga z kopnięciami na 30-stanowiskowym łańcuchu spinów 1D. Rozważany hamiltonjan to:

$H = -J\sum\limits_{\langle i,j \rangle} Z_iZ_j + h\sum\limits_iX_i$,

gdzie $J>0$ opisuje sprzężenie spinów najbliższych sąsiadów, $i<j$, a globalne poprzeczne pole, $h$, wynosi $\frac{\pi}{8}$. Im dalej $h$ jest od kąta Clifforda (tj. $\theta=n\frac{\pi}{2}, n \in \mathbb{Z}$), tym trudniejsza staje się propagacja generatorów anty-szumu przez obwód.

Jako obserwablę rozważymy średnie namagnesowanie na jednym stanowisku, $\frac{1}{N} \sum_{i=1}^{N} \langle z_i \rangle$, gdzie $N$ jest liczbą stanowisk.

```python
import numpy as np
from qiskit import QuantumCircuit
from qiskit.quantum_info import Pauli, SparsePauliOp

num_qubits = 30
num_trotter_steps = 10
rx_angle = np.pi / 8

# Avg single-site magnetization
id_pauli = Pauli("I" * num_qubits)
observable = SparsePauliOp([id_pauli.dot(Pauli("Z"), [i]) for i in range(num_qubits)]) / num_qubits

# Implement Trotterized kicked-Ising model
circuit = QuantumCircuit(num_qubits)
for _step in range(num_trotter_steps):
    circuit.rx(rx_angle, range(num_qubits))
    for first_qubit in (1, 2):
        for idx in range(first_qubit, num_qubits, 2):
            # equivalent to Rzz(-pi/2):
            circuit.sdg([idx - 1, idx])
            circuit.cz(idx - 1, idx)
circuit.compose(circuit.inverse(), inplace=True)
circuit.measure_active()
circuit.draw("mpl", fold=-1)
```

### Twirlingowanie warstw bramek 2-qubitowych i pomiarów oraz znalezienie unikalnych warstw {#twirlingowanie-warstw-bramek-2-qubitowych-i-pomiarow-oraz-znalezienie-unikalnych-warstw}

Tutaj upewniamy się, że menedżer przejść opatruje pudełka adnotacjami ``Twirl`` i ``InjectNoise``, które pozwalają nam nauczyć szumu wpływającego na nasz obwód i powiązać ten szum z odpowiadającą mu warstwą obwodu.

- ``enable_gates/enable_measure: True``: Opakuj wszystkie warstwy bramek 2Q i końcowe pomiary w pudełka. Bramki jednokubitowe zostaną ubrane od lewej strony wewnątrz pudełek.
- ``measure_annotations: all`` Dołącz adnotacje `Twirl` i `ChangeBasis` do pudełka pomiarowego.
- ``twirling_strategy: active``: Twirlinguj wszystkie aktywne qubity w każdym pudełku zawierającym bramki splątujące.
- ``inject_noise_targets: gates``: Adnotacje ``InjectNoise`` powinny być dodane do wszystkich pudełek z adnotacją ``Twirl`` zawierających bramki splątujące.
- ``inject_noise_strategy: uniform_modification``: Wszystkie warstwy szumu powinny być skalowane równomiernie.

#### Uczenie szumu {#uczenie-szumu}

Zanim uruchomimy eksperymenty, uczymy modelu szumu wpływającego na bramki splątujące i pomiary w obwodzie. Posiadanie dokładnego modelu szumu jest konieczne do skutecznej mitygacji błędów. Uczenie szumu tuż przed wykonaniem eksperymentów daje największą szansę, że model szumu wiernie opisuje rzeczywisty szum wpływający na bramki podczas wykonania.

Zanim nauczymy się szumu, musimy znaleźć unikalne warstwy 2-qubitowe w naszym obwodzie, aby zminimalizować liczbę strzałów potrzebnych do nauczenia szumu dla całego obwodu. Używamy ``find_unique_box_instructions`` z ``samplomatic``, aby uzyskać unikalne warstwy z obwodu w pudełkach, włącznie z warstwą pomiarową. To są warstwy, które przekazujemy do modułu uczenia szumu.

Gdy znamy już warstwy, możemy nauczyć się szumu. Rozważamy kilka parametrów:

- `num_randomizations`: Liczba losowych obwodów używanych na konfigurację obwodu uczącego.
- `shots_per_randomization`: Całkowita liczba strzałów na losowy obwód uczący.
- `layer_pair_depths`: Głębokości obwodu (mierzone liczbą par) używane w eksperymentach uczących.
- `post_selection`: Użyjemy post-selekcji opartej na krawędziach podczas uczenia, używając bramek `rx` do implementacji impulsów po pomiarze.

```python
from qiskit_ibm_runtime.noise_learner_v3.noise_learner_v3 import NoiseLearnerV3
from qiskit_ibm_runtime.options import NoiseLearnerV3Options
from samplomatic.utils import find_unique_box_instructions

# Load noise learner data from a shared job
load_saved_nl_result = True

# Noise learning parameters
num_randomizations_nl = 64
shots_per_randomization_nl = 128
strategy = "edge"
enable_postsel = True
x_pulse_type = "rx"

# Find the unique instructions (layers) from boxed-up circuit
unique_2q_layers_and_meas = find_unique_box_instructions(
    boxed_circuit, normalize_annotations=None, undress_boxes=True
)

noise_learner_params = {
    "num_randomizations": num_randomizations_nl,
    "shots_per_randomization": shots_per_randomization_nl,
    "layer_pair_depths": [1, 2, 4, 8, 12, 16, 24, 32, 40, 48],
    "post_selection": {
        "enable": enable_postsel,
        "strategy": strategy,
        "x_pulse_type": x_pulse_type,
    },
    "experimental": {},
}
# set the options
noise_learner_options = NoiseLearnerV3Options(**noise_learner_params)

# run the noise learner job
noise_learner = NoiseLearnerV3(backend, noise_learner_options)
noise_learner_job = noise_learner.run(unique_2q_layers_and_meas)
noise_learner_result = noise_learner_job.result()

nl_metadata = noise_learner_params | {"layout": layout}
```

#### Powiązanie pudełek obwodu z nauczonym szumem {#powiazanie-pudelek-obwodu-z-nauczonym-szumem}

Tutaj tworzymy mapowanie między identyfikatorami referencyjnymi ``InjectNoise`` każdego pudełka a nauczonym modelem szumu (`PauliLindbladMap`) wpływającym na bramki splątujące w tym pudełku.

```python
from samplomatic.annotations import InjectNoise
from samplomatic.utils import get_annotation

# map inject noise refs to pauli lindblad maps
refs_to_noise_models = {}
for instruction, result in zip(unique_2q_layers_and_meas, noise_learner_result, strict=False):
    if inject_noise_annot := get_annotation(instruction.operation, InjectNoise):
        refs_to_noise_models[inject_noise_annot.ref] = result.to_pauli_lindblad_map()
```

#### Propagowanie obserwabli przez nauczony anty-szum w celu uzyskania obserwabli mitygującej szum {#propagowanie-obserwabli-przez-nauczony-anty-szum}

Jak omówiono powyżej, odbywa się to w dwóch krokach. Najpierw propagujemy generator anty-szumu do końca obwodu. Następnie propagujemy obserwablę przez ten ewoluowany generator. Proces ten jest powtarzany dla każdego generatora anty-szumu w obwodzie. W tej implementacji każdy generator w danej warstwie jest propagowany do końca obwodu równolegle. Dodatkowo, wieloprocesorowość Pythona jest używana do równoległego wykonywania zarówno propagacji do przodu anty-szumu, jak i propagacji wstecz obserwabli. Zapobiega to gromadzeniu się ewoluowanych generatorów w pamięci i maksymalizuje zasoby obliczeniowe.

Przy uruchamianiu PNA zawsze musisz podać zaszumiony obwód i obserwablę. Jeśli twój zaszumiony obwód jest obwodem w pudełkach z adnotacjami `InjectNoise`, musisz podać mapowanie, które stworzyliśmy w powyższym kroku. Można również przekazać obwód bez pudełek zawierający instrukcje ``PauliLindbladError`` z ``qiskit-aer``. W takim przypadku nie trzeba podawać ``refs_to_noise_models``. Oprócz podstawowych danych wejściowych warto rozważyć:

- `max_err_terms`: Liczba wyrazów, które należy zachować w każdym generatorze anty-szumu podczas propagacji do przodu. Pozwolenie na większą wartość generalnie zwiększa dokładność, ale to zachowanie nie musi być monotoniczne.
- `max_obs_terms`: Liczba wyrazów, które należy zachować w obserwabli mitygującej szum, $\tilde{O}$, podczas propagacji wstecz przez ewoluowany anty-szum. Większe wartości generalnie zwiększają dokładność, ale nie jest to zagwarantowane w sposób monotoniczny.
- `num_processes`: Liczba rdzeni przeznaczonych do tego procesu. Pamiętaj, że generatory są propagowane do przodu i stosowane do obserwabli równolegle.
- `search_step`: Krok propagacji wstecz używa metody zachłannej do przybliżonego sprzężenia dwóch operatorów w bazie Pauliego. Metodę tę można przyspieszyć, zwiększając ``search_step``. Zobacz [dokumentację pauli-prop](https://qiskit.github.io/pauli-prop/) po więcej informacji.
- `num_to_measure`: Choć ta zmienna nie jest wejściem do ``generate_noise_mitigating_observable``, używamy jej do kontrolowania, ile wyrazów z $\tilde{O}$ faktycznie chcemy zmierzyć. Tutaj zmierzymy tylko 30 najważniejszych wyrazów, które są oryginalnymi wyrazami naszej obserwabli. Wyrazy zostały teraz przeskalowane tak, że ich pomiar ma efekt mitygowania nauczonych błędów bramek. Choć mierzymy tylko 30 wyrazów z $\tilde{O}$, często nadal warto pozwolić mu rosnąć, gdyż zwiększa to precyzję współczynników skalowania wiodących wyrazów.

```python
from qiskit_addon_pna import generate_noise_mitigating_observable

# PNA parameters
num_processes = 8
max_err_terms = 10_000
max_obs_terms = 10_000
num_to_measure = num_qubits

obs_tilde_isa = generate_noise_mitigating_observable(
    boxed_circuit,
    isa_observable,
    refs_to_noise_models,
    max_err_terms=max_err_terms,
    max_obs_terms=max_obs_terms,
    num_processes=num_processes,
    print_progress=True,
    search_step=8,
)
p_2_v = {p: v for v, p in enumerate(layout)}
obs_tilde_virtual = SparsePauliOp.from_sparse_list(
    [
        (pstr, [p_2_v[p] for p in p_qubits], coeff)
        for (pstr, p_qubits, coeff) in obs_tilde_isa.to_sparse_list()
    ],
    num_qubits=num_qubits,
)
obs_tilde_virtual = obs_tilde_virtual[np.argsort(np.abs(obs_tilde_virtual.coeffs))[::-1]][
    :num_to_measure
]
```

#### Przekształcenie baz pomiarowych do postaci kanonicznej {#przeksztalcenie-baz-pomiarowych-do-postaci-kanonicznej}

Następnie znajdziemy minimalny zestaw baz do zmierzenia, tak aby w pełni pokryć każdy wyraz Pauliego w mierzonej obserwabli (***wiele obserwabli może być mierzonych jednocześnie, jeśli dokonują kwantyzacji qubit po qubicie***). Ponieważ mierzymy tylko wyrazy naszej oryginalnej obserwabli, która jest sumą wszystkich pojedynczych Paulich-`Z`, potrzebna jest tylko jedna baza -- baza wszystkich-`Z`.

Oprócz znalezienia zestawu baz pomiarowych Pauliego musimy odwzorować te wyrazy Pauliego do postaci kanonicznej oczekiwanej przez prymityw ``Executor``. Aby uzyskać więcej informacji na temat kanonicznej kolejności qubitów, odwiedź [dokumentację samplomatic](https://qiskit.github.io/samplomatic/guides/samplex_io.html#qubit-ordering-convention).

```python
from qiskit_addon_utils.exp_vals.measurement_bases import get_measurement_bases

meas_box = boxed_circuit.data[-1]
canonical_qubits = [
    idx for idx, qubit in enumerate(boxed_circuit.qubits) if qubit in meas_box.qubits
]
c_2_p = {c: p for c, p in enumerate(canonical_qubits)}  # canonical -> physical
p_2_v = {p: v for v, p in enumerate(layout)}  # physical -> virtual
c_2_v = {c: p_2_v[p] for c, p in c_2_p.items()}  # canonical -> virtual
meas_bases, bases_reverser = get_measurement_bases(obs_tilde_virtual)
meas_bases_canonical = [
    np.array([base[c_2_v[c]] for c in range(num_qubits)], dtype=np.uint8) for base in meas_bases
]
```

#### Określenie sposobu próbkowania w ``QuantumProgram`` {#okreslenie-sposobu-probkowania-w-quantumprogram}

``QuantumProgram`` to miejsce, w którym określamy sposób próbkowania eksperymentu:

- ``template_circuit``: Obwód zawierający wszystkie bramki niezbędne do implementacji wszystkich pożądanych losowości (z losowości twirlingowania, parametrów itd.).
- ``samplex``: Obiekt definiujący rozkład prawdopodobieństwa nad wszystkimi możliwymi losowościami obwodu, z których należy próbkować.
- ``samplex_arguments``: Powiązania niezbędne do pełnego zdefiniowania samplex.
    - ``basis_changes``: Tutaj określamy zestaw baz do zmierzenia, które pokryją wszystkie wyrazy Pauliego w mierzonej obserwabli.
    - ``noise_scales.ref``: Ustawiamy skalę każdej warstwy szumu na `0.0`, aby zapobiec wstrzykiwaniu dodatkowego szumu do naszych próbek.
    - ``pauli_lindblad_maps``: Wymagane, jeśli przekazywane są ``noise_scales``. Mapuje warstwy szumu na powiązany model szumu.
- ``shape``: Krotka kształtu rozszerzająca niejawny kształt zdefiniowany przez ``samplex_arguments``. Nietrywalne osie wprowadzone przez to rozszerzenie wyliczają losowości.

```python
from qiskit_ibm_runtime import QuantumProgram

# Control the # of shots during execution
shots_per_randomization_exec = 64
num_randomizations_exec = 6144

# Zero out the noise to prevent noise from being injected during execution.
# We only added InjectNoise annotations so PNA could associate the noise
# to layers in the circuit
samplex_inputs = {f"noise_scales.{ref}": 0.0 for ref in refs_to_noise_models}
samplex_inputs |= {"pauli_lindblad_maps": refs_to_noise_models}

# Specify the bases to measure
bases_broadcastable = np.expand_dims(np.array(meas_bases_canonical), axis=1)
samplex_inputs |= {"basis_changes": {"basis0": bases_broadcastable}}

# Convert samplex_inputs into a dict to pass to QuantumProgram
samplex_arguments = samplex.inputs().make_broadcastable().bind(**samplex_inputs)

# Instantiate the QuantumProgram with the specified parameters
program = QuantumProgram(shots=shots_per_randomization_exec)
program.append(
    circuit=template_circuit,
    samplex=samplex,
    samplex_arguments=samplex_arguments,
    shape=(num_randomizations_exec),
)
```

#### Próbkowanie obwodu za pomocą prototypu prymitywu ``Executor`` {#probkowanie-obwodu-za-pomoca-prototypu-prymitywu-executor}

Teraz, gdy mamy zdefiniowany nasz ``QuantumProgram``, wykonanie eksperymentu jest proste. Wystarczy zainicjalizować obiekt ``Executor``, podać mu backend i uruchomić program.

```python
from qiskit_ibm_runtime import Executor

# Execute (sample) the circuit
executor = Executor(backend)
job_exec = executor.run(program)
exec_results = job_exec.result()
```

#### Przetworzenie próbek w celu obliczenia mitygowanej wartości oczekiwanej {#przetworzenie-probek-w-celu-obliczenia-mitygowanej-wartosci-oczekiwanej}

Aby obliczyć mitygowaną wartość oczekiwaną:

- Obliczymy współczynniki skalowania TREX na podstawie nauczonych szumów wpływających na pomiary.
- Wygenerujemy maskę do zachowania tylko próbek po post-selekcji.
- Użyjemy funkcji ``executor_expectation_values`` z ``qiskit-addon-utils`` do połączenia wszystkich danych w mitygowaną wartość oczekiwaną.

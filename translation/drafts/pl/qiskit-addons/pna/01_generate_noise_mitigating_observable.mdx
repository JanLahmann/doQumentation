### Czym jest pochłanianie propagowanego szumu (PNA)? {#what-is-propagated-noise-absorption-pna}

***Technika mitygacji błędów bramek polegająca na propagowaniu obserwowalnej przez odwrotny kanał szumu wpływający na 2-Qubitowe Gate, czego wynikiem jest obserwowalna mitygująca szum.***
Gate 2Q w eksperymencie, który chcemy przeprowadzić, będą dotknięte znacznym szumem.
![Zaszumiony eksperyment](../images/noisy_expt.png)
Jeśli wyuczymy model szumu, możemy zastosować jego odwrotność i wyeliminować szum.
![Eksperyment z mitygacją szumu](../images/noise_mitigated_expt.png)
Zamiast implementować odwrotny kanał szumu przez próbkowanie go na QPU jak w PEC, możemy zaimplementować go klasycznie w mierzonej obserwowalnej za pomocą propagacji Pauliego. Prowadzi to do bardziej złożonej obserwowalnej, która podczas pomiaru ma efekt mitygowania wyuczonego szumu bramek.
![Przegląd PNA](../images/pna_overview.png)

### Generowanie lustrzanego Circuit Trottera i obserwowalnej {#generate-the-mirrored-trotter-circuit-and-observable}

W tym eksperymencie będziemy badać dynamikę czasową modelu kopniętego Isinga na 30 węzłach w jednowymiarowym łańcuchu spinowym. Rozważany Hamiltonian to:

$H = -J\sum\limits_{\langle i,j \rangle} Z_iZ_j + h\sum\limits_iX_i$,

gdzie $J>0$ opisuje sprzężenie spinów najbliższych sąsiadów, $i<j$, a globalne poprzeczne pole $h$ jest ustawione na $\frac{\pi}{8}$. Im dalej $h$ jest od kąta Clifforda (tzn. $\theta=n\frac{\pi}{2}, n \in \mathbb{Z}$), tym trudniejsze staje się propagowanie generatorów anty-szumu przez Circuit.

Jako obserwowalną wybierzemy średnią jednowęzłową magnetyzację, $\frac{1}{N} \sum_{i=1}^{N} \langle z_i \rangle$, gdzie $N$ to liczba węzłów.

```python
import numpy as np
from qiskit import QuantumCircuit
from qiskit.quantum_info import Pauli, SparsePauliOp

num_qubits = 30
num_trotter_steps = 10
rx_angle = np.pi / 8

# Avg single-site magnetization
id_pauli = Pauli("I" * num_qubits)
observable = SparsePauliOp([id_pauli.dot(Pauli("Z"), [i]) for i in range(num_qubits)]) / num_qubits

# Implement Trotterized kicked-Ising model
circuit = QuantumCircuit(num_qubits)
for _step in range(num_trotter_steps):
    circuit.rx(rx_angle, range(num_qubits))
    for first_qubit in (1, 2):
        for idx in range(first_qubit, num_qubits, 2):
            # equivalent to Rzz(-pi/2):
            circuit.sdg([idx - 1, idx])
            circuit.cz(idx - 1, idx)
circuit.compose(circuit.inverse(), inplace=True)
circuit.measure_active()
circuit.draw("mpl", fold=-1)
```

### Twirlingowanie warstw 2-Qubitowych Gate i pomiarów oraz znajdowanie unikalnych warstw {#twirl-the-2-qubit-gate-layers-and-measurements-and-find-unique-layers}

Tutaj upewniamy się, że menedżer przejść opatruje boxy adnotacjami ``Twirl`` i ``InjectNoise``, które pozwalają nam wyuczyć szum wpływający na nasz Circuit i powiązać ten szum z odpowiadającą mu warstwą Circuit.

- ``enable_gates/enable_measure: True``: Opakuj wszystkie warstwy Gate 2Q i końcowe pomiary w boxy. Gate jednokubitowe będą lewostronnie ubierane wewnątrz boxów.
- ``measure_annotations: all`` Dołącz adnotacje `Twirl` i `ChangeBasis` do boksa pomiaru.
- ``twirling_strategy: active``: Twirlinguj wszystkie aktywne Qubity w każdym boksie zawierającym Gate splątujące.
- ``inject_noise_targets: gates``: Adnotacje ``InjectNoise`` powinny być dodane do wszystkich boxów opatrzonych adnotacją ``Twirl``, zawierających Gate splątujące.
- ``inject_noise_strategy: uniform_modification``: Wszystkie warstwy szumu powinny być skalowane równoważnie.

#### Uczenie szumu {#learn-the-noise}

Zanim uruchomimy eksperymenty, uczymy modelu szumu wpływającego na bramki splątujące i pomiary w Circuit. Posiadanie dokładnego modelu szumu jest niezbędne do skutecznego łagodzenia błędów. Uczenie szumu tuż przed wykonaniem eksperymentów daje najlepszą szansę na to, by model szumu wiernie opisywał rzeczywisty szum wpływający na bramki podczas wykonania.

Zanim przystąpimy do uczenia szumu, musimy znaleźć unikalne warstwy 2-Qubitowe w naszym Circuit, aby zminimalizować liczbę strzałów potrzebnych do nauczenia szumu dla całego Circuit. Używamy ``find_unique_box_instructions`` z ``samplomatic``, aby dostarczyć nam unikalne warstwy z Circuit podzielonego na bloki, łącznie z warstwą pomiarową. To są warstwy, które przekazujemy do narzędzia do uczenia szumu.

Gdy znamy już warstwy, możemy przystąpić do uczenia szumu. Jest kilka parametrów, które bierzemy pod uwagę:

- `num_randomizations`: Liczba losowych Circuit używanych na każdą konfigurację Circuit uczącego
- `shots_per_randomization`: Całkowita liczba strzałów używanych na każdy losowy Circuit uczący
- `layer_pair_depths`: Głębokości Circuit (mierzone liczbą par) używane w eksperymentach uczących.
- `post_selection`: Podczas uczenia będziemy używać post-selekcji opartej na krawędziach z użyciem bramek `rx` do implementacji impulsów po pomiarze

```python
from qiskit_ibm_runtime.noise_learner_v3.noise_learner_v3 import NoiseLearnerV3
from qiskit_ibm_runtime.options import NoiseLearnerV3Options
from samplomatic.utils import find_unique_box_instructions

# Load noise learner data from a shared job
load_saved_nl_result = True

# Noise learning parameters
num_randomizations_nl = 64
shots_per_randomization_nl = 128
strategy = "edge"
enable_postsel = True
x_pulse_type = "rx"

# Find the unique instructions (layers) from boxed-up circuit
unique_2q_layers_and_meas = find_unique_box_instructions(
    boxed_circuit, normalize_annotations=None, undress_boxes=True
)

noise_learner_params = {
    "num_randomizations": num_randomizations_nl,
    "shots_per_randomization": shots_per_randomization_nl,
    "layer_pair_depths": [1, 2, 4, 8, 12, 16, 24, 32, 40, 48],
    "post_selection": {
        "enable": enable_postsel,
        "strategy": strategy,
        "x_pulse_type": x_pulse_type,
    },
    "experimental": {},
}
# set the options
noise_learner_options = NoiseLearnerV3Options(**noise_learner_params)

# run the noise learner job
noise_learner = NoiseLearnerV3(backend, noise_learner_options)
noise_learner_job = noise_learner.run(unique_2q_layers_and_meas)
noise_learner_result = noise_learner_job.result()

nl_metadata = noise_learner_params | {"layout": layout}
```

#### Powiązanie bloków Circuit z nauczonym szumem {#associate-circuit-boxes-with-learned-noise}

Tutaj tworzymy mapowanie między identyfikatorami referencyjnymi ``InjectNoise`` każdego bloku a nauczonym modelem szumu (`PauliLindbladMap`) wpływającym na bramki splątujące w tym bloku.

```python
from samplomatic.annotations import InjectNoise
from samplomatic.utils import get_annotation

# map inject noise refs to pauli lindblad maps
refs_to_noise_models = {}
for instruction, result in zip(unique_2q_layers_and_meas, noise_learner_result, strict=False):
    if inject_noise_annot := get_annotation(instruction.operation, InjectNoise):
        refs_to_noise_models[inject_noise_annot.ref] = result.to_pauli_lindblad_map()
```

#### Propagacja obserwowalnej przez nauczony anty-szum w celu uzyskania obserwowalnej łagodzącej szum {#propagate-the-observable-through-the-learned-anti-noise-to-get-a-noise-mitigating-observable}

Jak omówiono powyżej, odbywa się to w dwóch krokach. Najpierw propagujemy generator anty-szumu do końca Circuit. Następnie propagujemy obserwowalną przez ten wyewoluowany generator. Proces ten jest powtarzany dla każdego generatora anty-szumu w Circuit. W tej implementacji każdy generator w danej warstwie jest propagowany do końca Circuit równolegle. Dodatkowo, wieloprocesowość Pythona jest używana do wykonywania zarówno propagacji do przodu anty-szumu, jak i propagacji wstecznej obserwowalnej równolegle. Zapobiega to gromadzeniu się wyewoluowanych generatorów w pamięci i maksymalizuje wykorzystanie zasobów obliczeniowych.

Podczas uruchamiania PNA zawsze musisz podać zaszumiony Circuit i obserwowalną. Jeśli twój zaszumiony Circuit jest Circuit podzielonym na bloki z adnotacjami `InjectNoise`, musisz podać mapowanie, które stworzyliśmy w powyższym kroku. Można też przekazać Circuit niepodzielony na bloki, zawierający instrukcje ``PauliLindbladError`` z ``qiskit-aer``. W takim przypadku nie trzeba podawać ``refs_to_noise_models``. Oprócz podstawowych danych wejściowych, użytkownicy powinni rozważyć:

- `max_err_terms`: Liczba wyrazów do zachowania w każdym generatorze anty-szumu podczas jego propagacji do przodu. Pozwolenie na większą wartość generalnie zwiększa dokładność, choć takie zachowanie nie musi być monotonicznie zagwarantowane.
- `max_obs_terms`: Liczba wyrazów do zachowania w obserwowalnej łagodzącej szum, $\tilde{O}$, podczas jej propagacji wstecznej przez wyewoluowany anty-szum. Większe wartości generalnie zwiększają dokładność, choć nie jest to monotonicznie gwarantowane.
- `num_processes`: Liczba rdzeni dedykowanych do procesu. Pamiętaj, że generatory są propagowane do przodu i stosowane do obserwowalnej równolegle.
- `search_step`: Krok propagacji wstecznej używa metody zachłannej do przybliżonego sprzężenia dwóch operatorów w bazie Pauliego. Metodę tę można przyspieszyć, zwiększając ``search_step``. Więcej informacji znajdziesz w [dokumentacji pauli-prop](https://qiskit.github.io/pauli-prop/).
- `num_to_measure`: Choć ta zmienna nie jest wejściem do ``generate_noise_mitigating_observable``, używamy jej do kontrolowania, ile wyrazów z $\tilde{O}$ faktycznie chcemy mierzyć. Tutaj zmierzymy tylko 30 najważniejszych wyrazów, które są oryginalnymi wyrazami naszej obserwowalnej. Wyrazy te zostały teraz przeskalowane tak, że ich pomiar ma efekt łagodzenia nauczonych błędów bramek. Choć mierzymy tylko 30 wyrazów z $\tilde{O}$, często nadal warto pozwolić mu rosnąć, ponieważ zwiększa to precyzję współczynników skalowania wiodących wyrazów.

```python
from qiskit_addon_pna import generate_noise_mitigating_observable

# PNA parameters
num_processes = 8
max_err_terms = 10_000
max_obs_terms = 10_000
num_to_measure = num_qubits

obs_tilde_isa = generate_noise_mitigating_observable(
    boxed_circuit,
    isa_observable,
    refs_to_noise_models,
    max_err_terms=max_err_terms,
    max_obs_terms=max_obs_terms,
    num_processes=num_processes,
    print_progress=True,
    search_step=8,
)
p_2_v = {p: v for v, p in enumerate(layout)}
obs_tilde_virtual = SparsePauliOp.from_sparse_list(
    [
        (pstr, [p_2_v[p] for p in p_qubits], coeff)
        for (pstr, p_qubits, coeff) in obs_tilde_isa.to_sparse_list()
    ],
    num_qubits=num_qubits,
)
obs_tilde_virtual = obs_tilde_virtual[np.argsort(np.abs(obs_tilde_virtual.coeffs))[::-1]][
    :num_to_measure
]
```

#### Przekształcenie baz pomiarowych do postaci kanonicznej {#transform-the-measurement-bases-to-canonical-form}

Następnie znajdziemy minimalny zestaw baz do pomiaru, który pozwoli w pełni pokryć każdy wyraz Pauliego w mierzonym obserwowalnym (***wiele obserwowalnych może być mierzonych jednocześnie, jeśli komutują one qubit po Qubit***). Ponieważ mierzymy wyłącznie wyrazy naszego oryginalnego obserwowalnego, będącego sumą wszystkich jednoQubitowych Paulich `Z`, wystarczy jedna baza — baza całkowicie-`Z`.

Oprócz wyznaczenia zbioru baz pomiarowych Pauliego, musimy odwzorować te wyrazy na postać kanoniczną oczekiwaną przez prymityw ``Executor``. Więcej informacji na temat kanonicznego porządkowania Qubitów znajdziesz w [dokumentacji samplomatic](https://qiskit.github.io/samplomatic/guides/samplex_io.html#qubit-ordering-convention).

```python
from qiskit_addon_utils.exp_vals.measurement_bases import get_measurement_bases

meas_box = boxed_circuit.data[-1]
canonical_qubits = [
    idx for idx, qubit in enumerate(boxed_circuit.qubits) if qubit in meas_box.qubits
]
c_2_p = {c: p for c, p in enumerate(canonical_qubits)}  # canonical -> physical
p_2_v = {p: v for v, p in enumerate(layout)}  # physical -> virtual
c_2_v = {c: p_2_v[p] for c, p in c_2_p.items()}  # canonical -> virtual
meas_bases, bases_reverser = get_measurement_bases(obs_tilde_virtual)
meas_bases_canonical = [
    np.array([base[c_2_v[c]] for c in range(num_qubits)], dtype=np.uint8) for base in meas_bases
]
```

#### Określenie sposobu próbkowania w ``QuantumProgram`` {#specify-how-to-sample-in-the-quantumprogram}

``QuantumProgram`` to miejsce, w którym określamy sposób próbkowania eksperymentu:

- ``template_circuit``: Circuit zawierający wszystkie Gate konieczne do implementacji wszystkich pożądanych randomizacji (wynikające z randomizacji twirlingiem, parametrów itd.).
- ``samplex``: Obiekt definiujący rozkład prawdopodobieństwa dla wszystkich możliwych randomizacji Circuit, z których będziemy próbkować.
- ``samplex_arguments``: Przypisania niezbędne do pełnego zdefiniowania samplex:
    - ``basis_changes``: Tu wskazujemy zestaw baz pomiarowych pokrywających wszystkie wyrazy Pauliego w mierzonym obserwowalnym.
    - ``noise_scales.ref``: Ustawiamy skalę każdej warstwy szumu na `0.0`, aby zapobiec wprowadzaniu dodatkowego szumu do próbek.
    - ``pauli_lindblad_maps``: Wymagane, gdy przekazywane są ``noise_scales``. Odwzorowuje warstwy szumu na powiązany model szumu.
- ``shape``: Krotka kształtu rozszerzająca niejawny kształt zdefiniowany przez ``samplex_arguments``. Osie nie-trywialne wprowadzone przez to rozszerzenie enumerują randomizacje.

```python
from qiskit_ibm_runtime import QuantumProgram
```

#### Próbkowanie Circuit z użyciem prototypu prymitywu ``Executor`` {#sample-the-circuit-using-the-executor-primitive-prototype}

Teraz, gdy zdefiniowaliśmy nasz ``QuantumProgram``, wykonanie eksperymentu jest proste. Wystarczy utworzyć instancję obiektu ``Executor``, podać mu Backend i uruchomić program.

```python
from qiskit_ibm_runtime import Executor
```

#### Przetwarzanie końcowe próbek w celu obliczenia wartości oczekiwanej z korekcją błędów {#post-process-the-samples-to-calculate-an-error-mitigated-expectation-value}

Aby obliczyć wartość oczekiwaną z korekcją błędów, wykonamy następujące kroki:

- Obliczymy czynniki skalujące TREX na podstawie wyuczonego szumu wpływającego na pomiary.
- Wygenerujemy maskę służącą do zachowania wyłącznie próbek po post-selekcji.
- Użyjemy funkcji ``executor_expectation_values`` z ``qiskit-addon-utils`` do połączenia wszystkich danych w wartość oczekiwaną z korekcją błędów.

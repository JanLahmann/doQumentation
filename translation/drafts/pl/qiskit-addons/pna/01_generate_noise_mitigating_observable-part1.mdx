### Czym jest pochłanianie propagowanego szumu (PNA)? {#what-is-propagated-noise-absorption-pna}

***Technika mitygacji błędów bramek polegająca na propagowaniu obserwowalnej przez odwrotny kanał szumu wpływający na 2-Qubitowe Gate, czego wynikiem jest obserwowalna mitygująca szum.***
Gate 2Q w eksperymencie, który chcemy przeprowadzić, będą dotknięte znacznym szumem.
![Zaszumiony eksperyment](../images/noisy_expt.png)
Jeśli wyuczymy model szumu, możemy zastosować jego odwrotność i wyeliminować szum.
![Eksperyment z mitygacją szumu](../images/noise_mitigated_expt.png)
Zamiast implementować odwrotny kanał szumu przez próbkowanie go na QPU jak w PEC, możemy zaimplementować go klasycznie w mierzonej obserwowalnej za pomocą propagacji Pauliego. Prowadzi to do bardziej złożonej obserwowalnej, która podczas pomiaru ma efekt mitygowania wyuczonego szumu bramek.
![Przegląd PNA](../images/pna_overview.png)

### Generowanie lustrzanego Circuit Trottera i obserwowalnej {#generate-the-mirrored-trotter-circuit-and-observable}

W tym eksperymencie będziemy badać dynamikę czasową modelu kopniętego Isinga na 30 węzłach w jednowymiarowym łańcuchu spinowym. Rozważany Hamiltonian to:

$H = -J\sum\limits_{\langle i,j \rangle} Z_iZ_j + h\sum\limits_iX_i$,

gdzie $J>0$ opisuje sprzężenie spinów najbliższych sąsiadów, $i<j$, a globalne poprzeczne pole $h$ jest ustawione na $\frac{\pi}{8}$. Im dalej $h$ jest od kąta Clifforda (tzn. $\theta=n\frac{\pi}{2}, n \in \mathbb{Z}$), tym trudniejsze staje się propagowanie generatorów anty-szumu przez Circuit.

Jako obserwowalną wybierzemy średnią jednowęzłową magnetyzację, $\frac{1}{N} \sum_{i=1}^{N} \langle z_i \rangle$, gdzie $N$ to liczba węzłów.

```python
import numpy as np
from qiskit import QuantumCircuit
from qiskit.quantum_info import Pauli, SparsePauliOp

num_qubits = 30
num_trotter_steps = 10
rx_angle = np.pi / 8

# Avg single-site magnetization
id_pauli = Pauli("I" * num_qubits)
observable = SparsePauliOp([id_pauli.dot(Pauli("Z"), [i]) for i in range(num_qubits)]) / num_qubits

# Implement Trotterized kicked-Ising model
circuit = QuantumCircuit(num_qubits)
for _step in range(num_trotter_steps):
    circuit.rx(rx_angle, range(num_qubits))
    for first_qubit in (1, 2):
        for idx in range(first_qubit, num_qubits, 2):
            # equivalent to Rzz(-pi/2):
            circuit.sdg([idx - 1, idx])
            circuit.cz(idx - 1, idx)
circuit.compose(circuit.inverse(), inplace=True)
circuit.measure_active()
circuit.draw("mpl", fold=-1)
```

### Twirlingowanie warstw 2-Qubitowych Gate i pomiarów oraz znajdowanie unikalnych warstw {#twirl-the-2-qubit-gate-layers-and-measurements-and-find-unique-layers}

Tutaj upewniamy się, że menedżer przejść opatruje boxy adnotacjami ``Twirl`` i ``InjectNoise``, które pozwalają nam wyuczyć szum wpływający na nasz Circuit i powiązać ten szum z odpowiadającą mu warstwą Circuit.

- ``enable_gates/enable_measure: True``: Opakuj wszystkie warstwy Gate 2Q i końcowe pomiary w boxy. Gate jednokubitowe będą lewostronnie ubierane wewnątrz boxów.
- ``measure_annotations: all`` Dołącz adnotacje `Twirl` i `ChangeBasis` do boksa pomiaru.
- ``twirling_strategy: active``: Twirlinguj wszystkie aktywne Qubity w każdym boksie zawierającym Gate splątujące.
- ``inject_noise_targets: gates``: Adnotacje ``InjectNoise`` powinny być dodane do wszystkich boxów opatrzonych adnotacją ``Twirl``, zawierających Gate splątujące.
- ``inject_noise_strategy: uniform_modification``: Wszystkie warstwy szumu powinny być skalowane równoważnie.

```python
from samplomatic.transpiler import generate_boxing_pass_manager

# Box up circuit with Twirl and InjectNoise annotations
pm = generate_boxing_pass_manager(
    enable_gates=True,
    enable_measures=True,
    measure_annotations="all",
    twirling_strategy="active",
    inject_noise_targets="gates",
    inject_noise_strategy="uniform_modification",
    remove_barriers=True,
)
boxed_circuit = pm.run(isa_circuit)
```


### 1c: Konfigurowanie obwodów Trottera {#1c-setting-up-the-trotter-circuits}

Na tym etapie znaleźliśmy nasze współczynniki rozwinięcia $x$ i pozostało już tylko wygenerować kwantowe obwody Trottera.
Po raz kolejny z pomocą przychodzi moduł [qiskit_addon_utils.problem_generators](https://qiskit.github.io/qiskit-addon-utils/stubs/qiskit_addon_utils.problem_generators.html):

```python
from qiskit.synthesis import SuzukiTrotter
from qiskit_addon_utils.problem_generators import generate_time_evolution_circuit

circuits = []
for k in trotter_steps:
    circ = generate_time_evolution_circuit(
        hamiltonian,
        synthesis=SuzukiTrotter(order=2, reps=k),
        time=time,
    )
    circuits.append(circ)
```

## Krok 2: Optymalizacja problemu {#step-2-optimize-the-problem}

Zazwyczaj jest to krok w wzorcu, podczas którego optymalizujesz swoje obwody pod kątem wykonania na sprzęcie.
Tutaj, ponieważ używamy tylko bezszumowego symulatora, po prostu transpilujemy nasz obwód dla [GenericBackendV2](https://quantum.cloud.ibm.com/docs/api/qiskit/qiskit.providers.fake_provider.GenericBackendV2).

```python
from qiskit.providers.fake_provider import GenericBackendV2
from qiskit.transpiler import generate_preset_pass_manager

backend = GenericBackendV2(num_qubits=10)
transpiler = generate_preset_pass_manager(optimization_level=2, backend=backend)

transpiled_circuits = [transpiler.run(circ) for circ in circuits]
```

## Krok 3: Wykonanie eksperymentów kwantowych {#step-3-execute-quantum-experiments}

Jak wyjaśniono na samym początku, pominiemy krok 2 optymalizacji, ponieważ po prostu obliczymy wartości oczekiwane naszego docelowego obserwabla za pomocą bezszumowego symulatora, czyli [StatevectorEstimator](https://quantum.cloud.ibm.com/docs/api/qiskit/qiskit.primitives.StatevectorEstimator).

```python
from qiskit.primitives import StatevectorEstimator

estimator = StatevectorEstimator()
job = estimator.run([(circ, observable) for circ in transpiled_circuits])
result = job.result()
```

## Krok 4: Rekonstrukcja wyników {#step-4-reconstruct-results}

Najpierw wyodrębniamy indywidualne wartości oczekiwane uzyskane dla każdego z obwodów Trottera:

```python
evs = [res.data.evs for res in result]
print(evs)
```

```text
[array(0.23799162), array(0.35754312), array(0.38649906)]
```

Następnie łączymy je po prostu ze współczynnikami MPF, aby uzyskać całkowite wartości oczekiwane MPF. Poniżej robimy to dla każdego z różnych sposobów, w jaki obliczyliśmy $x$.

```python
print("Analytical    solution:", evs @ coeffs_analytical)
print("Exact model   solution:", evs @ coeffs_exact.value)
print("Approx. model solution:", evs @ coeffs_approx.value)
```

```text
Analytical    solution: 0.3954847855980006
Exact model   solution: 0.39548478559800204
Approx. model solution: 0.42991214253489807
```

Na koniec, dla tego małego problemu możemy obliczyć dokładną wartość referencyjną używając [scipy.linalg.expm](https://docs.scipy.org/doc/scipy/reference/generated/scipy.linalg.expm.html) w następujący sposób:

```python
from scipy.linalg import expm

exp_H = expm(-1j * time * hamiltonian.to_matrix())

initial_state = np.zeros(exp_H.shape[0])
initial_state[0] = 1.0

time_evolved_state = exp_H @ initial_state

exact_obs = time_evolved_state.conj() @ observable.to_matrix() @ time_evolved_state
print(exact_obs.real)
```

```text
0.40060242487899755
```

Wyraźnie widać, że MPF zmniejszyło błąd Trottera w porównaniu z tym uzyskanym przy użyciu najgłębszej indywidualnej formuły PF z $k_j=19$.
Jednak widzimy też, że model przybliżony nie jest doskonały, ponieważ w rzeczywistości dał wartość oczekiwaną gorszą niż dokładne rozwiązanie. Pokazuje to znaczenie stosowania ścisłych kryteriów zbieżności w modelu przybliżonym, o czym dowiesz się w przewodniku [Jak używać modelu przybliżonego](https://qiskit.github.io/qiskit-addon-mpf/how_tos/using_approximate_model.html).

## Krok 2: Optymalizacja {#krok-2-optymalizacja}

### Tworzenie nowego Circuit z instrukcjami `Move` umieszczonymi w wybranych miejscach cięcia {#tworzenie-nowego-circuit-z-instrukcjami-move}

Mając powyższy Circuit, chcemy umieścić dwa cięcia linii kwantowych na środkowej linii Qubitów, tak aby Circuit mógł rozdzielić się na dwa Circuit po cztery Qubity każdy. Jednym ze sposobów na to jest ręczne umieszczenie dwuqubitowych instrukcji `Move`, które przenoszą stan z jednej linii Qubitu na drugą. Instrukcja `Move` jest koncepcyjnie równoważna operacji resetu na drugim Qubicie, po której następuje bramka SWAP. Efektem tej instrukcji jest przeniesienie stanu pierwszego (źródłowego) Qubitu na drugi (docelowy) Qubit przy jednoczesnym odrzuceniu przychodzącego stanu drugiego Qubitu. Aby działało to zgodnie z zamierzeniem, ważne jest, aby drugi (docelowy) Qubit nie był splątany z resztą systemu; w przeciwnym razie operacja resetu spowoduje częściowy kolaps stanu reszty systemu.

Tutaj budujemy nowy Circuit z jednym dodatkowym Qubitem i instrukcjami `Move` na miejscu. W tym przykładzie możemy ponownie użyć Qubitu: Qubit źródłowy pierwszej operacji `Move` staje się Qubitem docelowym drugiej operacji `Move`.

Uwaga: Jako alternatywę dla bezpośredniego korzystania z instrukcji `Move` można wybrać oznaczanie cięć linii kwantowych za pomocą jednoQubitowej instrukcji `CutWire`. Funkcja `cut_wires` służy do przekształcania instrukcji `CutWire` na instrukcje `Move` na nowo przydzielonych Qubitach. Jednak w przeciwieństwie do metody ręcznej, ta automatyczna metoda nie umożliwia ponownego użycia linii Qubitów. Szczegóły znajdziesz w [przewodniku jak to zrobić](../how-tos/how_to_specify_cut_wires.ipynb) dla `CutWire`.

```python
from qiskit_addon_cutting.instructions import Move

qc_1 = QuantumCircuit(8)
for i in [*range(4), *range(5, 8)]:
    qc_1.rx(np.pi / 4, i)
qc_1.cx(0, 3)
qc_1.cx(1, 3)
qc_1.cx(2, 3)
qc_1.append(Move(), [3, 4])
qc_1.cx(4, 5)
qc_1.cx(4, 6)
qc_1.cx(4, 7)
qc_1.append(Move(), [4, 3])
qc_1.cx(0, 3)
qc_1.cx(1, 3)
qc_1.cx(2, 3)

qc_1.draw("mpl")
```

### Generowanie subeksperymantów do uruchomienia na Backend {#generowanie-subeksperymantów}

`generate_cutting_experiments` przyjmuje argumenty `circuits`/`observables` jako słowniki odwzorowujące etykiety partycji Qubitów na odpowiednie `subcircuit`/`subobservables`.

Aby zasymulować wartość oczekiwaną pełnego Circuit, z rozłożonych bramek' wspólnego rozkładu quasiprawdopodobieństwa generowanych jest wiele subeksperymantów, które są następnie wykonywane na jednym lub kilku Backend. Liczba próbek pobranych z rozkładu jest kontrolowana przez `num_samples`, a jeden łączny współczynnik jest podawany dla każdej unikalnej próbki. Więcej informacji na temat sposobu obliczania współczynników znajdziesz w [materiałach wyjaśniających](../explanation/index.rst).

```python
from qiskit_addon_cutting import generate_cutting_experiments

subexperiments, coefficients = generate_cutting_experiments(
    circuits=subcircuits, observables=subobservables, num_samples=np.inf
)
```

### Wybór Backend {#wybór-backend}

Używamy tutaj fałszywego Backend, co spowoduje uruchomienie Qiskit Runtime w trybie lokalnym (tzn. na lokalnym symulatorze).

```python
from qiskit_ibm_runtime.fake_provider import FakeManilaV2

backend = FakeManilaV2()
```

### Przygotowanie subeksperymantów dla Backend {#przygotowanie-subeksperymantów}

Musimy transpilować Circuit z naszym Backend jako celem przed przesłaniem ich do Qiskit Runtime.

```python
from qiskit.transpiler import generate_preset_pass_manager

# Transpile the subexperiments to ISA circuits
pass_manager = generate_preset_pass_manager(optimization_level=1, backend=backend)
isa_subexperiments = {
    label: pass_manager.run(partition_subexpts)
    for label, partition_subexpts in subexperiments.items()
}
```

## Krok 3: Uruchomienie {#krok-3-uruchomienie}

### Uruchamianie subeksperymantów za pomocą primitywu Sampler Qiskit Runtime {#uruchamianie-subeksperymantów}

```python
from qiskit_ibm_runtime import SamplerV2, Batch

# Submit each partition's subexperiments to the Qiskit Runtime Sampler
# primitive, in a single batch so that the jobs will run back-to-back.
with Batch(backend=backend) as batch:
    sampler = SamplerV2(mode=batch)
    jobs = {
        label: sampler.run(subsystem_subexpts, shots=2**12)
        for label, subsystem_subexpts in isa_subexperiments.items()
    }
```

### Porównanie odtworzonej wartości oczekiwanej z dokładną wartością oczekiwaną dla oryginalnego Circuit i obserwabli {#porównanie-wartości-oczekiwanej}

```python
from qiskit_aer.primitives import EstimatorV2

estimator = EstimatorV2()
exact_expval = estimator.run([(qc_0, observable)]).result()[0].data.evs
print(f"Reconstructed expectation value: {np.real(np.round(reconstructed_expval, 8))}")
print(f"Exact expectation value: {np.round(exact_expval, 8)}")
print(f"Error in estimation: {np.real(np.round(reconstructed_expval-exact_expval, 8))}")
print(
    f"Relative error in estimation: {np.real(np.round((reconstructed_expval-exact_expval) / exact_expval, 8))}"
)
```

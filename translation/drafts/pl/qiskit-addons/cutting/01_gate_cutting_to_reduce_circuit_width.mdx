W tym notatniku przejdziemy przez kolejne kroki [wzorca Qiskit](https://quantum.cloud.ibm.com/docs/guides/intro-to-patterns), korzystając z **circuit cutting** w celu zmniejszenia liczby Qubitów w Circuit. Będziemy ciąć Gate'y, aby umożliwić rekonstrukcję wartości oczekiwanej czterokubitowego Circuit za pomocą wyłącznie eksperymentów dwukubitowych.

Oto kroki, które wykonamy:

- **Krok 1: Odwzorowanie problemu na Circuit kwantowe i operatory**:
    - Odwzorowanie hamiltonianu na Circuit kwantowy.
- **Krok 2: Optymalizacja pod kątem docelowego sprzętu** [_Używa dodatku cutting_]:
    - <font color='#0F62FE'>Przecięcie Circuit i obserwowalnej.</font>
    - Transpilacja subeksperymontów dla sprzętu.
- **Krok 3: Wykonanie na docelowym sprzęcie**:
    - Uruchomienie subeksperymontów uzyskanych w Kroku 2 za pomocą prymitywu `Sampler`.
- **Krok 4: Post-przetwarzanie wyników** [_Używa dodatku cutting_]:
    - <font color='#0F62FE'>Połączenie wyników z Kroku 3 w celu rekonstrukcji wartości oczekiwanej rozpatrywanej obserwowalnej.</font>

## Krok 2: Optymalizacja {#step-2-optimize}

### Rozdzielanie Circuit i obserwowalnej zgodnie z określonym podziałem Qubitów {#separate-the-circuit-and-observable-according-to-a-specified-qubit-partitioning}

Każda etykieta w `partition_labels` odpowiada Qubitowi `circuit` o tym samym indeksie. Qubity współdzielące wspólną etykietę partycji zostaną zgrupowane, a bramy nieloklane obejmujące więcej niż jedną partycję zostaną przecięte.

**Uwaga:** Argument ``observables`` funkcji `partition_problem` ma typ `PauliList`. Współczynniki i fazy składników obserwowalnej są ignorowane podczas dekompozycji problemu oraz wykonywania subeksperymontów. Mogą zostać ponownie zastosowane podczas rekonstrukcji wartości oczekiwanej.

```python
from qiskit_addon_cutting import partition_problem

partitioned_problem = partition_problem(
    circuit=qc, partition_labels="AABB", observables=observable.paulis
)
subcircuits = partitioned_problem.subcircuits
subobservables = partitioned_problem.subobservables
bases = partitioned_problem.bases
```

### Generowanie subeksperymontów do uruchomienia na Backend {#generate-the-subexperiments-to-run-on-the-backend}

`generate_cutting_experiments` przyjmuje argumenty `circuits`/`observables` jako słowniki odwzorowujące etykiety partycji Qubitów na odpowiednie `subcircuit`/`subobservables`.

Aby zasymulować wartość oczekiwaną pełnego Circuit, z rozkładu kwaziprawdopodobieństwa rozłożonych Gate'ów generowanych jest wiele subeksperymontów, które następnie są wykonywane na jednym lub kilku Backend. Liczba próbek pobieranych z rozkładu jest kontrolowana przez `num_samples`, a dla każdej unikalnej próbki podawany jest jeden połączony współczynnik. Więcej informacji na temat sposobu obliczania współczynników znajdziesz w [materiałach wyjaśniających](../explanation/index.rst).

```python
from qiskit_addon_cutting import generate_cutting_experiments

subexperiments, coefficients = generate_cutting_experiments(
    circuits=subcircuits, observables=subobservables, num_samples=np.inf
)
```

### Wybór Backend {#choose-a-backend}

Tutaj używamy fałszywego Backend, co spowoduje uruchomienie Qiskit Runtime w trybie lokalnym (tj. na lokalnym symulatorze).

```python
from qiskit_ibm_runtime.fake_provider import FakeManilaV2

backend = FakeManilaV2()
```

### Przygotowanie subeksperymontów dla Backend {#prepare-the-subexperiments-for-the-backend}

Przed przesłaniem Circuit do Qiskit Runtime musimy je transpilować z naszym Backend jako celem.

```python
from qiskit.transpiler import generate_preset_pass_manager

# Transpile the subexperiments to ISA circuits
pass_manager = generate_preset_pass_manager(optimization_level=1, backend=backend)
isa_subexperiments = {
    label: pass_manager.run(partition_subexpts)
    for label, partition_subexpts in subexperiments.items()
}
```

## Krok 3: Wykonanie {#step-3-execute}

### Uruchamianie subeksperymontów za pomocą prymitywu Sampler z Qiskit Runtime {#run-the-subexperiments-using-the-qiskit-runtime-sampler-primitive}

```python
from qiskit_ibm_runtime import SamplerV2, Batch

# Submit each partition's subexperiments to the Qiskit Runtime Sampler
# primitive, in a single batch so that the jobs will run back-to-back.
with Batch(backend=backend) as batch:
    sampler = SamplerV2(mode=batch)
    jobs = {
        label: sampler.run(subsystem_subexpts, shots=2**12)
        for label, subsystem_subexpts in isa_subexperiments.items()
    }
```

## Krok 4: Post-przetwarzanie {#step-4-post-process}

### Rekonstrukcja wartości oczekiwanej {#reconstruct-the-expectation-value}

Rekonstruuj wartości oczekiwane dla każdego składnika obserwowalnej i połącz je, aby odtworzyć wartość oczekiwaną pierwotnej obserwowalnej.

```python
from qiskit_addon_cutting import reconstruct_expectation_values

# Get expectation values for each observable term
reconstructed_expval_terms = reconstruct_expectation_values(
    results,
    coefficients,
    subobservables,
)

# Reconstruct final expectation value
reconstructed_expval = np.dot(reconstructed_expval_terms, observable.coeffs)
```

### Porównanie zrekonstruowanej wartości oczekiwanej z dokładną wartością oczekiwaną z oryginalnego Circuit i obserwowalnej {#compare-the-reconstructed-expectation-value-with-the-exact-expectation-value-from-the-original-circuit-and-observable}

```python
from qiskit_aer.primitives import EstimatorV2

estimator = EstimatorV2()
exact_expval = estimator.run([(qc, observable)]).result()[0].data.evs
print(f"Reconstructed expectation value: {np.real(np.round(reconstructed_expval, 8))}")
print(f"Exact expectation value: {np.round(exact_expval, 8)}")
print(f"Error in estimation: {np.real(np.round(reconstructed_expval-exact_expval, 8))}")
print(
    f"Relative error in estimation: {np.real(np.round((reconstructed_expval-exact_expval) / exact_expval, 8))}"
)
```

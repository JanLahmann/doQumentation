W tym notatniku przejdziemy przez kolejne kroki [wzorca Qiskit](https://quantum.cloud.ibm.com/docs/guides/intro-to-patterns), używając **przybliżonej kompilacji kwantowej z sieciami tensorowymi (AQC-Tensor)**, aby uzyskać mniejszą głębokość Circuit niż byłoby to normalnie potrzebne do przeprowadzenia ewolucji Trottera.

Oto kroki, które wykonamy:

- **Krok 1: Mapowanie na problem kwantowy**
    - Zainicjalizowanie Hamiltonianu problemu i obserwowalnych
    - <font color='#0F62FE'>Wygenerowanie docelowego stanu sieci tensorowej dla początkowej części Circuit</font>
    - <font color='#0F62FE'>Wygenerowanie Circuit o małej głębokości przybliżającej kompresowaną część</font>
    - <font color='#0F62FE'>Wygenerowanie ogólnego ansatz na podstawie tego Circuit</font>
    - <font color='#0F62FE'>Optymalizacja parametrów, aby ansatz był jak najbliżej celu</font>
    - <font color='#0F62FE'>Dodanie kolejnych kroków Trottera do zoptymalizowanego ansatz</font>
- **Krok 2: Optymalizacja pod kątem docelowego sprzętu**
    - Transpilacja Circuit dla sprzętu
- **Krok 3: Wykonanie eksperymentów**
    - Użycie fałszywego Backend dla uproszczenia
- **Krok 4: Rekonstrukcja wyników**
    - Nie dotyczy; zamiast tego po prostu wypisujemy zmierzoną obserwowalną
## Krok 1: Mapowanie na Circuit kwantowy i operator {#step-1-map-to-quantum-circuit-and-operator}

### Konfiguracja modelowego Hamiltonianu i obserwowalnej {#set-up-a-model-hamiltonian-and-observable}

W tym notatniku używamy modelu Isinga na okręgu złożonym z 10 węzłów:
$$
\hat{\mathcal{H}}_{\text{Ising}} = \sum_{i=1}^{10} J_{i,(i+1)} Z_i Z_{(i+1)} + h_i X_i \, ,
$$
gdzie periodyczne warunki brzegowe oznaczają, że dla $i=10$ otrzymujemy $i+1=11\rightarrow1$, $J$ to siła sprzężenia między dwoma węzłami, a $h$ to zewnętrzne pole magnetyczne.

```python
# Dodane przez doQumentation — pakiety wymagane w tym notatniku
!pip install -q qiskit qiskit-addon-aqc-tensor qiskit-addon-utils qiskit-ibm-runtime quimb scipy
```

```python
from qiskit.transpiler import CouplingMap
from qiskit_addon_utils.problem_generators import generate_xyz_hamiltonian

# Wygenerowanie przykładowej mapy sprzężeń dla tego przykładu
coupling_map = CouplingMap.from_heavy_hex(3, bidirectional=False)

# Wybór okręgu 10-qubitowego na tej mapie sprzężeń
reduced_coupling_map = coupling_map.reduce([0, 13, 1, 14, 10, 16, 4, 15, 3, 9])

# Pobranie operatora qubitowego opisującego model pola Isinga
hamiltonian = generate_xyz_hamiltonian(
    reduced_coupling_map,
    coupling_constants=(0.0, 0.0, 1.0),
    ext_magnetic_field=(0.4, 0.0, 0.0),
)
```

Obserwowalną, którą będziemy mierzyć, jest całkowite namagnesowanie.

```python
from qiskit.quantum_info import SparsePauliOp

L = reduced_coupling_map.size()
observable = SparsePauliOp.from_sparse_list([("Z", [i], 1 / L / 2) for i in range(L)], num_qubits=L)
```

### Określenie, ile ewolucji czasowej symulować klasycznie {#determine-how-much-of-the-time-evolution-to-simulate-classically}

Naszym głównym celem jest symulacja ewolucji czasowej powyższego modelowego Hamiltonianu. Robimy to przez ewolucję Trottera, którą dzielimy na dwie części:

1. Część początkową, którą można symulować za pomocą stanów product matrix (MPS). Tę część „skompilujemy" przy użyciu AQC zgodnie z opisem w https://arxiv.org/abs/2301.08609.
2. Kolejną część Circuit, która zostanie wykonana na sprzęcie.
Planujemy użyć AQC-Tensor do skompresowania naszego Circuit ewolucji czasowej do czasu $t=4$, a następnie ewoluować zwykłymi krokami Trottera do $t=5$.
### Generowanie Circuit przed i po podziale {#generate-circuits-before-and-after-split}

Skoro zdecydowaliśmy się podzielić w $t=4$, wygenerujemy dwa Circuit:

1. Circuit „docelowy" dla części AQC ewolucji, od $t_i=0$ do $t_f=4$. Ponieważ jest on symulowany przez symulator sieci tensorowej, liczba warstw wpływa na czas wykonania jedynie o stały czynnik, więc możemy śmiało użyć dużej liczby warstw, aby zminimalizować błąd Trottera.

```python
from qiskit.synthesis import SuzukiTrotter
from qiskit_addon_utils.problem_generators import generate_time_evolution_circuit

aqc_evolution_time = 4.0
aqc_target_num_trotter_steps = 45

aqc_target_circuit = generate_time_evolution_circuit(
    hamiltonian,
    synthesis=SuzukiTrotter(reps=aqc_target_num_trotter_steps),
    time=aqc_evolution_time,
)
```

2. Kolejny Circuit ewolucji, który ewoluuje od $t_i=4$ do $t_f=5$. Ponieważ będzie on uruchamiany na sprzęcie kwantowym, pożądane jest użycie jak najmniejszej liczby warstw Trottera.

```python
subsequent_evolution_time = 1.0
subsequent_num_trotter_steps = 5

subsequent_circuit = generate_time_evolution_circuit(
    hamiltonian,
    synthesis=SuzukiTrotter(reps=subsequent_num_trotter_steps),
    time=subsequent_evolution_time,
)
```

Dla celów późniejszego porównania wygenerujemy też trzeci Circuit: taki, który ewoluuje przez `aqc_evolution_time`, ale ma ten sam czas ewolucji na krok Trottera co kolejny Circuit. To ten Circuit, z którym pracowalibyśmy, gdybyśmy nie użyli dużej liczby kroków Trottera dla Circuit docelowego. Będziemy go nazywać _Circuit porównawczym_.

```python
aqc_comparison_num_trotter_steps = int(
    subsequent_num_trotter_steps / subsequent_evolution_time * aqc_evolution_time
)
aqc_comparison_num_trotter_steps
```

```text
20
```

```python
comparison_circuit = generate_time_evolution_circuit(
    hamiltonian,
    synthesis=SuzukiTrotter(reps=aqc_comparison_num_trotter_steps),
    time=aqc_evolution_time,
)
```

### Generowanie ansatz i parametrów początkowych z Circuit Trottera z mniejszą liczbą kroków {#generate-an-ansatz-and-initial-parameters-from-a-trotter-circuit-with-fewer-steps}

Na początku budujemy „dobry" Circuit o tym samym czasie ewolucji co Circuit docelowy, ale z mniejszą liczbą kroków Trottera (a zatem i mniejszą liczbą warstw).

Następnie przekazujemy ten „dobry" Circuit do funkcji `generate_ansatz_from_circuit` z AQC-Tensor. Funkcja ta analizuje dwuqubitową łączność Circuit i zwraca dwie rzeczy:
1. ogólny, sparametryzowany Circuit ansatz o tej samej dwuqubitowej łączności co Circuit wejściowy; oraz
2. parametry, które po podstawieniu do ansatz dają wejściowy (dobry) Circuit.

Wkrótce weźmiemy te parametry i będziemy je iteracyjnie dostosowywać, aby Circuit ansatz był jak najbliżej docelowego MPS.

```python
from qiskit_addon_aqc_tensor import generate_ansatz_from_circuit

aqc_ansatz_num_trotter_steps = 5

aqc_good_circuit = generate_time_evolution_circuit(
    hamiltonian,
    synthesis=SuzukiTrotter(reps=aqc_ansatz_num_trotter_steps),
    time=aqc_evolution_time,
)

aqc_ansatz, aqc_initial_parameters = generate_ansatz_from_circuit(
    aqc_good_circuit, qubits_initially_zero=True
)
aqc_ansatz.draw("mpl", fold=-1)
```

### Wybór ustawień dla symulacji sieci tensorowej {#choose-settings-for-tensor-network-simulation}

Tutaj używamy symulatora sieci tensorowej opartego na [quimb](http://quimb.readthedocs.io/). W tym przykładzie korzystamy z symulatora stanów product matrix (MPS) z quimb i używamy [JAX](https://docs.jax.dev/en/latest/) do automatycznego różniczkowania. Więcej informacji o korzystaniu z symulatora quimb znajdziesz w [dokumentacji API](../stubs/qiskit_addon_aqc_tensor.simulation.quimb.QuimbSimulator.rst).

```python
from functools import partial

import quimb.tensor

from qiskit_addon_aqc_tensor.simulation.quimb import QuimbSimulator

simulator_settings = QuimbSimulator(
    partial(quimb.tensor.CircuitMPS, max_bond=100, cutoff=1e-8),
    autodiff_backend="jax",
)
```

### Konstruowanie reprezentacji matrix-product state docelowego stanu AQC {#construct-matrix-product-state-representation-of-the-aqc-target-state}

Następnie budujemy reprezentację matrix-product stanu, który ma być przybliżony przez AQC.

```python
from qiskit_addon_aqc_tensor.simulation import tensornetwork_from_circuit

aqc_target_mps = tensornetwork_from_circuit(aqc_target_circuit, simulator_settings)
```

Zwróć uwagę, że ponieważ wybraliśmy dużą liczbę kroków Trottera dla stanu docelowego, ma on w rzeczywistości mniejszy błąd Trottera niż Circuit porównawczy. Możemy obliczyć wierność ($| \langle \psi_1 | \psi_2 \rangle |^2$) stanu przygotowanego przez Circuit porównawczy w stosunku do stanu docelowego:

```python
from qiskit_addon_aqc_tensor.simulation import compute_overlap

comparison_mps = tensornetwork_from_circuit(comparison_circuit, simulator_settings)
comparison_fidelity = abs(compute_overlap(comparison_mps, aqc_target_mps)) ** 2
comparison_fidelity
```

### Optymalizacja parametrów ansatz przy użyciu obliczeń MPS {#optimize-the-parameters-of-the-ansatz-using-mps-calculations}

Tutaj minimalizujemy najprostszą możliwą funkcję kosztu, `MaximizeStateFidelity`, używając optymalizatora L-BFGS z scipy.

Wybieramy punkt zatrzymania dla wierności taki, aby była ona wyższa niż wierność Circuit porównawczego bez użycia AQC. Gdy zostanie on osiągnięty, skompresowany Circuit ma mniejszy błąd Trottera _i_ mniejszą głębokość niż oryginalny Circuit. Przy większej ilości czasu obliczeniowego można wykonać dalsze kroki optymalizacji, aby jeszcze bardziej zwiększyć wierność.

```python
from scipy.optimize import OptimizeResult, minimize

from qiskit_addon_aqc_tensor.objective import MaximizeStateFidelity

objective = MaximizeStateFidelity(aqc_target_mps, aqc_ansatz, simulator_settings)

stopping_point = 1 - comparison_fidelity

def callback(intermediate_result: OptimizeResult):
    print(f"Intermediate result: Fidelity {1 - intermediate_result.fun:.8}")
    if intermediate_result.fun < stopping_point:
        # Wystarczająco dobre na teraz
        raise StopIteration

result = minimize(
    objective.loss_function,
    aqc_initial_parameters,
    method="L-BFGS-B",
    jac=True,
    options={"maxiter": 100},
    callback=callback,
)
if result.status not in (
    0,
    1,
    99,
):  # 0 => sukces; 1 => osiągnięto maks. iteracji; 99 => wczesne zakończenie przez StopIteration
    raise RuntimeError(f"Optimization failed: {result.message} (status={result.status})")

print(f"Done after {result.nit} iterations.")
aqc_final_parameters = result.x
```

## Krok 2: Transpilacja do wykonania na docelowym sprzęcie {#step-2-transpile-for-execution-on-target-hardware}

W kroku 2 [wzorca Qiskit](https://quantum.cloud.ibm.com/docs/guides/intro-to-patterns) transpilujemy ten Circuit oraz wszelkie pożądane obserwowalne do wykonania na docelowym urządzeniu. Tutaj używamy fałszywego Backend dostarczonego przez `qiskit-ibm-runtime`.

```python
from qiskit import transpile
from qiskit_ibm_runtime.fake_provider import FakeMelbourneV2

backend = FakeMelbourneV2()

isa_circuit = transpile(final_circuit, backend)
isa_observable = observable.apply_layout(isa_circuit.layout)
```

Wynikowy Circuit ISA może następnie zostać wysłany do wykonania na Backend (krok 3 [wzorca Qiskit](https://quantum.cloud.ibm.com/docs/guides/intro-to-patterns)).
## Krok 3: Wykonanie na sprzęcie kwantowym {#step-3-execute-on-quantum-hardware}

```python
from qiskit_ibm_runtime import EstimatorV2 as Estimator

estimator = Estimator(backend)
job = estimator.run([(isa_circuit, isa_observable)])
pub_result = job.result()[0]
```

W tym samouczku implementujemy [wzorzec Qiskit](https://quantum.cloud.ibm.com/docs/guides/intro-to-patterns) pokazujący, jak przetwarzać zaszumione próbki kwantowe w celu znalezienia przybliżenia stanu podstawowego hamiltoniana chemicznego: cząsteczki $N_2$ w równowadze w bazie 6-31G. Będziemy stosować [podejście kwantowej diagonalizacji opartej na próbkowaniu (SQD)](https://arxiv.org/abs/2405.05068) do przetwarzania próbek pobranych z obwodu kwantowego ansatzu o ``36`` qubitach (w tym przypadku obwód LUCJ). Aby uwzględnić wpływ szumu kwantowego, stosuje się technikę odtwarzania konfiguracji.

Wzorzec można opisać w czterech krokach:

1. **Krok 1: Odwzorowanie na problem kwantowy**
    - Wygeneruj ansatz do szacowania stanu podstawowego
2. **Krok 2: Optymalizacja problemu**
    - Transpiluj ansatz dla backendu
3. **Krok 3: Wykonanie eksperymentów**
    - Pobierz próbki z ansatzu za pomocą prymitywu ``Sampler``
4. **Krok 4: Postprzetwarzanie wyników**
   - Samospójna pętla odtwarzania konfiguracji
       - Postprzyprzetwórz pełny zbiór próbek bitstring, korzystając z wiedzy a priori o liczbie cząstek i średnim obsadzeniu orbitali obliczonym w poprzedniej iteracji.
       - Probabilistycznie twórz partie podpróbek z odtworzonych bitstringów.
       - Projektuj i diagonalizuj hamiltonan molekularny w każdej podprzestrzeni próbki.
       - Zapisuj minimalną znalezioną energię stanu podstawowego ze wszystkich partii i aktualizuj średnie obsadzenie orbitali.

Dla tego przykładu hamiltonian elektronów oddziałujących przyjmuje ogólną postać:

$$
\hat{H} = \sum_{ \substack{pr\\\sigma} } h_{pr} \, \hat{a}^\dagger_{p\sigma} \hat{a}_{r\sigma}
+ 
\sum_{ \substack{prqs\\\sigma\tau} }
\frac{(pr|qs)}{2} \, 
\hat{a}^\dagger_{p\sigma}
\hat{a}^\dagger_{q\tau}
\hat{a}_{s\tau}
\hat{a}_{r\sigma}
$$

$\hat{a}^\dagger_{p\sigma}$/$\hat{a}_{p\sigma}$ to fermiońskie operatory kreacji/anihilacji skojarzone z $p$-tym elementem bazy i spinem $\sigma$. $h_{pr}$ i $(pr|qs)$ to jedno- i dwuciałowe całki elektronowe.

Przepływ pracy SQD z samospójnym odtwarzaniem konfiguracji przedstawiono na poniższym schemacie.

![SQD diagram](../_static/images/sqd_diagram.png)

Wiadomo, że SQD działa dobrze, gdy docelowy stan własny jest rzadki: funkcja falowa jest zdefiniowana na zbiorze stanów bazowych $\mathcal{S} = \{|x\rangle \}$, którego rozmiar nie rośnie wykładniczo wraz z rozmiarem problemu. W tym scenariuszu diagonalizacja hamiltoniana rzutowanego na podprzestrzeń zdefiniowaną przez $\mathcal{S}$:
$$
H_\mathcal{S} = P_\mathcal{S} H  P_\mathcal{S} \textrm{ with } P_\mathcal{S} = \sum_{x \in \mathcal{S}} |x \rangle \langle x |;
$$
daje dobre przybliżenie docelowego stanu własnego. Rola urządzenia kwantowego polega na produkowaniu próbek elementów $\mathcal{S}$ wyłącznie. Najpierw obwód kwantowy przygotowuje stan $|\Psi\rangle$ na urządzeniu kwantowym. Stosowane jest kodowanie Jordana-Wignera. W konsekwencji elementy bazy obliczeniowej reprezentują stany Focka (konfiguracje/determinanty elektronowe). Obwód jest próbkowany w bazie obliczeniowej, dając zbiór zaszumionych konfiguracji $\tilde{\mathcal{X}}$. Konfiguracje są reprezentowane przez bitstrings. Zbiór $\tilde{\mathcal{X}}$ jest następnie przekazywany do bloku klasycznego postprzetwarzania, gdzie stosuje się [samospójną technikę odtwarzania konfiguracji](https://arxiv.org/abs/2405.05068). W ramach SQD rola urządzenia kwantowego polega na wytworzeniu rozkładu prawdopodobieństwa.
### Krok 1: Odwzorowanie problemu na obwód kwantowy {#step-1-map-problem-to-a-quantum-circuit}

W tym samouczku będziemy przybliżać energię stanu podstawowego cząsteczki $N_2$. Najpierw określimy cząsteczkę i jej właściwości. Następnie stworzymy ansatz [lokalnego unitarnego klastra Jastrow (LUCJ)](https://pubs.rsc.org/en/content/articlelanding/2023/sc/d3sc02516k) (obwód kwantowy), aby generować próbki z komputera kwantowego do szacowania energii stanu podstawowego.

Najpierw określimy cząsteczkę i jej właściwości.

```python
import warnings

warnings.filterwarnings("ignore")

import pyscf
import pyscf.cc
import pyscf.mcscf

# Specify molecule properties
open_shell = False
spin_sq = 0

# Build N2 molecule
mol = pyscf.gto.Mole()
mol.build(
    atom=[["N", (0, 0, 0)], ["N", (1.0, 0, 0)]],
    basis="6-31g",
    symmetry="Dooh",
)

# Define active space
n_frozen = 2
active_space = range(n_frozen, mol.nao_nr())

# Get molecular integrals
scf = pyscf.scf.RHF(mol).run()
num_orbitals = len(active_space)
n_electrons = int(sum(scf.mo_occ[active_space]))
num_elec_a = (n_electrons + mol.spin) // 2
num_elec_b = (n_electrons - mol.spin) // 2
cas = pyscf.mcscf.CASCI(scf, num_orbitals, (num_elec_a, num_elec_b))
mo = cas.sort_mo(active_space, base=0)
hcore, nuclear_repulsion_energy = cas.get_h1cas(mo)
eri = pyscf.ao2mo.restore(1, cas.get_h2cas(mo), num_orbitals)

# Compute exact energy
exact_energy = cas.run().e_tot
```

```text
CASCI E = -109.046671778080  E(CI) = -32.8155692383188  S^2 = 0.0000000
```

Następnie stworzymy ansatz. Ansatz ``LUCJ`` to parametryzowany obwód kwantowy, który zainicjalizujemy amplitudami `t2` i `t1` uzyskanymi z obliczenia CCSD.

```python
# Get CCSD t2 amplitudes for initializing the ansatz
ccsd = pyscf.cc.CCSD(scf, frozen=[i for i in range(mol.nao_nr()) if i not in active_space]).run()
t1 = ccsd.t1
t2 = ccsd.t2
```

```text
E(CCSD) = -109.0398256929734  E_corr = -0.2045891221988311
```

Użyjemy pakietu [ffsim](https://github.com/qiskit-community/ffsim/tree/main) do stworzenia i zainicjalizowania ansatzu amplitudami `t2` i `t1` obliczonymi powyżej. Ponieważ nasza cząsteczka ma zamknięty stan Hartree-Focka, użyjemy symetrycznie zrównoważonego wariantu ansatzu UCJ, [UCJOpSpinBalanced](https://qiskit-community.github.io/ffsim/api/ffsim.html#ffsim.UCJOpSpinBalanced).

Ponieważ docelowy sprzęt IBM ma topologię heavy-hex, przyjmiemy [wzorzec _zig-zag_ dla interakcji qubitów](https://pubs.rsc.org/en/content/articlehtml/2023/sc/d3sc02516k). W tym wzorcu orbitale (reprezentowane przez qubity) o tym samym spinie są połączone topologią liniową (czerwone i niebieskie kółka), gdzie każda linia przyjmuje kształt zig-zag ze względu na łączność heavy-hex docelowego sprzętu. Ponownie, ze względu na topologię heavy-hex, orbitale różnych spinów mają połączenia co każdy 4. orbital (0, 4, 8 itd.) (fioletowe kółka).

![lucj_ansatz](../_static/images/lucj_ansatz_zig_zag_pattern.jpg)

```python
import ffsim
from qiskit import QuantumCircuit, QuantumRegister

n_reps = 1
alpha_alpha_indices = [(p, p + 1) for p in range(num_orbitals - 1)]
alpha_beta_indices = [(p, p) for p in range(0, num_orbitals, 4)]

ucj_op = ffsim.UCJOpSpinBalanced.from_t_amplitudes(
    t2=t2,
    t1=t1,
    n_reps=n_reps,
    interaction_pairs=(alpha_alpha_indices, alpha_beta_indices),
)

nelec = (num_elec_a, num_elec_b)

# create an empty quantum circuit
qubits = QuantumRegister(2 * num_orbitals, name="q")
circuit = QuantumCircuit(qubits)

# prepare Hartree-Fock state as the reference state and append it to the quantum circuit
circuit.append(ffsim.qiskit.PrepareHartreeFockJW(num_orbitals, nelec), qubits)

# apply the UCJ operator to the reference state
circuit.append(ffsim.qiskit.UCJOpSpinBalancedJW(ucj_op), qubits)
circuit.measure_all()
```

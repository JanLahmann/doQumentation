W tym samouczku implementujemy [wzorzec Qiskit](https://quantum.cloud.ibm.com/docs/guides/intro-to-patterns) pokazujący, jak post-processować zaszumione próbki kwantowe w celu znalezienia przybliżenia stanu podstawowego fermionowego hamiltonianuu sieciowego, zwanego modelem Andersona z pojedynczą zanieczyszczeniem. Będziemy stosować podejście kwantowej diagonalizacji opartej na próbkach do przetwarzania próbek pobranych z zestawu ``16``-qubitowych stanów bazy Kryłowa w rosnących przedziałach czasowych. Stany te są realizowane na urządzeniu kwantowym przy użyciu trotteryzacji ewolucji czasowej. Aby uwzględnić wpływ szumu kwantowego, stosuje się technikę odtwarzania konfiguracji. Zakładając dobry stan początkowy i rzadkość stanu podstawowego, [podejście to jest udowodnione jako efektywnie zbieżne](https://arxiv.org/abs/2501.09702).

Wzorzec można opisać w czterech krokach:

1. **Krok 1: Odwzorowanie na problem kwantowy**
    - Wygenerowanie zestawu stanów bazy Kryłowa (tzn. trotteryzowanych Circuit ewolucji czasowej) w rosnących przedziałach czasowych dla estymacji stanu podstawowego
2. **Krok 2: Optymalizacja problemu**
    - Transpilacja Circuit dla Backend
3. **Krok 3: Wykonanie eksperymentów**
    - Pobieranie próbek z Circuit za pomocą primitywu ``Sampler``
4. **Krok 4: Post-processing wyników**
   - Pętla samokonsystentnego odtwarzania konfiguracji
       - Post-processing pełnego zestawu próbek bitstrings, z wykorzystaniem wcześniejszej wiedzy o liczbie cząstek i średnim zajęciu orbitali obliczonym w ostatniej iteracji
       - Probabilistyczne tworzenie wsadów podpróbek z odtworzonych bitstrings
       - Rzutowanie i diagonalizacja fermionowego hamiltonianuu sieciowego w każdej podprzestrzeni próbkowania
       - Zapisywanie minimalnej energii stanu podstawowego znalezionej we wszystkich wsadach i aktualizacja średniego zajęcia orbitali
### Krok 1: Odwzorowanie problemu na Circuit kwantowy {#krok-1-odwzorowanie-problemu-na-circuit-kwantowy}
Najpierw stworzymy jednoelektronowe i dwuelektronowe hamiltoniany opisujące jednowymiarowy model Andersona z pojedynczą zanieczyszczeniem (SIAM) z ``7`` miejscami kąpielowymi (``8`` elektronów w ``8`` orbitalach). Model ten służy do opisu magnetycznych zanieczyszczeń w metalach.

Następnie stworzymy ``16``-qubitowe Circuit Trottera używane do wygenerowania kwantowej podprzestrzeni Kryłowa.

```python
import numpy as np

n_bath = 7  # number of bath sites

V = 1  # hybridization amplitude
t = 1  # bath hopping amplitude
U = 10  # Impurity onsite repulsion
eps = -U / 2  # Chemical potential for the impurity

# Place the impurity on the first qubit
impurity_index = 0

# One body matrix elements in the "position" basis
h1e = -t * np.diag(np.ones(n_bath), k=1) - t * np.diag(np.ones(n_bath), k=-1)
h1e[impurity_index, impurity_index + 1] = -V
h1e[impurity_index + 1, impurity_index] = -V
h1e[impurity_index, impurity_index] = eps

# Two body matrix elements in the "position" basis
h2e = np.zeros((n_bath + 1, n_bath + 1, n_bath + 1, n_bath + 1))
h2e[impurity_index, impurity_index, impurity_index, impurity_index] = U
```

Następnie wygenerujemy kwantową podprzestrzeń Kryłowa za pomocą zestawu trotteryzowanych Circuit kwantowych. Tutaj tworzymy funkcje pomocnicze do generowania stanu początkowego (referencyjnego) oraz ewolucji czasowej dla jednoelektronowej i dwuelektronowej części hamiltonianuu. Aby uzyskać bardziej szczegółowy opis tego modelu i sposobu projektowania Circuit, zapoznaj się z [tą pracą](https://arxiv.org/abs/2501.09702).

```python
import ffsim
import scipy
from qiskit import QuantumCircuit, QuantumRegister
from qiskit.circuit.library import CPhaseGate, XGate, XXPlusYYGate

n_modes = n_bath + 1
nelec = (n_modes // 2, n_modes // 2)

dt = 0.2
Utar = scipy.linalg.expm(-1j * dt * h1e)

# The reference state
def initial_state(q_circuit, norb, nocc):
    """Prepare an initial state."""
    for i in range(nocc):
        q_circuit.append(XGate(), [i])
        q_circuit.append(XGate(), [norb + i])
    rot = XXPlusYYGate(np.pi / 2, -np.pi / 2)

    for i in range(3):
        for j in range(nocc - i - 1, nocc + i, 2):
            q_circuit.append(rot, [j, j + 1])
            q_circuit.append(rot, [norb + j, norb + j + 1])
    q_circuit.append(rot, [j + 1, j + 2])
    q_circuit.append(rot, [norb + j + 1, norb + j + 2])

# The one-body time evolution
free_fermion_evolution = ffsim.qiskit.OrbitalRotationJW(n_modes, Utar)

# The two-body time evolution
def append_diagonal_evolution(dt, U, impurity_qubit, num_orb, q_circuit):
    """Append two-body time evolution to a quantum circuit."""
    if U != 0:
        q_circuit.append(
            CPhaseGate(-dt / 2 * U),
            [impurity_qubit, impurity_qubit + num_orb],
        )
```

Generujemy ``d`` stanów po ewolucji czasowej, które określają kwantową podprzestrzeń Kryłowa. Tutaj wybraliśmy ``d=8``. Błąd wynikający z próbkowania stanów bazy Kryłowa zbiega się wraz ze wzrostem ``d``. Zwróć uwagę, że specyfika tej instancji problemu pozwala na efektywną kompilację ewolucji jednoelektronowej przy użyciu `OrbitalRotationJW`; jednak ogólnie można użyć [PauliEvolutionGate](https://quantum.cloud.ibm.com/docs/api/qiskit/qiskit.circuit.library.PauliEvolutionGate) z Qiskit do implementacji trotteryzowanej ewolucji czasowej pełnego hamiltonianuu.

```python
# Generate the initial state
qubits = QuantumRegister(2 * n_modes, name="q")
init_state = QuantumCircuit(qubits)
initial_state(init_state, n_modes, n_modes // 2)
init_state.draw("mpl", scale=0.4, fold=-1)

d = 8  # Number of Krylov basis states
circuits = []
for i in range(d):
    circ = init_state.copy()
    circuits.append(circ)
    for _ in range(i):
        append_diagonal_evolution(dt, U, impurity_index, n_modes, circ)
        circ.append(free_fermion_evolution, qubits)
        append_diagonal_evolution(dt, U, impurity_index, n_modes, circ)
    circ.measure_all()
```

### Krok 2: Optymalizacja problemu {#krok-2-optymalizacja-problemu}
Po stworzeniu trotteryzowanych Circuit zoptymalizujemy je dla docelowego sprzętu. Musimy wybrać urządzenie sprzętowe przed optymalizacją. Użyjemy fałszywego Backend 127-qubitowego z ``qiskit_ibm_runtime``, aby emulować prawdziwe urządzenie. Aby uruchomić na prawdziwym QPU, wystarczy zastąpić fałszywy Backend prawdziwym. Więcej informacji znajdziesz w [dokumentacji Qiskit IBM Runtime](https://quantum.cloud.ibm.com/docs/guides/get-started-with-primitives#get-started-with-sampler).

```python
from qiskit_ibm_runtime.fake_provider.backends import FakeSherbrooke

backend = FakeSherbrooke()
```

Następnie dokonamy transpilacji Circuit do docelowego Backend za pomocą Qiskit.

```python
from qiskit.transpiler import generate_preset_pass_manager

# The circuit needs to be transpiled to the AerSimulator target
pass_manager = generate_preset_pass_manager(optimization_level=3, backend=backend)
isa_circuits = pass_manager.run(circuits)
```

### Krok 3: Wykonanie eksperymentów {#krok-3-wykonanie-eksperymentów}
Po zoptymalizowaniu Circuit dla sprzętu jesteśmy gotowi do ich uruchomienia na docelowym sprzęcie i zbierania próbek do estymacji energii stanu podstawowego. Tutaj używamy ``SamplerV2`` z ``qiskit-ibm-runtime`` do symulowania zaszumionych próbek pobranych z Backend ``ibm_sherbrooke``. Następnie łączymy zliczenia z każdego ze stanów bazy Kryłowa w jeden słownik zliczeń i wykreślamy 20 najczęściej próbkowanych bitstrings.

***Uwaga: [Qiskit Aer](https://qiskit.github.io/qiskit-aer/index.html) jest wymagany do symulowania próbek z transpilowanych Circuit.***

```python
from qiskit.primitives import BitArray
from qiskit.visualization import plot_histogram
from qiskit_ibm_runtime import SamplerV2 as Sampler

# Sample from the circuits
noisy_sampler = Sampler(backend, options={"simulator": {"seed_simulator": 24}})
job = noisy_sampler.run(isa_circuits, shots=500)

# Combine the counts from the individual Trotter circuits
bit_array = BitArray.concatenate_shots([result.data.meas for result in job.result()])

plot_histogram(bit_array.get_counts(), number_to_keep=20)
```

### Krok 4: Post-processing wyników {#krok-4-post-processing-wyników}
Teraz uruchamiamy algorytm SQD za pomocą funkcji `diagonalize_fermionic_hamiltonian`. Objaśnienie argumentów tej funkcji znajdziesz w [dokumentacji API](../apidocs/qiskit_addon_sqd.fermion.rst#qiskit_addon_sqd.fermion.diagonalize_fermionic_hamiltonian).

```python
from qiskit_addon_sqd.fermion import SCIResult, diagonalize_fermionic_hamiltonian

# List to capture intermediate results
result_history = []

def callback(results: list[SCIResult]):
    result_history.append(results)
    iteration = len(result_history)
    print(f"Iteration {iteration}")
    for i, result in enumerate(results):
        print(f"\tSubsample {i}")
        print(f"\t\tEnergy: {result.energy}")
        print(f"\t\tSubspace dimension: {np.prod(result.sci_state.amplitudes.shape)}")

rng = np.random.default_rng(24)
result = diagonalize_fermionic_hamiltonian(
    h1e,
    h2e,
    bit_array,
    samples_per_batch=300,
    norb=n_modes,
    nelec=nelec,
    num_batches=3,
    max_iterations=10,
    symmetrize_spin=True,
    callback=callback,
    seed=rng,
)
```

Teraz wykreślimy wyniki. Pierwszy wykres pokazuje, że po kilku iteracjach uzyskujemy dokładną energię stanu podstawowego.

Ten przykład jest wystarczająco mały, aby móc zbadać pełną przestrzeń Hilberta, co widać w powyższych wydrukach. Pamiętaj, że pełna przestrzeń Hilberta ma wymiar ``(liczba_orbitali wybierz nelec_a) * (liczba_orbitali wybierz nelec_b)``. Dla tego problemu: `(8 wybierz 4)**2 = 4900`. Wymiary podprzestrzeni rosną dzięki ulepszonemu odtwarzaniu konfiguracji, a także dlatego, że algorytm SQD przenosi ważne konfiguracje z jednej iteracji do następnej. W ostatniej iteracji diagonalizujemy w całej przestrzeni Hilberta.

Drugi wykres pokazuje średnie zajęcie każdego orbitalu przestrzennego we wszystkich rozwiązaniach wsadowych. Widzimy, że z dużym prawdopodobieństwem każdy orbital zawiera jeden elektron.

# Automatyczne modyfikacje kodu

doQumentation automatycznie stosuje niewielką liczbę modyfikacji do treści
samouczków i przewodników Qiskit z repozytorium upstream, aby zapewnić
płynne, interaktywne doświadczenie. Ta strona dokumentuje każdą modyfikację,
dzięki czemu możesz dokładnie zrozumieć, co zostało zmienione w stosunku
do oryginalnej dokumentacji IBM Quantum.

## Kopie notatników (Otwórz w Colab / Binder / Code Engine) {#notebook-copies-open-in-colab--binder--code-engine}

Gdy klikniesz **Otwórz w Colab**, **Otwórz w JupyterLab** lub **Otwórz w Code
Engine**, otrzymujesz kopię oryginalnego notatnika z następującymi dodatkami:

### 1. Komórka informacyjna (markdown) {#1-setup-notice-cell-markdown}

Na samym początku wstawiana jest komórka blokowa (blockquote) wyjaśniająca,
że doQumentation dodało automatyczną komórkę konfiguracyjną. Zawiera ona
odsyłacz do tej strony.

### 2. Komórka wymagań wstępnych (kod) {#2-prerequisites-cell-code}

Po komórce informacyjnej wstawiana jest komórka kodu, która:

- **Instaluje wymagane pakiety** (`qiskit`, `qiskit-aer`, `qiskit-ibm-runtime`,
  `pylatexenc` oraz wszelkie pakiety specyficzne dla samouczka wykryte przez
  skanowanie importów). Instalacja jest **pomijana**, jeśli pakiety są już
  obecne (np. w Binderze lub Code Engine, gdzie są preinstalowane).
- **Dostarcza zakomentowany szablon poświadczeń** dla IBM Quantum, dzięki czemu
  użytkownicy chcący uruchomić kod na prawdziwym sprzęcie mogą odkomentować
  szablon i wpisać swój klucz API.

W Google Colab ta komórka uruchamia się automatycznie przy otwarciu notatnika
za pomocą flagi metadanych `cell_execution_strategy: setup`.

### 3. Przepisywanie ścieżek obrazków {#3-image-path-rewrites}

Względne ścieżki obrazków (`/docs/images/...`, `/learning/images/...`) są
przepisywane tak, aby działały poprawnie w samodzielnych środowiskach notatnikowych.

## Strony MDX (renderowanie w przeglądarce) {#mdx-pages-in-browser-rendering}

Samouczki wyświetlane na tej stronie są konwertowane z plików `.ipynb`
lub `.mdx` z repozytorium upstream. Stosowane są następujące transformacje:

- **Linie `pip install`** są dodawane do bloków kodu Python importujących
  pakiety innych firm, umożliwiając jednokllikowe wykonanie przez thebelab.
- **Sekcja IBM Tutorial Survey**: Dodawana jest notatka wyjaśniająca, że
  ankieta należy do IBM Quantum, z odsyłaczem do Issues na GitHubie doQumentation
  dla opinii dotyczących strony.
- **Widget opinii**: Na dole każdego samouczka dołączany jest widget „Czy to było pomocne?",
  śledzony przez przyjazną dla prywatności analitykę Umami.
- **Poprawki składni MDX**: Nawiasy klamrowe, hierarchia nagłówków i problemy
  ze zgodnością JSX są automatycznie korygowane dla renderowania Docusaurus.
- **OpenInLabBanner**: Pod tytułem wstawiany jest interaktywny baner z
  przyciskami do otwierania notatnika w Colab, Binderze lub Code Engine.

## Co NIE jest modyfikowane {#what-is-not-modified}

- **Sama treść samouczka** (wyjaśnienia, logika kodu, wyniki) nigdy nie jest
  zmieniana.
- **Przypisanie oryginalnym autorom** jest zachowywane przez frontmatter i plik
  NOTICE (licencje Apache 2.0 / CC BY-SA 4.0).
- **Żaden kod telemetryczny ani śledzący** nie jest wstrzykiwany do notatników.
  Analityka (Umami) działa wyłącznie na stronie doQumentation, nie w
  eksportowanych notatnikach.

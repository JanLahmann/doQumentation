# Zrozumienie wysokiej dostępności i odtwarzania po awarii dla Qiskit Runtime {#understanding-high-availability-and-disaster-recovery-for-qiskit-runtime}

Wysoka dostępność (HA) to zdolność usługi do pozostania operacyjną i dostępną w obliczu nieoczekiwanych awarii. Odtwarzanie po awarii to proces przywracania instancji usługi do stanu roboczego. Klienci Qiskit Runtime nie są odpowiedzialni za planowanie ani wykonywanie działań związanych z odtwarzaniem po awarii.

Qiskit Runtime jest usługą regionalną, która spełnia zdefiniowane [Cele poziomu usług (SLO)](https://cloud.ibm.com/docs/resiliency?topic=resiliency-slo) w ramach każdego płatnego planu. Więcej informacji znajdziesz w [Umowie o poziomie usług (SLA)](https://www.ibm.com/support/customer/csol/terms/?id=i126-9268&lc=en) dla IBM Cloud&reg;. Więcej informacji o dostępnych regionach IBM Cloud i centrach danych dla Qiskit Runtime znajdziesz w artykule [Dostępność usług i infrastruktury według lokalizacji.](https://cloud.ibm.com/docs/overview?topic=overview-services_region)

## Architektura wysokiej dostępności {#high-availability-architecture}

### Funkcje wysokiej dostępności {#high-availability-features}

Ta usługa jest udostępniana w trzech strefach w regionie wielostrefowym, bez pojedynczego punktu awarii. Żądania API są kierowane przez moduł równoważenia obciążenia do węzłów instancji HA, z których każdy znajduje się w innej strefie.

Jeśli węzeł lub strefa instancji HA ulegnie awarii, usługa nadal obsługuje żądania API za pośrednictwem pozostałych węzłów instancji HA. Podczas awarii strefowej nie dochodzi do zauważalnego pogorszenia jakości usługi.

Jest dostępna w następujących regionach:

- Waszyngton DC (us-east)
- Frankfurt (eu-de)

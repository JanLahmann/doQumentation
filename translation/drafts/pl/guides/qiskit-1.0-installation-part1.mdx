# Zainstaluj nowy pakiet Qiskit 1.0 {#install-the-new-qiskit-10-package}

Qiskit 1.0 używa innej struktury pakietów niż poprzednie wersje Qiskit i prawdopodobnie spowoduje problemy w środowiskach korzystających z pakietów, które nie są jeszcze gotowe na Qiskit 1.0.

<Admonition type="caution">
Nie próbuj uaktualniać istniejącego wirtualnego środowiska Python do Qiskit 1.0 w miejscu.

W przyszłości nie będziemy wprowadzać podobnych przełomowych zmian w pakowaniu. To jednorazowe zdarzenie, przy wydaniu Qiskit 1.0, specjalnie po to, aby zarządzanie pakietami było jak najprostsze w przyszłości.
</Admonition>

Ten przewodnik podzielony jest na następujące sekcje. Musisz zapoznać się tylko z sekcjami, które są dla ciebie istotne.

* Użytkownicy powinni przeczytać sekcję [Dla użytkowników](#for-users).
* Jeśli tworzysz lub utrzymujesz pakiet zależny od Qiskit, przeczytaj sekcję [Dla deweloperów](#for-developers).
* Jeśli napotkasz problemy z instalacją lub importowaniem Qiskit 1.0, zapoznaj się z sekcją [Rozwiązywanie problemów](#troubleshooting).

Jeśli chcesz zrozumieć starą strukturę pakietów i powód jej zmiany, zapoznaj się z [omówieniem przełomowych zmian w pakowaniu](./metapackage-migration).

<span id="for-users"></span>
## Dla użytkowników {#for-users}

Aby zainstalować Qiskit 1.0, musisz zacząć od nowego wirtualnego środowiska.
Uaktualnienie istniejącej instalacji do Qiskit 1.0 w miejscu jest bardzo trudne i podatne na błędy.

<Admonition type="note">
Przykłady w tej sekcji używają modułu `venv`, który jest częścią standardowej biblioteki Pythona.
Jeśli korzystasz z innego narzędzia, takiego jak `virtualenv` lub `conda`, zajrzyj do jego dokumentacji po pomoc.

W poleceniach dla systemu Linux i macOS używana jest składnia bash-podobna.
Do poleceń dla systemu Windows używany jest PowerShell.
</Admonition>

<span id="creating-a-venv"></span>
### Utwórz nowe środowisko {#create-the-new-environment}

1. Utwórz nowe wirtualne środowisko w każdym katalogu projektu, nad którym pracujesz, używając preferowanej wersji Pythona 3.8 lub nowszej.

        <OperatingSystemTabs>
          <TabItem value="mac" label="macOS">
             ```bash
             python3 -m venv .venv
             ```
          </TabItem>

          <TabItem value="linux" label="Linux">
             ```bash
             python3 -m venv .venv
             ```
          </TabItem>

          <TabItem value="win" label="Windows">
            ```powershell
            python -m venv .venv
            ```
          </TabItem>
         </OperatingSystemTabs>

<span id="activate"></span>
2. Aktywuj środowisko.

        <OperatingSystemTabs>
          <TabItem value="mac" label="macOS">
             ```bash
             source .venv/bin/activate
             ```
          </TabItem>

          <TabItem value="linux" label="Linux">
             ```bash
             source .venv/bin/activate
             ```
          </TabItem>

          <TabItem value="win" label="Windows">
            ```powershell
            .venv\Scripts\activate.ps1
            ```
          </TabItem>
         </OperatingSystemTabs>

3. Zainstaluj pakiety według potrzeb.
   Najlepiej zrobić to za pomocą jednego polecenia `pip install` zawierającego wszystkie zależności.

   ```bash
   pip install 'qiskit>=1'
   ```

   Opcjonalnie możesz uwzględnić dodatkowe pakiety, podając je jako argumenty. Na przykład:

   ```bash
   pip install 'qiskit>=1' jupyterlab pandas matplotlib
   ```

   Qiskit 1.0 zawiera przełomowe zmiany, dlatego kilka pakietów jest oznaczonych jako jeszcze niekompatybilnych z tą wersją. W związku z tym możesz napotkać błędy z `pip`, dopóki nie zostaną wydane nowe wersje tych pakietów. Stare wersje pakietów mogą też zależeć od przestarzałego pakietu `qiskit-terra`. Takie pakiety mogą nie zwracać błędów podczas wykonywania tego polecenia, ale mogą zgłaszać błąd przy uruchamianiu `import qiskit`. Nie należy instalować żadnych pakietów zależących bezpośrednio od `qiskit-terra`.

   <Admonition type="tip">
   Jednym ze sposobów, aby wymagać od `pip` zakazu instalowania `qiskit-terra` w poszczególnych poleceniach `install`, jest użycie [pliku ograniczeń](https://pip.pypa.io/en/stable/user_guide/#constraints-files), który wymusza niemożliwą wersję dla `qiskit-terra`.
   Na przykład plik ograniczeń zawierający wiersz `qiskit-terra>=1.0` sprawi, że jeśli jakaś zależność spróbuje zainstalować `qiskit-terra`, żadna opublikowana wersja nie będzie spełniać wymagań.

   Udostępniliśmy taki plik w GitHub Gist pod adresem [https://qisk.it/1-0-constraints](https://qisk.it/1-0-constraints), którego możesz używać w następujący sposób:

   ```bash
   pip install -c https://qisk.it/1-0-constraints qiskit [other packages]
   ```

   Jeśli jakiś pakiet wymaga `qiskit-terra`, zobaczysz [błąd rozwiązywania zależności](#pip-resolution-impossible).
   </Admonition>

   <Admonition type="caution">
   Nie instaluj pakietów niekompatybilnych z Qiskit 1.0 w tym wirtualnym środowisku. Jeśli musisz używać takich pakietów, zainstaluj je w osobnym wirtualnym środowisku z Qiskit 0.45 lub 0.46.
   </Admonition>

   Jeśli masz istniejące środowisko, możesz użyć [`pipdeptree`](https://github.com/tox-dev/pipdeptree/blob/main/README.md#pipdeptree), aby sprawdzić wymagania zainstalowanych pakietów i zobaczyć, czy któryś z nich wymaga `qiskit<1`. Dla tych, które wymagają `qiskit<1`, sprawdź dostępność aktualizacji zapewniających kompatybilność z Qiskit 1.0.

   Jeśli napotkasz problemy, zajrzyj do sekcji [rozwiązywania problemów](#troubleshooting), lub zapytaj na [Qiskit Slack](https://qisk.it/join-slack). Jeśli uważasz, że to błąd, możesz [zgłosić problem w Qiskit](https://github.com/Qiskit/qiskit/issues/new/choose).

4. Jeśli nie planujesz od razu korzystać ze środowiska, użyj polecenia `deactivate`, aby je opuścić.

<span id="activating-a-venv"></span>
### Korzystaj z nowego środowiska {#use-the-new-environment}

Za każdym razem, gdy rozpoczynasz nową sesję wiersza poleceń, musisz przejść do katalogu projektu i „aktywować" środowisko, uruchamiając polecenie `activate`:

        <OperatingSystemTabs>
          <TabItem value="mac" label="macOS">
             ```bash
             source .venv/bin/activate
             ```
          </TabItem>

          <TabItem value="linux" label="Linux">
             ```bash
             source .venv/bin/activate
             ```
          </TabItem>

          <TabItem value="win" label="Windows">
            ```powershell
            .venv\Scripts\activate.ps1
            ```
          </TabItem>
         </OperatingSystemTabs>

<span id="for-developers"></span>
## Dla deweloperów {#for-developers}

Jeśli utrzymujesz pakiet zależny od Qiskit, skorzystaj z tych informacji, aby dowiedzieć się, jak poprawnie wyrazić swoją kompatybilność i testować względem Qiskit 1.0.

### Zalecenia dotyczące wymagań {#recommendations-for-requirements}

Zalecamy, aby twój pakiet wymagał `qiskit>=0.45,<1` (lub innego odpowiedniego dolnego ograniczenia), jeśli nie masz pewności, czy pakiet jest kompatybilny z Qiskit 1.0.
To [ta sama rekomendacja, która jest stosowana w przypadku kompatybilności z NumPy 2.0.](https://github.com/numpy/numpy/issues/24300)

Kandydat do wydania Qiskit 1.0, wersja 1.0.0rc1, zostanie wydany 1 lutego 2024 roku.
Powinieneś przetestować swój pakiet względem tej wersji i jak najszybciej wydać nową (kompatybilną) wersję swojego pakietu z odpiętym górnym ograniczeniem wersji.

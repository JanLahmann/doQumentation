## Zacznij pracę z Estimator {#get-started-with-estimator}

{/*Verified the v2 examples 2/29/24 - updated 10/29/24*/}

### 1. Zainicjalizuj konto {#1-initialize-the-account}

Ponieważ Qiskit Runtime Estimator jest usługą zarządzaną, najpierw musisz zainicjalizować swoje konto. Następnie możesz wybrać QPU, którego chcesz użyć do obliczenia wartości oczekiwanej.

Postępuj zgodnie z krokami opisanymi w [temacie Instalacja i konfiguracja](install-qiskit), jeśli nie masz jeszcze skonfigurowanego konta.

```python
from qiskit_ibm_runtime import QiskitRuntimeService

service = QiskitRuntimeService()
backend = service.least_busy(
    operational=True, simulator=False, min_num_qubits=127
)

print(backend.name)
```

```text
ibm_torino
```

### 2. Utwórz Circuit i obserwowalną {#2-create-a-circuit-and-an-observable}

Do prymitywu Estimator potrzebny jest co najmniej jeden Circuit i jedna obserwowalna jako dane wejściowe.

```python
from qiskit.circuit.library import qaoa_ansatz
from qiskit.quantum_info import SparsePauliOp

entanglement = [tuple(edge) for edge in backend.coupling_map.get_edges()]
observable = SparsePauliOp.from_sparse_list(
    [("ZZ", [i, j], 0.5) for i, j in entanglement],
    num_qubits=backend.num_qubits,
)
circuit = qaoa_ansatz(observable, reps=2)
# the circuit is parametrized, so we will define the parameter values for execution
param_values = [0.1, 0.2, 0.3, 0.4]

print(f">>> Observable: {observable.paulis}")
```

Circuit i obserwowalna muszą zostać przekształcone tak, aby używały wyłącznie instrukcji obsługiwanych przez QPU (nazywanych *zestawem architektury instrukcji (ISA)* Circuitów). Użyjemy do tego Transpilera.

```python
from qiskit.transpiler import generate_preset_pass_manager

pm = generate_preset_pass_manager(optimization_level=1, backend=backend)
isa_circuit = pm.run(circuit)
isa_observable = observable.apply_layout(isa_circuit.layout)
print(f">>> Circuit ops (ISA): {isa_circuit.count_ops()}")
```

```text
>>> Circuit ops (ISA): OrderedDict([('rz', 3826), ('sx', 1601), ('cz', 968)])
```

### 3. Zainicjalizuj Qiskit Runtime Estimator {#3-initialize-qiskit-runtime-estimator}

Podczas inicjalizacji Estimatora użyj parametru `mode`, aby określić tryb, w którym ma działać. Możliwe wartości to obiekty `batch`, `session` lub `backend` — odpowiednio dla trybu wsadowego (batch), sesyjnego (session) i zadaniowego (job). Więcej informacji znajdziesz w artykule [Wprowadzenie do trybów wykonania Qiskit Runtime.](execution-modes)

```python
from qiskit_ibm_runtime import EstimatorV2 as Estimator

estimator = Estimator(mode=backend)
```

### 4. Wywołaj Estimator i pobierz wyniki {#4-invoke-the-estimator-and-get-results}

Następnie wywołaj metodę `run()`, aby obliczyć wartości oczekiwane dla podanych Circuitów i obserwowalnych. Circuit, obserwowalna oraz opcjonalne zestawy wartości parametrów są przekazywane jako krotki PUB (*primitive unified bloc*).

```python
job = estimator.run([(isa_circuit, isa_observable, param_values)])
print(f">>> Job ID: {job.job_id()}")
print(f">>> Job Status: {job.status()}")
```

```text
>>> Job ID: d5k96c4jt3vs73ds5smg
```


```python
result = job.result()
print(f">>> {result}")
print(f"  > Expectation value: {result[0].data.evs}")
print(f"  > Metadata: {result[0].metadata}")
```

## Zacznij pracę z Sampler {#get-started-with-sampler}

### 1. Zainicjalizuj konto {#1-initialize-the-account-1}

Ponieważ Qiskit Runtime Sampler jest usługą zarządzaną, najpierw musisz zainicjalizować swoje konto. Następnie możesz wybrać QPU, którego chcesz użyć do obliczenia wartości oczekiwanej.

Postępuj zgodnie z krokami opisanymi w [temacie Instalacja i konfiguracja](install-qiskit), jeśli nie masz jeszcze skonfigurowanego konta.

```python
from qiskit_ibm_runtime import QiskitRuntimeService

service = QiskitRuntimeService()
backend = service.least_busy(
    operational=True, simulator=False, min_num_qubits=127
)
```

### 2. Utwórz Circuit {#2-create-a-circuit}

Do prymitywu Sampler potrzebny jest co najmniej jeden Circuit jako dane wejściowe.

```python
import numpy as np
from qiskit.circuit.library import efficient_su2

circuit = efficient_su2(127, entanglement="linear")
circuit.measure_all()
# The circuit is parametrized, so we will define the parameter values for execution
param_values = np.random.rand(circuit.num_parameters)
```

Użyj Transpilera, aby uzyskać Circuit ISA.

```python
from qiskit.transpiler import generate_preset_pass_manager

pm = generate_preset_pass_manager(optimization_level=1, backend=backend)
isa_circuit = pm.run(circuit)
print(f">>> Circuit ops (ISA): {isa_circuit.count_ops()}")
```

```text
>>> Circuit ops (ISA): OrderedDict([('sx', 3089), ('rz', 3036), ('cz', 1092), ('measure', 127), ('barrier', 1)])
```

### 3. Zainicjalizuj Qiskit Runtime Sampler {#3-initialize-the-qiskit-runtime-sampler}

Podczas inicjalizacji Samplera użyj parametru `mode`, aby określić tryb, w którym ma działać. Możliwe wartości to obiekty `batch`, `session` lub `backend` — odpowiednio dla trybu wsadowego (batch), sesyjnego (session) i zadaniowego (job). Więcej informacji znajdziesz w artykule [Wprowadzenie do trybów wykonania Qiskit Runtime.](execution-modes) Pamiętaj, że użytkownicy planu Open Plan nie mogą przesyłać zadań sesyjnych.

```python
from qiskit_ibm_runtime import SamplerV2 as Sampler

sampler = Sampler(mode=backend)
```

### 4. Wywołaj Sampler i pobierz wyniki {#4-invoke-the-sampler-and-get-results}

Następnie wywołaj metodę `run()`, aby wygenerować dane wyjściowe. Circuit oraz opcjonalne zestawy wartości parametrów są przekazywane jako krotki PUB (*primitive unified bloc*).

```python
job = sampler.run([(isa_circuit, param_values)])
print(f">>> Job ID: {job.job_id()}")
print(f">>> Job Status: {job.status()}")
```

```text
>>> Job ID: d5k96rsjt3vs73ds5tig
>>> Job Status: QUEUED
```

```python
result = job.result()

# Get results for the first (and only) PUB
pub_result = result[0]
print(

)
```

## Zacznij pracę z prymitywami backend {#get-started-with-the-backend-primitives}

W przeciwieństwie do prymitywów specyficznych dla danego dostawcy, prymitywy backend to implementacje ogólne, które można używać z dowolnym
obiektem `backend`, o ile implementuje on interfejs [`Backend`](https://docs.quantum.ibm.com/api/qiskit/qiskit.providers.Backend).

- Prymityw Sampler można uruchomić u dowolnego dostawcy, używając [`qiskit.primitives.BackendSamplerV2`](https://docs.quantum.ibm.com/api/qiskit/qiskit.primitives.BackendSamplerV2).
- Prymityw Estimator można uruchomić u dowolnego dostawcy, używając [`qiskit.primitives.BackendEstimatorV2`](../api/qiskit/qiskit.primitives.BackendEstimatorV2).

Niektórzy dostawcy implementują prymitywy natywnie. Szczegóły znajdziesz na stronie [Qiskit Ecosystem](https://qiskit.github.io/ecosystem#provider).
### Przykład: BackendEstimator {#example-backendestimator}

```python
from qiskit.primitives import BackendEstimatorV2
from <some_qiskit_provider> import QiskitProvider

provider = QiskitProvider()
backend = provider.get_backend('backend_name')
estimator = BackendEstimatorV2(backend)
```

### Przykład: BackendSampler {#example-backendsampler}

```python
from qiskit.primitives import BackendSamplerV2
from <some_qiskit_provider> import QiskitProvider

provider = QiskitProvider()
backend = provider.get_backend('backend_name')
sampler = BackendSamplerV2(backend)
```
### Podobieństwa i różnice między prymitywami backend a Runtime {#similarities-and-differences-between-backend-and-runtime-primitives}

- Dane wejściowe i wyjściowe dla [`qiskit.primitives.BackendSamplerV2`](https://docs.quantum.ibm.com/api/qiskit/qiskit.primitives.BackendSamplerV2) i [`qiskit.primitives.BackendEstimatorV2`](../api/qiskit/qiskit.primitives.BackendEstimatorV2)
są zgodne z tym samym formatem PUB co prymitywy w Qiskit Runtime. Szczegóły znajdziesz w artykule [Dane wejściowe i wyjściowe prymitywów](primitive-input-output).
Mogą jednak występować różnice w polach zwracanych metadanych.

- Klasa [`qiskit.primitives.BackendEstimatorV2`](https://docs.quantum.ibm.com/api/qiskit/qiskit.primitives.BackendEstimatorV2) nie oferuje gotowych implementacji mitygacji błędów pomiarowych ani bramkowych, ponieważ
prymitywy backend są zaprojektowane do lokalnego uruchamiania na maszynie użytkownika.

- Klasa [`qiskit.primitives.BackendSamplerV2`](https://docs.quantum.ibm.com/api/qiskit/qiskit.primitives.BackendSamplerV2) wymaga backendu obsługującego opcję `memory`.

- Interfejsy prymitywów backend udostępniają własne opcje `Options` dla [`SamplerV2`](https://docs.quantum.ibm.com/api/qiskit/qiskit.primitives.BackendSamplerV2) i [`EstimatorV2`](https://docs.quantum.ibm.com/api/qiskit/qiskit.primitives.BackendEstimatorV2), które różnią się od implementacji Runtime.

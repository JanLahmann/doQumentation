Ten przewodnik przedstawia prosty przykład roboczy, który pozwoli ci zacząć korzystać z AQC-Tensor. W tym przykładzie weźmiemy obwód Trottera, który symuluje ewolucję poprzecznego modelu Isinga z polem magnetycznym, i użyjemy metody AQC-Tensor, aby zredukować wynikającą z niego głębokość obwodu. Ponadto ten przykład wymaga pakietu `qiskit-addon-utils` do generowania problemu, `qiskit-aer` do symulacji sieci tensorowej oraz `scipy` do optymalizacji parametrów.

Na początek przypomnijmy, że hamiltonian poprzecznego modelu Isinga z polem magnetycznym ma postać

$$ \mathcal{H}_{Ising} = \sum_{i=1}^N J_{i,(i+1)}Z_iZ_{i+1} + h_i X_i $$

gdzie zakładamy periodyczne warunki brzegowe, co oznacza, że dla $i=10$ otrzymujemy $i+1=11\rightarrow 1$, $J$ jest stałą sprzężenia między dwoma węzłami, a $h$ jest natężeniem zewnętrznego pola magnetycznego.

Poniższy fragment kodu wygeneruje hamiltonian 10-węzłowego łańcucha Isinga z periodycznymi warunkami brzegowymi.

```python

```

```python
from qiskit.transpiler import CouplingMap
from qiskit_addon_utils.problem_generators import generate_xyz_hamiltonian
from qiskit.synthesis import SuzukiTrotter
from qiskit_addon_utils.problem_generators import (
    generate_time_evolution_circuit,
)
from qiskit_addon_aqc_tensor import generate_ansatz_from_circuit
from qiskit_addon_aqc_tensor.simulation import tensornetwork_from_circuit
from qiskit_addon_aqc_tensor.simulation import compute_overlap
from qiskit_addon_aqc_tensor.objective import MaximizeStateFidelity
from qiskit_aer import AerSimulator
from scipy.optimize import OptimizeResult, minimize

# Generate some coupling map to use for this example
coupling_map = CouplingMap.from_heavy_hex(3, bidirectional=False)

# Choose a 10-qubit circle on this coupling map
reduced_coupling_map = coupling_map.reduce(
    [0, 13, 1, 14, 10, 16, 4, 15, 3, 9]
)

# Get a qubit operator describing the Ising field model
hamiltonian = generate_xyz_hamiltonian(
    reduced_coupling_map,
    coupling_constants=(0.0, 0.0, 1.0),
    ext_magnetic_field=(0.4, 0.0, 0.0),
)
```

## Podział ewolucji czasowej na dwie części: klasyczną i kwantową {#podzial-ewolucji-czasowej-na-dwie-czesci-klasyczna-i-kwantowa}

Ogólnym celem tego przykładu jest symulacja ewolucji czasowej hamiltoniana modelu. Realizujemy to za pomocą ewolucji Trottera, która zostanie podzielona na dwie części.

1. Pierwsza część, która jest symulowalna przy użyciu stanów produktu macierzowego (MPS). To właśnie ona zostanie „skompilowana" przy użyciu AQC-Tensor.
2. Kolejna część, która zostanie wykonana na sprzęcie kwantowym.

Wybierzemy ewolucję systemu do czasu $t_f=5$ i użyjemy AQC-Tensor do kompresji ewolucji czasowej do czasu $t=4$, a następnie będziemy ewoluować za pomocą zwykłych kroków Trottera aż do $t_f$.

Następnie wygenerujemy dwa obwody: jeden, który zostanie skompresowany przy użyciu AQC-Tensor, i jeden, który zostanie wykonany na QPU. Dla pierwszego obwodu, ponieważ będzie on symulowany klasycznie przy użyciu stanów produktu macierzowego, użyjemy dużej liczby warstw, aby zminimalizować błąd Trottera. Natomiast drugi obwód symulujący ewolucję czasową od $t_i=4$ do $t_f=5$ będzie używał znacznie mniejszej liczby warstw, aby zminimalizować głębokość.

```python
# Generate circuit to be compressed
aqc_evolution_time = 4.0
aqc_target_num_trotter_steps = 45

aqc_target_circuit = generate_time_evolution_circuit(
    hamiltonian,
    synthesis=SuzukiTrotter(reps=aqc_target_num_trotter_steps),
    time=aqc_evolution_time,
)

# Generate circuit to execute on hardware
subsequent_evolution_time = 1.0
subsequent_num_trotter_steps = 5

subsequent_circuit = generate_time_evolution_circuit(
    hamiltonian,
    synthesis=SuzukiTrotter(reps=subsequent_num_trotter_steps),
    time=subsequent_evolution_time,
)
```

Dla celów porównawczych wygenerujemy również trzeci obwód — taki, który ewoluuje do $t=4$, ale ma tę samą liczbę warstw co drugi obwód ewoluujący od $t_i=4$ do $t_f=5$. To obwód, który *wykonalibyśmy*, gdybyśmy nie skorzystali z techniki AQC-Tensor. Na razie nazwiemy go obwodem *porównawczym*.

```python
aqc_comparison_num_trotter_steps = int(
    subsequent_num_trotter_steps
    / subsequent_evolution_time
    * aqc_evolution_time
)
aqc_comparison_num_trotter_steps

comparison_circuit = generate_time_evolution_circuit(
    hamiltonian,
    synthesis=SuzukiTrotter(reps=aqc_comparison_num_trotter_steps),
    time=aqc_evolution_time,
)
```

## Generowanie ansatzu i budowa symulacji MPS {#generowanie-ansatzu-i-budowa-symulacji-mps}

Następnie wygenerujemy ansatz, który będziemy optymalizować. Będzie on ewoluował do tego samego czasu ewolucji co nasz pierwszy obwód (od $t_i=0$ do $t_f=4$), ale przy mniejszej liczbie kroków Trottera.

Gdy obwód zostanie wygenerowany, przekażemy go do funkcji `generate_ansatz_from_circuit()` z AQC-Tensor, która analizuje łączność dwuqubitową i zwraca dwie rzeczy. Pierwsza to wygenerowany obwód ansatzu o tej samej łączności dwuqubitowej, a druga to zbiór parametrów, które po podstawieniu do ansatzu odtwarzają obwód wejściowy.

```python
aqc_ansatz_num_trotter_steps = 5

aqc_good_circuit = generate_time_evolution_circuit(
    hamiltonian,
    synthesis=SuzukiTrotter(reps=aqc_ansatz_num_trotter_steps),
    time=aqc_evolution_time,
)

aqc_ansatz, aqc_initial_parameters = generate_ansatz_from_circuit(
    aqc_good_circuit, qubits_initially_zero=True
)
aqc_ansatz.draw("mpl", fold=-1)
```

Następnie zbudujemy reprezentację MPS stanu, który ma zostać przybliżony przez AQC. Obliczymy również wierność $|\langle\psi_1 | \psi_2 \rangle |^2$ między stanem przygotowanym przez obwód porównawczy a obwodem generującym stan docelowy (który używał większej liczby kroków Trottera).

```python
# Generate MPS simulator settings and obtain the MPS representation of the target state
simulator_settings = AerSimulator(
    method="matrix_product_state",
    matrix_product_state_max_bond_dimension=100,
)
aqc_target_mps = tensornetwork_from_circuit(
    aqc_target_circuit, simulator_settings
)

# Compute the fidelity between the MPS representation of the target state and the state produced by the comparison circuit
comparison_mps = tensornetwork_from_circuit(
    comparison_circuit, simulator_settings
)
comparison_fidelity = (
    abs(compute_overlap(comparison_mps, aqc_target_mps)) ** 2
)
print(f"Comparison fidelity: {comparison_fidelity}")
```

## Optymalizacja parametrów ansatzu przy użyciu MPS {#optymalizacja-parametrow-ansatzu-przy-uzyciu-mps}

Na koniec zoptymalizujemy obwód ansatzu tak, aby wytwarzał stan docelowy z wyższą wiernością niż nasz `comparison_fidelity`. Funkcją kosztu do minimalizacji będzie `MaximizeStateFidelity`, a do optymalizacji zostanie użyty optymalizator L-BFGS z biblioteki scipy.

```python
objective = MaximizeStateFidelity(
    aqc_target_mps, aqc_ansatz, simulator_settings
)

stopping_point = 1 - comparison_fidelity

def callback(intermediate_result: OptimizeResult):
    print(f"Intermediate result: Fidelity {1 - intermediate_result.fun:.8}")
    if intermediate_result.fun < stopping_point:
        # Good enough for now
        raise StopIteration

result = minimize(
    objective,
    aqc_initial_parameters,
    method="L-BFGS-B",
    jac=True,
    options={"maxiter": 100},
    callback=callback,
)
if (
    result.status
    not in (
        0,
        1,
        99,
    )
):  # 0 => success; 1 => max iterations reached; 99 => early termination via StopIteration
    raise RuntimeError(
        f"Optimization failed: {result.message} (status={result.status})"
    )

print(f"Done after {result.nit} iterations.")
aqc_final_parameters = result.x
```

Na tym etapie dysponujemy zestawem parametrów, które generują stan docelowy z wyższą wiernością niż ta, którą uzyskałby obwód porównawczy bez użycia AQC. Dzięki tym optymalnym parametrom skompresowany obwód ma teraz mniejszy błąd Trottera *i* mniejszą głębokość niż oryginalny obwód.

W ostatnim kroku poniższy fragment kodu buduje pełny obwód ewolucji czasowej, który można przekazać do potoku transpilatora i wykonać na sprzęcie kwantowym.

```python
final_circuit = aqc_ansatz.assign_parameters(aqc_final_parameters)
final_circuit.compose(subsequent_circuit, inplace=True)
final_circuit.draw("mpl", fold=-1)
```

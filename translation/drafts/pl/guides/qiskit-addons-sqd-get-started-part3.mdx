## Uzyskiwanie próbek z ansatzu

Gdy informacje o cząsteczce zostały już wczytane, należy określić Circuit ansatzu i zoptymalizować go do wykonania. Wiąże się to z uwzględnieniem symetrii problemu, wyborem ansatzu i jego optymalizacją pod kątem uruchamiania na sprzęcie kwantowym.

Jednak te wybory są specyficzne dla danego problemu i wykraczają poza zakres tego przewodnika. Zamiast tego, aby zademonstrować przepływ pracy SQD, wygenerujemy losowy zestaw zliczeń dla pętli odzyskiwania konfiguracji do post-processowania. Symuluje to dane pomiarowe 32-Qubitowego Circuit próbkowanego z 10 000 ujęciami. Po wygenerowaniu danych zliczeń metoda `recover_configurations()` wymaga, abyś przekonwertował dane zliczeń na macierz bitciągów zmierzonych w każdym ujęciu oraz tablicę prawdopodobieństw dla każdego zmierzonego stanu.

## Uruchamianie pętli odzyskiwania konfiguracji

Próbki pomiarowe można następnie udoskonalać, powtarzając odzyskiwanie konfiguracji i diagonalizując zestawy podpróbek w celu przybliżenia stanu podstawowego aż do osiągnięcia zbieżności.

Najpierw określ następujące opcje:

- `ITERATIONS`: Liczba iteracji samozgodnego odzyskiwania konfiguracji
- `NUM_BATCHES`: Liczba zestawów podpróbek do diagonalizacji
- `SAMPLES_PER_BATCH`: Liczba próbek do uwzględnienia w każdej partii
- `MAX_DAVIDSON_CYCLES`: Liczba iteracji solvera stanu własnego

```python
# SQD options
ITERATIONS = 5

# Eigenstate solver options
NUM_BATCHES = 1
SAMPLES_PER_BATCH = 500
MAX_DAVIDSON_CYCLES = 200
```

Następnie, w celu wykreślenia zbieżności, zdefiniuj tablice do przechowywania przybliżenia energii stanu podstawowego, wartości oczekiwanej $\langle S \rangle ^2$ oraz obsadzenia orbitalnego cząsteczki.

```python
# Self-consistent configuration recovery loop
energy_hist = np.zeros((ITERATIONS, NUM_BATCHES))  # energy history
spin_sq_hist = np.zeros((ITERATIONS, NUM_BATCHES))  # spin history
occupancy_hist = []
avg_occupancy = None
```

Teraz uruchom pętlę odzyskiwania konfiguracji. Każda pętla składa się z trzech kroków:

1. Użyj funkcji `recover_configurations()`, aby uzyskać udoskonaloną macierz bitciągów i tablicę prawdopodobieństw na podstawie średniego obsadzenia orbitalnego.
1. Użyj funkcji `postselect_and_subsample()`, aby zebrać partie podpróbek do diagonalizacji.
1. Następnie użyj partii podpróbek jako argumentów funkcji `solve_fermion()`, aby uzyskać przybliżenie stanu podstawowego.

Ważne jest, aby zwrócić uwagę na to, jak postępować w pierwszej iteracji pętli odzyskiwania konfiguracji. Ponieważ średnie obsadzenie orbitalne nie jest jeszcze dostępne, wywoływana jest tylko funkcja `postselect_and_subsample()`. Usuwa ona wszelkie próbki niefizyczne (próbki z niepoprawną wagą Hamminga) przed uruchomieniem solvera stanu własnego, `solve_fermion()`. Następnie obliczane jest średnie obsadzenie orbitalne we wszystkich partiach i używane jako wejście do metody `recover_configurations()`, która probabilistycznie odwraca poszczególne bity na podstawie tej średniej. Więcej informacji znajdziesz w sekcji **II-A** informacji uzupełniających w [artykule SQD](https://arxiv.org/abs/2405.05068).

```python
from qiskit_addon_sqd.configuration_recovery import recover_configurations
from qiskit_addon_sqd.subsampling import postselect_and_subsample
from qiskit_addon_sqd.fermion import (
    bitstring_matrix_to_ci_strs,
    solve_fermion,
)

for i in range(ITERATIONS):
    print(f"\nStarting configuration recovery iteration {i}")
    # On the first iteration, we have no orbital occupancy information from the
    # solver, so we just post-select from the full bitstring set based on Hamming weight.
    if avg_occupancy is None:
        bitstring_matrix_tmp = bitstring_matrix_full
        probs_array_tmp = probs_array_full

    # If there is average orbital occupancy information, use it to refine the full set of noisy configurations
    else:
        bitstring_matrix_tmp, probs_array_tmp = recover_configurations(
            bitstring_matrix_full,
            probs_array_full,
            avg_occupancy,
            num_alpha,
            num_beta,
            rand_seed=rng,
        )

    # Throw out configurations with an incorrect particle number in either the spin-up or spin-down systems
    batches = postselect_and_subsample(
        bitstring_matrix_tmp,
        probs_array_tmp,
        hamming_right=num_alpha,
        hamming_left=num_beta,
        samples_per_batch=SAMPLES_PER_BATCH,
        num_batches=NUM_BATCHES,
        rand_seed=rng,
    )

    # Run eigenstate solvers in a loop. This loop should be parallelized for larger problems.
    e_tmp = np.zeros(NUM_BATCHES)
    s_tmp = np.zeros(NUM_BATCHES)
    occs_tmp = []
    coeffs = []
    for j in range(NUM_BATCHES):
        strs_a, strs_b = bitstring_matrix_to_ci_strs(batches[j])
        print(f"Batch {j} subspace dimension: {len(strs_a) * len(strs_b)}")
        energy_sci, coeffs_sci, avg_occs, spin = solve_fermion(
            batches[j],
            core_hamiltonian,
            electron_repulsion_integrals,
            open_shell=open_shell,
            spin_sq=spin_sq,
            max_cycle=MAX_DAVIDSON_CYCLES,
        )
        energy_sci += nuclear_repulsion_energy
        e_tmp[j] = energy_sci
        s_tmp[j] = spin
        occs_tmp.append(avg_occs)
        coeffs.append(coeffs_sci)

    # Combine batch results
    avg_occupancy = tuple(np.mean(occs_tmp, axis=0))

    # Track optimization history
    energy_hist[i, :] = e_tmp
    spin_sq_hist[i, :] = s_tmp
    occupancy_hist.append(avg_occupancy)
    print()
```

### Wizualizacja wyników

Na koniec wyniki można zwizualizować, analizując przybliżoną energię i średnie obsadzenie orbitalne w każdej iteracji pętli odzyskiwania konfiguracji. Pierwszy wykres pokazuje, że po kilku iteracjach energia stanu podstawowego jest szacowana z dokładnością do około `27 mH` (dokładność chemiczna jest zazwyczaj przyjmowana jako `1 kcal/mol` $\approx$ `1,6 mH`). Warto pamiętać, że ta demonstracja wykorzystywała wyłącznie szum, a zdolność do przybliżenia energii w tym stopniu wynika z wcześniejszej wiedzy o cząsteczce i jej strukturze elektronowej.

Drugi wykres pokazuje średnie obsadzenie każdego orbitu przestrzennego po ostatniej iteracji. Zauważ, że zarówno elektrony ze spinem w górę, jak i elektrony ze spinem w dół obsadzają z wysokim prawdopodobieństwem pierwsze pięć orbitalów w rozwiązaniach.

Ta sekcja opisuje, jak używać `qiskit-ibm-catalog` do wyświetlania listy programów dostępnych w Qiskit Serverless, przekazywania do nich danych wejściowych, zdalnego uruchamiania, sprawdzania statusu oraz pobierania wyników i logów.

Upewnij się, że uwierzytelniłeś się w Qiskit Serverless przy użyciu swojego [klucza API](https://quantum.cloud.ibm.com/) (instrukcje znajdziesz w sekcji [Wdrażanie na IBM Quantum Platform](./serverless-first-program#deploy-to-ibm-quantum-platform)).
## Wyświetlanie listy dostępnych programów {#list-programs-available}

Możesz użyć `QiskitServerless.list()`, aby pobrać listę dostępnych programów do uruchomienia w Qiskit Serverless. Obejmuje to wcześniej przesłany program [`transpile_remote_serverless`](./serverless-first-program).

```python
# Added by doQumentation — required packages for this notebook
!pip install -q qiskit qiskit-ibm-catalog qiskit-ibm-runtime
```

```python
from qiskit_ibm_catalog import QiskitServerless

serverless = QiskitServerless()
next(
    program
    for program in serverless.list()
    if program.title == "transpile_remote_serverless"
)
```

```text
QiskitFunction(transpile_remote_serverless)
```

## Uruchamianie przesłanego programu i przekazywanie danych wejściowych {#run-an-uploaded-program-and-pass-inputs}

Najpierw skonfiguruj dane wejściowe. Twój program ma trzy wejścia: `circuits`, `backend` i `optimization_level`. Możesz użyć `random_circuit`, aby utworzyć 30 losowych obwodów:

```python
from qiskit.circuit.random import random_circuit

qc_random = [(random_circuit(4, 4, measure=True, seed=i)) for i in range(10)]
qc_random[0].draw(output="mpl", idle_wires=False)
```

![Wyjście poprzedniej komórki kodu](/docs/images/guides/serverless-run-first-workload/extracted-outputs/46108a95-11f8-4ecf-83bb-131c3e7ea4c2-0.svg)

Następnie użyj `QiskitRuntimeService` i `least_busy`, aby wybrać `backend`:

```python
from qiskit_ibm_runtime import QiskitRuntimeService

service = QiskitRuntimeService()
backend = service.least_busy(operational=True, simulator=False)
print(backend.name)
```

Ustaw poziom optymalizacji:

```python
optimization_level = 3
```

Wybierz swój program za pomocą `serverless.load('PROGRAM_NAME')`:

```python
transpile_remote_serverless = serverless.load("transpile_remote_serverless")
```

Następnie przekaż dane wejściowe i uruchom program w sposób pythonowy:

```python
job = transpile_remote_serverless.run(
    circuits=qc_random,
    backend=backend.name,
    optimization_level=optimization_level,
)
```

```python
job.job_id
```

```text
'118256c5-bbb0-4ea8-9e9f-51aac2220aef'
```

## Sprawdzanie statusu zadania {#check-job-status}

Mając `job_id` Qiskit Serverless, możesz sprawdzać status uruchomionych zadań. Obejmuje to następujące statusy:

- **`QUEUED`**: Zdalny program jest w kolejce Qiskit Serverless. Priorytet kolejki jest obecnie oparty na tym, ile już korzystałeś z Qiskit Serverless.
- **`INITIALIZING`**: Zdalny program jest uruchamiany; obejmuje to konfigurowanie zdalnego środowiska i instalowanie zależności.
- **`RUNNING`**: Program jest uruchomiony. Na tym etapie, jeśli w programie masz wyjścia `print()`, możesz pobierać logi za pomocą `job.logs()`.
- **`DONE`**: Program jest ukończony i możesz pobrać dane zapisane w `save_result()` za pomocą `job.results()`.

Możesz również ustawiać bardziej szczegółowe statusy zadań w sekcji [Zarządzanie zasobami obliczeniowymi i danymi w Qiskit Serverless](/guides/serverless-manage-resources).

```python
job.status()
```

```text
'QUEUED'
```

```python
# This cell is hidden from users, it checks the job status
assert _ in ["QUEUED", "INITIALIZING", "RUNNING", "DONE"]  # noqa: F821
```

<Admonition type="tip">
Obecnie tabela zadań IBM Quantum odzwierciedla tylko zadania Qiskit Runtime. Użyj `job.status()`, aby zobaczyć bieżący status zadania Qiskit Serverless.
</Admonition>

Pomyślnie uruchomiłeś swój pierwszy program Qiskit Serverless!
## Pobieranie logów i wyników {#retrieve-logs-and-results}

Jak wspomniano wcześniej, gdy program jest w stanie `RUNNING`, możesz użyć `job.logs()`, aby pobrać logi utworzone z wyjść `print()`:

```python
logs = job.logs()
print(logs)
```

```text
No logs yet.
```

W dowolnym momencie możesz również anulować zadanie:

```python
job.stop()
```

```text
'Job has been stopped.'
```

Gdy program osiągnie stan `DONE`, możesz użyć `job.results()`, aby pobrać wynik zapisany w `save_result()`:

```python
# We can't get results from a cancelled job, so we'll fetch a completed one instead
completed_job = next(
    job for job in serverless.jobs() if job.status() == "DONE"
)
completed_job.result()
```

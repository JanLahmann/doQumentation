# Zarządzaj kosztami {#manage-cost}

Ten przewodnik opisuje, jak ustawić limit kosztów dla instancji w planie Pay-As-You-Go. Możesz też zarządzać kosztami, [minimalizując czas kwantowy obciążenia](/guides/minimize-time) oraz ustawiając [maksymalny czas wykonania](/guides/max-execution-time) dla swoich obciążeń.

_Limit kosztów_ odnosi się do łącznego kosztu wszystkich obciążeń uruchomionych w ramach tej instancji od momentu jej utworzenia i zawsze będzie większy lub równy łącznemu kosztowi. Po osiągnięciu przez instancję określonej łącznej liczby sekund nie można uruchamiać kolejnych obciążeń i nie są naliczane żadne dodatkowe koszty.

    <Admonition type="note">
      Limit kosztów jest zawsze podawany w dolarach amerykańskich (USD), a następnie przeliczany na sekundy środowiska uruchomieniowego. Jednak do celów miesięcznego rozliczenia opłaty są naliczane w lokalnej walucie określonej na koncie IBM Cloud&reg;. Ponieważ kursy wymiany walut mogą się wahać, koszt X sekund środowiska uruchomieniowego obliczony początkowo w USD może różnić się od kwoty faktycznie pobranej w lokalnej walucie. W związku z tym, jeśli twoja lokalna waluta nie jest USD, łączna kwota naliczona za liczbę sekund określoną w tym polu może różnić się od podanej kwoty w dolarach.
    </Admonition>

<span id="cost-limit"></span>
## Ustaw lub zmień limit kosztów {#set-or-change-a-cost-limit}

Istnieje kilka sposobów ustawienia lub zmiany limitu wydatków:

* **IBM Quantum Platform**: Otwórz stronę [Instances](https://quantum.cloud.ibm.com/instances). Znajdź instancję, którą chcesz ograniczyć, kliknij menu Actions na końcu wiersza, a następnie kliknij **Edit details**. Ustaw łączny limit kosztów lub limit użycia, a następnie kliknij **Save changes**.
   <Admonition type="note">
   Limit kosztów lub limit użycia możesz też ustawić podczas tworzenia nowej instancji w planie Pay-As-You-Go. Po wybraniu kafelka Pay-As-You-Go pojawia się pole z etykietą „Total cost limit (Optional)". W tym polu możesz wprowadzić łączny limit kosztów.
   </Admonition>

* **API**: [Ustaw limit instancji w sekundach.](https://docs.quantum.ibm.com/api/qiskit-runtime-rest/tags/instances#tags__instances__operations__replace_instance_data)

   ```api
   PUT /v1/instances/configuration "instance_limit":<seconds>
   ```

* **IBM Cloud CLI**: Instrukcje dotyczące korzystania z CLI znajdziesz w [Getting started with the IBM Cloud CLI.](https://cloud.ibm.com/docs/cli?topic=cli-getting-started)

   ```cli
      ibmcloud resource service-instance-update <instance_CRN>
      -p '{"instance_limit_seconds":"<seconds>"}'
   ```

## Wyświetl limit kosztów {#view-the-cost-limit}

Istnieje kilka sposobów wyświetlenia aktualnego limitu kosztów:

* **IBM Quantum Platform**: Maksymalny limit kosztów jest wyświetlany na karcie Pay-As-You-Go w tabeli [Instances](https://quantum.cloud.ibm.com/instances).

* **API**:

   ```api
   GET /v1/instances/configuration
   ```
   Zobacz [dokumentację REST API](https://docs.quantum.ibm.com/api/qiskit-runtime-rest/tags/instances).

* **IBM Cloud CLI**: Instrukcje dotyczące korzystania z CLI znajdziesz w [Getting started with the IBM Cloud CLI.](https://cloud.ibm.com/docs/cli?topic=cli-getting-started)

   ```cli
   ibmcloud resource service-instance-update <instance_CRN> -p
   '{"instance_limit_seconds":null}'
   ```

## Usuń limit kosztów {#remove-a-cost-limit}

Limit kosztów możesz usunąć na kilka sposobów:

* **IBM Quantum Platform**: Otwórz tabelę [Instances](https://quantum.cloud.ibm.com/instances). Znajdź instancję, którą chcesz zaktualizować, kliknij menu przepełnienia na końcu wiersza, a następnie kliknij **Edit details**. Usuń łączny limit kosztów lub limit użycia, a następnie kliknij **Save changes**.

* **API**: [Ustaw limit instancji na `null`.](https://docs.quantum.ibm.com/api/qiskit-runtime-rest/tags/instances)

   ```api
   PUT /v1/instances/configuration "instance_limit":null
   ```

* **IBM Cloud CLI**: Instrukcje dotyczące korzystania z CLI znajdziesz w [Getting started with the IBM Cloud CLI.](https://cloud.ibm.com/docs/cli?topic=cli-getting-started)

   ```cli
   ibmcloud resource service-instance-update <instance_CRN> -p
   '{"instance_limit_seconds":null}'
   ```
<Admonition type="note">
Tylko administratorzy i właściciele mogą edytować instancję. Informacje na temat przypisywania dostępu znajdziesz w przewodniku [Tworzenie zasad dostępu i grup dostępu](/guides/access-groups).
</Admonition>

### Co się dzieje po osiągnięciu limitu kosztów {#what-happens-when-the-cost-limit-is-reached}

Po osiągnięciu limitu kosztów instancji bieżące uruchomione zadanie zostaje zatrzymane. Jego status jest ustawiany na Canceled z przyczyną Ran too long. Wszelkie dostępne częściowe wyniki są zachowywane.

Do momentu zwiększenia limitu kosztów za pomocą tej instancji nie można przesyłać kolejnych zadań.

## Jak sprawdzić, co jest ci naliczane {#how-to-see-what-youre-being-charged}

Co miesiąc otrzymasz fakturę zawierającą szczegóły dotyczące opłat za zasoby. W dowolnym momencie możesz sprawdzić, ile zostało wydane, na stronie [IBM Cloud Billing and usage](https://cloud.ibm.com/billing).

Ponadto w dowolnym momencie możesz określić koszt na instancję lub na zadanie.

### Wyświetl użycie {#view-usage}

Użycie instancji możesz wyświetlić na stronie [Analytics](https://quantum.cloud.ibm.com/analytics).

### Wyświetl koszt instancji {#view-instance-cost}

Aby sprawdzić, ile zostało naliczone dla instancji w bieżącym cyklu rozliczeniowym, na stronie [Instances](https://quantum.cloud.ibm.com/instances) kliknij instancję, aby otworzyć jej stronę szczegółów.

Oto pola istotne z punktu widzenia kosztów:

- **Billing cycle usage**: Ilość _czasu kwantowego_ wykorzystana przez tę instancję w bieżącym cyklu rozliczeniowym. Czas kwantowy to czas, przez który QPU jest zaangażowane w realizację żądania użytkownika.
- **Billing cycle cost**: Łączny koszt uruchamiania zadań w bieżącym cyklu rozliczeniowym.
- **Total usage**: Ilość czasu kwantowego wykorzystana przez tę instancję od momentu jej utworzenia.
- **Total cost**: Łączny koszt uruchamiania zadań w tej instancji od momentu jej utworzenia. Tylko administratorzy mogą ustawić tę wartość.

Swój cykl rozliczeniowy możesz wyświetlić na stronie [Billing and usage](https://cloud.ibm.com/billing).

### Wyświetl koszt zadania {#view-job-cost}

Aby sprawdzić, ile zostało naliczone dla każdego zadania powiązanego z instancją, na stronie [Instances](https://quantum.cloud.ibm.com/instances) kliknij instancję, aby otworzyć jej stronę szczegółów. Następnie po lewej stronie kliknij Jobs.

Oto kolumny istotne z punktu widzenia kosztów:

- **Usage**: Ilość czasu kwantowego wykorzystana przez to zadanie. Czas kwantowy to czas, przez który QPU jest zaangażowane w realizację żądania użytkownika.
- **Cost**: Łączny koszt uruchomienia tego zadania.

## Skonfiguruj powiadomienia o wydatkach {#set-up-spending-notifications}

Możesz skonfigurować powiadomienia o wydatkach, aby otrzymywać powiadomienia, gdy twoje konto lub określona usługa osiągnie ustawiony przez ciebie próg wydatków. Więcej informacji znajdziesz na stronie [IBM Cloud Account types](https://cloud.ibm.com/docs/account?topic=account-accounts). Powiadomienia o wydatkach IBM Cloud muszą być stosowane wraz z innymi metodami zarządzania kosztami z kilku powodów:

*   Powiadomienia są wyzwalane dopiero *po* przekroczeniu określonego limitu kosztów.
*   Koszty są przesyłane do systemu rozliczeniowego co godzinę. Dlatego między przesłaniem zadania a wysłaniem powiadomienia o wydatkach może wystąpić znaczne opóźnienie.
*   System rozliczeniowy może potrzebować kilku dni, aby przekazać informacje do systemu fakturowania, co może powodować dalsze opóźnienia w powiadomieniach. Więcej informacji o działaniu systemu rozliczeniowego IBM Cloud znajdziesz w [Setting spending notifications.](https://cloud.ibm.com/docs/billing-usage?topic=billing-usage-spending)

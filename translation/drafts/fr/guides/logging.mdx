# Journalisation pour Qiskit Runtime

Les services IBM Cloud&reg;, tels que Qiskit Runtime, génèrent des journaux de plateforme que tu peux utiliser pour examiner les activités anormales et les actions critiques dans ton compte, ainsi que pour résoudre des problèmes.

Tu peux utiliser [IBM Cloud Logs Routing,](https://cloud.ibm.com/docs/logs-router?topic=logs-router-about) un service de plateforme, pour acheminer les journaux de plateforme de ton compte vers la destination de ton choix en configurant un tenant qui définit l'endroit où les journaux de plateforme sont envoyés. Pour plus d'informations, consulte [À propos de Logs Routing.](https://cloud.ibm.com/docs/logs-router?topic=logs-router-about)

Tu peux utiliser [IBM Cloud Logs](https://cloud.ibm.com/docs/cloud-logs) pour visualiser et recevoir des alertes sur les journaux de plateforme générés dans ton compte et acheminés par IBM Cloud Logs Routing vers une instance IBM Cloud Logs.

## Emplacements où les journaux de plateforme sont générés {#locations-where-platform-logs-are-generated}

IBM Cloud Container Registry génère des journaux de plateforme qui s'affichent dans tes instances de journalisation. Le tableau suivant liste les emplacements où la collecte automatique des journaux du service Container Registry est activée.

| Emplacement                | Journaux des services de plateforme disponibles ? |
|----------------------------|---------------------------------------------------|
| Washington, D.C. (us-east) |                       Oui                        |
| Francfort (eu-de)          |                       Oui                        |

## Journaux de plateforme générés {#platform-logs-that-are-generated}

IBM Qiskit Runtime génère des journaux de services de plateforme qui s'affichent dans tes instances de journalisation dans les cas spécifiques suivants :

- Lorsque l'état d'un job change.

## Afficher les journaux depuis la page Observabilité {#viewing-logs-from-the-observability-page}

Pour en savoir plus sur le lancement de l'interface IBM Cloud Logs, consulte **Launching the UI** dans la [documentation IBM Cloud Logs.](https://cloud.ibm.com/docs/cloud-logs?topic=cloud-logs-instance-launch)

## Champs par type de journal {#fields-by-log-type}

Pour en savoir plus sur les champs inclus dans chaque journal de plateforme, consulte [Fields that are included in platform logs.](https://cloud.ibm.com/docs/logs-router?topic=logs-router-about-platform-logs#about-platform-logs-2)

Les champs suivants sont inclus dans l'enregistrement de journal.

| Champ           | Type      | Description                                                                              |
|-----------------|-----------|------------------------------------------------------------------------------------------|
| logSourceCRN    | Obligatoire | Définit le compte et l'instance de journal de flux où le journal est publié.            |
| saveServiceCopy | Obligatoire | Définit si IBM conserve une copie de l'enregistrement à des fins opérationnelles.       |
| message         | Obligatoire | Description du journal généré.                                                           |

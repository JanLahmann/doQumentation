Les workflows quantiques prennent souvent du temps à s'exécuter et peuvent s'étaler sur plusieurs sessions. Redémarrer ton kernel Python signifie que tu perdras tous les résultats stockés en mémoire. Pour éviter toute perte de données, tu peux sauvegarder les résultats dans un fichier et récupérer les résultats de jobs passés depuis IBM Quantum&reg;, afin que ta prochaine session puisse reprendre là où tu t'étais arrêté.
## Récupérer les résultats de jobs depuis IBM Quantum {#retrieve-job-results-from-ibm-quantum}

IBM Quantum stocke automatiquement les résultats de chaque job pour que tu puisses les récupérer ultérieurement. Utilise cette fonctionnalité pour continuer tes programmes quantiques après un redémarrage du kernel et pour consulter les résultats passés. Tu peux obtenir l'ID d'un job par programme via sa méthode `job_id`, ou tu peux voir tous tes jobs soumis et leurs ID sur la [page Workloads](https://quantum.cloud.ibm.com/workloads).

Pour trouver un job par programme, utilise la méthode [`QiskitRuntimeService.jobs`](https://docs.quantum.ibm.com/api/qiskit-ibm-runtime/qiskit-runtime-service#jobs). Par défaut, elle retourne les jobs soumis le plus récemment, mais tu peux également filtrer les jobs par nom de backend, date de création, et bien plus encore. La cellule suivante recherche tous les jobs soumis au cours des trois derniers mois. L'argument `created_after` doit être un objet [`datetime.datetime`](https://docs.python.org/3.8/library/datetime.html#datetime.datetime).

```python
# Added by doQumentation — required packages for this notebook
```

```python
import datetime
from qiskit_ibm_runtime import QiskitRuntimeService

three_months_ago = datetime.datetime.now() - datetime.timedelta(days=90)

service = QiskitRuntimeService()
jobs_in_last_three_months = service.jobs(created_after=three_months_ago)
jobs_in_last_three_months[:3]  # show first three jobs
```

Tu peux également filtrer par backend, état du job, session, et bien plus encore. Pour plus d'informations, consulte [`QiskitRuntimeService.jobs`](https://docs.quantum.ibm.com/api/qiskit-ibm-runtime/qiskit-runtime-service#jobs) dans la documentation de l'API.

Une fois que tu as l'ID du job, utilise la méthode [`QiskitRuntimeService.job`](https://docs.quantum.ibm.com/api/qiskit-ibm-runtime/qiskit-runtime-service#job) pour le récupérer.

```python
# Get ID of most recent successful job for demonstration.

successful_job = next(
    j for j in service.jobs(limit=1000) if j.status() == "DONE"
)
job_id = successful_job.job_id()
print(job_id)
```

## Sauvegarder les résultats sur disque {#save-results-to-disk}

Tu peux également vouloir sauvegarder les résultats sur disque. Pour ce faire, utilise la bibliothèque JSON intégrée de Python avec les encodeurs de Qiskit Runtime.

```python
import json
from qiskit_ibm_runtime import RuntimeEncoder

with open("result.json", "w") as file:
    json.dump(retrieved_job.result(), file, cls=RuntimeEncoder)
```

Tu peux ensuite charger ce tableau depuis le disque dans un kernel séparé.

```python
from qiskit_ibm_runtime import RuntimeDecoder

with open("result.json", "r") as file:
    result = json.load(file, cls=RuntimeDecoder)

result
```

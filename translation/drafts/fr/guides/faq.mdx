# Questions fréquentes (FAQ) {#frequently-asked-questions-faq}

Pour les questions fréquentes spécifiques aux modes d'exécution, consulte le guide [FAQ sur les modes d'exécution de Qiskit Runtime](/guides/execution-modes-faq).

## Comment citer Qiskit dans mes recherches ? {#how-do-i-cite-qiskit-in-my-research}

Cite Qiskit en utilisant le [fichier BibTeX](https://github.com/Qiskit/qiskit/blob/main/CITATION.bib) fourni.

## Comment citer un ordinateur IBM Quantum dans mes recherches ? {#how-do-i-cite-an-ibm-quantum-computer-in-my-research}

Pour les articles de recherche, nous encourageons les auteurs à mentionner IBM Quantum&reg; en utilisant :

>    Nous reconnaissons l'utilisation des services IBM Quantum pour ce travail. Les opinions exprimées sont celles des auteurs et ne reflètent pas la politique officielle ou la position d'IBM ou de l'équipe IBM Quantum.

Les références bibliographiques doivent utiliser la citation suivante :

> IBM Quantum. https://<span>quantum.cloud.ibm.com</span>/, YYYY

Les unités de traitement quantique dans l'article doivent être référencées par leur nom unique (par exemple, `ibmq_vigo`) en ajoutant éventuellement la version (par exemple, `ibmq_vigo` v1.0.2). Nous encourageons également à référencer le processeur.
Par exemple :

> Dans cet article, nous avons utilisé `ibmq_vigo`, qui est l'un des processeurs IBM Quantum Canary.

Exemple de citation d'un programme IBM Quantum :

> IBM Quantum (2025). Estimator primitive (Version x.y.z) [logiciel informatique]. https://<span>quantum.cloud.ibm.com</span>/

## Comment citer IBM Quantum Composer dans mes recherches ? {#how-do-i-cite-ibm-quantum-composer-in-my-research}

Exemple de citation pour IBM Quantum Composer :

> IBM Quantum Composer. 2025. url : https://<span>quantum.cloud.ibm.com/composer</span>

## Je ne vois pas quelque chose dans l'interface utilisateur, comme mes tâches d'administration. {#i-dont-see-something-in-the-user-interface-such-as-my-administration-tasks}

Essaie de choisir un autre compte depuis le sélecteur de comptes dans l'en-tête de l'interface [IBM Quantum Platform](https://quantum.cloud.ibm.com).

## J'utilise seulement le plan Open, mais j'ai dû entrer ma carte de crédit. Vais-je être facturé(e) ? {#im-just-using-the-open-plan-but-i-had-to-put-in-my-credit-card-will-i-be-charged}

Tu ne seras pas facturé(e) à moins que tu ne crées une [instance](/guides/instances) Pay-As-You-Go et que tu l'utilises pour envoyer des charges de travail aux QPUs. Consulte [Gérer les coûts](/guides/manage-cost) pour obtenir des instructions sur la définition d'une limite de coût et pour en savoir plus sur d'autres façons de gérer les coûts.

## Quelle est la différence entre une clé API et un jeton API ? {#whats-the-difference-between-an-api-key-and-an-api-token}

Il n'y a aucune différence. Les termes clé API et jeton API sont utilisés de façon interchangeable. Tu verras les deux termes dans la documentation, car différentes interfaces emploient des termes différents.

<span id="bearer"></span>
## Quelle est la différence entre un jeton API et un jeton bearer ? {#whats-the-difference-between-an-api-token-and-a-bearer-token}

Un jeton API est un identifiant de longue durée utilisé pour [s'authentifier auprès du service Qiskit Runtime.](/guides/hello-world#install-and-authenticate) Un jeton bearer est un identifiant temporaire utilisé pour [s'authentifier auprès de l'API REST dans un environnement non fiable.](/guides/cloud-setup-untrusted#authenticate-to-the-rest-api-in-an-untrusted-environment) Un jeton bearer ne peut pas être utilisé pour s'authentifier auprès du service Qiskit Runtime.

## Dois-je choisir la région dans laquelle je vis ? {#do-i-have-to-choose-the-region-that-i-live-in}

Non. Tu peux utiliser l'une ou l'autre des régions. La région détermine où les jobs sont exécutés et où les données sont conservées. De plus, tu ne peux voir et utiliser que les instances créées dans l'instance à laquelle tu es connecté(e). Note que seule la région us-east prend en charge le plan Open.

## Puis-je me connecter via un serveur proxy ? {#can-i-connect-through-a-proxy-server}

Oui, mais tu dois utiliser Qiskit Runtime v0.44.0 ou une version ultérieure.

## Quelle est la politique de conservation des données ? {#what-is-the-data-retention-policy}

IBM conserve tes données de jobs et d'utilisation (non supprimées) pendant trois ans tant que tu continues à utiliser le service Qiskit Runtime.

## Qu'est-il arrivé aux distributions Python telles que Qiskit Aer, Qiskit Nature et Qiskit Experiments ? {#what-happened-to-python-distributions-such-as-qiskit-aer-qiskit-nature-and-qiskit-experiments}

Depuis la sortie du Qiskit SDK v1.0, la commande `pip install qiskit` n'installera **que** le [Qiskit SDK](https://github.com/Qiskit/qiskit). Cela signifie que certains modules que tu as utilisés par le passé, tels que `qiskit-aer` ou `qiskit-nature`, doivent désormais être installés séparément.

Avant la sortie du Qiskit SDK v1.0, Qiskit était composé de nombreuses distributions Python différentes, telles que `qiskit-terra`, le cœur du compilateur ; `qiskit-aer`, le simulateur haute performance ; le fournisseur IBM Quantum&reg; d'origine ; et plusieurs packages désormais obsolètes fournissant des fonctionnalités algorithmiques exploratoires particulières ou d'exécution d'expériences.

Ces packages logiciels distincts ont été regroupés dans ce qu'on appelait le *métapackage* Qiskit, qui permettait aux utilisateurs d'installer tous ces composants ensemble via `pip install qiskit`. Cette architecture de métapackage a été supprimée avec la sortie du Qiskit SDK v1.0, et chacune des distributions logicielles individuelles doit maintenant être installée séparément.

Si tu veux installer les packages qui étaient précédemment inclus dans le métapackage Qiskit, visite l'[écosystème Qiskit](https://www.ibm.com/quantum/ecosystem) pour trouver une gamme de packages adaptés à tes besoins. Tu peux également lire le [guide de migration v1.0](/guides/metapackage-migration) pour plus d'informations sur ce changement.

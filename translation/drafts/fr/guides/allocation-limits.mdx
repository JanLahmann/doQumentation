# Définir des limites d'allocation pour une instance

Un compte peut disposer d'un temps alloué sur le plan Premium ou Flex. Étant donné qu'un compte peut avoir plusieurs instances sur un plan, définir des limites d'allocation te permet de contrôler quelles instances sont autorisées à consommer ce temps. Note que si l'allocation n'est pas définie comme une limite, l'instance peut utiliser plus que son temps alloué.

Le temps que tu alloues aux instances est utilisé par l'algorithme de partage équitable pour déterminer la priorité dans la file d'attente, en fonction de l'utilisation de tous les QPU sur une fenêtre temporelle glissante de 28 jours (pour les utilisateurs du plan Premium) ou sur la période d'abonnement (pour les utilisateurs du plan Flex).

L'allocation est définie lors de la création de l'instance. Cependant, tu peux consulter et modifier le temps alloué sur la [page des instances.](https://quantum.cloud.ibm.com/instances)

## Ce qui se passe lorsque la limite est atteinte {#what-happens-when-the-limit-is-reached}

Tu peux éventuellement définir l'allocation de temps comme une limite pour l'instance. Voici ce qui se produit lorsqu'une instance atteint sa limite :

- Les charges de travail actives continuent de s'exécuter.
- Les jobs en attente restent dans la file d'attente.
- Aucune charge de travail n'est annulée.
- Les collaborateurs peuvent soumettre de nouvelles charges de travail, qui resteront dans la file d'attente jusqu'à ce que du temps supplémentaire soit disponible.
- Pour le plan Flex, l'instance ne peut pas traiter de nouveaux jobs à moins que la limite ait été augmentée ou, pour le plan Premium, après le démarrage de la prochaine fenêtre d'utilisation.

## Définir des valeurs différentes pour la limite d'allocation et la limite d'utilisation {#set-different-values-for-the-allocation-limit-and-usage-limit}

Lorsque tu définis la limite d'allocation pour une instance dans IBM Quantum Platform, la même valeur est définie comme limite d'utilisation de l'instance. Si tu souhaites que les utilisateurs puissent continuer à soumettre et exécuter des jobs après que la limite d'allocation de l'instance est atteinte, tu peux définir une valeur plus élevée pour la limite d'utilisation en utilisant le paramètre [usage_allocation_seconds](/guides/access-instances-platform-apis#parameters) dans l'API [Update a resource instance](https://cloud.ibm.com/apidocs/resource-controller/resource-controller#update-resource-instance).

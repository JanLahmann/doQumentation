# Passer du plan Open {#upgrade-from-the-open-plan}

Ce guide s'adresse aux entités qui souhaitent passer du plan IBM Quantum&reg; Open au plan IBM Cloud&reg; Pay-As-You-Go. Pour configurer un compte cloud, créer une instance de service et gérer les utilisateurs, suis les étapes ci-dessous.

Si tu es un particulier qui configure le service pour la première fois, ou si tu as été invité à une instance par un administrateur, suis plutôt les étapes de [Installer Qiskit](./install-qiskit).

## Avant de commencer {#before-you-begin}

Si nécessaire, suis les étapes de la section [Configurer pour utiliser IBM Cloud](./cloud-setup#cloud) afin d'effectuer les opérations suivantes :

- Créer un compte IBM Cloud.
- Créer une instance IBM Quantum.

<span id="update-cloud-account"></span>
## Mettre à niveau ton compte IBM Cloud {#upgrade-your-ibm-cloud-account}

Ignore cette étape si tu possèdes déjà un compte IBM Cloud Pay-as-you-go. Tous les nouveaux comptes IBM Cloud sont configurés en Pay-as-you-go.

Tu dois mettre à niveau ton compte IBM Cloud dans les cas suivants :

- Tu possèdes un compte Trial.
- Tu possèdes un compte Lite (obsolète) créé avant le 25 octobre 2021.

Après la mise à niveau, tu peux continuer à utiliser toutes les instances que tu as créées avec ton compte Lite. Pour passer à un compte Pay-As-You-Go, effectue les étapes suivantes :

1. Va dans Gérer > [Compte](https://cloud.ibm.com/account) dans la console IBM Cloud.
2. Sélectionne **Paramètres du compte**, puis clique sur **Ajouter une carte de crédit**.
3. Saisis tes informations de paiement et clique sur **Suivant** pour les soumettre.
4. Une fois tes informations de paiement traitées, ton compte est mis à niveau.
   <Admonition type="note">
   Tu ne seras pas facturé à moins de créer une instance du plan Pay-As-You-Go et de l'utiliser pour envoyer des charges de travail aux QPUs.
   </Admonition>

## Accéder à ton plan Pay-As-You-Go {#access-your-pay-as-you-go-plan}

Pour utiliser ton plan IBM Cloud Pay-As-You-Go, crée une nouvelle [instance](/guides/instances) et affecte-la au plan Pay-As-You-Go. <ins>Tous les jobs envoyés aux QPUs en spécifiant cette instance entraîneront un coût.</ins> Pour utiliser les QPUs sans frais, tu dois utiliser une instance affectée au plan Open ou à un autre plan gratuit, si tu en as un.

Il est recommandé de configurer une [limite de coût](/guides/manage-cost#cost-limit) sur l'instance et de te familiariser avec les autres moyens de gérer tes coûts lors de l'utilisation du plan Pay-As-You-Go.

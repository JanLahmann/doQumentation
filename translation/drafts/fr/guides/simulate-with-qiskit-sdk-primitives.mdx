### Initialiser l'Estimator {#initialize-estimator}

Instancie un [`qiskit.primitives.StatevectorEstimator`](../api/qiskit/qiskit.primitives.StatevectorEstimator).


### ExГ©cuter et obtenir les rГ©sultats {#run-and-get-results}

Cet exemple n'utilise qu'un seul circuit (de type [`QuantumCircuit`](../api/qiskit/qiskit.circuit.QuantumCircuit)) et un seul
observable.

Lance l'estimation en appelant la mГ©thode [`StatevectorEstimator.run`](../api/qiskit/qiskit.primitives.StatevectorEstimator#run), qui retourne une instance d'un objet [`PrimitiveJob`](https://docs.quantum.ibm.com/api/qiskit/qiskit.primitives.PrimitiveJob). Tu peux obtenir les rГ©sultats du job (sous forme d'objet [`qiskit.primitives.PrimitiveResult`](../api/qiskit/qiskit.primitives.PrimitiveResult))
avec la mГ©thode [`qiskit.primitives.PrimitiveJob.result`](../api/qiskit/qiskit.primitives.PrimitiveJob#result).

```python
job = estimator.run([(circuit, observable, parameter_values)])

print(f" > Result class: {type(result)}")
```


### Configurer les options d'exГ©cution de l'Estimator {#set-estimator-run-options}

Par dГ©faut, l'Estimator de rГ©fГ©rence effectue un calcul exact par vecteur d'Г©tat en s'appuyant sur la classe
[`quantum_info.Statevector`](../api/qiskit/qiskit.quantum_info.Statevector).
Il est toutefois possible de le modifier pour introduire l'effet du surcoГ»t d'Г©chantillonnage (aussi appelГ© В« bruit de grenaille В»).

L'Estimator accepte un argument `precision` qui exprime les barres d'erreur que l'implГ©mentation primitive doit cibler pour les estimations de valeurs d'espГ©rance. Il s'agit du surcoГ»t d'Г©chantillonnage, dГ©fini exclusivement dans la mГ©thode `.run()`. Cela te permet d'affiner l'option jusqu'au niveau PUB.

```python

precise_job = estimator.run(
    [(circuit, observable, parameter_values)], precision=0.05
)
```

Les primitives acceptent plusieurs PUBs en entrГ©e, et chaque PUB obtient son propre rГ©sultat. Tu peux donc exГ©cuter diffГ©rents circuits avec diverses combinaisons de paramГЁtres et d'observables, et rГ©cupГ©rer les rГ©sultats des PUBs :

```python

circuit2 = circuit.copy()

# transpile circuits

 isa_circuit1 = pm.run(circuit1)
 isa_circuit2 = pm.run(circuit2)
# execute 2 circuits using Sampler
job = sampler.run([(isa_circuit1), (isa_circuit2)])
pub_result_1 = job.result()[0]
pub_result_2 = job.result()[1]

```

```text
> Result class: <class 'qiskit.primitives.containers.sampler_pub_result.SamplerPubResult'>
```

### Modifier les options d'exГ©cution {#change-run-options}

Par dГ©faut, le Sampler de rГ©fГ©rence effectue un calcul exact par vecteur d'Г©tat en s'appuyant sur la classe

## 2.1 Map the problem to a quantum-native format

Dans un programme quantique, les *circuits quantiques* sont le format natif pour représenter les instructions quantiques, et les *opérateurs* représentent les observables à mesurer. Quand tu crées un circuit, tu vas généralement créer un nouvel objet [`QuantumCircuit`](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.QuantumCircuit#quantumcircuit-class), puis y ajouter des instructions en séquence.

La cellule de code suivante crée un circuit qui produit l'état GHZ, qui est un état dans lequel trois qubits sont totalement intriqués les uns avec les autres.

<div class="alert alert-info">

  Le SDK Qiskit utilise la numérotation de bits LSb 0 où le $n^{th}$ chiffre a la valeur $1 \ll n$ ou $2^n$. Pour plus de détails, consulte le sujet [Bit-ordering in the Qiskit SDK](https://docs.quantum.ibm.com/guides/bit-ordering).

</div>

```python
# Create a GHZ state circuit

qc = QuantumCircuit(3)
qc.h(0)
qc.cx(0,1)
qc.cx(0,2)
# Draw the circuit
qc.draw("mpl")
```

![Quantum circuit diagram](./_04_Hands-on Introduction to Qiskit_imgs/output_15.png)

Consulte [`QuantumCircuit`](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.QuantumCircuit#quantumcircuit-class) dans la documentation pour toutes les opérations disponibles.

Quand tu crées des circuits quantiques, tu dois aussi réfléchir au type de données que tu veux récupérer après l'exécution. Qiskit offre deux façons de retourner des données : tu peux obtenir une distribution de probabilité pour un ensemble de qubits que tu choisis de mesurer, ou tu peux obtenir la valeur d'espérance d'un observable. Prépare ta charge de travail pour mesurer ton circuit de l'une de ces deux façons avec les [primitives Qiskit](https://docs.quantum.ibm.com/guides/get-started-with-primitives) (expliquées en détail à l'Étape 3).

Cet exemple mesure des valeurs d'espérance en utilisant le sous-module `qiskit.quantum_info`, qui est spécifié à l'aide d'opérateurs (objets mathématiques utilisés pour représenter une action ou un processus qui modifie un état quantique). La cellule de code suivante crée six opérateurs de Pauli à trois qubits : `ZZZ`, `ZZX`, `ZII`, `XXI`, `ZZI` et `III`.

```python
# Set up six different observables.

observables_labels = ["ZZZ", "ZZX", "ZII", "XXI", "ZZI", "III"]

observables = [SparsePauliOp(label) for label in observables_labels]
print(observables)
```

```text
[SparsePauliOp(['ZZZ'],
              coeffs=[1.+0.j]), SparsePauliOp(['ZZX'],
              coeffs=[1.+0.j]), SparsePauliOp(['ZII'],
              coeffs=[1.+0.j]), SparsePauliOp(['XXI'],
              coeffs=[1.+0.j]), SparsePauliOp(['ZZI'],
              coeffs=[1.+0.j]), SparsePauliOp(['III'],
              coeffs=[1.+0.j])]
```

<div class="alert alert-info">

  Ici, un opérateur comme `ZZI` est une notation abrégée pour le produit tensoriel $Z\otimes Z\otimes I$, ce qui signifie mesurer Z sur le qubit 2 et Z sur le qubit 1 ensemble, et obtenir des informations sur la corrélation entre le qubit 2 et le qubit 1. Les valeurs d'espérance comme celle-ci sont aussi typiquement notées $\langle Z_2 Z_1 \rangle$.

  Si l'état que l'on observe est l'état GHZ à trois qubits, alors la mesure de $\langle Z_2 Z_1 \rangle$ devrait être 1.

</div>

## 2.2 Optimize the circuits and operators

Quand tu exécutes des circuits sur un appareil, il est important d'optimiser l'ensemble des instructions que le circuit contient et de minimiser la profondeur globale (approximativement le nombre d'instructions) du circuit. Cela garantit que tu obtiens les meilleurs résultats possibles en réduisant les effets des erreurs et du bruit. De plus, les instructions du circuit doivent être conformes à l'[architecture du jeu d'instructions (ISA)](https://docs.quantum.ibm.com/guides/transpile#instruction-set-architecture) d'un Backend et doivent prendre en compte les portes de base et la connectivité des qubits de l'appareil.

Le code suivant instancie un vrai appareil auquel soumettre une tâche, et transforme le circuit et les observables pour correspondre à l'ISA de ce Backend.
Si tu n'as pas encore enregistré tes identifiants, suis les instructions [ici](https://docs.quantum.ibm.com/guides/setup-channel#iqp) pour t'authentifier avec ton jeton API.

```python
# Choose a real backend
service = QiskitRuntimeService(channel='ibm_quantum_platform',)

    f"Processor type: {backend.processor_type}\n"
)
```

## 2.3 Execute using the quantum primitives

Les ordinateurs quantiques peuvent produire des résultats aléatoires, donc tu collectes généralement un échantillon des sorties en exécutant le circuit plusieurs fois. Tu peux estimer la valeur de l'observable en utilisant la classe `Estimator`. `Estimator` est l'une des deux [primitives](https://docs.quantum.ibm.com/guides/get-started-with-primitives) ; l'autre est `Sampler`, qui peut être utilisé pour obtenir des données d'un ordinateur quantique. Ces objets possèdent une méthode `run()` qui exécute la sélection de circuits, d'observables et de paramètres (le cas échéant), en utilisant un [bloc unifié de primitives (PUB).](https://docs.quantum.ibm.com/guides/primitives#sampler)
Quand tu exécutes ce code sur du vrai matériel quantique, pense à appliquer des [techniques d'atténuation et de suppression d'erreurs](https://quantum.cloud.ibm.com/docs/en/guides/error-mitigation-and-suppression-techniques) pour réduire le bruit intrinsèque de l'ordinateur quantique.

```python
# Construct the Estimator instance.
estimator = Estimator(mode=backend)
estimator.options.resilience_level = 1
estimator.options.default_shots = 5000
```

Soumet une tâche en utilisant la primitive Estimator.

```python
# One pub, with one circuit to run against six different observables.
job = estimator.run([(isa_circuit, mapped_observables)]) 

# Use the job ID to retrieve your job data later
print(f">>> Job ID: {job.job_id()}")
```

Après avoir soumis une tâche, tu peux attendre que la tâche soit terminée dans ton instance Python courante, ou utiliser le `job_id` pour récupérer les données ultérieurement. (Consulte la [section sur la récupération des tâches](https://docs.quantum.ibm.com/guides/monitor-job#retrieve-job-results-at-a-later-time) pour plus de détails.)

Après la fin de la tâche, examine sa sortie via l'attribut `result()` de la tâche.

```python
# This is the result of the entire submission.  You submitted one Pub,
# so this contains one inner result (and some metadata of its own).
job_result = job.result()

# This is the result from our single pub, which had six observables,
# so contains information on all six.
pub_result = job.result()[0]
```

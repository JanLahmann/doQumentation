        vec = A @ ks[j].T
        ortho = orthoset(vec, ks)
        ks.append(ortho)
        ksarray = np.array(ks)
        Hs.append(ksarray @ A @ ksarray.T)
        eigs.append(np.linalg.eig(Hs[j + 1]).eigenvalues)
        k_tot_times.append(time_mus() - t0)

    # Returnează vectorii Krylov, Hamiltonianele proiectate, valorile proprii și timpul total necesar.
    return (ks, Hs, eigs, k_tot_times)
```

def errors(matrix, krylov_eigs):
    targ_min = min(np.linalg.eig(matrix).eigenvalues)
    err = []
    for i in range(len(matrix)):
        err.append(min(krylov_eigs[i]) - targ_min)
    return err
```

### 2.2 Scalarea în timp cu dimensiunea matricei {#time-scaling-with-matrix-dimension}

Să ne convingem că metoda Krylov poate fi avantajoasă față de solverele numerice exacte de valori proprii în felul următor:
* Construim matrice aleatorii (nu sparse, nu aplicația ideală pentru KQD)
* Determinăm valorile proprii folosind două metode: direct cu NumPy și utilizând un subspațiu Krylov.
* Alegem o limită pentru cât de precise trebuie să fie valorile noastre proprii înainte de a accepta estimările Krylov.
* Comparăm timpul de execuție necesar pentru a rezolva problema în cele două moduri.

__Avertismente:__ Așa cum vom discuta în detaliu mai jos, diagonalizarea cuantică Krylov este cel mai bine aplicată operatorilor ale căror reprezentări matriceale sunt sparse și/sau pot fi scrise folosind un număr mic de grupuri de operatori Pauli care comută. Matricele aleatorii pe care le folosim aici nu se potrivesc cu această descriere. Acestea sunt utile doar pentru a sonda scala la care metodele Krylov clasice ar putea fi utile.
În al doilea rând, în utilizarea metodei Krylov vom calcula valorile proprii folosind mai multe subspații de dimensiuni diferite. Vom raporta timpul necesar pentru subspațiul Krylov de dimensiune minimă care atinge precizia cerută pentru valoarea proprie a stării fundamentale. Din nou, acest lucru este ușor diferit de rezolvarea unei probleme care nu poate fi rezolvată de solverele exacte, deoarece folosim soluția exactă pentru a evalua dimensiunea necesară.

Începem prin a genera setul nostru de matrice aleatorii.

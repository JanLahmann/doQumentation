Vom vedea asta în mai multe exemple de mai jos. Extinderea la repetări multiple este aceeași ca în cazul `z_feature_map`:
$$
\mathscr{U}^{(r)}_{\text{ZZFM}}\left(\vec{x}\right)=\prod_{s=1}^{r}\left[U_{ZZ}(\vec{x})\left(\bigotimes_{k = 1}^N P(\vec{x}_k)\right)H^{\otimes N}\right].
$$
Deoarece operatorii au crescut în complexitate, să codificăm mai întâi un vector de date $\vec{x} = (x_0, x_1)$ cu un ZZFM pe doi qubiți și o singură repetare, folosind codul următor:

```python
from qiskit.circuit.library import zz_feature_map

feature_dim = 2
zzfeature_map = zz_feature_map(
    feature_dimension=feature_dim, entanglement="linear", reps=1
)
zzfeature_map.decompose(reps=1).draw("mpl", scale=1)
```

În mod implicit în Qiskit, caracteristicile $(\vec{x}_1, \vec{x}_2)$ sunt mapate împreună la $R_{ZZ}(\theta)$ prin această funcție de mapare $\theta_{1,2} = \phi(\vec{x}_1, \vec{x}_2) = 2(\pi-\vec{x}_1)(\pi-\vec{x}_2)$. Qiskit îi permite utilizatorului să personalizeze funcția $\phi$ (sau $\phi_S$, unde $S$ este mulțimea perechilor de qubiți cuplați prin porți $R_{ZZ}$) ca pas de preprocesare.

Trecând la un vector de date cu patru dimensiuni $\vec{x} = (\vec{x}_1, \vec{x}_2, \vec{x}_3, \vec{x}_4)$ și mapând la un ZZFM pe patru qubiți cu o singură repetare, putem începe să vedem maparea $\phi$ pentru diverse perechi de qubiți. Putem de asemenea să înțelegem sensul entanglamentului „linear":

```python
feature_dim = 4
zzfeature_map = zz_feature_map(
```

### Harta de caracteristici Pauli {#pauli-feature-map}

Harta de caracteristici Pauli (PFM) este generalizarea ZFM și ZZFM la utilizarea de porți Pauli arbitrare. Harta de caracteristici Pauli are o formă foarte asemănătoare cu cele două hărți de caracteristici anterioare. Pentru $r$ repetări ale codificării celor $N$ caracteristici ale vectorului $\vec{x},$

$$
\mathscr{U}_{\text{PFM}}(\vec{x}) = \prod_{s=1}^{r} U(\vec{x}) H^{\otimes n}.
$$

Pentru PFM, $U(\vec{x})$ este generalizat la un operator unitar de expansiune Pauli. Prezentăm aici o formă mai generală a hărților de caracteristici considerate până acum:

$$
U(\vec{x}) = \exp\left(i \sum_{S \in\mathcal{I}} \phi_S(\vec{x}) \prod_{i \in S} \sigma_i \right),
$$

unde $\sigma_i$ este un operator Pauli, $\sigma_i \in {I,X,Y,Z}$. Aici $\mathcal{I}$ este mulțimea tuturor conectivităților dintre qubiți, determinată de harta de caracteristici, inclusiv mulțimea qubiților acționați de porți cu un singur qubit. Adică, pentru o hartă de caracteristici în care qubitul 0 este acționat de o poartă de fază, iar qubiții 2 și 3 sunt acționați de o poartă $R_{ZZ}$, mulțimea $\mathcal{I}$ ar include $\{\{0\},\{2,3\}\}$. $S$ parcurge toate elementele acelei mulțimi. În hărțile de caracteristici anterioare, funcția $\phi_S(\vec{x})$ era implicată fie exclusiv cu porți cu un singur qubit, fie exclusiv cu porți cu doi qubiți. Aici o definim în general:
$$
\phi_S(\vec{x})=
    \begin{cases}
      x_i & \text{if } S= \{i\} \text{ (single-qubit)}\\
      \prod_{j\in{S}}(\pi-x_j) & \text{if } |S|\ge2 \text{ (multi-qubit)}\\
    \end{cases}
$$

Pentru documentație, consultă [documentația clasei Qiskit `Pauli feature map`](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.library.PauliFeatureMap)). În ZZFM, operatorul $\sigma_i$ este restricționat la $Z_i$.

Un mod de a înțelege unitarul de mai sus este prin analogie cu propagatorul dintr-un sistem fizic. Unitarul de mai sus este un operator de evoluție unitară, $\exp(it\mathcal{H})$, pentru un Hamiltonian, $\mathcal{H}$, similar modelului Ising, unde parametrul de timp, $t$, este înlocuit cu valorile datelor pentru a conduce evoluția. Expansiunea acestui operator unitar produce circuitul PFM. Conectivitățile de entanglament din $S$ pot fi interpretate ca cuplaje Ising într-o rețea de spini.
Să considerăm un exemplu cu operatorii Pauli $Y$ și $XX$ reprezentând acele interacțiuni de tip Ising. Qiskit oferă o clasă `pauli_feature_map` pentru instanțierea unui PFM cu o alegere de porți cu un singur qubit și cu $n$ qubiți, care în acest exemplu vor fi transmise ca șiruri Pauli `'Y'` și `'XX'`. De obicei, $n$ este 1 sau 2 pentru interacțiuni cu un singur qubit, respectiv cu doi qubiți. Schema de entanglament este „linear", ceea ce înseamnă că doar qubiții vecini din circuitul cuantic sunt cuplați. Reține că aceasta nu corespunde qubiților vecini pe calculatorul cuantic în sine, deoarece acest circuit cuantic este un nivel de abstractizare.

```python
from qiskit.circuit.library import pauli_feature_map

feature_dim = 3
pfmap = pauli_feature_map(
    feature_dimension=feature_dim, entanglement="linear", reps=1, paulis=["Y", "XX"]
)

pfmap.decompose().draw("mpl", scale=1.5)
```

![Output of the previous code cell](/learning/images/courses/quantum-machine-learning/data-encoding/extracted-outputs/5ba5df82-83c1-428c-a269-baea5f75c3dc-0.avif)

Qiskit oferă un parametru, $\alpha$, în hărțile de caracteristici Pauli pentru a controla scalarea rotațiilor Pauli.

$$
U(\bar{x}) = \exp\left(i \alpha \sum_{S\subseteq[n]} \phi_S(\bar{x}) \prod_{i \in S} \sigma_i \right)
$$

Valoarea implicită a lui $\alpha$ este $2$. Optimizând valoarea sa în intervalul, de exemplu, $[0,4],$ se poate alinia mai bine un kernel cuantic cu datele.

```
import numpy as np
from math import sqrt

init_list = [4, 8, 5, 9, 8, 6, 2, 9, 2, 5, 7, 0, 3, 7, 5]
qubits = round(np.log(len(init_list)) / np.log(2) + 0.4999999999)
need_length = 2**qubits
pad = need_length - len(init_list)
for i in range(0, pad):
    init_list.append(0)

init_array = np.array(init_list)  # Unnormalized data vector
length = sqrt(
    sum(init_array[i] ** 2 for i in range(0, len(init_array)))
)  # Vector length
norm_array = init_array / length  # Normalized array
print("Normalized array:")
print(norm_array)
print()

qubit_numbers = []
for i in range(0, qubits):
    qubit_numbers.append(i)
print(qubit_numbers)

qc = QuantumCircuit(qubits)
qc.initialize(norm_array, qubit_numbers)
qc.decompose(reps=7).draw(output="mpl")
```

Un avantaj al codificării prin amplitudine este că sunt necesari doar $\log_2(N)$ qubiți pentru un vector de date $\vec{x}\rightarrow|\vec{x}\rangle$ cu $N$ dimensiuni ($N$ caracteristici). Totuși, codificarea prin amplitudine este în general o procedură ineficientă ce necesită pregătirea unui stări arbitrare, care este exponențială în numărul de porți CNOT. Cu alte cuvinte, pregătirea stării are o complexitate de timp polinomială de $\mathcal O(N)$ în numărul de dimensiuni, unde $N = 2^n$, iar $n$ este numărul de qubiți. Codificarea prin amplitudine „oferă o economie exponențială în spațiu cu prețul unei creșteri exponențiale în timp"[\[3\]](#references); totuși, creșterile de timp de execuție la $\mathcal O(\log N)$ sunt realizabile în anumite cazuri[\[4\]](#references). Pentru o accelerare cuantică end-to-end, complexitatea de timp de execuție a încărcării datelor trebuie luată în considerare.

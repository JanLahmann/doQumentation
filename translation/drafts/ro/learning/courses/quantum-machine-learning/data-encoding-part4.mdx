### Codificare unghiulară {#angle-encoding}

Codificarea unghiulară prezintă interes în multe modele QML care utilizează hărți de caracteristici Pauli, cum ar fi mașinile cu vector de suport cuantice (QSVM) și circuitele cuantice variationale (VQC), printre altele. Codificarea unghiulară este strâns legată de codificarea de fază și codificarea unghiulară densă, prezentate mai jos. Vom folosi „codificare unghiulară" pentru a ne referi la o rotație în $\theta$, adică o rotație față de axa $z$, realizată, de exemplu, cu o poartă $R_X$ sau o poartă $R_Y$[\[1,3\]](#references). De fapt, datele pot fi codificate în *orice* rotație sau combinație de rotații. Totuși, $R_Y$ este frecventă în literatură, motiv pentru care o accentuăm aici.

Când este aplicată unui singur qubit, codificarea unghiulară imprimă o rotație pe axa Y proporțională cu valoarea datelor. Consideră codificarea unei singure caracteristici (a $k$-a) din cel de-al $j$-lea vector de date dintr-un set de date, $\vec{x}^{(j)}_k$:

$$
|\vec{x}^{(j)}_k\rangle = R_Y(\theta=\vec{x}^{(j)}_k)|0\rangle = \textstyle\cos\left(\frac{\vec{x}^{(j)}_k}{2}\right)|0\rangle + \sin\left(\frac{\vec{x}^{(j)}_k}{2}\right)|1\rangle.
$$

Ca alternativă, codificarea unghiulară poate fi realizată cu porți $R_X(\theta)$, deși starea codificată ar avea o fază relativă complexă față de $R_Y(\theta)$.

Codificarea unghiulară diferă de cele două metode anterioare în mai multe privințe. În codificarea unghiulară:
- Fiecare valoare de caracteristică este mapată la un qubit corespunzător, $\vec{x}^{(j)}_k \rightarrow Q_k$, lăsând qubiții într-o stare produs.
- Se codifică o singură valoare numerică la un moment dat, nu un întreg set de caracteristici dintr-un punct de date.
- Sunt necesari $n$ qubiți pentru $N$ caracteristici de date, unde $n\leq N$. Adesea egalitatea se respectă. Vom vedea cum este posibil $n<N$ în secțiunile următoare.
- Circuitul rezultat are adâncime constantă (de obicei adâncimea este 1 înainte de transpilare).

Circuitul cuantic cu adâncime constantă îl face deosebit de potrivit pentru hardware-ul cuantic actual. O caracteristică suplimentară a codificării datelor folosind $\theta$ (și, în special, alegerea de a folosi codificarea unghiulară pe axa Y) este că produce stări cuantice cu valori reale, utile pentru anumite aplicații. Pentru rotația pe axa Y, datele sunt mapate cu o poartă de rotație pe axa Y, $R_Y(\theta)$, printr-un unghi real $\theta \in (0, 2\pi]$ ([Qiskit RYGate](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.library.RYGate)). Ca și în cazul codificării de fază (vezi mai jos), recomandăm rescalarea datelor astfel încât $\vec{x}^{(j)}_k \in (0,2\pi]$, pentru a preveni pierderea de informații și alte efecte nedorite.

Următorul cod Qiskit rotește un singur qubit dintr-o stare inițială $|0\rangle$ pentru a codifica o valoare de date $\vec{x}^{(j)}_k=\frac{1}{2}\pi$.

```python
from qiskit.quantum_info import Statevector
from math import pi

qc = QuantumCircuit(1)
state1 = Statevector.from_instruction(qc)
qc.ry(pi / 2, 0)  # Phase gate rotates by an angle pi/2
state2 = Statevector.from_instruction(qc)
states = state1, state2
```

Vom defini o funcție pentru a vizualiza acțiunea asupra vectorului de stare. Detaliile definiției funcției nu sunt importante, dar capacitatea de a vizualiza vectorii de stare și modificările acestora este importantă.

```python
import numpy as np
from qiskit.visualization.bloch import Bloch
from qiskit.visualization.state_visualization import _bloch_multivector_data

def plot_Nstates(states, axis, plot_trace_points=True):
    """This function plots N states to 1 Bloch sphere"""
    bloch_vecs = [_bloch_multivector_data(s)[0] for s in states]

    if axis is None:
        bloch_plot = Bloch()
    else:
        bloch_plot = Bloch(axes=axis)

    bloch_plot.add_vectors(bloch_vecs)

    if len(states) > 1:

        def rgba_map(x, num):
            g = (0.95 - 0.05) / (num - 1)
            i = 0.95 - g * num
            y = g * x + i
            return (0.0, y, 0.0, 0.7)

        num = len(states)
        bloch_plot.vector_color = [rgba_map(x, num) for x in range(1, num + 1)]

    bloch_plot.vector_width = 3
    bloch_plot.vector_style = "simple"

    if plot_trace_points:

        def trace_points(bloch_vec1, bloch_vec2):
            # bloch_vec = (x,y,z)
            n_points = 15
            thetas = np.arccos([bloch_vec1[2], bloch_vec2[2]])
            phis = np.arctan2(
                [bloch_vec1[1], bloch_vec2[1]], [bloch_vec1[0], bloch_vec2[0]]
            )
            if phis[1] < 0:
                phis[1] = phis[1] + 2 * pi
            angles0 = np.linspace(phis[0], phis[1], n_points)
            angles1 = np.linspace(thetas[0], thetas[1], n_points)

            xp = np.cos(angles0) * np.sin(angles1)
            yp = np.sin(angles0) * np.sin(angles1)
            zp = np.cos(angles1)
            pnts = [xp, yp, zp]
            bloch_plot.add_points(pnts)
            bloch_plot.point_color = "k"
            bloch_plot.point_size = [4] * len(bloch_plot.points)
            bloch_plot.point_marker = ["o"]

        for i in range(len(bloch_vecs) - 1):
            trace_points(bloch_vecs[i], bloch_vecs[i + 1])

    bloch_plot.sphere_alpha = 0.05
    bloch_plot.frame_alpha = 0.15
    bloch_plot.figsize = [4, 4]

    bloch_plot.render()

plot_Nstates(states, axis=None, plot_trace_points=True)
```

Aceasta a fost doar o singură caracteristică a unui singur vector de date. Când se codifică $N$ caracteristici în unghiurile de rotație ale $n$ qubiți, de exemplu pentru cel de-al $j$-lea vector de date $\vec{x}^{(j)} = (x_1,...,x_N),$ starea produs codificată va arăta astfel:

$$
|\vec{x}^{(j)}\rangle = \bigotimes^N_{k=1} \cos(\vec{x}^{(j)}_k)|0\rangle + \sin(\vec{x}^{(j)}_k)|1\rangle
$$

Observăm că aceasta este echivalentă cu

$$
|\vec{x}^{(j)}\rangle = \bigotimes^N_{k=1} R_Y(2\vec{x}^{(j)}_k)|0\rangle.
$$

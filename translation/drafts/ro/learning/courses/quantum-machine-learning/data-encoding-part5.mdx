### Codificarea prin fază {#phase-encoding}

Codificarea prin fază este foarte similară cu codificarea prin unghi descrisă mai sus. Unghiul de fază al unui qubit este un unghi real $\phi$ în jurul axei $z$ față de axa +$x$. Datele sunt mapate cu o rotație de fază, $P(\phi) = e^{i\phi/2}R_Z(\phi)$, unde $\phi \in (0,2\pi]$ (consultă [Qiskit PhaseGate](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.library.PhaseGate) pentru mai multe informații). Se recomandă reescalarea datelor astfel încât $\vec{x}^{(j)}_k \in (0,2\pi]$. Aceasta previne pierderea de informații și alte efecte potențial nedorite[\[1,2\]](#references).

Un qubit este adesea inițializat în starea $|0\rangle$, care este o stare proprie a operatorului de rotație de fază, ceea ce înseamnă că starea qubitului trebuie mai întâi rotită pentru ca codificarea prin fază să poată fi implementată. Prin urmare, are sens să inițializezi starea cu o poartă Hadamard: $H|0\rangle = |+\rangle = \textstyle\frac{1}{\sqrt{2}}(|0\rangle + |1\rangle)$. Codificarea prin fază pe un singur qubit înseamnă imprimarea unei faze relative proporționale cu valoarea datelor:

$$
|\vec{x}^{(j)}_k\rangle = P(\phi=\vec{x}^{(j)}_k)|+\rangle = \textstyle\frac{1}{\sqrt{2}}\big(|0\rangle + e^{i\vec{x}^{(j)}_k}|1\rangle\big).
$$

Procedura de codificare prin fază mapează fiecare valoare de caracteristică la faza unui qubit corespunzător, $\vec{x}^{(j)}_k \rightarrow Q_k$. În total, codificarea prin fază are o adâncime de circuit de 2, inclusiv stratul Hadamard, ceea ce o face o schemă de codificare eficientă. Starea multi-qubit codificată prin fază ($n$ qubiți pentru $N=n$ caracteristici) este o stare produs:

$$
|\vec{x}^{(j)}\rangle = \bigotimes_{k=1}^{N} P_k(\phi = \vec{x}^{(j)}_k)|+\rangle^{\otimes N} = {\textstyle\frac{1}{\sqrt{2^N}}} \bigotimes_{k=1}^{N}\big(|0\rangle + e^{i\vec{x}^{(j)}_k}|1\rangle\big).
$$

Următorul cod Qiskit pregătește mai întâi starea inițială a unui singur qubit rotindu-l cu o poartă Hadamard, apoi îl rotește din nou folosind o poartă de fază pentru a codifica o caracteristică de date $\vec{x}^{(j)}_k=\frac{1}{2}\pi$.

```python
qc = QuantumCircuit(1)
qc.h(0)  # Hadamard gate rotates state down to Bloch equator
state1 = Statevector.from_instruction(qc)

qc.p(pi / 2, 0)  # Phase gate rotates by an angle pi/2
state2 = Statevector.from_instruction(qc)

states = state1, state2

qc.draw("mpl", scale=1)
```

### Codificarea densă prin unghi {#dense-angle-encoding}

Codificarea densă prin unghi (DAE) este o combinație de codificare prin unghi și codificare prin fază. DAE permite codificarea a două valori de caracteristici într-un singur qubit: un unghi cu o rotație pe axa Y, iar celălalt cu o rotație pe axa $z$: $\vec{x}^{(j)}_k,$ $\vec{x}^{(j)}_\ell \rightarrow \theta, \phi$. Codifică două caracteristici astfel:

$$
|\vec{x}^{(j)}_k,\vec{x}^{(j)}_\ell\rangle = R_Z(\phi=\vec{x}^{(j)}_\ell) R_Y(\theta=\vec{x}^{(j)}_k)|0\rangle = \cos\left(\frac{\vec{x}^{(j)}_k}{2}\right)|0\rangle + e^{i\vec{x}^{(j)}_\ell} \sin\left(\frac{\vec{x}^{(j)}_k}{2}\right)|1\rangle.
$$

Codificarea a două caracteristici de date într-un singur qubit duce la o reducere de $2\times$ a numărului de qubiți necesari pentru codificare. Extinzând aceasta la mai multe caracteristici, vectorul de date $\vec{x} = (x_1,...,x_N)$ poate fi codificat astfel:

$$
|\vec{x}\rangle = \bigotimes_{k=1}^{N/2} \cos(x_{2k-1})|0\rangle + e^{i x_{2k}}\sin(x_{2k-1})|1\rangle
$$

DAE poate fi generalizată la funcții arbitrare ale celor două caracteristici în loc de funcțiile sinusoidale folosite aici. Aceasta se numește codificare generală a qubitului[\[7\]](#references).

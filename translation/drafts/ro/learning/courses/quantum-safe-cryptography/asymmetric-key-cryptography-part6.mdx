
1. Creează un hash al textului cifrat folosind un algoritm de hashing.
2. Criptează hash-ul folosind cheia privată a lui Alice, ceea ce constituie o semnătură.

```python
# Alice signs the ciphertext using her private key
digest = hashes.Hash(hashes.SHA256())
digest.update(ciphertext)
hash_to_sign = digest.finalize()

signature = alice_private_key.sign(
    hash_to_sign,
    padding.PSS(mgf=padding.MGF1(hashes.SHA256()), salt_length=padding.PSS.MAX_LENGTH),
    Prehashed(hashes.SHA256()),
)

print("signature: ", signature)
```

Alice transmite apoi textul cifrat și semnătura printr-o rețea, astfel încât Bob să le poată intercepta pe ambele.

```python
# Bob receives the ciphertext and signature
received_ciphertext = ciphertext
received_signature = signature

# Send signature and ciphertext here
print("Sending ciphertext and signature.....")
```

De partea lui Bob, primul pas este să verifice <DefinitionTooltip definition="Integrity is maintained when a message is protected against unauthorized change.">integritatea</DefinitionTooltip> și <DefinitionTooltip definition="Authenticity is the proof that a message is from the source it claims to be from.">autenticitatea</DefinitionTooltip> textului cifrat primit. Pentru aceasta, Bob creează un hash al textului cifrat primit folosind același algoritm de hashing utilizat de Alice.

```python
# Bob creates a hash of the ciphertext using the same algorithm used by Alice
digest = hashes.Hash(hashes.SHA256())
digest.update(received_ciphertext)
hash_to_verify = digest.finalize()

print("hash to verify: ", hash_to_verify)
```

Apoi, Bob decriptează semnătura primită folosind cheia publică a lui Alice. Deoarece Alice și-a folosit cheia privată pentru a crea semnătura, Bob o poate decripta cu cheia publică a lui Alice. Semnătura decriptată nu este altceva decât un hash al textului cifrat creat de Alice. Dacă hash-ul creat de Bob coincide cu semnătura decriptată, atunci Bob a verificat că textul cifrat primit nu a fost alterat și că Alice este cea care a semnat textul cifrat.

În codul Python de mai jos, aceste operații sunt combinate într-o funcție utilitară numită ``verify``, oferită de un obiect asociat cheii publice a lui Alice.

```python
from cryptography.exceptions import InvalidSignature

def is_signature_valid(public_key, signature, data_hash):
    try:
        public_key.verify(
            signature,
            data_hash,
            padding.PSS(
                mgf=padding.MGF1(hashes.SHA256()), salt_length=padding.PSS.MAX_LENGTH
            ),
            Prehashed(hashes.SHA256()),
        )
        return True
    except InvalidSignature:
        return False

if is_signature_valid(alice_public_key, received_signature, hash_to_verify):
    print("The signature is valid.")
else:
    print("The signature is not valid.")
```

După ce a verificat <DefinitionTooltip definition="Integrity is maintained when a message is protected against unauthorized change.">integritatea</DefinitionTooltip> și <DefinitionTooltip definition="Authenticity is the proof that a message is from the source it claims to be from.">autenticitatea</DefinitionTooltip> textului cifrat primit, Bob îl poate decripta folosind cheia sa privată, deoarece Alice a creat textul cifrat cu cheia publică a lui Bob.

```python
# Bob decrypts the message using his private key
decrypted_message = bob_private_key.decrypt(
    received_ciphertext,
    padding.OAEP(
        mgf=padding.MGF1(algorithm=hashes.SHA256()),
        algorithm=hashes.SHA256(),
        label=None,
    ),
)

print("Decrypted message:", decrypted_message.decode())
```

În scenariul de semnătură digitală de mai sus, orice parte — nu doar Bob — poate verifica că Alice este expeditoarea textului cifrat, deoarece toată lumea poate accesa cheia publică a lui Alice, textul cifrat și semnătura digitală. Mai mult, după trimiterea textului cifrat și a semnăturii, Alice nu poate nega ulterior că a făcut-o, deoarece semnătura poate fi decriptată la un hash semnificativ folosind doar cheia sa publică. Aceasta stabilește <DefinitionTooltip definition="Non-repudiation refers to the ability to prove that a specific party has sent a message. In symmetric key cryptography, since the same key is used for both encryption and decryption, it is not possible to determine which party has created a particular ciphertext. In contrast, asymmetric key cryptography provides non-repudiation through the use of digital signatures.">non-repudierea</DefinitionTooltip>.

Prin facilitarea distribuirii securizate a cheilor și prin susținerea non-repudierii, criptografia cu cheie publică pune o bază solidă pentru comunicarea digitală securizată.

## Spargerea RSA {#breaking-rsa}

Utilitatea și securitatea algoritmului <DefinitionTooltip definition="A public-key cryptosystem named after Ron Rivest, Adi Shamir, and Leonard Adleman. Developed in 1977.">RSA</DefinitionTooltip> descris mai sus se bazează pe două ipoteze matematice:

1. Găsirea <DefinitionTooltip definition="Given an integer $a$, and modulus $m$, the remainder from ax/m is 1. This is only the case for coprimes.">inversului multiplicativ modular</DefinitionTooltip> $d$ știind că avem acces doar la $(e, n)$ este computațional infezabilă.

2. În contextul RSA, operația de exponențiere modulară se comportă ca o <DefinitionTooltip definition="A one-way function where it is easy to compute an output from the inputs, but hard to determine the inputs from the output.">funcție trapă unidirecțională</DefinitionTooltip>. Când este folosită pentru criptare, produce text cifrat de neînțeles, iar fără acces la cheia privată, inversarea operației pentru a recupera textul clar din textul cifrat nu este fezabilă. Cu toate acestea, cu acces la cheia privată, care acționează ca o trapă, textul cifrat poate fi decriptat cu ușurință.

Cel mai proeminent atac asupra algoritmului <DefinitionTooltip definition="A public-key cryptosystem named after Ron Rivest, Adi Shamir, and Leonard Adleman. Developed in 1977.">RSA</DefinitionTooltip> urmărește să submineze ipoteza 1 prin recuperarea eficientă a numărului cheii private $d$ via factorizarea <DefinitionTooltip definition="The modulus is the remainder left over after dividing by the value specified. modulo is the operation of doing this.">modulului</DefinitionTooltip> $n$. Așa cum va fi ilustrat mai jos, este ușor să recuperezi $d$ dacă ai acces la <DefinitionTooltip definition="prime factors of a number n, are those primes that are multiplied together to make n.">factorii primi</DefinitionTooltip> $p$ și $q$ ai lui $n$ sau la totientul <DefinitionTooltip definition="The greek symbol, phi, often used to indicate Euler's totient.">$φ$</DefinitionTooltip>$(n)$. Reamintim că $p$, $q$ și <DefinitionTooltip definition="The greek symbol, phi, often used to indicate Euler's totient.">$φ$</DefinitionTooltip>$(n)$ sunt ținute secrete în timpul generării cheilor și eliminate după aceea. O terță parte care recuperează aceste informații folosind un calculator clasic sau cuantic descoperă practic cheia privată, spărgând RSA. Prin urmare, factorizarea în numere prime este primitiva computațională esențială necesară pentru spargerea RSA.

### Calculul clasic și RSA {#classical-computing-and-rsa}

Se știe că factorizarea în numere prime a numerelor întregi mari prezintă o scalare <DefinitionTooltip definition="Growing faster than a polynomial; for example, an exponential function.">super-polinomială</DefinitionTooltip> sau <DefinitionTooltip definition="Scaling slower than exponential. For example polynomial and linear scaling.">subexponențială</DefinitionTooltip> pe calculatoarele clasice. În prezent, [cele mai mari numere întregi factorizate pe hardware clasic](https://en.wikipedia.org/w/index.php?title=RSA_Factoring_Challenge&oldid=1163481797) se situează în intervalul de 829 de biți sau 250 de cifre zecimale. Cu toate acestea, în viitorul previzibil, factorizarea numerelor întregi de 2048 de biți cu magnitudini în intervalul $10^{617}$ este în prezent considerată infezabilă pe sistemele clasice, sugerând o viabilitate continuă. Apariția calculatoarelor cuantice, însă, anulează această evaluare.

### Algoritmul cuantic al lui Shor și RSA {#shors-quantum-algorithm-and-rsa}

Probabil cel mai cunoscut algoritm cuantic din ziua de azi este <DefinitionTooltip definition="A quantum algorithm which can find prime factors efficiently. Developed by the mathematician, Peter Shor, in 1994.">algoritmul lui Shor</DefinitionTooltip> pentru găsirea factorilor primi ai numerelor întregi. Când [a fost introdus de Peter Shor în 1994](https://doi.org/10.1109/SFCS.1994.365700), a fost recunoscut ca primul algoritm cuantic care oferea o <DefinitionTooltip definition="Growing faster than a polynomial; for example, an exponential function.">accelerare super-polinomială</DefinitionTooltip> față de algoritmii clasici pe o problemă de mare importanță practică, și anume factorizarea în numere prime.

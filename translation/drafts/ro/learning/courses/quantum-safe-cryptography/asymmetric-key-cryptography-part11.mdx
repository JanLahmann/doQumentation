    m4 = pow(2, -1, 6)

Alice poate calcula acum semnătura.

- $r = (g^{k} \mod p) \mod\ q$
- $s = (k^{-1} (H(m) + xr)) \mod q$

Reține că există anumite condiții particulare care ne vor obliga să alegem o altă valoare pentru $k$ în cazul în care $r$ sau $s$ se calculează la $0$. În acest caz vom genera o nouă valoare și vom repeta procesul.

```python
# Start an infinite loop; we will 'break' out of it once a valid signature is found.
while True:
    k = randint(1, q - 1)  # Should be different for every message
    print("Using random k =", k)

    r = pow(g, k, p) % q
    # If r is 0, the value is invalid. Try again with a new k.
    if r == 0:
        print(f"{k} is not a good random value to use to calculate r. Trying another k")
        continue

    s = (pow(k, -1, q) * (alice_hash + alice_private_key * r)) % q
    # If s is 0, the value is also invalid. Try again with a new k.
    if s == 0:
        print(f"{k} is not a good random value to use to calculate s. Trying another k")
        continue

    # If we've reached this point, both r and s are valid. Break the loop.
    signature = (r, s)
    print(f"Alice's signature is : {(r,s)}")
    break

# After the loop, the valid r and s values can be used here.
# print(f"Generated Signature -> r: {r}, s: {s}")
```

Alice transmite mesajul și semnătura sa receptorului sau verificatorului, Bob.

Bob a interceptat anterior cheia publică a lui Alice și poate acum să verifice semnătura pentru a-i autentifica mesajul.

Pentru a face acest lucru, el efectuează câteva verificări, apoi regenerează un hash al mesajului transmis de Alice și calculează cantitățile auxiliare $w, u_1, u_2$ și $v$.
- $0 < r < q$
- $0 < s < q$
- $w = s^{-1} \mod\ q$
- $u_{1} = H(m) . w \mod\ q$
- $u_{2} = r . w \mod\ q$
- $v = (g^{u1}y^{u2} \mod\ p) \mod\ q$

În final, Bob verifică dacă $v$ este egal cu $r$. Dacă sunt egali, semnătura este verificată.

```python
# Bob re-generates message hash using Alice's broadcast message
bob_hash = mock_hash_func(alice_message)

# Bob computes auxiliary quantities w (using modular inverse), u1, u2 and v
w = (pow(s, -1, q)) % q
u1 = (bob_hash * w) % q
u2 = (r * w) % q
v = ((g**u1 * alice_public_key**u2) % p) % q

if v == r:
    print("Signature is valid!")
else:
    print("Signature is invalid!")
```

### Securitatea DSA {#security-of-the-dsa}

În calculul clasic, securitatea DSA poate fi influențată de mai mulți factori:

1. **Dimensiunea cheii**: Ca întotdeauna, dimensiunea cheii oferă o protecție de bază împotriva atacurilor prin forță brută. Aplicațiile moderne care utilizează DSA folosesc chei de cel puțin 2048 de biți.

2. **Calitatea lui $k$**: În DSA, fiecare semnătură are nevoie de o valoare $k$ unică, aleatoare și secretă (a se vedea mai sus). Unicitatea, entropia și confidențialitatea lui $k$ sunt esențiale, iar o slăbiciune în oricare dintre aceste aspecte ar putea duce la compromiterea cheii private $x$. Generatoarele de numere aleatoare folosite pentru generarea lui $k$ trebuie să fie ele însele securizate.

3. **Rezistența funcției hash**: Deoarece DSA folosește o funcție hash ca parte a procesului de generare și verificare a semnăturilor, o funcție hash slabă ar putea compromite securitatea semnăturii digitale. De exemplu, utilizarea SHA-1 cu DSA este depreciată și se recomandă SHA-2 sau o versiune superioară.

Pe lângă acestea, o implementare robustă a DSA ar trebui să se apere și împotriva altor tipuri de atacuri care vizează criptografia cu cheie asimetrică, așa cum s-a prezentat anterior.

```python
# Simple message to encode
msg = 9

# encrypt then decrypt
enc_msg = encrypt(msg)
dec_msg = decrypt(enc_msg)

print("Original Message:", msg)
print("Encrypted Message:", enc_msg)
print("Decrypted Message:", dec_msg)
```

Deși numerele întregi mici folosite mai sus sunt utile pentru a ilustra cu ușurință ideile de bază din algoritmul RSA, în aplicațiile reale RSA necesită utilizarea unor numere întregi foarte mari. De exemplu, RSA de 2048 de biți implică utilizarea unui <DefinitionTooltip definition="The modulus is the remainder left over after dividing by the value specified. modulo is the operation of doing this.">modul</DefinitionTooltip> $n$ de 2048 de biți, ai cărui echivalenți zecimali sunt în jur de 10$^616$. Aceste numere cu adevărat enorme sunt necesare pentru securitatea practică a RSA.
#### Schimbul de chei simetrice cu RSA {#symmetric-key-exchange-with-rsa}
Așa cum s-a discutat anterior, <DefinitionTooltip definition="Asynchronous key encryption.">AKC</DefinitionTooltip> le permite celor două părți care doresc să comunice să stabilească în mod securizat un *secret comun*, care poate fi folosit, de exemplu, drept cheie secretă pentru criptarea simetrică ulterioară a textului simplu în volum mare.

Să luăm în considerare următorul scenariu. Alice și Bob vor să folosească <DefinitionTooltip definition="Symmetric key encryption.">SKC</DefinitionTooltip> pentru a cripta și decripta mesaje. Totuși, înainte ca acest proces să poată fi inițializat, trebuie să cadă de acord asupra unei chei secrete comune. O opțiune este ca una dintre părți — să spunem, Alice — să genereze o cheie secretă și apoi să i-o transmită în mod securizat lui Bob. Pentru a realiza acest transfer securizat, Alice decide să folosească RSA drept <DefinitionTooltip definition="Key encapsulation mechanism. Secures a symmetric key by use of asymmetric key encryption. A simple approach is to encrypt a random symmetric key using a public key, but more advanced & secure kems are available, and new ones continually developed.">mecanism de încapsulare a cheii</DefinitionTooltip> (KEM).

Aceasta implică următorii pași:

- Mai întâi, Alice generează o cheie simetrică aleatorie, pe care intenționează să o partajeze cu Bob.
- Apoi, Bob generează o pereche de chei asimetrice și pune la dispoziție cheia sa publică pe un canal potrivit.
- În continuare, Alice folosește cheia publică a lui Bob pentru a cripta cheia simetrică, încapsulând-o astfel într-un ciphertext.
- Apoi, Alice transmite ciphertext-ul pe un canal fiabil, dar nu neapărat securizat.
- În final, Bob primește ciphertext-ul și îl decriptează folosind cheia sa privată. Acum are acces la cheia simetrică generată de Alice.

##### Exemplu de schimb de chei bazat pe padding în Python {#padding-based-key-exchange-example-in-python}
Un flux de lucru practic care utilizează RSA pentru schimbul de chei non-interactiv bazat pe padding este ilustrat acum folosind biblioteca Python ``cryptography``.

Importă modulele necesare din biblioteca Python ``cryptography``. Dacă este necesar, această bibliotecă poate fi instalată folosind comanda ``pip install cryptography``.

Apoi, Alice generează o cheie secretă aleatorie, pe care intenționează să i-o transmită lui Bob.

```python
# pip install cryptography
from cryptography.hazmat.primitives import serialization
from cryptography.hazmat.primitives.asymmetric import rsa, padding
from cryptography.fernet import Fernet
from cryptography.hazmat.primitives import hashes

symmetric_key = Fernet.generate_key()
print(f"\nSymmetric key generated by Alice: {symmetric_key}")
```

Folosind modulul ``rsa`` din biblioteca ``cryptography``, Bob generează o pereche de chei și transmite apoi cheia sa publică. Oricine poate intercepta cheia publică și citi numerele publice $(e,n)$ care formează cheia.

```python
# Bob generates a 2048-bit RSA key pair
bob_private_key = rsa.generate_private_key(public_exponent=65537, key_size=2048)
bob_public_key = bob_private_key.public_key()
print(f"Public key broadcast by Bob: {bob_public_key}")
print(f"\nPublic numbers in Bobs' public key: {bob_public_key.public_numbers()}")
```

În acest moment, presupunem că Alice a primit cheia publică transmisă de Bob. ``symmetric_key``-ul lui Alice de mai sus poate fi acum criptat folosind cheia publică a lui Bob pentru a produce ciphertext-ul. Într-un scenariu realist, Alice va folosi și metode suplimentare de <DefinitionTooltip definition="The addition of data to the beginning, middle, or end of a message. Often used in block ciphers to fill out blocks with insufficient data up to the required size.">padding</DefinitionTooltip>, precum [OAEP](https://en.wikipedia.org/w/index.php?title=Optimal_asymmetric_encryption_padding&oldid=1163481577), pentru a asigura <DefinitionTooltip definition="With only the ciphertext, and any relevant public information (such as public key for asymmetric key cryptography), and adversary cannot retrieve even partial information from the plaintext.">securitatea semantică</DefinitionTooltip> a comunicațiilor sale cu Bob.

```python
# Encryption
ciphertext = bob_public_key.encrypt(
    symmetric_key,
    padding.OAEP(
        mgf=padding.MGF1(algorithm=hashes.SHA256()),
        algorithm=hashes.SHA256(),
        label=None,
    ),
)

print("Ciphertext:", ciphertext)
```

Alice transmite apoi ciphertext-ul pe un canal deschis, sigură că numai Bob, cu cheia privată corespunzătoare, va putea să îl decripteze. Presupunem că Bob a primit ciphertext-ul și îl poate decripta acum folosind cheia sa privată confidențială.

```python
# Bob decrypts ciphertext to access the symmetric key
decrypted_symmetric_key = bob_private_key.decrypt(
    ciphertext,
    padding.OAEP(
        mgf=padding.MGF1(algorithm=hashes.SHA256()),
        algorithm=hashes.SHA256(),
        label=None,
    ),
)

print("Decrypted key:", decrypted_symmetric_key)
assert decrypted_symmetric_key == symmetric_key
```

În acest moment, atât Alice cât și Bob au acces la cheia simetrică secretă, pe care o pot folosi în aplicații <DefinitionTooltip definition="Symmetric key encryption.">SKC</DefinitionTooltip>.
##### Simularea unui Mecanism de Încapsulare a Cheii cu RSA în Python {#simulating-a-key-encapsulation-mechanism-with-rsa-in-python}
În următorul flux de lucru, ilustrăm utilizarea RSA pentru a simula un Mecanism de Încapsulare a Cheii (KEM), prin care un mesaj secret aleatoriu suficient de lung este schimbat în mod securizat și apoi convertit într-un secret comun de lungimea potrivită folosind un KDF.

Din nou, Alice și Bob vor să stabilească un secret comun în mod non-interactiv, iar Alice este cea care decide ce secret să folosească.

Începem prin a importa câteva biblioteci Python necesare.

Bob generează apoi perechea sa de chei RSA și transmite cheia sa publică.

```python
from cryptography.hazmat.primitives.asymmetric import rsa, padding
from cryptography.hazmat.primitives.kdf.hkdf import HKDF
from cryptography.hazmat.primitives import hashes
from os import urandom

# Bob's RSA key pair
private_key_Bob = rsa.generate_private_key(public_exponent=65537, key_size=2048)
public_key_Bob = private_key_Bob.public_key()

print("Bob's private and public keys created")
```

Alice generează mai întâi un secret aleatoriu lung din care va fi derivat în cele din urmă secretul comun. Într-un [KEM](https://en.wikipedia.org/w/index.php?title=Key_encapsulation_mechanism&oldid=1110687965) pur, secretul lung va fi un element aleatoriu din structura algebrică ce stă la baza criptosistemului. În cazul RSA de 2048 de biți, acesta ar fi un număr întreg aleatoriu modulo modulul RSA de 2048 de biți. Un astfel de KEM pur nu necesită padding suplimentar, dar în acest exemplu simulăm doar un KEM folosind RSA, iar biblioteca `cryptography` impune utilizarea padding-ului la criptarea cu RSA. Astfel, vom folosi un secret lung oarecum mai scurt, care este totuși mult mai lung decât o cheie AES standard de 256 de biți.

```python
Alice_long_secret = urandom(160)  # A 160 byte or 1280 bit random message
print("Alice's secret created")
```

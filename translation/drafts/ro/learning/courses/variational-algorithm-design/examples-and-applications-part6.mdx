### Modificarea betas {#change-betas}
După cum am menționat în lecția anterioară, valorile din $\vec{\beta}$ ar trebui să fie mai mari decât diferența dintre valorile proprii. Să vedem ce se întâmplă când această condiție nu este îndeplinită, cu $\hat{O}_2$

$$
\hat{O}_2 = 2 II - 3 XX + 2 YY - 4 ZZ
$$

cu valorile proprii

$$
\left\{
\begin{array}{c}
\lambda_0 = -7 \\
\lambda_1 = 3\\
\lambda_2 = 5 \\
\lambda_3 = 7
\end{array}
\right\}
$$

```python
from qiskit.quantum_info import SparsePauliOp

k = 4
betas = np.ones(3)
x0 = np.zeros(8)
```

```python
from scipy.optimize import minimize

prev_states = []
prev_opt_parameters = []
eigenvalues = []

realbackend = 0

for step in range(1, k + 1):
    if step > 1:
        prev_states.append(ansatz.assign_parameters(prev_opt_parameters))

    result = minimize(
        cost_func_vqd,
        x0,
        args=(
            ansatz,
            prev_states,
            step,
            betas,
            estimator,
            sampler,
            observable_2,
            realbackend,
            None,
        ),
        method="COBYLA",
        options={"tol": 0.01, "maxiter": 200},
    )
    print(result)

    prev_opt_parameters = result.x
    eigenvalues.append(result.fun)
```

De data aceasta, optimizatorul returnează aceeași stare $|\phi_0\rangle = \frac{1}{\sqrt{2}}(|00\rangle + |11\rangle)$ ca soluție propusă pentru toate stările proprii — ceea ce este evident greșit. Aceasta se întâmplă deoarece betas erau prea mici pentru a penaliza starea proprie minimă în funcțiile de cost successive. Prin urmare, ea nu a fost exclusă din spațiul efectiv de căutare în iterațiile ulterioare ale algoritmului și a fost întotdeauna aleasă ca cea mai bună soluție posibilă.

Îți recomandăm să experimentezi cu valorile din $\vec{\beta}$ și să te asiguri că sunt mai mari decât diferența dintre valorile proprii.

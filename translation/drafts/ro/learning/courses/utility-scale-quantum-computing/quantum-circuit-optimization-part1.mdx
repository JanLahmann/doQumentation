```python
from qiskit_ibm_runtime.fake_provider import FakeBrisbane
```

```python
num_qubits = 5
```

### 2.1 Nivelul de optimizare
Există 4 `optimization_level`-uri disponibile, de la 0 la 3. Cu cât nivelul de optimizare este mai ridicat, cu atât se depune mai mult efort de calcul pentru a optimiza Circuit-ul. Nivelul 0 nu efectuează nicio optimizare și face doar minimul necesar pentru ca Circuit-ul să poată rula pe Backend-ul selectat. Nivelul 3 depune cel mai mare efort (și, de obicei, cel mai mult timp de execuție) pentru a încerca să optimizeze Circuit-ul. Nivelul 1 este nivelul de optimizare implicit.
Transpilăm Circuit-ul fără optimizare (`optimization_level=0`) și cu optimizare (`optimization_level=2`).
Observăm o diferență mare în lungimea circuitelor transpilate.

```python
pm0 = generate_preset_pass_manager(
    optimization_level=0, backend=backend, seed_transpiler=777
)
pm2 = generate_preset_pass_manager(
    optimization_level=2, backend=backend, seed_transpiler=777
)
circ0 = pm0.run(ghz_circ)
circ2 = pm2.run(ghz_circ)
print("optimization_level=0:")
display(circ0.draw("mpl", idle_wires=False, fold=-1))
print("optimization_level=2:")
display(circ2.draw("mpl", idle_wires=False, fold=-1))
```

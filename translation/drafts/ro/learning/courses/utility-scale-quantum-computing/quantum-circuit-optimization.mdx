### 2.1 Nivelul de optimizare
Există 4 `optimization_level`-uri disponibile, de la 0 la 3. Cu cât nivelul de optimizare este mai ridicat, cu atât se depune mai mult efort de calcul pentru a optimiza Circuit-ul. Nivelul 0 nu efectuează nicio optimizare și face doar minimul necesar pentru ca Circuit-ul să poată rula pe Backend-ul selectat. Nivelul 3 depune cel mai mare efort (și, de obicei, cel mai mult timp de execuție) pentru a încerca să optimizeze Circuit-ul. Nivelul 1 este nivelul de optimizare implicit.
Transpilăm Circuit-ul fără optimizare (`optimization_level=0`) și cu optimizare (`optimization_level=2`).
Observăm o diferență mare în lungimea circuitelor transpilate.

```python
pm0 = generate_preset_pass_manager(
    optimization_level=0, backend=backend, seed_transpiler=777
)
pm2 = generate_preset_pass_manager(
    optimization_level=2, backend=backend, seed_transpiler=777
)
circ0 = pm0.run(ghz_circ)
circ2 = pm2.run(ghz_circ)
print("optimization_level=0:")
display(circ0.draw("mpl", idle_wires=False, fold=-1))
```

Noua sinteză produce un Circuit mai puțin adânc. De ce?

Acest lucru se datorează faptului că noul Circuit poate fi plasat pe Qubit-uri conectate liniar, deci și pe graful de cuplare de tip heavy-hexagon al IBM&reg; Brisbane, în timp ce Circuit-ul original necesită conectivitate în formă de stea (un nod de grad 4) și, prin urmare, nu poate fi plasat pe graful de cuplare heavy-hex, ale cărui noduri au gradul cel mult 3. Drept urmare, Circuit-ul original necesită rutarea Qubit-urilor, ceea ce adaugă porți SWAP și crește numărul de Gate-uri.

Ceea ce am făcut în noul Circuit poate fi privit ca o sinteză manuală a Circuit-ului „conștientă de constrângerile de cuplare". Cu alte cuvinte: rezolvăm manual sinteza Circuit-ului și maparea Circuit-ului în același timp.

În general, sinteza Circuit-ului depinde de aplicație și este prea dificil pentru un program software să acopere toate aplicațiile posibile. Qiskit Transpiler nu are, din întâmplare, funcții pentru sinteza Circuit-ului de pregătire a stării GHZ. Într-un astfel de caz, sinteza manuală a Circuit-ului, așa cum este prezentată mai sus, merită luată în considerare.
În această secțiune, analizăm în detaliu modul în care funcționează Qiskit Transpiler, folosind următorul Circuit exemplu simplificat.

```python
# Build a toy example circuit
from math import pi
import itertools
from qiskit.circuit import QuantumCircuit
from qiskit.circuit.library import excitation_preserving

circuit = QuantumCircuit(4, name="Example circuit")
circuit.append(excitation_preserving(4, reps=1, flatten=True), range(4))
circuit.measure_all()

value_cycle = itertools.cycle([0, pi / 4, pi / 2, 3 * pi / 4, pi, 2 * pi])
circuit.assign_parameters(
    [x[1] for x in zip(range(len(circuit.parameters)), value_cycle)], inplace=True
)
circuit.draw("mpl")
```

```python
from qiskit.transpiler import PassManager
from qiskit.transpiler.passes import (
    Collect2qBlocks,
    ConsolidateBlocks,

)

# Collect 2q blocks and consolidate to unitary when we expect that we can reduce the 2q gate count for that unitary
consolidate_pm = PassManager(
    [
        Collect2qBlocks(),
        ConsolidateBlocks(target=backend.target),

)
```

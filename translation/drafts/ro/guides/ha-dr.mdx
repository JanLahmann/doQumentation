# Înțelegerea disponibilității ridicate și a recuperării în caz de dezastru pentru Qiskit Runtime

Disponibilitatea ridicată (HA) reprezintă capacitatea unui serviciu de a rămâne operațional și accesibil în prezența unor defecțiuni neașteptate. Recuperarea în caz de dezastru este procesul de readucere a instanței de serviciu la o stare funcțională. Clienții Qiskit Runtime nu sunt responsabili pentru planificarea sau acțiunile de recuperare în caz de dezastru.

Qiskit Runtime este un serviciu regional care îndeplinește [Obiectivele de nivel de serviciu (SLO)](https://cloud.ibm.com/docs/resiliency?topic=resiliency-slo) definite cu orice plan plătit. Pentru mai multe informații, consultă [Acordul de nivel de serviciu (SLA)](https://www.ibm.com/support/customer/csol/terms/?id=i126-9268&lc=en) pentru IBM Cloud&reg;. Pentru mai multe informații despre regiunile IBM Cloud și centrele de date disponibile pentru Qiskit Runtime, consultă [Disponibilitatea serviciilor și infrastructurii după locație.](https://cloud.ibm.com/docs/overview?topic=overview-services_region)

## Arhitectura de disponibilitate ridicată {#high-availability-architecture}

### Caracteristici de disponibilitate ridicată {#high-availability-features}

Acest serviciu este provizionat în trei zone dintr-o regiune cu mai multe zone, fără niciun punct unic de defecțiune. Solicitările API sunt direcționate printr-un echilibrator de sarcină către nodurile instanței HA, fiecare aflat într-o zonă diferită.

Dacă un nod sau o zonă a instanței HA eșuează, serviciul continuă să proceseze solicitările API prin nodurile instanței HA rămase. Nu apare nicio degradare vizibilă a serviciului în timpul unei defecțiuni zonale.

Este disponibil în următoarele regiuni:

- Washington DC (us-east)
- Frankfurt (eu-de)

# Plugin SPANK pentru QRMI

[Plugin-ul SPANK](https://github.com/qiskit-community/spank-plugins/) pentru [Interfața de Gestionare a Resurselor Cuantice (QRMI)](/guides/qrmi) este utilizat pentru a configura accesul la resursele cuantice din joburile utilizatorilor într-un mediu de calcul administrat de managerul de sarcini [Slurm](https://slurm.schedmd.com/quickstart.html). Acesta gestionează dobândirea și eliberarea accesului la resursele cuantice și setează variabilele de mediu necesare pentru executarea sarcinilor de lucru cuantice. Resursele cuantice disponibile sunt specificate într-un fișier `qrmi_config.json`, gestionat de un administrator.

Odată instalat, acest plugin înregistrează următoarea opțiune. Un utilizator Slurm poate specifica ce resurse cuantice sunt folosite pentru scriptul jobului Slurm.

```bash
--qpu=names             Comma separated list of QPU resources to use.
```

De exemplu,
```bash
#!/bin/bash

#SBATCH --job-name=sampler_job
#SBATCH --ntasks=1
#SBATCH --cpus-per-task=1
#SBATCH --qpu=ibm_quebec,ibm_sherbrooke

# Your script goes here...
```

## Cerințe și configurare {#requirements-and-configuration}

Următoarele instrumente sunt necesare pentru mediul de compilare:
* [Compilatorul Rust](https://www.rust-lang.org/tools/install) 1.86 sau superior
* Un compilator C: de exemplu, GCC (gcc) pe Linux și Clang (clang-tools-extra) pentru ținte necunoscute Rust/compilări încrucișate. QRMI și plugin-ul său SPANK sunt compatibile cu un compilator conform standardului C11
* make/cmake (RPM-urile make/cmake pentru OS-uri compatibile RHEL)
* openssl (RPM-ul openssl-devel pentru OS-uri compatibile RHEL)
* zlib (RPM-ul zlib-devel pentru OS-uri compatibile RHEL)
* Fișierele antet Slurm (slurm/slurm.h și altele) trebuie să fie disponibile pe gazda ta

Mediul de rulare necesită:
* gcc (RPM-ul libgcc pentru OS-uri compatibile RHEL)
* openssl (RPM-ul openssl-libs pentru OS-uri compatibile RHEL)
* zlib (RPM-ul zlib pentru OS-uri compatibile RHEL)

### Configurarea resurselor cuantice disponibile {#configure-available-quantum-resources}

Depozitul conține un exemplu de configurare ([`qrmi_config.json.example`](https://github.com/qiskit-community/spank-plugins/blob/main/plugins/spank_qrmi/qrmi_config.json.example)).

Array-ul `resources` conține un set de resurse cuantice disponibile. Fiecare definiție de resursă cuantică conține următoarele:

| Proprietate | Descriere |
| ---- | ---- |
| name | Numele resursei cuantice (de exemplu, numele Backend-ului) |
| type | Tipul resursei (`direct-access`, `qiskit-runtime-service` și `pasqal-cloud`) |
| environment | Un set de variabile de mediu pentru a lucra cu QRMI. Implementările actuale presupun că endpoint-ul API și credențialele sunt specificate prin setarea variabilelor de mediu |

Dacă un utilizator specifică o resursă cu opțiunea `--qpu` care nu este definită în fișierul `qrmi_config.json`, specificația va fi ignorată.

Dacă utilizatorul setează el însuși variabilele de mediu necesare pentru executarea jobului, nu este necesar să le specifice în acest fișier. În acest caz, proprietatea environment va fi `{}`.

<Admonition type="note">
Dacă folosești o resursă QPU cu tipul de resursă `qiskit-runtime-service`, folosește un cont care suportă [deschiderea unei sesiuni](/guides/run-jobs-session#open-a-session), cum ar fi un cont Premium Plan.

Dacă folosești un cont care nu suportă deschiderea unei sesiuni, cum ar fi un cont Open Plan, adaugă `QRMI_IBM_QRS_SESSION_MODE="batch"` în lista de variabile de mediu din qrmi_config.json ca soluție alternativă.
</Admonition>

## Instalare {#installation}

Rulează build-ul folosind `make` și `cmake`:
```shell-session
. ~/.cargo/env
mkdir build
cd build
cmake ..
make
```

În mod implicit, fișierul `CMakeLists.txt` se așteaptă ca fișierul antet Slurm (`slurm.h`) să fie localizat în `/usr/include/slurm`, dar aceasta poate fi personalizată după cum se arată mai jos.

```shell-session
SLURM_INCLUDE_DIRS=<directory containing slurm/slurm.h> cmake ..
```

Dacă pasul de build de mai sus are succes, o bibliotecă partajată Linux numită `spank_qrmi.so` va fi creată în directorul `build/`.

În plus, adaugă următoarea linie în `/etc/slurm/plugstack.conf` pe nodurile unde este instalat acest plugin (rețineți că un administrator trebuie să creeze fișierul `qrmi_config.json` și să specifice calea ca argument al plugin-ului, ca în exemplul următor):

```bash
optional /usr/lib64/slurm/spank_qrmi.so /etc/slurm/qrmi_config.json
```

<Admonition type="note">
Poți folosi argumentele opționale disponibile pentru a adăuga variabile de mediu la procesul Slurm unde este încărcat plugin-ul SPANK. Formatul pentru specificarea variabilelor de mediu este definit după cum urmează.
```bash
--env:{variable name}={value}
```
De exemplu, când interacționezi cu resursele cuantice printr-un proxy HTTP, sunt necesare variabilele de mediu `http_proxy`, `https_proxy` și `no_proxy`. Acestea pot fi adăugate după cum se arată mai jos.
```bash
optional /usr/lib64/slurm/spank_qrmi.so /etc/slurm/qrmi_config.json --env:http_proxy=http://192.168.1.128:3128 --env:https_proxy=http://192.168.1.128:3128
```
</Admonition>
Pentru nodul alocator, nu trebuie să specifici calea către qrmi_config.json, ca în exemplul următor.

```bash
optional /usr/lib64/slurm/spank_qrmi.so
```

Tabelul următor arată la care noduri de context Slurm trebuie copiate aceste fișiere. Pentru mai multe detalii despre fiecare context, consultă [documentația](https://slurm.schedmd.com/spank.html#SECTION_SPANK-PLUGINS) pentru plugin-ul SPANK.
| Fișiere | Contexte Slurm |
| ------- | ------- |
| `plugstack.conf` | **local**, **remote**, **allocator**, **slurmd** și **job_script** |
| `qrmi_config.json` | **remote** (Noduri de calcul) |
| `spank_qrmi.so` | **allocator** și **remote** (Noduri de autentificare și noduri de calcul) |

<Admonition type="note">
Odată ce `plugstack.conf` este actualizat, plugin-urile SPANK vor fi încărcate la runtime în timpul următoarei lansări a jobului, ceea ce înseamnă că administratorii nu trebuie să repornească clusterul Slurm.
</Admonition>

Odată ce finalizezi instalarea, ar trebui să găsești opțiunea `--qpu=names` în mesajul de ajutor `sbatch`.

```bash
Options provided by plugins:
      --qpu=names             Comma-separated list of QPU resources to use.
```

|  Opțiune `sbatch`/`srun` | Nivel jurnal Slurm (SRUN_DEBUG) | Nivel jurnal QRMI (RUST_LOG) |
| ---- | ---- | ---- |
| (implicit) | 3 | info |

| `--verbose` | 4 | debug |
| `-vv` sau mai mult | 5 | trace |

## Considerații pentru mai multe QPU-uri {#considerations-for-multiple-qpus}

La runtime, fiecare instanță QRMI este legată de o singură resursă QPU. Pentru a permite utilizarea mai multor resurse cuantice într-un singur script de job, acest plugin setează variabilele de mediu cu numele resursei ca prefix. De exemplu, dacă este specificat `--qpu=qpu1,qpu2`, variabilele de mediu vor fi setate după cum urmează:

```bash
qpu1_QRMI_IBM_DA_ENDPOINT=http://test1
qpu2_QRMI_IBM_DA_ENDPOINT=http://test2
```

Aceasta asigură că fiecare instanță QRMI operează cu parametrii de configurare setați pentru resursa sa respectivă în timpul execuției jobului Slurm.

Setările variabilelor de mediu de mai sus se aplică doar joburilor în care este specificată opțiunea `--qpu=names`.

Acest plugin setează, de asemenea, următoarele două variabile de mediu, la care se face referire în codul primitivelor QRMI.

| Variabilă de mediu | Descriere |
| ---- | ---- |
| `SLURM_JOB_QPU_RESOURCES` | Listă separată prin virgule de resurse QPU de utilizat la runtime. Resursele nedocumentate vor fi filtrate. De exemplu, `qpu1,qpu2` |
| `SLURM_JOB_QPU_TYPES` | Listă separată prin virgule de tipuri de resurse (`direct-access`, `qiskit-runtime-service` și `pasqal-cloud`). De exemplu, `direct-access,pasqal-cloud` |

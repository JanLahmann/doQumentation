## Suport Qiskit Runtime

Deși desenatorul de cronologie integrat în Qiskit este util pentru Circuit-uri statice, este posibil să nu reflecte cu acuratețe cronologia [circuit-urilor dinamice](/guides/classical-feedforward-and-control-flow) din cauza operațiunilor implicite, cum ar fi difuzarea și determinarea ramurilor. Ca parte a suportului pentru circuit-uri dinamice, Qiskit Runtime returnează informațiile precise despre cronologia Circuit-ului în rezultatele jobului atunci când este solicitat.

<Admonition type="note" title="Note">
- Aceasta este o funcție experimentală. Se află în stadiu de previzualizare și poate suferi modificări.
- Această funcție se aplică doar joburilor Qiskit Runtime Sampler.
- Deși timpul total al Circuit-ului este returnat în metadatele „compilation", acesta NU este timpul utilizat pentru facturare (quantum time).
</Admonition>
### Activarea recuperării datelor de cronologie

Pentru a activa recuperarea datelor de cronologie, setează indicatorul experimental `scheduler_timing` la `True` când rulezi jobul primitiv.

```python
from qiskit_ibm_runtime import QiskitRuntimeService, SamplerV2

service = QiskitRuntimeService()
backend = service.least_busy(operational=True, simulator=False)

pm = generate_preset_pass_manager(backend=backend, optimization_level=1)
isa_circuit = pm.run(qc)

sampler = SamplerV2(backend)
sampler.options.experimental = {
    "execution": {
        "scheduler_timing": True,
    },
}

sampler_job = sampler.run([isa_circuit])
result = sampler_job.result()
```

### Accesarea datelor de cronologie ale Circuit-ului

Atunci când este solicitat, datele de cronologie ale Circuit-ului pentru fiecare PUB sunt returnate în metadatele rezultatului jobului, sub `["compilation"]["scheduler_timing"]["timing"]`. Acest câmp conține informațiile brute de cronologie. Pentru a afișa informațiile de cronologie, folosește instrumentul de vizualizare integrat, descris în secțiunea [Vizualizarea cronologiilor](#visualize-timings).

Folosește următorul cod pentru a accesa datele de cronologie ale Circuit-ului pentru primul PUB:

```python
job_result = sampler_job.result()
circuit_schedule = job_result[0].metadata["compilation"]["scheduler_timing"]
circuit_schedule_timing = circuit_schedule["timing"]
```

#### Înțelegerea datelor brute de cronologie

Deși vizualizarea datelor de cronologie ale Circuit-ului prin metoda `draw_circuit_schedule_timing` este cel mai frecvent caz de utilizare, poate fi util să înțelegi structura datelor brute de cronologie returnate. Acest lucru te-ar putea ajuta, de exemplu, să extragi informații programatic.

Datele de cronologie returnate în `["compilation"]["scheduler_timing"]["timing"]` reprezintă o listă de șiruri de caractere. Fiecare șir reprezintă o singură instrucțiune pe un anumit canal și este separat prin virgulă în următoarele tipuri de date:

- `Branch` - Determină dacă instrucțiunea se află într-un flux de control (then / else) sau pe ramura principală.
- `Instruction` - Gate-ul și Qubit-ul pe care se operează.
- `Channel` - Canalul căruia îi este atribuită instrucțiunea. Poate fi unul dintre următoarele:
      - `Qubit x` - Canalul de drive pentru Qubit-ul _x_.
      - `AWGRx_y` (generator de forme de undă arbitrare pentru citire) - Utilizat de canalele de citire pentru a comunica la măsurarea Qubit-urilor. Argumentele _x_ și _y_ corespund respectiv ID-ului instrumentului de citire și numărului Qubit-ului.
- `T0` - Timpul de început al instrucțiunii în cadrul întregului program.
- `Duration` - Durata instrucțiunii, în unități de _dt_ secunde, unde 1 dt = 1 ciclu de programare. Poți găsi valoarea `dt` a unui Backend folosind [`backend.dt`](https://docs.quantum.ibm.com/api/qiskit/qiskit.providers.BackendV2#dt).
- `Pulse` - Tipul operației de puls utilizate.

Exemplu:

```python
main,barrier,Qubit 0,7,0,barrier # A barrier on the main branch on qubit 0 at time 7 with 0 duration
main,reset_0,Qubit 0,7,64,play # A reset instruction on the main branch on qubit 0 at time 7 with duration 64 and a play operation
...
```

### Vizualizarea cronologiilor

Cu `qiskit-ibm-runtime` v0.43.0 sau o versiune ulterioară, poți vizualiza cronologiile Circuit-ului. Pentru a vizualiza cronologiile, trebuie mai întâi să convertești metadatele rezultatului în `fig` folosind [metoda `draw_circuit_schedule_timing`.](https://github.com/Qiskit/qiskit-ibm-runtime/blob/3d1bf1e1d49e5123841639fce259859c90ce9314/qiskit_ibm_runtime/visualization/draw_circuit_schedule_timings.py#L26) Această metodă returnează o figură `plotly`, pe care o poți afișa direct, salva într-un fișier sau ambele. Pentru mai multe informații despre comenzile `plotly` de utilizat, consultă [`fig.show()`](https://plotly.com/python-api-reference/generated/plotly.io.show.html) și [`fig.write_image("<path.format>")`.](https://plotly.com/python-api-reference/generated/plotly.io.write_image.html)

```python
from qiskit_ibm_runtime.visualization import draw_circuit_schedule_timing

# Create a figure from the metadata
fig = draw_circuit_schedule_timing(
    circuit_schedule=circuit_schedule_timing,
    included_channels=None,
    filter_readout_channels=False,
    filter_barriers=False,
    width=1000,
)

# Uncomment the following line to display the figure
# fig.show(renderer="notebook")

# Save to a file
# fig.write_html("scheduler_timing.html")
```

#### Înțelegerea figurii generate

Imaginea datelor de cronologie ale Circuit-ului generată de `draw_circuit_schedule_timing` transmite următoarele informații:

- Axa X reprezintă timpul în unități de _dt_ secunde, unde 1 dt = 1 ciclu de programare. Poți găsi valoarea `dt` a unui Backend folosind [`backend.dt`](https://docs.quantum.ibm.com/api/qiskit/qiskit.providers.BackendV2#dt).
- Axa Y reprezintă canalul (gândește-te la canale ca la instrumente care emit pulsuri).
    - `Receive channel` - Singurul canal care nu este un instrument în sine. Este o instrucțiune redată pe toate canalele care fac parte dintr-o procedură de comunicare cu hub-ul în acel moment.
    - `Qubit x` - Canalul de drive pentru Qubit-ul x.
    - `AWGRx_y` (generator de forme de undă arbitrare pentru citire) - Utilizat de canalele de citire pentru a comunica la măsurarea Qubit-urilor. Argumentele _x_ și _y_ corespund respectiv ID-ului instrumentului de citire și numărului Qubit-ului.
    - `Hub` - Controlează difuzarea.

În plus, fiecare instrucțiune are formatul *X_Y*, unde *X* este numele instrucțiunii și *Y* este tipul de puls. Un tip `play` aplică pulsuri de control, iar un `capture` înregistrează starea Qubit-ului. Poți, de asemenea, să treci cu cursorul peste fiecare instrucțiune pentru a obține mai multe detalii. De exemplu, figura anterioară arată un puls de control pentru Gate-ul X aplicat pe Qubit-ul 10 la 1161 dt.

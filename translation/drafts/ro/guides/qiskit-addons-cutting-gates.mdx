Acest ghid demonstrează două exemple funcționale de tăieri de Gate-uri cu pachetul `qiskit-addon-cutting`. Primul exemplu arată cum să reduci adâncimea Circuit-ului (numărul de instrucțiuni ale Circuit-ului) prin tăierea Gate-urilor de împletire pe Qubiți non-adiacenți, care altfel ar introduce o suprasarcină SWAP la transpilare. Al doilea exemplu acoperă modul de utilizare a tăierii de Gate-uri pentru a reduce lățimea Circuit-ului (numărul de Qubiți), împărțind un Circuit în mai multe Circuit-uri cu mai puțini Qubiți.

Ambele exemple vor folosi ansatz-ul [`efficient_su2`](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.library.efficient_su2) și reconstruiesc același observabil.
## Tăierea Gate-urilor pentru a reduce adâncimea Circuit-ului

Următorul flux de lucru reduce adâncimea unui Circuit prin tăierea Gate-urilor distante, evitând o serie lungă de Gate-uri SWAP care altfel ar fi introduse.

Începe cu ansatz-ul [`efficient_su2`](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.library.efficient_su2), cu împletire „circulară" pentru a introduce Gate-uri distante.

```python
# Added by doQumentation — required packages for this notebook
!pip install -q numpy qiskit qiskit-addon-cutting qiskit-aer qiskit-ibm-runtime
```

```python
import numpy as np
from qiskit.circuit.library import efficient_su2
from qiskit.quantum_info import SparsePauliOp
from qiskit.transpiler import generate_preset_pass_manager
from qiskit_ibm_runtime.fake_provider import FakeManilaV2
from qiskit_ibm_runtime import SamplerV2, Batch
from qiskit_aer.primitives import EstimatorV2
from qiskit_addon_cutting import (
    cut_gates,
    partition_problem,
    generate_cutting_experiments,
    reconstruct_expectation_values,
)

circuit = efficient_su2(num_qubits=4, entanglement="circular")
circuit.assign_parameters([0.4] * len(circuit.parameters), inplace=True)

observable = SparsePauliOp(["ZZII", "IZZI", "-IIZZ", "XIXI", "ZIZZ", "IXIX"])
print(f"Observable: {observable}")
circuit.draw("mpl", scale=0.8)
```

```text
Observable: SparsePauliOp(['ZZII', 'IZZI', 'IIZZ', 'XIXI', 'ZIZZ', 'IXIX'],
              coeffs=[ 1.+0.j,  1.+0.j, -1.+0.j,  1.+0.j,  1.+0.j,  1.+0.j])
```

![Output of the previous code cell](/docs/images/guides/qiskit-addons-cutting-gates/extracted-outputs/1551c440-c158-478a-a8fe-86df834c59bd-1.svg)

Fiecare Gate [`CNOT`](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.library.CXGate) dintre Qubiții $q_0$ și $q_3$ introduce două Gate-uri SWAP după transpilare (presupunând că Qubiții sunt conectați în linie dreaptă). Pentru a evita această creștere a adâncimii, poți înlocui aceste Gate-uri distante cu obiecte [`TwoQubitQPDGate`](https://docs.quantum.ibm.com/api/qiskit-addon-cutting/qpd-two-qubit-qpd-gate) folosind metoda [`cut_gates()`](https://docs.quantum.ibm.com/api/qiskit-addon-cutting/qiskit-addon-cutting#cut_gates). Această funcție returnează, de asemenea, o listă de instanțe [`QPDBasis`](../api/qiskit-addon-cutting/qpd-qpd-basis) — câte una pentru fiecare descompunere.

```python
# Find the indices of the distant gates
cut_indices = [
    i
    for i, instruction in enumerate(circuit.data)
    if {circuit.find_bit(q)[0] for q in instruction.qubits} == {0, 3}
]

# Decompose distant CNOTs into TwoQubitQPDGate instances
qpd_circuit, bases = cut_gates(circuit, cut_indices)

qpd_circuit.draw("mpl", scale=0.8)
```

![Output of the previous code cell](/docs/images/guides/qiskit-addons-cutting-gates/extracted-outputs/66dc0a14-ab51-4190-9cda-1c0373e91b9e-0.svg)

Acum că instrucțiunile Gate-urilor tăiate au fost adăugate, subexperimentele vor avea o adâncime mai mică după transpilare față de Circuit-ul original. Fragmentul de cod de mai jos generează subexperimentele folosind [`generate_cutting_experiments`](https://docs.quantum.ibm.com/api/qiskit-addon-cutting/qiskit-addon-cutting#generate_cutting_experiments), care primește Circuit-ul și observabilul de reconstruit.

<Admonition type="note" title="Notă despre numărul de mostre">
    Argumentul `num_samples` specifică câte mostre să fie extrase din distribuția quasi-probabilistică și determină acuratețea coeficienților folosiți pentru reconstrucție. Transmiterea valorii infinit (`np.inf`) va asigura că toți coeficienții sunt calculați exact. Citește documentația API despre [generarea ponderilor](https://docs.quantum.ibm.com/api/qiskit-addon-cutting/qpd#generate_qpd_weights) și [generarea experimentelor de tăiere](https://docs.quantum.ibm.com/api/qiskit-addon-cutting/qiskit-addon-cutting#generate_cutting_experiments) pentru mai multe informații.
</Admonition>

Odată ce subexperimentele sunt generate, le poți transpila și folosi primitivul `Sampler` pentru a eșantiona distribuția și a reconstitui valorile așteptate estimate. Următorul bloc de cod generează, transpilează și execută subexperimentele. Apoi reconstruiește rezultatele și le compară cu valoarea așteptată exactă.

```python
# Generate the subexperiments and sampling coefficients
subexperiments, coefficients = generate_cutting_experiments(
    circuits=qpd_circuit, observables=observable.paulis, num_samples=np.inf
)

# Set a backend to use and transpile the subexperiments
backend = FakeManilaV2()
pass_manager = generate_preset_pass_manager(
    optimization_level=1, backend=backend
)
isa_subexperiments = pass_manager.run(subexperiments)

# Set up the Qiskit Runtime Sampler primitive, submit the subexperiments, and retrieve the results
sampler = SamplerV2(backend)
job = sampler.run(isa_subexperiments, shots=4096 * 3)
results = job.result()

# Reconstruct the results
reconstructed_expval_terms = reconstruct_expectation_values(
    results,
    coefficients,
    observable.paulis,
)

# Apply the coefficients of the original observable
reconstructed_expval = np.dot(reconstructed_expval_terms, observable.coeffs)

estimator = EstimatorV2()
exact_expval = (
    estimator.run([(circuit, observable, [0.4] * len(circuit.parameters))])
    .result()[0]
    .data.evs
)
print(
    f"Reconstructed expectation value: {np.real(np.round(reconstructed_expval, 8))}"
)
print(f"Exact expectation value: {np.round(exact_expval, 8)}")
print(
    f"Error in estimation: {np.real(np.round(reconstructed_expval-exact_expval, 8))}"
)
print(
    f"Relative error in estimation: {np.real(np.round((reconstructed_expval-exact_expval) / exact_expval, 8))}"
)
```

```text
Reconstructed expectation value: 0.49812826
Exact expectation value: 0.50497603
Error in estimation: -0.00684778
Relative error in estimation: -0.0135606
```

<Admonition type="caution" title="Notă despre coeficienții observabilului">
    Pentru a reconstitui cu acuratețe valoarea așteptată, coeficienții observabilului original (care sunt diferiți de coeficienții din ieșirea `generate_cutting_experiments()`) trebuie aplicați ieșirii reconstrucției, deoarece aceste informații s-au pierdut când experimentele de tăiere au fost generate sau când observabilul a fost extins.

    De obicei, acești coeficienți pot fi aplicați prin `numpy.dot()` așa cum este arătat mai sus.
</Admonition>
## Tăierea Gate-urilor pentru a reduce lățimea Circuit-ului

Această secțiune demonstrează utilizarea tăierii de Gate-uri pentru a reduce lățimea Circuit-ului. Începe cu același [`efficient_su2`](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.library.efficient_su2), dar folosește împletire „lineară".

```python
qc = efficient_su2(4, entanglement="linear", reps=2)
qc.assign_parameters([0.4] * len(qc.parameters), inplace=True)

observable = SparsePauliOp(["ZZII", "IZZI", "-IIZZ", "XIXI", "ZIZZ", "IXIX"])
print(f"Observable: {observable}")

qc.draw("mpl", scale=0.8)
```

```text
Observable: SparsePauliOp(['ZZII', 'IZZI', 'IIZZ', 'XIXI', 'ZIZZ', 'IXIX'],
              coeffs=[ 1.+0.j,  1.+0.j, -1.+0.j,  1.+0.j,  1.+0.j,  1.+0.j])
```

![Output of the previous code cell](/docs/images/guides/qiskit-addons-cutting-gates/extracted-outputs/64010a14-8360-47e2-bb77-af9b2e0dbbfc-1.svg)

Apoi generează *subcircuitele* și *subobservabilele* pe care le vei executa folosind funcția [`partition_problem()`](https://docs.quantum.ibm.com/api/qiskit-addon-cutting/qiskit-addon-cutting#partition_problem). Această funcție primește Circuit-ul, observabilul și o schemă opțională de partiționare și returnează Circuit-urile și observabilele tăiate sub forma unui dicționar.

Partiționarea este definită printr-un șir de etichete de forma `"AABB"`, unde fiecare etichetă din acest șir corespunde Qubitului de la același index din argumentul `circuit`. Qubiții care împart o etichetă de partiție comună sunt grupați împreună, iar orice Gate-uri non-locale care acoperă mai mult de o partiție vor fi tăiate.

<Admonition type="info" title="Notă">
   Kwarg-ul `observables` al `partition_problem` este de tipul [`PauliList`](https://docs.quantum.ibm.com/api/qiskit/qiskit.quantum_info.PauliList). Coeficienții și fazele termenilor observabilului sunt ignorate în timpul descompunerii problemei și al execuției subexperimentelor. Ele pot fi re-aplicate în timpul reconstrucției valorii așteptate.
</Admonition>

```python
partitioned_problem = partition_problem(
    circuit=qc, partition_labels="AABB", observables=observable.paulis
)
subcircuits = partitioned_problem.subcircuits
subobservables = partitioned_problem.subobservables
bases = partitioned_problem.bases

print(f"Sampling overhead: {np.prod([basis.overhead for basis in bases])}")
print(f"Subobservables: {subobservables}")
subcircuits["A"].draw("mpl", scale=0.8)
```

```text
Sampling overhead: 81.0
Subobservables: {'A': PauliList(['II', 'ZI', 'ZZ', 'XI', 'ZZ', 'IX']), 'B': PauliList(['ZZ', 'IZ', 'II', 'XI', 'ZI', 'IX'])}
```

Următorul pas este să folosești subcircuitele și subobservabilele pentru a genera *subexperimentele* care vor fi executate pe un QPU folosind metoda [`generate_cutting_experiments`](https://docs.quantum.ibm.com/api/qiskit-addon-cutting/qiskit-addon-cutting#generate_cutting_experiments).

Pentru a estima valoarea așteptată a Circuit-ului de dimensiune completă, multe subexperimente sunt generate din distribuția quasi-probabilistică îmbinată a Gate-urilor descompuse și apoi executate pe unul sau mai multe QPU-uri. Numărul de mostre ce urmează a fi extrase din această distribuție este controlat prin argumentul `num_samples`.

Următorul bloc de cod generează subexperimentele și le execută folosind primitivul `Sampler` pe un simulator local. (Pentru a le rula pe un QPU, schimbă `backend`-ul cu resursa QPU dorită.)

```python
subexperiments, coefficients = generate_cutting_experiments(
    circuits=subcircuits, observables=subobservables, num_samples=np.inf
)

# Set a backend to use and transpile the subexperiments
backend = FakeManilaV2()
pass_manager = generate_preset_pass_manager(
    optimization_level=1, backend=backend
)
isa_subexperiments = {
    label: pass_manager.run(partition_subexpts)
    for label, partition_subexpts in subexperiments.items()
}

# Submit each partition's subexperiments to the Qiskit Runtime Sampler
# primitive, in a single batch so that the jobs will run back-to-back.
with Batch(backend=backend) as batch:
    sampler = SamplerV2(mode=batch)
    jobs = {
        label: sampler.run(subsystem_subexpts, shots=4096 * 3)
        for label, subsystem_subexpts in isa_subexperiments.items()
    }

# Retrieve results
results = {label: job.result() for label, job in jobs.items()}
```

În cele din urmă, valoarea așteptată a Circuit-ului complet este reconstruită folosind metoda [`reconstruct_expectation_values`](https://docs.quantum.ibm.com/api/qiskit-addon-cutting/qiskit-addon-cutting#reconstruct_expectation_values).

Blocul de cod de mai jos reconstruiește rezultatele și le compară cu valoarea așteptată exactă.

```python
# Get expectation values for each observable term
reconstructed_expval_terms = reconstruct_expectation_values(
    results,
    coefficients,
    subobservables,
)

# Reconstruct final expectation value
reconstructed_expval = np.dot(reconstructed_expval_terms, observable.coeffs)

estimator = EstimatorV2()
exact_expval = (
    estimator.run([(qc, observable, [0.4] * len(qc.parameters))])
    .result()[0]
    .data.evs
)
print(
    f"Reconstructed expectation value: {np.real(np.round(reconstructed_expval, 8))}"
)
print(f"Exact expectation value: {np.round(exact_expval, 8)}")
print(
    f"Error in estimation: {np.real(np.round(reconstructed_expval-exact_expval, 8))}"
)
print(
    f"Relative error in estimation: {np.real(np.round((reconstructed_expval-exact_expval) / exact_expval, 8))}"
)
```

```text
Reconstructed expectation value: 0.53571896
Exact expectation value: 0.56254612
Error in estimation: -0.02682716
Relative error in estimation: -0.04768882
```

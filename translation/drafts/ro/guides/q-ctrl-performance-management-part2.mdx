## Estimator primitive
### Exemplu Estimator

Folosește primitiva Estimator din Fire Opal Performance Management pentru a determina valoarea de așteptare a unei perechi circuit-observabil.
Pe lângă pachetele `qiskit-ibm-catalog` și `qiskit`, vei folosi și pachetul `numpy` pentru a rula acest exemplu. Poți instala acest pachet decomentând celula de mai jos, dacă rulezi exemplul într-un notebook folosind kernel-ul IPython.

```python
# %pip install numpy
```

**1. Creează Circuit-ul**

Ca exemplu, generează un operator Hermitian aleatoriu și un observabil pe care să le transmiți funcției Performance Management.

```python
import numpy as np
from qiskit.circuit.library import iqp
from qiskit.quantum_info import random_hermitian, SparsePauliOp

n_qubits = 50

# Generate a random circuit
mat = np.real(random_hermitian(n_qubits, seed=1234))
circuit = iqp(mat)
circuit.measure_all()

# Define observables as a string
observable = SparsePauliOp("Z" * n_qubits)
```

```python
# Create PUB tuple
estimator_pubs = [(circuit, observable)]
```

**2. Rulează Circuit-ul**

Rulează Circuit-ul și, opțional, definește Backend-ul și numărul de măsurători (shots).

```python
# This cell is hidden from users
from qiskit_ibm_runtime import QiskitRuntimeService

service = QiskitRuntimeService()
backend_name = service.least_busy().name
```

```python
# Run the circuit using the estimator
qctrl_estimator_job = perf_mgmt.run(
    primitive="estimator",
    pubs=estimator_pubs,
    backend_name=backend_name,
)
```

Poți folosi API-urile familiare [Qiskit Serverless](./serverless) pentru a verifica starea lucrării tale din Qiskit Function:

```python
qctrl_estimator_job.status()
```

```text
'QUEUED'
```

**3. Recuperează rezultatul**

```python
# Retrieve the counts from the result list
result = qctrl_estimator_job.result()
```

Rezultatele au același format ca un [rezultat Estimator](/guides/primitive-input-output#estimator-output):

```python
print(
    f"The result of the submitted job had {len(result)} PUB and has a value:\n {result}\n"
)
print(
    f"The associated PubResult of this job has the following DataBins:\n {result[0].data}\n"
)
print(f"And this DataBin has attributes: {result[0].data.keys()}")
print(
    f"The expectation values measured from this PUB are: \n{result[0].data.evs}"
)
```

### Intrări Estimator
<Admonition type="caution">Fire Opal Performance Management acceptă circuite abstracte, spre deosebire de primitivele native Qiskit Runtime, care acceptă doar circuite scrise în Setul de Instrucțiuni (ISA) al Backend-ului țintă. Pentru cele mai bune rezultate, nu transpila circuitele înainte de a le trimite prin funcția Performance Management. </Admonition>

| Nume  | Tip  | Descriere | Obligatoriu | Implicit  | Exemplu |
|------|------|----------|----------|-----------|-----------|
| pubs       | `QctrlEstimatorPubLike` sau `list[QctrlEstimatorPubLike]` | Unul sau mai multe tuple-uri ce conțin intrările listate la componentele `EstimatorPubLike`   | Da   | N/A   | `(circuit, observables, parameter_values)`   |
| backend_name| `str`  | Numele Backend-ului | Nu  |  Cel mai puțin ocupat Backend la care instanța ta are acces | `"ibm_fez"` |
| options| `dict`  | Opțiuni de intrare; vezi secțiunea Opțiuni pentru mai multe detalii | Nu  |  Vezi secțiunea Opțiuni pentru detalii | `{"default_shots": 2048}` |

**Componentele `QctrlEstimatorPubLike` (derivate din [definiția PUB Qiskit Runtime](/guides/primitive-input-output#estimator-pub)):**
- Un singur Circuit definit ca `QuantumCircuit` sau în format string OpenQASM 2.0 sau 3.0.
- Unul sau mai mulți observabili care specifică valorile de așteptare de estimat, în oricare dintre formatele indicate în lista „Formate de observabili acceptate".
- (Opțional) O colecție de valori de parametri pentru a lega Circuit-ul, care urmează aceleași [reguli de broadcasting](./primitive-input-output#broadcasting-rules) ca primitivele `QiskitRuntime`.
- (Opțional) O precizie țintă pentru valorile de așteptare de estimat.
- (Opțional) Un număr real reprezentând precizia sau un dicționar de opțiuni de rulare ce conține numărul de shots. De exemplu: `{"shots": <int>}`.

**Formate de observabili acceptate:**
- Oricare dintre formatele `ObservablesArrayLike`, cum ar fi `Pauli`, `SparsePauliOp`, `PauliList` sau `str`
- Un string Pauli: `"XY"`
- Un dicționar — stringuri Pauli cu coeficienți: `{"XY": 0.5, "YZ": 0.3}`
- O listă de stringuri Pauli: `["XY", "YZ", "ZX"]`
- O listă de stringuri Pauli cu coeficienți: `[("XY", 0.5), ("YZ", 0.3)]`
- O listă imbricată de stringuri Pauli: `[["XY", "YZ"], ["ZX", "XX"]]`
- O listă imbricată de stringuri Pauli cu coeficienți: `[[("XY", 0.1), ("YZ", 0.2)], [("ZX", 0.3), ("XX", 0.4)]]`

**Backend-uri acceptate:**
Lista de mai jos cuprinde Backend-urile acceptate în prezent. Dacă dispozitivul tău nu este inclus, [contactează Q-CTRL](https://form.typeform.com/to/iuujEAEI?typeform-source=q-ctrl.com) pentru a adăuga suport.

**Opțiuni:**
| Nume   | Tip   | Descriere  | Implicit |
|--------|----------|-----------------------|---------------------|
| session_id | `str`  | ID-ul unei Session Qiskit Runtime existente  | `"cw4r3je6f0t010870y3g"` |
| default_shots | `int`  | Numărul de shots de utilizat pentru fiecare Circuit  | `2048` |
| default_precision | `float`  | Precizia țintă pentru valorile de așteptare de estimat pentru fiecare Circuit  | `0.015625` |
| job_tags | `list[str]` | O listă de etichete de job | `[]`|

### Ieșiri Estimator
| Nume   | Tip   | Descriere  | Exemplu |
|--------|---------------------------------|------------------------------------|---------------------|
| N/A | `PrimitiveResult`  | [`PrimitiveResult`](https://docs.quantum.ibm.com/api/qiskit/qiskit.primitives.PrimitiveResult) corespunzător listei de PUB-uri de intrare | `PubResult(data=DataBin(evs=[0.1234], stds=[0.1]))` |

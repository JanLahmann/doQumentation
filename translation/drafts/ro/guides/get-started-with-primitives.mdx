## Începe cu Estimator

{/*Verified the v2 examples 2/29/24 - updated 10/29/24*/}

### 1. Inițializează contul

Deoarece Qiskit Runtime Estimator este un serviciu gestionat, mai întâi trebuie să îți inițializezi contul. Poți apoi să selectezi QPU-ul pe care vrei să îl folosești pentru a calcula valoarea de așteptare.

Urmează pașii din [subiectul Instalare și configurare](install-qiskit) dacă nu ai încă un cont.

```python
from qiskit_ibm_runtime import QiskitRuntimeService

service = QiskitRuntimeService()
backend = service.least_busy(
    operational=True, simulator=False, min_num_qubits=127
)

print(backend.name)
```

```text
ibm_torino
```

### 2. Creează un Circuit și un observabil

Ai nevoie de cel puțin un Circuit și un observabil ca intrări pentru primitiva Estimator.

```python
from qiskit.circuit.library import qaoa_ansatz
from qiskit.quantum_info import SparsePauliOp

entanglement = [tuple(edge) for edge in backend.coupling_map.get_edges()]
observable = SparsePauliOp.from_sparse_list(
    [("ZZ", [i, j], 0.5) for i, j in entanglement],
    num_qubits=backend.num_qubits,
)
circuit = qaoa_ansatz(observable, reps=2)
# the circuit is parametrized, so we will define the parameter values for execution
param_values = [0.1, 0.2, 0.3, 0.4]

print(f">>> Observable: {observable.paulis}")
```

Circuit-ul și observabilul trebuie transformate pentru a folosi doar instrucțiunile suportate de QPU (cunoscute ca circuite de *arhitectură a setului de instrucțiuni (ISA)*). Vom folosi Transpiler-ul pentru aceasta.

```python
from qiskit.transpiler import generate_preset_pass_manager

pm = generate_preset_pass_manager(optimization_level=1, backend=backend)
isa_circuit = pm.run(circuit)
isa_observable = observable.apply_layout(isa_circuit.layout)
print(f">>> Circuit ops (ISA): {isa_circuit.count_ops()}")
```

```text
>>> Circuit ops (ISA): OrderedDict([('rz', 3826), ('sx', 1601), ('cz', 968)])
```

### 3. Inițializează Qiskit Runtime Estimator

Când inițializezi Estimator, folosește parametrul `mode` pentru a specifica modul în care vrei să ruleze. Valorile posibile sunt obiectele `batch`, `session` sau `backend` pentru modul de execuție batch, Session și job, respectiv. Pentru mai multe informații, consultă [Introducere în modurile de execuție Qiskit Runtime.](execution-modes)

```python
from qiskit_ibm_runtime import EstimatorV2 as Estimator

estimator = Estimator(mode=backend)
```

### 4. Invocă Estimator și obține rezultate

Apoi, invocă metoda `run()` pentru a calcula valorile de așteptare pentru Circuit-urile și observabilele de intrare. Circuit-ul, observabilul și seturile opționale de valori ale parametrilor sunt introduse ca tuple-uri PUB (*primitive unified bloc*).

```python
job = estimator.run([(isa_circuit, isa_observable, param_values)])
print(f">>> Job ID: {job.job_id()}")
print(f">>> Job Status: {job.status()}")
```

```python
result = job.result()
print(f">>> {result}")
```

## Începe cu Sampler

### 1. Inițializează contul

Deoarece Qiskit Runtime Sampler este un serviciu gestionat, mai întâi trebuie să îți inițializezi contul. Poți apoi să selectezi QPU-ul pe care vrei să îl folosești pentru a calcula valoarea de așteptare.

Urmează pașii din [subiectul Instalare și configurare](install-qiskit) dacă nu ai încă un cont configurat.

```python
from qiskit_ibm_runtime import QiskitRuntimeService

service = QiskitRuntimeService()
backend = service.least_busy(
    operational=True, simulator=False, min_num_qubits=127
)
```

### 2. Creează un Circuit

Ai nevoie de cel puțin un Circuit ca intrare pentru primitiva Sampler.

```python
import numpy as np
from qiskit.circuit.library import efficient_su2

circuit = efficient_su2(127, entanglement="linear")
circuit.measure_all()
# The circuit is parametrized, so we will define the parameter values for execution
param_values = np.random.rand(circuit.num_parameters)
```

Folosește Transpiler-ul pentru a obține un Circuit ISA.

```python
from qiskit.transpiler import generate_preset_pass_manager

pm = generate_preset_pass_manager(optimization_level=1, backend=backend)
isa_circuit = pm.run(circuit)
print(f">>> Circuit ops (ISA): {isa_circuit.count_ops()}")
```

```text
>>> Circuit ops (ISA): OrderedDict([('sx', 3089), ('rz', 3036), ('cz', 1092), ('measure', 127), ('barrier', 1)])
```

### 3. Inițializează Qiskit Runtime Sampler

Când inițializezi Sampler, folosește parametrul `mode` pentru a specifica modul în care vrei să ruleze. Valorile posibile sunt obiectele `batch`, `session` sau `backend` pentru modul de execuție batch, Session și job, respectiv. Pentru mai multe informații, consultă [Introducere în modurile de execuție Qiskit Runtime.](execution-modes) Reține că utilizatorii planului Open nu pot trimite job-uri de tip Session.

```python
from qiskit_ibm_runtime import SamplerV2 as Sampler

sampler = Sampler(mode=backend)
```

### 4. Invocă Sampler și obține rezultate

Apoi, invocă metoda `run()` pentru a genera rezultatul. Circuit-ul și seturile opționale de valori ale parametrilor sunt introduse ca tuple-uri PUB (*primitive unified bloc*).

```python
job = sampler.run([(isa_circuit, param_values)])
print(f">>> Job ID: {job.job_id()}")
print(f">>> Job Status: {job.status()}")
```

```text
>>> Job ID: d5k96rsjt3vs73ds5tig
>>> Job Status: QUEUED
```

```python
result = job.result()

# Get results for the first (and only) PUB
pub_result = result[0]
print(
```

## Începe cu primitivele backend

Spre deosebire de primitivele specifice unui furnizor, primitivele backend sunt implementări generice care pot fi folosite cu orice obiect
`backend` arbitrar, atâta timp cât acesta implementează interfața [`Backend`](https://docs.quantum.ibm.com/api/qiskit/qiskit.providers.Backend).

- Primitiva Sampler poate fi rulată cu orice furnizor folosind [`qiskit.primitives.BackendSamplerV2`](https://docs.quantum.ibm.com/api/qiskit/qiskit.primitives.BackendSamplerV2).
- Primitiva Estimator poate fi rulată cu orice furnizor folosind [`qiskit.primitives.BackendEstimatorV2`](../api/qiskit/qiskit.primitives.BackendEstimatorV2).

Unii furnizori implementează primitive nativ. Consultă [pagina Qiskit Ecosystem](https://qiskit.github.io/ecosystem#provider) pentru detalii.
### Exemplu: BackendEstimator

```python
from qiskit.primitives import BackendEstimatorV2
from <some_qiskit_provider> import QiskitProvider

provider = QiskitProvider()
backend = provider.get_backend('backend_name')
estimator = BackendEstimatorV2(backend)
```

### Exemplu: BackendSampler

```python
from qiskit.primitives import BackendSamplerV2
from <some_qiskit_provider> import QiskitProvider

provider = QiskitProvider()
backend = provider.get_backend('backend_name')
sampler = BackendSamplerV2(backend)
```
### Asemănări și diferențe între primitivele backend și cele Runtime

- Intrările și ieșirile pentru [`qiskit.primitives.BackendSamplerV2`](https://docs.quantum.ibm.com/api/qiskit/qiskit.primitives.BackendSamplerV2) și [`qiskit.primitives.BackendEstimatorV2`](../api/qiskit/qiskit.primitives.BackendEstimatorV2)
urmează același format PUB ca primitivele din Qiskit Runtime. Consultă [Intrările și ieșirile primitivelor](primitive-input-output) pentru detalii.
Totuși, pot exista diferențe în câmpurile metadatelor returnate.

- Clasa [`qiskit.primitives.BackendEstimatorV2`](https://docs.quantum.ibm.com/api/qiskit/qiskit.primitives.BackendEstimatorV2) nu oferă implementări de atenuare a erorilor de măsurare sau Gate din cutie, deoarece
primitivele backend sunt proiectate să ruleze local pe mașina utilizatorului.

- Clasa [`qiskit.primitives.BackendSamplerV2`](https://docs.quantum.ibm.com/api/qiskit/qiskit.primitives.BackendSamplerV2) necesită un Backend care suportă opțiunea `memory`.

- Interfețele primitive backend expun opțiuni `Options` personalizate [`SamplerV2`](https://docs.quantum.ibm.com/api/qiskit/qiskit.primitives.BackendSamplerV2) și [`EstimatorV2`](https://docs.quantum.ibm.com/api/qiskit/qiskit.primitives.BackendEstimatorV2) care diferă de implementările Runtime.

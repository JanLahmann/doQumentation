## Listă de Backend-uri {#list-backends}

Pentru a vedea Backend-urile la care ai acces, poți fie să vizualizezi o listă pe [pagina Compute resources,](https://quantum.cloud.ibm.com/computers) fie să folosești metoda [`QiskitRuntimeService.backends()`](../api/qiskit-ibm-runtime/qiskit-runtime-service#backends). Această metodă returnează o listă de instanțe [`IBMBackend`](../api/qiskit-ibm-runtime/ibm-backend):

<Admonition type="note" title="Note">
- Dacă ești autentificat într-o instanță sau regiune specifică, sau dacă ai inițializat serviciul cu o instanță sau regiune specifică folosind `QiskitRuntimeService()`, vor fi returnate doar Backend-urile disponibile ție în acea instanță sau regiune. Altfel, vor fi returnate toate Backend-urile disponibile ție în orice instanță și în orice regiune.
- Lista de Backend-uri returnată ar putea să nu fie aceeași cu cea afișată pe pagina [Compute resources](https://quantum.cloud.ibm.com/computers) a IBM Quantum Platform. Lista de pe pagina Compute resources este întotdeauna filtrată după regiunea selectată în partea de sus a paginii.
</Admonition>
Pentru a rula codul următor, asigură-te că te-ai autentificat deja la serviciu. Consultă [Configurează-ți contul IBM Cloud](/guides/cloud-setup) pentru mai multe detalii.

```python
# Added by doQumentation — required packages for this notebook
```

```python
from qiskit_ibm_runtime import QiskitRuntimeService

service.backends()
```

Metoda [`QiskitRuntimeService.backend()`](../api/qiskit-ibm-runtime/qiskit-runtime-service#backend) (reține că aceasta este la singular: *backend*) primește numele Backend-ului ca parametru de intrare și returnează o instanță [`IBMBackend`](../api/qiskit-ibm-runtime/ibm-backend) reprezentând acel Backend specific:

## Filtrarea Backend-urilor {#filter-backends}

Poți filtra, de asemenea, Backend-urile disponibile după proprietățile lor. Pentru filtre mai generale, setează argumentul `filters` la o funcție care acceptă un obiect Backend și returnează `True` dacă acesta îndeplinește criteriile tale. Consultă [documentația API](../api/qiskit-ibm-runtime/qiskit-runtime-service#backends) pentru mai multe detalii.

Codul următor returnează doar Backend-urile care îndeplinesc aceste criterii și îți sunt disponibile _în instanța selectată curent_:

*   Sunt dispozitive cuantice reale (`simulator=False`)
*   Sunt în prezent operaționale (`operational=True`)
*   Au cel puțin 5 Qubiți (`min_num_qubits=5`)

```python
# Optionally pass in an instance, region, or both, to
# further filter the backends.
service = QiskitRuntimeService()

service.backends(simulator=False, operational=True, min_num_qubits=5)
```

```text
[<IBMBackend('ibm_pittsburgh')>,
 <IBMBackend('ibm_boston')>,
 <IBMBackend('ibm_fez')>,
 <IBMBackend('ibm_miami')>,
 <IBMBackend('ibm_marrakesh')>,
 <IBMBackend('ibm_torino')>,
 <IBMBackend('ibm_kingston')>]
```

Folosește aceste argumente cu cuvinte cheie pentru a filtra după orice atribut din configurația Backend-ului ([schema JSON](https://github.com/Qiskit/ibm-quantum-schemas/blob/main/schemas/backend_configuration_schema.json)) sau stare ([schema JSON](https://github.com/Qiskit/ibm-quantum-schemas/blob/main/schemas/backend_status_schema.json)). O metodă similară este [`QiskitRuntimeService.least_busy()`](../api/qiskit-ibm-runtime/qiskit-runtime-service#least_busy), care primește aceleași filtre ca `backends()`, dar returnează Backend-ul care corespunde filtrelor și are cel mai mic număr de joburi în așteptare în coadă:

```python
service.least_busy(operational=True, min_num_qubits=5)
```

```text
<IBMBackend('ibm_torino')>
```

## Informații statice despre Backend {#static-backend-information}

Unele informații despre un Backend nu se schimbă regulat, cum ar fi numele, versiunea, numărul de Qubiți pe care îi are și tipurile de funcționalități pe care le suportă. Aceste informații sunt disponibile ca atribute ale obiectului `backend`.

Celula următoare construiește o descriere a unui Backend.

```python
backend = service.backend("ibm_fez")

print(
    f"Name: {backend.name}\n"
    f"Version: {backend.version}\n"
    f"No. of qubits: {backend.num_qubits}\n"
)
```

```text
Name: ibm_fez
Version: 2
No. of qubits: 156
```

Pentru o listă completă de atribute, consultă [documentația API `IBMBackend`](https://docs.quantum.ibm.com/api/qiskit-ibm-runtime/ibm-backend).
## Gate-uri și operații native {#native-gates-and-operations}

Fiecare [familie de procesoare](/guides/processor-types) are un set nativ de Gate-uri. În mod implicit, QPU-urile din fiecare familie suportă doar rularea Gate-urilor și operațiilor din setul nativ de Gate-uri. Prin urmare, fiecare Gate din Circuit trebuie tradus (de către Transpiler) în elementele acestui set.

Poți vizualiza Gate-urile și operațiile native pentru un QPU fie [cu Qiskit](#native-gates-with-qiskit), fie pe pagina [Compute resources](/guides/qpu-information#native-gates-on-platform) a IBM Quantum&reg; Platform.

<span id="native-gates-with-qiskit"></span>
```python

from qiskit_ibm_runtime import QiskitRuntimeService

service = QiskitRuntimeService()

for backend in service.backends():
    config = backend.configuration()
    if "simulator" in config.backend_name:
        continue
    print(f"Backend: {config.backend_name}")
    print(f"    Processor type: {config.processor_type}")
    print(f"    Supported instructions:")
    for instruction in config.supported_instructions:
        print(f"        {instruction}")
    print()
```
## Informații dinamice despre Backend {#dynamic-backend-information}

Backend-urile pot avea, de asemenea, proprietăți care se schimbă ori de câte ori Backend-ul este calibrat, cum ar fi frecvența Qubitului și ratele de eroare ale operațiilor. Backend-urile sunt de obicei calibrate la fiecare 24 de ore, iar proprietățile lor se actualizează după finalizarea secvenței de calibrare. Aceste proprietăți pot fi utilizate la optimizarea circuitelor cuantice sau pentru a construi modele de zgomot pentru un simulator clasic.

### Proprietățile Qubitului {#qubit-properties}

`backend.properties().qubit_property()` returnează informații despre atributele fizice ale Qubiților. Conține un dicționar cu diverse proprietăți ale Qubitului, fiecare asociată cu valoarea sa și marca de timp a ultimei calibrări.

- `T1 (Timp de relaxare)`: Timpul T1 reprezintă durata medie în care un Qubit rămâne în starea sa excitată $|1\rangle$ înainte de a se dezintegra la starea sa fundamentală $|0\rangle$ din cauza relaxării energetice. Acest parametru este folosit pentru a caracteriza comportamentul de relaxare energetică al Qubitului și este exprimat în unități de secunde (s).

- `T2 (Timp de defazare)`: Timpul T2 denotă scala de timp pe parcursul căreia un Qubit menține coerența de fază a unei superpoziții între stările $|0\rangle$ și $|1\rangle$. Acesta ține cont atât de relaxarea energetică, cât și de procesele de defazare pură, oferind informații despre proprietățile de coerență ale Qubitului.

- `frequency`: Acest parametru specifică frecvența de rezonanță a Qubitului, indicând diferența de energie între stările $|0\rangle$ și $|1\rangle$, exprimată în hertz (Hz).

- `anharmonicity`: Anharmonicitatea este diferența de energie dintre primul și al doilea nivel excitat al Qubitului, exprimată, de asemenea, în hertz (Hz).

- `readout_error`: Eroarea de citire cuantifică probabilitatea medie de a măsura incorect starea unui Qubit. Este calculată în mod obișnuit ca media dintre prob_meas0_prep1 și prob_meas1_prep0, oferind o singură metrică pentru fidelitatea măsurătorii.

- `prob_meas0_prep1`: Acest parametru indică probabilitatea de a măsura un Qubit în starea 0 atunci când s-a intenționat să fie pregătit în starea $|1\rangle$, notată $P(0 | 1)$. Reflectă erorile de pregătire și măsurare a stării (SPAM), în special erorile de măsurare în Qubiții superconductori.

- `prob_meas1_prep0`: Similar, acest parametru reprezintă probabilitatea de a măsura un Qubit în starea 1 atunci când s-a intenționat să fie pregătit în starea $|0\rangle$, notată $P(1 | 0)$. La fel ca prob_meas0_prep1, reflectă erorile SPAM, erorile de măsurare fiind principalul factor contributiv în Qubiții superconductori.

- `readout_length`: readout_length specifică durata operației de citire pentru un Qubit. Măsoară timpul de la inițierea pulsului de măsurare până la finalizarea digitizării semnalului, după care sistemul este pregătit pentru operația următoare. Înțelegerea acestui parametru este crucială pentru optimizarea execuției circuitelor, în special atunci când se încorporează măsurători mid-circuit.

### Proprietățile instrucțiunilor {#instruction-properties}

Atributul `backend.target` este un obiect `qiskit.transpiler.Target`: un obiect care conține toate informațiile necesare pentru a transpila un Circuit pentru acel Backend. Aceasta include erorile și duratele instrucțiunilor. De exemplu, celula următoare obține proprietățile pentru un [Gate `cz`](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.library.CZGate) care acționează între Qubiții 1 și 0.

```python
backend.target["cz"][(1, 0)]
```

```text
InstructionProperties(duration=6.8e-08, error=0.007831625819164134)
```

Celula următoare arată proprietățile pentru o operație de măsurare (inclusiv eroarea de citire) pe Qubitul 0.

```python
backend.target["measure"][(0,)]
```

```text
InstructionProperties(duration=1.56e-06, error=0.013916015625)
```

Când pregătești un workload cuantic cu retropropagarea operatorilor (OBP), trebuie mai întâi să selectezi „felii de Circuit" (circuit slices), iar apoi să specifici un prag de trunchiere sau „buget de eroare" pentru a elimina termenii cu coeficienți mici din operatorul retropropagat, precum și să stabilești o limită superioară pentru dimensiunea totală a operatorului retropropagat. În timpul retropropagării, numărul de termeni din operatorul unui Circuit cu $N$ Qubiți va tinde spre $4^N$ rapid în cel mai defavorabil scenariu. Acest ghid demonstrează pașii implicați în aplicarea OBP unui workload cuantic.

Componenta principală a pachetului `qiskit-addons-obp` este funcția `backpropagate()`. Aceasta primește argumente pentru observabila finală de reconstruit, un set de felii de Circuit pentru calculul clasic și, opțional, un `TruncationErrorBudget` sau `OperatorBudget` pentru a impune constrângeri asupra trunchierilor efectuate. Odată specificate acestea, operatorul retropropagat calculat clasic $O'$ este calculat iterativ prin aplicarea porților din fiecare felie $s$ în felul următor:

$$ O'^{(s)} = \mathcal{U}_{S-s+1}^\dagger O'^{(s-1)} \mathcal{U}_{S-s+1} $$

unde $S$ este numărul total de felii și $\mathcal{U}_{s}$ reprezintă o singură felie a Circuit-ului. Acest exemplu folosește pachetul `qiskit-addons-utils` pentru a pregăti feliile de Circuit, precum și pentru a genera Circuit-ul exemplu.

Pentru a începe, ia în considerare evoluția în timp a unui lanț Heisenberg XYZ. Acest Hamiltonian are forma

$$ \hat{H} = \sum_{(j,k)} \left( J_xX_jX_k + J_yY_jY_k + J_z Z_jZ_k \right) + \sum_{j} \left(h_xX_j + h_yY_j + h_zZ_j\right)$$

iar valoarea de așteptare de măsurat va fi $\langle Z_0 \rangle$.

Fragmentul de cod următor generează Hamiltonianul sub forma unui `SparsePauliOp` folosind modulul `qiskit_addons_utils.problem_generators` și un `CouplingMap`. Setează constantele de cuplaj la $J_x=\pi/8$, $J_y=\pi/4$, $J_z=\pi/2$ și câmpurile magnetice externe la $h_x=\pi/3$, $h_y=\pi/6$, $h_z=\pi/9$, apoi generează un Circuit care modelează evoluția sa în timp.

```python
import numpy as np
from qiskit.transpiler import CouplingMap
from qiskit.synthesis import LieTrotter
from qiskit.transpiler import generate_preset_pass_manager
from qiskit_ibm_runtime import QiskitRuntimeService, EstimatorV2
from qiskit_ibm_runtime.fake_provider import FakeMelbourneV2
from qiskit.primitives import StatevectorEstimator
from qiskit.quantum_info import SparsePauliOp
from qiskit_addon_utils.problem_generators import (
    generate_time_evolution_circuit,
    generate_xyz_hamiltonian,
)
from qiskit_addon_utils.slicing import slice_by_gate_types
from qiskit_addon_obp.utils.simplify import OperatorBudget
from qiskit_addon_obp.utils.truncating import setup_budget
from qiskit_addon_obp import backpropagate
from qiskit_addon_utils.slicing import combine_slices

coupling_map = CouplingMap.from_heavy_hex(3, bidirectional=False)

# Choose a 10-qubit linear chain on this coupling map
reduced_coupling_map = coupling_map.reduce(
    [0, 13, 1, 14, 10, 16, 5, 12, 8, 18]
)

# Get a qubit operator describing the Heisenberg XYZ model
hamiltonian = generate_xyz_hamiltonian(
    reduced_coupling_map,
    coupling_constants=(np.pi / 8, np.pi / 4, np.pi / 2),
    ext_magnetic_field=(np.pi / 3, np.pi / 6, np.pi / 9),
)

# we evolve for some time
circuit = generate_time_evolution_circuit(
    hamiltonian, synthesis=LieTrotter(reps=2), time=0.2
)

circuit.draw("mpl")
```

### Pregătirea intrărilor pentru retropropagare {#prepare-inputs-to-backpropagate}

Apoi, generează feliile de Circuit pentru retropropagare. În general, modul în care alegi să tai Circuit-ul poate influența cât de bine funcționează retropropagarea pentru o anumită problemă. Aici, grupează Gate-urile de același tip în felii folosind funcția `qiskit_addons_utils.slice_by_gate_types`.

```python
slices = slice_by_gate_types(circuit)
print(f"Separated the circuit into {len(slices)} slices.")
```

```text
Separated the circuit into 18 slices.
```

Odată generate feliile, specifică un `OperatorBudget` pentru a furniza funcției `backpropagate()` o condiție de oprire a retropropagării operatorului și pentru a preveni creșterea în continuare a supraîncărcării clasice. Poți specifica, de asemenea, un buget de eroare de trunchiere pentru fiecare felie, unde termenii Pauli cu coeficienți mici vor fi trunchiați din fiecare felie până la epuizarea bugetului de eroare. Orice buget rămas va fi adăugat la bugetul felie următoare.

Aici, specifică faptul că retropropagarea ar trebui să se oprească atunci când numărul de grupuri Pauli comutante qubit-wise din operator depășește $8$ și alocă un buget de eroare de $0.005$ pentru fiecare felie.

```python
op_budget = OperatorBudget(max_qwc_groups=8)
truncation_error_budget = setup_budget(max_error_per_slice=0.005)
```

### Retropropagarea feliilor {#backpropagate-slices}

În acest pas vei defini observabila finală de măsurat și vei rula retropropagarea pe fiecare felie. Funcția `backpropagate()` returnează trei ieșiri: observabila retropropagată, feliile de Circuit rămase care nu au fost retropropagate (și care ar trebui rulate pe hardware cuantic), și metadate despre retropropagare.

Reține că atât `OperatorBudget`, cât și `TruncationErrorBudget` sunt parametri opționali pentru metoda `backpropagate()`. În general, cea mai bună alegere pentru ambele ar trebui stabilită euristic și necesită un anumit grad de experimentare. În acest exemplu vom efectua retropropagarea atât cu, cât și fără un `TruncationErrorBudget`.

<Admonition title="Notă" type="note">
  Implicit, `backpropagate()` folosește norma $L_1$ a coeficienților trunchiați pentru a limita eroarea totală acumulată din trunchiere, dar se pot folosi și alte norme $L_p$ dacă dorești să modifici modul în care se calculează eroarea de trunchiere.
</Admonition>

```python
# Specify a single-qubit observable
observable = SparsePauliOp("IIIIIIIIIZ")

# Backpropagate without the truncation error budget
backpropagated_observable, remaining_slices, metadata = backpropagate(
    observable,
    slices,
    operator_budget=op_budget,
)

# Recombine the slices remaining after backpropagation
bp_circuit = combine_slices(remaining_slices, include_barriers=True)

print(f"Backpropagated {metadata.num_backpropagated_slices} slices.")
print(
    f"New observable has {len(backpropagated_observable.paulis)} terms, which can be combined into "
    f"{len(backpropagated_observable.group_commuting(qubit_wise=True))} groups.\n"
    f"After truncation, the error in our observable is bounded by {metadata.accumulated_error(0):.3e}"
)
print(
    f"Note that backpropagating one more slice would result in {metadata.backpropagation_history[-1].num_paulis[0]} terms "
    f"across {metadata.backpropagation_history[-1].num_qwc_groups} groups."
)
```

### Transpilarea și execuția workload-ului cuantic {#transpile-and-execute-quantum-workload}

Acum că ai retropropagat operatorul, poți executa porțiunea rămasă a Circuit-ului pe un QPU. Workload-ul cuantic, folosind Estimator, ar trebui să includă Circuit-ul `bp_circuit_trunc` și trebuie să măsoare operatorul retropropagat `backpropagated_observable`.

Pentru a demonstra eficacitatea OBP de sine stătătoare, fragmentul de cod următor transpilează atât Circuit-ul original, cât și Circuit-ul retropropagat (cu și fără trunchiere) și simulează Circuit-urile clasic folosind `StatevectorEstimator`.

```python
# Specify a backend and a pass manager for transpilation
backend = FakeMelbourneV2()
# pm = generate_preset_pass_manager(backend=backend, optimization_level=1)

pm = generate_preset_pass_manager(backend=backend, optimization_level=3)

# Transpile original experiment
circuit_isa = pm.run(circuit)
observable_isa = observable.apply_layout(circuit_isa.layout)

# Transpile backpropagated experiment without truncation
bp_circuit_isa = pm.run(bp_circuit)
bp_obs_isa = backpropagated_observable.apply_layout(bp_circuit_isa.layout)

# Transpile the backpropagated experiment with truncated observable terms
bp_circuit_trunc_isa = pm.run(bp_circuit_trunc)
bp_obs_trunc_isa = backpropagated_observable_trunc.apply_layout(
    bp_circuit_trunc_isa.layout
)

estimator = StatevectorEstimator()

# Run the experiments using the exact statevector estimator
result_exact = (
    estimator.run([(circuit, observable)]).result()[0].data.evs.item()
)

result_bp = (
    estimator.run([(bp_circuit_isa, bp_obs_isa)]).result()[0].data.evs.item()
)
result_bp_trunc = (
    estimator.run([(bp_circuit_trunc_isa, bp_obs_trunc_isa)])
    .result()[0]
    .data.evs.item()
)

print(f"Exact expectation value: {result_exact}")
print(f"Backpropagated expectation value without truncation: {result_bp}")
print(f"Backpropagated expectation value with truncation: {result_bp_trunc}")
print(
    f"    - Expected Error for truncated observable: {metadata_trunc.accumulated_error(0):.3e}"
)
print(
    f"    - Observed Error for truncated observable: {abs(result_exact - result_bp_trunc):.3e}"
)
```

```text
Exact expectation value: 0.8854160687717517
Backpropagated expectation value without truncation: 0.8854160687717533
Backpropagated expectation value with truncation: 0.8850236647156081
    - Expected Error for truncated observable: 4.933e-02
    - Observed Error for truncated observable: 3.924e-04
```

În final, fragmentul de cod următor va transpila și executa Circuit-ul retropropagat pe un QPU (atât cu, cât și fără trunchiere).

```python
# Specify a backend and a pass manager for transpilation
service = QiskitRuntimeService()
backend = service.least_busy()
pm = generate_preset_pass_manager(backend=backend, optimization_level=3)

# Transpile backpropagated experiment without truncation
bp_circuit_isa = pm.run(bp_circuit)
bp_obs_isa = backpropagated_observable.apply_layout(bp_circuit_isa.layout)

# Transpile the backpropagated experiment with truncated observable terms
bp_circuit_trunc_isa = pm.run(bp_circuit_trunc)
bp_obs_trunc_isa = backpropagated_observable_trunc.apply_layout(
    bp_circuit_trunc_isa.layout
)

# Run the experiments using Estimator primitive
estimator = EstimatorV2(mode=backend)

result_bp_qpu = (
    estimator.run([(bp_circuit_isa, bp_obs_isa)]).result()[0].data.evs.item()
)

result_bp_trunc_qpu = (
    estimator.run([(bp_circuit_trunc_isa, bp_obs_trunc_isa)])
    .result()[0]
    .data.evs.item()
)

print(f"Exact expectation value: {result_exact}")
print(f"Backpropagated expectation value without truncation: {result_bp_qpu}")
print(
    f"Backpropagated expectation value with truncation: {result_bp_trunc_qpu}"
)
print(
    f"    - Observed Error for observable without truncation: {abs(result_exact - result_bp_qpu):.3e}"
)
print(
    f"    - Observed Error for truncated observable: {abs(result_exact - result_bp_trunc_qpu):.3e}"
)
```

Backpropagated expectation value with truncation: 0.8759838342768448
    - Observed Error for observable without truncation: 6.373e-03
    - Observed Error for truncated observable: 9.432e-03

# Jurnalizare pentru Qiskit Runtime

Serviciile IBM Cloud&reg;, precum Qiskit Runtime, generează jurnale de platformă pe care le poți folosi pentru a investiga activități anormale și acțiuni critice din contul tău, precum și pentru a depana probleme.

Poți folosi [IBM Cloud Logs Routing,](https://cloud.ibm.com/docs/logs-router?topic=logs-router-about) un serviciu de platformă, pentru a direcționa jurnalele de platformă din contul tău către o destinație la alegere, configurând un tenant care definește unde sunt trimise jurnalele de platformă. Pentru mai multe informații, consultă [Despre Logs Routing.](https://cloud.ibm.com/docs/logs-router?topic=logs-router-about)

Poți folosi [IBM Cloud Logs](https://cloud.ibm.com/docs/cloud-logs) pentru a vizualiza și a primi alerte despre jurnalele de platformă generate în contul tău și direcționate de IBM Cloud Logs Routing către o instanță IBM Cloud Logs.

## Locații unde sunt generate jurnalele de platformă {#locations-where-platform-logs-are-generated}

IBM Cloud Container Registry generează jurnale de platformă care sunt afișate în instanțele tale de jurnalizare. Tabelul următor listează locațiile unde este activată colectarea automată a jurnalelor serviciului Container Registry.

| Locație                    | Sunt disponibile jurnalele serviciilor de platformă? |
|----------------------------|------------------------------------------------------|
| Washington, D.C. (us-east) |                  Da                                  |
| Frankfurt (eu-de)          |                  Da                                  |

## Jurnalele de platformă generate {#platform-logs-that-are-generated}

IBM Qiskit Runtime generează jurnale de servicii de platformă care sunt afișate în instanțele tale de jurnalizare pentru următoarele cazuri specifice:

- Când starea unui job se schimbă.

## Vizualizarea jurnalelor din pagina Observability {#viewing-logs-from-the-observability-page}

Pentru informații despre lansarea interfeței IBM Cloud Logs, consultă **Lansarea interfeței** în [documentația IBM Cloud Logs.](https://cloud.ibm.com/docs/cloud-logs?topic=cloud-logs-instance-launch)

## Câmpuri în funcție de tipul de jurnal {#fields-by-log-type}

Pentru informații despre câmpurile incluse în fiecare jurnal de platformă, consultă [Câmpuri incluse în jurnalele de platformă.](https://cloud.ibm.com/docs/logs-router?topic=logs-router-about-platform-logs#about-platform-logs-2)

Următoarele câmpuri sunt incluse în înregistrarea de jurnal.

| Câmp            | Tip      | Descriere                                                                              |
|-----------------|----------|----------------------------------------------------------------------------------------|
| logSourceCRN    | Obligatoriu | Definește contul și instanța de jurnal de flux unde este publicat jurnalul.         |
| saveServiceCopy | Obligatoriu | Definește dacă IBM salvează o copie a înregistrării în scopuri operaționale.        |
| message         | Obligatoriu | Descrierea jurnalului generat.                                                       |

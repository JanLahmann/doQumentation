Acest ghid demonstrează cum să folosești pachetul `qiskit-addon-mpf`, utilizând evoluția în timp a unui model Ising ca exemplu. Cu acest pachet, poți construi o Formulă Multi-Produs (MPF) care poate obține o eroare Trotter mai mică la măsurătorile observabilelor. Instrumentele furnizate îți permit să determini ponderile unui MPF ales, care pot fi apoi folosite pentru a recombina valorile de așteptare estimate din mai multe circuite de evoluție în timp, fiecare cu un număr diferit de pași Trotter.

Începe prin a considera Hamiltonianul unui model Ising cu 10 site-uri:

$$ H_{\text{Ising}} = \sum_{i=1}^9 J_{i,(i+1)}Z_iZ_{i+1} + \sum_{i=1}^{10} h_i X_i $$

unde $J_{i,(i+1)}$ este intensitatea cuplajului și $h_i$ este intensitatea unui câmp magnetic extern. Pentru a defini problema, observabilul de măsurat va fi magnetizarea totală a sistemului

$$ \langle M \rangle = \sum_{i=1}^{10} \langle Z_i \rangle. $$

Fragmentul de cod de mai jos pregătește Hamiltonianul lanțului Ising folosind pachetul `qiskit-addon-utils` și definește observabilul care va fi măsurat.

```python
    generate_xyz_hamiltonian,
    generate_time_evolution_circuit,

    setup_exact_problem,

)
from qiskit_addon_mpf.static import setup_static_lse

from scipy.linalg import expm
import numpy as np

# Generate some coupling map to use for this example
coupling_map = CouplingMap.from_line(10, bidirectional=False)

# Get a qubit operator describing the Ising field model
hamiltonian = generate_xyz_hamiltonian(
    coupling_map,
    coupling_constants=(0.0, 0.0, 1.0),
    ext_magnetic_field=(0.4, 0.0, 0.0),

observable = SparsePauliOp.from_sparse_list(
    [("Z", [i], 1 / L / 2) for i in range(L)], num_qubits=L
)
```

În continuare, pregătești MPF-ul. Prima alegere de făcut este dacă coeficienții vor fi statici (independenți de timp) sau dinamici; acest tutorial folosește un MPF static. Următoarea alegere este setul de valori $k_j$. Aceasta determină câți termeni va conține MPF-ul, precum și câți pași Trotter folosește fiecare termen pentru a simula evoluția în timp. Alegerea valorilor $k_j$ este euristică, deci trebuie să obții propriul set de valori $k_j$ „bune". Citește recomandările pentru găsirea unui set bun de valori la sfârșitul [paginii de primii pași](/guides/qiskit-addons-mpf#determine-mpf-coefficients).

Apoi, odată ce valorile $k_j$ sunt determinate, poți pregăti sistemul de ecuații, $Ax=b$, de rezolvat. Matricea $A$ este determinată și de formula de produs de utilizat. Alegerile disponibile sunt *ordinul* acesteia, setat la $2$ în acest exemplu, precum și dacă formula de produs ar trebui să fie *simetrică*, setat la `True` în acest exemplu. Fragmentul de cod de mai jos selectează un timp total pentru a evolua sistemul, valorile $k_j$ de utilizat și setul de ecuații de rezolvat folosind metoda [`qiskit_addon_mpf.static.setup_static_lse`](../api/qiskit-addon-mpf/static).

```python
time = 8.0
trotter_steps = (8, 12, 19)

lse = setup_static_lse(trotter_steps, order=2, symmetric=True)
print(lse)
```

Odată ce sistemul liniar de ecuații a fost instanțiat, acesta poate fi rezolvat fie exact, fie printr-un model aproximativ folosind o sumă a pătratelor (sau norma Frobenius pentru coeficienți dinamici; consultă [referința API](../api/qiskit-addon-mpf/costs) pentru mai multe informații). Alegerea de a folosi un model aproximativ apare de obicei când norma coeficienților pentru setul ales de valori $k_j$ este considerată prea mare și nu poate fi ales un alt set de valori $k_j$. Acest ghid demonstrează ambele variante pentru a compara rezultatele.

```python
model_exact, coeffs_exact = setup_exact_problem(lse)
model_approx, coeffs_approx = setup_sum_of_squares_problem(
    lse, max_l1_norm=3.0

model_exact.solve()
model_approx.solve()
print(f"Exact solution: {coeffs_exact.value}")
print(f"Approximate solution: {coeffs_approx.value}")
```

Odată ce aceste circuite au fost construite, le poți transpila și executa folosind un QPU. Pentru acest exemplu, vom folosi unul dintre simulatoarele fără zgomot pentru a demonstra cum se reduce eroarea Trotter.

```python
backend = GenericBackendV2(num_qubits=10)
transpiler = generate_preset_pass_manager(
    optimization_level=2, backend=backend
)

transpiled_circuits = [transpiler.run(circ) for circ in circuits]

estimator = StatevectorEstimator()
job = estimator.run([(circ, observable) for circ in transpiled_circuits])
result = job.result()

mpf_evs = [res.data.evs for res in result]
print(mpf_evs)
```

## Reconstruirea rezultatelor {#reconstruct-results}

Acum că circuitele au fost executate, reconstruirea rezultatelor este destul de simplă. Așa cum este menționat pe pagina de [prezentare generală MPF](./qiskit-addons-mpf#theoretical-background), observabilul nostru este reconstruit prin suma ponderată

$$ \langle M \rangle = \sum_j x_j \langle M_j \rangle. $$

unde $x_j$ sunt coeficienții găsiți și $\langle M_j \rangle$ este estimarea observabilului $\sum_i \langle Z_i \rangle$ pentru fiecare Circuit. Putem apoi compara rezultatele obținute cu valoarea exactă folosind pachetul `scipy.linalg`.

```python
exp_H = expm(-1j * time * hamiltonian.to_matrix())
initial_state = np.zeros(exp_H.shape[0])
initial_state[0] = 1.0

time_evolved_state = exp_H @ initial_state
exact_obs = (
    time_evolved_state.conj() @ observable.to_matrix() @ time_evolved_state
)

# Print out the different observable measurements
print(f"Exact value: {exact_obs.real}")
print(f"PF with 19 steps: {mpf_evs[-1]}")
print(f"MPF using exact solution: {mpf_evs @ coeffs_exact.value}")
print(f"MPF using approximate solution: {mpf_evs @ coeffs_approx.value}")
```

```text
Exact value: 0.4006024248789992
PF with 19 steps: 0.3864990619977402
MPF using exact solution: 0.3954847855979902
MPF using approximate solution: 0.4299121425348959
```

Aici poți vedea că MPF-ul a redus eroarea Trotter față de cea obținută cu un singur PF cu $k_j=19$. Totuși, modelul aproximativ a rezultat într-o valoare de așteptare mai slabă decât modelul exact. Aceasta demonstrează importanța utilizării unor criterii de convergență stricte pentru modelul aproximativ și a găsirii unui set „bun" de valori $k_j$.

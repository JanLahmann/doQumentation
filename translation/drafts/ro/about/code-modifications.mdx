# Modificări automate de cod

doQumentation aplică automat un număr mic de modificări conținutului din tutoriale
și ghidurile Qiskit din amonte pentru a asigura o experiență interactivă fluidă.
Această pagină documentează fiecare modificare, astfel încât să poți înțelege
exact ce s-a schimbat față de documentația originală IBM Quantum.

## Copii ale notebook-urilor (Open in Colab / Binder / Code Engine)

Când dai clic pe **Open in Colab**, **Open in JupyterLab** sau **Open in Code
Engine**, primești o copie a notebook-ului original cu aceste adăugări:

### 1. Celulă de notificare de configurare (markdown)

O celulă blockquote este inserată chiar în partea de sus, explicând faptul că
doQumentation a adăugat o celulă de configurare automată. Aceasta trimite către
această pagină.

### 2. Celulă de cerințe preliminare (cod)

O celulă de cod este inserată după notificare, care:

- **Instalează pachetele necesare** (`qiskit`, `qiskit-aer`, `qiskit-ibm-runtime`,
  `pylatexenc`, plus orice pachete specifice tutorialului detectate prin scanarea
  importurilor). Instalarea este **omisă** dacă pachetele sunt deja prezente
  (de exemplu, pe Binder sau Code Engine, unde sunt preinstalate).
- **Oferă un șablon de credențiale comentat** pentru IBM Quantum, astfel încât
  utilizatorii care doresc să ruleze pe hardware real să poată decomenta și să
  completeze cheia lor de API.

Pe Google Colab, această celulă se rulează automat la deschiderea notebook-ului
prin indicatorul de metadate `cell_execution_strategy: setup`.

### 3. Rescrieri ale căilor imaginilor

Căile relative ale imaginilor (`/docs/images/...`, `/learning/images/...`) sunt
rescrise pentru a funcționa corect în medii de notebook-uri de sine stătătoare.

## Pagini MDX (randare în browser)

Tutorialele afișate pe acest site sunt convertite din notebook-uri `.ipynb` sau
fișiere `.mdx` din amonte. Se aplică următoarele transformări:

- **Liniile `pip install`** sunt adăugate la blocurile de cod Python care
  importă pachete terțe, permițând execuția printr-un singur clic prin thebelab.
- **Secțiunea IBM Tutorial Survey**: O notă este adăugată pentru a clarifica
  faptul că sondajul aparține IBM Quantum și pentru a trimite către Issues-urile
  GitHub ale doQumentation pentru feedback specific site-ului.
- **Widget de feedback**: Un widget „Ți-a fost util?" este adăugat în partea
  de jos a fiecărui tutorial, urmărit prin analiza Umami, care respectă
  confidențialitatea.
- **Corecții de sintaxă MDX**: Acoladele, ierarhia antetelor și problemele de
  compatibilitate JSX sunt corectate automat pentru randarea Docusaurus.
- **OpenInLabBanner**: Un banner interactiv este injectat sub titlu, cu butoane
  pentru a deschide notebook-ul în Colab, Binder sau Code Engine.

## Ce NU este modificat

- **Conținutul tutorialelor în sine** (explicații, logica codului, ieșiri) nu
  este niciodată modificat.
- **Atribuirea autorului original** este păstrată prin frontmatter și fișierul
  NOTICE (licențele Apache 2.0 / CC BY-SA 4.0).
- **Niciun cod de telemetrie sau urmărire** nu este injectat în notebook-uri.
  Analytics (Umami) rulează doar pe site-ul doQumentation, nu în notebook-urile
  exportate.

## Codul sursă

Toate transformările sunt implementate în

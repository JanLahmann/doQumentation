În acest tutorial implementăm un [Qiskit pattern](https://quantum.cloud.ibm.com/docs/guides/intro-to-patterns) care arată cum să post-procesezi mostre cuantice zgomotoase pentru a găsi o aproximare a stării fundamentale a unui Hamiltonian fermionic de tip lattice, cunoscut sub numele de modelul Anderson cu o singură impuritate. Vom urma o abordare de diagonalizare cuantică bazată pe mostre pentru a procesa mostre preluate dintr-un set de stări din baza Krylov cu ``16`` Qubiți, peste intervale de timp din ce în ce mai mari. Aceste stări sunt realizate pe dispozitivul cuantic folosind Trotterizarea evoluției temporale. Pentru a ține cont de efectul zgomotului cuantic, se utilizează tehnica de recuperare a configurației. Presupunând o stare inițială bună și raritatea stării fundamentale, [s-a demonstrat că această abordare converge eficient](https://arxiv.org/abs/2501.09702).

Modelul poate fi descris în patru pași:

1. **Pasul 1: Mapează problema la o problemă cuantică**
    - Generează un set de stări din baza Krylov (adică, circuite de evoluție temporală Trotterizată) peste intervale de timp din ce în ce mai mari pentru estimarea stării fundamentale
2. **Pasul 2: Optimizează problema**
    - Transpilează circuitele pentru Backend
3. **Pasul 3: Execută experimentele**
    - Extrage mostre din circuite folosind primitivul ``Sampler``
4. **Pasul 4: Post-procesează rezultatele**
   - Buclă de recuperare a configurației auto-consistentă
       - Post-procesează întregul set de mostre de șiruri de biți, folosind cunoașterea prealabilă a numărului de particule și a ocupației orbitale medii calculată la iterația cea mai recentă
       - Creează probabilistic loturi de sub-mostre din șirurile de biți recuperate
       - Proiectează și diagonalizează Hamiltonianul fermionic de tip lattice pentru fiecare subspațiu eșantionat
       - Salvează energia minimă a stării fundamentale găsită în toate loturile și actualizează ocupația orbitală medie
### Pasul 1: Mapează problema la un Circuit cuantic
Mai întâi, vom crea Hamiltonienii de un corp și de două corpuri care descriu modelul Anderson cu o singură impuritate unidimensional (SIAM) cu ``7`` situri de baie (``8`` electroni în ``8`` orbitale). Acest model este folosit pentru a descrie impurități magnetice înglobate în metale.

Apoi vom crea circuitele Trotter cu ``16`` Qubiți folosite pentru a genera subspațiul Krylov cuantic.

```python
import numpy as np

n_bath = 7  # number of bath sites

V = 1  # hybridization amplitude
t = 1  # bath hopping amplitude
U = 10  # Impurity onsite repulsion
eps = -U / 2  # Chemical potential for the impurity

# Place the impurity on the first qubit
impurity_index = 0

# One body matrix elements in the "position" basis
h1e = -t * np.diag(np.ones(n_bath), k=1) - t * np.diag(np.ones(n_bath), k=-1)
h1e[impurity_index, impurity_index + 1] = -V
h1e[impurity_index + 1, impurity_index] = -V
h1e[impurity_index, impurity_index] = eps

# Two body matrix elements in the "position" basis
h2e = np.zeros((n_bath + 1, n_bath + 1, n_bath + 1, n_bath + 1))
h2e[impurity_index, impurity_index, impurity_index, impurity_index] = U
```

În continuare, vom genera subspațiul Krylov cuantic cu un set de circuite cuantice Trotterizate. Aici creăm funcții ajutătoare pentru generarea stării inițiale (de referință), precum și a evoluției temporale pentru părțile de un corp și de două corpuri ale Hamiltonianului. Pentru o descriere mai detaliată a acestui model și a modului în care sunt proiectate circuitele, te rugăm să consulți [articolul](https://arxiv.org/abs/2501.09702).

```python
import ffsim
import scipy
from qiskit import QuantumCircuit, QuantumRegister
from qiskit.circuit.library import CPhaseGate, XGate, XXPlusYYGate

n_modes = n_bath + 1
nelec = (n_modes // 2, n_modes // 2)

dt = 0.2
Utar = scipy.linalg.expm(-1j * dt * h1e)

# The reference state
def initial_state(q_circuit, norb, nocc):
    """Prepare an initial state."""
    for i in range(nocc):
        q_circuit.append(XGate(), [i])
        q_circuit.append(XGate(), [norb + i])
    rot = XXPlusYYGate(np.pi / 2, -np.pi / 2)

    for i in range(3):
        for j in range(nocc - i - 1, nocc + i, 2):
            q_circuit.append(rot, [j, j + 1])
            q_circuit.append(rot, [norb + j, norb + j + 1])
    q_circuit.append(rot, [j + 1, j + 2])
    q_circuit.append(rot, [norb + j + 1, norb + j + 2])

# The one-body time evolution
free_fermion_evolution = ffsim.qiskit.OrbitalRotationJW(n_modes, Utar)

# The two-body time evolution
def append_diagonal_evolution(dt, U, impurity_qubit, num_orb, q_circuit):
    """Append two-body time evolution to a quantum circuit."""
    if U != 0:
        q_circuit.append(
            CPhaseGate(-dt / 2 * U),
            [impurity_qubit, impurity_qubit + num_orb],
        )
```

Generează ``d`` stări evoluate temporal care specifică subspațiul Krylov cuantic. Aici am ales ``d=8``. Eroarea din eșantionarea stărilor din baza Krylov converge odată cu creșterea valorii ``d``. Rețineți că particularitățile acestei instanțe de problemă permit o compilare eficientă a evoluției de un corp folosind `OrbitalRotationJW`; totuși, în general, s-ar putea folosi [PauliEvolutionGate](https://quantum.cloud.ibm.com/docs/api/qiskit/qiskit.circuit.library.PauliEvolutionGate) din Qiskit pentru a implementa evoluția temporală Trotterizată a Hamiltonianului complet.

```python
# Generate the initial state
qubits = QuantumRegister(2 * n_modes, name="q")
init_state = QuantumCircuit(qubits)
initial_state(init_state, n_modes, n_modes // 2)
init_state.draw("mpl", scale=0.4, fold=-1)

d = 8  # Number of Krylov basis states
circuits = []
for i in range(d):
    circ = init_state.copy()
    circuits.append(circ)
    for _ in range(i):
        append_diagonal_evolution(dt, U, impurity_index, n_modes, circ)
        circ.append(free_fermion_evolution, qubits)
        append_diagonal_evolution(dt, U, impurity_index, n_modes, circ)
    circ.measure_all()
```

### Pasul 2: Optimizează problema
După ce am creat circuitele Trotterizate, le vom optimiza pentru un hardware țintă. Trebuie să alegem dispozitivul hardware înainte de optimizare. Vom folosi un backend fals de 127 de qubiți din ``qiskit_ibm_runtime`` pentru a emula un dispozitiv real. Pentru a rula pe un QPU real, înlocuiește pur și simplu backend-ul fals cu unul real. Consultă [documentația Qiskit IBM Runtime](https://quantum.cloud.ibm.com/docs/guides/get-started-with-primitives#get-started-with-sampler) pentru mai multe informații.

```python
from qiskit_ibm_runtime.fake_provider.backends import FakeSherbrooke

backend = FakeSherbrooke()
```

În continuare, vom transpila circuitele către backend-ul țintă folosind Qiskit.

```python
from qiskit.transpiler import generate_preset_pass_manager

# The circuit needs to be transpiled to the AerSimulator target
pass_manager = generate_preset_pass_manager(optimization_level=3, backend=backend)
isa_circuits = pass_manager.run(circuits)
```

### Pasul 3: Execută experimentele
După optimizarea circuitelor pentru execuția pe hardware, suntem gata să le rulăm pe hardware-ul țintă și să colectăm mostre pentru estimarea energiei stării de bază. Aici folosim ``SamplerV2`` din ``qiskit-ibm-runtime`` pentru a simula mostre zgomotoase preluate din backend-ul ``ibm_sherbrooke``. Apoi combinăm numărătorile din fiecare dintre stările bazei Krylov într-un singur dicționar de numărători și reprezentăm grafic cele mai frecvente 20 de șiruri de biți.

***Reține: [Qiskit Aer](https://qiskit.github.io/qiskit-aer/index.html) este necesar pentru a simula mostre din circuite transpirate.***

```python
from qiskit.primitives import BitArray
from qiskit.visualization import plot_histogram
from qiskit_ibm_runtime import SamplerV2 as Sampler

# Sample from the circuits
noisy_sampler = Sampler(backend, options={"simulator": {"seed_simulator": 24}})
job = noisy_sampler.run(isa_circuits, shots=500)

# Combine the counts from the individual Trotter circuits
bit_array = BitArray.concatenate_shots([result.data.meas for result in job.result()])

plot_histogram(bit_array.get_counts(), number_to_keep=20)
```

### Pasul 4: Post-procesează rezultatele
Acum rulăm algoritmul SQD folosind funcția `diagonalize_fermionic_hamiltonian`. Consultă [documentația API](../apidocs/qiskit_addon_sqd.fermion.rst#qiskit_addon_sqd.fermion.diagonalize_fermionic_hamiltonian) pentru explicații despre argumentele acestei funcții.

```python
from qiskit_addon_sqd.fermion import SCIResult, diagonalize_fermionic_hamiltonian

# List to capture intermediate results
result_history = []

def callback(results: list[SCIResult]):
    result_history.append(results)
    iteration = len(result_history)
    print(f"Iteration {iteration}")
    for i, result in enumerate(results):
        print(f"\tSubsample {i}")
        print(f"\t\tEnergy: {result.energy}")
        print(f"\t\tSubspace dimension: {np.prod(result.sci_state.amplitudes.shape)}")

rng = np.random.default_rng(24)
result = diagonalize_fermionic_hamiltonian(
    h1e,
    h2e,
    bit_array,
    samples_per_batch=300,
    norb=n_modes,
    nelec=nelec,
    num_batches=3,
    max_iterations=10,
    symmetrize_spin=True,
    callback=callback,
    seed=rng,
)
```

Acum reprezentăm grafic rezultatele. Primul grafic arată că după câteva iterații obținem energia exactă a stării de bază.

Acest exemplu este suficient de mic pentru a explora întregul spațiu Hilbert, așa cum se vede în instrucțiunile de afișare de mai sus. Reține că spațiul Hilbert complet are dimensiunea ``(num_orbitals choose nelec_a) * (num_orbitals choose nelec_b)``. Deci pentru această problemă: `(8 choose 4)**2 = 4900`. Dimensiunile subspatiului cresc datorită recuperării îmbunătățite a configurației, dar și faptului că algoritmul SQD transferă configurațiile importante de la o iterație la alta. La ultima iterație, diagonalizăm în întreg spațiul Hilbert.

Al doilea grafic arată ocupanța medie a fiecărui orbital spațial pentru soluțiile tuturor loturilor. Observăm că, cu probabilitate ridicată, fiecare orbital conține un singur electron.

### Ce este absorbția propagată a zgomotului (PNA)?

***O tehnică de mitigare a erorilor de gate prin propagarea observabilului prin canalul de noise invers care afectează porțile de 2 qubiți, rezultând un observable care mitigă noise-ul.***
Porțile 2Q din experimentul pe care dorim să-l rulăm vor fi afectate de noise substanțial.
![Noisy experiment](../images/noisy_expt.png)
Dacă învățăm modelul de noise, putem aplica inversul său și anula noise-ul.
![Noise-mitigated experiment](../images/noise_mitigated_expt.png)
În loc să implementăm canalul de noise invers prin eșantionarea sa pe QPU ca în PEC, îl putem implementa clasic în observabilul măsurat folosind propagarea Pauli. Aceasta rezultă într-un observable mai complex care, atunci când este măsurat, are efectul de a mitiga noise-ul de gate învățat.
![PNA overview](../images/pna_overview.png)
### Generează Circuit-ul Trotter oglindit și observabilul

Pentru acest experiment, vom studia dinamica temporală a unui model Ising cu kick pe 30 de site-uri pe un lanț de spinuri 1D. Hamiltonianul considerat este:

$H = -J\sum\limits_{\langle i,j \rangle} Z_iZ_j + h\sum\limits_iX_i$,

unde $J>0$ descrie cuplajul spinurilor vecine, $i<j$, iar câmpul transversal global, $h$, este setat la $\frac{\pi}{8}$. Cu cât $h$ este mai departe de un unghi Clifford (adică $\theta=n\frac{\pi}{2}, n \in \mathbb{Z}$), cu atât devine mai dificil să propagăm generatorii anti-noise prin circuit.

Pentru alegerea observabilului, vom considera magnetizarea medie pe un singur site, $\frac{1}{N} \sum_{i=1}^{N} \langle z_i \rangle$, unde $N$ este numărul de site-uri.

```python
import numpy as np
from qiskit import QuantumCircuit
from qiskit.quantum_info import Pauli, SparsePauliOp

num_qubits = 30
num_trotter_steps = 10
rx_angle = np.pi / 8

# Avg single-site magnetization
id_pauli = Pauli("I" * num_qubits)
observable = SparsePauliOp([id_pauli.dot(Pauli("Z"), [i]) for i in range(num_qubits)]) / num_qubits

# Implement Trotterized kicked-Ising model
circuit = QuantumCircuit(num_qubits)
for _step in range(num_trotter_steps):
    circuit.rx(rx_angle, range(num_qubits))
    for first_qubit in (1, 2):
        for idx in range(first_qubit, num_qubits, 2):
            # equivalent to Rzz(-pi/2):
            circuit.sdg([idx - 1, idx])
            circuit.cz(idx - 1, idx)
circuit.compose(circuit.inverse(), inplace=True)
circuit.measure_active()
circuit.draw("mpl", fold=-1)
```

### Twirl straturile de Gate-uri pe 2 Qubit-uri și măsurătorile și găsește straturile unice

Aici ne asigurăm că pass manager-ul adnotează casetele cu ``Twirl`` și ``InjectNoise``, care ne permit să învățăm noise-ul ce va afecta circuitul nostru și să asociem acel noise cu stratul de circuit corespunzător.

- ``enable_gates/enable_measure: True``: Încadrează toate straturile de Gate-uri pe 2 Qubit-uri și măsurătorile terminale. Gate-urile pe un singur Qubit vor fi lăsate îmbrăcate în interiorul casetelor.
- ``measure_annotations: all`` Include adnotările `Twirl` și `ChangeBasis` pe caseta de măsurare
- ``twirling_strategy: active``: Twirl pe toți Qubit-urile active din fiecare casetă care conține Gate-uri de entanglement
- ``inject_noise_targets: gates``: Adnotările ``InjectNoise`` trebuie adăugate la toate casetele adnotate cu ``Twirl`` care conțin Gate-uri de entanglement
- ``inject_noise_strategy: uniform_modification``: Toate straturile de noise trebuie scalate în mod echivalent.

#### Învață zgomotul

Înainte de a rula experimentele, învățăm modelul de noise care afectează Gate-urile de entanglement și măsurătorile din circuit. Existența unui model de noise precis este necesară pentru a mitiga eficient erorile. Învățarea noise-ului imediat înainte de executarea experimentelor oferă cea mai bună șansă ca modelul de noise să descrie fidel noise-ul real care afectează Gate-urile în timpul execuției.

Înainte de a învăța noise-ul, trebuie să găsim straturile unice pe 2 Qubit-uri din circuitul nostru, pentru a minimiza numărul de măsurători necesare să învățăm noise-ul pentru întregul circuit. Folosim ``find_unique_box_instructions`` din ``samplomatic`` pentru a obține straturile unice din circuitul cu casete, inclusiv stratul de măsurare. Acestea sunt straturile pe care le transmitem noise learner-ului.

Odată ce cunoaștem straturile, putem învăța noise-ul. Există câțiva parametri pe care îi luăm în considerare:

- `num_randomizations`: Numărul de circuite aleatorii utilizate per configurație de circuit de învățare
- `shots_per_randomization`: Numărul total de măsurători utilizate per circuit aleatoriu de învățare
- `layer_pair_depths`: Adâncimile de circuit (măsurate în număr de perechi) utilizate în experimentele de învățare.
- `post_selection`: Vom folosi post-selecție bazată pe muchii în timpul învățării, utilizând Gate-uri `rx` pentru a implementa pulsurile post-măsurare


#### Asociază cutiile Circuit-ului cu zgomotul învățat

Aici, creăm o mapare între ID-urile de referință ``InjectNoise`` ale fiecărei cutii și modelul de zgomot învățat (`PauliLindbladMap`) care afectează porțile de împletire din acea cutie.

```python
from samplomatic.annotations import InjectNoise
from samplomatic.utils import get_annotation

# map inject noise refs to pauli lindblad maps
refs_to_noise_models = {}
for instruction, result in zip(unique_2q_layers_and_meas, noise_learner_result, strict=False):
    if inject_noise_annot := get_annotation(instruction.operation, InjectNoise):
        refs_to_noise_models[inject_noise_annot.ref] = result.to_pauli_lindblad_map()
```

#### Propagă observabilul prin anti-zgomotul învățat pentru a obține un observabil de atenuare a zgomotului

Așa cum s-a discutat mai sus, acest lucru se face în doi pași. Mai întâi, propagăm un generator anti-noise până la sfârșitul Circuit-ului. Apoi, propagăm observabilul prin acel generator evoluat. Acest proces se repetă pentru fiecare generator anti-noise din Circuit. În această implementare, fiecare generator dintr-un strat dat este propagat până la sfârșitul Circuit-ului în paralel. În plus, multiprocessing-ul Python este folosit pentru a efectua atât propagarea înainte a anti-noise-ului, cât și propagarea înapoi a observabilului în paralel. Aceasta previne acumularea generatorilor evoluați în memorie și maximizează resursele de calcul.

Când rulezi PNA, va trebui întotdeauna să furnizezi un Circuit zgomotos și un observable. Dacă Circuit-ul tău zgomotos este un Circuit în cutii cu adnotări `InjectNoise`, va trebui să furnizezi maparea creată în pasul de mai sus. Se poate transmite, de asemenea, un Circuit fără cutii care conține instrucțiuni ``PauliLindbladError`` din ``qiskit-aer``. În acel caz, ``refs_to_noise_models`` nu trebuie furnizat. Pe lângă intrările primare, utilizatorii vor dori să ia în considerare:

- `max_err_terms`: Numărul de termeni de păstrat în fiecare generator anti-noise pe măsură ce este propagat înainte. A permite ca acesta să fie mai mare crește, în general, precizia, dar nu se garantează că acest comportament este monotonic.
- `max_obs_terms`: Numărul de termeni de păstrat în observabilul de atenuare a zgomotului, $\tilde{O}$, pe măsură ce este propagat înapoi prin generatorul anti-noise evoluat. Valorile mai mari cresc, în general, precizia, dar nu este garantat că o fac în mod monotonic.
- `num_processes`: Numărul de nuclee dedicate procesului. Reține că generatorii sunt propagați înainte și aplicați observabilului în paralel.
- `search_step`: Pasul de propagare înapoi folosește o metodă greedy pentru a conjuga aproximativ doi operatori în baza Pauli. Această metodă poate fi accelerată prin mărirea ``search_step``. Vezi [documentația pauli-prop](https://qiskit.github.io/pauli-prop/) pentru mai multe informații.
- `num_to_measure`: Deși această variabilă nu este o intrare pentru ``generate_noise_mitigating_observable``, o folosim pentru a controla câți termeni din $\tilde{O}$ dorim să măsurăm efectiv. Aici vom măsura doar primii 30 de termeni, care sunt termenii originali din observabilul nostru. Termenii au fost acum re-scalați astfel încât măsurarea lor are efectul de a atenua zgomotul de poartă învățat. Deși măsurăm doar 30 de termeni din $\tilde{O}$, este adesea util să îi permitem să crească, deoarece aceasta crește precizia factorilor de scalare ai termenilor de frunte.

```python
from qiskit_addon_pna import generate_noise_mitigating_observable

# PNA parameters
num_processes = 8
max_err_terms = 10_000
max_obs_terms = 10_000
num_to_measure = num_qubits

obs_tilde_isa = generate_noise_mitigating_observable(
    boxed_circuit,
    isa_observable,
    refs_to_noise_models,
    max_err_terms=max_err_terms,
    max_obs_terms=max_obs_terms,
    num_processes=num_processes,
    print_progress=True,
    search_step=8,
)
p_2_v = {p: v for v, p in enumerate(layout)}
obs_tilde_virtual = SparsePauliOp.from_sparse_list(
    [
        (pstr, [p_2_v[p] for p in p_qubits], coeff)
        for (pstr, p_qubits, coeff) in obs_tilde_isa.to_sparse_list()
    ],
    num_qubits=num_qubits,
)
obs_tilde_virtual = obs_tilde_virtual[np.argsort(np.abs(obs_tilde_virtual.coeffs))[::-1]][
    :num_to_measure
]
```

#### Transformă bazele de măsurare în forma canonică

În continuare, vom găsi un set minimal de baze de măsurat astfel încât să putem acoperi complet fiecare termen Pauli din observabilul măsurat (***mulți observabili pot fi măsurați simultan dacă comutează qubit cu qubit***). Deoarece măsurăm doar termenii din observabilul nostru original, care este suma tuturor Pauli-ilor de tip `Z` singular, este necesară o singură bază -- baza all-`Z`.

Pe lângă găsirea unui set de baze de măsurare Pauli, trebuie să mapăm acești termeni Pauli la forma canonică așteptată de primitivul ``Executor``. Pentru mai multe informații despre ordonarea canonică a Qubit-urilor, vizitează [documentația samplomatic](https://qiskit.github.io/samplomatic/guides/samplex_io.html#qubit-ordering-convention).

```python
from qiskit_addon_utils.exp_vals.measurement_bases import get_measurement_bases

meas_box = boxed_circuit.data[-1]
canonical_qubits = [
    idx for idx, qubit in enumerate(boxed_circuit.qubits) if qubit in meas_box.qubits
]
c_2_p = {c: p for c, p in enumerate(canonical_qubits)}  # canonical -> physical
p_2_v = {p: v for v, p in enumerate(layout)}  # physical -> virtual
c_2_v = {c: p_2_v[p] for c, p in c_2_p.items()}  # canonical -> virtual
meas_bases, bases_reverser = get_measurement_bases(obs_tilde_virtual)
meas_bases_canonical = [
    np.array([base[c_2_v[c]] for c in range(num_qubits)], dtype=np.uint8) for base in meas_bases
]
```

#### Specifică cum să eșantionezi în ``QuantumProgram``

``QuantumProgram`` este locul în care specificăm cum să eșantionăm experimentul:

- ``template_circuit``: Circuit-ul care conține toate Gate-urile necesare pentru a implementa toate randomizările dorite (din randomizările de twirl, parametri etc.).
- ``samplex``: Un obiect care definește o distribuție de probabilitate peste toate randomizările posibile ale Circuit-ului din care să se eșantioneze.
- ``samplex_arguments``: Legăturile necesare pentru a defini complet samplex-ul
    - ``basis_changes``: Aici specificăm un set de baze de măsurat care vor acoperi toți termenii Pauli din observable-ul măsurat.
    - ``noise_scales.ref``: Setăm scala fiecărui strat de zgomot la `0.0` pentru a preveni injectarea oricărui zgomot suplimentar în eșantioanele noastre.
    - ``pauli_lindblad_maps``: Necesar dacă se transmit ``noise_scales``. Acesta mapează pur și simplu straturile de zgomot la modelul de zgomot asociat.
- ``shape``: Un tuplu de formă pentru a extinde forma implicită definită de ``samplex_arguments``. Axele non-triviale introduse de această extensie enumeră randomizările.

```python
from qiskit_ibm_runtime import QuantumProgram

# Control the # of shots during execution
shots_per_randomization_exec = 64
num_randomizations_exec = 6144

# Zero out the noise to prevent noise from being injected during execution.
# We only added InjectNoise annotations so PNA could associate the noise
# to layers in the circuit
samplex_inputs = {f"noise_scales.{ref}": 0.0 for ref in refs_to_noise_models}
samplex_inputs |= {"pauli_lindblad_maps": refs_to_noise_models}

# Specify the bases to measure
bases_broadcastable = np.expand_dims(np.array(meas_bases_canonical), axis=1)
samplex_inputs |= {"basis_changes": {"basis0": bases_broadcastable}}

# Convert samplex_inputs into a dict to pass to QuantumProgram
samplex_arguments = samplex.inputs().make_broadcastable().bind(**samplex_inputs)

# Instantiate the QuantumProgram with the specified parameters
program = QuantumProgram(shots=shots_per_randomization_exec)
program.append(
    circuit=template_circuit,
    samplex=samplex,
    samplex_arguments=samplex_arguments,
    shape=(num_randomizations_exec),
)
```

#### Post-procesează eșantioanele pentru a calcula o valoare de așteptare cu erori atenuate

Pentru a calcula o valoare de așteptare cu erori atenuate, vom:

- Calcula factorii de scalare TREX pe baza zgomotului învățat care afectează măsurătorile
- Genera o mască pentru păstrarea doar a eșantioanelor post-selectate
- Folosi funcția ``executor_expectation_values`` din ``qiskit-addon-utils`` pentru a combina toate datele într-o valoare de așteptare cu erori atenuate.

```python
from qiskit_addon_utils.exp_vals.expectation_values import executor_expectation_values
from qiskit_addon_utils.noise_management import trex_factors
from qiskit_addon_utils.noise_management.post_selection import PostSelector

# Computing the TREX factors
```

În acest notebook, vom parcurge pașii unui [pattern Qiskit](https://quantum.cloud.ibm.com/docs/guides/intro-to-patterns) folosind **circuit cutting** pentru a reduce numărul de qubiți dintr-un Circuit. Vom tăia gate-uri pentru a putea reconstrui valoarea de așteptare a unui Circuit cu patru qubiți folosind doar experimente cu doi qubiți.

Aceștia sunt pașii pe care îi vom urma:

- **Pasul 1: Maparea problemei pe circuite și operatori cuantici**:
    - Maparea hamiltonianului pe un Circuit cuantic.
- **Pasul 2: Optimizare pentru hardware-ul țintă** [_Folosește addon-ul de cutting_]:
    - <font color='#0F62FE'>Tăierea Circuitului și a observabilei.</font>
    - Transpilarea subexperimentelor pentru hardware.
- **Pasul 3: Execuție pe hardware-ul țintă**:
    - Rularea subexperimentelor obținute în Pasul 2 folosind primitiva `Sampler`.
- **Pasul 4: Post-procesarea rezultatelor** [_Folosește addon-ul de cutting_]:
    - <font color='#0F62FE'>Combinarea rezultatelor din Pasul 3 pentru a reconstrui valoarea de așteptare a observabilei în cauză.</font>

## Pasul 2: Optimizare

### Separarea Circuitului și a observabilei conform unei partiționări specificate a qubiților

Fiecare etichetă din `partition_labels` corespunde qubitului din `circuit` cu același index. Qubiții care au o etichetă de partiție comună vor fi grupați împreună, iar gate-urile non-locale care acoperă mai mult de o partiție vor fi tăiate.

**Notă:** Argumentul ``observables`` pentru `partition_problem` este de tipul `PauliList`. Coeficienții și fazele termenilor observabilei sunt ignorați în timpul descompunerii problemei și al execuției subexperimentelor. Aceștia pot fi reaplicați în timpul reconstrucției valorii de așteptare.

```python
from qiskit_addon_cutting import partition_problem

partitioned_problem = partition_problem(
    circuit=qc, partition_labels="AABB", observables=observable.paulis
)
subcircuits = partitioned_problem.subcircuits
subobservables = partitioned_problem.subobservables
bases = partitioned_problem.bases
```

### Generează subexperimentele pentru a rula pe Backend

`generate_cutting_experiments` acceptă argumentele `circuits`/`observables` ca dicționare ce mapează etichetele partițiilor de Qubit la `subcircuit`/`subobservables` respective.

Pentru a simula valoarea așteptată a circuitului de dimensiune completă, din distribuția de quasiprobabilitate a porților descompuse sunt generate multe subexperimente, care sunt apoi executate pe unul sau mai multe Backends. Numărul de eșantioane preluate din distribuție este controlat de `num_samples`, iar un coeficient combinat este dat pentru fiecare eșantion unic. Pentru mai multe informații despre modul în care sunt calculați coeficienții, consultă [materialul explicativ](../explanation/index.rst).

```python
from qiskit_addon_cutting import generate_cutting_experiments

subexperiments, coefficients = generate_cutting_experiments(
    circuits=subcircuits, observables=subobservables, num_samples=np.inf
)
```

### Alege un Backend

Aici folosim un Backend simulat, ceea ce va face ca Qiskit Runtime să ruleze în modul local (adică, pe un simulator local).

```python
from qiskit_ibm_runtime.fake_provider import FakeManilaV2

backend = FakeManilaV2()
```

### Pregătește subexperimentele pentru Backend

Trebuie să transpilăm Circuit-urile cu Backend-ul nostru ca țintă înainte de a le trimite către Qiskit Runtime.

```python
from qiskit.transpiler import generate_preset_pass_manager

# Transpile the subexperiments to ISA circuits
pass_manager = generate_preset_pass_manager(optimization_level=1, backend=backend)
isa_subexperiments = {
    label: pass_manager.run(partition_subexpts)
    for label, partition_subexpts in subexperiments.items()
}
```

## Pasul 3: Executare

### Rulează subexperimentele folosind primitiva Sampler din Qiskit Runtime

```python
from qiskit_ibm_runtime import SamplerV2, Batch

# Submit each partition's subexperiments to the Qiskit Runtime Sampler
# primitive, in a single batch so that the jobs will run back-to-back.
with Batch(backend=backend) as batch:
    sampler = SamplerV2(mode=batch)
    jobs = {
        label: sampler.run(subsystem_subexpts, shots=2**12)
        for label, subsystem_subexpts in isa_subexperiments.items()
    }
```

## Pasul 4: Post-procesare

### Reconstituie valoarea așteptată

Reconstituie valorile așteptate pentru fiecare termen al observabilului și combină-le pentru a reconstitui valoarea așteptată a observabilului original.

```python
from qiskit_addon_cutting import reconstruct_expectation_values

# Get expectation values for each observable term
reconstructed_expval_terms = reconstruct_expectation_values(
    results,
    coefficients,
    subobservables,
)

# Reconstruct final expectation value
reconstructed_expval = np.dot(reconstructed_expval_terms, observable.coeffs)
```

### Compară valoarea așteptată reconstituită cu valoarea așteptată exactă din Circuit-ul și observabilul original

```python
from qiskit_aer.primitives import EstimatorV2

estimator = EstimatorV2()
exact_expval = estimator.run([(qc, observable)]).result()[0].data.evs
print(f"Reconstructed expectation value: {np.real(np.round(reconstructed_expval, 8))}")
print(f"Exact expectation value: {np.round(exact_expval, 8)}")
print(f"Error in estimation: {np.real(np.round(reconstructed_expval-exact_expval, 8))}")
print(
    f"Relative error in estimation: {np.real(np.round((reconstructed_expval-exact_expval) / exact_expval, 8))}"
)
```

Neste notebook, você aprenderá como usar uma **Fórmula Multi-Produto (MPF)** para obter um erro de Trotter menor no nosso observável em comparação com o incorrido pelo circuito de Trotter mais profundo que de fato vamos executar.
Você fará isso seguindo as etapas de um **padrão Qiskit**:

- **Etapa 1: Mapear para o problema quântico**
    - Inicializar o Hamiltoniano do nosso problema
    - <font color="#0F62FE">Usar uma MPF para gerar os circuitos de evolução temporal Trotterizados</font>
- **Etapa 2: Otimizar o problema**
    - Aqui nós transpilamos nossos circuitos para um [GenericBackendV2](https://quantum.cloud.ibm.com/docs/api/qiskit/qiskit.providers.fake_provider.GenericBackendV2)
- **Etapa 3: Executar experimentos**
    - Usar um [StatevectorEstimator](https://quantum.cloud.ibm.com/docs/api/qiskit/qiskit.primitives.StatevectorEstimator) por simplicidade neste notebook
- **Etapa 4: Reconstruir os resultados**
    - <font color="#0F62FE">Calcular o valor esperado da MPF</font>
## Etapa 1: Mapear para o problema quântico

### 1a: Configurando nosso Hamiltoniano

Usamos o modelo de Ising em uma cadeia de 10 sítios:

$$
\hat{\mathcal{H}}_{\text{Ising}} = \sum_{i=1}^{9} J_{i,(i+1)} Z_i Z_{(i+1)} + \sum_{i=1}^{10} h_i X_i \, ,
$$

onde $J$ é a intensidade de acoplamento entre dois sítios e $h$ é o campo magnético externo.
O pacote [qiskit_addon_utils](https://qiskit.github.io/qiskit-addon-utils/) fornece algumas funcionalidades reutilizáveis para diversos fins.

Seu módulo [qiskit_addon_utils.problem_generators](https://qiskit.github.io/qiskit-addon-utils/stubs/qiskit_addon_utils.problem_generators.html) fornece funções para gerar Hamiltonianos do tipo Heisenberg em um dado grafo de conectividade.
Esse grafo pode ser tanto um [rustworkx.PyGraph](https://www.rustworkx.org/apiref/rustworkx.PyGraph.html) quanto um [CouplingMap](https://quantum.cloud.ibm.com/docs/api/qiskit/qiskit.transpiler.CouplingMap), tornando fácil o uso em fluxos de trabalho centrados no Qiskit.

A seguir, criamos uma simples cadeia de 10 qubits usando o método `CouplingMap.from_line`.

### 1b: Fórmulas Multi-Produto

MPFs reduzem o erro de Trotter da dinâmica hamiltoniana por meio de uma combinação ponderada de várias execuções de circuitos.

Para tornar isso mais concreto, definimos uma MPF como:

$$
\mu(t) = \sum_{j} x_j \rho^{k_j}_{j}\left(\frac{t}{k_j}\right) + \text{some remaining Trotter error} \, ,
$$

onde $x_j$ são nossos coeficientes de ponderação, $\rho^{k_j}_j$ é a matriz de densidade correspondente ao estado puro obtido pela evolução do estado inicial com a fórmula de produto, $S^{k_j}$, envolvendo $k_j$ passos de Trotter, e $j$ indexa o número de fórmulas de produto que compõem a MPF.

O ponto-chave aqui é que o erro de Trotter restante é menor do que o erro de Trotter que se obteria simplesmente usando o maior valor de $k_j$!

Você pode enxergar a utilidade da MPF de duas perspectivas:

1. Para um orçamento fixo de passos de Trotter que você consegue executar, é possível obter resultados com um erro de Trotter menor no total.
2. Para um número de passos de Trotter que resulta em circuitos profundos, você pode usar MPF para encontrar vários circuitos de profundidade menor para executar, que resultam em um erro de Trotter similar.
#### Uma introdução às MPFs estáticas

As MPFs _estáticas_ são aquelas em que os valores $x_j$ **NÃO** dependem do tempo de evolução, $t$.

Determinar os coeficientes de uma MPF estática para um dado conjunto de valores $k_j$ equivale a resolver um sistema linear de equações:
$Ax=b$, onde $x$ são nossos coeficientes de interesse, $A$ é uma matriz que depende de $k_j$ e do tipo de PF que usamos ($S$), e $b$ é um vetor de restrições.
Por brevidade, não vamos entrar em mais detalhes aqui e, em vez disso, referimos você à documentação de [LSE](https://qiskit.github.io/qiskit-addon-mpf/apidocs/qiskit_addon_mpf.costs.html#qiskit_addon_mpf.costs.LSE).

Podemos encontrar uma solução para $x$ analiticamente como $x = A^{-1}b$, veja por exemplo [Carrera Vazquez et al., 2023] ou [Zhuk et al., 2023].
No entanto, essa solução exata pode ser _"mal-condicionada"_, resultando em normas L1 muito grandes dos nossos coeficientes, $x$, o que pode levar a um mau desempenho da MPF.
Em vez disso, também é possível obter uma solução aproximada que minimiza a norma L1 de $x$ para tentar otimizar o comportamento da MPF.

A seguir, você aprenderá como fazer tudo isso.

[Carrera Vazquez et al., 2023]: https://quantum-journal.org/papers/q-2023-07-25-1067/
[Zhuk et al., 2023]: https://journals.aps.org/prresearch/abstract/10.1103/PhysRevResearch.6.033309
#### Escolhendo $k_j$

A escolha de $k_j$ fica a critério do usuário final.
Em princípio, quaisquer valores podem ser escolhidos, mas alguns $k_j$'s levarão a uma maior amplificação de ruído em dispositivos reais do que outras escolhas.
Portanto, é importante tentar encontrar valores _"bons"_ de $k_j$.

Aqui, simplesmente escolheremos alguns valores fixos para $k_j$.
O menor valor é motivado pelo tempo de evolução alvo de $t=8.0$, que normalmente nos diz que devemos satisfazer $t/k_{\text{min}} \lt 1$, mas empiricamente sabemos que definir igual a $1$ geralmente também funciona.
Se você quiser saber mais sobre isso e como escolher seus outros valores de $k_j$, consulte o guia correspondente: [How to choose the Trotter steps for an MPF](https://qiskit.github.io/qiskit-addon-mpf/how_tos/choose_trotter_steps.html).

```python
time = 8.0
trotter_steps = (8, 12, 19)
```

#### Configurando o LSE

Agora que escolhemos nossos $k_j$s, devemos primeiro construir o LSE, $Ax=b$ conforme explicado acima.
A matriz $A$ depende não só de $k_j$, mas também da nossa escolha de fórmula de produto (PF) — em particular sua _ordem_.
Além disso, pode-se levar em conta se a PF é simétrica ou não (veja [Carrera Vazquez et al., 2023]), definindo `symmetric=True`.
No entanto, isso não é obrigatório, conforme demonstrado por [Zhuk et al., 2023].

Aqui, vamos usar uma fórmula de Suzuki-Trotter de segunda ordem, resultando em `order=2`, e definiremos `symmetric=True`.

[Carrera Vazquez et al., 2023]: https://quantum-journal.org/papers/q-2023-07-25-1067/
[Zhuk et al., 2023]: https://journals.aps.org/prresearch/abstract/10.1103/PhysRevResearch.6.033309

```python
from qiskit_addon_mpf.static import setup_static_lse

lse = setup_static_lse(trotter_steps, order=2, symmetric=True)
print(lse)
```

```text
LSE(A=array([[1.00000000e+00, 1.00000000e+00, 1.00000000e+00],
       [1.56250000e-02, 6.94444444e-03, 2.77008310e-03],
       [2.44140625e-04, 4.82253086e-05, 7.67336039e-06]]), b=array([1., 0., 0.]))
```

#### Resolvendo $x$ analiticamente

Como mencionado antes, podemos encontrar $x$ analiticamente:

```python
import numpy as np

coeffs_analytical = lse.solve()
print(coeffs_analytical)
```

```text
[ 0.17239057 -1.19447005  2.02207947]
```

#### Otimizando $x$ com um modelo exato

Como alternativa ao cálculo de $x=A^{-1}b$, você também pode usar [setup_exact_problem](https://qiskit.github.io/qiskit-addon-mpf/stubs/qiskit_addon_mpf.costs.setup_exact_problem.html) para construir uma instância de [cvxpy.Problem](https://www.cvxpy.org/api_reference/cvxpy.problems.html#cvxpy.Problem) que usa o LSE como restrições e cuja solução ótima produzirá $x$.

Na próxima seção, ficará claro por que essa interface existe.

```python
from qiskit_addon_mpf.costs import setup_exact_problem

model_exact, coeffs_exact = setup_exact_problem(lse)
model_exact.solve()
print(coeffs_exact.value)
```

```text
[ 0.17239057 -1.19447005  2.02207947]
```

Como indicador de se uma MPF construída com esses coeficientes produzirá bons resultados, podemos usar a norma L1 (veja também [Carrera Vazquez et al., 2023]).

[Carrera Vazquez et al., 2023]: https://quantum-journal.org/papers/q-2023-07-25-1067/

```python
print(np.linalg.norm(coeffs_exact.value, ord=1))
```

```text
3.3889400921655914
```

#### Otimizando $x$ com um modelo aproximado

Pode acontecer que a norma L1 para o conjunto de valores $k_j$ escolhido seja considerada muito alta.
Nesse caso, se você não puder escolher um conjunto diferente de valores $k_j$, pode usar uma solução aproximada para o LSE em vez de uma solução exata.

Para fazer isso, basta usar [setup_sum_of_squares_problem](https://qiskit.github.io/qiskit-addon-mpf/stubs/qiskit_addon_mpf.costs.setup_sum_of_squares_problem.html) para construir uma instância diferente de [cvxpy.Problem](https://www.cvxpy.org/api_reference/cvxpy.problems.html#cvxpy.Problem) que restringe a norma L1 a um limite escolhido enquanto minimiza a diferença entre $Ax$ e $b$.

```python
from qiskit_addon_mpf.costs import setup_sum_of_squares_problem

model_approx, coeffs_approx = setup_sum_of_squares_problem(lse, max_l1_norm=3.0)
model_approx.solve()
print(coeffs_approx.value)
print(np.linalg.norm(coeffs_approx.value, ord=1))
```

```text
[-0.40454257  0.57553173  0.8290123 ]
1.8090865903790838
```

Observe que você tem total liberdade sobre como resolver esse problema de otimização, ou seja, pode alterar o solver de otimização, seus limiares de convergência, e assim por diante.
Confira o guia correspondente em [How to use the approximate model](https://qiskit.github.io/qiskit-addon-mpf/how_tos/using_approximate_model.html).
### 1c: Configurando os circuitos de Trotter

Neste ponto, encontramos nossos coeficientes de expansão, $x$, e tudo que resta fazer é gerar os circuitos quânticos Trotterizados.
Mais uma vez, o módulo [qiskit_addon_utils.problem_generators](https://qiskit.github.io/qiskit-addon-utils/stubs/qiskit_addon_utils.problem_generators.html) vem ao resgate para fazer exatamente isso:

```python
from qiskit.synthesis import SuzukiTrotter
from qiskit_addon_utils.problem_generators import generate_time_evolution_circuit

circuits = []
for k in trotter_steps:
    circ = generate_time_evolution_circuit(
        hamiltonian,
        synthesis=SuzukiTrotter(order=2, reps=k),
        time=time,
    )
    circuits.append(circ)
```

## Etapa 2: Otimizar o problema

Normalmente, esta é a etapa do padrão durante a qual você otimiza seus circuitos para execução em hardware.
Aqui, como usamos apenas um simulador sem ruído, simplesmente transpilamos nosso circuito para um [GenericBackendV2](https://quantum.cloud.ibm.com/docs/api/qiskit/qiskit.providers.fake_provider.GenericBackendV2).

```python
from qiskit.providers.fake_provider import GenericBackendV2
from qiskit.transpiler import generate_preset_pass_manager

backend = GenericBackendV2(num_qubits=10)
transpiler = generate_preset_pass_manager(optimization_level=2, backend=backend)

transpiled_circuits = [transpiler.run(circ) for circ in circuits]
```

## Etapa 3: Executar experimentos quânticos

Como explicado no início, vamos pular a etapa de otimização 2 porque simplesmente vamos calcular os valores esperados do nosso observável alvo usando um simulador sem ruído, a saber, o [StatevectorEstimator](https://quantum.cloud.ibm.com/docs/api/qiskit/qiskit.primitives.StatevectorEstimator).

```python
from qiskit.primitives import StatevectorEstimator

estimator = StatevectorEstimator()
job = estimator.run([(circ, observable) for circ in transpiled_circuits])
result = job.result()
```

## Etapa 4: Reconstruir os resultados

Primeiro, extraímos os valores esperados individuais obtidos para cada um dos circuitos de Trotter:

```python
evs = [res.data.evs for res in result]
print(evs)
```

```text
[array(0.23799162), array(0.35754312), array(0.38649906)]
```

Em seguida, simplesmente os recombinamos com nossos coeficientes MPF para obter os valores esperados totais da MPF. Abaixo, fazemos isso para cada uma das diferentes formas pelas quais calculamos $x$.

```python
print("Analytical    solution:", evs @ coeffs_analytical)
print("Exact model   solution:", evs @ coeffs_exact.value)
print("Approx. model solution:", evs @ coeffs_approx.value)
```

```text
Analytical    solution: 0.3954847855980006
Exact model   solution: 0.39548478559800204
Approx. model solution: 0.42991214253489807
```

Por fim, para este pequeno problema podemos calcular o valor de referência exato usando [scipy.linalg.expm](https://docs.scipy.org/doc/scipy/reference/generated/scipy.linalg.expm.html) da seguinte forma:

```python
from scipy.linalg import expm

exp_H = expm(-1j * time * hamiltonian.to_matrix())

initial_state = np.zeros(exp_H.shape[0])
initial_state[0] = 1.0

time_evolved_state = exp_H @ initial_state

exact_obs = time_evolved_state.conj() @ observable.to_matrix() @ time_evolved_state
print(exact_obs.real)
```

```text
0.40060242487899755
```

Podemos ver claramente que a MPF reduziu o erro de Trotter em comparação com o obtido pela PF individual mais profunda com $k_j=19$.
No entanto, também vemos que o modelo aproximado não é perfeito, já que de fato resultou em um valor esperado pior do que a solução exata. Isso mostra a importância de usar critérios de convergência rígidos no modelo aproximado, como você aprenderá no guia [How to use the approximate model](https://qiskit.github.io/qiskit-addon-mpf/how_tos/using_approximate_model.html).

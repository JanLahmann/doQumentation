A retropropagação de operadores é uma técnica que envolve absorver operações do final de um circuito quântico em um operador de Pauli, geralmente reduzindo a profundidade do circuito ao custo de termos adicionais no operador. O objetivo é retropropagar o máximo possível do circuito sem permitir que o operador cresça demais.

Uma forma de permitir uma retropropagação mais profunda no circuito, enquanto se impede que o operador cresça demais, é truncar termos com coeficientes pequenos, em vez de adicioná-los ao operador. Truncar termos pode resultar em menos circuitos quânticos a executar, mas fazê-lo resulta em algum erro no cálculo final do valor esperado proporcional à magnitude dos coeficientes dos termos truncados.
Neste tutorial vamos implementar um [padrão Qiskit](https://quantum.cloud.ibm.com/docs/guides/serverless#qiskit-patterns-with-quantum-serverless) para simular a dinâmica quântica de uma cadeia de spin de Heisenberg usando retropropagação de operadores:

- **Etapa 1: Mapear para o problema quântico**
    - Mapear o Hamiltoniano evoluído no tempo para um circuito quântico
- **Etapa 2: Otimizar o problema**
    - Fatiar o circuito
    - <font color="#0F62FE">Retropropagar fatias do circuito para um observável de Pauli</font>
    - Combinar as fatias restantes em um único circuito
    - Transpilar o circuito para o Backend
- **Etapa 3: Executar experimentos**
    - Calcular o valor esperado usando o circuito reduzido e o observável expandido com um [StatevectorEstimator](https://quantum.cloud.ibm.com/docs/api/qiskit/qiskit.primitives.StatevectorEstimator) por simplicidade neste notebook
- **Etapa 4: Reconstruir resultados**
    - N.A.

**Nota:** O Qiskit descreve genericamente [camadas](https://quantum.cloud.ibm.com/docs/api/qiskit/qiskit.dagcircuit.DAGCircuit) como partições de profundidade 1 do circuito em todos os Qubits. Este pacote utiliza o termo **fatias** para descrever camadas de profundidade arbitrária. A função [qiskit_addon_obp.backpropagate](https://qiskit.github.io/qiskit-addon-obp/stubs/qiskit_addon_obp.backpropagate.html) é projetada para retropropagar fatias inteiras de cada vez, portanto a escolha de como fatiar o circuito quântico pode ter um impacto significativo em quão bem a retropropagação funciona para um dado problema. Você vai aprender mais sobre **fatias** abaixo.
## Etapa 1: Mapear para o problema quântico
### Mapear a evolução temporal de um modelo quântico de Heisenberg para um experimento quântico.

O pacote [qiskit_addon_utils](https://qiskit.github.io/qiskit-addon-utils/) fornece algumas funcionalidades reutilizáveis para diversos fins.

Seu módulo [qiskit_addon_utils.problem_generators](https://qiskit.github.io/qiskit-addon-utils/stubs/qiskit_addon_utils.problem_generators.html) fornece funções para gerar Hamiltonianos do tipo Heisenberg em um dado grafo de conectividade.
Esse grafo pode ser um [rustworkx.PyGraph](https://www.rustworkx.org/apiref/rustworkx.PyGraph.html) ou um [CouplingMap](https://quantum.cloud.ibm.com/docs/api/qiskit/qiskit.transpiler.CouplingMap), tornando-o fácil de usar em fluxos de trabalho centrados no Qiskit.

Em seguida, geramos um operador de Pauli modelando um Hamiltoniano XYZ de Heisenberg.

$$\hat{H} = \sum_{(j,k)\in E} (J_{x} \sigma_j^{x} \sigma_{k}^{x} +
    J_{y} \sigma_j^{y} \sigma_{k}^{y} + J_{z} \sigma_j^{z} \sigma_{k}^{z}) +
    \sum_{j\in V} (h_{x} \sigma_j^{x} + h_{y} \sigma_j^{y} + h_{z} \sigma_j^{z})$$

Onde $G(V,E)$ é o grafo do mapa de acoplamento fornecido.

```python
import numpy as np

# Get a qubit operator describing the Heisenberg XYZ model
hamiltonian = generate_xyz_hamiltonian(
    reduced_coupling_map,
    coupling_constants=(np.pi / 8, np.pi / 4, np.pi / 2),
    ext_magnetic_field=(np.pi / 3, np.pi / 6, np.pi / 9),
)
print(hamiltonian)
```

## Etapa 2: Otimizar o problema
### Criar fatias do circuito para retropropagar

Lembre-se, a função ``backpropagate`` irá retropropagar fatias inteiras do circuito de cada vez, portanto a escolha de como fatiar pode impactar o desempenho da retropropagação para um dado problema. Aqui, vamos agrupar Gates do mesmo tipo em fatias usando a função [slice_by_gate_types](https://qiskit.github.io/qiskit-addon-utils/stubs/qiskit_addon_utils.slicing.slice_by_gate_types.html).

Para uma discussão mais detalhada sobre fatiamento de circuitos, confira este [guia prático](https://qiskit.github.io/qiskit-addon-utils/how_tos/create_circuit_slices.html) do pacote [qiskit-addon-utils](https://qiskit.github.io/qiskit-addon-utils/index.html).

```python
from qiskit_addon_utils.slicing import slice_by_gate_types

slices = slice_by_gate_types(circuit)
print(f"Separated the circuit into {len(slices)} slices.")
```

```text
Separated the circuit into 18 slices.
```

### Limitar o quanto o operador pode crescer durante a retropropagação

Durante a retropropagação, o número de termos no operador geralmente se aproxima de $4^N$ rapidamente, onde $N$ é o número de Qubits. O tamanho do operador pode ser limitado especificando o kwarg ``operator_budget`` da função ``backpropagate``, que aceita uma instância de [OperatorBudget](https://qiskit.github.io/qiskit-addon-obp/stubs/qiskit_addon_obp.utils.simplify.OperatorBudget.html).

Aqui especificamos que a retropropagação deve parar quando o número de grupos de Pauli com comutação qubit a qubit no operador ultrapassar 8.

```python
from qiskit_addon_obp.utils.simplify import OperatorBudget

op_budget = OperatorBudget(max_qwc_groups=8)
```

### Retropropagar fatias do circuito

Primeiro, vamos especificar o observável de Pauli-Z no Qubit 0, e vamos retropropagar fatias do circuito de evolução temporal até que os termos no observável não possam mais ser combinados em 8 ou menos grupos de Pauli com comutação qubit a qubit.

Abaixo você verá que retropropagamos 7 fatias, mas usamos apenas 6 dos 8 grupos de Pauli alocados. Isso implica que retropropagar mais uma fatia faria o número de grupos de Pauli ultrapassar 8. Podemos verificar que esse é o caso inspecionando os metadados retornados.

```python
from qiskit.quantum_info import SparsePauliOp
from qiskit_addon_obp import backpropagate
from qiskit_addon_utils.slicing import combine_slices

# Specify a single-qubit observable
observable = SparsePauliOp("IIIIIIIIIZ")

# Backpropagate slices onto the observable
bp_obs, remaining_slices, metadata = backpropagate(observable, slices, operator_budget=op_budget)
# Recombine the slices remaining after backpropagation
bp_circuit = combine_slices(remaining_slices, include_barriers=True)

print(f"Backpropagated {metadata.num_backpropagated_slices} slices.")
print(
    f"New observable has {len(bp_obs.paulis)} terms, which can be combined into {len(bp_obs.group_commuting(qubit_wise=True))} groups."
)
print(
    f"Note that backpropagating one more slice would result in {metadata.backpropagation_history[-1].num_paulis[0]} terms "
    f"across {metadata.backpropagation_history[-1].num_qwc_groups} groups."
)
print("The remaining circuit after backpropagation looks as follows:")
bp_circuit.draw("mpl", scale=0.6)
```

```text
Backpropagated 7 slices.
New observable has 18 terms, which can be combined into 8 groups.
Note that backpropagating one more slice would result in 27 terms across 12 groups.
The remaining circuit after backpropagation looks as follows:
```

Em seguida, vamos especificar o mesmo problema com as mesmas restrições sobre o tamanho do observável de saída. No entanto, desta vez, vamos alocar um orçamento de erro para cada fatia usando a função [setup_budet](https://qiskit.github.io/qiskit-addon-obp/stubs/qiskit_addon_obp.utils.truncating.setup_budget.html). Termos de Pauli com coeficientes pequenos serão truncados de cada fatia até que o orçamento de erro seja preenchido, e o orçamento restante será adicionado ao orçamento da fatia seguinte.

Para habilitar esse truncamento, precisamos configurar nosso orçamento de erro da seguinte forma:

```python
from qiskit_addon_obp.utils.truncating import setup_budget

truncation_error_budget = setup_budget(max_error_per_slice=0.005)
```

Note que ao alocar `5e-3` de erro por fatia para truncamento, somos capazes de remover 3 fatias a mais do circuito, permanecendo dentro do orçamento original de 8 grupos de Pauli comutativos no observável. Por padrão, `backpropagate` usa a norma L1 dos coeficientes truncados para limitar o erro total incorrido com o truncamento. Para outras opções, consulte o [guia prático sobre como especificar o p_norm](https://qiskit.github.io/qiskit-addon-obp/how_tos/bound_error_using_p_norm.html).

Neste exemplo específico onde retropropagamos 10 fatias, o erro total de truncamento não deve ultrapassar ``(5e-3 error/slice) * (10 slices) = 5e-2``.
Para mais discussão sobre como distribuir um orçamento de erro pelas suas fatias, confira [este guia prático](https://qiskit.github.io/qiskit-addon-obp/how_tos/truncate_operator_terms.html).

```python
# Run the same experiment but truncate observable terms with small coefficients
bp_obs_trunc, remaining_slices_trunc, metadata = backpropagate(
    observable, slices, operator_budget=op_budget, truncation_error_budget=truncation_error_budget
)

# Recombine the slices remaining after backpropagation
bp_circuit_trunc = combine_slices(remaining_slices_trunc, include_barriers=True)

print(f"Backpropagated {metadata.num_backpropagated_slices} slices.")
print(
    f"New observable has {len(bp_obs_trunc.paulis)} terms, which can be combined into {len(bp_obs_trunc.group_commuting(qubit_wise=True))} groups.\n"
    f"After truncation, the error in our observable is bounded by {metadata.accumulated_error(0):.3e}"
)
print(
    f"Note that backpropagating one more slice would result in {metadata.backpropagation_history[-1].num_paulis[0]} terms "
    f"across {metadata.backpropagation_history[-1].num_qwc_groups} groups."
)
print("The remaining circuit after backpropagation looks as follows:")
bp_circuit_trunc.draw("mpl", scale=0.6)
```

```text
New observable has 19 terms, which can be combined into 8 groups.
After truncation, the error in our observable is bounded by 4.933e-02
Note that backpropagating one more slice would result in 27 terms across 13 groups.
The remaining circuit after backpropagation looks as follows:
```

### Agora que temos nossos ansatzes reduzidos e observáveis expandidos, podemos transpilar nossos experimentos para o Backend.

Aqui vamos usar o [FakeMelbourneV2](https://quantum.cloud.ibm.com/docs/api/qiskit-ibm-runtime/fake-provider-fake-melbourne-v2) de 14 Qubits do [qiskit-ibm-runtime](https://quantum.cloud.ibm.com/docs/api/qiskit-ibm-runtime) para demonstrar como transpilar para um Backend de QPU.

```python
from qiskit.transpiler.preset_passmanagers import generate_preset_pass_manager
from qiskit_ibm_runtime.fake_provider import FakeMelbourneV2

# Specify a backend and a pass manager for transpilation
backend = FakeMelbourneV2()
pm = generate_preset_pass_manager(backend=backend, optimization_level=1)

# Transpile original experiment
circuit_isa = pm.run(circuit)
observable_isa = observable.apply_layout(circuit_isa.layout)

# Transpile backpropagated experiment
bp_circuit_isa = pm.run(bp_circuit)
bp_obs_isa = bp_obs.apply_layout(bp_circuit_isa.layout)

# Transpile the backpropagated experiment with truncated observable terms
bp_circuit_trunc_isa = pm.run(bp_circuit_trunc)
bp_obs_trunc_isa = bp_obs_trunc.apply_layout(bp_circuit_trunc_isa.layout)
```

## Etapa 3: Executar experimentos quânticos
### Calcular o valor esperado

Por fim, podemos executar os experimentos retropropagados e compará-los com o experimento completo usando o [StatevectorEstimator](https://quantum.cloud.ibm.com/docs/api/qiskit/qiskit.primitives.StatevectorEstimator) sem ruído. Podemos ver que o valor esperado retropropagado sem truncamento é equivalente ao valor exato dentro dos limites de precisão numérica.

O valor esperado do operador com termos truncados tem algum erro na ordem de ``1e-4``, que está dentro da tolerância esperada.

**Nota:** Usamos um ``Estimator`` primitivo baseado em statevector para ilustrar o efeito do truncamento na saída. Para executar no Backend para o qual os experimentos foram transpilados na Etapa 2, seria necessário importar o [EstimatorV2](https://quantum.cloud.ibm.com/docs/api/qiskit-ibm-runtime/estimator-v2) do ``qiskit-ibm-runtime`` e passar a instância do Backend para o construtor.

```python
from qiskit.primitives import StatevectorEstimator as Estimator

estimator = Estimator()

# Run the experiments using Estimator primitive
result_exact = estimator.run([(circuit_isa, observable_isa)]).result()[0].data.evs.item()
result_bp = estimator.run([(bp_circuit_isa, bp_obs_isa)]).result()[0].data.evs.item()
result_bp_trunc = (
    estimator.run([(bp_circuit_trunc_isa, bp_obs_trunc_isa)]).result()[0].data.evs.item()
)

print(f"Exact expectation value: {result_exact}")
print(f"Backpropagated expectation value: {result_bp}")
print(f"Backpropagated expectation value with truncation: {result_bp_trunc}")
print(f"    - Expected Error for truncated observable: {metadata.accumulated_error(0):.3e}")
print(f"    - Observed Error for truncated observable: {abs(result_exact - result_bp_trunc):.3e}")
```

```text
Exact expectation value: 0.8854160687717507
Backpropagated expectation value: 0.8854160687717532
Backpropagated expectation value with truncation: 0.8850236647156059
    - Expected Error for truncated observable: 4.933e-02
    - Observed Error for truncated observable: 3.924e-04
```

### O que é absorção de ruído propagado (PNA)?

***Uma técnica para mitigar erros de Gate propagando o observável através do canal de ruído inverso que afeta os Gates de 2 qubits, resultando em um observável de mitigação de ruído.***
Os Gates de 2Q no experimento que queremos executar serão afetados por um ruído substancial.
![Experimento ruidoso](../images/noisy_expt.png)
Se aprendermos o modelo de ruído, podemos aplicar o seu inverso e cancelar o ruído.
![Experimento com ruído mitigado](../images/noise_mitigated_expt.png)
Em vez de implementar o canal de ruído inverso amostrando-o no QPU como no PEC, podemos implementá-lo de forma clássica no observável medido usando a propagação de Pauli. Isso resulta em um observável mais complexo que, quando medido, tem o efeito de mitigar o ruído de Gate aprendido.
![Visão geral do PNA](../images/pna_overview.png)
### Gerar o Circuit Trotter espelhado e o observável

Para este experimento, estudaremos a dinâmica temporal de um modelo de Ising chutado de 30 sítios em uma cadeia de spin 1D. O Hamiltoniano considerado é:

$H = -J\sum\limits_{\langle i,j \rangle} Z_iZ_j + h\sum\limits_iX_i$,

onde $J>0$ descreve o acoplamento de spins de primeiros vizinhos, $i<j$, e o campo transversal global, $h$, é definido como $\frac{\pi}{8}$. Quanto mais $h$ se afasta de um ângulo de Clifford (ou seja, $\theta=n\frac{\pi}{2}, n \in \mathbb{Z}$), mais difícil se torna propagar os geradores anti-ruído através do Circuit.

Para a escolha do observável, consideraremos a magnetização média por sítio, $\frac{1}{N} \sum_{i=1}^{N} \langle z_i \rangle$, onde $N$ é o número de sítios.

```python
import numpy as np
from qiskit import QuantumCircuit
from qiskit.quantum_info import Pauli, SparsePauliOp

num_qubits = 30
num_trotter_steps = 10
rx_angle = np.pi / 8

# Avg single-site magnetization
id_pauli = Pauli("I" * num_qubits)
observable = SparsePauliOp([id_pauli.dot(Pauli("Z"), [i]) for i in range(num_qubits)]) / num_qubits

# Implement Trotterized kicked-Ising model
circuit = QuantumCircuit(num_qubits)
for _step in range(num_trotter_steps):
    circuit.rx(rx_angle, range(num_qubits))
    for first_qubit in (1, 2):
        for idx in range(first_qubit, num_qubits, 2):
            # equivalent to Rzz(-pi/2):
            circuit.sdg([idx - 1, idx])
            circuit.cz(idx - 1, idx)
circuit.compose(circuit.inverse(), inplace=True)
circuit.measure_active()
circuit.draw("mpl", fold=-1)
```

### Embaralhar as camadas de Gates de 2 Qubits e medições e encontrar camadas únicas

Aqui garantimos que o pass manager anote as caixas com anotações ``Twirl`` e ``InjectNoise``, o que nos permite aprender o ruído que afetará nosso Circuit e associá-lo à sua camada de Circuit correspondente.

- ``enable_gates/enable_measure: True``: Encaixotar todas as camadas de Gates de 2Q e medições terminais. Gates de um único Qubit serão vestidos à esquerda dentro das caixas.
- ``measure_annotations: all`` Incluir anotações `Twirl` e `ChangeBasis` na caixa de medição
- ``twirling_strategy: active``: Embaralhar todos os Qubits ativos em cada caixa contendo Gates de entrelaçamento
- ``inject_noise_targets: gates``: Anotações ``InjectNoise`` devem ser adicionadas a todas as caixas anotadas com ``Twirl`` que contenham Gates de entrelaçamento
- ``inject_noise_strategy: uniform_modification``: Todas as camadas de ruído devem ser escaladas de forma equivalente.

```python
from samplomatic.transpiler import generate_boxing_pass_manager

# Box up circuit with Twirl and InjectNoise annotations
pm = generate_boxing_pass_manager(
    enable_gates=True,
    enable_measures=True,
    measure_annotations="all",
    twirling_strategy="active",
    inject_noise_targets="gates",
    inject_noise_strategy="uniform_modification",
    remove_barriers=True,
)
boxed_circuit = pm.run(isa_circuit)
```

```python
draw_circ = QuantumCircuit(boxed_circuit.num_qubits)
draw_circ.append(boxed_circuit.data[0], qargs=boxed_circuit.data[0].qubits)
draw_circ.append(boxed_circuit.data[1], qargs=boxed_circuit.data[1].qubits)
draw_circ.draw("mpl", fold=-1, scale=0.3, idle_wires=False)
```

### Gerar o Circuit template e o samplex, definir como o Circuit será amostrado

Aqui também adicionamos medições de espectadores e de pós-seleção, que são necessárias para realizar a pós-seleção nas amostras geradas pelo ``Executor``.

```python
import samplomatic
from qiskit.transpiler import PassManager
from qiskit_addon_utils.noise_management.post_selection.transpiler.passes import (
    AddPostSelectionMeasures,
    AddSpectatorMeasures,
)

# Build template circuit and samplex for later use with the "Executor"
template_circuit, samplex = samplomatic.build(boxed_circuit)

# Add post-selection instructions to the template circuit
post_selection_pm = PassManager(
    [
        AddSpectatorMeasures(backend.coupling_map),
        AddPostSelectionMeasures(x_pulse_type="rx"),
    ]
)
template_circuit = post_selection_pm.run(template_circuit)
```

## 2.1 Mapeie o problema para um formato nativo quântico

Em um programa quântico, *circuitos quânticos* são o formato nativo para representar instruções quânticas, e *operadores* representam os observáveis a serem medidos. Ao criar um Circuit, você geralmente cria um novo objeto [`QuantumCircuit`](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.QuantumCircuit#quantumcircuit-class) e, em seguida, adiciona instruções a ele em sequência.

O código a seguir cria um Circuit que produz o estado GHZ, que é um estado em que três Qubits estão totalmente entrelaçados entre si.

<div class="alert alert-info">

  O Qiskit SDK usa a numeração de bits LSb 0, onde o $n^{th}$ dígito tem valor $1 \ll n$ ou $2^n$. Para mais detalhes, consulte o tópico [Ordenação de bits no Qiskit SDK](https://docs.quantum.ibm.com/guides/bit-ordering).

</div>

```python
# Create a GHZ state circuit

qc = QuantumCircuit(3)
qc.h(0)
qc.cx(0,1)
qc.cx(0,2)
# Draw the circuit
qc.draw("mpl")
```

![Quantum circuit diagram](./_04_Hands-on Introduction to Qiskit_imgs/output_15.png)

Consulte [`QuantumCircuit`](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.QuantumCircuit#quantumcircuit-class) na documentação para todas as operações disponíveis.

Ao criar circuitos quânticos, você também deve considerar que tipo de dados deseja retornar após a execução. O Qiskit oferece duas formas de retornar dados: você pode obter uma distribuição de probabilidade para um conjunto de Qubits que escolher medir, ou pode obter o valor esperado de um observável. Prepare sua carga de trabalho para medir seu Circuit de uma dessas duas formas usando as [primitivas Qiskit](https://docs.quantum.ibm.com/guides/get-started-with-primitives) (explicadas em detalhes no Passo 3).

Este exemplo mede valores esperados usando o submódulo `qiskit.quantum_info`, especificado por meio de operadores (objetos matemáticos usados para representar uma ação ou processo que muda um estado quântico). O código a seguir cria seis operadores de Pauli de três Qubits: `ZZZ`, `ZZX`, `ZII`, `XXI`, `ZZI` e `III`.

```python
# Set up six different observables.

observables_labels = ["ZZZ", "ZZX", "ZII", "XXI", "ZZI", "III"]

observables = [SparsePauliOp(label) for label in observables_labels]
print(observables)
```

```text
[SparsePauliOp(['ZZZ'],
              coeffs=[1.+0.j]), SparsePauliOp(['ZZX'],
              coeffs=[1.+0.j]), SparsePauliOp(['ZII'],
              coeffs=[1.+0.j]), SparsePauliOp(['XXI'],
              coeffs=[1.+0.j]), SparsePauliOp(['ZZI'],
              coeffs=[1.+0.j]), SparsePauliOp(['III'],
              coeffs=[1.+0.j])]
```

<div class="alert alert-info">

  Aqui, algo como o operador `ZZI` é uma abreviação para o produto tensorial $Z\otimes Z\otimes I$, o que significa medir Z no Qubit 2 e Z no Qubit 1 juntos, e obter informações sobre a correlação entre o Qubit 2 e o Qubit 1. Valores esperados como esse também são tipicamente escritos como $\langle Z_2 Z_1 \rangle$.

  Se o estado que observamos é o estado GHZ de três Qubits, então a medição de $\langle Z_2 Z_1 \rangle$ deve ser 1.

</div>

## 2.2 Otimize os circuitos e operadores

Ao executar circuitos em um dispositivo, é importante otimizar o conjunto de instruções que o Circuit contém e minimizar a profundidade total (aproximadamente o número de instruções) do Circuit. Isso garante que você obtenha os melhores resultados possíveis, reduzindo os efeitos de erro e ruído. Além disso, as instruções do Circuit devem estar em conformidade com a [Arquitetura do Conjunto de Instruções (ISA)](https://docs.quantum.ibm.com/guides/transpile#instruction-set-architecture) do dispositivo Backend e devem considerar as gates base e a conectividade de Qubits do dispositivo.

O código a seguir instancia um dispositivo real para enviar um job e transforma o Circuit e os observáveis para corresponder à ISA desse Backend.
Se você não salvou suas credenciais anteriormente, siga as instruções [aqui](https://docs.quantum.ibm.com/guides/setup-channel#iqp) para autenticar com seu token de API.

```python
# Choose a real backend
service = QiskitRuntimeService(channel='ibm_quantum_platform',)
backend = service.least_busy(min_num_qubits=156)
# print backend details
print(
    f"Name: {backend.name}\n"
    f"Version: {backend.backend_version}\n"
    f"No. of qubits: {backend.num_qubits}\n"
    f"Processor type: {backend.processor_type}\n"
)
```

## 2.3 Execute usando as primitivas quânticas

Computadores quânticos podem produzir resultados aleatórios, por isso você geralmente coleta uma amostra das saídas executando o Circuit muitas vezes. Você pode estimar o valor do observável usando a classe `Estimator`. O Estimator é uma das duas [primitivas](https://docs.quantum.ibm.com/guides/get-started-with-primitives); a outra é o Sampler, que pode ser usado para obter dados de um computador quântico. Esses objetos possuem um método `run()` que executa a seleção de circuitos, observáveis e parâmetros (se aplicável), usando um [bloco unificado primitivo (PUB).](https://docs.quantum.ibm.com/guides/primitives#sampler)
Ao executar este código em hardware quântico real, considere aplicar [técnicas de mitigação e supressão de erros](https://quantum.cloud.ibm.com/docs/en/guides/error-mitigation-and-suppression-techniques) para reduzir o ruído intrínseco do computador quântico.

```python
# Construct the Estimator instance.
estimator = Estimator(mode=backend)
estimator.options.resilience_level = 1
estimator.options.default_shots = 5000
```

Envie um job usando a primitiva Estimator.

```python
# One pub, with one circuit to run against six different observables.
job = estimator.run([(isa_circuit, mapped_observables)]) 

# Use the job ID to retrieve your job data later
print(f">>> Job ID: {job.job_id()}")
```

Após o envio de um job, você pode aguardar até que o job seja concluído na sua instância Python atual, ou usar o `job_id` para recuperar os dados em um momento posterior. (Consulte a [seção sobre recuperação de jobs](https://docs.quantum.ibm.com/guides/monitor-job#retrieve-job-results-at-a-later-time) para detalhes.)

Após a conclusão do job, examine sua saída por meio do atributo `result()` do job.

```python
# This is the result of the entire submission.  You submitted one Pub,
# so this contains one inner result (and some metadata of its own).
job_result = job.result()

# This is the result from our single pub, which had six observables,
# so contains information on all six.
pub_result = job.result()[0]
```

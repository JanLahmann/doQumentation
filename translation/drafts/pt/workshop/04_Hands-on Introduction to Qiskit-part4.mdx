## 3.1 Amplitude encoding

Agora que vimos como construir circuitos quânticos, é interessante explorar como podemos codificar informação clássica em estados quânticos. Um método poderoso é a codificação por amplitude, onde as amplitudes de um estado quântico representam os componentes de um vetor clássico.

Vamos considerar um exemplo simples. Suponha que queremos codificar o vetor clássico

$$
\vec{x} = \begin{bmatrix} x_0 \\ x_1 \\ x_2 \\ x_3 \end{bmatrix}
$$

em um estado quântico de dois qubits. O objetivo é preparar o estado quântico:

$$
\ket{\psi} = x_0\ket{00} + x_1\ket{01} + x_2\ket{10} + x_3\ket{11}
$$
onde $ x_0, x_1, x_2, x_3 \in \mathbb{R} $ (ou $ \mathbb{C} $) e o vetor é normalizado de forma que:

$$
|x_0|^2 + |x_1|^2 + |x_2|^2 + |x_3|^2 = 1
$$

Agora consideramos o exemplo particular: $ \vec{x} = [0.8924,  0.3696, 0.2391, 0.0990] $

O estado quântico correspondente é:

$$
\begin{aligned}
\ket{\psi} &= 0.8924\,\ket{00}
+ 0.3696\,\ket{01}
+ 0.2391\,\ket{10}
+ 0.0990\,\ket{11}
\end{aligned}
$$
Este estado pode ser preparado usando uma combinação de gates de rotação $R_y$ de ângulos $\pi/6$ e $\pi/4$ para os qubits 0 e 1 respectivamente

```python
from qiskit import QuantumCircuit
from qiskit_aer import AerSimulator
import numpy as np

qc = QuantumCircuit(2)

qc.ry(np.pi / 6, 0)
qc.ry(np.pi / 4, 1)

simulator = AerSimulator()
qc.save_statevector()
result = simulator.run(qc).result()
statevector = result.get_statevector()

print("Statevector:", statevector)
qc.draw(output="mpl")
```

```text
             0.09904576+0.j],
            dims=(2, 2))
```

## 3.2 Angle encoding e circuitos parametrizados

Uma forma particularmente interessante de codificar informação em um computador quântico é projetar um circuito quântico que contenha alguns ângulos de rotação $\vec{\theta}$ ou parâmetros que podem ser ajustados para representar uma família de funções $f(\vec{\theta})$. Vamos considerar, por exemplo, o seguinte circuito quântico parametrizado:

```python
from qiskit import QuantumCircuit
from qiskit.circuit import Parameter

# Define a symbolic parameter
theta = Parameter("θ")

qc = QuantumCircuit(2)
# We applied a parametrized RX gate
qc.rx(theta, 0)
qc.cx(0, 1)
qc.draw("mpl")
```

![Quantum circuit diagram](./_04_Hands-on Introduction to Qiskit_imgs/output_25.png)

Matematicamente, podemos analisar qual é a família de funções que podemos representar com este circuito:

$$ 
\text{CNOT}_{01} \, R_x^{\{0\}}(\theta) |00\rangle = \text{CNOT}_{01} \left( \cos(\theta/2)\ket{00} - i\sin(\theta/2)\ket{10} \right) = \cos(\theta/2)\ket{00} - i\sin(\theta/2)\ket{11}
$$
É bastante claro que o número de estados que podemos representar com este circuito quântico é limitado, pois não podemos representar os estados $\ket{10}$ ou $\ket{01}$, por exemplo. No entanto, a família de estados que podemos representar começa a crescer quando introduzimos mais rotações nos lugares adequados:

```python
from qiskit import QuantumCircuit
from qiskit.circuit import Parameter

# Define a symbolic parameter
theta1 = Parameter("θ1")
theta2 = Parameter("θ2")

qc = QuantumCircuit(2)
qc.rx(theta1, 0)
qc.rx(theta2, 1)
qc.cx(0, 1)
qc.draw("mpl")
```

![Quantum circuit diagram](./_04_Hands-on Introduction to Qiskit_imgs/output_26.png)

Neste caso, os estados quânticos que representaremos são:

$$
\begin{align*}
\text{CNOT}_{01} \, R_x^{\{1}}(\theta_2) R_x^{\{0}}(\theta_1) \ket{00}
&= \text{CNOT}_{01} \, R_x^{\{1}}(\theta_2)\left( \cos(\theta_1/2)\ket{00} - i\sin(\theta_1/2)\ket{10} \right) \\
&= \text{CNOT}_{01}\left( \cos(\theta_1/2)\cos(\theta_2/2)\ket{00} - i\cos(\theta_1/2)\sin(\theta_2/2)\ket{01} \right. \\
&\quad \left. - i\sin(\theta_1/2)\cos(\theta_2/2)\ket{10} + \sin(\theta_1/2)\sin(\theta_2/2)\ket{11} \right) \\
&= \cos(\theta_1/2)\cos(\theta_2/2)\ket{00} - i\cos(\theta_1/2)\sin(\theta_2/2)\ket{01} \\
&\quad + \sin(\theta_1/2)\sin(\theta_2/2)\ket{10} - i\sin(\theta_1/2)\cos(\theta_2/2)\ket{11} 
\end{align*}
$$
Podemos ver que este circuito gera uma família mais ampla de estados quânticos em comparação com o anterior. Em particular, agora ele pode produzir estados com amplitudes não nulas para $\ket{01}$ ou $\ket{10}$, o que não era possível com o circuito acima. No entanto, este circuito ainda não é um gerador universal de estados quânticos, embora possa ser suficientemente expressivo para projetar circuitos com alguma flexibilidade para representar certas funções. Em geral, quanto mais parâmetros independentes (ângulos) introduzimos, maior é a expressividade do circuito para aproximar estados quânticos arbitrários.

# Bônus: Teleportação Quântica
Quando você ouve o termo teleportação quântica, pode imaginar uma tecnologia futurística de ficção científica que desintegra um objeto em um lugar e o faz reaparecer em outro lugar distante. Mas a teleportação quântica não é nada disso. Na realidade, o que é teleportado não é matéria — é informação.

A teleportação quântica permite a transferência do estado quântico de um qubit de um local para outro. Embora essa transferência pareça instantânea, ela não viola as leis da física. Como isso é possível? Vamos explorar!

A teleportação quântica é um protocolo que permite a um remetente (Alice) transmitir o estado $|\psi\rangle$ de um qubit `q` a um receptor (Bob) usando dois recursos-chave: um par entrelaçado compartilhado de qubits `a` e `b` e dois bits de comunicação clássica `c0` e `c1`.

Essencialmente, o que o protocolo necessita é:
*   `q`: o qubit de Alice, inicialmente no estado $|\psi\rangle$ que queremos teleportar.
*   `a`: a metade de Alice de um par entrelaçado compartilhado.
*   `b`: a metade de Bob do par entrelaçado compartilhado.
*   `c0`, `c1`: bits clássicos para armazenar os resultados das medições de Alice.

E como funciona? O fluxo de trabalho é o seguinte

1.  **Preparar o estado de Alice $|\psi\rangle$ em `q`.** Criaremos um estado específico como $|+\rangle$ para verificação.
2.  **Criar entrelaçamento:** Gerar um par de Bell entre `a` e `b`.
3.  **Operações de Alice:** Alice realiza uma "medição de Bell" em seus dois qubits (`q` e `a`) e armazena os resultados clássicos em `c0` e `c1`.
4.  **Comunicação clássica:** Alice envia seus dois bits clássicos (`c0`, `c1`) para Bob.
5.  **Correções de Bob:** Bob aplica gates quânticos específicos (X e/ou Z) ao seu qubit (`b`), condicionados aos valores de `c0` e `c1` que recebeu.

Se tudo for feito corretamente, o qubit `b` de Bob terminará no estado $|\psi\rangle$, o estado original do `q` de Alice!

Para uma explicação e exploração mais aprofundada da teleportação quântica, incluindo a explicação matemática de por que esse protocolo funciona, você pode consultar os recursos do IBM Quantum Learning: [Teleportação Quântica](https://quantum.cloud.ibm.com/learning/courses/basics-of-quantum-information/entanglement-in-action/quantum-teleportation). Isso faz parte do curso [Noções Básicas de Informação Quântica](https://quantum.cloud.ibm.com/learning/courses/basics-of-quantum-information).

```python

import matplotlib.pyplot as plt
from qiskit import QuantumCircuit, QuantumRegister, ClassicalRegister
from qiskit_aer import AerSimulator
from qiskit.visualization import plot_histogram, plot_bloch_multivector

# Define individual quantum registers for each qubit
q = QuantumRegister(1, name='q')  # message qubit
a = QuantumRegister(1, name='a')  # Alice's entangled qubit
b = QuantumRegister(1, name='b')  # Bob's entangled qubit

# Classical register for Alice's measurements
cr_alice = ClassicalRegister(2, name='c_alice')

# Create quantum circuit
teleport_qc = QuantumCircuit(q, a, b, cr_alice, name='Teleportation')

# Step 1: Prepare message state |+⟩ on q
teleport_qc.h(q[0])
teleport_qc.barrier()

# Step 2: Create entanglement between a and b
teleport_qc.h(a[0])
teleport_qc.cx(a[0], b[0])
teleport_qc.barrier()

# Step 3: Alice's Bell measurement
teleport_qc.cx(q[0], a[0])
teleport_qc.h(q[0])
teleport_qc.barrier()

# Step 4: Alice measures q and a
teleport_qc.measure(q[0], cr_alice[0])
teleport_qc.measure(a[0], cr_alice[1])
teleport_qc.barrier()

# Step 5: Bob's conditional measurements
with teleport_qc.if_test((cr_alice[1], 1)):
    teleport_qc.x(b[0])
with teleport_qc.if_test((cr_alice[0], 1)):
    teleport_qc.z(b[0])

# Draw the circuit
teleport_qc.draw(output='mpl')
```

![Quantum circuit diagram](./_04_Hands-on Introduction to Qiskit_imgs/output_28.png)

Após executar o protocolo, surge uma questão fundamental: como verificamos que a teleportação funcionou? Não podemos "ver" diretamente o estado do qubit de Bob após o protocolo. No entanto, como *preparamos* o estado inicial de Alice $|\psi\rangle$ (escolhemos $|+\rangle$), podemos usar um tipo especial de simulação para verificar se o qubit `b` de Bob terminou nesse mesmo estado.

Usaremos o `AerSimulator` com `save_statevector` para verificar se o qubit `b` de Bob termina no estado original de Alice ($|+\rangle$). Este simulador calcula o vetor de estado quântico final.
e então o representa usando `plot_bloch_multivector` para visualizar o qubit de Bob (`b`) comparado ao estado inicial de Alice (`q`).

```python
# Simulate the teleportation circuit
sv_simulator = AerSimulator(method='statevector')
teleport_qc_sv = teleport_qc.copy()
teleport_qc_sv.save_statevector()

# Execute the circuit on the statevector simulator
job_sv = sv_simulator.run(teleport_qc_sv)
result_sv = job_sv.result()

# Get the final statevector
final_statevector = result_sv.get_statevector()
print("Visualizing final qubit states:")
display(plot_bloch_multivector(final_statevector))
print("Note that Alice's qubits have collapsed to |00⟩, |01⟩, |10⟩, or |11⟩, while Bob's qubit is in the original state |+⟩.")
```

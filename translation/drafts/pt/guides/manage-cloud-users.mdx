# Gerenciar usuários do IBM Cloud

Este guia demonstra como usar o IBM Cloud&reg; para habilitar usuários que possuem contas IBM Cloud e fornece instruções para que os usuários acessem o ambiente.

Para gerenciar usuários de provedores de identidade, siga as instruções no tópico [Gerenciar provedores de identidade](/guides/manage-appid).

## Passos {#steps}

Siga estes passos de alto nível para gerenciar usuários do IBM Cloud:

1. Certifique-se de que os usuários que você deseja convidar possuem contas IBM Cloud.
2. Vá para o IAM e [convide usuários](https://cloud.ibm.com/iam/users/invite_users)
   <Admonition type="note">
   Os usuários não podem ser gerenciados até que aceitem o convite e façam login pelo menos uma vez.
   </Admonition>
3. Opcional: Modifique as atribuições de projetos e grupos de acesso dos usuários pelo [IAM.](https://cloud.ibm.com/iam/users)

<span id="user-org"></span>
## Fluxo do usuário {#user-flow}

1. O usuário aceita um convite e, em seguida, faz login pelo [portal IBM Cloud](https://cloud.ibm.com/).
2. O usuário cria uma chave de API acessando ([Gerenciar → Acesso (IAM) → Chaves de API](https://cloud.ibm.com/iam/apikeys)).
3. Para mais informações, os usuários podem consultar [Instalar o Qiskit.](/guides/install-qiskit)

## Exemplo de cenário {#example-scenario}

Este exemplo cria a seguinte configuração:

* Existem dois projetos, `ml` e `finance`.
   * O projeto `ml` deve ter acesso às instâncias de serviço `QR-ml` e `QR-common`.
   * O projeto `finance` deve ter acesso às instâncias de serviço `QR-finance` e `QR-common`.
* Existem três usuários:
   * Fatima deve ter acesso ao projeto `ml`.
   * Ravi deve ter acesso ao projeto `finance`.
   * Amyra deve ter acesso a ambos os projetos.
* Utiliza grupos de acesso sem grupos de recursos.
* Os usuários são definidos no IBM Cloud e as atribuições de projetos também são feitas lá.
* Os usuários devem ser capazes de excluir jobs.

Os passos para implementar essa configuração são:

2. O administrador do Cloud cria três instâncias de serviço: `QR-ml`, `QR finance` e `QR-common`.
3. O administrador do Cloud cria uma regra personalizada que inclui a ação `quantum-computing.job.delete`.
1. O administrador do Cloud cria dois grupos de acesso:
   * O grupo de acesso `ml` pode acessar `QR-ml` e `QR-common`.
   * O grupo de acesso `finance` pode acessar `QR-finance` e `QR-common`.
1. O administrador do Cloud convida os usuários do cloud para o projeto adequado. Especificamente, ele convida e atribui os usuários a um grupo de acesso que inclui o projeto.
   * Fatima é adicionada ao grupo de acesso "ml".
   * Ravi é adicionado ao grupo de acesso "finance".
   * Amyra é adicionada a ambos os grupos de acesso "ml" e "finance".
1. Os usuários podem fazer login pelo portal IBM Cloud, criar chaves de API e trabalhar com as instâncias de serviço de seus projetos.

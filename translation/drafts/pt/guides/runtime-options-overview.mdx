# Introdução às opções

Você pode passar opções para as primitivas a fim de personalizá-las de acordo com suas necessidades. Esta seção foca nas opções das primitivas do Qiskit Runtime. Embora a interface do método `run()` das primitivas seja comum a todas as implementações, as opções não são. Consulte as referências de API correspondentes para obter informações sobre as opções de [`qiskit.primitives`](https://docs.quantum.ibm.com/api/qiskit/primitives#primitives) e [`qiskit_aer.primitives`](https://qiskit.github.io/qiskit-aer/apidocs/aer_primitives.html).

## Visão geral {#overview}

<span id="options-structure"></span>
### Estrutura {#structure}

Ao chamar as primitivas, você pode passar opções usando uma classe de opções ou um dicionário. Opções usadas com frequência, como `resilience_level`, ficam no primeiro nível. Outras opções são agrupadas em diferentes categorias, como `execution`. Veja a seção [Definir opções de primitivas](/guides/specify-runtime-options#pass-options) para todos os detalhes.

<span id="options-defaults"></span>
### Valores padrão {#defaults}

Se você não especificar um valor para uma opção, ela recebe um valor especial `Unset` e o valor padrão do servidor é utilizado. Assim, o valor padrão será o mesmo independentemente da versão do seu código.

As tabelas na seção [Resumo das classes de opções](#options-classes) listam os valores padrão.

<span id="options-precedence"></span>
## Definir opções {#set-options}

As opções podem ser definidas antes de uma primitiva ser construída e passadas para ela, que faz uma cópia delas. Isso pode ser feito como um dicionário aninhado ou usando as classes de opções. Além disso, após a primitiva ser construída, suas opções podem ser alteradas. Use o fluxo de trabalho que melhor se adapte à sua aplicação. Veja [Especificar opções](specify-runtime-options) para todos os detalhes.

## Compatibilidade de recursos {#feature-compatibility}

Devido a diferenças no processo de compilação do dispositivo, certos recursos do runtime não podem ser usados juntos em um único job. Clique na aba correspondente para ver a lista de recursos incompatíveis com o recurso selecionado:

<Tabs>
  <TabItem value="Dynamic" label="Dynamic circuits">
  Incompatível com:
  - ZNE por dobramento de Gate
  - PEA
  - PEC
  - Desacoplamento dinâmico

  Outras observações:
  - Pode ser usado com twirling de Gate para gates não condicionais.
  - Compatível com gates fracionárias ao usar `qiskit-ibm-runtime` v0.42.0 ou posterior.

  </TabItem>
  <TabItem value="DD" label="Dynamical decoupling">
  Incompatível com circuits dinâmicos.

  </TabItem>

  <TabItem value="Fractional" label="Fractional gates">
  Incompatível com:
  - Twirling de Gate
  - PEA
  - PEC

  Compatível com circuits dinâmicos ao usar `qiskit-ibm-runtime` v0.42.0 ou posterior.

  </TabItem>

  <TabItem value="ZNE" label="Gate-folding ZNE">
    Incompatível com:
  - Circuits dinâmicos
  - PEA
  - PEC

  Pode não funcionar ao usar gates personalizadas.
  </TabItem>
  <TabItem value="Twirling" label="Gate twirling">
  Incompatível com gates fracionárias ou com stretches.

  Outras observações:
  - Pode ser usado com circuits dinâmicos com gates não condicionais.
  - Não funciona com entrelaçadores não Clifford.

  </TabItem>

  <TabItem value="PEA" label="PEA">
    Incompatível com:
  - Circuits dinâmicos
  - Gates fracionárias
  - ZNE por dobramento de Gate

# Eventos de rastreamento de atividade para o Qiskit Runtime

Os serviços do IBM Cloud&reg;, como o Qiskit Runtime, geram eventos de rastreamento de atividade.

Os eventos de rastreamento de atividade registram atividades que alteram o estado de um serviço no IBM Cloud. Você pode usar esses eventos para investigar atividades suspeitas e ações críticas, além de cumprir com requisitos de auditoria regulatória.

Você pode usar o IBM Cloud Activity Tracker Event Routing, um serviço de plataforma, para rotear eventos de auditoria em sua conta para destinos de sua escolha, configurando alvos e rotas que definem para onde os eventos de rastreamento de atividade são enviados. Para mais informações, consulte [Sobre o IBM Cloud Activity Tracker Events Routing.](https://cloud.ibm.com/docs/atracker?topic=atracker-about)

Você pode usar o IBM Cloud Logs para visualizar e receber alertas sobre eventos gerados na sua conta e roteados pelo IBM Cloud Activity Tracker Event Routing para uma instância do IBM Cloud Logs.

## Locais onde os eventos de rastreamento de atividade são gerados {#locations-where-activity-tracking-events-are-generated}

O Qiskit Runtime envia eventos de rastreamento de atividade nas seguintes regiões:

- Washington ("us-east")
- Frankfurt ("eu-de")

## Visualizando eventos de rastreamento de atividade para o Qiskit Runtime {#viewing-activity-tracking-events-for-qiskit-runtime}

Você pode usar o IBM Cloud Logs para visualizar e receber alertas sobre eventos gerados na sua conta e roteados pelo IBM Cloud Activity Tracker Event Routing para uma instância do IBM Cloud Logs.

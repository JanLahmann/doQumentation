# Entendendo alta disponibilidade e recuperação de desastres para o Qiskit Runtime {#understanding-high-availability-and-disaster-recovery-for-qiskit-runtime}

Alta disponibilidade (HA) é a capacidade de um serviço permanecer operacional e acessível diante de falhas inesperadas. Recuperação de desastres é o processo de restaurar a instância do serviço a um estado funcional. Os clientes do Qiskit Runtime não são responsáveis pelo planejamento ou pelas ações de recuperação de desastres.

O Qiskit Runtime é um serviço regional que cumpre os [Objetivos de Nível de Serviço (SLO)](https://cloud.ibm.com/docs/resiliency?topic=resiliency-slo) definidos com qualquer plano pago. Para mais informações, consulte o [Acordo de Nível de Serviço (SLA)](https://www.ibm.com/support/customer/csol/terms/?id=i126-9268&lc=en) do IBM Cloud&reg;. Para mais informações sobre as regiões e os data centers do IBM Cloud disponíveis para o Qiskit Runtime, consulte [Disponibilidade de serviços e infraestrutura por localização.](https://cloud.ibm.com/docs/overview?topic=overview-services_region)

## Arquitetura de alta disponibilidade {#high-availability-architecture}

### Recursos de alta disponibilidade {#high-availability-features}

Este serviço é provisionado em três zonas de uma região multi-zona, sem nenhum ponto único de falha. As requisições de API são roteadas por um balanceador de carga para os nós de instância HA, cada um em uma zona diferente.

Se um nó ou zona de instância HA falhar, o serviço continua a processar as requisições de API pelos nós de instância HA restantes. Nenhuma degradação perceptível do serviço ocorre durante uma falha zonal.

Está disponível nas seguintes regiões:

- Washington DC (us-east)
- Frankfurt (eu-de)

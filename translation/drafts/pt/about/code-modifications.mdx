# Modificações Automáticas de Código

O doQumentation aplica automaticamente um pequeno número de modificações ao
conteúdo upstream de tutoriais e guias do Qiskit para garantir uma experiência
interativa e fluida. Esta página documenta cada modificação para que você possa
entender exatamente o que foi alterado em relação à documentação original do
IBM Quantum.

## Cópias de notebooks (Abrir no Colab / Binder / Code Engine)

Quando você clica em **Abrir no Colab**, **Abrir no JupyterLab** ou **Abrir no Code
Engine**, você recebe uma cópia do notebook original com estas adições:

### 1. Célula de aviso de configuração (markdown)

Uma célula de bloco de citação é inserida no topo explicando que o doQumentation
adicionou uma célula de configuração automática. Ela contém um link de volta para
esta página.

### 2. Célula de pré-requisitos (código)

Uma célula de código é inserida após o aviso que:

- **Instala os pacotes necessários** (`qiskit`, `qiskit-aer`, `qiskit-ibm-runtime`,
  `pylatexenc`, além de quaisquer pacotes específicos do tutorial detectados via
  varredura de imports). A instalação é **ignorada** se os pacotes já estiverem
  presentes (ex.: no Binder ou Code Engine, onde eles vêm pré-instalados).
- **Fornece um modelo de credenciais comentado** para o IBM Quantum, para que
  usuários que queiram executar em hardware real possam descomentar e preencher
  sua chave de API.

No Google Colab, esta célula é executada automaticamente na abertura do notebook
via a flag de metadados `cell_execution_strategy: setup`.

### 3. Reescrita de caminhos de imagens

Caminhos de imagens relativos (`/docs/images/...`, `/learning/images/...`) são
reescritos para funcionar corretamente em ambientes de notebook independentes.

## Páginas MDX (renderização no navegador)

Os tutoriais exibidos neste site são convertidos a partir de notebooks `.ipynb`
ou arquivos `.mdx` upstream. As seguintes transformações são aplicadas:

- **Linhas `pip install`** são adicionadas aos blocos de código Python que
  importam pacotes de terceiros, permitindo execução com um clique via thebelab.
- **Seção de Pesquisa de Tutorial IBM**: Uma nota é anexada esclarecendo que a
  pesquisa pertence ao IBM Quantum e com um link para os Issues do GitHub do
  doQumentation para feedback específico do site.
- **Widget de feedback**: Um widget "Isso foi útil?" é anexado ao final de cada
  tutorial, rastreado via análise de privacidade amigável Umami.
- **Correções de sintaxe MDX**: Chaves, hierarquia de títulos e problemas de
  compatibilidade com JSX são corrigidos automaticamente para renderização no
  Docusaurus.
- **OpenInLabBanner**: Um banner interativo é injetado abaixo do título com
  botões para abrir o notebook no Colab, Binder ou Code Engine.

## O que NÃO é modificado

- O **conteúdo do tutorial em si** (explicações, lógica de código, saídas) nunca
  é alterado.
- **A atribuição ao autor original** é preservada via frontmatter e o arquivo
  NOTICE (licenças Apache 2.0 / CC BY-SA 4.0).
- **Nenhum código de telemetria ou rastreamento** é injetado nos notebooks. A
  análise (Umami) é executada apenas no site do doQumentation, não nos notebooks
  exportados.

## Código-fonte

Todas as transformações são implementadas em
[`scripts/sync-content.py`](https://github.com/JanLahmann/doQumentation/blob/main/scripts/sync-content.py).

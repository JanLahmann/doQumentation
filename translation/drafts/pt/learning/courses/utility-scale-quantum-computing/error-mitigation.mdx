## 2. Simulação ruidosa sem mitigação de erros {#2-noisy-simulation-without-error-mitigation}

O Qiskit Aer é um simulador clássico para computação quântica. Ele consegue simular não só a execução ideal, mas também a execução ruidosa de circuitos quânticos. Este notebook demonstra como executar simulações ruidosas usando o Qiskit Aer:

1. Construir um modelo de ruído
2. Construir um sampler ruidoso (simulador) com o modelo de ruído
3. Executar um circuito quântico no sampler ruidoso

```
noise_model = NoiseModel()
...
noisy_sampler = Sampler(options={"backend_options": {"noise_model": noise_model}})
job = noisy_sampler.run([circuit])
```

### 2.1 Construir um circuito de teste {#21-build-a-test-circuit}

Vamos considerar circuitos de 1 qubit simples que apenas repetem portas X `d` vezes (`d`=0 ... 100) e medem o observável `Z`.

```python
from qiskit.circuit import QuantumCircuit

MAX_DEPTH = 100
circuits = []
for d in range(MAX_DEPTH + 1):
    circ = QuantumCircuit(1)
    for _ in range(d):
        circ.x(0)
        circ.barrier(0)
    circ.measure_all()
    circuits.append(circ)

display(circuits[3].draw(output="mpl"))
```

### 2.2 Construir um modelo de ruído {#22-build-a-noise-model}

Para fazer simulação ruidosa, precisamos especificar um `NoiseModel`. Nesta seção, mostramos como construir o `NoiseModel`.
Primeiro precisamos definir erros quânticos (ou de leitura) para adicionar ao modelo de ruído.

```python
from qiskit_aer.noise.errors import (
    coherent_unitary_error,
    amplitude_damping_error,
    ReadoutError,
)
from qiskit.circuit.library import RXGate

# Erro coerente (unitário): erro de sobre-rotação em X
# https://qiskit.github.io/qiskit-aer/stubs/qiskit_aer.noise.coherent_unitary_error.html#qiskit_aer.noise.coherent_unitary_error
OVER_ROTATION_ANGLE = 0.05
coherent_error = coherent_unitary_error(RXGate(OVER_ROTATION_ANGLE).to_matrix())

# Erro incoerente: erro de amortecimento de amplitude
# https://qiskit.github.io/qiskit-aer/stubs/qiskit_aer.noise.amplitude_damping_error.html#qiskit_aer.noise.amplitude_damping_error
AMPLITUDE_DAMPING_PARAM = 0.02  # em [0, 1] (0: sem erro)
incoherent_error = amplitude_damping_error(AMPLITUDE_DAMPING_PARAM)

# Erro de leitura (medição): erro de readout
# https://qiskit.github.io/qiskit-aer/stubs/qiskit_aer.noise.ReadoutError.html#qiskit_aer.noise.ReadoutError
PREP0_MEAS1 = 0.03  # P(1|0): Probabilidade de preparar 0 e medir 1
PREP1_MEAS0 = 0.08  # P(0|1): Probabilidade de preparar 1 e medir 0
readout_error = ReadoutError(
    [[1 - PREP0_MEAS1, PREP0_MEAS1], [PREP1_MEAS0, 1 - PREP1_MEAS0]]
)
```

```python
from qiskit_aer.noise import NoiseModel

noise_model = NoiseModel()
noise_model.add_quantum_error(coherent_error.compose(incoherent_error), "x", (0,))
noise_model.add_readout_error(readout_error, (0,))
```

### 2.3 Construir um sampler ruidoso com o modelo de ruído {#23-build-a-noisy-sampler-with-the-noise-model}

## 3. Computação quântica real com mitigação de erros {#3-real-quantum-computation-with-error-mitigation}

Nesta parte, demonstramos como obter resultados com mitigação de erros (valores esperados) usando o Qiskit Estimator.
Consideramos circuitos Trotterizados de 6 qubits para simular a evolução temporal do modelo de Ising unidimensional e observamos como o erro escala em relação ao número de passos de tempo.

```python
backend = service.least_busy(
    operational=True, simulator=False, min_num_qubits=127
)  # Eagle
backend
```

```text
<IBMBackend('ibm_strasbourg')>
```

```python
NUM_QUBITS = 6
NUM_TIME_STEPS = list(range(8))
RX_ANGLE = 0.1
RZZ_ANGLE = 0.1
```

### 3.1 Construir os circuitos {#31-build-circuits}

```python
# Construir circuitos com diferentes números de passos de tempo
circuits = []
for n_steps in NUM_TIME_STEPS:
    circ = QuantumCircuit(NUM_QUBITS)
    for i in range(n_steps):
        # camada rx
        for q in range(NUM_QUBITS):
            circ.rx(RX_ANGLE, q)
        # 1ª camada rzz
        for q in range(1, NUM_QUBITS - 1, 2):
            circ.rzz(RZZ_ANGLE, q, q + 1)
        # 2ª camada rzz
        for q in range(0, NUM_QUBITS - 1, 2):
            circ.rzz(RZZ_ANGLE, q, q + 1)
    circ.barrier()  # necessário para não otimizar o circuito
    # Etapa de descomputo
    for i in range(n_steps):
        for q in range(0, NUM_QUBITS - 1, 2):
            circ.rzz(-RZZ_ANGLE, q, q + 1)
        for q in range(1, NUM_QUBITS - 1, 2):
            circ.rzz(-RZZ_ANGLE, q, q + 1)
        for q in range(NUM_QUBITS):
            circ.rx(-RX_ANGLE, q)
    circuits.append(circ)
```

Para conhecer a saída ideal com antecedência, usamos circuitos de computo-descomputo, que consistem em uma primeira etapa onde o circuito original $U$ é aplicado, e uma segunda etapa onde ele é revertido $U^\dagger$.
Observe que o resultado ideal de tais circuitos será trivialmente o estado de entrada $|000000\rangle$, que tem valores esperados triviais para qualquer observável de Pauli, por exemplo, $\langle IIIIIZ \rangle = 1$.

### 3.3 Executar com o Estimator (com diferentes níveis de resiliência) {#33-execute-using-estimator-with-different-resilience-levels}

Definir o nível de resiliência (`estimator.options.resilience_level`) é a forma mais simples de aplicar mitigação de erros ao usar o Qiskit Estimator. O Estimator suporta os seguintes níveis de resiliência (a partir de 28/06/2024). Veja mais detalhes no guia [Configurar mitigação de erros](/guides/configure-error-mitigation).

![image.png](/learning/images/courses/utility-scale-quantum-computing/error-mitigation/res_level.avif)

```python
from qiskit_ibm_runtime import Batch
from qiskit_ibm_runtime import EstimatorV2 as Estimator

jobs = []
job_ids = []
with Batch(backend=backend):
    for resilience_level in [0, 1, 2]:
        estimator = Estimator()
        estimator.options.resilience_level = resilience_level
        job = estimator.run(
            [(circ, obs.apply_layout(circ.layout)) for circ in isa_circuits]
        )
        job_ids.append(job.job_id())
        print(f"Job ID (rl={resilience_level}): {job.job_id()}")
        jobs.append(job)
```

## 4. (Opcional) Personalizar as opções de mitigação de erros {#4-optional-customize-error-mitigation-options}

Podemos personalizar a aplicação das técnicas de mitigação de erros via opções, como mostrado abaixo.

```python
# TREX
estimator.options.twirling.enable_measure = True
estimator.options.twirling.num_randomizations = "auto"
estimator.options.twirling.shots_per_randomization = "auto"

# Gate twirling
estimator.options.twirling.enable_gates = True
# ZNE
estimator.options.resilience.zne_mitigation = True
estimator.options.resilience.zne.noise_factors = [1, 3, 5]
estimator.options.resilience.zne.extrapolator = ("exponential", "linear")

# Desacoplamento dinâmico
estimator.options.dynamical_decoupling.enable = True  # Padrão: False
estimator.options.dynamical_decoupling.sequence_type = "XX"

# Outras opções
estimator.options.default_shots = 10_000
```

## Latar Belakang {#background}
Tutorial ini memaparkan *Pauli Correlation Encoding* (PCE) [\[1\]](#references), sebuah pendekatan yang dirancang untuk mengkodekan masalah optimasi ke dalam qubit dengan efisiensi lebih tinggi untuk komputasi kuantum. PCE memetakan variabel klasik dalam masalah optimasi ke korelasi matriks Pauli multi-bodi, sehingga menghasilkan kompresi polinomial pada kebutuhan ruang masalah tersebut. Dengan menggunakan PCE, jumlah qubit yang diperlukan untuk pengkodean berkurang, sehingga sangat menguntungkan bagi perangkat kuantum near-term yang memiliki sumber daya qubit terbatas. Selain itu, secara analitis telah dibuktikan bahwa PCE secara inheren memitigasi barren plateaus, memberikan ketahanan super-polinomial terhadap fenomena ini. Fitur bawaan ini memungkinkan performa yang belum pernah ada sebelumnya dalam solver optimasi kuantum.
### Ikhtisar {#overview}
Pendekatan PCE terdiri dari tiga langkah utama, seperti yang diilustrasikan pada Gambar 1 dari [\[1\]](#references) di bawah ini:
1. Mengkodekan masalah optimasi ke dalam ruang korelasi Pauli.
2. Menyelesaikan masalah menggunakan solver optimasi kuantum-klasik.
3. Mendekodekan solusi kembali ke ruang optimasi asli.
Pendekatan PCE dapat diadaptasi ke solver optimasi kuantum apa pun yang mampu memproses matriks korelasi Pauli.
![pce-overview.png](/docs/images/tutorials/solving-maxcut-with-reduced-qubit-requirements-using-pauli-correlation-encoding/af2cb835-88db-4a3d-9c86-51424b1a4bd3.avif)
Pada Gambar 1 dari [\[1\]](#references), masalah Max-Cut digunakan sebagai contoh untuk mengilustrasikan pendekatan PCE. Masalah Max-Cut dengan $m=9$ simpul dikodekan ke dalam ruang korelasi Pauli, merepresentasikan masalah optimasi sebagai matriks korelasi, khususnya korelasi matriks Pauli 2-bodi pada $n=3$ qubit $(Q_1, Q_2, Q_3)$. Warna simpul menunjukkan string Pauli yang digunakan untuk setiap simpul yang dikodekan.
Misalnya, simpul 1 yang bersesuaian dengan variabel biner $x_1$ dikodekan oleh nilai ekspektasi dari $Z_1 \otimes Z_2 \otimes I_3$, sedangkan $x_8$ dikodekan oleh $I_1 \otimes Y_2 \otimes Y_3$.
Ini bersesuaian dengan kompresi $m$ variabel masalah menjadi $n = O(m^{1/2})$ qubit. Secara lebih umum, korelasi $k$-bodi memungkinkan kompresi polinomial berorde $k$. Set Pauli yang dipilih terdiri dari tiga subset string Pauli yang saling komutasi, sehingga semua $m$ korelasi dapat diestimasi secara eksperimental hanya dengan tiga pengaturan pengukuran.

Sebuah fungsi kerugian $\mathcal{L}$ dari nilai ekspektasi Pauli yang meniru fungsi objektif Max-Cut asli dikonstruksi. Fungsi kerugian kemudian dioptimalkan menggunakan solver optimasi kuantum-klasik, seperti Variational Quantum Eigensolver (VQE).

Setelah optimasi selesai, solusi didekodekan kembali ke ruang optimasi asli, menghasilkan solusi Max-Cut yang optimal.

```python
    cut_size = 0
    for edge0, edge1 in graph.edge_list():
        if edge0 in partition0 and edge1 in partition1:
```

### Masalah Max-Cut {#max-cut-problem}
Masalah Max-Cut adalah masalah optimasi kombinatorial yang didefinisikan pada graf $G = (V, E)$, di mana $V$ adalah himpunan simpul dan $E$ adalah himpunan sisi. Tujuannya adalah membagi simpul menjadi dua himpunan, $S$ dan $V \setminus S$, sedemikian sehingga jumlah sisi antara kedua himpunan tersebut dimaksimalkan.
Untuk deskripsi lengkap masalah Max-Cut, silakan rujuk tutorial ["Quantum approximate optimization algorithm"](./quantum-approximate-optimization-algorithm).
Selain itu, masalah Max-Cut juga digunakan sebagai contoh dalam tutorial ["Advanced Techniques for QAOA"](./advanced-techniques-for-qaoa).
Dalam tutorial-tutorial tersebut, algoritma QAOA digunakan untuk menyelesaikan masalah Max-Cut.

Kita mengkodekan graf dengan 1000 simpul ke dalam korelasi matriks Pauli 2-bodi pada 100 qubit. Graf direpresentasikan sebagai matriks korelasi, di mana setiap simpul dikodekan oleh sebuah string Pauli. Tanda nilai ekspektasi string Pauli menunjukkan partisi simpul tersebut. Misalnya, simpul 0 dikodekan oleh string Pauli, $\prod_0 = I_{19} \otimes ... I_2 \otimes X_1 \otimes X_0$. Tanda nilai ekspektasi string Pauli ini menunjukkan partisi simpul 0. Kita mendefinisikan sebuah *Pauli-correlation encoding* (PCE) relatif terhadap $\prod$ sebagai

$ x_i \coloneqq \textit{sgn}(\langle\prod_i \rangle) $

di mana $x_i$ adalah partisi simpul $i$ dan $\langle \prod_i \rangle \coloneqq  \langle \psi |\prod_i| \psi \rangle $ adalah nilai ekspektasi string Pauli yang mengkodekan simpul $i$ atas keadaan kuantum $|\psi \rangle$.
Sekarang, mari kita kodekan graf ke dalam Hamiltonian menggunakan PCE.
Kita membagi simpul menjadi tiga himpunan: $S_1$, $S_2$, dan $S_3$.
Kemudian, kita mengkodekan simpul-simpul di setiap himpunan menggunakan string Pauli dengan $X$, $Y$, dan $Z$, masing-masing.

```python
def build_pauli_correlation_encoding(pauli, node_list, n, k=2):
    pauli_correlation_encoding = []
    for idx, c in enumerate(combinations(range(n), k)):
        if idx >= len(node_list):
            break
        paulis = ["I"] * n
        paulis[c[0]], paulis[c[1]] = pauli, pauli
        pauli_correlation_encoding.append(("".join(paulis)[::-1], 1))

    hamiltonian = []
    for pauli, weight in pauli_correlation_encoding:
        hamiltonian.append(SparsePauliOp.from_list([(pauli, weight)]))

    return hamiltonian
```

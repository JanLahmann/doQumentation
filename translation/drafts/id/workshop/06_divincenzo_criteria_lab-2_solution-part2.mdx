## 2. Kriteria 2 – Inisialisasi Qubit

**Kriteria 2:** *"Kemampuan untuk menginisialisasi keadaan qubit ke suatu keadaan fiducial sederhana, seperti |000…〉."* Semua Qubit harus bisa dimulai dari keadaan referensi yang sudah diketahui (biasanya keadaan dasar |0〉 untuk tiap Qubit). Inisialisasi ini penting supaya algoritma bisa dimulai dari kondisi yang bersih. Di perangkat IBM Quantum, setiap Qubit secara otomatis di-reset ke |0〉 di awal setiap eksekusi Circuit. Qiskit juga menyediakan instruksi untuk me-reset Qubit atau menyiapkan keadaan kustom di tengah komputasi.

**Tujuan Demo:** Menunjukkan cara menginisialisasi Qubit di Qiskit, baik di awal maupun di tengah Circuit. Kita akan demonstrasikan penggunaan instruksi `reset` dan metode persiapan keadaan.
### Latihan 2: Siapkan Keadaan Tertentu
Di blok kode berikut, lengkapi `QuantumCircuit` untuk menyiapkan keadaan $|10\rangle$. Artinya, Qubit 0 harus berada dalam keadaan $|0\rangle$ dan Qubit 1 harus berada dalam keadaan $|1\rangle$. Gunakan Gate dan instruksi yang tepat untuk mencapai ini.

```python
from qiskit import QuantumCircuit
from qiskit_aer import AerSimulator

# Create a circuit to initialize qubits to |10> and verify by measurement
qc_init = QuantumCircuit(2, 2)

# --- YOUR CODE HERE ---

# 1. Set qubit 1 to the |1> state
qc_init.x(1)

# 2. Explicitly reset qubit 0 to the |0> state
qc_init.reset(0)

# --- END YOUR CODE ---

qc_init.measure([0, 1], [0, 1])
qc_init.draw('mpl')
```


Kamu harusnya melihat `10` (biner untuk qubit1=1, qubit0=0) dengan probabilitas 100% dari simulasi, yang artinya Qubit 1 berhasil disiapkan dalam keadaan |1〉 dan Qubit 0 dalam keadaan |0〉.

Sekarang, untuk persiapan keadaan yang lebih umum, Qiskit memungkinkan inisialisasi ke keadaan sembarang menggunakan metode `initialize`. Misalnya, mari kita siapkan sebuah Qubit dalam keadaan $|+\rangle = (|0\rangle+|1\rangle)/\sqrt{2}$, yaitu keadaan superposisi, dan sepasang Qubit dalam keadaan Bell $(|00\rangle+|11\rangle)/\sqrt{2}$:

```python
import numpy as np

# Initialize a single qubit in |+> state and measure in Z-basis
qc_plus = QuantumCircuit(1, 1)
state_plus = [1/np.sqrt(2), 1/np.sqrt(2)]   # amplitude for |0> and |1>
qc_plus.initialize(state_plus, 0)
qc_plus.measure(0, 0)

# Initialize two qubits in a Bell state manually
qc_bell = QuantumCircuit(2, 2)
bell_state = [1/np.sqrt(2), 0, 0, 1/np.sqrt(2)]  # amplitudes for |00>,|01>,|10>,|11>
qc_bell.initialize(bell_state, [0, 1])
qc_bell.measure([0, 1], [0, 1])

# Transpile and run the initialization circuits
isa_qc_plus = pm.run(qc_plus)
job_plus = sampler.run([isa_qc_plus], shots=1024)
result_plus = job_plus.result()
counts_plus = result_plus[0].data.c.get_counts()

print("Outcome of |+> state measured in Z-basis:", counts_plus)

isa_qc_bell = pm.run(qc_bell)
job_bell = sampler.run([isa_qc_bell], shots=1024)
result_bell = job_bell.result()
counts_bell = result_bell[0].data.c.get_counts()

print("Outcome of Bell state measured in Z-basis:", counts_bell)
```

**Hasil yang diharapkan:** Keadaan Qubit tunggal |+〉 saat diukur akan menghasilkan `0` dan `1` masing-masing dengan probabilitas sekitar 50%. Pengukuran keadaan Bell seharusnya memberikan sekitar 50% `00` dan 50% `11`. Kalau kamu melihat hasil ini, itu mengonfirmasi bahwa inisialisasi ke keadaan-keadaan tersebut berhasil.

**Inisialisasi di tengah Circuit:** `reset` milik Qiskit bisa digunakan di tengah Circuit untuk menginisialisasi ulang sebuah Qubit ke |0〉 saat proses berjalan. Misalnya, dalam kode koreksi kesalahan atau algoritma iteratif, seringkali kita mengukur sebuah Qubit lalu me-reset-nya untuk dipakai lagi. Operasi `reset` bersifat deterministik; ia membuang keadaan yang ada dan mengembalikan Qubit ke keadaan dasar.

**Contoh pada perangkat:** Di perangkat keras seperti **ibmq_brisbane** (127 Qubit) atau perangkat IBM manapun, semua Qubit dimulai dari |0〉 secara default saat suatu pekerjaan dijalankan. Kalau kamu butuh keadaan awal yang berbeda, kamu cukup menerapkan Gate di awal (seperti yang kita lakukan dengan X untuk mendapatkan |1〉). Inisialisasi ulang secara terus-menerus (untuk koreksi kesalahan kuantum) adalah topik riset aktif karena melakukannya dengan cepat cukup menantang. Untungnya, untuk penggunaan dasar, kemampuan untuk memulai dari |0…0〉 yang bersih sudah tersedia, dan kita sudah demonstrasikan cara mencapai keadaan awal yang diinginkan lainnya juga.
## 3. Kriteria 3 – Waktu Koherensi yang Panjang (Dekoherensi vs Waktu Gate)

**Kriteria 3:** *"Waktu dekoherensi yang relevan jauh lebih panjang dari waktu operasi Gate."* Ini menyangkut kebutuhan agar Qubit bisa mempertahankan keadaan kuantumnya cukup lama untuk menyelesaikan operasi yang diperlukan. Setiap Qubit punya **waktu T₁** (waktu relaksasi energi, seberapa cepat |1〉 meluruh ke |0〉) dan **waktu T₂** (waktu dephasing, seberapa cepat koherensi fase relatif hilang). Agar komputer kuantum bisa bekerja, skala waktu ini harus jauh melebihi durasi operasi Gate.

**Tujuan Demo:** Menyelidiki koherensi Qubit di Qiskit dengan menunjukkan bagaimana dekoherensi memengaruhi hasil Circuit seiring bertambahnya panjang eksekusi. Kita akan menggunakan Backend tiruan dengan nilai T1/T2 yang diketahui untuk mensimulasikan efek ini.

Untuk **mendemonstrasikan dampak koherensi yang terbatas**, kita akan mensimulasikan eksperimen peluruhan T1. Kita akan menyiapkan sebuah Qubit dalam keadaan |1〉, menunggu beberapa waktu menggunakan instruksi `delay`, lalu mengukurnya. Kita mengharapkan probabilitas pengukuran |1〉 berkurang seiring bertambahnya delay.

```python
# This part is done for you. We are creating a list of circuits,
# each with a different delay time.

time_delays_ns = [0, 50000, 100000, 150000, 200000, 250000, 300000]  # delay durations in ns

decay_expts = []
for delay in time_delays_ns:
    qc = QuantumCircuit(1, 1)
    qc.x(0)  # initialize qubit to |1>
    if delay > 0:
        qc.delay(delay, 0, unit='ns')  # wait 'delay' nanoseconds
    qc.measure(0, 0)
    decay_expts.append(qc)

decay_expts[1].draw('mpl') # Visualize one of the circuits
```

```python
from qiskit_aer import AerSimulator

# 2. Transpile the list of circuits for this simulator

# 3. Use the Sampler to run all the transpiled circuits in a single job

# --- END YOUR CODE ---

# This part is done for you to analyze and print the results.
for idx, (delay, qc) in enumerate(zip(time_delays_ns, isa_decay_expts)):
    counts = result[idx].data.c.get_counts()
    p1 = counts.get('1', 0) / 1000  # Assuming 1000 shots
    print(f"Delay {delay} ns: P(qubit=1) = {p1:.3f}")
```

## 4. Kriteria 4 – Set Gate Kuantum Universal

**Kriteria 4:** *"Sebuah set Gate kuantum yang 'universal'."* Ini berarti perangkat keras kita harus memungkinkan kita untuk melakukan *sembarang* komputasi kuantum dengan mengombinasikan sekumpulan Gate dasar yang terbatas. Dalam komputasi klasik, NAND bersifat universal; dalam kuantum, ada banyak pilihan set Gate universal (misalnya \{H, T, CNOT\} atau Gate-Gate native dari mesin tertentu). Perangkat IBM, misalnya, punya set operasi native seperti rotasi Qubit tunggal sembarang dan CNOT antar Qubit tertentu, yang bersama-sama bersifat universal. Tugas Qiskit sering kali adalah **mengompilasi Gate tingkat tinggi ke dalam Gate-Gate basis ini**.

**Tujuan Demo:** Mengilustrasikan universalitas Gate dengan menunjukkan cara Qiskit menguraikan Gate. Kita akan mengambil sebuah Gate yang bukan native (seperti Gate Toffoli 3-Qubit, CCX) dan melihat bagaimana ia dipecah menjadi Gate-Gate basis perangkat. Ini mendemonstrasikan bahwa set Gate yang disediakan memang *universal* – ia bisa menghasilkan operasi yang lebih kompleks.

Pertama, mari kita lihat apa saja Gate-Gate basis untuk Backend IBM yang umum. Kita akan mengkueri konfigurasi suatu perangkat (atau versi tiruannya). Misalnya, Gate-Gate basis ibmq_brisbane:
Kamu seharusnya mengamati bahwa probabilitas `P(qubit=1)` berkurang seiring bertambahnya waktu delay, mengikuti kurva peluruhan eksponensial yang merupakan karakteristik relaksasi T1. Ini secara langsung mendemonstrasikan bagaimana waktu koherensi yang terbatas menyebabkan kesalahan komputasi jika Circuit berjalan terlalu lama.

**Dampak pada algoritma:** Kalau kamu mencoba algoritma yang lebih panjang (dengan banyak Gate berurutan), total waktu eksekusi mungkin mendekati atau melebihi T2, menyebabkan keadaan kehilangan koherensi sebelum selesai. Itulah kenapa meningkatkan waktu koherensi dan membuat Gate lebih cepat adalah dua tujuan paling krusial dalam riset perangkat keras kuantum.

```python
from qiskit_ibm_runtime.fake_provider import FakeBrisbane
fake_brisbane = FakeBrisbane()
print("Basis gates for ibmq_brisbane:", fake_brisbane.configuration().basis_gates)
```

Ini mungkin mengeluarkan sesuatu seperti `['id', 'rz', 'sx', 'x', 'ecr']`. Ini adalah operasi primitif yang didukung perangkat keras secara native (Identity/no-op, rotasi RZ, Gate sqrt(X), Gate X, dan controlled-X). Gate lain manapun harus disusun dari ini. Set ini diketahui bersifat universal untuk komputasi kuantum (pada dasarnya rotasi Qubit tunggal ditambah Gate dua-Qubit yang menghasilkan keterjeratan membentuk set universal).

Sekarang, ambil **Gate Toffoli (CCX)** sebagai kasus uji. CCX membalik Qubit target hanya jika dua Qubit kontrol keduanya bernilai 1. Ini bukan Gate native di perangkat keras IBM. Qiskit menyediakan instruksi `ccx`, tapi di balik layar ia akan menguraikannya.

**Universalitas dalam praktik:** Latihan di atas menunjukkan bahwa sebuah Gate 3-Qubit yang kompleks dibangun dari Gate-Gate yang lebih sederhana. Secara umum, **sembarang** unitarian multi-Qubit bisa disusun dari Gate 1- dan 2-Qubit. Transpiler adalah komponen penting dari stack perangkat lunak kuantum manapun, karena ia menjembatani kesenjangan antara algoritma abstrak yang ingin kita jalankan dan operasi fisik yang sebenarnya bisa dilakukan oleh perangkat kuantum tertentu.

**Contoh pada perangkat:** Perangkat **ibmq_brisbane** menggunakan arsitektur Eagle dengan Gate-Gate basis seperti yang ditunjukkan di atas. Artinya, algoritma apapun yang dikirim ke mesin-mesin tersebut akan dikonversi menjadi urutan operasi-operasi tersebut. Kriteria ini pada dasarnya tentang **kemampuan kontrol**; kita punya cukup kenop kontrol untuk melakukan operasi apapun yang diperlukan pada Qubit kita.
## 5. Kriteria 5 – Pengukuran Qubit

**Kriteria 5:** *"Kemampuan pengukuran spesifik per Qubit."* Keadaan setiap Qubit harus bisa diukur (biasanya dalam basis komputasi, |0〉 atau |1〉). Dengan kata lain, setelah menjalankan Circuit kuantum, kita perlu membaca setiap Qubit sebagai bit klasik 0/1. Kriteria ini tentang memiliki detektor yang andal untuk setiap Qubit dan kemampuan untuk memilih Qubit mana yang akan diukur.

**Tujuan Demo:** Menunjukkan cara melakukan pengukuran di Qiskit pada simulator dan perangkat nyata, dan menyoroti perbedaannya (seperti noise pengukuran). Kita akan mengukur beberapa Qubit dalam berbagai keadaan dan memeriksa hasilnya. Kita juga akan mendemonstrasikan bagaimana kesalahan readout mungkin muncul dengan membandingkan hasil simulator vs perangkat keras.

Pertama, contoh pengukuran sederhana:

```python
qc_measure = QuantumCircuit(2, 2)
qc_measure.x(0)              # qubit 0 -> |1>, qubit 1 stays |0>
qc_measure.measure([0, 1], [0, 1])
qc_measure.draw('mpl')
```

```python
sim_backend = AerSimulator()
pm = generate_preset_pass_manager(backend=sim_backend, optimization_level=1)
```

Kita mengharapkan 1000 hitungan `01` pada simulator. Sekarang, mari kita lihat **kesalahan pengukuran** dalam aksi dengan mensimulasikannya. Kita bisa menambahkan kesalahan readout ke simulator Aer kita. Qiskit Aer memungkinkan kita mendefinisikan `ReadoutError` dan melampirkannya ke Qubit dalam model kebisingan.
### Latihan 5: Simulasikan Kesalahan Readout

Lengkapi kode untuk mendefinisikan model kesalahan readout sederhana di mana setiap Qubit punya kemungkinan 2% diukur secara salah (0 terbaca sebagai 1, atau 1 terbaca sebagai 0). Kemudian, jalankan Circuit pengukuran dengan model kebisingan ini.

```python
from qiskit_aer.noise import NoiseModel, ReadoutError

# --- YOUR CODE HERE ---

# 1. Define a 2% readout error for each single qubit.
# [[P(0|0), P(1|0)], [P(0|1), P(1|1)]]
ro_error = ReadoutError([[0.98, 0.02], [0.02, 0.98]])

# 2. Create a new noise model
noise_model_ro = NoiseModel()

# 3. Add the readout error to all qubits in the noise model
noise_model_ro.add_all_qubit_readout_error(ro_error)

# --- END YOUR CODE ---

sim_backend.set_options(noise_model=noise_model_ro)
pm = generate_preset_pass_manager(backend=sim_backend, optimization_level=1)
isa_qc_measure = pm.run(qc_measure)

# Run the measurement circuit with readout noise
sampler = Sampler(mode=sim_backend)

job = sampler.run([isa_qc_measure], shots=1024)
result = job.result()
counts = result[0].data.c.get_counts()

print("Simulation with 2% readout error:", counts)
```

Output simulasi ini akan menunjukkan beberapa hitungan yang salah (seperti `11`, `00`, `10`) mirip dengan apa yang mungkin dihasilkan perangkat keras nyata, mendemonstrasikan dampak pengukuran yang tidak sempurna.

**Contoh pada perangkat:** Di perangkat nyata seperti **ibmq_brisbane**, kamu bisa menjalankan Circuit yang sama dan kemungkinan akan melihat hitungan non-nol yang serupa untuk hasil yang salah. Data kalibrasi perangkat mencantumkan kesalahan readout untuk setiap Qubit. Kemampuan untuk menargetkan dan membaca Qubit tertentu sangat penting, dan memahami karakteristik kesalahannya adalah kunci untuk mendapatkan hasil yang bermakna. ## Kriteria Komunikasi Kuantum (Flying Qubit)

DiVincenzo juga mencantumkan dua kriteria khusus untuk komunikasi kuantum, yang penting jika membangun komputer kuantum berjejaring:

Teleportasi menunjukkan konversi keadaan Qubit stasioner menjadi informasi yang dibawa oleh pasangan yang keterjeratan ("flying" part) dan komunikasi klasik, yang kemudian digunakan untuk merekonstruksi keadaan pada Qubit stasioner lainnya di tempat lain.

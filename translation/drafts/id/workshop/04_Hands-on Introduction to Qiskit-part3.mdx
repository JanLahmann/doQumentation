## 2.5 Skalakan ke Jumlah Qubit yang Lebih Besar {#25-scale-to-large-numbers-of-qubits}

Dalam komputasi kuantum, pekerjaan skala utilitas sangat penting untuk kemajuan di bidang ini. Pekerjaan semacam itu membutuhkan komputasi dalam skala yang jauh lebih besar; bekerja dengan Circuit yang mungkin menggunakan lebih dari 100 Qubit dan lebih dari 1000 Gate. Contoh ini mengambil langkah kecil ke arah itu dengan menskalakan masalah GHZ ke $n=10$ Qubit. Ini menggunakan alur kerja Qiskit patterns dan diakhiri dengan mengukur nilai ekspektasi $\langle Z_0 Z_i \rangle$.

### Step 1. Petakan masalah

Tulis fungsi yang mengembalikan `QuantumCircuit` yang menyiapkan state GHZ $n$-Qubit (pada dasarnya Bell state yang diperluas), lalu gunakan fungsi itu untuk menyiapkan state GHZ 10 Qubit dan kumpulkan observable yang akan diukur.

```python
def get_qc_for_n_qubit_GHZ_state(n: int) -> QuantumCircuit:

    qc = QuantumCircuit(n) 
    qc.h(0)
    for i in range(n-1):
        qc.cx(i, i+1)
    return qc
n = 10
qc_n_GHZ = get_qc_for_n_qubit_GHZ_state(n)
qc_n_GHZ.draw("mpl")
```

```python
def better_ghz(n):
    "fan out"
    s = int(n / 2)
    qc = QuantumCircuit(n)
    qc.h(s)
    for m in range(s, 0, -1):
        qc.cx(m, m - 1)
        if not (n % 2 == 0 and m == s): 
            qc.cx(n - m - 1, n - m)
    return qc

better_ghz(n).draw("mpl")
```

# 3. Encoding Informasi {#3-encoding-information}

## 3.1 Amplitude encoding {#31-amplitude-encoding}

Setelah kita melihat cara membangun Circuit kuantum, menarik untuk mengeksplorasi bagaimana kita bisa mengkodekan informasi klasik ke dalam state kuantum. Salah satu metode yang powerful adalah amplitude encoding, di mana amplitudo state kuantum merepresentasikan komponen dari vektor klasik.

Mari kita lihat contoh sederhana. Misalkan kita ingin mengkodekan vektor klasik

$$
\vec{x} = \begin{bmatrix} x_0 \\ x_1 \\ x_2 \\ x_3 \end{bmatrix}
$$

ke dalam state kuantum dua Qubit. Tujuannya adalah menyiapkan state kuantum:

$$
\ket{\psi} = x_0\ket{00} + x_1\ket{01} + x_2\ket{10} + x_3\ket{11}
$$
di mana $ x_0, x_1, x_2, x_3 \in \mathbb{R} $ (atau $ \mathbb{C} $) dan vektornya dinormalisasi sehingga:

$$
|x_0|^2 + |x_1|^2 + |x_2|^2 + |x_3|^2 = 1
$$

Sekarang kita pertimbangkan contoh khusus: $ \vec{x} = [0.8924,  0.3696, 0.2391, 0.0990] $

Maka state kuantum yang sesuai adalah:

$$
\begin{aligned}
\ket{\psi} &= 0.8924\,\ket{00}
+ 0.3696\,\ket{01}
+ 0.2391\,\ket{10}
+ 0.0990\,\ket{11}
\end{aligned}
$$
State ini bisa disiapkan menggunakan kombinasi rotation Gate $R_y$ dengan sudut $\pi/6$ dan $\pi/4$ untuk Qubit 0 dan 1

```python
from qiskit import QuantumCircuit
from qiskit_aer import AerSimulator
import numpy as np

qc = QuantumCircuit(2)

qc.ry(np.pi / 6, 0)
qc.ry(np.pi / 4, 1)

simulator = AerSimulator()
qc.save_statevector()
result = simulator.run(qc).result()
statevector = result.get_statevector()

print("Statevector:", statevector)
qc.draw(output="mpl")
```

             0.09904576+0.j],
            dims=(2, 2))

## 3.2 Angle encoding dan parametrized circuits {#32-angle-encoding-and-parametrized-circuits}

Cara yang cukup menarik untuk mengkodekan informasi ke komputer kuantum adalah dengan merancang Circuit kuantum yang berisi beberapa sudut rotasi $\vec{\theta}$ atau parameter yang bisa disetel untuk merepresentasikan keluarga fungsi $f(\vec{\theta})$. Mari kita pertimbangkan misalnya Circuit kuantum berparameter berikut:

```python
from qiskit import QuantumCircuit
from qiskit.circuit import Parameter

# Define a symbolic parameter
theta = Parameter("θ")

qc = QuantumCircuit(2)
# We applied a parametrized RX gate
qc.rx(theta, 0)
qc.cx(0, 1)
qc.draw("mpl")
```

![Quantum circuit diagram](./_04_Hands-on Introduction to Qiskit_imgs/output_25.png)

Secara matematis, kita bisa menganalisis keluarga fungsi apa yang bisa kita representasikan dengan Circuit ini:

$$ 
\text{CNOT}_{01} \, R_x^{\{0\}}(\theta) |00\rangle = \text{CNOT}_{01} \left( \cos(\theta/2)\ket{00} - i\sin(\theta/2)\ket{10} \right) = \cos(\theta/2)\ket{00} - i\sin(\theta/2)\ket{11}
$$
Cukup jelas bahwa jumlah state yang bisa kita representasikan dengan Circuit kuantum ini terbatas, karena kita tidak bisa merepresentasikan state $\ket{10}$ atau $\ket{01}$ misalnya. Namun, keluarga state yang bisa kita representasikan mulai berkembang ketika kita memperkenalkan lebih banyak rotasi di tempat yang tepat:

```python
from qiskit import QuantumCircuit
from qiskit.circuit import Parameter

# Define a symbolic parameter
theta1 = Parameter("θ1")
theta2 = Parameter("θ2")

qc = QuantumCircuit(2)
qc.rx(theta1, 0)
qc.rx(theta2, 1)
qc.cx(0, 1)
qc.draw("mpl")
```

![Quantum circuit diagram](./_04_Hands-on Introduction to Qiskit_imgs/output_26.png)

Dalam kasus ini, state kuantum yang akan kita representasikan adalah:

$$
\begin{align*}
\text{CNOT}_{01} \, R_x^{\{1}}(\theta_2) R_x^{\{0}}(\theta_1) \ket{00}
&= \text{CNOT}_{01} \, R_x^{\{1}}(\theta_2)\left( \cos(\theta_1/2)\ket{00} - i\sin(\theta_1/2)\ket{10} \right) \\
&= \text{CNOT}_{01}\left( \cos(\theta_1/2)\cos(\theta_2/2)\ket{00} - i\cos(\theta_1/2)\sin(\theta_2/2)\ket{01} \right. \\
&\quad \left. - i\sin(\theta_1/2)\cos(\theta_2/2)\ket{10} + \sin(\theta_1/2)\sin(\theta_2/2)\ket{11} \right) \\
&= \cos(\theta_1/2)\cos(\theta_2/2)\ket{00} - i\cos(\theta_1/2)\sin(\theta_2/2)\ket{01} \\
&\quad + \sin(\theta_1/2)\sin(\theta_2/2)\ket{10} - i\sin(\theta_1/2)\cos(\theta_2/2)\ket{11} 
\end{align*}
$$
Kita bisa melihat bahwa Circuit ini menghasilkan keluarga state kuantum yang lebih luas dibandingkan yang sebelumnya. Khususnya, sekarang bisa menghasilkan state dengan amplitudo bukan nol untuk $\ket{01}$ atau $\ket{10}$ yang tidak mungkin dengan Circuit di atas. Namun, Circuit ini masih bukan generator state kuantum universal, meskipun mungkin cukup ekspresif untuk merancang Circuit dengan fleksibilitas tertentu dalam merepresentasikan fungsi-fungsi tertentu. Secara umum, semakin banyak parameter independen (sudut) yang kita perkenalkan, semakin besar ekspresivitas Circuit dalam mengaproksimasi state kuantum sembarang.

## 1. Kriteria 1 – Qubit yang Bisa Diskalakan dan Terkarakterisasi dengan Baik

**Kriteria 1:** *"Sistem fisik yang bisa diskalakan dengan Qubit yang terkarakterisasi dengan baik."* Ini berarti kita membutuhkan platform perangkat keras kuantum di mana kita bisa **menambah jumlah Qubit** dan tetap mengendalikannya dengan andal. Properti setiap Qubit (tingkat energi, laju kesalahan, konektivitas, dll.) harus dipahami dengan baik. Intinya, kita ingin membangun Circuit yang lebih besar tanpa sistem tersebut rusak. Dalam praktiknya, saat kita menambah jumlah Qubit atau kedalaman Circuit, kesalahan dan dekoherensi terakumulasi, sehingga mendemonstrasikan *skalabilitas* juga berarti memahami bagaimana peningkatan ukuran memengaruhi performa.

**Tujuan Demo:** Gunakan Qiskit untuk menunjukkan efek penskalaan Circuit (dalam jumlah Qubit atau kedalaman Gate) terhadap fidelitas keluaran. Kita akan mensimulasikan skenario ideal vs berisik untuk melihat bagaimana sistem yang lebih besar atau Circuit yang lebih dalam rentan terhadap dekoherensi dan kesalahan.

Pertama, mari kita buat keadaan terjalin (keadaan GHZ) kecil pada 3 Qubit, lalu yang lebih besar pada 5 Qubit, sebagai uji penskalaan sederhana. Keadaan GHZ dari *n* Qubit adalah $\frac{1}{\sqrt{2}}(|0...0\rangle + |1...1\rangle)$. Dalam simulasi ideal, mengukur GHZ n-Qubit hanya menghasilkan dua keluaran (semua 0 atau semua 1) dengan probabilitas yang sama. Kita akan membandingkan **keluaran ideal** dengan **keluaran berisik** saat kita menambah n atau kedalaman Circuit.

```python
from qiskit import QuantumCircuit
from qiskit_aer import AerSimulator
from qiskit.visualization import plot_histogram
from qiskit.transpiler.preset_passmanagers import generate_preset_pass_manager
from qiskit_ibm_runtime import SamplerV2 as Sampler

# 3-qubit GHZ circuit
qc3 = QuantumCircuit(3, 3)
qc3.h(0)
qc3.cx(0, 1)
qc3.cx(1, 2)
qc3.measure([0, 1, 2], [0, 1, 2])

# 5-qubit GHZ circuit (scaling up the number of qubits)
qc5 = QuantumCircuit(5, 5)
qc5.h(0)
qc5.cx(0, range(1, 5))    # entangle qubit 0 with all others
qc5.measure(range(5), range(5))

# Transpile for a simulator backend
sim_backend = AerSimulator()
pm = generate_preset_pass_manager(backend=sim_backend, optimization_level=1)
isa_qc3 = pm.run(qc3)
isa_qc5 = pm.run(qc5)

# Run ideal simulations (no noise)
sampler = Sampler(mode=sim_backend)

job3 = sampler.run([isa_qc3], shots=1024)
result3 = job3.result()
counts3 = result3[0].data.c.get_counts()

job5 = sampler.run([isa_qc5], shots=1024)
result5 = job5.result()
counts5 = result5[0].data.c.get_counts()

print("3-qubit GHZ counts (ideal):", counts3)
plot_histogram(counts3, legend=['3-qubit ideal'], figsize=(6,4))
```

```python
print("5-qubit GHZ counts (ideal):", counts5)
plot_histogram(counts5, legend=['5-qubit ideal'], figsize=(6,4))
```

**Hasil yang diharapkan (kasus ideal):** GHZ 3-Qubit secara ideal menghasilkan sekitar 50% `000` dan 50% `111` dalam hitungan. GHZ 5-Qubit menghasilkan ~50% `00000` dan 50% `11111`. Tidak ada rangkaian bit lain yang muncul karena keadaan idealnya sepenuhnya koheren dan terjalin. Kamu seharusnya melihat dua batang tinggi di histogram untuk setiap Circuit yang sesuai dengan keluaran semua-nol dan semua-satu.

Selanjutnya, mari lihat apa yang terjadi di **lingkungan berisik**. Kita akan menggunakan kemampuan model noise Qiskit Aer untuk meniru kesalahan perangkat nyata. Misalnya, kita bisa mengambil properti Backend IBM untuk membuat model noise yang mencakup kesalahan Gate, waktu Gate terbatas, relaksasi Qubit (T₁), dephasing (T₂), dan kesalahan readout. Di sini, kita akan menggunakan **fake Backend** yang merepresentasikan perangkat IBM Quantum Brisbane untuk menghasilkan model noise, dan menjalankan ulang Circuit GHZ melaluinya.
### Latihan 1a: Simulasi dengan Noise
Lengkapi kode di bawah ini untuk mensimulasikan Circuit GHZ pada simulator berisik berdasarkan Backend `FakeBrisbane`. Ini akan menunjukkan bagaimana performa menurun saat sistem diskalakan dalam lingkungan noise yang realistis.

```python
from qiskit_ibm_runtime.fake_provider import FakeBrisbane

# We will reuse the ideal circuits qc3 and qc5 and their results from the previous cell.

# 1. Create a fake backend for IBM Quantum Brisbane
brisbane_backend = FakeBrisbane()

# 2. Create a noisy AerSimulator from the fake backend's properties
noisy_sim = AerSimulator.from_backend(brisbane_backend)

# 3. Transpile the circuits for the noisy simulator
pm = generate_preset_pass_manager(backend=noisy_sim, optimization_level=1)

isa_qc3_noisy = pm.run(qc3)

isa_qc5_noisy = pm.run(qc5)

# 4. Run the noisy simulations using the Sampler and get the counts
sampler = Sampler(mode=noisy_sim)

job3 = sampler.run([isa_qc3_noisy], shots=1024)

result3_noisy = job3.result()

counts3_noisy = result3_noisy[0].data.meas.get_counts()

job5 = sampler.run([isa_qc5_noisy], shots=1024)

result5_noisy = job5.result()

counts5_noisy = result5_noisy[0].data.meas.get_counts()

# --- END YOUR CODE ---

# This part is done for you to visualize the results.
print("3-qubit GHZ counts (noisy):", counts3_noisy)
plot_histogram(counts3_noisy, legend=['3-qubit noisy'], figsize=(6,4))
```

```python
print("5-qubit GHZ counts (noisy):", counts5_noisy)
plot_histogram(counts5_noisy, legend=['5-qubit noisy'], figsize=(6,4))
```

**Hasil yang diharapkan (berisik vs ideal):** Dengan noise, baik yang disimulasikan maupun di perangkat nyata, keadaan GHZ **tidak sempurna**. Kamu akan melihat keluaran tambahan selain semua-0 dan semua-1. Untuk 3 Qubit, alih-alih 100% di `000`/`111`, sebagian probabilitas bocor ke rangkaian bit lain (misalnya `001`, `010`, dll.) akibat kesalahan Gate atau dekoherensi yang membalik beberapa Qubit. Untuk 5 Qubit, efeknya bahkan lebih terasa; Circuit yang lebih besar (lebih banyak Qubit dan Gate CNOT) mengakumulasi lebih banyak kesalahan, sehingga puncak semua-0 dan semua-1 lebih rendah, dan banyak keluaran lain yang muncul. Tren ini mengilustrasikan tantangan *skalabilitas*: saat kita menskalakan ke atas, mempertahankan fidelitas tinggi semakin sulit tanpa koreksi kesalahan.

**Wawasan:** Komputer kuantum yang bisa diskalakan perlu menjaga korelasi kuantum saat sistem bertumbuh. Contoh kita menunjukkan bagaimana peningkatan jumlah Qubit/kedalaman Gate menyebabkan fidelitas keluaran menurun ketika noise hadir. Kriteria-kriteria berikutnya akan membahas cara menjaga Qubit tersebut tetap baik (kesalahan rendah, bisa diinisialisasi, dll.) saat kita menskalakan.

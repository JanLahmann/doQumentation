# Kelola pengguna IBM Cloud

Panduan ini menunjukkan cara menggunakan IBM Cloud&reg; untuk mengaktifkan pengguna yang memiliki akun IBM Cloud dan memberikan instruksi bagi pengguna untuk mengakses lingkungan tersebut.

Untuk mengelola pengguna ID provider, ikuti instruksi di topik [Kelola ID providers](/guides/manage-appid).

## Langkah-langkah {#steps}

Ikuti langkah-langkah berikut untuk mengelola pengguna IBM Cloud:

1. Pastikan pengguna yang ingin kamu undang sudah memiliki akun IBM Cloud.
2. Buka IAM dan [undang pengguna](https://cloud.ibm.com/iam/users/invite_users)
   <Admonition type="note">
   Pengguna tidak bisa dikelola sampai mereka menerima undangan dan masuk setidaknya sekali.
   </Admonition>
3. Opsional: Modifikasi penetapan proyek dan grup akses pengguna dari [IAM.](https://cloud.ibm.com/iam/users)

<span id="user-org"></span>
## Alur pengguna {#user-flow}

1. Pengguna menerima undangan, lalu masuk melalui [portal IBM Cloud](https://cloud.ibm.com/).
2. Pengguna membuat API key dengan membuka ([Manage → Access (IAM) → API keys](https://cloud.ibm.com/iam/apikeys)).
3. Untuk informasi lebih lanjut, pengguna bisa membaca [Install Qiskit.](/guides/install-qiskit)

## Contoh skenario {#example-scenario}

Contoh ini membuat pengaturan berikut:

* Ada dua proyek, `ml` dan `finance`.
   * Proyek `ml` harus bisa mengakses instans layanan `QR-ml` dan `QR-common`.
   * Proyek `finance` harus bisa mengakses instans layanan `QR-finance` dan `QR-common`.
* Ada tiga pengguna:
   * Fatima harus bisa mengakses proyek `ml`.
   * Ravi harus bisa mengakses proyek `finance`.
   * Amyra harus bisa mengakses kedua proyek.
* Menggunakan grup akses tanpa grup sumber daya.
* Pengguna didefinisikan di IBM Cloud dan penetapan proyek dilakukan di sana juga.
* Pengguna harus bisa menghapus jobs.

Langkah-langkah untuk mengimplementasikan pengaturan ini adalah:

2. Administrator Cloud membuat tiga instans layanan: `QR-ml`, `QR finance`, dan `QR-common`.
3. Administrator Cloud membuat aturan kustom yang menyertakan aksi `quantum-computing.job.delete`.
1. Administrator Cloud membuat dua grup akses:
   * Grup akses `ml` bisa mengakses `QR-ml` dan `QR-common`.
   * Grup akses `finance` bisa mengakses `QR-finance` dan `QR-common`.
1. Administrator Cloud mengundang pengguna cloud ke proyek yang sesuai. Secara khusus, mereka mengundang dan menetapkan pengguna ke grup akses yang mencakup proyek tersebut.
   * Fatima ditambahkan ke grup akses "ml".
   * Ravi ditambahkan ke grup akses "finance".
   * Amyra ditambahkan ke grup akses "ml" dan "finance".
1. Pengguna bisa masuk melalui portal IBM Cloud, membuat API keys, dan bekerja dengan instans layanan proyek mereka.

# Siapkan akun IBM Cloud kamu

Kalau kamu menerima undangan melalui email untuk bergabung ke sebuah akun, ikuti langkah-langkah di [Siapkan akun IBM Cloud - pengguna yang diundang](/guides/cloud-setup-invited) saja.

Kamu bisa menjalankan beban kerja pada unit pemrosesan kuantum (QPU) IBM&reg; dengan menyiapkan akun di IBM Cloud&reg;. Kamu juga perlu [menginstal dan mengatur Qiskit serta Qiskit Runtime](/guides/install-qiskit#local).

Akun penggunamu dikaitkan dengan satu atau beberapa [instans,](./instances) yang diidentifikasi melalui _Cloud Resource Names_ (CRN) yang memberikan akses ke layanan IBM Quantum. Selain itu, setiap akun diberikan token API unik (juga disebut kunci) yang memungkinkan akses IBM Quantum dari Qiskit. Untuk petunjuk memilih instans tertentu, lihat [Sambungkan ke instans](./instances#connect-instance).

<span id="cloud"></span>
## Siapkan untuk menggunakan IBM Cloud {#set-up-to-use-ibm-cloud}

### Sebelum memulai {#before-you-begin}

1. Kalau belum punya, buat [akun IBM Cloud.](https://quantum.cloud.ibm.com/registration)
   <Admonition type="note">
   Kamu bisa menjadi bagian dari beberapa akun IBM Cloud sekaligus. Kamu bisa mengakses akun IBM Cloud mana pun kapan saja dari pemilih akun di header antarmuka [IBM Quantum Platform](https://quantum.cloud.ibm.com).
   </Admonition>
1. Masuk ke [IBM Quantum Platform](https://quantum.cloud.ibm.com) dengan IBMid atau akun Google. Kalau belum punya, kamu akan dipandu untuk membuatnya. (Kredensial login ini bukan kredensial akun IBM Cloud kamu.)
1. Pastikan akun dan wilayah yang benar sudah dipilih di pemilih akun pada header, seperti yang ditunjukkan pada gambar berikut.

    Wilayah mengontrol di mana job kamu dijalankan dan di mana data job disimpan. Kamu bisa mengakses kedua wilayah menggunakan kunci API yang sama, tetapi kamu hanya bisa melihat dan mengakses instans yang dibuat di wilayah tempat kamu masuk.

    ![Header IBM Quantum Platform ditampilkan. Pemilih akun berada di sebelah kanan bilah pencarian. Pemilih wilayah berada di sebelah kanan pemilih akun.](/docs/images/guides/cloud-setup/IQP-Header.svg 'Header IBM Quantum Platform')

### 1. Buat instans {#1-create-an-instance}

Kamu bisa membuat instans dari halaman [Instances](https://quantum.cloud.ibm.com/instances) di IBM Quantum Platform. Lihat [Buat instans](/guides/instances#add-instance) untuk petunjuk lengkapnya.

### 2. Opsional: Simpan kredensial akses kamu {#2-optional-save-your-access-credentials}

Jika kamu menggunakan lingkungan Python yang terpercaya, seperti laptop pribadi, disarankan untuk menggunakan metode `save_account()` guna [menyimpan kredensial secara lokal.](/guides/save-credentials)

<span id="cloud-save"></span>
### 3. Sambungkan Qiskit dengan instans layanan kamu {#3-connect-qiskit-with-your-service-instance}

- [Inisialisasi layanan](/guides/initialize-account) di **lingkungan Python yang terpercaya**
- [Inisialisasi layanan di **lingkungan yang tidak terpercaya**](/guides/cloud-setup-untrusted) (seperti komputer umum)

<Admonition type="note" title="Catatan">
   * Ikuti [petunjuk ini](/guides/cloud-setup-rest-api) jika kamu ingin menyambungkan menggunakan REST API, bukan Qiskit.
   * Jika diperlukan, gunakan [informasi ini](/guides/quickstart-steps-org#firewall) untuk mengonfigurasi firewall agar akses ke endpoint API IBM Quantum bisa diaktifkan.
</Admonition>

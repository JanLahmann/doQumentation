# Panduan migrasi Qiskit v2.0 {#qiskit-v20-migration-guide}

Qiskit v2.0 adalah langkah berikutnya dalam siklus hidup Qiskit SDK yang baru distabilkan.
Versi mayor ini memang memutus kompatibilitas dengan versi Qiskit sebelumnya, namun,
berbeda dengan rilis v1.0, tidak ada perubahan dalam struktur paketnya.

Panduan ini menjelaskan jalur migrasi untuk perubahan fitur terpenting di Qiskit v2.0, diorganisasikan per modul.
Gunakan daftar isi di sisi kanan untuk navigasi ke modul yang kamu butuhkan.

Baca lebih lanjut tentang manfaat Qiskit v2.0 di [blog IBM&reg;](https://www.ibm.com/quantum/blog/qiskit-2-0-release-summary).

<span id="qiskit.circuit"></span>
## qiskit.circuit

<span id="c_if"></span>
### Penghapusan kondisi `c_if` lama {#removal-of-legacy-c-if-conditions}

Kondisi klasik di Qiskit sudah lebih direpresentasikan oleh instruksi [`IfElseOp`](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.IfElseOp) sejak sebelum Qiskit v1.0.
Operasi-operasi ini biasanya ditambahkan ke [`QuantumCircuit`](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.QuantumCircuit) menggunakan metode [`if_test`](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.QuantumCircuit#qiskit.circuit.QuantumCircuit.if_test) sebagai context manager.

Ini juga berarti bahwa [`Instruction`](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.Instruction) tidak lagi memiliki atribut `condition`, dan kamu tidak bisa merantai panggilan `c_if` pada output dari pemanggilan metode pembuatan Gate di [`QuantumCircuit`](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.QuantumCircuit).

Satu-satunya instruksi yang mungkin memiliki kondisi sekarang adalah [`IfElseOp`](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.IfElseOp) dan [`WhileLoopOp`](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.WhileLoopOp).
Cara paling tepat untuk menguji kondisi adalah menggunakan pengecekan `isinstance` untuk memvalidasi bahwa kamu punya kelas yang benar.
Jika kamu ingin mendukung atribut `Instruction.condition` lama selama transisi library-mu ke Qiskit v2.0 dan seterusnya, kamu bisa menggunakan `getattr(operation, "condition", None)`, yang biasanya akan membuat instance `Instruction` berperilaku seperti di Qiskit v1.x dan sebelumnya.

Metode lama dan asli untuk mendukung kondisi kesetaraan register sederhana di Qiskit adalah menggunakan metode `c_if` pada instruksi individual atau set instruksi.
Cara ini secara ketat lebih sedikit fiturnya dibandingkan bentuk control-flow penuh, dan tidak pernah didukung dengan baik pada hardware.
Metode lama ini sudah dihapus di Qiskit v2.0; kamu harus menggunakan metode [`if_test`](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.QuantumCircuit#qiskit.circuit.QuantumCircuit.if_test) dengan kondisi yang tepat.
Kamu masih bisa menggunakan bentuk 2-tuple `(register, integer)` untuk merepresentasikan kesetaraan register, atau menggunakan [subsistem komputasi klasik](https://docs.quantum.ibm.com/api/qiskit/circuit_classical) yang lebih ekspresif untuk merepresentasikan kondisi yang lebih kompleks.

Misalnya, kamu mungkin pernah menulis:

```python
from qiskit.circuit import QuantumCircuit, QuantumRegister, ClassicalRegister

qr = QuantumRegister(2, "q")
cr = ClassicalRegister(2, "c")
qc = QuantumCircuit(qr, cr)
qc.h(0)
qc.h(1)
qc.measure(qr, cr)

# Apply an X to qubit 0 if both measurements were 1.
qc.x(0).c_if(cr, 3)
```

Terjemahan paling langsung dari ini adalah menggunakan bentuk 2-tuple dengan context manager [`if_test`](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.QuantumCircuit#qiskit.circuit.QuantumCircuit.if_test):

```python
from qiskit.circuit import QuantumCircuit, QuantumRegister, ClassicalRegister

qr = QuantumRegister(2, "q")
cr = ClassicalRegister(2, "c")
qc = QuantumCircuit(qr, cr)
qc.h(0)
qc.h(1)
qc.measure(qr, cr)

# Apply an X to qubit 0 if both measurements were 1.
with qc.if_test((cr, 3)):
    qc.x(0)
```

Perhatikan bahwa dalam bentuk ini, kamu juga bisa membuat Gate kondisional mengakses beberapa operasi (cukup tambahkan lebih banyak ke dalam blok yang diindentasi) tanpa mengevaluasi ulang kondisi.
Kamu juga bisa menyediakan pernyataan ``else``, dan menggunakan logika kondisional yang lebih kompleks:

```python
from qiskit.circuit import QuantumCircuit
from qiskit.circuit.classical import expr

qc = QuantumCircuit(2, 2)
qc.h(0)
qc.h(1)
qc.measure(qc.qubits, qc.clbits)

# Apply an X to qubit 0 if both measurements were the same:
with qc.if_test(expr.equal(qc.clbits[0], qc.clbits[1])) as else_:
    qc.x(0)
# ... or apply a Y to both qubits otherwise:
with else_:
    qc.y(0)
    qc.y(1)
```

<span id="instruction_durations"></span>
### Penghapusan `Instruction.duration` dan `Instruction.unit` {#removal-of-instructionduration-and-instructionunit}

Objek [`Instruction`](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.Instruction) Circuit sebelumnya memiliki atribut bernama `duration` dan `unit`.
Atribut-atribut ini tidak ada lagi di Qiskit v2.0.
Ini karena field tersebut umumnya hanya menduplikasi informasi yang merupakan properti tetap dari hardware, dan sudah tersimpan secara eksplisit di [`Target`](https://docs.quantum.ibm.com/api/qiskit/qiskit.transpiler.Target).
Menyimpan informasi secara terpisah untuk setiap instruksi individual secara signifikan meningkatkan kebutuhan memori Qiskit dan memperlambat bahkan operasi Circuit yang tidak perlu menyentuh informasi tersebut.

Instruksi [`Delay`](https://docs.quantum.ibm.com/api/qiskit/circuit#delay) tetap mengandung field-field ini, begitu pula dengan [`BoxOp`](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.QuantumCircuit#box) yang baru di v2.0, karena instruksi-instruksi ini memiliki delay yang _bervariasi_.

Kamu bisa mengambil durasi yang ditentukan hardware dari suatu [`Instruction`](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.Instruction) yang didukung hardware dengan mengquery [`Target`](https://docs.quantum.ibm.com/api/qiskit/qiskit.transpiler.Target).
Caranya, indeks [`Target`](https://docs.quantum.ibm.com/api/qiskit/qiskit.transpiler.Target) pertama dengan nama operasi, lalu indeks Qubit hardware sebagai tuple.
Ini akan mengembalikan `None` (jika instruksi didukung, tapi tidak ada properti spesifik yang ditetapkan), atau instance [`InstructionProperties`](https://docs.quantum.ibm.com/api/qiskit/qiskit.transpiler.InstructionProperties), yang field [`duration`](https://docs.quantum.ibm.com/api/qiskit/qiskit.transpiler.InstructionProperties#qiskit.transpiler.InstructionProperties.duration)-nya mungkin memberikan durasi dalam detik (atau `None`, jika tidak disediakan).

```python
target = "<some Target from a hardware vendor>"

# Query the duration of a CX instruction on hardware qubits (2, 4),
# returning `None` if the duration is not specified.
properties = target["cx"][2, 4]
duration = None if properties is None else properties.duration

# You can also do this in one step, using `getattr`:
duration = getattr(target["cx"][2, 4], "duration", None)
```

Secara umum, seiring control-flow klasik menjadi lebih umum dalam eksekusi Circuit, "durasi" sebuah Circuit akan semakin tidak terdefinisi dengan baik.
Meskipun Qiskit v2.x masih menyertakan atribut [`QuantumCircuit.duration`](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.QuantumCircuit#qiskit.circuit.QuantumCircuit.duration), disarankan untuk beralih darinya, karena kemungkinan akan dihapus di masa depan; asumsi-asumsi yang mendasarinya akan semakin sering dilanggar.
Sebagai gantinya, kamu bisa menggunakan metode baru [`QuantumCircuit.estimate_duration`](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.QuantumCircuit#qiskit.circuit.QuantumCircuit.estimate_duration), yang lebih aman dan lebih jelas tentang asumsi yang diperlukan.

## qiskit.transpiler

<span id="generate_preset_pass_manager"></span>
### Perubahan pada antarmuka `generate_preset_pass_manager` {#changes-to-generate-preset-pass-manager-interface}

Qiskit v2.0.0 menyertakan serangkaian pembaruan pada antarmuka preset pass manager (yaitu,
fungsi [`generate_preset_pass_manager`](https://docs.quantum.ibm.com/api/qiskit/qiskit.transpiler.generate_preset_pass_manager) dan
[`transpile`](https://docs.quantum.ibm.com/api/qiskit/compiler#transpile)) yang memengaruhi
penanganan input constraint bebas, seperti `coupling_map` atau `basis_gates`. Jumlah
kasus penggunaan transpilasi di mana constraint bebas diterima telah dikurangi demi
input `target` dan `backend` (di mana `backend` adalah [`BackendV2`](https://docs.quantum.ibm.com/api/qiskit/qiskit.providers.BackendV2) yang
mengandung instance [`Target`](https://docs.quantum.ibm.com/api/qiskit/qiskit.transpiler.Target)),
yang memungkinkan komunikasi constraint yang lebih ekspresif.

Contoh-contoh berikut menunjukkan perubahan utama dan alternatif yang disarankan untuk migrasi. Perhatikan bahwa semua
tips migrasi di bawah berlaku untuk `transpile` maupun `generate_preset_pass_manager`:

1. Argumen input `backend_properties` tidak lagi didukung, dan kelas `BackendProperties` telah
   dihapus. Kamu harus menggunakan argumen `target` atau `backend` sebagai gantinya untuk mengkomunikasikan
   durasi dan error Gate. Ada beberapa jalur untuk membangun [`Target`](https://docs.quantum.ibm.com/api/qiskit/qiskit.transpiler.Target) kustom:

  a. Bangun target dari awal dan tambahkan error dan durasi yang relevan ke Gate yang sesuai:

    ```python
    from qiskit.transpiler import Target, InstructionProperties, generate_preset_pass_manager
    from qiskit.circuit.library import XGate, CXGate

    target = Target(num_qubits=2)
    target.add_instruction(CXGate(), {(0, 1): InstructionProperties(error=.0001, duration=5e-7)})
    target.add_instruction(
        XGate(),
        {
            (0,): InstructionProperties(error=.00001, duration=5e-8),
            (1,): InstructionProperties(error=.00002, duration=6e-8)
        }
    )

    pm = generate_preset_pass_manager(target=target)
    ```

  b. Gunakan [`GenericBackendV2`](https://docs.quantum.ibm.com/api/qiskit/qiskit.providers.fake_provider.GenericBackendV2) untuk membangun target generik dengan cepat dengan error dan durasi yang sudah terisi. Ini juga bisa diperbarui setelah konstruksi:

    ```python
    from qiskit.providers.fake_provider import GenericBackendV2
    from qiskit.transpiler import generate_preset_pass_manager

    target = GenericBackendV2(num_qubits=2, basis_gates=["x", "cx"]).target
    target.update_instruction_properties("x", (0,), InstructionProperties(0.00001))
    target.update_instruction_properties("x", (1,), InstructionProperties(0.00002))
    target.update_instruction_properties("cx", (0, 1), InstructionProperties(0.0001))

    pm = generate_preset_pass_manager(target=target)
    ```

2. Argumen input `instruction_durations` dan `timing_constraints` tidak lagi didukung. Kamu harus menggunakan
  argumen `target` atau `backend` sebagai gantinya. Alternatifnya adalah membangun [`Target`](https://docs.quantum.ibm.com/api/qiskit/qiskit.transpiler.Target) kustom dari
  awal, membangun Backend generik, dan, selain itu, memungkinkan untuk membangun target kustom dengan
  `instruction_durations` dan/atau `timing_constraints` menggunakan metode `Target.from_configuration`:

    ```python
    from qiskit.transpiler import Target, generate_preset_pass_manager
    from qiskit.transpiler.timing_constraints import TimingConstraints
    from qiskit.transpiler.instruction_durations import InstructionDurations

    instruction_durations = InstructionDurations([("x", None, 200), ("cx", None, 1000)], dt=1e-7)

    timing_constraints = TimingConstraints(
            granularity=1, min_length=1, pulse_alignment=1, acquire_alignment=1
        )

    target = Target.from_configuration(num_qubits=2,
                                        basis_gates=["x", "cx"],
                                        instruction_durations=instruction_durations,
                                        timing_constraints=timing_constraints)

    pm = generate_preset_pass_manager(target=target)
    ```

3. Menyediakan `coupling_map` dan/atau `basis_gates` bersama dengan `backend` tidak disarankan, dan mulai
  v2.0 akan muncul `UserWarning`. Dalam kasus ini ada beberapa sumber kebenaran,
  dan informasi seperti error Gate dan durasi dari Backend tidak bisa diselesaikan berdasarkan
  informasi yang diberikan.
  Alternatif yang disarankan adalah mendefinisikan target kustom yang menggabungkan constraint yang dipilih baik
  dari awal, menggunakan [`Target.from_configuration`](https://docs.quantum.ibm.com/api/qiskit/qiskit.transpiler.Target#from_configuration) atau [`GenericBackendV2`](https://docs.quantum.ibm.com/api/qiskit/qiskit.providers.fake_provider.GenericBackendV2).

4. Argumen `basis_gates` tidak lagi menerima Gate basis kustom. Alternatif yang disarankan
  adalah argumen `target`, yang bisa dibangun dari awal atau melalui [`Target.from_configuration`](https://docs.quantum.ibm.com/api/qiskit/qiskit.transpiler.Target#from_configuration)
  seperti:

    ```python
    from qiskit.circuit.library import XGate
    from qiskit.transpiler.target import Target

    basis_gates = ["my_x", "cx"]
    custom_name_mapping = {"my_x": XGate()}
    target = Target.from_configuration(
        basis_gates=basis_gates, num_qubits=2, custom_name_mapping=custom_name_mapping
    )
    pm = generate_preset_pass_manager(target=target)
    ```

5. Kombinasi input di mana `coupling_map` kustom disediakan bersama `backend` atau `basis_gates`
  yang mengandung Gate dengan tiga Qubit atau lebih tidak lagi diizinkan. Coupling map
  tidak memberikan detail konektivitas yang diperlukan untuk menentukan aksi Gate.
  Dalam kasus ini, jalur migrasi yang direkomendasikan adalah pembuatan target kustom dari awal.

6. Argumen `inst_map` telah dihapus tanpa alternatif. Argumen ini digunakan untuk menyediakan informasi pulse
  ke Transpiler, dan modul `qiskit.pulse` penuh telah dihapus.

## qiskit.pulse

Modul `qiskit.pulse` telah sepenuhnya dihapus dari Qiskit, tanpa pengganti.
Ini termasuk item-item terkait:

- properti `calibrations` dari [`QuantumCircuit`](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.QuantumCircuit), [`DAGCircuit`](https://docs.quantum.ibm.com/api/qiskit/qiskit.dagcircuit.DAGCircuit) dan [`DAGDependency`](https://docs.quantum.ibm.com/api/qiskit/qiskit.dagcircuit.DAGDependency).
- metode `has_calibration_for` dan `add_calibration` dari [`QuantumCircuit`](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.QuantumCircuit).
- metode `has_calibration_for` dari [`DAGCircuit`](https://docs.quantum.ibm.com/api/qiskit/qiskit.dagcircuit.DAGCircuit).
- pass Transpiler `PulseGates`, `RXCalibrationBuilder`, `RZXCalibrationBuilder`, `EchoRZXWeylDecomposition`, `ValidatePulseGates`.
- modul `qiskit.scheduler` dan semua isinya.
- metode `instruction_schedule_map`, `drive_channel`, `measure_channel`, `acquire_channel` dan `control_channel` dari [`BackendV2`](https://docs.quantum.ibm.com/api/qiskit/qiskit.providers.BackendV2).
- metode `has_calibration`, `get_calibration`, `instruction_schedule_map` dan `update_from_instruction_schedule_map` dari [`Target`](https://docs.quantum.ibm.com/api/qiskit/qiskit.transpiler.Target).
- argumen `inst_map` dari [`generate_preset_pass_manager`](https://docs.quantum.ibm.com/api/qiskit/qiskit.transpiler.generate_preset_pass_manager), [`transpile`](https://docs.quantum.ibm.com/api/qiskit/compiler#transpile), dan [`Target.from_configuration`](https://docs.quantum.ibm.com/api/qiskit/qiskit.transpiler.Target#qiskit.transpiler.Target.from_configuration).
- field `calibration` dari [`InstructionProperties`](https://docs.quantum.ibm.com/api/qiskit/qiskit.transpiler.InstructionProperties).

Ini terjadi karena QPU IBM Quantum&reg; tidak lagi menawarkan akses tingkat pulse, dan tim Qiskit tidak lagi memiliki sumber daya untuk memelihara paket yang jarang digunakan ini.
Kamu masih bisa menggunakan modul `qiskit.pulse` sambil menggunakan seri v1.x dari Qiskit, yang memiliki dukungan perbaikan bug hingga September 2024 dan dukungan keamanan hingga Maret 2025, namun modul ini dihapus dari seri v2.x.

Sebagian besar penggunaan `qiskit.pulse` bisa digantikan dengan akses langsung ke fractional gates pada QPU IBM Quantum Heron (dan generasi selanjutnya, jika berlaku).
Kamu bisa membaca lebih lanjut di [panduan migrasi untuk penggunaan pulse yang dikonversi ke fractional gates](/guides/pulse-migration).

Kamu tidak lagi bisa memuat file QPY yang mengandung objek `ScheduleBlock` di Qiskit v2.0 karena penghapusan Qiskit Pulse.
Kamu masih bisa memuat file-file ini di seri Qiskit v1.x.

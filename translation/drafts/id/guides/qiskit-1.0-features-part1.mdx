# Perubahan fitur Qiskit 1.0

Panduan ini menjelaskan jalur migrasi untuk perubahan fitur terpenting di Qiskit 1.0, yang diorganisasi berdasarkan modul.
Gunakan daftar isi di sisi kanan untuk navigasi ke modul yang kamu butuhkan.

## Alat migrasi Qiskit 1.0 {#qiskit-10-migration-tool}

Untuk mempermudah proses migrasi, kamu bisa menggunakan alat
[`flake8-qiskit-migration`](https://github.com/qiskit-community/flake8-qiskit-migration)
untuk mendeteksi jalur impor yang sudah dihapus di kode kamu dan menyarankan alternatifnya.

<Tabs>
  <TabItem value="flake8-pipx" label="Jalankan dengan pipx" default>
      Kalau kamu sudah menginstal [`pipx`](https://github.com/pypa/pipx), cukup jalankan
      perintah berikut.
      ```sh
      pipx run flake8-qiskit-migration <path-to-source-directory>
      ```
      Ini akan menginstal paket ke virtual environment sementara dan menjalankannya
      pada kode kamu.
  </TabItem>
  <TabItem value="flake8-venv" label="Jalankan dengan venv" default>
      Kalau kamu tidak ingin menggunakan `pipx`, kamu bisa membuat environment baru
      secara manual untuk alat ini. Pendekatan ini juga memungkinkan kamu menggunakan
      [`nbqa`](https://github.com/nbQA-dev/nbQA) untuk memeriksa contoh kode di
      notebook Jupyter. Hapus environment-nya setelah selesai.
      ```sh
      # Make new environment and install
      python -m venv .flake8-qiskit-migration-venv
      source .flake8-qiskit-migration-venv/bin/activate
      pip install flake8-qiskit-migration

      # Run plugin on Python code
      flake8 --select QKT100 <path-to-source-directory>  # e.g. `src/`

      # (Optional) run plugin on notebooks
      pip install nbqa
      nbqa flake8 ./**/*.ipynb --select QKT100

      # Deactivate and delete environment
      deactivate
      rm -r .flake8-qiskit-migration-venv
      ```
  </TabItem>
</Tabs>

<Admonition type="note" title="Keterbatasan">
Alat ini hanya mendeteksi jalur impor yang sudah dihapus. Alat ini tidak mendeteksi penggunaan
metode yang sudah dihapus (seperti `QuantumCircuit.qasm`) maupun argumennya. Alat ini juga tidak
bisa melacak assignment seperti `qk = qiskit`, meskipun bisa menangani alias seperti
`import qiskit as qk`.
</Admonition>

Untuk informasi lebih lanjut, lihat [repositori
proyek ini](https://github.com/qiskit-community/flake8-qiskit-migration).

## Instance dan fungsi global {#global-instances-and-functions}

<span id="Aer"></span>
### Aer

Objek `qiskit.Aer` tidak tersedia di Qiskit 1.0. Sebagai gantinya, gunakan
objek yang sama dari namespace `qiskit_aer`, yang merupakan pengganti langsung.
Untuk menginstal `qiskit_aer`, jalankan:

```bash
pip install qiskit-aer
```

<span id="BasicAer"></span>
### BasicAer

Objek `qiskit.BasicAer` tidak tersedia di Qiskit 1.0. Lihat
[bagian migrasi `basicaer`](#providers.basicaer) untuk
pilihan migrasi.

<span id="execute"></span>
### execute

Fungsi `qiskit.execute` tidak tersedia di Qiskit 1.0. Fungsi ini
berperan sebagai wrapper tingkat tinggi di sekitar fungsi
[`transpile`](https://docs.quantum.ibm.com/api/qiskit/compiler#transpile) dan
[`run`](https://docs.quantum.ibm.com/api/qiskit/qiskit.providers.BackendV2#run) di Qiskit.
Alih-alih `qiskit.execute`, gunakan fungsi
[`transpile`](https://docs.quantum.ibm.com/api/qiskit/compiler#transpile) diikuti oleh
[`backend.run()`](https://docs.quantum.ibm.com/api/qiskit/qiskit.providers.BackendV2#run).

```python
# Legacy path
from qiskit import execute

job = execute(circuit, backend)

# New path
from qiskit import transpile

new_circuit = transpile(circuit, backend)
job = backend.run(new_circuit)
```

Alternatifnya, primitif [Sampler](https://docs.quantum.ibm.com/api/qiskit/1.4/qiskit.primitives.Sampler)
secara semantis setara dengan fungsi `qiskit.execute` yang sudah dihapus.
Kelas [`BackendSampler`](https://docs.quantum.ibm.com/api/qiskit/1.4/qiskit.primitives.BackendSampler) adalah
wrapper generik untuk Backend yang tidak mendukung primitif:

```python
from qiskit.primitives import BackendSampler

sampler = BackendSampler(backend)
job = sampler.run(circuit)
```

<span id="qiskit.circuit"></span>
## qiskit.circuit

<span id="QuantumCircuit.qasm"></span>
### QuantumCircuit.qasm

Metode `QuantumCircuit.qasm` telah dihapus. Sebagai gantinya, gunakan
[`qasm2.dump`](https://docs.quantum.ibm.com/api/qiskit/qasm2#dump) atau
[`qasm2.dumps`](https://docs.quantum.ibm.com/api/qiskit/qasm2#dumps).

Untuk output yang diformat dengan Pygments, lihat paket standalone
[`openqasm-pygments`](https://pypi.org/project/openqasm-pygments/),
karena `qasm2.dump` dan `qasm2.dumps` tidak menyediakan output berwarna Pygments.

```python
from qiskit import QuantumCircuit

qc = QuantumCircuit(1)

# Old
qasm_str = qc.qasm()

# Alternative
from qiskit.qasm2 import dumps

qasm_str = dumps(qc)

# Alternative: Write to file
from qiskit.qasm2 import dump

with open("my_file.qasm", "w") as f:
    dump(qc, f)
```

|------------------------------|----------------------|

| `QuantumCircuit.mct`         | [`QuantumCircuit.mcx`](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.QuantumCircuit#mcx)     |
| `QuantumCircuit.i`           | [`QuantumCircuit.id`](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.QuantumCircuit#id)       |

| `QuantumCircuit.ucrx`        | [`UCRXGate`](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.library.UCRXGate)           |
| `QuantumCircuit.ucry`        | [`UCRYGate`](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.library.UCRYGate)           |

Contohnya, untuk [`DiagonalGate`](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.library.DiagonalGate):

```python
from qiskit.circuit import QuantumCircuit
from qiskit.circuit.library import DiagonalGate  # new location in the circuit library

circuit = QuantumCircuit(2)
circuit.h([0, 1])  # some initial state

gate = DiagonalGate([1, -1, -1, 1])
qubits = [0, 1]  # qubit indices on which to apply the gate
circuit.append(gate, qubits)  # apply the gate
```

| Dihapus                          | Alternatif |
|----------------------------------|-------------|
| `QuantumCircuit.bind_parameters` | [`QuantumCircuit.assign_parameters`](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.QuantumCircuit#assign_parameters) |
| `QuantumCircuit.snapshot`        | [instruksi save](https://qiskit.org/ecosystem/aer/apidocs/circuit.html#saving-simulator-data) dari `qiskit-aer` |

<span id="qiskit.converters"></span>
## qiskit.converters

Fungsi `qiskit.converters.ast_to_dag` telah dihapus dari Qiskit. Fungsi ini mengonversi
abstract syntax tree yang dihasilkan oleh parser OpenQASM 2 lawas ke sebuah
[`DAGCircuit`](https://docs.quantum.ibm.com/api/qiskit/qiskit.dagcircuit.DAGCircuit). Karena parser
OpenQASM 2 lawas sudah dihapus (lihat [qiskit.qasm](#qiskit.qasm)), fungsi ini
tidak lagi berguna. Sebagai gantinya, parse file OpenQASM 2 kamu ke
`QuantumCircuit` menggunakan metode konstruktor
[`QuantumCircuit.from_qasm_file`](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.QuantumCircuit#from_qasm_file)
atau
[`QuantumCircuit.from_qasm_str`](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.QuantumCircuit#from_qasm_str)
(atau modul [`qiskit.qasm2`](https://docs.quantum.ibm.com/api/qiskit/qasm2)), lalu
konversi `QuantumCircuit` tersebut ke
[`DAGCircuit`](https://docs.quantum.ibm.com/api/qiskit/qiskit.dagcircuit.DAGCircuit) dengan
[`circuit_to_dag`](https://docs.quantum.ibm.com/api/qiskit/converters#circuit_to_dag).

```python
# Previous
from qiskit.converters import ast_to_dag
from qiskit.qasm import Qasm

dag = ast_to_dag(Qasm(filename="myfile.qasm").parse())

# Current alternative
import qiskit.qasm2
from qiskit.converters import circuit_to_dag

dag = circuit_to_dag(qiskit.qasm2.load("myfile.qasm"))
```

<span id="qiskit.extensions"></span>
## qiskit.extensions

Modul `qiskit.extensions` tidak lagi tersedia. Sebagian besar objeknya telah
diintegrasikan ke dalam circuit library
([`qiskit.circuit.library`](https://docs.quantum.ibm.com/api/qiskit/circuit_library)). Untuk migrasi ke
lokasi baru, cukup ganti `qiskit.extensions` dengan `qiskit.circuit.library`
di jalur impor objek. Ini adalah pengganti langsung.

```python
# Previous
from qiskit.extensions import DiagonalGate

# Current alternative
from qiskit.circuit.library import DiagonalGate
```

Kelas-kelas yang dipindahkan ke [`qiskit.circuit.library`](https://docs.quantum.ibm.com/api/qiskit/circuit_library) adalah:

* [`DiagonalGate`](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.library.DiagonalGate)
* [`HamiltonianGate`](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.library.HamiltonianGate)
* [`Initialize`](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.library.Initialize)
* [`Isometry`](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.library.Isometry)
* `qiskit.circuit.library.generalized_gates.mcg_up_diag.MCGupDiag`
* [`UCGate`](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.library.UCGate)
* [`UCPauliRotGate`](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.library.UCPauliRotGate)
* [`UCRXGate`](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.library.UCRXGate)
* [`UCRYGate`](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.library.UCRYGate)
* [`UCRZGate`](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.library.UCRZGate)
* [`UnitaryGate`](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.library.UnitaryGate)

Kelas-kelas berikut telah dihapus dari codebase, karena fungsinya
sudah redundan atau terkait dengan modul `extensions`:

Dihapus                          |   Alternatif
---------------------------------|--------------------------------------------
`SingleQubitUnitary`    |  [`qiskit.circuit.library.UnitaryGate`](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.library.UnitaryGate)
`Snapshot`             |  Gunakan [instruksi save](https://qiskit.org/ecosystem/aer/apidocs/circuit.html#saving-simulator-data) dari `qiskit-aer`
`ExtensionError` | Kelas error yang relevan

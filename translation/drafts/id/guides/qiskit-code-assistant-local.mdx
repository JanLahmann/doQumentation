# Gunakan Qiskit Code Assistant dalam mode lokal {#use-qiskit-code-assistant-in-local-mode}

Pelajari cara menginstal, mengonfigurasi, dan menggunakan model Qiskit Code Assistant di mesin lokalmu.

<Admonition type="note" title="Catatan">
    - Qiskit Code Assistant masih dalam status rilis preview dan dapat berubah sewaktu-waktu.
    - Kalau kamu punya masukan atau ingin menghubungi tim pengembang, gunakan [channel Qiskit Slack Workspace](https://qiskit.enterprise.slack.com/archives/C07LYA6PL83) atau repositori GitHub publik yang terkait.
</Admonition>

## Mulai cepat (direkomendasikan) {#quick-start-recommended}

Cara termudah untuk memulai Qiskit Code Assistant dalam mode lokal adalah menggunakan skrip setup otomatis untuk ekstensi VS Code atau JupyterLab. Skrip ini akan secara otomatis menginstal [Ollama](https://ollama.com/) untuk menjalankan LLM, mengunduh model yang direkomendasikan, dan mengonfigurasi ekstensi untukmu.

### Setup ekstensi VS Code {#vs-code-extension-setup}

Jalankan perintah berikut di terminal:

```bash
bash <(curl -fsSL https://raw.githubusercontent.com/Qiskit/qiskit-code-assistant-vscode/main/setup_local.sh)
```

Skrip ini melakukan langkah-langkah berikut:
- Menginstal Ollama (jika belum terinstal)
- Mengunduh dan mengonfigurasi model Qiskit Code Assistant yang direkomendasikan
- Menyiapkan ekstensi VS Code agar bekerja dengan deployment lokal

### Setup ekstensi JupyterLab {#jupyterlab-extension-setup}

Jalankan perintah berikut di terminal:

```bash
bash <(curl -fsSL https://raw.githubusercontent.com/Qiskit/qiskit-code-assistant-jupyterlab/main/setup_local.sh)
```

Skrip ini akan:
- Menginstal Ollama (jika belum terinstal)
- Mengunduh dan mengonfigurasi model Qiskit Code Assistant yang direkomendasikan
- Menyiapkan ekstensi JupyterLab agar bekerja dengan deployment lokal

### Menggunakan aplikasi Ollama {#using-the-ollama-application}

Aplikasi Ollama menyediakan solusi sederhana untuk menjalankan LLM secara lokal. Mudah digunakan, dengan CLI yang membuat seluruh proses setup, manajemen model, dan interaksi cukup mudah. Cocok untuk eksperimen cepat dan bagi pengguna yang ingin menangani lebih sedikit detail teknis.

#### Instal Ollama {#install-ollama}

1. Unduh [aplikasi Ollama](https://ollama.com/download)
1. Instal file yang sudah diunduh
1. Jalankan aplikasi Ollama yang sudah terinstal

    <Admonition type="info">Aplikasi berjalan dengan sukses ketika ikon Ollama muncul di menu bar desktop. Kamu juga bisa memverifikasi layanan berjalan dengan membuka `http://localhost:11434/`.</Admonition>

1. Coba Ollama di terminal dan mulai menjalankan model. Contoh:

    ```
    ollama run hf.co/Qiskit/Qwen2.5-Coder-14B-Qiskit
    ```

#### Siapkan Ollama menggunakan integrasi Hugging Face Hub {#set-up-ollama-using-the-hugging-face-hub-integration}

[Integrasi Ollama/Hugging Face Hub](https://huggingface.co/docs/hub/ollama) menyediakan cara untuk berinteraksi dengan model yang di-host di Hugging Face Hub tanpa perlu membuat modelfile baru atau mengunduh file GGUF atau safetensors secara manual. File `template` dan `params` default sudah disertakan untuk model di Hugging Face Hub.

1. Pastikan aplikasi Ollama sedang berjalan.

1. Buka halaman model yang diinginkan, dan salin URL-nya. Contoh, https://huggingface.co/Qiskit/Qwen2.5-Coder-14B-Qiskit-GGUF.

1. Dari terminal, jalankan perintah:

    ```
    ollama run hf.co/Qiskit/Qwen2.5-Coder-14B-Qiskit
    ```

Kamu bisa menggunakan model `hf.co/Qiskit/Qwen2.5-Coder-14B-Qiskit` atau salah satu model GGUF resmi yang direkomendasikan saat ini: `hf.co/Qiskit/mistral-small-3.2-24b-qiskit-GGUF` atau `hf.co/Qiskit/granite-3.3-8b-qiskit-GGUF`.

#### Siapkan Ollama dengan model GGUF Qiskit Code Assistant yang diunduh secara manual {#set-up-ollama-with-a-manually-downloaded-qiskit-code-assistant-gguf-model}

Kalau kamu sudah mengunduh model GGUF secara manual seperti https://huggingface.co/Qiskit/Qwen2.5-Coder-14B-Qiskit-GGUF dan ingin bereksperimen dengan template dan parameter yang berbeda, ikuti langkah-langkah berikut untuk memuatnya ke aplikasi Ollama lokalmu.

1. Buat `Modelfile` dengan memasukkan konten berikut dan pastikan untuk memperbarui `<PATH-TO-GGUF-FILE>` ke path sebenarnya dari model yang sudah diunduh.

    ```
    FROM <PATH-TO-GGUF-FILE>
    TEMPLATE """{{ if .System }}
    System:
    {{ .System }}

    {{ end }}{{ if .Prompt }}Question:
    {{ .Prompt }}

    {{ end }}Answer:
    ```python{{ .Response }}
    """

    PARAMETER stop "Question:"
    PARAMETER stop "Answer:"
    PARAMETER stop "System:"
    PARAMETER stop "```"

    PARAMETER temperature 0
    PARAMETER top_k 1
    ```

1. Jalankan perintah berikut untuk membuat instance model kustom berdasarkan `Modelfile`.

    ```
    ollama create Qwen2.5-Coder-14B-Qiskit -f ./path-to-model-file
    ```

    <Admonition type="note">Proses ini mungkin membutuhkan waktu karena Ollama perlu membaca file model, menginisialisasi instance model, dan mengonfigurasinya sesuai spesifikasi yang diberikan.</Admonition>

#### Jalankan model Qiskit Code Assistant yang diunduh secara manual di Ollama {#run-the-qiskit-code-assistant-model-manually-downloaded-in-ollama}

Setelah model `Qwen2.5-Coder-14B-Qiskit` sudah disiapkan di Ollama, jalankan perintah berikut untuk meluncurkan model dan berinteraksi dengannya di terminal (dalam mode chat).

```
ollama run Qwen2.5-Coder-14B-Qiskit
```

Beberapa perintah yang berguna:

- `ollama list` - Daftar model di komputermu
- `ollama rm Qwen2.5-Coder-14B-Qiskit` - Hapus model
- `ollama show Qwen2.5-Coder-14B-Qiskit` - Tampilkan informasi model
- `ollama stop Qwen2.5-Coder-14B-Qiskit` - Hentikan model yang sedang berjalan
- `ollama ps` - Daftar model yang sedang dimuat

<summary>Deploy model Qiskit Code Assistant secara lokal melalui paket `llama-cpp-python` secara manual</summary>

<span id="use-llama-cpp-python"></span>
Alternatif dari aplikasi Ollama adalah paket `llama-cpp-python`, yang merupakan binding Python untuk `llama.cpp`. Paket ini memberikan lebih banyak kontrol dan fleksibilitas untuk menjalankan model GGUF secara lokal, dan cocok untuk pengguna yang ingin mengintegrasikan model lokal dalam workflow dan aplikasi Python mereka.

1. Instal [`llama-cpp-python`](https://pypi.org/project/llama-cpp-python/)
2. Berinteraksi dengan model dari dalam aplikasimu menggunakan `llama_cpp`. Contoh:

```python
from llama_cpp import Llama

model_path = <PATH-TO-GGUF-FILE>

model = Llama(
        model_path,
        seed=17,
        n_ctx=10000,
        n_gpu_layers=37, # to offload in gpu, but put 0 if all in cpu
    )

input = 'Generate a quantum circuit with 2 qubits'
raw_pred = model(input)["choices"][0]["text"]
```

Kamu juga bisa menambahkan parameter text generation ke model untuk menyesuaikan inferensi:

```python
generation_kwargs = {
        "max_tokens": 512,
        "echo": False, # Echo the prompt in the output
        "top_k": 1
    }

raw_pred = model(input, **generation_kwargs)["choices"][0]["text"]
```

### Gunakan library `llama.cpp` {#use-the-llamacpp-library}

Alternatif lain adalah menggunakan `llama.cpp`, library open-source untuk melakukan inferensi LLM pada CPU dengan setup minimal.
Library ini memberikan kontrol level rendah atas eksekusi model dan biasanya dijalankan dari command line, menunjuk ke file model GGUF lokal.

Ada beberapa cara untuk menginstal `llama.cpp` di mesinmu:

- Instal llama.cpp menggunakan [brew, nix, atau winget](https://github.com/ggml-org/llama.cpp/blob/master/docs/install.md)
- Jalankan dengan Docker: Lihat [dokumentasi Docker oleh tim `llama.cpp`](https://github.com/ggml-org/llama.cpp/blob/master/docs/docker.md)
- Unduh binary yang sudah di-build dari [halaman releases](https://github.com/ggml-org/llama.cpp/releases)
- Build dari sumber dengan [meng-clone repositori ini](https://github.com/ggml-org/llama.cpp/blob/master/docs/build.md)

Setelah terinstal, kamu bisa menggunakan `llama.cpp` untuk berinteraksi dengan model GGUF dalam mode percakapan seperti berikut:
```bash
# Use a local model file
llama-cli -m my_model.gguf -cnv

# Or download and run a model directly from Hugging Face
llama-cli -hf Qiskit/Qwen2.5-Coder-14B-Qiskit-GGUF -cnv
```

Kamu juga bisa meluncurkan server API yang kompatibel dengan OpenAI untuk model dengan cara berikut:
```bash
llama-server -hf Qiskit/Qwen2.5-Coder-14B-Qiskit-GGUF
```

#### Parameter lanjutan {#advanced-parameters}
Dengan program `llama-cli`, kamu bisa mengontrol generasi model menggunakan opsi command-line. Misalnya, kamu bisa memberikan prompt "system" awal menggunakan flag `-p/--prompt`. Dalam mode percakapan (`-cnv`), prompt awal ini berperan sebagai pesan sistem. Jika tidak, kamu bisa langsung menambahkan instruksi yang diinginkan ke awal teks promptmu. Kamu juga bisa menyesuaikan parameter sampling - misalnya: temperature (`--temp`), top-k (`--top-k`), top-p (`--top-p`), repetition penalty (`--repeat-penalty`), dan seed yang digunakan (`--seed`). Berikut adalah contoh pemanggilan menggunakan opsi-opsi ini:

```bash
llama-cli -hf Qiskit/Qwen2.5-Coder-14B-Qiskit-GGUF \
  -p "You are a friendly assistant." -cnv \
  --temp 0.7 \
  --top-k 50 \
  --top-p 0.95 \
  --repeat-penalty 1.1 \
  --seed 42
```

Untuk memastikan fungsi model Qiskit kami berjalan dengan baik, kami merekomendasikan menggunakan system prompt yang tersedia di repositori HF GGUF kami: system prompt untuk [mistral-small-3.2-24b-qiskit-GGUF](https://huggingface.co/Qiskit/mistral-small-3.2-24b-qiskit-GGUF/blob/main/system), [Qwen2.5-Coder-14B-Qiskit-GGUF](https://huggingface.co/Qiskit/Qwen2.5-Coder-14B-Qiskit-GGUF/blob/main/system), [granite-3.3-8b-qiskit-GGUF](https://huggingface.co/Qiskit/granite-3.3-8b-qiskit-GGUF/blob/main/system), dan [granite-3.2-8b-qiskit-GGUF.](https://huggingface.co/Qiskit/granite-3.2-8b-qiskit-GGUF/blob/main/system)

<summary>Hubungkan ekstensi ke deployment lokal secara manual</summary>

Gunakan ekstensi VS Code dan ekstensi JupyterLab untuk Qiskit Code Assistant guna meminta model Qiskit Code Assistant yang di-deploy secara lokal. Setelah kamu menyiapkan aplikasi Ollama [dengan model](#using-the-ollama-application), kamu bisa mengonfigurasi ekstensi untuk terhubung ke layanan lokal.

### Hubungkan dengan ekstensi VS Code Qiskit Code Assistant {#connect-with-the-qiskit-code-assistant-vs-code-extension}

Dengan ekstensi VS Code Qiskit Code Assistant, kamu bisa berinteraksi dengan model dan melakukan code completion saat menulis kode. Ini bisa bekerja dengan baik untuk pengguna yang mencari bantuan menulis kode Qiskit untuk aplikasi Python mereka.

1. Instal [ekstensi VS Code Qiskit Code Assistant](/guides/qiskit-code-assistant-vscode).
1. Di VS Code, buka **User Settings** dan atur **Qiskit Code Assistant: Url** ke URL deployment Ollama lokalmu (misalnya, `http://localhost:11434`).
1. Muat ulang VS Code dengan membuka **View > Command Palette...** dan memilih **Developer: Reload Window**.

Model Qiskit Code Assistant yang dikonfigurasi di Ollama akan muncul di status bar dan siap digunakan.

### Hubungkan dengan ekstensi JupyterLab Qiskit Code Assistant {#connect-with-the-qiskit-code-assistant-jupyterlab-extension}

Dengan ekstensi JupyterLab Qiskit Code Assistant, kamu bisa berinteraksi dengan model dan melakukan code completion langsung di Jupyter Notebook. Pengguna yang terutama bekerja dengan Jupyter Notebook bisa memanfaatkan ekstensi ini untuk lebih meningkatkan pengalaman menulis kode Qiskit.

1. Instal [ekstensi JupyterLab Qiskit Code Assistant](/guides/qiskit-code-assistant-jupyterlab).
1. Di JupyterLab, buka **Settings Editor** dan atur **Qiskit Code Assistant Service API** ke URL deployment Ollama lokalmu (misalnya, `http://localhost:11434`).

Model Qiskit Code Assistant yang dikonfigurasi di Ollama akan muncul di status bar dan siap digunakan.

</details>

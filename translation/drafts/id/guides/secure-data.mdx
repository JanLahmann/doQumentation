# Mengamankan datamu {#securing-your-data}

Untuk memastikan kamu bisa mengelola datamu dengan aman saat menggunakan IBM Quantum&reg; Platform, penting untuk mengetahui data apa saja yang disimpan dan dienkripsi, serta bagaimana cara menghapus data yang tersimpan.

## Melindungi data sensitifmu {#protecting-your-sensitive-data}

Data yang kamu simpan di IBM Cloud&reg; dienkripsi saat tidak aktif menggunakan kunci yang dibuat secara acak.

## Menghapus datamu {#deleting-your-data}

Menghapus instansi layanan akan menghapus semua konten yang terkait dengan instansi tersebut, seperti job, hasil, parameter, dan program. Untuk menghapus instansi, dari [halaman Instances,](https://quantum.cloud.ibm.com/instances) temukan instansi yang ingin dihapus, klik menu overflow-nya, lalu klik **Delete**. Kamu akan diminta untuk mengonfirmasi penghapusan.

Jika kamu memiliki peran Manager, kamu juga bisa menghapus job individual beserta datanya. Lihat selengkapnya tentang kebijakan akses, grup, dan peran di [dokumentasi](/guides/access-groups).

## Kebijakan retensi data {#data-retention-policy}

IBM akan menyimpan datamu (yang belum dihapus) selama tiga tahun selama kamu masih menjadi pengguna aktif layanan ini. Untuk mengetahui informasi tentang berapa lama data disimpan oleh IBM setelah kamu berhenti menggunakan layanan, lihat [Qiskit Runtime data sheet](https://www.ibm.com/support/customer/csol/terms/?id=BB6EE1D06BDE11EBA199CCFE56FA6D9D), yang tersedia di situs IBM Terms.

## 2. קריטריון 2 – אתחול Qubit {#criterion-2-qubit-initialization}

**קריטריון 2:** *"היכולת לאתחל את מצב ה-Qubits למצב ייחוס פשוט, כמו |000…〉."* כל ה-Qubits צריכים להתחיל בצורה אמינה במצב ייחוס ידוע (בדרך כלל מצב הבסיס |0〉 לכל Qubit). האתחול חיוני כדי שהאלגוריתמים יתחילו מנקודת פתיחה נקייה. בפועל, במכשירים קוואנטיים של IBM, כל Qubit מאופס אוטומטית ל-|0〉 בתחילת כל ביצוע Circuit. Qiskit גם מספקת הוראות לאפס Qubits או להכין מצבים מותאמים אישית במהלך חישוב.

**מטרת ההדגמה:** להראות כיצד לאתחל Qubits ב-Qiskit, הן בתחילה והן באמצע ה-Circuit. נדגים שימוש בהוראת ה-`reset` ובשיטות הכנת מצב.
### תרגיל 2: הכנת מצב ספציפי {#exercise-2-prepare-a-specific-state}
בלוק הקוד שלמטה, השלם את ה-`QuantumCircuit` כדי להכין את המצב $|10\rangle$. כלומר, Qubit 0 צריך להיות במצב $|0\rangle$ ו-Qubit 1 במצב $|1\rangle$. השתמש בשער ובהוראה המתאימים כדי להשיג זאת.

```python
from qiskit import QuantumCircuit
from qiskit_aer import AerSimulator

# Create a circuit to initialize qubits to |10> and verify by measurement
qc_init = QuantumCircuit(2, 2)

# --- YOUR CODE HERE ---

# 1. Set qubit 1 to the |1> state
qc_init.x(1)

# 2. Explicitly reset qubit 0 to the |0> state
qc_init.reset(0)

# --- END YOUR CODE ---

qc_init.measure([0, 1], [0, 1])
qc_init.draw('mpl')
```

```python
# Run the circuit and check the outcome
sim_backend = AerSimulator()
pm = generate_preset_pass_manager(backend=sim_backend, optimization_level=1)
isa_qc_init = pm.run(qc_init)

sampler = Sampler(mode=sim_backend)

job = sampler.run([isa_qc_init], shots=1024)
result = job.result()
counts = result[0].data.c.get_counts()

print("Outcome of |10> state measured in Z-basis:", counts)
plot_histogram(counts)
```

אמורים לראות `10` (בינארי עבור qubit1=1, qubit0=0) עם הסתברות של 100% מהסימולציה, כלומר Qubit 1 הוכן בהצלחה ב-|1〉 ו-Qubit 0 ב-|0〉.

עכשיו, להכנת מצב כללי יותר, Qiskit מאפשרת אתחול למצבים שרירותיים בעזרת שיטת `initialize`. לדוגמה, בואו נכין Qubit במצב $|+\rangle = (|0\rangle+|1\rangle)/\sqrt{2}$, שהוא מצב סופרפוזיציה, וזוג Qubits במצב Bell $(|00\rangle+|11\rangle)/\sqrt{2}$:

```python
import numpy as np

# Initialize a single qubit in |+> state and measure in Z-basis
qc_plus = QuantumCircuit(1, 1)
state_plus = [1/np.sqrt(2), 1/np.sqrt(2)]   # amplitude for |0> and |1>
qc_plus.initialize(state_plus, 0)
qc_plus.measure(0, 0)

# Initialize two qubits in a Bell state manually
qc_bell = QuantumCircuit(2, 2)
bell_state = [1/np.sqrt(2), 0, 0, 1/np.sqrt(2)]  # amplitudes for |00>,|01>,|10>,|11>
qc_bell.initialize(bell_state, [0, 1])
qc_bell.measure([0, 1], [0, 1])

# Transpile and run the initialization circuits
isa_qc_plus = pm.run(qc_plus)
job_plus = sampler.run([isa_qc_plus], shots=1024)
result_plus = job_plus.result()
counts_plus = result_plus[0].data.c.get_counts()

print("Outcome of |+> state measured in Z-basis:", counts_plus)

isa_qc_bell = pm.run(qc_bell)
job_bell = sampler.run([isa_qc_bell], shots=1024)
result_bell = job_bell.result()
counts_bell = result_bell[0].data.c.get_counts()

print("Outcome of Bell state measured in Z-basis:", counts_bell)
```

**תוצאות צפויות:** מצב |+〉 של Qubit בודד, כשמודדים אותו, יניב `0` ו-`1` עם הסתברות של כ-50% כל אחד. מדידת מצב Bell אמורה לתת כ-50% `00` וכ-50% `11`. אם רואים את זה, הדבר מאשר שהאתחול לאותם מצבים הצליח.

**אתחול באמצע Circuit:** ה-`reset` של Qiskit ניתן לשימוש באמצע Circuit כדי לאתחל מחדש Qubit ל-|0〉 תוך כדי ריצה. לדוגמה, בקודי תיקון שגיאות או באלגוריתמים איטרטיביים, לעיתים קרובות מודדים Qubit ואז מאפסים אותו לשימוש חוזר. פעולת ה-`reset` היא דטרמיניסטית; היא מוחקת כל מצב קיים ומקררת את ה-Qubit חזרה למצב הבסיס.

**דוגמת מכשיר:** בחומרה כמו **ibmq_brisbane** (127 Qubits) או כל מכשיר של IBM, כל ה-Qubits מתחילים ב-|0〉 כברירת מחדל כשמריצים משימה. אם צריך מצב התחלה שונה, יש להחיל Gates בתחילת ה-Circuit (כמו שעשינו עם X כדי לקבל |1〉). אתחול מחדש רציף (לצורך תיקון שגיאות קוואנטיות) הוא נושא מחקר פעיל כי ביצועו במהירות הוא אתגר. למרבה המזל, לשימוש בסיסי, היכולת להתחיל מחדש ב-|0…0〉 זמינה, והראינו כיצד להשיג גם מצבים התחלתיים רצויים אחרים.
## 3. קריטריון 3 – זמן קוהרנטיות ארוך (דקוהרנטיות לעומת זמן Gate) {#criterion-3-long-coherence-time}

**קריטריון 3:** *"זמני דקוהרנטיות רלוונטיים ארוכים, ארוכים בהרבה מזמן פעולת ה-Gate."* זה מתייחס לצורך של Qubits לשמור על מצבם הקוואנטי זמן מספיק לביצוע הפעולות הנדרשות. לכל Qubit יש **זמן T₁** (זמן רגיעת אנרגיה, כמה מהר |1〉 דועך ל-|0〉) ו**זמן T₂** (זמן דיפאזינג, כמה מהר הקוהרנטיות של פאזה יחסית אובדת). כדי שמחשב קוואנטי יעבוד, סקאלות הזמן הללו צריכות לעלות בהרבה על משך פעולות ה-Gate.

**מטרת ההדגמה:** לחקור את קוהרנטיות Qubit ב-Qiskit על ידי הצגת השפעת הדקוהרנטיות על תוצאות Circuit כשאורך הביצוע גדל. נשתמש ב-Backend מדומה עם זמני T1/T2 ידועים כדי לדמות את ההשפעה הזו.

כדי **להדגים את ההשפעה של קוהרנטיות סופית**, נדמה ניסוי דעיכת T1. נכין Qubit במצב |1〉, נחכה זמן מה בעזרת הוראת `delay`, ואז נמדוד. אנחנו מצפים שהסתברות המדידה של |1〉 תקטן ככל שהעיכוב גדל.

```python
# This part is done for you. We are creating a list of circuits,
# each with a different delay time.

time_delays_ns = [0, 50000, 100000, 150000, 200000, 250000, 300000]  # delay durations in ns

decay_expts = []
for delay in time_delays_ns:
    qc = QuantumCircuit(1, 1)
    qc.x(0)  # initialize qubit to |1>
    if delay > 0:
        qc.delay(delay, 0, unit='ns')  # wait 'delay' nanoseconds
    qc.measure(0, 0)
    decay_expts.append(qc)

decay_expts[1].draw('mpl') # Visualize one of the circuits
```

### תרגיל 3: דמיית ניסוי דעיכת T1 {#exercise-3-simulate-a-t1-decay-experiment}

עכשיו, השתמש בסימולטור עם רעש המבוסס על `FakeVigo` (שיש לו זמני T1 של כ-50-100 µs) כדי להריץ את ה-Circuits האלה. הסימולטור יחיל אוטומטית שגיאות T1/T2 במהלך הוראות ה-`delay`. בצע Transpile על ה-Circuits לאותו Backend והפעל אותם.

```python
from qiskit_ibm_runtime.fake_provider import FakeVigoV2 as FakeVigo
from qiskit_aer import AerSimulator

# --- YOUR CODE HERE ---

# 1. Create a noisy simulator from the FakeVigo backend
sim_vigo = AerSimulator.from_backend(FakeVigo())

# 2. Transpile the list of circuits for this simulator
pm = generate_preset_pass_manager(backend=sim_vigo, optimization_level=1)

isa_decay_expts = pm.run(decay_expts)

# 3. Use the Sampler to run all the transpiled circuits in a single job
sampler = Sampler(mode=sim_vigo)

job = sampler.run(isa_decay_expts, shots=1000)

result = job.result()

# --- END YOUR CODE ---

# This part is done for you to analyze and print the results.
for idx, (delay, qc) in enumerate(zip(time_delays_ns, isa_decay_expts)):
    counts = result[idx].data.c.get_counts()
    p1 = counts.get('1', 0) / 1000  # Assuming 1000 shots
    print(f"Delay {delay} ns: P(qubit=1) = {p1:.3f}")
```

## 4. קריטריון 4 – סט אוניברסלי של Quantum Gates {#criterion-4-universal-set-of-quantum-gates}

**קריטריון 4:** *"סט 'אוניברסלי' של Quantum Gates."* כלומר, החומרה שלנו חייבת לאפשר לנו לבצע *כל* חישוב קוואנטי על ידי הרכבת סט סופי של Gates בסיסיים. בחישוב קלאסי, NAND הוא אוניברסלי; בקוואנטום יש אפשרויות רבות לסט Gates אוניברסלי (לדוגמה \{H, T, CNOT\} או ה-Gates הנייטיביים של מכשיר מסוים). מכשירי IBM, לדוגמה, מכילים סט של פעולות נייטיביות כמו סיבובים חד-Qubit שרירותיים ו-CNOTs בין Qubits מסוימים, שיחד הם אוניברסליים. תפקידו של Qiskit לעיתים קרובות הוא **לקמפל Gates ברמה גבוהה ל-Gates הבסיסיים האלה**.

**מטרת ההדגמה:** להמחיש את אוניברסליות ה-Gate על ידי הצגת כיצד Qiskit מפרק Gates. ניקח Gate לא נייטיבי (כמו Gate Toffoli בן 3 Qubits, CCX) ונראה כיצד הוא מתפרק ל-Gates הבסיסיים של המכשיר. זה מדגים שסט ה-Gates שסופק הוא אכן *אוניברסלי* – הוא יכול לייצר את הפעולה המורכבת יותר.

ראשית, בואו נראה מהם ה-Gates הבסיסיים של Backend טיפוסי של IBM. נשאל את תצורת המכשיר (או את הגרסה המדומה שלו). לדוגמה, ה-Gates הבסיסיים של ibmq_brisbane:
אמורים לצפות שההסתברות `P(qubit=1)` יורדת ככל שזמן העיכוב גדל, בעקבות עקומת דעיכה מעריכית האופיינית לרגיעת T1. זה מדגים ישירות כיצד זמן קוהרנטיות סופי מוביל לשגיאות חישוביות אם ה-Circuit פועל זמן רב מדי.

**השפעה על אלגוריתמים:** אם מנסים אלגוריתם ארוך יותר (עם Gates רבות בסדרה), זמן הביצוע הכולל עשוי להתקרב ל-T2 או לחרוג ממנו, וגורם למצב לאבד קוהרנטיות לפני הסיום. זו הסיבה מדוע שיפור זמני הקוהרנטיות והאצת ה-Gates הם שניים מהיעדים הקריטיים ביותר במחקר חומרה קוואנטית.

```python
from qiskit_ibm_runtime.fake_provider import FakeBrisbane
fake_brisbane = FakeBrisbane()
print("Basis gates for ibmq_brisbane:", fake_brisbane.configuration().basis_gates)
```

הפלט עשוי להיות משהו כמו `['id', 'rz', 'sx', 'x', 'ecr']`. אלו הפעולות הפרימיטיביות שהחומרה תומכת בהן באופן נייטיבי (Identity/ללא פעולה, סיבוב RZ, Gate sqrt(X), Gate X, ו-controlled-X). כל Gate אחרת חייבת להיות מורכבת מאלה. הסט הזה ידוע כאוניברסלי לחישוב קוואנטי (בעיקרות, סיבובים חד-Qubit בתוספת Gate סבוכת דו-Qubit יוצרים סט אוניברסלי).

עכשיו, ניקח **Gate Toffoli (CCX)** כמקרה בחינה. CCX הופך Qubit מטרה רק אם שני Qubits שליטה הם 1. זו לא Gate נייטיבית בחומרת IBM. Qiskit מספקת הוראת `ccx`, אבל מאחורי הקלעים היא תפרק אותה.
### תרגיל 4: פירוק Gate Toffoli {#exercise-4-decompose-a-toffoli-gate}
השלם את הקוד שלמטה כדי לבנות Circuit עם Gate Toffoli (CCX) ואז השתמש ב-Qiskit כדי לפרק אותה ל-Gates הבסיסיים הנייטיביים של Backend `FakeBrisbane`.

```python
from qiskit import QuantumCircuit
from qiskit_ibm_runtime.fake_provider import FakeBrisbane

# The fake_brisbane backend from the previous cell is reused here.

qc_toffoli = QuantumCircuit(3)

qc_toffoli.ccx(0, 1, 2)

# 2. Transpile the circuit to the fake Brisbane backend
```

בפלט לאחר Transpile, אמורים לראות את ה-CCX מוחלפת ברצף של Gates בסיסיים יותר כמו `rz`, `sx`, ו-`ecr`. זה מוכיח שה-Gates הנייטיביות מספיקות לביטוי ה-Toffoli.

**אוניברסליות בפועל:** התרגיל שלמעלה מראה שGate מורכבת בת 3 Qubits נבנתה מפשוטות יותר. באופן כללי, **כל** יוניטרי רב-Qubit ניתן להרכבה מ-Gates של 1 ו-2 Qubits. ה-Transpiler הוא רכיב חיוני בכל מחסנית תוכנה קוואנטית, שכן הוא גשר בין האלגוריתמים האבסטרקטיים שאנחנו רוצים להריץ לבין הפעולות הפיזיות שמכשיר קוואנטי ספציפי יכול לבצע בפועל.

**דוגמת מכשיר:** מכשיר **ibmq_brisbane** משתמש בארכיטקטורת Eagle עם Gates הבסיסיות שהוצגו לעיל. כלומר, כל אלגוריתם שנשלח למכשירים האלה יומר לרצפי פעולות אלה. קריטריון זה עוסק בעצם ב**שליטה**; יש לנו מספיק כפתורי שליטה לביצוע כל פעולה נדרשת על ה-Qubits שלנו.
## 5. קריטריון 5 – מדידת Qubit {#criterion-5-qubit-measurement}

**קריטריון 5:** *"יכולת מדידה ספציפית ל-Qubit."* המצב של כל Qubit חייב להיות ניתן למדידה (בדרך כלל בבסיס החישובי, |0〉 או |1〉). במילים אחרות, לאחר הרצת Circuit קוואנטי, אנחנו צריכים לקרוא כל Qubit כביט קלאסי 0/1. קריטריון זה עוסק בקיום גלאים אמינים לכל Qubit וביכולת לבחור אילו Qubits למדוד.

**מטרת ההדגמה:** להראות כיצד לבצע מדידות ב-Qiskit על סימולטורים ומכשירים אמיתיים, ולהדגיש את ההבדלים (כמו רעש מדידה). נמדוד כמה Qubits במצבים שונים ונבחן את התוצאות. נדגים גם כיצד שגיאות קריאה עשויות להופיע על ידי השוואת סימולטור לתוצאות חומרה.

ראשית, דוגמת מדידה פשוטה:

```python
qc_measure = QuantumCircuit(2, 2)
qc_measure.x(0)              # qubit 0 -> |1>, qubit 1 stays |0>
qc_measure.measure([0, 1], [0, 1])
qc_measure.draw('mpl')
```

```python
sim_backend = AerSimulator()
pm = generate_preset_pass_manager(backend=sim_backend, optimization_level=1)
isa_qc_measure = pm.run(qc_measure)
job = sampler.run([isa_qc_measure], shots=1000)
result = job.result()
counts = result[0].data.c.get_counts()

print("Simulator measurement counts:", counts)
```

אנחנו מצפים ל-1000 ספירות של `01` על הסימולטור. עכשיו, בואו נראה **שגיאת מדידה** בפעולה על ידי דמייה שלה. ניתן להוסיף שגיאת קריאה לסימולטור Aer שלנו. Qiskit Aer מאפשרת לנו להגדיר `ReadoutError` ולחבר אותה ל-Qubits במודל רעש.
### תרגיל 5: דמיית שגיאת קריאה {#exercise-5-simulate-readout-error}
השלם את הקוד כדי להגדיר מודל שגיאת קריאה פשוט שבו לכל Qubit יש סיכוי של 2% להימדד בצורה שגויה (0 נקרא כ-1, או 1 נקרא כ-0). לאחר מכן, הרץ את Circuit המדידה עם מודל הרעש הזה.

```python
from qiskit_aer.noise import NoiseModel, ReadoutError

# --- YOUR CODE HERE ---

# 1. Define a 2% readout error for each single qubit.
# [[P(0|0), P(1|0)], [P(0|1), P(1|1)]]
ro_error = ReadoutError([[0.98, 0.02], [0.02, 0.98]])

# 2. Create a new noise model
noise_model_ro = NoiseModel()

# 3. Add the readout error to all qubits in the noise model
noise_model_ro.add_all_qubit_readout_error(ro_error)

# --- END YOUR CODE ---

sim_backend.set_options(noise_model=noise_model_ro)
pm = generate_preset_pass_manager(backend=sim_backend, optimization_level=1)
isa_qc_measure = pm.run(qc_measure)

# Run the measurement circuit with readout noise
sampler = Sampler(mode=sim_backend)

job = sampler.run([isa_qc_measure], shots=1024)
result = job.result()
counts = result[0].data.c.get_counts()

print("Simulation with 2% readout error:", counts)
```

הפלט המדומה הזה יראה ספירות שגויות כלשהן (כמו `11`, `00`, `10`) בדומה למה שחומרה אמיתית עשויה לייצר, ומדגים את ההשפעה של מדידה לא מושלמת.

**דוגמת מכשיר:** במכשיר אמיתי כמו **ibmq_brisbane**, ניתן להריץ את אותו Circuit ולראות ספירות שאינן אפס לתוצאות השגויות. נתוני הכיול של המכשיר מפרטים שגיאת קריאה לכל Qubit. היכולת לכוון ולקרוא Qubits ספציפיים היא קריטית, והבנת מאפייני השגיאות שלהם היא מפתח לקבלת תוצאות משמעותיות. הרצה על מכשיר אמיתי הודגמה ב**תרגיל 1b: הרצה על מחשב קוואנטי אמיתי של IBM**.
## קריטריוני תקשורת קוואנטית (Qubits עפים) {#quantum-communication-criteria-flying-qubits}

DiVincenzo גם פירט שני קריטריונים הספציפיים לתקשורת קוואנטית, חשובים לבניית מחשב קוואנטי מרושת:

6. **היכולת להמיר בין Qubits נייחים ו-Qubits עפים.** (לדוגמה, למפות Qubit במעבד לפוטון שיכול לנסוע.)
7. **היכולת להעביר Qubits עפים בין מיקומים בנאמנות.** (לדוגמה, לשלוח Qubit פוטון דרך סיב אופטי מבלי לאבד מידע קוואנטי.)

אלה מעבר לשימוש הרגיל ב-Qiskit מכיוון ש-Qiskit עוסק בעיקר ב-Qubits נייחים על שבב. עם זאת, ניתן להמחיש את *הרעיון* של קריטריונים אלה עם דוגמה פשוטה: **טלפורטציה קוואנטית**. טלפורטציה מראה המרת מצבו של Qubit נייח למידע הנישא על ידי זוג סבוך (החלק ה"עף") ותקשורת קלאסית, שמשמשת לאחר מכן לשחזור המצב על Qubit נייח אחר במקום אחר.
### פעילות מומלצת: לקחת את מודול *טלפורטציה קוואנטית* של Qiskit in Classrooms {#recommended-activity-quantum-teleportation}

מודול [טלפורטציה קוואנטית](https://quantum.cloud.ibm.com/learning/en/modules/computer-science/quantum-teleportation) של Qiskit in Classrooms מאת ד"ר Katie McCormick ידריך אותך דרך אחד הפרוטוקולים המרתקים ביותר במידע קוואנטי: טלפורטציה קוואנטית, שבה מצב קוואנטי (Qubit) נשלח מ-Alice אל Bob באמצעות שזירה ורק שני ביטים קלאסיים. תלמד את הליך הטלפורטציה המלא שלב אחר שלב – כיצד להכין את זוג Bell הסבוך, לבצע מדידת בסיס Bell מצד Alice, להעביר תוצאות קלאסיות, ולהחיל את ה-Gate הקוואנטי הנכון על Qubit של Bob כדי לשחזר בדיוק את המצב המקורי. בדרך, תחקור מדוע טלפורטציית המידע של Qubit לא מפרת את משפט אי-השיבוט או חורגת ממהירות האור. דרך תרגילים מעשיים באמצעות חומרה או סימולטורים של IBM Quantum, תצבור הבנה מעשית של מדידה, שזירה ובקרת feed-forward בפעולה.

על ידי שליטה בטלפורטציה קוואנטית, תבין כיצד לקודד, להעביר ולשחזר מידע קוואנטי בין צמתים שונים – מה שמניח את הבסיס לרשתות קוואנטיות, מערכות ממסר, תוכניות תקשורת מאובטחות וחישוב קוואנטי מודולרי מדרגי.
**כיצד זה קשור לקריטריונים 6 ו-7:** ברשת קוואנטית אמיתית, זוג ה-Qubits הסבוך ייווצר על ידי הפצת Qubits "עפים" (כמו פוטונים) בין מיקומי Alice ו-Bob (קריטריון 7: העברה נאמנה). פרוטוקול הטלפורטציה עצמו משמש אז כדרך למפות את מצב ה-Qubit הנייח של Alice על חצי הזוג הסבוך שלה, ובכך "שולח" אותו אל Bob (קריטריון 6: המרה הדדית). Qiskit מאפשרת לנו לדמות את הלוגיקה של הפרוטוקול בצורה מושלמת, ומספקת מודל קונספטואלי לאופן שבו קריטריונים אלה מתממשים בארכיטקטורות תקשורת.
## סיכום ומסקנות {#conclusion-and-summary}

תכננו סדרת תרגילים ממוקדי קוד להמחשת קריטריוני DiVincenzo בעזרת Qiskit. דרך הדוגמאות המעשיות האלה, חקרת כיצד פלטפורמת חישוב קוואנטי אמיתית עומדת בכל דרישה:

- **מדרגיות**: בניית Circuits על יותר Qubits והבנת מדרגיות הרעש.
- **אתחול**: שימוש באיפוסים והכנת מצב כדי להתחיל חישובים בצורה אמינה במצבים ידועים.
- **Gates אוניברסליות**: ביצוע Transpile לפעולות מורכבות ל-Gates הבסיסיות של המכשיר, מה שמוכיח שניתן לבצע כל חישוב.
- **מדידה**: קריאת Qubits והתמודדות עם שגיאות קריאה ריאליסטיות.
- **קוהרנטיות**: ראיית ההשפעה של T₁, T₂ סופיים על נאמנות האלגוריתם והצורך בפעולות מהירות ביחס לדקוהרנטיות.

לשלמות, גם נגענו בהיבטי תקשורת קוואנטית דרך מודול [טלפורטציה קוואנטית](https://quantum.cloud.ibm.com/learning/en/modules/computer-science/quantum-teleportation) של Qiskit in Classrooms, ויצרנו קישור לשני הקריטריונים האחרונים (Qubits עפים).

לבסוף, ראוי לציין כיצד קריטריונים אלה מתחברים יחד במחשב קוואנטי אמיתי כמו זה של IBM. מכשיר כמו **ibmq_brisbane** מכיל 127 Qubits מוליכי-על (קריטריון 1), כל אחד מתחיל ב-|0〉 (קריטריון 2), עם סט Gates מכוייל ומקמפלים לאוניברסליות (קריטריון 4), רזוננטורי קריאה מיקרוגל לכל Qubit (קריטריון 5), וזמני קוהרנטיות בסדר גודל של מאות מיקרו-שניות לעומת פעולות בסדר ננו-שניות (קריטריון 3). לצורך ניסויי רשת קוואנטית, IBM ואחרים חוקרים המרת מיקרוגל-לאופטי עבור Qubits עפים, ושזירת Qubits מרוחקים (קריטריונים 6 ו-7); אלה תחומי מחקר פעילים.

על ידי השלמת התרגילים במחברת זו, לא רק ראית את הגדרות קריטריוני DiVincenzo, אלא *נגעת בהם* דרך קוד; בנית אינטואיציה לגבי מה שכל דרישה אומרת לחומרה ולאלגוריתמים קוואנטיים אמיתיים. אל תהסס להרחיב את הניסויים האלה, ובהצלחה בחישוב הקוואנטי!

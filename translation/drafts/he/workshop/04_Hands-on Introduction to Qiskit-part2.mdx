## 2.1 מיפוי הבעיה לפורמט קוונטי-נייטיב {#map-the-problem-to-a-quantum-native-format}

בתוכנית קוונטית, *Circuit קוונטיים* הם הפורמט הנייטיב לייצוג הוראות קוונטיות, ו-*אופרטורים* מייצגים את האובייקטים הנמדדים. כשיוצרים Circuit, בדרך כלל יוצרים אובייקט [`QuantumCircuit`](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.QuantumCircuit#quantumcircuit-class) חדש ואז מוסיפים אליו הוראות ברצף.

תא הקוד הבא יוצר Circuit שמייצר את מצב GHZ — מצב שבו שלושה Qubit שזורים לחלוטין אחד עם השני.

<div class="alert alert-info">

  ה-Qiskit SDK משתמש במספור סיביות LSb 0, שבו הספרה ה-$n$ בעלת הערך $1 \ll n$ או $2^n$. לפרטים נוספים, ראו את הנושא [Bit-ordering in the Qiskit SDK](https://docs.quantum.ibm.com/guides/bit-ordering).

</div>

```python
# Create a GHZ state circuit

qc = QuantumCircuit(3)
qc.h(0)
qc.cx(0,1)
qc.cx(0,2)
# Draw the circuit
qc.draw("mpl")
```

![Quantum circuit diagram](./_04_Hands-on Introduction to Qiskit_imgs/output_15.png)

ראו את [`QuantumCircuit`](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.QuantumCircuit#quantumcircuit-class) בתיעוד לכל הפעולות הזמינות.

כשיוצרים Circuit קוונטיים, צריך גם לשקול איזה סוג של נתונים רוצים לקבל בחזרה אחרי הרצה. Qiskit מציע שתי דרכים להחזרת נתונים: אפשר לקבל התפלגות הסתברות עבור קבוצת Qubit שבוחרים למדוד, או לקבל את ערך הציפייה של אובייקט נמדד. הכינו את העומס שלכם למדוד את ה-Circuit באחת משתי הדרכים האלה באמצעות [primitives של Qiskit](https://docs.quantum.ibm.com/guides/get-started-with-primitives) (מוסבר בפירוט בשלב 3).

הדוגמה הזו מודדת ערכי ציפייה באמצעות מודול המשנה `qiskit.quantum_info`, שמצוין באמצעות אופרטורים (אובייקטים מתמטיים המשמשים לייצוג פעולה או תהליך שמשנים מצב קוונטי). תא הקוד הבא יוצר שישה אופרטורי פאולי של שלושה Qubit: `ZZZ`, `ZZX`, `ZII`, `XXI`, `ZZI` ו-`III`.

```python
# Set up six different observables.

observables_labels = ["ZZZ", "ZZX", "ZII", "XXI", "ZZI", "III"]

observables = [SparsePauliOp(label) for label in observables_labels]
print(observables)
```

```text
[SparsePauliOp(['ZZZ'],
              coeffs=[1.+0.j]), SparsePauliOp(['ZZX'],
              coeffs=[1.+0.j]), SparsePauliOp(['ZII'],
              coeffs=[1.+0.j]), SparsePauliOp(['XXI'],
              coeffs=[1.+0.j]), SparsePauliOp(['ZZI'],
              coeffs=[1.+0.j]), SparsePauliOp(['III'],
              coeffs=[1.+0.j])]
```

<div class="alert alert-info">

  כאן, משהו כמו האופרטור `ZZI` הוא קיצור למכפלה הטנזורית $Z\otimes Z\otimes I$, כלומר מדידת Z על Qubit 2 ו-Z על Qubit 1 יחד, ולקבל מידע על הקורלציה בין Qubit 2 ל-Qubit 1. ערכי ציפייה כאלה נכתבים בדרך כלל גם כ-$\langle Z_2 Z_1 \rangle$.

  אם המצב שאנחנו מתבוננים בו הוא מצב GHZ של שלושה Qubit, אז המדידה של $\langle Z_2 Z_1 \rangle$ אמורה להיות 1.

</div>

## 2.2 אופטימיזציה של ה-Circuit והאופרטורים {#optimize-the-circuits-and-operators}

כשמריצים Circuit על מכשיר, חשוב לאמטב את קבוצת ההוראות שה-Circuit מכיל ולמזער את העומק הכולל (בגדול, מספר ההוראות) של ה-Circuit. הדבר מבטיח שתקבלו את התוצאות הטובות ביותר האפשריות על ידי הפחתת השפעות השגיאה והרעש. בנוסף, הוראות ה-Circuit חייבות לעמוד ב-[Instruction Set Architecture (ISA)](https://docs.quantum.ibm.com/guides/transpile#instruction-set-architecture) של ה-Backend ולקחת בחשבון את Gate הבסיסיים וקישוריות ה-Qubit של המכשיר.

הקוד הבא יוצר מופע של מכשיר אמיתי לשליחת עבודה אליו, ומתמיר את ה-Circuit והאופרטורים כך שיתאימו ל-ISA של אותו Backend.
אם לא שמרתם את האישורים שלכם קודם לכן, עקבו אחר ההוראות [כאן](https://docs.quantum.ibm.com/guides/setup-channel#iqp) לאימות עם ה-API token שלכם.

```python
# Choose a real backend
service = QiskitRuntimeService(channel='ibm_quantum_platform',)

# print backend details

    f"Name: {backend.name}\n"
    f"Version: {backend.backend_version}\n"
    f"No. of qubits: {backend.num_qubits}\n"
    f"Processor type: {backend.processor_type}\n"
)
```

```text
No. of qubits: 156
```


## 2.3 הרצה באמצעות ה-primitives הקוונטיים {#execute-using-the-quantum-primitives}

מחשבים קוונטיים יכולים לייצר תוצאות אקראיות, אז בדרך כלל אוספים דגימה מהפלטים על ידי הרצת ה-Circuit פעמים רבות. אפשר לאמוד את ערכו של האובייקט הנמדד באמצעות המחלקה `Estimator`. ה-`Estimator` הוא אחד משני [primitives](https://docs.quantum.ibm.com/guides/get-started-with-primitives); השני הוא `Sampler`, שניתן להשתמש בו כדי לקבל נתונים ממחשב קוונטי. לאובייקטים האלה יש מתודת `run()` שמריצה את ה-Circuit, האופרטורים והפרמטרים הנבחרים (אם רלוונטי), תוך שימוש ב-[primitive unified bloc (PUB).](https://docs.quantum.ibm.com/guides/primitives#sampler)
כשמריצים קוד זה על חומרת קוונטום אמיתית, כדאי להשתמש ב[טכניקות לצמצום ומיתון שגיאות](https://quantum.cloud.ibm.com/docs/en/guides/error-mitigation-and-suppression-techniques) כדי להפחית את הרעש הפנימי של המחשב הקוונטי.

```python
# Construct the Estimator instance.
estimator = Estimator(mode=backend)
estimator.options.resilience_level = 1
estimator.options.default_shots = 5000
```

שלחו עבודה באמצעות ה-Estimator primitive.

```python
# One pub, with one circuit to run against six different observables.
job = estimator.run([(isa_circuit, mapped_observables)]) 

# Use the job ID to retrieve your job data later
print(f">>> Job ID: {job.job_id()}")
```

# 3. קידוד מידע {#3-encoding-information}

## 3.1 קידוד אמפליטודה {#31-amplitude-encoding}

עכשיו שראינו איך לבנות מעגלים קוונטיים, מעניין לחקור איך אפשר לקדד מידע קלאסי לתוך מצבים קוונטיים. שיטה עוצמתית אחת היא קידוד אמפליטודה, שבו האמפליטודות של מצב קוונטי מייצגות את הרכיבים של וקטור קלאסי.

בוא נבחן דוגמה פשוטה. נניח שאנחנו רוצים לקדד את הוקטור הקלאסי

$$
\vec{x} = \begin{bmatrix} x_0 \\ x_1 \\ x_2 \\ x_3 \end{bmatrix}
$$

לתוך מצב קוונטי של שני Qubit. המטרה היא להכין את המצב הקוונטי:

$$
\ket{\psi} = x_0\ket{00} + x_1\ket{01} + x_2\ket{10} + x_3\ket{11}
$$
כאשר $ x_0, x_1, x_2, x_3 \in \mathbb{R} $ (או $ \mathbb{C} $) והוקטור מנורמל כך ש:

$$
|x_0|^2 + |x_1|^2 + |x_2|^2 + |x_3|^2 = 1
$$

עכשיו נבחן את הדוגמה הספציפית: $ \vec{x} = [0.8924,  0.3696, 0.2391, 0.0990] $

אז המצב הקוונטי המתאים הוא:

$$
\begin{aligned}
\ket{\psi} &= 0.8924\,\ket{00}
+ 0.3696\,\ket{01}
+ 0.2391\,\ket{10}
+ 0.0990\,\ket{11}
\end{aligned}
$$
אפשר להכין מצב זה באמצעות שילוב של Gate-ים של סיבוב $R_y$ בזוויות $\pi/6$ ו-$\pi/4$ עבור Qubit 0 ו-1 בהתאמה

```python
from qiskit import QuantumCircuit
from qiskit_aer import AerSimulator
import numpy as np

qc = QuantumCircuit(2)

qc.ry(np.pi / 6, 0)
qc.ry(np.pi / 4, 1)

simulator = AerSimulator()
qc.save_statevector()
result = simulator.run(qc).result()
statevector = result.get_statevector()

print("Statevector:", statevector)
qc.draw(output="mpl")
```

```text
             0.09904576+0.j],
            dims=(2, 2))
```

## 3.2 קידוד זווית ומעגלים פרמטריים {#32-angle-encoding-and-parametrized-circuits}

דרך מעניינת במיוחד לקדד מידע למחשב קוונטי היא לתכנן Circuit קוונטי שמכיל כמה זוויות סיבוב $\vec{\theta}$ או פרמטרים שניתן לכוונן כדי לייצג משפחה של פונקציות $f(\vec{\theta})$. בוא נבחן לדוגמה את ה-Circuit הקוונטי הפרמטרי הבא:

```python
from qiskit import QuantumCircuit
from qiskit.circuit import Parameter

# Define a symbolic parameter
theta = Parameter("θ")

qc = QuantumCircuit(2)
# We applied a parametrized RX gate
qc.rx(theta, 0)
qc.cx(0, 1)
qc.draw("mpl")
```

![Quantum circuit diagram](./_04_Hands-on Introduction to Qiskit_imgs/output_25.png)

מבחינה מתמטית, אפשר לנתח מהי משפחת הפונקציות שאפשר לייצג עם ה-Circuit הזה:

$$ 
\text{CNOT}_{01} \, R_x^{\{0\}}(\theta) |00\rangle = \text{CNOT}_{01} \left( \cos(\theta/2)\ket{00} - i\sin(\theta/2)\ket{10} \right) = \cos(\theta/2)\ket{00} - i\sin(\theta/2)\ket{11}
$$
די ברור שמספר המצבים שאפשר לייצג עם ה-Circuit הקוונטי הזה מוגבל, כי לא ניתן לייצג מצבים כמו $\ket{10}$ או $\ket{01}$ למשל. עם זאת, משפחת המצבים שאפשר לייצג מתרחבת כשמוסיפים סיבובים נוספים במקומות המתאימים:

```python
from qiskit import QuantumCircuit
from qiskit.circuit import Parameter

# Define a symbolic parameter
theta1 = Parameter("θ1")
theta2 = Parameter("θ2")

qc = QuantumCircuit(2)
qc.rx(theta1, 0)
qc.rx(theta2, 1)
qc.cx(0, 1)
qc.draw("mpl")
```

![Quantum circuit diagram](./_04_Hands-on Introduction to Qiskit_imgs/output_26.png)

במקרה זה, המצבים הקוונטיים שנייצג הם:

$$
\begin{align*}
\text{CNOT}_{01} \, R_x^{\{1}}(\theta_2) R_x^{\{0}}(\theta_1) \ket{00}
&= \text{CNOT}_{01} \, R_x^{\{1}}(\theta_2)\left( \cos(\theta_1/2)\ket{00} - i\sin(\theta_1/2)\ket{10} \right) \\
&= \text{CNOT}_{01}\left( \cos(\theta_1/2)\cos(\theta_2/2)\ket{00} - i\cos(\theta_1/2)\sin(\theta_2/2)\ket{01} \right. \\
&\quad \left. - i\sin(\theta_1/2)\cos(\theta_2/2)\ket{10} + \sin(\theta_1/2)\sin(\theta_2/2)\ket{11} \right) \\
&= \cos(\theta_1/2)\cos(\theta_2/2)\ket{00} - i\cos(\theta_1/2)\sin(\theta_2/2)\ket{01} \\
&\quad + \sin(\theta_1/2)\sin(\theta_2/2)\ket{10} - i\sin(\theta_1/2)\cos(\theta_2/2)\ket{11} 
\end{align*}
$$
אפשר לראות שה-Circuit הזה מייצר משפחה רחבה יותר של מצבים קוונטיים בהשוואה לקודמו. בפרט, עכשיו הוא יכול לייצר מצבים עם אמפליטודות שאינן אפס עבור $\ket{01}$ או $\ket{10}$ שלא היו אפשריים עם ה-Circuit שלמעלה. עם זאת, ה-Circuit הזה עדיין אינו מייצר מצבים קוונטיים כלליים, אם כי הוא עשוי להיות בעל מספיק ביטוי לתכנן Circuit-ים עם גמישות מסוימת לייצוג פונקציות מסוימות. באופן כללי, ככל שמוסיפים פרמטרים עצמאיים (זוויות) יותר, כך ל-Circuit יש יותר ביטויות לקרב מצבים קוונטיים שרירותיים.
## Ansatzes וספריית ה-Circuit {#ansatzes-and-circuit-library}

סוג זה של Circuit קוונטי פרמטרי יכול לשמש לבניית [Ansatzes](https://quantum.cloud.ibm.com/learning/en/courses/quantum-chem-with-vqe/ansatz), מצבים קוונטיים ניסיוניים שמטרתם לקרב את הפתרון של בעיה. Ansatzes אלה הם רכיב מרכזי של [אלגוריתמים קוונטיים וריאציוניים](https://quantum.cloud.ibm.com/learning/en/courses/variational-algorithm-design/variational-algorithms), מחלקה של אלגוריתמים היברידיים קוונטי-קלאסיים שמשתמשים במחשב קוונטי להערכת פונקציית עלות ובמייעל קלאסי למזעורה. נכנס לפרטים על נושאים אלה ביחידה מאוחרת יותר, אך כרגע נציג כיצד לבנות Ansatz פשוט באמצעות [ספריית ה-Circuit ב-Qiskit](https://quantum.cloud.ibm.com/docs/en/api/qiskit/circuit_library). 

```python
from qiskit.circuit.library import efficient_su2

SU2_ansatz = efficient_su2(4, su2_gates=["rx", "y"], entanglement="linear", reps=1)
SU2_ansatz.decompose().draw(output="mpl")
```

# בונוס: טלפורטציה קוונטית {#bonus-quantum-teleportation}
כשאתה שומע את המונח טלפורטציה קוונטית, אולי אתה מדמיין טכנולוגיית מדע בדיוני עתידנית שמפרקת עצם במקום אחד ומחזירה אותו לקיום במקום רחוק. אבל טלפורטציה קוונטית אינה כזאת כלל. במציאות, מה שמתבצעת עליו הטלפורטציה הוא לא חומר — זה מידע.

טלפורטציה קוונטית מאפשרת העברת המצב הקוונטי של Qubit ממיקום אחד לאחר. בעוד שהעברה זו נראית מיידית, היא אינה מפרה את חוקי הפיזיקה. איך זה אפשרי? בוא נחפור!

טלפורטציה קוונטית היא פרוטוקול שמאפשר לשולח (אליס) להעביר את המצב $|\psi\rangle$ של Qubit `q` למקבל (בוב) באמצעות שני משאבים מרכזיים: זוג Qubit שזורים משותף `a` ו-`b` ושני ביטים של תקשורת קלאסית `c0` ו-`c1`. 

בעצם מה שהפרוטוקול צריך הוא:
*   `q`: ה-Qubit של אליס, ממוקם בתחילה במצב $|\psi\rangle$ שאנחנו רוצים לטלפורט.
*   `a`: החצי של אליס מזוג שזור משותף.
*   `b`: החצי של בוב מהזוג השזור המשותף.
*   `c0`, `c1`: ביטים קלאסיים לאחסון תוצאות המדידה של אליס.

ואיך זה עובד? זרימת העבודה היא כדלקמן

1.  **הכנת המצב $|\psi\rangle$ של אליס על `q`.** ניצור מצב ספציפי כמו $|+\rangle$ לצורך אימות.
2.  **יצירת שזירה:** ייצור זוג Bell בין `a` ל-`b`.
3.  **פעולות אליס:** אליס מבצעת "מדידת Bell" על שני ה-Qubit שלה (`q` ו-`a`) ומאחסנת את התוצאות הקלאסיות ב-`c0` ו-`c1`.
4.  **תקשורת קלאסית:** אליס שולחת את שני הביטים הקלאסיים שלה (`c0`, `c1`) לבוב.
5.  **תיקוני בוב:** בוב מפעיל Gate-ים קוונטיים ספציפיים (X ו/או Z) על ה-Qubit שלו (`b`), בהתניה על ערכי `c0` ו-`c1` שקיבל.

אם הכל נעשה נכון, ה-Qubit `b` של בוב יסיים במצב $|\psi\rangle$, המצב המקורי של `q` של אליס!

להסבר ולחקירה מעמיקים יותר של טלפורטציה קוונטית, כולל מעבר על ההסבר המתמטי של מדוע פרוטוקול זה עובד, ניתן לפנות למשאבי הלמידה של IBM Quantum: [טלפורטציה קוונטית](https://quantum.cloud.ibm.com/learning/courses/basics-of-quantum-information/entanglement-in-action/quantum-teleportation). זה חלק מקורס [יסודות המידע הקוונטי](https://quantum.cloud.ibm.com/learning/courses/basics-of-quantum-information).

```python

import matplotlib.pyplot as plt
from qiskit import QuantumCircuit, QuantumRegister, ClassicalRegister
from qiskit_aer import AerSimulator
from qiskit.visualization import plot_histogram, plot_bloch_multivector

# Define individual quantum registers for each qubit
q = QuantumRegister(1, name='q')  # message qubit
a = QuantumRegister(1, name='a')  # Alice's entangled qubit
b = QuantumRegister(1, name='b')  # Bob's entangled qubit

# Classical register for Alice's measurements
cr_alice = ClassicalRegister(2, name='c_alice')

# Create quantum circuit
teleport_qc = QuantumCircuit(q, a, b, cr_alice, name='Teleportation')

# Step 1: Prepare message state |+⟩ on q
teleport_qc.h(q[0])
teleport_qc.barrier()

# Step 2: Create entanglement between a and b
teleport_qc.h(a[0])
teleport_qc.cx(a[0], b[0])
teleport_qc.barrier()

# Step 3: Alice's Bell measurement
teleport_qc.cx(q[0], a[0])
teleport_qc.h(q[0])
teleport_qc.barrier()

# Step 4: Alice measures q and a
teleport_qc.measure(q[0], cr_alice[0])
teleport_qc.measure(a[0], cr_alice[1])
teleport_qc.barrier()

# Step 5: Bob's conditional measurements
with teleport_qc.if_test((cr_alice[1], 1)):
    teleport_qc.x(b[0])
with teleport_qc.if_test((cr_alice[0], 1)):
    teleport_qc.z(b[0])

# Draw the circuit
teleport_qc.draw(output='mpl')
```

![Quantum circuit diagram](./_04_Hands-on Introduction to Qiskit_imgs/output_28.png)

לאחר הרצת הפרוטוקול עולה שאלה מרכזית: איך מאמתים שהטלפורטציה עבדה? לא ניתן 'לראות' ישירות את המצב של ה-Qubit של בוב לאחר הפרוטוקול. עם זאת, מכיוון ש*הכנו* את המצב הראשוני $|\psi\rangle$ של אליס (בחרנו $|+\rangle$), אנחנו יכולים להשתמש בסוג מיוחד של סימולציה כדי לבדוק אם ה-Qubit `b` של בוב הסתיים באותו מצב.

נשתמש ב-`AerSimulator` עם `save_statevector` כדי לבדוק אם ה-Qubit `b` של בוב מסיים במצב הראשוני של אליס ($|+\rangle$). הסימולטור מחשב את וקטור המצב הקוונטי הסופי.
ואז מייצג אותו באמצעות `plot_bloch_multivector` כדי להמחיש את ה-Qubit של בוב (`b`) בהשוואה למצב הראשוני של אליס (`q`).

```python
# Simulate the teleportation circuit
sv_simulator = AerSimulator(method='statevector')
teleport_qc_sv = teleport_qc.copy()
teleport_qc_sv.save_statevector()

# Execute the circuit on the statevector simulator
job_sv = sv_simulator.run(teleport_qc_sv)
result_sv = job_sv.result()

# Get the final statevector
final_statevector = result_sv.get_statevector()
print("Visualizing final qubit states:")
display(plot_bloch_multivector(final_statevector))
print("Note that Alice's qubits have collapsed to |00⟩, |01⟩, |10⟩, or |11⟩, while Bob's qubit is in the original state |+⟩.")
```

כפי שאנחנו רואים מהמחשה, שני ה-Qubit הראשונים (השייכים לאליס) התמוטטו ל-0 או 1. בינתיים, ה-Qubit השלישי (השייך לבוב), המיוצג בכדור Bloch השלישי, מצביע לאורך ציר x, מה שמעיד שהוא נמצא במצב $|+\rangle$, כך שיישמנו בהצלחה את פרוטוקול הטלפורטציה הקוונטית!

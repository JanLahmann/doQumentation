ה-primitives הייחוסיים ב-Qiskit SDK מבצעים סימולציות statevector מקומיות. סימולציות אלה אינן תומכות
בדגמול רעש של מכשיר, אך שימושיות לאב-טיפוס מהיר של אלגוריתמים לפני שפונים לטכניקות סימולציה מתקדמות יותר ([שימוש ב-Qiskit Aer](./simulate-stabilizer-circuits)) או להרצה על מכשירים אמיתיים ([Qiskit Runtime primitives](primitives)).

ה-Estimator primitive יכול לחשב ערכי ציפייה של מעגלים, וה-Sampler primitive יכול לדגום
מתפלוגות פלט של מעגלים.

הסעיפים הבאים מראים כיצד להשתמש ב-primitives הייחוסיים כדי להריץ את ה-workflow שלך באופן מקומי.

## שימוש ב-Estimator הייחוסי {#use-the-reference-estimator}

המימוש הייחוסי של `EstimatorV2` ב-`qiskit.primitives` שרץ על
סימולטורי statevector מקומיים הוא המחלקה [`StatevectorEstimator`](../api/qiskit/qiskit.primitives.StatevectorEstimator). היא יכולה לקבל מעגלים, observables ופרמטרים כקלט ומחזירה את ערכי הציפייה המחושבים באופן מקומי.

הקוד הבא מכין את הקלטים שישמשו בדוגמאות שיבואו. סוג הקלט הצפוי עבור
ה-observables הוא [`qiskit.quantum_info.SparsePauliOp`](../api/qiskit/qiskit.quantum_info.SparsePauliOp). שים לב ש-
המעגל בדוגמה מפורמט, אבל אפשר גם להריץ את ה-Estimator על מעגלים לא-מפורמטים.

<Admonition type="note">
  כל Circuit שמועבר ל-Estimator חייב **לא** לכלול **מדידות**.
</Admonition>

from qiskit import QuantumCircuit
from qiskit.circuit import Parameter

# circuit for which you want to obtain the expected value
circuit = QuantumCircuit(2)
circuit.ry(Parameter("theta"), 0)
circuit.h(0)
circuit.cx(0, 1)
circuit.draw("mpl", style="iqp")

from qiskit.quantum_info import SparsePauliOp
import numpy as np

# observable(s) whose expected values you want to compute

observable = SparsePauliOp(["II", "XX", "YY", "ZZ"], coeffs=[1, 1, -1, 1])

parameter_values = [[0], [np.pi / 6], [np.pi / 2]]

<Admonition type="tip" title="טרנספיל למעגלי ISA ו-observables">
  ה-workflow של Qiskit Runtime primitives דורש שמעגלים ו-observables יומרו כך שישתמשו רק בהוראות הנתמכות על ידי ה-QPU (המכונות מעגלי *instruction set architecture (ISA)* ו-observables). ה-primitives הייחוסיים עדיין מקבלים הוראות מופשטות, מאחר שהם מסתמכים על סימולציות statevector מקומיות, אך הטרנספיל של המעגל עשוי עדיין להועיל מבחינת אופטימיזציית המעגל.

  ```python
  # Generate a pass manager without providing a backend
  from qiskit.transpiler import generate_preset_pass_manager

  pm = generate_preset_pass_manager(optimization_level=1)
  isa_circuit = pm.run(circuit)
  isa_observable = observable.apply_layout(isa_circuit.layout)
  ```
</Admonition>

### אתחול Estimator {#initialize-estimator}

יצירת מופע של [`qiskit.primitives.StatevectorEstimator`](../api/qiskit/qiskit.primitives.StatevectorEstimator).

```python
from qiskit.primitives import StatevectorEstimator

estimator = StatevectorEstimator()
```

### הרצה וקבלת תוצאות {#run-and-get-results}

דוגמה זו משתמשת רק במעגל אחד (מסוג [`QuantumCircuit`](../api/qiskit/qiskit.circuit.QuantumCircuit)) ו-observable אחד.

מריצים את האומדן על ידי קריאה לשיטה [`StatevectorEstimator.run`](../api/qiskit/qiskit.primitives.StatevectorEstimator#run), שמחזירה מופע של אובייקט [`PrimitiveJob`](https://docs.quantum.ibm.com/api/qiskit/qiskit.primitives.PrimitiveJob). אפשר לקבל את התוצאות מהעבודה (כאובייקט [`qiskit.primitives.PrimitiveResult`](../api/qiskit/qiskit.primitives.PrimitiveResult))
עם השיטה [`qiskit.primitives.PrimitiveJob.result`](../api/qiskit/qiskit.primitives.PrimitiveJob#result).

```python
job = estimator.run([(circuit, observable, parameter_values)])
result = job.result()
print(f" > Result class: {type(result)}")
```

```text
> Result class: <class 'qiskit.primitives.containers.primitive_result.PrimitiveResult'>
```

#### קבלת ערך הציפייה מהתוצאה {#get-the-expected-value-from-the-result}

פלט תוצאות ה-primitives הוא מערך של אובייקטי [`PubResult`](https://docs.quantum.ibm.com/api/qiskit/qiskit.primitives.PubResult#pubresult), כאשר כל פריט במערך הוא אובייקט `PubResult` שמכיל בנתוניו את מערך ההערכות המתאים לכל שילוב מעגל-observable ב-PUB.

כדי לשחזר את ערכי הציפייה והמטא-דאטה עבור הערכת המעגל הראשונה (והיחידה במקרה זה), יש לגשת ל-[`data`](https://docs.quantum.ibm.com/api/qiskit/qiskit.primitives.PubResult#data) עבור PUB 0:

```python
print(f" > Expectation value: {result[0].data.evs}")
print(f" > Metadata: {result[0].metadata}")
```

```text
> Expectation value: [4.         3.73205081 2.        ]
 > Metadata: {'target_precision': 0.0, 'circuit_metadata': {}}
```

### הגדרת אפשרויות הרצה של Estimator {#set-estimator-run-options}

כברירת מחדל, ה-Estimator הייחוסי מבצע חישוב statevector מדויק המבוסס על
המחלקה [`quantum_info.Statevector`](../api/qiskit/qiskit.quantum_info.Statevector).
עם זאת, ניתן לשנות זאת כדי להכניס את ההשפעה של עומס הדגימה (המכונה גם "רעש ירייה").

ה-Estimator מקבל ארגומנט `precision` שמבטא את מרווחי השגיאה שמימוש ה-
primitive צריך לכוון אליהם עבור הערכות ערכי ציפייה. זהו עומס הדגימה ומוגדר אך ורק בשיטת `.run()`. זה מאפשר לך לכוון את האפשרות עד לרמת ה-PUB.

```python
# Estimate expectation values for two PUBs, both with 0.05 precision.
precise_job = estimator.run(
    [(circuit, observable, parameter_values)], precision=0.05
)
```

לדוגמה מלאה, ראו את הדף [דוגמאות Primitives](primitives-examples#estimator-examples).

## שימוש ב-Sampler הייחוסי {#use-the-reference-sampler}

המימוש הייחוסי של `SamplerV2` ב-`qiskit.primitives` הוא המחלקה [`StatevectorSampler`](../api/qiskit/qiskit.primitives.StatevectorSampler). היא מקבלת מעגלים ופרמטרים כקלט ומחזירה את התוצאות מדגימה של תפלוגות ההסתברות לפלט כתפלוגת קוואזי-הסתברות של מצבי פלט.

הקוד הבא מכין את הקלטים המשמשים בדוגמאות שיבואו. שים לב ש-
דוגמאות אלה מריצות מעגל מפורמט יחיד, אבל אפשר גם להריץ את ה-Sampler
על מעגלים לא-מפורמטים.

```python
from qiskit import QuantumCircuit

circuit = QuantumCircuit(2)
circuit.h(0)
circuit.cx(0, 1)
circuit.measure_all()
circuit.draw("mpl", style="iqp")
```

### אתחול `SamplerV2` {#initialize-samplerv2}

יצירת מופע של [`qiskit.primitives.StatevectorSampler`](../api/qiskit/qiskit.primitives.StatevectorSampler):

```python
from qiskit.primitives import StatevectorSampler

sampler = StatevectorSampler()
```

### הרצה וקבלת תוצאות {#run-and-get-results-1}

```python
# execute 1 circuit with Sampler
job = sampler.run([circuit])
pub_result = job.result()[0]
print(f" > Result class: {type(pub_result)}")
```

```text
> Result class: <class 'qiskit.primitives.containers.sampler_pub_result.SamplerPubResult'>
```

ה-Primitives מקבלים מספר PUBs כקלט, וכל PUB מקבל תוצאה משלו. לכן, אפשר להריץ מעגלים שונים עם שילובי פרמטרים/observables שונים, ולשחזר את תוצאות ה-PUB:

```python
from qiskit.transpiler import generate_preset_pass_manager

# create two circuits
circuit1 = circuit.copy()
circuit2 = circuit.copy()

# transpile circuits
pm = generate_preset_pass_manager(optimization_level=1)
 isa_circuit1 = pm.run(circuit1)
 isa_circuit2 = pm.run(circuit2)
# execute 2 circuits using Sampler
job = sampler.run([(isa_circuit1), (isa_circuit2)])
pub_result_1 = job.result()[0]
pub_result_2 = job.result()[1]
print(f" > Result class: {type(pub_result)}")
```

```text
> Result class: <class 'qiskit.primitives.containers.sampler_pub_result.SamplerPubResult'>
```

### קבלת תפלוגת ההסתברות או תוצאת המדידה {#get-the-probability-distribution-or-measurement-outcome}

דגימות תוצאות המדידה מוחזרות כ-**bitstrings** או **counts**. ה-bitstrings מציגות את תוצאות המדידה, ושומרות על סדר הירייות שבו נמדדו. אובייקטי תוצאות ה-Sampler מארגנים נתונים לפי שמות הרגיסטרים הקלאסיים של מעגלי הקלט שלהם, לתאימות עם מעגלים דינמיים.

<Admonition type="note">
שם הרגיסטר הקלאסי מוגדר כברירת מחדל ל-`"meas"`. שם זה ישמש בהמשך לגישה ל-bitstrings של המדידה.
 </Admonition>

```python
# Define quantum circuit with 2 qubits
circuit = QuantumCircuit(2)
circuit.h(0)
circuit.cx(0, 1)
circuit.measure_all()
circuit.draw()
```

```text
┌───┐      ░ ┌─┐
   q_0: ┤ H ├──■───░─┤M├───
        └───┘┌─┴─┐ ░ └╥┘┌─┐
   q_1: ─────┤ X ├─░──╫─┤M├
             └───┘ ░  ║ └╥┘
meas: 2/══════════════╩══╩═
                      0  1
```

```python
# Transpile circuit
pm = generate_preset_pass_manager(optimization_level=1)
 isa_circuit = pm.run(circuit)
# Run using sampler
result = sampler.run([circuit]).result()
# Access result data for PUB 0
data_pub = result[0].data
# Access bitstring for the classical register "meas"
bitstrings = data_pub.meas.get_bitstrings()
print(f"The number of bitstrings is: {len(bitstrings)}")
# Get counts for the classical register "meas"
counts = data_pub.meas.get_counts()
print(f"The counts are: {counts}")
```

```text
The number of bitstrings is: 1024
The counts are: {'11': 515, '00': 509}
```

### שינוי אפשרויות הרצה {#change-run-options}

כברירת מחדל, ה-Sampler הייחוסי מבצע חישוב statevector מדויק המבוסס על
המחלקה [`quantum_info.Statevector`](../api/qiskit/qiskit.quantum_info.Statevector).
עם זאת, ניתן לשנות זאת כדי להכניס את ההשפעה של עומס הדגימה (המכונה גם "רעש ירייה"). כדי לעזור בניהול עומס זה, ממשק ה-Sampler מקבל ארגומנט `shots` שניתן להגדיר ברמת ה-PUB.

דוגמה זו מניחה שהגדרת שני מעגלים.

```python
# Sample two circuits at 128 shots each.
sampler.run([isa_circuit1, isa_circuit2], shots=128)
# Sample two circuits at different amounts of shots. The "None"s are necessary
# as placeholders
# for the lack of parameter values in this example.
sampler.run([(isa_circuit1, None, 123), (isa_circuit2, None, 456)])
```

```text
<qiskit.primitives.primitive_job.PrimitiveJob at 0x7fa430e39dd0>
```

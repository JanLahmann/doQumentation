## משפחות מעבדים שפרשו {#retired-processor-families}

- **Canary**: משפחת Canary כללה עיצובים קטנים המכילים בין 5 ל-16 qubits.
- **Falcon**: משפחת מכשירי Falcon הציעה פלטפורמה ערכית עבור Circuit-ים בקנה מידה בינוני, ושימשה גם כפלטפורמה יקרת ערך להדגמת שיפורי ביצועים ומדרגיות לפני שהועברו למכשירים הגדולים יותר.
- **Egret**: Egret הביא את החידושים של couplers כוונניים לפלטפורמת 33 qubits, מה שהוביל ל-Gate-ים בין שני qubits מהירים יותר ובעלי נאמנות גבוהה יותר.
- **Hummingbird**: באמצעות פריסת qubit משושה-כבד, משפחת Hummingbird אפשרה עד 65 qubits.
- **Osprey**: Osprey היה כמעט פי ארבעה מגודל Eagle עם 433 qubits. גדלי השבב הגדולים יותר דרשו שיפורים נוספים באריזת המכשיר, כמו גם כבלים גמישים מותאמים בקריוסטט כדי להכיל את דרישות ה-I/O הגדולות יותר בתוך אותו טביעת רגל של חיווט.
- **Eagle**: משפחת מעבד Eagle שילבה טכנולוגיות אריזה מדרגיות יותר מהדורות הקודמים. בפרט, אותות עברו דרך שכבות שבב מרובות, מה שאפשר I/O בצפיפות גבוהה מבלי לוותר על ביצועים.

## אחזור עבודה מ-QPU שפרש {#retrieve-a-job-from-a-retired-qpu}

השתמש בקוד הבא כדי לאחזר עבודה מ-QPU שפרש.

<Admonition type="note">
העברת Cloud Resource Name (CRN) בקוד הבא היא אופציונלית. כאשר הוא מועבר, רק Backend-ים ועבודות מאותו instance זמינים. אם לא מצוין instance, כל ה-Backend-ים והעבודות בכל ה-instances בחשבונך זמינים. במקרה זה, כאשר נקבע Backend, נמצא ונעשה שימוש ב-instance שיכול לגשת לאותו Backend.
</Admonition>

```python
from qiskit_ibm_runtime import QiskitRuntimeService

# Load your IBM Quantum account. Replace "<IBM Cloud CRN>" with your desired instance's CRN.
service = QiskitRuntimeService(channel="ibm_quantum_platform", instance="<IBM Cloud CRN>")

# Retrieve a single job by id
job = service.job(<job_id>)

# Retrieve a batch of jobs. Filtering options can be found in the QiskitRuntimeService.jobs api reference
jobs = service.jobs(backend_name=<backend_name>)
```

<Admonition type="note" title="Deprecated provider packages">
  `service.jobs()` returns jobs run from both `qiskit-ibm-runtime` and the deprecated `qiskit-ibm-provider` package. Jobs submitted by the older (also deprecated) `qiskit-ibmq-provider` package are no longer available.
</Admonition>

# מדריך המעבר ל-Qiskit v2.0 {#qiskit-v20-migration-guide}

Qiskit v2.0 הוא הצעד הבא במחזור החיים המיוצב של Qiskit SDK.
גרסה ראשית זו אכן שוברת תאימות עם גרסאות קודמות של Qiskit, אך בניגוד לגרסה v1.0, אינה כוללת שינויים במבנה החבילות.

מדריך זה מתאר נתיבי מעבר עבור שינויי הפיצ'רים החשובים ביותר ב-Qiskit v2.0, מאורגנים לפי מודול.
השתמש בתוכן העניינים בצד ימין כדי לנווט למודול שמעניין אותך.

קרא עוד על היתרונות של Qiskit v2.0 ב[בלוג IBM&reg;](https://www.ibm.com/quantum/blog/qiskit-2-0-release-summary).

<span id="qiskit.circuit"></span>
## qiskit.circuit {#qiskitcircuit}

<span id="c_if"></span>
### הסרת תנאי `c_if` הישנים {#removal-of-legacy-c_if-conditions}

תנאים קלאסיים ב-Qiskit יוצגו בעדיפות על ידי ההוראה [`IfElseOp`](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.IfElseOp) עוד לפני Qiskit v1.0.
פעולות אלה מתווספות בדרך כלל ל-[`QuantumCircuit`](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.QuantumCircuit) באמצעות שימוש במתודה [`if_test`](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.QuantumCircuit#qiskit.circuit.QuantumCircuit.if_test) כמנהל הקשר.

משמעות הדבר היא גם של-[`Instruction`](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.Instruction) אין עוד מאפיין `condition`, ולא ניתן לשרשר קריאות `c_if` על פלט של קריאת מתודה ליצירת Gate ב-[`QuantumCircuit`](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.QuantumCircuit).

ההוראות היחידות שעשויות כיום לכלול תנאי הן [`IfElseOp`](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.IfElseOp) ו-[`WhileLoopOp`](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.WhileLoopOp).
הדרך הנכונה ביותר לבדוק קיום תנאי היא שימוש בבדיקת `isinstance` כדי לוודא שיש לך את המחלקה הנכונה.
אם ברצונך לתמוך במאפיין הישן `Instruction.condition` בזמן מעבר הספרייה שלך ל-Qiskit v2.0 ואילך, תוכל להשתמש ב-`getattr(operation, "condition", None)`, שבדרך כלל יגרום ל-instances מסוג `Instruction` להתנהג כפי שהתנהגו ב-Qiskit v1.x ומוקדם יותר.

השיטה המקורית הישנה לתמיכה בתנאי שוויון-רגיסטר פשוטים ב-Qiskit הייתה שימוש במתודה `c_if` על הוראות בודדות או על קבוצות הוראות.
שיטה זו הייתה מוגבלת יותר מצורת control-flow המלאה, ומעולם לא נתמכה היטב על חומרה.
שיטה ישנה זו הוסרה ב-Qiskit v2.0; עליך להשתמש במתודה [`if_test`](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.QuantumCircuit#qiskit.circuit.QuantumCircuit.if_test) עם תנאי מתאים.
תוכל להמשיך להשתמש בצורת ה-2-tuple של `(register, integer)` לייצוג שוויון רגיסטר, או להשתמש ב[מערכת החישוב הקלאסי](https://docs.quantum.ibm.com/api/qiskit/circuit_classical) הביטויית יותר לייצוג תנאים מורכבים.

לדוגמה, ייתכן שכתבת:

```python
from qiskit.circuit import QuantumCircuit, QuantumRegister, ClassicalRegister

qr = QuantumRegister(2, "q")
cr = ClassicalRegister(2, "c")
qc = QuantumCircuit(qr, cr)
qc.h(0)
qc.h(1)
qc.measure(qr, cr)

# Apply an X to qubit 0 if both measurements were 1.
qc.x(0).c_if(cr, 3)
```

ההמרה הישירה ביותר היא שימוש בצורת ה-2-tuple עם מנהל ההקשר [`if_test`](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.QuantumCircuit#qiskit.circuit.QuantumCircuit.if_test):

```python
from qiskit.circuit import QuantumCircuit, QuantumRegister, ClassicalRegister

qr = QuantumRegister(2, "q")
cr = ClassicalRegister(2, "c")
qc = QuantumCircuit(qr, cr)
qc.h(0)
qc.h(1)
qc.measure(qr, cr)

# Apply an X to qubit 0 if both measurements were 1.
with qc.if_test((cr, 3)):
    qc.x(0)
```

שים לב שבצורה זו תוכל גם לאפשר ל-Gate המותנה לגשת למספר פעולות (פשוט הוסף עוד לבלוק המוזחה) מבלי להעריך מחדש את התנאים.
תוכל גם לספק משפט ``else``, ולהשתמש בלוגיקה מותנית מורכבת יותר:

```python
from qiskit.circuit import QuantumCircuit
from qiskit.circuit.classical import expr

qc = QuantumCircuit(2, 2)
qc.h(0)
qc.h(1)
qc.measure(qc.qubits, qc.clbits)

# Apply an X to qubit 0 if both measurements were the same:
with qc.if_test(expr.equal(qc.clbits[0], qc.clbits[1])) as else_:
    qc.x(0)
# ... or apply a Y to both qubits otherwise:
with else_:
    qc.y(0)
    qc.y(1)
```

<span id="instruction_durations"></span>
### הסרת `Instruction.duration` ו-`Instruction.unit` {#removal-of-instructionduration-and-instructionunit}

לאובייקטים מסוג Circuit [`Instruction`](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.Instruction) היו בעבר מאפיינים הנקראים `duration` ו-`unit`.
אלה אינם קיימים עוד ב-Qiskit v2.0.
הסיבה לכך היא שהשדה בדרך כלל רק שכפל מידע שהיה מאפיין קבוע של החומרה, ונשמר במפורש ב-[`Target`](https://docs.quantum.ibm.com/api/qiskit/qiskit.transpiler.Target).
שמירת המידע בנפרד עבור כל הוראה בנפרד הגדילה משמעותית את דרישות הזיכרון של Qiskit והאטה אפילו פעולות Circuit שלא נדרשו לגעת במידע ישירות.

הוראת ה-[`Delay`](https://docs.quantum.ibm.com/api/qiskit/circuit#delay) עדיין מכילה שדות אלה, וכן [`BoxOp`](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.QuantumCircuit#box) החדש ב-v2.0, מכיוון שהוראות אלה כוללות עיכוב _משתנה_.

ניתן לאחזר את משך הפעולה שהחומרה מגדירה עבור [`Instruction`](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.Instruction) נתמך מסוים על ידי שאילתת ה-[`Target`](https://docs.quantum.ibm.com/api/qiskit/qiskit.transpiler.Target).
לשם כך, אנדקס את ה-[`Target`](https://docs.quantum.ibm.com/api/qiskit/qiskit.transpiler.Target) תחילה עם שם הפעולה, ולאחר מכן עם אינדקסי ה-Qubits של החומרה כ-tuple.
פעולה זו תחזיר `None` (אם ההוראה נתמכת, אך לא הוקצו לה מאפיינים ספציפיים), או instance מסוג [`InstructionProperties`](https://docs.quantum.ibm.com/api/qiskit/qiskit.transpiler.InstructionProperties), שממנו השדה [`duration`](https://docs.quantum.ibm.com/api/qiskit/qiskit.transpiler.InstructionProperties#qiskit.transpiler.InstructionProperties.duration) עשוי לספק את המשך בשניות (או `None`, אם לא סופק).

```python
target = "<some Target from a hardware vendor>"

# Query the duration of a CX instruction on hardware qubits (2, 4),
# returning `None` if the duration is not specified.
properties = target["cx"][2, 4]
duration = None if properties is None else properties.duration

# You can also do this in one step, using `getattr`:
duration = getattr(target["cx"][2, 4], "duration", None)
```

באופן כללי, ככל ש-control-flow קלאסי יהפוך נפוץ יותר בתוך ביצועי Circuit, "משך הפעולה" של Circuit יהפוך פחות ופחות מוגדר.
בעוד ש-Qiskit v2.x עדיין כולל את המאפיין [`QuantumCircuit.duration`](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.QuantumCircuit#qiskit.circuit.QuantumCircuit.duration), מומלץ לעבור ממנו, מכיוון שסביר שיוסר בעתיד; ההנחות שבהן הוא תקף יופרו בתדירות גוברת.
במקום זאת, תוכל להשתמש במתודה החדשה [`QuantumCircuit.estimate_duration`](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.QuantumCircuit#qiskit.circuit.QuantumCircuit.estimate_duration), שהיא בטוחה יותר ומפורשת יותר לגבי ההנחות הנדרשות.

## qiskit.transpiler {#qiskittranspiler}

<span id="generate_preset_pass_manager"></span>
### שינויים בממשק `generate_preset_pass_manager` {#changes-to-generate_preset_pass_manager-interface}

Qiskit v2.0.0 כולל סדרת עדכונים לממשק מנהל הפאסים המוגדר מראש (כלומר, הפונקציות
[`generate_preset_pass_manager`](https://docs.quantum.ibm.com/api/qiskit/qiskit.transpiler.generate_preset_pass_manager) ו-
[`transpile`](https://docs.quantum.ibm.com/api/qiskit/compiler#transpile)) המשפיעים על
טיפול בקלטי אילוצים רופפים, כגון `coupling_map` או `basis_gates`. מספר
מקרי השימוש בטרנספילציה שבהם מתקבלים אילוצים רופפים צומצם לטובת
קלטי `target` ו-`backend` (כאשר `backend` הוא [`BackendV2`](https://docs.quantum.ibm.com/api/qiskit/qiskit.providers.BackendV2) המכיל instance מסוג
[`Target`](https://docs.quantum.ibm.com/api/qiskit/qiskit.transpiler.Target)),
המאפשרים תקשורת ביטויית יותר של אילוצים אלה.

הדוגמאות הבאות מציגות את השינויים העיקריים ואלטרנטיבות מוצעות למעבר. שים לב שכל
טיפים למעבר שלהלן חלים גם על `transpile` וגם על `generate_preset_pass_manager`:

1. ארגומנט הקלט `backend_properties` אינו נתמך עוד, והמחלקה `BackendProperties` הוסרה.
   עליך להשתמש בארגומנטים `target` או `backend` במקום לתקשר עם משך Gate ושגיאות.
   קיימים מספר נתיבים לבניית [`Target`](https://docs.quantum.ibm.com/api/qiskit/qiskit.transpiler.Target) מותאם:

  a. בנה Target מאפס והוסף שגיאות ומשכים רלוונטיים ל-Gates המתאימים:

    ```python
    from qiskit.transpiler import Target, InstructionProperties, generate_preset_pass_manager
    from qiskit.circuit.library import XGate, CXGate

    target = Target(num_qubits=2)
    target.add_instruction(CXGate(), {(0, 1): InstructionProperties(error=.0001, duration=5e-7)})
    target.add_instruction(
        XGate(),
        {
            (0,): InstructionProperties(error=.00001, duration=5e-8),
            (1,): InstructionProperties(error=.00002, duration=6e-8)
        }
    )

    pm = generate_preset_pass_manager(target=target)
    ```

  b. השתמש ב-[`GenericBackendV2`](https://docs.quantum.ibm.com/api/qiskit/qiskit.providers.fake_provider.GenericBackendV2) לבניה מהירה של
     Target גנרי עם שגיאות ומשכים מולאים מראש. ניתן גם לעדכן אותם לאחר הבנייה:

    ```python
    from qiskit.providers.fake_provider import GenericBackendV2
    from qiskit.transpiler import generate_preset_pass_manager

    target = GenericBackendV2(num_qubits=2, basis_gates=["x", "cx"]).target
    target.update_instruction_properties("x", (0,), InstructionProperties(0.00001))
    target.update_instruction_properties("x", (1,), InstructionProperties(0.00002))
    target.update_instruction_properties("cx", (0, 1), InstructionProperties(0.0001))

    pm = generate_preset_pass_manager(target=target)
    ```

2. ארגומנטי הקלט `instruction_durations` ו-`timing_constraints` אינם נתמכים עוד. עליך להשתמש
  בארגומנטים `target` או `backend` במקום. האלטרנטיבות הן בניית [`Target`](https://docs.quantum.ibm.com/api/qiskit/qiskit.transpiler.Target) מותאם
  מאפס, בניית Backend גנרי, ובנוסף לאלה, ניתן לבנות Target מותאם עם
  `instruction_durations` ו/או `timing_constraints` באמצעות המתודה `Target.from_configuration`:

    ```python
    from qiskit.transpiler import Target, generate_preset_pass_manager
    from qiskit.transpiler.timing_constraints import TimingConstraints
    from qiskit.transpiler.instruction_durations import InstructionDurations

    instruction_durations = InstructionDurations([("x", None, 200), ("cx", None, 1000)], dt=1e-7)

    timing_constraints = TimingConstraints(
            granularity=1, min_length=1, pulse_alignment=1, acquire_alignment=1
        )

    target = Target.from_configuration(num_qubits=2,
                                        basis_gates=["x", "cx"],
                                        instruction_durations=instruction_durations,
                                        timing_constraints=timing_constraints)

    pm = generate_preset_pass_manager(target=target)
    ```

3. לא מומלץ לספק `coupling_map` ו/או `basis_gates` יחד עם `backend`, ומ-v2.0 יוצג `UserWarning`.
  במקרים אלה קיימות מספר מקורות אמת,
  ולא ניתן לפתור מידע כגון שגיאות ומשכי Gate מה-Backend על בסיס המידע שניתן.
  האלטרנטיבה המוצעת היא הגדרת Target מותאם המשלב את האילוצים הנבחרים,
  בין אם מאפס, באמצעות [`Target.from_configuration`](https://docs.quantum.ibm.com/api/qiskit/qiskit.transpiler.Target#from_configuration) או [`GenericBackendV2`](https://docs.quantum.ibm.com/api/qiskit/qiskit.providers.fake_provider.GenericBackendV2).

4. ארגומנט `basis_gates` אינו מקבל עוד Gates בסיס מותאמים. האלטרנטיבה המוצעת
  היא ארגומנט `target`, שניתן לבנות מאפס או דרך [`Target.from_configuration`](https://docs.quantum.ibm.com/api/qiskit/qiskit.transpiler.Target#from_configuration)
  כגון:

    ```python
    from qiskit.circuit.library import XGate
    from qiskit.transpiler.target import Target

    basis_gates = ["my_x", "cx"]
    custom_name_mapping = {"my_x": XGate()}
    target = Target.from_configuration(
        basis_gates=basis_gates, num_qubits=2, custom_name_mapping=custom_name_mapping
    )
    pm = generate_preset_pass_manager(target=target)
    ```

5. שילוב הקלטים שבו מסופק `coupling_map` מותאם יחד עם `backend` או `basis_gates`
  המכילים Gates עם שלושה Qubits או יותר אינו מותר עוד. מפת ה-coupling
  אינה מספקת את פרטי הקישוריות הנדרשים כדי לקבוע את פעולת ה-Gate.
  במקרים אלה, נתיב המעבר המומלץ הוא יצירת Target מותאם מאפס.

6. ארגומנט `inst_map` הוסר ללא אלטרנטיבה. הוא שימש לספק מידע Pulse ל-Transpiler,
  ומודול `qiskit.pulse` המלא הוסר.

## qiskit.pulse {#qiskitpulse}

המודול `qiskit.pulse` הוסר לחלוטין מ-Qiskit, ללא תחליף.
הדבר כולל את הפריטים הקשורים:

- המאפיין `calibrations` של [`QuantumCircuit`](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.QuantumCircuit), [`DAGCircuit`](https://docs.quantum.ibm.com/api/qiskit/qiskit.dagcircuit.DAGCircuit) ו-[`DAGDependency`](https://docs.quantum.ibm.com/api/qiskit/qiskit.dagcircuit.DAGDependency).
- המתודות `has_calibration_for` ו-`add_calibration` של [`QuantumCircuit`](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.QuantumCircuit).
- המתודה `has_calibration_for` של [`DAGCircuit`](https://docs.quantum.ibm.com/api/qiskit/qiskit.dagcircuit.DAGCircuit).
- פאסי ה-Transpiler: `PulseGates`, `RXCalibrationBuilder`, `RZXCalibrationBuilder`, `EchoRZXWeylDecomposition`, `ValidatePulseGates`.
- המודול `qiskit.scheduler` וכל תכניו.
- המתודות `instruction_schedule_map`, `drive_channel`, `measure_channel`, `acquire_channel` ו-`control_channel` של [`BackendV2`](https://docs.quantum.ibm.com/api/qiskit/qiskit.providers.BackendV2).
- המתודות `has_calibration`, `get_calibration`, `instruction_schedule_map` ו-`update_from_instruction_schedule_map` של [`Target`](https://docs.quantum.ibm.com/api/qiskit/qiskit.transpiler.Target).
- ארגומנט `inst_map` של [`generate_preset_pass_manager`](https://docs.quantum.ibm.com/api/qiskit/qiskit.transpiler.generate_preset_pass_manager), [`transpile`](https://docs.quantum.ibm.com/api/qiskit/compiler#transpile), ו-[`Target.from_configuration`](https://docs.quantum.ibm.com/api/qiskit/qiskit.transpiler.Target#qiskit.transpiler.Target.from_configuration).
- השדה `calibration` של [`InstructionProperties`](https://docs.quantum.ibm.com/api/qiskit/qiskit.transpiler.InstructionProperties).

הסיבה לכך היא שמעבדי ה-QPU של IBM Quantum&reg; הפסיקו להציע גישה ברמת Pulse, וצוות Qiskit לא היה לו עוד משאבים לתחזק את החבילה הנדירה בשימוש.
תוכל להמשיך להשתמש במודול `qiskit.pulse` בזמן שימוש בסדרת v1.x של Qiskit, שיש לה תמיכת תיקוני באגים עד ספטמבר 2024 ותמיכת אבטחה עד מרץ 2025, אך המודול הוסר מסדרת v2.x.

רוב השימושים ב-`qiskit.pulse` ניתנים להחלפה בגישה ישירה ל-Gates חלקיים ב-QPU מסוג IBM Quantum Heron (ודורות מאוחרים יותר, אם רלוונטי).
ניתן לקרוא עוד ב[מדריך המעבר לשימוש ב-Pulse לעומת Gates חלקיים](/guides/pulse-migration).

לא ניתן עוד לטעון קבצי QPY המכילים אובייקטים מסוג `ScheduleBlock` ב-Qiskit v2.0 בגלל הסרת Qiskit Pulse.
ניתן להמשיך לטעון קבצים אלה בסדרת Qiskit v1.x.

# טבלת תכונות OpenQASM 3 {#openqasm-3-feature-table}

להלן רשימת תכונות השפה `OpenQASM 3`.

לפרטים נוספים על יכולות אלו, ראה את [המפרט החי של OpenQASM 3.X](https://openqasm.com/).

מפתח:

*   ❌ לא נתמך
*   🟡 תמיכה חלקית
*   ✅ נתמך

משמעות סימן "נתמך" לחלוטין תלויה בעמודה:

* *Qiskit SDK*: ניתן לנתח את התכונה באמצעות [`qiskit.qasm3.loads`](https://docs.quantum.ibm.com/api/qiskit/qasm3) (על ידי שימוש בתוסף `qiskit-qasm3-import`), לייצג אותה ב-[`QuantumCircuit`,](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.QuantumCircuit)
  ולייצא אותה ל-OpenQASM 3 באמצעות [`qiskit.qasm3.dumps`.](https://docs.quantum.ibm.com/api/qiskit/qasm3#dumps)

* *IBM Qiskit Runtime*: Circuit המכיל את תכונת Qiskit המתאימה ניתן לביצוע על חומרה דרך IBM&reg; Qiskit Runtime.

משמעות "תמיכה חלקית" תלויה בדרך כלל בהערות המקושרות.

<Admonition type="note">
השיטה הנפוצה ביותר לשליחת Circuits ל-IBM Qiskit Runtime היא יצירת ה-Circuit בממשק Python של Qiskit SDK.
Circuits שנבנו ונשלחו בדרך זו אינם צריכים להיטען מקבצי OpenQASM 3 לתוך Qiskit SDK.

אם אינך משתמש ב-OpenQASM 3 ישירות, תוכל להשתמש בבטחה בתכונות הנתמכות לייצוג ב-Qiskit SDK, לייצוא ל-OpenQASM 3, ולשליחה ל-IBM Qiskit Runtime.
זה כולל תכונות שלא ניתן לטעון עם Qiskit SDK מ-OpenQASM 3.
</Admonition>

## הערות {#notes}

<span id="1"></span>
1.  לתכונות תוכנית OpenQASM 3 אלו אין השפעה על הביצוע, ו-Qiskit מסיר אותן כחלק מניתוח הקבצים. ניתן לשלוח קבצים שמשתמשים בהן, אך לא תהיה להן כל השפעה. לגבי קבצי `include`, `stdgates.inc` נתמך כיום כקלט ל-Qiskit, וביצוע על Backend תמיד דורש שה-Circuits עברו קומפילציה ל-Backend Instruction Set Architecture (ISA), שם קבצי `include` אינם רלוונטיים.

<span id="2"></span>
2. Qiskit SDK תומך בניתוח ויצוא קבצי OpenQASM 3 עם כל הצהרות ``qubit``. לביצוע על חומרה, רק Circuits המוגדרים במונחים של qubits פיזיים (לדוגמה, `$0`) תקפים. Qiskit SDK מייצא אוטומטית OpenQASM 3 במונחים של מזהי hardware-qubit הנתמכים אם ה-Circuit עבר Transpilation עבור Backend עם מידע פריסה.

<span id="3"></span>
3. הצהרות משתנים מסוג `bit`- ו-`bit[n]` ב-Qiskit SDK מתאימות להצהרות `Clbit` ו-`ClassicalRegister`.

<span id="4"></span>
4. נכון ליולי 2025, Qiskit SDK יכול לייצג משתנים מקומיים מסוג מוגבל, יכול לייצג פעולות runtime רבות על אובייקטים אלו, ותומך בייצואם ל-OpenQASM 3. עם זאת, Qiskit SDK (דרך `qiskit-qasm3-import` v0.6.0) אינו תומך בניתוח קבצי OpenQASM 3 המכילים הצהרות משתנים, ותמיכתו בניתוח ביטויי משתנים מוגבלת מאוד. באופן כללי, רוב מה ש-Qiskit יכול לייצג במערכת הביטויים שלו ניתן לביצוע על חומרת dynamic circuits מתאימה, גם אם הביטוי עדיין לא ניתן לניתוח על ידי Qiskit SDK. ראה [תיעוד Qiskit של המודול `qiskit.circuit.classical`](https://docs.quantum.ibm.com/api/qiskit/circuit_classical) לקבלת המידע העדכני ביותר.

<span id="5"></span>
5. Qiskit SDK יכול לייצג כינוי רגיסטרים (register aliasing) עבור רגיסטרים קוונטיים וקלאסיים כאחד, אך מומלץ מאוד להימנע משימוש בכינויים של רגיסטרים קלאסיים. רוב הביטויים על רגיסטרים קלאסיים אינם עובדים עם כינויים, ורגיסטרים קלאסיים עם כינויים אינם נתמכים לביצוע על חומרה. מנתח OpenQASM 3 של Qiskit יכול לפתור הצהרות כינוי `let` הקושרות תוצאה של שרשור רגיסטרים.

<span id="6"></span>
6. Qiskit SDK תומך בעיכובים מפורשים דרך `QuantumCircuit.delay`, ולתיבות Circuit (`QuantumCircuit.box`) יכולות להיות משכי זמן מפורשים. משכי זמן אלו יכולים לכלול ביטויים קלאסיים של משתני `stretch`. Qiskit SDK (נכון ליולי 2025 דרך `qiskit-qasm3-import` v0.6.0) אינו תומך בניתוח הצהרות מסוג `duration` או `stretch` מקבצי OpenQASM 3. לחומרה יש תמיכה מוגבלת במשכי זמן הכוללים `stretch`.

<span id="7"></span>
7. יש לבצע Transpilation של Circuits ל-Backend ISA כדי להריץ אותם על חומרת IBM. הדבר מונע ממנהגי `gate` מותאמים אישית ומבני רמה גבוהה כמו Gate modifiers (כגון `inv @`) מלהיות תקפים לביצוע ישיר על חומרה, אך תהליך ה-`transpile` מפתור אותם ל-ISA Circuits תקפים. Qiskit SDK (נכון ליולי 2025, דרך `qiskit-qasm3-import` v0.6.0) יעריך Gate modifiers באופן נלהב במהלך הניתוח, ולכן אלו לא יהיו גלויים ב-`QuantumCircuit` המתקבל, ועלולים לגרום לעלות ביצוע בזמן ריצה.

<span id="8"></span>
8. Qiskit SDK יכול לייצג זרימת בקרה מובנית ולייצא אותה ל-OpenQASM 3. הצהרות `continue` ו-`break` יכולות מבחינה טכנית להיות מיוצגות על ידי Qiskit, אך אינן נתמכות היטב אפילו בתוך Qiskit SDK. לולאות `for` ב-Qiskit v2.1.0 אינן נתמכות היטב. זרימת בקרה מקוננת (כגון `if` בתוך `if` נוסף, או הצהרת `else if`) אינה כשירה לביצוע על חומרה.

<span id="9"></span>
9. Qiskit SDK תומך בהצהרת כל סוג קלאסי נתמך כמשתנה `input` על ה-Circuit. משתנים כאלה אינם כשירים כיום לביצוע על חומרה, ולא ניתן לטעון אותם על ידי מייבא OpenQASM 3 של Qiskit. אובייקטי `Parameter` לא-קשורים המצויים ב-`QuantumCircuit` מיוצאים כמשתני `input float[64]`. אפשרויות תצורת runtime מסוימות יכולות לאפשר ביצוע של Circuits כאלה על Backend-ים מסוימים.

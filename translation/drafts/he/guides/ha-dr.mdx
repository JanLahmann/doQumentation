# הבנת זמינות גבוהה והתאוששות מאסון עבור Qiskit Runtime {#understanding-high-availability-and-disaster-recovery-for-qiskit-runtime}

זמינות גבוהה (HA) היא היכולת של שירות להישאר פעיל ונגיש בנוכחות כשלים בלתי צפויים. התאוששות מאסון היא תהליך החזרת מופע השירות למצב תקין. לקוחות Qiskit Runtime אינם אחראים לתכנון או לביצוע פעולות התאוששות מאסון.

Qiskit Runtime הוא שירות אזורי העומד ב[יעדי רמת שירות (SLO)](https://cloud.ibm.com/docs/resiliency?topic=resiliency-slo) המוגדרים בכל תכנית בתשלום. למידע נוסף, ראה את [הסכם רמת השירות (SLA)](https://www.ibm.com/support/customer/csol/terms/?id=i126-9268&lc=en) עבור IBM Cloud&reg;. למידע נוסף על אזורי IBM Cloud ומרכזי הנתונים הזמינים עבור Qiskit Runtime, ראה [זמינות שירות ותשתית לפי מיקום.](https://cloud.ibm.com/docs/overview?topic=overview-services_region)

## ארכיטקטורת זמינות גבוהה {#high-availability-architecture}

### תכונות זמינות גבוהה {#high-availability-features}

שירות זה מסופק על פני שלושה אזורים באזור מרובה-אזורים ללא נקודת כשל בודדת. בקשות API מנותבות דרך מאזן עומסים לצמתי מופע ה-HA, כאשר כל אחד מהם נמצא באזור שונה.

אם צומת מופע HA או אזור כלשהו נכשל, השירות ממשיך לנתב בקשות API דרך צמתי מופע ה-HA הפעילים. לא מתרחשת כל הידרדרות ניכרת בשירות במהלך כשל אזורי.

השירות זמין באזורים הבאים:

- וושינגטון די.סי. (us-east)
- פרנקפורט (eu-de)

`Neat` ממירה את מעגלי הקלט ל-Clifford לצורך סימולציה יעילה, תוך שמירה על המבנה והעומק שלהם. מעגלי Clifford סובלים מרמות רעש דומות ומשמשים כפרוקסי טוב לחקר המעגל המקורי שמעניין אותך.

הדוגמאות הבאות ממחישות מצבים שבהם `Neat` יכולה להיות משאב שימושי.
ראשית, ייבא את החבילות הרלוונטיות ו[אמת את עצמך מול שירות Qiskit Runtime.](/guides/cloud-setup)

```python
import numpy as np
import random

from qiskit.circuit import QuantumCircuit
from qiskit.transpiler import generate_preset_pass_manager
from qiskit.quantum_info import SparsePauliOp

from qiskit_ibm_runtime import QiskitRuntimeService, EstimatorV2 as Estimator

```

```python
# Choose the least busy backend
service = QiskitRuntimeService()
backend = service.least_busy(operational=True, simulator=False)

# Generate a preset pass manager
# This will be used to convert the abstract circuit to an equivalent Instruction Set Architecture (ISA) circuit.
pm = generate_preset_pass_manager(backend=backend, optimization_level=0)

# Set the random seed
random.seed(10)
```

## אתחל Circuit יעד {#initialize-a-target-circuit}

שקול Circuit בן שישה Qubits עם המאפיינים הבאים:

* מתחלף בין סיבובי `RZ` אקראיים ושכבות של Gate-ים `CNOT`.
* בעל מבנה מראה, כלומר מבצע אוניטרי `U` ולאחריו ההופכי שלו.

```python
def generate_circuit(n_qubits, n_layers):
    r"""
    A function to generate a pseudo-random a circuit with ``n_qubits`` qubits and
    ``2*n_layers`` entangling layers of the type used in this notebook.
    """
    # An array of random angles
    angles = [
        [random.random() for q in range(n_qubits)] for s in range(n_layers)
    ]

    qc = QuantumCircuit(n_qubits)
    qubits = list(range(n_qubits))

    # do random circuit
    for layer in range(n_layers):
        # rotations
        for q_idx, qubit in enumerate(qubits):
            qc.rz(angles[layer][q_idx], qubit)

        # cx gates
        control_qubits = (
            qubits[::2] if layer % 2 == 0 else qubits[1 : n_qubits - 1 : 2]
        )
        for qubit in control_qubits:
            qc.cx(qubit, qubit + 1)

    # undo random circuit
    for layer in range(n_layers)[::-1]:
        # cx gates
        control_qubits = (
            qubits[::2] if layer % 2 == 0 else qubits[1 : n_qubits - 1 : 2]
        )
        for qubit in control_qubits:
            qc.cx(qubit, qubit + 1)

        # rotations
        for q_idx, qubit in enumerate(qubits):
            qc.rz(-angles[layer][q_idx], qubit)

    return qc

# Generate a random circuit
qc = generate_circuit(6, 3)
# Convert the abstract circuit to an equivalent ISA circuit.
 isa_qc = pm.run(qc)

qc.draw("mpl", idle_wires=0)
```

## המר את המעגלים ל-Clifford {#cliffordize-the-circuits}

מעגלי ה-PUB שהוגדרו קודם אינם Clifford, מה שמקשה על הסימולציה הקלאסית שלהם. עם זאת, תוכל להשתמש בשיטת `Neat` [`to_clifford`](https://docs.quantum.ibm.com/api/qiskit-ibm-runtime/debug-tools-neat#to_clifford) כדי למפות אותם למעגלי Clifford לסימולציה יעילה יותר. השיטה [`to_clifford`](https://docs.quantum.ibm.com/api/qiskit-ibm-runtime/debug-tools-neat#to_clifford) היא עטיפה סביב מעבר ה-Transpiler [`ConvertISAToClifford`](https://docs.quantum.ibm.com/api/qiskit-ibm-runtime/transpiler-passes-convert-isa-to-clifford), שניתן להשתמש בו גם באופן עצמאי. בפרט, הוא מחליף Gate-ים חד-Qubit שאינם Clifford במעגל המקורי ב-Gate-ים חד-Qubit של Clifford, אך אינו משנה את ה-Gate-ים דו-Qubit, מספר ה-Qubits, או עומק המעגל.

ראה [סימולציה יעילה של מעגלי מייצב עם פרימיטיבים של Qiskit Aer](/guides/simulate-stabilizer-circuits) למידע נוסף על סימולציית מעגלי Clifford.
ראשית, אתחל את `Neat`.

```python
# You could specify a custom `NoiseModel` here. If `None`, `Neat`
# pulls the noise model from the given backend
noise_model = None

# Initialize `Neat`
analyzer = Neat(backend, noise_model)
```

לאחר מכן, המר את ה-PUBs ל-Clifford.

```python
clifford_pubs = analyzer.to_clifford(pubs)

clifford_pubs[0].circuit.draw("mpl", idle_wires=0)
```

## יישום 1: ניתוח השפעת הרעש על פלטי המעגל {#application-1-analyze-the-impact-of-noise-on-the-circuit-outputs}

דוגמה זו מראה כיצד להשתמש ב-`Neat` כדי לחקור את השפעת מודלי רעש שונים על PUBs כפונקציה של עומק המעגל, באמצעות הרצת סימולציות בתנאים אידיאליים ([`ideal_sim`](https://docs.quantum.ibm.com/api/qiskit-ibm-runtime/debug-tools-neat#ideal_sim)) ורועשים ([`noisy_sim`](https://docs.quantum.ibm.com/api/qiskit-ibm-runtime/debug-tools-neat#noisy_sim)). זה יכול להיות שימושי להגדרת ציפיות לגבי איכות התוצאות הניסיוניות לפני הרצת עבודה על QPU. למידע נוסף על מודלי רעש, ראה [סימולציה מדויקת ורועשת עם פרימיטיבים של Qiskit Aer.](/guides/simulate-with-qiskit-aer#exact-and-noisy-simulation-with-qiskit-aer-primitives)

התוצאות המדומות תומכות בפעולות מתמטיות, ולכן ניתן להשוות ביניהן (או עם תוצאות ניסיוניות) לחישוב מדדי ביצועים.

<Admonition type="caution">
QPU יכול להיות מושפע מסוגים שונים של רעש. מודל הרעש של Qiskit Aer המשמש כאן מדמה רק חלק מהם ולכן צפוי להיות פחות חמור מהרעש ב-QPU אמיתי.

לפרטים על אילו שגיאות נכללות בעת אתחול מודל רעש מ-QPU, ראה את תיעוד ה-API של Aer [`NoiseModel`](https://qiskit.github.io/qiskit-aer/stubs/qiskit_aer.noise.NoiseModel.html#qiskit_aer.noise.NoiseModel.from_backend).
</Admonition>

```python
# Figure of merit: Absolute difference
def rdiff(res1, re2):
    r"""The absolute difference between `res1` and re2`.

    --> The closer to `0`, the better.
    """
    d = abs(res1 - re2)
    return np.round(d.vals * 100, 2)

for idx, (ideal_res, noisy_res) in enumerate(

    vals = rdiff(ideal_res, noisy_res)

    )
```

- אם ההפרש המוחלט הממוצע גדול מ-90%, הפחתת שגיאות כנראה לא תעזור.

## יישום 2: השוואת אסטרטגיות שונות {#application-2-benchmark-different-strategies}

דוגמה זו משתמשת ב-`Neat` כדי לסייע בזיהוי האפשרויות הטובות ביותר עבור ה-PUBs שלך. לשם כך, שקול הרצת בעיית הערכה עם PEA, שאינה ניתנת לסימולציה עם `qiskit_aer`. תוכל להשתמש ב-`Neat` כדי לסייע בקביעה אילו גורמי הגברת רעש יעבדו הכי טוב, ולאחר מכן להשתמש בגורמים אלו בעת הרצת הניסוי המקורי על QPU.

```python
# Generate a circuit with six qubits and six layers
 isa_qc = pm.run(generate_circuit(6, 3))

# Use the same observables as previously
pubs = [(isa_qc, isa_obs)]
clifford_pubs = analyzer.to_clifford(pubs)
```

```python
noise_factors = [
    [1, 1.1],
    [1, 1.1, 1.2],
    [1, 1.5, 2],
    [1, 1.5, 2, 2.5, 3],
    [1, 4],
]
```

```python
# Run the PUBs on a QPU
estimator = Estimator(backend)
estimator.options.default_shots = 100000
estimator.options.twirling.enable_gates = True
estimator.options.twirling.enable_measure = True
estimator.options.twirling.shots_per_randomization = 100
estimator.options.resilience.measure_mitigation = True
estimator.options.resilience.zne_mitigation = True
estimator.options.resilience.zne.amplifier = "pea"

jobs = []
for factors in noise_factors:
    estimator.options.resilience.zne.noise_factors = factors
    jobs.append(estimator.run(clifford_pubs))

results = [job.result() for job in jobs]
```

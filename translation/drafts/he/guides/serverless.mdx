# מהו Qiskit Serverless? {#what-is-qiskit-serverless}

Qiskit Serverless מספק ממשק פשוט להרצת עומסי עבודה על משאבים קוונטיים-קלאסיים. זה כולל פריסת תוכניות ל-IBM Quantum&reg; Platform והרצת עומסי עבודה מרחוק, כמו גם ניהול משאבים קל לשימושים של מולטי-ענן ומחשוב-על ממוקד קוונטום.

משתמשי Premium יכולים לבנות, לפרוס ולהריץ את עומסי העבודה שלהם מרחוק על מחשוב קלאסי הזמין דרך IBM Quantum Platform.

<Admonition type="note">
    This is an experimental feature available only for IBM Quantum Premium Plan, Flex Plan, and On-Prem (via IBM Quantum Platform API) Plan users.
</Admonition>

## מתי כדאי להשתמש ב-Qiskit Serverless? {#when-should-i-use-qiskit-serverless}

Qiskit Serverless עוזר לנהל משאבים קלאסיים וקוונטיים לאורך כל זרימת העבודה של [Qiskit patterns](./intro-to-patterns). זה כולל כמה מהדוגמאות הבאות:

- הקבלה של משימות קלאסיות, כגון עיבוד מקדים ועיבוד לאחר
- שמירת עומסי עבודה ממושכים בענן, גם אם המחשב הנייד שלך כבוי
- פריסת תוכניות לשימוש חוזר

## מה כלול? {#what-is-included}

משאבי המחשוב הבאים כלולים כברירת מחדל בסביבת Qiskit Serverless. משתמשים יכולים ליצור קשר עם [support](/guides/support) להגדלת כל אחד מאלה.

- 16 ליבות vCPU
- 64GB RAM
- מקסימום שתי עבודות פעילות למשתמש בכל עת

## התקנת Qiskit Serverless {#install-qiskit-serverless}

כדי להשתמש ב-Qiskit Serverless ב-IBM Quantum Platform, התקן את החבילות הבאות:

```bash
pip install qiskit-ibm-catalog qiskit_serverless
```

`qiskit_ibm_catalog` מספקת כלים בצד הלקוח להעלאה והרצה של תוכניות מרוחקות, בעוד `qiskit_serverless` מספקת כלים בצד השרת לפיזור מחשוב וניהול נתונים. חבילות אלה דורשות `python` 3.11+. למשתמשים וארגונים שרוצים להריץ את Qiskit Serverless על תשתית מותאמת אישית, עקוב אחר מדריך [Cloud infrastructure setup](https://qiskit.github.io/qiskit-serverless/).

<Admonition type="tip">
Currently, the IBM Quantum workloads table only reflects Qiskit Runtime workloads. Use `job.status()` to see your Qiskit Serverless workload's current status.
</Admonition>

# ניהול משתמשי IBM Cloud

מדריך זה מדגים כיצד להשתמש ב-IBM Cloud&reg; כדי לאפשר למשתמשים שיש להם חשבונות IBM Cloud גישה לסביבה, ומספק הוראות למשתמשים לגישה לסביבה.

לניהול משתמשי ספק זהות במקום זאת, עקוב אחר ההוראות בנושא [ניהול ספקי זהות](/guides/manage-appid).

## שלבים {#steps}

עקוב אחר השלבים ברמה גבוהה הבאים לניהול משתמשי IBM Cloud:

1. ודא שלמשתמשים שברצונך להזמין יש חשבונות IBM Cloud.
2. עבור ל-IAM ו[הזמן משתמשים](https://cloud.ibm.com/iam/users/invite_users)
   <Admonition type="note">
   לא ניתן לנהל משתמשים עד שהם מקבלים את ההזמנה ומתחברים לפחות פעם אחת.
   </Admonition>
3. אופציונלי: שנה את הקצאות הפרויקטים וקבוצות הגישה של המשתמשים מ-[IAM.](https://cloud.ibm.com/iam/users)

<span id="user-org"></span>
## תהליך משתמש {#user-flow}

1. המשתמש מקבל הזמנה, ואז מתחבר דרך [פורטל IBM Cloud](https://cloud.ibm.com/).
2. המשתמש יוצר מפתח API על ידי מעבר אל ([Manage → Access (IAM) → API keys](https://cloud.ibm.com/iam/apikeys)).
3. למידע נוסף, משתמשים יכולים לעיין ב-[התקנת Qiskit.](/guides/install-qiskit)

<span id="steps-cloud-org"></span>
## תרחיש לדוגמה {#example-scenario}

דוגמה זו יוצרת את ההגדרה הבאה:

* ישנם שני פרויקטים, `ml` ו-`finance`.
   * לפרויקט `ml` צריכה להיות גישה לאינסטנסים של השירות `QR-ml` ו-`QR-common`.
   * לפרויקט `finance` צריכה להיות גישה לאינסטנסים של השירות `QR-finance` ו-`QR-common`.
* ישנם שלושה משתמשים:
   * לפאטימה צריכה להיות גישה לפרויקט `ml`.
   * לראווי צריכה להיות גישה לפרויקט `finance`.
   * לאמירה צריכה להיות גישה לשני הפרויקטים.
* הדוגמה משתמשת בקבוצות גישה ללא קבוצות משאבים.
* המשתמשים מוגדרים ב-IBM Cloud והקצאות הפרויקטים מתבצעות שם גם כן.
* המשתמשים צריכים להיות מסוגלים למחוק משימות.

השלבים ליישום הגדרה זו הם:

2. מנהל ה-Cloud יוצר שלושה אינסטנסים של שירות: `QR-ml`, `QR finance`, ו-`QR-common`.
3. מנהל ה-Cloud יוצר כלל מותאם אישית הכולל את הפעולה `quantum-computing.job.delete`.
1. מנהל ה-Cloud יוצר שתי קבוצות גישה:
   * קבוצת הגישה `ml` יכולה לגשת ל-`QR-ml` ו-`QR-common`.
   * קבוצת הגישה `finance` יכולה לגשת ל-`QR-finance` ו-`QR-common`.
1. מנהל ה-Cloud מזמין משתמשי ענן לפרויקט המתאים. באופן ספציפי, הוא מזמין ומקצה משתמשים לקבוצת גישה הכוללת את הפרויקט.
   * פאטימה מתווספת לקבוצת הגישה "ml".
   * ראווי מתווסף לקבוצת הגישה "finance".
   * אמירה מתווספת לשתי קבוצות הגישה "ml" ו-"finance".
1. משתמשים יכולים להתחבר דרך פורטל IBM Cloud, ליצור מפתחות API, ולעבוד עם אינסטנסים של השירות של הפרויקטים שלהם.

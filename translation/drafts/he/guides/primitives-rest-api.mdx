## פרימיטיב Estimator עם REST API {#estimator-primitive-with-rest-api}

### 1. אתחול החשבון {#1-initialize-the-account}

מאחר ש-Qiskit Runtime Estimator הוא שירות מנוהל, תחילה עליך לאתחל את החשבון שלך. לאחר מכן תוכל לבחור את המכשיר שבו ברצונך להשתמש לחישוב ערך הציפייה.

מצא פרטים על אופן אתחול החשבון, צפייה ב-Backends הזמינים, וביטול תוקף אסימונים בנושא [זה](/guides/cloud-setup-rest-api).

### 2. יצירת Circuit מסוג QASM {#2-create-a-qasm-circuit}

אתה זקוק לפחות ל-Circuit אחד כקלט לפרימיטיב Estimator.

הגדר Circuit קוונטי מסוג QASM. לדוגמה:

```python
qasm_string='''
OPENQASM 3;
include "stdgates.inc";
qreg q[2];
creg c[2];
x q[0];
cx q[0], q[1];
c[0] = measure q[0];
c[1] = measure q[1];
'''
```

קטעי הקוד הבאים מניחים ש-`qasm_string` עבר transpilation למחרוזת חדשה `resulting_qasm`.

### 3. הרצת ה-Circuit הקוונטי באמצעות Estimator V2 API {#3-run-the-quantum-circuit-using-the-estimator-v2-api}

<Admonition type="note">
    המשימות הבאות משתמשות ב[פרימיטיבים Qiskit Runtime V2](/guides/v2-primitives). גם `SamplerV2` וגם `EstimatorV2` מקבלים כקלט PUB אחד או יותר (primitive unified blocs). כל PUB הוא tuple המכיל Circuit אחד ואת הנתונים המשודרים אל ה-Circuit, שיכולים לכלול מספר observables ופרמטרים. כל PUB מחזיר תוצאה.
</Admonition>

```python
import requests

url = 'https://quantum.cloud.ibm.com/api/v1/jobs'
auth_id = "Bearer <YOUR_BEARER_TOKEN>"
crn = "<SERVICE-CRN>"
backend = "<BACKEND_NAME>"

headers = {
    'Content-Type': 'application/json',
    'Authorization':auth_id,
    'Service-CRN': crn
    }

job_input = {
    'program_id': 'estimator',
    "backend": backend,
    "params": {
        "pubs": [ #primitive unified blocs (PUBs) containing one circuit each.
            [resulting_qasm, # QASM circuit
             {"IIZII": 1, "XIZZZ": 2.3}, # Observable
             None # parameter values
             ]]
}}

response = requests.post(url, headers=headers, json=job_input)

if response.status_code == 200:
    job_id = response.json().get('id')
    print("Job created:",response.text)
else:
    print(f"Error: {response.status_code}")
```

### 4. בדיקת סטטוס המשימה וקבלת תוצאות {#4-check-job-status-and-get-results}

לאחר מכן, העבר את `job_id` ל-API:

```python
response_status_singlejob= requests.get(url+'/'+job_id, headers=headers)
response_status_singlejob.json().get('state')
```
פלט

```text
>>> Job ID: 58223448-5100-4dec-a47a-942fb30edcad
>>> Job Status: JobStatus.RUNNING
```

קבל תוצאות משימה:
```python
response_result= requests.get(url+'/'+job_id+'/results', headers=headers)

res_dict=response_result.json()

estimator_result=res_dict['results']
print(estimator_result)
```
פלט
```text
[{'data': {'evs': 0.7428980350102542, 'stds': 0.029884014518789213, 'ensemble_standard_error': 0.03261147170624149}, 'metadata': {'shots': 10016, 'target_precision': 0.01, 'circuit_metadata': {}, 'resilience': {}, 'num_randomizations': 32}}]
```

### 5. עבודה עם אפשרויות Runtime {#5-work-with-runtime-options}

טכניקות הפחתת שגיאות מאפשרות למשתמשים להפחית שגיאות Circuit על ידי מידול רעש המכשיר בזמן הביצוע. לרוב זה גורם לתקורה בעיבוד מקדים קוונטי הקשורה לאימון מודל, ולתקורה בעיבוד לאחר מכן כדי להפחית שגיאות בתוצאות הגולמיות בעזרת המודל שנוצר.

טכניקות הפחתת השגיאות המובנות בפרימיטיבים הן אפשרויות resilience מתקדמות. כדי לציין אפשרויות אלו, השתמש באפשרות `resilience_level` בעת שליחת המשימה שלך.

הדוגמאות הבאות מדגימות את אפשרויות ברירת המחדל עבור dynamical decoupling, twirling, ו-TREX + ZNE. מצא אפשרויות נוספות ופרטים נוספים בנושא [טכניקות הפחתה ודיכוי שגיאות](./error-mitigation-and-suppression-techniques).

## פרימיטיב Sampler עם REST API ו-Circuits עם פרמטרים {#sampler-primitive-with-rest-api-and-parameterized-circuits}

### 1. אתחול החשבון {#1-initialize-the-account}

מאחר ש-Qiskit Runtime הוא שירות מנוהל, תחילה עליך לאתחל את החשבון שלך. לאחר מכן תוכל לבחור את המכשיר שבו ברצונך להשתמש להרצת החישובים שלך.

מצא פרטים על אופן אתחול החשבון, צפייה ב-Backends הזמינים, וביטול תוקף אסימונים בנושא [זה](/guides/cloud-setup-rest-api).

### 2. הגדרת פרמטרים {#2-define-parameters}

```python
import requests
import qiskit_ibm_runtime
from qiskit_ibm_runtime import QiskitRuntimeService
from qiskit.transpiler import generate_preset_pass_manager
from qiskit.qasm3 import dumps
from qiskit import QuantumCircuit
from qiskit.circuit import Parameter
from qiskit import transpile

service = QiskitRuntimeService(channel='ibm_quantum')
backend = service.backend("<SPECIFY BACKEND>")

pm = generate_preset_pass_manager(backend=backend, optimization_level=1)

theta = Parameter('theta')
phi = Parameter('phi')
parameter_values = {'theta': 1.57, 'phi': 3.14}   # In case we want to pass a dictionary
```

### 3. יצירת Circuit קוונטי והוספת Gates עם פרמטרים {#3-create-a-quantum-circuit-and-add-parameterized-gates}

```python
qc = QuantumCircuit(2)

# Add parameterized gates
qc.rx(theta, 0)
qc.ry(phi, 1)
qc.cx(0, 1)
qc.measure_all()

# Draw the original circuit
qc.draw('mpl')

# Get an ISA circuit
 isa_circuit = pm.run(qc)
```

### 4. יצירת קוד QASM 3 {#4-generate-qasm-3-code}

```python
qasm_str = dumps(isa_circuit)
print("Generated QASM 3 code:")
print(qasm_str)
```

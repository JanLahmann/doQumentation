# הגדרת חשבון IBM Cloud שלך {#set-up-your-ibm-cloud-account}

אם קיבלת הזמנה בדוא"ל להצטרף לחשבון, עקוב במקום זאת אחרי השלבים ב-[הגדרת חשבון IBM Cloud שלך - משתמשים מוזמנים](/guides/cloud-setup-invited).

אפשר להריץ עומסי עבודה על יחידות עיבוד קוונטיות (QPUs) של IBM&reg; על-ידי הגדרת חשבון ב-IBM Cloud&reg;. בנוסף תצטרך [להתקין ולהגדיר את Qiskit ו-Qiskit Runtime](/guides/install-qiskit#local).

חשבון המשתמש שלך משויך לאחד או יותר [מופעים (instances),](./instances) המזוהים לפי _שמות משאבי ענן_ (CRNs) שמעניקים גישה לשירותי IBM Quantum. בנוסף, אסימון API ייחודי (המכונה גם מפתח) מוקצה לכל חשבון, ומאפשר גישה ל-IBM Quantum מתוך Qiskit. להוראות בחירת מופע ספציפי, ראה [התחברות למופע](./instances#connect-instance).

<span id="cloud"></span>
## הגדרה לשימוש ב-IBM Cloud {#set-up-to-use-ibm-cloud}

### לפני שמתחילים {#before-you-begin}

1. אם עדיין אין לך חשבון, הגדר [חשבון IBM Cloud.](https://quantum.cloud.ibm.com/registration)
   <Admonition type="note">
   אפשר להיות חלק ממספר חשבונות IBM Cloud. ניתן לגשת לכל אחד מחשבונות ה-IBM Cloud שלך בכל עת דרך בורר החשבון שבכותרת ממשק [IBM Quantum Platform](https://quantum.cloud.ibm.com).
   </Admonition>
1. התחבר ל-[IBM Quantum Platform](https://quantum.cloud.ibm.com) עם IBMid או חשבון Google. אם אין לך כזה, תודרך ליצור אחד. (פרטי הכניסה האלו אינם פרטי חשבון ה-IBM Cloud שלך.)
1. ודא שהחשבון והאזור הנכונים נבחרו בבורר החשבון שבכותרת, כפי שמוצג בתמונה הבאה.

    האזור קובע היכן המשימות שלך מורצות והיכן נתוני המשימה נשמרים. ניתן לגשת לכל אחד מהאזורים באמצעות אותו מפתח API, אך תוכל לראות ולגשת רק למופעים שנוצרו באזור שאליו אתה מחובר.

    ![כותרת IBM Quantum Platform מוצגת. בורר החשבון נמצא מימין לסרגל החיפוש. בורר האזור נמצא מימין לבורר החשבון.](/docs/images/guides/cloud-setup/IQP-Header.svg 'IBM Quantum Platform header')

### 1. יצירת מופע {#1-create-an-instance}

ניתן ליצור מופע מדף [המופעים](https://quantum.cloud.ibm.com/instances) של IBM Quantum Platform. ראה [יצירת מופע](/guides/instances#add-instance) להוראות מפורטות.

### 2. אופציונלי: שמירת פרטי הגישה שלך {#2-optional-save-your-access-credentials}

אם אתה משתמש בסביבת Python מהימנה, כמו מחשב נייד אישי, מומלץ להשתמש בשיטת `save_account()` כדי [לשמור את פרטי הגישה שלך באופן מקומי.](/guides/save-credentials)

<span id="cloud-save"></span>
### 3. חיבור Qiskit למופע השירות שלך {#3-connect-qiskit-with-your-service-instance}

- [אתחול השירות](/guides/initialize-account) ב**סביבת Python מהימנה**
- [אתחול השירות ב**סביבה לא מהימנה**](/guides/cloud-setup-untrusted) (כמו מחשב ציבורי)

<Admonition type="note" title="הערות">
   * עקוב אחרי [ההוראות האלו](/guides/cloud-setup-rest-api) אם ברצונך להתחבר באמצעות REST API במקום להשתמש ב-Qiskit.
   * במידת הצורך, השתמש ב[מידע זה](/guides/quickstart-steps-org#firewall) כדי להגדיר את חומת האש שלך ולאפשר גישה לנקודות הקצה של IBM Quantum API.
</Admonition>

# ניהול משתמשי ספק זהות

אתה יכול להשתלב עם ספק הזהות החיצוני שלך (IDP) כדי לאמת בצורה מאובטחת משתמשים חיצוניים לחשבון IBM Cloud שלך. על ידי שימוש בספק חיצוני, אתה יכול לספק למשתמשים בחברה שלך אפשרות להשתמש בכניסה יחידה (SSO). אתה יכול לחבר את חשבון הענן שלך לספק IDP חיצוני באחת מהדרכים הבאות:

- השתמש ב-[IBM Cloud&reg; SAML service provider (SP)](https://cloud.ibm.com/docs/account?topic=account-ibm-idp-integration#federate-cloud-saml)
- השתמש ב-[App ID](https://cloud.ibm.com/docs/account?topic=account-ibm-idp-integration#federate-appid) (עשוי לכרוך בעלות)
- [אחד את המשתמשים שלך לכל מוצרי IBM&reg; עם IBMid](https://cloud.ibm.com/docs/account?topic=account-ibm-idp-integration#federate-ibmid)

<Admonition type="tip">
לאחר חיבור ה-IDP שלך, אתה יכול להגדיר כללים דינמיים כדי שלא תצטרך להוסיף ידנית כל משתמש לכל קבוצה כשהוא מצטרף. ראה [Enabling authentication from an external identity provider](https://cloud.ibm.com/docs/account?topic=account-ibm-idp-integration#app-id-dynamic-rules) להוראות.
</Admonition>

## הצעדים הבאים {#next-steps}

   * עיין בתיעוד המלא של [App ID.](https://cloud.ibm.com/docs/appid?topic=appid-getting-started)
</Admonition>

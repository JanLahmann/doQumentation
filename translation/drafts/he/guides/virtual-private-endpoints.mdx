# שימוש ב-Virtual Private Endpoints עבור VPC לחיבור פרטי ל-IBM Quantum Platform {#use-virtual-private-endpoints-for-vpc-to-privately-connect-to-ibm-quantum-platform}

השתמש ב-Virtual Private Endpoints‏ (VPE) עבור VPC כדי להתחבר ל-IBM Quantum&reg; מרשת ה-VPC שלך, באמצעות כתובת IP שתבחר, המוקצית מ-subnet בתוך ה-VPC שלך.

VPEs הם ממשקי IP וירטואליים הקשורים ל-endpoint gateway שנוצר לכל שירות בנפרד. ה-endpoint gateway הוא פונקציה וירטואלית שמתרחבת אופקית, יתירה וזמינה ביותר, ומשתרעת על כל אזורי הזמינות של ה-VPC שלך. Endpoint gateways מאפשרים תקשורת בין מופעי שרת וירטואלי בתוך ה-VPC לבין שירות IBM Cloud&reg; בגב הרשת הפרטית. VPE עבור VPC מאפשר לך לשלוט בכל הכתובות הפרטיות בתוך הענן שלך. למידע נוסף, ראה [About virtual private endpoint gateways](https://cloud.ibm.com/docs/vpc?topic=vpc-about-vpe).

בתוך IBM Quantum Platform, כל נתוני הלקוח מועברים דרך הרשת הפרטית, בין אם הגישה מתבצעת דרך endpoint ציבורי ובין אם דרך VPE.

חיבור ל-IBM Quantum Platform דרך endpoints ציבוריים משדר את כל נתוני הבקשה והתגובה דרך האינטרנט הציבורי. כדי להתחבר ל-IBM Quantum Platform באמצעות VPE, עליך להשתמש ב-IBM Quantum Platform API או ב-SDK. ניתן לגשת ל-IBM Quantum Platform דרך הרשת הציבורית בלבד.

## לפני שמתחילים {#before-you-begin}

לפני שמכוונים VPE ל-IBM Quantum Platform, יש להשלים את הצעדים הבאים:

* [יצירת Virtual Private Cloud](https://cloud.ibm.com/docs/vpc?topic=vpc-getting-started).
* [תכנון טופולוגיית הרשת](https://cloud.ibm.com/docs/vpc?topic=vpc-planning-considerations) לחיבור ל-VPEs.
* [הגדרת בקרות גישה](https://cloud.ibm.com/docs/vpc?topic=vpc-configure-acls-sgs-endpoint-gateways) עבור ה-VPE שלך.
* הכרת [המגבלות](https://cloud.ibm.com/docs/vpc?topic=vpc-limitations-vpe) של שימוש ב-VPE.
* הכרת אופן [הצגת פרטי VPE](https://cloud.ibm.com/docs/vpc?topic=vpc-vpe-viewing-details-of-an-endpoint-gateway).

## הגדרת VPE עבור IBM Quantum Platform {#set-up-a-vpe-for-ibm-quantum-platform}

ישנן מספר דרכים ליצור VPE gateway. אם משתמשים ב-[CLI](https://cloud.ibm.com/docs/vpc?topic=vpc-vpc-reference&interface=cli#vpe-clis) או ב-[API](https://cloud.ibm.com/apidocs/vpc/latest#create-endpoint-gateway), יש לציין את [Cloud Resource Name‏ (CRN)](https://cloud.ibm.com/docs/account?topic=account-crn) של האזור שאליו רוצים להתחבר ב-IBM Quantum Platform. עיין בטבלה הבאה לקבלת רשימת האזורים והCRNים הזמינים.

| אזור | תוכניות | Fully Qualified Domain Name (FDQN) | Cloud Resource Name (CRN) |
|-----------------|-----------------|-----------------|-----------------|
| `us-east` | `Standard` | `private.us-east.quantum.cloud.ibm.com` | `crn:v1:bluemix:public:quantum-computing:us-east:::endpoint:private.us-east.quantum.cloud.ibm.com` |
| `eu-de` | `Standard` | `private.eu-de.quantum.cloud.ibm.com` | `crn:v1:bluemix:public:quantum-computing:eu-de:::endpoint:private.eu-de.quantum.cloud.ibm.com` |

### הגדרת endpoint gateway {#configuring-an-endpoint-gateway}

כדי להגדיר VPE gateway, בצע את הצעדים הבאים:

1. הצג את רשימת השירותים הזמינים, כולל שירותי תשתית IBM Cloud הזמינים (כברירת מחדל) לכל משתמשי ה-VPC.
1. [צור endpoint gateway](https://cloud.ibm.com/docs/vpc?topic=vpc-ordering-endpoint-gateway) עבור IBM Quantum Platform שאותו רוצים להנגיש באופן פרטי ל-VPC.
1. [קשר כתובת IP שמורה](https://cloud.ibm.com/docs/vpc?topic=vpc-bind-unbind-reserved-ip) ל-endpoint gateway.
1. [צפה ב-VPE gateways שנוצרו](https://cloud.ibm.com/docs/vpc?topic=vpc-vpe-viewing-details-of-an-endpoint-gateway) המשויכים ל-IBM Quantum Platform.

עכשיו מופעי השרת הוירטואלי שב-VPC שלך יכולים לגשת למופע IBM Quantum Platform שלך באופן פרטי.

## שימוש ב-VPE עבור IBM Quantum Platform {#use-your-vpe-for-ibm-quantum-platform}

לאחר יצירת endpoint gateway עבור IBM Quantum Platform, בצע את הצעדים הבאים:

### שימוש ב-VPE עם `qiskit-ibm-runtime` (Python SDK) {#use-the-vpe-with-qiskit-ibm-runtime-python-sdk}

<Admonition type="note">
תמיכה ב-VPE דורשת `qiskit-ibm-runtime` בגרסה 0.24.0 ומעלה.
</Admonition>

בעת יצירת מופע של `QiskitRuntimeService`, ציין `private_endpoint=True`.

```python
service = QiskitRuntimeService(token="your-API_KEY", # Use the 44-character API_KEY you created and saved from the IBM Quantum Platform Home dashboard
                              instance="INSTANCE_CRN",
                              channel="ibm_cloud",
                              private_endpoint=True)
```

### שימוש ב-VPE עם IBM Quantum API {#use-the-vpe-with-the-ibm-quantum-api}

לאחר יצירת endpoint gateway עבור IBM Quantum Platform, השתמש ב-FQDN של service endpoint של האזור הרצוי.

```bash
  curl -X POST https://private.us-east.quantum.cloud.ibm.com/api/v1/jobs -H "Authorization: Bearer $BEARER_TOKEN" -H "Service-CRN: $SERVICE_INSTANCE_CRN" -d '{
    "backend": "backend",
    "program_id": "sampler"
  }'
```

## מצב ברירת המחדל {#default-state}

*מצב ייחוס* מתייחס לנקודת ההתחלה הקבועה של הבעיה שלנו. כדי להכין מצב ייחוס, עלינו להחיל את האוניטרי המתאים, הבלתי-פרמטרי $U_R$ בתחילת מעגל הקוונטי שלנו, כך ש-$|\rho\rangle = U_R |0\rangle$. אם יש לכם השערה מושכלת או נקודת נתונים מפתרון אופטימלי קיים, סביר שהאלגוריתם הוריאציוני יתכנס מהר יותר אם תשתמשו בה כנקודת מוצא.

מצב ייחוס הפשוט ביותר הוא מצב ברירת המחדל, שבו אנו משתמשים במצב ההתחלתי של מעגל קוונטי בן $n$ קיוביטים: $|0\rangle^{\otimes n}$. עבור מצב ברירת המחדל, האופרטור האוניטרי שלנו $U_R \equiv I$. בזכות פשטותו, מצב ברירת המחדל הוא מצב ייחוס תקף שנפוץ בשימוש בתרחישים רבים.
## מצב ייחוס קלאסי {#classical-reference-state}

נניח שיש לכם מערכת של שלושה קיוביטים ואתם רוצים להתחיל במצב $|001\rangle$ במקום במצב ברירת המחדל $|000\rangle$. זהו דוגמה למצב ייחוס קלאסי טהור, וכדי לבנות אותו, פשוט מחילים [שער X](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.library.XGate) על הקיוביט $0$ (לפי סדר הקיוביטים של Qiskit), שכן $|001\rangle = X_0 |000\rangle$.

במקרה זה, האופרטור האוניטרי שלנו הוא $U_R \equiv X_0$, שמוביל למצב ייחוס $|\rho\rangle \equiv |001\rangle$.

## מצב ייחוס קוונטי {#quantum-reference-state}

נניח שאתם רוצים להתחיל במצב מורכב יותר הכולל סופרפוזיציה ו/או שזירה, כגון $\frac{1}{\sqrt{2}}(|100\rangle+|111\rangle)$.

כדי להגיע למצב זה מ-$|000\rangle$, אחת הגישות היא להשתמש ב-[שער Hadamard](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.library.HGate) על קיוביט $0$ ($H_0$), בשער [CNOT (CX)](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.library.CXGate) כאשר קיוביט $0$ הוא קיוביט הבקרה וקיוביט $1$ הוא קיוביט המטרה ($CNOT_{01}$), ולבסוף שער $X$ המוחל על קיוביט $2$ ($X_2$).

בתרחיש זה, האופרטור האוניטרי שלנו הוא $U_{R} \equiv X_2CNOT_{01}H_0|000\rangle$, ומצב הייחוס שלנו הוא $|\rho\rangle \equiv \frac{1}{\sqrt{2}}(|100\rangle+|111\rangle)$.

```python
qc = QuantumCircuit(3)
qc.h(0)
qc.cx(0, 1)
qc.x(2)

qc.draw("mpl")
```

## בניית מצבי ייחוס באמצעות מעגלי תבנית {#constructing-reference-states-using-template-circuits}

ניתן גם להשתמש במגוון מעגלי תבנית, כגון [`TwoLocal`](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.library.TwoLocal), המאפשר לבטא פרמטרים מכווננים ושזירות בקלות. נעמיק במעגלי תבנית אלה בשיעור הבא, אך ניתן להשתמש בהם למצבי הייחוס שלנו *בתנאי* שכובלים את הפרמטרים:

```python
from qiskit.circuit.library import TwoLocal
from math import pi

reference_circuit = TwoLocal(2, "rx", "cz", entanglement="linear", reps=1)
theta_list = [pi / 2, pi / 3, pi / 3, pi / 2]

reference_circuit = reference_circuit.assign_parameters(theta_list)

reference_circuit.decompose().draw("mpl")
```

## מצבי ייחוס ספציפיים לאפליקציה {#application-specific-reference-states}
### למידת מכונה קוונטית {#quantum-machine-learning}

בהקשר של מסווג קוונטי וריאציוני (VQC), נתוני האימון מקודדים למצב קוונטי באמצעות מעגל פרמטרי הידוע בשם *מפת מאפיינים*, שבו כל ערך פרמטר מייצג נקודת נתונים מסט האימון. ה-`zz_feature_map` הוא סוג של מעגל פרמטרי שניתן להשתמש בו כדי להעביר את נקודות הנתונים שלנו ($x$) למפת המאפיינים.

```python
from qiskit.circuit.library import zz_feature_map

data = [0.1, 0.2]

zz_feature_map_reference = zz_feature_map(feature_dimension=2, reps=2)
zz_feature_map_reference = zz_feature_map_reference.assign_parameters(data)
zz_feature_map_reference.decompose().draw("mpl")
```

## 1. מבוא {#1-introduction}

כדי לפתור כל בעיה קוונטית בסדר גודל שימושי, נצטרך להעביר מידע ברחבי המחשב הקוונטי מ-Qubit אחד לאחר. קיימים פרוטוקולים מוכרים לכך, אך חלק מהיסודיים שבהם נוסחו בהקשר של שליחת מידע בין צדדים מרוחקים. לאורך השיעור הזה נשתמש לעיתים בשפה המתאימה להקשר זה, כמו "חברים מרוחקים שמשתפים מידע". אך יש לזכור שלפרוטוקולים האלה יש משמעות רחבה יותר בתחום המחשוב הקוונטי. בשיעור זה נבחן את פרוטוקולי התקשורת הקוונטית הבאים:
- **טלפורטציה קוונטית**
    שימוש במצב שזור משותף (המכונה לעיתים e-bit) לשליחת מצב קוונטי לא ידוע לחבר מרוחק, תוך צורך בתקשורת קלאסית משלימה.
- **קידוד סופר-צפוף קוונטי**
    כיצד לשלוח שני סיביות מידע על ידי שליחת Qubit יחיד לחבר מרוחק (שוב בשימוש ב-Qubit שזורים משותפים מראש).

לרקע נוסף הרלוונטי לנושאים אלה, אנו ממליצים על שיעור 4 ב"יסודות מידע קוונטי" בנושא [שזירה בפעולה](/learning/courses/basics-of-quantum-information/entanglement-in-action/introduction).

בתיאור לעיל, "מצב קוונטי לא ידוע" מתייחס פשוט למצב בצורה המתוארת בשיעור הקודם:
$$
|\psi\rangle =\alpha|0\rangle+\beta|1\rangle
$$
כאשר $\alpha$ ו-$\beta$ הם מספרים מרוכבים כך ש-$|\alpha|^2+|\beta|^2 = 1$. זה מאפשר לנו לכתוב את המצב הקוונטי בצורה
$$
|\psi\rangle =\cos\frac{\theta}{2}|0\rangle+e^{i\varphi}\sin\frac{\theta}{2}|1\rangle=
\left(
\begin{matrix}
\cos\frac{\theta}{2}\\
e^{i\varphi}\sin\frac{\theta}{2}
\end{matrix}
\right)
$$
מכיוון שאנחנו רוצים להיות מסוגלים להעביר את המידע בכל מצב קוונטי אקראי, יצירת מצב כזה היא הנקודה שממנה נתחיל את השיעור הזה.
## 2. מטריצות צפיפות {#2-density-matrices}

ניתן לכתוב גם את המצב הקוונטי $|\psi \rangle$ כמטריצת הצפיפות שלו. צורה זו שימושית לציון תערובת הסתברותית של מצבים קוונטיים טהורים. במקרה של Qubit יחיד, ניתן לכתוב

$$
|\psi \rangle \langle \psi| \equiv  \rho = \left(
\begin{pmatrix}
\cos\frac{\theta}{2}\\
e^{i\varphi}\sin\frac{\theta}{2}
\end{pmatrix}
\right)
\left(
\begin{pmatrix}
\cos\frac{\theta}{2} & e^{-i\varphi}\sin\frac{\theta}{2}
\end{pmatrix}
\right)
=\frac{1}{2}\left(\begin{pmatrix}
1+\cos\theta & e^{-i\varphi}\sin\theta\\
e^{-i\varphi}\sin\theta & 1-\cos\theta
\end{pmatrix}\right)
$$

שימו לב שמטריצת הצפיפות $\rho$ היא סכום לינארי של מטריצות פאולי, כפי שמוצג להלן,

$$
\rho = \frac{1}{2}\bigl( \textbf{I} + (\sin{\theta}\cos{\varphi})\textbf{X}+  (\sin{\theta}\sin{\varphi})\textbf{Y} +  (\cos{\theta})\textbf{Z} \bigr)
$$

או, באופן כללי,

$$
\rho = \frac{1}{2}(\textbf{I} + r_{x}\textbf{X}+ r_{y}\textbf{Y} + r_{z}\textbf{Z})
$$

כאשר
$r_{x}^2+r_{y}^2+r_{z}^2=1$.

וקטור בלוך הוא $\textbf{r} = (r_{x}, r_{y}, r_{z})$.

עכשיו, בואו ניצור מצב קוונטי שרירותי באמצעות מספרים אקראיים.


```python
import numpy as np

# create a random 1-qubit state from a random (theta, varphi) to define r vector
np.random.seed(1)  # fixing seed for repeatibility

theta = np.random.uniform(0.0, 1.0) * np.pi  # from 0 to pi
varphi = np.random.uniform(0.0, 2.0) * np.pi  # from 0 to 2*pi

def get_r_vec(theta, varphi):
    rx = np.sin(theta) * np.cos(varphi)
    ry = np.sin(theta) * np.sin(varphi)
    rz = np.cos(theta)
    return (rx, ry, rz)

# get r vector
rx, ry, rz = get_r_vec(theta, varphi)

print("theta=" + str(theta), ",varphi=" + str(varphi))
print("(rx, ry, rz) = (" + str(rx) + ", " + str(ry) + ", " + str(rz) + ")")
```


## 3. טומוגרפיה של מצב קוונטי {#3-quantum-state-tomography}

אם תמדדו את המצב הקוונטי רק בבסיס החישובי ($|0 \rangle$ ו-$|1 \rangle$), מידע הפאזה (מידע המספרים המרוכבים) יאבד. אך אם יש לנו עותקים רבים של $|\psi \rangle$ על ידי חזרה על תהליך ההכנה (לא ניתן לשכפל מצבים, אך ניתן לחזור על תהליכי הכנה), נוכל להעריך את ערכי $r_{x}, r_{y}, r_{z}$ על ידי ביצוע *טומוגרפיה של מצב קוונטי* עבור מטריצת הצפיפות $\rho$. בהתחשב בצורה:

$$
\rho = \frac{1}{2}(\textbf{I} + r_{x}\textbf{X}+ r_{y}\textbf{Y} + r_{z}\textbf{Z})
$$

מתקיים

$$
Tr(\textbf{X} \rho) =  r_{x}, \quad Tr(\textbf{Y} \rho) =  r_{y}, \quad Tr(\textbf{Z} \rho) =  r_{z}
$$

במקרה של $r_{z}$,

$$
Tr(\textbf{Z} \rho) =  \langle 0|\textbf{Z} \rho|0 \rangle +  \langle 1|\textbf{Z} \rho|1 \rangle
$$
$$
= \langle 0|(|0 \rangle\langle 0|-|1 \rangle\langle 1|) \rho|0 \rangle +\langle 1|(|0 \rangle\langle 0|-|1 \rangle\langle 1|) \rho|1 \rangle
$$
$$
=\langle 0|\rho|0 \rangle- \langle 1| \rho|1 \rangle
$$
$$
=\langle 0|\psi\rangle\langle \psi|0 \rangle -  \langle 1| \psi\rangle\langle \psi|1 \rangle
$$
$$
=|\alpha|^2-|\beta|^2
$$

הטרנספורמציה האחרונה במשוואה מיועדת ל-$|\psi \rangle  =\alpha|0\rangle+\beta|1\rangle$. לכן, ניתן לקבל את $r_{z}$ על ידי הסתברות של $|0 \rangle$ פחות הסתברות של $|1 \rangle$.

### הערכת ערך $r_z$ {#estimate-r_z-value}

כדי להעריך את $r_z$, ניצור מצב קוונטי ונמדוד אותו. לאחר מכן נחזור על ההכנה והמדידה פעמים רבות. לבסוף נשתמש בסטטיסטיקה של המדידות להערכת ההסתברויות לעיל ובכך נעריך את $r_z$.

ליצירת המצב הקוונטי האקראי, נשתמש ב-Gate האוניטרי הכללי $U$ עם הפרמטרים $\theta, \varphi$. (ראו [U-gate](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.library.UGate) למידע נוסף.)

```python
from qiskit import QuantumCircuit

# create a 1-qubit quantum state psi from theta, varphi parameters
qc = QuantumCircuit(1, 1)
qc.u(theta, varphi, 0.0, 0)

# measure in computational basis
qc.measure(0, 0)

qc.draw(output="mpl")
```

## 4. טלפורטציה קוונטית {#4-quantum-teleportation}

בואו נשקול את המצב שבו דמות בשם אליס רוצה לשלוח מצב קוונטי לא ידוע $|\psi \rangle$ לחברה בוב, שנמצא רחוק. נניח שהם יכולים לתקשר רק בתקשורת קלאסית (כמו אימייל או טלפון). אליס לא יכולה להעתיק את המצב הקוונטי (בגלל משפט אי-השכפול). אם היא חזרה על אותו תהליך הכנה פעמים רבות, היא יכלה לצבור סטטיסטיקות כפי שעשינו זה עתה. אבל מה אם יש רק מצב יחיד לא ידוע? מצב זה אולי צץ מתהליך פיזי שאתם רוצים לחקור. או שהוא יכול להיות חלק מחישוב קוונטי גדול יותר. במקרה כזה, כיצד יכולה אליס לשלוח את המצב לבוב? היא יכולה, אם היא ובוב חולקים משאב קוונטי יקר ערך: מצב שזור משותף, כמו מצב בל שהוצג בשיעור הקודם: $\frac {|00\rangle + |11\rangle}{\sqrt 2}.$ לפעמים תראו גם התייחסות לזה כ"זוג EPR" או "e-bit" (יחידה בסיסית של שזירה). אם אליס חולקת מצב שזור כזה עם בוב, היא יכולה *לטלפורט* את המצב הקוונטי הלא ידוע לבוב על ידי ביצוע סדרת פעולות קוונטיות ושליחת שני סיביות של מידע קלאסי.

### 4.1 הפרוטוקול של טלפורטציה קוונטית {#41-the-protocol-of-quantum-teleportation}
**הנחה**: לאליס יש מצב קוונטי לא ידוע $|\psi \rangle$ לשליחה לבוב. אליס ובוב חולקים מצב שזור של 2 Qubit, או e-bit, כשלכל אחד מהם יש אחד מה-Qubit פיזית במיקומו.

הנה אנו מתארים את ההליך ללא הסבר. פרטים אלה ייושמו להלן.
1. אליס שוזרת את $|\psi \rangle$ עם החלק שלה מה-e-bit באמצעות שער CNOT.
2. אליס מפעילה שער Hadamard על $|\psi \rangle$, ומודדת את שני ה-Qubit שלה בבסיס החישובי.
3. אליס שולחת לבוב את תוצאות המדידה שלה (אחת מבין "00", "01", "10", או "11")
4. בוב מבצע אופרטור *תיקון* בהתבסס על שני הסיביות של מידע אליס על החלק שלו מזוג ה-e-bit.
    - אם "00", בוב לא עושה כלום
    - אם "01", בוב מפעיל שער X
    - אם "10", בוב מפעיל שער Z
    - אם "11", בוב מפעיל שער iY = ZX
5. החלק של בוב מה-e-bit הופך ל-$|\psi \rangle$.

זה גם מפורט יותר ב[יסודות המידע הקוונטי](/learning/courses/basics-of-quantum-information/entanglement-in-action/introduction). אבל המצב יהיה ברור יותר כשנממש זאת ב-Qiskit.

### 4.3 הרצה ואישור התוצאה על ידי הפעלת ה-U ההפוך {#43-execute-it-and-confirm-the-result-by-applying-u-inverse}

### 4.5 נקודות מפתח בטלפורטציה קוונטית {#45-key-takeaways-on-quantum-teleportation}

אנחנו יכולים להעביר מצב קוונטי לחבר מרוחק על ידי שיתוף זוג Qubit שזורים (e-bit).

1. האם טלפורטציה קוונטית יכולה לשלוח את המצב הקוונטי מהר יותר מהאור?
	לא, כי אליס חייבת לומר לבוב את תוצאות המדידה בדרך קלאסית.

2. האם טלפורטציה קוונטית תשבור את "משפט אי-השכפול", האוסר על העתקת מצב קוונטי?
	לא, כי המצב הקוונטי המקורי שניתן לאליס על אחד מה-Qubit שלה אבד במדידה. הוא קרס ל-$|0\rangle$ או $|1\rangle$.
## 5. קידוד צפוף קוונטי {#superdense-coding}

כמעט אותה הגדרה יכולה לשמש למטרה שונה. נניח שאליס רוצה לשלוח לבוב שני סיביות של מידע קלאסי, אבל אין לה אפשרות לתקשר עם בוב בערוץ קלאסי. עם זאת, הם חולקים זוג שזור ומותר לה לשלוח את ה-Qubit שלה למיקומו של בוב. שימו לב להבדל מפרוטוקול הטלפורטציה הקוונטית. בטלפורטציה, תקשורת קלאסית _הייתה_ זמינה לחברים, והמטרה הייתה לשלוח מצב קוונטי. כאן, תקשורת קלאסית אינה נגישה והם משתמשים בהעברת Qubit כדי לשתף שתי סיביות של מידע קלאסי.

### 5.1 הפרוטוקול של קידוד צפוף קוונטי {#the-protocol-of-superdense-coding}

**הנחה**: לאליס יש שתי סיביות מידע, נניח $a_1a_2 \in \{00, 01, 10, 11\}$. אליס ובוב חולקים זוג שזור (e-bit), אבל הם לא יכולים לתקשר בערוץ קלאסי.

1. אליס מבצעת אחת מהפעולות הבאות על החלק שלה מה-e-bit.
    - אם $a_1a_2 = 00$, היא לא עושה כלום
    - אם $a_1a_2 = 01$, היא מפעילה את Gate Z
    - אם $a_1a_2 = 10$, היא מפעילה את Gate X
    - אם $a_1a_2 = 11$, היא מפעילה את Gate Z ואת Gate X.
2. אליס שולחת את החלק שלה מה-e-bit למיקומו של בוב.
3. בוב מפעיל CNOT Gate עם ה-Qubit של אליס כשליטה וה-Qubit שלו כיעד, לאחר מכן מפעיל Gate H על ה-Qubit של אליס, ומודד את שני ה-Qubits. המצבים ההתחלתיים האפשריים ותוצאות פעולות בוב הן:
$$
\frac {|00\rangle + |11\rangle}{\sqrt 2} \rightarrow CX_{01}\otimes H_0 \rightarrow |00\rangle
$$
$$
\frac {|00\rangle - |11\rangle}{\sqrt 2} \rightarrow CX_{01}\otimes H_0 \rightarrow |01\rangle
$$
$$
\frac {|10\rangle + |01\rangle}{\sqrt 2} \rightarrow CX_{01}\otimes H_0 \rightarrow |10\rangle
$$
$$
\frac {|10\rangle - |01\rangle}{\sqrt 2} \rightarrow CX_{01}\otimes H_0 \rightarrow -|11\rangle
$$

שימו לב שסימן השלילי של $-|11\rangle$ הוא פאזה גלובלית, לכן הוא אינו מדיד.

### 5.2 Circuit קוונטי המדמה קידוד צפוף קוונטי {#quantum-circuit-simulating-the-superdense-coding}

על בסיס הפרוטוקול של קידוד צפוף קוונטי, אפשר לבנות את ה-Circuit של הקידוד הצפוף כמפורט להלן. נסו לשנות את ההודעה, `msg`, שאליס רוצה להעביר לבוב.

```python
from qiskit import QuantumCircuit
```

שלבי דפוס Qiskit מזוהים בתגובות הקוד.

```python
# Step 1: Map problem to quantum circuits and operators
# Create 2-qubits circuit
qc = QuantumCircuit(2, 2)

# Eve creates EPR pair and send q0 to Alice and q1 to Bob
qc.h(0)
qc.cx(0, 1)
qc.barrier()

# set message which Alice wants to transform to Bob
msg = "11"  # You can change the message

if msg == "00":
    pass
elif msg == "10":
    qc.x(0)
elif msg == "01":
    qc.z(0)
elif msg == "11":
    qc.z(0)
    qc.x(0)

qc.barrier()
# Bob receives EPR qubit from Alice and performs unitary operations
qc.cx(0, 1)
qc.h(0)
qc.barrier()

# Bob measures q0 and q1
qc.measure(0, 0)
qc.measure(1, 1)

qc.draw(output="mpl")
```

```python
# We will execute on a simulator first
from qiskit_aer import AerSimulator
from qiskit.transpiler.preset_passmanagers import generate_preset_pass_manager
from qiskit_ibm_runtime import Sampler

# Define backend
backend = AerSimulator()
shots = 1000

# Transpile to backend
pm = generate_preset_pass_manager(backend=backend, optimization_level=1)
isa_qc = pm.run(qc)

# Run the job
sampler = Sampler(mode=backend)
job_sim = sampler.run([isa_qc], shots=shots)
result_sim = job_sim.result()

# Extract counts data
counts = result_sim[0].data.c.get_counts()
print(counts)
```

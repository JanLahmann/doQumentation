## 1. מבוא {#1-introduction}

שיעור זה יעסוק במספר היבטים של אופטימיזציית מעגלים בחישוב קוונטי. באופן ספציפי, נראה את הערך של אופטימיזציית מעגלים באמצעות הגדרות אופטימיזציה המובנות ב-Qiskit. לאחר מכן נעמיק קצת ונראה מה ניתן לעשות בתור מומחה בתחום היישום שלך כדי לבנות מעגלים בצורה חכמה. לבסוף, נבחן מקרוב מה קורה במהלך הטרנספילציה שעוזרת לנו לבצע אופטימיזציה של המעגלים שלנו.

## 2. אופטימיזציית מעגלים חשובה {#2-circuit-optimization-matters}

ראשית נשווה את תוצאות הרצת מעגלי הכנת מצב GHZ של 5-Qubit ($\frac{1}{\sqrt{2}} \left( |00000\rangle + |11111\rangle \right)$) עם ובלי אופטימיזציה.

### 2.1 רמת אופטימיזציה {#21-optimization-level}
קיימות 4 `optimization_level` זמינות מ-0 עד 3. ככל שרמת האופטימיזציה גבוהה יותר, כך מושקע יותר מאמץ חישובי לאופטימיזציה של המעגל. רמה 0 לא מבצעת אופטימיזציה כלל ועושה רק את הכמות המינימלית של עבודה כדי להפוך את המעגל לניתן להרצה על ה-Backend הנבחר. רמה 3 משקיעה את הכמות הגדולה ביותר של מאמץ (ובדרך כלל זמן ריצה) כדי לנסות לבצע אופטימיזציה של המעגל. רמה 1 היא רמת האופטימיזציה ברירת המחדל.
אנו מבצעים טרנספילציה של המעגל ללא אופטימיזציה (`optimization_level=0`) ועם אופטימיזציה (`optimization_level=2`).
אנו רואים הבדל גדול באורך המעגל של מעגלים מטרנספלים.

```python
pm0 = generate_preset_pass_manager(
    optimization_level=0, backend=backend, seed_transpiler=777
)
pm2 = generate_preset_pass_manager(
    optimization_level=2, backend=backend, seed_transpiler=777
)
circ0 = pm0.run(ghz_circ)
circ2 = pm2.run(ghz_circ)
print("optimization_level=0:")
display(circ0.draw("mpl", idle_wires=False, fold=-1))
print("optimization_level=2:")
display(circ2.draw("mpl", idle_wires=False, fold=-1))
```

הסינתזה החדשה מייצרת מעגל רדוד יותר. למה?

הסיבה היא שהמעגל החדש ניתן למיפוי על Qubit-ים מחוברים ליניארית, ולכן גם על גרף הצימוד heavy-hexagon של IBM&reg; Brisbane, בעוד שהמעגל המקורי דורש קישוריות בצורת כוכב (צומת מדרגה 4) ולכן לא ניתן למיפוי על גרף הצימוד heavy-hex, שצמתיו הם לכל היותר מדרגה 3. כתוצאה מכך, המעגל המקורי דורש ניתוב Qubit שמוסיף Gate-ים מסוג SWAP, ומגדיל את מספר ה-Gate-ים.

מה שעשינו במעגל החדש ניתן לראות כסינתזת מעגל "מודעת לאילוץ צימוד" ידנית. במילים אחרות: פתרון ידני של סינתזת מעגל ומיפוי מעגל בו-זמנית.

```python
# Build a toy example circuit
from math import pi
import itertools
from qiskit.circuit import QuantumCircuit
from qiskit.circuit.library import excitation_preserving

circuit = QuantumCircuit(4, name="Example circuit")
circuit.append(excitation_preserving(4, reps=1, flatten=True), range(4))
circuit.measure_all()

value_cycle = itertools.cycle([0, pi / 4, pi / 2, 3 * pi / 4, pi, 2 * pi])
circuit.assign_parameters(
    [x[1] for x in zip(range(len(circuit.parameters)), value_cycle)], inplace=True
)
circuit.draw("mpl")
```

## 4. דוגמאות מעמיקות {#4-in-depth-examples}
### 4.1 אופטימיזציה של בלוקים דו-Qubit באמצעות סינתזת יוניטרי דו-Qubit {#41-two-qubit-block-optimization-using-two-qubit-unitary-synthesis}

עבור רמות 2 ו-3, יש לנו יותר פאסים (`Collect2qBlocks`, `ConsolidateBlocks`, `UnitarySynthesis`) לאופטימיזציה מעמיקה יותר, כלומר אופטימיזציה של בלוקים דו-Qubit. (השווה את זרימת שלב האופטימיזציה לרמה 2 עם זו שלמעלה לרמה 1)

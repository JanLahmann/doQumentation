## הכנת המילטוניאנים לכימיה קוונטית {#preparing-hamiltonians-for-quantum-chemistry}

צעד ראשון טוב ביישום מחשוב קוונטי על בעיה כימית הוא הגדרת המילטוניאן עבור המערכת שמעניינת אותנו. כאן נגביל את הדיון להמילטוניאנים של כימיה קוונטית, מכיוון שאותם המילטוניאנים מצריכים מיפוי ספציפי למערכות של פרמיונים זהים.

כמי שעוסק בכימיה קוונטית, כנראה שכבר יש לך תוכנה מועדפת לדוגמנות מולקולות, שיכולה לייצר המילטוניאן המתאר את המערכת שמעניינת אותך. כאן נשתמש בקוד המבוסס אך ורק על PySCF, numpy ו-Qiskit. אבל תהליך הכנת ההמילטוניאן ניתן להעברה גם לפתרונות ארוזים מראש. ההבדל היחיד בין גישה זו לתוכנות אחרות יהיה הבדלי תחביר קלים; חלקם מטופלים בסעיף "תוכנות צד שלישי" כדי לאפשר שילוב של תהליכי עבודה קיימים.

יצירת המילטוניאן לכימיה קוונטית לשימוש ב-QPUs של IBM Quantum&reg; כוללת את השלבים הבאים:

1. הגדר את המולקולה שלך (גיאומטריה, ספין, מרחב פעיל וכדומה)
2. צור את ההמילטוניאן הפרמיוני (אופרטורי יצירה ואנהילציה)
3. מפה מהמילטוניאן הפרמיוני לאופרטור בוזוני (בהקשר זה, באמצעות אופרטורי פאולי)
4. אם משתמשים בתוכנת צד שלישי: טפל בכל אי-התאמות תחביר בין תוכנת היצירה ל-Qiskit

ההמילטוניאן הפרמיוני כתוב במונחים של אופרטורים פרמיוניים, ובפרט, מתחשב בכך שאלקטרונים הם פרמיונים בלתי ניתנים להבחנה. משמעות הדבר היא שהם ציתים לסטטיסטיקה שונה לחלוטין מ-Qubits ניתנים להבחנה בוזוניים. מכאן הצורך בתהליך המיפוי.

1. הגדר את המולקולה שלך

כאן נציין תכונות של המולקולה שמעניינת אותנו. בדוגמה זו בחרנו במימן דו-אטומי (מכיוון שההמילטוניאנים המתקבלים קצרים מספיק להצגה).
ל-Python-based Simulations of Chemistry Framework ([PySCF](https://pyscf.org/)) יש אוסף רחב של מודולים לסטרוקטורה אלקטרונית שניתן להשתמש בהם, בין היתר, לייצור המילטוניאנים מולקולריים המתאימים לחישוב קוונטי. [מדריך ה-Quickstart של PySCF](https://pyscf.org/quickstart.html) הוא משאב מעולה לתיאור מלא של כל המשתנים והפונקציונליות. נספק רק סקירה שטחית ביותר, מכיוון שרבים מכם כבר מכירים אותה. כדי להבין אותם טוב יותר, בקרו ב-[PySCF.](https://pyscf.org/)
בקצרה:

__distance__ ניתן לשימוש עבור מולקולות דו-אטומיות, או פשוט ציין קואורדינטות קרטזיות לכל אטום. המרחקים הם ביחידות של אנגסטרום.

__gto__ מייצר אורביטלים מסוג גאוסיאן.

__basis__ מתייחס לפונקציות המשמשות לדוגמנות אורביטלים מולקולריים. כאן 'sto-6g' הוא בסיס מינימלי נפוץ, שנקרא על שם התאמת אורביטלים מסוג סלייטר באמצעות 6 אורביטלים גאוסיאניים פרימיטיביים.

__spin__ ערך שלם המציין את מספר האלקטרונים הלא-מזווגים (שווה ל-$2S$). שים לב שחלק מהתוכנות משתמשות במולטיפליסיטי במקום ($2S+1$).

__charge__ המטען של המולקולה.

__symmetry__ - קבוצת הסימטריה הנקודתית של המולקולה, שניתן לציין עם מחרוזת או לזהות אוטומטית על ידי הגדרת "symmetry = True". כאן "Dooh" היא קבוצת הסימטריה המתאימה למולקולות דו-אטומיות עם שני אטומים מאותו מין.

2. צור המילטוניאן פרמיוני

__scf__ מתייחס למגוון רחב של שיטות שדה עצמי-עקבי.

__rhf__ כמו ב-__mf__ = scf.RHF(mol) ב-mf הוא פותר המשתמש בחישוב Restricted Hartree Fock. הגרעין שלו (E, להלן) הוא האנרגיה הכוללת, כולל דחיית גרעין ואורביטלים מולקולריים.

__mcscf__ הוא חבילה לשדות עצמי-עקביים רב-תצורתיים.

__ao2mo__ הוא טרנספורמציה מאורביטלים אטומיים לאורביטלים מולקולריים.

אנו גם משתמשים במשתנים הבאים:

__ncas__: מספר האורביטלים במרחב הפעיל השלם

__nelecas__: מספר האלקטרונים במרחב הפעיל השלם

המילטוניאנים אלה הם כרגע אופרטורים פרמיוניים (יצירה ואנהילציה), הישימים למערכות של פרמיונים (בלתי ניתנים להבחנה), ובהתאם לכך כפופים לאנטי-סימטריה תחת החלפה. הדבר מוביל לסטטיסטיקה שונה מזו שתחול על מערכת ניתנת להבחנה או בוזונית. כדי להריץ חישובים על QPUs של IBM Quantum, אנחנו צריכים אופרטור בוזוני המתאר את האנרגיה. תוצאת מיפוי כזה נכתבת באופן מקובל במונחים של אופרטורי פאולי, מכיוון שהם הרמיטיים וגם אוניטריים. ישנם מספר מיפויים שניתן להשתמש בהם. אחד הפשוטים ביותר הוא טרנספורמציית Jordan-Wigner.

3. מיפוי ההמילטוניאן

ראוי לציין שישנם כלים רבים לצורך מיפוי המילטוניאן כימי לכזה המתאים להרצה על מחשב קוונטי. כאן נממש את מיפוי Jordan-Wigner ישירות באמצעות PySCF, numpy ו-Qiskit בלבד. נגיב להלן על שיקולי תחביר עבור פתרונות אחרים.
פונקציית Cholesky עוזרת לנו לקבל פירוק דרגה נמוכה של האיברים הדו-אלקטרוניים בהמילטוניאן.

```python
def cholesky(V, eps):
    # see https://arxiv.org/pdf/1711.02242.pdf section B2
    # see https://arxiv.org/abs/1808.02625
    # see https://arxiv.org/abs/2104.08957
    no = V.shape[0]
    chmax, ng = 20 * no, 0
    W = V.reshape(no**2, no**2)
    L = np.zeros((no**2, chmax))
    Dmax = np.diagonal(W).copy()
    nu_max = np.argmax(Dmax)
    vmax = Dmax[nu_max]
    while vmax > eps:
        L[:, ng] = W[:, nu_max]
        if ng > 0:
            L[:, ng] -= np.dot(L[:, 0:ng], (L.T)[0:ng, nu_max])
        L[:, ng] /= np.sqrt(vmax)
        Dmax[: no**2] -= L[: no**2, ng] ** 2
        ng += 1
        nu_max = np.argmax(Dmax)
        vmax = Dmax[nu_max]
    L = L[:, :ng].reshape((no, no, ng))
    print(
        "accuracy of Cholesky decomposition ",
        np.abs(np.einsum("prg,qsg->prqs", L, L) - V).max(),
    )
    return L, ng
```

הפונקציות `identity` ו-`creators_destructors` מחליפות אופרטורי יצירה ואנהילציה בהמילטוניאן הפרמיוני באופרטורי פאולי; `creators_destructors` משתמשת במיפוי Jordan-Wigner.

```python
def identity(n):
    return SparsePauliOp.from_list([("I" * n, 1)])

def creators_destructors(n, mapping="jordan_wigner"):
    c_list = []
    if mapping == "jordan_wigner":
        for p in range(n):
            if p == 0:
                ell, r = "I" * (n - 1), ""
            elif p == n - 1:
                ell, r = "", "Z" * (n - 1)
            else:
                ell, r = "I" * (n - p - 1), "Z" * p
            cp = SparsePauliOp.from_list([(ell + "X" + r, 0.5), (ell + "Y" + r, -0.5j)])
            c_list.append(cp)
    else:
        raise ValueError("Unsupported mapping.")
    d_list = [cp.adjoint() for cp in c_list]
    return c_list, d_list
```

לבסוף, `build_hamiltonian` משתמשת בפונקציות `cholesky`, `identity` ו-`creators_destructors` כדי ליצור את ההמילטוניאן הסופי המתאים להרצה על מחשב קוונטי.

```python
def build_hamiltonian(ecore: float, h1e: np.ndarray, h2e: np.ndarray) -> SparsePauliOp:
    ncas, _ = h1e.shape

    C, D = creators_destructors(2 * ncas, mapping="jordan_wigner")
    Exc = []
    for p in range(ncas):
        Excp = [C[p] @ D[p] + C[ncas + p] @ D[ncas + p]]
        for r in range(p + 1, ncas):
            Excp.append(
                C[p] @ D[r]
                + C[ncas + p] @ D[ncas + r]
                + C[r] @ D[p]
                + C[ncas + r] @ D[ncas + p]
            )
        Exc.append(Excp)
```
    # פירוק דרגה-נמוכה של ה-Hamiltonian
    Lop, ng = cholesky(h2e, 1e-6)
    t1e = h1e - 0.5 * np.einsum("pxxr->pr", h2e)

    H = ecore * identity(2 * ncas)
    # איבר חד-גופי
    for p in range(ncas):
        for r in range(p, ncas):
            H += t1e[p, r] * Exc[p][r - p]
    # איבר דו-גופי
    for g in range(ng):
        Lg = 0 * identity(2 * ncas)
        for p in range(ncas):
            for r in range(p, ncas):
                Lg += Lop[p, r, g] * Exc[p][r - p]
        H += 0.5 * Lg @ Lg

    return H.chop().simplify()
```

לבסוף, אנו משתמשים ב-`build_hamiltonian` כדי לבנות את ה-Hamiltonian הקוביטי שלנו מאופרטורי Pauli באמצעות טרנספורמציית Jordan-Wigner. זה גם נותן לנו את דיוק פירוק ה-Cholesky שבו השתמשנו.

```python
H = build_hamiltonian(ecore, h1e, h2e)
print(H)

4. תוכנות צד שלישי

ישנם מספר חבילות תוכנה שפותחו לכימיה קוונטית, חלקן מציעות ממפים מרובים וכלים להגבלת מרחבים פעילים. השלבים המתוארים לעיל הם כלליים וחלים גם על תוכנות צד שלישי. אבל תוכנות אחרות אלה עשויות להחזיר Hamiltonians בפורמט שאינו מתקבל על ידי Qiskit. לדוגמה, חלק מהתוכנות מחזירות Hamiltonians בצורה:

`H = -0.042 [] + -0.045 [X0 X1 Y2 Y3] + ... + 0.178 [Z0] + ... + 0.176 [Z2 Z3] + -0.243 [Z3]`
שים לב בפרט שה-Gates ממוספרים, ואופרטורי הזהות אינם מוצגים. זה בניגוד ל-Hamiltonians הנהוגים ב-Qiskit, שבהם האיבר `[Z2 Z3]` יכתב כ-`ZZII` (קוביטים 0 ו-1 פועלים תחת אופרטור הזהות, קוביטים 2 ו-3 פועלים תחת אופרטור Z, בסדר שבו קוביט 0 נמצא הכי ימינה).

כדי להתאים לכל זרימות עבודה קיימות שיש לך, בלוק הקוד שלהלן ממיר מתחביר אחד לאחר. הפונקציה `convert_openfermion_to_qiskit` מקבלת כארגומנטים שלה Hamiltonian שנוצר ב-OpenFermion או ב-Tangelo (וכבר מומפה לאופרטורי Pauli באמצעות כל ממפה זמין), ואת מספר הקוביטים הדרושים למולקולה.

```python
from openfermion import QubitOperator
from qiskit.quantum_info import SparsePauliOp

def convert_openfermion_to_qiskit(
    openfermion_operator: QubitOperator, num_qubits: int
) -> SparsePauliOp:
    terms = openfermion_operator.terms

    labels = []
    coefficients = []

    for term, constant in terms.items():
        # Default set to identity
        operator = list("I" * num_qubits)

        # Iterate through PauliSum and replace I with Pauli
        for index, pauli in term:
            operator[index] = pauli
        label = "".join(operator)
        labels.append(label)
        coefficients.append(constant)

    return SparsePauliOp(labels, coefficients)
```

## IBM Quantum sing open-source Inhälde {#ibm-quantums-open-source-content}

IBM stellt en Masse Lernmaterial för Quantencomputing zur Verfüjung — alles open source unger CC BY-SA 4.0:

- **[Learning](https://quantum.cloud.ibm.com/learning)** — Strukturierte Kurse vun de Quantegrundlage bes zo de fortjeschrittene Thematike
- **[Tutorials](https://quantum.cloud.ibm.com/docs/en/tutorials)** — Över 40 Tutorials övver Transpilation, Fählerminderung un mieh
- **[Documentation](https://quantum.cloud.ibm.com/docs/en/guides)** — Aleitunge un API-Referenz för Qiskit
- **[Source repo](https://github.com/Qiskit/documentation)** — All Inhalt op GitHub

Ehr [Quantum Platform](https://quantum.cloud.ibm.com) es emmer aktuell un jot jestallt — der beste Plaaz för Läse, Liere un Nohkieke.

## Wat dit Projäk dozo deit {#what-this-project-adds}

IBM sing [Qiskit-Dokumentation](https://github.com/Qiskit/documentation) es open source (CC BY-SA 4.0), ävver ehr Webawendung es et nit. doQumentation deit en open-source Frontend met live Code-Usföhrung, automatische Credential-Injektion un Simulator-Modus dozo.

Övverall enzesäze — vun [GitHub Pages](https://doqumentation.org) övver [Docker](https://github.com/JanLahmann/doQumentation/pkgs/container/doqumentation) bes zo [RasQberry](https://rasqberry.org/). Luurt ens [all Funktione](/features) aan.

## Code-Usföhrung {#code-execution}

Klickt bei jedem Codeblock op **Run**. Dä eeste Klick startet enne kostenlose Jupyter-Kernel övver [Binder](https://mybinder.org) (1–2 Min.). Donoh jeiht et direkt.

- **Simulator-Modus (kei Konto nüüdich)** — Aktiviert dä [Simulator-Modus](/jupyter-settings#simulator-mode), domet Ehr all Notebooks met AerSimulator udder FakeBackends met Ruschmodell ußföhre künnt. Null Enrichtungsopwand nüüdich.

- **IBM Quantum Hardware** — Jitt üch API-Token un CRN eimol en de [Enstellunge](/jupyter-settings#ibm-quantum) en — de Zogjangsdaate wääde bei jeder Usföhrung automatisch enjefött. [Erstellt e kostenlos Konto](https://quantum.cloud.ibm.com/registration), falls Ehr noch keins hatt.

<details>
<summary>Verfüjbare Usföhrungs-Backends</summary>

Jedes Tutorial hät ußföhrbare Codeblöcke. Klickt op **Run**, domet Ehr se met einem vun drei Backends ußföhre künnt:

1. **Binder** (Standard op GitHub Pages) — Kostenlose Remote-Jupyter-Kernel övver [mybinder.org](https://mybinder.org)
2. **Lokales Jupyter (Docker / RasQberry)** — Verbingt Üch met däm lokale Jupyter-Server met vorinstalliertem Qiskit
3. **Benutzerdefinierte Server** — Verwies en de [Enstellunge](/jupyter-settings#advanced) op enne beliebjije Jupyter-Endpunkt

</details>

<details>
<summary>Bereitstellungsoptione</summary>

doQumentation es verfüjbar als:

- **[GitHub Pages](https://doqumentation.org)** — Statische Website met [Binder](https://mybinder.org) för Remote-Code-Usföhrung
- **[Docker](https://github.com/JanLahmann/doQumentation/pkgs/container/doqumentation)** — Vollständije Stack lokal, offline-fähich
- **[RasQberry](https://rasqberry.org/)** — Selvs jehostet op enem Raspberry Pi met lokalem Jupyter-Kernel

</details>

<details>
<summary>Lokal ußföhre met Podman / Docker</summary>

Installiert [Podman Desktop](https://podman-desktop.io/) udder [Docker Desktop](https://www.docker.com/products/docker-desktop/) (Mac / Windows / Linux), dann föhrt uß:

```bash
# Vollständije Stack: Website + Jupyter + Qiskit (~3 GB)
podman run -p 8080:80 -p 8888:8888 ghcr.io/janlahmann/doqumentation:jupyter
```

Maht [http://localhost:8080](http://localhost:8080) op — Code-Usföhrung funktioneert lokal, kein Binder-Waadezigg.

För en leichtjewichtije Version oohne lokale Code-Usföhrung (~60 MB):

```bash
# Nur statische Website — Code-Usföhrung funktioneert wiggerhin övver Binder
podman run -p 8080:80 ghcr.io/janlahmann/doqumentation:latest
```

Benutzt Ehr Docker? Ersäzt enfach `podman` durch `docker` — de Befehle sin identisch. De Images sin Multi-Arch (`linux/amd64` + `linux/arm64`), esu dat Apple Silicon Macs nativ funktioniere.

</details>

## Hengerjrund {#background}

Dä Tutorial zeisch, wi mer e higher-order binary optimization (HOBO) Problem met dem [Optimization Solver, ene Qiskit Function vun Q-CTRL Fire Opal](/guides/q-ctrl-optimization-solver) löse kann. Dat Beispill us däm Tutorial es e Optimierungsproblem, dat doför jedaach es, de Jrundstandes-Energie vun nem 156-Qubit Ising-Modell met kubische Terme ze fingke. Dä Optimization Solver kann för allgemein Optimierungsprobleme jebruch wäde, die als Ziellfunkzjon definiert sin.

Dä Optimization Solver automatisiert komplett de hardware-bewusste Implementierungsschrete för Optimierungsprobleme op Quantenhardware, un durch et Usnuutze vun [Performance Management](/guides/q-ctrl-performance-management) för de Quantenausführung, erreich hä jenaue Lösunge op utility scale. För en detaillierte Zosammefassung vum Optimization Solver Workflow un Benchmarking-Erjebnisse, luurt ens op [dat veröffentlichte Manuskript](https://arxiv.org/abs/2406.01743).

Dä Tutorial jeiht durch die Schrete:

1. Dat Problem als Ziellfunkzjon definiere
2. Dä Hybridalgorithmus met dem Fire Opal Optimization Solver loufe losse
3. Erjebnisse evaluiere

## Schrett 2: Dä Hybridalgorithmus met dem Fire Opal Optimization Solver loufe losse {#step-2-run-the-hybrid-algorithm-using-the-fire-opal-optimization-solver}
Jetz bruuch mer de Optimization Solver Qiskit Function för dä Algorithmus loufe ze losse. Henger de Kulisse kümmert sich dä Optimization Solver öm et Mapping vum Problem op en hybrid Quantenalgorithmus, et loufe losse vun de Quantenschaltkreise met Fehlerunterdrückung, un et Durchföhre vun de klassische Optimierung.

```python
# Authenticate to the Qiskit Functions Catalog
catalog = QiskitFunctionsCatalog(
    token=token,
    instance=instance,
)

# Load the function
solver = catalog.load("q-ctrl/optimization_solver")
```

Prööf, dat dat jowählte Jerät winnichstens 156 Qubits hätt.

```python
# Specify the target backend name
backend_name = "<CHOOSE_A_BACKEND>"
```

Dä Solver akzepteert en String-Repräsentazjon vun de Ziellfunkzjon.

```python
# Convert the objective function to string format
spin_glass_poly_as_str = srepr(spin_glass_poly)
```

```python
# Run the problem
spin_glass_job = solver.run(
    problem=spin_glass_poly_as_str,
    run_options={"backend_name": backend_name},
)
```

De kanns de bekannte [Qiskit Serverless APIs](/guides/serverless) bruche för dinge Qiskit Function Workload Status ze prööfe:

```python
# Get job status
spin_glass_job.status()
```

Dä Solver jitt e Dictionary met de Lösung un assoziierte Metadaten retuur, wi zum Beispill de Lösungs-Bitstring, de Zahl vun Iterazjone, un et Mapping vun Variable op Bitstring. För en komplette Definizjon vun däm Solver singe Eingabe un Ausgabe, luurt ens op de [Dokumentazjon]().

```python
# Poll for results
result = spin_glass_job.result()
```


Dä Solver hätt de korrekte Lösung jefunge, wat met klassischer Optimierungssoftware validiert wood. De Komplexität vun däm utility-scale Problem bruch en fortjeschrittene Optimierungssoftware för klassisch jelöst ze wäde, wi zum Beispill [IBM ILOG CPLEX Optimization Studio (CPLEX)](https://www.ibm.com/products/ilog-cplex-optimization-studio) oder [Gurobi Optimization](https://www.gurobi.com/).
Als visuelle Analyse vun de Qualität vun de Erjebnisse kanns de Erjebnisse plotte, indem de de Kostenweerte vun de Bitstrings un ehre Wahrscheinlichkeite berächnes. Zem Verglisch plott de Erjebnisse nebben en Verteilung vun zufällich jesamplete Bitstrings, wat glich ener "brute-force" klassische Lösung es. Wann dä Algorithmus konsequent niedrijere Koste fingk, düttet et drop hen, dat dä Quantenalgorithmus dat Optimierungsproblem effektiv löst.

```python
def plot_cost_histogram(
    costs, probabilities, distribution, qubit_count, bitstring_cost
```

## Wos des Projekt dazuafügt {#what-this-project-adds}

IBM ihre [Qiskit-Dokumentation](https://github.com/Qiskit/documentation) is open source (CC BY-SA 4.0), oba ihre Webanwendung is's net. doQumentation fügt a open-source Frontend mit Live-Code-Ausführung, automatischer Credential-Injektion und Simulator-Modus dazua.

Überoll einsetzbar — von [GitHub Pages](https://doqumentation.org) üba [Docker](https://github.com/JanLahmann/doQumentation/pkgs/container/doqumentation) bis zu [RasQberry](https://rasqberry.org/). Schaugt's eich [olle Funktionen](/features) o.

## Code-Ausführung {#code-execution}

Klickt's bei jedn Codeblock auf **Run**. Da erste Klick startet an kostenlosen Jupyter-Kernel üba [Binder](https://mybinder.org) (1–2 Min.). Donoch geht's sofort.

- **Simulator-Modus (koa Konto notwendig)** — Aktiviert's den [Simulator-Modus](/jupyter-settings#simulator-mode), damit ihr olle Notebooks mit AerSimulator oda FakeBackends mit Rauschmodell ausführn kennt's. Null Einrichtungsaufwand notwendig.

- **IBM Quantum Hardware** — Gebt's euern API-Token und CRN oanmol in de [Einstellungen](/jupyter-settings#ibm-quantum) ei — de Zugangsdaten werdn bei jeda Ausführung automatisch eigfügt. [Erstellt's a kostenloses Konto](https://quantum.cloud.ibm.com/registration), falls ihr no koans habt's.

<details>
<summary>Vafügbare Ausführungs-Backends</summary>

Jedes Tutorial hot ausführbare Codeblöcke. Klickt's auf **Run**, damit ihr's mit oam von drei Backends ausführn kennt's:

1. **Binder** (Standard auf GitHub Pages) — Kostenlosen Remote-Jupyter-Kernel üba [mybinder.org](https://mybinder.org)
2. **Lokales Jupyter (Docker / RasQberry)** — Vabindet's eich mitn lokalen Jupyter-Server mit vorinstalliertem Qiskit
3. **Benutzerdefinierter Server** — Vaweist's in de [Einstellungen](/jupyter-settings#advanced) auf an beliebigen Jupyter-Endpunkt

</details>

<details>
<summary>Bereitstellungsoptionen</summary>

doQumentation is vafügbar ois:

- **[GitHub Pages](https://doqumentation.org)** — Statische Website mit [Binder](https://mybinder.org) für Remote-Code-Ausführung
- **[Docker](https://github.com/JanLahmann/doQumentation/pkgs/container/doqumentation)** — Vollständiger Stack lokal, offline-fähig
- **[RasQberry](https://rasqberry.org/)** — Söbst g'hostet auf am Raspberry Pi mit lokalem Jupyter-Kernel

</details>

<details>
<summary>Lokal ausführn mit Podman / Docker</summary>

Installiert's [Podman Desktop](https://podman-desktop.io/) oda [Docker Desktop](https://www.docker.com/products/docker-desktop/) (Mac / Windows / Linux), donn führt's aus:

```bash
# Vollständiger Stack: Website + Jupyter + Qiskit (~3 GB)
podman run -p 8080:80 -p 8888:8888 ghcr.io/janlahmann/doqumentation:jupyter
```

Mocht's [http://localhost:8080](http://localhost:8080) auf — Code-Ausführung funktioniert lokal, kane Binder-Wartezeit.

Für a leichtgwichtige Variante ohne lokale Code-Ausführung (~60 MB):

```bash
# Nur statische Website — Code-Ausführung funktioniert weiterhin üba Binder
podman run -p 8080:80 ghcr.io/janlahmann/doqumentation:latest
```

Benutzt's ihr Docker? Ersetzt's anfoch `podman` durch `docker` — de Befehle san identisch. De Images san Multi-Arch (`linux/amd64` + `linux/arm64`), sodass Apple Silicon Macs nativ funktionieren.

</details>

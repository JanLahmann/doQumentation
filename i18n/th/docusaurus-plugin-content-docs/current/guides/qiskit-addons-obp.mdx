# Operator backpropagation (OBP)

Operator backpropagation (OBP) คือเทคนิคสำหรับลดความลึกของ Circuit โดยการตัดการดำเนินการออกจากด้านท้ายของ Circuit แลกกับการวัด operator เพิ่มขึ้น มีหลายวิธีในการทำ operator backpropagation และแพ็กเกจนี้ใช้วิธีที่อิงจาก Clifford perturbation theory [[1]](#references)

เมื่อ propagate operator ผ่าน Circuit ไปไกลขึ้น ขนาดของ observable ที่ต้องวัดจะเติบโตแบบ exponential ส่งผลให้เกิดค่าใช้จ่ายด้าน resource ทั้งแบบ classical และ quantum อย่างไรก็ตาม สำหรับบาง Circuit การกระจายของ Pauli observable เพิ่มเติมที่ได้มีความกระจุกตัวมากกว่า worst-case exponential scaling ซึ่งหมายความว่าบาง term ใน observable ที่มี coefficient เล็กน้อยสามารถถูก truncate ออกเพื่อลดค่าใช้จ่ายด้าน quantum ได้ error ที่เกิดจากการทำเช่นนี้สามารถควบคุมได้เพื่อหาจุดสมดุลที่เหมาะสมระหว่าง precision และประสิทธิภาพ

## การติดตั้ง {#installation}

ติดตั้งแพ็กเกจ OBP ได้สองวิธี: ผ่าน PyPI หรือ build จาก source แนะนำให้ติดตั้งแพ็กเกจเหล่านี้ใน [virtual environment](https://docs.python.org/3.10/tutorial/venv.html) เพื่อแยก dependency ของแพ็กเกจออกจากกัน

### ติดตั้งจาก PyPI {#install-from-pypi}

วิธีที่ตรงไปตรงมาที่สุดในการติดตั้งแพ็กเกจ `qiskit-addon-obp` คือผ่าน PyPI

```bash
pip install qiskit-addon-obp
```

### Build จาก source {#build-from-source}

ผู้ใช้ที่ต้องการมีส่วนร่วมใน package นี้หรือต้องการติดตั้งด้วยตัวเองสามารถทำได้โดยการ clone repository ก่อน:

```bash
git clone git@github.com:Qiskit/qiskit-addon-obp.git
```_

แล้วติดตั้งแพ็กเกจผ่าน `pip` repository ยังมี example notebook ด้วย หากวางแผนจะพัฒนาใน repository ให้ติดตั้ง dependency แบบ `dev` ด้วย

ปรับ option ตามต้องการ:

```bash
pip install tox notebook -e '.[notebook-dependencies, dev]'
```

## พื้นฐานทางทฤษฎี {#theoretical-background}

กระบวนการ OBP ที่ implement ในแพ็กเกจนี้อธิบายไว้ในรายละเอียดใน [[1]](#references) เมื่อใช้ Estimator primitive ผลลัพธ์ของ quantum workload คือการประมาณค่า expectation value หนึ่งค่าหรือมากกว่า $\langle O \rangle$ สำหรับ state ที่เตรียมโดย QPU ส่วนนี้สรุปขั้นตอนของกระบวนการ

เริ่มต้นด้วยการเขียน expectation value measurement ของ observable $O$ ในรูปของ initial state $|\psi\rangle$ และ quantum circuit $U_Q$:

$$ \langle O \rangle_{U|\psi\rangle} = \langle\psi | U^\dagger O U |\psi \rangle. $$

เพื่อกระจายปัญหานี้ไปยัง resource ทั้งแบบ classical และ quantum ให้แบ่ง Circuit $U$ ออกเป็นสอง subcircuit คือ $U_C$ และ $U_Q$ จากนั้นจำลอง Circuit $U_C$ แบบ classical แล้วรัน Circuit $U_Q$ บน quantum hardware และใช้ผลของการจำลอง classical เพื่อสร้าง observable $O$ ขึ้นมาใหม่

![แผนภาพ OBP แสดงการแบ่ง circuit เป็นสอง subcircuit จำลอง subcircuit หนึ่งแบบ classical แล้ววัดอีก circuit หนึ่งด้วย quantum hardware](/docs/images/guides/qiskit-addons/obp-diagram.avif)

ควรเลือก subcircuit $U_C$ ให้สามารถจำลองได้แบบ classical ซึ่งจะคำนวณ expectation value

$$ \langle O' \rangle \equiv U_C^\dagger O U_C, $$

ซึ่งเป็น operator เริ่มต้น $O$ ที่ถูก evolve ผ่าน Circuit $U_C$ เมื่อได้ $O'$ แล้ว quantum workload จะถูกเตรียมโดยเริ่มต้น state $|\psi\rangle$ ใช้ Circuit $U_Q$ กับมัน แล้ววัด expectation value $O'$ สามารถแสดงให้เห็นว่านี่เทียบเท่ากับการวัด $\langle O \rangle$ โดยเขียนว่า:

$$ \langle \psi | U_Q^\dagger O' U_Q \psi \rangle = \langle \psi | U_Q^\dagger U_C^\dagger O U_CU_Q \psi \rangle = \langle\psi | U^\dagger O U |\psi \rangle = \langle O \rangle_{U|\psi\rangle}$$

สุดท้าย เพื่อวัด expectation value $\langle O' \rangle$ จำเป็นต้องกำหนดให้สามารถ decompose ออกเป็นผลรวมของ Pauli string ได้

$$ O' = \sum_P c_P P, $$

โดยที่ $c_P$ คือ coefficient จริงของการ decompose และ $P$ คือ Pauli string ที่ประกอบด้วย operator $I$, $X$, $Y$ และ $Z$ ซึ่งช่วยให้สามารถสร้าง expectation value ของ $O$ ขึ้นมาใหม่ได้โดย

$$ \langle \psi | U_Q^\dagger O' |\psi \rangle = \sum_P c_P \langle \psi | U_Q^\dagger P U_Q | \psi \rangle. $$

### การ truncate term {#truncating-terms}

แนวทางนี้เสนอการแลกเปลี่ยนระหว่างความลึกของ Circuit $U_Q$ ที่ต้องการ จำนวนการรัน Circuit บน quantum hardware และปริมาณ resource classical ที่จำเป็นสำหรับการคำนวณ $O'$ โดยทั่วไป เมื่อเลือก backpropagate ผ่าน Circuit ไปไกลขึ้น จำนวน Pauli string ที่ต้องวัดรวมถึงค่าใช้จ่ายด้าน error-mitigation ก็จะเติบโตแบบ exponential (พร้อมกับ resource classical ที่ต้องการสำหรับการจำลอง $U_C$)

โชคดีที่ การ decompose ของ $O'$ มักมี coefficient ขนาดเล็กที่สามารถถูก truncate ออกจากการวัดสุดท้ายที่ใช้สร้าง $O$ ขึ้นมาใหม่ได้โดยไม่เกิด error มากนัก แพ็กเกจ `qiskit-addon-obp` มีฟังก์ชันสำหรับระบุ error budget ซึ่งสามารถค้นหา term ที่สามารถ truncate ได้โดยอัตโนมัติ ภายในค่า error tolerance ที่กำหนด

### Clifford perturbation theory

แพ็กเกจ `qiskit-addon-obp` เข้าถึง operator backpropagation โดยอิงจาก Clifford perturbation theory วิธีนี้มีข้อดีตรงที่ค่าใช้จ่ายที่เกิดจากการ backpropagate Gate ต่างๆ จะ scale ตาม non-Cliffordness ของ $U_C$ (กล่าวคือ ส่วนที่ประกอบด้วย non-Clifford instruction ใน U C)

แนวทาง OBP นี้เริ่มต้นด้วยการแบ่ง Circuit ที่จำลอง $U_C$ ออกเป็น *slice*:

$$ U_C = \prod_{s=1}^S \mathcal{U}_s = \mathcal{U}_S...\mathcal{U}_2\mathcal{U}_1, $$

โดยที่ $S$ แทนจำนวน slice ทั้งหมด และ $\mathcal{U}_s$ หมายถึง slice เดียวของ Circuit $U_C$ แต่ละ slice เหล่านี้จะถูกนำไปใช้ในลำดับแบบ analytic เพื่อวัด operator $O'$ ที่ถูก back propagate และอาจส่งผลหรือไม่ส่งผลต่อขนาดรวมของผลรวม ขึ้นอยู่กับว่า slice นั้นเป็น Clifford หรือ non-Clifford operation หาก [error budget](../api/qiskit-addon-obp/utils-truncating#setup_budget) ถูกจัดสรรไว้ การ truncation จะเกิดขึ้นระหว่างการใช้แต่ละ slice

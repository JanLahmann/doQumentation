# คำถามที่พบบ่อย (FAQ) {#frequently-asked-questions-faq}

สำหรับ FAQ เฉพาะเกี่ยวกับ execution modes ดูคู่มือ [Qiskit Runtime execution modes FAQ](/guides/execution-modes-faq)

## จะอ้างอิง Qiskit ในงานวิจัยอย่างไร? {#how-do-i-cite-qiskit-in-my-research}

อ้างอิง Qiskit โดยใช้ [ไฟล์ BibTeX](https://github.com/Qiskit/qiskit/blob/main/CITATION.bib) ที่รวมมาด้วย

## จะอ้างอิงคอมพิวเตอร์ IBM Quantum ในงานวิจัยอย่างไร? {#how-do-i-cite-an-ibm-quantum-computer-in-my-research}

สำหรับบทความวิจัย เราส่งเสริมให้ผู้เขียนระบุขอบคุณ IBM Quantum&reg; โดยใช้:

>    We acknowledge the use of IBM Quantum services for this work. The views expressed are those of the authors, and do not reflect the official policy or position of IBM or the IBM Quantum team.

การอ้างอิงในบทความควรใช้การอ้างอิงต่อไปนี้:

> IBM Quantum. https://<span>quantum.cloud.ibm.com</span>/, YYYY

Quantum processing units ในบทความควรอ้างอิงด้วยชื่อเฉพาะ (เช่น `ibmq_vigo`) และอาจเพิ่มเวอร์ชัน (เช่น `ibmq_vigo` v1.0.2) เราส่งเสริมให้อ้างอิง processor ด้วย
ตัวอย่างเช่น:

> In this paper we used `ibmq_vigo`, which is one of the IBM Quantum Canary processors.

ตัวอย่างการอ้างอิงโปรแกรม IBM Quantum:

> IBM Quantum (2025). Estimator primitive (Version x.y.z) [computer software]. https://<span>quantum.cloud.ibm.com</span>/

## จะอ้างอิง IBM Quantum Composer ในงานวิจัยอย่างไร? {#how-do-i-cite-ibm-quantum-composer-in-my-research}

ตัวอย่างการอ้างอิง IBM Quantum Composer:

> IBM Quantum Composer. 2025. url: https://<span>quantum.cloud.ibm.com/composer</span>

## ฉันไม่เห็นสิ่งที่ต้องการใน user interface เช่น งานบริหารของฉัน {#i-dont-see-something-in-the-user-interface-such-as-my-administration-tasks}

ลองเลือก account อื่นจาก account switcher ในส่วนหัวของ[IBM Quantum Platform](https://quantum.cloud.ibm.com)

## ฉันใช้แค่ Open Plan แต่ต้องกรอกข้อมูลบัตรเครดิต จะถูกเรียกเก็บเงินไหม? {#im-just-using-the-open-plan-but-i-had-to-put-in-my-credit-card-will-i-be-charged}

จะไม่ถูกเรียกเก็บเงิน เว้นแต่จะสร้าง Pay-As-You-Go [instance](/guides/instances) และใช้เพื่อส่ง workload ไปยัง QPU ดู[จัดการค่าใช้จ่าย](/guides/manage-cost)สำหรับคำแนะนำในการตั้งค่าขีดจำกัดค่าใช้จ่ายและวิธีอื่น ๆ ในการจัดการค่าใช้จ่าย

## API key กับ API token ต่างกันอย่างไร? {#whats-the-difference-between-an-api-key-and-an-api-token}

ไม่ต่างกัน คำว่า API key และ API token ใช้แทนกันได้ คุณจะเห็นทั้งสองคำในเอกสารเพราะ interface ต่าง ๆ ใช้คำศัพท์ต่างกัน

<span id="bearer"></span>
## API token กับ bearer token ต่างกันอย่างไร? {#whats-the-difference-between-an-api-token-and-a-bearer-token}

API token คือ credential ที่ใช้งานได้นาน สำหรับ[การยืนยันตัวตนกับ Qiskit Runtime service](/guides/hello-world#install-and-authenticate) ส่วน bearer token คือ credential ชั่วคราวที่ใช้[ยืนยันตัวตนกับ REST API ในสภาพแวดล้อมที่ไม่น่าเชื่อถือ](/guides/cloud-setup-untrusted#authenticate-to-the-rest-api-in-an-untrusted-environment) bearer token ไม่สามารถใช้ยืนยันตัวตนกับ Qiskit Runtime service ได้

## ต้องเลือก region ที่ตัวเองอยู่ไหม? {#do-i-have-to-choose-the-region-that-i-live-in}

ไม่ต้อง สามารถใช้ region ใดก็ได้ region จะกำหนดว่า job ทำงานที่ไหนและข้อมูลเก็บที่ไหน นอกจากนี้คุณสามารถเห็นและใช้เฉพาะ instance ที่สร้างใน instance ที่คุณ login อยู่เท่านั้น โปรดทราบว่าเฉพาะ region us-east เท่านั้นที่รองรับ Open Plan

## สามารถเชื่อมต่อผ่าน proxy server ได้ไหม? {#can-i-connect-through-a-proxy-server}

ได้ แต่ต้องใช้ Qiskit Runtime v0.44.0 หรือใหม่กว่า

## นโยบายการเก็บรักษาข้อมูลเป็นอย่างไร? {#what-is-the-data-retention-policy}

IBM เก็บรักษาข้อมูล job และการใช้งาน (ที่ยังไม่ถูกลบ) ของคุณเป็นเวลาสามปีตราบเท่าที่ยังใช้ Qiskit Runtime service อยู่

## เกิดอะไรขึ้นกับ Python distributions เช่น Qiskit Aer, Qiskit Nature และ Qiskit Experiments? {#what-happened-to-python-distributions-such-as-qiskit-aer-qiskit-nature-and-qiskit-experiments}

ตั้งแต่การเปิดตัว Qiskit SDK v1.0 คำสั่ง `pip install qiskit` จะ**ติดตั้งเฉพาะ** [Qiskit SDK](https://github.com/Qiskit/qiskit) เท่านั้น ซึ่งหมายความว่า module บางอย่างที่เคยใช้มาก่อน เช่น `qiskit-aer` หรือ `qiskit-nature` ต้องติดตั้งแยกต่างหากแล้ว

ก่อนการเปิดตัว Qiskit SDK v1.0 Qiskit ประกอบด้วย Python distributions หลายตัว เช่น `qiskit-terra` ซึ่งเป็น compiler core; `qiskit-aer` ซึ่งเป็น high-performance simulator; IBM Quantum&reg; provider ดั้งเดิม; และ package อื่น ๆ ที่ล้าสมัยแล้วซึ่งให้คุณสมบัติ algorithmic หรือการรันการทดลองเฉพาะทาง

software package แยกต่าง ๆ เหล่านี้ถูกรวมเข้าเป็น Qiskit *metapackage* ซึ่งอนุญาตให้ผู้ใช้ติดตั้ง component ทั้งหมดผ่าน `pip install qiskit` สถาปัตยกรรม metapackage นี้ถูกลบออกพร้อมการเปิดตัว Qiskit SDK v1.0 และตอนนี้ software distribution แต่ละตัวต้องติดตั้งแยกกัน

หากต้องการติดตั้ง package ที่เคยรวมอยู่ใน Qiskit metapackage ให้เยี่ยมชม [Qiskit ecosystem](https://www.ibm.com/quantum/ecosystem) เพื่อค้นหา package ที่เหมาะกับความต้องการ นอกจากนี้ยังสามารถอ่าน[คู่มือการย้าย v1.0](/guides/metapackage-migration) สำหรับข้อมูลเพิ่มเติมเกี่ยวกับการเปลี่ยนแปลงนี้

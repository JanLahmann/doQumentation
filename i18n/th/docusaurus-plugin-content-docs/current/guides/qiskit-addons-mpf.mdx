Multi-product formulas (MPF) สามารถใช้จำลองไดนามิกส์ของระบบควอนตัมได้แม่นยำยิ่งขึ้น แลกกับการรัน Circuit ที่เพิ่มขึ้น นี่คือเทคนิคการประมวลผลหลัง (post-processing) ที่ช่วยลด error ของค่า expectation values สำหรับสถานะที่วิวัฒนาการตามเวลา

การคำนวณแบบ classical ถูกนำมาใช้แก้ระบบสมการเชิงเส้นที่ให้ coefficients สำหรับการรวมแบบ weighted ของการรัน Circuit หลายครั้ง การใช้การรวมแบบ weighted นี้ช่วยลด error ที่เกิดจากการจำลองการวิวัฒนาการตามเวลา โดยต้องเลือก Trotter steps อย่างเหมาะสม เครื่องมือ MPF จะรับข้อมูล ได้แก่ จำนวน Trotter steps และ order ของการประมาณ Trotter เพื่อเตรียมและแก้ (หรือประมาณค่า) ระบบสมการเชิงเส้นที่เกี่ยวข้อง ซึ่งคุณสามารถนำไปใช้ประมวลผลค่า expectation values ของสถานะที่วิวัฒนาการตามเวลาต่อไปได้
## ติดตั้งแพ็กเกจ MPF {#install-the-mpf-package}

มีสองวิธีในการติดตั้งแพ็กเกจ MPF: ผ่าน PyPI และการ build จาก source แนะนำให้ติดตั้งใน[สภาพแวดล้อมเสมือน](https://docs.python.org/3.10/tutorial/venv.html) เพื่อให้แน่ใจว่าแพ็กเกจ dependencies แยกออกจากกัน

### ติดตั้งจาก PyPI {#install-from-pypi}

วิธีที่ตรงไปตรงมาที่สุดในการติดตั้งแพ็กเกจ `qiskit-addon-mpf` คือผ่าน PyPI
```bash
pip install qiskit-addon-mpf
```

### Build จาก source {#build-from-source}
ผู้ใช้ที่ต้องการพัฒนาใน repository หรือต้องการติดตั้งด้วยตัวเองสามารถทำได้โดยเริ่มจากการ clone repository:
```bash
git clone git@github.com:Qiskit/qiskit-addon-mpf.git
```

และติดตั้งแพ็กเกจผ่าน `pip` repository ยังมี optional dependencies หลายตัวที่เปิดใช้งานฟีเจอร์บางอย่างด้วย

ปรับตัวเลือกตามความต้องการ
```bash
pip install tox notebook -e '.[notebook-dependencies,dev]'
```
## พื้นฐานทางทฤษฎี {#theoretical-background}

MPF สามารถลด Trotter approximation error ที่เกิดจากการจำลองไดนามิกส์ของระบบควอนตัมได้ผ่านการรวมแบบ weighted ของการรัน Circuit หลายครั้ง weighted sum นี้นิยามเป็น:

$$ \mu(t) = \sum_j x_j\rho_j^{k_j}\left(\frac{t}{k_j}\right) + \text{some remaining Trotter error}, $$

โดยที่ $x_j$ คือ weighting coefficients, $\rho_j^{k_j}$ คือ density matrix ที่สอดคล้องกับ pure state ที่ได้จากการ evolve สถานะเริ่มต้นผ่าน product formula $S^{k_j}$ โดยประมาณ time-evolution operator ด้วย $k_j$ Trotter steps และ $j$ เป็น index ของแต่ละ product formula ที่ใช้ในผลรวม

![Circuit หลายตัวที่มีจำนวน Trotter steps ต่างกันถูกใช้คำนวณ observable เป้าหมาย](/docs/images/guides/qiskit-addons/mpf-diagram.avif)

โดยทั่วไปแล้ว เป้าหมายของการจำลองไดนามิกส์ควอนตัมคือการวัด observable บางตัว $\mathcal{O}(t)$ ซึ่งเป็นฟังก์ชันของเวลา เมื่อใช้ MPF จะรัน Circuit หลายตัว โดยแต่ละตัวใช้ $k_j$ Trotter steps เพื่อได้การวัด observable เป้าหมาย $\mathcal{O}_{k_j}(t)$ หลายครั้ง จากนั้นได้การวัด observable เป้าหมายโดยการคำนวณ:

$$ \langle \mathcal{O}(t) \rangle = \sum_j x_j(t) \langle \mathcal{O}_{k_j}(t) \rangle. $$

โดยพื้นฐานแล้ว คุณสามารถลด Trotter error รวมได้โดยการประมาณ time-evolution operator ด้วย product formulas หลายตัวที่มีจำนวน Trotter steps ต่างกัน แทนที่จะใช้ product formula เพียงตัวเดียว คุณสร้าง Circuit สำหรับแต่ละ term ใน weighted sum ซึ่ง evolve ระบบตามจำนวน $k_j$ Trotter steps แต่ละ Circuit จะถูกรันแยกกันบน QPU เพื่อสร้างผลลัพธ์ใหม่ในขั้นตอนการประมวลผลหลัง ประโยชน์ของเทคนิคนี้มองได้จากสองมุมมอง:
1. สำหรับจำนวน Trotter steps ที่กำหนดไว้ที่รัน คุณสามารถได้ผลลัพธ์ที่มี Trotter error รวมน้อยลง
2. สำหรับจำนวน Trotter steps ที่ส่งผลให้ Circuit ลึกมาก คุณสามารถใช้ MPF เพื่อหา Circuit ที่มีความลึกสั้นกว่าหลายตัวมาแทน ซึ่งให้ Trotter approximation error ที่ใกล้เคียงกัน

## กำหนด MPF coefficients {#determine-mpf-coefficients}

ฟังก์ชันหลักของแพ็กเกจ `qiskit-addon-mpf` อยู่ที่การกำหนด MPF coefficients $x_j(t)$ (ซึ่งอาจขึ้นกับเวลา) กระบวนการได้แต่ละ $x_j(t)$ เกี่ยวข้องกับการแก้ระบบสมการเชิงเส้น $Ax=b$ โดยที่ $x$ คือเวกเตอร์ของ coefficients ที่ต้องการหา $A$ คือเมทริกซ์ที่ขึ้นอยู่กับค่า $k_j$ และ product formula $S$ ที่ใช้ (ได้แก่ approximation order และจำนวน Trotter steps) และ $b$ คือเวกเตอร์ของเงื่อนไขบังคับ ระบบสมการนี้สามารถแก้ได้แบบ exact หรือด้วยโมเดลประมาณที่ minimize 1-norm ของ coefficients นอกจากนี้ การเลือกแต่ละ $k_j$ เป็นกระบวนการ heuristic แต่สามารถกำหนดขอบเขตด้วยเงื่อนไขต่อไปนี้:

1. ค่า $k_j$ ที่**ใหญ่ที่สุด**ถูกจำกัดโดยความลึกสูงสุดของ Circuit ที่สามารถรันได้อย่างน่าเชื่อถือ
2. $k_j$ ที่**เล็กที่สุด**ควรตอบสนอง $dt = t/k_j < 1$ เนื่องจากนั่นคือจุดที่ Trotter error มีพฤติกรรมที่ดีที่สุด
3. ไม่มี coefficients $x_j$ ตัวใดควรใกล้ $0$ เพราะนั่นหมายความว่ามันไม่มีส่วนสำคัญต่อ MPF
4. ในทำนองเดียวกัน coefficient ที่เกี่ยวข้องกับค่า $k_j$ ที่ใหญ่ที่สุดไม่ควรครอบงำ เพราะนั่นหมายความว่าคุณกำลังใช้ product formula เพียงตัวเดียว
5. สุดท้าย norm ของ coefficients $x_j$ ที่ได้ควรมีค่าน้อย เพราะนี่บ่งชี้ว่า MPF มีเงื่อนไขที่ดี [1](#references)

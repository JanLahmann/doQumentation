# เริ่มต้นใช้งาน backend primitives {#get-started-with-the-backend-primitives}

ต่างจาก primitives เฉพาะ provider, backend primitives คือ implementation ทั่วไปที่สามารถใช้กับ
`backend` object ใดก็ได้ ตราบใดที่ implement interface [`BackendV2`](https://docs.quantum.ibm.com/api/qiskit/qiskit.providers.BackendV2) บาง provider implement primitives ไว้ในตัว ดูรายละเอียดที่ [Qiskit Ecosystem page](https://www.ibm.com/quantum/ecosystem?tag=provider)

<span id="estimator"></span>
## เริ่มต้นใช้งาน Estimator backend primitive {#get-started-with-the-estimator-backend-primitive}

Estimator primitive สามารถรันกับ provider ใดก็ได้โดยใช้คลาส [`qiskit.primitives.BackendEstimatorV2`](https://docs.quantum.ibm.com/api/qiskit/qiskit.primitives.BackendEstimatorV2) อย่างไรก็ตาม ไม่มี measurement หรือ gate error mitigation implementations แบบ "สำเร็จรูป" เนื่องจาก backend primitives ออกแบบมาให้รันในเครื่องของผู้ใช้

### ตัวอย่าง: {#example}

```python
from qiskit.primitives import BackendEstimatorV2
from <some_qiskit_provider> import QiskitProvider

provider = QiskitProvider()
backend = provider.get_backend('backend_name')
estimator = BackendEstimatorV2(backend)
```

<span id="sampler"></span>
## เริ่มต้นใช้งาน Sampler backend primitive {#get-started-with-the-sampler-backend-primitive}

Sampler primitive สามารถรันกับ provider ใดก็ได้โดยใช้ [`qiskit.primitives.BackendSamplerV2`](https://docs.quantum.ibm.com/api/qiskit/qiskit.primitives.BackendSamplerV2) อย่างไรก็ตาม ต้องใช้ backend ที่รองรับตัวเลือก `memory`

### ตัวอย่าง: {#example}

```python
from qiskit.primitives import BackendSamplerV2
from <some_qiskit_provider> import QiskitProvider

provider = QiskitProvider()
backend = provider.get_backend('backend_name')
sampler = BackendSamplerV2(backend)
```

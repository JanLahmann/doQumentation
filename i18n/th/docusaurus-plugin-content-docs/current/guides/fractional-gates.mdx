## วิธีใช้ fractional gates {#how-to-use-fractional-gates}

ภายใน fractional gate เหล่านี้ทำงานโดยดำเนินการหมุน $R_{ZZ}(\theta)$ และ $R_X(\theta)$ โดยตรงสำหรับมุมใด ๆ การใช้ $R_X(\theta)$ gate สามารถลดระยะเวลาและ error สำหรับการหมุน single-qubit ของมุมใด ๆ ได้สูงสุดถึงสองเท่า การดำเนินการหมุน $R_{ZZ}(\theta)$ gate โดยตรงหลีกเลี่ยงการแยกส่วนเป็นหลาย `CZGate` ซึ่งลดระยะเวลาและ error ของ Circuit เช่นกัน สิ่งนี้มีประโยชน์อย่างยิ่งสำหรับ Circuit ที่มีการหมุน single- และ two-qubit จำนวนมาก เช่น เมื่อจำลอง dynamics ของระบบควอนตัมหรือเมื่อใช้ variational ansatz ที่มีหลาย parameter

ในขณะที่ gate ประเภทเหล่านี้อยู่ใน[ไลบรารี standard gate](./circuit-library) ที่ `QuantumCircuit` สามารถมีได้ แต่สามารถใช้บน IBM Quantum QPU เฉพาะบางตัวเท่านั้น และต้องโหลดด้วย flag `use_fractional_gates` ตั้งค่าเป็น `True` (แสดงด้านล่าง) flag นี้จะรวม fractional gate ไว้ใน `Target` ของ Backend สำหรับ Transpiler

```python
service = QiskitRuntimeService()
backend = service.backend('ibm_torino', use_fractional_gates=True)
```

ตัวอย่างโค้ดนี้แสดงวิธีใช้ fractional gate ในบริบทของ workflow ที่จำลอง dynamics ของ Ising chain โดยใช้ fractional gate จากนั้นเปรียบเทียบระยะเวลา Circuit กับ Backend ที่ไม่ใช้ fractional gate

<Admonition title="หมายเหตุเกี่ยวกับอัตรา error ที่รายงาน" type="note">
  ค่า error ที่รายงานใน `Target` ของ Backend ที่เปิดใช้ fractional gate เป็นเพียงสำเนาของคู่หูที่ไม่ใช่ fractional gate (ซึ่งอาจไม่เหมือนกัน) เนื่องจากยังไม่รองรับการรายงานอัตรา error บน fractional gate

  อย่างไรก็ตาม เนื่องจาก gate time ของ fractional gate และ non-fractional gate เท่ากัน จึงสมเหตุสมผลที่จะสมมติว่าอัตรา error ของทั้งสองเทียบเคียงได้ โดยเฉพาะเมื่อแหล่ง error หลักใน Circuit เกิดจาก relaxation
</Admonition>

```python
from qiskit import QuantumCircuit
from qiskit.circuit import Parameter
from qiskit.transpiler import generate_preset_pass_manager
from qiskit.visualization.timeline import draw as draw_timeline, IQXSimple

from qiskit_ibm_runtime import QiskitRuntimeService

num_qubits = 5
num_time_steps = 3
rx_angle = 0.1
rzz_angle = 0.1

ising_circuit = QuantumCircuit(num_qubits)
for i in range(num_time_steps):
    # rx layer
    for q in range(num_qubits):
        ising_circuit.rx(rx_angle, q)
    for q in range(1, num_qubits - 1, 2):
        ising_circuit.rzz(rzz_angle, q, q + 1)
    # 2nd rzz layer
    for q in range(0, num_qubits - 1, 2):
        ising_circuit.rzz(rzz_angle, q, q + 1)
    ising_circuit.barrier()
ising_circuit.draw("mpl")
```

กำหนด Backend object สองตัว: ตัวหนึ่งเปิดใช้ fractional gate และอีกตัวปิดใช้ จากนั้น transpile ทั้งสองตัว

```python
service = QiskitRuntimeService()
backend_fractional = service.backend("ibm_torino", use_fractional_gates=True)
backend_conventional = service.backend(
    "ibm_torino", use_fractional_gates=False
)

pm_fractional = generate_preset_pass_manager(
    optimization_level=3, backend=backend_fractional, scheduling_method="alap"
)
pm_conventional = generate_preset_pass_manager(
    optimization_level=3,
    backend=backend_conventional,
    scheduling_method="alap",
)

ising_circuit_fractional = pm_fractional.run(ising_circuit)
ising_circuit_conventional = pm_conventional.run(ising_circuit)
```

แสดง timeline ของ Circuit โดยใช้ gate สองประเภท

```python
)
```

### ข้อจำกัดของมุม {#angle-constraints}

สำหรับ two-qubit $R_{ZZ}(\theta)$ gate สามารถดำเนินการได้เฉพาะมุมระหว่าง $0$ ถึง $\pi/2$ บน IBM Quantum hardware หาก Circuit มี $R_{ZZ}(\theta)$ gate ใด ๆ ที่มีมุมอยู่นอกช่วงนี้ pipeline transpilation มาตรฐานโดยทั่วไปจะแก้ไขด้วยการแปลง Circuit ที่เหมาะสม (ผ่าน pass [`FoldRzzAngle`](../api/qiskit-ibm-runtime/transpiler-passes-fold-rzz-angle)) อย่างไรก็ตาม สำหรับ $R_{ZZ}(\theta)$ gate ใด ๆ ที่มี [`Parameter`](../api/qiskit/qiskit.circuit.Parameter) หนึ่งตัวหรือมากกว่า Transpiler จะสมมติว่า parameter เหล่านี้จะถูกกำหนดมุมภายในช่วงนี้ที่ runtime Job จะล้มเหลวหากค่า parameter ใด ๆ ที่ระบุใน PUB ที่ส่งไปยัง Qiskit Runtime อยู่นอกช่วงนี้
## ควรใช้ fractional gates ที่ไหน {#where-to-use-fractional-gates}

ในอดีต basis gate ที่มีบน IBM Quantum QPU คือ **`CZ`**, **`X`**, **`RZ`**, **`SX`** และ **`ID`** ซึ่งไม่สามารถแสดง Circuit ที่มีการหมุน single- และ two-qubit ที่ไม่ใช่ทวีคูณของ $\pi / 2$ ได้อย่างมีประสิทธิภาพ ตัวอย่างเช่น $R_X(\theta)$ gate เมื่อ transpile ต้องแยกส่วนเป็นชุดของ $RZ$ และ $\sqrt{X}$ gate ซึ่งสร้าง Circuit ที่มีสอง gate ที่มีระยะเวลาจำกัดแทนที่จะเป็นหนึ่งตัว

ในทำนองเดียวกัน เมื่อ two-qubit rotation เช่น $R_{ZZ}(\theta)$ gate ถูก transpile การแยกส่วนต้องใช้ **`CZ`** gate สองตัวและ single-qubit gate หลายตัว ซึ่งเพิ่ม circuit depth การแยกส่วนเหล่านี้แสดงในโค้ดต่อไปนี้

```python
qc = QuantumCircuit(1)
param = Parameter("θ")
qc.rx(param, 0)
qc.draw("mpl")
```

สำหรับ workflow ที่ต้องการการหมุน $R_X(\theta)$ single-qubit หรือ two-qubit จำนวนมาก (เช่น ใน variational ansatz หรือเมื่อจำลอง time evolution ของระบบควอนตัม) ข้อจำกัดนี้ทำให้ circuit depth เติบโตอย่างรวดเร็ว อย่างไรก็ตาม fractional gate ลบข้อกำหนดนี้ เนื่องจากการหมุน single- และ two-qubit ถูกดำเนินการโดยตรง และสร้าง quantum circuit ที่มีประสิทธิภาพมากขึ้น (และด้วยเหตุนี้จึงลด error ได้)

## เมื่อ*ไม่ควร*ใช้ fractional gates {#when-not-to-use}

สิ่งสำคัญที่ต้องทราบคือ fractional gate เป็นคุณสมบัติ*เชิงทดลอง* และพฤติกรรมของ flag `use_fractional_gates` อาจเปลี่ยนแปลงในอนาคต ดู[release notes](https://docs.quantum.ibm.com/api/qiskit-ibm-runtime/release-notes) สำหรับเวอร์ชันใหม่ของ Qiskit Runtime เพื่อข้อมูลเพิ่มเติม ดูเอกสาร API reference สำหรับ [QiskitRuntimeService.backend](https://docs.quantum.ibm.com/api/qiskit-ibm-runtime/qiskit-runtime-service#backend) ซึ่งอธิบาย `use_fractional_gates` ด้วย

นอกจากนี้ Qiskit Transpiler มีความสามารถจำกัดในการใช้ $R_{ZZ}(\theta)$ ใน optimization pass ซึ่งต้องระมัดระวังมากขึ้นในการสร้างและปรับ Circuit ที่มีคำสั่งเหล่านี้

- [Probabilistic error cancellation](/guides/error-mitigation-and-suppression-techniques#probabilistic-error-cancellation-pec)
- [Zero-noise extrapolation (using probabilistic error amplification)](/guides/error-mitigation-and-suppression-techniques#probabilistic-error-amplification-pea)

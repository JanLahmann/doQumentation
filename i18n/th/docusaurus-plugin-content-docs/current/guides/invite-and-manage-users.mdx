# เชิญผู้ใช้และจัดการการเข้าถึง {#invite-users-and-manage-their-access}

<Admonition type="tip" title="ก่อนเริ่มต้น">
สร้าง Instance ของ Account ก่อนที่จะเชิญผู้ใช้ สำหรับทุก Instance ที่สร้างผ่าน IBM Quantum Platform จะมี [access group](/guides/access-groups) ที่เกี่ยวข้องถูกสร้างขึ้นโดยอัตโนมัติ เมื่อเชิญผู้ใช้เข้า Instance สามารถกำหนดให้พวกเขาเข้า Access group ของ Instance นั้นๆ ได้ในระหว่างกระบวนการเชิญ
</Admonition>

<span id="invite"></span>
## เชิญผู้ใช้เข้า Account ขององค์กร {#invite-users-to-your-organizations-account}

<Admonition note="tip" title="ผู้ใช้ที่มี IBM Cloud account อยู่แล้ว">
ถ้าผู้ใช้มี IBM Cloud account ที่มีอยู่ก่อนแล้ว ให้ทำตามคำแนะนำในส่วน [จัดการผู้ใช้ IBM Cloud](/guides/manage-cloud-users)
</Admonition>

1. ก่อนอื่น ตรวจสอบให้แน่ใจว่าเลือก Account และ Region ที่ถูกต้องใน Account switcher และ Region switcher ในส่วน Header
1. เลือก [*Access management*](https://quantum.cloud.ibm.com/users) จากเมนูหลัก
1. ในแท็บ Users คลิก "Invite user"
1. ป้อนที่อยู่อีเมลของผู้ใช้และเลือกกำหนด Access group ให้ผู้ถูกเชิญได้ตามต้องการ
1. ผู้ใช้จะได้รับคำเชิญให้สร้าง IBM Cloud account และเข้าร่วม Instance หลังจากที่พวกเขายอมรับคำเชิญ พวกเขาจะเห็น Account ใหม่ใน Account switcher ที่ด้านบนของหน้า IBM Quantum Platform

## จัดการ Access group ของผู้ใช้ {#manage-a-users-access-groups}

ในการเปลี่ยน Access group ของผู้ใช้ ไปที่ [*Access management*](https://quantum.cloud.ibm.com/users) จากเมนูหลัก คลิกเมนู Overflow ที่ท้ายแถวของผู้ใช้และเลือก *Manage access* ในโมดอลที่เปิดขึ้น เลือก Access group ที่ผู้ใช้ควรเป็นสมาชิก

## ลบผู้ใช้ {#remove-a-user}

ในการลบผู้ใช้ออกจาก Instance ไปที่ [*Access management*](https://quantum.cloud.ibm.com/users) จากเมนูหลัก คลิกเมนู Overflow ที่ท้ายแถวของผู้ใช้ แล้วเลือก *Delete* ระบบจะขอให้ยืนยันการลบ

<Admonition type="note">
ถ้าผู้ใช้ถูกลบออกจาก Instance Workload และผลลัพธ์ที่เกี่ยวข้องจะไม่แสดงให้พวกเขาเห็นอีกต่อไป ผู้ใช้คนอื่นๆ ใน Instance เดียวกันยังสามารถเข้าถึงข้อมูล Workload และผลลัพธ์ได้
</Admonition>

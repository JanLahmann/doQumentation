## IBM Quantum Platform {#ibm-quantum-platform}

ส่วนนี้สรุปการปรับปรุงล่าสุดและฟีเจอร์ใหม่สำหรับ [IBM Quantum Platform](https://quantum.cloud.ibm.com/) ใหม่

**มกราคม 2026**

- การปรับปรุงล่าสุดในหน้า [Compute resources](https://quantum.cloud.ibm.com/computers) รวมถึง:

  - **มุมมอง QPU แบบรวม** - QPU ทั้งหมดในทุกภูมิภาคตอนนี้แสดงในตารางเดียว ทำให้ดูสถานะและการ calibrate ของ fleet ที่ใช้ร่วมกันทั้งสองภูมิภาคได้ง่ายขึ้น (ยังสามารถกรองตารางทรัพยากรคอมพิวเตอร์ตามภูมิภาคโดยใช้ไอคอน funnel ได้)

  - **ตารางการบำรุงรักษาเดียว** - ตารางการบำรุงรักษาตอนนี้อยู่ที่ด้านบนของหน้า แม้ว่าจะไม่มีเหตุการณ์ที่วางแผนไว้ในปัจจุบัน

  - **รายละเอียด "Instance access"** - เมื่อดูรายละเอียดของ QPU ส่วน Instance access ใหม่จะแสดง instance ทั้งหมดที่ให้สิทธิ์เข้าถึง QPU นั้นสำหรับบัญชีที่เลือก

- **QPU Nighthawk ตัวแรกและ QPU Heron ล่าสุดพร้อมใช้งานแล้ว** processor ใหม่สองตัวที่ทรงพลังเหล่านี้เปิดตัวใน [IBM Quantum Developer Conference](https://www.ibm.com/quantum/blog/qdc-2025) ในเดือนพฤศจิกายน

  - Premium และ Flex Plans ตอนนี้มีการเข้าถึงล่วงหน้าสู่ `ibm_miami` ซึ่งเป็น IBM Quantum Nighthawk QPU ตัวแรก เป็น quantum processor ที่ก้าวหน้าที่สุดของเราจนถึงปัจจุบัน Nighthawk เป็นอุปกรณ์ 120 Qubit ที่มี topology แบบ square-lattice (218 coupler เทียบกับ 176 ของ Heron) ที่ออกแบบมาเพื่อรองรับ workload ที่ซับซ้อนมากขึ้น Nighthawk มี coherence สูงสุดใน fleet ของเรา — สูงกว่าที่รายงานใน QDC — โดยมีค่ามัธยฐาน T1 ที่ 350 &#181;s

  - โปรดทราบว่า `ibm_miami` ขณะนี้เป็น QPU เชิงสำรวจที่ยังคงได้รับการทดสอบและปรับแต่งเป็นส่วนหนึ่งของวงจรการเรียนรู้อย่างต่อเนื่องของเรา ดังนั้น จึงมีข้อจำกัดชั่วคราวต่อไปนี้:

    - เวลาทำซ้ำเริ่มต้นได้รับการเพิ่มชั่วคราวจาก 250 &#181;s เป็น 4 ms ซึ่งจะส่งผลต่อเวลา QPU ที่จำเป็นในการรัน workload
    - Dynamic circuit ยังไม่รองรับ และคุณภาพของการวัดกลางวงจรยังมีข้อจำกัด

  - นอกจากนี้ Premium, Flex และ Pay-As-You-Go Plans ตอนนี้มีการเข้าถึง `ibm_boston` ซึ่งเป็น IBM Quantum Heron r3 ที่มีประสิทธิภาพสูงสุดของเราจนถึงปัจจุบัน มีค่า EPLG ที่ 100 Qubit ที่ 2.15e-3 โดยมี 57 จาก 176 two-qubit gate ที่ต้องการที่มีระดับข้อผิดพลาดต่ำกว่า 1e-3

- **แนะนำการ archive instance สำหรับผู้ดูแลระบบบน IBM Quantum Platform** ด้วยการอัปเดตนี้ ผู้ดูแลระบบสามารถ archive instance ใดก็ได้ในบัญชีของตน การ archive instance ช่วยให้:

    - ผู้ร่วมงานยังคงมีการเข้าถึงผลลัพธ์และ metrics, properties และ calibrate ของ QPU จาก workload ที่ผ่านมา
    - ไม่สามารถส่ง workload ใหม่ไปยัง instance ที่ถูก archive ได้ ป้องกันการใช้งานโดยไม่ตั้งใจ
    - Instance ที่ถูก archive ไม่หักออกจากการจัดสรร plan ของบัญชี
    - Instance ที่ถูก archive ทั้งหมดจัดระเบียบไว้ใต้แท็บ "Archived" ที่กำหนดไว้
    - ผู้ดูแลระบบสามารถ reactivate instance ได้ตลอดเวลา

**ตุลาคม 2025**

- **เรียนรู้การคำนวณควอนตัมในภาษาของคุณเอง**: เราตื่นเต้นที่จะประกาศว่าการแปลตอนนี้พร้อมใช้งานในหน้าเอกสารและการเรียนรู้ทั้งหมดบน IBM Quantum Platform! การแปลเป็นฟีเจอร์ที่ได้รับการร้องขอมากที่สุดในความคิดเห็นของผู้ใช้ และตอนนี้ การเรียนรู้และพัฒนาควอนตัมสามารถเข้าถึงได้มากกว่าที่เคย

  - การแปล **tutorial, course, เอกสาร และประกาศ** ตอนนี้มีให้บริการใน 7 ภาษา: ฝรั่งเศส เยอรมัน อิตาลี ญี่ปุ่น เกาหลี โปรตุเกส และสเปน

  - **การค้นหาหลายภาษา**: ใช้แถบค้นหาเพื่อหาสิ่งที่คุณต้องการในภาษาที่ต้องการ

  - **ทรัพยากรที่ดาวน์โหลดได้**: ดาวน์โหลดทรัพยากรเช่น guide หรือ tutorial ในภาษาที่เหมาะกับคุณที่สุด

  - **API reference ที่แปลแล้ว**: สำรวจ API reference เวอร์ชันล่าสุดและ REST API พร้อมเอกสารที่แปลครบถ้วน (เวอร์ชันปัจจุบันเท่านั้น)

  - **ข้อความแสดงข้อผิดพลาดและวิธีแก้ไข**: เข้าใจปัญหาได้เร็วขึ้นด้วย error code และวิธีแก้ไขที่แปลแล้ว

  พบข้อผิดพลาดในการแปล? แจ้งให้เราทราบ! [ส่ง translation issue](https://github.com/Qiskit/documentation/issues/new?template=translations.yaml) ใน GitHub repo ของเรา

  *โปรดทราบว่ามีความล่าช้าในการประมวลผลระหว่างการเผยแพร่เนื้อหาใหม่และการแปล ดังนั้นคุณอาจเห็นเฉพาะเวอร์ชันภาษาอังกฤษของเนื้อหาที่เพิ่งเผยแพร่สองสามวันในขณะที่กำลังดำเนินการแปล*

- **การเรียนรู้บน IBM Quantum Platform** ตอนนี้มีหน้าหลักที่อัปเดตเพื่อช่วยนำทางทรัพยากรการศึกษาที่มีให้ รวมถึงหน้า catalog ใหม่สองหน้าเพื่อช่วยเรียกดู course และ module [ดูการอัปเดต!](/learning)

**24 มิถุนายน 2025**

- หน้า Profile: ตอนนี้คุณสามารถดูรายละเอียด profile แบบรวมโดยตรงใน platform ใหม่ ดู profile ของคุณโดยคลิกที่ไอคอนผู้ใช้ที่มุมขวาบนหรือเยี่ยมชม [หน้า profile ของคุณ](https://quantum.cloud.ibm.com/profile)
- Public Backend: ดูรายการ Backend ของสหรัฐอเมริกาโดยไม่ต้องเข้าสู่ระบบ มีให้บริการบนหน้า [Compute Resources](https://quantum.cloud.ibm.com/computers) ที่อัปเดต
- การเรียนรู้: course การคำนวณควอนตัมได้รับการเผยแพร่ใน platform ที่อัปเกรดแล้ว พร้อมกับ educational module ใหม่ที่ออกแบบมาเพื่อช่วยให้ครูนำ Qiskit เข้าสู่ห้องเรียน นี่เป็นก้าวแรกในการย้าย IBM Quantum Learning ไปยัง platform ใหม่ โดยจะมีทรัพยากรเพิ่มเติมในสัปดาห์ข้างหน้า สามารถเข้าถึงทรัพยากรโดยไม่ต้องเข้าสู่ระบบที่ [IBM Quantum Learning](/learning) ดูเนื้อหาใหม่และอัปเดต รวมถึง [Quantum diagonalization algorithms](/learning/courses/quantum-diagonalization-algorithms), [Quantum computing in practice](/learning/courses/quantum-computing-in-practice) และ Qiskit in Classrooms module เช่น [Quantum mechanics](/learning/modules/quantum-mechanics) และ [Computer science](/learning/modules/computer-science)
- IBM Quantum Composer: [Composer](https://quantum.cloud.ibm.com/composer) ที่สร้าง Circuit ได้ย้ายไปยัง platform ใหม่แล้ว ประสบการณ์ส่วนใหญ่เหมือนเดิม ยกเว้นว่าไฟล์จะไม่ถูกเก็บไว้ใน platform อีกต่อไป และจะต้องดาวน์โหลดในเครื่อง ดูรายละเอียดเพิ่มเติมใน [เอกสาร Composer](/guides/composer)
- ประกาศ: ตอนนี้คุณสามารถเข้าถึงข่าว การอัปเดตซอฟต์แวร์ และการแจ้งเตือนบริการบน platform ใหม่ คลิกไอคอนระฆังบนแถบนำทางด้านบนหรือลิงก์ใต้ Featured resources บน [หน้าหลัก](https://quantum.cloud.ibm.com/) เพื่อเยี่ยมชมหน้า [Announcements](/announcements) ใหม่

- เพิ่มตัวกรอง instance job: ตอนนี้ platform แสดงเฉพาะ workload ของคุณโดยค่าเริ่มต้น บน dashboard คุณจะเห็นเฉพาะ workload ล่าสุดที่คุณส่งไปยัง instance ในบัญชีของคุณ

  - รองรับ private job สำหรับนักพัฒนา Qiskit Functions
  - channel ใหม่สำหรับ platform ที่ทำให้การเลือก instance เป็น optional ซึ่งช่วยให้การตั้งค่าบัญชีและการเข้าถึง Backend ง่ายขึ้นโดยการตรวจสอบ instance ที่ใช้งานได้ทั้งหมดโดยอัตโนมัติ การอัปเดตนี้เป็นประโยชน์สำหรับผู้ใช้ที่จัดการ instance หลายตัวหรือต้องการ workflow ที่เรียบง่ายโดยไม่ต้องระบุรายละเอียด instance ทุกครั้ง นักพัฒนาและนักวิจัยที่ทำงานในโปรเจ็กต์ต่างๆ ที่มีระดับการเข้าถึงและ Backend หลายตัวจะพบว่าสิ่งนี้มีประโยชน์อย่างยิ่ง

**8 พฤษภาคม 2025**

- [Flex Plan](https://www.ibm.com/quantum/blog/flex-plan) ซึ่งเป็น offering การใช้ทรัพยากรใหม่ได้รับการเผยแพร่แล้ว
- [Tutorial](https://docs.quantum.ibm.com/tutorials) ตอนนี้มีให้บริการบน platform เพื่อช่วยผู้ใช้ในการสำรวจ use case

- อัปเดต OpenAPI schema ของ Session: ฟิลด์ `backend_name` ตอนนี้สามารถเป็น null ได้

**24 เมษายน 2025**

- ขีดจำกัด Instance: บุคคลสามารถดู [เวลาที่ใช้ได้และการใช้งาน](https://quantum.cloud.ibm.com/instances) ซึ่งแสดงการแจ้งเตือนเมื่อใกล้ถึงขีดจำกัด
- สิทธิ์ผู้ดูแลระบบ: ผู้ดูแลระบบสามารถดูการใช้งานโดยรวม รอบระยะเวลา และเวลาที่เหลือต่อ plan ในหน้า Instances
- การค้นหา: การค้นหาตอนนี้รวมอยู่และพร้อมใช้งานสำหรับกรองเนื้อหาเอกสาร รวมถึง guide และ API reference
- การเรียกเก็บเงิน: ผู้ใช้ Pay-As-You-Go สามารถดูลิงก์ไปยังการเรียกเก็บเงินโดยคลิก overflow menu สำหรับแต่ละ instance บนหน้า [Instances](https://quantum.cloud.ibm.com/instances)
- ตัวเลือกการเข้าสู่ระบบใหม่: การเข้าสู่ระบบด้วย Google ตอนนี้มีให้บน [หน้าเข้าสู่ระบบ](https://quantum.cloud.ibm.com/signin)
- การแก้ไขบั๊ก:
  - แก้ไข batch Session ที่ล้มเหลวสำหรับผู้ใช้ Open Plan
  - แก้ไข batch Session ที่ไม่ปิดอย่างถูกต้อง
  - ปรับปรุงความเสถียรของ platform โดยปรับปรุงการจัดการข้อผิดพลาดและการกู้คืนที่ดีขึ้น
  - แก้ไขปัญหาที่ทำให้ QPU metrics ไม่สามารถแสดงสำหรับ QPU ในภูมิภาค eu-de
- การแจ้งเตือนการเลิกใช้: วันที่เลิกใช้ IBM Quantum Platform Classic ถูกกำหนดเป็น 1 กรกฎาคม 2025

**10 เมษายน 2025**

- อัปเดต navigation เพื่อลบหน้าที่ซ้ำกัน:
  - ส่วน Administration ถูกเปลี่ยนชื่อเป็น "Access management" และตอนนี้มุ่งเน้นเฉพาะการจัดการผู้ใช้และกลุ่มการเข้าถึงเท่านั้น
  - มีเพียงหน้าเดียวสำหรับจัดการ instance และ workload แต่ละอย่าง ไม่มีหน้าแยกต่างหากสำหรับแต่ละอย่างในส่วน Administration อีกต่อไป แต่สิ่งที่คุณสามารถดูและจัดการในแต่ละหน้าขึ้นอยู่กับการเข้าถึงและสิทธิ์ของคุณ
  - หน้า analytics (มองเห็นได้เฉพาะผู้ที่มีการเข้าถึงบางอย่าง) ถูกย้ายเข้าสู่เมนู navigation หลัก เพื่อให้ค้นหาได้ง่ายขึ้น
- เปิดตัว QPU ใหม่สองตัว: ibm_aachen (ภูมิภาค eu-de) และ ibm_kingston (ภูมิภาค us-east)
- บุคคลที่สามารถดูหน้า analytics ตอนนี้สามารถกรองข้อมูลเพื่อดูข้อมูลต่อ plan ได้
- ผู้ดูแลระบบตอนนี้สามารถเพิ่มบุคคลหลายคนในกลุ่มการเข้าถึงได้ง่ายขึ้น เนื่องจากมีการเพิ่มตาราง existing account members
- ผู้ดูแลระบบ instance ของ Premium plan สามารถตั้งค่าการจัดสรร instance เป็นขีดจำกัดการใช้งานได้ เพื่อให้การใช้งานใน instance เหล่านั้นไม่เกินเวลาที่จัดสรร ดูคำแนะนำใน [ตั้งขีดจำกัดการจัดสรร instance](/guides/allocation-limits)
- โปรดทราบว่าวันที่ sunset สำหรับ IBM Quantum Platform Classic ถูกกำหนดเป็น 1 กรกฎาคม 2025

  การอัปเดตหลักได้แก่:

  - **การเลือก hardware layout ที่เร็วขึ้น** - การอัปเกรดและการปรับปรุงประสิทธิภาพที่ขับเคลื่อนด้วย Rust สำหรับ `VF2Layout` และ `VF2PostLayout` ปรับปรุงความเร็วและความสามารถในการขยายขนาดของการเลือก layout circuit-ต่อ-hardware ลด compilation overhead และปรับปรุง fidelity บน quantum hardware

  - **PauliProductMeasurement แบบ multi-qubit ใหม่** - คำสั่ง `PauliProductMeasurement` ใหม่ช่วยให้การวัดแบบ joint projective ข้าม Qubit หลายตัวในการดำเนินการเดียว ปลดล็อกการ compile ไปยัง Pauli-based computation ซึ่งเป็นการแสดงแทนทั่วไปในการคำนวณควอนตัมแบบ fault-tolerant

  - **การปรับปรุงใน early fault-tolerant transpile** - การปรับปรุงประสิทธิภาพและฟีเจอร์สำหรับการ transpile ไปยัง early fault-tolerant target โดยเฉพาะ target Clifford+T ช่วยให้ผู้ใช้สร้าง transpile pipeline สำหรับ QPU ในอนาคตได้อย่างมีประสิทธิภาพมากขึ้น

  ความสามารถเหล่านี้ให้ความยืดหยุ่นมากขึ้นแก่นักพัฒนาในการปรับแต่ง quantum circuit optimization และรวมเข้ากับ HPC workflow ที่มีอยู่ [เยี่ยมชม blog](https://www.ibm.com/quantum/blog/qiskit-2-3-release-summary) สำหรับสรุปทางเทคนิคเต็มรูปแบบ

## Qiskit Runtime service {#qiskit-runtime-service}

สรุปการเปลี่ยนแปลงล่าสุดของ Qiskit Runtime service ใช้ได้กับทุกคนที่ใช้ Qiskit Runtime ไม่ว่าจะสื่อสารกับบริการอย่างไร (โดยใช้ `qiskit-ibm-runtime` หรืออื่นๆ) หรือใช้ client SDK เวอร์ชันใด

**มกราคม 2026**

- [**Directed execution model**](/guides/directed-execution-model) execution model ใหม่นี้ช่วยให้คุณปรับแต่ง error mitigation และเทคนิคอื่นๆ ได้อย่างละเอียดโดยไม่เสียสละประสิทธิภาพ โดยจัดเตรียม ingredient เพื่อ capture design intent ฝั่ง client และย้ายการสร้าง circuit variant ที่มีค่าใช้จ่ายสูงไปยัง server side ตัวอย่างเช่น คุณสามารถใช้ [Samplomatic library](https://qiskit.github.io/samplomatic/index.html) เพื่อกำหนด layer ที่แน่นอนใน circuit ของคุณสำหรับใช้ Pauli-twirling และ inject noise primitive [Executor](/guides/directed-execution-model#executor-primitive) ใหม่จะทำตามคำสั่งของคุณเพื่อสร้างและรัน randomized circuit variant ใน Qiskit Runtime

## Qiskit addons {#qiskit-addons}

- เมื่อเร็วๆ นี้เราได้ release addon ใหม่สองตัวที่ใช้ทรัพยากร classical เพื่อลดทรัพยากร quantum ที่จำเป็นสำหรับ error mitigation:

  Qiskit Functions Catalog preview ให้การเข้าถึงผู้ใช้ Premium Plan เพื่อสำรวจฟังก์ชันที่มีให้ รวมถึงฟังก์ชันที่เขียนโดย IBM และฟังก์ชันที่เขียนโดยสมาชิกอื่นๆ ของ ecosystem catalog มีฟังก์ชันสองประเภท: circuit function และ application function

    - **Circuit function** มอบ interface ที่เรียบง่ายสำหรับรัน circuit รับ abstract circuit และ observable ที่ผู้ใช้ให้มาเป็น input จากนั้นจัดการการสังเคราะห์ การปรับให้เหมาะสม และการ execute circuit ISA ที่เป็นตัวแทน Circuit function รวมความสามารถล่าสุดใน transpile, error suppression และ error mitigation เพื่อให้ประสิทธิภาพระดับ utility สามารถเข้าถึงได้ทันที ซึ่งช่วยให้นักวิทยาศาสตร์คอมพิวเตอร์มุ่งเน้นการ map ปัญหาของตนไปยัง circuit แทนที่จะสร้าง pattern สำหรับแต่ละปัญหาตั้งแต่ต้น

    - **Application function** ครอบคลุมงานระดับสูงกว่า เช่น การสำรวจ algorithm และ use case เฉพาะโดเมน นักพัฒนาองค์กรและนักวิทยาศาสตร์ข้อมูลอาจไม่มีความรู้พื้นฐาน quantum information science สำหรับทำงานกับ circuit แต่ต้องการนำความรู้ด้านโดเมนของตนมาพัฒนา quantum computing algorithm และแอปพลิเคชัน ด้วย application function ผู้ใช้สามารถป้อน classical input และรับคำตอบเพื่อให้ทดลองนำ quantum เข้าสู่ workflow เฉพาะโดเมนของตนได้ง่ายขึ้น

- ด้วยการเปิดตัว Qiskit Functions Catalog นักพัฒนา Premium Plan สามารถสำรวจ IBM Circuit function ได้ IBM Circuit function รวม AI-powered extension ล่าสุดสำหรับ Qiskit สำหรับการสังเคราะห์ การปรับให้เหมาะสม และการจัดตาราง circuit รวมถึง advanced error mitigation method เพื่อส่งคืนค่าประมาณที่แม่นยำที่สุดที่เป็นไปได้ด้วย hardware ปัจจุบัน

  ผู้ใช้สามารถซื้อใบอนุญาตสำหรับฟังก์ชันต่อไปนี้ที่ contributed โดยพันธมิตรของเราที่ Q-CTRL, QEDMA และ Algorithmiq

- **Circuit function**

  - Q-CTRL กำลัง release circuit function ที่ใช้เทคนิค AI-driven quantum control ซึ่งผู้ใช้สามารถขยายขนาดไปยังปัญหาที่ใหญ่ขึ้นได้สำเร็จ
  - Algorithmiq กำลัง release circuit function ที่ใช้ TEM (tensor-network error mitigation) ซึ่งเป็น error mitigation method สำหรับรับ estimator ด้วย shot ที่น้อยกว่าวิธี PEC (probabilistic error cancellation)
  - QEDMA กำลัง release circuit function ที่ใช้ protocol เฉพาะสำหรับการ characterize QPU operation ที่มี noise อย่างมีประสิทธิภาพและแม่นยำ และใช้ error suppression และ error mitigation ตาม characterization data

- **Application function**

  - Q-CTRL กำลัง release optimization solver ที่ผู้ใช้สามารถส่ง graph หรือ objective และรับ solution cost

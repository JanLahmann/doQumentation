เพื่ออำนวยความสะดวกในการพัฒนาและนำโค้ด transpilation แบบกำหนดเองกลับมาใช้ซ้ำโดยชุมชน Qiskit ในวงกว้าง Qiskit SDK รองรับ interface แบบ plugin ที่ช่วยให้แพ็กเกจ Python ของบุคคลที่สามสามารถประกาศว่าตนมีฟังก์ชัน transpilation ที่ขยายเพิ่มเติมซึ่งเข้าถึงได้ผ่าน Qiskit

ปัจจุบัน plugin ของบุคคลที่สามสามารถให้ฟังก์ชัน transpilation ที่ขยายเพิ่มเติมได้สามวิธี:

- [transpiler stage plugin](https://docs.quantum.ibm.com/api/qiskit/transpiler_plugins) ให้ pass manager ที่สามารถใช้แทนหนึ่งใน [6 ขั้นตอน](transpiler-stages) ของ preset staged pass manager ได้แก่: `init`, `layout`, `routing`, `translation`, `optimization` และ `scheduling`
- [unitary synthesis plugin](https://docs.quantum.ibm.com/api/qiskit/qiskit.transpiler.passes.synthesis.plugin.UnitarySynthesisPlugin) ให้ฟังก์ชันที่ขยายเพิ่มเติมสำหรับการสังเคราะห์ unitary gate
- [high-level synthesis plugin](https://docs.quantum.ibm.com/api/qiskit/qiskit.transpiler.passes.synthesis.plugin.HighLevelSynthesisPlugin) ให้ฟังก์ชันที่ขยายเพิ่มเติมสำหรับการสังเคราะห์ "high-level objects" เช่น linear functions หรือ Clifford operators โดย high-level objects แทนด้วย subclass ของคลาส [Operation](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.Operation)

ส่วนที่เหลือของหน้านี้อธิบายวิธีแสดงรายการ plugin ที่มีอยู่ ติดตั้ง plugin ใหม่ และใช้งาน plugin เหล่านั้น
## แสดงรายการ plugin ที่มีอยู่และติดตั้ง plugin ใหม่ {#list-available-plugins-and-install-new-ones}

Qiskit มี plugin ในตัวสำหรับ transpilation อยู่แล้ว หากต้องการติดตั้งเพิ่มเติม สามารถใช้ Python package manager ได้ ตัวอย่างเช่น อาจรัน `pip install qiskit-toqm` เพื่อติดตั้ง [Qiskit TOQM](https://github.com/qiskit-toqm/qiskit-toqm) routing stage plugin มี plugin ของบุคคลที่สามหลายตัวที่เป็นส่วนหนึ่งของ [Qiskit ecosystem](https://qiskit.github.io/ecosystem/#transpiler_plugin)

## ใช้งาน plugin {#use-a-plugin}

ในส่วนนี้จะแสดงวิธีใช้งาน transpiler plugins ในตัวอย่างโค้ด เราใช้ plugin ที่มากับ Qiskit แต่ plugin ที่ติดตั้งจากแพ็กเกจของบุคคลที่สามก็ใช้งานในลักษณะเดียวกัน

### ใช้งาน transpiler stage plugin {#use-a-transpiler-stage-plugin}

ในการใช้งาน transpiler stage plugin ให้ระบุชื่อของ plugin พร้อมกับ argument ที่เหมาะสมใน [`generate_preset_pass_manager`](https://docs.quantum.ibm.com/api/qiskit/qiskit.transpiler.generate_preset_pass_manager#qiskit.transpiler.generate_preset_pass_manager) หรือ [`transpile`](https://docs.quantum.ibm.com/api/qiskit/compiler#qiskit.compiler.transpile) โดย argument จะเกิดจากการต่อ `_method` ต่อท้ายชื่อขั้นตอน transpilation ตัวอย่างเช่น หากต้องการใช้ `lookahead` routing plugin ให้ระบุ `lookahead` ใน argument `routing_method`:

```python
from qiskit.transpiler import generate_preset_pass_manager
from qiskit_ibm_runtime import QiskitRuntimeService

service = QiskitRuntimeService()
backend = service.backend("ibm_fez")

pass_manager = generate_preset_pass_manager(
    optimization_level=3, backend=backend, routing_method="lookahead"
)
```

### ใช้งาน unitary synthesis plugin {#use-a-unitary-synthesis-plugin}

ในการใช้งาน unitary synthesis plugin ให้ระบุชื่อของ plugin เป็น argument `unitary_synthesis_method` ใน [`generate_preset_pass_manager`](https://docs.quantum.ibm.com/api/qiskit/qiskit.transpiler.generate_preset_pass_manager#qiskit.transpiler.generate_preset_pass_manager) หรือ [`transpile`](https://docs.quantum.ibm.com/api/qiskit/compiler#qiskit.compiler.transpile):

```python
pass_manager = generate_preset_pass_manager(
    optimization_level=3,
    backend=backend,
    unitary_synthesis_method="sk",
    unitary_synthesis_plugin_config=dict(
        basis_gates=["cz", "id", "rz", "sx", "x"]
    ),
)
```

Unitary synthesis ถูกใช้ในขั้นตอน `init`, `translation` และ `optimization` ของ staged pass manager ที่คืนค่าจาก [`generate_preset_pass_manager`](https://docs.quantum.ibm.com/api/qiskit/qiskit.transpiler.generate_preset_pass_manager#qiskit.transpiler.generate_preset_pass_manager) หรือที่ใช้ใน [`transpile`](https://docs.quantum.ibm.com/api/qiskit/compiler#qiskit.compiler.transpile) ดู [Transpiler stages](transpiler-stages) สำหรับคำอธิบายของขั้นตอนเหล่านี้

ใช้ argument `unitary_synthesis_plugin_config` ซึ่งเป็น dictionary รูปแบบเสรี เพื่อส่งตัวเลือกสำหรับวิธี unitary synthesis เอกสารของวิธี synthesis ควรอธิบายตัวเลือกที่รองรับ ดู [รายการนี้](https://docs.quantum.ibm.com/api/qiskit/transpiler_synthesis_plugins#unitary-synthesis-plugins-2) สำหรับลิงก์ไปยังเอกสารของ unitary synthesis plugins ในตัว
### ใช้งาน high-level synthesis plugin {#use-a-high-level-synthesis-plugin}

ขั้นแรก สร้าง [HLSConfig](https://docs.quantum.ibm.com/api/qiskit/qiskit.transpiler.passes.HLSConfig) เพื่อ
จัดเก็บชื่อของ plugin ที่จะใช้กับ high-level objects ต่าง ๆ ตัวอย่างเช่น:

```python
from qiskit.transpiler.passes import HLSConfig

hls_config = HLSConfig(clifford=["layers"], linear_function=["pmh"])
```

โค้ดเซลล์นี้สร้างการตั้งค่า high-level synthesis ที่ใช้ `layers` plugin
สำหรับการสังเคราะห์ออบเจ็กต์ [Clifford](https://docs.quantum.ibm.com/api/qiskit/qiskit.quantum_info.Clifford#clifford) และ `pmh` plugin สำหรับการสังเคราะห์
ออบเจ็กต์ [LinearFunction](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.library.LinearFunction#linearfunction) ชื่อของ keyword arguments สอดคล้องกับ
attribute [`name`](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.Operation#name) ของคลาส [Operation](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.Operation) ที่แทนประเภทของออบเจ็กต์ที่กำลังสังเคราะห์
สำหรับ high-level object แต่ละตัว รายการของ plugin ที่กำหนดไว้จะถูกลองตามลำดับจนกว่าจะมีตัวหนึ่งสำเร็จ (ในตัวอย่างข้างต้น แต่ละรายการมี plugin เพียงตัวเดียว)

นอกจากการระบุ
plugin โดยชื่อ ยังสามารถส่ง tuple `(name, options)` โดยที่ element ที่สองของ tuple คือ dictionary ที่มีตัวเลือกสำหรับ plugin เอกสารของวิธี synthesis ควรอธิบายตัวเลือกที่รองรับ ดู [รายการนี้](https://docs.quantum.ibm.com/api/qiskit/transpiler_synthesis_plugins#high-level-synthesis-plugins-2) สำหรับลิงก์ไปยังเอกสารของ high-level synthesis plugins ในตัว

เมื่อสร้างออบเจ็กต์ `HLSConfig` แล้ว ให้ส่งเป็น
argument `hls_config` ใน [`generate_preset_pass_manager`](https://docs.quantum.ibm.com/api/qiskit/qiskit.transpiler.generate_preset_pass_manager#qiskit.transpiler.generate_preset_pass_manager) หรือ [`transpile`](https://docs.quantum.ibm.com/api/qiskit/compiler#qiskit.compiler.transpile):

```python
pass_manager = generate_preset_pass_manager(
    optimization_level=3, backend=backend, hls_config=hls_config
)
```

High-level synthesis ถูกใช้ในขั้นตอน `init`, `translation` และ `optimization` ของ staged pass manager ที่คืนค่าจาก [`generate_preset_pass_manager`](https://docs.quantum.ibm.com/api/qiskit/qiskit.transpiler.generate_preset_pass_manager#qiskit.transpiler.generate_preset_pass_manager) หรือที่ใช้ใน [`transpile`](https://docs.quantum.ibm.com/api/qiskit/compiler#qiskit.compiler.transpile) ดู [Transpiler stages](transpiler-stages) สำหรับคำอธิบายของขั้นตอนเหล่านี้

# การตัดวงจร {#circuit-cutting}

การตัดวงจร (Circuit cutting) คือเทคนิคที่ใช้เพิ่มขนาดของวงจรที่สามารถรันบนฮาร์ดแวร์ควอนตัมได้ โดยแลกมาด้วยค่าใช้จ่ายในการสุ่มตัวอย่างที่เพิ่มขึ้น addon นี้ใช้งานเทคนิคดังกล่าว โดยตัดเกตบางส่วน, สาย, หรือทั้งสองอย่าง ทำให้ได้วงจรที่เล็กลงและเหมาะสมกว่าสำหรับการรันบนฮาร์ดแวร์ จากนั้นจึงรันวงจรย่อยเหล่านี้ และนำผลลัพธ์ของวงจรดั้งเดิมกลับมาประกอบกันผ่านการประมวลผลแบบคลาสสิก อย่างไรก็ตาม ข้อแลกเปลี่ยนคือจำนวน shot ทั้งหมดต้องเพิ่มขึ้นตามปัจจัยที่ขึ้นอยู่กับจำนวนและประเภทของการตัดที่ทำ (เรียกว่า sampling overhead) การตัดวงจรยังสามารถใช้สร้างเกตระหว่าง Qubit ที่อยู่ห่างกัน ซึ่งหากไม่ใช้วิธีนี้จะต้องแลกด้วย SWAP overhead จำนวนมาก

## คำศัพท์สำคัญ {#important-terms}

- **Subcircuits**: ชุดของวงจรที่เกิดจากการตัดเกตใน `QuantumCircuit` แล้วแยกกลุ่ม Qubit ที่ไม่เชื่อมต่อกันออกเป็นวงจรย่อย วงจรเหล่านี้มีอ็อบเจกต์ [`SingleQubitQPDGate`](https://docs.quantum.ibm.com/api/qiskit-addon-cutting/qpd-single-qubit-qpd-gate) และใช้สำหรับสร้างแต่ละ subexperiment

- **Subexperiment**: คำที่ใช้อธิบายตัวอย่างวงจรที่ไม่ซ้ำกันซึ่งเกี่ยวข้องกับ subcircuit แต่ละตัว ซึ่งจะถูกส่งไปยัง QPU เพื่อรัน

## ติดตั้งแพ็กเกจตัดวงจร {#install-the-circuit-cutting-package}

มีสามวิธีในการติดตั้งแพ็กเกจตัดวงจร ได้แก่ PyPI, การสร้างจากซอร์สโค้ด, และการรันในสภาพแวดล้อม container แนะนำให้ติดตั้งแพ็กเกจเหล่านี้ใน [virtual environment](https://docs.python.org/3.10/tutorial/venv.html) เพื่อให้แน่ใจว่า dependencies ของแพ็กเกจแยกจากกัน

### ติดตั้งจาก PyPI {#install-from-pypi}

วิธีที่ตรงไปตรงมาที่สุดในการติดตั้งแพ็กเกจ `qiskit-addon-cutting` คือผ่าน PyPI:
```bash
pip install qiskit-addon-cutting
```

## พื้นฐานทางทฤษฎี {#theoretical-background}

ในกระบวนการตัดวงจร มีการตัดสองประเภท ได้แก่ การตัด **เกต** หรือ "space-like" ซึ่งตัดผ่านเกตที่ทำงานบน Qubit สองตัว (หรือมากกว่า) และการตัด **สาย** หรือ "time-like" ซึ่งตัดตรงผ่านสายของ Qubit (โดยพื้นฐานคือเกต identity ของ Qubit เดียวที่ถูกตัดเป็นสองส่วน)

แผนภาพด้านล่างแสดงตัวอย่างการตัดเกตเพื่อแบ่งวงจรออกเป็นสองส่วนที่เล็กลงพร้อม Qubit ที่น้อยลง

![แผนภาพการตัดเกตโดยนำวงจรขนาดใหญ่หนึ่งวงจรมาตัดเป็นสองวงจรเล็กที่มีป้ายกำกับ "A" และ "B"](/docs/images/guides/qiskit-addons/circuit-cutting-diagram.avif)

มีสามสถานการณ์ที่ต้องพิจารณาเมื่อเตรียมกระบวนการทำงานในการตัดวงจร ซึ่งเกี่ยวข้องกับความพร้อมใช้งานของการสื่อสารแบบคลาสสิกระหว่างการรันวงจร สถานการณ์แรกคือมีเฉพาะการดำเนินงานแบบโลคัล (LO) เท่านั้น ส่วนอีกสองสถานการณ์จะมีการสื่อสารแบบคลาสสิกระหว่างการรัน เรียกว่าการดำเนินงานแบบโลคัลและการสื่อสารแบบคลาสสิก (LOCC) สถานการณ์ LOCC แบ่งออกเป็นการสื่อสารแบบ near-time ทิศทางเดียวระหว่างการรันวงจร หรือการสื่อสารแบบ real-time สองทิศทาง (ซึ่งอาจพบในสภาพแวดล้อม multi-QPU)

แม้ว่าการตัดวงจรสามารถใช้รันวงจรควอนตัมที่มีขนาดใหญ่กว่าที่ฮาร์ดแวร์ปัจจุบันรองรับได้ แต่ก็มีค่าใช้จ่าย เนื่องจากเทคนิคนี้สามารถกำหนดเป็นปัญหา quasi-probability decomposition (QPD) จึงมี sampling overhead แบบ exponential ที่จำเป็นในการประกอบผลลัพธ์กลับคืน overhead นี้คือปัจจัยที่จำนวน shot ทั้งหมดต้องเพิ่มขึ้นเพื่อให้ quasi-probability decomposition ให้ค่าความผิดพลาด $\epsilon$ เท่ากับที่จะได้จากการรันวงจรดั้งเดิม เกตที่ถูกตัดแต่ละตัวมีส่วนในการเพิ่ม overhead และปริมาณ overhead ที่เพิ่มขึ้นขึ้นอยู่กับประเภทของเกตที่ถูกตัด (รายละเอียดเพิ่มเติมเกี่ยวกับ sampling overhead อยู่ในภาคผนวกสุดท้ายของ [[1]](#references))

ตัวอย่างเช่น เกต CNOT ที่ถูกตัดหนึ่งตัวมี sampling overhead เป็น 9 [[2,6]](#references) และวงจรที่มีการตัดสาย $n$ ครั้งมี sampling overhead เป็น $\mathcal{O}(16^n)$ เมื่อไม่มีการสื่อสารแบบคลาสสิก (สถานการณ์ LO) ซึ่งลดลงเป็น $\mathcal{O}(4^n)$ เมื่อมีการสื่อสารแบบคลาสสิก (สถานการณ์ LOCC) [[4]](#references) อย่างไรก็ตาม การตัดสายพร้อมการสื่อสารแบบคลาสสิก (LOCC) ไม่ได้รับการสนับสนุนโดยแพ็กเกจนี้

ในเชิงทฤษฎี ปัญหา QPD ของการตัดวงจรสามารถแสดงได้ดังนี้:

$$ \mathcal{U} = \sum_i a_i \mathcal{F}_i,$$

โดยที่ $\mathcal{U}$ คือช่องทางควอนตัมที่ใช้งานการดำเนินงานที่ต้องการ และ $a_i$ แต่ละตัวคือค่าสัมประสิทธิ์จริงที่สอดคล้องกับช่องทาง $\mathcal{F}_i$ ที่รันได้บนฮาร์ดแวร์

ผลลัพธ์ที่เทียบเท่ากับช่องทางที่ต้องการ $\mathcal{U}$ ได้รับโดยการสร้างค่าสัมประสิทธิ์ $a_i$ ก่อน จากนั้นรัน subexperiment เพื่อรับผลลัพธ์ของช่องทางต่าง ๆ $\mathcal{F}_i$ เพื่อประกอบค่า expectation values ที่สอดคล้องกับ $\mathcal{U}$

### ตัวอย่างสั้น: การตัด RZZGate {#a-short-example-cutting-a-rzzgate}

เป็นตัวอย่างพื้นฐานที่ชัดเจน ลองพิจารณา decomposition ของ RZZGate ที่ถูกตัด (รายละเอียดอยู่ใน [[2]](#references)) วงจรควอนตัมที่มี RZZGate สามารถจำลองได้โดยการรัน subexperiment หกตัว ซึ่ง RZZGate ถูกแทนที่ด้วยการดำเนินงาน Qubit เดียวเท่านั้น (นี่คือ $\mathcal{F}_i$ จากสมการด้านบน) ผลลัพธ์ของวงจรนี้ประกอบขึ้นโดยการรวมผลลัพธ์ของแต่ละ subexperiment กับชุดค่าสัมประสิทธิ์ ($a_i$ จากสมการด้านบน) ซึ่งอาจเป็นบวกหรือลบก็ได้

สำหรับพารามิเตอร์ $\theta$ ที่เลือกของ RZZGate subexperiment ทั้งหกมีดังนี้:
1. โดยมีค่าสัมประสิทธิ์ $a_1 = \cos^2(\theta/2)$ ไม่ทำอะไร ($I\otimes I$)
1. โดยมีค่าสัมประสิทธิ์ $a_2 = \sin^2(\theta/2)$ ทำ ZGate บน Qubit แต่ละตัว ($Z\otimes Z$)
1. โดยมีค่าสัมประสิทธิ์ $a_3 = -\sin(\theta)/2$ ทำการวัดแบบ projective ในฐาน $Z$ บน Qubit แรก และ $S$ บน Qubit ที่สอง ($M_z\otimes S$) หากผลการวัดเป็น $1$ ให้พลิกเครื่องหมายของส่วนที่ผลลัพธ์นั้นมีส่วนในระหว่างการประกอบ
1. โดยมีค่าสัมประสิทธิ์ $a_4 = \sin(\theta)/2$ ทำการวัดแบบ projective ในฐาน $Z$ บน Qubit แรก และ $S^\dagger$ บน Qubit ที่สอง ($M_z\otimes S^\dagger$) หากผลการวัดเป็น 1 ให้พลิกเครื่องหมายของส่วนที่ผลลัพธ์นั้นมีส่วนในระหว่างการประกอบ
1. เหมือนกับข้อ 3. ($a_5=a_3$) แต่สลับ Qubit (ทำ $S\otimes M_z$ แทน)
1. เหมือนกับข้อ 4. ($a_6=a_4$) แต่สลับ Qubit (ทำ $S^\dagger\otimes M_z$ แทน)

### ตารางอ้างอิง sampling overhead {#sampling-overhead-reference-table}

ตารางต่อไปนี้แสดงปัจจัย sampling overhead สำหรับคำสั่ง two-qubit หลายประเภท โดยกำหนดว่าตัดคำสั่งเพียงหนึ่งตัว
| คำสั่ง | KAK decomposition angles | ปัจจัย sampling overhead |
| --- | --- | --- |
| CSGate, CSdgGate, CSXGate | $\left(\pi/8, 0, 0\right)$ | $3+2\sqrt(2) \approx 2.828$ |
| CXGate, CYGate, CZGate, GHGate, ECRGate | $\left(\pi/4, 0, 0\right)$ | $3^2=9$ |
| iSwapGate, DCXGate | $\left(\pi/4, \pi/4, 0\right)$ | $7^2 = 49$ |
| SwapGate | $\left(\pi/4, \pi/4, \pi/4\right)$ | $7^2 = 49$ |
| RXXGate, RYYGate, RZZGate, RZXGate | $\left(\lvert\theta/2\rvert, 0, 0, \right)$ | $\left(1 + 2\lvert\sin(\theta)\rvert\right)^2$ |
| CRXGate, CRYGate, CRZGate, CPhaseGate | $\left(\lvert\theta/4\rvert, 0, 0\right)$ |  $\left(1 + 2\lvert\sin(\theta/2)\rvert\right)^2$ |
| XXPlusYYGate, XXMinusYYGate | $\left(\vert\theta/4\rvert, \lvert\theta/4\rvert, 0\right)$ | $\left(1 + 4\lvert\sin(\theta/2)\rvert + 2\sin^2(\theta/2)\right)^2$ (independent of $\beta$) |
| Move (ตัดสายในสถานการณ์ LO) | N/A | $4^2 = 16$ |

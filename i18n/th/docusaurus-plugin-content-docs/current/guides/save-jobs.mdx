Quantum workflow มักใช้เวลานานและอาจทำงานข้ามหลาย Session รีสตาร์ท Python kernel หมายความว่าผลลัพธ์ที่เก็บไว้ในหน่วยความจำจะหายไปทั้งหมด เพื่อป้องกันการสูญเสียข้อมูล สามารถบันทึกผลลัพธ์ลงไฟล์ และดึงผลลัพธ์ของ Job ที่ผ่านมาจาก IBM Quantum&reg; ได้ เพื่อให้ Session ถัดไปทำงานต่อจากที่ค้างไว้

## ดึงผลลัพธ์ Job จาก IBM Quantum {#retrieve-job-results-from-ibm-quantum}

IBM Quantum เก็บผลลัพธ์ของทุก Job ไว้ให้โดยอัตโนมัติ เพื่อให้ดึงกลับมาได้ในภายหลัง ใช้ฟีเจอร์นี้เพื่อทำงานกับ quantum program ต่อเนื่องข้ามการรีสตาร์ท kernel และทบทวนผลลัพธ์ในอดีต สามารถรับ ID ของ Job แบบ programmatic ผ่าน method `job_id` หรือดูรายการ Job ที่ส่งทั้งหมดพร้อม ID ได้ที่ [หน้า Workloads](https://quantum.cloud.ibm.com/workloads)

ในการค้นหา Job แบบ programmatic ให้ใช้ method [`QiskitRuntimeService.jobs`](https://docs.quantum.ibm.com/api/qiskit-ibm-runtime/qiskit-runtime-service#jobs) ค่าเริ่มต้นจะคืน Job ที่ส่งล่าสุด แต่ยังสามารถกรอง Job ตามชื่อ Backend วันที่สร้าง และอื่น ๆ ได้ด้วย cell ต่อไปนี้ค้นหา Job ที่ส่งในช่วงสามเดือนที่ผ่านมา โดย argument `created_after` ต้องเป็น object ประเภท [`datetime.datetime`](https://docs.python.org/3.8/library/datetime.html#datetime.datetime)

```python
# Added by doQumentation — required packages for this notebook
!pip install -q qiskit-ibm-runtime
```

```python
import datetime
from qiskit_ibm_runtime import QiskitRuntimeService

three_months_ago = datetime.datetime.now() - datetime.timedelta(days=90)

service = QiskitRuntimeService()
jobs_in_last_three_months = service.jobs(created_after=three_months_ago)
jobs_in_last_three_months[:3]  # show first three jobs
```

ยังสามารถเลือกตาม Backend สถานะ Job Session และอื่น ๆ ได้อีก ดูข้อมูลเพิ่มเติมได้ที่ [`QiskitRuntimeService.jobs`](https://docs.quantum.ibm.com/api/qiskit-ibm-runtime/qiskit-runtime-service#jobs) ในเอกสาร API

เมื่อได้ job ID แล้ว ให้ใช้ method [`QiskitRuntimeService.job`](https://docs.quantum.ibm.com/api/qiskit-ibm-runtime/qiskit-runtime-service#job) เพื่อดึงข้อมูลกลับมา

```python
# Get ID of most recent successful job for demonstration.
# This will not work if you've never successfully run a job.
successful_job = next(
    j for j in service.jobs(limit=1000) if j.status() == "DONE"
)
job_id = successful_job.job_id()
print(job_id)
```

## บันทึกผลลัพธ์ลงดิสก์ {#save-results-to-disk}

อาจต้องการบันทึกผลลัพธ์ลงดิสก์ด้วย ทำได้โดยใช้ไลบรารี JSON ในตัวของ Python ร่วมกับ encoder จาก Qiskit Runtime

```python
import json
from qiskit_ibm_runtime import RuntimeEncoder

with open("result.json", "w") as file:
    json.dump(retrieved_job.result(), file, cls=RuntimeEncoder)
```

จากนั้นสามารถโหลด array นี้จากดิสก์ใน kernel แยกต่างหากได้

```python
from qiskit_ibm_runtime import RuntimeDecoder

with open("result.json", "r") as file:
    result = json.load(file, cls=RuntimeDecoder)

result
```

# จัดการผู้ใช้ IBM Cloud {#manage-ibm-cloud-users}

คู่มือนี้แสดงวิธีใช้ IBM Cloud&reg; เพื่อเปิดใช้งานผู้ใช้ที่มีบัญชี IBM Cloud และให้คำแนะนำสำหรับผู้ใช้ในการเข้าถึงสภาพแวดล้อม

หากต้องการจัดการผู้ใช้ผ่าน ID provider แทน ให้ทำตามคำแนะนำในหัวข้อ [จัดการ ID providers](/guides/manage-appid)

## ขั้นตอน {#steps}

ทำตามขั้นตอนระดับสูงเหล่านี้เพื่อจัดการผู้ใช้ IBM Cloud:

1. ตรวจสอบว่าผู้ใช้ที่ต้องการเชิญมีบัญชี IBM Cloud แล้ว
2. ไปที่ IAM และ[เชิญผู้ใช้](https://cloud.ibm.com/iam/users/invite_users)
   <Admonition type="note">
   ผู้ใช้จะยังไม่สามารถจัดการได้จนกว่าจะยอมรับคำเชิญและเข้าสู่ระบบอย่างน้อยหนึ่งครั้ง
   </Admonition>
3. (ไม่บังคับ) แก้ไขการกำหนดโปรเจกต์และกลุ่มสิทธิ์การเข้าถึงของผู้ใช้จาก [IAM](https://cloud.ibm.com/iam/users)

<span id="user-org"></span>
## ขั้นตอนของผู้ใช้ {#user-flow}

1. ผู้ใช้ยอมรับคำเชิญ จากนั้นเข้าสู่ระบบผ่าน [IBM Cloud portal](https://cloud.ibm.com/)
2. ผู้ใช้สร้าง API key โดยไปที่ ([Manage → Access (IAM) → API keys](https://cloud.ibm.com/iam/apikeys))
3. สำหรับข้อมูลเพิ่มเติม ผู้ใช้สามารถอ่าน [ติดตั้ง Qiskit](/guides/install-qiskit)

<span id="steps-cloud-org"></span>
## ตัวอย่างสถานการณ์ {#example-scenario}

ตัวอย่างนี้สร้างการตั้งค่าดังต่อไปนี้:

* มีสองโปรเจกต์คือ `ml` และ `finance`
   * โปรเจกต์ `ml` ควรมีสิทธิ์เข้าถึง service instance `QR-ml` และ `QR-common`
   * โปรเจกต์ `finance` ควรมีสิทธิ์เข้าถึง service instance `QR-finance` และ `QR-common`
* มีผู้ใช้สามคน:
   * Fatima ควรมีสิทธิ์เข้าถึงโปรเจกต์ `ml`
   * Ravi ควรมีสิทธิ์เข้าถึงโปรเจกต์ `finance`
   * Amyra ควรมีสิทธิ์เข้าถึงทั้งสองโปรเจกต์
* ใช้ access group โดยไม่มี resource group
* ผู้ใช้ถูกกำหนดใน IBM Cloud และการกำหนดโปรเจกต์ก็ทำที่นั่นด้วย
* ผู้ใช้ควรสามารถลบ job ได้

ขั้นตอนในการดำเนินการตั้งค่านี้คือ:

2. Cloud administrator สร้าง service instance สามรายการ: `QR-ml`, `QR finance` และ `QR-common`
3. Cloud administrator สร้าง custom rule ที่รวม action `quantum-computing.job.delete`
1. Cloud administrator สร้าง access group สองกลุ่ม:
   * access group `ml` สามารถเข้าถึง `QR-ml` และ `QR-common`
   * access group `finance` สามารถเข้าถึง `QR-finance` และ `QR-common`
1. Cloud administrator เชิญผู้ใช้ cloud เข้าสู่โปรเจกต์ที่เหมาะสม โดยเฉพาะอย่างยิ่ง เชิญและกำหนดผู้ใช้ให้กับ access group ที่รวมโปรเจกต์นั้น
   * Fatima ถูกเพิ่มเข้า access group "ml"
   * Ravi ถูกเพิ่มเข้า access group "finance"
   * Amyra ถูกเพิ่มเข้าทั้ง access group "ml" และ "finance"
1. ผู้ใช้สามารถเข้าสู่ระบบผ่าน IBM Cloud portal สร้าง API key และทำงานกับ service instance ของโปรเจกต์ได้

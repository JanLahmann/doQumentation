# เริ่มต้น service ใน environment ที่ไม่เชื่อถือ {#initialize-the-service-in-an-untrusted-environment}

ทำตามคำแนะนำเหล่านี้หากใช้งาน IBM Cloud หรือ IBM Qiskit Runtime REST API ใน environment ที่ไม่เชื่อถือ เช่น คอมพิวเตอร์สาธารณะ

**หลีกเลี่ยงการรันโค้ดบนเครื่องที่ไม่เชื่อถือหรือ cloud Python environment ภายนอกเพื่อลดความเสี่ยงด้านความปลอดภัย**  หากจำเป็นต้องใช้ environment ที่ไม่เชื่อถือ (เช่น คอมพิวเตอร์ที่ใช้ร่วมกันหรือสาธารณะ) ให้เปลี่ยน API key หลังจากแต่ละการใช้งานโดยการลบบน [หน้า API keys](https://cloud.ibm.com/iam/apikeys) และสร้างใหม่

    <Admonition type="caution">
      **ปกป้อง API key ของคุณ!** ห้ามใส่ key ในซอร์สโค้ด, Python script หรือไฟล์ notebook  เมื่อแชร์โค้ดกับผู้อื่น ตรวจสอบให้แน่ใจว่า API key ไม่ได้ฝังโดยตรงใน Python script  แต่ควรแชร์ script โดยไม่มี token และให้คำแนะนำสำหรับการตั้งค่าอย่างปลอดภัยแทน

      ถ้าเผลอแชร์ key กับผู้อื่นหรือใส่ไว้ใน version control เช่น Git ให้เพิกถอน key ทันทีโดยทำตาม [Deleting an API key](https://cloud.ibm.com/docs/account?topic=account-userapikey&interface=ui#delete_user_key) บน IBM Cloud
    </Admonition>

<span id="access-credentials"></span>
## ค้นหา access credentials {#find-your-access-credentials}

1. ค้นหา API key (เรียกอีกชื่อว่า _API token_)  จาก [dashboard](https://quantum.cloud.ibm.com/) สร้าง API key แล้วคัดลอกไปเก็บในที่ปลอดภัยเพื่อใช้ยืนยันตัวตน  Token จะไม่แสดงอีกครั้ง  โปรดทราบว่าสามารถใช้ API key เดียวเพื่อเชื่อมต่อกับ region ใดก็ได้
2. ไม่บังคับ: ค้นหา instance ที่ต้องการใช้จากหน้า [Instances](https://quantum.cloud.ibm.com/instances)  วางเมาส์เหนือ CRN คลิกไอคอนเพื่อคัดลอก แล้วบันทึกในที่ปลอดภัยเพื่อใช้ระบุ instance

## ยืนยันตัวตนกับ IBM Cloud ใน environment ที่ไม่เชื่อถือ {#authenticate-to-ibm-cloud-in-an-untrusted-environment}

หากต้องการเริ่มต้น service ในสถานการณ์นี้ ให้ใช้โค้ดดังนี้:

    ```python
    from qiskit_ibm_runtime import QiskitRuntimeService

    # The credentials you enter will override any saved
    # account credentials that might be available locally.
    service = QiskitRuntimeService(
      # Use the 44-character API_KEY you created and saved from
      # the IBM Quantum Platform dashboard, and then delete
      # the key on the API keys page after entering this code:
      token="<your-API-key>",

      # Optionally specify an instance to use
      instance="<IBM Cloud CRN or instance name>"
      )
    ```

<span id="rest-untrusted"></span>
## ยืนยันตัวตนกับ REST API ใน environment ที่ไม่เชื่อถือ {#authenticate-to-the-rest-api-in-an-untrusted-environment}

ทำตามขั้นตอนเหล่านี้เพื่อใช้ API key โดยตรงในการยืนยันตัวตนกับ Qiskit Runtime REST API โดยที่ `<your_API_KEY>` คือ API_KEY 44 ตัวอักษรที่สร้างและบันทึกไว้จาก IBM Quantum Platform Home dashboard

1. ใช้ API key เพื่อสร้าง [bearer token](https://docs.quantum.ibm.com/api/qiskit-runtime-rest#authentication)  _bearer token_ คือ credential ชั่วคราวที่หมดอายุภายในไม่เกินหนึ่งชั่วโมง  หลังจาก token หมดอายุ ต้องสร้างใหม่เพื่อดำเนินการเรียก IBM Cloud หรือ service APIs อื่น ๆ ต่อ:
  ```bash
  curl -X POST 'https://iam.cloud.ibm.com/identity/token' \
  -H 'Content-Type: application/x-www-form-urlencoded' \
  -d 'grant_type=urn:ibm:params:oauth:grant-type:apikey&apikey=your_API_KEY'
  ```
  คัดลอกและบันทึก bearer token ที่ได้รับ: `"access_token": "<NEW_BEARER_TOKEN>"`
1. ยืนยันตัวตนคำขอไปยัง Qiskit Runtime REST API โดยรวม CRN และ bearer token ใน headers ของคำขอ

  ```bash
  curl -X 'GET' \
     'https://quantum.cloud.ibm.com/api/v1/usage' \
     '-H accept: application/json' \
     '-H authorization: Bearer <BEARER_TOKEN>' \
     '-H Service-CRN: <INSTANCE_CRN>'
  ```

1. เปลี่ยน API key หลังจากแต่ละการใช้งานโดยสร้าง API key ใหม่บนหน้าหลัก IBM Quantum Platform หรือทำตามคำแนะนำ [Deleting an API key](https://cloud.ibm.com/docs/account?topic=account-userapikey&interface=ui#delete_user_key) บน IBM Cloud

# จัดการค่าใช้จ่าย {#manage-cost}

คู่มือนี้อธิบายวิธีตั้งค่าขีดจำกัดค่าใช้จ่ายบน instance แผน Pay-As-You-Go นอกจากนี้ยังสามารถจัดการค่าใช้จ่ายได้โดย[ลดเวลา quantum ของ workload](/guides/minimize-time) และตั้งค่า[เวลาดำเนินการสูงสุด](/guides/max-execution-time) บน workload ของตัวเอง

_ขีดจำกัดค่าใช้จ่าย_ หมายถึงค่าใช้จ่ายรวมของ workload ทั้งหมดที่รันด้วย instance นี้นับตั้งแต่สร้างขึ้น และจะมีค่ามากกว่าหรือเท่ากับค่าใช้จ่ายรวมเสมอ หลังจาก instance ถึงจำนวนวินาทีรวมที่กำหนด จะไม่สามารถรัน workload เพิ่มเติมได้และไม่มีค่าใช้จ่ายเพิ่มขึ้นอีก

    <Admonition type="note">
      ขีดจำกัดค่าใช้จ่ายระบุเป็นดอลลาร์สหรัฐ (USD) เสมอ แล้วจึงแปลงเป็นวินาที runtime อย่างไรก็ตาม สำหรับการเรียกเก็บเงินรายเดือน จะมีการเรียกเก็บในสกุลเงินท้องถิ่นของตัวเอง ตามที่ระบุในบัญชี IBM Cloud&reg; เนื่องจากอัตราแลกเปลี่ยนสามารถผันผวนได้ ค่าใช้จ่ายสำหรับ X วินาที runtime อาจแตกต่างกันเมื่อคำนวณในตอนแรกเป็น USD กับเมื่อถูกเรียกเก็บจริงในสกุลเงินท้องถิ่น ดังนั้น หากสกุลเงินท้องถิ่นไม่ใช่ USD จำนวนเงินรวมที่ถูกเรียกเก็บสำหรับจำนวนวินาทีที่ระบุในฟิลด์นี้อาจแตกต่างจากจำนวนเงินดอลลาร์ที่ระบุ
    </Admonition>

<span id="cost-limit"></span>
## ตั้งค่าหรือเปลี่ยนขีดจำกัดค่าใช้จ่าย {#set-or-change-a-cost-limit}

มีหลายวิธีในการตั้งค่าหรือเปลี่ยนขีดจำกัดการใช้จ่าย:

* **IBM Quantum Platform**: เปิดหน้า [Instances](https://quantum.cloud.ibm.com/instances) หา instance ที่ต้องการจำกัด คลิกเมนู Actions ที่ปลายแถว แล้วคลิก **Edit details** ตั้งค่าขีดจำกัดค่าใช้จ่ายรวมหรือขีดจำกัดการใช้งาน แล้วคลิก **Save changes**
   <Admonition type="note">
   สามารถตั้งค่าขีดจำกัดค่าใช้จ่ายหรือการใช้งานได้เมื่อสร้าง instance ใหม่ภายใต้แผน Pay-As-You-Go เมื่อเลือก tile Pay-As-You-Go จะมีฟิลด์ปรากฏขึ้นพร้อมป้ายกำกับ "Total cost limit (Optional)" สามารถป้อนขีดจำกัดค่าใช้จ่ายรวมในฟิลด์นี้
   </Admonition>

* **API**: [ตั้งค่าขีดจำกัด instance เป็นวินาที](https://docs.quantum.ibm.com/api/qiskit-runtime-rest/tags/instances#tags__instances__operations__replace_instance_data)

   ```api
   PUT /v1/instances/configuration "instance_limit":<seconds>
   ```

* **IBM Cloud CLI**: สำหรับคำแนะนำในการใช้ CLI อ้างอิง [เริ่มต้นใช้งาน IBM Cloud CLI](https://cloud.ibm.com/docs/cli?topic=cli-getting-started)

   ```cli
      ibmcloud resource service-instance-update <instance_CRN>
      -p '{"instance_limit_seconds":"<seconds>"}'
   ```

## ดูขีดจำกัดค่าใช้จ่าย {#view-the-cost-limit}

มีหลายวิธีในการดูขีดจำกัดค่าใช้จ่ายปัจจุบัน:

* **IBM Quantum Platform**: ขีดจำกัดค่าใช้จ่ายสูงสุดจะแสดงบนแท็บ Pay-As-You-Go ของตาราง [Instances](https://quantum.cloud.ibm.com/instances)

* **API**:

   ```api
   GET /v1/instances/configuration
   ```
   ดู [เอกสาร REST API](https://docs.quantum.ibm.com/api/qiskit-runtime-rest/tags/instances)

* **IBM Cloud CLI**: สำหรับคำแนะนำในการใช้ CLI อ้างอิง [เริ่มต้นใช้งาน IBM Cloud CLI](https://cloud.ibm.com/docs/cli?topic=cli-getting-started)

   ```cli
   ibmcloud resource service-instance-update <instance_CRN> -p
   '{"instance_limit_seconds":null}'
   ```

## ลบขีดจำกัดค่าใช้จ่าย {#remove-a-cost-limit}

สามารถลบขีดจำกัดค่าใช้จ่ายได้หลายวิธี:

* **IBM Quantum Platform**: เปิดตาราง [Instances](https://quantum.cloud.ibm.com/instances) หา instance ที่ต้องการอัปเดต คลิกเมนู overflow ที่ปลายแถว แล้วคลิก **Edit details** ลบขีดจำกัดค่าใช้จ่ายรวมหรือการใช้งาน แล้วคลิก **Save changes**

* **API**: [ตั้งค่าขีดจำกัด instance เป็น `null`](https://docs.quantum.ibm.com/api/qiskit-runtime-rest/tags/instances)

   ```api
   PUT /v1/instances/configuration "instance_limit":null
   ```

* **IBM Cloud CLI**: สำหรับคำแนะนำในการใช้ CLI อ้างอิง [เริ่มต้นใช้งาน IBM Cloud CLI](https://cloud.ibm.com/docs/cli?topic=cli-getting-started)

   ```cli
   ibmcloud resource service-instance-update <instance_CRN> -p
   '{"instance_limit_seconds":null}'
   ```
<Admonition type="note">
เฉพาะ administrator และเจ้าของเท่านั้นที่สามารถแก้ไข instance ได้ ดูคู่มือ [สร้าง access policy และ access group](/guides/access-groups) สำหรับข้อมูลเกี่ยวกับการกำหนดสิทธิ์การเข้าถึง
</Admonition>

### สิ่งที่เกิดขึ้นเมื่อถึงขีดจำกัดค่าใช้จ่าย {#what-happens-when-the-cost-limit-is-reached}

เมื่อ instance ถึงขีดจำกัดค่าใช้จ่าย job ที่กำลังรันอยู่จะหยุดทำงาน สถานะจะถูกตั้งเป็น Canceled พร้อมเหตุผล Ran too long และผลลัพธ์บางส่วนที่มีอยู่จะถูกเก็บไว้

จะไม่สามารถส่ง job เพิ่มเติมโดยใช้ instance นี้ได้จนกว่าจะเพิ่มขีดจำกัดค่าใช้จ่าย

## วิธีดูว่าถูกเรียกเก็บเงินเท่าไร {#how-to-see-what-youre-being-charged}

จะได้รับใบแจ้งหนี้รายเดือนที่ให้รายละเอียดเกี่ยวกับค่าใช้จ่ายทรัพยากร สามารถตรวจสอบยอดใช้จ่ายได้ทุกเมื่อที่หน้า [IBM Cloud Billing and usage](https://cloud.ibm.com/billing)

นอกจากนี้ยังสามารถกำหนดค่าใช้จ่ายต่อ instance หรือต่อ job ได้ทุกเมื่อ

### ดูการใช้งาน {#view-usage}

ดูการใช้งานของ instance ที่หน้า [Analytics](https://quantum.cloud.ibm.com/analytics)

### ดูค่าใช้จ่าย instance {#view-instance-cost}

เพื่อกำหนดว่ามีการเรียกเก็บเงินจาก instance เท่าไรในรอบการเรียกเก็บเงินปัจจุบัน จากหน้า [Instances](https://quantum.cloud.ibm.com/instances) คลิก instance เพื่อเปิดหน้ารายละเอียด

ฟิลด์ที่เกี่ยวข้องกับค่าใช้จ่าย:

- **Billing cycle usage**: จำนวน _quantum time_ ที่ instance นี้ใช้ในรอบการเรียกเก็บเงินปัจจุบัน Quantum time คือระยะเวลาที่ QPU ทุ่มเทในการดำเนินการตามคำขอของผู้ใช้
- **Billing cycle cost**: ค่าใช้จ่ายรวมในการรัน job ในรอบการเรียกเก็บเงินปัจจุบัน
- **Total usage**: จำนวน quantum time ที่ instance นี้ใช้นับตั้งแต่สร้างขึ้น
- **Total cost**: ค่าใช้จ่ายรวมในการรัน job บน instance นี้นับตั้งแต่สร้างขึ้น เฉพาะ administrator เท่านั้นที่สามารถตั้งค่านี้ได้

สามารถดูรอบการเรียกเก็บเงินได้ที่ [หน้า Billing and usage](https://cloud.ibm.com/billing)

### ดูค่าใช้จ่าย job {#view-job-cost}

เพื่อกำหนดว่ามีการเรียกเก็บเงินจาก job แต่ละงานที่เกี่ยวข้องกับ instance เท่าไร จากหน้า [Instances](https://quantum.cloud.ibm.com/instances) คลิก instance เพื่อเปิดหน้ารายละเอียด จากนั้นที่ด้านซ้าย คลิก Jobs

คอลัมน์ที่เกี่ยวข้องกับค่าใช้จ่าย:

- **Usage**: จำนวน quantum time ที่ job นี้ใช้ Quantum time คือระยะเวลาที่ QPU ทุ่มเทในการดำเนินการตามคำขอของผู้ใช้
- **Cost**: ค่าใช้จ่ายรวมในการรัน job นี้

## ตั้งค่าการแจ้งเตือนค่าใช้จ่าย {#set-up-spending-notifications}

สามารถตั้งค่าการแจ้งเตือนค่าใช้จ่ายเพื่อรับแจ้งเมื่อบัญชีหรือ service เฉพาะถึงขีดจำกัดการใช้จ่ายที่กำหนด สำหรับข้อมูล ดู หน้า [IBM Cloud Account types](https://cloud.ibm.com/docs/account?topic=account-accounts) การแจ้งเตือนค่าใช้จ่าย IBM Cloud ต้องใช้ร่วมกับวิธีการจัดการค่าใช้จ่ายอื่นๆ ด้วยเหตุผลหลายประการ:

*   การแจ้งเตือนจะทำงานเฉพาะ *หลังจาก* ค่าใช้จ่ายเกินขีดจำกัดที่กำหนดเท่านั้น
*   ค่าใช้จ่ายถูกส่งไปยังระบบเรียกเก็บเงินทุกชั่วโมง ดังนั้น อาจมีความล่าช้านานระหว่างการส่ง job และการส่งการแจ้งเตือนค่าใช้จ่าย
*   ระบบเรียกเก็บเงินอาจใช้เวลาหลายวันในการส่งข้อมูลไปยังระบบออกใบแจ้งหนี้ ซึ่งอาจทำให้การแจ้งเตือนล่าช้าเพิ่มขึ้น สำหรับข้อมูลเพิ่มเติมเกี่ยวกับวิธีที่ระบบเรียกเก็บเงิน IBM Cloud ทำงาน ดู [การตั้งค่าการแจ้งเตือนค่าใช้จ่าย](https://cloud.ibm.com/docs/billing-usage?topic=billing-usage-spending)

# ย้ายจาก `BackendV1` ไปยัง `BackendV2` {#migrate-from-backendv1-to-backendv2}

คลาส [`BackendV1`](https://docs.quantum.ibm.com/api/qiskit/1.4/qiskit.providers.BackendV1) ของ Qiskit ถูก deprecated แล้วและจะถูกลบออกจากบริการ คู่มือการย้ายนี้อธิบายการปรับเปลี่ยนเล็กน้อยที่จำเป็น หากคุณใช้ provider ที่อัปเกรดจาก [`BackendV1`](https://docs.quantum.ibm.com/api/qiskit/1.4/qiskit.providers.BackendV1) ไปเป็น [`BackendV2`](https://docs.quantum.ibm.com/api/qiskit/qiskit.providers.BackendV2)

<Admonition type="note">
 หากคุณใช้เฉพาะ `qiskit_ibm_runtime` และ `qiskit_aer` ไม่จำเป็นต้องดำเนินการใด ๆ แพ็กเกจ `qiskit_ibm_runtime` ใช้ `BackendV2` มาตลอด และ `qiskit_aer` ใช้ `BackendV2` ตั้งแต่เวอร์ชัน 0.13
 </Admonition>

## การเปลี่ยนแปลงระดับสูงใน `BackendV2` {#high-level-changes-in-backendv2}

โมเดล [`Backend`](https://docs.quantum.ibm.com/api/qiskit/qiskit.providers.Backend) ของ Qiskit
ถูกออกแบบมาเพื่อให้ Qiskit SDK มี abstraction layer ที่ช่วยให้สามารถใช้เหตุผล
เกี่ยวกับคอมพิวเตอร์ควอนตัมภายในขอบเขตของ SDK การวนซ้ำครั้งแรกของโมเดลถูกนำเสนอ
พร้อมกับคลาส
[`BackendV1`](https://docs.quantum.ibm.com/api/qiskit/1.4/qiskit.providers.BackendV1)
คลาสนี้เก็บข้อมูล Backend ไว้ในชุดของ data container ได้แก่คลาส
[`BackendConfiguration`](https://docs.quantum.ibm.com/api/qiskit/1.4/qiskit.providers.models.BackendConfiguration)
และ [`BackendProperties`](https://docs.quantum.ibm.com/api/qiskit/1.4/qiskit.providers.models.BackendProperties)

คลาส [`BackendV2`](https://docs.quantum.ibm.com/api/qiskit/qiskit.providers.BackendV2)
ได้นิยามการเข้าถึงของผู้ใช้สำหรับคุณสมบัติ Backend ส่วนใหญ่ใหม่ เพื่อให้ทำงานร่วมกับ
data structure ของ Qiskit แบบ native และมีรูปแบบการเข้าถึงที่แบนราบกว่า แกนหลัก
ของโมเดล [`BackendV2`](https://docs.quantum.ibm.com/api/qiskit/qiskit.providers.BackendV2)
คือคลาส [`Target`](https://docs.quantum.ibm.com/api/qiskit/qiskit.transpiler.Target)
ซึ่งเป็นตัวแทนของ QPU ที่มีข้อจำกัดในการ transpile ที่ Qiskit ใช้เพื่อปรับแต่ง Circuit
สำหรับการรัน

Qiskit SDK ได้รับการอัปเดตให้ทำงานเฉพาะกับอินพุต [`BackendV2`](https://docs.quantum.ibm.com/api/qiskit/qiskit.providers.BackendV2) และ provider ส่วนใหญ่ได้อัปเกรดจาก [`BackendV1`](https://docs.quantum.ibm.com/api/qiskit/1.4/qiskit.providers.BackendV1) เป็น [`BackendV2`](https://docs.quantum.ibm.com/api/qiskit/qiskit.providers.BackendV2) แล้ว คาดว่า provider ที่มีอยู่จะ deprecated การเข้าถึงแบบเก่าเมื่อเป็นไปได้เพื่อให้การย้ายเป็นไปอย่างราบรื่น แต่ในที่สุดผู้ใช้จำเป็นต้องปรับโค้ดของตน

หลักการเบื้องหลัง [`BackendV2`](https://docs.quantum.ibm.com/api/qiskit/qiskit.providers.BackendV2)
คือข้อมูลส่วนใหญ่เกี่ยวกับ Backend จะอยู่ใน object
[`Target`](https://docs.quantum.ibm.com/api/qiskit/qiskit.transpiler.Target)
ของมัน และ attribute ของ Backend มักจะ query attribute `BackendV2.target`
เพื่อคืนข้อมูล อย่างไรก็ตาม ในหลายกรณี attribute เหล่านี้ให้เพียงส่วนหนึ่งของข้อมูล
ที่ target สามารถมีได้ ตัวอย่างเช่น `backend.coupling_map` คืน object
[`CouplingMap`](https://docs.quantum.ibm.com/api/qiskit/qiskit.transpiler.CouplingMap)
ที่สร้างจาก [`Target`](https://docs.quantum.ibm.com/api/qiskit/qiskit.transpiler.Target)
ที่เข้าถึงได้ใน attribute `BackendV2.target` อย่างไรก็ตาม target อาจมีคำสั่ง
ที่ทำงานบน Qubit มากกว่าสอง Qubit (ซึ่งไม่สามารถแทนใน
[`CouplingMap`](https://docs.quantum.ibm.com/api/qiskit/qiskit.transpiler.CouplingMap)
ได้) หรืออาจมีคำสั่งที่ทำงานบน Qubit เพียงบางส่วน (หรือ link สอง Qubit สำหรับ
คำสั่งสอง Qubit) ซึ่งจะไม่ถูกระบุรายละเอียดใน coupling map เต็มรูปแบบที่คืนโดย
`BackendV2.coupling_map` ดังนั้นขึ้นอยู่กับ use case ของคุณ อาจจำเป็นต้องมองลึก
กว่าแค่การเข้าถึงที่เทียบเท่าด้วย
[`BackendV2`](https://docs.quantum.ibm.com/api/qiskit/qiskit.providers.BackendV2)

## การเปลี่ยนแปลงเฉพาะใน `BackendV2` {#specific-changes-in-backendv2}

attribute ส่วนใหญ่มีการแทนที่โดยตรง ทำให้การย้ายง่ายขึ้น จุดเดียวที่ไม่ตรงกันระหว่าง interface คือใน [`CouplingMap`](https://docs.quantum.ibm.com/api/qiskit/qiskit.transpiler.CouplingMap)

ต่อไปนี้คือตารางตัวอย่างรูปแบบการเข้าถึงใน [`BackendV1`](https://docs.quantum.ibm.com/api/qiskit/1.4/qiskit.providers.BackendV1) และรูปแบบใหม่ด้วย [`BackendV2`](https://docs.quantum.ibm.com/api/qiskit/qiskit.providers.BackendV2)

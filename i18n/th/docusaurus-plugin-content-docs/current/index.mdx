## คอนเทนต์โอเพนซอร์สของ IBM Quantum {#ibm-quantums-open-source-content}

IBM มีสื่อการเรียนรู้ด้านควอนตัมคอมพิวติ้งมากมาย — ทั้งหมดเป็นโอเพนซอร์สภายใต้ CC BY-SA 4.0:

- **[Learning](https://quantum.cloud.ibm.com/learning)** — คอร์สแบบมีโครงสร้างตั้งแต่พื้นฐานควอนตัมจนถึงหัวข้อขั้นสูง
- **Develop locally** — [install Qiskit](https://docs.quantum.ibm.com/guides/install-qiskit) on your own laptop for serious development work.
- **[Tutorials](https://quantum.cloud.ibm.com/docs/en/tutorials)** — กว่า 40 tutorials เรื่อง transpilation, error mitigation และอื่นๆ
- **[Documentation](https://quantum.cloud.ibm.com/docs/en/guides)** — คู่มือและ API reference สำหรับ Qiskit
- **[Source repo](https://github.com/Qiskit/documentation)** — คอนเทนต์ทั้งหมดบน GitHub

[Quantum Platform](https://quantum.cloud.ibm.com) ของเขาอัปเดตอยู่เสมอและออกแบบมาดี — เป็นสถานที่ที่ดีที่สุดสำหรับการอ่าน เรียน และค้นหาข้อมูล

## โปรเจกต์นี้เพิ่มอะไรบ้าง {#what-this-project-adds}

[เอกสาร Qiskit](https://github.com/Qiskit/documentation) ของ IBM เป็นโอเพนซอร์ส (CC BY-SA 4.0) แต่เว็บแอปพลิเคชันของเขาไม่ใช่ doQumentation เพิ่ม frontend แบบโอเพนซอร์สพร้อมการรันโค้ด live, การฉีด credentials อัตโนมัติ และ simulator mode

ติดตั้งได้ทุกที่ — ตั้งแต่ [GitHub Pages](https://doqumentation.org) ถึง [Docker](https://github.com/JanLahmann/doQumentation/pkgs/container/doqumentation) ถึง [RasQberry](https://rasqberry.org/) ดู[ฟีเจอร์ทั้งหมด](/features)

## พร้อมแล้วก็ไปต่อได้เลย {#when-youre-ready-for-more}

doQumentation ช่วยให้คุณ**ก้าวแรกกับ Qiskit** — อ่าน รันโค้ด และสำรวจ — โดยไม่ต้องติดตั้งอะไรเลย พอพร้อมแล้วก็ไปที่แพลตฟอร์มอย่างเป็นทางการของ IBM ได้เลย:

- **รันบนฮาร์ดแวร์จริง** — สร้าง [บัญชี IBM Cloud](https://quantum.cloud.ibm.com/registration) ฟรีและ Open Plan instance เพื่อรัน circuits บนควอนตัมคอมพิวเตอร์จริง
- **คอนเทนต์ล่าสุด** — [แพลตฟอร์มการเรียนรู้ Qiskit](https://quantum.cloud.ibm.com/learning) ของ IBM มีคอร์ส tutorials และสิ่งใหม่ๆ ล่าสุด (บางอย่างอาจยังไม่อยู่ใน doQumentation)
- **พัฒนาในเครื่อง** — [ติดตั้ง Qiskit](https://docs.quantum.ibm.com/guides/install-qiskit) บนแล็ปท็อปของคุณเองสำหรับงานพัฒนาจริงจัง

*สร้างจาก [เอกสารโอเพนซอร์สของ IBM Quantum](https://github.com/Qiskit/documentation) ขอบคุณทีม IBM Quantum ที่เปิดให้ใช้คอนเทนต์นี้ฟรีภายใต้ CC BY-SA 4.0*

## การรันโค้ด {#code-execution}

คลิก **Run** ที่บล็อกโค้ดไหนก็ได้ การคลิกครั้งแรกจะเริ่ม Jupyter kernel ผ่าน [Binder](https://mybinder.org) หรือ [IBM Code Engine](/jupyter-settings#code-engine) หลังจากนั้นการรันจะทันที

- **Simulator Mode (ไม่ต้องมีบัญชี)** — เปิดใช้ [Simulator Mode](/jupyter-settings#simulator-mode) เพื่อรัน notebooks ทั้งหมดด้วย AerSimulator หรือ FakeBackends แบบ noise-model ไม่ต้องตั้งค่าใดๆ

- **IBM Quantum Hardware** — ใส่ API token และ CRN ของคุณใน [การตั้งค่า](/jupyter-settings#ibm-quantum) ครั้งเดียว — credentials จะถูกฉีดอัตโนมัติในทุกครั้งที่รัน [สร้างบัญชีฟรี](https://quantum.cloud.ibm.com/registration) ถ้ายังไม่มี

<details>
<summary>Execution backends ที่ใช้ได้</summary>

ทุก tutorial มีบล็อกโค้ดที่รันได้ คลิก **Run** เพื่อรันด้วย backend หนึ่งในสี่ตัวนี้:

1. **Binder** (ค่าเริ่มต้นบน GitHub Pages) — Jupyter kernel ระยะไกลฟรีผ่าน [mybinder.org](https://mybinder.org)
2. **IBM Code Engine** — Serverless Jupyter บนบัญชี IBM Cloud ของคุณ Cold start ใช้เวลาเป็นวินาทีแทนนาที [ตั้งค่า](/jupyter-settings#code-engine)
3. **Local Jupyter (Docker / RasQberry)** — เชื่อมต่อกับ Jupyter server ภายในเครื่องที่ติดตั้ง Qiskit ไว้แล้ว
4. **Custom server** — ชี้ไปยัง Jupyter endpoint ใดก็ได้ใน [การตั้งค่า](/jupyter-settings#advanced)

  <strong>No IBM Quantum account?</strong> Enable <a href="/jupyter-settings#execution-mode">Simulator Mode</a> in Settings to run all code without signing up.

เมื่อมี backend หลายตัว ให้เลือกว่าจะใช้ตัวไหนใน [การตั้งค่า](/jupyter-settings#backend-selection)

</details>

<details>
<summary>ตัวเลือกการ deploy</summary>

doQumentation มีให้ใช้ในรูปแบบ:

- **[GitHub Pages](https://doqumentation.org)** — ไซต์แบบ static พร้อม [Binder](https://mybinder.org) หรือ [IBM Code Engine](/jupyter-settings#code-engine) สำหรับการรันโค้ดระยะไกล
- **[Docker](https://github.com/JanLahmann/doQumentation/pkgs/container/doqumentation)** — Full stack ในเครื่อง ใช้งานออฟไลน์ได้
- **[RasQberry](https://rasqberry.org/)** — Self-hosted บน Raspberry Pi พร้อม Jupyter kernel ภายในเครื่อง

</details>

<details>
<summary>รันในเครื่องด้วย Podman / Docker</summary>

ติดตั้ง [Podman Desktop](https://podman-desktop.io/) หรือ [Docker Desktop](https://www.docker.com/products/docker-desktop/) (Mac / Windows / Linux) แล้วรัน:

```bash
# Full stack: site + Jupyter + Qiskit (~3 GB)
podman run -p 8080:80 -p 8888:8888 ghcr.io/janlahmann/doqumentation:jupyter
```

เปิด [http://localhost:8080](http://localhost:8080) — การรันโค้ดทำงานในเครื่อง ไม่ต้องรอ Binder

สำหรับเวอร์ชันเบาที่ไม่มีการรันโค้ดในเครื่อง (~60 MB):

```bash
# Static site only — code execution still works via Binder
podman run -p 8080:80 ghcr.io/janlahmann/doqumentation:latest
```

ใช้ Docker แทน? แค่เปลี่ยน `podman` เป็น `docker` — คำสั่งเหมือนกันทุกอย่าง Images รองรับหลาย arch (`linux/amd64` + `linux/arm64`) ทำให้ Apple Silicon Macs ทำงานได้แบบ native

</details>

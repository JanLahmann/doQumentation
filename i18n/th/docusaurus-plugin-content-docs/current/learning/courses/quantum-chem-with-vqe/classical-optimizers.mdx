## เขียนโค้ด classical optimizer {#coding-a-classical-optimizer}

ในบทเรียนก่อนหน้า คุณได้เรียนรู้วิธีสร้าง Hamiltonian ที่เหมาะสำหรับใช้บนคอมพิวเตอร์ควอนตัมและวิธีสร้าง variational circuit คุณยังรู้ด้วยว่า variational circuit (หรือ ansatz) มีพารามิเตอร์ที่ต้องปรับ และพารามิเตอร์ที่ดีที่สุดคือสิ่งที่ให้ค่า cost function หรือพลังงานต่ำสุด ดังนั้นปัญหาของเราลดเหลือการค้นหาใน parameter space เพื่อหาชุดที่ดีที่สุด งานส่วนใหญ่ใน classical optimizers ได้ถูกทำเสร็จแล้วสำหรับเรา เนื่องจาก optimizer ที่ยอดเยี่ยมมีให้ใช้จากหลายแหล่ง

ในบทเรียนนี้ คุณจะได้เรียนรู้:
- วิธีที่ classical optimizers เข้าร่วมในการคำนวณ VQE
- classical optimizers ที่มีให้ใช้จาก SciPy มีอะไรบ้าง
- optimizer ใดที่ __ยังไม่มี__ ใน SciPy และวิธีเสริมโดยใช้ `qiskit.algorithms` ในระหว่างนี้
- ตัวเลือกที่มีสำหรับ optimizer เหล่านี้และความสำคัญสำหรับการคำนวณควอนตัม

[SciPy](https://scipy.org/) คือ Python library โอเพนซอร์สฟรีที่มีแพ็กเกจสำหรับหลายด้านของการคำนวณทางวิทยาศาสตร์ รวมถึงการ optimization โดยเฉพาะ SciPy มีแพ็กเกจ optimization ที่รวม [minimize](https://docs.scipy.org/doc/scipy/reference/generated/scipy.optimize.minimize.html) ไว้ด้วย:

```from scipy.optimize import minimize```
This minimize function has several arguments, but the most relevant arguments for quantum chemistry are:
- The cost function (`cost_func`). This is related to the Hamiltonian, but also includes some complexities, such as determining the expectation value by using Estimator, and in the case of excited state calculations, might include orthogonality conditions.
- An initial state (x0) for the system, often the Hartree Fock state
- Other arguments, including arguments of the cost function itself
- The method set to the classical optimizer you select
- Options for the classical optimizer (not to be confused with Session options discussed in the next section)

Some example code is shown below. We restrict our discussion here to the last two arguments.

```minimize(
    cost_func,
    x0,
    args=(ansatz, hamiltonian, estimator),
    method="cobyla",
    options={"maxiter": 200})
```
SciPy มีเอกสารสำหรับ minimize methods ที่มีทั้งหมด นี่คือตัวอย่างที่น่าสนใจไม่กี่ตัว ซึ่งล้วนเป็นวิธีสำหรับการหาค่าต่ำสุดของฟังก์ชัน scalar ที่มีหนึ่งหรือมากกว่าตัวแปร:
- [cobyla](https://docs.scipy.org/doc/scipy/reference/optimize.minimize-cobyla.html#optimize-minimize-cobyla): อัลกอริทึม Optimization BY Linear Approximation (COBYLA)
- [slsqp](https://docs.scipy.org/doc/scipy/reference/optimize.minimize-slsqp.html#optimize-minimize-slsqp): Sequential Least Squares Programming (SLSQP)
- [nelder-mead](https://docs.scipy.org/doc/scipy/reference/optimize.minimize-neldermead.html#optimize-minimize-neldermead) อัลกอริทึม Nelder-Mead

classical optimization algorithms ที่มีส่วนใหญ่เป็น local minimizer กล่าวคือใช้วิธีต่าง ๆ เพื่อหา local minima แต่ไม่ได้รับประกันว่าจะหา global minima ได้ classical optimizer บางตัวประมาณ gradient โดยตรงและใช้มันเพื่อหา local minima ตัวอื่น ๆ อาจใช้การประมาณค่าเชิงเส้นหรือเชิงกำลังสองต่อเนื่องของ objective function เพื่อหาค่าต่ำสุด

อัลกอริทึมเหล่านี้มีหลาย options ที่ร่วมกัน แต่มีความแตกต่างเล็กน้อย ตัวอย่างเช่น ทุกตัวมี option ให้ระบุจำนวนการวนซ้ำสูงสุดโดยใช้ notation `'maxiter': 200` จากข้างต้น ทุกตัวมี option บางอย่างที่ระบุเกณฑ์หยุดตามค่าฟังก์ชันหรือค่าตัวแปร แม้เกณฑ์เหล่านี้จะต่างกันเล็กน้อยสำหรับแต่ละอัลกอริทึม ตัวอย่างเช่น COBYLA ให้คุณระบุ tolerance (เช่น `'tol': 0.0001`) ที่เป็นขอบล่างของ "trust region" ส่วน SLSQP ให้คุณระบุเป้าหมายความแม่นยำของฟังก์ชันที่ใช้ในเกณฑ์หยุด ('ftol') ส่วน Nelder-Mead ให้คุณระบุ tolerance ในความแตกต่างระหว่าง parameter ($x$) ที่เดาต่อเนื่องกัน (xatol) หรือ tolerance ในความแตกต่างระหว่างค่าที่ได้จาก cost function $f(x)$ ต่อเนื่องกัน (fatol) (หรือทั้งสองอย่าง)
สำหรับรายการครบถ้วนของอัลกอริทึมและ options ที่มี เยี่ยมชม [SciPy's minimize documentation](https://docs.scipy.org/doc/scipy/reference/generated/scipy.optimize.minimize.html)

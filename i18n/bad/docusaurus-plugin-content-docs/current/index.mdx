## Was des Projekt hinzuefüegt {#what-this-project-adds}

Em IBM sini [Qiskit-Dokumentation](https://github.com/Qiskit/documentation) isch Open Source (CC BY-SA 4.0), awwer ihri Webawwendung isch's nit. doQumentation fiegt e Open-Source-Frontend mit Live-Code-Usfierung, automatischer Credential-Injektion un Simulator-Modus hinzue.

Iberall iisetzbor – vo [GitHub Pages](https://doqumentation.org) iber [Docker](https://github.com/JanLahmann/doQumentation/pkgs/container/doqumentation) bis zu [RasQberry](https://rasqberry.org/). Luege Sie [alli Funktione](/features).

## Code-Usfierung {#code-execution}

Klicket bi jedem Codeblock uf **Run**. Dr erscht Klick startet e koschtelose Jupyter-Kernel iber [Binder](https://mybinder.org) (1–2 Min.). Dnoch laufet d Usfierige sofort.

- **Simulator-Modus (kei Konto neetig)** — Aktivieret de [Simulator-Modus](/jupyter-settings#simulator-mode), zum alli Notebooks mit AerSimulator oder FakeBackends mit Ruschmodell uszfiere. Null Iirichtungsufwand neetig.

- **IBM Quantum Hardware** — Gebet eiere API-Token un CRN emol in de [Iistellige](/jupyter-settings#ibm-quantum) ii — d Zugangsdatte werret bi jeder Usfierung automatisch iigfiegt. [Erschtellet e koschteloses Konto](https://quantum.cloud.ibm.com/registration), falls ihr noch keis hent.

<details>
<summary>Verfiegbori Usfierungs-Backends</summary>

Jedes Tutorial het usfierbare Codebleck. Klicket uf **Run**, zum sie mit einem vo drei Backends uszfiere:

1. **Binder** (Standard uf GitHub Pages) — Koschtelose Remote-Jupyter-Kernel iber [mybinder.org](https://mybinder.org)
2. **Lokals Jupyter (Docker / RasQberry)** — Verbindet sich mit em lokale Jupyter-Server mit vorinstalliertem Qiskit
3. **Benutzerdefinierte Server** — Verwiiset in de [Iistellige](/jupyter-settings#advanced) uf e beliebige Jupyter-Endpunkt

</details>

<details>
<summary>Bereitstellungsoptione</summary>

doQumentation isch verfiegbor als:

- **[GitHub Pages](https://doqumentation.org)** — Statischi Websitte mit [Binder](https://mybinder.org) fir Remote-Code-Usfierung
- **[Docker](https://github.com/JanLahmann/doQumentation/pkgs/container/doqumentation)** — Vollständige Stack lokal, offline-fehig
- **[RasQberry](https://rasqberry.org/)** — Selwer ghostet uf Raspberry Pi mit lokalem Jupyter-Kernel

</details>

<details>
<summary>Lokal usfiere mit Podman / Docker</summary>

Installieret [Podman Desktop](https://podman-desktop.io/) oder [Docker Desktop](https://www.docker.com/products/docker-desktop/) (Mac / Windows / Linux), deno fieret us:

```bash
# Vollständige Stack: Websitte + Jupyter + Qiskit (~3 GB)
podman run -p 8080:80 -p 8888:8888 ghcr.io/janlahmann/doqumentation:jupyter
```

Offnet [http://localhost:8080](http://localhost:8080) — Code-Usfierung funktioniert lokal, kei Binder-Wartezitt.

Fir e leichtgwichtigi Version ohni lokali Code-Usfierung (~60 MB):

```bash
# Numme statischi Websitte — Code-Usfierung funktioniert wiiterhin iber Binder
podman run -p 8080:80 ghcr.io/janlahmann/doqumentation:latest
```

Ihr verwennet Docker? Ersetzet eifach `podman` durch `docker` — d Befehle sin identisch. D Images sin Multi-Arch (`linux/amd64` + `linux/arm64`), so dass Apple Silicon Macs nativ funktioniere.

</details>

## Code-Ausfierung {#code-execution}

Gliggt bei jeedn Codeblock uff **Run**. Der erschde Gligg startet en gostenlosen Jupyter-Kernel ieeber [Binder](https://mybinder.org) (1–2 Min.). Danach geht's sofort.

- **Simulator-Modus (gee Gonto neetich)** — Aggdiviert den [Simulator-Modus](/jupyter-settings#simulator-mode), damitt ihr alle Notebooks mit AerSimulator oder FakeBackends mit Rauschmodell ausfier'n gennt. Null Einrichtungsaufwand neetich.

- **IBM Quantum Hardware** — Gebbt eiern API-Token un CRN eemol in de [Einstellungen](/jupyter-settings#ibm-quantum) ei — de Zugangsdaten werd'n bei jeder Ausfierung automatisch eingefigt. [Erschdellt e gostenloses Gonto](https://quantum.cloud.ibm.com/registration), falls ihr noch keens habbt.

<details>
<summary>Verfiegbare Ausfierungs-Backends</summary>

Jeedes Tutorial hat ausfierbare Codeblegge. Gliggt uff **Run**, damitt ihr se mit eenm von drei Backends ausfier'n gennt:

1. **Binder** (Standard uff GitHub Pages) — Gostenlosen Remote-Jupyter-Kernel ieeber [mybinder.org](https://mybinder.org)
2. **Logales Jupyter (Docker / RasQberry)** — Verbindet eich mit'm logalen Jupyter-Server mit vorinstalliertem Qiskit
3. **Benutzerdefinierder Server** — Verwiest in de [Einstellungen](/jupyter-settings#advanced) uff en beliebich'n Jupyter-Endpunkt

</details>

<details>
<summary>Bereitstellungsoptschjoon'n</summary>

doQumentation is verfiegbar als:

- **[GitHub Pages](https://doqumentation.org)** — Statische Webseid mit [Binder](https://mybinder.org) fier Remote-Code-Ausfierung
- **[Docker](https://github.com/JanLahmann/doQumentation/pkgs/container/doqumentation)** — Vollständiger Stack logal, offline-fähich
- **[RasQberry](https://rasqberry.org/)** — Selbst gehostet uff nem Raspberry Pi mit logalem Jupyter-Kernel

</details>

<details>
<summary>Logal ausfier'n mit Podman / Docker</summary>

Installiert [Podman Desktop](https://podman-desktop.io/) oder [Docker Desktop](https://www.docker.com/products/docker-desktop/) (Mac / Windows / Linux), dann fiert aus:

```bash
# Vollständicher Stack: Webseid + Jupyter + Qiskit (~3 GB)
podman run -p 8080:80 -p 8888:8888 ghcr.io/janlahmann/doqumentation:jupyter
```

Macht [http://localhost:8080](http://localhost:8080) uff — Code-Ausfierung funggsjooniert logal, gee Binder-Warte-Zeit.

Fier e leichtgewichtiche Varjante ohne logale Code-Ausfierung (~60 MB):

```bash
# Nur statische Webseid — Code-Ausfierung funggsjooniert weiterhin ieeber Binder
podman run -p 8080:80 ghcr.io/janlahmann/doqumentation:latest
```

Benutzt ihr Docker? Ersetzt einfach `podman` durch `docker` — de Befehle sin identisch. De Images sin Multi-Arch (`linux/amd64` + `linux/arm64`), sodass Apple Silicon Macs nativ funggsjoonieren.

</details>

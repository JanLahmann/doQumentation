## Contenuti open-source di IBM Quantum {#ibm-quantums-open-source-content}

IBM fornisce una vasta quantità di materiale didattico sul quantum computing — tutto open source sotto licenza CC BY-SA 4.0:

- **[Learning](https://quantum.cloud.ibm.com/learning)** — Corsi strutturati dalle basi quantistiche agli argomenti avanzati
- **[Tutorials](https://quantum.cloud.ibm.com/docs/en/tutorials)** — Oltre 40 tutorial su transpilazione, mitigazione degli errori e altro
- **[Documentation](https://quantum.cloud.ibm.com/docs/en/guides)** — Guide e riferimenti API per Qiskit
- **[Source repo](https://github.com/Qiskit/documentation)** — Tutti i contenuti su GitHub

La loro [Quantum Platform](https://quantum.cloud.ibm.com) è sempre aggiornata e ben progettata — il miglior posto per leggere, imparare e consultare.

## Cosa aggiunge questo progetto {#what-this-project-adds}

La [documentazione Qiskit](https://github.com/Qiskit/documentation) di IBM è open source (CC BY-SA 4.0), ma la loro applicazione web non lo è. doQumentation aggiunge un frontend open-source con esecuzione di codice dal vivo, iniezione automatica delle credenziali e modalità simulatore.

Distribuibile ovunque — da [GitHub Pages](https://doqumentation.org) a [Docker](https://github.com/JanLahmann/doQumentation/pkgs/container/doqumentation) a [RasQberry](https://rasqberry.org/). Veda [tutte le funzionalità](/features).

## Esecuzione del codice {#code-execution}

Clicchi **Run** su qualsiasi blocco di codice. Il primo click avvia un kernel Jupyter gratuito tramite [Binder](https://mybinder.org) (1–2 min). Successivamente, le esecuzioni sono istantanee.

- **Modalità Simulatore (nessun account necessario)** — Attivi la [Modalità Simulatore](/jupyter-settings#simulator-mode) per eseguire tutti i notebook con AerSimulator o FakeBackends con modelli di rumore. Nessuna configurazione richiesta.

- **Hardware IBM Quantum** — Inserisca il suo token API e CRN nelle [Impostazioni](/jupyter-settings#ibm-quantum) una volta — le credenziali vengono iniettate automaticamente ad ogni esecuzione. [Crei un account gratuito](https://quantum.cloud.ibm.com/registration) se non ne ha uno.

<details>
<summary>Backend di esecuzione disponibili</summary>

Ogni tutorial ha blocchi di codice eseguibili. Clicchi **Run** per eseguirli usando uno dei tre backend:

1. **Binder** (predefinito su GitHub Pages) — Kernel Jupyter remoto gratuito tramite [mybinder.org](https://mybinder.org)
2. **Jupyter locale (Docker / RasQberry)** — Si connette al server Jupyter locale con Qiskit preinstallato
3. **Server personalizzato** — Punti a qualsiasi endpoint Jupyter nelle [Impostazioni](/jupyter-settings#advanced)

</details>

<details>
<summary>Opzioni di distribuzione</summary>

doQumentation è disponibile come:

- **[GitHub Pages](https://doqumentation.org)** — Sito statico con [Binder](https://mybinder.org) per l'esecuzione remota del codice
- **[Docker](https://github.com/JanLahmann/doQumentation/pkgs/container/doqumentation)** — Stack completo localmente, utilizzabile offline
- **[RasQberry](https://rasqberry.org/)** — Self-hosted su Raspberry Pi con kernel Jupyter locale

</details>

<details>
<summary>Esegua localmente con Podman / Docker</summary>

Installi [Podman Desktop](https://podman-desktop.io/) o [Docker Desktop](https://www.docker.com/products/docker-desktop/) (Mac / Windows / Linux), poi esegua:

```bash
# Stack completo: sito + Jupyter + Qiskit (~3 GB)
podman run -p 8080:80 -p 8888:8888 ghcr.io/janlahmann/doqumentation:jupyter
```

Apra [http://localhost:8080](http://localhost:8080) — l'esecuzione del codice funziona localmente, nessun tempo di attesa per Binder.

Per una versione leggera senza esecuzione locale del codice (~60 MB):

```bash
# Solo sito statico — l'esecuzione del codice funziona comunque tramite Binder
podman run -p 8080:80 ghcr.io/janlahmann/doqumentation:latest
```

Usa Docker invece? Sostituisca semplicemente `podman` con `docker` — i comandi sono identici. Le immagini sono multi-arch (`linux/amd64` + `linux/arm64`), quindi i Mac Apple Silicon funzionano nativamente.

</details>

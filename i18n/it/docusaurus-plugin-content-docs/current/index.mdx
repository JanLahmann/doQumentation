## Contenuti open-source di IBM Quantum {#ibm-quantums-open-source-content}

IBM fornisce una vasta quantità di materiale didattico sul quantum computing — tutto open source sotto licenza CC BY-SA 4.0:

- **[Learning](https://quantum.cloud.ibm.com/learning)** — Corsi strutturati dalle basi quantistiche agli argomenti avanzati
- **[Tutorials](https://quantum.cloud.ibm.com/docs/en/tutorials)** — Oltre 40 tutorial su transpilazione, mitigazione degli errori e altro
- **[Documentation](https://quantum.cloud.ibm.com/docs/en/guides)** — Guide e riferimenti API per Qiskit
- **[Source repo](https://github.com/Qiskit/documentation)** — Tutti i contenuti su GitHub

La loro [Quantum Platform](https://quantum.cloud.ibm.com) è sempre aggiornata e ben progettata — il miglior posto per leggere, imparare e consultare.

## Cosa aggiunge questo progetto {#what-this-project-adds}

La [documentazione Qiskit](https://github.com/Qiskit/documentation) di IBM è open source (CC BY-SA 4.0), ma la loro applicazione web non lo è. doQumentation aggiunge un frontend open-source con esecuzione di codice dal vivo, iniezione automatica delle credenziali e modalità simulatore.

Distribuibile ovunque — da [GitHub Pages](https://doqumentation.org) a [Docker](https://github.com/JanLahmann/doQumentation/pkgs/container/doqumentation) a [RasQberry](https://rasqberry.org/). Vedi [tutte le funzionalità](/features).

## Quando sei pronto per qualcosa di più {#when-youre-ready-for-more}

doQumentation ti aiuta a fare i tuoi **primi passi con Qiskit** — leggere, eseguire codice ed esplorare — senza dover installare nulla. Quando sei pronto per andare oltre, vai sulla piattaforma ufficiale di IBM:

- **Esegui su hardware reale** — crea un [account IBM Cloud](https://quantum.cloud.ibm.com/registration) gratuito e un'istanza Open Plan per eseguire circuiti su veri computer quantistici.
- **Contenuti più recenti** — la [piattaforma di apprendimento Qiskit](https://quantum.cloud.ibm.com/learning) di IBM offre i corsi, i tutorial e le novità più recenti (alcuni potrebbero non essere ancora in doQumentation).
- **Sviluppa localmente** — [installa Qiskit](https://docs.quantum.ibm.com/guides/install-qiskit) sul tuo laptop per un lavoro di sviluppo serio.

*Costruito sulla [documentazione open source di IBM Quantum](https://github.com/Qiskit/documentation). Grazie al team di IBM Quantum per aver reso questi contenuti liberamente disponibili sotto licenza CC BY-SA 4.0.*

## Esecuzione del codice {#code-execution}

Clicca **Run** su qualsiasi blocco di codice. Il primo click avvia un kernel Jupyter tramite [Binder](https://mybinder.org) o [IBM Code Engine](/jupyter-settings#code-engine). Successivamente, le esecuzioni sono istantanee.

- **Modalità Simulatore (nessun account necessario)** — Attiva la [Modalità Simulatore](/jupyter-settings#execution-mode) per eseguire tutti i notebook con AerSimulator o FakeBackends con modelli di rumore. Nessuna configurazione richiesta.

- **Hardware IBM Quantum** — Inserisci il tuo token API e CRN nelle [Impostazioni](/jupyter-settings#ibm-quantum) una volta — le credenziali vengono iniettate automaticamente ad ogni esecuzione. [Crea un account gratuito](https://quantum.cloud.ibm.com/registration) se non ne hai uno.

<details>
<summary>Backend di esecuzione disponibili</summary>

Ogni tutorial ha blocchi di codice eseguibili. Clicca **Run** per eseguirli usando uno dei quattro backend:

1. **Binder** (predefinito su GitHub Pages) — Kernel Jupyter remoto gratuito tramite [mybinder.org](https://mybinder.org)
2. **IBM Code Engine** — Jupyter serverless sul tuo account IBM Cloud. Avvio a freddo in pochi secondi anziché minuti. [Configura](/jupyter-settings#code-engine)
3. **Jupyter locale (Docker / RasQberry)** — Si connette al server Jupyter locale con Qiskit preinstallato
4. **Server personalizzato** — Punta a qualsiasi endpoint Jupyter nelle [Impostazioni](/jupyter-settings#advanced)

Quando più backend sono disponibili, scegli quale usare dalle [Impostazioni](/jupyter-settings#backend-selection).

</details>

<details>
<summary>Opzioni di distribuzione</summary>

doQumentation è disponibile come:

- **[GitHub Pages](https://doqumentation.org)** — Sito statico con [Binder](https://mybinder.org) o [IBM Code Engine](/jupyter-settings#code-engine) per l'esecuzione remota del codice
- **[Docker](https://github.com/JanLahmann/doQumentation/pkgs/container/doqumentation)** — Stack completo localmente, utilizzabile offline
- **[RasQberry](https://rasqberry.org/)** — Self-hosted su Raspberry Pi con kernel Jupyter locale

</details>

<details>
<summary>Esegui localmente con Podman / Docker</summary>

Installa [Podman Desktop](https://podman-desktop.io/) o [Docker Desktop](https://www.docker.com/products/docker-desktop/) (Mac / Windows / Linux), poi esegui:

```bash
# Full stack: site + Jupyter + Qiskit (~3 GB)
podman run -p 8080:80 -p 8888:8888 ghcr.io/janlahmann/doqumentation:jupyter
```

Apri [http://localhost:8080](http://localhost:8080) — l'esecuzione del codice funziona localmente, senza tempi di attesa per Binder.

Per una versione leggera senza esecuzione locale del codice (~60 MB):

```bash
# Static site only — code execution still works via Binder
podman run -p 8080:80 ghcr.io/janlahmann/doqumentation:latest
```

Usi Docker invece? Sostituisci semplicemente `podman` con `docker` — i comandi sono identici. Le immagini sono multi-arch (`linux/amd64` + `linux/arm64`), quindi i Mac Apple Silicon funzionano nativamente.

</details>

## Contesto {#background}

Questo tutorial dimostra come implementare la **Compilazione Quantistica Approssimata** utilizzando reti tensoriali (AQC-Tensor) con Qiskit per migliorare le prestazioni dei circuiti quantistici. AQC-Tensor comprime i circuiti Trotter profondi in circuiti più superficiali e compatibili con l'hardware, preservando l'accuratezza della simulazione.

### Come funziona AQC-Tensor {#how-aqc-tensor-works}

Considera la simulazione di un hamiltoniano $H$ per un tempo totale $t$ utilizzando $k$ passi Trotter. Il circuito Trotter completo è:

$$
U_{\text{full}} = \left[U_{\text{Trotter}}(t/k)\right]^k
$$

Un approccio ingenuo utilizza pochi passi Trotter per mantenere la profondità del circuito gestibile, ma questo introduce un errore Trotter significativo. AQC-Tensor risolve questa tensione separando accuratezza da profondità:

1. **Circuito target (alta precisione, profondo):** Costruire un circuito Trotter con molti passi&mdash;ad esempio, $10k$&mdash;per lo stesso tempo di evoluzione. Questo circuito ha molto meno errore Trotter, ma è troppo profondo per l'hardware. Poiché viene simulato solo classicamente come stato a prodotto matriciale (MPS), la profondità non è un problema.

2. **Circuito ansatz (bassa profondità, parametrizzato):** Definire un circuito parametrizzato $V(\theta)$ con la stessa struttura di un circuito Trotter a singolo passo. Inizializzarlo in modo che $V(\theta_{\text{init}}) = U_{\text{Trotter}}(t/k)$, poi ottimizzare iterativamente $\theta$ in modo che $V(\theta)$ riproduca lo stato target ad alta precisione nel modo più fedele possibile.

Il risultato è un circuito che mantiene la profondità di un singolo passo Trotter ma raggiunge la precisione di molti, rendendolo fattibile per l'hardware quantistico a breve termine.

### Quando utilizzare AQC-Tensor {#when-to-use-aqc-tensor}

AQC-Tensor è più efficace quando:

- **La profondità del circuito supera i tempi di coerenza dell'hardware.** Se una simulazione Trotter richiede più passi Trotter di quanti il dispositivo possa supportare, AQC-Tensor può comprimere l'evoluzione in un circuito più superficiale.
- **L'entanglement rimane trattabile classicamente.** L'entanglement totale in uno stato evoluto nel tempo dipende principalmente dal tempo di evoluzione $t$, non dal numero di passi Trotter $k$. Ciò significa che un circuito target con $10k$ passi è tipicamente non più difficile da rappresentare come MPS rispetto a uno con $k$ passi, purché $t$ sia sufficientemente breve da mantenere gestibili le dimensioni dei bond.
- **Esiste un ansatz naturale.** Poiché l'ansatz rispecchia la struttura di un circuito Trotter, fornisce un punto di partenza fisicamente motivato con parametri iniziali ben definiti, evitando i problemi di convergenza che possono affliggere ansatz variazionali arbitrari.

Questo approccio si contrappone alla compressione generica dei circuiti: invece di cercare di approssimare un unitario arbitrario con meno porte, AQC-Tensor mantiene la stessa struttura di porte e ottimizza i suoi parametri per ridurre l'errore Trotter. Consulta la [documentazione AQC-Tensor](https://qiskit.github.io/qiskit-addon-aqc-tensor/) per ulteriori informazioni.

Questo tutorial ti guida attraverso il flusso di lavoro completo di preparazione degli stati con AQC-Tensor: definizione di un hamiltoniano, generazione di circuiti Trotter, loro compressione tramite ottimizzazione a rete tensoriale ed esecuzione del risultato su hardware IBM Quantum&reg;.

## Esempio su simulatore su piccola scala {#small-scale-simulator-example}

Questa sezione utilizza un sistema a 10 siti per illustrare passo dopo passo il flusso di lavoro AQC-Tensor. Simuliamo le dinamiche di una catena di spin XXZ a 10 siti, un modello ampiamente studiato per esaminare le interazioni di spin e le proprietà magnetiche.

L'hamiltoniano è il seguente:

$$
\hat{\mathcal{H}}_{XXZ} = \sum_{i=1}^{L-1} J_{i,(i+1)}\left(X_i X_{(i+1)}+Y_i Y_{(i+1)}+ 2\cdot Z_i Z_{(i+1)} \right) \, ,
$$

dove $J_{i,(i+1)}$ è un coefficiente casuale per il legame $(i, i+1)$ e $L=10$.
### Passo 1: Mappare gli input classici a un problema quantistico {#step-1-map-classical-inputs-to-a-quantum-problem}

In questo passo:
1. Definiamo l'hamiltoniano, l'osservabile e lo stato iniziale.
2. Calcoliamo il valore di aspettazione esatto classicamente per un confronto successivo.
3. Generiamo un circuito Trotter ad alta precisione (il target AQC) e lo comprimiamo in un ansatz a bassa profondità usando AQC-Tensor.
#### Configurare l'hamiltoniano, l'osservabile e lo stato iniziale {#set-up-the-hamiltonian-observable-and-initial-state}

```python
# L is the number of sites in the 1D spin chain
L = 10

# Generate the coupling map
edge_list = [(i - 1, i) for i in range(1, L)]
even_edges = edge_list[::2]
odd_edges = edge_list[1::2]
coupling_map = CouplingMap(edge_list)

# Generate random coefficients for our XXZ Hamiltonian
np.random.seed(0)
Js = np.random.rand(L - 1) + 0.5 * np.ones(L - 1)
hamiltonian = SparsePauliOp(Pauli("I" * L))
for i, edge in enumerate(even_edges + odd_edges):
    hamiltonian += SparsePauliOp.from_sparse_list(
        [
            ("XX", (edge), Js[i] / 2),
            ("YY", (edge), Js[i] / 2),
            ("ZZ", (edge), Js[i]),
        ],
        num_qubits=L,
    )

# Generate a ZZ observable between the two middle qubits
observable = SparsePauliOp.from_sparse_list(
    [("ZZ", (L // 2 - 1, L // 2), 1.0)], num_qubits=L
)

# Generate an initial Néel state |1010101010⟩
initial_state_circuit = QuantumCircuit(L)
for i in range(L):
    if i % 2:
        initial_state_circuit.x(i)

print("Hamiltonian:", hamiltonian)
print("Observable:", observable)
graphviz_draw(coupling_map.graph, method="circo")
```

#### Calcolare il valore di aspettazione esatto {#compute-the-exact-expectation-value}

Per un sistema di questa dimensione, possiamo calcolare il valore di aspettazione esatto evoluto nel tempo direttamente usando l'esponenziazione matriciale. Questo serve come riferimento per valutare la precisione del circuito AQC.

```python
aqc_evolution_time = 0.2

# Each baseline Trotter step covers dt = aqc_evolution_time / 3
# The subsequent (uncompressed) step covers 1 additional dt
subsequent_evolution_time = aqc_evolution_time / 3
total_evolution_time = aqc_evolution_time + subsequent_evolution_time

# Compute exact expectation value via matrix exponentiation
H_matrix = hamiltonian.to_matrix()
U_exact = expm(-1j * H_matrix * total_evolution_time)

# Build the initial state vector (Néel state)
initial_state_vec = np.zeros(2**L)
state_idx = sum(2**i for i in range(L) if i % 2)
initial_state_vec[state_idx] = 1.0

# Evolve and compute expectation value
evolved_state = U_exact @ initial_state_vec
obs_matrix = observable.to_matrix()
exact_expval = (evolved_state.conj() @ obs_matrix @ evolved_state).real

print(f"AQC evolution time: {aqc_evolution_time}")
print(f"Subsequent evolution time: {subsequent_evolution_time:.6f}")
print(f"Total evolution time: {total_evolution_time:.6f}")
print(f"Exact expectation value: {exact_expval:.6f}")
```

#### Generare il circuito target AQC {#generate-the-aqc-target-circuit}

Ora costruiamo il circuito Trotter che servirà come target AQC. Questo circuito utilizza molti passi Trotter (32) per alta precisione. Poiché verrà simulato solo classicamente come MPS&mdash;non eseguito su hardware&mdash;la grande profondità non è un problema.

```python
aqc_target_num_trotter_steps = 32

aqc_target_circuit = initial_state_circuit.copy()
aqc_target_circuit.compose(
    generate_time_evolution_circuit(
        hamiltonian,
        synthesis=SuzukiTrotter(reps=aqc_target_num_trotter_steps),
        time=aqc_evolution_time,
    ),
    inplace=True,
)
```

#### Generare un ansatz, parametri iniziali, circuito successivo e circuito di riferimento {#generate-an-ansatz-initial-parameters-subsequent-circuit-and-a-baseline-circuit}

Successivamente, costruiamo un circuito "buono" con lo stesso tempo di evoluzione del target AQC ma con molti meno passi Trotter (solo uno). Passiamo questo circuito a `generate_ansatz_from_circuit`, che restituisce:

1. Un circuito **ansatz** generale e parametrizzato con la stessa connettività a due qubit.
2. **Parametri iniziali** che riproducono il circuito di input quando inseriti nell'ansatz.

Costruiamo anche:
- Un **circuito successivo** con un passo Trotter che verrà aggiunto (non compresso) dopo la porzione ottimizzata con AQC, seguendo l'approccio nel [tutorial AQC-Tensor sullo stato iniziale](https://qiskit.github.io/qiskit-addon-aqc-tensor/tutorials/01_initial_state_aqc.html).
- Un **circuito Trotter di riferimento** usando quattro passi Trotter sull'intero tempo di evoluzione (`aqc_evolution_time + subsequent_evolution_time`). Questo serve come confronto: rappresenta ciò che si eseguirebbe su hardware senza AQC. L'ansatz AQC (3 passi compressi + 1 passo non compresso) raggiunge una migliore precisione a profondità inferiore.

```python
aqc_ansatz_num_trotter_steps = 1

aqc_good_circuit = initial_state_circuit.copy()
aqc_good_circuit.compose(
    generate_time_evolution_circuit(
        hamiltonian,
        synthesis=SuzukiTrotter(reps=aqc_ansatz_num_trotter_steps),
        time=aqc_evolution_time,
    ),
    inplace=True,
)

aqc_ansatz, aqc_initial_parameters = generate_ansatz_from_circuit(
    aqc_good_circuit
)

# Subsequent circuit: 1 non-compressed Trotter step appended after AQC
subsequent_num_trotter_steps = 1
subsequent_circuit = generate_time_evolution_circuit(
    hamiltonian,
    synthesis=SuzukiTrotter(reps=subsequent_num_trotter_steps),
    time=subsequent_evolution_time,
)

# Baseline Trotter circuit: 4 Trotter steps over total evolution time, no AQC
baseline_num_trotter_steps = 4
baseline_circuit = initial_state_circuit.copy()
baseline_circuit.compose(
    generate_time_evolution_circuit(
        hamiltonian,
        synthesis=SuzukiTrotter(reps=baseline_num_trotter_steps),
        time=total_evolution_time,
    ),
    inplace=True,
)

print(
    f"Target circuit:      depth {aqc_target_circuit.depth(lambda x: x.operation.num_qubits == 2)}"
)
print(
    f"Baseline circuit:    depth {baseline_circuit.depth(lambda x: x.operation.num_qubits == 2)} ({baseline_num_trotter_steps} Trotter steps, time={total_evolution_time:.4f})"
)
print(
    f"Subsequent circuit:  depth {subsequent_circuit.depth(lambda x: x.operation.num_qubits == 2)} ({subsequent_num_trotter_steps} Trotter step, time={subsequent_evolution_time:.4f})"
)
print(
    f"Ansatz circuit:      depth {aqc_ansatz.depth(lambda x: x.operation.num_qubits == 2)}, with {len(aqc_initial_parameters)} parameters"
)
aqc_ansatz.draw("mpl", fold=-1)
```

#### Configurare la simulazione a rete tensoriale e costruire l'MPS target {#set-up-tensor-network-simulation-and-build-the-target-mps}

Utilizziamo il simulatore di circuiti a stato prodotto matriciale (MPS) [quimb](https://github.com/jcmgray/quimb), con JAX che fornisce la differenziazione automatica per l'ottimizzazione basata sul gradiente. Costruiamo quindi una rappresentazione MPS dello stato target e valutiamo la fedeltà iniziale tra l'ansatz iniziale e il target. Poiché il problema è un esempio relativamente piccolo, la fedeltà iniziale risulta molto alta.


#### Ottimizzare i parametri dell'ansatz {#optimize-the-ansatz-parameters}

Minimizziamo la funzione di costo `MaximizeStateFidelity` usando l'ottimizzatore L-BFGS-B. L'ottimizzatore regola iterativamente i parametri dell'ansatz per massimizzare la fedeltà tra il circuito ansatz e l'MPS target.

```python
aqc_stopping_fidelity = 1
aqc_max_iterations = 500

stopping_point = 1.0 - aqc_stopping_fidelity
objective = MaximizeStateFidelity(
    aqc_target_mps, aqc_ansatz, simulator_settings
)

def callback(intermediate_result: OptimizeResult):
    fidelity = 1 - intermediate_result.fun
    print(
        f"{datetime.datetime.now()} Intermediate result: Fidelity {fidelity:.8f}"
    )
    if intermediate_result.fun < stopping_point:
        raise StopIteration

result = minimize(
    objective,
    aqc_initial_parameters,
    method="L-BFGS-B",
    jac=True,
    options={"maxiter": aqc_max_iterations},
    callback=callback,
)
if result.status not in (0, 1, 99):
    raise RuntimeError(
        f"Optimization failed: {result.message} (status={result.status})"
    )

print(f"Done after {result.nit} iterations.")
aqc_final_parameters = result.x
```

## Esempio su hardware su grande scala {#large-scale-hardware-example}

Ora scalliamo a un modello XXZ a 50 siti per dimostrare AQC-Tensor su una dimensione di problema più realistica. Il flusso di lavoro è lo stesso dell'esempio su piccola scala: comprimiamo tre passi Trotter tramite AQC e aggiungiamo un passo non compresso.

Per un sistema di questa dimensione, l'esponenziazione matriciale è impraticabile (dimensioni $2^{50}$), quindi calcoliamo il valore di aspettazione di riferimento direttamente da un MPS ad alta precisione evoluto per l'intero tempo.

### Passi 1–4 combinati {#steps-1-4-combined}

```python
# -------------------------Step 1-------------------------

# Define the 50-site spin chain
L = 50
edge_list = [(i - 1, i) for i in range(1, L)]
even_edges = edge_list[::2]
odd_edges = edge_list[1::2]
coupling_map = CouplingMap(edge_list)

# Random XXZ Hamiltonian
np.random.seed(0)
Js = np.random.rand(L - 1) + 0.5 * np.ones(L - 1)
hamiltonian = SparsePauliOp(Pauli("I" * L))
for i, edge in enumerate(even_edges + odd_edges):
    hamiltonian += SparsePauliOp.from_sparse_list(
        [
            ("XX", (edge), Js[i] / 2),
            ("YY", (edge), Js[i] / 2),
            ("ZZ", (edge), Js[i]),
        ],
        num_qubits=L,
    )

observable = SparsePauliOp.from_sparse_list(
    [("ZZ", (L // 2 - 1, L // 2), 1.0)], num_qubits=L
)

# Initial Néel state
initial_state_circuit = QuantumCircuit(L)
for i in range(L):
    if i % 2:
        initial_state_circuit.x(i)

# Time parameters
aqc_evolution_time = 0.2
subsequent_evolution_time = aqc_evolution_time / 3
total_evolution_time = aqc_evolution_time + subsequent_evolution_time

# AQC target circuit (high-accuracy, 32 Trotter steps for AQC portion)
aqc_target_num_trotter_steps = 32

aqc_target_circuit = initial_state_circuit.copy()
aqc_target_circuit.compose(
    generate_time_evolution_circuit(
        hamiltonian,
        synthesis=SuzukiTrotter(reps=aqc_target_num_trotter_steps),
        time=aqc_evolution_time,
    ),
    inplace=True,
)

# Generate ansatz from 1-step Trotter circuit
aqc_good_circuit = initial_state_circuit.copy()
aqc_good_circuit.compose(
    generate_time_evolution_circuit(
        hamiltonian,
        synthesis=SuzukiTrotter(reps=1),
        time=aqc_evolution_time,
    ),
    inplace=True,
)

aqc_ansatz, aqc_initial_parameters = generate_ansatz_from_circuit(
    aqc_good_circuit
)

# Subsequent circuit: 1 non-compressed Trotter step
subsequent_circuit = generate_time_evolution_circuit(
    hamiltonian,
    synthesis=SuzukiTrotter(reps=1),
    time=subsequent_evolution_time,
)

# Baseline Trotter circuit: 4 Trotter steps over total evolution time, no AQC
baseline_num_trotter_steps = 4
baseline_circuit = initial_state_circuit.copy()
baseline_circuit.compose(
    generate_time_evolution_circuit(
        hamiltonian,
        synthesis=SuzukiTrotter(reps=baseline_num_trotter_steps),
        time=total_evolution_time,
    ),
    inplace=True,
)
print(
    f"Target circuit:  depth {aqc_target_circuit.depth(lambda x: x.operation.num_qubits == 2)}"
)
print(
    f"Ansatz circuit:  depth {aqc_ansatz.depth(lambda x: x.operation.num_qubits == 2)}, with {len(aqc_initial_parameters)} parameters"
)
print(
    f"Subsequent circuit: depth {subsequent_circuit.depth(lambda x: x.operation.num_qubits == 2)}"
)
print(
    f"Baseline circuit:   depth {baseline_circuit.depth(lambda x: x.operation.num_qubits == 2)} ({baseline_num_trotter_steps} steps, time={total_evolution_time:.4f})"
)

# Build target MPS and compute reference expectation value
simulator_settings = QuimbSimulator(
    quimb.tensor.CircuitMPS, autodiff_backend="jax"
)
aqc_target_mps = tensornetwork_from_circuit(
    aqc_target_circuit, simulator_settings
)
print("Target MPS maximum bond dimension:", aqc_target_mps.psi.max_bond())

# For the reference expectation value, we need the full evolution (AQC + subsequent)
# Build a high-accuracy full circuit for MPS reference
full_target_circuit = initial_state_circuit.copy()
full_target_circuit.compose(
    generate_time_evolution_circuit(
        hamiltonian,
        synthesis=SuzukiTrotter(reps=aqc_target_num_trotter_steps),
        time=total_evolution_time,
    ),
    inplace=True,
)
full_target_mps = tensornetwork_from_circuit(
    full_target_circuit, simulator_settings
)
exact_expval = full_target_mps.local_expectation(
    quimb.pauli("Z") & quimb.pauli("Z"), (L // 2 - 1, L // 2)
).real.item()
print(f"Reference expectation value (from MPS): {exact_expval:.6f}")

# Optimize ansatz parameters
objective = MaximizeStateFidelity(
    aqc_target_mps, aqc_ansatz, simulator_settings
)

def callback(intermediate_result: OptimizeResult):
    fidelity = 1 - intermediate_result.fun
    print(
        f"{datetime.datetime.now()} Intermediate result: Fidelity {fidelity:.8f}"
    )

result = minimize(
    objective,
    aqc_initial_parameters,
    method="L-BFGS-B",
    jac=True,
    options={"maxiter": 500},
    callback=callback,
)
if result.status not in (0, 1, 99):
    raise RuntimeError(
        f"Optimization failed: {result.message} (status={result.status})"
    )
print(f"Done after {result.nit} iterations.")

# Assemble the final AQC circuit: optimized ansatz + subsequent Trotter step
aqc_final_circuit = aqc_ansatz.assign_parameters(result.x)
aqc_final_circuit.compose(subsequent_circuit, inplace=True)

# -------------------------Step 2-------------------------

service = QiskitRuntimeService()
backend = service.least_busy(min_num_qubits=127)
print(backend)

pass_manager = generate_preset_pass_manager(
    backend=backend, optimization_level=3
)
isa_circuit = pass_manager.run(aqc_final_circuit)
isa_observable = observable.apply_layout(isa_circuit.layout)
print(
    "AQC circuit depth:",
    isa_circuit.depth(lambda x: x.operation.num_qubits == 2),
)

# Also transpile the baseline Trotter circuit (4 Trotter steps, no AQC)
isa_baseline_circuit = pass_manager.run(baseline_circuit)
isa_baseline_observable = observable.apply_layout(isa_baseline_circuit.layout)
print(
    "Baseline Trotter circuit depth:",
    isa_baseline_circuit.depth(lambda x: x.operation.num_qubits == 2),
)

# -------------------------Step 3-------------------------

# Submit both circuits in a single job
estimator = Estimator(backend)
estimator.options.environment.job_tags = ["TUT_AQCTE"]

job = estimator.run(
    [
        (isa_circuit, isa_observable),
        (isa_baseline_circuit, isa_baseline_observable),
    ]
)
print("Job ID:", job.job_id())
```

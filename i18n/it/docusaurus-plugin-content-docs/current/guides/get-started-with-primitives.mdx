## Iniziare con Estimator {#get-started-with-estimator}

{/*Verified the v2 examples 2/29/24 - updated 10/29/24*/}

### 1. Inizializzare l'account {#1-initialize-the-account}

Poiché Qiskit Runtime Estimator è un servizio gestito, devi prima inizializzare il tuo account. Puoi quindi selezionare il QPU che vuoi usare per calcolare il valore atteso.

Segui i passi nell'[argomento Installazione e configurazione](install-qiskit) se non hai ancora un account.

```python
from qiskit_ibm_runtime import QiskitRuntimeService

service = QiskitRuntimeService()
backend = service.least_busy(
    operational=True, simulator=False, min_num_qubits=127
)

print(backend.name)
```

```text
ibm_torino
```

### 2. Creare un circuito e un'osservabile {#2-create-a-circuit-and-an-observable}

Hai bisogno di almeno un circuito e un'osservabile come input per la primitiva Estimator.

```python
from qiskit.circuit.library import qaoa_ansatz
from qiskit.quantum_info import SparsePauliOp

entanglement = [tuple(edge) for edge in backend.coupling_map.get_edges()]
observable = SparsePauliOp.from_sparse_list(
    [("ZZ", [i, j], 0.5) for i, j in entanglement],
    num_qubits=backend.num_qubits,
)
circuit = qaoa_ansatz(observable, reps=2)
# the circuit is parametrized, so we will define the parameter values for execution
param_values = [0.1, 0.2, 0.3, 0.4]

print(f">>> Observable: {observable.paulis}")
```

Il circuito e l'osservabile devono essere trasformati in modo da usare solo le istruzioni supportate dal QPU (denominati circuiti con *instruction set architecture (ISA)*). Useremo il transpiler per farlo.

```python
from qiskit.transpiler import generate_preset_pass_manager

pm = generate_preset_pass_manager(optimization_level=1, backend=backend)
isa_circuit = pm.run(circuit)
isa_observable = observable.apply_layout(isa_circuit.layout)
print(f">>> Circuit ops (ISA): {isa_circuit.count_ops()}")
```

```text
>>> Circuit ops (ISA): OrderedDict([('rz', 3826), ('sx', 1601), ('cz', 968)])
```

### 3. Inizializzare Qiskit Runtime Estimator {#3-initialize-qiskit-runtime-estimator}

Quando inizializzi Estimator, usa il parametro `mode` per specificare la modalità in cui vuoi eseguirlo. I valori possibili sono oggetti `batch`, `session` o `backend` per la modalità di esecuzione in batch, in sessione e per singolo job, rispettivamente. Per ulteriori informazioni, consulta [Introduzione alle modalità di esecuzione di Qiskit Runtime.](execution-modes)

```python
from qiskit_ibm_runtime import EstimatorV2 as Estimator

estimator = Estimator(mode=backend)
```

### 4. Invocare Estimator e ottenere i risultati {#4-invoke-the-estimator-and-get-results}

Successivamente, invoca il metodo `run()` per calcolare i valori attesi per i circuiti e le osservabili di input. Il circuito, l'osservabile e gli eventuali set di valori dei parametri vengono passati come tuple *primitive unified bloc* (PUB).

```python
job = estimator.run([(isa_circuit, isa_observable, param_values)])
print(f">>> Job ID: {job.job_id()}")
print(f">>> Job Status: {job.status()}")
```

```text

```


```python
result = job.result()
print(f">>> {result}")
print(f"  > Expectation value: {result[0].data.evs}")
print(f"  > Metadata: {result[0].metadata}")
```

## Iniziare con Sampler {#get-started-with-sampler}

### 1. Inizializzare l'account {#1-initialize-the-account}

Poiché Qiskit Runtime Sampler è un servizio gestito, devi prima inizializzare il tuo account. Puoi quindi selezionare il QPU che vuoi usare per calcolare il valore atteso.

Segui i passi nell'[argomento Installazione e configurazione](install-qiskit) se non hai ancora configurato un account.

```python
from qiskit_ibm_runtime import QiskitRuntimeService

service = QiskitRuntimeService()
backend = service.least_busy(
    operational=True, simulator=False, min_num_qubits=127
)
```

### 2. Creare un circuito {#2-create-a-circuit}

Hai bisogno di almeno un circuito come input per la primitiva Sampler.

```python
import numpy as np
from qiskit.circuit.library import efficient_su2

circuit = efficient_su2(127, entanglement="linear")
circuit.measure_all()
# The circuit is parametrized, so we will define the parameter values for execution
param_values = np.random.rand(circuit.num_parameters)
```

Usa il transpiler per ottenere un circuito ISA.

```python
from qiskit.transpiler import generate_preset_pass_manager

pm = generate_preset_pass_manager(optimization_level=1, backend=backend)
isa_circuit = pm.run(circuit)
print(f">>> Circuit ops (ISA): {isa_circuit.count_ops()}")
```

```text
>>> Circuit ops (ISA): OrderedDict([('sx', 3089), ('rz', 3036), ('cz', 1092), ('measure', 127), ('barrier', 1)])
```

### 3. Inizializzare Qiskit Runtime Sampler {#3-initialize-the-qiskit-runtime-sampler}

Quando inizializzi Sampler, usa il parametro `mode` per specificare la modalità in cui vuoi eseguirlo. I valori possibili sono oggetti `batch`, `session` o `backend` per la modalità di esecuzione in batch, in sessione e per singolo job, rispettivamente. Per ulteriori informazioni, consulta [Introduzione alle modalità di esecuzione di Qiskit Runtime.](execution-modes) Nota che gli utenti del piano Open non possono inviare job in sessione.

```python
from qiskit_ibm_runtime import SamplerV2 as Sampler

sampler = Sampler(mode=backend)
```

### 4. Invocare Sampler e ottenere i risultati {#4-invoke-the-sampler-and-get-results}

Successivamente, invoca il metodo `run()` per generare l'output. Il circuito e gli eventuali set di valori dei parametri vengono passati come tuple *primitive unified bloc* (PUB).

```python
job = sampler.run([(isa_circuit, param_values)])
print(f">>> Job ID: {job.job_id()}")
print(f">>> Job Status: {job.status()}")
```

```text
>>> Job ID: d5k96rsjt3vs73ds5tig
>>> Job Status: QUEUED
```

```python
result = job.result()

# Get results for the first (and only) PUB
pub_result = result[0]
print(
```

## Iniziare con le primitive backend {#get-started-with-the-backend-primitives}

A differenza delle primitive specifiche per un provider, le primitive backend sono implementazioni generiche che possono essere usate con un qualsiasi oggetto `backend`, purché implementi l'interfaccia [`Backend`](https://docs.quantum.ibm.com/api/qiskit/qiskit.providers.Backend).

- La primitiva Sampler può essere eseguita con qualsiasi provider usando [`qiskit.primitives.BackendSamplerV2`](https://docs.quantum.ibm.com/api/qiskit/qiskit.primitives.BackendSamplerV2).
- La primitiva Estimator può essere eseguita con qualsiasi provider usando [`qiskit.primitives.BackendEstimatorV2`](../api/qiskit/qiskit.primitives.BackendEstimatorV2).

Alcuni provider implementano le primitive in modo nativo. Consulta la [pagina dell'ecosistema Qiskit](https://qiskit.github.io/ecosystem#provider) per i dettagli.
### Esempio: BackendEstimator {#example-backendestimator}

```python
from qiskit.primitives import BackendEstimatorV2
from <some_qiskit_provider> import QiskitProvider

provider = QiskitProvider()
backend = provider.get_backend('backend_name')
estimator = BackendEstimatorV2(backend)
```

### Esempio: BackendSampler {#example-backendsampler}

```python
from qiskit.primitives import BackendSamplerV2
from <some_qiskit_provider> import QiskitProvider

provider = QiskitProvider()
backend = provider.get_backend('backend_name')
sampler = BackendSamplerV2(backend)
```
### Somiglianze e differenze tra le primitive backend e quelle Runtime {#similarities-and-differences-between-backend-and-runtime-primitives}

- Gli input e gli output di [`qiskit.primitives.BackendSamplerV2`](https://docs.quantum.ibm.com/api/qiskit/qiskit.primitives.BackendSamplerV2) e [`qiskit.primitives.BackendEstimatorV2`](../api/qiskit/qiskit.primitives.BackendEstimatorV2)
seguono lo stesso formato PUB delle primitive in Qiskit Runtime. Consulta [Input e output delle primitive](primitive-input-output) per i dettagli.
Possono tuttavia esserci differenze nei campi dei metadati restituiti.

- La classe [`qiskit.primitives.BackendEstimatorV2`](https://docs.quantum.ibm.com/api/qiskit/qiskit.primitives.BackendEstimatorV2) non offre implementazioni predefinite di mitigazione degli errori di misura o di gate, poiché le primitive backend sono progettate per essere eseguite localmente sul computer dell'utente.

- La classe [`qiskit.primitives.BackendSamplerV2`](https://docs.quantum.ibm.com/api/qiskit/qiskit.primitives.BackendSamplerV2) richiede un backend che supporti l'opzione `memory`.

- Le interfacce delle primitive backend espongono `Options` personalizzate per [`SamplerV2`](https://docs.quantum.ibm.com/api/qiskit/qiskit.primitives.BackendSamplerV2) ed [`EstimatorV2`](https://docs.quantum.ibm.com/api/qiskit/qiskit.primitives.BackendEstimatorV2) che differiscono dalle implementazioni Runtime.

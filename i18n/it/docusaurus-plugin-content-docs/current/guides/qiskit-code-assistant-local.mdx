# Usare Qiskit Code Assistant in modalità locale {#use-qiskit-code-assistant-in-local-mode}

Scopri come installare, configurare e usare uno qualsiasi dei modelli di Qiskit Code Assistant sul tuo computer locale.

<Admonition type="note" title="Note">
    - Qiskit Code Assistant è in stato di anteprima e potrebbe subire modifiche.
    - Per inviare feedback o contattare il team di sviluppo, usa il [canale Qiskit Slack Workspace](https://qiskit.enterprise.slack.com/archives/C07LYA6PL83) o i repository GitHub pubblici correlati.
</Admonition>

## Avvio rapido (consigliato) {#quick-start-recommended}

Il modo più semplice per iniziare a usare Qiskit Code Assistant in modalità locale è ricorrere agli script di configurazione automatica per l'estensione VS Code o JupyterLab. Questi script installano automaticamente [Ollama](https://ollama.com/) per eseguire i modelli LLM, scaricano il modello consigliato e configurano l'estensione per te.

### Configurazione dell'estensione VS Code {#vs-code-extension-setup}

Esegui il comando seguente nel tuo terminale:

```bash
bash <(curl -fsSL https://raw.githubusercontent.com/Qiskit/qiskit-code-assistant-vscode/main/setup_local.sh)
```

Questo script esegue i passaggi seguenti:
- Installa Ollama (se non è già installato)
- Scarica e configura il modello Qiskit Code Assistant consigliato
- Configura l'estensione VS Code per funzionare con la tua installazione locale

### Configurazione dell'estensione JupyterLab {#jupyterlab-extension-setup}

Esegui il comando seguente nel tuo terminale:

```bash
bash <(curl -fsSL https://raw.githubusercontent.com/Qiskit/qiskit-code-assistant-jupyterlab/main/setup_local.sh)
```

Questo script:
- Installa Ollama (se non è già installato)
- Scarica e configura il modello Qiskit Code Assistant consigliato
- Configura l'estensione JupyterLab per funzionare con la tua installazione locale

Esistono più modi per distribuire e interagire con il modello Qiskit Code Assistant scaricato. ### Usare l'applicazione Ollama {#using-the-ollama-application}

L'applicazione Ollama offre una soluzione semplice per eseguire i modelli LLM in locale. È facile da usare, con una CLI che rende piuttosto semplice l'intero processo di configurazione, la gestione dei modelli e l'interazione. È ideale per la sperimentazione rapida e per gli utenti che desiderano gestire meno dettagli tecnici.

#### Installare Ollama {#install-ollama}

1. Scarica l'[applicazione Ollama](https://ollama.com/download)
1. Installa il file scaricato
1. Avvia l'applicazione Ollama installata

    <Admonition type="info">L'applicazione è in esecuzione correttamente quando l'icona di Ollama appare nella barra dei menu del desktop. Puoi anche verificare che il servizio sia in esecuzione visitando `http://localhost:11434/`.</Admonition>

1. Prova Ollama nel tuo terminale e inizia a eseguire modelli. Ad esempio:

    ```
    ollama run hf.co/Qiskit/Qwen2.5-Coder-14B-Qiskit
    ```

#### Configurare Ollama tramite l'integrazione con Hugging Face Hub {#set-up-ollama-using-the-hugging-face-hub-integration}

L'[integrazione Ollama/Hugging Face Hub](https://huggingface.co/docs/hub/ollama) consente di interagire con i modelli ospitati su Hugging Face Hub senza dover creare un nuovo modelfile né scaricare manualmente i file GGUF o safetensors. I file `template` e `params` predefiniti sono già inclusi per il modello su Hugging Face Hub.

1. Assicurati che l'applicazione Ollama sia in esecuzione.

1. Vai alla pagina del modello desiderato e copia l'URL. Ad esempio, https://huggingface.co/Qiskit/Qwen2.5-Coder-14B-Qiskit-GGUF.

1. Dal tuo terminale, esegui il comando:

    ```
    ollama run hf.co/Qiskit/Qwen2.5-Coder-14B-Qiskit
    ```

Puoi usare il modello `hf.co/Qiskit/Qwen2.5-Coder-14B-Qiskit` o uno qualsiasi degli altri modelli GGUF ufficiali attualmente consigliati: `hf.co/Qiskit/mistral-small-3.2-24b-qiskit-GGUF` o `hf.co/Qiskit/granite-3.3-8b-qiskit-GGUF`.

#### Configurare Ollama con un modello GGUF di Qiskit Code Assistant scaricato manualmente {#set-up-ollama-with-a-manually-downloaded-qiskit-code-assistant-gguf-model}

Se hai scaricato manualmente un modello GGUF come https://huggingface.co/Qiskit/Qwen2.5-Coder-14B-Qiskit-GGUF e vuoi sperimentare con template e parametri diversi, puoi seguire questi passaggi per caricarlo nella tua applicazione Ollama locale.

1. Crea un `Modelfile` inserendo il contenuto seguente e assicurati di aggiornare `<PATH-TO-GGUF-FILE>` con il percorso effettivo del modello scaricato.

    ```
    FROM <PATH-TO-GGUF-FILE>
    TEMPLATE """{{ if .System }}
    System:
    {{ .System }}

    {{ end }}{{ if .Prompt }}Question:
    {{ .Prompt }}

    {{ end }}Answer:
    ```python{{ .Response }}
    """

    PARAMETER stop "Question:"
    PARAMETER stop "Answer:"
    PARAMETER stop "System:"
    PARAMETER stop "```"

    PARAMETER temperature 0
    PARAMETER top_k 1
    ```

1. Run the following command to create a custom model instance based on the `Modelfile`.

    ```
    ollama create Qwen2.5-Coder-14B-Qiskit -f ./path-to-model-file
    ```

    <Admonition type="note">This process may take some time for Ollama to read the model file, initialize the model instance, and configure it according to the specifications provided.</Admonition>

#### Run the Qiskit Code Assistant model manually downloaded in Ollama

After the `Qwen2.5-Coder-14B-Qiskit` model has been set up in Ollama, run the following command to launch the model and interact with it in the terminal (in chat mode).

```
ollama run Qwen2.5-Coder-14B-Qiskit
```

Some useful commands:

- `ollama list` - List models on your computer
- `ollama rm Qwen2.5-Coder-14B-Qiskit` - Delete the model
- `ollama show Qwen2.5-Coder-14B-Qiskit` - Show model information
- `ollama stop Qwen2.5-Coder-14B-Qiskit` - Stop a model that is currently running
- `ollama ps` - List which models are currently loaded

<summary>Manually deploy the Qiskit Code Assistant models in local through the `llama-cpp-python` package</summary>

<span id="use-llama-cpp-python"></span>
An alternative to the Ollama application is the `llama-cpp-python` package, which is a Python binding for `llama.cpp`. It gives you more control and flexibility to run the GGUF model locally, and is ideal for users who wish to integrate the local model in their workflows and Python applications.

1. Install [`llama-cpp-python`](https://pypi.org/project/llama-cpp-python/)
2. Interact with the model from within your application using `llama_cpp`. For example:

```python
from llama_cpp import Llama

model_path = <PATH-TO-GGUF-FILE>

model = Llama(
        model_path,
        seed=17,
        n_ctx=10000,
        n_gpu_layers=37, # to offload in gpu, but put 0 if all in cpu
    )

input = 'Generate a quantum circuit with 2 qubits'
raw_pred = model(input)["choices"][0]["text"]
```

You can also add text generation parameters to the model to customize the inference:

```python
generation_kwargs = {
        "max_tokens": 512,
        "echo": False, # Echo the prompt in the output
        "top_k": 1
    }

raw_pred = model(input, **generation_kwargs)["choices"][0]["text"]
```

### Use the `llama.cpp` library

Another alternative is to use `llama.cpp`, an open-source library for performing LLM inference on a CPU with minimal setup.
It provides low-level control over the model execution and is typically run from the command line, pointing to a local GGUF model file.

There are several ways to install `llama.cpp` on your machine:

- Install llama.cpp using [brew, nix, or winget](https://github.com/ggml-org/llama.cpp/blob/master/docs/install.md)
- Run with Docker: See out the [Docker documentation by `llama.cpp` team](https://github.com/ggml-org/llama.cpp/blob/master/docs/docker.md)
- Download pre-built binaries from the [releases page](https://github.com/ggml-org/llama.cpp/releases)
- Build from source by [cloning this repository](https://github.com/ggml-org/llama.cpp/blob/master/docs/build.md)

Once installed, you can use `llama.cpp` to interact with GGUF models in conversation mode as follows:
```bash
# Use a local model file
llama-cli -m my_model.gguf -cnv

# Or download and run a model directly from Hugging Face
llama-cli -hf Qiskit/Qwen2.5-Coder-14B-Qiskit-GGUF -cnv
```

You can also launch an OpenAI-compatible API server for the model in the following way:
```bash
llama-server -hf Qiskit/Qwen2.5-Coder-14B-Qiskit-GGUF
```

#### Advanced parameters
With the `llama-cli` program, you can control the model generation using command-line options. For example, you can provide an initial “system” prompt using the `-p/--prompt` flag. In conversation mode (`-cnv`), this initial prompt acts as the system message. Otherwise, you can simply prepend any desired instruction to your prompt text. You can also adjust sampling parameters - for instance: temperature (`--temp`), top-k (`--top-k`), top-p (`--top-p`), repetition penalty (`--repeat-penalty`), and the seed to use (`--seed`). The following is an example invocation using these options:

```bash
llama-cli -hf Qiskit/Qwen2.5-Coder-14B-Qiskit-GGUF \
  -p "You are a friendly assistant." -cnv \
  --temp 0.7 \
  --top-k 50 \
  --top-p 0.95 \
  --repeat-penalty 1.1 \
  --seed 42
```

Per garantire il corretto funzionamento dei nostri modelli Qiskit, consigliamo di usare il prompt di sistema fornito nei nostri repository HF GGUF: prompt di sistema per [mistral-small-3.2-24b-qiskit-GGUF](https://huggingface.co/Qiskit/mistral-small-3.2-24b-qiskit-GGUF/blob/main/system), [Qwen2.5-Coder-14B-Qiskit-GGUF](https://huggingface.co/Qiskit/Qwen2.5-Coder-14B-Qiskit-GGUF/blob/main/system), [granite-3.3-8b-qiskit-GGUF](https://huggingface.co/Qiskit/granite-3.3-8b-qiskit-GGUF/blob/main/system) e [granite-3.2-8b-qiskit-GGUF.](https://huggingface.co/Qiskit/granite-3.2-8b-qiskit-GGUF/blob/main/system)

<summary>Collegare manualmente le estensioni alla distribuzione locale</summary>

Usa l'estensione VS Code e l'estensione JupyterLab per Qiskit Code Assistant per inviare prompt al modello Qiskit Code Assistant distribuito localmente. Una volta che hai [configurato l'applicazione Ollama con il modello](#using-the-ollama-application), puoi configurare le estensioni per connettersi al servizio locale.

### Collegare l'estensione VS Code di Qiskit Code Assistant {#connect-with-the-qiskit-code-assistant-vs-code-extension}

Con l'estensione VS Code di Qiskit Code Assistant, puoi interagire con il modello ed eseguire il completamento del codice mentre scrivi. È particolarmente utile per gli utenti che cercano assistenza nella scrittura di codice Qiskit per le loro applicazioni Python.

1. Installa l'[estensione VS Code di Qiskit Code Assistant](/guides/qiskit-code-assistant-vscode).
1. In VS Code, vai alle **Impostazioni utente** e imposta **Qiskit Code Assistant: Url** sull'URL della tua distribuzione Ollama locale (ad esempio, `http://localhost:11434`).
1. Ricarica VS Code andando su **Visualizza > Riquadro comandi...** e selezionando **Sviluppatore: Ricarica finestra**.

Il modello Qiskit Code Assistant configurato in Ollama dovrebbe apparire nella barra di stato ed è quindi pronto all'uso.

### Collegare l'estensione JupyterLab di Qiskit Code Assistant {#connect-with-the-qiskit-code-assistant-jupyterlab-extension}

Con l'estensione JupyterLab di Qiskit Code Assistant, puoi interagire con il modello ed eseguire il completamento del codice direttamente nel tuo Jupyter Notebook. Gli utenti che lavorano prevalentemente con Jupyter Notebook possono sfruttare questa estensione per migliorare ulteriormente la propria esperienza nella scrittura di codice Qiskit.

1. Installa l'[estensione JupyterLab di Qiskit Code Assistant](/guides/qiskit-code-assistant-jupyterlab).
1. In JupyterLab, vai all'**Editor delle impostazioni** e imposta **Qiskit Code Assistant Service API** sull'URL della tua distribuzione Ollama locale (ad esempio, `http://localhost:11434`).

Il modello Qiskit Code Assistant configurato in Ollama dovrebbe apparire nella barra di stato ed è quindi pronto all'uso.

</details>

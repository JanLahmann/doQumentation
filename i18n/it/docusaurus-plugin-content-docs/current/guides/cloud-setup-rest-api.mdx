# Configurazione per utilizzare IBM Quantum Platform con la REST API {#set-up-to-use-ibm-quantum-platform-with-rest-api}

Puoi accedere ai processori quantistici tramite le REST API, permettendoti di lavorare con le QPU usando qualsiasi linguaggio di programmazione o framework.

### 1. Ottieni l'accesso {#1-get-access}

1. Se non hai già un account utente, creane uno dalla [pagina di accesso a IBM Quantum.](https://quantum.cloud.ibm.com/login)
1. Crea una chiave API (chiamata anche token) nella [dashboard.](https://quantum.cloud.ibm.com) Tieni presente che la stessa chiave API può essere utilizzata per entrambe le regioni.
1. Genera un bearer token IBM Cloud Identity and Access Management (IAM). Si tratta di un token a breve durata utilizzato per autenticare le richieste alla REST API. Per generarne uno, chiama la [IAM Identity Services API](https://cloud.ibm.com/apidocs/iam-identity-token-api#create-api-key) come mostrato nella seguente richiesta di esempio:
<Tabs>

   <TabItem value="Curl" label="Curl">
   ```bash
   curl -X POST 'https://iam.cloud.ibm.com/identity/token' \
   -H 'Content-Type: application/x-www-form-urlencoded' \
   -d 'grant_type=urn:ibm:params:oauth:grant-type:apikey&apikey=MY_APIKEY'
   ```
   Risposta attesa
   ```text
   {
      "access_token": "eyJhbGciOiJIUz......sgrKIi8hdFs",
      "refresh_token": "SPrXw5tBE3......KBQ+luWQVY=",
      "token_type": "Bearer",
      "expires_in": 3600,
      "expiration": 1473188353
   }
   ```
   </TabItem>

   <TabItem value="Python" label="Python">
   ```python
   # Use 'service' to invoke operations.
   import requests
   import json

   url = 'https://iam.cloud.ibm.com/identity/token'
   api_key = 'MY_APIKEY'
   headers = {
    'Content-Type': 'application/x-www-form-urlendcoded',
   }
   data = f'grant_type=urn:ibm:params:oauth:grant-type:apikey&apikey={api_key}'
   response = requests.post(url, headers=headers, data=data)

   # Bearer token to authorize requests to the REST API
   bearer_token = response.json()['access_token']
   ```
   </TabItem>
</Tabs>

### 2. Scegli un metodo di autenticazione {#2-choose-an-authentication-method}

Scegli il metodo di autenticazione appropriato in base al tuo ambiente di lavoro:

* [Crea una variabile d'ambiente per la tua chiave API (ambienti Python affidabili)](#environment_variable)
* [Usa la tua chiave API direttamente (ambiente non affidabile)](/guides/cloud-setup-untrusted#rest-untrusted)

<span id="environment_variable"></span>
#### Crea una variabile d'ambiente (ambiente affidabile) {#create-an-environment-variable-trusted-environment}

1. Per impostare la variabile d'ambiente `IQP_API_TOKEN` nel tuo sistema, puoi aggiungere la riga seguente al profilo della tua shell (ad esempio `.bashrc` o `.zshrc`) oppure impostarla direttamente nel terminale:

    ```shell
    export IQP_API_TOKEN=<your-API_KEY> # Use the 44-character API_KEY you created and saved from the IBM Quantum Platform Home dashboard
    ```
    Quando richiami la variabile d'ambiente nel tuo codice, includi `import os`, come in questo esempio:

    ```python
    import os
    api_token = os.environ['IQP_API_TOKEN']
    ```
    Tieni presente che, quando crei una variabile d'ambiente, la tua chiave API viene comunque salvata localmente in testo normale e deve essere protetta adeguatamente.

1. Autentica le richieste alla REST API di Qiskit Runtime includendo il CRN e il bearer token negli header della richiesta.
```bash
curl -X 'GET' \
   'https://quantum.cloud.ibm.com/api/v1/usage' \
   '-H accept: application/json' \
   '-H authorization: Bearer <BEARER_TOKEN>' \
   '-H Service-CRN: <INSTANCE_CRN>'
```

# Comprendere l'alta disponibilità e il disaster recovery per Qiskit Runtime {#understanding-high-availability-and-disaster-recovery-for-qiskit-runtime}

L'alta disponibilità (HA) è la capacità di un servizio di rimanere operativo e accessibile anche in presenza di guasti imprevisti. Il disaster recovery è il processo di ripristino dell'istanza del servizio a uno stato funzionante. I clienti di Qiskit Runtime non sono responsabili della pianificazione o delle azioni di disaster recovery.

Qiskit Runtime è un servizio regionale che rispetta gli [Obiettivi del livello di servizio (SLO)](https://cloud.ibm.com/docs/resiliency?topic=resiliency-slo) definiti con qualsiasi piano a pagamento. Per ulteriori informazioni, consulta il [Contratto sul livello di servizio (SLA)](https://www.ibm.com/support/customer/csol/terms/?id=i126-9268&lc=en) per IBM Cloud&reg;. Per ulteriori informazioni sulle regioni e sui data center IBM Cloud disponibili per Qiskit Runtime, consulta [Disponibilità dei servizi e dell'infrastruttura per area geografica.](https://cloud.ibm.com/docs/overview?topic=overview-services_region)

## Architettura ad alta disponibilità {#high-availability-architecture}

### Funzionalità di alta disponibilità {#high-availability-features}

Questo servizio è distribuito su tre zone in una regione multi-zona, senza alcun punto di guasto singolo. Le richieste API vengono indirizzate tramite un load balancer ai nodi dell'istanza HA, ciascuno in una zona diversa.

Se un nodo o una zona dell'istanza HA si guasta, il servizio continua a elaborare le richieste API attraverso i nodi dell'istanza HA rimanenti. Non si verifica alcun degrado percettibile del servizio durante un guasto zonale.

È disponibile nelle seguenti regioni:

- Washington DC (us-east)
- Francoforte (eu-de)

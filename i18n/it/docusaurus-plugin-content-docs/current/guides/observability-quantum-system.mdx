# Eventi di tracciamento delle attività per il servizio IBM Quantum System {#activity-tracking-events-for-the-ibm-quantum-system-service}

I servizi IBM Cloud&reg;, come il servizio IBM&reg; Quantum System, generano eventi di tracciamento delle attività.

Gli eventi di tracciamento delle attività riportano le attività che modificano lo stato di un servizio in IBM Cloud. Puoi utilizzare gli eventi per esaminare attività anomale e azioni critiche e per soddisfare i requisiti di audit normativi.

Puoi utilizzare IBM Cloud Activity Tracker Event Routing, un servizio della piattaforma, per instradare gli eventi di audit nel tuo account verso destinazioni di tua scelta configurando target e route che definiscono dove vengono inviati gli eventi di tracciamento delle attività. Per ulteriori informazioni, consulta [About IBM Cloud Activity Tracker Events Routing](https://cloud.ibm.com/docs/atracker?topic=atracker-about).

Puoi utilizzare IBM Cloud Logs per visualizzare e ricevere avvisi sugli eventi generati nel tuo account e instradati da IBM Cloud Activity Tracker Event Routing a un'istanza di IBM Cloud Logs.

## Visualizzazione degli eventi di tracciamento delle attività per il servizio IBM Quantum System {#viewing-activity-tracking-events-for-the-ibm-quantum-system-service}

Puoi utilizzare IBM Cloud Logs per visualizzare e ricevere avvisi sugli eventi generati nel tuo account e instradati da IBM Cloud Activity Tracker Event Routing a un'istanza di IBM Cloud Logs.

### Avvio di IBM Cloud Logs dalla pagina Observability {#launching-ibm-cloud-logs-from-the-observability-page}

Per informazioni sull'avvio dell'interfaccia utente di IBM Cloud Logs, consulta [Launching the UI](https://cloud.ibm.com/docs/cloud-logs?topic=cloud-logs-instance-launch#instance-launch-cloud-ui) nella documentazione di IBM Cloud Logs.

| Azione | Attivata da |
| -- | -- |
| `quantum-computing.direct-access-backend-configuration.read` | Ottieni la configurazione del backend (`GET /v1/backends/{backend-name}/configuration`) |
| `quantum-computing.direct-access-backend-properties.read` | Ottieni le proprietà del backend (`GET /v1/backends/{backend-name}/properties`) |
| `quantum-computing.direct-access-backend.list` | Ottieni l'elenco dei backend (`GET /v1/backends`) |
| `quantum-computing.direct-access-backend.read` | Ottieni i dettagli del backend (`GET /v1/backends/{backend-name}`) |
| `quantum-computing.direct-access-job.cancel` | Annulla un job (`POST /v1/jobs/{job-id}/cancel`) |
| `quantum-computing.direct-access-job.create` | Esegui un job (`POST /v1/jobs`) |
| `quantum-computing.direct-access-job.delete` | Elimina un job (`DELETE /v1/jobs/{job-id}`) |
| `quantum-computing.direct-access-job.list` | Ottieni i job (`GET /v1/jobs`) |
| `quantum-computing.direct-access-job.read` | Ottieni il job tramite ID (`GET /v1/jobs/{job-id}`) |
| `quantum-computing.direct-access-lane-configuration.list` | Ottieni la configurazione delle corsie del backend (`GET /v1/backends/{backend-name}/lanes`) |

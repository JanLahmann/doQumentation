# Configurare IBM Quantum Platform per un'organizzazione {#configure-ibm-quantum-platform-for-an-organization}

Segui questi passaggi per configurare IBM Quantum Platform.

<span id="iam-org"></span>
## 1. Configurare le impostazioni IAM {#iam-org}

Prima di tutto, configura le seguenti impostazioni nell'account IAM dell'amministratore. Per rivedere e configurare queste impostazioni, vai su [Gestisci → IAM → Impostazioni](https://cloud.ibm.com/iam/settings).

* **Visibilità dell'elenco utenti** determina se gli utenti possono vedersi a vicenda, indipendentemente dall'assegnazione al progetto. L'impostazione `enabled` limita la visibilità degli utenti. In altre parole, gli utenti nel tuo account non possono vedersi tra loro, anche se hanno accesso alle stesse risorse. Scegli il valore appropriato per il tuo ambiente. Consulta [Controllo della visibilità degli utenti](https://cloud.ibm.com/docs/account?topic=account-iam-user-setting) per ulteriori informazioni.
* **Creazione di chiavi API** controlla se gli utenti possono creare chiavi API. In IBM Quantum Platform, l'uso delle chiavi API è comune. Se le chiavi API vengono utilizzate, scegli `disabled`. In alternativa, puoi assegnare autorizzazioni specifiche a ciascun utente.

<span id="crt-rsc-grp-org"></span>
## 2. (Facoltativo) Creare gruppi di risorse {#crt-rsc-grp-org}

Salta questo passaggio se utilizzi gruppi di accesso che fanno riferimento direttamente alle istanze del servizio IBM Quantum.

Se hai scelto di utilizzare i gruppi di risorse, vai su [Gestisci → Account → Gruppi di risorse (in Risorse account)](https://cloud.ibm.com/account/resource-groups) e fai clic su **Crea**.

<span id="create-instance-org"></span>
## 3. Creare istanze del servizio IBM Quantum {#create-instance-org}

Se hai già creato istanze del servizio IBM Quantum, salta questo passaggio.

* Se utilizzi i gruppi di risorse, assicurati di creare le istanze del servizio nel gruppo di risorse appropriato.
* Il nome dell'istanza del servizio, ad esempio `QR-ml`, è necessario per i riferimenti ai gruppi di accesso.

Segui i passaggi in [Configurare l'uso di IBM Cloud](/guides/cloud-setup#cloud) per le istruzioni.

<span id="create-group-org"></span>
## 4. Creare gruppi di accesso per i progetti {#create-group-org}

Prima di tutto, [crea un ruolo personalizzato](/guides/custom-roles#custom) che consenta agli utenti di eseguire azioni per lavorare con le istanze del servizio IBM Quantum.
Poi, [crea un gruppo di accesso](/guides/access-groups) per ciascun progetto e assegna a quel gruppo il set minimo di autorizzazioni necessarie per lavorare con le risorse del progetto.

<span id="setup-idp-org"></span>
## 5. Configurare il provider di identità e assegnare gli utenti {#setup-idp-org}

Segui i passaggi nell'argomento appropriato, a seconda del provider di identità che hai scelto di usare:

* [Usare IBM Cloud&reg; come provider di identità per gli utenti che hanno account IBM Cloud](/guides/manage-cloud-users)
* [Usare IBM Cloud come provider di identità per gli utenti che non hanno account IBM Cloud](/guides/manage-appid)
* [Usare un provider di identità diverso da IBM Cloud per gli utenti che non hanno account IBM Cloud](/guides/manage-appid)
* [Invitare utenti e gestire il loro accesso](/guides/invite-and-manage-users)

<span id="firewall"></span>
## 6. Configurare il firewall {#firewall}

Per configurare il tuo firewall e abilitare l'accesso agli endpoint API di IBM Quantum, aggiungi i seguenti URL alla tua lista di autorizzati:

- Servizi globali IBM Cloud:
    ```text
    cloud.ibm.com
    iam.cloud.ibm.com
    api.global-search-tagging.cloud.ibm.com
    globalcatalog.cloud.ibm.com
    resource-controller.cloud.ibm.com
    ```
- Servizi IBM Quantum Qiskit Runtime - regione us-east:
    ```text
    quantum.cloud.ibm.com
    s3.us-east.cloud-object-storage.appdomain.cloud
    ```
- Servizi IBM Quantum Qiskit Runtime - regione eu-de:
    ```text
    eu-de.quantum.cloud.ibm.com
    s3.direct.eu-de.cloud-object-storage.appdomain.cloud
    ```
- Servizio di anteprima IBM Quantum Qiskit Functions:
    ```text
    qiskit-serverless.quantum.ibm.com
    ```

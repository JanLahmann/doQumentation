## IBM Quantum se Open-Source-Inhalte {#ibm-quantums-open-source-content}

IBM stellt a Haufa Lernmaterial zum Quantencomputing bereit – alles Open Source unter CC BY-SA 4.0:

- **[Learning](https://quantum.cloud.ibm.com/learning)** — Strukturierte Kurse von de Quantegrundlaga bis zu de fortgschrittena Thema
- **[Tutorials](https://quantum.cloud.ibm.com/docs/en/tutorials)** — Über 40 Tutorials zu Transpilation, Fehlerminderung ond mehr
- **[Documentation](https://quantum.cloud.ibm.com/docs/en/guides)** — Aleidunga ond API-Referenz für Qiskit
- **[Source repo](https://github.com/Qiskit/documentation)** — Alle Inhalt uff GitHub

Ihre [Quantum Platform](https://quantum.cloud.ibm.com) isch immer aktuell ond guad gstaltet – der bescht Ort zum Lesa, Lerna ond Nachgucka.

## Was des Projekt drzua macht {#what-this-project-adds}

IBM se [Qiskit-Dokumentation](https://github.com/Qiskit/documentation) isch Open Source (CC BY-SA 4.0), aber ihre Webawendung isch's net. doQumentation macht a Open-Source-Frontend mit Live-Code-Ausführung, automatischer Credential-Injektion ond Simulator-Modus drzua.

Überall eisetzbar – von [GitHub Pages](https://doqumentation.org) über [Docker](https://github.com/JanLahmann/doQumentation/pkgs/container/doqumentation) bis zu [RasQberry](https://rasqberry.org/). Gucket se [alle Funktiona](/features) a.

## Code-Ausführung {#code-execution}

Klicket se bei jedem Codeblock uff **Run**. Dr erscht Klick startet an kostelosa Jupyter-Kernel über [Binder](https://mybinder.org) (1–2 Min.). Danach goht's sofort.

- **Simulator-Modus (koi Konto braucht)** — Aktivieret se dr [Simulator-Modus](/jupyter-settings#simulator-mode), damit se alle Notebooks mit AerSimulator oder FakeBackends mit Rauschmodell ausführa könnet. Null Eirichtungsaufwand braucht.

- **IBM Quantum Hardware** — Gebet se eure API-Token ond CRN oimol in de [Eistellunga](/jupyter-settings#ibm-quantum) ei — d'Zugangsdaten werdet bei jeder Ausführung automatisch eigfügt. [Erstellt a kosteloses Konto](https://quantum.cloud.ibm.com/registration), falls se no kois habet.

<details>
<summary>Verfügbare Ausführungs-Backends</summary>

Jedes Tutorial hot ausführbare Codeblöcke. Klicket se uff **Run**, damit se se mit oim vo drei Backends ausführa könnet:

1. **Binder** (Standard uff GitHub Pages) — Kostelosa Remote-Jupyter-Kernel über [mybinder.org](https://mybinder.org)
2. **Lokales Jupyter (Docker / RasQberry)** — Verbindet se mit'm lokala Jupyter-Server mit vorinstalliertem Qiskit
3. **Benutzerdefinierter Server** — Verweist se in de [Eistellunga](/jupyter-settings#advanced) uff an beliebiga Jupyter-Endpunkt

</details>

<details>
<summary>Bereitstellungsoptiona</summary>

doQumentation isch verfügbar als:

- **[GitHub Pages](https://doqumentation.org)** — Statische Website mit [Binder](https://mybinder.org) für Remote-Code-Ausführung
- **[Docker](https://github.com/JanLahmann/doQumentation/pkgs/container/doqumentation)** — Vollständiger Stack lokal, offline-fähig
- **[RasQberry](https://rasqberry.org/)** — Selbscht ghostet uff am Raspberry Pi mit lokalem Jupyter-Kernel

</details>

<details>
<summary>Lokal ausführa mit Podman / Docker</summary>

Installieret se [Podman Desktop](https://podman-desktop.io/) oder [Docker Desktop](https://www.docker.com/products/docker-desktop/) (Mac / Windows / Linux), dann führet se aus:

```bash
# Vollständiger Stack: Website + Jupyter + Qiskit (~3 GB)
podman run -p 8080:80 -p 8888:8888 ghcr.io/janlahmann/doqumentation:jupyter
```

Machet se [http://localhost:8080](http://localhost:8080) uff — Code-Ausführung funktioniert lokal, koine Binder-Wartezeit.

Für a leichtgewichtige Version ohne lokale Code-Ausführung (~60 MB):

```bash
# Nur statische Website — Code-Ausführung funktioniert weiterhin über Binder
podman run -p 8080:80 ghcr.io/janlahmann/doqumentation:latest
```

Benützet se Docker? Ersetzet se oifach `podman` durch `docker` — d'Befehle send identisch. D'Images send Multi-Arch (`linux/amd64` + `linux/arm64`), sodass Apple Silicon Macs nativ funktioniere.

</details>

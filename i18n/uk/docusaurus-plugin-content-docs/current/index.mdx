## Відкритий контент IBM Quantum {#ibm-quantums-open-source-content}

IBM надає величезну кількість навчальних матеріалів з квантових обчислень — усе відкрите під ліцензією CC BY-SA 4.0:

- **[Навчання](https://quantum.cloud.ibm.com/learning)** — Структуровані курси від основ квантової механіки до складних тем
- **[Посібники](https://quantum.cloud.ibm.com/docs/en/tutorials)** — Понад 40 посібників з транспіляції, зменшення помилок та інше
- **[Документація](https://quantum.cloud.ibm.com/docs/en/guides)** — Довідники та API-довідка для Qiskit
- **[Вихідний репозиторій](https://github.com/Qiskit/documentation)** — Весь контент на GitHub

Їхня [Quantum Platform](https://quantum.cloud.ibm.com) завжди актуальна та добре розроблена — найкраще місце для читання, навчання та довідкової інформації.

## Що додає цей проект {#what-this-project-adds}

[Документація Qiskit](https://github.com/Qiskit/documentation) від IBM є відкритою (CC BY-SA 4.0), але їхній веб-додаток — ні. doQumentation додає відкритий інтерфейс з можливістю виконання коду в реальному часі, автоматичним впровадженням облікових даних та режимом симулятора.

Може бути розгорнутий будь-де — від [GitHub Pages](https://doqumentation.org) до [Docker](https://github.com/JanLahmann/doQumentation/pkgs/container/doqumentation) до [RasQberry](https://rasqberry.org/). Дивіться [всі можливості](/features).

## Виконання коду {#code-execution}

Натисніть **Run** на будь-якому блоці коду. Перше натискання запускає безкоштовне ядро Jupyter через [Binder](https://mybinder.org) (1–2 хв). Після цього виконання миттєве.

- **Режим симулятора (не потрібен обліковий запис)** — Увімкніть [Режим симулятора](/jupyter-settings#simulator-mode), щоб виконувати всі ноутбуки з AerSimulator або FakeBackends з моделями шуму. Налаштування не потрібне.

- **Апаратне забезпечення IBM Quantum** — Введіть свій API-токен та CRN у [Налаштуваннях](/jupyter-settings#ibm-quantum) один раз — облікові дані автоматично впроваджуються при кожному запуску. [Створіть безкоштовний обліковий запис](https://quantum.cloud.ibm.com/registration), якщо у вас його немає.

<details>
<summary>Доступні бекенди для виконання</summary>

Кожен посібник має виконувані блоки коду. Натисніть **Run**, щоб виконати їх, використовуючи один із трьох бекендів:

1. **Binder** (за замовчуванням на GitHub Pages) — Безкоштовне віддалене ядро Jupyter через [mybinder.org](https://mybinder.org)
2. **Локальний Jupyter (Docker / RasQberry)** — Підключається до локального сервера Jupyter з попередньо встановленим Qiskit
3. **Власний сервер** — Вкажіть будь-яку кінцеву точку Jupyter у [Налаштуваннях](/jupyter-settings#advanced)

</details>

<details>
<summary>Варіанти розгортання</summary>

doQumentation доступний як:

- **[GitHub Pages](https://doqumentation.org)** — Статичний сайт з [Binder](https://mybinder.org) для віддаленого виконання коду
- **[Docker](https://github.com/JanLahmann/doQumentation/pkgs/container/doqumentation)** — Повний стек локально, здатний працювати офлайн
- **[RasQberry](https://rasqberry.org/)** — Самостійно розміщений на Raspberry Pi з локальним ядром Jupyter

</details>

<details>
<summary>Запуск локально з Podman / Docker</summary>

Встановіть [Podman Desktop](https://podman-desktop.io/) або [Docker Desktop](https://www.docker.com/products/docker-desktop/) (Mac / Windows / Linux), потім виконайте:

```bash
# Повний стек: сайт + Jupyter + Qiskit (~3 ГБ)
podman run -p 8080:80 -p 8888:8888 ghcr.io/janlahmann/doqumentation:jupyter
```

Відкрийте [http://localhost:8080](http://localhost:8080) — виконання коду працює локально, без очікування Binder.

Для легкої версії без локального виконання коду (~60 МБ):

```bash
# Тільки статичний сайт — виконання коду все одно працює через Binder
podman run -p 8080:80 ghcr.io/janlahmann/doqumentation:latest
```

Використовуєте Docker? Просто замініть `podman` на `docker` — команди ідентичні. Образи мультиархітектурні (`linux/amd64` + `linux/arm64`), тому Apple Silicon Mac працюють нативно.

</details>

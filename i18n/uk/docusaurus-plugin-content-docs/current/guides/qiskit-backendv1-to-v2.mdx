# Міграція з `BackendV1` на `BackendV2` {#migrate-from-backendv1-to-backendv2}

Клас Qiskit [`BackendV1`](https://docs.quantum.ibm.com/api/qiskit/1.4/qiskit.providers.BackendV1) є застарілим і буде вилучений з обслуговування. Цей посібник з міграції описує невеликі зміни, які тобі потрібно внести, якщо ти використовуєш провайдера, що оновився з [`BackendV1`](https://docs.quantum.ibm.com/api/qiskit/1.4/qiskit.providers.BackendV1) на [`BackendV2`](https://docs.quantum.ibm.com/api/qiskit/qiskit.providers.BackendV2).

<Admonition type="note">
 Якщо ти використовуєш виключно `qiskit_ibm_runtime` та `qiskit_aer`, жодних дій не потрібно. Пакет `qiskit_ibm_runtime` завжди використовував `BackendV2`, а `qiskit_aer` використовує `BackendV2` починаючи з версії 0.13.
 </Admonition>

## Основні зміни в `BackendV2` {#high-level-changes-in-backendv2}

Модель Qiskit [`Backend`](https://docs.quantum.ibm.com/api/qiskit/qiskit.providers.Backend) була розроблена для забезпечення Qiskit SDK рівнем абстракції,
який дозволяв міркувати про квантові комп'ютери в межах SDK. Перша ітерація моделі була представлена класом
[`BackendV1`](https://docs.quantum.ibm.com/api/qiskit/1.4/qiskit.providers.BackendV1). Цей клас зберігав інформацію про Backend у серії
контейнерів даних, а саме класах [`BackendConfiguration`](https://docs.quantum.ibm.com/api/qiskit/1.4/qiskit.providers.models.BackendConfiguration) та
[`BackendProperties`](https://docs.quantum.ibm.com/api/qiskit/1.4/qiskit.providers.models.BackendProperties).

Клас [`BackendV2`](https://docs.quantum.ibm.com/api/qiskit/qiskit.providers.BackendV2) переглянув
доступ користувача до більшості властивостей Backend, щоб вони працювали з нативними структурами даних Qiskit і мали більш плоскі
шаблони доступу. Ядром моделі [`BackendV2`](https://docs.quantum.ibm.com/api/qiskit/qiskit.providers.BackendV2) є клас
[`Target`](https://docs.quantum.ibm.com/api/qiskit/qiskit.transpiler.Target) — представлення QPU, що містить обмеження транспіляції, які Qiskit може використовувати для оптимізації Circuit для виконання.

Qiskit SDK було оновлено для роботи виключно з вхідними даними [`BackendV2`](https://docs.quantum.ibm.com/api/qiskit/qiskit.providers.BackendV2),
і більшість провайдерів оновилися з [`BackendV1`](https://docs.quantum.ibm.com/api/qiskit/1.4/qiskit.providers.BackendV1) на [`BackendV2`](https://docs.quantum.ibm.com/api/qiskit/qiskit.providers.BackendV2). Очікується, що існуючі провайдери визнають застарілим старий доступ, де це можливо, для плавної міграції, але врешті-решт користувачам потрібно буде адаптувати свій код.

Принцип [`BackendV2`](https://docs.quantum.ibm.com/api/qiskit/qiskit.providers.BackendV2) полягає в тому,
що більша частина інформації про Backend міститься в його об'єкті
[`Target`](https://docs.quantum.ibm.com/api/qiskit/qiskit.transpiler.Target), і атрибути Backend часто звертаються
до атрибута `BackendV2.target` для повернення інформації. Проте в багатьох випадках атрибути надають лише
підмножину інформації, яку може містити Target. Наприклад, `backend.coupling_map`
повертає [`CouplingMap`](https://docs.quantum.ibm.com/api/qiskit/qiskit.transpiler.CouplingMap), побудовану з
[`Target`](https://docs.quantum.ibm.com/api/qiskit/qiskit.transpiler.Target), доступного в атрибуті
`BackendV2.target`. Проте Target може містити
інструкції, які працюють з більш ніж двома Qubit (що не може бути представлено в
[`CouplingMap`](https://docs.quantum.ibm.com/api/qiskit/qiskit.transpiler.CouplingMap)), або може мати інструкції, які працюють лише з
підмножиною Qubit (або двокубітних зв'язків для двокубітної інструкції), що не буде
детально відображено в повній карті зв'язків, що повертається
`BackendV2.coupling_map`. Тому, залежно від твого випадку використання,
може бути необхідно заглибитися далі, ніж просто еквівалентний доступ через
[`BackendV2`](https://docs.quantum.ibm.com/api/qiskit/qiskit.providers.BackendV2).

## Конкретні зміни в `BackendV2` {#specific-changes-in-backendv2}

Більшість атрибутів мають пряму заміну, що спрощує зусилля з міграції. Єдиною точкою розбіжності між інтерфейсами є [`CouplingMap`](https://docs.quantum.ibm.com/api/qiskit/qiskit.transpiler.CouplingMap).

Нижче наведена таблиця прикладів шаблонів доступу в [`BackendV1`](https://docs.quantum.ibm.com/api/qiskit/1.4/qiskit.providers.BackendV1) та нова форма
з [`BackendV2`](https://docs.quantum.ibm.com/api/qiskit/qiskit.providers.BackendV2).

Прокрути праворуч, щоб побачити важливі примітки.
</Admonition>

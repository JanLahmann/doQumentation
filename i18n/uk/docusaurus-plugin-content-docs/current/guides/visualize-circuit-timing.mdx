Окрім [візуалізації інструкцій на схемі](/guides/visualize-circuits), ти можеш також візуалізувати планування схеми за допомогою методу Qiskit [`timeline_drawer`](https://docs.quantum.ibm.com/api/qiskit/qiskit.visualization.timeline_drawer). Ця візуалізація може допомогти, наприклад, швидко виявити час простою кубітів. Однак цей метод не повертає точних результатів для динамічних схем. Для візуалізації планування динамічних схем використовуй метод `draw_circuit_schedule_timing`, як описано в розділі [Підтримка Qiskit Runtime](#qr-support).

## Підтримка Qiskit Runtime {#qiskit-runtime-support}

Хоча вбудований у Qiskit планувальник часових ліній корисний для статичних схем, він може не точно відображати часові параметри [динамічних схем](/guides/classical-feedforward-and-control-flow) через неявні операції, такі як broadcasting та визначення гілок. У рамках підтримки динамічних схем Qiskit Runtime повертає точну інформацію про часові параметри схеми в результатах завдань, якщо це було запрошено.

<Admonition type="note" title="Примітки">
- Це експериментальна функція. Вона перебуває у статусі попереднього релізу і тому може змінюватись.
- Ця функція застосовується лише до завдань Sampler у Qiskit Runtime.
- Хоча загальний час схеми повертається в метаданих «compilation», це НЕ є часом, що використовується для виставлення рахунків (квантовий час).
</Admonition>
### Увімкнення отримання даних про час виконання {#enable-timing-data-retrieval}

Щоб увімкнути отримання даних про час виконання, встанови експериментальний прапорець `scheduler_timing` в `True` під час запуску примітивного завдання.

```python
from qiskit_ibm_runtime import QiskitRuntimeService, SamplerV2

service = QiskitRuntimeService()
backend = service.least_busy(operational=True, simulator=False)

pm = generate_preset_pass_manager(backend=backend, optimization_level=1)
isa_circuit = pm.run(qc)

sampler = SamplerV2(backend)
sampler.options.experimental = {
    "execution": {
        "scheduler_timing": True,
    },
}

sampler_job = sampler.run([isa_circuit])
result = sampler_job.result()
```

### Доступ до даних про час виконання схеми {#access-the-circuit-timing-data}

За запитом дані про час виконання схеми для кожного PUB повертаються в метаданих результату завдання за ключем `["compilation"]["scheduler_timing"]["timing"]`. Це поле містить необроблену інформацію про час. Для відображення інформації про час використовуй вбудований інструмент візуалізації, як описано в розділі [Візуалізація часових параметрів](#visualize-timings).

Використовуй наступний код для доступу до даних про час виконання схеми для першого PUB:

```python
job_result = sampler_job.result()
circuit_schedule = job_result[0].metadata["compilation"]["scheduler_timing"]
circuit_schedule_timing = circuit_schedule["timing"]
```

#### Розуміння необроблених даних про час виконання {#understand-the-raw-timing-data}

Хоча візуалізація даних про час виконання схеми за допомогою методу `draw_circuit_schedule_timing` є найбільш поширеним випадком використання, може бути корисно розуміти структуру необроблених даних про час, що повертаються. Це може допомогти тобі, наприклад, програмно витягати інформацію.

Дані про час, що повертаються в `["compilation"]["scheduler_timing"]["timing"]`, є списком рядків. Кожен рядок представляє одну інструкцію на якомусь каналі і розділений комами на такі типи даних:

- `Branch` — визначає, чи знаходиться інструкція в керуючому потоці (then / else) або в головній гілці.
- `Instruction` — вентиль і кубіт, на якому потрібно виконати операцію.
- `Channel` — канал, якому призначається інструкція. Може бути одним із таких:
      - `Qubit x` — канал керування для кубіта _x_.
      - `AWGRx_y` (arbitrary waveform generator readout, генератор довільних форм хвиль зчитування) — використовується каналами зчитування для передачі даних при вимірюванні кубітів. Аргументи _x_ і _y_ відповідають ідентифікатору інструменту зчитування та номеру кубіта відповідно.
- `T0` — час початку інструкції в межах повного розкладу.
- `Duration` — тривалість інструкції в одиницях _dt_ секунд, де 1 dt = 1 цикл планування. Значення `dt` бекенду можна знайти за допомогою [`backend.dt`](https://docs.quantum.ibm.com/api/qiskit/qiskit.providers.BackendV2#dt).
- `Pulse` — тип імпульсної операції, що використовується.

Приклад:

```python
main,barrier,Qubit 0,7,0,barrier # A barrier on the main branch on qubit 0 at time 7 with 0 duration
main,reset_0,Qubit 0,7,64,play # A reset instruction on the main branch on qubit 0 at time 7 with duration 64 and a play operation
...
```

### Візуалізація часових параметрів {#visualize-the-timings}

Починаючи з `qiskit-ibm-runtime` v0.43.0 або новіших версій, ти можеш візуалізувати часові параметри схеми. Для цього спочатку потрібно перетворити метадані результату на `fig` за допомогою [методу `draw_circuit_schedule_timing`.](https://github.com/Qiskit/qiskit-ibm-runtime/blob/3d1bf1e1d49e5123841639fce259859c90ce9314/qiskit_ibm_runtime/visualization/draw_circuit_schedule_timings.py#L26) Цей метод повертає фігуру `plotly`, яку можна відобразити безпосередньо, зберегти у файл або зробити і те, й інше. #### Розуміння згенерованої фігури {#understand-the-generated-figure}

Зображення даних про час виконання схеми, що виводиться методом `draw_circuit_schedule_timing`, передає наступну інформацію:

- Вісь X — час в одиницях _dt_ секунд, де 1 dt = 1 цикл планування. Значення `dt` бекенду можна знайти за допомогою [`backend.dt`](https://docs.quantum.ibm.com/api/qiskit/qiskit.providers.BackendV2#dt).
- Вісь Y — канал (думай про канали як про інструменти, що генерують імпульси).
    - `Receive channel` — єдиний канал, що не є самостійним інструментом. Це інструкція, що відтворюється на всіх каналах, які беруть участь у процедурі зв'язку з хабом у цей момент часу.
    - `Qubit x` — канал керування для кубіта x.
    - `AWGRx_y` (arbitrary waveform generator readout, генератор довільних форм хвиль зчитування) — використовується каналами зчитування для передачі даних при вимірюванні кубітів. Аргументи _x_ і _y_ відповідають ідентифікатору інструменту зчитування та номеру кубіта відповідно.
    - `Hub` — керує broadcasting.

Крім того, кожна інструкція має формат *X_Y*, де *X* — назва інструкції, а *Y* — тип імпульсу. Тип `play` застосовує керуючі імпульси, а `capture` записує стан кубіта. Ти також можеш навести курсор на кожну інструкцію, щоб отримати більше деталей. Наприклад, попередня фігура показує керуючий імпульс для вентиля X, застосованого до кубіта 10 в момент 1161 dt.
### Наскрізний приклад {#end-to-end-example}

Цей приклад показує, як увімкнути опцію, отримати інформацію про часові параметри схеми з метаданих і відобразити її як зображення.

Спочатку налаштуй середовище, визнач схеми та перетвори їх на ISA-схеми, визнач завдання та запусти їх.

```python
from qiskit_ibm_runtime import SamplerV2, QiskitRuntimeService
from qiskit.circuit import QuantumCircuit, QuantumRegister, ClassicalRegister
from qiskit.transpiler import generate_preset_pass_manager

service = QiskitRuntimeService()
backend = service.least_busy(operational=True, simulator=False)

# Create a dynamic circuit

qubits = QuantumRegister(1)
clbits = ClassicalRegister(1)
qc = QuantumCircuit(qubits, clbits)
(q0,) = qubits
(c0,) = clbits

qc.h(q0)
qc.measure(q0, c0)
with qc.if_test((c0, 1)):
    qc.x(q0)
qc.measure(q0, c0)

# Convert to an ISA circuit for the given backend

pm = generate_preset_pass_manager(backend=backend, optimization_level=1)
isa_circuit = pm.run(qc)

# Generate samplers for backend targets
sampler = SamplerV2(backend)
sampler.options.experimental = {"execution": {"scheduler_timing": True}}

# Submit jobs
sampler_job = sampler.run([isa_circuit])
result = sampler_job.result()

print(
    f">>> {' Job ID:':<10}  {sampler_job.job_id()} ({sampler_job.status()})"
)
```

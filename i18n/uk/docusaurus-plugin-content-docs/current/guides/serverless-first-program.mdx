## Огляд робочого процесу {#workflow-overview}

1. Створи локальний каталог і порожній файл програми (`./source_files/transpile_remote.py`)

1. Використай `qiskit-ibm-catalog` для автентифікації в Qiskit Serverless
1. Завантаж програму до Qiskit Serverless

Після завантаження програми можна запустити її для транспіляції схеми, дотримуючись інструкцій посібника [Запусти своє перше робоче навантаження Qiskit Serverless віддалено](/guides/serverless-run-first-workload).

## Приклад: Віддалена транспіляція з Qiskit Serverless {#example-remote-transpilation-with-qiskit-serverless}

У цьому прикладі наведено кроки зі створення та наповнення файлу програми, який після завантаження до Qiskit Serverless транспілюватиме `circuit` для заданого `backend` і цільового `optimization_level`.

:::tip

Qiskit Serverless вимагає організувати `.py`-файли робочого навантаження в окремий каталог. Наступна структура є прикладом хорошої практики:

```text
serverless_program
├── program_uploader.ipynb
└── source_files
├── transpile_remote.py
└── *.py
```

:::

Serverless завантажує вміст конкретного каталогу (у цьому прикладі — каталогу `source_files`) для віддаленого запуску. Після налаштування можна скоригувати `transpile_remote.py` для отримання вхідних даних і повернення результатів.

### Створи каталог і порожній файл програми {#create-the-directory-and-an-empty-program-file}

Спочатку створи каталог `source_files`, а потім файл програми в ньому так, щоб його шлях був `./source_files/transpile_remote.py`. Це файл, який ти завантажиш до Qiskit Serverless.
### Додай код до файлу програми {#add-code-to-your-program-file}

Заповни файл програми наведеним нижче кодом, а потім збережи його.

<Admonition type="caution">
Якщо ти читаєш блоки коду локально в ноутбуці, ти побачиш [магічну команду](https://ipython.readthedocs.io/en/stable/interactive/magics.html#cellmagic-writefile) `%%writefile`. Виконання блоків з цією магічною командою зберігає їх на диск, а не виконує.
</Admonition>

```python
# This cell is hidden from users, it creates a new folder
from pathlib import Path

Path("./source_files").mkdir(exist_ok=True)
```

```python
%%writefile ./source_files/transpile_remote.py
# If you include the preceding `%%writefile` command (visible only when you read this
# locally in a notebook), running this cell saves to disk rather than executing the code.

from qiskit.transpiler import generate_preset_pass_manager

def transpile_remote(circuit, optimization_level, backend):
    """Transpiles an abstract circuit into an ISA circuit for a given backend."""
    pass_manager = generate_preset_pass_manager(
        optimization_level=optimization_level,
		backend=backend
    )
    isa_circuit = pass_manager.run(circuit)
    return isa_circuit
```

### Додай код для отримання аргументів програми {#add-code-to-get-program-arguments}

Тепер додай такий код до файлу програми, що налаштовує аргументи програми.

Початковий `transpile_remote.py` має три вхідні параметри: `circuits`, `backend_name` і `optimization_level`. Serverless наразі підтримує лише серіалізовані вхідні й вихідні дані. З цієї причини не можна передати `backend` безпосередньо — замість цього використовуй `backend_name` як рядок.

```python
%%writefile --append ./source_files/transpile_remote.py
# If you include the preceding `%%writefile` command (visible only when you read this
# locally in a notebook), running this cell saves to disk rather than executing the code.

from qiskit_serverless import get_arguments, save_result, distribute_task, get

# Get program arguments
arguments = get_arguments()
circuits = arguments.get("circuits")
backend_name = arguments.get("backend_name")
optimization_level = arguments.get("optimization_level")
```

### Додай код для виклику бекенду {#add-code-that-calls-the-backend}

Додай такий код до файлу програми, що викликає бекенд через `QiskitRuntimeService`.

Наведений нижче код передбачає, що ти вже виконав процедуру збереження облікових даних за допомогою `QiskitRuntimeService.save_account`, і завантажить твій обліковий запис за замовчуванням, якщо не вказано інше. Дивись [Збережи свої облікові дані для входу](/guides/save-credentials) і [Ініціалізуй обліковий запис сервісу Qiskit Runtime](/guides/initialize-account) для отримання додаткової інформації.

```python
%%writefile --append ./source_files/transpile_remote.py
# If you include the preceding `%%writefile` command (visible only when you read this
# locally in a notebook), running this cell saves to disk rather than executing the code.

from qiskit_ibm_runtime import QiskitRuntimeService

service = QiskitRuntimeService()
backend = service.backend(backend_name)
```

### Додай код для транспіляції {#add-code-to-transpile}

Нарешті, додай такий код до файлу програми. Цей код запускає `transpile_remote()` для всіх переданих `circuits` і повертає `transpiled_circuits` як результат:

```python
%%writefile --append ./source_files/transpile_remote.py
# If you include the preceding `%%writefile` command (visible only when you read this
# locally in a notebook), running this cell saves to disk rather than executing the code.

# Each circuit is being transpiled and will populate the array
results = [
    transpile_remote(circuit, 1, backend)
    for circuit in circuits
]

save_result({
    "transpiled_circuits": results
})
```

### Автентифікація в Qiskit Serverless {#authenticate-to-qiskit-serverless}

Використовуй `qiskit-ibm-catalog` для автентифікації в `QiskitServerless` за допомогою свого API-ключа (можна використовувати API-ключ `QiskitRuntimeService` або створити новий на [дашборді IBM Quantum Platform](https://quantum.cloud.ibm.com)).

```python
from qiskit_ibm_catalog import QiskitServerless, QiskitFunction

# Authenticate to the remote cluster and submit the pattern for remote execution
serverless = QiskitServerless()
```

### Запусти код для завантаження {#run-code-to-upload}

Запусти наведений нижче код для завантаження програми. Qiskit Serverless стискає вміст `working_dir` (у цьому випадку `source_files`) у `tar`-архів, який завантажується, а після цього очищається. `entrypoint` визначає головний виконуваний файл програми для Qiskit Serverless.

```python
transpile_remote_demo = QiskitFunction(
    title="transpile_remote_serverless",
    entrypoint="transpile_remote.py",
    working_dir="./source_files/",
)
```

```python
serverless.upload(transpile_remote_demo)
```

```text
QiskitFunction(transpile_remote_serverless)
```

### Перевір завантаження {#verify-upload}

Щоб перевірити успішність завантаження, використовуй `serverless.list()`, як у наведеному нижче коді:

```python
# Get program from serverless.list() that matches the title of the one we uploaded
next(
    program
    for program in serverless.list()
    if program.title == "transpile_remote_serverless"
)
```

```text
QiskitFunction(transpile_remote_serverless)
```

```python
# This cell is hidden from users, it checks the program uploaded correctly
assert _.title == "transpile_remote_serverless"  # noqa: F821
```

```python
# This cell is hidden from users, it checks the program executes correctly
from time import sleep
from qiskit import QuantumCircuit

qc = QuantumCircuit(2)
transpile_remote_serverless = serverless.load("transpile_remote_serverless")
job = transpile_remote_serverless.run(
    circuits=[qc],
    backend="ibm_sherbrooke",
    optimization_level=1,
)
while True:
    sleep(5)
    status = job.status()
    if status not in ["QUEUED", "INITIALIZING", "RUNNING", "DONE"]:
        raise Exception(
            f"Unexpected job status: '{status}'\n"
            + "Here are the logs:\n"
            + job.logs()
        )
    if status == "DONE":
        break
```

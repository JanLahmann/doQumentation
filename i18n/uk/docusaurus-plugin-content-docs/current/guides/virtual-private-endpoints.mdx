# Використання віртуальних приватних кінцевих точок для VPC для приватного підключення до IBM Quantum Platform {#use-virtual-private-endpoints-for-vpc-to-privately-connect-to-ibm-quantum-platform}

Використовуй Virtual Private Endpoints (VPE) для VPC, щоб підключатися до IBM Quantum&reg; зі своєї мережі VPC за допомогою IP-адреси, яку ти вказуєш і яку виділено з підмережі твого VPC.

VPE — це віртуальні IP-інтерфейси, прив'язані до шлюзу кінцевої точки, що створюється для кожного сервісу окремо. Шлюз кінцевої точки — це віртуалізована функція, що масштабується горизонтально, є надлишковою та високодоступною, і охоплює всі зони доступності твого VPC. Шлюзи кінцевих точок забезпечують зв'язок між екземплярами віртуальних серверів у твоєму VPC і сервісом IBM Cloud&reg; через приватну магістраль. VPE для VPC дає тобі контроль над усією приватною адресацією в хмарі. Для отримання додаткової інформації дивись [About virtual private endpoint gateways](https://cloud.ibm.com/docs/vpc?topic=vpc-about-vpe).

У межах IBM Quantum Platform усі дані клієнтів передаються через приватну мережу незалежно від того, чи здійснюється доступ через публічну кінцеву точку, чи через VPE.

Підключення до IBM Quantum Platform через публічні кінцеві точки передає всі дані запитів і відповідей через відкритий інтернет. Щоб підключитися до IBM Quantum Platform за допомогою VPE, необхідно використовувати API IBM Quantum Platform або SDK. Доступ до IBM Quantum Platform можна отримати лише через публічну мережу.

## Перед початком роботи {#before-you-begin}

Перед тим як налаштовувати VPE для IBM Quantum Platform, виконай такі кроки:

* [Створи Virtual Private Cloud](https://cloud.ibm.com/docs/vpc?topic=vpc-getting-started).
* [Сплануй топологію мережі](https://cloud.ibm.com/docs/vpc?topic=vpc-planning-considerations) для підключення до VPE.
* [Налаштуй контроль доступу](https://cloud.ibm.com/docs/vpc?topic=vpc-configure-acls-sgs-endpoint-gateways) для свого VPE.
* Ознайомся з [обмеженнями](https://cloud.ibm.com/docs/vpc?topic=vpc-limitations-vpe) використання VPE.
* Дізнайся, як [переглядати деталі VPE](https://cloud.ibm.com/docs/vpc?topic=vpc-vpe-viewing-details-of-an-endpoint-gateway).

## Налаштування VPE для IBM Quantum Platform {#set-up-a-vpe-for-ibm-quantum-platform}

Є кілька способів створення шлюзу VPE. Якщо ти використовуєш [CLI](https://cloud.ibm.com/docs/vpc?topic=vpc-vpc-reference&interface=cli#vpe-clis) або [API](https://cloud.ibm.com/apidocs/vpc/latest#create-endpoint-gateway), необхідно вказати [Cloud Resource Name (CRN)](https://cloud.ibm.com/docs/account?topic=account-crn) регіону, до якого хочеш підключитися з IBM Quantum Platform. Перегляньте наведену нижче таблицю з доступними регіонами та CRN.

| Регіон | Плани   | Повне доменне ім'я (FDQN) | Cloud Resource Name (CRN) |
|-----------------|-----------------|-----------------|-----------------|
| `us-east` | `Standard` | `private.us-east.quantum.cloud.ibm.com` | `crn:v1:bluemix:public:quantum-computing:us-east:::endpoint:private.us-east.quantum.cloud.ibm.com` |
| `eu-de` | `Standard` | `private.eu-de.quantum.cloud.ibm.com` | `crn:v1:bluemix:public:quantum-computing:eu-de:::endpoint:private.eu-de.quantum.cloud.ibm.com` |

### Налаштування шлюзу кінцевої точки {#configuring-an-endpoint-gateway}

Щоб налаштувати шлюз VPE, виконай такі кроки:

1. Перегляньте список доступних сервісів, включаючи сервіси інфраструктури IBM Cloud, що доступні (за замовчуванням) усім користувачам VPC.
1. [Створи шлюз кінцевої точки](https://cloud.ibm.com/docs/vpc?topic=vpc-ordering-endpoint-gateway) для IBM Quantum Platform, який має бути приватно доступний для VPC.
1. [Прив'яжи зарезервовану IP-адресу](https://cloud.ibm.com/docs/vpc?topic=vpc-bind-unbind-reserved-ip) до шлюзу кінцевої точки.
1. [Переглянь створені шлюзи VPE](https://cloud.ibm.com/docs/vpc?topic=vpc-vpe-viewing-details-of-an-endpoint-gateway), пов'язані з IBM Quantum Platform.

Тепер екземпляри віртуальних серверів у твоєму VPC можуть приватно отримувати доступ до екземпляра IBM Quantum Platform.

## Використання VPE для IBM Quantum Platform {#use-your-vpe-for-ibm-quantum-platform}

Після створення шлюзу кінцевої точки для IBM Quantum Platform виконай такі кроки:

### Використання VPE з `qiskit-ibm-runtime` (Python SDK) {#use-the-vpe-with-qiskit-ibm-runtime-python-sdk}

<Admonition type="note">
Підтримка VPE потребує `qiskit-ibm-runtime` v0.24.0 або новішої версії.
</Admonition>

При створенні екземпляра `QiskitRuntimeService` вкажи `private_endpoint=True`.

```python
service = QiskitRuntimeService(token="your-API_KEY", # Use the 44-character API_KEY you created and saved from the IBM Quantum Platform Home dashboard
                              instance="INSTANCE_CRN",
                              channel="ibm_cloud",
                              private_endpoint=True)
```

### Використання VPE з IBM Quantum API {#use-the-vpe-with-the-ibm-quantum-api}

Після створення шлюзу кінцевої точки для IBM Quantum Platform використовуй FQDN кінцевої точки сервісу для цільового регіону.

```bash
  curl -X POST https://private.us-east.quantum.cloud.ibm.com/api/v1/jobs -H "Authorization: Bearer $BEARER_TOKEN" -H "Service-CRN: $SERVICE_INSTANCE_CRN" -d '{
    "backend": "backend",
    "program_id": "sampler"
  }'
```

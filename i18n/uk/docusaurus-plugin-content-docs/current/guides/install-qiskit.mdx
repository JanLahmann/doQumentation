# Встановлення Qiskit {#install-qiskit}

Незалежно від того, чи будете Випрацювати локально або в хмарному середовищі, першим кроком для всіх користувачів є встановлення Qiskit.

<Admonition type="caution" title="Оновлення з Qiskit 0.x до Qiskit 1.0 і вище">
  (Якщо Вивстановлюєте Qiskit вперше, перейдіть до розділу [Встановлення та налаштування](#local). Це повідомлення стосується лише користувачів, які раніше встановлювали Qiskit.)

  **Для тих, хто оновлюється з версії 0.x до 1.0 або пізнішої**: зверніть увагу, що оскільки Qiskit v1.0 використовує нову структуру пакетів, Ви**не можете** використовувати `pip install -U qiskit` для оновлення з будь-якої версії Qiskit 0.x до 1.0.

  Докладніше та інструкції дивіться у [посібнику з міграції на Qiskit 1.0](/guides/qiskit-1.0).

  Майбутні оновлення, починаючи з Qiskit 1.0, дозволятимуть оновлення на місці.
</Admonition>

## Встановлення Qiskit SDK та клієнта Qiskit Runtime {#install-the-qiskit-sdk-and-the-qiskit-runtime-client}

1. Встановіть Python. Перевірте розділ "Programming Language" на [сторінці проєкту Qiskit PyPI](https://pypi.org/project/qiskit/), щоб визначити, які версії Python підтримуються останнім випуском. Інструкції зі завантаження дивіться в [Посібнику для початківців Python.](https://wiki.python.org/moin/BeginnersGuide/Download)

    Рекомендується використовувати [віртуальні середовища Python](https://docs.python.org/3.10/tutorial/venv.html) для відокремлення Qiskit від інших додатків.

    <Admonition type="note">
    Ці інструкції використовують стандартний дистрибутив Python з [pypi.org](https://pypi.org/). Однак Виможете використовувати інші дистрибутиви Python, такі як [Anaconda](https://docs.anaconda.com/anaconda/) або [miniconda](https://docs.anaconda.com/miniconda/), а також інші робочі процеси управління залежностями, як-от [Poetry](https://python-poetry.org/docs/).
    </Admonition>

    <details>
    <summary>
    Якщо Виновачок у віртуальних середовищах, натисніть тут для отримання додаткової інформації.
    </summary>
    Віртуальне середовище Python — це ізольований простір для роботи з Python для конкретної мети — тож Виможете встановлювати будь-які пакети, налаштовувати бібліотеки, залежності тощо, не впливаючи на "базове" середовище Python на Вашій машині.

    Одна важлива перевага віртуального середовища полягає в тому, що якщо Ваше середовище Python десь пошкодиться, Виможете легко видалити віртуальне середовище та почати знову!

    Виберіть бажане місце для зберігання інформації про Ваші віртуальні середовища. Зазвичай вони зберігаються в каталозі з назвою `.venv` у кожному каталозі проєкту, над яким Випрацюєте.
    </details>

    Спочатку перейдіть до каталогу Вашого проєкту та створіть мінімальне середовище лише з встановленим Python.

          <TabItem value="linux" label="Linux">

          </TabItem>

          <TabItem value="win" label="Windows">
            ```text
            python -m venv .venv
            ```
          </TabItem>
         </OperatingSystemTabs>

    Далі активуйте Ваше нове середовище.

         <OperatingSystemTabs>
           <TabItem value="mac" label="macOS">
             ```shell
             source .venv/bin/activate
             ```
           </TabItem>

           <TabItem value="linux" label="Linux">
             ```shell
             source .venv/bin/activate
             ```
           </TabItem>

           <TabItem value="win" label="Windows">
              Якщо використовуєте PowerShell:

             ```text

             ```
             Якщо використовуєте Git Bash:

             ```text
             source .venv/scripts/activate
             ```
              Якщо використовуєте командний рядок:

             ```text
             .venv\Scripts\activate
             ```
           </TabItem>

1. [Встановіть pip](https://pip.pypa.io/en/stable/installation/), якщо він ще не встановлений у Вашому середовищі. Pip — це менеджер пакетів Python, який Вивикористовуєте для встановлення Qiskit та інших пакетів Python. Використовуйте `pip list`, щоб побачити, що знаходиться у Вашому віртуальному середовищі. У більшості середовищ Python pip вже встановлено.

1. Встановіть Qiskit SDK. Якщо Виплануєте виконувати завдання на квантовому обладнанні, також встановіть Qiskit Runtime.

      ```shell
      pip install qiskit
      ```

      ```shell
      pip install qiskit-ibm-runtime
      ```

      <Admonition type="tip" title="Додайте підтримку візуалізації">
      Якщо Вимаєте намір використовувати функціонал візуалізації або Jupyter notebooks, також встановіть модуль `qiskit.visualization`. Докладніше дивіться у [довідці API візуалізацій](https://docs.quantum.ibm.com/api/qiskit/visualization).
      </Admonition>

<span id="local-jupyter"></span>
1. Якщо Вихочете запустити Jupyter notebook з пакетами Qiskit, які Вищойно встановили, Вам потрібно встановити Jupyter у Вашому середовищі.

      ```shell
      pip install jupyter
      ```
    Потім відкрийте свій notebook наступним чином:

      ```shell
      jupyter notebook <path/to/notebook.ipynb>
      ```

Якщо Виплануєте працювати локально та використовувати симулятори, вбудовані в Qiskit, то Ваше встановлення завершено. Якщо Вихочете виконувати завдання на QPU IBM, перейдіть до [налаштування облікового запису IBM Cloud](cloud-setup).

<Admonition type="note" title="Будьте в курсі останніх версій">
    Періодично перевіряйте [примітки до випуску Qiskit](../api/qiskit/release-notes) та [примітки до випуску Qiskit Runtime](../api/qiskit-ibm-runtime/release-notes), щоб побачити нові випуски. Ми рекомендуємо часто оновлювати Ваші вимоги для `qiskit` та `qiskit-ibm-runtime`, наприклад, змінюючи версії в `requirements.txt` на останні версії, а потім запускаючи `pip install -r requirements.txt` або відповідну команду для Вашого робочого процесу управління залежностями.
</Admonition>

## Виправлення проблем {#troubleshooting}

<details>
  <summary>
    Помилка "No Module 'qiskit'" з Jupyter Notebook
  </summary>
  Якщо Вивикористовували ``pip install qiskit`` і налаштували своє віртуальне середовище в
  Anaconda, то Виможете отримати помилку ``No Module 'qiskit'``, коли запускаєте підручник
  в Jupyter Notebook. Якщо Вине встановили Qiskit або не налаштували своє
  віртуальне середовище, Виможете виконати кроки [встановлення](#qiskit-install).

  Помилка виникає при спробі імпортувати пакет Qiskit в
  середовищі, де Qiskit не встановлено. Якщо Визапустили Jupyter Notebook
  з Anaconda-Navigator, можливо, що Jupyter Notebook працює
  в базовому (root) середовищі, а не у Вашому віртуальному
  середовищі. Виберіть віртуальне середовище в Anaconda-Navigator з
  випадаючого меню **Applications on**. У цьому меню Виможете побачити
  всі віртуальні середовища в Anaconda, і Виможете
  вибрати середовище, де у Вас встановлено Qiskit, щоб запустити Jupyter
  Notebook.
</details>

<details>
  <summary>
    Помилки компіляції під час встановлення
  </summary>
  Qiskit залежить від низки інших пакетів Python з відкритим кодом, які
  автоматично встановлюються при виконанні ``pip install qiskit``. Залежно від
  платформи Вашої системи та версії Python, можливо, що певний
  пакет не надає попередньо зібраних бінарних файлів для Вашої системи. Ви можете звернутися
  до [Підтримка операційних систем](#operating-system-support) для списку платформ, що підтримуються Qiskit, деякі
  з яких можуть потребувати додатковий компілятор. У випадках, коли немає
  доступних попередньо скомпільованих бінарних файлів, ``pip`` спробує скомпілювати пакет
  з джерельного коду, що, у свою чергу, може вимагати деяких додаткових залежностей, які потрібно
  встановити вручну.

  Якщо вивід ``pip install qiskit`` містить подібні рядки до:

  ```
  Failed building wheel for SOME_PACKAGE
  ...
  build/temp.linux-x86_64-3.5/_openssl.c:498:30: fatal error
  compilation terminated.
  error: command 'x86_64-linux-gnu-gcc' failed with exit status 1
  ```

  будь ласка, перевірте документацію пакета, який не вдалося встановити (у
  прикладі коду, ``SOME_PACKAGE``) для інформації про те, як встановити бібліотеки,
  необхідні для компіляції з джерельного коду.
</details>

## Підтримка операційних систем {#operating-system-support}

Qiskit прагне підтримувати якомога більше операційних систем, але через обмеження у доступних ресурсах тестування та доступності операційних систем не всі операційні системи можуть підтримуватися. Підтримка операційних систем для Qiskit поділяється на три рівні з різними рівнями підтримки для кожного рівня. Для платформ поза цими, таких як FreeBSD або WebAssembly (WASI), Qiskit все ще може бути встановлений, але він не тестується, і Вам доведеться зібрати Qiskit (і, ймовірно, залежності Qiskit) з джерельного коду.

Крім того, Qiskit підтримує лише реалізацію CPython мови Python. Запуск з іншими інтерпретаторами Python, такими як PyPy, не підтримується.

### Qiskit v2.x

У серії випусків Qiskit v2.x підтримуються платформи:

<details>
  <summary>
    Рівень 1
  </summary>
  Операційні системи рівня 1 повністю тестуються як частина процесів розробки, щоб гарантувати, що будь-яка запропонована зміна буде функціонувати коректно. Попередньо скомпільовані бінарні файли створюються, тестуються та публікуються на PyPI як частина процесу випуску. Зазвичай, за умови наявності функціонуючого середовища Python, Qiskit можна встановити на цих операційних системах без необхідності встановлення додаткових залежностей.

  Операційні системи рівня 1:

  - Linux x86_64 (дистрибутиви, сумісні зі специфікацією пакування [manylinux 2014](https://www.python.org/dev/peps/pep-0599/)).
  - macOS ARM64 (11.0 або новіша)
  - Windows 64-bit (підтримується Windows 10 і пізніші)
  - Linux AArch64 (дистрибутиви, сумісні зі специфікацією пакування [manylinux 2014](https://www.python.org/dev/peps/pep-0599/))
</details>

<details>
  <summary>
    Рівень 2
  </summary>
  Операційні системи рівня 2 не тестуються як частина процесу розробки. Однак попередньо скомпільовані бінарні файли створюються, тестуються та публікуються на PyPI як частина процесу випуску, і ці пакети можна очікувати встановити лише з функціонуючим середовищем Python.
  Може бути затримка у випуску пакетів для цих систем, оскільки помилки тестування можуть бути виявлені лише пізніше, і помилки під час публікації не блокуватимуть публікацію пакета для систем рівня 1.

  Операційні системи рівня 2:

  - macOS x86_64 (10.12 або пізніша)
</details>

<details>
  <summary>
    Рівень 3
  </summary>
  Операційні системи рівня 3 не тестуються як частина процесу розробки. Попередньо скомпільовані бінарні файли створюються та публікуються на PyPI як частина процесу випуску, але не тестуються. Вони можуть не встановлюватися лише з функціонуючим середовищем Python і можуть вимагати компілятор C/C++ або додаткові програми для збирання залежностей з джерельного коду як частина процесу встановлення. Підтримка цих операційних систем надається за принципом максимальних зусиль.

  Операційні системи рівня 3:

  - Linux ppc64le (дистрибутиви, сумісні зі специфікацією пакування [manylinux 2014](https://www.python.org/dev/peps/pep-0599/))
  - Linux s390x (дистрибутиви, сумісні зі специфікацією пакування [manylinux 2014](https://www.python.org/dev/peps/pep-0599/))
</details>

Починаючи з Qiskit v2.0.0, підтримуються лише 64-бітні платформи, а 32-бітні платформи не підтримуються. Ви також не зможете зібрати з джерельного коду
на 32-бітних платформах, оскільки внутрішньо код Rust у Qiskit припускає 64-бітну ширину вказівника.

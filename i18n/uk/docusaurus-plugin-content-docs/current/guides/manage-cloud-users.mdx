# Керування користувачами IBM Cloud {#manage-ibm-cloud-users}

Цей посібник показує, як використовувати IBM Cloud&reg; для надання доступу користувачам, які мають облікові записи IBM Cloud, та містить інструкції для користувачів щодо входу в середовище.

Щоб керувати користувачами через провайдер ідентифікаторів, дотримуйся інструкцій у розділі [Керування провайдерами ідентифікаторів](/guides/manage-appid).

## Кроки {#steps}

Виконай ці основні кроки для керування користувачами IBM Cloud:

1. Переконайся, що користувачі, яких ти хочеш запросити, мають облікові записи IBM Cloud.
2. Перейди до IAM і [запроси користувачів](https://cloud.ibm.com/iam/users/invite_users)
   <Admonition type="note">
   Керувати користувачами неможливо, поки вони не приймуть запрошення та не увійдуть хоча б один раз.
   </Admonition>
3. Необов'язково: змінюй призначення проєктів користувачів і групи доступу через [IAM.](https://cloud.ibm.com/iam/users)

<span id="user-org"></span>
## Процес для користувача {#user-flow}

1. Користувач приймає запрошення, після чого входить через [портал IBM Cloud](https://cloud.ibm.com/).
2. Користувач створює ключ API, перейшовши до ([Manage → Access (IAM) → API keys](https://cloud.ibm.com/iam/apikeys)).
3. Для отримання додаткової інформації користувачі можуть ознайомитись із розділом [Встановлення Qiskit.](/guides/install-qiskit)

<span id="steps-cloud-org"></span>
## Приклад сценарію {#example-scenario}

У цьому прикладі створюється така конфігурація:

* Є два проєкти: `ml` і `finance`.
   * Проєкт `ml` повинен мати доступ до екземплярів сервісу `QR-ml` і `QR-common`.
   * Проєкт `finance` повинен мати доступ до екземплярів сервісу `QR-finance` і `QR-common`.
* Є три користувачі:
   * Fatima повинна мати доступ до проєкту `ml`.
   * Ravi повинен мати доступ до проєкту `finance`.
   * Amyra повинна мати доступ до обох проєктів.
* Використовуються групи доступу без груп ресурсів.
* Користувачі визначені в IBM Cloud, і там само виконується призначення проєктів.
* Користувачі повинні мати можливість видаляти завдання.

Кроки для реалізації цієї конфігурації:

2. Адміністратор Cloud створює три екземпляри сервісу: `QR-ml`, `QR finance` і `QR-common`.
3. Адміністратор Cloud створює власне правило, яке включає дію `quantum-computing.job.delete`.
1. Адміністратор Cloud створює дві групи доступу:
   * Група доступу `ml` має доступ до `QR-ml` і `QR-common`.
   * Група доступу `finance` має доступ до `QR-finance` і `QR-common`.
1. Адміністратор Cloud запрошує хмарних користувачів до відповідного проєкту. Зокрема, він запрошує та призначає користувачів до групи доступу, що включає цей проєкт.
   * Fatima додається до групи доступу "ml".
   * Ravi додається до групи доступу "finance".
   * Amyra додається до груп доступу "ml" і "finance".
1. Користувачі можуть входити через портал IBM Cloud, створювати ключі API та працювати з екземплярами сервісу своїх проєктів.

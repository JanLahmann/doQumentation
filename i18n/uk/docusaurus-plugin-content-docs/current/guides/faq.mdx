# Часті запитання (FAQ) {#frequently-asked-questions-faq}

Питання, що стосуються режимів виконання, дивись у посібнику [FAQ щодо режимів виконання Qiskit Runtime](/guides/execution-modes-faq).

## Як процитувати Qiskit у дослідженні? {#how-do-i-cite-qiskit-in-my-research}

Цитуй Qiskit, використовуючи вкладений [файл BibTeX](https://github.com/Qiskit/qiskit/blob/main/CITATION.bib).

## Як процитувати комп'ютер IBM Quantum у дослідженні? {#how-do-i-cite-an-ibm-quantum-computer-in-my-research}

У наукових статтях ми рекомендуємо авторам подякувати IBM Quantum&reg;, використовуючи:

>    We acknowledge the use of IBM Quantum services for this work. The views expressed are those of the authors, and do not reflect the official policy or position of IBM or the IBM Quantum team.

У посиланнях у статті слід використовувати таке цитування:

> IBM Quantum. https://<span>quantum.cloud.ibm.com</span>/, YYYY

Квантові процесорні блоки у статті слід посилатися за їхньою унікальною назвою (наприклад, `ibmq_vigo`) і за бажанням
додавати версію (наприклад, `ibmq_vigo` v1.0.2). Ми також заохочуємо посилатися на процесор.
Наприклад:

> In this paper we used `ibmq_vigo`, which is one of the IBM Quantum Canary processors.

Приклад цитування програми IBM Quantum:

> IBM Quantum (2025). Estimator primitive (Version x.y.z) [computer software]. https://<span>quantum.cloud.ibm.com</span>/

## Я не бачу щось в інтерфейсі користувача, наприклад завдань адміністрування. {#i-dont-see-something-in-the-user-interface-such-as-my-administration-tasks}

Спробуй вибрати інший акаунт за допомогою перемикача акаунтів у заголовку інтерфейсу [IBM Quantum Platform](https://quantum.cloud.ibm.com).

## Я користуюся лише планом Open Plan, але мені довелося вказати кредитну картку. Чи буде здійснено списання? {#im-just-using-the-open-plan-but-i-had-to-put-in-my-credit-card-will-i-be-charged}

Кошти не буде списано, якщо ти не створиш [інстанс](/guides/instances) з оплатою по факту використання (Pay-As-You-Go) і не будеш надсилати навантаження на QPU через нього. Дивись [Керування витратами](/guides/manage-cost), щоб дізнатися, як встановити ліміт витрат та інші способи управління ними.

## У чому різниця між ключем API та токеном API? {#whats-the-difference-between-an-api-key-and-an-api-token}

Різниці немає. Терміни «ключ API» та «токен API» використовуються як взаємозамінні. Ти побачиш обидва терміни в документації, оскільки різні інтерфейси використовують різні терміни.

<span id="bearer"></span>
## У чому різниця між токеном API та токеном Bearer? {#whats-the-difference-between-an-api-token-and-a-bearer-token}

Токен API — це довгостроковий обліковий запис, що використовується для [автентифікації в сервісі Qiskit Runtime.](/guides/hello-world#install-and-authenticate) Токен Bearer — це тимчасовий обліковий запис, що використовується для [автентифікації до REST API в ненадійному середовищі.](/guides/cloud-setup-untrusted#authenticate-to-the-rest-api-in-an-untrusted-environment) Токен Bearer не може бути використаний для автентифікації в сервісі Qiskit Runtime.

## Чи потрібно вибирати регіон, у якому я живу? {#do-i-have-to-choose-the-region-that-i-live-in}

Ні. Ти можеш використовувати будь-який регіон. Регіон визначає, де виконуються завдання та де зберігаються дані. Крім того, ти можеш бачити та використовувати лише ті інстанси, які були створені в тому інстансі, до якого ти увійшов. Зверни увагу, що лише регіон us-east підтримує план Open Plan.

## Чи можу я підключитися через проксі-сервер? {#can-i-connect-through-a-proxy-server}

Так, але ти маєш використовувати Qiskit Runtime v0.44.0 або пізнішу версію.

## Яка політика зберігання даних? {#what-is-the-data-retention-policy}

IBM зберігає твої (невидалені) дані про завдання та використання протягом трьох років, поки ти продовжуєш користуватися сервісом Qiskit Runtime.

## Що сталося з дистрибутивами Python, такими як Qiskit Aer, Qiskit Nature та Qiskit Experiments? {#what-happened-to-python-distributions-such-as-qiskit-aer-qiskit-nature-and-qiskit-experiments}

Починаючи з виходу Qiskit SDK v1.0, команда `pip install qiskit` встановлюватиме **лише** [Qiskit SDK](https://github.com/Qiskit/qiskit). Це означає, що деякі модулі, які ти раніше використовував, наприклад `qiskit-aer` або `qiskit-nature`, тепер потрібно встановлювати окремо.

До виходу Qiskit SDK v1.0 Qiskit складався з багатьох різних дистрибутивів Python, таких як `qiskit-terra` — ядро компілятора; `qiskit-aer` — високопродуктивний симулятор; оригінальний провайдер IBM Quantum&reg;; а також кілька нині застарілих пакетів, що надавали певні дослідницькі алгоритмічні функції або засоби для запуску експериментів.

Ці окремі програмні пакети були об'єднані в так званий *метапакет* Qiskit, що дозволяло користувачам встановлювати всі ці компоненти разом за допомогою `pip install qiskit`. Ця архітектура метапакету була видалена з виходом Qiskit SDK v1.0, і тепер кожен із окремих програмних дистрибутивів потрібно встановлювати окремо.

Якщо ти хочеш встановити пакети, що раніше входили до метапакету Qiskit, відвідай [екосистему Qiskit](https://www.ibm.com/quantum/ecosystem), щоб знайти широкий вибір пакетів для своїх потреб. Також можна прочитати [посібник з міграції v1.0](/guides/metapackage-migration), щоб отримати більше інформації про цю зміну.

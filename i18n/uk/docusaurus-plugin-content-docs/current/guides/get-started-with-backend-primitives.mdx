# Початок роботи з backend-примітивами {#get-started-with-the-backend-primitives}

На відміну від примітивів, специфічних для певного провайдера, backend-примітиви — це загальні реалізації, які можна використовувати з довільним об'єктом `backend`, якщо він реалізує інтерфейс [`BackendV2`](https://docs.quantum.ibm.com/api/qiskit/qiskit.providers.BackendV2). Деякі провайдери реалізують примітиви нативно. Докладніше на [сторінці екосистеми Qiskit](https://www.ibm.com/quantum/ecosystem?tag=provider).

<span id="estimator"></span>
## Початок роботи з backend-примітивом Estimator {#get-started-with-the-estimator-backend-primitive}

Примітив Estimator можна запускати з будь-яким провайдером, використовуючи клас [`qiskit.primitives.BackendEstimatorV2`](https://docs.quantum.ibm.com/api/qiskit/qiskit.primitives.BackendEstimatorV2). Однак він не пропонує реалізацій пом'якшення помилок вимірювання або вентилів "з коробки", оскільки backend-примітиви призначені для локального запуску на машині користувача.

### Приклад: {#example}

```python
from qiskit.primitives import BackendEstimatorV2
from <some_qiskit_provider> import QiskitProvider

provider = QiskitProvider()
backend = provider.get_backend('backend_name')
estimator = BackendEstimatorV2(backend)
```

<span id="sampler"></span>
## Початок роботи з backend-примітивом Sampler {#get-started-with-the-sampler-backend-primitive}

Примітив Sampler можна запускати з будь-яким провайдером, використовуючи [`qiskit.primitives.BackendSamplerV2`](https://docs.quantum.ibm.com/api/qiskit/qiskit.primitives.BackendSamplerV2). Однак він потребує backend, що підтримує опцію `memory`.

### Приклад: {#example}

```python
from qiskit.primitives import BackendSamplerV2
from <some_qiskit_provider> import QiskitProvider

provider = QiskitProvider()
backend = provider.get_backend('backend_name')
sampler = BackendSamplerV2(backend)
```

# Розуміння високої доступності та відновлення після збоїв для Qiskit Runtime {#understanding-high-availability-and-disaster-recovery-for-qiskit-runtime}

Висока доступність (HA) — це здатність сервісу залишатися працездатним і доступним у разі непередбачених збоїв. Відновлення після збоїв — це процес повернення екземпляра сервісу до робочого стану. Клієнти Qiskit Runtime не несуть відповідальності за планування відновлення після збоїв або пов'язані дії.

Qiskit Runtime — це регіональний сервіс, що виконує визначені [Цілі рівня обслуговування (SLO)](https://cloud.ibm.com/docs/resiliency?topic=resiliency-slo) для будь-якого платного плану. Докладніше — в [Угоді про рівень обслуговування (SLA)](https://www.ibm.com/support/customer/csol/terms/?id=i126-9268&lc=en) для IBM Cloud&reg;. Інформацію про доступні регіони та центри обробки даних IBM Cloud для Qiskit Runtime дивись у розділі [Доступність сервісів та інфраструктури за місцезнаходженням.](https://cloud.ibm.com/docs/overview?topic=overview-services_region)

### Функції високої доступності {#high-availability-features}

Цей сервіс розгорнуто в трьох зонах у регіоні з кількома зонами — без єдиної точки відмови. API-запити маршрутизуються через балансувальник навантаження до вузлів HA-екземпляра, кожен з яких знаходиться в окремій зоні.

Якщо вузол або зона HA-екземпляра виходить з ладу, сервіс продовжує обробляти API-запити через вузли HA-екземпляра, що залишились. Під час зональної відмови помітного погіршення якості обслуговування не відбувається.

Сервіс доступний у таких регіонах:

- Вашингтон (us-east)
- Франкфурт (eu-de)

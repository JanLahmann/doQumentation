# Події відстеження активності для сервісу IBM Quantum System {#activity-tracking-events-for-the-ibm-quantum-system-service}

Сервіси IBM Cloud&reg;, такі як сервіс IBM&reg; Quantum System, генерують події відстеження активності.

Події відстеження активності повідомляють про дії, що змінюють стан сервісу в IBM Cloud. Ці події можна використовувати для розслідування аномальної активності та критичних дій, а також для відповідності нормативним вимогам аудиту.

Можна використовувати IBM Cloud Activity Tracker Event Routing — платформний сервіс — для маршрутизації подій аудиту у своєму акаунті до потрібних призначень шляхом налаштування цілей та маршрутів, що визначають, куди надсилаються події відстеження активності. Докладніше див. у [Про IBM Cloud Activity Tracker Events Routing](https://cloud.ibm.com/docs/atracker?topic=atracker-about).

Можна використовувати IBM Cloud Logs для візуалізації та оповіщення про події, що генеруються у твоєму акаунті та маршрутизуються IBM Cloud Activity Tracker Event Routing до екземпляра IBM Cloud Logs.

## Перегляд подій відстеження активності для сервісу IBM Quantum System {#viewing-activity-tracking-events-for-the-ibm-quantum-system-service}

Можна використовувати IBM Cloud Logs для візуалізації та оповіщення про події, що генеруються у твоєму акаунті та маршрутизуються IBM Cloud Activity Tracker Event Routing до екземпляра IBM Cloud Logs.

### Запуск IBM Cloud Logs зі сторінки Observability {#launching-ibm-cloud-logs-from-the-observability-page}

Інформацію про запуск інтерфейсу IBM Cloud Logs дивись у розділі [Launching the UI](https://cloud.ibm.com/docs/cloud-logs?topic=cloud-logs-instance-launch#instance-launch-cloud-ui) документації IBM Cloud Logs.

| Дія | Ініціюється |
| -- | -- |
| `quantum-computing.direct-access-backend-configuration.read` | Отримання конфігурації backend (`GET /v1/backends/{backend-name}/configuration`) |
| `quantum-computing.direct-access-backend-properties.read` | Отримання властивостей backend (`GET /v1/backends/{backend-name}/properties`) |
| `quantum-computing.direct-access-backend.list` | Отримання списку backends (`GET /v1/backends`) |
| `quantum-computing.direct-access-backend.read` | Отримання деталей backend (`GET /v1/backends/{backend-name}`) |
| `quantum-computing.direct-access-job.cancel` | Скасування завдання (`POST /v1/jobs/{job-id}/cancel`) |
| `quantum-computing.direct-access-job.create` | Запуск завдання (`POST /v1/jobs`) |
| `quantum-computing.direct-access-job.delete` | Видалення завдання (`DELETE /v1/jobs/{job-id}`) |
| `quantum-computing.direct-access-job.list` | Отримання завдань (`GET /v1/jobs`) |
| `quantum-computing.direct-access-job.read` | Отримання завдання за ID (`GET /v1/jobs/{job-id}`) |

# Керування витратами {#manage-cost}

Цей посібник описує, як встановити ліміт витрат для інстансу тарифного плану «Оплата за використання». Також можна керувати витратами, [мінімізуючи квантовий час роботи](/guides/minimize-time) та встановлюючи [максимальний час виконання](/guides/max-execution-time) для своїх робочих навантажень.

_Ліміт витрат_ — це загальна вартість усіх робочих навантажень, запущених у цьому інстансі з моменту його створення; він завжди більший або рівний загальній вартості. Після того як інстанс досягає вказаної кількості загальних секунд, подальші робочі навантаження не можуть бути запущені й додаткові витрати не нараховуються.

    <Admonition type="note">
      Ліміт витрат завжди вказується в доларах США (USD), а потім конвертується у секунди виконання. Однак для щомісячного виставлення рахунків тебе стягують плату в місцевій валюті, яка вказана у твоєму обліковому записі IBM Cloud&reg;. Оскільки обмінні курси можуть коливатися, вартість X секунд виконання може відрізнятися від суми, початково розрахованої в USD, та від суми, яку фактично стягнуть у місцевій валюті. Тому, якщо твоя місцева валюта не є USD, загальна сума, стягнута за кількість секунд, зазначену в цьому полі, може відрізнятися від указаної суми в доларах.
    </Admonition>

## Встановлення або зміна ліміту витрат {#set-or-change-a-cost-limit}

Існує кілька способів встановити або змінити ліміт витрат:

* **IBM Quantum Platform**: Відкрий сторінку [Instances](https://quantum.cloud.ibm.com/instances). Знайди інстанс, для якого хочеш встановити ліміт, натисни меню «Actions» в кінці рядка, а потім — **Edit details**. Встанови загальний ліміт витрат або ліміт використання, а потім натисни **Save changes**.
   <Admonition type="note">
   Ліміт витрат або використання також можна встановити при створенні нового інстансу в тарифному плані «Оплата за використання». Коли ти вибираєш плитку Pay-As-You-Go, з'являється поле з міткою "Total cost limit (Optional)". У це поле можна ввести загальний ліміт витрат.
   </Admonition>

* **API**: [Встановлення ліміту інстансу в секундах.](https://docs.quantum.ibm.com/api/qiskit-runtime-rest/tags/instances#tags__instances__operations__replace_instance_data)

   ```api
   PUT /v1/instances/configuration "instance_limit":<seconds>
   ```

* **IBM Cloud CLI**: Інструкції щодо використання CLI наведені у розділі [Getting started with the IBM Cloud CLI.](https://cloud.ibm.com/docs/cli?topic=cli-getting-started)

   ```cli
      ibmcloud resource service-instance-update <instance_CRN>
      -p '{"instance_limit_seconds":"<seconds>"}'
   ```

## Перегляд ліміту витрат {#view-the-cost-limit}

Існує кілька способів переглянути поточний ліміт витрат:

* **IBM Quantum Platform**: Максимальний ліміт витрат відображається на вкладці Pay-As-You-Go у таблиці [Instances](https://quantum.cloud.ibm.com/instances).

* **API**:

   ```api
   GET /v1/instances/configuration
   ```
   Дивись [документацію REST API](https://docs.quantum.ibm.com/api/qiskit-runtime-rest/tags/instances).

* **IBM Cloud CLI**: Інструкції щодо використання CLI наведені у розділі [Getting started with the IBM Cloud CLI.](https://cloud.ibm.com/docs/cli?topic=cli-getting-started)

   ```cli
   ibmcloud resource service-instance-update <instance_CRN> -p
   '{"instance_limit_seconds":null}'
   ```

## Видалення ліміту витрат {#remove-a-cost-limit}

Ліміт витрат можна видалити кількома способами:

* **IBM Quantum Platform**: Відкрий таблицю [Instances](https://quantum.cloud.ibm.com/instances). Знайди інстанс, який хочеш оновити, натисни меню переповнення в кінці рядка, а потім — **Edit details**. Видали загальний ліміт витрат або ліміт використання, а потім натисни **Save changes**.

* **API**: [Встанови ліміт інстансу на `null`.](https://docs.quantum.ibm.com/api/qiskit-runtime-rest/tags/instances)

   ```api
   PUT /v1/instances/configuration "instance_limit":null
   ```

* **IBM Cloud CLI**: Інструкції щодо використання CLI наведені у розділі [Getting started with the IBM Cloud CLI.](https://cloud.ibm.com/docs/cli?topic=cli-getting-started)

   ```cli
   ibmcloud resource service-instance-update <instance_CRN> -p
   '{"instance_limit_seconds":null}'
   ```
<Admonition type="note">
Редагувати інстанс можуть лише адміністратори та власники. Інформацію про призначення доступу дивись у посібнику [Створення політик і груп доступу](/guides/access-groups).
</Admonition>

### Що відбувається при досягненні ліміту витрат {#what-happens-when-the-cost-limit-is-reached}

Коли ліміт витрат інстансу досягнуто, поточне завдання, що виконується, зупиняється. Його статус змінюється на «Скасовано» з причиною «Виконувалося надто довго». Усі доступні часткові результати зберігаються.

Жодні подальші завдання не можуть бути подані з використанням цього інстансу, доки ліміт витрат не буде збільшено.

## Як дізнатися, скільки з тебе стягують {#how-to-see-what-youre-being-charged}

Щомісяця ти отримуватимеш рахунок із детальною інформацією про витрати за ресурси. Перевірити, скільки витрачено, можна в будь-який час на сторінці [IBM Cloud Billing and usage](https://cloud.ibm.com/billing).

Крім того, у будь-який момент можна визначити вартість для конкретного інстансу або завдання.

### Перегляд використання {#view-usage}

Переглядай використання інстансу на сторінці [Analytics](https://quantum.cloud.ibm.com/analytics).

### Перегляд вартості інстансу {#view-instance-cost}

Щоб дізнатися, яку суму було виставлено до рахунку інстансу в поточному розрахунковому циклі, на сторінці [Instances](https://quantum.cloud.ibm.com/instances) натисни на інстанс, щоб відкрити сторінку його деталей.

Нижче наведено поля, що стосуються вартості:

- **Billing cycle usage**: Обсяг _квантового часу_, використаного цим інстансом у поточному розрахунковому циклі. Квантовий час — це тривалість, протягом якої QPU зайнятий виконанням запиту користувача.
- **Billing cycle cost**: Загальна вартість виконання завдань у поточному розрахунковому циклі.
- **Total usage**: Обсяг квантового часу, використаного цим інстансом з моменту його створення.
- **Total cost**: Загальна вартість виконання завдань у цьому інстансі з моменту його створення. Лише адміністратори можуть встановлювати це значення.

Свій розрахунковий цикл можна переглянути на [сторінці «Billing and usage»](https://cloud.ibm.com/billing).

### Перегляд вартості завдання {#view-job-cost}

Щоб дізнатися, яку суму виставлено до рахунку для кожного завдання, пов'язаного з інстансом, на сторінці [Instances](https://quantum.cloud.ibm.com/instances) натисни на інстанс, щоб відкрити сторінку його деталей. Потім у лівій частині натисни «Jobs».

Нижче наведено стовпці, що стосуються вартості:

- **Usage**: Обсяг квантового часу, використаного цим завданням. Квантовий час — це тривалість, протягом якої QPU зайнятий виконанням запиту користувача.
- **Cost**: Загальна вартість виконання цього завдання.

## Налаштування сповіщень про витрати {#set-up-spending-notifications}

Можна налаштувати сповіщення про витрати, щоб отримувати повідомлення, коли твій обліковий запис або конкретний сервіс досягає встановленого тобою порогу витрат. Для отримання інформації дивись сторінку [IBM Cloud Account types](https://cloud.ibm.com/docs/account?topic=account-accounts). Сповіщення про витрати IBM Cloud слід використовувати разом з іншими методами управління витратами з кількох причин:

*   Сповіщення спрацьовують лише *після* того, як витрати перевищують указаний ліміт.
*   Дані про витрати передаються до системи виставлення рахунків щогодини. Тому між поданням завдання і відправленням сповіщення про витрати може виникнути значна затримка.
*   Системі виставлення рахунків може знадобитися кілька днів, щоб передати інформацію до системи формування рахунків-фактур, що може спричинити додаткові затримки у надходженні сповіщень. Для отримання додаткової інформації про те, як працює система виставлення рахунків IBM Cloud, дивись [Setting spending notifications.](https://cloud.ibm.com/docs/billing-usage?topic=billing-usage-spending)

## IBM Quantum Platform {#latest-platform}

У цьому розділі підсумовано останні вдосконалення та нові функції нової [IBM Quantum Platform.](https://quantum.cloud.ibm.com/)

**Січень 2026**

- Останні вдосконалення сторінки [Обчислювальні ресурси](https://quantum.cloud.ibm.com/computers) включають таке:

  - **Єдиний вигляд QPU** — усі QPU в різних регіонах тепер відображаються в одній таблиці, що спрощує перегляд статусу та калібрувань спільного парку між двома регіонами. (Ти все ще можеш фільтрувати таблицю обчислювальних ресурсів за регіоном за допомогою значка лійки.)

  - **Єдиний графік технічного обслуговування** — графік технічного обслуговування тепер знаходиться на початку сторінки, навіть якщо на даний момент не заплановано жодних подій.

  - **Деталі «Доступ до інстанції»** — при перегляді деталей QPU новий розділ «Доступ до інстанції» показує всі інстанції, які надають користувачу доступ до цього QPU для вибраного облікового запису.

- **Перший QPU Nighthawk та останній QPU Heron вже доступні.** Ці два потужні нові процесори були представлені на [IBM Quantum Developer Conference](https://www.ibm.com/quantum/blog/qdc-2025) у листопаді.

  - Premium та Flex плани тепер мають ранній доступ до `ibm_miami` — першого IBM Quantum QPU Nighthawk. Наш найпросунутіший квантовий процесор на сьогодні, Nighthawk — це 120-кубітний пристрій з топологією квадратної решітки (218 зв'язувачів порівняно з 176 у Heron), розроблений для підтримки складніших робочих навантажень. Nighthawk має найвищу когерентність нашого парку — навіть вищу, ніж було повідомлено на QDC — із середнім значенням T1 350 &#181;s.

  - Зверни увагу, що `ibm_miami` наразі є дослідницьким QPU, який ще тестується та оптимізується в рамках наших постійних циклів навчання. Тому він тимчасово має такі обмеження:

    - Час повторення за замовчуванням тимчасово збільшено з 250 &#181;s до 4 мс, що вплине на необхідний час QPU для виконання робочих навантажень
    - Динамічні схеми ще не підтримуються, а якість вимірювань у середині схеми обмежена

  - Крім того, плани Premium, Flex та Pay-As-You-Go тепер мають доступ до `ibm_boston` — нашого найпродуктивнішого IBM Quantum Heron r3 на сьогодні. Він продемонстрував EPLG на 100 кубітах рівня 2.15e-3, із 57 із 176 можливих двокубітних гейтів нижче рівня похибки 1e-3.

- **Представляємо архівування інстанцій для адміністраторів на IBM Quantum Platform.** З цим оновленням адміністратори можуть архівувати будь-яку інстанцію в межах свого облікового запису. Архівування інстанції гарантує, що:

    - Учасники зберігають доступ до результатів і метрик QPU, властивостей та калібрувань з попередніх робочих навантажень.
    - До архівованої інстанції не можна надсилати нові робочі навантаження, що запобігає ненавмисному використанню.
    - Архівовані інстанції не зараховуються до розподілу плану облікового запису.
    - Усі архівовані інстанції акуратно організовані під спеціальною вкладкою «Архів».
    - Адміністратори можуть реактивувати інстанцію в будь-який час.

## Qiskit SDK v2.3 {#latest-qiskit-sdk}

**Січень 2026**

- Спираючись на імпульс, закладений у серії v2.x, яка зосередилася на позиціонуванні Qiskit як провідної платформи для гібридних квантово-HPC робочих процесів, [Qiskit SDK v2.3](https://www.ibm.com/quantum/blog/qiskit-2-3-release-summary) забезпечує значне розширення C API разом із швидшими та гнучкішими інструментами для побудови та оптимізації квантових схем.

  Основні оновлення включають:

  - **Власні транспіляційні проходи на C** — пиши власні транспіляційні проходи через C API Qiskit за допомогою нового об'єкта `QkDag` і розширеного об'єкта `QkTarget`, які дозволяють покроково перевіряти, модифікувати та розширювати процес компіляції — без перебудови всього конвеєра компілятора.

  - **Швидший вибір розміщення на апаратному забезпеченні** — оновлення та покращення продуктивності на основі Rust для `VF2Layout` і `VF2PostLayout` підвищують швидкість і масштабованість вибору розміщення схема-апаратне забезпечення, скорочуючи накладні витрати на компіляцію та покращуючи точність на квантовому апаратному забезпеченні.

  - **Нове багатокубітне вимірювання Паулі** — нова інструкція `PauliProductMeasurement` дозволяє виконувати спільне проективне вимірювання кількох кубітів в одній операції, відкриваючи можливість компіляції для обчислень на основі Паулі — поширеного представлення в квантових обчисленнях з виправленням помилок.

  - **Вдосконалення ранньої відмовостійкої транспіляції** — покращення продуктивності та функцій для транспіляції до ранніх відмовостійких цілей — особливо цілей Clifford+T — допомагають користувачам ефективніше будувати транспіляційні конвеєри для майбутніх QPU.

  Ці можливості надають розробникам більшу гнучкість для налаштування оптимізацій квантових схем та інтеграції з наявними HPC-робочими процесами. [Відвідай блог](https://www.ibm.com/quantum/blog/qiskit-2-3-release-summary) для повного технічного огляду.

  Попередній перегляд Каталогу Qiskit Functions надає доступ користувачам Premium Plan для вивчення доступних функцій, включаючи написані IBM та написані іншими членами нашої екосистеми. Каталог містить два типи функцій: функції схем та функції застосунків.

    - **Функції схем** забезпечують спрощений інтерфейс для запуску схем. Вони отримують надані користувачем абстрактні схеми та спостережувані як вхідні дані, а потім керують синтезом, оптимізацією та виконанням відповідної ISA-схеми. Функції схем поєднують найновіші можливості транспіляції, придушення помилок та пом'якшення помилок, щоб зробити продуктивність рівня корисності доступною з коробки. Це дозволяє обчислювальним вченим зосередитися на відображенні своїх задач у схеми, а не на побудові патерну для кожної задачі з нуля.

    - **Функції застосунків** охоплюють завдання вищого рівня, як-от вивчення алгоритмів та сценаріїв використання для конкретних предметних областей. Корпоративні розробники та науковці з обробки даних можуть не мати базових знань з квантової інформатики для роботи зі схемами і натомість сподіваються привнести свої предметні знання для просування квантових обчислювальних алгоритмів і застосунків. За допомогою функцій застосунків користувачі можуть вводити класичні дані та отримувати рішення, щоб легше експериментувати з інтеграцією квантових можливостей у свої предметно-орієнтовані робочі процеси.

- З запуском Каталогу Qiskit Functions розробники Premium Plan можуть вивчити IBM Circuit function. IBM Circuit function включає найновіші розширення Qiskit на основі ШІ для синтезу, оптимізації та планування схем, а також передові методи пом'якшення помилок для отримання найбільш точних оцінок, можливих із сучасним апаратним забезпеченням.

  Користувачі можуть придбати ліцензії на такі функції, надані нашими партнерами Q-CTRL, QEDMA та Algorithmiq.

- **Функції схем**

  - Q-CTRL випускає функцію схеми, яка застосовує методи квантового управління на основі ШІ, завдяки чому користувачі можуть успішно масштабуватися до більших задач.
  - Algorithmiq випускає функцію схеми, яка застосовує TEM (пом'якшення помилок за допомогою тензорних мереж) — метод пом'якшення помилок для отримання оцінок з меншою кількістю вимірювань, ніж метод PEC (скасування імовірнісних помилок).
  - QEDMA випускає функцію схеми, яка використовує власні протоколи для ефективної та точної характеризації шумових операцій QPU, та застосовує придушення та пом'якшення помилок на основі даних характеризації.

- **Функції застосунків**

  - Q-CTRL випускає розв'язувач задач оптимізації, за допомогою якого користувачі можуть передавати граф або цільову функцію та отримувати вартості рішень.
</details>

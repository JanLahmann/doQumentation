# Використання IBM Cloud Platform API для доступу до інстансів {#use-ibm-cloud-platform-apis-to-access-instances}

Цей посібник призначений для тих, хто використовує IBM Cloud Platform API для підготовки, оновлення або керування доступом до інстансів. Тут описано [параметри](#parameters), які можна задати під час підготовки інстансу, а також набір [полів розширень](#extensions), що повертаються при доступі до інстансу.

<Admonition type="note">
Для отримання додаткової інформації звернись до документації IBM Cloud&reg; ResourceController API.

Деякі корисні сторінки:
- [Створення (підготовка) нового інстансу ресурсу](https://cloud.ibm.com/apidocs/resource-controller/resource-controller#create-resource-instance)
- [Оновлення інстансу ресурсу](https://cloud.ibm.com/apidocs/resource-controller/resource-controller#update-resource-instance)
- [Отримання інстансу ресурсу](https://cloud.ibm.com/apidocs/resource-controller/resource-controller#get-resource-instance)
- [Отримання списку всіх інстансів ресурсів](https://cloud.ibm.com/apidocs/resource-controller/resource-controller#list-resource-instances)

    При отриманні списку всіх інстансів ресурсів зверни увагу, що можна відфільтрувати лише інстанси Qiskit Runtime за допомогою `resource_id = "b6049020-80f4-11eb-a0f7-e35ec9b4054f"`

</Admonition>

## `parameters` {#parameters}

У тілі запиту при підготовці або оновленні інстансу:

- **instance_limit_seconds** (рядок або null, `^[0-9]+$`): Ліміт, що встановлюється для інстансу. Для Open Plan: якщо не вказано, за замовчуванням дорівнює 10 хвилинам.
.
- **usage_allocation_seconds** (рядок або null, `^[0-9]+$`): Виділений час для інстансу, який використовується планувальником справедливого розподілу для визначення пріоритету в черзі на основі використання всіх QPU. (Не застосовується до інстансів Pay-As-You-Go.)
- **backends** (масив списків бекендів (рядки або null) або масив з "ANY"): список дозволених імен бекендів, доступних для цього інстансу. За замовчуванням: `["ANY"]` — всі бекенди, доступні в цьому плані. `[]` — жодних бекендів.

## `extensions` {#extensions}

У відповіді при підготовці або при отриманні деталей інстансу:

Додаткові властивості інстансу, надані сервісом, представлені у вигляді пар ключ-значення. Ці властивості зберігаються разом з інстансом у властивості extensions.

- **instance_limit_seconds** (ціле число або null, `>= 0`): Ліміт часу, встановлений для інстансу.
- **usage_allocation_seconds** (ціле число або null, `>= 0`): Виділений час для інстансу, який використовується планувальником справедливого розподілу для визначення пріоритету в черзі на основі використання всіх QPU.
- **backends** (Обов'язково; масив списків бекендів (рядки) або масив з "ANY" (рядки)): Список дозволених імен бекендів, доступних для цього інстансу. За замовчуванням: `["ANY"]` — всі бекенди доступні в цьому плані. `[]` — жодних бекендів.

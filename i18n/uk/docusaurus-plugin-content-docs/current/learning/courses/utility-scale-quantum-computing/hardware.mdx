## 2. Backend та Target {#2-backend-and-target}

Qiskit надає API для отримання інформації про квантовий пристрій — як статичної, так і динамічної. Ми використовуємо екземпляр Backend для взаємодії з пристроєм; він включає екземпляр Target — абстрактну модель машини, яка узагальнює ключові характеристики, такі як архітектура набору інструкцій (ISA) та будь-які пов'язані властивості або обмеження.
Скористаємось цими екземплярами backend, щоб отримати деяку інформацію, яку можна побачити на сторінці [Compute resources](https://quantum.cloud.ibm.com/computers) IBM Quantum&reg; Platform. Спочатку створимо екземпляр backend для пристрою, що нас цікавить. Нижче ми вибираємо «ibm_kyoto», «ibm_kawasaki» або найменш зайнятий пристрій Eagle. Твій доступ до QPU може відрізнятися; оновлюй назву backend відповідно.

```python
service = QiskitRuntimeService()
# backend = service.backend("ibm_kawasaki") # an Eagle, if you have access to ibm_kawasaki
backend = service.least_busy(
    operational=True, simulator=False, min_num_qubits=127
)  # Eagle
backend.name
```

```text
'ibm_strasbourg'
```

Почнімо з деякої базової (статичної) інформації про пристрій.

```python
print(
    f"""

processor type = {backend.processor_type}
basis gates = {backend.basis_gates}
"""
)
```

```text

processor type = {'family': 'Eagle', 'revision': 3} 
basis gates = ['ecr', 'id', 'rz', 'sx', 'x']
```

## 3. Властивості кубітів {#3-qubit-properties}

Пристрій Eagle має 127 кубітів. Отримаємо властивості деяких із них.

```python
for qn in range(backend.num_qubits):
```

### 3.2 Помилки гейтів та зчитування {#32-gate-and-readout-errors}

```python
target = backend.target
target.keys()
```


Значення також є словниками. Розглянемо деякі елементи значення (словника) для операції 'sx'.

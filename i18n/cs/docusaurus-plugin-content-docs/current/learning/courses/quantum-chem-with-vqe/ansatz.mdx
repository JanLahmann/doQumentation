## Kód ansatzu {#ansatz-code}

V předchozí lekci jsi vytvořil/a __Hamiltonián__, který popisuje energii molekuly, která tě zajímá, a namapoval/a jej do formátu vhodného pro kvantový počítač. VQE používá _variační obvod_ k přípravě kvantových stavů. Tyto stavy pak použijeme k určení střední hodnoty Hamiltoniánu (energie). Parametry ve variačním obvodu jsou měněny, dokud výpočet nekonverguje k minimální střední hodnotě. V kontextu kvantové chemie by to měla být energie základního stavu. Tato lekce se zaměřuje na variační obvod, nazývaný také __ansatz__ (německé slovo znamenající „přístup" nebo „metoda").
V této lekci se naučíš
- sadu předpřipravených ansaetzů dostupných v knihovně obvodů
- jak specifikovat nebo upravit vlastnosti ansatzu
- jak sestavit vlastní ansatz
- příklady dobrých a špatných ansaetzů

Knihovna obvodů Qiskit má mnoho kategorií obvodů, které lze použít jako ansatz. Zde omezíme naši diskuzi na dvoulokální obvody (obvody sestávající z hradel, která působí nanejvýš na dva qubity najednou). __Efficient SU2__ je běžně používaný ansatz.

Obvod `efficient_su_2` se skládá z vrstev jednoqubitových operací rozložených přes SU(2) (speciální unitární skupina stupně 2, jako jsou Pauliho rotační hradla) a CX provázání. Jedná se o heuristický vzor, který může být užitečný ve variačních kvantových algoritmech jako VQE a klasifikačních obvodech v kvantovém strojovém učení (QML).

Začneme čtyřqubitovým příkladem obvodu `efficient_su2` se dvěma typy SU(2) hradel, řekněme rx a y. Také specifikujeme schéma provázání a počet opakování. Pokud jednoduše zavoláš `.draw()` na obvodu, dostaneš poměrně abstraktní reprezentaci. Přehlednější diagram obvodu získáš pomocí `.decompose().draw()`, a zde použijeme `output = "mpl"`.


```python
from qiskit.circuit.library import efficient_su2

SU2_ansatz = efficient_su2(4, su2_gates=["rx", "y"], entanglement="linear", reps=1)
print(SU2_ansatz.draw())
SU2_ansatz.decompose().draw(output="mpl")
```

Hradla SU(2) se objevují na začátku a na konci v pořadí a prvcích specifikovaných v su2_gates = [...]. Schéma provázání `linear` znamená, že hradlo CX prochází číslovanými qubity, provazuje 0 a 1, pak 1 a 2 a tak dále po diagonální linii v obvodu. Jak asi očekáváš, nastavení `reps = 2` jednoduše přidá vrstvu provázání a závěrečnou vrstvu SU(2). Nastavení `reps = n` odpovídá `n` vrstvám provázání, s vrstvami SU(2) mezi nimi a na každém konci.

```python
SU2_ansatz2 = efficient_su2(
    4, su2_gates=["rx", "y", "z"], entanglement="linear", reps=2
)
SU2_ansatz2.decompose().draw(output="mpl")
```

Existuje několik dalších schémat provázání. Dvě, která stojí za zmínku, jsou `circular` a `full`. Kruhové provázání je identické s lineárním provázáním, ale s přidaným hradlem CX provazujícím první a poslední qubit. Schéma plného provázání zahrnuje hradlo CX mezi každým párem qubitů. Poznamenejme, že pro N-qubitový obvod je to $N(N-1)/2$ hradel $CX$, a může být výpočetně nákladné.

```python
SU2_ansatz3 = efficient_su2(
    4, su2_gates=["rx", "y", "z"], entanglement="circular", reps=1
)
SU2_ansatz3.decompose().draw(output="mpl")
```

Zobecněním efficient_su2 je dvoulokální obvod, který je sám o sobě speciálním případem n-lokálních obvodů. Dvoulokální obvod také obsahuje bloky SU(2) (nebo rotační bloky) a bloky provázání. Zde máme volnost specifikovat typ entanglovacího hradla, které chceme použít, například hradlo CRX. V tomto příkladu všechna hradla přijímají parametr, ale to nemusí být vždy nutné. Například by bylo možné použít hradla Y rotace a hradlo CX pro provázání.

```python
from qiskit.circuit.library import n_local

rotation_blocks = ["ry"]
entanglement_blocks = ["crx"]
two_ansatz = n_local(
    4, rotation_blocks, entanglement_blocks, "linear", insert_barriers=True, reps=2
)
two_ansatz.decompose().draw(output="mpl")
```

Posledním ansatzem, o kterém budeme mluvit jménem, je Pauli-two-design. Tento obvod obsahuje počáteční rotaci $RY(pi/4)$ a rotační vrstvy obsahují jednoqubitové Pauliho rotace, kde osa je zvolena rovnoměrně náhodně jako X, Y nebo Z. Vrstvy provázání jsou tvořeny párově hradly CZ s celkovou hloubkou dvě. Všimni si rozdílu v hloubce provázání (a celkového obvodu) mezi tímto pauli_two_design a například efficient_su2.

```python
from qiskit.circuit.library import pauli_two_design

PtwoD_ansatz = pauli_two_design(5, reps=1, seed=10599, insert_barriers=True)
PtwoD_ansatz.decompose().draw(output="mpl")
```

Tyto předpřipravené variační obvody jsou užitečné heuristiky jak z hlediska dosažení požadované úrovně provázání, tak z hlediska omezení hloubky obvodu. Ale není na nich nic magického. Máš svobodu vytvořit si vlastní variační obvod. To může být výhodné v případech, kdy víš něco o provázání cílového stavu svého systému.

Chceš-li si vytvořit vlastní ansatz, jednoduše sestavíš kvantový obvod, kde určitá podmnožina hradel je funkcí prvků vektoru parametrů (v tříqubitovém příkladu níže „theta").

```python
from qiskit import QuantumCircuit
from qiskit.circuit import ParameterVector

n = 3

theta = ParameterVector("θ", length=n)
qc = QuantumCircuit(n)
qc.h(0)
qc.h(2)
for i in range(n - 1):
    qc.cx(i, i + 1)
qc.cz(0, n - 1)
qc.barrier()
for i in range(n):
    qc.ry(theta[i], i)
qc.barrier()
qc.cz(0, n - 1)
for i in reversed(range(n - 1)):
    qc.cx(i, i + 1)
qc.h(0)
qc.h(1)
own_ansatz = qc
print(own_ansatz.depth())
qc.draw("mpl")
```

```text
9
```

Obecně platí, že výběr nejlepšího ansatzu je umění; nejlepší ansatz je ten, který ti pomůže dosáhnout cíle v co nejmenším počtu optimalizačních kroků. Snáze se identifikují ansaetze, které pravděpodobně budou špatné. Například větší hloubka obvodu má tendenci vést k hromadění chyb. Zmírnění chyb s tím může pomoci, ale je dobrou praxí udržovat hloubku obvodu co nejnižší, jak je rozumné. Ale nevynechávej provázání, které je nezbytné. Můžeš mít cílový stav, který vyžaduje schéma plného provázání. Níže jsou uvedeny dva příklady, které jsou pravděpodobně špatnými volbami z celkem zřejmých důvodů. Výběr dobrého ansatzu bude znovu probrán v pozdějších sekcích v kontextu testů konvergence.

Tento první obvod bude pravděpodobně špatnou volbou, protože poslední qubit není vůbec propojen s ostatními. Ve skutečnosti na posledním qubitu neexistuje žádná výpočetně smysluplná akce. S největší pravděpodobností by měl být poslední qubit buď propojen s ostatními, nebo odstraněn z výpočtu.

```python
n = 4

theta = ParameterVector("θ", length=n)
qc = QuantumCircuit(n)
qc.h(0)
qc.h(2)
for i in range(n - 2):
    qc.cx(i, i + 1)
qc.cz(0, n - 2)
qc.barrier()
for i in range(n):
    qc.ry(theta[i], i)
qc.barrier()
qc.cz(0, n - 2)
for i in reversed(range(n - 2)):
    qc.cx(i, i + 1)
qc.h(0)
qc.h(1)
own_ansatz2 = qc
print(own_ansatz2.depth())
qc.draw("mpl")
```

```text
9
```

Tento poslední obvod je pravděpodobně špatnou volbou, protože hloubka hradel je velmi vysoká a čtyřnásobné opakování vrstvy provázání pravděpodobně nepovede k podstatně lepší shodě s cílovým stavem než dvě nebo tři opakování.

```python
su2_ansatz_long = efficient_su2(
    4, su2_gates=["rx", "y", "z"], entanglement="linear", reps=4
)
print(su2_ansatz_long.decompose().depth())
su2_ansatz_long.decompose().draw(output="mpl")
```

```text
24
```

Tyto obvody nejsou „úplné" v tom smyslu, že do mnoha hradel je stále nutné vložit neznámé a proměnné parametry. Tyto parametry jsou voleny postupnými odhady a aktualizací parametrů za účelem snížení střední hodnoty účelové funkce (v chemickém kontextu typicky energie základního stavu). V jedné nebo dokonce několika dimenzích je to triviální. Ale výše uvedený obvod má 20 variačních parametrů, což znamená, že nalezení cílového stavu s minimální energií vyžaduje prohledávání 20-dimenzionálního prostoru stavů (další důvod nezahrnovat zbytečná hradla do obvodu). Zde přicházejí ke slovu klasické optimalizační algoritmy, a to je téma další lekce.

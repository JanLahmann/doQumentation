### 2.1 Úroveň optimalizace {#optimization-level}
K dispozici jsou 4 úrovně `optimization_level` v rozsahu 0–3. Čím vyšší je úroveň optimalizace, tím více výpočetního úsilí je vynaloženo na optimalizaci Circuit. Úroveň 0 neprovádí žádnou optimalizaci a pouze vykoná minimální množství práce potřebné ke spuštění Circuit na zvoleném Backend. Úroveň 3 vynakládá největší úsilí (a zpravidla i čas) na pokus o optimalizaci Circuit. Úroveň 1 je výchozí úrovní optimalizace.
Circuit transpilujeme bez optimalizace (`optimization_level=0`) a s optimalizací (`optimization_level=2`).
Uvidíme velký rozdíl v délce transpilovaných Circuit.

```python
pm0 = generate_preset_pass_manager(
    optimization_level=0, backend=backend, seed_transpiler=777
)
pm2 = generate_preset_pass_manager(
    optimization_level=2, backend=backend, seed_transpiler=777
)
circ0 = pm0.run(ghz_circ)
circ2 = pm2.run(ghz_circ)
print("optimization_level=0:")
display(circ0.draw("mpl", idle_wires=False, fold=-1))
print("optimization_level=2:")
display(circ2.draw("mpl", idle_wires=False, fold=-1))
```

Nová syntéza produkuje mělčí Circuit. Proč?

Je to proto, že nový Circuit lze rozložit na lineárně propojené Qubit, a tedy i na coupling grafu typu heavy-hexagon procesoru IBM&reg; Brisbane, zatímco původní Circuit vyžaduje hvězdicové propojení (uzel stupně 4), a proto jej na coupling grafu heavy-hex nelze rozložit, neboť uzly mají nejvýše stupeň 3. V důsledku toho původní Circuit vyžaduje směrování Qubit, které přidává Gate SWAP a zvyšuje celkový počet Gate.

To, co jsme provedli v novém Circuit, lze chápat jako ruční syntézu Circuit „s ohledem na omezení propojení". Jinými slovy: ručně jsme vyřešili syntézu Circuit a mapování Circuit zároveň.

Obecně platí, že syntéza Circuit závisí na konkrétní aplikaci a je příliš obtížné, aby software pokryl všechny možné aplikace. Qiskit Transpiler nemá funkce pro syntézu Circuit pro přípravu GHZ stavu. V takovém případě stojí za zvážení ruční syntéza Circuit, jak je ukázáno výše.
V této části se podrobněji podíváme na to, jak Qiskit Transpiler funguje, na následujícím jednoduchém příkladovém Circuit.

```python
# Build a toy example circuit
from math import pi
import itertools
from qiskit.circuit import QuantumCircuit
from qiskit.circuit.library import excitation_preserving

circuit = QuantumCircuit(4, name="Example circuit")
circuit.append(excitation_preserving(4, reps=1, flatten=True), range(4))
circuit.measure_all()

value_cycle = itertools.cycle([0, pi / 4, pi / 2, 3 * pi / 4, pi, 2 * pi])
circuit.assign_parameters(
    [x[1] for x in zip(range(len(circuit.parameters)), value_cycle)], inplace=True
)
circuit.draw("mpl")
```

```python
from qiskit.transpiler.passes import (
```

```python
from qiskit.transpiler.preset_passmanagers import generate_preset_pass_manager
from qiskit_ibm_runtime import QiskitRuntimeService, Sampler
```

```python
service = QiskitRuntimeService()
```

```python
# backend = service.backend('ibm_brisbane')
backend = service.least_busy(
    operational=True, simulator=False, min_num_qubits=127
)  # Eagle
backend
```

Nejprve použijeme Circuit GHZ naivně syntetizovaný následujícím způsobem.

```python
num_qubits = 5

ghz_circ = QuantumCircuit(num_qubits)
ghz_circ.h(0)
[ghz_circ.cx(0, i) for i in range(1, num_qubits)]
ghz_circ.measure_all()
ghz_circ.draw("mpl")
```

Circuit transpilujeme bez optimalizace (`optimization_level=0`) a s optimalizací (`optimization_level=2`).
Jak vidíš, délka transpilovaných Circuit se výrazně liší.

```python
pm0 = generate_preset_pass_manager(
    optimization_level=0, backend=backend, seed_transpiler=777
)
pm2 = generate_preset_pass_manager(
    optimization_level=2, backend=backend, seed_transpiler=777
)
circ0 = pm0.run(ghz_circ)
circ2 = pm2.run(ghz_circ)
print("optimization_level=0:")
display(circ0.draw("mpl", idle_wires=False, fold=-1))
print("optimization_level=2:")
display(circ2.draw("mpl", idle_wires=False, fold=-1))
```

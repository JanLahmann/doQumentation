## Open-source obsah IBM Quantum {#ibm-quantums-open-source-content}

IBM poskytuje bohaté výukové materiály o kvantovém výpočetnictví — vše open-source pod licencí CC BY-SA 4.0:

- **[Learning](https://quantum.cloud.ibm.com/learning)** — Strukturované kurzy od základů kvantové mechaniky po pokročilá témata
- **Develop locally** — [install Qiskit](https://docs.quantum.ibm.com/guides/install-qiskit) on your own laptop for serious development work.
- **[Tutorials](https://quantum.cloud.ibm.com/docs/en/tutorials)** — Přes 40 tutoriálů o transpilaci, zmírnění chyb a dalším
- **[Documentation](https://quantum.cloud.ibm.com/docs/en/guides)** — Průvodci a reference API pro Qiskit
- **[Source repo](https://github.com/Qiskit/documentation)** — Veškerý obsah na GitHubu

Jejich [Quantum Platform](https://quantum.cloud.ibm.com) je vždy aktuální a dobře navržená — nejlepší místo pro čtení, učení a vyhledávání informací.

## Co tento projekt přidává {#what-this-project-adds}

[Dokumentace Qiskit](https://github.com/Qiskit/documentation) od IBM je open-source (CC BY-SA 4.0), ale jejich webová aplikace není. doQumentation přidává open-source frontend s živým spouštěním kódu, automatickým vkládáním přihlašovacích údajů a simulátorovým módem.

Nasaditelný kdekoliv — od [GitHub Pages](https://doqumentation.org) přes [Docker](https://github.com/JanLahmann/doQumentation/pkgs/container/doqumentation) až po [RasQberry](https://rasqberry.org/). Viz [všechny funkce](/features).

## Až budeš připravený/á na víc {#when-youre-ready-for-more}

doQumentation ti pomáhá udělat **první kroky s Qiskit** — číst, spouštět kód a objevovat — bez nutnosti cokoliv instalovat. Až budeš pokročilejší, přejdi na oficiální platformu IBM:

- **Spuštění na skutečném hardwaru** — vytvoř si bezplatný [účet IBM Cloud](https://quantum.cloud.ibm.com/registration) a instanci Open Plan, abys mohl/a spouštět obvody na skutečných kvantových počítačích.
- **Nejnovější obsah** — IBM's [Qiskit learning platform](https://quantum.cloud.ibm.com/learning) obsahuje nejnovější kurzy, tutoriály a přídavky (některé ještě nemusí být v doQumentation).
- **Vývoj lokálně** — [nainstaluj Qiskit](https://docs.quantum.ibm.com/guides/install-qiskit) na svůj vlastní laptop pro vážnou vývojovou práci.

*Postaveno na [open-source dokumentaci IBM Quantum](https://github.com/Qiskit/documentation). Díky týmu IBM Quantum za to, že tento obsah volně zpřístupnil pod licencí CC BY-SA 4.0.*

## Spouštění kódu {#code-execution}

Klikni **Run** na libovolném bloku kódu. První kliknutí spustí kernel Jupyter přes [Binder](https://mybinder.org) nebo [IBM Code Engine](/jupyter-settings#code-engine). Poté jsou spuštění okamžitá.

- **Simulátorový mód (bez účtu)** — Zapni [Simulátorový mód](/jupyter-settings#simulator-mode), aby ses mohl spouštět všechny notebooky s AerSimulator nebo FakeBackends s modelem šumu. Žádné nastavení není potřeba.

- **Hardware IBM Quantum** — Zadej svůj API token a CRN v [Nastavení](/jupyter-settings#ibm-quantum) jednou — přihlašovací údaje se automaticky vkládají při každém spuštění. [Vytvoř si bezplatný účet](https://quantum.cloud.ibm.com/registration), pokud ho ještě nemáš.

<details>
<summary>Dostupné backendy pro spouštění</summary>

Každý tutoriál má spustitelné bloky kódu. Klikni **Run** a spusť je pomocí jednoho ze čtyř backendů:

1. **Binder** (výchozí na GitHub Pages) — Zdarma vzdálený kernel Jupyter přes [mybinder.org](https://mybinder.org)
2. **IBM Code Engine** — Bezserverový Jupyter na tvém IBM Cloud účtu. Cold start v sekundách místo minut. [Nastavit](/jupyter-settings#code-engine)
3. **Lokální Jupyter (Docker / RasQberry)** — Připojuje se k lokálnímu serveru Jupyter s předinstalovaným Qiskit
4. **Vlastní server** — Nasměruj na libovolný Jupyter endpoint v [Nastavení](/jupyter-settings#advanced)

  <strong>No IBM Quantum account?</strong> Enable <a href="/jupyter-settings#execution-mode">Simulator Mode</a> in Settings to run all code without signing up.

Když je dostupných více backendů, vyber který použít v [Nastavení](/jupyter-settings#backend-selection).

</details>

<details>
<summary>Možnosti nasazení</summary>

doQumentation je k dispozici jako:

- **[GitHub Pages](https://doqumentation.org)** — Statický web s [Binder](https://mybinder.org) nebo [IBM Code Engine](/jupyter-settings#code-engine) pro vzdálené spouštění kódu
- **[Docker](https://github.com/JanLahmann/doQumentation/pkgs/container/doqumentation)** — Plný stack lokálně, schopný fungovat offline
- **[RasQberry](https://rasqberry.org/)** — Self-hosted na Raspberry Pi s lokálním kernelem Jupyter

</details>

<details>
<summary>Spusť lokálně s Podman / Docker</summary>

Nainstaluj [Podman Desktop](https://podman-desktop.io/) nebo [Docker Desktop](https://www.docker.com/products/docker-desktop/) (Mac / Windows / Linux), pak spusť:

```bash
# Full stack: site + Jupyter + Qiskit (~3 GB)
podman run -p 8080:80 -p 8888:8888 ghcr.io/janlahmann/doqumentation:jupyter
```

Otevři [http://localhost:8080](http://localhost:8080) — spouštění kódu funguje lokálně, žádné čekání na Binder.

Pro odlehčenou verzi bez lokálního spouštění kódu (~60 MB):

```bash
# Static site only — code execution still works via Binder
podman run -p 8080:80 ghcr.io/janlahmann/doqumentation:latest
```

Používáš Docker? Stačí nahradit `podman` za `docker` — příkazy jsou identické. Obrazy jsou multi-arch (`linux/amd64` + `linux/arm64`), takže Apple Silicon Macy fungují nativně.

</details>

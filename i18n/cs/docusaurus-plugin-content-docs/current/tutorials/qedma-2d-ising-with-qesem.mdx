## Pozadí {#background}

Tento tutoriál ukazuje, jak používat [QESEM](/guides/qedma-qesem), Qiskit Function od Qedmy, k simulaci dynamiky kanonického kvantového spinového modelu – 2D Isingova modelu s naklopeným polem (TFI) s non-Cliffordovými úhly:

$$
H = J \sum_{\langle i,j \rangle} Z_i Z_j + g_x \sum_i X_i + g_z \sum_i Z_i ,
$$

kde $ \langle i,j \rangle $ označuje nejbližší sousedy na mřížce. Simulace časového vývoje mnohatělových kvantových systémů je pro klasické počítače výpočetně náročný úkol. Kvantové počítače jsou naopak přirozeně navrženy k efektivnímu provádění tohoto úkolu. TFI model se zejména stal populárním benchmarkem na kvantovém hardwaru díky svému bohatému fyzikálnímu chování a implementaci přátelské k hardwaru.

Místo simulace dynamiky v kontinuálním čase používáme úzce příbuzný model „kicked Ising". Dynamiku lze vyjádřit přesně jako periodický kvantový Circuit, kde každý krok vývoje se skládá ze tří vrstev zlomkových dvoQubitových Gate $ R_{ZZ} (\alpha_{ZZ}) $, proložených vrstvami jednoQubitových Gate $ R_X (\alpha_X) $ a $ R_Z (\alpha_Z) $.

Použijeme obecné úhly, které jsou náročné jak pro klasickou simulaci, tak pro chybovou mitigaci. Konkrétně jsme zvolili $ \alpha_{ZZ} = 1.0 $, $ \alpha_X = 0.53 $ a $ \alpha_Z = 0.1 $, čímž model umísťujeme daleko od jakéhokoli integrabilního bodu.

V tomto tutoriálu provedeme následující:
* Odhadneme očekávanou dobu běhu QPU pro plnou chybovou mitigaci pomocí analytických a empirických funkcí odhadování času QESEM.
* Sestavíme a nasimulujeme Circuit 2D Isingova modelu s naklopeným polem s využitím rozložení Qubitů inspirovaných hardwarem a vrstev Gate.
* Vizualizujeme konektivitu Qubitů zařízení a vybrané podgrafy pro náš experiment.
* Ukážeme použití [zpětné propagace operátorů (OBP)](/guides/qiskit-addons-obp) ke snížení hloubky Circuitu. Tato technika ořezává operace z konce Circuitu za cenu více měření operátorů.
* Provedeme nestrannou chybovou mitigaci (EM) pro více observabilů současně pomocí QESEM a porovnáme ideální, zašuměné a mitigované výsledky.
* Analyzujeme a vykreslíme dopad chybové mitigace na magnetizaci při různých hloubkách Circuitu.

Poznámka: OBP obecně vrátí sadu možných nekomutujících observabilů. QESEM automaticky optimalizuje měřicí báze, když cílové observably obsahují nekomutující členy. Generuje kandidátní sady měřicích bází pomocí několika heuristických algoritmů a vybere sadu, která minimalizuje počet různých bází. To znamená, že QESEM seskupuje kompatibilní observably do společných bází, aby snížil celkový počet potřebných konfigurace měření a zlepšil efektivitu.
## O QESEM {#about-qesem}

QESEM je spolehlivý, vysoce přesný software založený na charakterizaci, implementující efektivní, nestrannou kvaziprabilistickou chybovou mitigaci.
Je navržen pro mitigaci chyb v obecných kvantových Circuit a je aplikačně nezávislý. Byl ověřen na různých hardwarových platformách, včetně experimentů v utility měřítku na zařízeních IBM&reg; Eagle a Heron. Fáze pracovního postupu QESEM jsou následující:

1. Charakterizace zařízení – mapuje věrnosti Gate a identifikuje koherentní chyby, poskytuje kalibrační data v reálném čase. Tato fáze zajišťuje, že mitigace využívá operace s nejvyšší dostupnou věrností.
2. Transpilace zohledňující šum – generuje a vyhodnocuje alternativní mapování Qubitů, sady operací a měřicí báze, vybírá variantu minimalizující odhadovanou dobu běhu QPU s volitelnou paralelizací pro urychlení sběru dat.
3. Potlačení chyb – předefinuje nativní Gate, aplikuje Pauliho twirling a optimalizuje řízení na úrovni pulzů (na podporovaných platformách) pro zlepšení věrnosti.
4. Charakterizace Circuitu – sestaví přizpůsobený lokální chybový model a přizpůsobí ho měřením QPU pro kvantifikaci reziduálního šumu.
5. Chybová mitigace – konstruuje kvaziprabilistické dekompozice více typů a vzorkuje z nich v adaptivním procesu, který minimalizuje dobu QPU pro mitigaci a citlivost na výkyvy hardwaru, dosahuje vysoké přesnosti při velkých objemech Circuitu.

Další informace o QESEM a experimentu v utility měřítku tohoto modelu na 103-Qubitovém podgrafu s vysokou konektivitou nativní geometrie heavy-hex zařízení `ibm_marrakesh` najdeš v [Reliable high-accuracy error mitigation for utility-scale quantum circuits.](https://arxiv.org/abs/2508.10997)

## Krok 1: Mapování klasických vstupů na kvantový problém {#step-1-map-classical-inputs-to-a-quantum-problem}
Začneme definováním funkce, která vytvoří Trotterův Circuit:

```python
def trotter_circuit_from_layers(
    steps: int,
    theta_x: float,
    theta_z: float,
    theta_zz: float,
    layers: Sequence[Sequence[tuple[int, int]]],
    init_state: str | None = None,
) -> qiskit.QuantumCircuit:
    """
    Generates an ising trotter circuit
    :param steps: trotter steps
    :param theta_x: RX angle
    :param theta_z: RZ angle
    :param theta_zz: RZZ angle
    :param layers: list of layers (can be list of layers in device)
    :param init_state: Initial state to prepare. If None, will not prepare any state. If "+", will
     add Hadamard gates to all qubits.
    :return: QuantumCircuit
    """
    qubits = sorted({i for layer in layers for edge in layer for i in edge})
    circ = qiskit.QuantumCircuit(max(qubits) + 1)

    if init_state == "+":
        print("init_state = +")
        for q in qubits:
            circ.h(q)

    for _ in range(steps):
        for q in qubits:
            circ.rx(theta_x, q)
            circ.rz(theta_z, q)

        for layer in layers:
            for edge in layer:
                circ.rzz(theta_zz, *edge)
        circ.barrier(qubits)

    return circ
```

Dále vytvoříme funkci pro výpočet ideálních očekávaných hodnot pomocí `AerSimulator`.

Poznámka: pro velké Circuit (30 a více Qubitů) doporučujeme použít předpočítané hodnoty ze simulací belief-propagation (BP) PEPS. Tento kód zahrnuje předpočítané hodnoty pro 35 Qubitů jako příklad, založené na přístupu BP pro vývoj tenzorové sítě PEPS představeném [v tomto článku](https://www.science.org/doi/10.1126/sciadv.adk4321) (který označujeme jako PEPS-BP), s využitím Python balíčku pro tenzorové sítě [quimb.](https://joss.theoj.org/papers/10.21105/joss.00819)

```python
def calculate_ideal_evs(circ, obs, num_qubits, step):
    # Predefined results for large circuits - calculated using bppeps for 3, 5, 7, 9 trotter steps
    predefined_35 = [
        0.79537,
        0.78653,
        0.79699,
    ]

    if num_qubits == 35:
        print(
            "Using precalculated ideal values for large circuits calculated with belief propagation PEPS. Currently only for 35 qubits."
        )
        return predefined_35[step]

    else:
        simulator = AerSimulator()

        # Use Estimator primitive to get expectation value
        estimator = Estimator(simulator)
        sim_result = estimator.run([(circ, [obs])], precision=0.0001).result()

        # Extracting the result
        ideal_values = sim_result[0].data.evs[0]
        return ideal_values
```

## Krok 2: Optimalizace problému pro spuštění na kvantovém hardwaru {#step-2-optimize-problem-for-quantum-hardware-execution}
### Odhad času QPU s OBP a bez OBP {#qpu-time-estimation-with-and-without-obp}
Uživatelé obvykle chtějí vědět, kolik času QPU jejich experiment vyžaduje.
Jde však o problém, který je pro klasické počítače obtížně řešitelný.

QESEM nabízí dva režimy odhadu času, které tě informují o proveditelnosti tvých experimentů:
1. Analytický odhad času – poskytuje velmi hrubý odhad a nevyžaduje žádný čas QPU. Lze ho využít k testování, zda by transpilační průchod potenciálně snížil čas QPU.
2. Empirický odhad času (ukázaný zde) – poskytuje docela dobrý odhad a spotřebuje několik minut času QPU.

V obou případech QESEM vrátí odhad času potřebného k dosažení požadované přesnosti pro **všechny** pozorovatelné veličiny.

```python
def run_backpropagation(circ_vec, observable, steps_vec, max_qwc_groups=8):
    """
    Runs backpropagation for a list of circuits and observables.
    Returns lists of backpropagated circuits and observables.
    """
    op_budget = OperatorBudget(max_qwc_groups=max_qwc_groups)
    bp_circuit_vec = []
    bp_observable_vec = []

    for i, circ in enumerate(circ_vec):
        slices = slice_by_gate_types(circ)
        bp_observable, remaining_slices, metadata = backpropagate(
            observable,
            slices,
            operator_budget=op_budget,
        )
        bp_circuit = combine_slices(remaining_slices, include_barriers=True)
        bp_circuit_vec.append(bp_circuit)
        bp_observable_vec.append(bp_observable)
        print(f"n.o. steps: {steps_vec[i]}")
        print(f"Backpropagated {metadata.num_backpropagated_slices} slices.")
        print(
            f"New observable has {len(bp_observable.paulis)} terms, which can be combined into "
            f"{len(bp_observable.group_commuting(qubit_wise=True))} groups.\n"
            f"After truncation, the error in our observable is bounded by {metadata.accumulated_error(0):.3e}"
        )
        print("-----------------")
    return bp_circuit_vec, bp_observable_vec
```

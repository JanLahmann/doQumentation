# Často kladené otázky (FAQ) {#frequently-asked-questions-faq}

Nejčastější dotazy týkající se konkrétních režimů spouštění najdeš v průvodci [FAQ k režimům spouštění Qiskit Runtime](/guides/execution-modes-faq).

## Jak mám citovat Qiskit ve svém výzkumu? {#how-do-i-cite-qiskit-in-my-research}

Qiskit cituj pomocí přiloženého [souboru BibTeX](https://github.com/Qiskit/qiskit/blob/main/CITATION.bib).

## Jak mám citovat počítač IBM Quantum ve svém výzkumu? {#how-do-i-cite-an-ibm-quantum-computer-in-my-research}

Ve vědeckých pracích doporučujeme autorům poděkovat IBM Quantum&reg; tímto textem:

>    We acknowledge the use of IBM Quantum services for this work. The views expressed are those of the authors, and do not reflect the official policy or position of IBM or the IBM Quantum team.

Bibliografické reference by měly používat následující citaci:

> IBM Quantum. https://<span>quantum.cloud.ibm.com</span>/, YYYY

Kvantové procesory (QPU) použité v práci by měly být uváděny svým jedinečným názvem (například `ibmq_vigo`) a volitelně i verzí (například `ibmq_vigo` v1.0.2). Doporučujeme rovněž uvádět odkaz na příslušný procesor. Příklad:

> In this paper we used `ibmq_vigo`, which is one of the IBM Quantum Canary processors.

Příklad citace programu IBM Quantum:

> IBM Quantum (2025). Estimator primitive (Version x.y.z) [computer software]. https://<span>quantum.cloud.ibm.com</span>/

## Jak mám citovat IBM Quantum Composer ve svém výzkumu? {#how-do-i-cite-ibm-quantum-composer-in-my-research}

Příklad citace pro IBM Quantum Composer:

> IBM Quantum Composer. 2025. url: https://<span>quantum.cloud.ibm.com/composer</span>

## V uživatelském rozhraní nevidím něco, co hledám, například své úkoly správce. {#i-dont-see-something-in-the-user-interface-such-as-my-administration-tasks}

Zkus v záhlaví rozhraní [IBM Quantum Platform](https://quantum.cloud.ibm.com) přepnout na jiný účet pomocí přepínače účtů.

## Používám pouze Open Plan, přesto jsem musel zadat číslo kreditní karty. Bude mi něco účtováno? {#im-just-using-the-open-plan-but-i-had-to-put-in-my-credit-card-will-i-be-charged}

Nic ti nebude účtováno, pokud si nevytvoříš [instanci](/guides/instances) s modelem Pay-As-You-Go a nezačneš ji používat k odesílání úloh na QPU. Postup nastavení cenového limitu a další způsoby řízení nákladů najdeš v části [Správa nákladů](/guides/manage-cost).

## Jaký je rozdíl mezi API klíčem a API tokenem? {#whats-the-difference-between-an-api-key-and-an-api-token}

Žádný rozdíl neexistuje. Pojmy API klíč a API token se používají zaměnitelně. V dokumentaci se setkáš s oběma, protože různá rozhraní používají různé termíny.

<span id="bearer"></span>
## Jaký je rozdíl mezi API tokenem a bearer tokenem? {#whats-the-difference-between-an-api-token-and-a-bearer-token}

API token je dlouhodobé přihlašovací údaje používané k [ověření u služby Qiskit Runtime](/guides/hello-world#install-and-authenticate). Bearer token je dočasné přihlašovací údaje sloužící k [ověření u REST API v nedůvěryhodném prostředí](/guides/cloud-setup-untrusted#authenticate-to-the-rest-api-in-an-untrusted-environment). Bearer token nelze použít k ověření u služby Qiskit Runtime.

## Musím si vybrat region, ve kterém žiji? {#do-i-have-to-choose-the-region-that-i-live-in}

Ne. Můžeš použít kterýkoli region. Region určuje, kde jsou úlohy spouštěny a kde jsou data uchovávána. Navíc vidíš a můžeš používat pouze instance vytvořené v té instanci, do které jsi přihlášen. Upozorňujeme, že Open Plan podporuje pouze region us-east.

## Mohu se připojit přes proxy server? {#can-i-connect-through-a-proxy-server}

Ano, ale musíš používat Qiskit Runtime ve verzi 0.44.0 nebo novější.

## Jaká je politika uchovávání dat? {#what-is-the-data-retention-policy}

IBM uchovává tvá (nesmazaná) data o úlohách a využití po dobu tří let, dokud nadále používáš službu Qiskit Runtime.

## Co se stalo s Python distribucemi jako Qiskit Aer, Qiskit Nature a Qiskit Experiments? {#what-happened-to-python-distributions-such-as-qiskit-aer-qiskit-nature-and-qiskit-experiments}

Od vydání Qiskit SDK v1.0 příkaz `pip install qiskit` nainstaluje **pouze** [Qiskit SDK](https://github.com/Qiskit/qiskit). To znamená, že některé moduly, které jsi dříve používal, například `qiskit-aer` nebo `qiskit-nature`, je nyní nutné instalovat samostatně.

Před vydáním Qiskit SDK v1.0 se Qiskit skládal z mnoha různých Python distribucí, jako například `qiskit-terra` (jádro překladače), `qiskit-aer` (vysokovýkonný simulátor), původní poskytovatel IBM Quantum&reg; a několik dnes již zastaralých balíčků poskytujících konkrétní průzkumné algoritmické funkce nebo funkce pro spouštění experimentů.

Tyto samostatné softwarové balíčky byly sloučeny do tzv. Qiskit *metabalíčku*, který uživatelům umožňoval nainstalovat všechny tyto komponenty najednou příkazem `pip install qiskit`. Tato architektura metabalíčku byla s vydáním Qiskit SDK v1.0 zrušena a nyní je každou z jednotlivých softwarových distribucí nutné instalovat samostatně.

Pokud chceš nainstalovat balíčky, které byly dříve součástí Qiskit metabalíčku, navštiv [ekosystém Qiskit](https://www.ibm.com/quantum/ecosystem) a najdi tam celou řadu balíčků, které vyhovují tvým potřebám. Více informací o této změně si také můžeš přečíst v [průvodci migrací na v1.0](/guides/metapackage-migration).

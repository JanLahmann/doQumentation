# Konfigurace IBM Quantum Platform pro organizaci {#configure-ibm-quantum-platform-for-an-organization}

Postupuj podle těchto kroků a nastav IBM Quantum Platform.

<span id="iam-org"></span>
## 1. Konfigurace nastavení IAM {#1-configure-iam-settings}

Nejprve nakonfiguruj následující nastavení v účtu IAM administrátora. Nastavení si můžeš prohlédnout a upravit na stránce [Manage → IAM → Settings](https://cloud.ibm.com/iam/settings).

* **Viditelnost seznamu uživatelů** určuje, zda uživatelé vidí navzájem, bez ohledu na přiřazení k projektu. Nastavení `enabled` omezuje viditelnost uživatelů. To znamená, že uživatelé ve tvém účtu se navzájem neuvidí, i když mají přístup ke stejným prostředkům. Zvol hodnotu vhodnou pro tvoje prostředí. Další informace najdeš v článku [Controlling user visibility](https://cloud.ibm.com/docs/account?topic=account-iam-user-setting).
* **Vytváření API klíčů** určuje, zda mohou uživatelé vytvářet API klíče. V IBM Quantum Platform je používání API klíčů běžné. Pokud se API klíče používají, zvol `disabled`. Alternativně můžeš každému uživateli přidělit konkrétní oprávnění.

<span id="crt-rsc-grp-org"></span>
## 2. (Volitelné) Vytvoření skupin prostředků {#2-optional-create-resource-groups}

Tento krok přeskoč, pokud používáš přístupové skupiny, které přímo odkazují na instance služby IBM Quantum.

Pokud jsi se rozhodl/a používat skupiny prostředků, přejdi na [Manage → Account → Resource groups (in Account resources)](https://cloud.ibm.com/account/resource-groups) a klikni na **Create**.

<span id="create-instance-org"></span>
## 3. Vytvoření instancí služby IBM Quantum {#3-create-ibm-quantum-service-instances}

Pokud jsi již instance služby IBM Quantum vytvořil/a, tento krok přeskoč.

* Pokud používáš skupiny prostředků, ujisti se, že instance služby vytváříš ve správné skupině prostředků.
* Název instance služby, například `QR-ml`, je potřeba pro odkazy v přístupových skupinách.

Postupuj podle kroků v části [Set up to use IBM Cloud](/guides/cloud-setup#cloud).

<span id="create-group-org"></span>
## 4. Vytvoření přístupových skupin pro projekty {#4-create-access-groups-for-projects}

Nejprve [vytvoř vlastní roli](/guides/custom-roles#custom), která uživatelům umožní provádět akce potřebné pro práci s instancemi služby IBM Quantum.
Poté [vytvoř přístupovou skupinu](/guides/access-groups) pro každý projekt a přiděl jí minimální sadu oprávnění potřebných pro práci s prostředky projektu.

<span id="firewall"></span>
## 6. Konfigurace firewallu {#6-configure-your-firewall}

Aby tvůj firewall umožnil přístup ke koncovým bodům API IBM Quantum, přidej na seznam povolených adres následující URL:

- Globální služby IBM Cloud:
    ```text
    cloud.ibm.com
    iam.cloud.ibm.com
    api.global-search-tagging.cloud.ibm.com
    globalcatalog.cloud.ibm.com
    resource-controller.cloud.ibm.com
    ```
- Služby IBM Quantum Qiskit Runtime – region us-east:
    ```text
    quantum.cloud.ibm.com
    s3.us-east.cloud-object-storage.appdomain.cloud
    ```
- Služby IBM Quantum Qiskit Runtime – region eu-de:
    ```text
    eu-de.quantum.cloud.ibm.com
    s3.direct.eu-de.cloud-object-storage.appdomain.cloud
    ```
- Náhledová služba IBM Quantum Qiskit Functions:
    ```text
    qiskit-serverless.quantum.ibm.com
    ```

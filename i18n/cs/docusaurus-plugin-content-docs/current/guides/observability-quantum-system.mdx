# Události sledování aktivity pro službu IBM Quantum System {#activity-tracking-events-for-the-ibm-quantum-system-service}

Služby IBM Cloud&reg;, jako je služba IBM&reg; Quantum System, generují události sledování aktivity.

Události sledování aktivity hlásí aktivity, které mění stav služby v IBM Cloud. Události můžeš použít k prošetření neobvyklé aktivity a kritických akcí a ke splnění požadavků na regulační audit.

Pomocí směrování událostí IBM Cloud Activity Tracker Event Routing, platformní služby, můžeš směrovat auditovací události ve svém účtu do cílů dle vlastního výběru konfigurací cílů a tras, které definují, kam se události sledování aktivity odesílají. Další informace najdeš v části [About IBM Cloud Activity Tracker Events Routing](https://cloud.ibm.com/docs/atracker?topic=atracker-about).

Pomocí IBM Cloud Logs můžeš vizualizovat a upozorňovat na události, které jsou generovány ve tvém účtu a směrovány přes IBM Cloud Activity Tracker Event Routing do instance IBM Cloud Logs.

## Zobrazení událostí sledování aktivity pro službu IBM Quantum System {#viewing-activity-tracking-events-for-the-ibm-quantum-system-service}

Pomocí IBM Cloud Logs můžeš vizualizovat a upozorňovat na události, které jsou generovány ve tvém účtu a směrovány přes IBM Cloud Activity Tracker Event Routing do instance IBM Cloud Logs.

### Spuštění IBM Cloud Logs ze stránky Observability {#launching-ibm-cloud-logs-from-the-observability-page}

Informace o spuštění uživatelského rozhraní IBM Cloud Logs najdeš v části [Launching the UI](https://cloud.ibm.com/docs/cloud-logs?topic=cloud-logs-instance-launch#instance-launch-cloud-ui) v dokumentaci IBM Cloud Logs.

| Akce | Spuštěno prostřednictvím |
| -- | -- |
| `quantum-computing.direct-access-backend-configuration.read` | Získat konfiguraci backendu (`GET /v1/backends/{backend-name}/configuration`) |
| `quantum-computing.direct-access-backend-properties.read` | Získat vlastnosti backendu (`GET /v1/backends/{backend-name}/properties`) |
| `quantum-computing.direct-access-backend.list` | Získat seznam backendů (`GET /v1/backends`) |
| `quantum-computing.direct-access-backend.read` | Získat podrobnosti backendu (`GET /v1/backends/{backend-name}`) |
| `quantum-computing.direct-access-job.cancel` | Zrušit úlohu (`POST /v1/jobs/{job-id}/cancel`) |
| `quantum-computing.direct-access-job.create` | Spustit úlohu (`POST /v1/jobs`) |
| `quantum-computing.direct-access-job.delete` | Smazat úlohu (`DELETE /v1/jobs/{job-id}`) |
| `quantum-computing.direct-access-job.list` | Získat úlohy (`GET /v1/jobs`) |
| `quantum-computing.direct-access-job.read` | Získat úlohu podle ID (`GET /v1/jobs/{job-id}`) |

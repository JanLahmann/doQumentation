Tato příručka demonstruje funkční příklad přeříznutí drátů s balíčkem `qiskit-addon-cutting`. Popisuje rekonstrukci očekávaných hodnot sedmiqubitového Circuit pomocí přeříznutí drátů.

Přeříznutí drátu je v tomto balíčku reprezentováno jako dvoiqubitová instrukce [`Move`](https://docs.quantum.ibm.com/api/qiskit-addon-cutting/instructions-move), která je definována jako reset druhého Qubit, na který instrukce působí, následovaný záměnou obou Qubitů. Tato operace je ekvivalentní přenesení stavu prvního Qubit na druhý Qubit, přičemž se zároveň zahodí příchozí stav druhého Qubit.

Balíček je navržen tak, aby byl konzistentní se způsobem, jakým musíš zacházet s přeříznutími drátů při práci s fyzickými Qubity. Například přeříznutí drátu může vzít stav fyzického Qubit $n$ a pokračovat s ním jako fyzický Qubit $m$ po přeříznutí. Můžeš si „řezání instrukcí" představit jako sjednocený rámec pro uvažování o přeříznutích drátů i Gate v rámci stejného formalismu (protože přeříznutí drátu je jen přeříznutá instrukce [`Move`](https://docs.quantum.ibm.com/api/qiskit-addon-cutting/instructions-move)). Použití tohoto rámce pro přeříznutí drátů také umožňuje opakované využití Qubitů, což je vysvětleno v části o [ručním přeříznutí drátů](#cut-wires-using-the-low-level-move-instruction).

Jednoqubitová instrukce [`CutWire`](https://docs.quantum.ibm.com/api/qiskit-addon-cutting/instructions-cut-wire) slouží jako abstraktovanější a jednodušší rozhraní pro práci s přeříznutími drátů. Umožňuje ti označit na vysoké úrovni, kde v Circuit má být drát přeříznut, a doplněk pro řezání obvodů za tebe vloží příslušné instrukce [`Move`](https://docs.quantum.ibm.com/api/qiskit-addon-cutting/instructions-move).

Následující příklad demonstruje rekonstrukci očekávané hodnoty po přeříznutí drátů. Vytvoříš Circuit s několika nelokálními Gate a definuješ observovatelné veličiny k odhadu.

```python
import numpy as np
from qiskit import QuantumCircuit
from qiskit.transpiler import generate_preset_pass_manager
from qiskit.quantum_info import SparsePauliOp
from qiskit_ibm_runtime.fake_provider import FakeManilaV2
from qiskit_ibm_runtime import SamplerV2, Batch
from qiskit_aer.primitives import EstimatorV2

from qiskit_addon_cutting.instructions import Move, CutWire
from qiskit_addon_cutting import (
    partition_problem,
    generate_cutting_experiments,
    cut_wires,
    expand_observables,
    reconstruct_expectation_values,
)

qc_0 = QuantumCircuit(7)
for i in range(7):
    qc_0.rx(np.pi / 4, i)
qc_0.cx(0, 3)
qc_0.cx(1, 3)
qc_0.cx(2, 3)
qc_0.cx(3, 4)
qc_0.cx(3, 5)
qc_0.cx(3, 6)
qc_0.cx(0, 3)
qc_0.cx(1, 3)
qc_0.cx(2, 3)

# Define observable
observable = SparsePauliOp(["ZIIIIII", "IIIZIII", "IIIIIIZ"])

# Draw circuit
qc_0.draw("mpl")
```

## Přeříznutí drátů pomocí vysokoúrovňové instrukce `CutWire` {#cut-wires-using-the-high-level-cutwire-instruction}

Dále proveď přeříznutí drátů pomocí jednoqubitové instrukce [`CutWire`](https://docs.quantum.ibm.com/api/qiskit-addon-cutting/instructions-cut-wire) na Qubit $q_3$. Jakmile jsou subexperimenty připraveny ke spuštění, použij funkci [`cut_wires()`](https://docs.quantum.ibm.com/api/qiskit-addon-cutting/qiskit-addon-cutting#cut_wires) k transformaci instrukcí `CutWire` na instrukce [`Move`](https://docs.quantum.ibm.com/api/qiskit-addon-cutting/instructions-move) na nově alokovaných Qubitech.

```python
qc_1 = QuantumCircuit(7)
for i in range(7):
    qc_1.rx(np.pi / 4, i)
qc_1.cx(0, 3)
qc_1.cx(1, 3)
qc_1.cx(2, 3)
qc_1.append(CutWire(), [3])
qc_1.cx(3, 4)
qc_1.cx(3, 5)
qc_1.cx(3, 6)
qc_1.append(CutWire(), [3])
qc_1.cx(0, 3)
qc_1.cx(1, 3)
qc_1.cx(2, 3)

qc_1.draw("mpl")
```

<Admonition type="info" title="Poznámka k rozšiřování observovatelných veličin">
    Když je Circuit rozšířen přes jedno nebo více přeříznutí drátů, je potřeba observovatelnou veličinu aktualizovat, aby zohledňovala přidané Qubity. Balíček `qiskit-addon-cutting` obsahuje pomocnou funkci [`expand_observables()`](https://docs.quantum.ibm.com/api/qiskit-addon-cutting/qiskit-addon-cutting#expand_observables), která přijímá objekty `PauliList` a původní a rozšířené Circuit jako argumenty a vrací nový `PauliList`.

    Vrácený `PauliList` nebude obsahovat žádné informace o koeficientech původní observovatelné veličiny, ale ty lze ignorovat až do rekonstrukce výsledné očekávané hodnoty.
</Admonition>

```python
# Transform CutWire instructions to Move instructions
qc_2 = cut_wires(qc_1)

# Expand the observable to match the new circuit size
expanded_observable = expand_observables(observable.paulis, qc_0, qc_2)
print(f"Expanded Observable: {expanded_observable}")
qc_2.draw("mpl")
```

### Rozdělení Circuit a observovatelné veličiny {#partition-the-circuit-and-observable}

Nyní lze problém rozdělit do oddílů. Toho se dosáhne pomocí funkce [`partition_problem()`](https://docs.quantum.ibm.com/api/qiskit-addon-cutting/qiskit-addon-cutting#partition_problem) s volitelnou sadou popisků oddílů, které určují způsob rozdělení Circuit. Qubity sdílející společný popisek oddílu jsou seskupeny dohromady a veškeré nelokální Gate přesahující více než jeden oddíl jsou přeříznuty.

Pokud nejsou zadány žádné popisky oddílů, bude rozdělení automaticky určeno na základě konektivity Circuit. Přečti si další část o [ručním přeříznutí drátů](#cut-wires-using-the-low-level-move-instruction) pro více informací o zadávání popisků oddílů.

```python
partitioned_problem = partition_problem(
    circuit=qc_2,
    observables=expanded_observable,
)
subcircuits = partitioned_problem.subcircuits
subobservables = partitioned_problem.subobservables
bases = partitioned_problem.bases

print(f"Subobservables to measure: \n{subobservables}\n")
print(f"Sampling overhead: {np.prod([basis.overhead for basis in bases])}")
subcircuits[0].draw("mpl")
```


### Generování subexperimentů ke spuštění a následné zpracování výsledků {#generate-subexperiments-to-execute-and-post-process-results}

Pro odhad očekávané hodnoty Circuit v plné velikosti je z kvaziprávděpodobnostního rozdělení rozložených Gate vygenerováno několik subexperimentů, které jsou poté spuštěny na jednom nebo více QPU. Metoda [`generate_cutting_experiments`](https://docs.quantum.ibm.com/api/qiskit-addon-cutting/qiskit-addon-cutting#generate_cutting_experiments) to provede přijetím argumentů pro slovníky `subcircuits` a `subobservables`, které jsi vytvořil výše, a také pro počet vzorků odebraných z distribuce.

<Admonition type="note" title="Poznámka k počtu vzorků">
    Argument `num_samples` určuje, kolik vzorků se odebere z kvaziprávděpodobnostního rozdělení, a ovlivňuje přesnost koeficientů používaných pro rekonstrukci. Předání nekonečna (`np.inf`) zajistí, že všechny koeficienty budou vypočítány přesně. Přečti si dokumentaci API o [generování vah](https://docs.quantum.ibm.com/api/qiskit-addon-cutting/qpd#generate_qpd_weights) a [generování řezacích experimentů](https://docs.quantum.ibm.com/api/qiskit-addon-cutting/qiskit-addon-cutting#generate_cutting_experiments) pro více informací.
</Admonition>

```python
# Generate subexperiments
subexperiments, coefficients = generate_cutting_experiments(
    circuits=subcircuits, observables=subobservables, num_samples=np.inf
)

# Set a backend to use and transpile the subexperiments
backend = FakeManilaV2()
pass_manager = generate_preset_pass_manager(
    optimization_level=1, backend=backend
)
isa_subexperiments = {
    label: pass_manager.run(partition_subexpts)
    for label, partition_subexpts in subexperiments.items()
}

# Submit each partition's subexperiments to the Qiskit Runtime Sampler
# primitive, in a single batch so that the jobs will run back-to-back.
with Batch(backend=backend) as batch:
    sampler = SamplerV2(mode=batch)
    jobs = {
        label: sampler.run(subsystem_subexpts, shots=2**12)
        for label, subsystem_subexpts in isa_subexperiments.items()
    }
    # Retrieve results
    results = {label: job.result() for label, job in jobs.items()}
```

Nakonec lze očekávanou hodnotu celého Circuit rekonstruovat pomocí metody [`reconstruct_expectation_values()`](https://docs.quantum.ibm.com/api/qiskit-addon-cutting/qiskit-addon-cutting#reconstruct_expectation_values).

Následující blok kódu rekonstruuje výsledky a porovnává je s přesnou očekávanou hodnotou.

```python
reconstructed_expval_terms = reconstruct_expectation_values(
    results,
    coefficients,
    subobservables,
)
# Apply the coefficients of the original observable
reconstructed_expval = np.dot(reconstructed_expval_terms, observable.coeffs)

# Compute the exact expectation value using the `qiskit_aer` package.
estimator = EstimatorV2()
exact_expval = estimator.run([(qc_0, observable)]).result()[0].data.evs
print(
    f"Reconstructed expectation value: {np.real(np.round(reconstructed_expval, 8))}"
)
print(f"Exact expectation value: {np.round(exact_expval, 8)}")
print(
    f"Error in estimation: {np.real(np.round(reconstructed_expval-exact_expval, 8))}"
)
print(
    f"Relative error in estimation: {np.real(np.round((reconstructed_expval-exact_expval) / exact_expval, 8))}"
)
```

```text
Reconstructed expectation value: 1.45965266
Exact expectation value: 1.59099026
Error in estimation: -0.1313376
Relative error in estimation: -0.08255085
```

<Admonition type="caution" title="Poznámka ke koeficientům observovatelné veličiny">
    Pro přesnou rekonstrukci očekávané hodnoty musí být na výstup rekonstrukce aplikovány koeficienty původní observovatelné veličiny (které se liší od výstupu `generate_cutting_experiments()`), protože tyto informace byly ztraceny při generování řezacích experimentů nebo při rozšíření observovatelné veličiny.

    Tyto koeficienty lze obvykle aplikovat pomocí `numpy.dot()`, jak bylo ukázáno výše.
</Admonition>
## Přeříznutí drátů pomocí nízkoúrovňové instrukce `Move` {#cut-wires-using-the-low-level-move-instruction}

Jedním omezením použití vysokoúrovňové instrukce `CutWire` je, že neumožňuje opakované využití Qubitů. Pokud je to pro řezací experiment žádoucí, můžeš místo toho ručně umístit instrukce [`Move`](https://docs.quantum.ibm.com/api/qiskit-addon-cutting/instructions-move). Protože však instrukce `Move` zahodí stav cílového Qubit, je důležité, aby tento Qubit nesdílel žádné provázání se zbytkem systému. V opačném případě způsobí operace resetování částečný kolaps stavu Circuit po přeříznutí drátu.

Následující blok kódu provede přeříznutí drátu na Qubit $q_3$ pro stejný příklad Circuit jako výše. Rozdíl spočívá v tom, že Qubit lze znovu použít obrácením operace `Move` tam, kde bylo provedeno druhé přeříznutí drátu (to však není vždy možné a závisí na přeřezávaném Circuit).

```python
qc_1 = QuantumCircuit(8)
for i in [*range(4), *range(5, 8)]:
    qc_1.rx(np.pi / 4, i)
qc_1.cx(0, 3)
qc_1.cx(1, 3)
qc_1.cx(2, 3)
qc_1.append(Move(), [3, 4])
qc_1.cx(4, 5)
qc_1.cx(4, 6)
qc_1.cx(4, 7)
qc_1.append(Move(), [4, 3])
qc_1.cx(0, 3)
qc_1.cx(1, 3)
qc_1.cx(2, 3)

# Expand observable
observable_expanded = SparsePauliOp(["ZIIIIIII", "IIIIZIII", "IIIIIIIZ"])
qc_1.draw("mpl")
```

Výše uvedený Circuit lze nyní rozdělit do oddílů a vygenerovat řezací experimenty. Chceš-li explicitně specifikovat způsob rozdělení Circuit, můžeš do funkce `partition_problem()` přidat popisky oddílů. Qubity sdílející společný popisek oddílu jsou seskupeny dohromady a veškeré nelokální Gate přesahující více než jeden oddíl jsou přeříznuty. Klíče slovníku vráceného funkcí `partition_problem()` budou odpovídat těm, které jsou uvedeny v řetězci popisku.

```python
partitioned_problem = partition_problem(
    circuit=qc_1,
    partition_labels="AAAABBBB",
    observables=observable_expanded.paulis,
)
subcircuits = partitioned_problem.subcircuits
subobservables = partitioned_problem.subobservables
bases = partitioned_problem.bases

print(f"Subobservables to measure: \n{subobservables}\n")
print(f"Sampling overhead: {np.prod([basis.overhead for basis in bases])}")
```

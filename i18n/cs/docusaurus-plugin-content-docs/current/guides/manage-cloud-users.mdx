# Správa uživatelů IBM Cloud {#manage-ibm-cloud-users}

Tento průvodce ukazuje, jak používat IBM Cloud&reg; k přidávání uživatelů s účty IBM Cloud, a obsahuje pokyny pro uživatele, jak přistupovat k prostředí.

Pokud chceš místo toho spravovat uživatele přes poskytovatele identit, postupuj podle pokynů v tématu [Správa poskytovatelů identit](/guides/manage-appid).

## Kroky {#kroky}

Postupuj podle těchto kroků ke správě uživatelů IBM Cloud:

1. Ujisti se, že uživatelé, které chceš pozvat, mají účty IBM Cloud.
2. Přejdi do IAM a [pozvi uživatele](https://cloud.ibm.com/iam/users/invite_users)
   <Admonition type="note">
   Uživatelé nemohou být spravováni, dokud nepřijmou pozvánku a alespoň jednou se nepřihlásí.
   </Admonition>
3. Volitelné: Uprav přiřazení projektů a přístupové skupiny uživatelů v [IAM.](https://cloud.ibm.com/iam/users)

<span id="user-org"></span>
## Postup uživatele {#user-flow}

1. Uživatel přijme pozvánku a přihlásí se přes [portál IBM Cloud](https://cloud.ibm.com/).
2. Uživatel vytvoří API klíč v sekci ([Správa → Přístup (IAM) → API klíče](https://cloud.ibm.com/iam/apikeys)).
3. Další informace najdou uživatelé v průvodci [Instalace Qiskit.](/guides/install-qiskit)

<span id="steps-cloud-org"></span>
## Příklad použití {#example-scenario}

Tento příklad vytváří následující konfiguraci:

* Existují dva projekty: `ml` a `finance`.
   * Projekt `ml` by měl mít přístup k instancím služby `QR-ml` a `QR-common`.
   * Projekt `finance` by měl mít přístup k instancím služby `QR-finance` a `QR-common`.
* Existují tři uživatelé:
   * Fatima by měla mít přístup k projektu `ml`.
   * Ravi by měl mít přístup k projektu `finance`.
   * Amyra by měla mít přístup k oběma projektům.
* Konfigurace používá přístupové skupiny bez skupin prostředků.
* Uživatelé jsou definováni v IBM Cloud a přiřazení projektů probíhá tamtéž.
* Uživatelé by měli mít možnost mazat úlohy.

Kroky k implementaci této konfigurace jsou:

2. Správce cloudu vytvoří tři instance služby: `QR-ml`, `QR finance` a `QR-common`.
3. Správce cloudu vytvoří vlastní pravidlo zahrnující akci `quantum-computing.job.delete`.
1. Správce cloudu vytvoří dvě přístupové skupiny:
   * Přístupová skupina `ml` může přistupovat k `QR-ml` a `QR-common`.
   * Přístupová skupina `finance` může přistupovat k `QR-finance` a `QR-common`.
1. Správce cloudu pozve uživatele cloudu do příslušného projektu. Konkrétně je pozve a přiřadí do přístupové skupiny, která zahrnuje daný projekt.
   * Fatima je přidána do přístupové skupiny „ml".
   * Ravi je přidán do přístupové skupiny „finance".
   * Amyra je přidána do přístupových skupin „ml" i „finance".
1. Uživatelé se mohou přihlásit přes portál IBM Cloud, vytvořit API klíče a pracovat s instancemi služeb svých projektů.

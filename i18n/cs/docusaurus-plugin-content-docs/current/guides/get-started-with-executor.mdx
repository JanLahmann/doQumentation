Podobně jako [Sampler](/guides/get-started-with-sampler) primitiv, Executor vzorkuje výstupní registry ze spuštění kvantových obvodů, ale nemá žádné vestavěné potlačování ani zmírňování chyb. Místo toho je součástí [modelu řízeného spouštění](/guides/directed-execution-model), který poskytuje ingredience pro zachycení záměrů návrhu na straně klienta a přesouvá nákladné generování variant obvodů na stranu serveru. Executor sleduje direktivy uvedené v anotacích obvodů a možnostech, generuje a váže hodnoty parametrů, spouští vázané obvody na hardwaru a vrací výsledky spuštění a metadata. Neprovádí žádná implicitní rozhodnutí za tebe a dává ti plnou kontrolu a transparentnost.

## Kroky k použití primitivu Executor {#steps-to-use-the-executor-primitive}

### 1. Inicializace účtu {#1-initialize-the-account}

Protože Qiskit Runtime je spravovaná služba, musíš nejprve inicializovat svůj účet. Poté můžeš vybrat QPU, které chceš použít k výpočtu očekávané hodnoty.

Pokud ještě nemáš účet, postupuj podle kroků v [Nastavení účtu IBM Cloud&reg;](/guides/cloud-setup).

```python
from qiskit_ibm_runtime import QiskitRuntimeService, Executor
from qiskit_ibm_runtime.quantum_program import QuantumProgram
from qiskit.circuit import QuantumCircuit
from qiskit.transpiler import generate_preset_pass_manager
from samplomatic.transpiler import generate_boxing_pass_manager
from samplomatic import build

# Initialize the service and choose a backend
service = QiskitRuntimeService()
backend = service.least_busy(operational=True, simulator=False)
```

```python
print(backend)
```


### 2. Vytvoření a transpilace obvodu {#2-create-and-transpile-a-circuit}

K použití primitivu Executor potřebuješ alespoň jeden obvod. Volitelně může mít parametry.

```python
# Generate the circuit
circuit = QuantumCircuit(2)
circuit.h(0)
circuit.h(1)
circuit.cz(0, 1)
circuit.h(1)

# Using `measure_all` automatically creates the necessary
# classical registers.
circuit.measure_all()
```

Circuit musí být transformován tak, aby používal pouze instrukce podporované QPU (označované jako obvody *instrukční sady architektury (ISA)*). K tomu použij Transpiler.

```python
# Transpile the circuit
preset_pass_manager = generate_preset_pass_manager(
    backend=backend, optimization_level=0
)
isa_circuit = preset_pass_manager.run(circuit)
```

### 3. Inicializace `QuantumProgram` {#3-initialize-a-quantumprogram}

Inicializuj `QuantumProgram` se svou pracovní zátěží. `QuantumProgram` se skládá z `QuantumProgramItems`. Každá položka se obvykle skládá z obvodu, sady hodnot parametrů a případně `samplex` pro randomizaci obsahu obvodu. Podrobnosti najdeš v části [Vstupy a výstupy Executor](/guides/executor-input-output).

Následující buňka inicializuje `QuantumProgram` a specifikuje provedení 25 snímků. Dále přidá transpilovaný cílový Circuit.

```python
# Initialize an empty program
program = QuantumProgram(shots=25)

# Append the circuit to the program
program.append_circuit_item(isa_circuit)
```

### 4. Volitelně: Seskupení Gate a měření do anotovaných boxů {#4-optional-group-gates-and-measurements-into-annotated-boxes}

Seskupení instrukcí do boxů a jejich anotování je primárním způsobem, jak specifikovat svůj záměr. V následujícím příkladu používáme `generate_boxing_pass_manager` a jeho parametry twirling pro seskupení dvouQubitových Gate a měření do boxů a aplikaci anotace twirling.

```python
# Generate a boxing pass manager to group gates
# and measurements into boxes and add
# a`Twirl` annotation.
boxes_pm = generate_boxing_pass_manager(
    # Add gate twirling
    enable_gates=True,
    # Add measurement twirling
    enable_measures=True,
)

boxed_circuit = boxes_pm.run(isa_circuit)
boxed_circuit.draw("mpl", idle_wires=False)
```

### 5. Volitelně: Vytvoření šablony obvodu a samplex a jejich přidání do programu {#5-optional-build-a-template-circuit-and-samplex-and-add-them-to-the-program}

Dále použij metodu Samplomatic [build](https://qiskit.github.io/samplomatic/api/auto/samplomatic.build.html#samplomatic.build) k vygenerování páru _šablonového obvodu_ a _samplex_. Šablonový obvod je strukturálně ekvivalentní původnímu obvodu. Jednoqubitové Gate jsou však nahrazeny parametrizovanými Gate za účelem implementace předepsaných anotací (twirling Gate a měření v tomto příkladu). Samplex kóduje veškeré informace potřebné ke generování randomizovaných parametrů pro šablonový obvod.

Po vygenerování páru šablonového obvodu a samplex použij metodu `append_samplex_item` k přidání páru do programu.

Podrobnosti o `samplomatic.samplex.Samplex` a jeho argumentech najdeš v dokumentaci [API](https://qiskit.github.io/samplomatic/api/index.html) Samplomatic.

```python
# Build the template circuit and the samplex
template_circuit, samplex = build(boxed_circuit)

# Append the template circuit and samplex as a `samplex_item`
program.append_samplex_item(
    template_circuit,
    samplex=samplex,
    shape=(num_randomizations := 20,),
)
```

### 6. Spuštění Executor a získání výsledků {#6-invoke-executor-and-get-results}

Spusť `QuantumProgram` na backendu IBM&reg; pomocí primitivu `Executor` s výchozími možnostmi. Informace o dostupných možnostech najdeš v části [Možnosti Executor](/guides/executor-options).

```python
# Initialize an Executor with the default options
executor = Executor(mode=backend)

# Submit the job
job = executor.run(program)
job
```

# Qiskit Code Assistant

Qiskit Code Assistant si klade za cíl zpřístupnit kvantové výpočty novým uživatelům Qiskie a zlepšit zážitek z programování pro stávající uživatele. Jde o generativního AI asistenta pro psaní kódu, poháněného platformou [watsonx](https://www.ibm.com/products/watsonx-ai). Je natrénován na milionech textových tokenů z Qiskit SDK, letech příkladů Qiskit kódu a funkcích IBM Quantum&reg;. Qiskit Code Assistant ti může pomoci ve vývojovém procesu kvantových aplikací tím, že nabízí návrhy generované LLM na základě [IBM Granite modelů](https://www.ibm.com/products/watsonx-ai/foundation-models), které zahrnují nejnovější funkce a možnosti od IBM&reg;.

<Admonition type="note" title="Poznámky">
- Toto je experimentální funkce dostupná uživatelům IBM Quantum Premium Plan zaregistrovaným na nové [IBM Quantum Platform](https://quantum.cloud.ibm.com/).
- Qiskit Code Assistant je ve stavu předběžného vydání a může se měnit.
- Pokud máš zpětnou vazbu nebo chceš kontaktovat tým vývojářů, použij [kanál Qiskit Slack Workspace](https://qiskit.enterprise.slack.com/archives/C07LYA6PL83) nebo příslušné veřejné repozitáře na GitHubu.
</Admonition>

## Funkce {#features}

Následující funkce jsou součástí rozšíření pro [Visual Studio Code](https://code.visualstudio.com/) (VS Code) a kompatibilní editory, jakož i pro [JupyterLab](https://jupyterlab.readthedocs.io/en/latest/):

* Urychluje generování kódu v Qiskie díky využití generativní AI postavené na modelech specializovaných na generování Qiskit kódu.
* Umožňuje zadávat abstraktní i konkrétní prompty pro generování doporučení.
* Zobrazuje návrhy, které můžeš zkontrolovat, přijmout nebo odmítnout.
* Podporuje soubory Python a Jupyter notebooky.
* Obsahuje ochranné mechanismy, které brání odpovídání na otázky představující potenciální riziko pro uživatele, jako je nenávistný projev.

Pokud chceš Qiskit Code Assistant integrovat přímo do svého vývojového prostředí, postupuj podle pokynů v příslušném tématu:
- [JupyterLab](/guides/qiskit-code-assistant-jupyterlab)
- [VS Code](/guides/qiskit-code-assistant-vscode)

## Velký jazykový model (LLM) za Qiskit Code Assistantem {#the-large-language-model-llm-behind-qiskit-code-assistant}

Pro poskytování návrhů kódu využívá Qiskit Code Assistant velký jazykový model (LLM). V tomto případě se Qiskit Code Assistant aktuálně opírá o model `mistral-small-3.2-24b-qiskit`, postavený na [modelu Mistral-Small-3.2-24B-Qiskit](https://huggingface.co/Qiskit/mistral-small-3.2-24b-qiskit). Model `mistral-small-3.2-24b-qiskit` vylepšuje schopnosti generování kódu modelu `Mistral-Small-3.2-24B-Instruct-2506` pro Qiskit prostřednictvím rozšířeného předtrénování a doladění na vysoce kvalitních datech Qiskit, commitech Pythonu a chatu. Více informací o rodině modelů Mistral AI najdeš v [dokumentaci Mistral AI](https://docs.mistral.ai/getting-started/models). Další podrobnosti o modelech `.*-qiskit` najdeš v článku [Qiskit Code Assistant: Training LLMs for generating Quantum Computing Code](https://arxiv.org/abs/2405.19495).

Naše LLM specializované pro Qiskit jsou dostupné také jako open-source modely. Všechny dostupné modely najdeš na https://huggingface.co/Qiskit.

## Benchmarky Qiskit HumanEval a Qiskit HumanEval Hard {#the-qiskit-humaneval-and-qiskit-humaneval-hard-benchmarks}

Pro testování modelu `mistral-small-3.2-24b-qiskit` a dalších modelů jsme spolupracovali s Qiskit Advocates a odborníky na vytvoření benchmarků založených na spouštění kódu, nazvaných Qiskit HumanEval (QHE) a Qiskit HumanEval Hard (QHE Hard), a spustili jsme je na těchto modelech. Tyto benchmarky jsou podobné [HumanEval](https://arxiv.org/abs/2107.03374) a zahrnují řadu náročných programovacích úloh, všechny postavené na oficiálních knihovnách Qiskit.

Benchmarky se skládají přibližně z 150 testů, přičemž každý je tvořen definicí funkce, za níž následuje docstring popisující úkol, který má model vyřešit. Každý příklad také obsahuje referenční kanonické řešení a jednotkové testy pro hodnocení správnosti generovaných řešení. Testy mají tři úrovně obtížnosti: základní, středně pokročilou a obtížnou. Benchmark Qiskit HumanEval Hard je variantou Qiskit HumanEval, ale odstraňuje informace týkající se importů kódu, takže LLM musí sám přijít na správné importy metod nebo tříd. Tato změna podle našich testů a počátečních výsledků datovou sadu pro LLM výrazně ztěžuje.

Datové sady pro Qiskit HumanEval a Qiskit HumanEval Hard jsou dostupné na těchto stránkách: [Qiskit HumanEval](https://huggingface.co/datasets/Qiskit/qiskit_humaneval) a [Qiskit HumanEval](https://huggingface.co/datasets/Qiskit/qiskit_humaneval_hard). K rozvoji těchto benchmarků můžeš přispět v [repozitáři na GitHubu](https://github.com/qiskit-community/qiskit-human-eval).

### Rychlý start (doporučeno) {#quick-start-recommended}

Nejjednodušší způsob, jak začít s Qiskit Code Assistantem v lokálním režimu, je použít automatizované instalační skripty pro rozšíření VS Code nebo JupyterLab. Tyto skripty automaticky nainstalují [Ollama](https://ollama.com/) pro spuštění LLM, stáhnou doporučený model a nakonfigurují rozšíření za tebe.

#### Nastavení rozšíření pro VS Code {#vs-code-extension-setup}

Spusť v terminálu následující příkaz:

```bash
bash <(curl -fsSL https://raw.githubusercontent.com/Qiskit/qiskit-code-assistant-vscode/main/setup_local.sh)
```

Tento skript provede následující kroky:
- Nainstaluje Ollama (pokud ještě není nainstalována)
- Stáhne a nakonfiguruje doporučený model Qiskit Code Assistantu
- Nastaví rozšíření VS Code pro práci s tvým lokálním nasazením

#### Nastavení rozšíření pro JupyterLab {#jupyterlab-extension-setup}

Spusť v terminálu následující příkaz:

```bash
bash <(curl -fsSL https://raw.githubusercontent.com/Qiskit/qiskit-code-assistant-jupyterlab/main/setup_local.sh)
```

Tento skript:
- Nainstaluje Ollama (pokud ještě není nainstalována)
- Stáhne a nakonfiguruje doporučený model Qiskit Code Assistantu
- Nastaví rozšíření JupyterLab pro práci s tvým lokálním nasazením

Existuje několik způsobů, jak nasadit a pracovat se staženým modelem Qiskit Code Assistantu. Tato příručka demonstruje použití [Ollama](https://ollama.com) takto: buď pomocí [aplikace Ollama](#using-the-ollama-application) s využitím integrace Hugging Face Hub nebo lokálního modelu, nebo pomocí [balíčku `llama-cpp-python`](#use-llama-cpp-python).

#### Použití aplikace Ollama {#using-the-ollama-application}

Aplikace Ollama poskytuje jednoduché řešení pro lokální spouštění LLM. Je snadná na použití, s CLI, které celý proces nastavení, správu modelů a interakci s nimi činí poměrně přímočarými. Je ideální pro rychlé experimentování a pro uživatele, kteří chtějí mít méně technických detailů na starosti.

##### Instalace Ollamy {#install-ollama}

1. Stáhni [aplikaci Ollama](https://ollama.com/download)
1. Nainstaluj stažený soubor
1. Spusť nainstalovanou aplikaci Ollama

    <Admonition type="info">
        Aplikace běží úspěšně, když se ikona Ollamy zobrazí v pruhu nabídek plochy. Funkčnost služby můžeš také ověřit přechodem na `http://localhost:11434/`.
    </Admonition>

1. Vyzkoušej Ollamu v terminálu a začni spouštět modely. Například:

    ```
    ollama run hf.co/Qiskit/Qwen2.5-Coder-14B-Qiskit
    ```

##### Nastavení Ollamy pomocí integrace Hugging Face Hub {#set-up-ollama-using-the-hugging-face-hub-integration}

[Integrace Ollama/Hugging Face Hub](https://huggingface.co/docs/hub/ollama) poskytuje způsob, jak pracovat s modely hostovanými na Hugging Face Hub, aniž by bylo nutné vytvářet nový modelfile nebo ručně stahovat soubory GGUF či safetensors. Výchozí soubory `template` a `params` jsou pro model na Hugging Face Hub již zahrnuty.

1. Ujisti se, že aplikace Ollama běží.

1. Přejdi na stránku požadovaného modelu a zkopíruj URL. Například https://huggingface.co/Qiskit/Qwen2.5-Coder-14B-Qiskit-GGUF.

1. V terminálu spusť příkaz:

    ```
    ollama run hf.co/Qiskit/Qwen2.5-Coder-14B-Qiskit
    ```

Můžeš použít model `hf.co/Qiskit/Qwen2.5-Coder-14B-Qiskit` nebo kterýkoli z dalších aktuálně doporučených oficiálních modelů GGUF `hf.co/Qiskit/mistral-small-3.2-24b-qiskit-GGUF` nebo `hf.co/Qiskit/granite-3.3-8b-qiskit-GGUF`.

##### Nastavení Ollamy s ručně staženým modelem GGUF Qiskit Code Assistantu {#set-up-ollama-with-a-manually-downloaded-qiskit-code-assistant-gguf-model}

Pokud jsi ručně stáhl model GGUF, například https://huggingface.co/Qiskit/Qwen2.5-Coder-14B-Qiskit-GGUF, a chceš experimentovat s různými šablonami a parametry, můžeš jej načíst do lokální aplikace Ollama podle těchto kroků.

1. Vytvoř soubor `Modelfile` s následujícím obsahem a nezapomeň aktualizovat `<PATH-TO-GGUF-FILE>` na skutečnou cestu ke staženému modelu.

    ```
    FROM <PATH-TO-GGUF-FILE>
    TEMPLATE """{{ if .System }}
    System:
    {{ .System }}

    {{ end }}{{ if .Prompt }}Question:
    {{ .Prompt }}

    {{ end }}Answer:
    ```python{{ .Response }}
    """

    PARAMETER stop "Question:"
    PARAMETER stop "Answer:"
    PARAMETER stop "System:"
    PARAMETER stop "```"

    PARAMETER temperature 0
    PARAMETER top_k 1
    ```

1. Spusť následující příkaz k vytvoření vlastní instance modelu na základě souboru `Modelfile`.

    ```
    ollama create Qwen2.5-Coder-14B-Qiskit -f ./path-to-model-file
    ```

    <Admonition type="note">
        Tento proces může trvat určitou dobu, než Ollama přečte soubor modelu, inicializuje jeho instanci a nakonfiguruje ji podle zadaných specifikací.
    </Admonition>

##### Spuštění ručně staženého modelu Qiskit Code Assistantu v Ollama {#run-qiskit-code-assistant-model-manually-downloaded-in-ollama}

Jakmile je model `Qwen2.5-Coder-14B-Qiskit` nastaven v Ollama, spusť jej a interaguj s ním v terminálu (v režimu chatu) pomocí následujícího příkazu.

```
ollama run Qwen2.5-Coder-14B-Qiskit
```

Užitečné příkazy:

- `ollama list` – Zobrazí seznam modelů na tvém počítači
- `ollama rm Qwen2.5-Coder-14B-Qiskit` – Odstraní model
- `ollama show Qwen2.5-Coder-14B-Qiskit` – Zobrazí informace o modelu
- `ollama stop Qwen2.5-Coder-14B-Qiskit` – Zastaví aktuálně běžící model
- `ollama ps` – Zobrazí seznam aktuálně načtených modelů

</AccordionItem>

<AccordionItem title="Ruční nasazení modelů Qiskit Code Assistantu lokálně pomocí balíčku `llama-cpp-python`">

<span id="use-llama-cpp-python"></span>
Alternativou k aplikaci Ollama je balíček `llama-cpp-python`, což je Python binding pro `llama.cpp`. Poskytuje ti větší kontrolu a flexibilitu při lokálním spouštění modelu GGUF a je ideální pro uživatele, kteří chtějí lokální model integrovat do svých pracovních postupů a Python aplikací.

1. Nainstaluj [`llama-cpp-python`](https://pypi.org/project/llama-cpp-python/)
2. Interaguj s modelem ve své aplikaci pomocí `llama_cpp`. Například:

```python
from llama_cpp import Llama

model_path = <PATH-TO-GGUF-FILE>

model = Llama(
        model_path,
        seed=17,
        n_ctx=10000,
        n_gpu_layers=37, # to offload in gpu, but put 0 if all in cpu
    )

input = 'Generate a quantum circuit with 2 qubits'
raw_pred = model(input)["choices"][0]["text"]
```

Lze také přidat parametry generování textu pro přizpůsobení inference:

```python
generation_kwargs = {
        "max_tokens": 512,
        "echo": False, # Echo the prompt in the output
        "top_k": 1
    }

raw_pred = model(input, **generation_kwargs)["choices"][0]["text"]
```

</AccordionItem>

<AccordionItem title="Ruční nasazení modelů Qiskit Code Assistantu lokálně pomocí llama.cpp">
#### Použití knihovny `llama.cpp` {#use-the-llama-cpp-library}

Další alternativou je použití `llama.cpp`, open-source knihovny pro inferenci LLM na CPU s minimálním nastavením.
Poskytuje nízkoúrovňovou kontrolu nad spouštěním modelu a typicky se spouští z příkazové řádky s odkazem na lokální soubor modelu ve formátu GGUF.

Existuje několik způsobů, jak nainstalovat `llama.cpp` na svém počítači:

- Instalace llama.cpp pomocí [brew, nix nebo winget](https://github.com/ggml-org/llama.cpp/blob/master/docs/install.md)
- Spuštění v Dockeru: viz [dokumentace k Dockeru od týmu `llama.cpp`](https://github.com/ggml-org/llama.cpp/blob/master/docs/docker.md)
- Stažení předkompilovaných binárních souborů ze [stránky vydání](https://github.com/ggml-org/llama.cpp/releases)
- Sestavení ze zdrojového kódu [naklonováním tohoto repozitáře](https://github.com/ggml-org/llama.cpp/blob/master/docs/build.md)

Po instalaci můžeš pomocí `llama.cpp` interagovat s modely GGUF v konverzačním režimu takto:
```bash
# Use a local model file
llama-cli -m my_model.gguf -cnv

# Or download and run a model directly from Hugging Face
llama-cli -hf Qiskit/Qwen2.5-Coder-14B-Qiskit-GGUF -cnv
```

Lze také spustit server s API kompatibilním s OpenAI pro tento model takto:
```bash
llama-server -hf Qiskit/Qwen2.5-Coder-14B-Qiskit-GGUF
```

##### Pokročilé parametry {#advanced-parameters}
S programem `llama-cli` můžeš řídit generování modelu pomocí voleb příkazové řádky. Například počáteční „systémový” prompt lze zadat příznakem `-p/--prompt`. V konverzačním režimu (`-cnv`) tento počáteční prompt funguje jako systémová zpráva. Jinak lze jednoduše přidat libovolnou instrukci před text promptu. Lze také upravit parametry vzorkování – například: teplotu (`--temp`), top-k (`--top-k`), top-p (`--top-p`), penalizaci opakování (`--repeat-penalty`) a seed (`--seed`). Níže je ukázka volání s těmito možnostmi:

```bash
llama-cli -hf Qiskit/Qwen2.5-Coder-14B-Qiskit-GGUF \
  -p "You are a friendly assistant." -cnv \
  --temp 0.7 \
  --top-k 50 \
  --top-p 0.95 \
  --repeat-penalty 1.1 \
  --seed 42
```

Pro zajištění správné funkčnosti našich modelů Qiskit doporučujeme použít systémový prompt uvedený v našich HF GGUF repozitářích: systémový prompt pro [mistral-small-3.2-24b-qiskit-GGUF](https://huggingface.co/Qiskit/mistral-small-3.2-24b-qiskit-GGUF/blob/main/system), [Qwen2.5-Coder-14B-Qiskit-GGUF](https://huggingface.co/Qiskit/Qwen2.5-Coder-14B-Qiskit-GGUF/blob/main/system), [granite-3.3-8b-qiskit-GGUF](https://huggingface.co/Qiskit/granite-3.3-8b-qiskit-GGUF/blob/main/system) a [granite-3.2-8b-qiskit-GGUF](https://huggingface.co/Qiskit/granite-3.2-8b-qiskit-GGUF/blob/main/system).

<AccordionItem title="Ruční propojení rozšíření s lokálním nasazením">

Pro komunikaci s lokálně nasazeným modelem Qiskit Code Assistantu použij rozšíření VS Code a JupyterLab pro Qiskit Code Assistant. Jakmile máš nastavenou aplikaci Ollama [s modelem](#using-the-ollama-application), můžeš rozšíření nakonfigurovat pro připojení k lokální službě.

#### Propojení s rozšířením Qiskit Code Assistant pro VS Code {#connect-with-the-qiskit-code-assistant-vs-code-extension}

S rozšířením Qiskit Code Assistant pro VS Code můžeš komunikovat s modelem a provádět doplňování kódu při psaní. To se hodí pro uživatele, kteří hledají pomoc při psaní Qiskit kódu pro své Python aplikace.

1. Nainstaluj [rozšíření Qiskit Code Assistant pro VS Code](/guides/qiskit-code-assistant-vscode).
1. Ve VS Code přejdi do **Uživatelských nastavení** a nastav **Qiskit Code Assistant: Url** na URL tvého lokálního nasazení Ollamy (například `http://localhost:11434`).
1. Znovu načti VS Code přechodem na **View > Command Palette...** a výběrem **Developer: Reload Window**.

Model Qiskit Code Assistantu nakonfigurovaný v Ollama by se měl zobrazit ve stavovém řádku a je připraven k použití.

#### Propojení s rozšířením Qiskit Code Assistant pro JupyterLab {#connect-with-the-qiskit-code-assistant-jupyterlab-extension}

S rozšířením Qiskit Code Assistant pro JupyterLab můžeš komunikovat s modelem a provádět doplňování kódu přímo v Jupyter Notebooku. Uživatelé, kteří pracují převážně s Jupyter Notebooky, mohou využít toto rozšíření k dalšímu vylepšení svého zážitku při psaní Qiskit kódu.

1. Nainstaluj [rozšíření Qiskit Code Assistant pro JupyterLab](/guides/qiskit-code-assistant-jupyterlab).
1. V JupyterLab přejdi do **Settings Editor** a nastav **Qiskit Code Assistant Service API** na URL tvého lokálního nasazení Ollamy (například `http://localhost:11434`).

Model Qiskit Code Assistantu nakonfigurovaný v Ollama by se měl zobrazit ve stavovém řádku a je připraven k použití.

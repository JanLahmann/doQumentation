Tento průvodce ukazuje dva funkční příklady řezů Gate s balíčkem `qiskit-addon-cutting`. První příklad ukazuje, jak snížit hloubku Circuit (počet instrukcí Circuit) řezáním propojovacích Gate na nesousedních Qubitech, které by jinak při transpilaci na hardware způsobily overhead ve formě SWAP operací. Druhý příklad popisuje, jak pomocí řezání Gate snížit šířku Circuit (počet Qubitů) rozdělením Circuit na několik Circuit s menším počtem Qubitů.

Oba příklady použijí ansatz [`efficient_su2`](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.library.efficient_su2) a rekonstruují stejnou pozorovatelnou veličinu.
## Řezání Gate pro snížení hloubky Circuit {#gate-cutting-to-reduce-circuit-depth}

Následující postup snižuje hloubku Circuit řezáním vzdálených Gate a zabraňuje tak dlouhé sérii SWAP Gate, která by jinak byla zavedena.

Začni s ansatzem [`efficient_su2`](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.library.efficient_su2) s „kruhovým" provázáním, které zavádí vzdálené Gate.

```python
# Added by doQumentation — required packages for this notebook
!pip install -q numpy qiskit qiskit-addon-cutting qiskit-aer qiskit-ibm-runtime
```

```python
import numpy as np
from qiskit.circuit.library import efficient_su2
from qiskit.quantum_info import SparsePauliOp
from qiskit.transpiler import generate_preset_pass_manager
from qiskit_ibm_runtime.fake_provider import FakeManilaV2
from qiskit_ibm_runtime import SamplerV2, Batch
from qiskit_aer.primitives import EstimatorV2
from qiskit_addon_cutting import (
    cut_gates,
    partition_problem,
    generate_cutting_experiments,
    reconstruct_expectation_values,
)

circuit = efficient_su2(num_qubits=4, entanglement="circular")
circuit.assign_parameters([0.4] * len(circuit.parameters), inplace=True)

observable = SparsePauliOp(["ZZII", "IZZI", "-IIZZ", "XIXI", "ZIZZ", "IXIX"])
print(f"Observable: {observable}")
circuit.draw("mpl", scale=0.8)
```

```text
Observable: SparsePauliOp(['ZZII', 'IZZI', 'IIZZ', 'XIXI', 'ZIZZ', 'IXIX'],
              coeffs=[ 1.+0.j,  1.+0.j, -1.+0.j,  1.+0.j,  1.+0.j,  1.+0.j])
```

![Output of the previous code cell](/docs/images/guides/qiskit-addons-cutting-gates/extracted-outputs/1551c440-c158-478a-a8fe-86df834c59bd-1.svg)

Každá z Gate [`CNOT`](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.library.CXGate) mezi Qubity $q_0$ a $q_3$ zavede po transpilaci dvě SWAP Gate (za předpokladu, že Qubity jsou propojeny v přímé linii). Abys předešl tomuto nárůstu hloubky, můžeš tyto vzdálené Gate nahradit objekty [`TwoQubitQPDGate`](https://docs.quantum.ibm.com/api/qiskit-addon-cutting/qpd-two-qubit-qpd-gate) pomocí metody [`cut_gates()`](https://docs.quantum.ibm.com/api/qiskit-addon-cutting/qiskit-addon-cutting#cut_gates). Tato funkce také vrací seznam instancí [`QPDBasis`](../api/qiskit-addon-cutting/qpd-qpd-basis) – jednu pro každý rozklad.

```python
# Find the indices of the distant gates
cut_indices = [
    i
    for i, instruction in enumerate(circuit.data)
    if {circuit.find_bit(q)[0] for q in instruction.qubits} == {0, 3}
]

# Decompose distant CNOTs into TwoQubitQPDGate instances
qpd_circuit, bases = cut_gates(circuit, cut_indices)

qpd_circuit.draw("mpl", scale=0.8)
```

![Output of the previous code cell](/docs/images/guides/qiskit-addons-cutting-gates/extracted-outputs/66dc0a14-ab51-4190-9cda-1c0373e91b9e-0.svg)

Nyní, když byly přidány instrukce pro řez Gate, budou mít subexperimenty po transpilaci menší hloubku než původní Circuit. Níže uvedený fragment kódu generuje subexperimenty pomocí [`generate_cutting_experiments`](https://docs.quantum.ibm.com/api/qiskit-addon-cutting/qiskit-addon-cutting#generate_cutting_experiments), které přijímá Circuit a pozorovatelnou veličinu k rekonstrukci.

<Admonition type="note" title="Poznámka k počtu vzorků">
    Argument `num_samples` určuje, kolik vzorků se má odebrat z kvazi-pravděpodobnostního rozdělení, a určuje přesnost koeficientů použitých pro rekonstrukci. Předání nekonečna (`np.inf`) zajistí přesný výpočet všech koeficientů. Více informací najdeš v dokumentaci API k [generování vah](https://docs.quantum.ibm.com/api/qiskit-addon-cutting/qpd#generate_qpd_weights) a [generování řezacích experimentů](https://docs.quantum.ibm.com/api/qiskit-addon-cutting/qiskit-addon-cutting#generate_cutting_experiments).
</Admonition>

Jakmile jsou subexperimenty vygenerovány, můžeš je transpilovat a použít primitivum `Sampler` k vzorkování rozdělení a rekonstrukci odhadovaných hodnot očekávání. Následující blok kódu subexperimenty generuje, transpiluje a spustí. Poté rekonstruuje výsledky a porovná je s přesnou hodnotou očekávání.

```python
# Generate the subexperiments and sampling coefficients
subexperiments, coefficients = generate_cutting_experiments(
    circuits=qpd_circuit, observables=observable.paulis, num_samples=np.inf
)

# Set a backend to use and transpile the subexperiments
backend = FakeManilaV2()
pass_manager = generate_preset_pass_manager(
    optimization_level=1, backend=backend
)
isa_subexperiments = pass_manager.run(subexperiments)

# Set up the Qiskit Runtime Sampler primitive, submit the subexperiments, and retrieve the results
sampler = SamplerV2(backend)
job = sampler.run(isa_subexperiments, shots=4096 * 3)
results = job.result()

# Reconstruct the results
reconstructed_expval_terms = reconstruct_expectation_values(
    results,
    coefficients,
    observable.paulis,
)

# Apply the coefficients of the original observable
reconstructed_expval = np.dot(reconstructed_expval_terms, observable.coeffs)

estimator = EstimatorV2()
exact_expval = (
    estimator.run([(circuit, observable, [0.4] * len(circuit.parameters))])
    .result()[0]
    .data.evs
)
print(
    f"Reconstructed expectation value: {np.real(np.round(reconstructed_expval, 8))}"
)
print(f"Exact expectation value: {np.round(exact_expval, 8)}")
print(
    f"Error in estimation: {np.real(np.round(reconstructed_expval-exact_expval, 8))}"
)
print(
    f"Relative error in estimation: {np.real(np.round((reconstructed_expval-exact_expval) / exact_expval, 8))}"
)
```

```text
Reconstructed expectation value: 0.49812826
Exact expectation value: 0.50497603
Error in estimation: -0.00684778
Relative error in estimation: -0.0135606
```

<Admonition type="caution" title="Poznámka ke koeficientům pozorovatelné veličiny">
    Pro přesnou rekonstrukci hodnoty očekávání je nutné na výstup rekonstrukce aplikovat koeficienty původní pozorovatelné veličiny (které se liší od koeficientů ve výstupu `generate_cutting_experiments()`), protože tato informace byla ztracena při generování řezacích experimentů nebo při rozkladu pozorovatelné veličiny.

    Tyto koeficienty lze obvykle aplikovat pomocí `numpy.dot()`, jak je ukázáno výše.
</Admonition>
## Řezání Gate pro snížení šířky Circuit {#gate-cutting-to-reduce-circuit-width}

Tato část ukazuje použití řezání Gate ke snížení šířky Circuit. Začni se stejným ansatzem [`efficient_su2`](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.library.efficient_su2), ale s „lineárním" provázáním.

```python
qc = efficient_su2(4, entanglement="linear", reps=2)
qc.assign_parameters([0.4] * len(qc.parameters), inplace=True)

observable = SparsePauliOp(["ZZII", "IZZI", "-IIZZ", "XIXI", "ZIZZ", "IXIX"])
print(f"Observable: {observable}")

qc.draw("mpl", scale=0.8)
```

```text

              coeffs=[ 1.+0.j,  1.+0.j, -1.+0.j,  1.+0.j,  1.+0.j,  1.+0.j])
```

Poté vygeneruj *podcircuity* a *podzornovatelné veličiny*, které budeš spouštět, pomocí funkce [`partition_problem()`](https://docs.quantum.ibm.com/api/qiskit-addon-cutting/qiskit-addon-cutting#partition_problem). Tato funkce přijímá Circuit, pozorovatelnou veličinu a volitelné schéma rozdělení a vrací řezané Circuit a pozorovatelné veličiny ve formě slovníku.

Rozdělení je definováno řetězcem štítků ve tvaru `"AABB"`, kde každý štítek v tomto řetězci odpovídá Qubitu na stejném indexu argumentu `circuit`. Qubity sdílející společný štítek oddílu jsou seskupeny dohromady a jakékoli nelokální Gate zahrnující více než jeden oddíl budou přeříznuty.

<Admonition type="info" title="Poznámka">
   Argument `observables` funkce `partition_problem` je typu [`PauliList`](https://docs.quantum.ibm.com/api/qiskit/qiskit.quantum_info.PauliList). Koeficienty a fáze členů pozorovatelné veličiny jsou při rozkladu problému a spuštění subexperimentů ignorovány. Mohou být znovu aplikovány při rekonstrukci hodnoty očekávání.
</Admonition>

```python
partitioned_problem = partition_problem(
    circuit=qc, partition_labels="AABB", observables=observable.paulis
)
subcircuits = partitioned_problem.subcircuits
subobservables = partitioned_problem.subobservables
bases = partitioned_problem.bases

print(f"Sampling overhead: {np.prod([basis.overhead for basis in bases])}")
print(f"Subobservables: {subobservables}")
subcircuits["A"].draw("mpl", scale=0.8)
```

```text
Sampling overhead: 81.0
Subobservables: {'A': PauliList(['II', 'ZI', 'ZZ', 'XI', 'ZZ', 'IX']), 'B': PauliList(['ZZ', 'IZ', 'II', 'XI', 'ZI', 'IX'])}
```

Dalším krokem je použití podcircuitů a podzornovatelných veličin k vygenerování *subexperimentů*, které budou spuštěny na QPU pomocí metody [`generate_cutting_experiments`](https://docs.quantum.ibm.com/api/qiskit-addon-cutting/qiskit-addon-cutting#generate_cutting_experiments).

Pro odhadnutí hodnoty očekávání Circuit plné velikosti je z jointního kvazi-pravděpodobnostního rozdělení rozložených Gate vygenerováno mnoho subexperimentů, které jsou pak spuštěny na jednom nebo více QPU. Počet vzorků odebraných z tohoto rozdělení je řízen argumentem `num_samples`.

Následující blok kódu generuje subexperimenty a spouští je pomocí primitiva `Sampler` na lokálním simulátoru. (Chceš-li je spustit na QPU, změň `backend` na zvolený QPU prostředek.)

```python
subexperiments, coefficients = generate_cutting_experiments(
    circuits=subcircuits, observables=subobservables, num_samples=np.inf
)

# Set a backend to use and transpile the subexperiments
backend = FakeManilaV2()
pass_manager = generate_preset_pass_manager(
    optimization_level=1, backend=backend
)
isa_subexperiments = {
    label: pass_manager.run(partition_subexpts)
    for label, partition_subexpts in subexperiments.items()
}

# Submit each partition's subexperiments to the Qiskit Runtime Sampler
# primitive, in a single batch so that the jobs will run back-to-back.
with Batch(backend=backend) as batch:
    sampler = SamplerV2(mode=batch)
    jobs = {
        label: sampler.run(subsystem_subexpts, shots=4096 * 3)
        for label, subsystem_subexpts in isa_subexperiments.items()
    }

# Retrieve results
results = {label: job.result() for label, job in jobs.items()}
```

Nakonec je hodnota očekávání celého Circuit rekonstruována pomocí metody [`reconstruct_expectation_values`](https://docs.quantum.ibm.com/api/qiskit-addon-cutting/qiskit-addon-cutting#reconstruct_expectation_values).

Níže uvedený blok kódu rekonstruuje výsledky a porovná je s přesnou hodnotou očekávání.

```python
# Get expectation values for each observable term
reconstructed_expval_terms = reconstruct_expectation_values(
    results,
    coefficients,
    subobservables,
)

# Reconstruct final expectation value
reconstructed_expval = np.dot(reconstructed_expval_terms, observable.coeffs)

estimator = EstimatorV2()
exact_expval = (
    estimator.run([(qc, observable, [0.4] * len(qc.parameters))])
    .result()[0]
    .data.evs
)
print(
    f"Reconstructed expectation value: {np.real(np.round(reconstructed_expval, 8))}"
)
print(f"Exact expectation value: {np.round(exact_expval, 8)}")
print(
    f"Error in estimation: {np.real(np.round(reconstructed_expval-exact_expval, 8))}"
)
print(
    f"Relative error in estimation: {np.real(np.round((reconstructed_expval-exact_expval) / exact_expval, 8))}"
)
```

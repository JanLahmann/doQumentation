## 2. Backend at Target {#2-backend-and-target}

Nagbibigay ang Qiskit ng API para makuha ang impormasyon, parehong static at dynamic, tungkol sa isang quantum device. Gumagamit tayo ng Backend instance para makipag-ugnayan sa isang device, na kinabibilangan ng isang Target instance — isang abstract na modelo ng makina na nagbubuod ng mahahalagang katangian tulad ng instruction set architecture (ISA) nito at anumang mga katangian o hadlang na nauugnay dito.
Gamitin natin ang mga backend instance na ito para makuha ang ilan sa impormasyong makikita sa pahina ng [Compute resources](https://quantum.cloud.ibm.com/computers)
sa IBM Quantum&reg; Platform. Una, gumagawa tayo ng backend instance para sa isang device na gusto natin. Sa sumusunod, pinipili natin ang "ibm_kyoto", "ibm_kawasaki", o ang pinaka-hindi abala na Eagle machine. Maaaring mag-iba ang iyong access sa mga QPU; i-update ang pangalan ng backend ayon sa pagkakataon.

```python
service = QiskitRuntimeService()
# backend = service.backend("ibm_kawasaki") # an Eagle, if you have access to ibm_kawasaki
backend = service.least_busy(
    operational=True, simulator=False, min_num_qubits=127
)  # Eagle
backend.name
```

```text
'ibm_strasbourg'
```

Magsisimula tayo sa ilang pangunahing (static) na impormasyon tungkol sa device.

```python
print(
    f"""
{backend.name}, {backend.num_qubits} qubits
processor type = {backend.processor_type}
basis gates = {backend.basis_gates}
"""
)
```

```text
ibm_strasbourg, 127 qubits
processor type = {'family': 'Eagle', 'revision': 3}
basis gates = ['ecr', 'id', 'rz', 'sx', 'x']
```

### 2.2 Coupling map {#22-coupling-map}

Iguguhit natin ngayon ang coupling map ng device. Tulad ng makikita, ang mga node ay mga qubit na may numero. Ang mga gilid ay nagpapakita ng mga pares kung saan maaari kang direktang mag-apply ng 2-qubit entangling gate. Ang topology ay tinatawag na "heavy-hex lattice".

```python
# This function requires that Graphviz is installed. If you need to install Graphviz you can refer to:
# https://graphviz.org/download/#executable-packages for instructions.
try:
    fig = backend.coupling_map.draw()
except RuntimeError as ex:
    print(ex)
fig
```

### 3.2 Mga error ng gate at readout {#32-gate-and-readout-errors}

Lilipat tayo ngayon sa mga gate error. Una, pag-aaralan natin ang istraktura ng data ng target instance. Ito ay isang dictionary na ang mga key ay mga pangalan ng operasyon.

```python
target = backend.target
target.keys()
```

```text
dict_keys(['measure', 'id', 'sx', 'delay', 'x', 'for_loop', 'rz', 'if_else', 'ecr', 'reset', 'switch_case'])
```

Ang mga value nito ay mga dictionary rin. Tingnan natin ang ilan sa mga item ng value (dictionary) para sa operasyong 'sx'.

```python
for i, qq in enumerate(target["sx"]):

    print(i, qq, target["sx"][qq])
```

Dahil naiintindihan na natin ang mga istraktura ng data, handa na tayong kalkulahin ang mga median error para sa mga gate na 'sx' at 'ecr'. Muli, ihambing ang mga resulta sa ipinapakita para sa device sa [IBM Quantum Platform.](https://quantum.cloud.ibm.com/)

```python
sx_errors = [inst_prop.error for inst_prop in target["sx"].values()]
```

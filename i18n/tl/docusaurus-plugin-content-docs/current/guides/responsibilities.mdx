# Pag-unawa sa iyong mga responsibilidad sa paggamit ng Qiskit Runtime {#understanding-your-responsibilities-when-using-qiskit-runtime}

Suriin ang mga sumusunod na seksyon para sa mga tiyak na responsibilidad mo at ng IBM&reg; kapag ginagamit mo ang Qiskit Runtime. Para sa pangkalahatang mga tuntunin ng paggamit, tingnan ang [Terms and Notices](https://cloud.ibm.com/docs/overview/terms-of-use?topic=overview-terms) para sa IBM Cloud&reg;.

## Pamamahala ng insidente at operasyon {#incident-and-operations-management}

Ang pamamahala ng insidente at operasyon ay kinabibilangan ng mga gawaing tulad ng monitoring, pamamahala ng event, high availability, pagtukoy ng problema, pagbawi, at buong backup at recovery.

| Gawain | Mga responsibilidad ng IBM | Mga responsibilidad mo |
| --- | --- | --- |
|Pamamahala ng kalusugan ng imprastraktura|<ul><li>Patuloy na maghatid ng mga bagong feature, update, at bug fix ayon sa pangangailangan nang malinaw.</li><li>Magbigay ng mga abiso para sa naplanong maintenance, mga security bulletin, o mga hindi inaasahang pagkabigo.</li></ul> | <ul><li>[Itakda ang mga kagustuhan para makatanggap ng mga email](https://cloud.ibm.com/user/notifications) tungkol sa mga abiso ng serbisyo.</li><li>Subaybayan ang [IBM Cloud status](https://cloud.ibm.com/status) na pahina para sa mga pangkalahatang anunsyo.</li></ul> |
|Monitoring at abiso sa kalusugan ng imprastraktura| <ul><li>Ayusin ang lahat ng isyu sa kapaligiran ng imprastraktura.</li><li>Abisuhan ang mga customer ng mga naaangkop na insidente.</li></ul> | <ul><li>Tukuyin ang epekto ng bawat iniulat na insidente.</li><li>Makipag-ugnayan sa [IBM Support](/guides/support) kung kinakailangan.</li></ul>|
|Panatilihin ang IBM Cloud high availability SLA| Patakbuhin ang Qiskit Runtime alinsunod sa [IBM Cloud Public Service Level Agreements](https://cloud.ibm.com/docs/overview?topic=overview-slas) (SLAs). | Sundin ang mga pinakamahusay na gawi para sa high availability. |

## Pamamahala ng pagbabago {#change-management}

Ang pamamahala ng pagbabago ay kinabibilangan ng mga gawaing tulad ng deployment, configuration, mga upgrade, patching, mga pagbabago sa configuration, at pagtanggal.

| Gawain | Mga responsibilidad ng IBM | Mga responsibilidad mo |
| --- | --- | --- |
| Mga major version upgrade | Magbigay ng availability at mga tool para sa mga major version update. | Lumipat sa isang sinusuportahang bersyon bago matapos ang suporta para sa lumang bersyon. |
| Mga pagbabago sa API version | Pag-version ng mga REST API. | Mag-integrate sa gustong bersyon ng mga REST API. |
| Mga update, fix, at bagong feature | <ul><li>Magbigay ng regular na mga update, bug fix, at bagong mga feature, nang malinaw ayon sa isang tuluy-tuloy na modelo ng delivery.</li><li>Mag-post ng mga abiso para sa mga pagbabagong nakakaapekto sa mga customer.</li></ul> | <ul><li>Itakda ang mga kagustuhan para [makatanggap ng mga email notification](https://cloud.ibm.com/user/notifications) at subaybayan ang [IBM Cloud status](https://cloud.ibm.com/status) na pahina para sa mga pangkalahatang anunsyo.</li><li>Basahin ang mga system-generated na abiso at [mga anunsyo sa platform,](https://quantum.cloud.ibm.com/announcements) at gumawa ng naaangkop na aksyon.</li></ul> |

## Pamamahala ng pagkakakilanlan at access {#identity-and-access-management}

Ang pamamahala ng pagkakakilanlan at access ay kinabibilangan ng mga gawaing tulad ng authentication, authorization, pamamahala ng mga patakaran sa kontrol ng access, at pag-apruba, pagbibigay, at pagbawi ng access.

| Gawain | Mga responsibilidad ng IBM | Mga responsibilidad mo |
| --- | --- | --- |
| Kontrol ng pagkakakilanlan at access | Magbigay ng kakayahang pamahalaan at limitahan ang access sa mga resource sa pamamagitan ng IBM Cloud. | Ibigay, bawiin, at pamahalaan ang access sa mga resource sa pamamagitan ng [Cloud Identity and Access Management](/guides/access-groups) (IAM). |
| Authentication ng serbisyo | Tiyakin na ang mga authenticated na user lamang ang may access sa iyong mga serbisyo at resource ng Qiskit Runtime. | <ul><li>[Pangalagaan ang iyong API key](/guides/cloud-setup-untrusted#initialize-the-service-in-an-untrusted-environment) mula sa hindi awtorisadong access.</li><li>I-rotate at i-update ang iyong mga [API key](https://cloud.ibm.com/iam/apikeys) ayon sa mga kinakailangan ng iyong patakaran sa seguridad.</li></ul> |
| Observability | Pahintulutan ang integration ng IBM Cloud Activity Tracker Event Routing para ma-audit ang mga aksyon ng user. | Mag-set up ng [IBM Cloud Activity Tracker Event Routing](https://cloud.ibm.com/docs/vmware-service?topic=vmware-service-at_events) o iba pang mga function para subaybayan ang aktibidad ng user. |

## Seguridad at pagsunod sa regulasyon {#security-and-regulation-compliance}

Ang seguridad at pagsunod sa regulasyon ay kinabibilangan ng mga gawaing tulad ng pagpapatupad ng mga kontrol sa seguridad at sertipikasyon ng pagsunod.

| Gawain | Mga responsibilidad ng IBM | Mga responsibilidad mo |
| --- | --- | --- |
| Pangkalahatan | <ul><li>Panatilihin ang mga kontrol na naaangkop sa iba't ibang pamantayan ng pagsunod sa industriya.</li><li>Subaybayan, ihiwalay, at ibalik ang functionality ng serbisyo.</li><li>Subaybayan at iulat ang kalusugan ng mga interface ng serbisyo.</li></ul> | Tiyakin na hindi ibinibigay ang personal at regulated na data. |
| Mga kahinaan | Subaybayan para matukoy ang mga isyu sa kahinaan at pagsunod sa seguridad. | <ul><li>Manatiling may kaalaman tungkol sa mga posibleng kahinaan at isyu sa seguridad sa pamamagitan ng mga security bulletin na nagbibigay ng mga potensyal na aksyon sa pagwawasto.</li><li>Tingnan ang [security section ng Qiskit GitHub repository.](https://github.com/Qiskit/qiskit/security) Lahat ng mga kahinaan sa seguridad ay naka-post doon.</li><li>Tingnan ang [IBM Cloud status website](https://cloud.ibm.com/status) para mahanap ang mga anunsyo at security bulletin tungkol sa mga pangunahing event na nakakaapekto sa IBM Cloud platform, imprastraktura, at mga pangunahing serbisyo.</li></ul> |
| Encryption | I-encrypt ang data kapag ipinapadala ito sa anumang pampublikong network at sa loob ng pribadong data center network ng Cloud Service. Ang mga paraan ng encryption tulad ng HTTPS at TLS ay ginagamit para protektahan ang data habang gumagalaw. | Tiyakin, kung kinakailangan, na ang iyong koneksyon ay naka-encrypt mula dulo hanggang dulo. |
| Seguridad | <ul><li>Tiyakin ang seguridad ng data sa disk at data habang gumagalaw sa loob ng imprastraktura nito.</li><li>Magbigay ng ligtas na serbisyo na sumusunod sa mga pangunahing pamantayan.</li></ul> | Limitahan ang [access ng user](/guides/access-groups) sa mga naaangkop na resource at resource group. |

## Pagbawi mula sa sakuna {#disaster-recovery}

| Gawain | Mga responsibilidad ng IBM | Mga responsibilidad mo |
| --- | --- | --- |
| Pangkalahatan | I-restore o muling buuin ang mga kapaligiran ng provisioning at mga resource sa mga apektadong rehiyon. | <ul><li>Subaybayan ang mga estado ng instance at resource.</li><li>Mag-provision ng bagong Qiskit Runtime service [instances](/guides/instances) sa kahaliling rehiyon, kung available.</li></ul> |
| Pagbawi ng mga serbisyo | Awtomatikong bawiin at ibalik ang mga kakayahan ng serbisyo ng Qiskit Runtime sa normal na operasyon. | Muling isumite ang anumang mga apektadong job pagkatapos ng pagbawi. |
| Backup ng data ng configuration ng user | Hindi responsable ang IBM para sa backup ng data ng configuration ng user. | Backup ng anumang data ng configuration, kung kinakailangan. |
| Backup at pagbawi ng mga workload | Hindi responsable ang IBM para sa backup o pagbawi ng data ng workload ng user. | Backup at pagbawi ng iyong mga workload, kung kinakailangan. |

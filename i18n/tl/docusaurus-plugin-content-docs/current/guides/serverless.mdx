# Panimula sa Qiskit Serverless {#introduction-to-qiskit-serverless}

Ang Qiskit Serverless ay nagbibigay ng simpleng interface para patakbuhin ang mga workload sa quantum-classical na mga resources. Kasama dito ang pag-deploy ng mga programa sa IBM Quantum&reg; Platform at ang pagpapatakbo ng mga workload nang remote, gayundin ang madaling pamamahala ng resources para sa mga use case ng multi-cloud at quantum-centric supercomputing.

Ang mga premium na gumagamit ay maaaring mag-build, mag-deploy, at magpatakbo ng kanilang mga workload nang remote sa classical compute na ibinibigay sa pamamagitan ng IBM Quantum Platform.

<Admonition type="tip">
**Ang Qiskit Serverless ay nagpa-upgrade, at mabilis na nagbabago ang mga feature nito.** Sa panahon ng development phase na ito, hanapin ang mga release note at pinakabagong dokumentasyon sa pahina ng [Qiskit Serverless GitHub](https://qiskit.github.io/qiskit-serverless/index.html).
</Admonition>

<Admonition type="note">
Ito ay isang experimental na feature na available lamang para sa mga gumagamit ng IBM Quantum Premium Plan, Flex Plan, at On-Prem (sa pamamagitan ng IBM Quantum Platform API) Plan.
</Admonition>

## Kailan dapat gamitin ang Qiskit Serverless? {#when-should-i-use-qiskit-serverless}

Tinutulungan ng Qiskit Serverless ang pamamahala ng classical at quantum na mga resources sa buong [Qiskit patterns](/guides/intro-to-patterns) na workflow. Kasama dito ang ilan sa mga sumusunod na halimbawa:

- Pag-parallelize ng mga classical na gawain, tulad ng pre-processing at post-processing
- Pagpapanatili ng mga matagal na workload sa cloud, kahit nakapatay na ang iyong laptop
- Pag-deploy ng mga reusable na programa

## Ano ang kasama? {#what-is-included}

Ang mga sumusunod na compute resources ay kasama bilang default sa kapaligiran ng Qiskit Serverless. Maaaring makipag-ugnayan ang mga gumagamit sa [support](/guides/support) para palakihin ang alinman sa mga ito.

- 16 vCPU cores
- 64GB RAM
- Maximum na dalawang aktibong job bawat gumagamit sa anumang oras

## I-install ang Qiskit Serverless {#install-qiskit-serverless}

Para gamitin ang Qiskit Serverless sa IBM Quantum Platform, i-install ang mga sumusunod na package:

```bash
pip install qiskit-ibm-catalog qiskit_serverless
```

Ang `qiskit_ibm_catalog` ay nagbibigay ng mga client-side na tool para mag-upload at magpatakbo ng mga remote na programa, habang ang `qiskit_serverless` ay nagbibigay ng mga server-side na tool para ipamahagi ang compute at pamahalaan ang data. Ang mga package na ito ay nangangailangan ng `python` 3.11+. Para sa mga gumagamit at organisasyon na gustong patakbuhin ang Qiskit Serverless sa custom na imprastraktura, sundin ang gabay na [Cloud infrastructure setup](https://qiskit.github.io/qiskit-serverless/).

<Admonition type="tip">
Sa kasalukuyan, ang talahanayan ng IBM Quantum workloads ay nagpapakita lamang ng mga Qiskit Runtime workload. Gamitin ang `job.status()` para makita ang kasalukuyang status ng iyong Qiskit Serverless workload. Hanapin ang isang halimbawa sa gabay na [Patakbuhin ang iyong unang Qiskit Serverless workload nang remote](/guides/serverless-run-first-workload#serverless-job-status).
</Admonition>

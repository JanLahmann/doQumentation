# Pagprotekta sa mga resources ng Qiskit Runtime Service gamit ang context-based restrictions {#protecting-qiskit-runtime-service-resources-with-context-based-restrictions}

Ang context-based restrictions ay nagbibigay sa mga may-ari ng account at mga administrator ng kakayahang tukuyin at ipatupad ang mga paghihigpit sa access para sa mga resources ng IBM Cloud&reg; batay sa konteksto ng mga kahilingang access. Ang access sa mga resources ng Qiskit Runtime Service ay maaaring kontrolin gamit ang context-based restrictions at mga patakaran ng identity and access management (IAM).

Ang mga paghihigpit na ito ay gumagana kasabay ng mga tradisyonal na patakaran ng IAM, na batay sa pagkakakilanlan, para magbigay ng karagdagang antas ng proteksyon. Hindi tulad ng mga patakaran ng IAM, ang context-based restrictions ay hindi nagtatalaga ng access. Sinusuri ng context-based restrictions kung ang isang kahilingang access ay nagmumula sa isang pinahintulutang konteksto na iyong na-configure. Dahil parehong nagpapatupad ng access ang IAM access at context-based restrictions, ang context-based restrictions ay nagbibigay ng proteksyon kahit na may mga nakompromisong o maling pinamamahalaang kredensyal. Para sa karagdagang impormasyon, tingnan ang paksa na [What are context-based restrictions](https://cloud.ibm.com/docs/account?topic=account-context-restrictions-whatis).

Ang isang user ay dapat magkaroon ng Administrator role sa serbisyo ng Qiskit Runtime Service para lumikha, mag-update, o magtanggal ng mga panuntunan. Dapat din magkaroon ang isang user ng Editor o Administrator role sa serbisyo ng Context-based restrictions para lumikha, mag-update, o magtanggal ng mga network zone. Ang isang user na may Viewer role sa serbisyo ng Context-based restrictions ay makakapagdagdag lamang ng mga network zone sa isang panuntunan.

Ang anumang mga kaganapan ng Activity Tracker o audit log na nabuo ay nagmumula sa serbisyo ng context-based restrictions, hindi sa Qiskit Runtime Service. Para sa karagdagang impormasyon, tingnan ang pahina ng [Monitoring context-based restrictions](https://cloud.ibm.com/docs/account?topic=account-cbr-monitor).

Para makapagsimula sa pagprotekta ng iyong mga resources ng Qiskit Runtime Service gamit ang context-based restrictions, tingnan ang tutorial na [Leveraging context-based restrictions to secure your resources](https://cloud.ibm.com/docs/account?topic=account-context-restrictions-tutorial).

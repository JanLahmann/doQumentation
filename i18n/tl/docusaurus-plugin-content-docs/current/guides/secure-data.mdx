# Pag-secure ng iyong data {#securing-your-data}

Para masigurado na ligtas mong mapamamahalaan ang iyong data kapag gumagamit ng IBM Quantum&reg; Platform, mahalagang malaman kung anong data ang nakaimbak at ine-encrypt, at kung paano mo mabubura ang anumang nakaimbak na data.

## Pagprotekta sa iyong sensitibong data {#protecting-your-sensitive-data}

Ang data na iniimbak mo sa IBM Cloud&reg; ay ine-encrypt habang nakaimbak gamit ang isang random na nabuong key.

## Pagbura ng iyong data {#deleting-your-data}

Ang pagbura ng isang service instance ay aalisin ang lahat ng content na nauugnay sa instance na iyon, tulad ng iyong mga job, resulta, parameter, at programa. Para burahin ang isang instance, mula sa [pahina ng Instances,](https://quantum.cloud.ibm.com/instances) hanapin ang instance na gusto mong alisin, i-click ang overflow menu nito, pagkatapos ay i-click ang **Delete**. Hihilingin sa iyo na kumpirmahin ang pagbura.

Kung mayroon kang Manager role, maaari ka ring magbura ng isang indibidwal na job at ang data nito. Alamin ang higit pa tungkol sa mga access policy, grupo, at role sa [dokumentasyon](/guides/access-groups).

## Patakaran sa pagpapanatili ng data {#data-retention-policy}

Itatago ng IBM ang iyong data (na hindi nabura) sa loob ng tatlong taon habang ikaw ay kasalukuyang gumagamit ng serbisyo. Para mahanap ang impormasyon tungkol sa kung gaano katagal pinananatili ng IBM ang data pagkatapos mong itigil ang paggamit ng serbisyo, tingnan ang [Qiskit Runtime data sheet](https://www.ibm.com/support/customer/csol/terms/?id=BB6EE1D06BDE11EBA199CCFE56FA6D9D), na makikita sa IBM Terms site.

# I-configure ang IBM Quantum Platform para sa isang organisasyon {#configure-ibm-quantum-platform-for-an-organization}

Sundin ang mga hakbang na ito para i-set up ang IBM Quantum Platform.

<span id="iam-org"></span>
## 1. I-configure ang mga setting ng IAM {#iam-org}

Una, i-configure ang mga sumusunod na setting sa IAM account ng administrator. Para suriin at i-configure ang mga setting na ito, pumunta sa [Manage → IAM → Settings](https://cloud.ibm.com/iam/settings).

* **User list visibility** — tinutukoy nito kung makikita ng mga user ang isa't isa, anuman ang project assignment. Ang setting na `enabled` ay nagre-restrict ng visibility ng user. Ibig sabihin, hindi makikita ng mga user sa iyong account ang isa't isa, kahit na may access sila sa parehong mga resources. Piliin ang angkop na value para sa iyong environment. Tingnan ang [Controlling user visibility](https://cloud.ibm.com/docs/account?topic=account-iam-user-setting) para sa karagdagang impormasyon.
* **API key creation** — kinokontrol nito kung maaaring gumawa ng API keys ang mga user. Sa IBM Quantum Platform, karaniwan ang paggamit ng API keys. Kung ginagamit ang API keys, piliin ang `disabled`. Bilang alternatibo, maaari kang magbigay ng mga partikular na permiso sa bawat user.

<span id="crt-rsc-grp-org"></span>
## 2. (Opsyonal) Gumawa ng mga resource group {#crt-rsc-grp-org}

Laktawan ang hakbang na ito kung gumagamit ka ng mga access group na direktang nagre-reference ng mga IBM Quantum service instance.

Kung pinili mong gumamit ng mga resource group, pumunta sa [Manage → Account → Resource groups (in Account resources)](https://cloud.ibm.com/account/resource-groups) at i-click ang **Create**.

<span id="create-instance-org"></span>
## 3. Gumawa ng mga IBM Quantum service instance {#create-instance-org}

Kung nakagawa ka na ng mga IBM Quantum service instance, laktawan ang hakbang na ito.

* Kung gumagamit ka ng mga resource group, siguraduhing gawin ang mga service instance sa angkop na resource group.
* Ang pangalan ng service instance, tulad ng `QR-ml`, ay kailangan para sa mga reference ng access group.

Sundin ang mga hakbang sa [Set up to use IBM Cloud](/guides/cloud-setup#cloud) para sa mga tagubilin.

<span id="create-group-org"></span>
## 4. Gumawa ng mga access group para sa mga proyekto {#create-group-org}

Una, [gumawa ng custom role](/guides/custom-roles#custom) na nagbibigay-daan sa mga user na magsagawa ng mga aksyon para magtrabaho gamit ang mga IBM Quantum service instance.
Susunod, [gumawa ng access group](/guides/access-groups) para sa bawat proyekto at bigyan ang group na iyon ng pinakamaliit na hanay ng mga permisong kailangan para magtrabaho sa mga resources ng proyekto.

<span id="setup-idp-org"></span>

<span id="firewall"></span>
## 6. I-configure ang iyong firewall {#firewall}

Para i-configure ang iyong firewall na payagan ang access sa mga IBM Quantum API endpoint, idagdag ang mga sumusunod na URL sa iyong allowlist:

- Mga IBM Cloud global service:
    ```text
    cloud.ibm.com
    iam.cloud.ibm.com
    api.global-search-tagging.cloud.ibm.com
    globalcatalog.cloud.ibm.com
    resource-controller.cloud.ibm.com
    ```
- IBM Quantum Qiskit Runtime services - rehiyon us-east:
    ```text
    quantum.cloud.ibm.com
    s3.us-east.cloud-object-storage.appdomain.cloud
    ```
- IBM Quantum Qiskit Runtime services - rehiyon eu-de:
    ```text
    eu-de.quantum.cloud.ibm.com
    s3.direct.eu-de.cloud-object-storage.appdomain.cloud
    ```
- IBM Quantum Qiskit Functions preview service:
    ```text
    qiskit-serverless.quantum.ibm.com
    ```

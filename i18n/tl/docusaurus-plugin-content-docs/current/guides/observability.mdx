# Mga kaganapan sa pagsubaybay ng aktibidad para sa Qiskit Runtime {#activity-tracking-events-for-qiskit-runtime}

Ang mga serbisyo ng IBM Cloud&reg;, tulad ng Qiskit Runtime, ay nagge-generate ng mga kaganapan sa pagsubaybay ng aktibidad.

Ang mga kaganapan sa pagsubaybay ng aktibidad ay nag-uulat ng mga aktibidad na nagbabago ng estado ng isang serbisyo sa IBM Cloud. Maaari mong gamitin ang mga kaganapan upang imbestigahan ang abnormal na aktibidad at kritikal na mga aksyon, at upang sumunod sa mga kinakailangan ng regulatoryang pag-audit.

Maaari mong gamitin ang IBM Cloud Activity Tracker Event Routing, isang platform na serbisyo, upang i-route ang mga kaganapan sa pag-audit sa iyong account patungo sa mga destinasyon ng iyong pagpili sa pamamagitan ng pag-configure ng mga target at ruta na nagtatakda kung saan ipinapadala ang mga kaganapan sa pagsubaybay ng aktibidad. Para sa karagdagang impormasyon, tingnan ang [About IBM Cloud Activity Tracker Events Routing.](https://cloud.ibm.com/docs/atracker?topic=atracker-about)

Maaari mong gamitin ang IBM Cloud Logs upang mag-visualize at mag-alerto sa mga kaganapan na nabuo sa iyong account at ini-route ng IBM Cloud Activity Tracker Event Routing patungo sa isang IBM Cloud Logs na instance.

## Mga lokasyon kung saan nalilikha ang mga kaganapan sa pagsubaybay ng aktibidad {#locations-where-activity-tracking-events-are-generated}

Nagpapadala ang Qiskit Runtime ng mga kaganapan sa pagsubaybay ng aktibidad sa mga sumusunod na rehiyon:

- Washington ("us-east")
- Frankfurt ("eu-de")

## Pagtingin sa mga kaganapan sa pagsubaybay ng aktibidad para sa Qiskit Runtime {#viewing-activity-tracking-events-for-qiskit-runtime}

Maaari mong gamitin ang IBM Cloud Logs upang mag-visualize at mag-alerto sa mga kaganapan na nabuo sa iyong account at ini-route ng IBM Cloud Activity Tracker Event Routing patungo sa isang IBM Cloud Logs na instance.

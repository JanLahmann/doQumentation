# Mga kaganapan sa pagsubaybay ng aktibidad para sa serbisyo ng IBM Quantum System {#activity-tracking-events-for-the-ibm-quantum-system-service}

Ang mga serbisyo ng IBM Cloud&reg;, tulad ng serbisyo ng IBM&reg; Quantum System, ay nagbubuo ng mga kaganapan sa pagsubaybay ng aktibidad.

Ang mga kaganapan sa pagsubaybay ng aktibidad ay nag-uulat ng mga aktibidad na nagbabago ng estado ng isang serbisyo sa IBM Cloud. Magagamit mo ang mga kaganapan para imbestigahan ang abnormal na aktibidad at mga kritikal na aksyon at para sumunod sa mga kinakailangan ng regulasyon sa pag-audit.

Maaari kang gumamit ng IBM Cloud Activity Tracker Event Routing, isang platform na serbisyo, para i-ruta ang mga kaganapan sa pag-audit sa iyong account patungo sa mga destinasyon ng iyong pagpili sa pamamagitan ng pag-configure ng mga target at ruta na nagtatakda kung saan ipinapadala ang mga kaganapan sa pagsubaybay ng aktibidad. Para sa karagdagang impormasyon, tingnan ang [About IBM Cloud Activity Tracker Events Routing](https://cloud.ibm.com/docs/atracker?topic=atracker-about).

Maaari kang gumamit ng IBM Cloud Logs para ma-visualize at ma-alert sa mga kaganapang nabuo sa iyong account at ini-ruta ng IBM Cloud Activity Tracker Event Routing sa isang IBM Cloud Logs instance.

## Pagtingin sa mga kaganapan sa pagsubaybay ng aktibidad para sa serbisyo ng IBM Quantum System {#viewing-activity-tracking-events-for-the-ibm-quantum-system-service}

Maaari kang gumamit ng IBM Cloud Logs para ma-visualize at ma-alert sa mga kaganapang nabuo sa iyong account at ini-ruta ng IBM Cloud Activity Tracker Event Routing sa isang IBM Cloud Logs instance.

### Paglulunsad ng IBM Cloud Logs mula sa pahina ng Observability {#launching-ibm-cloud-logs-from-the-observability-page}

Para sa impormasyon tungkol sa paglulunsad ng IBM Cloud Logs UI, tingnan ang [Launching the UI](https://cloud.ibm.com/docs/cloud-logs?topic=cloud-logs-instance-launch#instance-launch-cloud-ui) sa dokumentasyon ng IBM Cloud Logs.

| Aksyon | Itinaas ng |
| -- | -- |
| `quantum-computing.direct-access-backend-configuration.read` | Kumuha ng backend configuration (`GET /v1/backends/{backend-name}/configuration`) |
| `quantum-computing.direct-access-backend-properties.read` | Kumuha ng backend properties (`GET /v1/backends/{backend-name}/properties`) |
| `quantum-computing.direct-access-backend.list` | Kumuha ng listahan ng mga backend (`GET /v1/backends`) |
| `quantum-computing.direct-access-backend.read` | Kumuha ng mga detalye ng backend (`GET /v1/backends/{backend-name}`) |
| `quantum-computing.direct-access-job.cancel` | Kanselahin ang isang job (`POST /v1/jobs/{job-id}/cancel`) |
| `quantum-computing.direct-access-job.create` | Magpatakbo ng isang job (`POST /v1/jobs`) |
| `quantum-computing.direct-access-job.delete` | Tanggalin ang isang job (`DELETE /v1/jobs/{job-id}`) |
| `quantum-computing.direct-access-job.list` | Kumuha ng mga job (`GET /v1/jobs`) |
| `quantum-computing.direct-access-job.read` | Kumuha ng Job ayon sa ID (`GET /v1/jobs/{job-id}`) |

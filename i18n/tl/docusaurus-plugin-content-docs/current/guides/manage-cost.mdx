# Pamahalaan ang gastos {#manage-cost}

Inilalarawan ng gabay na ito kung paano magtakda ng limitasyon sa gastos sa iyong Pay-As-You-Go Plan na instance. Maaari mo ring pamahalaan ang gastos sa pamamagitan ng [pagpapaliit ng quantum time ng workload](/guides/minimize-time), at pagtakda ng [maximum na oras ng execution](/guides/max-execution-time) sa iyong mga workload.

Ang _limitasyon sa gastos_ ay tumutukoy sa kabuuang gastos ng lahat ng workload na pinatakbo gamit ang instance na ito mula nang ito ay nilikha, at palaging magiging mas malaki o katumbas ng kabuuang gastos. Kapag naabot na ng instance ang tinukoy na kabuuang bilang ng segundo, hindi na maaaring magpatakbo ng karagdagang workload at hindi na madadagdag ang gastos.

    <Admonition type="note">
      Ang limitasyon sa gastos ay palaging tinutukoy sa US dollars (USD), pagkatapos ay kino-convert sa runtime seconds. Gayunpaman, para sa buwanang billing, sinisingil ka sa iyong lokal na pera, na tinutukoy sa iyong IBM Cloud&reg; account. Dahil maaaring magbago-bago ang mga exchange rate ng pera, ang gastos para sa X runtime seconds ay maaaring magkaiba kapag unang kinakalkula sa USD kaysa kapag talagang sinisingil ka sa iyong lokal na pera. Kaya, kung ang iyong lokal na pera ay hindi USD, ang kabuuang halaga na sinisingil para sa bilang ng segundo na tinukoy sa field na ito ay maaaring mag-iba mula sa halaga sa dolyar na iyong tinukoy.
    </Admonition>

## Magtakda o baguhin ang limitasyon sa gastos {#set-or-change-a-cost-limit}

Maraming paraan para magtakda o baguhin ang limitasyon sa gastos:

* **IBM Quantum Platform**: Buksan ang pahina ng [Mga Instance](https://quantum.cloud.ibm.com/instances). Hanapin ang instance na gusto mong limitahan, i-click ang menu ng Mga Aksyon sa dulo ng row, pagkatapos ay i-click ang **Edit details**. Itakda ang kabuuang limitasyon sa gastos o ang limitasyon sa paggamit, pagkatapos ay i-click ang **Save changes**.
   <Admonition type="note">
   Maaari ka ring magtakda ng limitasyon sa gastos o paggamit kapag gumagawa ng bagong instance sa ilalim ng Pay-As-You-Go Plan. Kapag pinili mo ang Pay-As-You-Go tile, may lalabas na field na may label na "Total cost limit (Optional)". Maaari kang maglagay ng kabuuang limitasyon sa gastos sa field na ito.
   </Admonition>

* **API**: [Itakda ang limitasyon ng instance sa segundo.](https://docs.quantum.ibm.com/api/qiskit-runtime-rest/tags/instances#tags__instances__operations__replace_instance_data)

   ```api
   PUT /v1/instances/configuration "instance_limit":<seconds>
   ```

* **IBM Cloud CLI**: Para sa mga tagubilin sa paggamit ng CLI, tingnan ang [Getting started with the IBM Cloud CLI.](https://cloud.ibm.com/docs/cli?topic=cli-getting-started)

   ```cli
      ibmcloud resource service-instance-update <instance_CRN>
      -p '{"instance_limit_seconds":"<seconds>"}'
   ```

## Tingnan ang limitasyon sa gastos {#view-the-cost-limit}

Maraming paraan para tingnan ang kasalukuyang limitasyon sa gastos:

* **IBM Quantum Platform**: Ang Max cost limit ay ipinapakita sa Pay-As-You-Go tab ng talahanayan ng [Mga Instance](https://quantum.cloud.ibm.com/instances).

* **API**:

   ```api
   GET /v1/instances/configuration
   ```
   Tingnan ang [dokumentasyon ng REST API](https://docs.quantum.ibm.com/api/qiskit-runtime-rest/tags/instances).

* **IBM Cloud CLI**: Para sa mga tagubilin sa paggamit ng CLI, tingnan ang [Getting started with the IBM Cloud CLI.](https://cloud.ibm.com/docs/cli?topic=cli-getting-started)

   ```cli
   ibmcloud resource service-instance-update <instance_CRN> -p
   '{"instance_limit_seconds":null}'
   ```

## Alisin ang limitasyon sa gastos {#remove-a-cost-limit}

Maaari mong alisin ang limitasyon sa gastos sa ilang paraan:

* **IBM Quantum Platform**: Buksan ang talahanayan ng [Mga Instance](https://quantum.cloud.ibm.com/instances). Hanapin ang instance na gusto mong i-update, i-click ang overflow menu sa dulo ng row, pagkatapos ay i-click ang **Edit details**. Burahin ang kabuuang limitasyon sa gastos o ang paggamit, pagkatapos ay i-click ang **Save changes**.

* **API**: [Itakda ang limitasyon ng instance sa `null`.](https://docs.quantum.ibm.com/api/qiskit-runtime-rest/tags/instances)

   ```api
   PUT /v1/instances/configuration "instance_limit":null
   ```

* **IBM Cloud CLI**: Para sa mga tagubilin sa paggamit ng CLI, tingnan ang [Getting started with the IBM Cloud CLI.](https://cloud.ibm.com/docs/cli?topic=cli-getting-started)

   ```cli
   ibmcloud resource service-instance-update <instance_CRN> -p
   '{"instance_limit_seconds":null}'
   ```
<Admonition type="note">
Ang mga administrator at may-ari lamang ang maaaring mag-edit ng instance. Tingnan ang gabay na [Lumikha ng mga patakaran sa access at mga grupo ng access](/guides/access-groups) para sa impormasyon tungkol sa pag-assign ng access.
</Admonition>

### Ano ang mangyayari kapag naabot ang limitasyon sa gastos {#what-happens-when-the-cost-limit-is-reached}

Kapag naabot na ang limitasyon sa gastos ng instance, ihihinto ang kasalukuyang tumatakbong job. Ang status nito ay itatatakda sa Canceled na may dahilang Ran too long. Ang anumang available na partial na resulta ay mananatili.

Hindi na maaaring magsumite ng karagdagang job gamit ang instance na ito hanggang sa taasan ang limitasyon sa gastos.

## Paano makita kung magkano ang sinisingil sa iyo {#how-to-see-what-youre-being-charged}

Makakatanggap ka ng buwanang invoice na nagbibigay ng detalye tungkol sa iyong mga singil sa resource. Maaari mong tingnan kung magkano na ang nagastos anumang oras sa pahina ng [IBM Cloud Billing and usage](https://cloud.ibm.com/billing).

Bukod dito, maaari mong tukuyin ang gastos bawat instance o bawat job anumang oras.

### Tingnan ang paggamit {#view-usage}

Tingnan ang paggamit ng instance sa pahina ng [Analytics](https://quantum.cloud.ibm.com/analytics).

### Tingnan ang gastos ng instance {#view-instance-cost}

Para malaman kung magkano ang na-bill sa isang instance sa kasalukuyang billing cycle, mula sa pahina ng [Mga Instance](https://quantum.cloud.ibm.com/instances), i-click ang instance para buksan ang pahina ng detalye nito.

Ito ang mga field na may kaugnayan sa gastos:

- **Billing cycle usage**: Ang dami ng _quantum time_ na ginamit ng instance na ito sa kasalukuyang billing cycle. Ang quantum time ay ang tagal na naka-commit ang QPU sa pagsasakatuparan ng kahilingan ng user.
- **Billing cycle cost**: Ang kabuuang gastos ng pagpapatakbo ng mga job sa kasalukuyang billing cycle.
- **Total usage**: Ang dami ng quantum time na ginamit ng instance na ito mula nang ito ay nilikha.
- **Total cost**: Ang kabuuang gastos ng pagpapatakbo ng mga job sa instance na ito mula nang ito ay nilikha. Ang mga administrator lamang ang maaaring magtakda ng halagang ito.

Maaari mong tingnan ang iyong billing cycle sa [pahina ng Billing and usage](https://cloud.ibm.com/billing).

### Tingnan ang gastos ng job {#view-job-cost}

Para malaman kung magkano ang na-bill sa bawat job na nauugnay sa isang instance, mula sa pahina ng [Mga Instance](https://quantum.cloud.ibm.com/instances), i-click ang instance para buksan ang pahina ng detalye nito. Pagkatapos, sa kaliwang bahagi, i-click ang Jobs.

Ito ang mga column na may kaugnayan sa gastos:

- **Usage**: Ang dami ng quantum time na ginamit ng job na ito. Ang quantum time ay ang tagal na naka-commit ang QPU sa pagsasakatuparan ng kahilingan ng user.
- **Cost**: Ang kabuuang gastos ng pagpapatakbo ng job na ito.

## Mag-set up ng mga notification sa paggastos {#set-up-spending-notifications}

Maaari kang mag-set up ng mga notification sa paggastos para maabisuhan kapag ang iyong account o isang partikular na serbisyo ay umabot sa isang tiyak na threshold ng paggastos na iyong itinakda. Para sa impormasyon, tingnan ang pahina ng [IBM Cloud Account types](https://cloud.ibm.com/docs/account?topic=account-accounts). Ang mga notification sa paggastos ng IBM Cloud ay kailangang gamitin kasama ang iba pang mga paraan ng pamamahala ng gastos sa ilang dahilan:

*   Ang mga notification ay nati-trigger lamang *pagkatapos* lumampas ang gastos sa tinukoy na limitasyon.
*   Ang gastos ay isinusumite sa sistema ng billing bawat oras. Kaya, maaaring magkaroon ng matagal na pagkaantala sa pagitan ng pagsusumite ng job at ng pagpapadala ng notification sa paggastos.
*   Ang sistema ng billing ay maaaring tumagal ng ilang araw bago maipadala ang impormasyon sa sistema ng invoicing, na maaaring magdulot ng karagdagang pagkaantala sa mga notification. Para sa karagdagang impormasyon tungkol sa kung paano gumagana ang sistema ng billing ng IBM Cloud, tingnan ang [Setting spending notifications.](https://cloud.ibm.com/docs/billing-usage?topic=billing-usage-spending)

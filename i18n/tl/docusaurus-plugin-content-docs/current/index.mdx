## Ang open-source na nilalaman ng IBM Quantum {#ibm-quantums-open-source-content}

Nagbibigay ang IBM ng napakaraming quantum computing na materyales sa pag-aaral — lahat ay open source sa ilalim ng CC BY-SA 4.0:

- **[Learning](https://quantum.cloud.ibm.com/learning)** — Mga nakaayos na kurso mula sa mga basic ng quantum hanggang sa mga advanced na paksa
- **[Tutorials](https://quantum.cloud.ibm.com/docs/en/tutorials)** — 40+ na mga tutorial tungkol sa transpilation, error mitigation, at iba pa
- **[Documentation](https://quantum.cloud.ibm.com/docs/en/guides)** — Mga gabay at API reference para sa Qiskit
- **[Source repo](https://github.com/Qiskit/documentation)** — Lahat ng nilalaman sa GitHub

Ang kanilang [Quantum Platform](https://quantum.cloud.ibm.com) ay palaging napapanahon at magandang-disenyo — ang pinakamahusay na lugar para sa pagbabasa, pag-aaral, at reference.

## Ano ang idadagdag ng proyektong ito {#what-this-project-adds}

Ang [Qiskit documentation](https://github.com/Qiskit/documentation) ng IBM ay open source (CC BY-SA 4.0), ngunit ang kanilang web application ay hindi. Ang doQumentation ay nagdadagdag ng open-source na frontend na may live code execution, awtomatikong credential injection, at simulator mode.

Pwedeng i-deploy kahit saan — mula sa [GitHub Pages](https://doqumentation.org) hanggang sa [Docker](https://github.com/JanLahmann/doQumentation/pkgs/container/doqumentation) hanggang sa [RasQberry](https://rasqberry.org/). Tingnan ang [lahat ng features](/features).

## Pagpapatakbo ng code {#code-execution}

Mag-click ng **Run** sa kahit anong code block. Ang unang click ay magsisimula ng libreng Jupyter kernel sa pamamagitan ng [Binder](https://mybinder.org) (1–2 min). Pagkatapos noon, instant na ang mga pagpapatakbo.

- **Simulator Mode (walang kailangang account)** — I-enable ang [Simulator Mode](/jupyter-settings#simulator-mode) upang magpatakbo ng lahat ng notebooks gamit ang AerSimulator o noise-model FakeBackends. Walang kailangang setup.

- **IBM Quantum Hardware** — Ilagay ang inyong API token at CRN sa [Settings](/jupyter-settings#ibm-quantum) nang isang beses lamang — ang mga credentials ay awtomatikong ini-inject sa bawat pagpapatakbo. [Gumawa ng libreng account](https://quantum.cloud.ibm.com/registration) kung wala pa kayong account.

<details>
<summary>Available na mga execution backend</summary>

Bawat tutorial ay may mga executable code blocks. Mag-click ng **Run** upang magpatakbo gamit ang isa sa tatlong backend:

1. **Binder** (default sa GitHub Pages) — Libreng remote Jupyter kernel sa pamamagitan ng [mybinder.org](https://mybinder.org)
2. **Local Jupyter (Docker / RasQberry)** — Kumokonekta sa lokal na Jupyter server na may pre-installed na Qiskit
3. **Custom server** — Ituro sa kahit anong Jupyter endpoint sa [Settings](/jupyter-settings#advanced)

</details>

<details>
<summary>Mga opsyon sa deployment</summary>

Ang doQumentation ay available bilang:

- **[GitHub Pages](https://doqumentation.org)** — Static site na may [Binder](https://mybinder.org) para sa remote code execution
- **[Docker](https://github.com/JanLahmann/doQumentation/pkgs/container/doqumentation)** — Kumpletong stack sa lokal, maaaring mag-offline
- **[RasQberry](https://rasqberry.org/)** — Self-hosted sa Raspberry Pi na may lokal na Jupyter kernel

</details>

<details>
<summary>Magpatakbo nang lokal gamit ang Podman / Docker</summary>

Mag-install ng [Podman Desktop](https://podman-desktop.io/) o [Docker Desktop](https://www.docker.com/products/docker-desktop/) (Mac / Windows / Linux), pagkatapos ay magpatakbo ng:

```bash
# Kumpletong stack: site + Jupyter + Qiskit (~3 GB)
podman run -p 8080:80 -p 8888:8888 ghcr.io/janlahmann/doqumentation:jupyter
```

Buksan ang [http://localhost:8080](http://localhost:8080) — gumagana ang code execution nang lokal, walang paghihintay sa Binder.

Para sa magaan na bersyon na walang lokal na code execution (~60 MB):

```bash
# Static site lamang — gumagana pa rin ang code execution sa pamamagitan ng Binder
podman run -p 8080:80 ghcr.io/janlahmann/doqumentation:latest
```

Gumagamit ng Docker? Palitan lamang ang `podman` ng `docker` — pareho ang mga command. Ang mga image ay multi-arch (`linux/amd64` + `linux/arm64`), kaya gumagana nang natural ang Apple Silicon Macs.

</details>

## الحالة الافتراضية {#default-state}

تشير *الحالة المرجعية* إلى نقطة البداية الثابتة الأولية لمسألتنا. لتحضير حالة مرجعية، نحتاج إلى تطبيق العملية الوحدوية غير المُعاملَة $U_R$ في بداية دائرتنا الكمومية، بحيث $|\rho\rangle = U_R |0\rangle$. إذا توفرت لديك تخمين مستنير أو نقطة بيانات من حل أمثل موجود، فمن المرجح أن تتقارب الخوارزمية التبايُنية بشكل أسرع إذا استخدمت ذلك كنقطة بداية.

أبسط حالة مرجعية ممكنة هي الحالة الافتراضية، حيث نستخدم الحالة الابتدائية لدائرة كمومية مؤلفة من $n$ كيوبت: $|0\rangle^{\otimes n}$. بالنسبة للحالة الافتراضية، يكون العامل الوحدوي $U_R \equiv I$. نظرًا لبساطتها، تُعدّ الحالة الافتراضية حالة مرجعية صالحة تُستخدم في سيناريوهات كثيرة.
## الحالة المرجعية الكلاسيكية {#classical-reference-state}

لنفترض أن لديك نظامًا مؤلفًا من ثلاثة كيوبتات وتريد البدء في الحالة $|001\rangle$ عوضًا عن الحالة الافتراضية $|000\rangle$. هذا مثال على حالة مرجعية كلاسيكية خالصة، ولبنائها تحتاج فقط إلى تطبيق [بوابة X](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.library.XGate) على الكيوبت $0$ (وفق ترتيب Qiskit للكيوبتات)، إذ إن $|001\rangle = X_0 |000\rangle$.

في هذه الحالة، يكون عاملنا الوحدوي $U_R \equiv X_0$، مما يعطي الحالة المرجعية $|\rho\rangle \equiv |001\rangle$.

```python
# Added by doQumentation — required packages for this notebook
```

```python
from qiskit import QuantumCircuit

qc = QuantumCircuit(3)
qc.x(0)

qc.draw("mpl")
```

## الحالة المرجعية الكمومية {#quantum-reference-state}

لنفترض أنك تهدف إلى البدء بحالة أكثر تعقيدًا تتضمن التراكب و/أو التشابك، كـ$\frac{1}{\sqrt{2}}(|100\rangle+|111\rangle)$.

للحصول على هذه الحالة من $|000\rangle$، أحد الأساليب هو استخدام [بوابة Hadamard](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.library.HGate) على الكيوبت $0$ ($H_0$)، وبوابة [CNOT (CX)](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.library.CXGate) مع الكيوبت $0$ كمتحكم والكيوبت $1$ كهدف ($CNOT_{01}$)، وأخيرًا بوابة $X$ على الكيوبت $2$ ($X_2$).

في هذا السيناريو، يكون عاملنا الوحدوي $U_{R} \equiv X_2CNOT_{01}H_0|000\rangle$، وحالتنا المرجعية هي $|\rho\rangle \equiv \frac{1}{\sqrt{2}}(|100\rangle+|111\rangle)$.

```python
qc = QuantumCircuit(3)
qc.h(0)
qc.cx(0, 1)
qc.x(2)

qc.draw("mpl")
```

## بناء الحالات المرجعية باستخدام دوائر القوالب {#constructing-reference-states-using-template-circuits}

يمكننا أيضًا استخدام دوائر قوالب متنوعة، مثل [`TwoLocal`](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.library.TwoLocal) التي تتيح التعبير عن معاملات قابلة للضبط وتشابكات متعددة بسهولة. سنتناول هذه الدوائر النموذجية بمزيد من التفصيل في الدرس التالي، لكن يمكننا استخدامها للحالات المرجعية *إذا* ربطنا المعاملات:

```python
from qiskit.circuit.library import TwoLocal
from math import pi

reference_circuit = TwoLocal(2, "rx", "cz", entanglement="linear", reps=1)
theta_list = [pi / 2, pi / 3, pi / 3, pi / 2]

reference_circuit = reference_circuit.assign_parameters(theta_list)

reference_circuit.decompose().draw("mpl")
```

## الحالات المرجعية الخاصة بالتطبيق {#application-specific-reference-states}
### تعلم الآلة الكمومي {#quantum-machine-learning}

في سياق المُصنِّف الكمومي التبايُني (VQC)، تُشفَّر بيانات التدريب في حالة كمومية من خلال دائرة مُعاملَة تُعرف بـ*خريطة الميزات*، حيث تمثل كل قيمة معامل نقطة بيانات من مجموعة بيانات التدريب. `zz_feature_map` هو نوع من الدوائر المُعاملَة يمكن استخدامه لتمرير نقاط بياناتنا ($x$) إلى خريطة الميزات هذه.

```python
from qiskit.circuit.library import zz_feature_map

data = [0.1, 0.2]

zz_feature_map_reference = zz_feature_map(feature_dimension=2, reps=2)
zz_feature_map_reference = zz_feature_map_reference.assign_parameters(data)
zz_feature_map_reference.decompose().draw("mpl")
```

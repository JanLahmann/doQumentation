## المحتوى مفتوح المصدر من IBM Quantum {#ibm-quantums-open-source-content}

توفر IBM ثروة من المواد التعليمية في الحوسبة الكمومية — وجميعها مفتوحة المصدر بموجب ترخيص CC BY-SA 4.0:

- **[التعلّم](https://quantum.cloud.ibm.com/learning)** — دورات منظمة من أساسيات الكم إلى الموضوعات المتقدمة
- **[الدروس التعليمية](https://quantum.cloud.ibm.com/docs/en/tutorials)** — أكثر من 40 درسًا تعليميًا حول التحويل البرمجي وتخفيف الأخطاء والمزيد
- **[الوثائق](https://quantum.cloud.ibm.com/docs/en/guides)** — أدلة إرشادية ومرجع API لـ Qiskit
- **[مستودع المصدر](https://github.com/Qiskit/documentation)** — جميع المحتويات على GitHub

[منصة Quantum](https://quantum.cloud.ibm.com) الخاصة بهم محدّثة دائمًا وحسنة التصميم — وهي أفضل مكان للقراءة والتعلم والمرجع.

## ما يضيفه هذا المشروع {#what-this-project-adds}

[وثائق Qiskit](https://github.com/Qiskit/documentation) من IBM مفتوحة المصدر (CC BY-SA 4.0)، لكن تطبيق الويب الخاص بهم ليس كذلك. يضيف doQumentation واجهة أمامية مفتوحة المصدر مع تنفيذ مباشر للكود وحقن تلقائي لبيانات الاعتماد ووضع المحاكاة.

قابل للنشر في أي مكان — من [GitHub Pages](https://doqumentation.org) إلى [Docker](https://github.com/JanLahmann/doQumentation/pkgs/container/doqumentation) إلى [RasQberry](https://rasqberry.org/). انظر [جميع الميزات](/features).

## تنفيذ الكود {#code-execution}

انقر على **Run** في أي كتلة برمجية. النقرة الأولى تبدأ نواة Jupyter عبر [Binder](https://mybinder.org) أو [IBM Code Engine](/jupyter-settings#code-engine). بعد ذلك، يكون التنفيذ فوريًا.

- **وضع المحاكاة (لا يتطلب حسابًا)** — فعّل [وضع المحاكاة](/jupyter-settings#simulator-mode) لتشغيل جميع الدفاتر باستخدام AerSimulator أو FakeBackends بنموذج الضوضاء. لا يتطلب أي إعداد.

- **أجهزة IBM Quantum** — أدخل رمز API و CRN في [الإعدادات](/jupyter-settings#ibm-quantum) مرة واحدة — يتم حقن بيانات الاعتماد تلقائيًا في كل تشغيل. [أنشئ حسابًا مجانيًا](https://quantum.cloud.ibm.com/registration) إذا لم يكن لديك حساب.

<details>
<summary>الخلفيات المتاحة للتنفيذ</summary>

كل درس تعليمي يحتوي على كتل برمجية قابلة للتنفيذ. انقر على **Run** لتنفيذها باستخدام إحدى الخلفيات الأربع:

1. **Binder** (الافتراضي على GitHub Pages) — نواة Jupyter عن بُعد مجانية عبر [mybinder.org](https://mybinder.org)
2. **IBM Code Engine** — Jupyter بدون خادم على حساب IBM Cloud الخاص بك. البدء البارد في ثوانٍ بدلاً من دقائق. [الإعداد](/jupyter-settings#code-engine)
3. **Jupyter محلي (Docker / RasQberry)** — يتصل بخادم Jupyter المحلي مع Qiskit مثبت مسبقًا
4. **خادم مخصص** — وجّه إلى أي نقطة نهاية Jupyter في [الإعدادات](/jupyter-settings#advanced)

عند توفر خلفيات متعددة، اختر الخلفية المراد استخدامها من [الإعدادات](/jupyter-settings#backend-selection).

</details>

<details>
<summary>خيارات النشر</summary>

doQumentation متاح كـ:

- **[GitHub Pages](https://doqumentation.org)** — موقع ثابت مع [Binder](https://mybinder.org) أو [IBM Code Engine](/jupyter-settings#code-engine) لتنفيذ الكود عن بُعد
- **[Docker](https://github.com/JanLahmann/doQumentation/pkgs/container/doqumentation)** — حزمة كاملة محليًا، قادر على العمل بدون إنترنت
- **[RasQberry](https://rasqberry.org/)** — مستضاف ذاتيًا على Raspberry Pi مع نواة Jupyter محلية

</details>

<details>
<summary>التشغيل محليًا باستخدام Podman / Docker</summary>

ثبّت [Podman Desktop](https://podman-desktop.io/) أو [Docker Desktop](https://www.docker.com/products/docker-desktop/) (Mac / Windows / Linux)، ثم شغّل:

```bash
# Full stack: site + Jupyter + Qiskit (~3 GB)
podman run -p 8080:80 -p 8888:8888 ghcr.io/janlahmann/doqumentation:jupyter
```

افتح [http://localhost:8080](http://localhost:8080) — تنفيذ الكود يعمل محليًا، بدون وقت انتظار Binder.

للحصول على نسخة خفيفة بدون تنفيذ كود محلي (~60 MB):

```bash
# Static site only — code execution still works via Binder
podman run -p 8080:80 ghcr.io/janlahmann/doqumentation:latest
```

هل تستخدم Docker بدلاً من ذلك؟ فقط استبدل `podman` بـ `docker` — الأوامر متطابقة. الصور متعددة البنيات (`linux/amd64` + `linux/arm64`)، لذا تعمل أجهزة Apple Silicon Mac بشكل أصلي.

</details>

# إدارة مستخدمي موفّر الهوية {#manage-id-provider-users}

يمكنك الدمج مع موفّر الهوية الخارجي (IDP) الخاص بك لمصادقة المستخدمين الخارجيين بأمان على حساب IBM Cloud. باستخدام موفّر خارجي، يمكنك توفير طريقة لمستخدمي شركتك لاستخدام تسجيل الدخول الأحادي (SSO). يمكنك ربط حساب السحابة الخاص بك بـ IDP خارجي بإحدى الطرق التالية:

- استخدام [IBM Cloud&reg; SAML service provider (SP)](https://cloud.ibm.com/docs/account?topic=account-ibm-idp-integration#federate-cloud-saml)
- استخدام [App ID](https://cloud.ibm.com/docs/account?topic=account-ibm-idp-integration#federate-appid) (قد يترتب عليه تكلفة)
- [ربط مستخدميك بجميع منتجات IBM&reg; عبر IBMid](https://cloud.ibm.com/docs/account?topic=account-ibm-idp-integration#federate-ibmid)

<Admonition type="tip">
بعد ربط موفّر الهوية الخاص بك، يمكنك إعداد قواعد ديناميكية حتى لا تضطر إلى إضافة كل مستخدم يدويًا إلى كل مجموعة عند انضمامه. راجع [تفعيل المصادقة من موفّر هوية خارجي](https://cloud.ibm.com/docs/account?topic=account-ibm-idp-integration#app-id-dynamic-rules) للاطلاع على التعليمات.
</Admonition>

## الخطوات التالية {#next-steps}

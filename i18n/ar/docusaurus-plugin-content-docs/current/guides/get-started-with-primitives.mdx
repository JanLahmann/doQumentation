## ابدأ مع Estimator {#start-estimator}

{/*Verified the v2 examples 2/29/24 - updated 10/29/24*/}

### 1. تهيئة الحساب {#1-initialize-the-account}

بما أن Qiskit Runtime Estimator هو خدمة مُدارة، فأنت بحاجة أولًا إلى تهيئة حسابك. يمكنك بعد ذلك اختيار وحدة المعالجة الكمية (QPU) التي تريد استخدامها لحساب قيمة التوقع.

اتبع الخطوات في [موضوع التثبيت والإعداد](install-qiskit) إذا لم يكن لديك حساب بعد.

```python
from qiskit_ibm_runtime import QiskitRuntimeService

service = QiskitRuntimeService()
backend = service.least_busy(
    operational=True, simulator=False, min_num_qubits=127
)

print(backend.name)
```

```text
ibm_torino
```

### 2. إنشاء دائرة ومؤثِّر (Observable) {#2-create-a-circuit-and-an-observable}

تحتاج إلى دائرة واحدة على الأقل ومؤثِّر واحد على الأقل كمدخلات لأولية Estimator.

```python
from qiskit.circuit.library import qaoa_ansatz
from qiskit.quantum_info import SparsePauliOp

entanglement = [tuple(edge) for edge in backend.coupling_map.get_edges()]
observable = SparsePauliOp.from_sparse_list(
    [("ZZ", [i, j], 0.5) for i, j in entanglement],
    num_qubits=backend.num_qubits,
)
circuit = qaoa_ansatz(observable, reps=2)
# the circuit is parametrized, so we will define the parameter values for execution
param_values = [0.1, 0.2, 0.3, 0.4]

print(f">>> Observable: {observable.paulis}")
```

يجب تحويل الدائرة والمؤثِّر ليستخدما فقط التعليمات التي تدعمها وحدة المعالجة الكمية (يُشار إليها بدوائر *معمارية مجموعة التعليمات (ISA)*). سنستخدم Transpiler للقيام بذلك.

```python
from qiskit.transpiler import generate_preset_pass_manager

pm = generate_preset_pass_manager(optimization_level=1, backend=backend)
isa_circuit = pm.run(circuit)
isa_observable = observable.apply_layout(isa_circuit.layout)
print(f">>> Circuit ops (ISA): {isa_circuit.count_ops()}")
```


### 3. تهيئة Qiskit Runtime Estimator {#3-initialize-qiskit-runtime-estimator}

عند تهيئة Estimator، استخدم المعامل `mode` لتحديد الوضع الذي تريد تشغيله فيه. القيم الممكنة هي كائنات `batch` أو `session` أو `backend` لوضع التنفيذ الدُفعي والجلسي والوظيفي على التوالي. لمزيد من المعلومات، راجع [مقدمة إلى أوضاع تنفيذ Qiskit Runtime.](execution-modes)

```python
from qiskit_ibm_runtime import EstimatorV2 as Estimator

estimator = Estimator(mode=backend)
```

### 4. استدعاء Estimator والحصول على النتائج {#4-invoke-the-estimator-and-get-results}

بعد ذلك، استدعِ الدالة `run()` لحساب قيم التوقع للدوائر والمؤثِّرات المُدخَلة. تُدخَل الدائرة والمؤثِّر ومجموعات قيم المعاملات الاختيارية كصِيَغ *كتلة أولية موحدة* (PUB).

```python
job = estimator.run([(isa_circuit, isa_observable, param_values)])
print(f">>> Job ID: {job.job_id()}")
print(f">>> Job Status: {job.status()}")
```

## ابدأ مع Sampler {#start-sampler}

### 1. تهيئة الحساب {#1-initialize-the-account}

بما أن Qiskit Runtime Sampler هو خدمة مُدارة، فأنت بحاجة أولًا إلى تهيئة حسابك. يمكنك بعد ذلك اختيار وحدة المعالجة الكمية (QPU) التي تريد استخدامها لحساب قيمة التوقع.

اتبع الخطوات في [موضوع التثبيت والإعداد](install-qiskit) إذا لم يكن لديك حساب مُعدّ بعد.

```python
from qiskit_ibm_runtime import QiskitRuntimeService

service = QiskitRuntimeService()
backend = service.least_busy(
    operational=True, simulator=False, min_num_qubits=127
)
```

### 2. إنشاء دائرة (Circuit) {#2-create-a-circuit}

تحتاج إلى دائرة واحدة على الأقل كمدخل لأولية Sampler.

```python
import numpy as np
from qiskit.circuit.library import efficient_su2

circuit = efficient_su2(127, entanglement="linear")
circuit.measure_all()
# The circuit is parametrized, so we will define the parameter values for execution
param_values = np.random.rand(circuit.num_parameters)
```

استخدم Transpiler للحصول على دائرة ISA.

```python
from qiskit.transpiler import generate_preset_pass_manager

pm = generate_preset_pass_manager(optimization_level=1, backend=backend)
isa_circuit = pm.run(circuit)
print(f">>> Circuit ops (ISA): {isa_circuit.count_ops()}")
```

```text
>>> Circuit ops (ISA): OrderedDict([('sx', 3089), ('rz', 3036), ('cz', 1092), ('measure', 127), ('barrier', 1)])
```

### 3. تهيئة Qiskit Runtime Sampler {#3-initialize-the-qiskit-runtime-sampler}

عند تهيئة Sampler، استخدم المعامل `mode` لتحديد الوضع الذي تريد تشغيله فيه. القيم الممكنة هي كائنات `batch` أو `session` أو `backend` لوضع التنفيذ الدُفعي والجلسي والوظيفي على التوالي. لمزيد من المعلومات، راجع [مقدمة إلى أوضاع تنفيذ Qiskit Runtime.](execution-modes) لاحظ أن مستخدمي الخطة المفتوحة (Open Plan) لا يمكنهم إرسال وظائف الجلسة.

```python
from qiskit_ibm_runtime import SamplerV2 as Sampler

sampler = Sampler(mode=backend)
```

### 4. استدعاء Sampler والحصول على النتائج {#4-invoke-the-sampler-and-get-results}

بعد ذلك، استدعِ الدالة `run()` لتوليد المخرجات. تُدخَل الدائرة ومجموعات قيم المعاملات الاختيارية كصِيَغ *كتلة أولية موحدة* (PUB).

```python
job = sampler.run([(isa_circuit, param_values)])
```

## ابدأ مع أوليات backend {#backend}

على عكس الأوليات الخاصة بمزوِّد معين، أوليات backend هي تطبيقات عامة يمكن استخدامها مع أي كائن
`backend` اعتباطي، طالما أنه يُنفِّذ واجهة [`Backend`](https://docs.quantum.ibm.com/api/qiskit/qiskit.providers.Backend).

- يمكن تشغيل أولية Sampler مع أي مزوِّد باستخدام [`qiskit.primitives.BackendSamplerV2`](https://docs.quantum.ibm.com/api/qiskit/qiskit.primitives.BackendSamplerV2).
- يمكن تشغيل أولية Estimator مع أي مزوِّد باستخدام [`qiskit.primitives.BackendEstimatorV2`](../api/qiskit/qiskit.primitives.BackendEstimatorV2).

بعض المزوِّدين يُطبِّقون الأوليات بصورة أصلية. راجع [صفحة نظام Qiskit البيئي](https://qiskit.github.io/ecosystem#provider) لمزيد من التفاصيل.
### مثال: BackendEstimator {#example-backendestimator}

```python
from qiskit.primitives import BackendEstimatorV2
from <some_qiskit_provider> import QiskitProvider

provider = QiskitProvider()
backend = provider.get_backend('backend_name')
estimator = BackendEstimatorV2(backend)
```

### مثال: BackendSampler {#example-backendsampler}

```python
from qiskit.primitives import BackendSamplerV2
from <some_qiskit_provider> import QiskitProvider

provider = QiskitProvider()
backend = provider.get_backend('backend_name')
sampler = BackendSamplerV2(backend)
```
### أوجه التشابه والاختلاف بين أوليات backend وأوليات Runtime {#similarities-and-differences-between-backend-and-runtime-primitives}

- تتبع المدخلات والمخرجات لكل من [`qiskit.primitives.BackendSamplerV2`](https://docs.quantum.ibm.com/api/qiskit/qiskit.primitives.BackendSamplerV2) و [`qiskit.primitives.BackendEstimatorV2`](../api/qiskit/qiskit.primitives.BackendEstimatorV2)
نفس تنسيق PUB الخاص بأوليات Qiskit Runtime. راجع [مدخلات ومخرجات الأوليات](primitive-input-output) لمزيد من التفاصيل.
غير أنه قد تكون هناك اختلافات في حقول البيانات الوصفية المُعادة.

- لا توفر فئة [`qiskit.primitives.BackendEstimatorV2`](https://docs.quantum.ibm.com/api/qiskit/qiskit.primitives.BackendEstimatorV2) أي تطبيقات لتخفيف أخطاء القياس أو البوابة جاهزةً للاستخدام، إذ إن أوليات backend مُصمَّمة للتشغيل محليًا على جهاز المستخدم.

- تتطلب فئة [`qiskit.primitives.BackendSamplerV2`](https://docs.quantum.ibm.com/api/qiskit/qiskit.primitives.BackendSamplerV2) backend يدعم خيار `memory`.

- تُظهر واجهات أولية backend خيارات `Options` مخصصة لـ [`SamplerV2`](https://docs.quantum.ibm.com/api/qiskit/qiskit.primitives.BackendSamplerV2) و [`EstimatorV2`](https://docs.quantum.ibm.com/api/qiskit/qiskit.primitives.BackendEstimatorV2) تختلف عن تطبيقات Runtime.

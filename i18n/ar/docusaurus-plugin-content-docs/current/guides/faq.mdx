## كيف أستشهد بحاسوب IBM Quantum في أبحاثي؟ {#how-do-i-cite-an-ibm-quantum-computer-in-my-research}

بالنسبة للأوراق البحثية، نشجع المؤلفين على الإقرار باستخدام IBM Quantum&reg; على النحو التالي:

>    نُقرّ باستخدام خدمات IBM Quantum في هذا العمل. الآراء الواردة هنا تعبّر عن وجهات نظر المؤلفين، ولا تعكس السياسة الرسمية لشركة IBM أو فريق IBM Quantum.

يجب أن تستخدم المراجع الببليوغرافية في الورقة الاستشهاد التالي:

> IBM Quantum. https://<span>quantum.cloud.ibm.com</span>/, YYYY

يجب الإشارة إلى وحدات المعالجة الكمية في الورقة باسمها الفريد (مثلاً `ibmq_vigo`) مع إمكانية إضافة رقم الإصدار اختياريًا (مثلاً `ibmq_vigo` v1.0.2). كما نشجع على الإشارة إلى المعالج. على سبيل المثال:

> في هذه الورقة استخدمنا `ibmq_vigo`، وهو أحد معالجات IBM Quantum Canary.

مثال على الاستشهاد ببرنامج IBM Quantum:

> IBM Quantum (2025). Estimator primitive (Version x.y.z) [computer software]. https://<span>quantum.cloud.ibm.com</span>/

## لا أرى شيئًا في واجهة المستخدم، مثل مهام الإدارة الخاصة بي. {#i-dont-see-something-in-the-user-interface-such-as-my-administration-tasks}

جرّب تحديد حساب مختلف من محوّل الحسابات في رأس واجهة [IBM Quantum Platform](https://quantum.cloud.ibm.com).

## أنا أستخدم الخطة المجانية فقط، لكنني اضطررت لإدخال بطاقة الائتمان. هل سيتم خصم أي مبالغ؟ {#im-just-using-the-open-plan-but-i-had-to-put-in-my-credit-card-will-i-be-charged}

لن يتم خصم أي مبالغ منك ما لم تنشئ [نسخة](/guides/instances) بنظام الدفع حسب الاستخدام وتستخدمها لإرسال أعباء العمل إلى وحدات QPU. راجع [إدارة التكاليف](/guides/manage-cost) للاطلاع على تعليمات تحديد حد للتكلفة ومعرفة طرق أخرى لإدارة النفقات.

## ما الفرق بين مفتاح API ورمز API؟ {#whats-the-difference-between-an-api-key-and-an-api-token}

لا يوجد فرق. يُستخدم مصطلحا مفتاح API ورمز API بالتبادل. ستجد كلا المصطلحين في الوثائق لأن الواجهات المختلفة تستخدم مصطلحات مختلفة.

<span id="bearer"></span>
## ما الفرق بين رمز API ورمز Bearer؟ {#whats-the-difference-between-an-api-token-and-a-bearer-token}

رمز API هو بيانات اعتماد طويلة الأمد تُستخدم للـ[مصادقة على خدمة Qiskit Runtime.](/guides/hello-world#install-and-authenticate) أما رمز Bearer فهو بيانات اعتماد مؤقتة تُستخدم للـ[مصادقة على REST API في بيئة غير موثوقة.](/guides/cloud-setup-untrusted#authenticate-to-the-rest-api-in-an-untrusted-environment) لا يمكن استخدام رمز Bearer للمصادقة على خدمة Qiskit Runtime.

## هل يجب أن أختار المنطقة التي أعيش فيها؟ {#do-i-have-to-choose-the-region-that-i-live-in}

لا. يمكنك استخدام أي من المنطقتين. تحدد المنطقة المكان الذي تُشغَّل فيه الوظائف وتُحفظ فيه البيانات. بالإضافة إلى ذلك، لا يمكنك رؤية واستخدام إلا النسخ التي أُنشئت في النسخة التي قمت بتسجيل الدخول إليها. لاحظ أن منطقة us-east فقط هي التي تدعم الخطة المجانية.

## هل يمكنني الاتصال عبر خادم وكيل (Proxy)؟ {#can-i-connect-through-a-proxy-server}

نعم، لكن يجب أن تستخدم Qiskit Runtime v0.44.0 أو إصدارًا أحدث.

## ما سياسة الاحتفاظ بالبيانات؟ {#what-is-the-data-retention-policy}

تحتفظ IBM ببياناتك (غير المحذوفة) المتعلقة بالوظائف والاستخدام لمدة ثلاث سنوات طالما تواصل استخدام خدمة Qiskit Runtime.

## ماذا حدث لتوزيعات Python مثل Qiskit Aer وQiskit Nature وQiskit Experiments؟ {#what-happened-to-python-distributions-such-as-qiskit-aer-qiskit-nature-and-qiskit-experiments}

منذ إصدار Qiskit SDK v1.0، سيقوم الأمر `pip install qiskit` بتثبيت [Qiskit SDK](https://github.com/Qiskit/qiskit) **فقط**. هذا يعني أن بعض الوحدات التي استخدمتها في الماضي، مثل `qiskit-aer` أو `qiskit-nature`، أصبح يجب تثبيتها بشكل منفصل.

قبل إصدار Qiskit SDK v1.0، كان Qiskit يتكون من العديد من توزيعات Python المختلفة، مثل `qiskit-terra` وهو نواة المترجم؛ و`qiskit-aer` وهو المحاكي عالي الأداء؛ ومزوّد IBM Quantum&reg; الأصلي؛ وعدة حزم أصبحت قديمة الآن كانت توفر ميزات خوارزمية استكشافية أو تجريبية.

جُمعت هذه الحزم البرمجية المنفصلة في ما كان يُعرف بـ Qiskit *metapackage*، مما أتاح للمستخدمين تثبيت جميع هذه المكونات معًا عبر `pip install qiskit`. تم إلغاء هذه البنية القائمة على الحزمة الموحدة مع إصدار Qiskit SDK v1.0، والآن يجب تثبيت كل توزيع برمجي بشكل منفصل.

إذا أردت تثبيت الحزم التي كانت مضمّنة سابقًا في Qiskit metapackage، فقم بزيارة [نظام Qiskit البيئي](https://www.ibm.com/quantum/ecosystem) للعثور على مجموعة من الحزم التي تناسب احتياجاتك. يمكنك أيضًا قراءة [دليل الانتقال إلى v1.0](/guides/metapackage-migration) للمزيد من المعلومات حول هذا التغيير.

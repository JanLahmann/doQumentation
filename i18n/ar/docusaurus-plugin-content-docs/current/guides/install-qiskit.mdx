# تثبيت Qiskit {#install-qiskit}

سواء كنت ستعمل محلياً أو في بيئة سحابية، فإن الخطوة الأولى لجميع المستخدمين هي تثبيت Qiskit.

<Admonition type="caution" title="الترقية من Qiskit 0.x إلى Qiskit 1.0 وما بعده">
  (إذا كنت تثبّت Qiskit للمرة الأولى، تخطَّ إلى قسم [التثبيت والإعداد](#local). هذا التنبيه مخصص فقط للمستخدمين الذين سبق لهم تثبيت Qiskit.)

  **للمستخدمين الذين يرقّون من الإصدار 0.x إلى 1.0 أو أحدث**: لاحظ أن Qiskit v1.0 يستخدم هيكل حزم جديداً، لذا **لا يمكنك** استخدام `pip install -U qiskit` للترقية من أي إصدار Qiskit 0.x إلى 1.0.

  راجع [دليل الترحيل إلى Qiskit 1.0](/guides/qiskit-1.0) للاطلاع على التفاصيل والتعليمات.

  ستتيح التحديثات المستقبلية بدءاً من Qiskit 1.0 إجراء الترقيات في مكانها مباشرةً.
</Admonition>

<span id="local"></span>
## تثبيت Qiskit SDK وعميل Qiskit Runtime {#install-the-qiskit-sdk-and-the-qiskit-runtime-client}

1. ثبّت Python. راجع قسم "Programming Language" في [صفحة مشروع Qiskit على PyPI](https://pypi.org/project/qiskit/) لتحديد إصدارات Python المدعومة من أحدث إصدار. لتعليمات التنزيل، راجع [دليل المبتدئين في Python.](https://wiki.python.org/moin/BeginnersGuide/Download)

    يُنصح باستخدام [البيئات الافتراضية في Python](https://docs.python.org/3.10/tutorial/venv.html) لعزل Qiskit عن التطبيقات الأخرى.

    <Admonition type="note">
    تستخدم هذه التعليمات توزيعة Python القياسية من [pypi.org](https://pypi.org/). ومع ذلك، يمكنك استخدام توزيعات Python أخرى مثل [Anaconda](https://docs.anaconda.com/anaconda/) أو [miniconda](https://docs.anaconda.com/miniconda/)، إلى جانب سير عمل إدارة الاعتمادات الأخرى مثل [Poetry](https://python-poetry.org/docs/).
    </Admonition>

    <details>
    <summary>
    إذا كنت جديداً على البيئات الافتراضية، انقر هنا لمزيد من المعلومات.
    </summary>
    البيئة الافتراضية في Python هي مساحة معزولة للعمل مع Python لغرض محدد — بحيث يمكنك تثبيت أي حزم تريدها وإعداد المكتبات والاعتمادات وما إلى ذلك، دون التأثير على بيئة Python "الأساسية" على جهازك.

    أحد المزايا المهمة للبيئة الافتراضية هو أنه إذا تعطّلت بيئة Python في مرحلة ما، يمكنك ببساطة حذف البيئة الافتراضية والبدء من جديد!

    اختر موقعاً مفضلاً لتخزين معلومات البيئات الافتراضية. عادةً ما تُخزَّن في مجلد باسم `.venv` داخل كل مجلد مشروع تعمل عليه.
    </details>

    أولاً، انتقل إلى مجلد مشروعك وأنشئ بيئة بسيطة تحتوي فقط على Python.

        <OperatingSystemTabs>
          <TabItem value="mac" label="macOS">
            ```shell
            python3 -m venv .venv
            ```
          </TabItem>

          <TabItem value="linux" label="Linux">
            ```shell
            python3 -m venv .venv
            ```
          </TabItem>

          <TabItem value="win" label="Windows">
            ```text
            python -m venv .venv
            ```
          </TabItem>
         </OperatingSystemTabs>

    بعد ذلك، فعّل بيئتك الجديدة.

         <OperatingSystemTabs>
           <TabItem value="mac" label="macOS">
             ```shell
             source .venv/bin/activate
             ```
           </TabItem>

           <TabItem value="linux" label="Linux">
             ```shell
             source .venv/bin/activate
             ```
           </TabItem>

           <TabItem value="win" label="Windows">
              إذا كنت تستخدم PowerShell:

             ```text
             .venv\Scripts\Activate.ps1
             ```
             إذا كنت تستخدم Git Bash:

             ```text
             source .venv/scripts/activate
             ```
              إذا كنت تستخدم موجه الأوامر:

             ```text
             .venv\Scripts\activate
             ```
           </TabItem>
         </OperatingSystemTabs>

1. [ثبّت pip](https://pip.pypa.io/en/stable/installation/) إذا لم يكن مثبتاً في بيئتك. pip هو مدير حزم Python الذي تستخدمه لتثبيت Qiskit وغيره من حزم Python. استخدم `pip list` لرؤية ما هو موجود في بيئتك الافتراضية. في معظم بيئات Python، يكون pip مثبتاً مسبقاً.

1. ثبّت Qiskit SDK. إذا كنت تخطط لتشغيل المهام على أجهزة الحوسبة الكمومية، ثبّت أيضاً Qiskit Runtime.

      ```shell
      pip install qiskit
      ```

      ```shell
      pip install qiskit-ibm-runtime
      ```

      <Admonition type="tip" title="إضافة دعم التصور البياني">
      إذا كنت تنوي استخدام وظائف التصور البياني أو دفاتر Jupyter، ثبّت أيضاً وحدة `qiskit.visualization`. راجع [مرجع واجهة برمجة التصور البياني](https://docs.quantum.ibm.com/api/qiskit/visualization) لمزيد من المعلومات.
      </Admonition>

<span id="local-jupyter"></span>
1. إذا أردت تشغيل دفتر Jupyter باستخدام حزم Qiskit التي ثبّتها للتو، تحتاج إلى تثبيت Jupyter في بيئتك.

      ```shell
      pip install jupyter
      ```
    ثم افتح دفترك على النحو التالي:

      ```shell
      jupyter notebook <path/to/notebook.ipynb>
      ```

إذا كنت تخطط للعمل محلياً واستخدام المحاكيات المدمجة في Qiskit، فقد انتهيت من التثبيت. إذا أردت تشغيل المهام على وحدات المعالجة الكمومية من IBM، تابع إلى [إعداد حساب IBM Cloud](cloud-setup).

<Admonition type="note" title="ابقَ على اطلاع بأحدث الإصدارات">
    تحقق دورياً من [ملاحظات إصدار Qiskit](../api/qiskit/release-notes) و[ملاحظات إصدار Qiskit Runtime](../api/qiskit-ibm-runtime/release-notes) للاطلاع على الإصدارات الجديدة. نوصي بتحديث متطلباتك لـ `qiskit` و`qiskit-ibm-runtime` بشكل متكرر، على سبيل المثال بتغيير الإصدارات في `requirements.txt` إلى أحدث الإصدارات، ثم تشغيل `pip install -r requirements.txt` أو الأمر المناسب لسير عمل إدارة الاعتمادات لديك.
</Admonition>

## استكشاف الأخطاء وإصلاحها {#troubleshooting}

<details>
  <summary>
    خطأ "No Module 'qiskit'" مع Jupyter Notebook
  </summary>
  إذا استخدمت ``pip install qiskit`` وأعددت بيئتك الافتراضية في
  Anaconda، فقد تظهر لك رسالة الخطأ ``No Module 'qiskit'`` عند تشغيل درس تعليمي
  في Jupyter Notebook. إذا لم تثبّت Qiskit بعد أو لم تُعِدّ بيئتك
  الافتراضية، يمكنك اتباع [خطوات التثبيت](#qiskit-install).

  يحدث هذا الخطأ عند محاولة استيراد حزمة Qiskit في بيئة
  لم يُثبَّت فيها Qiskit. إذا شغّلت Jupyter Notebook من
  Anaconda-Navigator، فمن المحتمل أن Jupyter Notebook يعمل
  في البيئة الأساسية (root)، بدلاً من بيئتك الافتراضية.
  اختر بيئة افتراضية في Anaconda-Navigator من
  قائمة **Applications on** المنسدلة. في هذه القائمة، يمكنك رؤية
  جميع البيئات الافتراضية داخل Anaconda، ويمكنك
  تحديد البيئة التي ثبّتّ فيها Qiskit لتشغيل Jupyter
  Notebook.
</details>

<details>
  <summary>
    أخطاء التصريف أثناء التثبيت
  </summary>
  يعتمد Qiskit على عدد من حزم Python مفتوحة المصدر الأخرى، والتي
  تُثبَّت تلقائياً عند تنفيذ ``pip install qiskit``. بناءً على
  منصة نظامك وإصدار Python، من المحتمل أن حزمة معينة
  لا توفر ملفات ثنائية مجمّعة مسبقاً لنظامك. يمكنك الرجوع إلى
  [دعم أنظمة التشغيل](#operating-system-support) للاطلاع على قائمة المنصات المدعومة من Qiskit، والتي
  قد تحتاج بعضها إلى مُصرِّف إضافي. في الحالات التي لا توجد فيها
  ملفات ثنائية مجمّعة مسبقاً، سيحاول ``pip`` تصريف الحزمة
  من المصدر، مما قد يستلزم بعض الاعتمادات الإضافية التي تحتاج إلى
  تثبيتها يدوياً.

  إذا كان ناتج ``pip install qiskit`` يحتوي على أسطر مشابهة لما يلي:

  ```
  Failed building wheel for SOME_PACKAGE
  ...
  build/temp.linux-x86_64-3.5/_openssl.c:498:30: fatal error
  compilation terminated.
  error: command 'x86_64-linux-gnu-gcc' failed with exit status 1
  ```

  يرجى مراجعة وثائق الحزمة التي فشل تثبيتها (في مثال الكود أعلاه، ``SOME_PACKAGE``) للاطلاع على كيفية تثبيت المكتبات
  اللازمة للتصريف من المصدر.
</details>

## دعم أنظمة التشغيل {#operating-system-support}

يسعى Qiskit إلى دعم أكبر عدد ممكن من أنظمة التشغيل، ولكن بسبب القيود المتعلقة بموارد الاختبار المتاحة وتوافر أنظمة التشغيل، لا يمكن دعم جميع الأنظمة. ينقسم دعم أنظمة التشغيل في Qiskit إلى ثلاث مستويات بمستويات مختلفة من الدعم لكل منها. بالنسبة للمنصات خارج هذه المستويات مثل FreeBSD أو WebAssembly (WASI)، قد يكون Qiskit قابلاً للتثبيت ولكنه غير مختبر وستحتاج إلى بناء Qiskit (وعلى الأرجح اعتمادات Qiskit) من المصدر.

علاوةً على ذلك، يدعم Qiskit تطبيق CPython فقط للغة Python. لا يُدعم التشغيل بمفسرات Python أخرى مثل PyPy.

### Qiskit v2.x

في سلسلة إصدارات Qiskit v2.x، المنصات المدعومة هي:

<details>
  <summary>
    المستوى الأول
  </summary>
  تُختبر أنظمة تشغيل المستوى الأول بالكامل كجزء من عمليات التطوير لضمان عمل أي تغيير مقترح بشكل صحيح. تُبنى الملفات الثنائية المجمّعة مسبقاً وتُختبر وتُنشر على PyPI كجزء من عملية الإصدار. عادةً، طالما توجد بيئة Python تعمل بشكل صحيح، يمكن تثبيت Qiskit على هذه الأنظمة دون الحاجة إلى تثبيت اعتمادات إضافية.

  أنظمة تشغيل المستوى الأول:

  - Linux x86_64 (التوزيعات المتوافقة مع مواصفات التعبئة [manylinux 2014](https://www.python.org/dev/peps/pep-0599/)).
  - macOS ARM64 (11.0 أو أحدث)
  - Windows 64-bit (يدعم Windows 10 وما بعده)
  - Linux AArch64 (التوزيعات المتوافقة مع مواصفات التعبئة [manylinux 2014](https://www.python.org/dev/peps/pep-0599/))
</details>

<details>
  <summary>
    المستوى الثاني
  </summary>
  لا تُختبر أنظمة تشغيل المستوى الثاني كجزء من عملية التطوير. ومع ذلك، تُبنى الملفات الثنائية المجمّعة مسبقاً وتُختبر وتُنشر على PyPI كجزء من عملية الإصدار، ويمكن توقع تثبيت هذه الحزم بوجود بيئة Python تعمل فقط.
  قد يكون هناك تأخير في إصدار الحزم لهذه الأنظمة، إذ قد لا تُكتشف أخطاء الاختبار إلا في وقت متأخر، ولن تعيق الأخطاء أثناء النشر إصدار الحزمة لأنظمة المستوى الأول.

  أنظمة تشغيل المستوى الثاني:

  - macOS x86_64 (10.12 أو أحدث)
</details>

<details>
  <summary>
    المستوى الثالث
  </summary>
  لا تُختبر أنظمة تشغيل المستوى الثالث كجزء من عملية التطوير. تُبنى الملفات الثنائية المجمّعة مسبقاً وتُنشر على PyPI كجزء من عملية الإصدار ولكنها غير مختبرة. قد لا تكون قابلة للتثبيت بوجود بيئة Python تعمل فقط وقد تستلزم مُصرِّف C/C++ أو برامج إضافية لبناء الاعتمادات من المصدر أثناء التثبيت. يقتصر الدعم لهذه الأنظمة على أفضل جهد ممكن فقط.

  أنظمة تشغيل المستوى الثالث:

  - Linux ppc64le (التوزيعات المتوافقة مع مواصفات التعبئة [manylinux 2014](https://www.python.org/dev/peps/pep-0599/))
  - Linux s390x (التوزيعات المتوافقة مع مواصفات التعبئة [manylinux 2014](https://www.python.org/dev/peps/pep-0599/))
</details>

بدءاً من Qiskit v2.0.0، تُدعم المنصات 64-bit فقط ولا تُدعم المنصات 32-bit. لن تتمكن أيضاً من البناء من المصدر على المنصات 32-bit، إذ يفترض كود Rust الداخلي في Qiskit عرض مؤشر 64-bit.

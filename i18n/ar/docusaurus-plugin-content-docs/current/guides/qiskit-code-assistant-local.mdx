# استخدام Qiskit Code Assistant في الوضع المحلي {#use-qiskit-code-assistant-in-local-mode}

تعلّم كيفية تثبيت وإعداد واستخدام أيٍّ من نماذج Qiskit Code Assistant على جهازك المحلي.

<Admonition type="note" title="ملاحظات">
    - Qiskit Code Assistant في مرحلة الإصدار التجريبي وقد يتغير.
    - إذا كان لديك ملاحظات أو أردت التواصل مع فريق التطوير، استخدم [قناة Qiskit على Slack](https://qiskit.enterprise.slack.com/archives/C07LYA6PL83) أو مستودعات GitHub العامة ذات الصلة.
</Admonition>

## البدء السريع (موصى به) {#quick-start-recommended}

أسهل طريقة للبدء مع Qiskit Code Assistant في الوضع المحلي هي استخدام سكريبتات الإعداد التلقائي الخاصة بإضافة VS Code أو JupyterLab. تقوم هذه السكريبتات تلقائيًا بتثبيت [Ollama](https://ollama.com/) لتشغيل نماذج LLM، وتنزيل النموذج الموصى به، وإعداد الإضافة نيابةً عنك.

### إعداد إضافة VS Code {#vs-code-extension-setup}

شغّل الأمر التالي في طرفيتك:

```bash
bash <(curl -fsSL https://raw.githubusercontent.com/Qiskit/qiskit-code-assistant-vscode/main/setup_local.sh)
```

يُنفّذ هذا السكريبت الخطوات التالية:
- تثبيت Ollama (إذا لم يكن مثبّتًا مسبقًا)
- تنزيل نموذج Qiskit Code Assistant الموصى به وإعداده
- تهيئة إضافة VS Code للعمل مع نشرك المحلي

### إعداد إضافة JupyterLab {#jupyterlab-extension-setup}

شغّل الأمر التالي في طرفيتك:

```bash
bash <(curl -fsSL https://raw.githubusercontent.com/Qiskit/qiskit-code-assistant-jupyterlab/main/setup_local.sh)
```

سيقوم هذا السكريبت بـ:
- تثبيت Ollama (إذا لم يكن مثبّتًا مسبقًا)
- تنزيل نموذج Qiskit Code Assistant الموصى به وإعداده
- تهيئة إضافة JupyterLab للعمل مع نشرك المحلي

هناك عدة طرق لنشر نموذج Qiskit Code Assistant الذي نزّلته والتفاعل معه. يوضح هذا الدليل استخدام [Ollama](https://ollama.com) على النحو التالي: إما مع [تطبيق Ollama](#using-the-ollama-application) عبر تكامل Hugging Face Hub أو نموذج محلي، أو مع [حزمة `llama-cpp-python`](#use-llama-cpp-python).

### استخدام تطبيق Ollama {#using-the-ollama-application}

يوفر تطبيق Ollama حلًا بسيطًا لتشغيل نماذج LLM محليًا. سهل الاستخدام، مع واجهة CLI تجعل عملية الإعداد بأكملها وإدارة النماذج والتفاعل معها أمرًا مباشرًا. مثالي للتجريب السريع وللمستخدمين الذين يريدون تفاصيل تقنية أقل.

#### تثبيت Ollama {#install-ollama}

1. نزّل [تطبيق Ollama](https://ollama.com/download)
1. ثبّت الملف الذي تم تنزيله
1. شغّل تطبيق Ollama المثبّت

    <Admonition type="info">يعمل التطبيق بنجاح عندما تظهر أيقونة Ollama في شريط القوائم على سطح المكتب. يمكنك أيضًا التحقق من أن الخدمة تعمل بالانتقال إلى `http://localhost:11434/`.</Admonition>

1. جرّب Ollama في طرفيتك وابدأ تشغيل النماذج. مثلًا:

    ```
    ollama run hf.co/Qiskit/Qwen2.5-Coder-14B-Qiskit
    ```

#### إعداد Ollama باستخدام تكامل Hugging Face Hub {#set-up-ollama-using-the-hugging-face-hub-integration}

يوفر [تكامل Ollama/Hugging Face Hub](https://huggingface.co/docs/hub/ollama) طريقة للتفاعل مع النماذج المستضافة على Hugging Face Hub دون الحاجة إلى إنشاء modelfile جديد أو تنزيل ملفات GGUF أو safetensors يدويًا. ملفات `template` و`params` الافتراضية مُضمَّنة مسبقًا للنموذج على Hugging Face Hub.

1. تأكد من أن تطبيق Ollama يعمل.

1. اذهب إلى صفحة النموذج المطلوب وانسخ الرابط. مثلًا: https://huggingface.co/Qiskit/Qwen2.5-Coder-14B-Qiskit-GGUF.

1. من طرفيتك، شغّل الأمر:

    ```
    ollama run hf.co/Qiskit/Qwen2.5-Coder-14B-Qiskit
    ```

يمكنك استخدام نموذج `hf.co/Qiskit/Qwen2.5-Coder-14B-Qiskit` أو أي من النماذج الرسمية GGUF الموصى بها حاليًا `hf.co/Qiskit/mistral-small-3.2-24b-qiskit-GGUF` أو `hf.co/Qiskit/granite-3.3-8b-qiskit-GGUF`.

#### إعداد Ollama مع نموذج GGUF لـ Qiskit Code Assistant تم تنزيله يدويًا {#set-up-ollama-with-a-manually-downloaded-qiskit-code-assistant-gguf-model}

إذا قمت بتنزيل نموذج GGUF يدويًا مثل https://huggingface.co/Qiskit/Qwen2.5-Coder-14B-Qiskit-GGUF وأردت تجربة templates وإعدادات مختلفة، يمكنك اتباع هذه الخطوات لتحميله في تطبيق Ollama المحلي.

1. أنشئ `Modelfile` بالمحتوى التالي وتأكد من تحديث `<PATH-TO-GGUF-FILE>` بالمسار الفعلي للنموذج الذي نزّلته.

    ```
    FROM <PATH-TO-GGUF-FILE>
    TEMPLATE """{{ if .System }}
    System:
    {{ .System }}

    {{ end }}{{ if .Prompt }}Question:
    {{ .Prompt }}

    {{ end }}Answer:
    ```python{{ .Response }}
    """

    PARAMETER stop "Question:"
    PARAMETER stop "Answer:"
    PARAMETER stop "System:"
    PARAMETER stop "```"

    PARAMETER temperature 0
    PARAMETER top_k 1
    ```

1. شغّل الأمر التالي لإنشاء مثيل نموذج مخصص بناءً على `Modelfile`.

    ```
    ollama create Qwen2.5-Coder-14B-Qiskit -f ./path-to-model-file
    ```

    <Admonition type="note">قد تستغرق هذه العملية بعض الوقت حتى يقرأ Ollama ملف النموذج ويُهيّئ مثيل النموذج ويضبطه وفقاً للمواصفات المحددة.</Admonition>

#### تشغيل نموذج Qiskit Code Assistant المُحمَّل يدوياً في Ollama {#run-the-qiskit-code-assistant-model-manually-downloaded-in-ollama}

بعد إعداد نموذج `Qwen2.5-Coder-14B-Qiskit` في Ollama، شغّل الأمر التالي لتشغيل النموذج والتفاعل معه في الطرفية (في وضع المحادثة).

```
ollama run Qwen2.5-Coder-14B-Qiskit
```

بعض الأوامر المفيدة:

- `ollama list` - عرض قائمة النماذج على جهازك
- `ollama rm Qwen2.5-Coder-14B-Qiskit` - حذف النموذج
- `ollama show Qwen2.5-Coder-14B-Qiskit` - عرض معلومات النموذج
- `ollama stop Qwen2.5-Coder-14B-Qiskit` - إيقاف نموذج قيد التشغيل حالياً
- `ollama ps` - عرض النماذج المحمّلة حالياً

<summary>نشر نماذج Qiskit Code Assistant محلياً عبر حزمة `llama-cpp-python`</summary>

<span id="use-llama-cpp-python"></span>
بديل عن تطبيق Ollama هو حزمة `llama-cpp-python`، وهي واجهة ربط Python لمكتبة `llama.cpp`. توفّر لك مزيداً من التحكم والمرونة لتشغيل نموذج GGUF محلياً، وهي مثالية للمستخدمين الراغبين في دمج النموذج المحلي في سير عملهم وتطبيقات Python الخاصة بهم.

1. ثبّت [`llama-cpp-python`](https://pypi.org/project/llama-cpp-python/)
2. تفاعل مع النموذج من داخل تطبيقك باستخدام `llama_cpp`. على سبيل المثال:

```python
from llama_cpp import Llama

model_path = <PATH-TO-GGUF-FILE>

model = Llama(
        model_path,
        seed=17,
        n_ctx=10000,
        n_gpu_layers=37, # to offload in gpu, but put 0 if all in cpu
    )

input = 'Generate a quantum circuit with 2 qubits'
raw_pred = model(input)["choices"][0]["text"]
```

يمكنك أيضاً إضافة معاملات توليد النص إلى النموذج لتخصيص الاستدلال:

```python
generation_kwargs = {
        "max_tokens": 512,
        "echo": False, # Echo the prompt in the output
        "top_k": 1
    }

raw_pred = model(input, **generation_kwargs)["choices"][0]["text"]
```

### استخدام مكتبة `llama.cpp` {#use-the-llama-cpp-library}

بديل آخر هو استخدام `llama.cpp`، وهي مكتبة مفتوحة المصدر لتنفيذ استدلال نماذج اللغة الكبيرة على المعالج المركزي بأقل قدر من الإعداد.
توفّر تحكماً منخفض المستوى في تنفيذ النموذج وتُشغَّل عادةً من سطر الأوامر مع الإشارة إلى ملف نموذج GGUF محلي.

هناك عدة طرق لتثبيت `llama.cpp` على جهازك:

- تثبيت llama.cpp باستخدام [brew أو nix أو winget](https://github.com/ggml-org/llama.cpp/blob/master/docs/install.md)
- التشغيل عبر Docker: راجع [توثيق Docker من فريق `llama.cpp`](https://github.com/ggml-org/llama.cpp/blob/master/docs/docker.md)
- تنزيل الملفات التنفيذية الجاهزة من [صفحة الإصدارات](https://github.com/ggml-org/llama.cpp/releases)
- البناء من المصدر عبر [استنساخ هذا المستودع](https://github.com/ggml-org/llama.cpp/blob/master/docs/build.md)

بعد التثبيت، يمكنك استخدام `llama.cpp` للتفاعل مع نماذج GGUF في وضع المحادثة كالتالي:
```bash
# Use a local model file
llama-cli -m my_model.gguf -cnv

# Or download and run a model directly from Hugging Face
llama-cli -hf Qiskit/Qwen2.5-Coder-14B-Qiskit-GGUF -cnv
```

يمكنك أيضاً تشغيل خادم API متوافق مع OpenAI للنموذج بالطريقة التالية:
```bash
llama-server -hf Qiskit/Qwen2.5-Coder-14B-Qiskit-GGUF
```

#### معاملات متقدمة {#advanced-parameters}
باستخدام برنامج `llama-cli`، يمكنك التحكم في توليد النموذج عبر خيارات سطر الأوامر. على سبيل المثال، يمكنك تقديم prompt نظام أولي باستخدام العلم `-p/--prompt`. في وضع المحادثة (`-cnv`)، يعمل هذا الـ prompt الأولي كرسالة النظام. وإلا يمكنك ببساطة إضافة أي تعليمات مطلوبة في بداية نص الـ prompt. يمكنك أيضاً ضبط معاملات أخذ العينات - مثلاً: درجة الحرارة (`--temp`)، وtop-k (`--top-k`)، وtop-p (`--top-p`)، وعقوبة التكرار (`--repeat-penalty`)، والبذرة (`--seed`). فيما يلي مثال على الاستدعاء باستخدام هذه الخيارات:

```bash
llama-cli -hf Qiskit/Qwen2.5-Coder-14B-Qiskit-GGUF \
  -p "You are a friendly assistant." -cnv \
  --temp 0.7 \
  --top-k 50 \
  --top-p 0.95 \
  --repeat-penalty 1.1 \
  --seed 42
```

لضمان الأداء الصحيح لنماذج Qiskit لدينا، نوصي باستخدام prompt النظام المقدَّم في مستودعات HF GGUF الخاصة بنا: prompt النظام لـ [mistral-small-3.2-24b-qiskit-GGUF](https://huggingface.co/Qiskit/mistral-small-3.2-24b-qiskit-GGUF/blob/main/system)، و[Qwen2.5-Coder-14B-Qiskit-GGUF](https://huggingface.co/Qiskit/Qwen2.5-Coder-14B-Qiskit-GGUF/blob/main/system)، و[granite-3.3-8b-qiskit-GGUF](https://huggingface.co/Qiskit/granite-3.3-8b-qiskit-GGUF/blob/main/system)، و[granite-3.2-8b-qiskit-GGUF.](https://huggingface.co/Qiskit/granite-3.2-8b-qiskit-GGUF/blob/main/system)

### الاتصال بإضافة Qiskit Code Assistant لـ VS Code {#connect-with-the-qiskit-code-assistant-vs-code-extension}

مع إضافة Qiskit Code Assistant لـ VS Code، يمكنك التفاعل مع النموذج وإجراء إكمال الكود أثناء كتابته. يمكن أن يكون هذا مفيدًا جدًا للمستخدمين الذين يبحثون عن مساعدة في كتابة كود Qiskit لتطبيقات Python الخاصة بهم.

1. ثبّت [إضافة Qiskit Code Assistant لـ VS Code](/guides/qiskit-code-assistant-vscode).
1. في VS Code، اذهب إلى **User Settings** واضبط **Qiskit Code Assistant: Url** على رابط نشر Ollama المحلي لديك (مثلًا: `http://localhost:11434`).

# إنشاء الـ instances وإدارتها {#create-and-manage-instances}

الوصول إلى خدمات IBM Quantum Platform يتحكم فيه **الـ instances** التي تنتمي إليها. بإمكانك الانتماء إلى instances متعددة.

الـ instances هي خوادم افتراضية (نشرات من Qiskit Runtime) تُدير تنفيذ أحمال عملك، بما في ذلك تشغيل البرامج الكمومية ومهام الحوسبة الكلاسيكية (مثل معالجة تخفيف الأخطاء). يُعرَّف كل instance بـ *Cloud Resource Name* (CRN) الخاص به، ويمكن استخدام هذا الـ CRN عند إرسال أحمال العمل من برامج Qiskit.

يمكنك الاطلاع على الـ instances التي تملك صلاحية الوصول إليها من لوحة التحكم، أو بالضغط على تبويب "Instances" في لوحة التحكم. كل instance مُدرج مع معرّف CRN الخاص به.

تحتاج إلى instance مختلف لكل خطة خدمة (مثل Open أو Pay-As-You-Go) في كل منطقة تريد استخدامها. يستطيع مدير الحساب وأي شخص يملك صلاحيات كافية إنشاء instance وتحديد إعداداته، مثل التخصيصات وحدود الاستخدام وأجهزة الحوسبة الكمومية (QPUs) وأذونات المستخدمين.

تعرض [لوحة تحكم](https://quantum.cloud.ibm.com/) IBM Quantum Platform الـ instances التي تملك صلاحية الوصول إليها. بحسب صلاحياتك، يمكنك عرض التفاصيل الكاملة وتعديل وحذف الـ instances من [صفحة Instances](https://quantum.cloud.ibm.com/instances).

من المهم أن تعرف أي instance تستخدم، لأن جميع الـ instances غير المرتبطة بالخطة المجانية (Open Plan) تُكبّدك تكلفة (أو تُحتسب من الوقت المخصص لخطتك).

<Admonition type="note">
المهام التي تُشغّلها أنت والمهام التي يُشغّلها متعاونون آخرون في نفس الـ instance تُحتسب ضمن الاستخدام المُبلَّغ عنه لذلك الـ instance.

يمكنك الاطلاع على استخدام instance معين من صفحة [Instances](https://quantum.cloud.ibm.com/instances)، أو، لمن يملك الصلاحية المناسبة، من صفحة [Analytics](https://quantum.cloud.ibm.com/analytics). لاحظ أن الصفحتين قد تعرضان أرقام استخدام مختلفة، إذ تعرض صفحة Instance إجمالي الاستخدام قبل الطابع الزمني الحالي، بينما تستخدم صفحة Analytics حدود يوم كامل.
</Admonition>

<span id="open-plan"></span>
## الـ instance الخاص بالخطة المجانية (Open Plan) {#open-plan}

يستطيع مستخدمو الخطة المجانية تشغيل الدوائر الكمومية على أفضل QPUs في العالم مجاناً (حتى 10 دقائق خلال نافذة متجددة مدتها 28 يوماً). يمكن تتبع استخدام QPU على [لوحة تحكم المنصة](https://quantum.cloud.ibm.com/) وصفحة [Workloads](https://quantum.cloud.ibm.com/workloads).

<span id="standard-plan"></span>
## الـ instance الخاص بخطة الدفع حسب الاستخدام (Pay-As-You-Go) {#standard-plan}

خطة Pay-As-You-Go هي خطة مدفوعة. لاستخدامها، يجب إنشاء instance وربطه بهذه الخطة. جميع أحمال العمل المُرسَلة إلى QPUs عبر هذه الخطة تُحتسب عليها رسوم. يمكن لـ instances خطة Pay-As-You-Go أن يكون لها حد إجمالي للتكلفة (اختياري).

<span id="flex-plan"></span>
## الـ instance الخاص بخطة Flex {#flex-plan}

خطة Flex هي خطة مدفوعة. لاستخدامها، يجب أولاً [التواصل مع IBM](https://www.ibm.com/account/reg/us-en/signup?formid=MAIL-quantum) للتسجيل. ثم أنشئ instance وربطه بهذه الخطة. جميع أحمال العمل المُرسَلة إلى QPUs عبر هذه الخطة تُحتسب من اشتراكك المخصص.

يستطيع المسؤول تعيين حد استخدام على instance خطة Flex يسري طوال مدة العقد. إذا صنّف مسؤولك الـ instance على أنه "محدود" (ستجد عمود "Remaining" في منطقة الاستخدام) وتجاوز الـ instance تخصيصه (المحدد من قِبَل مسؤولك)، سيستمر أي حمل عمل نشط في التشغيل (بما في ذلك الجلسات)، لكن أحمال العمل المعلّقة ستبقى في قائمة الانتظار حتى يتوفر المزيد من الوقت. أما إذا لم يكن الـ instance محدوداً وتجاوز تخصيصه، فمن المرجح أن المهام التي تعمل معه ستحظى بأولوية أدنى وستواجه أوقات انتظار أطول في قائمة الانتظار.

يظهر تنبيه في منطقة استخدام الـ instance عند تجاوزه لتخصيصه.

<span id="premium-plan"></span>
## الـ instance الخاص بالخطة المميزة (Premium Plan) {#premium-plan}

الخطة المميزة هي خطة مدفوعة. لاستخدامها، يجب أولاً [التواصل مع IBM](https://www.ibm.com/account/reg/us-en/signup?formid=MAIL-quantum) للتسجيل. ثم أنشئ instance وربطه بهذه الخطة. جميع أحمال العمل المُرسَلة إلى QPUs عبر هذه الخطة تُحتسب من اشتراكك المخصص.

يستطيع المسؤول تعيين حد استخدام على instance الخطة المميزة يسري خلال نافذة الاستخدام المتجددة مدتها 28 يوماً. إذا صنّف مسؤولك الـ instance على أنه "محدود" (ستجد عمود "Remaining" في منطقة الاستخدام) وتجاوز الـ instance تخصيصه (المحدد من قِبَل مسؤولك)، سيستمر أي حمل عمل نشط في التشغيل (بما في ذلك الجلسات)، لكن أحمال العمل المعلّقة ستبقى في قائمة الانتظار حتى يتوفر المزيد من الوقت. أما إذا لم يكن الـ instance محدوداً وتجاوز تخصيصه، فمن المرجح أن المهام التي تعمل معه ستحظى بأولوية أدنى وستواجه أوقات انتظار أطول في قائمة الانتظار.

يظهر تنبيه في منطقة استخدام الـ instance عند تجاوزه لتخصيصه.

<span id="connect-instance"></span>
## تحديد instance في كودك {#connect-instance}

يمكنك استخدام الـ CRN أو اسم الـ instance لتحديد الـ instance الذي تريد استخدامه — سواء عند تهيئة الخدمة أو عند إرسال حمل العمل إلى QPU (مثلاً عبر [`QiskitRuntimeService.backend()](https://docs.quantum.ibm.com/api/qiskit-ibm-runtime/qiskit-runtime-service#backend)). للاطلاع على تعليمات تحديد الـ instance في كودك، راجع [تهيئة حسابك](/guides/initialize-account).

عند تمرير instance محدد، يصبح متاحاً فقط الـ backends والمهام التابعة لذلك الـ instance (مسموح به في Qiskit Runtime الإصدار v0.40.1 وما بعده).

إذا لم يُحدَّد instance عند تهيئة الخدمة، تكون جميع الـ backends والمهام عبر جميع الـ instances في حسابك متاحة. في هذه الحالة، عند تحديد backend معين، يُعثر على instance يستطيع الوصول إلى ذلك الـ backend ويُستخدم كما هو موضح في [الاختيار التلقائي للـ instance](/guides/initialize-account#auto-select).

<Admonition type="important" title="الإصدارات المطلوبة">
يُشترط وجود Qiskit Runtime الإصدار v0.40.1 أو أحدث للتالي:

- استخدام اسم الـ instance لتحديده
- عدم تمرير instance
- تعيين نوع instance مفضّل ومنطقة مفضّلة

بالنسبة لـ Qiskit Runtime الإصدار v0.42 أو أحدث، تحظى الخطط المجانية بالأولوية تلقائياً عند استخدام [الاختيار التلقائي للـ instance](/guides/initialize-account#auto-select).
</Admonition>

<span id="add-instance"></span>
## إنشاء instance {#add-instance}

اتبع هذه الخطوات لإنشاء instance وإضافته إلى حسابك. إذا كنت مسؤولاً تحتاج إلى إعداد Qiskit Runtime على Cloud لمؤسستك، فارجع إلى [التخطيط لـ Qiskit Runtime في بيئة المؤسسات](/guides/considerations-set-up-runtime).

<span id="switcher"></span>

1. لإضافة instance إلى حسابك، تأكد أولاً من تحديد الحساب والمنطقة الصحيحَين في محوّل الحساب الموجود في الرأس.
        * يمكنك أن تمتلك (أو تكون جزءاً من) حسابات IBM Cloud متعددة. إذا لم تجد الـ instances أو المهام التي تتوقعها، جرّب حساباً مختلفاً.
        * يمكنك الوصول إلى أي من المنطقتَين باستخدام نفس مفتاح API، لكنك تستطيع فقط رؤية الـ instances التي أُنشئت في المنطقة التي أنت مسجّل الدخول إليها.
        * إذا أردت إنشاء Open instance، يجب عليك اختيار منطقة us-east.

        ![The IBM Quantum Platform header is shown.  The account switcher is immediately to the right of the search bar.  The region switcher is immediately to the right of the account switcher.](/docs/images/guides/cloud-setup/IQP-Header.svg 'IBM Quantum Platform header')

   <span id="regions"></span>

   <Admonition type="note" title="حول المناطق">
   سيُنشأ الـ instance في المنطقة المحددة. تحدد هذه المنطقة الخصائص المهمة التالية:
     * QPUs المتاحة.
     * أين تجري الحوسبة الكلاسيكية للمهام، مثل التجميع (compilation).
     * أين تبقى بيانات سير عمل المستخدم.

    تشمل بيانات سير العمل الدوائر المُدخَلة ومعاملات الدوائر ونتائج الحوسبة الكمومية. ولا تشمل البيانات الوصفية مثل عدد اللقطات (shots) أو حجم الدوائر أو إحصائيات المهام الأخرى.

   لتجنب كشف بيانات سير عمل المستخدم أمام طبقة حماية موزعة من هجمات الحرمان من الخدمة (DDoS) على مستوى عالمي، يمكنك استخدام [Virtual Private Endpoints](https://cloud.ibm.com/docs/overview?topic=overview-endpoints-support). يدعم `qiskit-ibm-runtime` هذا من خلال [المعامل `private_endpoint`](https://docs.quantum.ibm.com/api/qiskit-ibm-runtime/qiskit-runtime-service).
   </Admonition>

1. من لوحة مهام التحكم، في قسم Instances، اضغط على **View all**، أو انتقل إلى صفحة [Instances](https://quantum.cloud.ibm.com/instances) من القائمة الرئيسية.
1. من صفحة Instances، اضغط على **Create instance**.
    * أدخل اسماً وأضف وسوماً (tags) اختيارياً.
    * إذا كان بإمكانك الوصول إلى أكثر من مجموعة موارد (resource group) في IBM Cloud&reg;، يمكنك تغيير مجموعة الموارد التي ينتمي إليها هذا الـ instance. راجع [Managing resource groups](https://cloud.ibm.com/docs/account?topic=account-rgs&interface=ui) و[Giving access to resources in resource groups](https://cloud.ibm.com/docs/account?topic=account-rgs_manage_access) للمزيد من المعلومات.
1. اختر خطة التسعير التي سيرتبط بها هذا الـ instance (مثل Pay-As-You-Go). بحسب خطة التسعير المختارة، يمكنك تحديد خيارات أخرى مثل حد التكلفة أو عدد الدقائق المخصصة لهذا الـ instance. لاحظ أنه إذا لم يكن هذا الـ instance مرتبطاً بخطة مجانية، فستُحتسب تكلفة عند استخدامه لتشغيل أحمال العمل.
1. اضغط على **Next**. ستظهر قائمة بالـ QPUs التي يمكن للـ instance الوصول إليها.
    - إذا أردت للـ instance أن يصل إلى جميع الـ QPUs المدرجة (وجميع الـ QPUs التي ستُضاف إلى هذه الخطة مستقبلاً)، اضغط على **Next**.
    - إذا لم تكن تُعدّ open instance، يمكنك تخصيص الـ QPUs التي يمكن الوصول إليها عبر هذا الـ instance. اضغط على مفتاح "Customize allocated compute resources"، وحدد الـ QPUs المعينة التي يمكن لهذا الـ instance الوصول إليها، ثم اضغط **Next**.
        <Admonition type="note">
      إذا خصّصت موارد الحوسبة المخصصة، لن تُحدَّث الـ QPUs المتاحة تلقائياً أبداً، بصرف النظر عن أي تغييرات تُجرى على الخطة الأصلية. لكن يمكنك إضافة أو إزالة الـ QPUs يدوياً لاحقاً. أما إذا لم تقم بتخصيصها، فستظل دائماً تملك صلاحية الوصول إلى أي QPU يملك الحساب صلاحية الوصول إليه في الخطة. لذا إذا أُضيف QPU جديد إلى الخطة مستقبلاً، سيحصل الـ instance عليه تلقائياً.
        </Admonition>
1. يفتح قسم Access group. يُنشأ تلقائياً مجموعة وصول (access group) باسم "collaborators" لهذا الـ instance. المستخدمون المضافون إلى مجموعة الوصول هذه يستطيعون استخدام الوقت المخصص لهذا الـ instance. يمكنك [تعديل أو إنشاء مجموعات وصول إضافية](/guides/access-groups) باستخدام [IBM Cloud console](https://cloud.ibm.com/iam/groups). راجع [Setting up access groups](https://cloud.ibm.com/docs/account?topic=account-groups&interface=ui) للتعليمات.
1. اضغط على **Create instance**. يمكنك الآن عرض الـ instance في صفحة [Instances](https://quantum.cloud.ibm.com/instances). إذا قمت بإعداد خطط متعددة في حسابك، فكل نوع خطة له تبويبه الخاص في جدول Instances.

<Admonition type="note" title="أحمال العمل المعلّقة">
يعرض عمود أحمال العمل المعلّقة (Pending workloads) في جدول Instances عدد أحمال العمل المنتظرة حالياً في هذا الـ instance عبر جميع الـ QPUs.
</Admonition>

## تعديل instance وتخصيصه {#edit-an-instance-and-its-allocation}

ابحث عن الـ instance في صفحة [Instances](https://quantum.cloud.ibm.com/instances)، ثم اضغط على قائمة الفائض (overflow menu) في نهاية صف الـ instance واختر Edit details.

<Admonition type="note">
فقط المسؤولون والمالكون يستطيعون تعديل الـ instance. راجع دليل [إنشاء سياسات الوصول ومجموعات الوصول](/guides/access-groups) للاطلاع على معلومات حول تعيين الصلاحيات.
</Admonition>

## الـ instances المؤرشفة {#archived-instances}

الـ instance المؤرشف هو نتيجة أحد السيناريوهين التاليين.

### قام مستخدم ذو صلاحيات عالية بأرشفة الحساب {#a-privileged-user-has-archived-the-account}

قام مستخدم يمتلك صلاحيات كافية للحساب بأرشفة الـ instance.
    - تُلغى أي مهام معلّقة في الـ instance.
    - تظهر الـ instances المؤرشفة في قسم "Archive" منفصل في صفحة Instances.
    - لا يزال بالإمكان استرداد المهام.
    - لا يستطيع المستخدمون تقديم مهام جديدة إلى الـ instance المؤرشف.
    - لا يُحتسب الـ instance المؤرشف ضمن تخصيص خطتك.
    - يستطيع مستخدم ذو صلاحيات كافية إعادة تفعيل instance قام هو (أو مستخدم آخر ذو صلاحيات) بأرشفته في أي وقت.

لأرشفة instance تديره، انتقل إلى صفحة [Instances](https://quantum.cloud.ibm.com/instances). في نهاية صف ذلك الـ instance، اضغط على قائمة الفائض (ثلاث نقاط) على اليمين، ثم اختر *Archive instance*.

### لم يعد الحساب يملك صلاحية الوصول إلى خطة الـ instance {#account-no-longer-has-access-to-the-instances-plan}

لم يعد الحساب يملك صلاحية الوصول إلى الخطة التي أُنشئ تحتها الـ instance.

- تظهر الـ instances المؤرشفة في قسم "Archive" منفصل في صفحة Instances.
- لا يزال بالإمكان استرداد المهام.
- لا يستطيع المستخدمون تقديم مهام جديدة إلى الـ instance المؤرشف.
- لا يُحتسب الـ instance المؤرشف ضمن تخصيص خطتك.
- لا يمكن إعادة تفعيل الـ instances التي أُرشفت لأنها لم تعد تملك رؤية لخطة الـ instance.

# أحداث تتبع النشاط لخدمة IBM Quantum System {#activity-tracking-events-for-the-ibm-quantum-system-service}

تُولِّد خدمات IBM Cloud&reg;، مثل خدمة IBM&reg; Quantum System، أحداثَ تتبع النشاط.

تُبلِّغ أحداث تتبع النشاط عن الأنشطة التي تُغيِّر حالة خدمة ما في IBM Cloud. يمكنك استخدام هذه الأحداث للتحقيق في الأنشطة غير الطبيعية والإجراءات الحرجة، والامتثال لمتطلبات التدقيق التنظيمي.

يمكنك استخدام IBM Cloud Activity Tracker Event Routing، وهي خدمة منصة، لتوجيه أحداث التدقيق في حسابك إلى الوجهات التي تختارها عبر تهيئة الأهداف والمسارات التي تُحدِّد إلى أين تُرسَل أحداث تتبع النشاط. لمزيد من المعلومات، راجع [عن IBM Cloud Activity Tracker Events Routing](https://cloud.ibm.com/docs/atracker?topic=atracker-about).

يمكنك استخدام IBM Cloud Logs لتصوير الأحداث وإصدار تنبيهات بشأنها، وهي الأحداث المُولَّدة في حسابك والمُوجَّهة بواسطة IBM Cloud Activity Tracker Event Routing إلى نسخة IBM Cloud Logs.

## عرض أحداث تتبع النشاط لخدمة IBM Quantum System {#viewing-activity-tracking-events-for-the-ibm-quantum-system-service}

يمكنك استخدام IBM Cloud Logs لتصوير الأحداث وإصدار تنبيهات بشأنها، وهي الأحداث المُولَّدة في حسابك والمُوجَّهة بواسطة IBM Cloud Activity Tracker Event Routing إلى نسخة IBM Cloud Logs.

### تشغيل IBM Cloud Logs من صفحة الرصد {#launching-ibm-cloud-logs-from-the-observability-page}

للحصول على معلومات حول تشغيل واجهة مستخدم IBM Cloud Logs، راجع [تشغيل واجهة المستخدم](https://cloud.ibm.com/docs/cloud-logs?topic=cloud-logs-instance-launch#instance-launch-cloud-ui) في وثائق IBM Cloud Logs.

| الإجراء | مُشغَّل بواسطة |
| -- | -- |
| `quantum-computing.direct-access-backend-configuration.read` | الحصول على تهيئة Backend (`GET /v1/backends/{backend-name}/configuration`) |
| `quantum-computing.direct-access-backend-properties.read` | الحصول على خصائص Backend (`GET /v1/backends/{backend-name}/properties`) |
| `quantum-computing.direct-access-backend.list` | الحصول على قائمة Backends (`GET /v1/backends`) |
| `quantum-computing.direct-access-backend.read` | الحصول على تفاصيل Backend (`GET /v1/backends/{backend-name}`) |
| `quantum-computing.direct-access-job.cancel` | إلغاء مهمة (`POST /v1/jobs/{job-id}/cancel`) |
| `quantum-computing.direct-access-job.create` | تشغيل مهمة (`POST /v1/jobs`) |
| `quantum-computing.direct-access-job.delete` | حذف مهمة (`DELETE /v1/jobs/{job-id}`) |
| `quantum-computing.direct-access-job.list` | الحصول على المهام (`GET /v1/jobs`) |
| `quantum-computing.direct-access-job.read` | الحصول على مهمة بمعرِّفها (`GET /v1/jobs/{job-id}`) |
| `quantum-computing.direct-access-lane-configuration.list` | الحصول على تهيئة مسارات Backend (`GET /v1/backends/{backend-name}/lanes`) |

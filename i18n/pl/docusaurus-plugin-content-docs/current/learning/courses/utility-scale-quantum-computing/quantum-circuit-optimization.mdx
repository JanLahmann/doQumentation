### 2.1 Poziom optymalizacji {#21-poziom-optymalizacji}
Dostępne są 4 poziomy `optimization_level` od 0 do 3. Im wyższy poziom optymalizacji, tym więcej wysiłku obliczeniowego poświęca się na optymalizację obwodu. Poziom 0 nie wykonuje żadnej optymalizacji i robi tylko minimum konieczne do uruchomienia obwodu na wybranym backendzie. Poziom 3 wkłada największy wysiłek (i zazwyczaj czas działania) w optymalizację obwodu. Poziom 1 jest domyślnym poziomem optymalizacji.
Transpilujemy obwód bez optymalizacji (`optimization_level=0`) i z optymalizacją (`optimization_level=2`).
Widzimy dużą różnicę w długości obwodów po transpilacji.

```python
pm0 = generate_preset_pass_manager(
    optimization_level=0, backend=backend, seed_transpiler=777
)
pm2 = generate_preset_pass_manager(
    optimization_level=2, backend=backend, seed_transpiler=777
)
circ0 = pm0.run(ghz_circ)
circ2 = pm2.run(ghz_circ)
print("optimization_level=0:")
display(circ0.draw("mpl", idle_wires=False, fold=-1))
print("optimization_level=2:")
display(circ2.draw("mpl", idle_wires=False, fold=-1))
```

Nowa synteza daje płytszy obwód. Dlaczego?

Dzieje się tak, ponieważ nowy obwód może być ułożony na qubitach połączonych liniowo, a więc również na grafie sprzężeń heavy-hexagon IBM&reg; Brisbane, podczas gdy oryginalny obwód wymaga łączności w kształcie gwiazdy (węzeł stopnia 4) i dlatego nie może być ułożony na grafie heavy-hex, który ma węzły o stopniu co najwyżej 3. W rezultacie oryginalny obwód wymaga routingu qubitów, który dodaje bramki SWAP, zwiększając liczbę bramek.

To, co zrobiliśmy w nowym obwodzie, można postrzegać jako ręczną syntezę obwodu „świadomą ograniczeń sprzężeń". Innymi słowy: ręczne rozwiązanie syntezy obwodu i mapowania obwodu jednocześnie.

Ogólnie rzecz biorąc, synteza obwodów zależy od zastosowania i oprogramowanie nie jest w stanie pokryć wszystkich możliwych zastosowań. Transpiler Qiskit akurat nie posiada funkcji syntezy obwodu przygotowania stanu GHZ. W takim przypadku warto rozważyć ręczną syntezę obwodu, jak pokazano powyżej.
W tej sekcji przyjrzymy się szczegółom działania transpilera Qiskit na poniższym przykładowym obwodzie.

```python
# Build a toy example circuit
from math import pi
import itertools
from qiskit.circuit import QuantumCircuit
from qiskit.circuit.library import excitation_preserving

circuit = QuantumCircuit(4, name="Example circuit")
circuit.append(excitation_preserving(4, reps=1, flatten=True), range(4))
circuit.measure_all()

value_cycle = itertools.cycle([0, pi / 4, pi / 2, 3 * pi / 4, pi, 2 * pi])
circuit.assign_parameters(
    [x[1] for x in zip(range(len(circuit.parameters)), value_cycle)], inplace=True
)
circuit.draw("mpl")
```

### 3.4 Etap optymalizacji {#34-etap-optymalizacji}

Ostatnim domyślnym etapem w potoku jest optymalizacja. Po osadzeniu obwodu w docelowym urządzeniu obwód znacznie się rozrósł. Dla poziomów 2 i 3 dostępne są dodatkowe przejścia (`Collect2qBlocks`, `ConsolidateBlocks`, `UnitarySynthesis`) umożliwiające głębszą optymalizację, czyli optymalizację bloków 2-qubitowych. (Porównaj przepływ etapu optymalizacji dla poziomu 2 z powyższym dla poziomu 1)

Optymalizacja bloków 2-qubitowych składa się z dwóch kroków: zbierania i konsolidowania bloków 2-qubitowych oraz syntezy unitarnych macierzy 2-qubitowych.

```python
from qiskit.transpiler import PassManager
from qiskit.transpiler.passes import (
    Collect2qBlocks,
    ConsolidateBlocks,
    UnitarySynthesis,
)

# Collect 2q blocks and consolidate to unitary when we expect that we can reduce the 2q gate count for that unitary
consolidate_pm = PassManager(
    [
        Collect2qBlocks(),
        ConsolidateBlocks(target=backend.target),
    ]
)
```

```python
display(basis_out.draw("mpl", idle_wires=False, fold=-1))

consolidated = consolidate_pm.run(basis_out)
```

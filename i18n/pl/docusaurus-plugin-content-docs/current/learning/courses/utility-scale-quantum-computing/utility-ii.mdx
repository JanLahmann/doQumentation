## 1. Wprowadzenie i przegląd ewolucji w czasie {#1-introduction-and-review-of-time-evolution}

Ten notatnik wykorzystuje metody i techniki z lekcji 7. Naszym celem jest numeryczne rozwiązanie zależnego od czasu równania Schrödingera. Jak omówiono w lekcji 7, trotteryzacja polega na kolejnym stosowaniu bramki lub bramek kwantowych, wybranych tak, aby przybliżyć ewolucję systemu w czasie dla wycinka czasu. Powtarzamy tę dyskusję tutaj dla wygody. Jeśli niedawno powtarzałeś lekcję 7, możesz przejść bezpośrednio do komórek z kodem poniżej.

Zgodnie z równaniem Schrödingera, ewolucja w czasie systemu początkowo będącego w stanie $\vert\psi(0)\rangle$ przyjmuje postać:

$$
\vert \psi(t) \rangle = e^{-i H t} \vert \psi(0) \rangle \text{,}
$$

gdzie $H$ to niezależny od czasu Hamiltonian rządzący systemem. Rozważamy Hamiltonian, który można zapisać jako ważoną sumę wyrazów Pauli $H=\sum_j a_j P_j$, przy czym $P_j$ reprezentuje iloczyn tensorowy wyrazów Pauli działających na $n$ kubitach. W szczególności te wyrazy Pauli mogą ze sobą komutować lub nie. Mając stan w czasie $t=0$, jak uzyskamy stan systemu w późniejszym czasie $|\psi(t)\rangle$ za pomocą komputera kwantowego? Eksponentę operatora można najłatwiej zrozumieć poprzez jej szereg Taylora:

$$
e^{-i H t} = 1-iHt-\frac{1}{2}H^2t^2+...
$$

Niektóre bardzo podstawowe eksponenty, takie jak $e^{iZ}$, można łatwo zaimplementować na komputerach kwantowych przy użyciu zwartego zestawu bramek kwantowych. Większość interesujących Hamiltonianów nie będzie miała tylko jednego wyrazu, ale wiele wyrazów. Zwróć uwagę, co się dzieje, jeśli $H = H_1+H_2$:

$$
e^{-i H t} = 1-i(H_1+H_2)t-\frac{1}{2}(H_1+H_2)^2t^2+...
$$

Gdy $H_1$ i $H_2$ komutują, mamy znany przypadek (co jest również prawdą dla liczb oraz zmiennych $a$ i $b$ poniżej):

$$
e^{-i (a+b) t} = e^{-i a t}e^{-i b t}
$$

Ale kiedy operatory nie komutują, wyrazów w szeregu Taylora nie można przegrupować, aby uprościć w ten sposób. Dlatego wyrażanie skomplikowanych Hamiltonianów w postaci bramek kwantowych jest wyzwaniem.

Jednym z rozwiązań jest rozważenie bardzo małego czasu $t$, tak aby wyraz pierwszego rzędu w rozwinięciu Taylora dominował. Przy tym założeniu:

$$
e^{-i (H_1+H_2) t} \approx 1-i(H_1+H_2)t \approx (1-i H_1 t)(1-i H_2 t) \approx e^{-i H_1 t}e^{-i H_2 t}
$$

Oczywiście może być konieczne ewoluowanie naszego stanu przez dłuższy czas. Osiąga się to, stosując wiele takich małych kroków w czasie. Ten proces nazywa się trotteryzacją:

$$
\vert \psi(t) \rangle \approx \left(\prod_j e^{-i a_j P_j t/r} \right)^r \vert\psi(0) \rangle \text{,}
$$

Tutaj $t/r$ to wycinek czasu (krok ewolucji), który wybieramy. W rezultacie powstaje bramka, która ma być stosowana $r$ razy. Mniejszy krok czasowy prowadzi do dokładniejszego przybliżenia. Jednak prowadzi to również do głębszych obwodów, co w praktyce prowadzi do większej akumulacji błędów (niezaniedbywalny problem w przypadku urządzeń kwantowych bliskiej przyszłości).

Dzisiaj przestudiujemy ewolucję w czasie [modelu Isinga](https://en.wikipedia.org/wiki/Ising_model) na liniowych siatkach z $N=2$ oraz $N=6$ węzłami. Te siatki składają się z szeregu spinów $\sigma_i$, które oddziałują tylko z najbliższymi sąsiadami. Te spiny mogą mieć dwie orientacje: $\uparrow$ i $\downarrow$, które odpowiadają namagnesowaniu odpowiednio $+1$ i $-1$.

$$
H = - J \sum_{i=0}^{N-2} Z_i Z_{i+1} - h \sum_{i=0}^{N-1} X_i  \text{,}
$$

gdzie $J$ opisuje energię oddziaływania, a $h$ wielkość pola zewnętrznego (w kierunku x powyżej, ale zmodyfikujemy to). Zapiszmy to wyrażenie przy użyciu macierzy Pauli, biorąc pod uwagę, że pole zewnętrzne ma kąt $\alpha$ względem kierunku poprzecznego,

$$
H = -J \sum_{i=0}^{N-2} Z_i Z_{i+1} -h \sum_{i=0}^{N-1} (\sin\alpha Z_i + \cos\alpha X_i) \text{.}
$$

Ten Hamiltonian jest przydatny, ponieważ pozwala nam łatwo badać wpływ pola zewnętrznego. W bazie obliczeniowej system będzie zakodowany w następujący sposób:

| Stan kwantowy            | Reprezentacja spinowa                      |
|:------------------------:|:------------------------------------------:|
| $\lvert 0 0 0 0 \rangle$ | $\uparrow\uparrow\uparrow\uparrow$         |
| $\lvert 1 0 0 0 \rangle$ | $\downarrow\uparrow\uparrow\uparrow$       |
| $\ldots$                 | $\ldots$                                   |
| $\lvert 1 1 1 1 \rangle$ | $\downarrow\downarrow\downarrow\downarrow$ |

Zaczniemy badać ewolucję w czasie takiego systemu kwantowego. Dokładniej, zwizualizujemy ewolucję w czasie pewnych właściwości systemu, takich jak namagnesowanie.

## 2. Definiowanie hamiltonianu Isinga w poprzecznym polu {#defining-the-transverse-field-ising-hamiltonian}

Rozważamy tu jednowymiarowy model Isinga w poprzecznym polu.

Najpierw utworzymy funkcję, która przyjmuje parametry układu $N$, $J$ oraz $h$ i zwraca nasz Hamiltonian jako `SparsePauliOp`. [SparsePauliOp](https://docs.quantum.ibm.com/api/qiskit/qiskit.quantum_info.SparsePauliOp) to rzadka reprezentacja operatora w postaci ważonych wyrazów [Pauli](https://docs.quantum.ibm.com/api/qiskit/qiskit.quantum_info.Pauli).
### 2.1 Ćwiczenie 1 {#activity-1}
Skonstruuj funkcję budującą Hamiltonian Isinga w poprzecznym polu (zobacz równanie powyżej), przyjmującą argumenty: „liczba kubitów”, „parametr J” oraz „parametr h”. Spróbuj wykonać to samodzielnie, korzystając z wcześniejszych przykładów. Przewiń w dół, aby zobaczyć rozwiązanie.

```python
def get_hamiltonian(nqubits, J, h):
    # List of Hamiltonian terms as 3-tuples containing
    # (1) the Pauli string,
    # (2) the qubit indices corresponding to the Pauli string,
    # (3) the coefficient.
    ZZ_tuples = [("ZZ", [i, i + 1], -J) for i in range(0, nqubits - 1)]
    X_tuples = [("X", [i], -h) for i in range(0, nqubits)]

    # We create the Hamiltonian as a SparsePauliOp, via the method
    # `from_sparse_list`, and multiply by the interaction term.
    hamiltonian = SparsePauliOp.from_sparse_list(
        [*ZZ_tuples, *X_tuples], num_qubits=nqubits
    )
    return hamiltonian.simplify()
```

#### Wykonanie symulacji ewolucji czasowej {#perform-time-evolution-simulation}
Będziemy monitorować magnetyzację (wartość oczekiwaną operatora magnetyzacji). Użyjemy symulatorów Statevector i MPS oraz porównamy wyniki.

```python
# Step 1. Map the problem
# Initiate the circuit
evolved_state = QuantumCircuit(initial_circuit.num_qubits)
# Start from the initial spin configuration
evolved_state.append(initial_circuit, evolved_state.qubits)

# Define backend (simulator)
# MPS
backend_mps = AerSimulator(method="matrix_product_state")
# Statevector
backend_sv = AerSimulator(method="statevector")

# Set Runtime Estimator
# MPS
estimator_mps = Estimator(mode=backend_mps)
# Statevector
estimator_sv = Estimator(mode=backend_sv)

# Step 2. Optimize
# Set pass manager
# MPS
pm_mps = generate_preset_pass_manager(optimization_level=3, backend=backend_mps)
# Statevector
pm_sv = generate_preset_pass_manager(optimization_level=3, backend=backend_sv)

# Transpile initial circuit
# MPS
evolved_state_mps = pm_mps.run(evolved_state)
# Statevector
evolved_state_sv = pm_sv.run(evolved_state)

# Apply layout to the operator
# MPS
magnetization_mps = magnetization.apply_layout(evolved_state_mps.layout)
# Statevector
magnetization_sv = magnetization.apply_layout(evolved_state_sv.layout)

mag_mps_list = []
mag_sv_list = []

# Step 3. Run the circuit
# Estimate expectation values for t=0.0: MPS
job = estimator_mps.run([(evolved_state_mps, [magnetization_mps])])
# Get estimated expectation values: MPS
evs = job.result()[0].data.evs
# Collect data: MPS
mag_mps_list.append(evs[0])

# Estimate expectation values for t=0.0: Statevector
job = estimator_sv.run([(evolved_state_sv, [magnetization_sv])])
# Get estimated expectation values: Statevector
evs = job.result()[0].data.evs
# Collect data: Statevector
mag_sv_list.append(evs[0])

# Start time evolution
for n in range(num_timesteps):
    # Step 1. Map the problem
    # Expand the circuit to describe delta-t
    evolved_state.append(single_step_evolution_lt, evolved_state.qubits)
    # Step 2. Optimize
    # Transpile the circuit: MPS
    evolved_state_mps = pm_mps.run(evolved_state)
    # Apply the physical layout of the qubits to the operator: MPS
    magnetization_mps = magnetization.apply_layout(evolved_state_mps.layout)
    # Step 3. Run the circuit
    # Estimate expectation values at delta-t: MPS
    job = estimator_mps.run([(evolved_state_mps, [magnetization_mps])])
    # Get estimated expectation values: MPS
    evs = job.result()[0].data.evs
    # Collect data: MPS
    mag_mps_list.append(evs[0])

    # Step 2. Optimize
    # Transpile the circuit: Statevector
    evolved_state_sv = pm_sv.run(evolved_state)
    # Apply the physical layout of the qubits to the operator: Statevector
    magnetization_sv = magnetization.apply_layout(evolved_state_sv.layout)
    # Step 3. Run the circuit
    # Estimate expectation values at delta-t: Statevector
    job = estimator_sv.run([(evolved_state_sv, [magnetization_sv])])
    # Get estimated expectation values: Statevector
    evs = job.result()[0].data.evs
    # Collect data: Statevector
    mag_sv_list.append(evs[0])

# Transform the list of expectation values (at each time step) to arrays
mag_mps_array = np.array(mag_mps_list)
mag_sv_array = np.array(mag_sv_list)
```

# Define J
J = 1.0
# Define h
h = -5.0
# Create instruction for rotation around ZZ:
# Initiate the circuit (use 2 qubits)
Rzz_circ = QuantumCircuit(2)
# Add Rzz gate (do not forget to multiply the angle by 2.0)
Rzz_circ.rzz(-J * dt * 2.0, 0, 1)
# Transform the QuantumCircuit to instruction (QuantumCircuit.to_instruction())
Rzz_instr = Rzz_circ.to_instruction(label="RZZ")

# Create instruction for rotation around X:
# Initiate the circuit (use 1 qubit)
Rx_circ = QuantumCircuit(1)
# Add Rx gate (do not forget to multiply the angle by 2.0)
Rx_circ.rx(-h * dt * 2.0, 0)
# Transform the QuantumCircuit to instruction (QuantumCircuit.to_instruction())
Rx_instr = Rx_circ.to_instruction(label="RX")

# Define the interaction list
interaction_list = [
    [[i, i + 1] for i in range(0, n_qubits2 - 1, 2)],
    [[i, i + 1] for i in range(1, n_qubits2 - 1, 2)],
]  # linear chain

# Define the registers
qr = QuantumRegister(n_qubits2)
# Initiate the circuit
single_step_evolution_sh = QuantumCircuit(qr)
# Construct the Rzz gates
for i, color in enumerate(interaction_list):
    for interaction in color:
        single_step_evolution_sh.append(Rzz_instr, interaction)

# Construct the Rx gates
for i in range(0, n_qubits2):
    single_step_evolution_sh.append(Rx_instr, [i])

print(

-----------------------------
Depth: {single_step_evolution_sh.decompose(reps=3).depth()}
Gate count: {len(single_step_evolution_sh.decompose(reps=3))}
Nonlocal gate count: {single_step_evolution_sh.decompose(reps=3).num_nonlocal_gates()}
Gate breakdown: {", ".join([f"{k.upper()}: {v}" for k, v in single_step_evolution_sh.decompose(reps=3).count_ops().items()])}
"""
)

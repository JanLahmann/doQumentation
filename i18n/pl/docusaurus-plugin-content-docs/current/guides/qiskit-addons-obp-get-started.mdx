Przygotowując kwantowe zadanie z wsteczną propagacją operatorów (OBP), najpierw musisz wybrać „plasterki obwodu", a następnie określić próg obcięcia lub „budżet błędu", aby usunąć człony z małymi współczynnikami w propagowanym wstecznie operatorze, a także ustawić górne ograniczenie całkowitego rozmiaru propagowanego wstecznie operatora. Podczas wstecznej propagacji liczba członów w operatorze obwodu $N$-qubitowego w najgorszym przypadku szybko zbliża się do $4^N$. Ten przewodnik przedstawia kroki stosowania OBP do kwantowego zadania.

Głównym składnikiem pakietu `qiskit-addons-obp` jest funkcja `backpropagate()`. Przyjmuje ona argumenty: końcową obserwowalną do odtworzenia, zestaw plasterków obwodu do obliczenia klasycznie, a opcjonalnie `TruncationErrorBudget` lub `OperatorBudget`, które określają ograniczenia obcięcia. Po ich podaniu, klasycznie obliczony propagowany wstecznie operator $O'$ jest wyznaczany iteracyjnie przez zastosowanie bramek z każdego plasterka $s$ w następujący sposób:

$$ O'^{(s)} = \mathcal{U}_{S-s+1}^\dagger O'^{(s-1)} \mathcal{U}_{S-s+1} $$

gdzie $S$ jest całkowitą liczbą plasterków, a $\mathcal{U}_{s}$ reprezentuje jeden plasterek obwodu. Niniejszy przykład korzysta z pakietu `qiskit-addons-utils` do przygotowania plasterków obwodu oraz generowania przykładowego obwodu.

Na początek rozważmy ewolucję czasową łańcucha Heisenberga XYZ. Ten hamiltonian ma postać

$$ \hat{H} = \sum_{(j,k)} \left( J_xX_jX_k + J_yY_jY_k + J_z Z_jZ_k \right) + \sum_{j} \left(h_xX_j + h_yY_j + h_zZ_j\right)$$

a wartość oczekiwaną do zmierzenia będzie $\langle Z_0 \rangle$.

Poniższy fragment kodu generuje hamiltonian w postaci `SparsePauliOp` przy użyciu modułu `qiskit_addons_utils.problem_generators` i `CouplingMap`. Ustaw stałe sprzężenia na $J_x=\pi/8$, $J_y=\pi/4$, $J_z=\pi/2$ oraz zewnętrzne pola magnetyczne na $h_x=\pi/3$, $h_y=\pi/6$, $h_z=\pi/9$, a następnie wygeneruj obwód modelujący jego ewolucję czasową.

```python
import numpy as np
from qiskit.transpiler import CouplingMap
from qiskit.synthesis import LieTrotter
from qiskit.transpiler import generate_preset_pass_manager
from qiskit_ibm_runtime import QiskitRuntimeService, EstimatorV2
from qiskit_ibm_runtime.fake_provider import FakeMelbourneV2
from qiskit.primitives import StatevectorEstimator
from qiskit.quantum_info import SparsePauliOp
from qiskit_addon_utils.problem_generators import (
    generate_time_evolution_circuit,
    generate_xyz_hamiltonian,
)
from qiskit_addon_utils.slicing import slice_by_gate_types
from qiskit_addon_obp.utils.simplify import OperatorBudget
from qiskit_addon_obp.utils.truncating import setup_budget
from qiskit_addon_obp import backpropagate
from qiskit_addon_utils.slicing import combine_slices

coupling_map = CouplingMap.from_heavy_hex(3, bidirectional=False)

# Choose a 10-qubit linear chain on this coupling map
reduced_coupling_map = coupling_map.reduce(
    [0, 13, 1, 14, 10, 16, 5, 12, 8, 18]
)

# Get a qubit operator describing the Heisenberg XYZ model
hamiltonian = generate_xyz_hamiltonian(
    reduced_coupling_map,
    coupling_constants=(np.pi / 8, np.pi / 4, np.pi / 2),
    ext_magnetic_field=(np.pi / 3, np.pi / 6, np.pi / 9),
)

# we evolve for some time
circuit = generate_time_evolution_circuit(
    hamiltonian, synthesis=LieTrotter(reps=2), time=0.2
)

circuit.draw("mpl")
```

### Przygotowanie danych wejściowych do wstecznej propagacji {#przygotowanie-danych-wejsciowych-do-wstecznej-propagacji}

Następnie wygeneruj plasterki obwodu do wstecznej propagacji. Ogólnie rzecz biorąc, wybór sposobu cięcia może mieć wpływ na efektywność wstecznej propagacji dla danego problemu. Tutaj zgrupuj bramki tego samego typu w plasterki, używając funkcji `qiskit_addons_utils.slice_by_gate_types`.

```python
slices = slice_by_gate_types(circuit)
print(f"Separated the circuit into {len(slices)} slices.")
```

```text
Separated the circuit into 18 slices.
```

Po wygenerowaniu plasterków określ `OperatorBudget`, aby zapewnić funkcji `backpropagate()` warunek zatrzymania wstecznej propagacji operatora i zapobiec dalszemu wzrostowi narzutu klasycznego. Możesz również określić budżet błędu obcięcia dla każdego plasterka, w ramach którego człony Pauliego z małymi współczynnikami będą obcinane z każdego plasterka do wyczerpania budżetu błędu. Wszelki pozostały budżet zostanie dodany do budżetu następnego plasterka.

Tutaj określ, że wsteczna propagacja powinna się zatrzymać, gdy liczba komutujących parami grup Pauliego w operatorze przekroczy $8$, i przydziel budżet błędu $0.005$ dla każdego plasterka.

```python
op_budget = OperatorBudget(max_qwc_groups=8)
truncation_error_budget = setup_budget(max_error_per_slice=0.005)
```

### Wsteczna propagacja plasterków {#wsteczna-propagacja-plasterkow}

W tym kroku zdefiniujesz końcową obserwowalną do zmierzenia i uruchomisz wsteczną propagację przez każdy plasterek. Funkcja `backpropagate()` zwraca trzy wyniki: propagowaną wstecznie obserwowalną, pozostałe plasterki obwodu, które nie zostały propagowane wstecznie (i które powinny być uruchomione na sprzęcie kwantowym), oraz metadane dotyczące wstecznej propagacji.

Zwróć uwagę, że zarówno `OperatorBudget`, jak i `TruncationErrorBudget` są opcjonalnymi parametrami metody `backpropagate()`. Ogólnie rzecz biorąc, najlepszy wybór dla obu powinien być dobierany heurystycznie i wymaga pewnych eksperymentów. W tym przykładzie dokonamy wstecznej propagacji zarówno z budżetem `TruncationErrorBudget`, jak i bez niego.

<Admonition title="Uwaga" type="note">
  Domyślnie `backpropagate()` używa normy $L_1$ obciętych współczynników do ograniczenia całkowitego błędu wynikającego z obcięcia, ale można użyć innych norm $L_p$, jeśli chcesz zmodyfikować sposób obliczania błędu obcięcia.
</Admonition>

```python
# Specify a single-qubit observable
observable = SparsePauliOp("IIIIIIIIIZ")

# Backpropagate without the truncation error budget
backpropagated_observable, remaining_slices, metadata = backpropagate(
    observable,
    slices,
    operator_budget=op_budget,
)

# Recombine the slices remaining after backpropagation
bp_circuit = combine_slices(remaining_slices, include_barriers=True)

print(f"Backpropagated {metadata.num_backpropagated_slices} slices.")
print(
    f"New observable has {len(backpropagated_observable.paulis)} terms, which can be combined into "
    f"{len(backpropagated_observable.group_commuting(qubit_wise=True))} groups.\n"
    f"After truncation, the error in our observable is bounded by {metadata.accumulated_error(0):.3e}"
)
print(
    f"Note that backpropagating one more slice would result in {metadata.backpropagation_history[-1].num_paulis[0]} terms "
    f"across {metadata.backpropagation_history[-1].num_qwc_groups} groups."
)
```

Note that backpropagating one more slice would result in 27 terms across 12 groups.

### Transpilacja i wykonanie kwantowego zadania {#transpilacja-i-wykonanie-kwantowego-zadania}

Teraz, gdy dokonałeś wstecznej propagacji operatora, możesz wykonać pozostałą część obwodu na QPU. Kwantowe zadanie, korzystające z Estimatora, powinno zawierać obwód `bp_circuit_trunc` i musi mierzyć propagowaną wstecznie obserwowalną `backpropagated_observable`.

Aby zademonstrować skuteczność OBP samego w sobie, poniższy fragment kodu transpiluje zarówno oryginalny, jak i propagowany wstecznie obwód (z obcięciem i bez) oraz symuluje obwody klasycznie przy użyciu `StatevectorEstimator`.

```python
# Specify a backend and a pass manager for transpilation
backend = FakeMelbourneV2()
# pm = generate_preset_pass_manager(backend=backend, optimization_level=1)

pm = generate_preset_pass_manager(backend=backend, optimization_level=3)

# Transpile original experiment
circuit_isa = pm.run(circuit)
observable_isa = observable.apply_layout(circuit_isa.layout)

# Transpile backpropagated experiment without truncation
bp_circuit_isa = pm.run(bp_circuit)
bp_obs_isa = backpropagated_observable.apply_layout(bp_circuit_isa.layout)

# Transpile the backpropagated experiment with truncated observable terms
bp_circuit_trunc_isa = pm.run(bp_circuit_trunc)
bp_obs_trunc_isa = backpropagated_observable_trunc.apply_layout(
    bp_circuit_trunc_isa.layout
)

estimator = StatevectorEstimator()

# Run the experiments using the exact statevector estimator
result_exact = (
    estimator.run([(circuit, observable)]).result()[0].data.evs.item()
)

result_bp = (
    estimator.run([(bp_circuit_isa, bp_obs_isa)]).result()[0].data.evs.item()
)
result_bp_trunc = (
    estimator.run([(bp_circuit_trunc_isa, bp_obs_trunc_isa)])
    .result()[0]
    .data.evs.item()
)

print(f"Exact expectation value: {result_exact}")
print(f"Backpropagated expectation value without truncation: {result_bp}")
print(f"Backpropagated expectation value with truncation: {result_bp_trunc}")
print(
    f"    - Expected Error for truncated observable: {metadata_trunc.accumulated_error(0):.3e}"
)
print(
    f"    - Observed Error for truncated observable: {abs(result_exact - result_bp_trunc):.3e}"
)
```

```text
Exact expectation value: 0.8854160687717517
Backpropagated expectation value without truncation: 0.8854160687717533
Backpropagated expectation value with truncation: 0.8850236647156081
    - Expected Error for truncated observable: 4.933e-02
    - Observed Error for truncated observable: 3.924e-04
```

Na koniec poniższy fragment kodu transpiluje i wykonuje propagowany wstecznie obwód na QPU (zarówno z obcięciem, jak i bez).

```python
# Specify a backend and a pass manager for transpilation
service = QiskitRuntimeService()
backend = service.least_busy()
pm = generate_preset_pass_manager(backend=backend, optimization_level=3)

# Transpile backpropagated experiment without truncation
bp_circuit_isa = pm.run(bp_circuit)
bp_obs_isa = backpropagated_observable.apply_layout(bp_circuit_isa.layout)

# Transpile the backpropagated experiment with truncated observable terms
bp_circuit_trunc_isa = pm.run(bp_circuit_trunc)
bp_obs_trunc_isa = backpropagated_observable_trunc.apply_layout(
    bp_circuit_trunc_isa.layout
)

# Run the experiments using Estimator primitive
estimator = EstimatorV2(mode=backend)

result_bp_qpu = (
    estimator.run([(bp_circuit_isa, bp_obs_isa)]).result()[0].data.evs.item()
)

result_bp_trunc_qpu = (
    estimator.run([(bp_circuit_trunc_isa, bp_obs_trunc_isa)])
    .result()[0]
    .data.evs.item()
)

print(f"Exact expectation value: {result_exact}")
print(f"Backpropagated expectation value without truncation: {result_bp_qpu}")
print(
    f"Backpropagated expectation value with truncation: {result_bp_trunc_qpu}"
)
print(
    f"    - Observed Error for observable without truncation: {abs(result_exact - result_bp_qpu):.3e}"
)
print(
    f"    - Observed Error for truncated observable: {abs(result_exact - result_bp_trunc_qpu):.3e}"
)
```

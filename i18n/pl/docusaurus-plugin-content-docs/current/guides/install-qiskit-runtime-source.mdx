# Instalacja klienta Qiskit Runtime ze źródła {#install-the-qiskit-runtime-client-from-source}

Gdy instalujesz Qiskit Runtime ze źródła, możesz uzyskać dostęp do bieżącej wersji deweloperskiej, zamiast korzystać z wersji dostępnej w repozytorium Python Package Index (PyPI). Dzięki temu możesz sprawniej sprawdzać i rozszerzać najnowszą wersję kodu.

## Tworzenie i aktywacja nowego środowiska wirtualnego {#create-and-activate-a-new-virtual-environment}

1. Przejdź do katalogu projektu i utwórz minimalne środowisko z zainstalowanym tylko Pythonem.

        <OperatingSystemTabs>
          <TabItem value="mac" label="macOS">
            ```shell
            python3 -m venv .venv
            ```
          </TabItem>

          <TabItem value="linux" label="Linux">
            ```shell
            python3 -m venv .venv
            ```
          </TabItem>

          <TabItem value="win" label="Windows">
            ```text
            python -m venv .venv
            ```
          </TabItem>
         </OperatingSystemTabs>

1. Aktywuj nowe środowisko.

         <OperatingSystemTabs>
           <TabItem value="mac" label="macOS">
             ```shell
             source .venv/bin/activate
             ```
           </TabItem>

           <TabItem value="linux" label="Linux">

         </OperatingSystemTabs>

## Instalacja Qiskit Runtime {#install-qiskit-runtime}

Wykonaj następujące kroki, aby zainstalować Qiskit Runtime:

1. Sklonuj repozytorium Qiskit Runtime.

```bash
git clone https://github.com/Qiskit/qiskit-ibm-runtime.git
```

2. Przejdź do katalogu `qiskit-ibm-runtime`.

```bash
cd qiskit-ibm-runtime
```

3. Zainstaluj `qiskit-runtime`. Zalecamy używanie [środowiska wirtualnego](https://docs.python.org/3/library/venv.html), aby uniknąć zanieczyszczenia globalnej instalacji Pythona.

  * **Standardowa instalacja**:

    ```bash
    pip install .
    ```

  * **Tryb edytowalny**: W tym trybie nie musisz ponownie instalować Qiskit po wprowadzeniu zmian w kodzie projektu.

    ```bash
    pip install -e .
    ```

    W trybie edytowalnym skompilowane rozszerzenia są budowane w _trybie debugowania_ bez optymalizacji.

4. (Opcjonalnie) Jeśli chcesz uruchamiać testy lub sprawdzenia lintingu, zainstaluj wymagania deweloperskie. Zalecamy używanie [środowiska wirtualnego](https://docs.python.org/3/library/venv.html), aby uniknąć zanieczyszczenia globalnej instalacji Pythona.

```bash
pip install -e ".[dev]"
```

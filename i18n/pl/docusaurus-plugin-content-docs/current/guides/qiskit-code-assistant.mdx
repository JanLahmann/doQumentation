# Qiskit Code Assistant

Qiskit Code Assistant ma na celu zwiększenie dostępności obliczeń kwantowych dla nowych użytkowników Qiskit oraz poprawę doświadczenia programistycznego dla obecnych użytkowników. Jest to generatywny asystent kodu AI oparty na [watsonx](https://www.ibm.com/products/watsonx-ai). Został wytrenowany na milionach tokenów tekstowych z Qiskit SDK, wieloletnich przykładach kodu Qiskit i funkcjach IBM Quantum&reg;. Qiskit Code Assistant może wspierać twój przepływ pracy w zakresie obliczeń kwantowych, oferując sugestie generowane przez LLM na podstawie [modeli IBM Granite](https://www.ibm.com/products/watsonx-ai/foundation-models), które uwzględniają najnowsze funkcje i możliwości IBM&reg;.

<Admonition type="note" title="Uwagi">
- To jest funkcja eksperymentalna dostępna dla użytkowników IBM Quantum Premium Plan zarejestrowanych na nowej [IBM Quantum Platform](https://quantum.cloud.ibm.com/).
- Qiskit Code Assistant jest w fazie wstępnego wydania i może ulec zmianie.
- Jeśli masz opinię lub chcesz skontaktować się z zespołem deweloperów, użyj [kanału Qiskit Slack Workspace](https://qiskit.enterprise.slack.com/archives/C07LYA6PL83) lub powiązanych publicznych repozytoriów GitHub.
</Admonition>

## Funkcje {#features}

Następujące funkcje są dostępne w rozszerzeniach dla [Visual Studio Code](https://code.visualstudio.com/) (VS Code) i kompatybilnych edytorów, a także [JupyterLab](https://jupyterlab.readthedocs.io/en/latest/):

* Przyspiesza generowanie kodu Qiskit poprzez wykorzystanie generatywnej AI opartej na modelach wyspecjalizowanych w generowaniu kodu Qiskit.
* Obsługuje abstrakcyjne i szczegółowe podpowiedzi do generowania rekomendacji.
* Prezentuje sugestie, które możesz przeglądać, akceptować lub odrzucać.
* Obsługuje pliki z kodem Python i notatniki Jupyter.
* Zawiera zabezpieczenia chroniące przed odpowiadaniem na pytania stanowiące potencjalne zagrożenie dla użytkowników, takie jak mowa nienawiści.

Aby zintegrować Qiskit Code Assistant bezpośrednio ze swoim środowiskiem deweloperskim, postępuj zgodnie z instrukcjami w odpowiednim temacie:
- [JupyterLab](/guides/qiskit-code-assistant-jupyterlab)
- [VS Code](/guides/qiskit-code-assistant-vscode)

## Duży Model Językowy (LLM) stojący za Qiskit Code Assistant {#the-large-language-model-llm-behind-qiskit-code-assistant}

Aby dostarczać sugestie kodu, Qiskit Code Assistant używa Dużego Modelu Językowego (LLM). W tym przypadku Qiskit Code Assistant opiera się obecnie na modelu `mistral-small-3.2-24b-qiskit`, zbudowanym na [modelu Mistral-Small-3.2-24B-Qiskit](https://huggingface.co/Qiskit/mistral-small-3.2-24b-qiskit). Model `mistral-small-3.2-24b-qiskit` poprawia możliwości generowania kodu modelu `Mistral-Small-3.2-24B-Instruct-2506` dla Qiskit poprzez rozszerzone wstępne trenowanie i dostrajanie na wysokiej jakości danych Qiskit, a także commitach Pythona i czacie. Aby uzyskać więcej informacji o rodzinie modeli Mistral AI, zapoznaj się z [dokumentacją Mistral AI](https://docs.mistral.ai/getting-started/models). Aby dowiedzieć się więcej o modelach `.*-qiskit`, zobacz [Qiskit Code Assistant: Training LLMs for generating Quantum Computing Code](https://arxiv.org/abs/2405.19495).

Nasze LLM wyspecjalizowane dla Qiskit są dostępne również jako modele open-source. Sprawdź wszystkie dostępne modele na https://huggingface.co/Qiskit.

## Benchmarki Qiskit HumanEval i Qiskit HumanEval Hard {#the-qiskit-humaneval-and-qiskit-humaneval-hard-benchmarks}

Aby przetestować `mistral-small-3.2-24b-qiskit` i inne modele, współpracowaliśmy z Adwokatami i ekspertami Qiskit, aby stworzyć benchmarki oparte na wykonaniu kodu o nazwie Qiskit HumanEval (QHE) i Qiskit HumanEval Hard (QHE Hard), i uruchomiliśmy je na modelach. Benchmarki te są podobne do [HumanEval](https://arxiv.org/abs/2107.03374), zawierają wiele wymagających problemów kodowania do rozwiązania, wszystkie oparte na oficjalnych bibliotekach Qiskit.

Benchmarki składają się z około 150 testów, z których każdy tworzony jest z definicji funkcji, po której następuje docstring szczegółowo opisujący zadanie, które model ma rozwiązać. Każdy przykład zawiera również referencyjne rozwiązanie kanoniczne oraz testy jednostkowe do oceny poprawności generowanych rozwiązań. Istnieją trzy poziomy trudności testów: podstawowy, pośredni i trudny. Benchmark Qiskit HumanEval Hard jest wariantem Qiskit HumanEval, ale usuwa informacje dotyczące importów kodu, więc LLM musi samodzielnie ustalić właściwe importy metod lub klas. Ta zmiana sprawia, że zestaw danych jest znacznie bardziej wymagający dla LLM, zgodnie z naszymi testami i wstępnymi wynikami.

Zestawy danych dla Qiskit HumanEval i Qiskit HumanEval Hard są dostępne na tych stronach: [Qiskit HumanEval](https://huggingface.co/datasets/Qiskit/qiskit_humaneval) i [Qiskit HumanEval](https://huggingface.co/datasets/Qiskit/qiskit_humaneval_hard). Możesz przyczynić się do rozwoju tych benchmarków w [repozytorium GitHub](https://github.com/qiskit-community/qiskit-human-eval).

### Szybki start (zalecane) {#quick-start-recommended}

Najprostszym sposobem na rozpoczęcie pracy z Qiskit Code Assistant w trybie lokalnym jest użycie automatycznych skryptów konfiguracyjnych dla rozszerzenia VS Code lub JupyterLab. Skrypty te automatycznie zainstalują [Ollama](https://ollama.com/) do uruchamiania LLM, pobiorą zalecany model i skonfigurują rozszerzenie za ciebie.

#### Konfiguracja rozszerzenia VS Code {#vs-code-extension-setup}

Uruchom następujące polecenie w terminalu:

```bash
bash <(curl -fsSL https://raw.githubusercontent.com/Qiskit/qiskit-code-assistant-vscode/main/setup_local.sh)
```

Ten skrypt wykonuje następujące kroki:
- Instaluje Ollama (jeśli nie jest jeszcze zainstalowana)
- Pobiera i konfiguruje zalecany model Qiskit Code Assistant
- Konfiguruje rozszerzenie VS Code do pracy z lokalnym wdrożeniem

#### Konfiguracja rozszerzenia JupyterLab {#jupyterlab-extension-setup}

Uruchom następujące polecenie w terminalu:

```bash
bash <(curl -fsSL https://raw.githubusercontent.com/Qiskit/qiskit-code-assistant-jupyterlab/main/setup_local.sh)
```

Ten skrypt wykona:
- Instalację Ollama (jeśli nie jest jeszcze zainstalowana)
- Pobranie i konfigurację zalecanego modelu Qiskit Code Assistant
- Konfigurację rozszerzenia JupyterLab do pracy z lokalnym wdrożeniem

##### Skonfiguruj Ollama z ręcznie pobranym modelem GGUF Qiskit Code Assistant {#set-up-ollama-with-a-manually-downloaded-qiskit-code-assistant-gguf-model}

Jeśli ręcznie pobrałeś model GGUF, taki jak https://huggingface.co/Qiskit/Qwen2.5-Coder-14B-Qiskit-GGUF, i chcesz eksperymentować z różnymi szablonami i parametrami, możesz wykonać następujące kroki, aby załadować go do lokalnej aplikacji Ollama.

1. Utwórz plik `Modelfile` wpisując następującą zawartość i pamiętaj, aby zaktualizować `<PATH-TO-GGUF-FILE>` do rzeczywistej ścieżki pobranego modelu.

    ```
    FROM <PATH-TO-GGUF-FILE>
    TEMPLATE """{{ if .System }}
    System:
    {{ .System }}

    {{ end }}{{ if .Prompt }}Question:
    {{ .Prompt }}

    {{ end }}Answer:
    ```python{{ .Response }}
    """

    PARAMETER stop "Question:"
    PARAMETER stop "Answer:"
    PARAMETER stop "System:"
    PARAMETER stop "```"

    PARAMETER temperature 0
    PARAMETER top_k 1
    ```

1. Uruchom następujące polecenie, aby utworzyć niestandardową instancję modelu na podstawie pliku `Modelfile`.

    ```
    ollama create Qwen2.5-Coder-14B-Qiskit -f ./path-to-model-file
    ```

    <Admonition type="note">
        Ten proces może chwilę potrwać, ponieważ Ollama musi odczytać plik modelu, zainicjalizować instancję modelu i skonfigurować ją zgodnie z dostarczonymi specyfikacjami.
    </Admonition>

##### Uruchamianie ręcznie pobranego modelu Qiskit Code Assistant w Ollama {#run-the-qiskit-code-assistant-model-manually-downloaded-in-ollama}

Po skonfigurowaniu modelu `Qwen2.5-Coder-14B-Qiskit` w Ollama uruchom następujące polecenie, aby uruchomić model i wchodzić z nim w interakcję w terminalu (w trybie czatu).

```
ollama run Qwen2.5-Coder-14B-Qiskit
```

Kilka przydatnych poleceń:

- `ollama list` — wyświetl listę modeli na swoim komputerze
- `ollama rm Qwen2.5-Coder-14B-Qiskit` — usuń model
- `ollama show Qwen2.5-Coder-14B-Qiskit` — wyświetl informacje o modelu
- `ollama stop Qwen2.5-Coder-14B-Qiskit` — zatrzymaj aktualnie działający model
- `ollama ps` — wyświetl listę aktualnie załadowanych modeli

</AccordionItem>

<AccordionItem title="Manually deploy the Qiskit Code Assistant models in local through the `llama-cpp-python` package">

<span id="use-llama-cpp-python"></span>
Alternatywą dla aplikacji Ollama jest pakiet `llama-cpp-python`, będący powiązaniem Pythona z `llama.cpp`. Zapewnia on większą kontrolę i elastyczność w lokalnym uruchamianiu modelu GGUF i jest idealny dla użytkowników, którzy chcą zintegrować lokalny model ze swoimi procesami roboczymi i aplikacjami Python.

1. Zainstaluj [`llama-cpp-python`](https://pypi.org/project/llama-cpp-python/)
2. Wchodź w interakcję z modelem z poziomu swojej aplikacji za pomocą `llama_cpp`. Na przykład:

```python
from llama_cpp import Llama

model_path = <PATH-TO-GGUF-FILE>

model = Llama(
        model_path,
        seed=17,
        n_ctx=10000,
        n_gpu_layers=37, # to offload in gpu, but put 0 if all in cpu
    )

input = 'Generate a quantum circuit with 2 qubits'
raw_pred = model(input)[“choices”][0][“text”]
```

Możesz również dodać do modelu parametry generowania tekstu, aby dostosować wnioskowanie:

```python
generation_kwargs = {
        “max_tokens”: 512,
        “echo”: False, # Echo the prompt in the output
        “top_k”: 1
    }

raw_pred = model(input, **generation_kwargs)[“choices”][0][“text”]
```

</AccordionItem>

<AccordionItem title="Manually deploy the Qiskit Code Assistant models in local through llama.cpp">
#### Użycie biblioteki `llama.cpp` {#use-llama-cpp-python}

Kolejną alternatywą jest `llama.cpp` — biblioteka open-source do wykonywania wnioskowania LLM na procesorze przy minimalnej konfiguracji.
Zapewnia ona niskopoziomową kontrolę nad wykonaniem modelu i jest zazwyczaj uruchamiana z wiersza poleceń, wskazując na lokalny plik modelu GGUF.

Istnieje kilka sposobów instalacji `llama.cpp` na swoim komputerze:

- Zainstaluj llama.cpp za pomocą [brew, nix lub winget](https://github.com/ggml-org/llama.cpp/blob/master/docs/install.md)
- Uruchom z Dockerem: zapoznaj się z [dokumentacją Docker przygotowaną przez zespół `llama.cpp`](https://github.com/ggml-org/llama.cpp/blob/master/docs/docker.md)
- Pobierz gotowe pliki binarne ze [strony wydań](https://github.com/ggml-org/llama.cpp/releases)
- Zbuduj ze źródeł, [klonując to repozytorium](https://github.com/ggml-org/llama.cpp/blob/master/docs/build.md)

Po instalacji możesz użyć `llama.cpp` do interakcji z modelami GGUF w trybie konwersacji w następujący sposób:
```bash
# Use a local model file
llama-cli -m my_model.gguf -cnv

# Or download and run a model directly from Hugging Face
llama-cli -hf Qiskit/Qwen2.5-Coder-14B-Qiskit-GGUF -cnv
```

Możesz również uruchomić serwer API zgodny z OpenAI dla modelu w następujący sposób:
```bash
llama-server -hf Qiskit/Qwen2.5-Coder-14B-Qiskit-GGUF
```

##### Zaawansowane parametry {#advanced-parameters}
Za pomocą programu `llama-cli` możesz kontrolować generowanie przez model, używając opcji wiersza poleceń. Na przykład możesz podać wstępny prompt „systemowy” za pomocą flagi `-p/--prompt`. W trybie konwersacji (`-cnv`) ten wstępny prompt działa jako wiadomość systemowa. W przeciwnym razie możesz po prostu dołączyć dowolną pożądaną instrukcję na początku tekstu promptu. Możesz również dostosować parametry próbkowania — na przykład: temperaturę (`--temp`), top-k (`--top-k`), top-p (`--top-p`), karę za powtórzenia (`--repeat-penalty`) i ziarno (`--seed`). Poniżej znajduje się przykładowe wywołanie z użyciem tych opcji:

```bash
llama-cli -hf Qiskit/Qwen2.5-Coder-14B-Qiskit-GGUF \
  -p "You are a friendly assistant." -cnv \
  --temp 0.7 \
  --top-k 50 \
  --top-p 0.95 \
  --repeat-penalty 1.1 \
  --seed 42
```

Aby zapewnić prawidłowe działanie naszych modeli Qiskit, zalecamy używanie promptu systemowego dostępnego w naszych repozytoriach HF GGUF: prompt systemowy dla [mistral-small-3.2-24b-qiskit-GGUF](https://huggingface.co/Qiskit/mistral-small-3.2-24b-qiskit-GGUF/blob/main/system), [Qwen2.5-Coder-14B-Qiskit-GGUF](https://huggingface.co/Qiskit/Qwen2.5-Coder-14B-Qiskit-GGUF/blob/main/system), [granite-3.3-8b-qiskit-GGUF](https://huggingface.co/Qiskit/granite-3.3-8b-qiskit-GGUF/blob/main/system) i [granite-3.2-8b-qiskit-GGUF](https://huggingface.co/Qiskit/granite-3.2-8b-qiskit-GGUF/blob/main/system).

</AccordionItem>

<AccordionItem title="Ręcznie połącz rozszerzenia z lokalnym wdrożeniem">

Użyj rozszerzenia VS Code i rozszerzenia JupyterLab dla Qiskit Code Assistant, aby wysyłać zapytania do lokalnie wdrożonego modelu Qiskit Code Assistant. Po [skonfigurowaniu aplikacji Ollama z modelem](#using-the-ollama-application) możesz skonfigurować rozszerzenia, aby połączyć się z lokalną usługą.

#### Połącz z rozszerzeniem Qiskit Code Assistant VS Code {#connect-with-the-qiskit-code-assistant-vs-code-extension}

Za pomocą rozszerzenia Qiskit Code Assistant VS Code możesz wchodzić w interakcję z modelem i wykonywać uzupełnianie kodu podczas pisania. Może to dobrze sprawdzić się dla użytkowników poszukujących pomocy przy pisaniu kodu Qiskit dla swoich aplikacji Python.

1. Zainstaluj [rozszerzenie Qiskit Code Assistant VS Code](/guides/qiskit-code-assistant-vscode).
1. W VS Code przejdź do **User Settings** i ustaw **Qiskit Code Assistant: Url** na URL swojego lokalnego wdrożenia Ollama (na przykład `http://localhost:11434`).
1. Przeładuj VS Code, przechodząc do **View > Command Palette...** i wybierając **Developer: Reload Window**.

Model Qiskit Code Assistant skonfigurowany w Ollama powinien pojawić się na pasku stanu i jest gotowy do użycia.

#### Połącz z rozszerzeniem Qiskit Code Assistant JupyterLab {#connect-with-the-qiskit-code-assistant-jupyterlab-extension}

Za pomocą rozszerzenia Qiskit Code Assistant JupyterLab możesz wchodzić w interakcję z modelem i wykonywać uzupełnianie kodu bezpośrednio w swoim notatniku Jupyter. Użytkownicy pracujący głównie z notatnikami Jupyter mogą skorzystać z tego rozszerzenia, aby jeszcze bardziej poprawić swoje doświadczenie podczas pisania kodu Qiskit.

1. Zainstaluj [rozszerzenie Qiskit Code Assistant JupyterLab](/guides/qiskit-code-assistant-jupyterlab).
1. W JupyterLab przejdź do **Settings Editor** i ustaw **Qiskit Code Assistant Service API** na URL swojego lokalnego wdrożenia Ollama (na przykład `http://localhost:11434`).

Model Qiskit Code Assistant skonfigurowany w Ollama powinien pojawić się na pasku stanu i jest gotowy do użycia.

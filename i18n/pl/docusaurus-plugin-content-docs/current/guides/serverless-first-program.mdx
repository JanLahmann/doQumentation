Ten przykład pokazuje, jak używać narzędzi `qiskit-serverless` do tworzenia programu do równoległej transpilacji, a następnie implementować `qiskit-ibm-catalog`, aby wdrożyć program na IBM Quantum Platform do używania jako wielokrotny zdalny serwis.
## Przykład: zdalna transpilacja z Qiskit Serverless {#example-remote-transpilation-with-qiskit-serverless}

Zacznij od poniższego przykładu, który transpiluje `circuit` względem danego `backend` i docelowego `optimization_level`, i stopniowo dodawaj więcej elementów, aby wdrożyć swoje zadanie do Qiskit Serverless.

Umieść poniższą komórkę kodu w pliku `./source_files/transpile_remote.py`. Ten plik to program do przesłania do Qiskit Serverless.



```python
%%writefile ./source_files/transpile_remote.py

from qiskit.transpiler import generate_preset_pass_manager

def transpile_remote(circuit, optimization_level, backend):
    """Transpiles an abstract circuit into an ISA circuit for a given backend."""
    pass_manager = generate_preset_pass_manager(
        optimization_level=optimization_level,
		backend=backend
    )
    isa_circuit = pass_manager.run(circuit)
    return isa_circuit
```

## Konfiguracja plików {#set-up-your-files}

Qiskit Serverless wymaga umieszczenia plików `.py` zadania w dedykowanym katalogu. Następująca struktura jest przykładem dobrej praktyki:

```text
serverless_program
├── program_uploader.ipynb
└── source_files
    ├── transpile_remote.py
    └── *.py
```

Serverless przesyła zawartość `source_files`, aby uruchomić zdalnie. Po ich skonfigurowaniu możesz dostosować `transpile_remote.py`, aby pobierało dane wejściowe i zwracało wyjścia.

### Pobieranie argumentów programu {#get-program-arguments}

Twój początkowy `transpile_remote.py` ma trzy dane wejściowe: `circuits`, `backend_name` i `optimization_level`. Serverless jest obecnie ograniczony do przyjmowania wyłącznie danych wejściowych i wyjściowych, które można serializować. Z tego powodu nie możesz bezpośrednio przekazać `backend`, więc zamiast tego użyj `backend_name` jako ciągu znaków.

```python
%%writefile --append ./source_files/transpile_remote.py

from qiskit_serverless import get_arguments, save_result, distribute_task, get

# Get program arguments
arguments = get_arguments()
circuits = arguments.get("circuits")
backend_name = arguments.get("backend_name")
optimization_level = arguments.get("optimization_level")
```

```text
Appending to ./source_files/transpile_remote.py
```

W tym momencie możesz uzyskać Backend za pomocą `QiskitRuntimeService` i dodać istniejący program za pomocą poniższego kodu. Poniższy kod wymaga, abyś już [zapisał swoje dane uwierzytelniające](/guides/cloud-setup).

```python
%%writefile --append ./source_files/transpile_remote.py

from qiskit_ibm_runtime import QiskitRuntimeService

service = QiskitRuntimeService()
backend = service.backend(backend_name)
```

```text
Appending to ./source_files/transpile_remote.py
```

Na koniec możesz uruchomić `transpile_remote()` dla wszystkich przekazanych `circuits` i zwrócić `transpiled_circuits` jako wynik:

```python
%%writefile --append ./source_files/transpile_remote.py

results = [
    transpile_remote(circuit, 1, backend)
    for circuit in circuits
]

save_result({
    "transpiled_circuits": results
})
```

```text
Appending to ./source_files/transpile_remote.py
```

## Wdrożenie na IBM Quantum Platform {#deploy-to-ibm-quantum-platform}

Poprzednia sekcja stworzyła program do zdalnego uruchomienia. Komórki kodu w tej sekcji przesyłają ten program do Qiskit Serverless.

Użyj `qiskit-ibm-catalog`, aby uwierzytelnić się w `QiskitServerless` swoim kluczem API, który możesz znaleźć na [panelu IBM Quantum](https://quantum.cloud.ibm.com), i przesłać program.

Możesz użyć `save_account()`, aby zapisać swoje dane uwierzytelniające (patrz sekcja [Konfiguracja w IBM Cloud](/guides/cloud-setup#cloud-save)). Pamiętaj, że zapisuje to Twoje dane uwierzytelniające do tego samego pliku co [`QiskitRuntimeService.save_account()`](https://docs.quantum.ibm.com/api/qiskit-ibm-runtime/qiskit-runtime-service#save_account).

```python
from qiskit_ibm_catalog import QiskitServerless, QiskitFunction

# Authenticate to the remote cluster and submit the pattern for remote execution
serverless = QiskitServerless()
```

Qiskit Serverless kompresuje zawartość `working_dir` (w tym przypadku `source_files`) do archiwum `tar`, które jest przesyłane i czyszczone po zakończeniu. `entrypoint` identyfikuje główny plik wykonywalny programu, który Qiskit Serverless ma uruchomić. Ponadto, jeśli Twój program ma niestandardowe zależności `pip`, możesz dodać je do tablicy `dependencies`:

```python
transpile_remote_demo = QiskitFunction(
    title="transpile_remote_serverless",
    entrypoint="transpile_remote.py",
    working_dir="./source_files/",
)
```

```python
serverless.upload(transpile_remote_demo)
```

```text
QiskitFunction(transpile_remote_serverless)
```

Aby sprawdzić, czy przesłanie się powiodło, użyj `serverless.list()`:

```python
# Get program from serverless.list() that matches the title of the one we uploaded
```

```python
# This cell is hidden from users, it checks the program executes correctly
from time import sleep
from qiskit import QuantumCircuit

qc = QuantumCircuit(2)
transpile_remote_serverless = serverless.load("transpile_remote_serverless")
job = transpile_remote_serverless.run(
    circuits=[qc],
    backend="ibm_sherbrooke",
    optimization_level=1,
)
while True:
    sleep(5)
    status = job.status()
    if status not in ["QUEUED", "INITIALIZING", "RUNNING", "DONE"]:
        raise Exception(
            f"Unexpected job status: '{status}'\n"
            + "Here are the logs:\n"
            + job.logs()
        )
    if status == "DONE":
        break
```

## Tworzenie i uruchamianie prostego programu kwantowego {#create-and-run-a-simple-quantum-program}
Cztery kroki pisania programu kwantowego przy użyciu wzorców Qiskit to:

1.  Odwzorowanie problemu na format natywny dla komputerów kwantowych.

2.  Optymalizacja Circuit i operatorów.

3.  Wykonanie przy użyciu kwantowej funkcji prymitywnej.

4.  Analiza wyników.

### Krok 1. Odwzorowanie problemu na format natywny dla komputerów kwantowych {#step-1-map-the-problem-to-a-quantum-native-format}

W programie kwantowym *obwody kwantowe* (Circuit) są natywnym formatem reprezentowania instrukcji kwantowych, a *operatory* reprezentują obserwable, które mają być mierzone. Tworząc Circuit, zazwyczaj tworzysz nowy obiekt [`QuantumCircuit`](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.QuantumCircuit#quantumcircuit-class), a następnie sekwencyjnie dodajesz do niego instrukcje.
Poniższy blok kodu tworzy Circuit, który generuje *stan Bella* — stan, w którym dwa Qubit są ze sobą w pełni splątane.

<Admonition type="note" title="Uwaga: kolejność bitów">
  Qiskit SDK używa numeracji bitów LSb 0, gdzie $n$-ta cyfra ma wartość $1 \ll n$ lub $2^n$. Więcej szczegółów znajdziesz w temacie [Kolejność bitów w Qiskit SDK](/guides/bit-ordering).
</Admonition>

```python
from qiskit import QuantumCircuit
from qiskit.quantum_info import SparsePauliOp
from qiskit.transpiler import generate_preset_pass_manager
from qiskit_ibm_runtime import QiskitRuntimeService
from qiskit_ibm_runtime import EstimatorOptions
from qiskit_ibm_runtime import EstimatorV2 as Estimator
from matplotlib import pyplot as plt
# Uncomment the next line if you want to use a simulator:
# from qiskit_ibm_runtime.fake_provider import FakeBelemV2

# Create a new circuit with two qubits
qc = QuantumCircuit(2)

# Add a Hadamard gate to qubit 0
qc.h(0)

# Perform a controlled-X gate on qubit 1, controlled by qubit 0
qc.cx(0, 1)

# Return a drawing of the circuit using MatPlotLib ("mpl").
# These guides are written by using Jupyter notebooks, which
# display the output of the last line of each cell.
# If you're running this in a script, use `print(qc.draw())` to
# print a text drawing.
qc.draw("mpl")
```

![Output of the previous code cell](/docs/images/guides/hello-world/extracted-outputs/930ca3b6-0.svg)

Pełną listę dostępnych operacji znajdziesz w dokumentacji [`QuantumCircuit`](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.QuantumCircuit#quantumcircuit-class).
Tworząc obwody kwantowe, musisz także zastanowić się, jaki rodzaj danych ma być zwracany po wykonaniu. Qiskit oferuje dwa sposoby zwracania danych: możesz uzyskać rozkład prawdopodobieństwa dla wybranego zestawu Qubit lub wartość oczekiwaną obserwabla. Przygotuj swoje zadanie, aby mierzyć Circuit za pomocą jednego z tych dwóch podejść, korzystając z [prymitywów Qiskit](/guides/get-started-with-primitives) (szczegółowo omówionych w [Kroku 3](#step-3-execute-using-the-quantum-primitives)).

Ten przykład mierzy wartości oczekiwane przy użyciu podmodułu `qiskit.quantum_info`, który określa się za pomocą operatorów (obiektów matematycznych reprezentujących działanie lub proces zmieniający stan kwantowy). Poniższy blok kodu tworzy sześć dwuqubitowych operatorów Pauliego: `IZ`, `IX`, `ZI`, `XI`, `ZZ` i `XX`.

```python
# Set up six different observables.

observables_labels = ["IZ", "IX", "ZI", "XI", "ZZ", "XX"]
observables = [SparsePauliOp(label) for label in observables_labels]
```

<Admonition type="note" title="Notacja operatorów">
Tutaj operator taki jak `ZZ` jest skrótem dla iloczynu tensorowego $Z\otimes Z$, co oznacza jednoczesny pomiar Z na Qubit 1 i Z na Qubit 0, dający informacje o korelacji między nimi. Wartości oczekiwane tego typu są zazwyczaj zapisywane jako $\langle Z_1 Z_0 \rangle$.

Jeśli stan jest splątany, pomiar $\langle Z_1 Z_0 \rangle$ powinien różnić się od iloczynu pomiarów $\langle I_1 \otimes Z_0 \rangle \langle Z_1 \otimes I_0 \rangle$. Dla konkretnego stanu splątanego tworzonego przez opisany powyżej Circuit, pomiar $\langle Z_1 Z_0 \rangle$ powinien wynosić 1, a iloczyn $\langle I_1 \otimes Z_0 \rangle \langle Z_1 \otimes I_0 \rangle$ powinien być równy zero.
</Admonition>

### Krok 2. Optymalizacja Circuit i operatorów {#step-2-optimize-the-circuits-and-operators}

Wykonując Circuit na urządzeniu, ważne jest, aby zoptymalizować zestaw instrukcji zawartych w Circuit i zminimalizować całkowitą głębokość (w przybliżeniu liczbę instrukcji) Circuit. Dzięki temu uzyskasz jak najlepsze wyniki poprzez ograniczenie wpływu błędów i szumów. Ponadto instrukcje w Circuit muszą być zgodne z [architekturą zestawu instrukcji (ISA)](/guides/transpile#instruction-set-architecture) urządzenia Backend i muszą uwzględniać bramki bazowe oraz łączność Qubit tego urządzenia.

Poniższy kod tworzy instancję rzeczywistego urządzenia, do którego zostanie przesłane zadanie, i przekształca Circuit oraz operatory tak, aby były zgodne z ISA tego Backend. Wymaga to wcześniejszego [zapisania danych uwierzytelniających](/guides/cloud-setup).

```python
service = QiskitRuntimeService()

backend = service.least_busy(simulator=False, operational=True)

# Convert to an ISA circuit and layout-mapped observables.
pm = generate_preset_pass_manager(backend=backend, optimization_level=1)
isa_circuit = pm.run(qc)

isa_circuit.draw("mpl", idle_wires=False)
```

### Krok 3. Wykonanie przy użyciu kwantowych prymitywów {#step-3-execute-using-the-quantum-primitives}

Komputery kwantowe mogą generować losowe wyniki, dlatego zazwyczaj zbiera się próbkę wyjść, uruchamiając Circuit wiele razy. Możesz oszacować wartość obserwabla przy użyciu klasy `Estimator`. `Estimator` jest jednym z dwóch [prymitywów](/guides/get-started-with-primitives); drugim jest `Sampler`, który służy do pobierania danych z komputera kwantowego. Obiekty te posiadają metodę `run()`, która wykonuje wybrany zestaw Circuit, obserwabli i parametrów (jeśli dotyczy) przy użyciu [bloku ujednoliconego prymitywu (PUB)](/guides/primitives#sampler).

```python
# Construct the Estimator instance.

estimator = Estimator(mode=backend)
estimator.options.resilience_level = 1
estimator.options.default_shots = 5000

mapped_observables = [
    observable.apply_layout(isa_circuit.layout) for observable in observables
]

# One pub, with one circuit to run against five different observables.
job = estimator.run([(isa_circuit, mapped_observables)])

# Use the job ID to retrieve your job data later
print(f">>> Job ID: {job.job_id()}")
```

```text
>>> Job ID: d5k96q4jt3vs73ds5tgg
```

## Skalowanie do dużej liczby Qubitów {#scale-to-large-numbers-of-qubits}

W obliczeniach kwantowych praca na skalę użytkową ma kluczowe znaczenie dla postępu w tej dziedzinie. Wymaga ona wykonywania obliczeń na znacznie większą skalę — z Circuit, które mogą wykorzystywać ponad 100 Qubitów i ponad 1000 Gate. Ten przykład pokazuje, jak wykonywać pracę na skalę użytkową na procesorach QPU IBM&reg; poprzez tworzenie i analizę stanu GHZ złożonego ze 100 Qubitów. Wykorzystuje przepływ pracy Qiskit patterns i kończy się pomiarem wartości oczekiwanej $\langle Z_0 Z_i \rangle$ dla każdego Qubitu.

### Krok 1. Sformułowanie problemu {#step-1-map-the-problem}

Napisz funkcję zwracającą `QuantumCircuit`, który przygotowuje $n$-Qubitowy stan GHZ (będący rozszerzeniem stanu Bella), a następnie użyj tej funkcji do przygotowania 100-Qubitowego stanu GHZ i zebrania obserwabli do zmierzenia.

```python
def get_qc_for_n_qubit_GHZ_state(n: int) -> QuantumCircuit:
    """This function will create a qiskit.QuantumCircuit (qc) for an n-qubit GHZ state.

    Args:
        n (int): Number of qubits in the n-qubit GHZ state

    Returns:
        QuantumCircuit: Quantum circuit that generate the n-qubit GHZ state, assuming all qubits start in the 0 state
    """
    if isinstance(n, int) and n >= 2:
        qc = QuantumCircuit(n)
        qc.h(0)
        for i in range(n - 1):
            qc.cx(i, i + 1)
    else:
        raise Exception("n is not a valid input")
    return qc

# Create a new circuit with two qubits (first argument) and two classical
# bits (second argument)
n = 100
qc = get_qc_for_n_qubit_GHZ_state(n)
```

Następnie przypisz interesujące operatory. W tym przykładzie użyto operatorów `ZZ` między Qubitami, aby zbadać zachowanie wraz ze wzrostem odległości między nimi. Coraz mniej dokładne (zaburzone) wartości oczekiwane pomiędzy odległymi Qubitami ujawnią poziom obecnego szumu.

### Krok 4. Przetwarzanie końcowe wyników {#step-4-post-process-results}

Po zakończeniu zadania narysuj wykres wyników i zauważ, że $\langle Z_0 Z_i \rangle$ maleje wraz ze wzrostem $i$, choć w idealnej symulacji wszystkie $\langle Z_0 Z_i \rangle$ powinny wynosić 1.

```python
# data
data = list(range(1, len(operators) + 1))  # Distance between the Z operators
result = job.result()[0]
values = result.data.evs  # Expectation value at each Z operator.
values = [
    v / values[0] for v in values
]  # Normalize the expectation values to evaluate how they decay with distance.

# plotting graph
plt.plot(data, values, marker="o", label="100-qubit GHZ state")
plt.xlabel("Distance between qubits $i$")
plt.ylabel(r"$\langle Z_i Z_0 \rangle / \langle Z_1 Z_0 \rangle $")
plt.legend()
plt.show()
```

Ten przewodnik demonstruje, jak używać pakietu `qiskit-addon-mpf`, wykorzystując jako przykład ewolucję czasową modelu Isinga. Dzięki temu pakietowi możesz zbudować Formułę Wieloproduktową (MPF), która pozwala osiągnąć mniejszy błąd Trottera przy pomiarach obserwowalnych. Dostępne narzędzia umożliwiają wyznaczenie wag wybranej MPF, które następnie można wykorzystać do ponownego łączenia oszacowanych wartości oczekiwanych z kilku obwodów ewolucji czasowej, z których każdy ma inną liczbę kroków Trottera.

Zacznij od rozważenia hamiltonianu modelu Isinga z 10 węzłami:

$$ H_{\text{Ising}} = \sum_{i=1}^9 J_{i,(i+1)}Z_iZ_{i+1} + \sum_{i=1}^{10} h_i X_i $$

gdzie $J_{i,(i+1)}$ to siła sprzężenia, a $h_i$ to siła zewnętrznego pola magnetycznego. Aby sformułować problem, obserwowalną do zmierzenia będzie całkowite namagnesowanie układu

$$ \langle M \rangle = \sum_{i=1}^{10} \langle Z_i \rangle. $$

Poniższy fragment kodu przygotowuje hamiltonian łańcucha Isinga przy użyciu pakietu `qiskit-addon-utils` i definiuje obserwowalną, która będzie mierzona.

```python
from qiskit.transpiler import CouplingMap
from qiskit.synthesis import SuzukiTrotter
from qiskit.quantum_info import SparsePauliOp
from qiskit.primitives import StatevectorEstimator
from qiskit.providers.fake_provider import GenericBackendV2
from qiskit.transpiler import generate_preset_pass_manager
from qiskit_addon_utils.problem_generators import (
    generate_xyz_hamiltonian,
    generate_time_evolution_circuit,
)
from qiskit_addon_mpf.costs import (
    setup_exact_problem,
    setup_sum_of_squares_problem,
)
from qiskit_addon_mpf.static import setup_static_lse

from scipy.linalg import expm
import numpy as np

# Generate some coupling map to use for this example
coupling_map = CouplingMap.from_line(10, bidirectional=False)

# Get a qubit operator describing the Ising field model
hamiltonian = generate_xyz_hamiltonian(
    coupling_map,
    coupling_constants=(0.0, 0.0, 1.0),
    ext_magnetic_field=(0.4, 0.0, 0.0),
)
print(f"Hamiltonian:\n {hamiltonian}\n")

L = coupling_map.size()
observable = SparsePauliOp.from_sparse_list(
    [("Z", [i], 1 / L / 2) for i in range(L)], num_qubits=L
)
print(f"Observable:\n {observable}")
```

Następnie przygotuj MPF. Pierwszym wyborem jest to, czy współczynniki będą statyczne (niezależne od czasu) czy dynamiczne; ten samouczek korzysta ze statycznej MPF. Kolejny wybór to zestaw wartości $k_j$. Określa on, ile wyrazów będzie w MPF, a także ile kroków Trottera każdy wyraz używa do symulacji ewolucji czasowej. Wybór wartości $k_j$ ma charakter heurystyczny, dlatego musisz samodzielnie znaleźć „dobry" zestaw wartości $k_j$. Wskazówki dotyczące znajdowania dobrego zestawu wartości znajdziesz na końcu [strony z pierwszymi krokami](/guides/qiskit-addons-mpf#determine-mpf-coefficients).

Gdy wartości $k_j$ zostaną wyznaczone, możesz przygotować układ równań $Ax=b$ do rozwiązania. Macierz $A$ jest również wyznaczana przez formułę produktową. Do wyboru są tutaj jej *rząd*, który w tym przykładzie wynosi $2$, a także to, czy formuła produktowa powinna być *symetryczna* — co w tym przykładzie jest ustawione na `True`. Poniższy fragment kodu wybiera łączny czas ewolucji układu, wartości $k_j$ oraz układ równań do rozwiązania, korzystając z metody [`qiskit_addon_mpf.static.setup_static_lse`](../api/qiskit-addon-mpf/static).

```python
time = 8.0
trotter_steps = (8, 12, 19)

lse = setup_static_lse(trotter_steps, order=2, symmetric=True)
print(lse)
```

```text

```

Po zinstancjonowaniu liniowego układu równań można go rozwiązać dokładnie lub przy użyciu modelu przybliżonego opartego na sumie kwadratów (lub normie Frobeniusa dla współczynników dynamicznych; więcej informacji znajdziesz w [dokumentacji API](../api/qiskit-addon-mpf/costs)). Wybór modelu przybliżonego zazwyczaj pojawia się wtedy, gdy norma współczynników dla wybranego zestawu wartości $k_j$ jest uznana za zbyt dużą i nie można wybrać innego zestawu wartości $k_j$. Poniższy przewodnik demonstruje obie metody w celu porównania wyników.

```python
model_exact, coeffs_exact = setup_exact_problem(lse)
model_approx, coeffs_approx = setup_sum_of_squares_problem(
    lse, max_l1_norm=3.0
)
model_exact.solve()
model_approx.solve()
print(f"Exact solution: {coeffs_exact.value}")
print(f"Approximate solution: {coeffs_approx.value}")
```

```text
Approximate solution: [-0.40454257  0.57553173  0.8290123 ]
```

## Konfiguracja i wykonanie obwodów Trottera {#set-up-and-execute-trotter-circuits}

```python
circuits = []
```

Po skonstruowaniu tych obwodów możesz je poddać transpilacji i wykonać na QPU. W tym przykładzie użyjemy jednego z symulatorów bez szumu, aby zademonstrować, jak błąd Trottera jest redukowany.

```python
backend = GenericBackendV2(num_qubits=10)
transpiler = generate_preset_pass_manager(
    optimization_level=2, backend=backend
)

transpiled_circuits = [transpiler.run(circ) for circ in circuits]

estimator = StatevectorEstimator()
job = estimator.run([(circ, observable) for circ in transpiled_circuits])
result = job.result()

mpf_evs = [res.data.evs for res in result]
print(mpf_evs)
```

## Rekonstrukcja wyników {#reconstruct-results}

Teraz, gdy obwody zostały wykonane, rekonstrukcja wyników jest stosunkowo prosta. Jak wspomniano na stronie [przeglądu MPF](./qiskit-addons-mpf#theoretical-background), nasza obserwowalna jest rekonstruowana przez ważoną sumę

$$ \langle M \rangle = \sum_j x_j \langle M_j \rangle. $$

gdzie $x_j$ to wyznaczone przez nas współczynniki, a $\langle M_j \rangle$ to oszacowanie obserwowalnej $\sum_i \langle Z_i \rangle$ dla każdego obwodu. Możemy następnie porównać uzyskane wyniki z wartością dokładną, korzystając z pakietu `scipy.linalg`.

```python
exp_H = expm(-1j * time * hamiltonian.to_matrix())
initial_state = np.zeros(exp_H.shape[0])
initial_state[0] = 1.0

time_evolved_state = exp_H @ initial_state
exact_obs = (
    time_evolved_state.conj() @ observable.to_matrix() @ time_evolved_state
)

# Print out the different observable measurements
print(f"Exact value: {exact_obs.real}")
print(f"PF with 19 steps: {mpf_evs[-1]}")
print(f"MPF using exact solution: {mpf_evs @ coeffs_exact.value}")
print(f"MPF using approximate solution: {mpf_evs @ coeffs_approx.value}")
```

```text
Exact value: 0.4006024248789992
PF with 19 steps: 0.3864990619977402
MPF using exact solution: 0.3954847855979902
MPF using approximate solution: 0.4299121425348959
```

Tutaj możesz zobaczyć, że MPF zmniejszyła błąd Trottera w porównaniu z błędem uzyskanym przy użyciu jednej PF z $k_j=19$. Jednak model przybliżony dał wartość oczekiwaną gorszą niż model dokładny. Pokazuje to, jak ważne jest stosowanie ścisłych kryteriów zbieżności w modelu przybliżonym oraz znalezienie „dobrego" zestawu wartości $k_j$.

# Używanie Qiskit Code Assistant w trybie lokalnym {#uzywanie-qiskit-code-assistant-w-trybie-lokalnym}

Dowiedz się, jak zainstalować, skonfigurować i używać dowolnego modelu Qiskit Code Assistant na swoim lokalnym komputerze.

<Admonition type="note" title="Uwagi">
    - Qiskit Code Assistant jest w fazie wersji zapoznawczej i może ulec zmianie.
    - Jeśli masz uwagi lub chcesz skontaktować się z zespołem deweloperów, użyj [kanału Qiskit Slack Workspace](https://qiskit.enterprise.slack.com/archives/C07LYA6PL83) lub powiązanych publicznych repozytoriów GitHub.
</Admonition>

## Szybki start (zalecany) {#szybki-start-zalecany}

Najprostszym sposobem na rozpoczęcie pracy z Qiskit Code Assistant w trybie lokalnym jest użycie automatycznych skryptów konfiguracyjnych dla rozszerzenia VS Code lub JupyterLab. Skrypty te automatycznie zainstalują [Ollama](https://ollama.com/), aby uruchamiać modele językowe, pobiorą zalecany model i skonfigurują rozszerzenie.

### Konfiguracja rozszerzenia VS Code {#konfiguracja-rozszerzenia-vs-code}

Uruchom następujące polecenie w terminalu:

```bash
bash <(curl -fsSL https://raw.githubusercontent.com/Qiskit/qiskit-code-assistant-vscode/main/setup_local.sh)
```

Skrypt wykonuje następujące kroki:
- Instaluje Ollama (jeśli nie jest jeszcze zainstalowane)
- Pobiera i konfiguruje zalecany model Qiskit Code Assistant
- Konfiguruje rozszerzenie VS Code do pracy z lokalnym wdrożeniem

### Konfiguracja rozszerzenia JupyterLab {#konfiguracja-rozszerzenia-jupyterlab}

Uruchom następujące polecenie w terminalu:

```bash
bash <(curl -fsSL https://raw.githubusercontent.com/Qiskit/qiskit-code-assistant-jupyterlab/main/setup_local.sh)
```

Ten skrypt:
- Instaluje Ollama (jeśli nie jest jeszcze zainstalowane)
- Pobiera i konfiguruje zalecany model Qiskit Code Assistant
- Konfiguruje rozszerzenie JupyterLab do pracy z lokalnym wdrożeniem

<summary>Ręczne wdrożenie modeli Qiskit Code Assistant lokalnie przez Ollama</summary>

### Korzystanie z aplikacji Ollama {#using-the-ollama-application}

Aplikacja Ollama zapewnia proste rozwiązanie do lokalnego uruchamiania modeli językowych. Jest łatwa w użyciu, a jej interfejs wiersza poleceń sprawia, że cały proces konfiguracji, zarządzanie modelami i interakcja są całkiem proste. Idealnie nadaje się do szybkiego eksperymentowania oraz dla użytkowników, którzy chcą uniknąć zbędnych szczegółów technicznych.

#### Instalacja Ollama {#instalacja-ollama}

1. Pobierz [aplikację Ollama](https://ollama.com/download)
1. Zainstaluj pobrany plik
1. Uruchom zainstalowaną aplikację Ollama

    <Admonition type="info">Aplikacja działa poprawnie, gdy ikona Ollama pojawi się na pasku menu pulpitu. Możesz też sprawdzić, czy usługa działa, przechodząc na adres `http://localhost:11434/`.</Admonition>

1. Wypróbuj Ollama w terminalu i zacznij uruchamiać modele. Na przykład:

    ```
    ollama run hf.co/Qiskit/Qwen2.5-Coder-14B-Qiskit
    ```

#### Konfiguracja Ollama z integracją Hugging Face Hub {#konfiguracja-ollama-z-integracja-hugging-face-hub}

[Integracja Ollama/Hugging Face Hub](https://huggingface.co/docs/hub/ollama) umożliwia interakcję z modelami hostowanymi na Hugging Face Hub bez konieczności tworzenia nowego pliku modelfile ani ręcznego pobierania plików GGUF lub safetensors. Domyślne pliki `template` i `params` są już dołączone do modelu na Hugging Face Hub.

1. Upewnij się, że aplikacja Ollama działa.

1. Przejdź do strony wybranego modelu i skopiuj adres URL. Na przykład: https://huggingface.co/Qiskit/Qwen2.5-Coder-14B-Qiskit-GGUF.

1. W terminalu uruchom polecenie:

    ```
    ollama run hf.co/Qiskit/Qwen2.5-Coder-14B-Qiskit
    ```

Możesz używać modelu `hf.co/Qiskit/Qwen2.5-Coder-14B-Qiskit` lub dowolnego z innych aktualnie zalecanych oficjalnych modeli GGUF: `hf.co/Qiskit/mistral-small-3.2-24b-qiskit-GGUF` lub `hf.co/Qiskit/granite-3.3-8b-qiskit-GGUF`.

#### Konfiguracja Ollama z ręcznie pobranym modelem GGUF Qiskit Code Assistant {#konfiguracja-ollama-z-recznie-pobranym-modelem-gguf-qiskit-code-assistant}

Jeśli ręcznie pobrałeś model GGUF, taki jak https://huggingface.co/Qiskit/Qwen2.5-Coder-14B-Qiskit-GGUF, i chcesz eksperymentować z różnymi szablonami i parametrami, możesz wykonać poniższe kroki, aby załadować go do lokalnej aplikacji Ollama.

1. Utwórz plik `Modelfile` o następującej zawartości, pamiętając o zaktualizowaniu `<PATH-TO-GGUF-FILE>` do rzeczywistej ścieżki pobranego modelu.

    ```
    FROM <PATH-TO-GGUF-FILE>
    TEMPLATE """{{ if .System }}
    System:
    {{ .System }}

    {{ end }}{{ if .Prompt }}Question:
    {{ .Prompt }}

    {{ end }}Answer:
    ```python{{ .Response }}
    """

    PARAMETER stop "Question:"
    PARAMETER stop "Answer:"
    PARAMETER stop "System:"
    PARAMETER stop "```"

    PARAMETER temperature 0
    PARAMETER top_k 1
    ```

1. Run the following command to create a custom model instance based on the `Modelfile`.

    ```
    ollama create Qwen2.5-Coder-14B-Qiskit -f ./path-to-model-file
    ```

    <Admonition type="note">This process may take some time for Ollama to read the model file, initialize the model instance, and configure it according to the specifications provided.</Admonition>

#### Run the Qiskit Code Assistant model manually downloaded in Ollama

After the `Qwen2.5-Coder-14B-Qiskit` model has been set up in Ollama, run the following command to launch the model and interact with it in the terminal (in chat mode).

```
ollama run Qwen2.5-Coder-14B-Qiskit
```

Some useful commands:

- `ollama list` - List models on your computer
- `ollama rm Qwen2.5-Coder-14B-Qiskit` - Delete the model
- `ollama show Qwen2.5-Coder-14B-Qiskit` - Show model information
- `ollama stop Qwen2.5-Coder-14B-Qiskit` - Stop a model that is currently running
- `ollama ps` - List which models are currently loaded

<summary>Manually deploy the Qiskit Code Assistant models in local through the `llama-cpp-python` package</summary>

<span id="use-llama-cpp-python"></span>
An alternative to the Ollama application is the `llama-cpp-python` package, which is a Python binding for `llama.cpp`. It gives you more control and flexibility to run the GGUF model locally, and is ideal for users who wish to integrate the local model in their workflows and Python applications.

1. Install [`llama-cpp-python`](https://pypi.org/project/llama-cpp-python/)
2. Interact with the model from within your application using `llama_cpp`. For example:

```python
from llama_cpp import Llama

model_path = <PATH-TO-GGUF-FILE>

model = Llama(
        model_path,
        seed=17,
        n_ctx=10000,
        n_gpu_layers=37, # to offload in gpu, but put 0 if all in cpu
    )

input = 'Generate a quantum circuit with 2 qubits'
raw_pred = model(input)["choices"][0]["text"]
```

You can also add text generation parameters to the model to customize the inference:

```python
generation_kwargs = {
        "max_tokens": 512,
        "echo": False, # Echo the prompt in the output
        "top_k": 1
    }

raw_pred = model(input, **generation_kwargs)["choices"][0]["text"]
```

### Use the `llama.cpp` library

Another alternative is to use `llama.cpp`, an open-source library for performing LLM inference on a CPU with minimal setup.
It provides low-level control over the model execution and is typically run from the command line, pointing to a local GGUF model file.

There are several ways to install `llama.cpp` on your machine:

- Install llama.cpp using [brew, nix, or winget](https://github.com/ggml-org/llama.cpp/blob/master/docs/install.md)
- Run with Docker: See out the [Docker documentation by `llama.cpp` team](https://github.com/ggml-org/llama.cpp/blob/master/docs/docker.md)
- Download pre-built binaries from the [releases page](https://github.com/ggml-org/llama.cpp/releases)
- Build from source by [cloning this repository](https://github.com/ggml-org/llama.cpp/blob/master/docs/build.md)

Once installed, you can use `llama.cpp` to interact with GGUF models in conversation mode as follows:
```bash
# Use a local model file
llama-cli -m my_model.gguf -cnv

# Or download and run a model directly from Hugging Face
llama-cli -hf Qiskit/Qwen2.5-Coder-14B-Qiskit-GGUF -cnv
```

You can also launch an OpenAI-compatible API server for the model in the following way:
```bash
llama-server -hf Qiskit/Qwen2.5-Coder-14B-Qiskit-GGUF
```

#### Advanced parameters
With the `llama-cli` program, you can control the model generation using command-line options. For example, you can provide an initial “system” prompt using the `-p/--prompt` flag. In conversation mode (`-cnv`), this initial prompt acts as the system message. Otherwise, you can simply prepend any desired instruction to your prompt text. You can also adjust sampling parameters - for instance: temperature (`--temp`), top-k (`--top-k`), top-p (`--top-p`), repetition penalty (`--repeat-penalty`), and the seed to use (`--seed`). The following is an example invocation using these options:

```bash
llama-cli -hf Qiskit/Qwen2.5-Coder-14B-Qiskit-GGUF \
  -p "You are a friendly assistant." -cnv \
  --temp 0.7 \
  --top-k 50 \
  --top-p 0.95 \
  --repeat-penalty 1.1 \
  --seed 42
```

Aby zapewnić prawidłowe działanie naszych modeli Qiskit, zalecamy używanie promptu systemowego udostępnionego w naszych repozytoriach HF GGUF: prompt systemowy dla [mistral-small-3.2-24b-qiskit-GGUF](https://huggingface.co/Qiskit/mistral-small-3.2-24b-qiskit-GGUF/blob/main/system), [Qwen2.5-Coder-14B-Qiskit-GGUF](https://huggingface.co/Qiskit/Qwen2.5-Coder-14B-Qiskit-GGUF/blob/main/system), [granite-3.3-8b-qiskit-GGUF](https://huggingface.co/Qiskit/granite-3.3-8b-qiskit-GGUF/blob/main/system) i [granite-3.2-8b-qiskit-GGUF.](https://huggingface.co/Qiskit/granite-3.2-8b-qiskit-GGUF/blob/main/system)

### Połączenie z rozszerzeniem Qiskit Code Assistant dla VS Code {#polaczenie-z-rozszerzeniem-qiskit-code-assistant-dla-vs-code}

Za pomocą rozszerzenia Qiskit Code Assistant dla VS Code możesz wchodzić w interakcję z modelem i uzupełniać kod podczas pisania. Sprawdza się to dobrze dla użytkowników szukających pomocy przy pisaniu kodu Qiskit w swoich aplikacjach Python.

1. Zainstaluj [rozszerzenie Qiskit Code Assistant dla VS Code](/guides/qiskit-code-assistant-vscode).
1. W VS Code przejdź do **User Settings** i ustaw **Qiskit Code Assistant: Url** na adres URL lokalnego wdrożenia Ollama (na przykład `http://localhost:11434`).
1. Przeładuj VS Code, przechodząc do **View > Command Palette...** i wybierając **Developer: Reload Window**.

Model Qiskit Code Assistant skonfigurowany w Ollama powinien pojawić się na pasku statusu i jest gotowy do użycia.

### Połączenie z rozszerzeniem Qiskit Code Assistant dla JupyterLab {#polaczenie-z-rozszerzeniem-qiskit-code-assistant-dla-jupyterlab}

Za pomocą rozszerzenia Qiskit Code Assistant dla JupyterLab możesz wchodzić w interakcję z modelem i uzupełniać kod bezpośrednio w Jupyter Notebook. Użytkownicy, którzy głównie pracują z Jupyter Notebooks, mogą skorzystać z tego rozszerzenia, aby jeszcze bardziej usprawnić pracę z kodem Qiskit.

1. Zainstaluj [rozszerzenie Qiskit Code Assistant dla JupyterLab](/guides/qiskit-code-assistant-jupyterlab).
1. W JupyterLab przejdź do **Settings Editor** i ustaw **Qiskit Code Assistant Service API** na adres URL lokalnego wdrożenia Ollama (na przykład `http://localhost:11434`).

Model Qiskit Code Assistant skonfigurowany w Ollama powinien pojawić się na pasku statusu i jest gotowy do użycia.

</details>

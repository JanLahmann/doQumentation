# Zdarzenia śledzenia aktywności dla usługi IBM Quantum System {#activity-tracking-events-for-the-ibm-quantum-system-service}

Usługi IBM Cloud&reg;, takie jak usługa IBM&reg; Quantum System, generują zdarzenia śledzenia aktywności.

Zdarzenia śledzenia aktywności raportują działania, które zmieniają stan usługi w IBM Cloud. Możesz używać tych zdarzeń do badania nieprawidłowej aktywności i krytycznych działań oraz do spełniania wymagań audytu regulacyjnego.

Możesz używać IBM Cloud Activity Tracker Event Routing — usługi platformy — do kierowania zdarzeń audytowych na swoim koncie do wybranych miejsc docelowych poprzez konfigurowanie celów i tras określających, gdzie są wysyłane zdarzenia śledzenia aktywności. Więcej informacji znajdziesz w artykule [About IBM Cloud Activity Tracker Events Routing](https://cloud.ibm.com/docs/atracker?topic=atracker-about).

Możesz używać IBM Cloud Logs do wizualizowania i alertowania o zdarzeniach generowanych na twoim koncie i kierowanych przez IBM Cloud Activity Tracker Event Routing do instancji IBM Cloud Logs.

## Przeglądanie zdarzeń śledzenia aktywności dla usługi IBM Quantum System {#viewing-activity-tracking-events-for-the-ibm-quantum-system-service}

Możesz używać IBM Cloud Logs do wizualizowania i alertowania o zdarzeniach generowanych na twoim koncie i kierowanych przez IBM Cloud Activity Tracker Event Routing do instancji IBM Cloud Logs.

### Uruchamianie IBM Cloud Logs ze strony Observability {#launching-ibm-cloud-logs-from-the-observability-page}

Informacje o uruchamianiu interfejsu użytkownika IBM Cloud Logs znajdziesz w sekcji [Launching the UI](https://cloud.ibm.com/docs/cloud-logs?topic=cloud-logs-instance-launch#instance-launch-cloud-ui) w dokumentacji IBM Cloud Logs.

| Działanie | Wyzwalane przez |
| -- | -- |
| `quantum-computing.direct-access-backend-configuration.read` | Pobieranie konfiguracji backendu (`GET /v1/backends/{backend-name}/configuration`) |
| `quantum-computing.direct-access-backend-properties.read` | Pobieranie właściwości backendu (`GET /v1/backends/{backend-name}/properties`) |
| `quantum-computing.direct-access-backend.list` | Pobieranie listy backendów (`GET /v1/backends`) |
| `quantum-computing.direct-access-backend.read` | Pobieranie szczegółów backendu (`GET /v1/backends/{backend-name}`) |
| `quantum-computing.direct-access-job.cancel` | Anulowanie zadania (`POST /v1/jobs/{job-id}/cancel`) |
| `quantum-computing.direct-access-job.create` | Uruchomienie zadania (`POST /v1/jobs`) |
| `quantum-computing.direct-access-job.delete` | Usunięcie zadania (`DELETE /v1/jobs/{job-id}`) |
| `quantum-computing.direct-access-job.list` | Pobieranie zadań (`GET /v1/jobs`) |
| `quantum-computing.direct-access-job.read` | Pobieranie zadania po ID (`GET /v1/jobs/{job-id}`) |
| `quantum-computing.direct-access-lane-configuration.list` | Pobieranie konfiguracji pasów backendu (`GET /v1/backends/{backend-name}/lanes`) |

# Logowanie dla Qiskit Runtime {#logging-for-qiskit-runtime}

Usługi IBM Cloud&reg;, takie jak Qiskit Runtime, generują logi platformy, których możesz używać do badania nieprawidłowej aktywności i krytycznych działań na swoim koncie oraz rozwiązywania problemów.

Możesz skorzystać z [IBM Cloud Logs Routing,](https://cloud.ibm.com/docs/logs-router?topic=logs-router-about) która jest usługą platformy, aby kierować logi platformy na swoim koncie do wybranego miejsca docelowego, konfigurując dzierżawcę (tenant) określającego, dokąd są wysyłane logi platformy. Więcej informacji znajdziesz w artykule [Informacje o Logs Routing.](https://cloud.ibm.com/docs/logs-router?topic=logs-router-about)

Możesz skorzystać z [IBM Cloud Logs](https://cloud.ibm.com/docs/cloud-logs), aby wizualizować logi platformy generowane na swoim koncie i kierowane przez IBM Cloud Logs Routing do instancji IBM Cloud Logs oraz ustawiać alerty na ich podstawie.

## Lokalizacje, w których generowane są logi platformy {#locations-where-platform-logs-are-generated}

IBM Cloud Container Registry generuje logi platformy wyświetlane w Twoich instancjach logowania. Poniższa tabela zawiera listę lokalizacji, w których włączone jest automatyczne zbieranie logów usługi Container Registry.

| Lokalizacja                     | Czy logi usług platformy są dostępne? |
|---------------------------------|---------------------------------------|
| Waszyngton, D.C. (us-east)      |                  Tak                  |
| Frankfurt (eu-de)               |                  Tak                  |

## Generowane logi platformy {#platform-logs-that-are-generated}

IBM Qiskit Runtime generuje logi usług platformy wyświetlane w Twoich instancjach logowania w następujących konkretnych przypadkach:

- Gdy zmienia się stan zadania (job).

## Przeglądanie logów ze strony Observability {#viewing-logs-from-the-observability-page}

Informacje o uruchamianiu interfejsu IBM Cloud Logs znajdziesz w sekcji **Launching the UI** w [dokumentacji IBM Cloud Logs.](https://cloud.ibm.com/docs/cloud-logs?topic=cloud-logs-instance-launch)

## Pola według typu logu {#fields-by-log-type}

Informacje o polach zawartych w każdym logu platformy znajdziesz w artykule [Fields that are included in platform logs.](https://cloud.ibm.com/docs/logs-router?topic=logs-router-about-platform-logs#about-platform-logs-2)

Poniższe pola są zawarte w rekordzie logu.

| Pole            | Typ      | Opis                                                                                    |
|-----------------|----------|-----------------------------------------------------------------------------------------|
| logSourceCRN    | Wymagane | Definiuje konto i instancję dziennika przepływu, w której publikowany jest log.         |
| saveServiceCopy | Wymagane | Określa, czy IBM zapisuje kopię rekordu do celów operacyjnych.                          |
| message         | Wymagane | Opis wygenerowanego logu.                                                               |

# Pierwsze kroki z prymitywami backendu {#get-started-with-the-backend-primitives}

W przeciwieństwie do prymitywów specyficznych dla dostawcy, prymitywy backendu to ogólne implementacje, które można stosować z dowolnym obiektem `backend`, o ile implementuje on interfejs [`BackendV2`](https://docs.quantum.ibm.com/api/qiskit/qiskit.providers.BackendV2). Niektórzy dostawcy implementują prymitywy natywnie. Szczegóły znajdziesz na [stronie ekosystemu Qiskit](https://www.ibm.com/quantum/ecosystem?tag=provider).

<span id="estimator"></span>
## Pierwsze kroki z prymitywem backendu Estimator {#get-started-with-the-estimator-backend-primitive}

Prymityw Estimator można uruchamiać z dowolnym dostawcą przy użyciu klasy [`qiskit.primitives.BackendEstimatorV2`](https://docs.quantum.ibm.com/api/qiskit/qiskit.primitives.BackendEstimatorV2). Nie oferuje on jednak żadnej implementacji mitygacji błędów pomiarowych ani bramkowych „z automatu", ponieważ prymitywy backendu są zaprojektowane do lokalnego uruchamiania na maszynie użytkownika.

### Przykład: {#example}

```python
from qiskit.primitives import BackendEstimatorV2
from <some_qiskit_provider> import QiskitProvider

provider = QiskitProvider()
backend = provider.get_backend('backend_name')
estimator = BackendEstimatorV2(backend)
```

<span id="sampler"></span>
## Pierwsze kroki z prymitywem backendu Sampler {#get-started-with-the-sampler-backend-primitive}

Prymityw Sampler można uruchamiać z dowolnym dostawcą przy użyciu [`qiskit.primitives.BackendSamplerV2`](https://docs.quantum.ibm.com/api/qiskit/qiskit.primitives.BackendSamplerV2). Wymaga on jednak backendu obsługującego opcję `memory`.

### Przykład: {#example-1}

```python
from qiskit.primitives import BackendSamplerV2
from <some_qiskit_provider> import QiskitProvider

provider = QiskitProvider()
backend = provider.get_backend('backend_name')
sampler = BackendSamplerV2(backend)
```

# Zdarzenia śledzenia aktywności dla Qiskit Runtime {#activity-tracking-events-for-qiskit-runtime}

Usługi IBM Cloud&reg;, takie jak Qiskit Runtime, generują zdarzenia śledzenia aktywności.

Zdarzenia śledzenia aktywności rejestrują działania, które zmieniają stan usługi w IBM Cloud. Możesz używać tych zdarzeń do badania nieprawidłowej aktywności i krytycznych działań oraz do spełnienia wymagań regulacyjnych dotyczących audytu.

Możesz użyć IBM Cloud Activity Tracker Event Routing – usługi platformy – do kierowania zdarzeń audytowych na swoim koncie do wybranych miejsc docelowych, konfigurując cele i trasy definiujące, dokąd są wysyłane zdarzenia śledzenia aktywności. Więcej informacji znajdziesz w artykule [About IBM Cloud Activity Tracker Events Routing.](https://cloud.ibm.com/docs/atracker?topic=atracker-about)

Możesz używać IBM Cloud Logs do wizualizowania i tworzenia alertów dotyczących zdarzeń generowanych na twoim koncie i kierowanych przez IBM Cloud Activity Tracker Event Routing do instancji IBM Cloud Logs.

## Lokalizacje, w których generowane są zdarzenia śledzenia aktywności {#locations-where-activity-tracking-events-are-generated}

Qiskit Runtime wysyła zdarzenia śledzenia aktywności w następujących regionach:

- Waszyngton ("us-east")
- Frankfurt ("eu-de")

## Wyświetlanie zdarzeń śledzenia aktywności dla Qiskit Runtime {#viewing-activity-tracking-events-for-qiskit-runtime}

Możesz używać IBM Cloud Logs do wizualizowania i tworzenia alertów dotyczących zdarzeń generowanych na twoim koncie i kierowanych przez IBM Cloud Activity Tracker Event Routing do instancji IBM Cloud Logs.

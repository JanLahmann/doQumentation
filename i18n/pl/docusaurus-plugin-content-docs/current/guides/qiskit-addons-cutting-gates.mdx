Ten przewodnik przedstawia dwa działające przykłady cięć bramek z użyciem pakietu `qiskit-addon-cutting`. Pierwszy przykład pokazuje, jak zmniejszyć głębokość Circuit (liczbę instrukcji Circuit) przez cięcie splątujących bramek na nie-sąsiadujących Qubitach, które w przeciwnym razie wiązałyby się z narzutem operacji SWAP podczas transpilacji do sprzętu. Drugi przykład omawia, jak użyć cięcia bramek do zmniejszenia szerokości Circuit (liczby Qubitów) przez podzielenie Circuit na kilka mniejszych Circuit z mniejszą liczbą Qubitów.

Oba przykłady korzystają z ansatzu [`efficient_su2`](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.library.efficient_su2) i rekonstruują tę samą obserwowalną.
## Cięcie bramek w celu zmniejszenia głębokości Circuit {#gate-cutting-to-reduce-circuit-depth}

Poniższy przepływ pracy zmniejsza głębokość Circuit przez cięcie odległych bramek, unikając dużej serii bramek SWAP, które w przeciwnym razie zostałyby wprowadzone.

Zacznij od ansatzu [`efficient_su2`](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.library.efficient_su2) z „kołowym" splątaniem, aby wprowadzić odległe bramki.


```python
import numpy as np
from qiskit.circuit.library import efficient_su2
from qiskit.quantum_info import SparsePauliOp
from qiskit.transpiler import generate_preset_pass_manager
from qiskit_ibm_runtime.fake_provider import FakeManilaV2
from qiskit_ibm_runtime import SamplerV2, Batch
from qiskit_aer.primitives import EstimatorV2
from qiskit_addon_cutting import (
    cut_gates,
    partition_problem,
    generate_cutting_experiments,
    reconstruct_expectation_values,
)

circuit = efficient_su2(num_qubits=4, entanglement="circular")
circuit.assign_parameters([0.4] * len(circuit.parameters), inplace=True)

observable = SparsePauliOp(["ZZII", "IZZI", "-IIZZ", "XIXI", "ZIZZ", "IXIX"])
print(f"Observable: {observable}")
circuit.draw("mpl", scale=0.8)
```

```text
Observable: SparsePauliOp(['ZZII', 'IZZI', 'IIZZ', 'XIXI', 'ZIZZ', 'IXIX'],
              coeffs=[ 1.+0.j,  1.+0.j, -1.+0.j,  1.+0.j,  1.+0.j,  1.+0.j])
```

![Wyjście poprzedniej komórki kodu](/docs/images/guides/qiskit-addons-cutting-gates/extracted-outputs/1551c440-c158-478a-a8fe-86df834c59bd-1.svg)

Każda bramka [`CNOT`](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.library.CXGate) między Qubitami $q_0$ i $q_3$ wprowadza dwie bramki SWAP po transpilacji (zakładając, że Qubity są połączone w linii prostej). Aby uniknąć tego wzrostu głębokości, możesz zastąpić te odległe bramki obiektami [`TwoQubitQPDGate`](https://docs.quantum.ibm.com/api/qiskit-addon-cutting/qpd-two-qubit-qpd-gate) przy użyciu metody [`cut_gates()`](https://docs.quantum.ibm.com/api/qiskit-addon-cutting/qiskit-addon-cutting#cut_gates). Funkcja ta zwraca również listę instancji [`QPDBasis`](../api/qiskit-addon-cutting/qpd-qpd-basis) — po jednej dla każdego rozkładu.

```python
# Find the indices of the distant gates
cut_indices = [
    i
    for i, instruction in enumerate(circuit.data)
    if {circuit.find_bit(q)[0] for q in instruction.qubits} == {0, 3}
]

# Decompose distant CNOTs into TwoQubitQPDGate instances
qpd_circuit, bases = cut_gates(circuit, cut_indices)

qpd_circuit.draw("mpl", scale=0.8)
```

![Wyjście poprzedniej komórki kodu](/docs/images/guides/qiskit-addons-cutting-gates/extracted-outputs/66dc0a14-ab51-4190-9cda-1c0373e91b9e-0.svg)

Po dodaniu instrukcji ciętych bramek podeksperymenty będą miały mniejszą głębokość po transpilacji niż oryginalny Circuit. Poniższy fragment kodu generuje podeksperymenty przy użyciu [`generate_cutting_experiments`](https://docs.quantum.ibm.com/api/qiskit-addon-cutting/qiskit-addon-cutting#generate_cutting_experiments), który przyjmuje Circuit oraz obserwowalną do zrekonstruowania.

<Admonition type="note" title="Uwaga dotycząca liczby próbek">
    Argument `num_samples` określa, ile próbek należy pobrać z quasi-rozkładu prawdopodobieństwa i wyznacza dokładność współczynników używanych do rekonstrukcji. Przekazanie nieskończoności (`np.inf`) zapewni dokładne obliczenie wszystkich współczynników. Przeczytaj dokumentację API na temat [generowania wag](https://docs.quantum.ibm.com/api/qiskit-addon-cutting/qpd#generate_qpd_weights) i [generowania eksperymentów cięcia](https://docs.quantum.ibm.com/api/qiskit-addon-cutting/qiskit-addon-cutting#generate_cutting_experiments), aby uzyskać więcej informacji.
</Admonition>

Po wygenerowaniu podeksperymentów możesz je transpilować i użyć prymitywu `Sampler` do próbkowania rozkładu oraz rekonstrukcji szacowanych wartości oczekiwanych. Poniższy blok kodu generuje, transpiluje i wykonuje podeksperymenty. Następnie rekonstruuje wyniki i porównuje je z dokładną wartością oczekiwaną.

```python
# Generate the subexperiments and sampling coefficients
subexperiments, coefficients = generate_cutting_experiments(
    circuits=qpd_circuit, observables=observable.paulis, num_samples=np.inf
)

# Set a backend to use and transpile the subexperiments
backend = FakeManilaV2()
pass_manager = generate_preset_pass_manager(
    optimization_level=1, backend=backend
)
isa_subexperiments = pass_manager.run(subexperiments)

# Set up the Qiskit Runtime Sampler primitive, submit the subexperiments, and retrieve the results
sampler = SamplerV2(backend)
job = sampler.run(isa_subexperiments, shots=4096 * 3)
results = job.result()

# Reconstruct the results
reconstructed_expval_terms = reconstruct_expectation_values(
    results,
    coefficients,
    observable.paulis,
)

# Apply the coefficients of the original observable
reconstructed_expval = np.dot(reconstructed_expval_terms, observable.coeffs)

estimator = EstimatorV2()
exact_expval = (
    estimator.run([(circuit, observable, [0.4] * len(circuit.parameters))])
    .result()[0]
    .data.evs
)
print(
    f"Reconstructed expectation value: {np.real(np.round(reconstructed_expval, 8))}"
)
print(f"Exact expectation value: {np.round(exact_expval, 8)}")
print(
    f"Error in estimation: {np.real(np.round(reconstructed_expval-exact_expval, 8))}"
)
print(
    f"Relative error in estimation: {np.real(np.round((reconstructed_expval-exact_expval) / exact_expval, 8))}"
)
```

```text
Reconstructed expectation value: 0.49812826
Exact expectation value: 0.50497603
Error in estimation: -0.00684778
Relative error in estimation: -0.0135606
```

<Admonition type="caution" title="Uwaga dotycząca współczynników obserwowalnej">
    Aby dokładnie zrekonstruować wartość oczekiwaną, współczynniki oryginalnej obserwowalnej (które różnią się od współczynników w wyjściu `generate_cutting_experiments()`) muszą być zastosowane do wyjścia rekonstrukcji, ponieważ ta informacja została utracona podczas generowania eksperymentów cięcia lub gdy oberwowalna była rozwijana.

    Zazwyczaj te współczynniki można zastosować przez `numpy.dot()`, jak pokazano powyżej.
</Admonition>
## Cięcie bramek w celu zmniejszenia szerokości Circuit {#gate-cutting-to-reduce-circuit-width}

Ta sekcja demonstruje użycie cięcia bramek do zmniejszenia szerokości Circuit. Zacznij od tego samego ansatzu [`efficient_su2`](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.library.efficient_su2), ale użyj splątania „liniowego".

```python
qc = efficient_su2(4, entanglement="linear", reps=2)
qc.assign_parameters([0.4] * len(qc.parameters), inplace=True)

observable = SparsePauliOp(["ZZII", "IZZI", "-IIZZ", "XIXI", "ZIZZ", "IXIX"])
print(f"Observable: {observable}")

qc.draw("mpl", scale=0.8)
```

```text
Observable: SparsePauliOp(['ZZII', 'IZZI', 'IIZZ', 'XIXI', 'ZIZZ', 'IXIX'],
              coeffs=[ 1.+0.j,  1.+0.j, -1.+0.j,  1.+0.j,  1.+0.j,  1.+0.j])
```

![Wyjście poprzedniej komórki kodu](/docs/images/guides/qiskit-addons-cutting-gates/extracted-outputs/64010a14-8360-47e2-bb77-af9b2e0dbbfc-1.svg)

Następnie wygeneruj *podcircuity* i *podobserwowalnych*, które wykonasz przy użyciu funkcji [`partition_problem()`](https://docs.quantum.ibm.com/api/qiskit-addon-cutting/qiskit-addon-cutting#partition_problem). Funkcja ta przyjmuje Circuit, obserwowalną i opcjonalny schemat podziału, a następnie zwraca podzielone Circuit i obserwowalnych w postaci słownika.

Podział jest zdefiniowany przez ciąg etykiet w postaci `"AABB"`, gdzie każda etykieta w tym ciągu odpowiada Qubitowi o tym samym indeksie w argumencie `circuit`. Qubity dzielące wspólną etykietę partycji są grupowane razem, a wszelkie nielokalnych bramki obejmujące więcej niż jedną partycję będą cięte.

<Admonition type="info" title="Uwaga">
   Argument `observables` funkcji `partition_problem` jest typu [`PauliList`](https://docs.quantum.ibm.com/api/qiskit/qiskit.quantum_info.PauliList). Współczynniki i fazy składników obserwowalnej są ignorowane podczas rozkładu problemu i wykonania podeksperymentów. Mogą być ponownie zastosowane podczas rekonstrukcji wartości oczekiwanej.
</Admonition>

```python
partitioned_problem = partition_problem(
    circuit=qc, partition_labels="AABB", observables=observable.paulis
)
subcircuits = partitioned_problem.subcircuits
subobservables = partitioned_problem.subobservables
bases = partitioned_problem.bases

print(f"Sampling overhead: {np.prod([basis.overhead for basis in bases])}")
print(f"Subobservables: {subobservables}")
subcircuits["A"].draw("mpl", scale=0.8)
```

```text
Sampling overhead: 81.0
Subobservables: {'A': PauliList(['II', 'ZI', 'ZZ', 'XI', 'ZZ', 'IX']), 'B': PauliList(['ZZ', 'IZ', 'II', 'XI', 'ZI', 'IX'])}
```

Kolejnym krokiem jest użycie podcircuitów i podobserwowalnych do wygenerowania *podeksperymentów*, które będą wykonywane na QPU przy użyciu metody [`generate_cutting_experiments`](https://docs.quantum.ibm.com/api/qiskit-addon-cutting/qiskit-addon-cutting#generate_cutting_experiments).

Aby oszacować wartość oczekiwaną pełnego Circuit, wiele podeksperymentów jest generowanych z wspólnego quasi-rozkładu prawdopodobieństwa rozłożonych bramek, a następnie wykonywanych na jednym lub kilku QPU. Liczba próbek pobieranych z tego rozkładu jest kontrolowana przez argument `num_samples`.

Poniższy blok kodu generuje podeksperymenty i wykonuje je przy użyciu prymitywu `Sampler` na lokalnym symulatorze. (Aby uruchomić je na QPU, zmień `backend` na wybrany zasób QPU.)

```python
subexperiments, coefficients = generate_cutting_experiments(
    circuits=subcircuits, observables=subobservables, num_samples=np.inf
)

# Set a backend to use and transpile the subexperiments
backend = FakeManilaV2()
pass_manager = generate_preset_pass_manager(
    optimization_level=1, backend=backend
)
isa_subexperiments = {
    label: pass_manager.run(partition_subexpts)
    for label, partition_subexpts in subexperiments.items()
}

# Submit each partition's subexperiments to the Qiskit Runtime Sampler
# primitive, in a single batch so that the jobs will run back-to-back.
with Batch(backend=backend) as batch:
    sampler = SamplerV2(mode=batch)
    jobs = {
        label: sampler.run(subsystem_subexpts, shots=4096 * 3)
        for label, subsystem_subexpts in isa_subexperiments.items()
    }

# Retrieve results
results = {label: job.result() for label, job in jobs.items()}
```

Na koniec wartość oczekiwana pełnego Circuit jest rekonstruowana przy użyciu metody [`reconstruct_expectation_values`](https://docs.quantum.ibm.com/api/qiskit-addon-cutting/qiskit-addon-cutting#reconstruct_expectation_values).

Poniższy blok kodu rekonstruuje wyniki i porównuje je z dokładną wartością oczekiwaną.

```python
# Get expectation values for each observable term
reconstructed_expval_terms = reconstruct_expectation_values(
    results,
    coefficients,
    subobservables,
)

# Reconstruct final expectation value
reconstructed_expval = np.dot(reconstructed_expval_terms, observable.coeffs)

estimator = EstimatorV2()
exact_expval = (
    estimator.run([(qc, observable, [0.4] * len(qc.parameters))])
    .result()[0]
    .data.evs
)
print(
    f"Reconstructed expectation value: {np.real(np.round(reconstructed_expval, 8))}"
)
print(f"Exact expectation value: {np.round(exact_expval, 8)}")
print(
    f"Error in estimation: {np.real(np.round(reconstructed_expval-exact_expval, 8))}"
)
print(
    f"Relative error in estimation: {np.real(np.round((reconstructed_expval-exact_expval) / exact_expval, 8))}"
)
```

```text
Reconstructed expectation value: 0.53571896
Exact expectation value: 0.56254612
Error in estimation: -0.02682716
Relative error in estimation: -0.04768882
```

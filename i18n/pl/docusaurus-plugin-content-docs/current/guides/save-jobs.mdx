Kwantowe przepływy pracy często trwają długo i mogą być uruchamiane w wielu sesjach. Ponowne uruchomienie jądra Pythona oznacza utratę wszelkich wyników przechowywanych w pamięci. Aby uniknąć utraty danych, możesz zapisywać wyniki do pliku i pobierać wyniki poprzednich zadań z IBM Quantum&reg;, aby następna sesja mogła kontynuować od miejsca, w którym skończyłeś.
## Pobieranie wyników zadań z IBM Quantum {#retrieve-job-results-from-ibm-quantum}

IBM Quantum automatycznie przechowuje wyniki każdego zadania, abyś mógł je pobrać w późniejszym terminie. Korzystaj z tej funkcji, aby kontynuować programy kwantowe po ponownym uruchomieniu jądra i przeglądać poprzednie wyniki. Możesz uzyskać ID zadania programowo za pomocą jego metody `job_id` lub zobaczyć wszystkie przesłane zadania i ich ID na [stronie Workloads](https://quantum.cloud.ibm.com/workloads).

Aby znaleźć zadanie programowo, użyj metody [`QiskitRuntimeService.jobs`](https://docs.quantum.ibm.com/api/qiskit-ibm-runtime/qiskit-runtime-service#jobs). Domyślnie zwraca ona ostatnio przesłane zadania, ale możesz też filtrować zadania według nazwy Backend, daty utworzenia i innych kryteriów. Poniższa komórka wyszukuje zadania przesłane w ciągu ostatnich trzech miesięcy. Argument `created_after` musi być obiektem [`datetime.datetime`](https://docs.python.org/3.8/library/datetime.html#datetime.datetime).

```python
# Added by doQumentation — required packages for this notebook
```

```python
import datetime
from qiskit_ibm_runtime import QiskitRuntimeService

three_months_ago = datetime.datetime.now() - datetime.timedelta(days=90)

service = QiskitRuntimeService()
jobs_in_last_three_months = service.jobs(created_after=three_months_ago)
jobs_in_last_three_months[:3]  # show first three jobs
```

Możesz także filtrować według Backend, stanu zadania, sesji i nie tylko. Więcej informacji znajdziesz w dokumentacji API [`QiskitRuntimeService.jobs`](https://docs.quantum.ibm.com/api/qiskit-ibm-runtime/qiskit-runtime-service#jobs).

Gdy masz już ID zadania, użyj metody [`QiskitRuntimeService.job`](https://docs.quantum.ibm.com/api/qiskit-ibm-runtime/qiskit-runtime-service#job), aby je pobrać.

```python
# Get ID of most recent successful job for demonstration.
# This will not work if you've never successfully run a job.
successful_job = next(
    j for j in service.jobs(limit=1000) if j.status() == "DONE"
)
job_id = successful_job.job_id()
print(job_id)
```

```python
retrieved_job = service.job(job_id)
```

## Zapisywanie wyników na dysk {#save-results-to-disk}

Możesz również chcieć zapisać wyniki na dysk. W tym celu użyj wbudowanej biblioteki JSON Pythona z koderami z Qiskit Runtime.

```python
import json
from qiskit_ibm_runtime import RuntimeEncoder

with open("result.json", "w") as file:
    json.dump(retrieved_job.result(), file, cls=RuntimeEncoder)
```

Następnie możesz załadować tę tablicę z dysku w osobnym jądrze.

```python
from qiskit_ibm_runtime import RuntimeDecoder

with open("result.json", "r") as file:
    result = json.load(file, cls=RuntimeDecoder)

result
```

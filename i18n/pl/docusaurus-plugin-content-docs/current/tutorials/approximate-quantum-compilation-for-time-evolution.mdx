## Wprowadzenie {#background}

Ten samouczek pokazuje, jak zaimplementować **Przybliżoną Kompilację Kwantową** z użyciem sieci tensorowych (AQC-Tensor) z Qiskit, aby poprawić wydajność układów kwantowych. AQC-Tensor kompresuje głębokie układy Trottera do płytszych, bardziej przyjaznych dla sprzętu układów, zachowując przy tym dokładność symulacji.

### Jak działa AQC-Tensor {#how-aqc-tensor-works}

Rozważ symulację Hamiltonianu $H$ przez całkowity czas $t$ z użyciem $k$ kroków Trottera. Pełny układ Trottera to:

$$
U_{\text{full}} = \left[U_{\text{Trotter}}(t/k)\right]^k
$$

Naiwne podejście używa niewielu kroków Trottera, aby utrzymać zarządzalną głębokość układu, lecz wprowadza to znaczący błąd Trottera. AQC-Tensor rozwiązuje to napięcie, oddzielając dokładność od głębokości:

1. **Układ docelowy (wysoka dokładność, głęboki):** Konstruuje układ Trottera z wieloma krokami — na przykład $10k$ — dla tego samego czasu ewolucji. Ten układ ma znacznie mniejszy błąd Trottera, ale jest zbyt głęboki dla sprzętu. Ponieważ jest on symulowany klasycznie wyłącznie jako stan iloczynu macierzowego (MPS), głębokość nie stanowi problemu.

2. **Układ ansatz (mała głębokość, sparametryzowany):** Definiuje sparametryzowany układ $V(\theta)$ o tej samej strukturze co układ Trottera z jednym krokiem. Inicjalizuje go tak, aby $V(\theta_{\text{init}}) = U_{\text{Trotter}}(t/k)$, a następnie iteracyjnie optymalizuje $\theta$, by $V(\theta)$ jak najbliżej odtwarzał stan docelowy o wysokiej dokładności.

Wynikiem jest układ zachowujący głębokość pojedynczego kroku Trottera, lecz osiągający dokładność wielu kroków, co czyni go wykonalnym na sprzęcie kwantowym bliskiej przyszłości.

### Kiedy używać AQC-Tensor {#when-to-use-aqc-tensor}

AQC-Tensor jest najbardziej efektywny, gdy:

- **Głębokość układu przekracza czasy koherencji sprzętu.** Jeśli symulacja Trottera wymaga więcej kroków Trottera niż urządzenie może obsłużyć, AQC-Tensor może skompresować ewolucję do płytszego układu.
- **Splątanie pozostaje klasycznie zarządzalne.** Całkowite splątanie w stanie ewoluowanym w czasie zależy przede wszystkim od czasu ewolucji $t$, a nie od liczby kroków Trottera $k$. Oznacza to, że układ docelowy z $10k$ krokami jest zazwyczaj równie łatwy do reprezentacji jako MPS jak układ z $k$ krokami, o ile $t$ jest wystarczająco krótkie, by wymiary wiązania pozostały zarządzalne.
- **Istnieje naturalny ansatz.** Ponieważ ansatz odzwierciedla strukturę układu Trottera, zapewnia fizycznie umotywowany punkt startowy z dobrze zdefiniowanymi parametrami początkowymi, unikając problemów ze zbieżnością, które mogą trapić arbitralne warianty ansatze.

To podejście różni się od generycznej kompresji układu: zamiast próbować aproksymować arbitralną unitarność mniejszą liczbą bramek, AQC-Tensor zachowuje tę samą strukturę bramek i optymalizuje jej parametry, aby zmniejszyć błąd Trottera. Więcej informacji można znaleźć w [dokumentacji AQC-Tensor](https://qiskit.github.io/qiskit-addon-aqc-tensor/).

Ten samouczek przeprowadzi cię przez pełny przepływ pracy AQC-Tensor przygotowania stanu: definiowanie Hamiltonianu, generowanie układów Trottera, kompresowanie ich przez optymalizację sieciami tensorowymi i wykonywanie wyniku na sprzęcie IBM Quantum&reg;.

## Przykład symulatora w małej skali {#small-scale-simulator-example}

Ta sekcja używa układu 10-węzłowego, aby krok po kroku zilustrować przepływ pracy AQC-Tensor. Symulujemy dynamikę łańcucha spinowego XXZ z 10 węzłami, szeroko badanego modelu do badania oddziaływań spinowych i właściwości magnetycznych.

Hamiltonian jest następujący:

$$
\hat{\mathcal{H}}_{XXZ} = \sum_{i=1}^{L-1} J_{i,(i+1)}\left(X_i X_{(i+1)}+Y_i Y_{(i+1)}+ 2\cdot Z_i Z_{(i+1)} \right) \, ,
$$

gdzie $J_{i,(i+1)}$ jest losowym współczynnikiem dla krawędzi $(i, i+1)$, a $L=10$.
### Krok 1: Odwzorowanie klasycznych danych wejściowych na problem kwantowy {#step-1-map-classical-inputs-to-a-quantum-problem}

W tym kroku:
1. Definiujemy Hamiltonian, obserwablę i stan początkowy.
2. Klasycznie obliczamy dokładną wartość oczekiwaną do późniejszego porównania.
3. Generujemy wysoce dokładny układ Trottera (cel AQC) i kompresujemy go do ansatzu o małej głębokości za pomocą AQC-Tensor.
#### Konfiguracja Hamiltonianu, obserwabli i stanu początkowego {#set-up-the-hamiltonian-observable-and-initial-state}

```python
# L is the number of sites in the 1D spin chain
L = 10

# Generate the coupling map
edge_list = [(i - 1, i) for i in range(1, L)]
even_edges = edge_list[::2]
odd_edges = edge_list[1::2]
coupling_map = CouplingMap(edge_list)

# Generate random coefficients for our XXZ Hamiltonian
np.random.seed(0)
Js = np.random.rand(L - 1) + 0.5 * np.ones(L - 1)
hamiltonian = SparsePauliOp(Pauli("I" * L))
for i, edge in enumerate(even_edges + odd_edges):
    hamiltonian += SparsePauliOp.from_sparse_list(
        [
            ("XX", (edge), Js[i] / 2),
            ("YY", (edge), Js[i] / 2),
            ("ZZ", (edge), Js[i]),
        ],
        num_qubits=L,
    )

# Generate a ZZ observable between the two middle qubits
observable = SparsePauliOp.from_sparse_list(
    [("ZZ", (L // 2 - 1, L // 2), 1.0)], num_qubits=L
)

# Generate an initial Néel state |1010101010⟩
initial_state_circuit = QuantumCircuit(L)
for i in range(L):
    if i % 2:
        initial_state_circuit.x(i)

print("Hamiltonian:", hamiltonian)
print("Observable:", observable)
graphviz_draw(coupling_map.graph, method="circo")
```

#### Obliczenie dokładnej wartości oczekiwanej {#compute-the-exact-expectation-value}

Dla układu tej wielkości możemy bezpośrednio obliczyć dokładną wartość oczekiwaną stanu ewoluowanego w czasie, używając potęgowania macierzy. Służy to jako punkt odniesienia do oceny dokładności układu AQC.

```python
aqc_evolution_time = 0.2

# Each baseline Trotter step covers dt = aqc_evolution_time / 3
# The subsequent (uncompressed) step covers 1 additional dt
subsequent_evolution_time = aqc_evolution_time / 3
total_evolution_time = aqc_evolution_time + subsequent_evolution_time

# Compute exact expectation value via matrix exponentiation
H_matrix = hamiltonian.to_matrix()
U_exact = expm(-1j * H_matrix * total_evolution_time)

# Build the initial state vector (Néel state)
initial_state_vec = np.zeros(2**L)
state_idx = sum(2**i for i in range(L) if i % 2)
initial_state_vec[state_idx] = 1.0

# Evolve and compute expectation value
evolved_state = U_exact @ initial_state_vec
obs_matrix = observable.to_matrix()
exact_expval = (evolved_state.conj() @ obs_matrix @ evolved_state).real

print(f"AQC evolution time: {aqc_evolution_time}")
print(f"Subsequent evolution time: {subsequent_evolution_time:.6f}")
print(f"Total evolution time: {total_evolution_time:.6f}")
print(f"Exact expectation value: {exact_expval:.6f}")
```

#### Generowanie docelowego układu AQC {#generate-the-aqc-target-circuit}

Konstruujemy teraz układ Trottera, który będzie służył jako cel AQC. Ten układ używa wielu kroków Trottera (32) dla wysokiej dokładności. Ponieważ będzie on symulowany klasycznie wyłącznie jako MPS — a nie wykonywany na sprzęcie — duża głębokość nie stanowi problemu.

```python
aqc_target_num_trotter_steps = 32

aqc_target_circuit = initial_state_circuit.copy()
aqc_target_circuit.compose(
    generate_time_evolution_circuit(
        hamiltonian,
        synthesis=SuzukiTrotter(reps=aqc_target_num_trotter_steps),
        time=aqc_evolution_time,
    ),
    inplace=True,
)
```

#### Generowanie ansatzu, parametrów początkowych, układu kolejnego i układu bazowego {#generate-an-ansatz-initial-parameters-subsequent-circuit-and-a-baseline-circuit}

Następnie konstruujemy „dobry" układ o tym samym czasie ewolucji co cel AQC, ale ze znacznie mniejszą liczbą kroków Trottera (tylko jednym). Przekazujemy ten układ do `generate_ansatz_from_circuit`, który zwraca:

1. Ogólny, sparametryzowany układ **ansatz** o tej samej łączności dwu-Qubitowej.
2. **Parametry początkowe** odtwarzające układ wejściowy po podstawieniu do ansatzu.

Konstruujemy również:
- **Układ kolejny** z jednym krokiem Trottera, który zostanie dołączony (nieskompresowany) po zoptymalizowanej przez AQC części, zgodnie z podejściem opisanym w [samouczku AQC-Tensor dotyczącym stanu początkowego](https://qiskit.github.io/qiskit-addon-aqc-tensor/tutorials/01_initial_state_aqc.html).
- **Bazowy układ Trottera** używający czterech kroków Trottera dla całego czasu ewolucji (`aqc_evolution_time + subsequent_evolution_time`). Służy to jako punkt porównawczy: reprezentuje to, co uruchomiłbyś na sprzęcie bez AQC. Ansatz AQC (3 skompresowane kroki + 1 nieskompresowany krok) osiąga lepszą dokładność przy mniejszej głębokości.

```python
aqc_ansatz_num_trotter_steps = 1

aqc_good_circuit = initial_state_circuit.copy()
aqc_good_circuit.compose(
    generate_time_evolution_circuit(
        hamiltonian,
        synthesis=SuzukiTrotter(reps=aqc_ansatz_num_trotter_steps),
        time=aqc_evolution_time,
    ),
    inplace=True,
)

aqc_ansatz, aqc_initial_parameters = generate_ansatz_from_circuit(
    aqc_good_circuit
)

# Subsequent circuit: 1 non-compressed Trotter step appended after AQC
subsequent_num_trotter_steps = 1
subsequent_circuit = generate_time_evolution_circuit(
    hamiltonian,
    synthesis=SuzukiTrotter(reps=subsequent_num_trotter_steps),
    time=subsequent_evolution_time,
)

# Baseline Trotter circuit: 4 Trotter steps over total evolution time, no AQC
baseline_num_trotter_steps = 4
baseline_circuit = initial_state_circuit.copy()
baseline_circuit.compose(
    generate_time_evolution_circuit(
        hamiltonian,
        synthesis=SuzukiTrotter(reps=baseline_num_trotter_steps),
        time=total_evolution_time,
    ),
    inplace=True,
)

print(
    f"Target circuit:      depth {aqc_target_circuit.depth(lambda x: x.operation.num_qubits == 2)}"
)
print(
    f"Baseline circuit:    depth {baseline_circuit.depth(lambda x: x.operation.num_qubits == 2)} ({baseline_num_trotter_steps} Trotter steps, time={total_evolution_time:.4f})"
)
print(
    f"Subsequent circuit:  depth {subsequent_circuit.depth(lambda x: x.operation.num_qubits == 2)} ({subsequent_num_trotter_steps} Trotter step, time={subsequent_evolution_time:.4f})"
)
print(
    f"Ansatz circuit:      depth {aqc_ansatz.depth(lambda x: x.operation.num_qubits == 2)}, with {len(aqc_initial_parameters)} parameters"
)
aqc_ansatz.draw("mpl", fold=-1)
```

#### Optymalizacja parametrów ansatzu {#optimize-the-ansatz-parameters}

Minimalizujemy funkcję kosztu `MaximizeStateFidelity` za pomocą optymalizatora L-BFGS-B. Optymalizator iteracyjnie dostosowuje parametry ansatzu, aby zmaksymalizować wierność między układem ansatz a docelowym MPS.

```python
aqc_stopping_fidelity = 1
aqc_max_iterations = 500

stopping_point = 1.0 - aqc_stopping_fidelity
objective = MaximizeStateFidelity(
    aqc_target_mps, aqc_ansatz, simulator_settings
)

def callback(intermediate_result: OptimizeResult):
    fidelity = 1 - intermediate_result.fun
    print(
        f"{datetime.datetime.now()} Intermediate result: Fidelity {fidelity:.8f}"
    )
    if intermediate_result.fun < stopping_point:
        raise StopIteration

result = minimize(
    objective,
    aqc_initial_parameters,
    method="L-BFGS-B",
    jac=True,
    options={"maxiter": aqc_max_iterations},
    callback=callback,
)
if result.status not in (0, 1, 99):
    raise RuntimeError(
        f"Optimization failed: {result.message} (status={result.status})"
    )

print(f"Done after {result.nit} iterations.")
aqc_final_parameters = result.x
```

#### Składanie finalnego układu AQC {#assemble-the-final-aqc-circuit}

Mając zoptymalizowane parametry, przypisujemy je do ansatzu, a następnie dołączamy kolejny (nieskompresowany) krok Trottera. Wynikowy układ ma głębokość pojedynczego skompresowanego kroku Trottera plus jeden nieskompresowany krok, lecz skompresowana część aproksymuje dokładność 32 kroków Trottera.

```python
aqc_final_circuit = aqc_ansatz.assign_parameters(aqc_final_parameters)
aqc_final_circuit.compose(subsequent_circuit, inplace=True)
aqc_final_circuit.draw("mpl", fold=-1)
```

### Krok 2: Optymalizacja problemu pod kątem wykonania na sprzęcie kwantowym {#step-2-optimize-problem-for-quantum-hardware-execution}

Dla tego przykładu w małej skali używamy fałszywego backendu (`FakeKyiv`), aby lokalnie zasymulować wykonanie sprzętowe. Transpilujemy zarówno układ zoptymalizowany przez AQC (`aqc_final_circuit`), jak i bazowy układ Trottera (`baseline_circuit`, cztery kroki Trottera dla całego czasu ewolucji, bez AQC) do architektury zestawu instrukcji (ISA) backendu, z `optimization_level=3` w celu dalszego zmniejszenia głębokości układu.

## Przykład sprzętowy w dużej skali {#large-scale-hardware-example}

Skalujemy teraz do modelu XXZ z 50 węzłami, aby zademonstrować AQC-Tensor na bardziej realistycznym rozmiarze problemu. Przepływ pracy jest taki sam jak w przykładzie w małej skali: kompresujemy trzy kroki Trottera za pomocą AQC i dołączamy jeden nieskompresowany krok.

Dla układu tej wielkości potęgowanie macierzy jest niewykonalne (wymiary $2^{50}$), więc obliczamy referencyjną wartość oczekiwaną bezpośrednio z wysoce dokładnego MPS ewoluowanego przez cały czas.

### Kroki 1–4 łącznie {#steps-14-combined}

```python
# -------------------------Step 1-------------------------

# Define the 50-site spin chain
L = 50
edge_list = [(i - 1, i) for i in range(1, L)]
even_edges = edge_list[::2]
odd_edges = edge_list[1::2]
coupling_map = CouplingMap(edge_list)

# Random XXZ Hamiltonian
np.random.seed(0)
Js = np.random.rand(L - 1) + 0.5 * np.ones(L - 1)
hamiltonian = SparsePauliOp(Pauli("I" * L))
for i, edge in enumerate(even_edges + odd_edges):
    hamiltonian += SparsePauliOp.from_sparse_list(
        [
            ("XX", (edge), Js[i] / 2),
            ("YY", (edge), Js[i] / 2),
            ("ZZ", (edge), Js[i]),
        ],
        num_qubits=L,
    )

observable = SparsePauliOp.from_sparse_list(
    [("ZZ", (L // 2 - 1, L // 2), 1.0)], num_qubits=L
)

# Initial Néel state
initial_state_circuit = QuantumCircuit(L)
for i in range(L):
    if i % 2:
        initial_state_circuit.x(i)

# Time parameters
aqc_evolution_time = 0.2
subsequent_evolution_time = aqc_evolution_time / 3
total_evolution_time = aqc_evolution_time + subsequent_evolution_time

# AQC target circuit (high-accuracy, 32 Trotter steps for AQC portion)
aqc_target_num_trotter_steps = 32

aqc_target_circuit = initial_state_circuit.copy()
aqc_target_circuit.compose(
    generate_time_evolution_circuit(
        hamiltonian,
        synthesis=SuzukiTrotter(reps=aqc_target_num_trotter_steps),
        time=aqc_evolution_time,
    ),
    inplace=True,
)

# Generate ansatz from 1-step Trotter circuit
aqc_good_circuit = initial_state_circuit.copy()
aqc_good_circuit.compose(
    generate_time_evolution_circuit(
        hamiltonian,
        synthesis=SuzukiTrotter(reps=1),
        time=aqc_evolution_time,
    ),
    inplace=True,
)

aqc_ansatz, aqc_initial_parameters = generate_ansatz_from_circuit(
    aqc_good_circuit
)

# Subsequent circuit: 1 non-compressed Trotter step
subsequent_circuit = generate_time_evolution_circuit(
    hamiltonian,
    synthesis=SuzukiTrotter(reps=1),
    time=subsequent_evolution_time,
)

# Baseline Trotter circuit: 4 Trotter steps over total evolution time, no AQC
baseline_num_trotter_steps = 4
baseline_circuit = initial_state_circuit.copy()
baseline_circuit.compose(
    generate_time_evolution_circuit(
        hamiltonian,
        synthesis=SuzukiTrotter(reps=baseline_num_trotter_steps),
        time=total_evolution_time,
    ),
    inplace=True,
)
print(
    f"Target circuit:  depth {aqc_target_circuit.depth(lambda x: x.operation.num_qubits == 2)}"
)
print(
    f"Ansatz circuit:  depth {aqc_ansatz.depth(lambda x: x.operation.num_qubits == 2)}, with {len(aqc_initial_parameters)} parameters"
)
print(
    f"Subsequent circuit: depth {subsequent_circuit.depth(lambda x: x.operation.num_qubits == 2)}"
)
print(
    f"Baseline circuit:   depth {baseline_circuit.depth(lambda x: x.operation.num_qubits == 2)} ({baseline_num_trotter_steps} steps, time={total_evolution_time:.4f})"
)

# Build target MPS and compute reference expectation value
simulator_settings = QuimbSimulator(
    quimb.tensor.CircuitMPS, autodiff_backend="jax"
)
aqc_target_mps = tensornetwork_from_circuit(
    aqc_target_circuit, simulator_settings
)
print("Target MPS maximum bond dimension:", aqc_target_mps.psi.max_bond())

# For the reference expectation value, we need the full evolution (AQC + subsequent)
# Build a high-accuracy full circuit for MPS reference
full_target_circuit = initial_state_circuit.copy()
full_target_circuit.compose(
    generate_time_evolution_circuit(
        hamiltonian,
        synthesis=SuzukiTrotter(reps=aqc_target_num_trotter_steps),
        time=total_evolution_time,
    ),
    inplace=True,
)
full_target_mps = tensornetwork_from_circuit(
    full_target_circuit, simulator_settings
)
exact_expval = full_target_mps.local_expectation(
    quimb.pauli("Z") & quimb.pauli("Z"), (L // 2 - 1, L // 2)
).real.item()
print(f"Reference expectation value (from MPS): {exact_expval:.6f}")

# Optimize ansatz parameters
objective = MaximizeStateFidelity(
    aqc_target_mps, aqc_ansatz, simulator_settings
)

def callback(intermediate_result: OptimizeResult):
    fidelity = 1 - intermediate_result.fun
    print(
        f"{datetime.datetime.now()} Intermediate result: Fidelity {fidelity:.8f}"
    )

result = minimize(
    objective,
    aqc_initial_parameters,
    method="L-BFGS-B",
    jac=True,
    options={"maxiter": 500},
    callback=callback,
)
if result.status not in (0, 1, 99):
    raise RuntimeError(
        f"Optimization failed: {result.message} (status={result.status})"
    )
print(f"Done after {result.nit} iterations.")

# Assemble the final AQC circuit: optimized ansatz + subsequent Trotter step
aqc_final_circuit = aqc_ansatz.assign_parameters(result.x)
aqc_final_circuit.compose(subsequent_circuit, inplace=True)

# -------------------------Step 2-------------------------

service = QiskitRuntimeService()
backend = service.least_busy(min_num_qubits=127)
print(backend)

pass_manager = generate_preset_pass_manager(
    backend=backend, optimization_level=3
)
isa_circuit = pass_manager.run(aqc_final_circuit)
isa_observable = observable.apply_layout(isa_circuit.layout)
print(
    "AQC circuit depth:",
    isa_circuit.depth(lambda x: x.operation.num_qubits == 2),
)

# Also transpile the baseline Trotter circuit (4 Trotter steps, no AQC)
isa_baseline_circuit = pass_manager.run(baseline_circuit)
isa_baseline_observable = observable.apply_layout(isa_baseline_circuit.layout)
print(
    "Baseline Trotter circuit depth:",
    isa_baseline_circuit.depth(lambda x: x.operation.num_qubits == 2),
)

# -------------------------Step 3-------------------------

# Submit both circuits in a single job
estimator = Estimator(backend)
estimator.options.environment.job_tags = ["TUT_AQCTE"]

job = estimator.run(
    [
        (isa_circuit, isa_observable),
        (isa_baseline_circuit, isa_baseline_observable),
    ]
)
print("Job ID:", job.job_id())
```

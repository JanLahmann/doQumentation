## Tło {#background}
Ten samouczek przedstawia *Kodowanie Korelacji Pauliego* (PCE) [\[1\]](#references), podejście zaprojektowane w celu kodowania problemów optymalizacyjnych w qubitach z większą wydajnością dla obliczeń kwantowych. PCE odwzorowuje zmienne klasyczne w problemach optymalizacyjnych na korelacje wielociałowych macierzy Pauliego, co skutkuje wielomianową kompresją wymagań przestrzennych problemu. Dzięki zastosowaniu PCE zmniejsza się liczba qubitów potrzebnych do kodowania, co jest szczególnie korzystne dla urządzeń kwantowych bliskiej przyszłości z ograniczonymi zasobami qubitowymi. Ponadto analitycznie wykazano, że PCE z natury łagodzi jałowe plateau, oferując super-wielomianową odporność na to zjawisko. Ta wbudowana cecha umożliwia bezprecedensową wydajność kwantowych solverów optymalizacyjnych.
### Przegląd {#overview}
Podejście PCE składa się z trzech głównych kroków, jak zilustrowano na Rysunku 1 z [\[1\]](#references) poniżej:
1. Kodowanie problemu optymalizacyjnego w przestrzeni korelacji Pauliego.
2. Rozwiązywanie problemu przy użyciu kwantowo-klasycznego solvera optymalizacyjnego.
3. Dekodowanie rozwiązania z powrotem do oryginalnej przestrzeni optymalizacyjnej.
Podejście PCE jest adaptowalne do dowolnego kwantowego solvera optymalizacyjnego zdolnego do przetwarzania macierzy korelacji Pauliego.
![Przegląd PCE.](/docs/images/tutorials/solving-maxcut-with-reduced-qubit-requirements-using-pauli-correlation-encoding/af2cb835-88db-4a3d-9c86-51424b1a4bd3.avif)
Na Rysunku 1 z [\[1\]](#references) problem [max-cut](/tutorials/quantum-approximate-optimization-algorithm) jest używany jako przykład do zilustrowania podejścia PCE. Problem max-cut z $m=9$ węzłami jest zakodowany w przestrzeni korelacji Pauliego, reprezentując problem optymalizacyjny jako macierz korelacji &mdash; konkretnie dwuciałowe korelacje macierzy Pauliego na $n=3$ qubitach $(Q_1, Q_2, Q_3)$. Kolory węzłów wskazują łańcuch Pauliego używany dla każdego zakodowanego węzła.
Na przykład węzeł 1, który odpowiada zmiennej binarnej $x_1$, jest zakodowany przez wartość oczekiwaną $Z_1 \otimes Z_2 \otimes I_3$, podczas gdy $x_8$ jest zakodowany przez $I_1 \otimes Y_2 \otimes Y_3$.
Odpowiada to kompresji $m$ zmiennych problemu do $n = O(m^{1/2})$ qubitów. Ogólniej, korelacje $k$-ciałowe umożliwiają wielomianowe kompresje rzędu $k$, gdzie $k>1$. Wybrany zbiór Pauliego składa się z trzech podzbiorów wzajemnie przemiennych łańcuchów Pauliego, umożliwiając eksperymentalne szacowanie wszystkich $m$ korelacji przy użyciu tylko trzech ustawień pomiaru.

Konstruuje się funkcję straty $\mathcal{L}$ wartości oczekiwanych Pauliego, która imituje oryginalną funkcję celu max-cut. Funkcja straty jest następnie optymalizowana przy użyciu kwantowo-klasycznego solvera optymalizacyjnego, takiego jak [Wariacyjny Eigensolver Kwantowy (VQE)](/learning/courses/quantum-diagonalization-algorithms/vqe).

Po zakończeniu optymalizacji rozwiązanie jest dekodowane z powrotem do oryginalnej przestrzeni optymalizacyjnej, dając optymalne rozwiązanie max-cut.

```python
    cut_size = 0
    for edge0, edge1 in graph.edge_list():
        if edge0 in partition0 and edge1 in partition1:
            cut_size += 1
        elif edge0 in partition1 and edge1 in partition0:
```

#### Problem max-cut {#the-max-cut-problem}
Problem max-cut jest kombinatorycznym problemem optymalizacyjnym zdefiniowanym na grafie $G = (V, E)$, gdzie $V$ jest zbiorem wierzchołków, a $E$ jest zbiorem krawędzi. Celem jest podział wierzchołków na dwa zbiory, $S$ i $V \setminus S$, tak aby liczba krawędzi między dwoma zbiorami była maksymalna.
Szczegółowy opis problemu max-cut znajduje się w samouczku [Kwantowy algorytm przybliżonej optymalizacji](/tutorials/quantum-approximate-optimization-algorithm).
Problem max-cut jest również używany jako przykład w samouczku [Zaawansowane techniki dla QAOA](/tutorials/advanced-techniques-for-qaoa).
W tych samouczkach algorytm QAOA jest używany do rozwiązania problemu max-cut.

Kodujemy graf ze 100 węzłami na dwuciałowe korelacje macierzy Pauliego na dziewięciu qubitach (patrz wyjaśnienie poniżej). Graf jest reprezentowany jako macierz korelacji, gdzie każdy węzeł jest zakodowany przez łańcuch Pauliego. Znak wartości oczekiwanej łańcucha Pauliego wskazuje podział węzła. Na przykład węzeł 0 jest zakodowany przez łańcuch Pauliego, $\prod_0 = I_{8} \otimes ... I_2 \otimes X_1 \otimes X_0$. Znak wartości oczekiwanej tego łańcucha Pauliego wskazuje podział węzła 0. Definiujemy *kodowanie korelacji Pauliego* (PCE) względem $\prod$ jako

$$ x_i \coloneqq \textit{sgn}(\langle\prod_i \rangle), $$

gdzie $x_i$ jest podziałem węzła $i$, a $\langle \prod_i \rangle \coloneqq  \langle \psi |\prod_i| \psi \rangle $ jest wartością oczekiwaną łańcucha Pauliego kodującego węzeł $i$ dla stanu kwantowego $|\psi \rangle$.
Teraz zakodujmy graf w Hamiltonianie za pomocą PCE.
Dzielimy węzły na trzy zbiory: $S_1$, $S_2$ i $S_3$.
Następnie kodujemy węzły w każdym zbiorze za pomocą łańcuchów Pauliego odpowiednio z $X$, $Y$ i $Z$.
Musimy wyodrębnić relację między liczbą węzłów a qubitów potrzebnych do zakodowania wszystkich węzłów. Użycie wszystkich możliwych permutacji dla kodowania daje:

$$
m=3\binom{n}{k}.
$$

W tym przykładzie rozważamy $k=2$, zatem:

$$
m  = \frac{3}{2} n(n-1).
$$

Dlatego liczba qubitów $n$ potrzebna do wyrażenia pewnej liczby węzłów $m$ wynosi:

$$
n = \left\lceil \frac{1 + \sqrt{1 + \tfrac{8}{3}m}}{2} \right\rceil.
$$

*Zauważ, że symbol $\lceil \cdot \rceil$ reprezentuje funkcję sufitu, która zaokrągla każdą liczbę rzeczywistą w górę do następnej liczby całkowitej. Zapewnia to, że liczba qubitów jest liczbą całkowitą.*

```python
def build_pauli_correlation_encoding(pauli, node_list, n, k=2):
    pauli_correlation_encoding = []
    for idx, c in enumerate(combinations(range(n), k)):
        if idx >= len(node_list):
            break
        paulis = ["I"] * n
        paulis[c[0]], paulis[c[1]] = pauli, pauli
        pauli_correlation_encoding.append(("".join(paulis)[::-1], 1))

    hamiltonian = []
    for pauli, weight in pauli_correlation_encoding:
        hamiltonian.append(SparsePauliOp.from_list([(pauli, weight)]))

    return hamiltonian

pauli_correlation_encoding_x = build_pauli_correlation_encoding(
    "X", node_x, num_qubits
)
pauli_correlation_encoding_y = build_pauli_correlation_encoding(
    "Y", node_y, num_qubits
)
pauli_correlation_encoding_z = build_pauli_correlation_encoding(
    "Z", node_z, num_qubits
)
```

#### Funkcja straty {#loss-function}
Dla funkcji straty $\mathcal{L}$ używamy relaksacji funkcji celu max-cut, jak opisano w [\[1\]](#references), która jest zdefiniowana jako $\mathcal{V}(\mathbf{x}) \coloneqq \sum_{(i, j) \in E} W_{i, j}(1-x_i x_j)$. Tutaj $W_{i, j}$ oznacza wagę krawędzi $(i, j)$, a $x_i$ reprezentuje podział węzła $i$.
Funkcja straty $\mathcal{L}$ jest dana przez:

$\mathcal{L}\coloneqq \sum_{(i, j) \in E} W_{i, j} \text{tanh} (\alpha \langle\prod_i \rangle) \text{tanh} (\alpha \langle\prod_j \rangle) + \mathcal{L}^{(\text{reg})},$

gdzie funkcja celu max-cut jest zastąpiona przez gładkie tangensy hiperboliczne wartości oczekiwanych łańcuchów Pauliego kodujących węzły. Człon regularyzacyjny $\mathcal{L}^{(\text{reg})}$ i współczynnik skalowania $\alpha$, proporcjonalny do liczby qubitów, są wprowadzone w celu poprawy wydajności solvera.

Człon regularyzacyjny jest zdefiniowany jako:

$\mathcal{L}^{(\text{reg})}$ jest zdefiniowany jako $\mathcal{L}^{(\text{reg})} \coloneqq \beta \nu \lbrack \frac{1}{m} \sum_{i \in V} \text{tanh} (\alpha \langle\prod_i \rangle)^2 \rbrack ^2$

gdzie $\beta=1/2$, $\nu = |E|/2 + (m -1) /4$, $|E|$ jest liczbą krawędzi, a $m$ jest liczbą węzłów w grafie.

```python
def loss_func_estimator(x, ansatz, hamiltonian, estimator, graph):
    """
    Calculates the specified loss function for the given ansatz, Hamiltonian, and graph.

    The expectation values of each Pauli string in the Hamiltonian are first obtained
    by running the ansatz on the quantum backend. These expectation values are then
    passed through the nonlinear function tanh(alpha * prod_i). The loss function is
    subsequently computed from these transformed values.
    """
    job = estimator.run(
        [
            (ansatz, hamiltonian[0], x),
            (ansatz, hamiltonian[1], x),
            (ansatz, hamiltonian[2], x),
        ]
    )
    result = job.result()

    # calculate the loss function
    node_exp_map = {}
    idx = 0
    for r in result:
        for ev in r.data.evs:
            node_exp_map[idx] = ev
            idx += 1

    loss = 0
    alpha = num_qubits
    for edge0, edge1 in graph.edge_list():
        loss += np.tanh(alpha * node_exp_map[edge0]) * np.tanh(
            alpha * node_exp_map[edge1]
        )

    regulation_term = 0
    for i in range(len(graph.nodes())):
        regulation_term += np.tanh(alpha * node_exp_map[i]) ** 2
    regulation_term = regulation_term / len(graph.nodes())
    regulation_term = regulation_term**2
    beta = 1 / 2
    v = len(graph.edges()) / 2 + (len(graph.nodes()) - 1) / 4
    regulation_term = beta * v * regulation_term

    loss = loss + regulation_term

    global experiment_result
    print(f"Iter {len(experiment_result)}: {loss}")
    experiment_result.append({"loss": loss, "exp_map": node_exp_map})
    return loss
```

### Krok 4: Post-processing i zwrócenie wyniku w żądanym klasycznym formacie {#step-4-post-process-and-return-result-in-desired-classical-format}

Podziały węzłów są określane przez ocenę znaku wartości oczekiwanych łańcuchów Pauliego kodujących węzły.

```python
# Calculate the partitions based on the final expectation values
# If the expectation value is positive, the node belongs to partition 0 (par0)
# Otherwise, the node belongs to partition 1 (par1)
def get_partitions(experiment_result):
    par0, par1 = set(), set()
    best_index = min(
        range(len(experiment_result)),
        key=lambda i: experiment_result[i]["loss"],
    )
    for i in experiment_result[best_index]["exp_map"]:
        if experiment_result[best_index]["exp_map"][i] >= 0:
            par0.add(i)
        else:
            par1.add(i)
    return par0, par1, best_index

par0, par1, best_index = get_partitions(experiment_result)
print(par0, par1)
```

```python
# Swap the partitions and calculate the cut size

def swap_partitions(graph, cur_bits):
    best_cut = 0
    best_bits = []
    for edge0, edge1 in graph.edge_list():
        swapped_bits = cur_bits.copy()
        swapped_bits[edge0], swapped_bits[edge1] = (
            swapped_bits[edge1],
            swapped_bits[edge0],
        )

        cur_partition = [set(), set()]
        for i, bit in enumerate(swapped_bits):
            if bit > 0:
                cur_partition[0].add(i)
            else:
                cur_partition[1].add(i)
        cut_size = calc_cut_size(graph, cur_partition[0], cur_partition[1])
        if best_cut < cut_size:
            best_cut = cut_size
            best_bits = swapped_bits
    return best_cut, best_bits

best_cut, best_bits = swap_partitions(graph, cur_bits)
print(best_cut, best_bits)
```

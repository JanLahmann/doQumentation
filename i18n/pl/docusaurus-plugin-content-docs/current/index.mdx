## Otwarte zasoby IBM Quantum {#ibm-quantums-open-source-content}

IBM udostępnia bogaty zbiór materiałów edukacyjnych z zakresu obliczeń kwantowych — wszystkie na otwartej licencji CC BY-SA 4.0:

- **[Learning](https://quantum.cloud.ibm.com/learning)** — Ustrukturyzowane kursy od podstaw kwantowych po zaawansowane tematy
- **[Tutorials](https://quantum.cloud.ibm.com/docs/en/tutorials)** — Ponad 40 tutoriali o transpilacji, mitygacji błędów i nie tylko
- **[Documentation](https://quantum.cloud.ibm.com/docs/en/guides)** — Przewodniki i dokumentacja API dla Qiskit
- **[Source repo](https://github.com/Qiskit/documentation)** — Wszystkie treści na GitHubie

Ich [Quantum Platform](https://quantum.cloud.ibm.com) jest zawsze aktualna i dobrze zaprojektowana — najlepsze miejsce do czytania, nauki i szukania informacji.

## Co ten projekt dodaje {#what-this-project-adds}

[Dokumentacja Qiskit](https://github.com/Qiskit/documentation) IBM jest otwarta (CC BY-SA 4.0), ale ich aplikacja webowa już nie. doQumentation dodaje otwartoźródłowy frontend z uruchamianiem kodu na żywo, automatycznym wstrzykiwaniem poświadczeń i trybem symulatora.

Możliwy do wdrożenia wszędzie — od [GitHub Pages](https://doqumentation.org) przez [Docker](https://github.com/JanLahmann/doQumentation/pkgs/container/doqumentation) po [RasQberry](https://rasqberry.org/). Zobacz [wszystkie funkcje](/features).

## Gdy jesteś gotowy na więcej {#when-youre-ready-for-more}

doQumentation pomaga ci postawić **pierwsze kroki z Qiskit** — czytać, uruchamiać kod i odkrywać — bez konieczności niczego instalować. Gdy będziesz iść dalej, przejdź na oficjalną platformę IBM:

- **Uruchom na prawdziwym sprzęcie** — utwórz darmowe [konto IBM Cloud](https://quantum.cloud.ibm.com/registration) i instancję Open Plan, aby wykonywać obwody na prawdziwych komputerach kwantowych.
- **Najnowsza zawartość** — [platforma edukacyjna Qiskit](https://quantum.cloud.ibm.com/learning) firmy IBM zawiera najnowsze kursy, samouczki i dodatki (niektóre mogą jeszcze nie być w doQumentation).
- **Rozwijaj lokalnie** — [zainstaluj Qiskit](https://docs.quantum.ibm.com/guides/install-qiskit) na własnym laptopie do poważnej pracy programistycznej.

*Zbudowane na bazie [dokumentacji IBM Quantum o otwartym kodzie źródłowym](https://github.com/Qiskit/documentation). Dziękujemy zespołowi IBM Quantum za udostępnienie tej zawartości na licencji CC BY-SA 4.0.*

## Uruchamianie kodu {#code-execution}

Kliknij **Run** na dowolnym bloku kodu. Pierwsze kliknięcie uruchamia kernel Jupyter przez [Binder](https://mybinder.org) lub [IBM Code Engine](/jupyter-settings#code-engine). Potem uruchomienia są natychmiastowe.

- **Tryb Symulatora (bez konta)** — Włącz [Tryb Symulatora](/jupyter-settings#simulator-mode), żeby uruchamiać wszystkie notebooki z AerSimulator lub FakeBackends z modelem szumu. Zero konfiguracji.

- **Sprzęt IBM Quantum** — Wpisz swój token API i CRN w [Ustawieniach](/jupyter-settings#ibm-quantum) raz — poświadczenia są automatycznie wstrzykiwane przy każdym uruchomieniu. [Utwórz darmowe konto](https://quantum.cloud.ibm.com/registration), jeśli jeszcze go nie masz.

<details>
<summary>Dostępne backendy uruchomieniowe</summary>

Każdy tutorial ma wykonywalne bloki kodu. Kliknij **Run**, żeby uruchomić je na jednym z czterech backendów:

1. **Binder** (domyślny na GitHub Pages) — Darmowy zdalny kernel Jupyter przez [mybinder.org](https://mybinder.org)
2. **IBM Code Engine** — Bezserwerowy Jupyter na twoim koncie IBM Cloud. Cold start w sekundy zamiast minut. [Skonfiguruj](/jupyter-settings#code-engine)
3. **Lokalny Jupyter (Docker / RasQberry)** — Łączy się z lokalnym serwerem Jupyter z preinstalowanym Qiskit
4. **Własny serwer** — Wskaż dowolny endpoint Jupyter w [Ustawieniach](/jupyter-settings#advanced)

  <strong>No IBM Quantum account?</strong> Enable <a href="/jupyter-settings#execution-mode">Simulator Mode</a> in Settings to run all code without signing up.

Gdy dostępnych jest kilka backendów, wybierz który używać w [Ustawieniach](/jupyter-settings#backend-selection).

</details>

<details>
<summary>Opcje wdrożenia</summary>

doQumentation jest dostępny jako:

- **[GitHub Pages](https://doqumentation.org)** — Statyczna strona z [Binder](https://mybinder.org) lub [IBM Code Engine](/jupyter-settings#code-engine) do zdalnego uruchamiania kodu
- **[Docker](https://github.com/JanLahmann/doQumentation/pkgs/container/doqumentation)** — Pełny stack lokalnie, możliwość pracy offline
- **[RasQberry](https://rasqberry.org/)** — Samodzielny hosting na Raspberry Pi z lokalnym kernelem Jupyter

</details>

<details>
<summary>Uruchom lokalnie z Podman / Docker</summary>

Zainstaluj [Podman Desktop](https://podman-desktop.io/) lub [Docker Desktop](https://www.docker.com/products/docker-desktop/) (Mac / Windows / Linux), a następnie uruchom:

```bash
# Full stack: site + Jupyter + Qiskit (~3 GB)
podman run -p 8080:80 -p 8888:8888 ghcr.io/janlahmann/doqumentation:jupyter
```

Otwórz [http://localhost:8080](http://localhost:8080) — uruchamianie kodu działa lokalnie, bez czekania na Binder.

Dla lekkiej wersji bez lokalnego uruchamiania kodu (~60 MB):

```bash
# Static site only — code execution still works via Binder
podman run -p 8080:80 ghcr.io/janlahmann/doqumentation:latest
```

Używasz Dockera? Po prostu zamień `podman` na `docker` — komendy są identyczne. Obrazy są wieloarchitekturowe (`linux/amd64` + `linux/arm64`), więc Apple Silicon Mac działa natywnie.

</details>

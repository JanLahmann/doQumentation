## IBM Quantum ihri Open-Source-Inhält {#ibm-quantums-open-source-content}

IBM stellt es Huufe Lernmaterial für Quantencomputing zur Verfüegig — alles open source under CC BY-SA 4.0:

- **[Learning](https://quantum.cloud.ibm.com/learning)** — Strukturierti Kurs vo de Quantegrundlage bis zu de fortgschrittene Themene
- **[Tutorials](https://quantum.cloud.ibm.com/docs/en/tutorials)** — Über 40 Tutorials zu Transpilation, Fehlerminderig und meh
- **[Documentation](https://quantum.cloud.ibm.com/docs/en/guides)** — Aaleitige und API-Referänz für Qiskit
- **[Source repo](https://github.com/Qiskit/documentation)** — Alli Inhält uf GitHub

Ihri [Quantum Platform](https://quantum.cloud.ibm.com) isch immer aktuell und guet gstaltet — de best Ort zum Läse, Lehrne und Naaluege.

## Was das Projäkt derzue füegt {#what-this-project-adds}

IBM ihri [Qiskit-Dokumentation](https://github.com/Qiskit/documentation) isch open source (CC BY-SA 4.0), aber ihri Webaawändig isch's nöd. doQumentation füegt es open-source Frontend mit Live-Code-Usfüehrig, automatischer Credential-Injektion und Simulator-Modus derzue.

Überall iisetzbaa — vo [GitHub Pages](https://doqumentation.org) über [Docker](https://github.com/JanLahmann/doQumentation/pkgs/container/doqumentation) bis zu [RasQberry](https://rasqberry.org/). Lueg dir [alli Funktione](/features) aa.

## Code-Usfüehrig {#code-execution}

Klick bi jedem Codeblock uf **Run**. De erscht Klick startet en koschtelos Jupyter-Kernel über [Binder](https://mybinder.org) (1–2 Min.). Dernaa gohts sofort.

- **Simulator-Modus (kei Konto nötig)** — Aktivier de [Simulator-Modus](/jupyter-settings#simulator-mode), demit du alli Notebooks mit AerSimulator oder FakeBackends mit Ruuschmodell chasch usfüehre. Null Yyrichtigsufwand nötig.

- **IBM Quantum Hardware** — Gib dii API-Token und CRN einisch ide [Yystellige](/jupyter-settings#ibm-quantum) yy — d'Zuegangsdate wärde bi jeder Usfüehrig automatisch yygfüegt. [Erstell es koschteloses Konto](https://quantum.cloud.ibm.com/registration), falls du no keis hesch.

<details>
<summary>Verfüegbari Usfüehrigs-Backends</summary>

Jedes Tutorial het usfüehrbare Codeblöck. Klick uf **Run**, demit du si mit einem vo drüü Backends chasch usfüehre:

1. **Binder** (Standard uf GitHub Pages) — Koschtelos Remote-Jupyter-Kernel über [mybinder.org](https://mybinder.org)
2. **Lokals Jupyter (Docker / RasQberry)** — Verbindet mit em lokale Jupyter-Server mit vorinstalliertm Qiskit
3. **Benutzerdefinierts Server** — Zeiget ide [Yystellige](/jupyter-settings#advanced) uf en beliiebige Jupyter-Endpunkt

</details>

<details>
<summary>Bereitstelligs-Optione</summary>

doQumentation isch verfüegbar als:

- **[GitHub Pages](https://doqumentation.org)** — Statischi Website mit [Binder](https://mybinder.org) für Remote-Code-Usfüehrig
- **[Docker](https://github.com/JanLahmann/doQumentation/pkgs/container/doqumentation)** — Vollständige Stack lokal, offline-fähig
- **[RasQberry](https://rasqberry.org/)** — Sälber ghehoschtet uf emene Raspberry Pi mit lokalem Jupyter-Kernel

</details>

<details>
<summary>Lokal usfüehre mit Podman / Docker</summary>

Installier [Podman Desktop](https://podman-desktop.io/) oder [Docker Desktop](https://www.docker.com/products/docker-desktop/) (Mac / Windows / Linux), denn füehr us:

```bash
# Vollständige Stack: Website + Jupyter + Qiskit (~3 GB)
podman run -p 8080:80 -p 8888:8888 ghcr.io/janlahmann/doqumentation:jupyter
```

Mach [http://localhost:8080](http://localhost:8080) uf — Code-Usfüehrig funktioniert lokal, kei Binder-Wartezyt.

Für e liicht Version ohni lokali Code-Usfüehrig (~60 MB):

```bash
# Nur statischi Website — Code-Usfüehrig funktioniert wyterhin über Binder
podman run -p 8080:80 ghcr.io/janlahmann/doqumentation:latest
```

Bruuchsch Docker? Ersetze eifach `podman` dur `docker` — d'Befähl si identisch. D'Images si Multi-Arch (`linux/amd64` + `linux/arm64`), sodass Apple Silicon Macs nativ funktioniere.

</details>

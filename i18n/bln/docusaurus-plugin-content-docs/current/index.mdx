## IBM Quantum ihre Open-Source-Inhalte {#ibm-quantums-open-source-content}

IBM stellt ne Menge Lernmaterial für Quantencomputing zur Verfügung — allet open source untern CC BY-SA 4.0:

- **[Learning](https://quantum.cloud.ibm.com/learning)** — Strukturierte Kurse von de Quantenjrundlagen bis zu de fortjeschrittenen Themen
- **[Tutorials](https://quantum.cloud.ibm.com/docs/en/tutorials)** — Über 40 Tutorials zu Transpilation, Fehlerminderung und mehr
- **[Documentation](https://quantum.cloud.ibm.com/docs/en/guides)** — Anleitungen und API-Referenz für Qiskit
- **[Source repo](https://github.com/Qiskit/documentation)** — Alle Inhalte uff GitHub

Ihre [Quantum Platform](https://quantum.cloud.ibm.com) is immer aktuell und jut jestaltet — de beste Platz zum Lesen, Lernen und Nachjucken.

## Wat dit Projekt dazumacht {#what-this-project-adds}

IBM ihre [Qiskit-Doku](https://github.com/Qiskit/documentation) is open source (CC BY-SA 4.0), aber ihre Webanwendung is't nich. doQumentation macht een open-source Frontend mit Live-Code-Ausführung, automatischer Credential-Injektion und Simulator-Modus dazu.

Überall einsetzbar — von [GitHub Pages](https://doqumentation.org) über [Docker](https://github.com/JanLahmann/doQumentation/pkgs/container/doqumentation) bis zu [RasQberry](https://rasqberry.org/). Kuckt euch [alle Funktionen](/features) an.

## Code-Ausführung {#code-execution}

Klickt bei jeden Codeblock uff **Run**. Der erste Klick startet een kostenlosen Jupyter-Kernel über [Binder](https://mybinder.org) (1–2 Min.). Danach jeht't sofort.

- **Simulator-Modus (keen Konto nötig)** — Aktiviert den [Simulator-Modus](/jupyter-settings#simulator-mode), damit ihr alle Notebooks mit AerSimulator oder FakeBackends mit Rauschmodell ausführen könnt. Null Einrichtungsaufwand nötig.

- **IBM Quantum Hardware** — Jebt euern API-Token und CRN eenmal in de [Einstellungen](/jupyter-settings#ibm-quantum) ein — de Zugangsdaten werden bei jeder Ausführung automatisch eingefügt. [Erstellt een kostenloset Konto](https://quantum.cloud.ibm.com/registration), falls ihr noch keenet habt.

<details>
<summary>Verfügbare Ausführungs-Backends</summary>

Jedet Tutorial hat ausführbare Codeblöcke. Klickt uff **Run**, damit ihr se mit eenem von drei Backends ausführen könnt:

1. **Binder** (Standard uff GitHub Pages) — Kostenlosen Remote-Jupyter-Kernel über [mybinder.org](https://mybinder.org)
2. **Lokalet Jupyter (Docker / RasQberry)** — Verbindet euch mitn lokalen Jupyter-Server mit vorinstalliertem Qiskit
3. **Benutzerdefinierter Server** — Verweist in de [Einstellungen](/jupyter-settings#advanced) uff een beliebijet Jupyter-Endpunkt

</details>

<details>
<summary>Bereitstellungsoptionen</summary>

doQumentation is verfügbar als:

- **[GitHub Pages](https://doqumentation.org)** — Statische Website mit [Binder](https://mybinder.org) für Remote-Code-Ausführung
- **[Docker](https://github.com/JanLahmann/doQumentation/pkgs/container/doqumentation)** — Vollständiger Stack lokal, offline-fähig
- **[RasQberry](https://rasqberry.org/)** — Selbst jehostet uff nem Raspberry Pi mit lokalem Jupyter-Kernel

</details>

<details>
<summary>Lokal ausführen mit Podman / Docker</summary>

Installiert [Podman Desktop](https://podman-desktop.io/) oder [Docker Desktop](https://www.docker.com/products/docker-desktop/) (Mac / Windows / Linux), dann führt aus:

```bash
# Vollständiger Stack: Website + Jupyter + Qiskit (~3 GB)
podman run -p 8080:80 -p 8888:8888 ghcr.io/janlahmann/doqumentation:jupyter
```

Macht [http://localhost:8080](http://localhost:8080) uff — Code-Ausführung funktioniert lokal, keene Binder-Wartezeit.

Für ne leichtjewichtige Variante ohne lokale Code-Ausführung (~60 MB):

```bash
# Nur statische Website — Code-Ausführung funktioniert weiterhin über Binder
podman run -p 8080:80 ghcr.io/janlahmann/doqumentation:latest
```

Benutzt ihr Docker? Ersetzt einfach `podman` durch `docker` — de Befehle sind identisch. De Images sind Multi-Arch (`linux/amd64` + `linux/arm64`), sodass Apple Silicon Macs nativ funktionieren.

</details>

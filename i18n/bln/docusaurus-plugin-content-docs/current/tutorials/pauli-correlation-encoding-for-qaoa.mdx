## Hintergrund {#background}
Dit Tutorial stellt die *Pauli-Korrelations-Codierung* (PCE) [\[1\]](#references) vor — 'n Ansatz, der dafür jemacht is, Optimierungsprobleme mit größerer Effizienz in Qubits zu vaschlüsseln für die Quantenberechnung. PCE bildet klassische Variablen in Optimierungsproblemen auf Mehrkörper-Pauli-Matrix-Korrelationen ab, wat zu 'ner polynomiellen Kompression von de Platzanfordаrungen des Problems führt. Durch de Verwendung von PCE wird die Anzahl der Qubits für die Codierung reduziert, wat det besondas vorteilhaft macht für Quantenjеräte der nahen Zukunft mit begrenzten Qubit-Ressourcen. Außadem wird analytisch gezeigt, dass PCE von Natur aus kahle Plateaus mindert un überpotenzielle Widerstandsfähigkeit jejenüber diesem Phänomen bietet. Dieset eingebautet Feature ermöglicht noch nie dajesehene Leistung bei Quantenoptimierungssolvern.
### Übablick {#overview}
Der PCE-Ansatz besteht aus drei Hauptschritten, wie in Abbildung 1 aus [\[1\]](#references) unten darjestellt:
1. Codierung des Optimierungsproblems in 'nen Pauli-Korrelationsraum.
2. Lösung des Problems mit'm quantenklassischen Optimierungssolver.
3. Dekodierung der Lösung zurück in den ursärünglichen Optimierungsraum.
Der PCE-Ansatz is uff jeden Quantenoptimierungssolver anpassbar, der Pauli-Korrelationsmatrizen vaarbeiten kann.
![pce-overview.png](/docs/images/tutorials/solving-maxcut-with-reduced-qubit-requirements-using-pauli-correlation-encoding/af2cb835-88db-4a3d-9c86-51424b1a4bd3.avif)
In Abbildung 1 aus [\[1\]](#references) wird det Max-Cut-Problem als Beispiel jenutzt, um den PCE-Ansatz zu veranschaulichn. Det Max-Cut-Problem mit $m=9$ Knoten wird in 'nen Pauli-Korrelationsraum codiert, der det Optimierungsproblem als Korrelationsmatrix darstelln tut — konkret als 2-Körper-Pauli-Matrix-Korrelationen üba $n=3$ Qubits $(Q_1, Q_2, Q_3)$. De Knotenfarben zeign den jeweilign Pauli-String, der für jeden kodierten Knoten jenutzt wird.
Zum Beispiel wird Knoten 1, der da binären Variable $x_1$ entspricht, durch den Erwartungswert von $Z_1 \otimes Z_2 \otimes I_3$ codiert, während $x_8$ durch $I_1 \otimes Y_2 \otimes Y_3$ codiert wird.
Det entspricht 'ner Kompression der $m$ Variablen des Problems in $ n = O(m^{1/2})$ Qubits. Jenauer jesagt ermöglichn $k$-Körper-Korrelationen polynomielle Kompressionn der Ordnung $k$. Det jewählte Pauli-Set besteht aus drei Teilmenjen von jejensitig kommutierenden Pauli-Strings, wodurch alle $m$ Korrelationen experimentell mit nur drei Messeinstellungen geschätzt wern können.

'Ne Verlustfunktion $\mathcal{L}$ von Pauli-Erwartungswerten, die die ursärüngliche Max-Cut-Zielfunktion nachahmt, wird konstruiert. Die Verlustfunktion wird dann mit'm quantenklassischen Optimierungssolver optimiert, zum Beispiel mit dem Variational Quantum Eigensolver (VQE).

Sobald die Optimierung abgeschlossen is, wird die Lösung zurück in den ursärünglichen Optimierungsraum dekodiert, un die optimale Max-Cut-Lösung kommt raus.

Wir codieren den Graphen mit 1000 Knoten in 2-Körper-Pauli-Matrix-Korrelationen üba 100 Qubits. Der Graph wird als Korrelationsmatrix darjestellt, wo jeder Knoten durch 'nen Pauli-String codiert is. Det Vorzeichen des Erwartungswerts des Pauli-Strings zeigt die Partition des Knotens an. Zum Beispiel is Knoten 0 durch 'nen Pauli-String codiert: $\prod_0 = I_{19} \otimes ... I_2 \otimes X_1 \otimes X_0$. Det Vorzeichen des Erwartungswerts von diesem Pauli-String zeigt die Partition von Knoten 0 an. Wir definieren 'ne *Pauli-Korrelations-Codierung* (PCE) relativ zu $\prod$ als

$ x_i \coloneqq \textit{sgn}(\langle\prod_i \rangle) $

def build_pauli_correlation_encoding(pauli, node_list, n, k=2):
    pauli_correlation_encoding = []
    for idx, c in enumerate(combinations(range(n), k)):
        if idx >= len(node_list):
            break
        paulis = ["I"] * n
        paulis[c[0]], paulis[c[1]] = pauli, pauli
        pauli_correlation_encoding.append(("".join(paulis)[::-1], 1))

    hamiltonian = []
    for pauli, weight in pauli_correlation_encoding:
        hamiltonian.append(SparsePauliOp.from_list([(pauli, weight)]))

    return hamiltonian

### Verlustfunktion {#loss-function}
Für die Verlustfunktion $\mathcal{L}$ verwenden wir 'ne Relaxation der Max-Cut-Zielfunktion, wie in [\[1\]](#references) beschrieben, die als $\mathcal{V}(\mathbf{x}) \coloneqq \sum_{(i, j) \in E} W_{i, j}(1-x_i x_j)$ definiert is. Hierbei bezeichnet $W_{i, j}$ det Jewicht der Kante $(i, j)$, un $x_i$ stellt die Partition von Knoten $i$ dar.
Die Verlustfunktion $\mathcal{L}$ is jejebn durch:

$\mathcal{L}\coloneqq \sum_{(i, j) \in E} W_{i, j} \text{tanh} (\alpha \langle\prod_i \rangle) \text{tanh} (\alpha \langle\prod_j \rangle) + \mathcal{L}^{(\text{reg})}$

wobei die Max-Cut-Zielfunktion durch die glattan hyperbolischen Tangens der Erwartungswerte der Pauli-Strings, die die Knoten codieren, ersetzt wird. Der Regularisierungsterm $\mathcal{L}^{(\text{reg})}$ un der Skalierungsfaktor $\alpha$, proportional zur Anzahl der Qubits, werden einjeführt, um die Leistung des Solvers zu vabessern.

Der Regularisierungsterm is definiert als:

$\mathcal{L}^{(\text{reg})}$ is definiert als $\mathcal{L}^{(\text{reg})} \coloneqq \beta \nu \lbrack \frac{1}{m} \sum_{i \in V} \text{tanh} (\alpha \langle\prod_i \rangle)^2 \rbrack ^2$

wo $\beta=1/2$, $\nu = |E|/2 + (m -1) /4$ un $m$ die Anzahl der Knoten im Graphen is.

```python
def loss_func_estimator(x, ansatz, hamiltonian, estimator, graph):
    """
    Calculates the specified loss function for the given ansatz, Hamiltonian, and graph.

    The expectation values of each Pauli string in the Hamiltonian are first obtained
    by running the ansatz on the quantum backend. These expectation values are then
    passed through the nonlinear function tanh(alpha * prod_i). The loss function is
    subsequently computed from these transformed values.
    """
    job = estimator.run(
        [
            (ansatz, hamiltonian[0], x),
            (ansatz, hamiltonian[1], x),
            (ansatz, hamiltonian[2], x),
        ]
    )
    result = job.result()

    # calculate the loss function
    node_exp_map = {}
    idx = 0
    for r in result:
        for ev in r.data.evs:
            node_exp_map[idx] = ev
            idx += 1

    loss = 0
    alpha = num_qubits
    for edge0, edge1 in graph.edge_list():
        loss += np.tanh(alpha * node_exp_map[edge0]) * np.tanh(
            alpha * node_exp_map[edge1]
        )

    regulation_term = 0
    for i in range(len(graph.nodes())):
        regulation_term += np.tanh(alpha * node_exp_map[i]) ** 2
    regulation_term = regulation_term / len(graph.nodes())
    regulation_term = regulation_term**2
    beta = 1 / 2
    v = len(graph.edges()) / 2 + (len(graph.nodes()) - 1) / 4
    regulation_term = beta * v * regulation_term

    loss = loss + regulation_term

    global experiment_result
    print(f"Iter {len(experiment_result)}: {loss}")
    experiment_result.append({"loss": loss, "exp_map": node_exp_map})
    return loss
```

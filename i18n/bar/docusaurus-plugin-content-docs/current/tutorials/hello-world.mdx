Bauts an [Bell-Zustand](https://en.wikipedia.org/wiki/Bell_state) (zwoa verschränkte Qubits) und führts eam auf drei Arten aus:

1. **Ideale Simulation** — perfekte Ergebnisse, koa Konto ned braucht
2. **Simulation mit Rauschen** — simuliert a echtes Gerät, koa Konto ned braucht
3. **Echte Quantenhardware** — braucht a [IBM Quantum-Konto](https://janlahmann.github.io/Qiskit-documentation/#setting-up-your-ibm-quantum-account)

## Schaltkreis erstellen {#schaltkreis-erstellen}

```python
from qiskit import QuantumCircuit

qc = QuantumCircuit(2)
qc.h(0)
qc.cx(0, 1)
qc.measure_all()

qc.draw(output="mpl")
```

## Option 1: Ideale Simulation (koa Konto nötig) {#option-1-ideal-simulation-no-account-needed}

Verwendet `StatevectorSampler` — a lokaler Simulator mit perfekte, rauschfreie Ergebnisse.

```python
from qiskit.primitives import StatevectorSampler

result = StatevectorSampler().run([qc], shots=1024).result()
counts = result[0].data.meas.get_counts()
counts
```

```python
from qiskit.visualization import plot_histogram
plot_histogram(counts)
```

## Option 2: Simulation mit Rauschen (koa Konto nötig) {#option-2-noisy-simulation-no-account-needed}

Verwendet `FakeManilaV2` — a lokaler Simulator, der a echtes IBM-Quantengerät nachahmt, aa mit seine Rauscheigenschaften. Da Schaltkreis muass erst transpiliert (angepasst) wern, damit er zum Gate-Satz und zur Qubit-Konnektivität vom Gerät passt.

```python
from qiskit_ibm_runtime import SamplerV2
from qiskit_ibm_runtime.fake_provider import FakeManilaV2
from qiskit.transpiler.preset_passmanagers import generate_preset_pass_manager

backend = FakeManilaV2()
pm = generate_preset_pass_manager(backend=backend, optimization_level=1)
isa_qc = pm.run(qc)

result = SamplerV2(mode=backend).run([isa_qc], shots=1024).result()
counts = result[0].data.meas.get_counts()
counts
```

```python
plot_histogram(counts)
```

## Option 3: Echte Quantenhardware {#option-3-echte-quantenhardware}

Braucht a IBM Quantum-Konto. Schaugts bei [Setting up your IBM Quantum account](https://janlahmann.github.io/Qiskit-documentation/#setting-up-your-ibm-quantum-account) für de Details.

Falls Eahna Zugangsdaten no net in dera Binder-Sitzung gspeichert san, führts des z'erscht aus:

```python
from qiskit_ibm_runtime import QiskitRuntimeService

QiskitRuntimeService.save_account(
    token="<your-api-key>",
    instance="<your-crn>",
    overwrite=True
)
```

**Hinweis:** Jobs auf echter Hardware kenna je nach Wartezeiten in da Warteschlang a bisserl dauern. Falls de Zell no lauft, kenna mia den Job-Status überpriafn und de Ergebnisse auf [quantum.cloud.ibm.com/workloads](https://quantum.cloud.ibm.com/workloads?user=me) anschaun.

```python
from qiskit_ibm_runtime import QiskitRuntimeService, SamplerV2
from qiskit.transpiler.preset_passmanagers import generate_preset_pass_manager

service = QiskitRuntimeService()
backend = service.least_busy(operational=True, simulator=False)
print(f"Running on {backend.name}")

pm = generate_preset_pass_manager(backend=backend, optimization_level=1)
isa_qc = pm.run(qc)

result = SamplerV2(mode=backend).run([isa_qc], shots=1024).result()
counts = result[0].data.meas.get_counts()
counts
```

```python
plot_histogram(counts)
```

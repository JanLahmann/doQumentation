## Code-Ausführung {#code-execution}

Klick bei jedem Codeblock auf **Run**. Da erste Klick startet an kostenlosen Jupyter-Kernel üba [Binder](https://mybinder.org) (1–2 Min.). Danoch gehts sofort.

- **Simulator-Modus (koa Konto nötig)** — Aktivier'n [Simulator-Modus](/jupyter-settings#simulator-mode), um olle Notebooks mit AerSimulator oda FakeBackends mit Rauschmodell auszführen. Koa Einrichtung nötig.

- **IBM Quantum Hardware** — Gib dein API-Token und CRN oamoi in de [Einstellungen](/jupyter-settings#ibm-quantum) ei — de Zugangsdotn wern bei jeder Ausführung automatisch eigfügt. [Erstell a kostenloses Konto](https://quantum.cloud.ibm.com/registration), falls'd no koas host.

<details>
<summary>Verfügbare Ausführungs-Backends</summary>

Jedes Tutorial hot ausführbare Codeblöck. Klick auf **Run**, um's mit oam vo drei Backends auszführen:

1. **Binder** (Standard auf GitHub Pages) — Kostenloser Remote-Jupyter-Kernel üba [mybinder.org](https://mybinder.org)
2. **Lokales Jupyter (Docker / RasQberry)** — Verbindt si mit'm lokalen Jupyter-Server mit vorinstalliertem Qiskit
3. **Benutzerdefinierter Server** — Verweis in de [Einstellungen](/jupyter-settings#advanced) auf an beliebigen Jupyter-Endpunkt

<details>
<summary>Lokal ausführen mit Podman / Docker</summary>

Installier [Podman Desktop](https://podman-desktop.io/) oda [Docker Desktop](https://www.docker.com/products/docker-desktop/) (Mac / Windows / Linux), donn führ aus:

```bash
# Vollständiger Stack: Website + Jupyter + Qiskit (~3 GB)
podman run -p 8080:80 -p 8888:8888 ghcr.io/janlahmann/doqumentation:jupyter
```

Öffne [http://localhost:8080](http://localhost:8080) — Code-Ausführung funktioniert lokal, koa Binder-Wortezeit.

Fia a leichtgewichtige Version ohne lokale Code-Ausführung (~60 MB):

```bash
# Nur statische Website — Code-Ausführung funktioniert weiterhin üba Binder
podman run -p 8080:80 ghcr.io/janlahmann/doqumentation:latest
```

Du verwendest Docker? Ersetze einfach `podman` durch `docker` — de Befehle san identisch. De Images san Multi-Arch (`linux/amd64` + `linux/arm64`), sodass Apple Silicon Macs nativ funktionieren.

</details>

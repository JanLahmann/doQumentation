## Contexte {#background}

Ce tutoriel démontre comment implémenter la **Compilation Quantique Approximative** à l'aide de réseaux de tenseurs (AQC-Tensor) avec Qiskit pour améliorer les performances des circuits quantiques. AQC-Tensor compresse les circuits de Trotter profonds en circuits plus courts et plus compatibles avec le matériel, tout en préservant la précision de la simulation.

### Fonctionnement d'AQC-Tensor {#how-aqc-tensor-works}

Considère la simulation d'un hamiltonien $H$ pour un temps total $t$ à l'aide de $k$ pas de Trotter. Le circuit de Trotter complet est :

$$
U_{\text{full}} = \left[U_{\text{Trotter}}(t/k)\right]^k
$$

Une approche naïve utilise peu de pas de Trotter pour maintenir la profondeur du circuit gérable, mais cela introduit une erreur de Trotter significative. AQC-Tensor résout cette tension en séparant la précision de la profondeur :

1. **Circuit cible (haute précision, profond) :** Construire un circuit de Trotter avec de nombreux pas&mdash;disons, $10k$&mdash;pour le même temps d'évolution. Ce circuit a bien moins d'erreur de Trotter, mais est trop profond pour le matériel. Comme il est uniquement simulé classiquement comme un état produit de matrices (MPS), la profondeur n'est pas un problème.

2. **Circuit ansatz (faible profondeur, paramétré) :** Définir un circuit paramétré $V(\theta)$ avec la même structure qu'un circuit de Trotter à un pas. L'initialiser de façon que $V(\theta_{\text{init}}) = U_{\text{Trotter}}(t/k)$, puis optimiser itérativement $\theta$ de façon que $V(\theta)$ reproduise l'état cible de haute précision aussi fidèlement que possible.

Le résultat est un circuit qui conserve la profondeur d'un seul pas de Trotter mais atteint la précision de nombreux pas, le rendant réalisable sur du matériel quantique actuel.

### Quand utiliser AQC-Tensor {#when-to-use-aqc-tensor}

AQC-Tensor est le plus efficace lorsque :

- **La profondeur du circuit dépasse les temps de cohérence du matériel.** Si une simulation de Trotter nécessite plus de pas de Trotter que le dispositif peut en supporter, AQC-Tensor peut compresser l'évolution en un circuit moins profond.
- **L'intrication reste classiquement tractable.** L'intrication totale dans un état soumis à une évolution temporelle dépend principalement du temps d'évolution $t$, et non du nombre de pas de Trotter $k$. Cela signifie qu'un circuit cible avec $10k$ pas est généralement aussi facile à représenter comme un MPS qu'un circuit avec $k$ pas, tant que $t$ est assez court pour que les dimensions de lien restent gérables.
- **Un ansatz naturel existe.** Parce que l'ansatz reflète la structure d'un circuit de Trotter, il fournit un point de départ physiquement motivé avec des paramètres initiaux bien définis, évitant les problèmes de convergence pouvant affecter les ansatze variationnels arbitraires.

Cette approche contraste avec la compression générique de circuits : plutôt que d'essayer d'approximer un unitaire arbitraire avec moins de portes, AQC-Tensor conserve la même structure de portes et optimise ses paramètres pour réduire l'erreur de Trotter. Consulte la [documentation AQC-Tensor](https://qiskit.github.io/qiskit-addon-aqc-tensor/) pour plus d'informations.

Ce tutoriel te guide à travers le flux de travail complet AQC-Tensor pour la préparation d'état : définir un hamiltonien, générer des circuits de Trotter, les compresser via l'optimisation par réseau de tenseurs, et exécuter le résultat sur du matériel IBM Quantum&reg;.

## Exemple à petite échelle sur simulateur {#small-scale-simulator-example}

Cette section utilise un système à 10 sites pour illustrer pas à pas le flux de travail AQC-Tensor. Nous simulons la dynamique d'une chaîne de spins XXZ à 10 sites, un modèle largement étudié pour examiner les interactions de spin et les propriétés magnétiques.

Le hamiltonien est le suivant :

$$
\hat{\mathcal{H}}_{XXZ} = \sum_{i=1}^{L-1} J_{i,(i+1)}\left(X_i X_{(i+1)}+Y_i Y_{(i+1)}+ 2\cdot Z_i Z_{(i+1)} \right) \, ,
$$

où $J_{i,(i+1)}$ est un coefficient aléatoire pour l'arête $(i, i+1)$ et $L=10$.
### Étape 1 : Formuler les entrées classiques en un problème quantique {#step-1-map-classical-inputs-to-a-quantum-problem}

Dans cette étape, nous :
1. Définissons le hamiltonien, l'observable et l'état initial.
2. Calculons la valeur d'espérance exacte classiquement pour une comparaison ultérieure.
3. Générons un circuit de Trotter de haute précision (la cible AQC) et le compressons en un ansatz de faible profondeur à l'aide d'AQC-Tensor.
#### Configuration du hamiltonien, de l'observable et de l'état initial {#set-up-the-hamiltonian-observable-and-initial-state}

```python
# L is the number of sites in the 1D spin chain
L = 10

# Generate the coupling map
edge_list = [(i - 1, i) for i in range(1, L)]
even_edges = edge_list[::2]
odd_edges = edge_list[1::2]
coupling_map = CouplingMap(edge_list)

# Generate random coefficients for our XXZ Hamiltonian
np.random.seed(0)
Js = np.random.rand(L - 1) + 0.5 * np.ones(L - 1)
hamiltonian = SparsePauliOp(Pauli("I" * L))
for i, edge in enumerate(even_edges + odd_edges):
    hamiltonian += SparsePauliOp.from_sparse_list(
        [
            ("XX", (edge), Js[i] / 2),
            ("YY", (edge), Js[i] / 2),
            ("ZZ", (edge), Js[i]),
        ],
        num_qubits=L,
    )

# Generate a ZZ observable between the two middle qubits
observable = SparsePauliOp.from_sparse_list(
    [("ZZ", (L // 2 - 1, L // 2), 1.0)], num_qubits=L
)

# Generate an initial Néel state |1010101010⟩
initial_state_circuit = QuantumCircuit(L)
for i in range(L):
    if i % 2:
        initial_state_circuit.x(i)

print("Hamiltonian:", hamiltonian)
print("Observable:", observable)
graphviz_draw(coupling_map.graph, method="circo")
```

#### Calcul de la valeur d'espérance exacte {#compute-the-exact-expectation-value}

Pour un système de cette taille, nous pouvons calculer la valeur d'espérance exacte soumise à l'évolution temporelle directement par exponentiation de matrice. Cela sert de référence pour évaluer la précision du circuit AQC.

```python
aqc_evolution_time = 0.2

# Each baseline Trotter step covers dt = aqc_evolution_time / 3
# The subsequent (uncompressed) step covers 1 additional dt
subsequent_evolution_time = aqc_evolution_time / 3
total_evolution_time = aqc_evolution_time + subsequent_evolution_time

# Compute exact expectation value via matrix exponentiation
H_matrix = hamiltonian.to_matrix()
U_exact = expm(-1j * H_matrix * total_evolution_time)

# Build the initial state vector (Néel state)
initial_state_vec = np.zeros(2**L)
state_idx = sum(2**i for i in range(L) if i % 2)
initial_state_vec[state_idx] = 1.0

# Evolve and compute expectation value
evolved_state = U_exact @ initial_state_vec
obs_matrix = observable.to_matrix()
exact_expval = (evolved_state.conj() @ obs_matrix @ evolved_state).real

print(f"AQC evolution time: {aqc_evolution_time}")
print(f"Subsequent evolution time: {subsequent_evolution_time:.6f}")
print(f"Total evolution time: {total_evolution_time:.6f}")
print(f"Exact expectation value: {exact_expval:.6f}")
```

#### Génération du circuit cible AQC {#generate-the-aqc-target-circuit}

Nous construisons maintenant le circuit de Trotter qui servira de cible AQC. Ce circuit utilise de nombreux pas de Trotter (32) pour une haute précision. Comme il ne sera simulé que classiquement comme un MPS&mdash;et non exécuté sur du matériel&mdash;la grande profondeur n'est pas un problème.

```python
aqc_target_num_trotter_steps = 32

aqc_target_circuit = initial_state_circuit.copy()
aqc_target_circuit.compose(
    generate_time_evolution_circuit(
        hamiltonian,
        synthesis=SuzukiTrotter(reps=aqc_target_num_trotter_steps),
        time=aqc_evolution_time,
    ),
    inplace=True,
)
```

#### Génération d'un ansatz, des paramètres initiaux, du circuit subséquent et d'un circuit de référence {#generate-an-ansatz-initial-parameters-subsequent-circuit-and-a-baseline-circuit}

Nous construisons ensuite un « bon » circuit avec le même temps d'évolution que la cible AQC mais bien moins de pas de Trotter (un seul). Nous passons ce circuit à `generate_ansatz_from_circuit`, qui retourne :

1. Un circuit **ansatz** général et paramétré ayant la même connectivité à deux qubits.
2. Des **paramètres initiaux** qui reproduisent le circuit d'entrée lorsqu'ils sont injectés dans l'ansatz.

Nous construisons également :
- Un **circuit subséquent** avec un pas de Trotter qui sera ajouté (non compressé) après la partie optimisée par AQC, en suivant l'approche du [tutoriel AQC-Tensor sur l'état initial](https://qiskit.github.io/qiskit-addon-aqc-tensor/tutorials/01_initial_state_aqc.html).
- Un **circuit de Trotter de référence** utilisant quatre pas de Trotter sur la durée d'évolution totale (`aqc_evolution_time + subsequent_evolution_time`). Il sert de comparaison : il représente ce qu'on exécuterait sur le matériel sans AQC. L'ansatz AQC (3 pas compressés + 1 pas non compressé) atteint une meilleure précision avec une profondeur plus faible.

```python
aqc_ansatz_num_trotter_steps = 1

aqc_good_circuit = initial_state_circuit.copy()
aqc_good_circuit.compose(
    generate_time_evolution_circuit(
        hamiltonian,
        synthesis=SuzukiTrotter(reps=aqc_ansatz_num_trotter_steps),
        time=aqc_evolution_time,
    ),
    inplace=True,
)

aqc_ansatz, aqc_initial_parameters = generate_ansatz_from_circuit(
    aqc_good_circuit
)

# Subsequent circuit: 1 non-compressed Trotter step appended after AQC
subsequent_num_trotter_steps = 1
subsequent_circuit = generate_time_evolution_circuit(
    hamiltonian,
    synthesis=SuzukiTrotter(reps=subsequent_num_trotter_steps),
    time=subsequent_evolution_time,
)

# Baseline Trotter circuit: 4 Trotter steps over total evolution time, no AQC
baseline_num_trotter_steps = 4
baseline_circuit = initial_state_circuit.copy()
baseline_circuit.compose(
    generate_time_evolution_circuit(
        hamiltonian,
        synthesis=SuzukiTrotter(reps=baseline_num_trotter_steps),
        time=total_evolution_time,
    ),
    inplace=True,
)

print(
    f"Target circuit:      depth {aqc_target_circuit.depth(lambda x: x.operation.num_qubits == 2)}"
)
print(
    f"Baseline circuit:    depth {baseline_circuit.depth(lambda x: x.operation.num_qubits == 2)} ({baseline_num_trotter_steps} Trotter steps, time={total_evolution_time:.4f})"
)
print(
    f"Subsequent circuit:  depth {subsequent_circuit.depth(lambda x: x.operation.num_qubits == 2)} ({subsequent_num_trotter_steps} Trotter step, time={subsequent_evolution_time:.4f})"
)
print(
    f"Ansatz circuit:      depth {aqc_ansatz.depth(lambda x: x.operation.num_qubits == 2)}, with {len(aqc_initial_parameters)} parameters"
)
aqc_ansatz.draw("mpl", fold=-1)
```

#### Configuration de la simulation par réseau de tenseurs et construction du MPS cible {#set-up-tensor-network-simulation-and-build-the-target-mps}

Nous utilisons le simulateur de circuits à état produit de matrices (MPS) de [quimb](https://github.com/jcmgray/quimb), avec JAX fournissant la différentiation automatique pour l'optimisation par gradient. #### Optimisation des paramètres de l'ansatz {#optimize-the-ansatz-parameters}

Nous minimisons la fonction de coût `MaximizeStateFidelity` à l'aide de l'optimiseur L-BFGS-B. L'optimiseur ajuste itérativement les paramètres de l'ansatz pour maximiser la fidélité entre le circuit ansatz et le MPS cible.

```python
aqc_stopping_fidelity = 1
aqc_max_iterations = 500

stopping_point = 1.0 - aqc_stopping_fidelity
objective = MaximizeStateFidelity(
    aqc_target_mps, aqc_ansatz, simulator_settings
)

def callback(intermediate_result: OptimizeResult):
    fidelity = 1 - intermediate_result.fun
    print(
        f"{datetime.datetime.now()} Intermediate result: Fidelity {fidelity:.8f}"
    )
    if intermediate_result.fun < stopping_point:
        raise StopIteration

result = minimize(
    objective,
    aqc_initial_parameters,
    method="L-BFGS-B",
    jac=True,
    options={"maxiter": aqc_max_iterations},
    callback=callback,
)
if result.status not in (0, 1, 99):
    raise RuntimeError(
        f"Optimization failed: {result.message} (status={result.status})"
    )

print(f"Done after {result.nit} iterations.")
aqc_final_parameters = result.x
```

## Exemple à grande échelle sur matériel réel {#large-scale-hardware-example}

Nous passons maintenant à un modèle XXZ à 50 sites pour démontrer AQC-Tensor sur une taille de problème plus réaliste. Le flux de travail est le même que dans l'exemple à petite échelle : nous compressons trois pas de Trotter via AQC et ajoutons un pas non compressé.

Pour un système de cette taille, l'exponentiation de matrice n'est pas réalisable ($2^{50}$ dimensions), donc nous calculons la valeur d'espérance de référence directement à partir d'un MPS de haute précision soumis à l'évolution pour la durée totale.

### Étapes 1 à 4 combinées {#steps-14-combined}

```python
# -------------------------Step 1-------------------------

# Define the 50-site spin chain
L = 50
edge_list = [(i - 1, i) for i in range(1, L)]
even_edges = edge_list[::2]
odd_edges = edge_list[1::2]
coupling_map = CouplingMap(edge_list)

# Random XXZ Hamiltonian
np.random.seed(0)
Js = np.random.rand(L - 1) + 0.5 * np.ones(L - 1)
hamiltonian = SparsePauliOp(Pauli("I" * L))
for i, edge in enumerate(even_edges + odd_edges):
    hamiltonian += SparsePauliOp.from_sparse_list(
        [
            ("XX", (edge), Js[i] / 2),
            ("YY", (edge), Js[i] / 2),
            ("ZZ", (edge), Js[i]),
        ],
        num_qubits=L,
    )

observable = SparsePauliOp.from_sparse_list(
    [("ZZ", (L // 2 - 1, L // 2), 1.0)], num_qubits=L
)

# Initial Néel state
initial_state_circuit = QuantumCircuit(L)
for i in range(L):
    if i % 2:
        initial_state_circuit.x(i)

# Time parameters
aqc_evolution_time = 0.2
subsequent_evolution_time = aqc_evolution_time / 3
total_evolution_time = aqc_evolution_time + subsequent_evolution_time

# AQC target circuit (high-accuracy, 32 Trotter steps for AQC portion)
aqc_target_num_trotter_steps = 32

aqc_target_circuit = initial_state_circuit.copy()
aqc_target_circuit.compose(
    generate_time_evolution_circuit(
        hamiltonian,
        synthesis=SuzukiTrotter(reps=aqc_target_num_trotter_steps),
        time=aqc_evolution_time,
    ),
    inplace=True,
)

# Generate ansatz from 1-step Trotter circuit
aqc_good_circuit = initial_state_circuit.copy()
aqc_good_circuit.compose(
    generate_time_evolution_circuit(
        hamiltonian,
        synthesis=SuzukiTrotter(reps=1),
        time=aqc_evolution_time,
    ),
    inplace=True,
)

aqc_ansatz, aqc_initial_parameters = generate_ansatz_from_circuit(
    aqc_good_circuit
)

# Subsequent circuit: 1 non-compressed Trotter step
subsequent_circuit = generate_time_evolution_circuit(
    hamiltonian,
    synthesis=SuzukiTrotter(reps=1),
    time=subsequent_evolution_time,
)

# Baseline Trotter circuit: 4 Trotter steps over total evolution time, no AQC
baseline_num_trotter_steps = 4
baseline_circuit = initial_state_circuit.copy()
baseline_circuit.compose(
    generate_time_evolution_circuit(
        hamiltonian,
        synthesis=SuzukiTrotter(reps=baseline_num_trotter_steps),
        time=total_evolution_time,
    ),
    inplace=True,
)
print(
    f"Target circuit:  depth {aqc_target_circuit.depth(lambda x: x.operation.num_qubits == 2)}"
)
print(
    f"Ansatz circuit:  depth {aqc_ansatz.depth(lambda x: x.operation.num_qubits == 2)}, with {len(aqc_initial_parameters)} parameters"
)
print(
    f"Subsequent circuit: depth {subsequent_circuit.depth(lambda x: x.operation.num_qubits == 2)}"
)
print(
    f"Baseline circuit:   depth {baseline_circuit.depth(lambda x: x.operation.num_qubits == 2)} ({baseline_num_trotter_steps} steps, time={total_evolution_time:.4f})"
)

# Build target MPS and compute reference expectation value
simulator_settings = QuimbSimulator(
    quimb.tensor.CircuitMPS, autodiff_backend="jax"
)
aqc_target_mps = tensornetwork_from_circuit(
    aqc_target_circuit, simulator_settings
)
print("Target MPS maximum bond dimension:", aqc_target_mps.psi.max_bond())

# For the reference expectation value, we need the full evolution (AQC + subsequent)
# Build a high-accuracy full circuit for MPS reference
full_target_circuit = initial_state_circuit.copy()
full_target_circuit.compose(
    generate_time_evolution_circuit(
        hamiltonian,
        synthesis=SuzukiTrotter(reps=aqc_target_num_trotter_steps),
        time=total_evolution_time,
    ),
    inplace=True,
)
full_target_mps = tensornetwork_from_circuit(
    full_target_circuit, simulator_settings
)
exact_expval = full_target_mps.local_expectation(
    quimb.pauli("Z") & quimb.pauli("Z"), (L // 2 - 1, L // 2)
).real.item()
print(f"Reference expectation value (from MPS): {exact_expval:.6f}")

# Optimize ansatz parameters
objective = MaximizeStateFidelity(
    aqc_target_mps, aqc_ansatz, simulator_settings
)

def callback(intermediate_result: OptimizeResult):
    fidelity = 1 - intermediate_result.fun
    print(
        f"{datetime.datetime.now()} Intermediate result: Fidelity {fidelity:.8f}"
    )

result = minimize(
    objective,
    aqc_initial_parameters,
    method="L-BFGS-B",
    jac=True,
    options={"maxiter": 500},
    callback=callback,
)
if result.status not in (0, 1, 99):
    raise RuntimeError(
        f"Optimization failed: {result.message} (status={result.status})"
    )
print(f"Done after {result.nit} iterations.")

# Assemble the final AQC circuit: optimized ansatz + subsequent Trotter step
aqc_final_circuit = aqc_ansatz.assign_parameters(result.x)
aqc_final_circuit.compose(subsequent_circuit, inplace=True)

# -------------------------Step 2-------------------------

service = QiskitRuntimeService()
backend = service.least_busy(min_num_qubits=127)
print(backend)

pass_manager = generate_preset_pass_manager(
    backend=backend, optimization_level=3
)
isa_circuit = pass_manager.run(aqc_final_circuit)
isa_observable = observable.apply_layout(isa_circuit.layout)
print(
    "AQC circuit depth:",
    isa_circuit.depth(lambda x: x.operation.num_qubits == 2),
)

# Also transpile the baseline Trotter circuit (4 Trotter steps, no AQC)
isa_baseline_circuit = pass_manager.run(baseline_circuit)
isa_baseline_observable = observable.apply_layout(isa_baseline_circuit.layout)
print(
    "Baseline Trotter circuit depth:",
    isa_baseline_circuit.depth(lambda x: x.operation.num_qubits == 2),
)

# -------------------------Step 3-------------------------

# Submit both circuits in a single job
estimator = Estimator(backend)
estimator.options.environment.job_tags = ["TUT_AQCTE"]

job = estimator.run(
    [
        (isa_circuit, isa_observable),
        (isa_baseline_circuit, isa_baseline_observable),
    ]
)
print("Job ID:", job.job_id())
```

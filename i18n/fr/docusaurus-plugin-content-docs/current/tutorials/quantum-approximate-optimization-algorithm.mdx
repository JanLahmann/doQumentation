#### Graphe &rarr; problÃĻme d'optimisation {#graph-optimization-problem}

La premiÃĻre ÃĐtape de la transposition est un changement de notation. Ce qui suit exprime le problÃĻme en notation QUBO :

oÃđ $Q$ est une matrice $n\times n$ de nombres rÃĐels, $n$ correspond au nombre de nÅuds dans votre graphe, $x$ est le vecteur de variables binaires introduit ci-dessus, et $x^T$ indique la transposÃĐe du vecteur $x$.

### ProblÃĻme d'optimisation &rarr; hamiltonien {#optimization-problem-hamiltonian}

Vous pouvez ensuite reformuler le problÃĻme QUBO sous forme d'un **hamiltonien** (ici, une matrice qui reprÃĐsente l'ÃĐnergie d'un systÃĻme) :

$$

Pour montrer comment le problÃĻme QAOA peut ÃŠtre rÃĐÃĐcrit de cette maniÃĻre, remplacez d'abord les variables binaires $x_i$ par un nouvel ensemble de variables $z_i\in\{-1, 1\}$ via

$$

$$

Ici, vous pouvez voir que si $x_i$ vaut $0$, alors $z_i$ doit valoir $1$. Lorsque les $x_i$ sont substituÃĐs par les $z_i$ dans le problÃĻme d'optimisation ($x^TQx$), une formulation ÃĐquivalente peut ÃŠtre obtenue.

$$
x^TQx=\sum_{ij}Q_{ij}x_ix_j \\ =\frac{1}{4}\sum_{ij}Q_{ij}(1-z_i)(1-z_j) \\=\frac{1}{4}\sum_{ij}Q_{ij}z_iz_j-\frac{1}{4}\sum_{ij}(Q_{ij}+Q_{ji})z_i + \frac{n^2}{4}.
$$

Maintenant, si nous dÃĐfinissons $b_i=-\sum_{j}(Q_{ij}+Q_{ji})$, supprimons le prÃĐfacteur et le terme constant $n^2$, nous obtenons les deux formulations ÃĐquivalentes du mÃŠme problÃĻme d'optimisation.

$$
\min_{x\in\{0,1\}^n} x^TQx\Longleftrightarrow \min_{z\in\{-1,1\}^n}z^TQz + b^Tz
$$

Ici, $b$ dÃĐpend de $Q$. Notez que pour obtenir $z^TQz + b^Tz$ nous avons supprimÃĐ le facteur de 1/4 et un dÃĐcalage constant de $n^2$ qui ne jouent aucun rÃīle dans l'optimisation.

Maintenant, pour obtenir une formulation quantique du problÃĻme, promouvez les variables $z_i$ en une matrice de Pauli $Z$, telle qu'une matrice $2\times 2$ de la forme

$$
Z_i = \begin{pmatrix}1 & 0 \\ 0 & -1\end{pmatrix}.
$$

Lorsque vous substituez ces matrices dans le problÃĻme d'optimisation ci-dessus, vous obtenez l'hamiltonien suivant

$$

$$

*Rappelez-vous ÃĐgalement que les matrices $Z$ sont intÃĐgrÃĐes dans l'espace de calcul de l'ordinateur quantique, c'est-Ã -dire un espace de Hilbert de taille $2^n\times 2^n$. Par consÃĐquent, vous devez comprendre des termes tels que $Z_iZ_j$ comme le produit tensoriel $Z_i\otimes Z_j$ intÃĐgrÃĐ dans l'espace de Hilbert $2^n\times 2^n$. Par exemple, dans un problÃĻme avec cinq variables de dÃĐcision, le terme $Z_1Z_3$ doit ÃŠtre compris comme $I\otimes Z_3\otimes I\otimes Z_1\otimes I$ oÃđ $I$ est la matrice identitÃĐ $2\times 2$.*

Cet hamiltonien est appelÃĐ l'**hamiltonien de la fonction de coÃŧt**. Il a la propriÃĐtÃĐ que son ÃĐtat fondamental correspond Ã  la solution qui **minimise la fonction de coÃŧt $f(x)$**.
Par consÃĐquent, pour rÃĐsoudre votre problÃĻme d'optimisation, vous devez maintenant prÃĐparer l'ÃĐtat fondamental de $H_C$ (ou un ÃĐtat ayant un fort recouvrement avec celui-ci) sur l'ordinateur quantique. ConsidÃĐrons maintenant l'hamiltonien $H_C$ pour le problÃĻme **Max-Cut**. Soit chaque sommet du graphe associÃĐ Ã  un qubit dans l'ÃĐtat $|0\rangle$ ou $|1\rangle$, oÃđ la valeur indique l'ensemble auquel le sommet appartient. L'objectif du problÃĻme est de maximiser le nombre d'arÃŠtes $(v_1, v_2)$ pour lesquelles $v_1 = |0\rangle$ et $v_2 = |1\rangle$, ou vice-versa. Si nous associons l'opÃĐrateur $Z$ Ã  chaque qubit, oÃđ

$$
    Z|0\rangle = |0\rangle \qquad Z|1\rangle = -|1\rangle
$$

alors une arÃŠte $(v_1, v_2)$ appartient Ã  la coupe si la valeur propre de $(Z_1|v_1\rangle) \cdot (Z_2|v_2\rangle) = -1$ ; en d'autres termes, les qubits associÃĐs Ã  $v_1$ et $v_2$ sont diffÃĐrents. De mÃŠme, $(v_1, v_2)$ n'appartient pas Ã  la coupe si la valeur propre de $(Z_1|v_1\rangle) \cdot (Z_2|v_2\rangle) = 1$.     graph: rx.PyGraph,
) -> list[tuple[str, list[int], float]]:
    """Convert the graph to Pauli list.

    This function does the inverse of `build_max_cut_graph`

    pauli_list = []

        pauli_list.append(("ZZ", [edge[0], edge[1]], weight))

_PARITY = np.array(
    [-1 if bin(i).count("1") % 2 else 1 for i in range(256)],
    dtype=np.complex128,
)

def evaluate_sparse_pauli(state: int, observable: SparsePauliOp) -> complex:
    """Utility for the evaluation of the expectation value of a measured state."""
    packed_uint8 = np.packbits(observable.paulis.z, axis=1, bitorder="little")
    state_bytes = np.frombuffer(
        state.to_bytes(packed_uint8.shape[1], "little"), dtype=np.uint8
    )
    reduced = np.bitwise_xor.reduce(packed_uint8 & state_bytes, axis=1)
    return np.sum(observable.coeffs * _PARITY[reduced])

def best_solution(samples, hamiltonian):
    """Find solution with lowest cost"""
    min_cost = 1000
    min_sol = None
    for bit_str in samples.keys():
        # Qiskit use little endian hence the [::-1]
        candidate_sol = int(bit_str)
        # fval = qp.objective.evaluate(candidate_sol)
        fval = evaluate_sparse_pauli(candidate_sol, hamiltonian).real
        if fval <= min_cost:
            min_sol = candidate_sol

    return min_sol

# auxiliary function to help plot cumulative distribution functions
def _plot_cdf(objective_values: dict, ax, color):
    x_vals = sorted(objective_values.keys(), reverse=True)
    y_vals = np.cumsum([objective_values[x] for x in x_vals])
    ax.plot(x_vals, y_vals, color=color)

def plot_cdf(dist, ax, title):
    _plot_cdf(
        dist,
        ax,
        "C1",
    )
    ax.vlines(min(list(dist.keys())), 0, 1, "C1", linestyle="--")

    ax.set_title(title)
    ax.set_xlabel("Objective function value")
    ax.set_ylabel("Cumulative distribution function")
    ax.grid(alpha=0.3)

# auxiliary function to convert bit-strings to objective values
def samples_to_objective_values(samples, hamiltonian):
    """Convert the samples to values of the objective function."""

    objective_values = defaultdict(float)
    for bit_str, prob in samples.items():
        candidate_sol = int(bit_str)
        fval = evaluate_sparse_pauli(candidate_sol, hamiltonian).real
        objective_values[fval] += prob

    return objective_values

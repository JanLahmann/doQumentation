### 2.1 Niveau d'optimisation {#21-optimization-level}

Il existe 4 niveaux `optimization_level` disponibles, de 0 à 3. Plus le niveau d'optimisation est élevé, plus l'effort de calcul consacré à l'optimisation du circuit est important. Le niveau 0 n'effectue aucune optimisation et fait juste le minimum nécessaire pour rendre le circuit exécutable sur le backend sélectionné. Le niveau 3 consacre le plus d'effort (et généralement de temps d'exécution) pour tenter d'optimiser le circuit. Le niveau 1 est le niveau d'optimisation par défaut.

Nous transpilons le circuit sans optimisation (`optimization_level=0`) et avec optimisation (`optimization_level=2`).
On observe une grande différence dans la longueur des circuits transpilés.

```python
pm0 = generate_preset_pass_manager(
    optimization_level=0, backend=backend, seed_transpiler=777
)
pm2 = generate_preset_pass_manager(
    optimization_level=2, backend=backend, seed_transpiler=777
)
circ0 = pm0.run(ghz_circ)
circ2 = pm2.run(ghz_circ)
print("optimization_level=0:")
display(circ0.draw("mpl", idle_wires=False, fold=-1))
print("optimization_level=2:")
display(circ2.draw("mpl", idle_wires=False, fold=-1))

La nouvelle synthèse produit un circuit moins profond. Pourquoi ?

C'est parce que le nouveau circuit peut être disposé sur des qubits connectés linéairement, et donc aussi sur le graphe de couplage hexagonal lourd d'IBM&reg; Brisbane, tandis que le circuit original nécessite une connectivité en étoile (un nœud de degré 4) et ne peut donc pas être disposé sur le graphe de couplage heavy-hex, dont les nœuds ont au maximum le degré 3. En conséquence, le circuit original nécessite un routage de qubits qui ajoute des portes SWAP, ce qui augmente le nombre de portes.

Ce que nous avons fait dans le nouveau circuit peut être vu comme une synthèse de circuit « consciente des contraintes de couplage » effectuée manuellement. En d'autres termes : résoudre manuellement la synthèse du circuit et le mapping du circuit en même temps.

En général, la synthèse de circuit dépend de l'application et il est trop difficile pour un logiciel de couvrir toutes les applications possibles. Le Transpiler de Qiskit ne dispose pas de fonctions pour synthétiser le circuit de préparation d'état GHZ. from math import pi
import itertools
from qiskit.circuit import QuantumCircuit
from qiskit.circuit.library import excitation_preserving

circuit = QuantumCircuit(4, name="Example circuit")
circuit.append(excitation_preserving(4, reps=1, flatten=True), range(4))
circuit.measure_all()

value_cycle = itertools.cycle([0, pi / 4, pi / 2, 3 * pi / 4, pi, 2 * pi])
circuit.assign_parameters(
    [x[1] for x in zip(range(len(circuit.parameters)), value_cycle)], inplace=True
)
circuit.draw("mpl")
```

### 3.4 Étape d'optimisation {#34-optimization-stage}

L'étape d'optimisation vise à minimiser la taille et la profondeur du circuit. Elle exécute une série de passes dans une boucle ```do while``` jusqu'à atteindre une sortie stable.

## État par défaut {#default-state}

Un *état de référence* désigne le point de départ initial et fixe pour notre problème. Pour préparer un état de référence, nous devons appliquer l'unitaire non paramétrisé approprié $U_R$ au début de notre circuit quantique, de sorte que $|\rho\rangle = U_R |0\rangle$. Si tu disposes d'une estimation éclairée ou d'un point de données d'une solution optimale existante, l'algorithme variationnel convergera probablement plus vite si tu l'utilises comme point de départ.

L'état de référence le plus simple possible est l'état par défaut, où l'on utilise l'état initial d'un circuit quantique à $n$ qubits : $|0\rangle^{\otimes n}$. Pour l'état par défaut, notre opérateur unitaire $U_R \equiv I$. Du fait de sa simplicité, l'état par défaut est un état de référence valide utilisé dans de nombreux scénarios.

## État de référence classique {#classical-reference-state}

Supposons que tu aies un système à trois qubits et que tu veuilles démarrer dans l'état $|001\rangle$ au lieu de l'état par défaut $|000\rangle$. C'est un exemple d'état de référence purement classique, et pour le construire, il suffit d'appliquer une [porte X](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.library.XGate) au qubit $0$ (en suivant l'ordre des qubits de Qiskit), puisque $|001\rangle = X_0 |000\rangle$.

Dans ce cas, notre opérateur unitaire est $U_R \equiv X_0$, ce qui conduit à l'état de référence $|\rho\rangle \equiv |001\rangle$.

## État de référence quantique {#quantum-reference-state}

Supposons que tu souhaites démarrer avec un état plus complexe impliquant une superposition et/ou une intrication, comme $\frac{1}{\sqrt{2}}(|100\rangle+|111\rangle)$.

Pour obtenir cet état à partir de $|000\rangle$, une approche consiste à utiliser une [porte de Hadamard](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.library.HGate) sur le qubit $0$ ($H_0$), une porte [CNOT (CX)](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.library.CXGate) avec le qubit $0$ comme qubit de contrôle et le qubit $1$ comme qubit cible ($CNOT_{01}$), et enfin une porte $X$ appliquée au qubit $2$ ($X_2$).

Dans ce scénario, notre opérateur unitaire est $U_{R} \equiv X_2CNOT_{01}H_0|000\rangle$, et notre état de référence est $|\rho\rangle \equiv \frac{1}{\sqrt{2}}(|100\rangle+|111\rangle)$.

```python
qc = QuantumCircuit(3)
qc.h(0)
qc.cx(0, 1)
qc.x(2)

qc.draw("mpl")
```

## Construire des états de référence à l'aide de circuits modèles {#constructing-reference-states-using-template-circuits}

Nous pouvons également utiliser divers circuits modèles, comme [`TwoLocal`](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.library.TwoLocal) qui permet d'exprimer facilement de multiples paramètres réglables et des intrications. Nous couvrirons ces circuits modèles plus en détail dans la prochaine leçon, mais nous pouvons les utiliser pour nos états de référence *si* nous lions les paramètres :

```python
from qiskit.circuit.library import TwoLocal
from math import pi

reference_circuit = TwoLocal(2, "rx", "cz", entanglement="linear", reps=1)
theta_list = [pi / 2, pi / 3, pi / 3, pi / 2]

reference_circuit = reference_circuit.assign_parameters(theta_list)

reference_circuit.decompose().draw("mpl")
```

## États de référence spécifiques à l'application {#application-specific-reference-states}
### Apprentissage automatique quantique {#quantum-machine-learning}

Dans le contexte d'un classificateur quantique variationnel (VQC), les données d'entraînement sont encodées dans un état quantique avec un circuit paramétrisé connu sous le nom de *feature map*, où chaque valeur de paramètre représente un point de données de l'ensemble d'entraînement. La `zz_feature_map` est un type de circuit paramétrisé qui peut être utilisé pour passer nos points de données ($x$) à cette feature map.

```python
from qiskit.circuit.library import zz_feature_map

data = [0.1, 0.2]

zz_feature_map_reference = zz_feature_map(feature_dimension=2, reps=2)
zz_feature_map_reference = zz_feature_map_reference.assign_parameters(data)
zz_feature_map_reference.decompose().draw("mpl")
```

## Contenu open-source d'IBM Quantum {#ibm-quantums-open-source-content}

IBM fournit une abondance de matériel d'apprentissage sur l'informatique quantique — tout est open-source sous licence CC BY-SA 4.0 :

- **[Learning](https://quantum.cloud.ibm.com/learning)** — Cours structurés, des bases quantiques aux sujets avancés
- **[Tutorials](https://quantum.cloud.ibm.com/docs/en/tutorials)** — Plus de 40 tutoriels sur la transpilation, l'atténuation d'erreurs, et plus encore
- **[Documentation](https://quantum.cloud.ibm.com/docs/en/guides)** — Guides et référence API pour Qiskit
- **[Source repo](https://github.com/Qiskit/documentation)** — Tout le contenu sur GitHub

Leur [Quantum Platform](https://quantum.cloud.ibm.com) est toujours à jour et bien conçue — le meilleur endroit pour lire, apprendre et consulter.

## Ce que ce projet ajoute {#what-this-project-adds}

La [documentation Qiskit](https://github.com/Qiskit/documentation) d'IBM est open-source (CC BY-SA 4.0), mais leur application web ne l'est pas. doQumentation ajoute une interface open-source avec exécution de code en direct, injection automatique des identifiants, et mode simulateur.

Déployable partout — de [GitHub Pages](https://doqumentation.org) à [Docker](https://github.com/JanLahmann/doQumentation/pkgs/container/doqumentation) en passant par [RasQberry](https://rasqberry.org/). Découvrez [toutes les fonctionnalités](/features).

## Exécution de code {#code-execution}

Cliquez sur **Exécuter** sur n'importe quel bloc de code. Le premier clic démarre un noyau Jupyter gratuit via [Binder](https://mybinder.org) (1 à 2 min). Ensuite, les exécutions sont instantanées.

- **Mode Simulateur (aucun compte requis)** — Activez le [Mode Simulateur](/jupyter-settings#simulator-mode) pour exécuter tous les notebooks avec AerSimulator ou des FakeBackends avec modèle de bruit. Aucune configuration requise.

- **Matériel IBM Quantum** — Entrez votre jeton API et votre CRN dans les [Paramètres](/jupyter-settings#ibm-quantum) une fois — les identifiants sont injectés automatiquement à chaque exécution. [Créez un compte gratuit](https://quantum.cloud.ibm.com/registration) si vous n'en avez pas.

<details>
<summary>Backends d'exécution disponibles</summary>

Chaque tutoriel contient des blocs de code exécutables. Cliquez sur **Exécuter** pour les exécuter en utilisant l'un des trois backends :

1. **Binder** (par défaut sur GitHub Pages) — Noyau Jupyter distant gratuit via [mybinder.org](https://mybinder.org)
2. **Jupyter local (Docker / RasQberry)** — Se connecte au serveur Jupyter local avec Qiskit pré-installé
3. **Serveur personnalisé** — Pointez vers n'importe quel point de terminaison Jupyter dans les [Paramètres](/jupyter-settings#advanced)

</details>

<details>
<summary>Options de déploiement</summary>

doQumentation est disponible sous forme de :

- **[GitHub Pages](https://doqumentation.org)** — Site statique avec [Binder](https://mybinder.org) pour l'exécution de code à distance
- **[Docker](https://github.com/JanLahmann/doQumentation/pkgs/container/doqumentation)** — Stack complète localement, capable de fonctionner hors ligne
- **[RasQberry](https://rasqberry.org/)** — Auto-hébergé sur Raspberry Pi avec noyau Jupyter local

</details>

<details>
<summary>Exécuter localement avec Podman / Docker</summary>

Installez [Podman Desktop](https://podman-desktop.io/) ou [Docker Desktop](https://www.docker.com/products/docker-desktop/) (Mac / Windows / Linux), puis exécutez :

```bash
# Stack complète : site + Jupyter + Qiskit (~3 Go)
podman run -p 8080:80 -p 8888:8888 ghcr.io/janlahmann/doqumentation:jupyter
```

Ouvrez [http://localhost:8080](http://localhost:8080) — l'exécution de code fonctionne localement, sans temps d'attente Binder.

Pour une version légère sans exécution de code locale (~60 Mo) :

```bash
# Site statique uniquement — l'exécution de code fonctionne toujours via Binder
podman run -p 8080:80 ghcr.io/janlahmann/doqumentation:latest
```

Vous utilisez Docker plutôt que Podman ? Remplacez simplement `podman` par `docker` — les commandes sont identiques. Les images sont multi-architecture (`linux/amd64` + `linux/arm64`), donc les Mac Apple Silicon fonctionnent nativement.

</details>

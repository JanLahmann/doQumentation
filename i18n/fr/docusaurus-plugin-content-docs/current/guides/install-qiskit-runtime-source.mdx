# Installer le client Qiskit Runtime depuis les sources {#install-the-qiskit-runtime-client-from-source}

Lorsque tu installes Qiskit Runtime depuis les sources, tu peux accéder à la version de développement actuelle, au lieu d'utiliser la version du référentiel Python Package Index (PyPI). Tu peux donc inspecter et étendre plus efficacement la dernière version du code.

## Créer et activer un nouvel environnement virtuel {#create-and-activate-a-new-virtual-environment}

1. Navigue vers le répertoire de ton projet et crée un environnement minimal avec seulement Python installé.

          <TabItem value="mac" label="macOS">

## Installer Qiskit Runtime {#install-qiskit-runtime}

Suis ces étapes pour installer Qiskit Runtime :

1. Clone le dépôt Qiskit Runtime.

```bash
git clone https://github.com/Qiskit/qiskit-ibm-runtime.git
```

2. Change vers le répertoire `qiskit-ibm-runtime`.

```bash
cd qiskit-ibm-runtime
```

3. Installe `qiskit-runtime`. Nous recommandons d'utiliser un [environnement virtuel](https://docs.python.org/3/library/venv.html) pour éviter de polluer ton installation Python globale.

  * **Installation standard** :

    ```bash
    pip install .
    ```

  * **Mode éditable** : Dans ce mode, tu n'as pas besoin de réinstaller Qiskit lorsqu'il y a des modifications de code dans le projet.

    ```bash
    pip install -e .
    ```

    En mode éditable, les extensions compilées sont construites en _mode débogage_ sans optimisations.

4. (Facultatif) Si tu veux exécuter des tests ou des vérifications de formatage, installe les dépendances de développement. Nous recommandons d'utiliser un [environnement virtuel](https://docs.python.org/3/library/venv.html) pour éviter de polluer ton installation Python globale.

```bash
pip install -e ".[dev]"
```

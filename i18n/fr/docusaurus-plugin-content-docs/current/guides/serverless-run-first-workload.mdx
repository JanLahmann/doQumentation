## Exécuter un programme téléversé et lui passer des entrées {#run-an-uploaded-program-and-pass-inputs}

D'abord, configure tes entrées. Ton programme attend trois entrées : `circuits`, `backend` et `optimization_level`. Tu peux utiliser `random_circuit` pour créer 30 circuits aléatoires :

```python
from qiskit.circuit.random import random_circuit

qc_random = [(random_circuit(4, 4, measure=True, seed=i)) for i in range(10)]
qc_random[0].draw(output="mpl", idle_wires=False)
```

![Sortie de la cellule de code précédente](/docs/images/guides/serverless-run-first-workload/extracted-outputs/46108a95-11f8-4ecf-83bb-131c3e7ea4c2-0.svg)

Ensuite, utilise `QiskitRuntimeService` et `least_busy` pour sélectionner un `backend` :

```python
from qiskit_ibm_runtime import QiskitRuntimeService

service = QiskitRuntimeService()
backend = service.least_busy(operational=True, simulator=False)
print(backend.name)
```

Définis ton niveau d'optimisation :

```python
optimization_level = 3
```

Sélectionne ton programme avec `serverless.load('PROGRAM_NAME')` :

```python
transpile_remote_serverless = serverless.load("transpile_remote_serverless")
```

Ensuite, passe tes entrées et exécute-le de manière pythonique comme suit :

```python
job = transpile_remote_serverless.run(
    circuits=qc_random,
    backend=backend.name,
    optimization_level=optimization_level,
)
```

## Vérifier le statut du job {#check-job-status}

Avec le `job_id` de ton job Qiskit Serverless, tu peux vérifier le statut des jobs en cours. Cela inclut les statuts suivants :

- **`QUEUED`** : Le programme distant est dans la file d'attente de Qiskit Serverless. La priorité de la file d'attente est actuellement basée sur ton utilisation de Qiskit Serverless
- **`INITIALIZING`** : Le programme distant démarre ; cela inclut la configuration de l'environnement distant et l'installation des dépendances
- **`RUNNING`** : Le programme est en cours d'exécution. À ce stade, si ton programme contient des sorties `print()`, tu peux récupérer les journaux avec `job.logs()`
- **`DONE`** : Le programme est terminé, et tu peux récupérer les données enregistrées dans `save_result()` avec `job.results()`

Tu peux également définir des statuts de job plus détaillés dans [Gérer les ressources de calcul et de données Qiskit Serverless](/guides/serverless-manage-resources).

```python
job.status()
```

```text
'QUEUED'
```

```python
# This cell is hidden from users, it checks the job status
assert _ in ["QUEUED", "INITIALIZING", "RUNNING", "DONE"]  # noqa: F821
```

## Récupérer les journaux et les résultats {#retrieve-logs-and-results}

Comme mentionné précédemment, une fois qu'un programme est en état `RUNNING`, tu peux utiliser `job.logs()` pour récupérer les journaux créés à partir des sorties `print()` :

```python
logs = job.logs()
print(logs)
```

```text
No logs yet.
```

À tout moment, tu peux également annuler un job :

```python
job.stop()
```

```text
'Job has been stopped.'
```

Une fois qu'un programme est en état `DONE`, tu peux utiliser `job.results()` pour récupérer le résultat enregistré dans `save_result()` :

```python
# We can't get results from a cancelled job, so we'll fetch a completed one instead
completed_job = next(
    job for job in serverless.jobs() if job.status() == "DONE"
)
completed_job.result()
```

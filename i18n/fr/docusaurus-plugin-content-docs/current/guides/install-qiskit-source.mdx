# Installer le SDK Qiskit et Qiskit Runtime depuis les sources {#install-the-qiskit-sdk-and-qiskit-runtime-from-source}

Installer le SDK Qiskit depuis les sources te permet d'accéder à la version de développement actuelle, au lieu d'utiliser la version disponible dans le dépôt Python Package Index (PyPI). Cela te permet d'inspecter et d'étendre la dernière version du code Qiskit de manière plus efficace.

## Créer et activer un nouvel environnement virtuel {#create-and-activate-a-new-virtual-environment}

1. Navigue vers le répertoire de ton projet et crée un environnement minimal avec uniquement Python d'installé.

## Installer le compilateur Rust {#install-the-rust-compiler}

Un compilateur Rust doit être installé sur ton système pour compiler Qiskit. Pour installer le compilateur Rust, utilise l'installateur Rust multiplateforme [`rustup`](https://rustup.rs/) ou [une autre méthode d'installation.](https://forge.rust-lang.org/infra/other-installation-methods.html)

## Installer Qiskit {#install-qiskit}

Suis ces étapes pour installer Qiskit :

1. Clone le dépôt Qiskit.

```bash
git clone https://github.com/Qiskit/qiskit.git
```

2. Accède au répertoire `qiskit`.

```bash
cd qiskit
```

3. (Optionnel) Si tu veux exécuter des tests ou des vérifications de linting, installe les dépendances de développement.

```bash
pip install -r requirements-dev.txt
```

4. Installe `qiskit`.

  * **Installation standard** :

    ```bash
    pip install .
    ```

  * **Mode éditable** : Dans ce mode, tu n'as pas besoin de réinstaller Qiskit lorsqu'il y a des modifications de code dans le projet.

    ```bash
    pip install -e .
    ```

    En mode éditable, les extensions compilées sont construites en _mode debug_ sans optimisations. Cela affecte les performances à l'exécution du code compilé. Pour construire les extensions compilées avec les optimisations activées, exécute la commande suivante pour reconstruire le binaire en _mode release_ :

    ```bash
    python setup.py build_rust --release --inplace
    ```

    <Admonition type="note">
    Si tu travailles sur du code Rust dans Qiskit, tu dois reconstruire le code de l'extension à chaque fois que tu effectues une modification locale. En mode éditable, l'extension Rust n'est construite que lors de l'exécution de la commande d'installation, donc les modifications locales apportées au code Rust ne sont pas reflétées dans le package installé, sauf si tu reconstruis l'extension en réexécutant `build_rust` (avec ou sans `--release`, selon que tu veux construire en mode release ou debug).
    </Admonition>

## Installer Qiskit Runtime {#install-qiskit-runtime}

Suis ces étapes si tu veux installer Qiskit Runtime :

1. Clone le dépôt Qiskit Runtime.

```bash
git clone https://github.com/Qiskit/qiskit-ibm-runtime.git
```

2. Accède au répertoire `qiskit-ibm-runtime`.

```bash
cd qiskit-ibm-runtime
```

3. Installe `qiskit-runtime`. Nous te recommandons d'utiliser un [environnement virtuel](https://docs.python.org/3/library/venv.html) pour éviter de polluer ton installation Python globale.

  * **Installation standard** :

    ```bash
    pip install .
    ```

  * **Mode éditable** : Dans ce mode, tu n'as pas besoin de réinstaller Qiskit lorsqu'il y a des modifications de code dans le projet.

    ```bash
    pip install -e .
    ```

    En mode éditable, les extensions compilées sont construites en _mode debug_ sans optimisations.

4. (Optionnel) Si tu veux exécuter des tests ou des vérifications de linting, installe les dépendances de développement. Nous te recommandons d'utiliser un [environnement virtuel](https://docs.python.org/3/library/venv.html) pour éviter de polluer ton installation Python globale.

```bash
pip install -e ".[dev]"
```

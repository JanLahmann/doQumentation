## Débuter avec Estimator {#get-started-with-estimator}

{/*Verified the v2 examples 2/29/24 - updated 10/29/24*/}

### 1. Initialiser le compte {#1-initialize-the-account}

Comme Qiskit Runtime Estimator est un service géré, tu dois d'abord initialiser ton compte. Tu peux ensuite sélectionner le QPU que tu souhaites utiliser pour calculer la valeur d'espérance.

Suis les étapes du [sujet Installation et configuration](install-qiskit) si tu n'as pas encore de compte.

```python
from qiskit_ibm_runtime import QiskitRuntimeService

service = QiskitRuntimeService()
backend = service.least_busy(
    operational=True, simulator=False, min_num_qubits=127
)

print(backend.name)
```

```text
ibm_torino
```

### 2. Créer un Circuit et un observable {#2-create-a-circuit-and-an-observable}

Tu as besoin d'au moins un circuit et un observable comme entrées pour la primitive Estimator.

```python
from qiskit.circuit.library import qaoa_ansatz
from qiskit.quantum_info import SparsePauliOp

entanglement = [tuple(edge) for edge in backend.coupling_map.get_edges()]
observable = SparsePauliOp.from_sparse_list(
    [("ZZ", [i, j], 0.5) for i, j in entanglement],
    num_qubits=backend.num_qubits,
)
circuit = qaoa_ansatz(observable, reps=2)
# the circuit is parametrized, so we will define the parameter values for execution
param_values = [0.1, 0.2, 0.3, 0.4]

print(f">>> Observable: {observable.paulis}")
```

Le Circuit et l'observable doivent être transformés pour n'utiliser que les instructions prises en charge par le QPU (appelées circuits à *architecture d'ensemble d'instructions (ISA)*). Nous allons utiliser le transpileur pour ce faire.

```python
from qiskit.transpiler import generate_preset_pass_manager

pm = generate_preset_pass_manager(optimization_level=1, backend=backend)
isa_circuit = pm.run(circuit)
isa_observable = observable.apply_layout(isa_circuit.layout)
print(f">>> Circuit ops (ISA): {isa_circuit.count_ops()}")
```

```text
>>> Circuit ops (ISA): OrderedDict([('rz', 3826), ('sx', 1601), ('cz', 968)])
```

### 3. Initialiser Qiskit Runtime Estimator {#3-initialize-qiskit-runtime-estimator}

Lors de l'initialisation d'Estimator, utilise le paramètre `mode` pour spécifier le mode dans lequel tu souhaites l'exécuter. Les valeurs possibles sont des objets `batch`, `session` ou `backend` pour les modes d'exécution par lot, session et job, respectivement. Pour plus d'informations, consulte [Introduction aux modes d'exécution de Qiskit Runtime.](execution-modes)

```python
from qiskit_ibm_runtime import EstimatorV2 as Estimator

estimator = Estimator(mode=backend)
```

### 4. Invoquer Estimator et obtenir les résultats {#4-invoke-the-estimator-and-get-results}

Ensuite, invoque la méthode `run()` pour calculer les valeurs d'espérance pour les circuits d'entrée et les observables. Le Circuit, l'observable et les ensembles de valeurs de paramètres optionnels sont fournis sous forme de tuples *primitive unified bloc* (PUB).

```python
job = estimator.run([(isa_circuit, isa_observable, param_values)])
print(f">>> Job ID: {job.job_id()}")
print(f">>> Job Status: {job.status()}")
```

```text
>>> Job ID: d5k96c4jt3vs73ds5smg
```

```text

```

```python
result = job.result()
print(f">>> {result}")
print(f"  > Expectation value: {result[0].data.evs}")
print(f"  > Metadata: {result[0].metadata}")
```

## Débuter avec Sampler {#get-started-with-sampler}

### 1. Initialiser le compte {#1-initialize-the-account-sampler}

Comme Qiskit Runtime Sampler est un service géré, tu dois d'abord initialiser ton compte. Tu peux ensuite sélectionner le QPU que tu souhaites utiliser pour calculer la valeur d'espérance.

Suis les étapes du [sujet Installation et configuration](install-qiskit) si tu n'as pas encore de compte configuré.

```python
from qiskit_ibm_runtime import QiskitRuntimeService

service = QiskitRuntimeService()
backend = service.least_busy(
    operational=True, simulator=False, min_num_qubits=127
)
```

### 2. Créer un Circuit {#2-create-a-circuit}

Tu as besoin d'au moins un circuit comme entrée pour la primitive Sampler.

```python
import numpy as np
from qiskit.circuit.library import efficient_su2

circuit = efficient_su2(127, entanglement="linear")
circuit.measure_all()
# The circuit is parametrized, so we will define the parameter values for execution
param_values = np.random.rand(circuit.num_parameters)
```

Utilise le transpileur pour obtenir un circuit ISA.

```python
from qiskit.transpiler import generate_preset_pass_manager

pm = generate_preset_pass_manager(optimization_level=1, backend=backend)
isa_circuit = pm.run(circuit)
print(f">>> Circuit ops (ISA): {isa_circuit.count_ops()}")
```

```text
>>> Circuit ops (ISA): OrderedDict([('sx', 3089), ('rz', 3036), ('cz', 1092), ('measure', 127), ('barrier', 1)])
```

### 3. Initialiser le Qiskit Runtime Sampler {#3-initialize-the-qiskit-runtime-sampler}

Lors de l'initialisation du Sampler, utilise le paramètre `mode` pour spécifier le mode dans lequel tu souhaites l'exécuter. Les valeurs possibles sont des objets `batch`, `session` ou `backend` pour les modes d'exécution par lot, session et job, respectivement. Pour plus d'informations, consulte [Introduction aux modes d'exécution de Qiskit Runtime.](execution-modes) Note que les utilisateurs du plan Open ne peuvent pas soumettre de jobs de session.

```python
from qiskit_ibm_runtime import SamplerV2 as Sampler

sampler = Sampler(mode=backend)
```

### 4. Invoquer le Sampler et obtenir les résultats {#4-invoke-the-sampler-and-get-results}

Ensuite, invoque la méthode `run()` pour générer la sortie. Le Circuit et les ensembles de valeurs de paramètres optionnels sont fournis sous forme de tuples *primitive unified bloc* (PUB).

```python
job = sampler.run([(isa_circuit, param_values)])
print(f">>> Job ID: {job.job_id()}")
print(f">>> Job Status: {job.status()}")
```

```text
>>> Job ID: d5k96rsjt3vs73ds5tig
>>> Job Status: QUEUED
```

```python
result = job.result()

# Get results for the first (and only) PUB
pub_result = result[0]
print(
    f"First ten results for the 'meas' output register: {pub_result.data.meas.get_bitstrings()[:10]}"
)
```

## Débuter avec les primitives backend {#get-started-with-the-backend-primitives}

Contrairement aux primitives propres à un fournisseur, les primitives backend sont des implémentations génériques utilisables avec n'importe quel objet `backend`, à condition qu'il implémente l'interface [`Backend`](https://docs.quantum.ibm.com/api/qiskit/qiskit.providers.Backend).

- La primitive Sampler peut être exécutée avec n'importe quel fournisseur en utilisant [`qiskit.primitives.BackendSamplerV2`](https://docs.quantum.ibm.com/api/qiskit/qiskit.primitives.BackendSamplerV2).
- La primitive Estimator peut être exécutée avec n'importe quel fournisseur en utilisant [`qiskit.primitives.BackendEstimatorV2`](../api/qiskit/qiskit.primitives.BackendEstimatorV2).

Certains fournisseurs implémentent les primitives nativement. Consulte la [page de l'écosystème Qiskit](https://qiskit.github.io/ecosystem#provider) pour les détails.
### Exemple : BackendEstimator {#example-backendestimator}

```python
from qiskit.primitives import BackendEstimatorV2
from <some_qiskit_provider> import QiskitProvider

provider = QiskitProvider()
backend = provider.get_backend('backend_name')
estimator = BackendEstimatorV2(backend)
```

### Exemple : BackendSampler {#example-backendsampler}

```python
from qiskit.primitives import BackendSamplerV2
from <some_qiskit_provider> import QiskitProvider

provider = QiskitProvider()
backend = provider.get_backend('backend_name')
sampler = BackendSamplerV2(backend)
```
### Similitudes et différences entre les primitives backend et Runtime {#similarities-and-differences-between-backend-and-runtime-primitives}

- Les entrées et sorties de [`qiskit.primitives.BackendSamplerV2`](https://docs.quantum.ibm.com/api/qiskit/qiskit.primitives.BackendSamplerV2) et [`qiskit.primitives.BackendEstimatorV2`](../api/qiskit/qiskit.primitives.BackendEstimatorV2)
suivent le même format PUB que les primitives dans Qiskit Runtime. Consulte [Entrées et sorties des primitives](primitive-input-output) pour les détails.
Des différences peuvent néanmoins apparaître dans les champs des métadonnées retournées.

- La classe [`qiskit.primitives.BackendEstimatorV2`](https://docs.quantum.ibm.com/api/qiskit/qiskit.primitives.BackendEstimatorV2) n'offre pas d'implémentations d'atténuation des erreurs de mesure ou de porte intégrées, car les primitives backend sont conçues pour s'exécuter localement sur la machine de l'utilisateur.

- La classe [`qiskit.primitives.BackendSamplerV2`](https://docs.quantum.ibm.com/api/qiskit/qiskit.primitives.BackendSamplerV2) nécessite un backend qui prend en charge l'option `memory`.

- Les interfaces des primitives backend exposent des `Options` personnalisées pour [`SamplerV2`](https://docs.quantum.ibm.com/api/qiskit/qiskit.primitives.BackendSamplerV2) et [`EstimatorV2`](https://docs.quantum.ibm.com/api/qiskit/qiskit.primitives.BackendEstimatorV2) qui diffèrent des implémentations Runtime.

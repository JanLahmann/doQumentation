# Gérer les utilisateurs IBM Cloud {#manage-ibm-cloud-users}

Ce guide explique comment utiliser IBM Cloud&reg; pour activer les utilisateurs qui possèdent des comptes IBM Cloud et donne des instructions pour qu'ils accèdent à l'environnement.

Pour gérer les utilisateurs d'un fournisseur d'identité à la place, suis les instructions dans la rubrique [Gérer les fournisseurs d'identité](/guides/manage-appid).

## Étapes {#steps}

Suis ces étapes générales pour gérer les utilisateurs IBM Cloud :

1. Assure-toi que les utilisateurs que tu souhaites inviter ont des comptes IBM Cloud.
2. Va dans IAM et [invite des utilisateurs](https://cloud.ibm.com/iam/users/invite_users)
   <Admonition type="note">
   Les utilisateurs ne peuvent pas être gérés tant qu'ils n'ont pas accepté l'invitation et ne se sont pas connectés au moins une fois.
   </Admonition>
3. Facultatif : Modifie les affectations de projets et les groupes d'accès des utilisateurs depuis [IAM.](https://cloud.ibm.com/iam/users)

<span id="user-org"></span>
## Parcours utilisateur {#user-flow}

1. L'utilisateur accepte une invitation, puis se connecte via le [portail IBM Cloud](https://cloud.ibm.com/).
2. L'utilisateur crée une clé API en allant dans ([Gérer → Accès (IAM) → Clés API](https://cloud.ibm.com/iam/apikeys)).
3. Pour plus d'informations, les utilisateurs peuvent consulter [Installer Qiskit.](/guides/install-qiskit)

## Exemple de scénario {#example-scenario}

Cet exemple crée la configuration suivante :

* Il y a deux projets, `ml` et `finance`.
   * Le projet `ml` doit avoir accès aux instances de service `QR-ml` et `QR-common`.
   * Le projet `finance` doit avoir accès aux instances de service `QR-finance` et `QR-common`.
* Il y a trois utilisateurs :
   * Fatima doit avoir accès au projet `ml`.
   * Ravi doit avoir accès au projet `finance`.
   * Amyra doit avoir accès aux deux projets.
* La configuration utilise des groupes d'accès sans groupes de ressources.
* Les utilisateurs sont définis dans IBM Cloud et les affectations de projets y sont également effectuées.
* Les utilisateurs doivent pouvoir supprimer des jobs.

Les étapes pour mettre en œuvre cette configuration sont :

2. L'administrateur Cloud crée trois instances de service : `QR-ml`, `QR finance` et `QR-common`.
3. L'administrateur Cloud crée une règle personnalisée qui inclut l'action `quantum-computing.job.delete`.
1. L'administrateur Cloud crée deux groupes d'accès :
   * Le groupe d'accès `ml` peut accéder à `QR-ml` et `QR-common`.
   * Le groupe d'accès `finance` peut accéder à `QR-finance` et `QR-common`.
1. L'administrateur Cloud invite les utilisateurs cloud dans le projet approprié. Plus précisément, il les invite et les assigne à un groupe d'accès qui inclut le projet.
   * Fatima est ajoutée au groupe d'accès « ml ».
   * Ravi est ajouté au groupe d'accès « finance ».
   * Amyra est ajoutée aux groupes d'accès « ml » et « finance ».
1. Les utilisateurs peuvent se connecter via le portail IBM Cloud, créer des clés API et travailler avec les instances de service de leurs projets.

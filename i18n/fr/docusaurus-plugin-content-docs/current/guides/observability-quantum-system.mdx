# Événements de suivi d'activité pour le service IBM Quantum System {#activity-tracking-events-for-the-ibm-quantum-system-service}

Les services IBM Cloud&reg;, tels que le service IBM&reg; Quantum System, génèrent des événements de suivi d'activité.

Les événements de suivi d'activité signalent les activités qui modifient l'état d'un service dans IBM Cloud. Tu peux utiliser ces événements pour détecter les activités anormales et les actions critiques, et pour répondre aux exigences d'audit réglementaire.

Tu peux utiliser IBM Cloud Activity Tracker Event Routing, un service de plateforme, pour acheminer les événements d'audit de ton compte vers les destinations de ton choix en configurant des cibles et des routes qui définissent où les événements de suivi d'activité sont envoyés. Pour plus d'informations, voir [About IBM Cloud Activity Tracker Events Routing](https://cloud.ibm.com/docs/atracker?topic=atracker-about).

Tu peux utiliser IBM Cloud Logs pour visualiser et générer des alertes sur les événements générés dans ton compte et acheminés par IBM Cloud Activity Tracker Event Routing vers une instance IBM Cloud Logs.

## Affichage des événements de suivi d'activité pour le service IBM Quantum System {#viewing-activity-tracking-events-for-the-ibm-quantum-system-service}

Tu peux utiliser IBM Cloud Logs pour visualiser et générer des alertes sur les événements générés dans ton compte et acheminés par IBM Cloud Activity Tracker Event Routing vers une instance IBM Cloud Logs.

### Lancement d'IBM Cloud Logs depuis la page Observabilité {#launching-ibm-cloud-logs-from-the-observability-page}

Pour obtenir des informations sur le lancement de l'interface utilisateur IBM Cloud Logs, consulte [Launching the UI](https://cloud.ibm.com/docs/cloud-logs?topic=cloud-logs-instance-launch#instance-launch-cloud-ui) dans la documentation IBM Cloud Logs.

| Action | Déclenchée par |
| -- | -- |
| `quantum-computing.direct-access-backend-configuration.read` | Obtenir la configuration du Backend (`GET /v1/backends/{backend-name}/configuration`) |
| `quantum-computing.direct-access-backend-properties.read` | Obtenir les propriétés du Backend (`GET /v1/backends/{backend-name}/properties`) |
| `quantum-computing.direct-access-backend.list` | Obtenir la liste des Backends (`GET /v1/backends`) |
| `quantum-computing.direct-access-backend.read` | Obtenir les détails du Backend (`GET /v1/backends/{backend-name}`) |
| `quantum-computing.direct-access-job.cancel` | Annuler un job (`POST /v1/jobs/{job-id}/cancel`) |
| `quantum-computing.direct-access-job.create` | Exécuter un job (`POST /v1/jobs`) |
| `quantum-computing.direct-access-job.delete` | Supprimer un job (`DELETE /v1/jobs/{job-id}`) |
| `quantum-computing.direct-access-job.list` | Obtenir les jobs (`GET /v1/jobs`) |
| `quantum-computing.direct-access-job.read` | Obtenir un job par ID (`GET /v1/jobs/{job-id}`) |
| `quantum-computing.direct-access-lane-configuration.list` | Obtenir la configuration des lanes du backend (`GET /v1/backends/{backend-name}/lanes`) |

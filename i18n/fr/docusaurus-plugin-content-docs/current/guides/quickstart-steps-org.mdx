# Configurer IBM Quantum Platform pour une organisation {#configure-ibm-quantum-platform-for-an-organization}

Suis ces étapes pour configurer IBM Quantum Platform.

<span id="iam-org"></span>
## 1. Configurer les paramètres IAM {#1-configure-iam-settings}

Commence par configurer les paramètres suivants dans le compte IAM de l'administrateur. Pour consulter et modifier ces paramètres, accède à [Manage → IAM → Settings](https://cloud.ibm.com/iam/settings).

* **User list visibility** détermine si les utilisateurs peuvent se voir mutuellement, indépendamment de l'affectation au projet. Le paramètre `enabled` restreint la visibilité des utilisateurs. Autrement dit, les utilisateurs de ton compte ne peuvent pas se voir les uns les autres, même s'ils ont accès aux mêmes ressources. Choisis la valeur adaptée à ton environnement. Consulte [Controlling user visibility](https://cloud.ibm.com/docs/account?topic=account-iam-user-setting) pour plus d'informations.
* **API key creation** contrôle si les utilisateurs peuvent créer des clés API. Dans IBM Quantum Platform, il est courant d'utiliser des clés API. Si des clés API sont utilisées, choisis `disabled`. Tu peux également accorder des permissions spécifiques à chaque utilisateur.

<span id="crt-rsc-grp-org"></span>
## 2. (Facultatif) Créer des groupes de ressources {#2-optional-create-resource-groups}

Ignore cette étape si tu utilises des groupes d'accès qui référencent directement des instances du service IBM Quantum.

Si tu as choisi d'utiliser des groupes de ressources, accède à [Manage → Account → Resource groups (in Account resources)](https://cloud.ibm.com/account/resource-groups) et clique sur **Create**.

<span id="create-instance-org"></span>
## 3. Créer des instances du service IBM Quantum {#3-create-ibm-quantum-service-instances}

Si tu as déjà créé des instances du service IBM Quantum, ignore cette étape.

* Si tu utilises des groupes de ressources, assure-toi de créer les instances de service dans le groupe de ressources approprié.
* Le nom de l'instance de service, par exemple `QR-ml`, est nécessaire pour les références aux groupes d'accès.

Suis les étapes décrites dans [Configurer pour utiliser IBM Cloud](/guides/cloud-setup#cloud) pour obtenir des instructions.

<span id="create-group-org"></span>
## 4. Créer des groupes d'accès pour les projets {#4-create-access-groups-for-projects}

Commence par [créer un rôle personnalisé](/guides/custom-roles#custom) qui permet aux utilisateurs d'effectuer des actions pour travailler avec les instances du service IBM Quantum.
Ensuite, [crée un groupe d'accès](/guides/access-groups) pour chaque projet et accorde à ce groupe l'ensemble minimal de permissions nécessaires pour travailler avec les ressources du projet.

<span id="setup-idp-org"></span>
## 5. Configurer ton fournisseur d'identité et assigner des utilisateurs {#5-set-up-your-id-provider-and-assign-users}

Suis les étapes décrites dans la rubrique appropriée, selon le fournisseur d'identité que tu as choisi d'utiliser :

* [Utiliser IBM Cloud&reg; comme fournisseur d'identité pour les utilisateurs disposant d'un compte IBM Cloud](/guides/manage-cloud-users)
* [Utiliser IBM Cloud comme fournisseur d'identité pour les utilisateurs ne disposant pas d'un compte IBM Cloud](/guides/manage-appid)
* [Utiliser un fournisseur d'identité autre qu'IBM Cloud pour les utilisateurs ne disposant pas d'un compte IBM Cloud](/guides/manage-appid)
* [Inviter des utilisateurs et gérer leurs accès](/guides/invite-and-manage-users)

<span id="firewall"></span>
## 6. Configurer ton pare-feu {#6-configure-your-firewall}

Pour configurer ton pare-feu afin d'autoriser l'accès aux points de terminaison de l'API IBM Quantum, ajoute les URLs suivantes à ta liste d'autorisation :

- Services globaux IBM Cloud :
    ```text
    cloud.ibm.com
    iam.cloud.ibm.com
    api.global-search-tagging.cloud.ibm.com
    globalcatalog.cloud.ibm.com
    resource-controller.cloud.ibm.com
    ```
- Services IBM Quantum Qiskit Runtime - région us-east :
    ```text
    quantum.cloud.ibm.com
    s3.us-east.cloud-object-storage.appdomain.cloud
    ```
- Services IBM Quantum Qiskit Runtime - région eu-de :
    ```text
    eu-de.quantum.cloud.ibm.com
    s3.direct.eu-de.cloud-object-storage.appdomain.cloud
    ```
- Service de prévisualisation IBM Quantum Qiskit Functions :
    ```text
    qiskit-serverless.quantum.ibm.com
    ```

# Événements de suivi d'activité pour Qiskit Runtime {#activity-tracking-events-for-qiskit-runtime}

Les services IBM Cloud&reg;, tels que Qiskit Runtime, génèrent des événements de suivi d'activité.

Les événements de suivi d'activité rendent compte des activités qui modifient l'état d'un service dans IBM Cloud. Tu peux utiliser ces événements pour enquêter sur des activités anormales et des actions critiques, et pour répondre aux exigences des audits réglementaires.

Tu peux utiliser IBM Cloud Activity Tracker Event Routing, un service de plateforme, pour acheminer les événements d'audit de ton compte vers les destinations de ton choix en configurant des cibles et des routes qui définissent où les événements de suivi d'activité sont envoyés. Pour plus d'informations, consulte [À propos du routage des événements IBM Cloud Activity Tracker.](https://cloud.ibm.com/docs/atracker?topic=atracker-about)

Tu peux utiliser IBM Cloud Logs pour visualiser et recevoir des alertes sur les événements générés dans ton compte et acheminés par IBM Cloud Activity Tracker Event Routing vers une instance IBM Cloud Logs.

## Emplacements où les événements de suivi d'activité sont générés {#locations-where-activity-tracking-events-are-generated}

Qiskit Runtime envoie des événements de suivi d'activité dans les régions suivantes :

- Washington (« us-east »)
- Francfort (« eu-de »)

## Consultation des événements de suivi d'activité pour Qiskit Runtime {#viewing-activity-tracking-events-for-qiskit-runtime}

Tu peux utiliser IBM Cloud Logs pour visualiser et recevoir des alertes sur les événements générés dans ton compte et acheminés par IBM Cloud Activity Tracker Event Routing vers une instance IBM Cloud Logs.

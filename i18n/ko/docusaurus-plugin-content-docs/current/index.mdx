## IBM Quantum의 오픈소스 콘텐츠 {#ibm-quantums-open-source-content}

IBM은 양자 컴퓨팅 학습 자료를 풍부하게 제공합니다 — 모두 CC BY-SA 4.0 라이선스의 오픈소스입니다:

- **[Learning](https://quantum.cloud.ibm.com/learning)** — 양자 기초부터 고급 주제까지 체계적인 강좌
- **[Tutorials](https://quantum.cloud.ibm.com/docs/en/tutorials)** — transpilation, error mitigation 등에 관한 40개 이상의 tutorials
- **[Documentation](https://quantum.cloud.ibm.com/docs/en/guides)** — Qiskit 가이드 및 API 레퍼런스
- **[Source repo](https://github.com/Qiskit/documentation)** — GitHub의 모든 콘텐츠

그들의 [Quantum Platform](https://quantum.cloud.ibm.com)은 항상 최신 상태이며 잘 설계되어 있습니다 — 읽고, 배우고, 참고하기에 가장 좋은 곳입니다.

## 이 프로젝트가 추가하는 것 {#what-this-project-adds}

IBM의 [Qiskit 문서](https://github.com/Qiskit/documentation)는 오픈소스(CC BY-SA 4.0)이지만, 그들의 웹 애플리케이션은 그렇지 않습니다. doQumentation은 라이브 코드 실행, 자동 자격 증명 주입, 시뮬레이터 모드를 갖춘 오픈소스 프론트엔드를 추가합니다.

어디서든 배포 가능합니다 — [GitHub Pages](https://doqumentation.org)부터 [Docker](https://github.com/JanLahmann/doQumentation/pkgs/container/doqumentation), [RasQberry](https://rasqberry.org/)까지. [모든 기능](/features)을 확인해 보세요.

## 더 많은 것을 원할 준비가 되면 {#when-youre-ready-for-more}

doQumentation은 아무것도 설치하지 않고 **Qiskit으로 첫 발걸음**을 내딛도록 도와줍니다 — 읽고, 코드를 실행하고, 탐색하세요. 더 나아갈 준비가 되면 IBM의 공식 플랫폼으로 이동하세요:

- **실제 하드웨어에서 실행** — 무료 [IBM Cloud 계정](https://quantum.cloud.ibm.com/registration)과 Open Plan 인스턴스를 만들어 실제 양자 컴퓨터에서 회로를 실행하세요.
- **최신 콘텐츠** — IBM의 [Qiskit 학습 플랫폼](https://quantum.cloud.ibm.com/learning)에는 최신 코스, 튜토리얼 및 추가 사항이 있습니다(일부는 아직 doQumentation에 없을 수 있습니다).
- **로컬에서 개발** — 본격적인 개발 작업을 위해 자신의 노트북에 [Qiskit을 설치](https://docs.quantum.ibm.com/guides/install-qiskit)하세요.

*[IBM Quantum의 오픈소스 문서](https://github.com/Qiskit/documentation)를 기반으로 합니다. 이 콘텐츠를 CC BY-SA 4.0 라이선스로 자유롭게 사용할 수 있게 해 주신 IBM Quantum 팀에 감사드립니다.*

## 코드 실행 {#code-execution}

코드 블록에서 **Run**을 클릭하세요. 첫 번째 클릭은 [Binder](https://mybinder.org) 또는 [IBM Code Engine](/jupyter-settings#code-engine)을 통해 Jupyter 커널을 시작합니다. 그 이후 실행은 즉각적입니다.

- **시뮬레이터 모드 (계정 불필요)** — [시뮬레이터 모드](/jupyter-settings#simulator-mode)를 활성화하면 AerSimulator 또는 노이즈 모델 FakeBackends로 모든 노트북을 실행할 수 있습니다. 설정이 전혀 필요 없습니다.

- **IBM Quantum 하드웨어** — [설정](/jupyter-settings#ibm-quantum)에 API 토큰과 CRN을 한 번만 입력하면 — 실행할 때마다 자격 증명이 자동으로 주입됩니다. 계정이 없다면 [무료 계정을 만드세요](https://quantum.cloud.ibm.com/registration).

<details>
<summary>사용 가능한 실행 백엔드</summary>

모든 tutorial에는 실행 가능한 코드 블록이 있습니다. **Run**을 클릭해서 네 가지 백엔드 중 하나로 실행하세요:

1. **Binder** (GitHub Pages 기본값) — [mybinder.org](https://mybinder.org)를 통한 무료 원격 Jupyter 커널
2. **IBM Code Engine** — IBM Cloud 계정의 서버리스 Jupyter. 콜드 스타트가 분 단위가 아닌 초 단위. [설정](/jupyter-settings#code-engine)
3. **로컬 Jupyter (Docker / RasQberry)** — Qiskit이 사전 설치된 로컬 Jupyter 서버에 연결
4. **커스텀 서버** — [설정](/jupyter-settings#advanced)에서 임의의 Jupyter 엔드포인트 지정

여러 백엔드가 사용 가능한 경우, [설정](/jupyter-settings#backend-selection)에서 사용할 백엔드를 선택하세요.

</details>

<details>
<summary>배포 옵션</summary>

doQumentation은 다음과 같이 제공됩니다:

- **[GitHub Pages](https://doqumentation.org)** — [Binder](https://mybinder.org) 또는 [IBM Code Engine](/jupyter-settings#code-engine)을 이용한 원격 코드 실행이 가능한 정적 사이트
- **[Docker](https://github.com/JanLahmann/doQumentation/pkgs/container/doqumentation)** — 로컬 풀 스택, 오프라인 사용 가능
- **[RasQberry](https://rasqberry.org/)** — 로컬 Jupyter 커널을 갖춘 Raspberry Pi 자체 호스팅

</details>

<details>
<summary>Podman / Docker로 로컬 실행하기</summary>

[Podman Desktop](https://podman-desktop.io/) 또는 [Docker Desktop](https://www.docker.com/products/docker-desktop/) (Mac / Windows / Linux)을 설치한 후 실행하세요:

```bash
# Full stack: site + Jupyter + Qiskit (~3 GB)
podman run -p 8080:80 -p 8888:8888 ghcr.io/janlahmann/doqumentation:jupyter
```

[http://localhost:8080](http://localhost:8080)을 열면 — 코드 실행이 로컬에서 작동하고, Binder 대기 시간이 없습니다.

로컬 코드 실행 없이 가벼운 버전을 원한다면 (~60 MB):

```bash
# Static site only — code execution still works via Binder
podman run -p 8080:80 ghcr.io/janlahmann/doqumentation:latest
```

Docker를 사용하시나요? `podman`을 `docker`로 바꾸기만 하면 됩니다 — 명령어는 동일합니다. 이미지는 멀티 아치(`linux/amd64` + `linux/arm64`)이므로 Apple Silicon Mac에서도 네이티브로 작동합니다.

</details>

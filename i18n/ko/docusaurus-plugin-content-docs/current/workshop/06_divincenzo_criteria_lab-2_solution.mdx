## 소개 {#introduction}

물리학자 David DiVincenzo는 양자 컴퓨터의 물리적 구현을 위한 다섯 가지 핵심 요건과, 양자 통신을 위한 두 가지 추가 기준을 제시했습니다. 이 노트북에서는 **Qiskit 실습 데모를 통해 각 DiVincenzo 기준을 직접 경험**해 봅니다. 이론을 깊이 파고들기보다는, 각 섹션에서 하나의 기준을 간략히 설명한 후 Qiskit 2를 사용하는 코드 실습을 제공합니다. 시뮬레이터와 실제 IBM Quantum 장치에서 Circuit을 실행하며 **각 원리를 직접 탐구**해 볼 수 있습니다.

**DiVincenzo의 양자 계산을 위한 다섯 가지 기준**:

1. **잘 특성화된 Qubit을 갖춘 확장 가능한 물리 시스템.**
2. **Qubit을 초기화하는 능력** — 단순한 기준 상태(예: |00…0〉)로 초기화.
3. **긴 결어긋남 시간** — Qubit 결맞음 시간이 Gate 동작 시간보다 훨씬 길어야 함.
4. **보편적인 양자 Gate 집합** — 임의의 유니타리 연산을 수행 가능해야 함.
5. **Qubit별 측정 능력** — 각 Qubit의 상태를 읽어낼 수 있어야 함.

*(DiVincenzo는 양자 통신을 위한 두 가지 기준도 제시했습니다: 정지형 Qubit과 "비행" Qubit 간의 상호 변환 능력, 그리고 비행 Qubit을 위치 간에 충실하게 전송하는 능력. 이 내용은 이 노트북 끝에 있는 추천 활동에 포함되어 있습니다.)*

다음 각 섹션은 하나의 기준에 대응합니다. Qiskit을 사용해 코드와 **인터랙티브 실험**으로 개념을 설명합니다. 예를 들어, Qubit 수와 Circuit 깊이를 늘렸을 때 결과에 미치는 영향(기준 1), Qubit 상태를 리셋하고 준비하는 방법(기준 2), 시뮬레이터와 실제 장치에서 Qubit을 측정하는 방법(기준 4), Qiskit이 보편 Gate를 구성하는 방식(기준 3), 그리고 유한한 결맞음(T₁, T₂)이 계산에 미치는 영향(기준 5)을 살펴볼 것입니다. 이 과정을 통해 각 DiVincenzo 기준이 실제로 무엇을 의미하는지, 그리고 Qiskit이 어떻게 실험을 가능하게 하는지에 대한 더 깊은 직관을 얻을 수 있습니다.

## 1. 기준 1 – 확장 가능하고 잘 특성화된 Qubit {#1-criterion-1-scalable-well-characterized-qubits}

**기준 1:** *"잘 특성화된 Qubit을 갖춘 확장 가능한 물리 시스템."* 이는 Qubit 수를 **늘려도** 안정적으로 제어할 수 있는 양자 하드웨어 플랫폼이 필요하다는 뜻입니다. 각 Qubit의 특성(에너지 준위, 오류율, 연결성 등)이 잘 파악되어 있어야 합니다. 즉, 시스템이 망가지지 않으면서 더 큰 Circuit을 구성할 수 있어야 합니다. 실제로 Qubit 수나 Circuit 깊이가 증가하면 오류와 결어긋남이 누적되므로, *확장성*을 입증하는 것은 크기 증가가 성능에 미치는 영향을 이해하는 것도 포함합니다.

**데모 목표:** Qiskit을 사용하여 Circuit 규모를 늘렸을 때(Qubit 수 또는 Gate 깊이) 출력 충실도에 미치는 영향을 보여줍니다. 이상적 시나리오와 노이즈가 있는 시나리오를 시뮬레이션하여, 더 큰 시스템이나 더 깊은 Circuit이 어떻게 결어긋남과 오류에 취약해지는지 살펴봅니다.

먼저, 간단한 확장성 테스트로 3개의 Qubit에서 작은 얽힘 상태(GHZ 상태)를 생성하고, 그 다음 5개의 Qubit에서 더 큰 상태를 생성해 보겠습니다. *n*개의 Qubit으로 이루어진 GHZ 상태는 $\frac{1}{\sqrt{2}}(|0...0\rangle + |1...1\rangle)$입니다. 이상적인 시뮬레이션에서 n-Qubit GHZ를 측정하면 두 가지 결과(모두 0이거나 모두 1)만 동일한 확률로 나타납니다. n이나 Circuit 깊이를 늘리면서 **이상적 출력**과 **노이즈가 있는 출력**을 비교해 볼 것입니다.

```python
from qiskit import QuantumCircuit
from qiskit_aer import AerSimulator
from qiskit.visualization import plot_histogram
from qiskit.transpiler.preset_passmanagers import generate_preset_pass_manager
from qiskit_ibm_runtime import SamplerV2 as Sampler

# 3-qubit GHZ circuit
qc3 = QuantumCircuit(3, 3)
qc3.h(0)
qc3.cx(0, 1)
qc3.cx(1, 2)
qc3.measure([0, 1, 2], [0, 1, 2])

# 5-qubit GHZ circuit (scaling up the number of qubits)
qc5 = QuantumCircuit(5, 5)
qc5.h(0)
qc5.cx(0, range(1, 5))    # entangle qubit 0 with all others
qc5.measure(range(5), range(5))

# Transpile for a simulator backend
sim_backend = AerSimulator()
pm = generate_preset_pass_manager(backend=sim_backend, optimization_level=1)
isa_qc3 = pm.run(qc3)
isa_qc5 = pm.run(qc5)

# Run ideal simulations (no noise)
sampler = Sampler(mode=sim_backend)

job3 = sampler.run([isa_qc3], shots=1024)
result3 = job3.result()
counts3 = result3[0].data.c.get_counts()

job5 = sampler.run([isa_qc5], shots=1024)
result5 = job5.result()
counts5 = result5[0].data.c.get_counts()

print("3-qubit GHZ counts (ideal):", counts3)
plot_histogram(counts3, legend=['3-qubit ideal'], figsize=(6,4))
```

```python
print("5-qubit GHZ counts (ideal):", counts5)
plot_histogram(counts5, legend=['5-qubit ideal'], figsize=(6,4))
```

**예상 결과(이상적인 경우):** 3-Qubit GHZ는 이상적으로 카운트에서 `000`이 약 50%, `111`이 약 50%로 나타납니다. 5-Qubit GHZ는 `00000`이 약 50%, `11111`이 약 50%로 나타납니다. 상태가 이상적으로 완전히 결맞음 상태이고 얽혀 있기 때문에 다른 비트 문자열은 나타나지 않습니다. 각 Circuit의 히스토그램에서 모두 0인 결과와 모두 1인 결과에 해당하는 두 개의 높은 막대가 보여야 합니다.

다음으로, **노이즈 환경**에서 어떤 일이 벌어지는지 살펴보겠습니다. Qiskit Aer의 노이즈 모델 기능을 사용하여 실제 장치의 오류를 모방해 보겠습니다. 예를 들어, IBM Backend의 속성을 가져와서 Gate 오류, 유한 Gate 시간, Qubit 이완(T₁), 위상 이탈(T₂), 판독 오류를 포함하는 노이즈 모델을 만들 수 있습니다. 여기서는 IBM Quantum Brisbane 장치를 나타내는 **가짜 Backend**를 사용하여 노이즈 모델을 생성하고, GHZ Circuit을 다시 실행해 보겠습니다.

## 2. 기준 2 – Qubit 초기화 {#2-criterion-2-qubit-initialization}

**기준 2:** *"Qubit의 상태를 |000…〉과 같은 단순한 기준 상태로 초기화하는 능력."* 모든 Qubit은 알려진 기준 상태(일반적으로 각 Qubit의 기저 상태 |0〉)에서 안정적으로 시작해야 합니다. 알고리즘이 깨끗한 상태에서 시작하려면 초기화가 필수적입니다. 실제로 IBM 양자 장치에서는 각 Circuit 실행 시작 시 각 Qubit이 자동으로 |0〉으로 리셋됩니다. Qiskit은 계산 중에 Qubit을 리셋하거나 사용자 정의 상태를 준비하는 명령어도 제공합니다.

**데모 목표:** Qiskit에서 Qubit을 초기화하는 방법을 처음과 회로 중간 모두에서 보여줍니다. `reset` 명령어와 상태 준비 방법을 사용하는 방법을 시연합니다.
### 실습 2: 특정 상태 준비하기 {#exercise-2-prepare-a-specific-state}
아래 코드 블록에서 `QuantumCircuit`을 완성하여 상태 $|10\rangle$을 준비해 보세요. 이는 Qubit 0이 $|0\rangle$ 상태에 있고 Qubit 1이 $|1\rangle$ 상태에 있어야 함을 의미합니다. 이를 달성하기 위해 적절한 Gate와 명령어를 사용하세요.

```python
from qiskit import QuantumCircuit
from qiskit_aer import AerSimulator

# Create a circuit to initialize qubits to |10> and verify by measurement
qc_init = QuantumCircuit(2, 2)

# --- YOUR CODE HERE ---

# 1. Set qubit 1 to the |1> state
qc_init.x(1)

# 2. Explicitly reset qubit 0 to the |0> state
qc_init.reset(0)

# --- END YOUR CODE ---

qc_init.measure([0, 1], [0, 1])
qc_init.draw('mpl')
```

```python
# Run the circuit and check the outcome
sim_backend = AerSimulator()
pm = generate_preset_pass_manager(backend=sim_backend, optimization_level=1)
isa_qc_init = pm.run(qc_init)

sampler = Sampler(mode=sim_backend)

job = sampler.run([isa_qc_init], shots=1024)
result = job.result()
counts = result[0].data.c.get_counts()

print("Outcome of |10> state measured in Z-basis:", counts)
plot_histogram(counts)
```

시뮬레이션 결과, `10`(qubit1=1, qubit0=0에 해당하는 이진수)이 100% 확률로 나타나야 합니다. 이는 Qubit 1이 성공적으로 |1〉로, Qubit 0이 |0〉로 준비되었음을 의미합니다.

이제 더 일반적인 상태 준비를 위해, Qiskit은 `initialize` 메서드를 사용하여 임의의 상태로 초기화할 수 있습니다. 예를 들어, Qubit을 중첩 상태인 $|+\rangle = (|0\rangle+|1\rangle)/\sqrt{2}$로 준비하고, 두 Qubit을 벨 상태 $(|00\rangle+|11\rangle)/\sqrt{2}$로 준비해 보겠습니다:

```python
import numpy as np

# Initialize a single qubit in |+> state and measure in Z-basis
qc_plus = QuantumCircuit(1, 1)
state_plus = [1/np.sqrt(2), 1/np.sqrt(2)]   # amplitude for |0> and |1>
qc_plus.initialize(state_plus, 0)
qc_plus.measure(0, 0)

# Initialize two qubits in a Bell state manually
qc_bell = QuantumCircuit(2, 2)
bell_state = [1/np.sqrt(2), 0, 0, 1/np.sqrt(2)]  # amplitudes for |00>,|01>,|10>,|11>
qc_bell.initialize(bell_state, [0, 1])
qc_bell.measure([0, 1], [0, 1])

# Transpile and run the initialization circuits
isa_qc_plus = pm.run(qc_plus)
job_plus = sampler.run([isa_qc_plus], shots=1024)
result_plus = job_plus.result()
counts_plus = result_plus[0].data.c.get_counts()

print("Outcome of |+> state measured in Z-basis:", counts_plus)

isa_qc_bell = pm.run(qc_bell)
job_bell = sampler.run([isa_qc_bell], shots=1024)
result_bell = job_bell.result()
counts_bell = result_bell[0].data.c.get_counts()

print("Outcome of Bell state measured in Z-basis:", counts_bell)
```

**예상 결과:** 단일 Qubit |+〉 상태를 측정하면 `0`과 `1`이 각각 약 50% 확률로 나타납니다. 벨 상태 측정 결과는 `00`이 약 50%, `11`이 약 50%로 나타납니다. 이 결과를 확인하면 해당 상태로의 초기화가 성공적으로 이루어졌음을 확인할 수 있습니다.

**회로 중간 초기화:** Qiskit의 `reset`은 회로 중간에 Qubit을 즉석에서 |0〉으로 재초기화하는 데 사용할 수 있습니다. 예를 들어, 오류 수정 코드나 반복 알고리즘에서는 Qubit을 측정한 후 재사용을 위해 리셋하는 경우가 많습니다. `reset` 연산은 결정론적입니다. 기존 상태를 제거하고 Qubit을 기저 상태로 냉각시킵니다.

**장치 예시:** **ibmq_brisbane**(127 Qubit) 또는 IBM 장치에서는 작업이 실행될 때 모든 Qubit이 기본적으로 |0〉에서 시작합니다. 다른 시작 상태가 필요한 경우 처음에 Gate를 적용하면 됩니다(|1〉을 얻기 위해 X Gate를 적용한 것처럼). 지속적인 재초기화(양자 오류 수정을 위한)는 빠르게 수행하기 어렵기 때문에 활발한 연구 분야입니다. 다행히 기본 사용을 위해 |0…0〉에서 새로 시작하는 기능은 사용 가능하며, 다른 원하는 시작 상태를 달성하는 방법도 시연했습니다.
## 3. 기준 3 – 긴 결맞음 시간 (결어긋남 vs. Gate 동작 시간) {#3-criterion-3--long-coherence-time-decoherence-vs-gate-time}

**기준 3:** *"Gate 동작 시간보다 훨씬 긴 적절한 결어긋남 시간."* 이 기준은 필요한 연산을 수행하기에 충분할 만큼 오랫동안 Qubit이 양자 상태를 유지해야 한다는 요건을 다룹니다. 각 Qubit에는 **T₁ 시간**(에너지 이완 시간, |1〉이 |0〉으로 붕괴되는 속도)과 **T₂ 시간**(위상 결어긋남 시간, 상대적인 위상 결맞음이 손실되는 속도)이 있습니다. 양자 컴퓨터가 제대로 작동하려면 이 시간 척도가 Gate 동작 시간을 훨씬 초과해야 합니다.

**데모 목표:** 실행 길이가 증가함에 따라 결어긋남이 Circuit 결과에 어떤 영향을 미치는지 보여줌으로써 Qiskit에서 Qubit 결맞음을 조사합니다. 알려진 T1/T2 시간을 가진 가짜 Backend를 사용하여 이 효과를 시뮬레이션합니다.

**유한 결맞음의 영향을 시연하기** 위해 T1 감쇠 실험을 시뮬레이션합니다. Qubit을 |1〉 상태로 준비하고, `delay` 명령을 사용하여 일정 시간 기다린 후 측정합니다. 지연 시간이 길어질수록 |1〉을 측정할 확률이 감소할 것으로 예상됩니다.

```python
# This part is done for you. We are creating a list of circuits,
# each with a different delay time.

time_delays_ns = [0, 50000, 100000, 150000, 200000, 250000, 300000]  # delay durations in ns

decay_expts = []
for delay in time_delays_ns:
    qc = QuantumCircuit(1, 1)
    qc.x(0)  # initialize qubit to |1>
    if delay > 0:
        qc.delay(delay, 0, unit='ns')  # wait 'delay' nanoseconds
    qc.measure(0, 0)
    decay_expts.append(qc)

decay_expts[1].draw('mpl') # Visualize one of the circuits
```

### 연습 3: T1 감쇠 실험 시뮬레이션 {#exercise-3-simulate-a-t1-decay-experiment}

이제 T1 시간이 약 50~100 µs인 `FakeVigo`를 기반으로 한 노이즈 시뮬레이터를 사용하여 이 Circuit들을 실행합니다. 시뮬레이터는 `delay` 명령 실행 중에 T1/T2 오류를 자동으로 적용합니다. 이 Backend에 맞게 Circuit들을 Transpile하고 실행하세요.

```python

from qiskit_aer import AerSimulator
```

## 4. 기준 4 – 보편적인 양자 Gate 집합 {#4-criterion-4--universal-set-of-quantum-gates}

**기준 4:** *"'보편적인' 양자 Gate 집합."* 이는 하드웨어가 유한한 기본 Gate 집합을 조합하여 *임의의* 양자 계산을 수행할 수 있어야 한다는 것을 의미합니다. 고전 컴퓨팅에서 NAND가 보편적인 것처럼, 양자에서도 보편적인 Gate 집합에는 여러 선택지가 있습니다(예: \{H, T, CNOT\} 또는 특정 기계의 기본 Gate). IBM 장치는 예를 들어 임의의 단일 Qubit 회전과 특정 Qubit 간의 CNOT 같은 기본 연산 집합을 가지며, 이들을 함께 사용하면 보편적입니다. Qiskit의 역할은 종종 고수준 Gate를 이러한 기저 Gate로 **컴파일**하는 것입니다.

**데모 목표:** Gate 보편성을 설명하기 위해 Qiskit이 Gate를 어떻게 분해하는지 보여줍니다. 비기본 Gate(예: 3-Qubit Toffoli Gate, CCX)를 가져와 장치의 기저 Gate로 어떻게 분해되는지 살펴봅니다. 이를 통해 제공된 Gate 집합이 실제로 *보편적*임을 입증합니다 — 더 복잡한 연산을 만들어낼 수 있습니다.

먼저, 일반적인 IBM Backend의 기저 Gate가 무엇인지 살펴봅니다. 장치(또는 가짜 버전)의 구성을 조회합니다. 예를 들어, ibmq_brisbane의 기저 Gate:
`P(qubit=1)` 확률이 지연 시간이 증가함에 따라 감소하여 T1 이완의 특성인 지수 감쇠 곡선을 따르는 것을 확인할 수 있습니다. 이는 유한한 결맞음 시간이 Circuit이 너무 오래 실행될 경우 계산 오류로 이어짐을 직접 보여줍니다.

**알고리즘에 미치는 영향:** 많은 순차적 Gate가 포함된 더 긴 알고리즘을 시도하면 총 실행 시간이 T2에 근접하거나 초과하여 끝나기 전에 상태의 결맞음이 손실될 수 있습니다. 이것이 결맞음 시간을 개선하고 Gate를 더 빠르게 만드는 것이 양자 하드웨어 연구에서 가장 중요한 목표 중 두 가지인 이유입니다.

```python
from qiskit_ibm_runtime.fake_provider import FakeBrisbane
fake_brisbane = FakeBrisbane()
print("Basis gates for ibmq_brisbane:", fake_brisbane.configuration().basis_gates)
```

이 출력은 `['id', 'rz', 'sx', 'x', 'ecr']`과 같이 나타날 수 있습니다. 이것들이 하드웨어가 기본적으로 지원하는 기본 연산입니다(항등/무연산, RZ 회전, sqrt(X) Gate, X Gate, 제어-X). 다른 Gate는 모두 이것들로 구성되어야 합니다. 이 집합은 양자 컴퓨팅에 보편적인 것으로 알려져 있습니다(기본적으로 단일 Qubit 회전과 얽힘 2-Qubit Gate가 보편적인 집합을 형성합니다).

이제 **Toffoli (CCX) Gate**를 테스트 케이스로 사용합니다. CCX는 두 제어 Qubit이 모두 1일 때만 대상 Qubit을 뒤집습니다. IBM 하드웨어의 기본 Gate가 아닙니다. Qiskit은 `ccx` 명령을 제공하지만, 내부적으로 이를 분해합니다.

### 연습 4: Toffoli Gate 분해 {#exercise-4-decompose-a-toffoli-gate}

아래 코드를 완성하여 Toffoli (CCX) Gate가 있는 Circuit을 구성하고, Qiskit을 사용하여 `FakeBrisbane` Backend의 기본 기저 Gate로 분해하세요.

```python
from qiskit import QuantumCircuit
from qiskit_ibm_runtime.fake_provider import FakeBrisbane

# The fake_brisbane backend from the previous cell is reused here.

# --- YOUR CODE HERE ---

# 1. Create a circuit that can accommodate a Toffoli gate
qc_toffoli = QuantumCircuit(3)

# Apply a CCX gate with controls on qubits 0, 1 and target on qubit 2
qc_toffoli.ccx(0, 1, 2)

# 2. Transpile the circuit to the fake Brisbane backend
pm = generate_preset_pass_manager(backend=FakeBrisbane(), optimization_level=1)

isa_qc_toffoli = pm.run(qc_toffoli)

# --- END YOUR CODE ---

print("Toffoli circuit before decomposition:")
print(qc_toffoli)

print("\nToffoli circuit after transpiling to Brisbane basis:")
# The .draw() method will now show the decomposed circuit
print(isa_qc_toffoli.draw(fold=120))
```

Transpile된 출력에서 CCX가 `rz`, `sx`, `ecr`과 같은 더 기본적인 Gate의 시퀀스로 대체된 것을 볼 수 있습니다. 이는 기본 Gate만으로 Toffoli를 표현하기에 충분함을 증명합니다.

**실제 보편성:** 위의 연습은 복잡한 3-Qubit Gate가 더 단순한 Gate들로 구성되었음을 보여줍니다. 일반적으로 **임의의** 다중 Qubit 유니터리는 1- 및 2-Qubit Gate로 구성될 수 있습니다. Transpiler는 모든 양자 소프트웨어 스택의 핵심 구성 요소로, 우리가 실행하고자 하는 추상적인 알고리즘과 특정 양자 장치가 실제로 수행할 수 있는 물리적 연산 사이의 간격을 연결합니다.

**장치 예:** **ibmq_brisbane** 장치는 위에서 표시된 기저 Gate를 사용하는 Eagle 아키텍처를 사용합니다. 즉, 해당 기계로 전송된 모든 알고리즘은 이러한 연산의 시퀀스로 변환됩니다. 이 기준은 본질적으로 **제어 가능성**에 관한 것입니다; Qubit에 필요한 연산을 수행하기 위한 충분한 제어 수단이 있습니다.

## 5. 기준 5 – Qubit 측정 {#5-criterion-5--qubit-measurement}

**기준 5:** *"Qubit별 측정 기능."* 모든 Qubit의 상태가 측정 가능해야 합니다(일반적으로 계산 기저, |0〉 또는 |1〉에서). 다시 말해, 양자 Circuit을 실행한 후 각 Qubit을 0/1 고전 비트로 읽어낼 수 있어야 합니다. 이 기준은 각 Qubit에 대한 신뢰할 수 있는 검출기를 갖추고 측정할 Qubit을 선택할 수 있는 것에 관한 것입니다.

**데모 목표:** Qiskit에서 시뮬레이터와 실제 장치에서 측정을 수행하는 방법을 보여주고, 차이점(측정 노이즈 등)을 강조합니다. 다양한 상태의 일부 Qubit을 측정하고 결과를 검사합니다. 또한 시뮬레이터 대 하드웨어 결과를 비교하여 리드아웃 오류가 어떻게 나타날 수 있는지 시연합니다.

먼저, 간단한 측정 예:

```python
qc_measure = QuantumCircuit(2, 2)
qc_measure.x(0)              # qubit 0 -> |1>, qubit 1 stays |0>
qc_measure.measure([0, 1], [0, 1])
qc_measure.draw('mpl')
```

```python
sim_backend = AerSimulator()
pm = generate_preset_pass_manager(backend=sim_backend, optimization_level=1)
isa_qc_measure = pm.run(qc_measure)
job = sampler.run([isa_qc_measure], shots=1000)
result = job.result()
counts = result[0].data.c.get_counts()

print("Simulator measurement counts:", counts)
```

시뮬레이터에서 `01`의 1000회 카운트를 예상합니다. 이제 이를 시뮬레이션하여 **측정 오류**를 실제로 확인해 봅니다. Aer 시뮬레이터에 리드아웃 오류를 추가할 수 있습니다. Qiskit Aer는 `ReadoutError`를 정의하고 노이즈 모델에서 Qubit에 연결할 수 있게 합니다.

### 연습 5: 리드아웃 오류 시뮬레이션 {#exercise-5-simulate-readout-error}

각 Qubit이 2% 확률로 잘못 측정되는(0이 1로, 또는 1이 0으로 읽히는) 간단한 리드아웃 오류 모델을 정의하는 코드를 완성하세요. 그런 다음 이 노이즈 모델로 측정 Circuit을 실행하세요.

```python
from qiskit_aer.noise import NoiseModel, ReadoutError

# --- YOUR CODE HERE ---

# 1. Define a 2% readout error for each single qubit.
# [[P(0|0), P(1|0)], [P(0|1), P(1|1)]]
ro_error = ReadoutError([[0.98, 0.02], [0.02, 0.98]])

# 2. Create a new noise model
noise_model_ro = NoiseModel()

# 3. Add the readout error to all qubits in the noise model
noise_model_ro.add_all_qubit_readout_error(ro_error)

# --- END YOUR CODE ---

sim_backend.set_options(noise_model=noise_model_ro)
pm = generate_preset_pass_manager(backend=sim_backend, optimization_level=1)
isa_qc_measure = pm.run(qc_measure)

# Run the measurement circuit with readout noise
sampler = Sampler(mode=sim_backend)

job = sampler.run([isa_qc_measure], shots=1024)
result = job.result()
counts = result[0].data.c.get_counts()

print("Simulation with 2% readout error:", counts)
```

이 시뮬레이션 출력은 실제 하드웨어가 생성할 수 있는 것과 유사하게 일부 오류 카운트(`11`, `00`, `10` 등)를 보여주어 불완전한 측정의 영향을 시연합니다.

**장치 예:** **ibmq_brisbane**과 같은 실제 장치에서 동일한 Circuit을 실행하면 잘못된 결과에 대해 유사한 0이 아닌 카운트를 볼 수 있습니다. 장치 교정 데이터는 각 Qubit의 리드아웃 오류를 나열합니다. 특정 Qubit을 대상으로 읽어낼 수 있는 것이 중요하며, 오류 특성을 이해하는 것이 의미 있는 결과를 얻는 핵심입니다. 실제 장치에서의 실행은 **연습 1b: 실제 IBM 양자 컴퓨터에서 실행하기**에서 시연되었습니다.

## 양자 통신 기준 (비행 Qubit) {#quantum-communication-criteria-flying-qubits}

DiVincenzo는 또한 네트워크로 연결된 양자 컴퓨터를 구축할 때 중요한 양자 통신에 특화된 두 가지 기준을 나열했습니다:

6. **고정 Qubit과 비행 Qubit 간의 변환 능력.** (예: 프로세서의 Qubit을 이동 가능한 광자에 매핑하는 것.)
7. **위치 간에 비행 Qubit을 충실하게 전송하는 능력.** (예: 양자 정보를 잃지 않고 광섬유를 통해 광자 Qubit을 전송하는 것.)

Qiskit은 주로 칩의 고정 Qubit을 다루기 때문에 이것들은 표준 Qiskit 사용 범위를 벗어납니다. 그러나 간단한 예로 이러한 기준의 *개념*을 설명할 수 있습니다: **양자 텔레포테이션**. 텔레포테이션은 고정 Qubit의 상태를 얽힌 쌍에 의해 전달되는 정보("비행" 부분)와 고전 통신으로 변환하여 다른 곳의 또 다른 고정 Qubit에서 상태를 재구성하는 것을 보여줍니다.

### 권장 활동: Qiskit in Classrooms *양자 텔레포테이션* 모듈 수강 {#recommended-activity-take-the-qiskit-in-classrooms-quantum-teleportation-module}

Dr. Katie McCormick의 Qiskit in Classrooms [Quantum Teleportation](https://quantum.cloud.ibm.com/learning/en/modules/computer-science/quantum-teleportation) 모듈은 양자 정보에서 가장 흥미로운 프로토콜 중 하나인 양자 텔레포테이션을 안내합니다. 여기서는 얽힘과 단 두 개의 고전 비트를 사용하여 양자 상태(Qubit)가 Alice에서 Bob으로 전송됩니다. 얽힌 Bell 쌍을 준비하는 방법, Alice 쪽에서 Bell 기저 측정을 수행하는 방법, 고전적 결과를 전송하는 방법, Bob의 Qubit에 올바른 양자 Gate를 적용하여 원래 상태를 완벽하게 복구하는 방법 등 전체 텔레포테이션 절차를 단계별로 배웁니다. 그 과정에서 Qubit의 정보를 텔레포테이션하는 것이 복제 불가 정리를 위반하거나 빛의 속도를 초과하지 않는 이유를 탐구합니다. IBM 양자 하드웨어나 시뮬레이터를 사용한 실습 연습을 통해 측정, 얽힘, 피드포워드 제어의 실제 적용에 대한 실질적인 이해를 얻게 됩니다.

양자 텔레포테이션을 마스터함으로써 서로 다른 노드 간에 양자 정보를 인코딩, 전송, 복구하는 방법을 이해하게 되며, 이는 양자 네트워크, 중계기 시스템, 보안 통신 체계, 그리고 확장 가능한 모듈식 양자 컴퓨팅의 기초를 다집니다.

**기준 6 & 7과의 관계:** 실제 양자 네트워크에서 공유된 얽힌 쌍은 Alice와 Bob의 위치 사이에 "비행" Qubit(예: 광자)을 배포하여 생성됩니다(기준 7: 충실한 전송). 텔레포테이션 프로토콜 자체는 Alice의 고정 Qubit 상태를 얽힌 쌍의 절반에 매핑하는 방법으로 작동하여 효과적으로 Bob에게 '전송'합니다(기준 6: 변환). Qiskit을 사용하면 프로토콜의 논리를 완벽하게 시뮬레이션할 수 있어 통신 아키텍처에서 이러한 기준이 어떻게 충족되는지에 대한 개념적 모델을 제공합니다.

## 결론 및 요약 {#conclusion--summary}

우리는 Qiskit을 사용하여 DiVincenzo 기준을 설명하는 일련의 코드 중심 연습을 설계했습니다. 이 실습 예제들을 통해 실제 양자 컴퓨팅 플랫폼이 각 요건을 어떻게 충족하는지 탐구했습니다:

- **확장성**: 더 많은 Qubit에서 Circuit을 구축하고 노이즈 확장을 이해합니다.
- **초기화**: 리셋과 상태 준비를 사용하여 알려진 상태에서 계산을 안정적으로 시작합니다.
- **보편적 Gate**: 복잡한 연산을 기계의 기저 Gate로 Transpile하여 임의의 계산을 수행할 수 있음을 증명합니다.
- **측정**: Qubit을 읽어내고 실제 리드아웃 오류를 처리합니다.
- **결맞음**: 알고리즘 충실도에 미치는 유한한 T₁, T₂의 영향과 결어긋남에 비해 연산이 빠르게 이루어져야 할 필요성을 확인합니다.

완결성을 위해 Qiskit in Classrooms [Quantum Teleportation](https://quantum.cloud.ibm.com/learning/en/modules/computer-science/quantum-teleportation) 모듈을 통해 양자 통신 측면도 다루어 마지막 두 기준(비행 Qubit)을 연결했습니다.

마지막으로, 이러한 기준들이 IBM과 같은 실제 양자 컴퓨터에서 어떻게 결합되는지 주목할 필요가 있습니다. **ibmq_brisbane**과 같은 장치는 127개의 초전도 Qubit을 갖추고 있으며(기준 1), 각각 |0〉에서 시작하고(기준 2), 보편성을 위한 교정된 Gate 집합과 컴파일러를 갖추고 있으며(기준 4), 각 Qubit에 대한 마이크로파 리드아웃 공진기를 가지고 있고(기준 5), 나노초 연산 대비 수백 마이크로초 수준의 결맞음 시간을 가집니다(기준 3). 양자 네트워킹 실험을 위해 IBM과 다른 기관들은 비행 Qubit을 위한 마이크로파-광학 변환기와 원거리 Qubit 얽힘을 탐구하고 있습니다(기준 6 & 7); 이것들은 현재 활발히 연구되는 분야입니다.

이 노트북의 연습을 완료함으로써 DiVincenzo 기준의 정의를 보았을 뿐만 아니라 코드를 통해 *직접 체험*했습니다; 각 요건이 실제 양자 하드웨어와 알고리즘에 어떤 의미인지에 대한 직관을 키웠습니다. 이 실험들을 확장해 보세요, 양자 컴퓨팅을 즐기세요!

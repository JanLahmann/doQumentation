## Backend 목록 {#list-backends}

접근 가능한 Backend를 확인하려면 [컴퓨팅 리소스 페이지](https://quantum.cloud.ibm.com/computers)에서 목록을 확인하거나, [`QiskitRuntimeService.backends()`](../api/qiskit-ibm-runtime/qiskit-runtime-service#backends) 메서드를 사용할 수 있습니다. 이 메서드는 [`IBMBackend`](../api/qiskit-ibm-runtime/ibm-backend) 인스턴스의 목록을 반환합니다.

<Admonition type="note" title="참고 사항">
- 특정 인스턴스 또는 지역에 로그인했거나, `QiskitRuntimeService()`를 사용하여 특정 인스턴스 또는 지역으로 서비스를 초기화한 경우, 해당 인스턴스 또는 지역에서 사용 가능한 Backend만 반환됩니다. 그렇지 않은 경우, 모든 인스턴스 및 모든 지역에서 사용 가능한 Backend가 반환됩니다.
- 반환된 Backend 목록은 IBM Quantum Platform [컴퓨팅 리소스](https://quantum.cloud.ibm.com/computers) 페이지에 표시된 것과 다를 수 있습니다. 컴퓨팅 리소스 페이지의 목록은 항상 페이지 상단에서 선택한 지역을 기준으로 필터링됩니다.
</Admonition>

## Backend 필터링 {#filter-backends}

사용 가능한 Backend를 속성별로 필터링할 수도 있습니다. 더 일반적인 필터를 사용하려면 `filters` 인수를 Backend 객체를 받아 조건을 충족하면 `True`를 반환하는 함수로 설정합니다. 자세한 내용은 [API 문서](../api/qiskit-ibm-runtime/qiskit-runtime-service#backends)를 참조하세요.

다음 코드는 아래 기준을 충족하고 _현재 선택된 인스턴스에서_ 사용 가능한 Backend만 반환합니다.

*   실제 양자 장치임 (`simulator=False`)
*   현재 운영 중임 (`operational=True`)
*   최소 5개의 Qubit을 보유함 (`min_num_qubits=5`)

```python
# Optionally pass in an instance, region, or both, to
# further filter the backends.
service = QiskitRuntimeService()

service.backends(simulator=False, operational=True, min_num_qubits=5)
```

```text
[<IBMBackend('ibm_pittsburgh')>,
 <IBMBackend('ibm_boston')>,
 <IBMBackend('ibm_fez')>,
 <IBMBackend('ibm_miami')>,
 <IBMBackend('ibm_marrakesh')>,
 <IBMBackend('ibm_torino')>,
 <IBMBackend('ibm_kingston')>]
```

이러한 키워드 인수를 사용하여 Backend 구성([JSON 스키마](https://github.com/Qiskit/ibm-quantum-schemas/blob/main/schemas/backend_configuration_schema.json)) 또는 상태([JSON 스키마](https://github.com/Qiskit/ibm-quantum-schemas/blob/main/schemas/backend_status_schema.json))의 모든 속성을 기준으로 필터링할 수 있습니다. 유사한 메서드로 [`QiskitRuntimeService.least_busy()`](../api/qiskit-ibm-runtime/qiskit-runtime-service#least_busy)가 있으며, 이는 `backends()`와 동일한 필터를 사용하지만 필터 조건을 충족하면서 대기열에서 보류 중인 작업 수가 가장 적은 Backend를 반환합니다.

```python
service.least_busy(operational=True, min_num_qubits=5)
```

```text
<IBMBackend('ibm_torino')>
```

## 정적 Backend 정보 {#static-backend-information}

Backend에 대한 일부 정보(예: 이름, 버전, Qubit 수, 지원하는 기능 유형)는 자주 변경되지 않습니다. 이 정보는 `backend` 객체의 속성으로 사용할 수 있습니다.

다음 코드는 Backend에 대한 설명을 작성합니다.

```python
backend = service.backend("ibm_fez")

print(
    f"Name: {backend.name}\n"
    f"Version: {backend.version}\n"
    f"No. of qubits: {backend.num_qubits}\n"
)
```

```text
Name: ibm_fez
Version: 2
No. of qubits: 156
```

전체 속성 목록은 [`IBMBackend` API 문서](https://docs.quantum.ibm.com/api/qiskit-ibm-runtime/ibm-backend)를 참조하세요.
## 네이티브 Gate 및 연산 {#native-gates-and-operations}

각 [프로세서 제품군](/guides/processor-types)에는 네이티브 Gate 세트가 있습니다. 기본적으로 각 제품군의 QPU는 네이티브 Gate 세트에 있는 Gate와 연산만 실행할 수 있습니다. 따라서 Circuit의 모든 Gate는 Transpiler에 의해 이 세트의 요소로 변환되어야 합니다.

QPU의 네이티브 Gate와 연산은 [Qiskit으로 확인](#native-gates-with-qiskit)하거나 IBM Quantum&reg; Platform [컴퓨팅 리소스 페이지](/guides/qpu-information#native-gates-on-platform)에서 확인할 수 있습니다.

<span id="native-gates-with-qiskit"></span>
```python

from qiskit_ibm_runtime import QiskitRuntimeService

service = QiskitRuntimeService()

for backend in service.backends():
    config = backend.configuration()
    if "simulator" in config.backend_name:
        continue
    print(f"Backend: {config.backend_name}")
    print(f"    Processor type: {config.processor_type}")
    print(f"    Supported instructions:")
    for instruction in config.supported_instructions:
        print(f"        {instruction}")
    print()
```
## 동적 Backend 정보 {#dynamic-backend-information}

Backend는 Qubit 주파수 및 연산 오류율과 같이 보정될 때마다 변경되는 속성을 가질 수도 있습니다. Backend는 보통 24시간마다 보정되며, 보정 과정이 완료된 후 속성이 업데이트됩니다. 이러한 속성은 양자 Circuit을 최적화하거나 고전 시뮬레이터용 노이즈 모델을 구성하는 데 사용할 수 있습니다.

### Qubit 속성 {#qubit-properties}

`backend.properties().qubit_property()`는 Qubit의 물리적 속성에 대한 정보를 반환합니다. 여기에는 Qubit의 다양한 속성이 딕셔너리 형태로 포함되며, 각 속성은 값과 마지막 보정의 타임스탬프와 함께 제공됩니다.

- `T1 (이완 시간)`: T1 시간은 Qubit이 에너지 이완으로 인해 바닥 상태 $|0\rangle$으로 붕괴하기 전까지 들뜬 상태 $|1\rangle$을 유지하는 평균 시간을 나타냅니다. 이 매개변수는 Qubit의 에너지 이완 동작을 특성화하는 데 사용되며, 단위는 초(s)입니다.

- `T2 (위상 이완 시간)`: T2 시간은 Qubit이 $|0\rangle$과 $|1\rangle$ 상태의 중첩에서 위상 결맞음을 유지하는 시간 척도를 나타냅니다. 에너지 이완과 순수 위상 이완 과정을 모두 고려하여 Qubit의 결맞음 특성에 대한 통찰을 제공합니다.

- `frequency`: 이 매개변수는 Qubit의 공진 주파수를 지정하며, $|0\rangle$과 $|1\rangle$ 상태 사이의 에너지 차이를 나타내고 단위는 헤르츠(Hz)입니다.

- `anharmonicity`: 비조화성(Anharmonicity)은 Qubit의 첫 번째와 두 번째 들뜬 상태 사이의 에너지 차이로, 단위는 헤르츠(Hz)입니다.

- `readout_error`: 읽기 오류는 Qubit 상태를 잘못 측정할 평균 확률을 정량화합니다. 일반적으로 prob_meas0_prep1과 prob_meas1_prep0의 평균으로 계산되며, 측정 충실도에 대한 단일 지표를 제공합니다.

- `prob_meas0_prep1`: 이 매개변수는 $|1\rangle$ 상태로 준비하려 했을 때 Qubit을 0 상태로 측정할 확률, 즉 $P(0 | 1)$을 나타냅니다. 상태 준비 및 측정(SPAM) 오류, 특히 초전도 Qubit에서의 측정 오류를 반영합니다.

- `prob_meas1_prep0`: 마찬가지로, 이 매개변수는 $|0\rangle$ 상태로 준비하려 했을 때 Qubit을 1 상태로 측정할 확률, 즉 $P(1 | 0)$을 나타냅니다. prob_meas0_prep1과 마찬가지로 SPAM 오류를 반영하며, 초전도 Qubit에서는 측정 오류가 주요 원인입니다.

- `readout_length`: readout_length는 Qubit의 읽기 연산 지속 시간을 지정합니다. 측정 펄스 시작부터 신호 디지털화 완료까지의 시간을 측정하며, 그 이후 시스템은 다음 연산을 수행할 준비가 됩니다. 특히 회로 중간 측정을 포함할 때 회로 실행을 최적화하는 데 이 매개변수를 이해하는 것이 중요합니다.

### 명령 속성 {#instruction-properties}

`backend.target` 속성은 `qiskit.transpiler.Target` 객체로, 해당 Backend에 대한 Circuit을 트랜스파일하는 데 필요한 모든 정보를 포함합니다. 여기에는 명령 오류 및 지속 시간이 포함됩니다. 예를 들어, 다음 코드는 Qubit 1과 0 사이에서 작동하는 [`cz` Gate](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.library.CZGate)의 속성을 가져옵니다.

```python
backend.target["cz"][(1, 0)]
```

```text
InstructionProperties(duration=6.8e-08, error=0.007831625819164134)
```

다음 코드는 Qubit 0에서 측정 연산(읽기 오류 포함)의 속성을 보여줍니다.

```python
backend.target["measure"][(0,)]
```

```text
InstructionProperties(duration=1.56e-06, error=0.013916015625)
```

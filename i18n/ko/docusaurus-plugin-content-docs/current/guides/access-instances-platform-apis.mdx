# IBM Cloud Platform API를 사용하여 인스턴스에 액세스하기 {#use-ibm-cloud-platform-apis-to-access-instances}

이 가이드는 IBM Cloud Platform API를 사용하여 인스턴스를 프로비저닝하거나, 업데이트하거나, 액세스를 관리하는 모든 분들을 위한 것입니다. 인스턴스를 프로비저닝할 때 설정할 수 있는 [파라미터](#parameters)와 인스턴스에 액세스할 때 반환되는 [확장 필드](#extensions) 세트에 대한 자세한 내용을 제공합니다.

<Admonition type="note">
자세한 내용은 IBM Cloud&reg; ResourceController API 문서를 참고하세요.

관련 페이지:
- [새 리소스 인스턴스 생성(프로비저닝)](https://cloud.ibm.com/apidocs/resource-controller/resource-controller#create-resource-instance)
- [리소스 인스턴스 업데이트](https://cloud.ibm.com/apidocs/resource-controller/resource-controller#update-resource-instance)
- [리소스 인스턴스 가져오기](https://cloud.ibm.com/apidocs/resource-controller/resource-controller#get-resource-instance)
- [모든 리소스 인스턴스 목록 가져오기](https://cloud.ibm.com/apidocs/resource-controller/resource-controller#list-resource-instances)  
    
    모든 리소스 인스턴스 목록을 가져올 때, `resource_id = "b6049020-80f4-11eb-a0f7-e35ec9b4054f"`를 사용하면 Qiskit Runtime 인스턴스만 필터링할 수 있습니다.

</Admonition>

## `parameters` {#parameters}

인스턴스를 프로비저닝하거나 업데이트할 때 요청 본문에 사용합니다:

- **instance_limit_seconds** (문자열 또는 null, `^[0-9]+$`): 인스턴스에 설정할 제한 시간입니다. Open Plan의 경우: 생략하면 기본값은 10분입니다.
.
- **usage_allocation_seconds** (문자열 또는 null, `^[0-9]+$`): 인스턴스에 할당된 시간으로, 모든 QPU 사용량을 기반으로 큐에서의 우선순위를 결정하는 공정 분배 스케줄러에 사용됩니다. (종량제 인스턴스에는 적용되지 않습니다.)
- **backends** (Backend 목록 배열(문자열 또는 null) 또는 "ANY"가 포함된 배열): 이 인스턴스에서 사용 가능한 Backend 이름의 허용 목록입니다. 기본값: `["ANY"]`는 이 플랜에서 사용 가능한 모든 Backend를 의미합니다. `[]`는 Backend 없음을 의미합니다.

## `extensions` {#extensions}

인스턴스를 프로비저닝하거나 인스턴스 세부 정보를 가져올 때의 응답에 포함됩니다:

서비스에서 제공하는 추가 인스턴스 속성으로, 키-값 쌍으로 표현됩니다. 이 속성들은 extensions 속성 하위에 인스턴스와 함께 유지됩니다.

- **instance_limit_seconds** (정수 또는 null, `>= 0`): 인스턴스에 설정할 시간 제한입니다.
- **usage_allocation_seconds** (정수 또는 null, `>= 0`): 인스턴스에 할당된 시간으로, 모든 QPU 사용량을 기반으로 큐에서의 우선순위를 결정하는 공정 분배 스케줄러에 사용됩니다.
- **backends** (필수; Backend 목록 배열(문자열) 또는 "ANY"가 포함된 배열(문자열)): 이 인스턴스에서 사용 가능한 Backend 이름의 허용 목록입니다. 기본값: `["ANY"]`는 이 플랜에서 모든 Backend를 사용할 수 있음을 의미합니다. `[]`는 Backend 없음을 의미합니다.

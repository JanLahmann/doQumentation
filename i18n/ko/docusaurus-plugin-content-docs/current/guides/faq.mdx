# 자주 묻는 질문 (FAQ) {#frequently-asked-questions-faq}

실행 모드에 관한 자주 묻는 질문은 [Qiskit Runtime 실행 모드 FAQ](/guides/execution-modes-faq) 가이드를 참조하세요.

## 연구에서 Qiskit을 어떻게 인용하나요? {#how-do-i-cite-qiskit-in-my-research}

포함된 [BibTeX 파일](https://github.com/Qiskit/qiskit/blob/main/CITATION.bib)을 사용하여 Qiskit을 인용하세요.

## 연구에서 IBM Quantum 컴퓨터를 어떻게 인용하나요? {#how-do-i-cite-an-ibm-quantum-computer-in-my-research}

연구 논문의 경우, 저자들이 다음과 같이 IBM Quantum&reg;을 인정해 주시길 권장합니다:

>    이 연구에서 IBM Quantum 서비스를 사용하였음을 인정합니다. 여기에 표현된 견해는 저자들의 것이며, IBM 또는 IBM Quantum 팀의 공식 정책이나 입장을 반영하지 않습니다.

논문 참고 문헌에는 다음 인용 형식을 사용하세요:

> IBM Quantum. https://<span>quantum.cloud.ibm.com</span>/, YYYY

논문에서 양자 처리 장치(Quantum processing units)는 고유한 이름(예: `ibmq_vigo`)으로 참조해야 하며, 선택적으로 버전(예: `ibmq_vigo` v1.0.2)을 추가할 수 있습니다. 프로세서를 참조하는 것도 권장합니다. 예를 들어:

> 이 논문에서는 IBM Quantum Canary 프로세서 중 하나인 `ibmq_vigo`를 사용하였습니다.

IBM Quantum 프로그램 인용 예시:

> IBM Quantum (2025). Estimator primitive (Version x.y.z) [computer software]. https://<span>quantum.cloud.ibm.com</span>/

## 연구에서 IBM Quantum Composer를 어떻게 인용하나요? {#how-do-i-cite-ibm-quantum-composer-in-my-research}

IBM Quantum Composer 인용 예시:

> IBM Quantum Composer. 2025. url: https://<span>quantum.cloud.ibm.com/composer</span>

## 관리 작업 등 사용자 인터페이스에서 일부 항목이 보이지 않아요. {#i-dont-see-something-in-the-user-interface-such-as-my-administration-tasks}

[IBM Quantum Platform](https://quantum.cloud.ibm.com) 인터페이스의 헤더에 있는 계정 전환기에서 다른 계정을 선택해 보세요.

## Open Plan만 사용하고 있는데 신용카드를 입력해야 했어요. 요금이 청구되나요? {#im-just-using-the-open-plan-but-i-had-to-put-in-my-credit-card-will-i-be-charged}

종량제(Pay-As-You-Go) [인스턴스](/guides/instances)를 생성하고 이를 사용하여 QPU에 워크로드를 전송하지 않는 한 요금이 청구되지 않습니다. 비용 한도를 설정하는 방법과 비용을 관리하는 다른 방법에 대해서는 [비용 관리](/guides/manage-cost)를 참조하세요.

## API 키와 API 토큰의 차이는 무엇인가요? {#whats-the-difference-between-an-api-key-and-an-api-token}

차이가 없습니다. API 키와 API 토큰이라는 용어는 같은 의미로 사용됩니다. 서로 다른 인터페이스에서 서로 다른 용어를 사용하기 때문에 문서에서 두 용어가 모두 사용됩니다.

<span id="bearer"></span>
## API 토큰과 bearer 토큰의 차이는 무엇인가요? {#whats-the-difference-between-an-api-token-and-a-bearer-token}

API 토큰은 [Qiskit Runtime 서비스에 인증](/guides/hello-world#install-and-authenticate)하는 데 사용되는 장기 자격 증명입니다. bearer 토큰은 [신뢰할 수 없는 환경에서 REST API에 인증](/guides/cloud-setup-untrusted#authenticate-to-the-rest-api-in-an-untrusted-environment)하는 데 사용되는 임시 자격 증명입니다. bearer 토큰은 Qiskit Runtime 서비스에 인증하는 데 사용할 수 없습니다.

## 거주하는 지역을 선택해야 하나요? {#do-i-have-to-choose-the-region-that-i-live-in}

아니요. 어느 지역이든 사용할 수 있습니다. 지역은 작업이 실행되는 위치와 데이터가 보관되는 위치를 결정합니다. 또한, 현재 로그인한 인스턴스에서 생성된 인스턴스만 보고 사용할 수 있습니다. us-east 지역만 Open Plan을 지원한다는 점에 유의하세요.

## 프록시 서버를 통해 연결할 수 있나요? {#can-i-connect-through-a-proxy-server}

네, 하지만 Qiskit Runtime v0.44.0 이상을 사용해야 합니다.

## 데이터 보존 정책은 무엇인가요? {#what-is-the-data-retention-policy}

IBM은 Qiskit Runtime 서비스를 계속 사용하는 동안 귀하의 (삭제되지 않은) 작업 및 사용 데이터를 3년간 보유합니다.

## Qiskit Aer, Qiskit Nature, Qiskit Experiments와 같은 Python 배포판은 어떻게 되었나요? {#what-happened-to-python-distributions-such-as-qiskit-aer-qiskit-nature-and-qiskit-experiments}

Qiskit SDK v1.0 출시 이후, `pip install qiskit` 명령은 [Qiskit SDK](https://github.com/Qiskit/qiskit)**만** 설치합니다. 이는 과거에 사용했던 `qiskit-aer` 또는 `qiskit-nature`와 같은 일부 모듈을 이제 별도로 설치해야 한다는 것을 의미합니다.

Qiskit SDK v1.0 출시 이전에는 Qiskit이 `qiskit-terra`(컴파일러 코어), `qiskit-aer`(고성능 시뮬레이터), 기존 IBM Quantum&reg; 프로바이더, 그리고 특정 탐색적 알고리즘이나 실험 실행 기능을 제공하는 여러 현재는 더 이상 사용되지 않는 패키지 등 다양한 Python 배포판으로 구성되어 있었습니다.

이러한 별도의 소프트웨어 패키지들은 Qiskit *메타패키지*라고 불리는 것으로 통합되어, 사용자들이 `pip install qiskit`을 통해 모든 구성 요소를 함께 설치할 수 있었습니다. 이 메타패키지 아키텍처는 Qiskit SDK v1.0 출시와 함께 제거되었으며, 이제 각 개별 소프트웨어 배포판은 별도로 설치해야 합니다.

이전에 Qiskit 메타패키지에 포함되어 있던 패키지를 설치하려면 [Qiskit 에코시스템](https://www.ibm.com/quantum/ecosystem)을 방문하여 필요에 맞는 다양한 패키지를 찾아보세요. 이 변경에 대한 자세한 정보는 [v1.0 마이그레이션 가이드](/guides/metapackage-migration)에서도 확인할 수 있습니다.

[계정 닫기](https://cloud.ibm.com/docs/account?topic=account-account-close) 페이지에서 안내를 확인하세요.

# IBM Quantum System 서비스의 활동 추적 이벤트 {#activity-tracking-events-for-the-ibm-quantum-system-service}

IBM Cloud&reg; 서비스(예: IBM&reg; Quantum System 서비스)는 활동 추적 이벤트를 생성합니다.

활동 추적 이벤트는 IBM Cloud에서 서비스 상태를 변경하는 활동을 보고합니다. 이 이벤트를 사용하여 비정상적인 활동과 중요한 작업을 조사하고 규정 감사 요구 사항을 준수할 수 있습니다.

IBM Cloud Activity Tracker Event Routing(플랫폼 서비스)을 사용하여 대상 및 경로를 구성함으로써 활동 추적 이벤트가 전송될 위치를 정의하고 계정의 감사 이벤트를 원하는 목적지로 라우팅할 수 있습니다. 자세한 내용은 [IBM Cloud Activity Tracker Events Routing 소개](https://cloud.ibm.com/docs/atracker?topic=atracker-about)를 참조하세요.

IBM Cloud Logs를 사용하여 계정에서 생성되고 IBM Cloud Activity Tracker Event Routing에 의해 IBM Cloud Logs 인스턴스로 라우팅된 이벤트를 시각화하고 알림을 받을 수 있습니다.

## IBM Quantum System 서비스의 활동 추적 이벤트 보기 {#viewing-activity-tracking-events-for-the-ibm-quantum-system-service}

IBM Cloud Logs를 사용하여 계정에서 생성되고 IBM Cloud Activity Tracker Event Routing에 의해 IBM Cloud Logs 인스턴스로 라우팅된 이벤트를 시각화하고 알림을 받을 수 있습니다.

### Observability 페이지에서 IBM Cloud Logs 실행 {#launching-ibm-cloud-logs-from-the-observability-page}

IBM Cloud Logs UI 실행에 대한 정보는 IBM Cloud Logs 문서의 [UI 실행](https://cloud.ibm.com/docs/cloud-logs?topic=cloud-logs-instance-launch#instance-launch-cloud-ui)을 참조하세요.

| 작업 | 트리거 조건 |
| -- | -- |
| `quantum-computing.direct-access-backend-configuration.read` | 백엔드 구성 가져오기 (`GET /v1/backends/{backend-name}/configuration`) |
| `quantum-computing.direct-access-backend-properties.read` | 백엔드 속성 가져오기 (`GET /v1/backends/{backend-name}/properties`) |
| `quantum-computing.direct-access-backend.list` | 백엔드 목록 가져오기 (`GET /v1/backends`) |
| `quantum-computing.direct-access-backend.read` | 백엔드 상세 정보 가져오기 (`GET /v1/backends/{backend-name}`) |
| `quantum-computing.direct-access-job.cancel` | 작업 취소 (`POST /v1/jobs/{job-id}/cancel`) |
| `quantum-computing.direct-access-job.create` | 작업 실행 (`POST /v1/jobs`) |
| `quantum-computing.direct-access-job.delete` | 작업 삭제 (`DELETE /v1/jobs/{job-id}`) |
| `quantum-computing.direct-access-job.list` | 작업 목록 가져오기 (`GET /v1/jobs`) |
| `quantum-computing.direct-access-job.read` | ID로 작업 가져오기 (`GET /v1/jobs/{job-id}`) |
| `quantum-computing.direct-access-lane-configuration.list` | 백엔드 레인 구성 가져오기 (`GET /v1/backends/{backend-name}/lanes`) |

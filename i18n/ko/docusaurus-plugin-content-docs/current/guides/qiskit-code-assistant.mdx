# Qiskit Code Assistant {#qiskit-code-assistant}

Qiskit Code Assistant는 새로운 Qiskit 사용자들에게 양자 컴퓨팅을 보다 쉽게 접근할 수 있도록 하고, 기존 사용자들의 코딩 경험을 향상시키는 것을 목표로 합니다. 이것은 [watsonx](https://www.ibm.com/products/watsonx-ai)를 기반으로 하는 생성형 AI 코드 어시스턴트입니다. Qiskit SDK의 수백만 개의 텍스트 토큰, 수년간의 Qiskit 코드 예제, 그리고 IBM Quantum&reg; 기능을 활용하여 훈련되었습니다. Qiskit Code Assistant는 IBM&reg;의 최신 기능과 기능성을 통합한 [IBM Granite 모델](https://www.ibm.com/products/watsonx-ai/foundation-models)을 기반으로 LLM이 생성한 제안을 제공함으로써 양자 개발 워크플로우를 지원합니다.

<Admonition type="note" title="참고 사항">
- 이 기능은 새로운 [IBM Quantum Platform](https://quantum.cloud.ibm.com/)에 등록된 IBM Quantum Premium Plan 사용자에게 제공되는 실험적 기능입니다.
- Qiskit Code Assistant는 미리 보기 릴리스 상태이며 변경될 수 있습니다.
- 피드백이 있거나 개발팀에 문의하고 싶으시면 [Qiskit Slack Workspace 채널](https://qiskit.enterprise.slack.com/archives/C07LYA6PL83) 또는 관련 공개 GitHub 저장소를 이용해 주세요.
</Admonition>

## 기능 {#features}

다음 기능들은 [Visual Studio Code](https://code.visualstudio.com/) (VS Code) 및 호환 편집기, 그리고 [JupyterLab](https://jupyterlab.readthedocs.io/en/latest/) 확장 프로그램에 포함되어 있습니다:

* Qiskit 코드 생성에 특화된 모델을 기반으로 한 생성형 AI를 활용하여 Qiskit 코드 생성을 가속화합니다.
* 추상적이고 구체적인 프롬프트를 통해 추천 사항을 생성할 수 있습니다.
* 검토, 수락 또는 거부할 수 있는 제안을 제시합니다.
* Python 코드 및 Jupyter 노트북 파일을 지원합니다.
* 혐오 발언과 같이 사용자에게 잠재적인 위험을 초래할 수 있는 질문에 답변하지 않도록 하는 가드레일이 포함되어 있습니다.

Qiskit Code Assistant를 개발 환경에 직접 통합하는 방법은 해당 항목의 지침을 따르세요:
- [JupyterLab](/guides/qiskit-code-assistant-jupyterlab)
- [VS Code](/guides/qiskit-code-assistant-vscode)

## Qiskit Code Assistant의 대규모 언어 모델(LLM) {#the-large-language-model-llm-behind-qiskit-code-assistant}

코드 제안을 제공하기 위해 Qiskit Code Assistant는 대규모 언어 모델(LLM)을 사용합니다. 현재 Qiskit Code Assistant는 [Mistral-Small-3.2-24B-Qiskit 모델](https://huggingface.co/Qiskit/mistral-small-3.2-24b-qiskit)을 기반으로 구축된 `mistral-small-3.2-24b-qiskit` 모델을 사용합니다. `mistral-small-3.2-24b-qiskit` 모델은 고품질 Qiskit 데이터, Python 커밋 및 채팅에 대한 확장 사전 훈련과 미세 조정을 통해 `Mistral-Small-3.2-24B-Instruct-2506` 모델의 Qiskit 코드 생성 능력을 향상시킵니다. Mistral AI 모델 패밀리에 대한 자세한 내용은 [Mistral AI 문서](https://docs.mistral.ai/getting-started/models)를 참조하세요. `.*-qiskit` 모델에 대한 자세한 내용은 [Qiskit Code Assistant: Training LLMs for generating Quantum Computing Code](https://arxiv.org/abs/2405.19495)를 확인하세요.
2. [qiskit/qwen2.5-coder-14b-qiskit](https://huggingface.co/Qiskit/qwen2.5-coder-14b-qiskit) - Released June 2025
3. [qiskit/granite-3.3-8b-qiskit](https://huggingface.co/Qiskit/granite-3.3-8b-qiskit) - Released June 2025
4. [qiskit/granite-3.2-8b-qiskit](https://huggingface.co/Qiskit/granite-3.2-8b-qiskit) - Released June 2025

Qiskit에 특화된 LLM은 오픈 소스 모델로도 제공됩니다. https://huggingface.co/Qiskit 에서 모든 사용 가능한 모델을 확인하세요.

## Qiskit HumanEval 및 Qiskit HumanEval Hard 벤치마크 {#the-qiskit-humaneval-and-qiskit-humaneval-hard-benchmarks}

`mistral-small-3.2-24b-qiskit` 및 기타 모델을 테스트하기 위해, 저희는 Qiskit Advocates 및 전문가들과 협력하여 Qiskit HumanEval(QHE) 및 Qiskit HumanEval Hard(QHE Hard)라는 실행 기반 벤치마크를 만들고 모델에 적용했습니다. 이 벤치마크들은 공식 Qiskit 라이브러리를 기반으로 한 여러 도전적인 코드 문제를 포함하는 [HumanEval](https://arxiv.org/abs/2107.03374)과 유사합니다.

벤치마크는 약 150개의 테스트로 구성되어 있으며, 각 테스트는 함수 정의로 시작하고 모델이 해결해야 하는 작업을 자세히 설명하는 독스트링이 뒤따릅니다. 각 예제에는 생성된 솔루션의 정확성을 평가하기 위한 참조 표준 솔루션과 단위 테스트도 포함되어 있습니다. 테스트의 난이도는 기초, 중급, 고급의 세 단계로 나뉩니다. Qiskit HumanEval Hard 벤치마크는 Qiskit HumanEval의 변형이지만, 코드 임포트와 관련된 정보를 제거하여 LLM이 올바른 메서드나 클래스 임포트를 스스로 파악해야 합니다. 저희의 테스트와 초기 결과에 따르면, 이 변경으로 인해 LLM에게 데이터셋이 훨씬 더 도전적이 됩니다.

Qiskit HumanEval 및 Qiskit HumanEval Hard 데이터셋은 다음 웹사이트에서 확인할 수 있습니다: [Qiskit HumanEval](https://huggingface.co/datasets/Qiskit/qiskit_humaneval) 및 [Qiskit HumanEval](https://huggingface.co/datasets/Qiskit/qiskit_humaneval_hard). [GitHub 저장소](https://github.com/qiskit-community/qiskit-human-eval)에서 이 벤치마크의 개발에 기여할 수 있습니다.

### 빠른 시작 (권장) {#quick-start-recommended}

로컬 모드에서 Qiskit Code Assistant를 시작하는 가장 쉬운 방법은 VS Code 또는 JupyterLab 확장 프로그램용 자동화된 설정 스크립트를 사용하는 것입니다. 이 스크립트는 LLM을 실행하기 위한 [Ollama](https://ollama.com/)를 자동으로 설치하고, 권장 모델을 다운로드하며, 확장 프로그램을 구성합니다.

#### VS Code 확장 프로그램 설정 {#vs-code-extension-setup}

터미널에서 다음 명령을 실행하세요:

```bash
bash <(curl -fsSL https://raw.githubusercontent.com/Qiskit/qiskit-code-assistant-vscode/main/setup_local.sh)
```

이 스크립트는 다음 단계를 수행합니다:
- Ollama 설치 (아직 설치되지 않은 경우)
- 권장 Qiskit Code Assistant 모델 다운로드 및 구성
- 로컬 배포와 함께 작동하도록 VS Code 확장 프로그램 설정

#### JupyterLab 확장 프로그램 설정 {#jupyterlab-extension-setup}

터미널에서 다음 명령을 실행하세요:

```bash
bash <(curl -fsSL https://raw.githubusercontent.com/Qiskit/qiskit-code-assistant-jupyterlab/main/setup_local.sh)
```

이 스크립트는 다음을 수행합니다:
- Ollama 설치 (아직 설치되지 않은 경우)
- 권장 Qiskit Code Assistant 모델 다운로드 및 구성
- 로컬 배포와 함께 작동하도록 JupyterLab 확장 프로그램 설정

1. Hugging Face 계정에 로그인

    ```
    huggingface-cli login
    ```

1. 이전 목록에서 원하는 모델 다운로드

    ```
    huggingface-cli download <HF REPO NAME> <MODEL PATH> --local-dir <LOCAL PATH>
    ```

다운로드한 Qiskit Code Assistant 모델을 배포하고 상호 작용하는 방법은 여러 가지가 있습니다. 이 가이드는 [Ollama](https://ollama.com) 사용을 다음과 같이 설명합니다: Hugging Face Hub 통합 또는 로컬 모델을 사용하는 [Ollama 애플리케이션](#using-the-ollama-application), 또는 [`llama-cpp-python` 패키지](#use-llama-cpp-python)를 사용합니다.

#### Ollama 애플리케이션 사용 {#using-the-ollama-application}

Ollama 애플리케이션은 LLM을 로컬에서 실행하는 간단한 솔루션을 제공합니다. CLI를 통해 전체 설정 프로세스, 모델 관리 및 상호 작용을 상당히 간단하게 만들어 사용하기 쉽습니다. 빠른 실험과 기술적인 세부 사항을 적게 다루고 싶은 사용자에게 이상적입니다.

##### Ollama 설치 {#install-ollama}

1. [Ollama 애플리케이션](https://ollama.com/download) 다운로드
1. 다운로드한 파일 설치
1. 설치된 Ollama 애플리케이션 실행

    <Admonition type="info">
        데스크톱 메뉴 바에 Ollama 아이콘이 나타나면 애플리케이션이 성공적으로 실행 중입니다. `http://localhost:11434/`로 이동하여 서비스가 실행 중인지 확인할 수도 있습니다.
    </Admonition>

1. 터미널에서 Ollama를 사용해 보고 모델 실행을 시작합니다. 예를 들어:

    ```
    ollama run hf.co/Qiskit/Qwen2.5-Coder-14B-Qiskit
    ```

##### Hugging Face Hub 통합을 사용한 Ollama 설정 {#set-up-ollama-using-the-hugging-face-hub-integration}

[Ollama/Hugging Face Hub 통합](https://huggingface.co/docs/hub/ollama)은 새 modelfile을 만들거나 GGUF 또는 safetensors 파일을 수동으로 다운로드하지 않고도 Hugging Face Hub에 호스팅된 모델과 상호 작용하는 방법을 제공합니다. 기본 `template` 및 `params` 파일은 이미 Hugging Face Hub의 모델에 포함되어 있습니다.

1. Ollama 애플리케이션이 실행 중인지 확인합니다.

1. 원하는 모델 페이지로 이동하여 URL을 복사합니다. 예: https://huggingface.co/Qiskit/Qwen2.5-Coder-14B-Qiskit-GGUF

1. 터미널에서 다음 명령을 실행합니다:

    ```
    ollama run hf.co/Qiskit/Qwen2.5-Coder-14B-Qiskit
    ```

`hf.co/Qiskit/Qwen2.5-Coder-14B-Qiskit` 모델 또는 현재 권장되는 다른 GGUF 공식 모델인 `hf.co/Qiskit/mistral-small-3.2-24b-qiskit-GGUF` 또는 `hf.co/Qiskit/granite-3.3-8b-qiskit-GGUF`를 사용할 수 있습니다.

##### 수동으로 다운로드한 Qiskit Code Assistant GGUF 모델로 Ollama 설정 {#set-up-ollama-with-a-manually-downloaded-qiskit-code-assistant-gguf-model}

https://huggingface.co/Qiskit/Qwen2.5-Coder-14B-Qiskit-GGUF와 같은 GGUF 모델을 수동으로 다운로드했고 다른 템플릿과 매개변수로 실험하고 싶다면, 다음 단계에 따라 로컬 Ollama 애플리케이션에 로드할 수 있습니다.

1. 다음 내용을 입력하여 `Modelfile`을 만들고 `<PATH-TO-GGUF-FILE>`을 다운로드한 모델의 실제 경로로 업데이트하세요.

    ```
    FROM <PATH-TO-GGUF-FILE>
    TEMPLATE """{{ if .System }}
    System:
    {{ .System }}

    {{ end }}{{ if .Prompt }}Question:
    {{ .Prompt }}

    {{ end }}Answer:
    ```python{{ .Response }}
    """

    PARAMETER stop "Question:"
    PARAMETER stop "Answer:"
    PARAMETER stop "System:"
    PARAMETER stop "```"

    PARAMETER temperature 0
    PARAMETER top_k 1
    ```

1. Run the following command to create a custom model instance based on the `Modelfile`.

    ```
    ollama create Qwen2.5-Coder-14B-Qiskit -f ./path-to-model-file
    ```

    <Admonition type="note">
        This process may take some time for Ollama to read the model file, initialize the model instance, and configure it according to the specifications provided.
    </Admonition>

##### Run the Qiskit Code Assistant model manually downloaded in Ollama

After the `Qwen2.5-Coder-14B-Qiskit` model has been set up in Ollama, run the following command to launch the model and interact with it in the terminal (in chat mode).

```
ollama run Qwen2.5-Coder-14B-Qiskit
```

Some useful commands:

- `ollama list` - List models on your computer
- `ollama rm Qwen2.5-Coder-14B-Qiskit` - Delete the model
- `ollama show Qwen2.5-Coder-14B-Qiskit` - Show model information
- `ollama stop Qwen2.5-Coder-14B-Qiskit` - Stop a model that is currently running
- `ollama ps` - List which models are currently loaded

<AccordionItem title="Manually deploy the Qiskit Code Assistant models in local through the `llama-cpp-python` package">

<span id="use-llama-cpp-python"></span>
An alternative to the Ollama application is the `llama-cpp-python` package, which is a Python binding for `llama.cpp`. It gives you more control and flexibility to run the GGUF model locally, and is ideal for users who wish to integrate the local model in their workflows and Python applications.

1. Install [`llama-cpp-python`](https://pypi.org/project/llama-cpp-python/)
2. Interact with the model from within your application using `llama_cpp`. For example:

```python
from llama_cpp import Llama

model_path = <PATH-TO-GGUF-FILE>

model = Llama(
        model_path,
        seed=17,
        n_ctx=10000,
        n_gpu_layers=37, # to offload in gpu, but put 0 if all in cpu
    )

input = 'Generate a quantum circuit with 2 qubits'
raw_pred = model(input)["choices"][0]["text"]
```

You can also add text generation parameters to the model to customize the inference:

```python
generation_kwargs = {
        "max_tokens": 512,
        "echo": False, # Echo the prompt in the output
        "top_k": 1
    }

raw_pred = model(input, **generation_kwargs)["choices"][0]["text"]
```

</AccordionItem>

#### Use the `llama.cpp` library

Another alternative is to use `llama.cpp`, an open-source library for performing LLM inference on a CPU with minimal setup.
It provides low-level control over the model execution and is typically run from the command line, pointing to a local GGUF model file.

There are several ways to install `llama.cpp` on your machine:

- Install llama.cpp using [brew, nix, or winget](https://github.com/ggml-org/llama.cpp/blob/master/docs/install.md)
- Run with Docker: See out the [Docker documentation by `llama.cpp` team](https://github.com/ggml-org/llama.cpp/blob/master/docs/docker.md)
- Download pre-built binaries from the [releases page](https://github.com/ggml-org/llama.cpp/releases)
- Build from source by [cloning this repository](https://github.com/ggml-org/llama.cpp/blob/master/docs/build.md)

Once installed, you can use `llama.cpp` to interact with GGUF models in conversation mode as follows:
```bash
# Use a local model file
llama-cli -m my_model.gguf -cnv

# Or download and run a model directly from Hugging Face
llama-cli -hf Qiskit/Qwen2.5-Coder-14B-Qiskit-GGUF -cnv
```

You can also launch an OpenAI-compatible API server for the model in the following way:
```bash
llama-server -hf Qiskit/Qwen2.5-Coder-14B-Qiskit-GGUF
```

##### Advanced parameters
With the `llama-cli` program, you can control the model generation using command-line options. For example, you can provide an initial “system” prompt using the `-p/--prompt` flag. In conversation mode (`-cnv`), this initial prompt acts as the system message. Otherwise, you can simply prepend any desired instruction to your prompt text. You can also adjust sampling parameters - for instance: temperature (`--temp`), top-k (`--top-k`), top-p (`--top-p`), repetition penalty (`--repeat-penalty`), and the seed to use (`--seed`). The following is an example invocation using these options:

```bash
llama-cli -hf Qiskit/Qwen2.5-Coder-14B-Qiskit-GGUF \
  -p "You are a friendly assistant." -cnv \
  --temp 0.7 \
  --top-k 50 \
  --top-p 0.95 \
  --repeat-penalty 1.1 \
  --seed 42
```

Qiskit 모델의 올바른 기능을 보장하기 위해, HF GGUF 저장소에 제공된 시스템 프롬프트 사용을 권장합니다: [mistral-small-3.2-24b-qiskit-GGUF](https://huggingface.co/Qiskit/mistral-small-3.2-24b-qiskit-GGUF/blob/main/system), [Qwen2.5-Coder-14B-Qiskit-GGUF](https://huggingface.co/Qiskit/Qwen2.5-Coder-14B-Qiskit-GGUF/blob/main/system), [granite-3.3-8b-qiskit-GGUF](https://huggingface.co/Qiskit/granite-3.3-8b-qiskit-GGUF/blob/main/system), 및 [granite-3.2-8b-qiskit-GGUF](https://huggingface.co/Qiskit/granite-3.2-8b-qiskit-GGUF/blob/main/system)의 시스템 프롬프트.

</AccordionItem>

<AccordionItem title="로컬 배포에 확장 프로그램 수동 연결">

VS Code 확장 프로그램 및 JupyterLab 확장 프로그램을 사용하여 로컬에 배포된 Qiskit Code Assistant 모델에 프롬프트를 보냅니다. Ollama 애플리케이션을 [모델과 함께 설정](#using-the-ollama-application)한 후, 확장 프로그램이 로컬 서비스에 연결되도록 구성할 수 있습니다.

#### Qiskit Code Assistant VS Code 확장 프로그램으로 연결 {#connect-with-the-qiskit-code-assistant-vs-code-extension}

Qiskit Code Assistant VS Code 확장 프로그램을 사용하면 모델과 상호 작용하고 코드를 작성하는 동안 코드 완성을 수행할 수 있습니다. Python 애플리케이션용 Qiskit 코드 작성에 도움이 필요한 사용자에게 잘 맞을 수 있습니다.

1. [Qiskit Code Assistant VS Code 확장 프로그램](/guides/qiskit-code-assistant-vscode)을 설치합니다.
1. VS Code에서 **사용자 설정**으로 이동하여 **Qiskit Code Assistant: Url**을 로컬 Ollama 배포의 URL로 설정합니다 (예: `http://localhost:11434`).
1. **보기 > 명령 팔레트...**로 이동하여 **Developer: Reload Window**를 선택하여 VS Code를 다시 로드합니다.

Ollama에 구성된 Qiskit Code Assistant 모델이 상태 표시줄에 나타나고 사용할 준비가 됩니다.

#### Qiskit Code Assistant JupyterLab 확장 프로그램으로 연결 {#connect-with-the-qiskit-code-assistant-jupyterlab-extension}

Qiskit Code Assistant JupyterLab 확장 프로그램을 사용하면 모델과 상호 작용하고 Jupyter Notebook에서 직접 코드 완성을 수행할 수 있습니다. 주로 Jupyter Notebook을 사용하는 사용자는 이 확장 프로그램을 활용하여 Qiskit 코드 작성 경험을 더욱 향상시킬 수 있습니다.

1. [Qiskit Code Assistant JupyterLab 확장 프로그램](/guides/qiskit-code-assistant-jupyterlab)을 설치합니다.
1. JupyterLab에서 **설정 편집기**로 이동하여 **Qiskit Code Assistant Service API**를 로컬 Ollama 배포의 URL로 설정합니다 (예: `http://localhost:11434`).

Ollama에 구성된 Qiskit Code Assistant 모델이 상태 표시줄에 나타나고 사용할 준비가 됩니다.

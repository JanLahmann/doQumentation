# Qiskit Serverless란 무엇인가요? {#what-is-qiskit-serverless}

Qiskit Serverless는 양자-고전 리소스 전반에서 워크로드를 실행할 수 있는 간단한 인터페이스를 제공합니다. 여기에는 IBM Quantum&reg; Platform에 프로그램을 배포하고 워크로드를 원격으로 실행하는 것은 물론, 멀티 클라우드 및 양자 중심 슈퍼컴퓨팅 사용 사례를 위한 손쉬운 리소스 관리도 포함됩니다.

프리미엄 사용자는 IBM Quantum Platform을 통해 제공되는 고전 컴퓨팅 자원에서 워크로드를 구축, 배포 및 원격으로 실행할 수 있습니다.

<Admonition type="note">
    이 기능은 IBM Quantum Premium Plan, Flex Plan, On-Prem(IBM Quantum Platform API 경유) Plan 사용자에게만 제공되는 실험적 기능입니다.
</Admonition>

## Qiskit Serverless는 언제 사용해야 하나요? {#when-should-i-use-qiskit-serverless}

Qiskit Serverless는 전체 [Qiskit patterns](./intro-to-patterns) 워크플로우에 걸쳐 고전 및 양자 리소스를 관리하는 데 도움을 드립니다. 다음과 같은 예시들이 포함됩니다:

- 전처리 및 후처리와 같은 고전적 작업의 병렬 처리
- 노트북이 꺼진 경우에도 클라우드에서 장시간 실행 워크로드 유지
- 재사용 가능한 프로그램 배포

## 포함된 항목은 무엇인가요? {#what-is-included}

다음 컴퓨팅 리소스가 Qiskit Serverless 환경에 기본으로 포함되어 있습니다. 사용자는 [지원](/guides/support) 팀에 문의하여 이러한 리소스를 늘릴 수 있습니다.

- 16 vCPU 코어
- 64GB RAM
- 사용자당 동시에 활성화할 수 있는 최대 2개의 작업

## Qiskit Serverless 설치 {#install-qiskit-serverless}

IBM Quantum Platform에서 Qiskit Serverless를 사용하려면 다음 패키지를 설치하세요:

```bash
pip install qiskit-ibm-catalog qiskit_serverless
```

`qiskit_ibm_catalog`는 원격 프로그램을 업로드하고 실행하는 클라이언트 측 도구를 제공하며, `qiskit_serverless`는 컴퓨팅을 분산하고 데이터를 관리하는 서버 측 도구를 제공합니다. 이 패키지들은 `python` 3.11 이상을 필요로 합니다. 맞춤형 인프라에서 Qiskit Serverless를 실행하려는 사용자와 조직은 [클라우드 인프라 설정](https://qiskit.github.io/qiskit-serverless/) 가이드를 참고하시기 바랍니다.

<Admonition type="tip">
현재 IBM Quantum 워크로드 테이블에는 Qiskit Runtime 워크로드만 표시됩니다. Qiskit Serverless 워크로드의 현재 상태를 확인하려면 `job.status()`를 사용하세요.
</Admonition>

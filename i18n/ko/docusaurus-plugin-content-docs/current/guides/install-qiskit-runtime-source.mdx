# 소스에서 Qiskit Runtime 클라이언트 설치 {#install-the-qiskit-runtime-client-from-source}

소스에서 Qiskit Runtime을 설치하면 Python Package Index(PyPI) 저장소의 버전 대신 현재 개발 버전에 접근할 수 있습니다. 따라서 최신 버전의 코드를 더 효율적으로 검사하고 확장할 수 있습니다.

## 새 가상 환경 생성 및 활성화 {#create-and-activate-a-new-virtual-environment}

1. 프로젝트 디렉토리로 이동하고 Python만 설치된 최소 환경을 생성하세요.

        <OperatingSystemTabs>
          <TabItem value="mac" label="macOS">
            ```shell
            python3 -m venv .venv
            ```
          </TabItem>

          <TabItem value="linux" label="Linux">
            ```shell
            python3 -m venv .venv
            ```
          </TabItem>

          <TabItem value="win" label="Windows">
            ```text
            python -m venv .venv
            ```
          </TabItem>
         </OperatingSystemTabs>

1. 새 환경을 활성화하세요.

         <OperatingSystemTabs>
           <TabItem value="mac" label="macOS">
             ```shell
             source .venv/bin/activate
             ```
           </TabItem>

           <TabItem value="linux" label="Linux">
             ```shell
             source .venv/bin/activate
             ```
           </TabItem>

           <TabItem value="win" label="Windows">

           </TabItem>
         </OperatingSystemTabs>

## Qiskit Runtime 설치 {#install-qiskit-runtime}

다음 단계에 따라 Qiskit Runtime을 설치하세요:

1. Qiskit Runtime 저장소를 클론합니다.

```bash
git clone https://github.com/Qiskit/qiskit-ibm-runtime.git
```

2. `qiskit-ibm-runtime` 디렉토리로 이동합니다.

```bash
cd qiskit-ibm-runtime
```

3. `qiskit-runtime`을 설치합니다. 글로벌 Python 설치를 오염시키지 않으려면 [가상 환경](https://docs.python.org/3/library/venv.html)을 사용하는 것을 권장합니다.

  * **표준 설치**:

    ```bash
    pip install .
    ```

  * **편집 가능 모드**: 이 모드에서는 프로젝트에 코드 변경이 있을 때 Qiskit을 다시 설치할 필요가 없습니다.

    ```bash
    pip install -e .
    ```

    편집 가능 모드에서는 컴파일된 확장이 최적화 없이 _디버그 모드_로 빌드됩니다.

4. (선택 사항) 테스트 또는 린팅 검사를 실행하려면 개발자 요구 사항을 설치하세요. 글로벌 Python 설치를 오염시키지 않으려면 [가상 환경](https://docs.python.org/3/library/venv.html)을 사용하는 것을 권장합니다.

```bash
pip install -e ".[dev]"
```

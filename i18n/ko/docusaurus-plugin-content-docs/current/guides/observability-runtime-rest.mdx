# Qiskit Runtime의 활동 추적 이벤트 {#activity-tracking-events-for-qiskit-runtime}

Qiskit Runtime과 같은 IBM Cloud&reg; 서비스는 활동 추적 이벤트를 생성합니다.

활동 추적 이벤트는 IBM Cloud에서 서비스의 상태를 변경하는 활동을 보고합니다. 이 이벤트를 사용하여 비정상적인 활동 및 중요한 작업을 조사하고, 규제 감사 요구 사항을 준수할 수 있습니다.

IBM Cloud Activity Tracker Event Routing(플랫폼 서비스)을 사용하면 대상 및 경로를 구성하여 계정의 감사 이벤트를 원하는 목적지로 라우팅할 수 있습니다. 자세한 내용은 [IBM Cloud Activity Tracker Events Routing 정보](https://cloud.ibm.com/docs/atracker?topic=atracker-about)를 참조하세요.

IBM Cloud Logs를 사용하면 계정에서 생성되고 IBM Cloud Activity Tracker Event Routing을 통해 IBM Cloud Logs 인스턴스로 라우팅된 이벤트를 시각화하고 알림을 설정할 수 있습니다.

## 활동 추적 이벤트가 생성되는 위치 {#locations-where-activity-tracking-events-are-generated}

Qiskit Runtime은 다음 지역에서 활동 추적 이벤트를 전송합니다:

- 워싱턴 ("us-east")
- 프랑크푸르트 ("eu-de")

## Qiskit Runtime의 활동 추적 이벤트 보기 {#viewing-activity-tracking-events-for-qiskit-runtime}

IBM Cloud Logs를 사용하면 계정에서 생성되고 IBM Cloud Activity Tracker Event Routing을 통해 IBM Cloud Logs 인스턴스로 라우팅된 이벤트를 시각화하고 알림을 설정할 수 있습니다.

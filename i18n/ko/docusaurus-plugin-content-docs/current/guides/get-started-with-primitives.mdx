## Estimator 시작하기 {#get-started-with-estimator}

{/*Verified the v2 examples 2/29/24 - updated 10/29/24*/}

### 1. 계정 초기화 {#initialize-the-account}

Qiskit Runtime Estimator는 관리형 서비스이므로, 먼저 계정을 초기화해야 합니다. 그런 다음 기댓값을 계산하는 데 사용할 QPU를 선택할 수 있습니다.

아직 계정이 없다면 [설치 및 설정 항목](install-qiskit)의 단계를 따르세요.

```python
from qiskit_ibm_runtime import QiskitRuntimeService

service = QiskitRuntimeService()
backend = service.least_busy(
    operational=True, simulator=False, min_num_qubits=127
)

print(backend.name)
```

```text
ibm_torino
```

### 2. Circuit 및 관측 가능량 생성 {#create-a-circuit-and-an-observable}

Estimator 프리미티브의 입력으로 Circuit과 관측 가능량이 각각 하나 이상 필요합니다.

```python
from qiskit.circuit.library import qaoa_ansatz
from qiskit.quantum_info import SparsePauliOp

entanglement = [tuple(edge) for edge in backend.coupling_map.get_edges()]
observable = SparsePauliOp.from_sparse_list(
    [("ZZ", [i, j], 0.5) for i, j in entanglement],
    num_qubits=backend.num_qubits,
)
circuit = qaoa_ansatz(observable, reps=2)
# the circuit is parametrized, so we will define the parameter values for execution
param_values = [0.1, 0.2, 0.3, 0.4]

print(f">>> Observable: {observable.paulis}")
```

Circuit과 관측 가능량은 QPU가 지원하는 명령어만 사용하도록 변환되어야 합니다(이를 *명령어 집합 아키텍처 (ISA)* Circuit이라고 합니다). 이를 위해 Transpiler를 사용합니다.

```python
from qiskit.transpiler import generate_preset_pass_manager

pm = generate_preset_pass_manager(optimization_level=1, backend=backend)
isa_circuit = pm.run(circuit)
isa_observable = observable.apply_layout(isa_circuit.layout)
print(f">>> Circuit ops (ISA): {isa_circuit.count_ops()}")
```

```text

```

### 3. Qiskit Runtime Estimator 초기화 {#initialize-qiskit-runtime-estimator}

Estimator를 초기화할 때, `mode` 파라미터를 사용하여 실행할 모드를 지정하세요. 가능한 값은 각각 배치, 세션, 작업 실행 모드에 해당하는 `batch`, `session`, 또는 `backend` 객체입니다. 자세한 내용은 [Qiskit Runtime 실행 모드 소개](execution-modes)를 참조하세요.

```python
from qiskit_ibm_runtime import EstimatorV2 as Estimator

estimator = Estimator(mode=backend)
```

### 4. Estimator 호출 및 결과 가져오기 {#invoke-the-estimator-and-get-results}

다음으로, `run()` 메서드를 호출하여 입력 Circuit과 관측 가능량에 대한 기댓값을 계산합니다. Circuit, 관측 가능량, 그리고 선택적인 파라미터 값 집합은 *primitive unified bloc* (PUB) 튜플로 입력됩니다.

```python
job = estimator.run([(isa_circuit, isa_observable, param_values)])
print(f">>> Job ID: {job.job_id()}")
print(f">>> Job Status: {job.status()}")
```

result = job.result()

## Sampler 시작하기 {#get-started-with-sampler}

### 1. 계정 초기화 {#initialize-the-account-sampler}

Qiskit Runtime Sampler는 관리형 서비스이므로, 먼저 계정을 초기화해야 합니다. 그런 다음 기댓값을 계산하는 데 사용할 QPU를 선택할 수 있습니다.

아직 계정이 설정되어 있지 않다면 [설치 및 설정 항목](install-qiskit)의 단계를 따르세요.

```python
from qiskit_ibm_runtime import QiskitRuntimeService

service = QiskitRuntimeService()
backend = service.least_busy(
    operational=True, simulator=False, min_num_qubits=127
)
```

### 2. Circuit 생성 {#create-a-circuit}

Sampler 프리미티브의 입력으로 Circuit이 하나 이상 필요합니다.

```python
import numpy as np
from qiskit.circuit.library import efficient_su2

circuit = efficient_su2(127, entanglement="linear")
circuit.measure_all()
# The circuit is parametrized, so we will define the parameter values for execution
param_values = np.random.rand(circuit.num_parameters)
```

Transpiler를 사용하여 ISA Circuit을 가져옵니다.

```python
from qiskit.transpiler import generate_preset_pass_manager

pm = generate_preset_pass_manager(optimization_level=1, backend=backend)
isa_circuit = pm.run(circuit)
print(f">>> Circuit ops (ISA): {isa_circuit.count_ops()}")
```

```text
>>> Circuit ops (ISA): OrderedDict([('sx', 3089), ('rz', 3036), ('cz', 1092), ('measure', 127), ('barrier', 1)])
```

### 3. Qiskit Runtime Sampler 초기화 {#initialize-the-qiskit-runtime-sampler}

Sampler를 초기화할 때, `mode` 파라미터를 사용하여 실행할 모드를 지정하세요. 가능한 값은 각각 배치, 세션, 작업 실행 모드에 해당하는 `batch`, `session`, 또는 `backend` 객체입니다. 자세한 내용은 [Qiskit Runtime 실행 모드 소개](execution-modes)를 참조하세요. Open Plan 사용자는 세션 작업을 제출할 수 없습니다.

```python
from qiskit_ibm_runtime import SamplerV2 as Sampler

sampler = Sampler(mode=backend)
```

### 4. Sampler 호출 및 결과 가져오기 {#invoke-the-sampler-and-get-results}

다음으로, `run()` 메서드를 호출하여 출력을 생성합니다. Circuit과 선택적인 파라미터 값 집합은 *primitive unified bloc* (PUB) 튜플로 입력됩니다.

```python
job = sampler.run([(isa_circuit, param_values)])

print(f">>> Job Status: {job.status()}")
```

## Backend 프리미티브 시작하기 {#get-started-with-the-backend-primitives}

프로바이더별 프리미티브와 달리, Backend 프리미티브는 [`Backend`](https://docs.quantum.ibm.com/api/qiskit/qiskit.providers.Backend) 인터페이스를 구현하는 임의의 `backend` 객체와 함께 사용할 수 있는 범용 구현입니다.

- Sampler 프리미티브는 [`qiskit.primitives.BackendSamplerV2`](https://docs.quantum.ibm.com/api/qiskit/qiskit.primitives.BackendSamplerV2)를 사용하여 모든 프로바이더에서 실행할 수 있습니다.
- Estimator 프리미티브는 [`qiskit.primitives.BackendEstimatorV2`](../api/qiskit/qiskit.primitives.BackendEstimatorV2)를 사용하여 모든 프로바이더에서 실행할 수 있습니다.

일부 프로바이더는 프리미티브를 네이티브로 구현합니다. 자세한 내용은 [Qiskit 에코시스템 페이지](https://qiskit.github.io/ecosystem#provider)를 참조하세요.
### 예시: BackendEstimator {#example-backendestimator}

```python
from qiskit.primitives import BackendEstimatorV2
from <some_qiskit_provider> import QiskitProvider

provider = QiskitProvider()
backend = provider.get_backend('backend_name')
estimator = BackendEstimatorV2(backend)
```

### 예시: BackendSampler {#example-backendsampler}

```python
from qiskit.primitives import BackendSamplerV2
from <some_qiskit_provider> import QiskitProvider

provider = QiskitProvider()
backend = provider.get_backend('backend_name')
sampler = BackendSamplerV2(backend)
```
### Backend 프리미티브와 Runtime 프리미티브의 유사점 및 차이점 {#similarities-and-differences-between-backend-and-runtime-primitives}

- [`qiskit.primitives.BackendSamplerV2`](https://docs.quantum.ibm.com/api/qiskit/qiskit.primitives.BackendSamplerV2) 및 [`qiskit.primitives.BackendEstimatorV2`](../api/qiskit/qiskit.primitives.BackendEstimatorV2)의 입력과 출력은 Qiskit Runtime의 프리미티브와 동일한 PUB 형식을 따릅니다. 자세한 내용은 [프리미티브 입력 및 출력](primitive-input-output)을 참조하세요.
단, 반환되는 메타데이터의 필드에는 차이가 있을 수 있습니다.

- [`qiskit.primitives.BackendEstimatorV2`](https://docs.quantum.ibm.com/api/qiskit/qiskit.primitives.BackendEstimatorV2) 클래스는 기본적으로 측정 또는 Gate 오류 완화 구현을 제공하지 않습니다. 이는 Backend 프리미티브가 사용자의 로컬 머신에서 실행되도록 설계되었기 때문입니다.

- [`qiskit.primitives.BackendSamplerV2`](https://docs.quantum.ibm.com/api/qiskit/qiskit.primitives.BackendSamplerV2) 클래스는 `memory` 옵션을 지원하는 Backend가 필요합니다.

- Backend 프리미티브 인터페이스는 Runtime 구현과 다른 커스텀 [`SamplerV2`](https://docs.quantum.ibm.com/api/qiskit/qiskit.primitives.BackendSamplerV2) 및 [`EstimatorV2`](https://docs.quantum.ibm.com/api/qiskit/qiskit.primitives.BackendEstimatorV2) `Options`를 제공합니다.

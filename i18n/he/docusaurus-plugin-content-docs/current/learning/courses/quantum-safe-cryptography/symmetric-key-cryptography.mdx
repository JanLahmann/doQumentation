## מבוא לקריפטוגרפיית מפתח סימטרי {#introduction-to-symmetric-key-cryptography}

*קריפטוגרפיית מפתח סימטרי (SKC)* היא הצורה הוותיקה והאינטואיטיבית ביותר של קריפטוגרפיה. עם SKC, מידע סודי מאובטח באמצעות הצפנת מפתח סימטרי (SKE), כלומר, שימוש ב<DefinitionTooltip definition="אותו מפתח משמש הן להצפנה והן לפענוח נתונים. בשיטה זו השולח והמקבל חולקים מפתח סודי המשמש להמרת טקסט ברור (נתונים מקוריים) לטקסט מוצפן (ciphertext) ולהיפך.">*מפתח סודי יחיד*</DefinitionTooltip> הן להצפנה והן לפענוח.

SKC כוללת:
- פונקציית הצפנה הממירה מופע טקסט ברור נתון ל-<DefinitionTooltip definition="התוצאה של יישום אלגוריתם הצפנה על הטקסט הברור באמצעות מפתח הצפנה ספציפי; יש צורך במפתח פענוח הנכון כדי להמיר חזרה את ה-ciphertext לטקסט ברור.">ciphertext</DefinitionTooltip> תוך שימוש במפתח סודי
- פונקציית פענוח ההופכת את הפעולה על ידי המרת ה-ciphertext חזרה לטקסט ברור באמצעות אותו מפתח סודי

טקסט ברור יכול להתכוון לכל סוג של נתונים לא מוצפנים כגון טקסט שפה טבעית או קוד בינארי שתוכן המידע שלו נגיש באופן עקרוני ישירות, ואילו <DefinitionTooltip definition="התוצאה של יישום אלגוריתם הצפנה על הטקסט הברור באמצעות מפתח הצפנה ספציפי; יש צורך במפתח פענוח הנכון כדי להמיר חזרה את ה-ciphertext לטקסט ברור.">ciphertext</DefinitionTooltip> מתייחס לנתונים מוצפנים שתוכן המידע שלהם אמור להיות בלתי נגיש לפני הפענוח.

אלגוריתם המתאר את פעולות ההצפנה והפענוח באמצעות מפתח סודי משותף נקרא גם צופן סימטרי.

## תכונות של מערכות קריפטוגרפיות מפתח סימטרי {#properties-of-symmetric-key-cryptosystems}

מערכת קריפטוגרפית מפתח סימטרי צריכה להבטיח את התכונות הבאות לאבטחת הודעות — הן נתונים המאוחסנים בשקיפה והן תקשורת על פני ערוץ שידור כלשהו:

- **סודיות:** מתייחסת לתכונה לפיה תוכן המידע של הודעות מוצפנות מוגן מפני גישה בלתי מורשית.
- **שלמות:** מתייחסת לתכונה לפיה כל חבלה בהודעות מוצפנות במהלך אחסון או שידור ניתנת לזיהוי.
- **אותנטיות:** מתייחסת לתכונה לפיה מקבל הודעה יכול לאמת את זהות השולח ולזהות התחזות מצד גורם בלתי מורשה.

בנוסף, תכונות אלה צריכות להתממש בסביבה שבה האלגוריתמים או הצופנים המשמשים להצפנה ולפענוח עשויים להיות ציבוריים ושבה הגישה לתוכן המידע של הודעות מוצפנות נשלטת אך ורק דרך גישה למפתח הסודי.

יישום מערכת קריפטוגרפית מפתח סימטרי מאובטחת כולל לכן שתי משימות עיקריות:

1. שימוש באלגוריתם הצפנת מפתח סימטרי חזק העמיד בפני התקפות קריפטוגרפיות.
2. הבטחת סודיות בהפצה ובניהול המפתחות הסודיים.

בשיעור זה נדון בהיבטים הקשורים למשימה הראשונה, המהווה את הדאגה העיקרית של טכנולוגיית SKC. המשימה השנייה, עם זאת, דורשת פתרונות שאינם בתחום SKC עצמה ויוצגו בהמשך.

### צופן ההזזה של קיסר: {#caesar-shift-cipher}

הצפנת ההזזה של קיסר כוללת הגדרת
 - אלפבית של תווים אפשריים לקידוד
 - *ערך הזזה* שיכול להיות בין 0 (לא מוצפן) לאורך האלפבית. אנו מתייחסים לכך כ*מפתח*.

הוא ידוע כ*צופן החלפה חד-אלפביתי* מכיוון שכל אות מהטקסט הברור מוחלפת באחרת ב-ciphertext.

בדוגמה זו נשתמש באותיות קטנות של האלפבית.

נתחיל בהגדרת הדברים.

```python
# initialize the required python object for doing Caesar shift encryption
caesar_cipher = Caesar()

# Define the shift, ie the key
caesar_key = 5
print(f"Caesar shift secret key: {caesar_key}")

# Define the alphabet
alphabet = (
    "a",
    "b",
    "c",
    "d",
    "e",
    "f",
    "g",
    "h",
    "i",
    "j",
    "k",
    "l",
    "m",
    "n",
    "o",
    "p",
    "q",
    "r",
    "s",
    "t",
    "u",
    "v",
    "w",
    "x",
    "y",
    "z",
    " ",
)
print(f"alphabet: {alphabet}")
```

נצפין את הטקסט הברור כדי לקבל ciphertext עבור צופן קיסר.

```python
caeser_ciphertext = caesar_cipher.encrypt(plaintext, caesar_key, alphabet)
print(f"Encrypted caeser shift ciphertext: {caeser_ciphertext}")
```

נפענח את ה-ciphertext חזרה לטקסט הברור המקורי באמצעות אותו מפתח שהשתמשנו בו להצפנה.

```python
caeser_plaintext = caesar_cipher.decrypt(caeser_ciphertext, caesar_key, alphabet)
print(f"Decrypted caeser shift plaintext: {caeser_plaintext}\n")
```

### צופן תקן ההצפנה המתקדם (AES) {#advanced-encryption-standard-aes-cipher}

כעת נצפין את הטקסט הברור באמצעות AES, אלגוריתם הצפנת מפתח סימטרי פופולרי.

נתחיל ביצירת המפתח, במקרה זה, מחרוזת של 16 תווים אקראיים.

```python
# lambda defines an inline function in this case that takes two values a,b with the resulting expression of a+b
# reduce uses a two-argument function(above), and applies this to all the entries in the list (random alphabet characters) cumulatively
aes_key = reduce(lambda a, b: a + b, [np.random.choice(alphabet) for i in range(16)])

print(f"AES secret key: {aes_key}")
```

AES תומך במספר מצבי פעולה ומחייב אותנו לציין איזה להשתמש.

אנו בוחרים במצב *Cipher Block Chaining* (CBC) המסופק על ידי מחלקה `modes.CBC` של ספריית `cryptography`. מצב CBC של AES משתמש באקראיות לאבטחה נוספת. זה מחייב ציון *וקטור אתחול* אקראי (IV), הנקרא גם <DefinitionTooltip definition="מספר המשמש פעם אחת.">*nonce*</DefinitionTooltip>. נשתמש גם כאן במחרוזת אקראית, בדיוק כפי שעשינו עבור המפתח.

```python
aes_initialization_vector = reduce(
    lambda a, b: a + b, [np.random.choice(alphabet) for i in range(16)]
)
print(f"AES initialization vector: {aes_initialization_vector}")
```

כעת נוכל ליצור מופע של צופן AES בשם שולח ההודעה הסודית. שים לב שוקטור האתחול מועבר למחלקה `modes.CBC` להגדרת מצב הפעולה CBC.

לאחר מכן נצפין את הטקסט הברור לשליחה.

```python
# The encryptor is setup using the key and CBC. In both cases we need to convert the string (utf-8) into bytes
sender_aes_cipher = Cipher(
    algorithms.AES(bytes(aes_key, "utf-8")),
    modes.CBC(bytes(aes_initialization_vector, "utf-8")),
)
aes_encryptor = sender_aes_cipher.encryptor()

# update can add text to encypt in chunks, and then finalize is needed to complete the encryption process
aes_ciphertext = (
    aes_encryptor.update(bytes(plaintext, "utf-8")) + aes_encryptor.finalize()
)

# Note the output is a string of bytes
print(f"Encrypted AES ciphertext: {aes_ciphertext}")
```

כדי לפענח אותו, ניצור מופע של צופן AES בשם המקבל. שים לב שלמקבל המיועד יש גישה הן למפתח הסודי והן לוקטור האתחול, אך האחרון אינו נדרש להיות סודי.

```python
# Similar setup of AES to what we did for encryption, but this time, for decryption
receiver_aes_cipher = Cipher(
    algorithms.AES(bytes(aes_key, "utf-8")),
    modes.CBC(bytes(aes_initialization_vector, "utf-8")),
)
aes_decryptor = receiver_aes_cipher.decryptor()

# Do the decryption
aes_plaintext_bytes = aes_decryptor.update(aes_ciphertext) + aes_decryptor.finalize()

# convert back to a character string (we assume utf-8)
aes_plaintext = aes_plaintext_bytes.decode("utf-8")

print(f"Decrypted AES plaintext: {aes_plaintext}")
```

## יישומים של קריפטוגרפיית מפתח סימטרי {#applications-of-symmetric-key-cryptography}

בעוד שצופנים קלאסיים כגון <DefinitionTooltip definition="צופן ההזזה של קיסר הוא שיטת הצפנה פשוטה בשימוש מאז ימי רומא, או קודם לכן. כל אות מוחלפת על ידי אחרת, זזה מספר אותיות. זהו סוג של צופן החלפה חד-אלפביתי.">צופן קיסר</DefinitionTooltip> יצאו מכלל שימוש מזמן, מערכות קריפטוגרפיות סימטריות מודרניות כגון AES [פרוסות במגוון רחב של יישומים](https://books.google.com/books?id=01vfjgEACAAJ), כולל:

1. **הצפנה ופענוח של נתונים:** SKC נמצאת בשימוש נרחב להגנה על נתונים רגישים, בין אם מאוחסנים בשקיפה על מכשיר ובין אם מועברים על פני רשת. דוגמאות כוללות הגנה על אישורי משתמשים, הצפנת הודעות דואר אלקטרוני ואבטחת עסקאות פיננסיות, בין היתר.

2. **תקשורת מאובטחת:** פרוטוקולי תקשורת נפוצים כגון SSL/[TLS](https://en.wikipedia.org/w/index.php?title=Transport_Layer_Security&oldid=1155601531) משתמשים בשילוב של הצפנת מפתח סימטרי וא-סימטרי כדי להבטיח את הסודיות והשלמות של הנתונים המוחלפים בין שני צדדים. הודעות אלו מוצפנות ומפוענחות באמצעות הצפנת מפתח סימטרי הבשתמשת במפתח משותף. המפתח המשמש בהצפנת מפתח סימטרי מוחלף בצורה מאובטחת באמצעות הצפנת מפתח א-סימטרי המשתמשת בזוג מפתחות ציבורי-פרטי. הצפנת מפתח סימטרי מהירה הרבה יותר ולכן ניתן להשתמש בה להצפנת הודעות בגודל גדול.

3. **אימות אותנטיות:** בסביבות מסוימות, SKC מיושמת באמצעות טכניקות כגון [קודי אימות הודעות](https://en.wikipedia.org/w/index.php?title=Message_authentication_code&oldid=1151014755) (MACs) וקודי MAC עם גיבוב ממוין (HMAC) כדי לאמת את <DefinitionTooltip definition="אותנטיות היא ההוכחה שהודעה מגיעה מהמקור שהיא טוענת לגביו.">האותנטיות</DefinitionTooltip> ו<DefinitionTooltip definition="שלמות נשמרת כאשר הודעה מוגנת מפני שינוי בלתי מורשה.">השלמות</DefinitionTooltip> של הודעות, ומבטיחה תקשורת עמידה בפני חבלה.

4. **הצפנת קבצים ודיסקים:** תוכנות הצפנת דיסק מלא וכלי הצפנת קבצים משתמשים ב-SKC להגנה על נתונים רגישים המאוחסנים על דיסקים קשיחים או התקני אחסון ניידים.

5. **רשתות פרטיות וירטואליות:** טכנולוגיות [VPN](https://en.wikipedia.org/w/index.php?title=Virtual_private_network&oldid=1154342615), שנועדו לספק ערוצי תקשורת סודיים ללא ציתות, יכולות להשתמש בהצפנת מפתח סימטרי או א-סימטרי לחיבור משתמשים מרוחקים וכן רשתות ארגוניות.

מגוון היישומים הרחב שבהם SKC פרוסה מחייב בתורה שמערכות קריפטוגרפיות סימטריות יעמדו בסט מסוים של קריטריונים.
## עקרונות הצפנת מפתח סימטרי {#principles-of-symmetric-key-encryption}
בחלק זה נדון בכמה מה[עקרונות הבסיסיים הבסיסיים העומדים בבסיס אבטחת אלגוריתמי הצפנת מפתח סימטרי](https://doi.org/10.1080/07468342.1987.11973000).

**עמידות בפני התקפת כוח גס:** הדרישה הבסיסית ביותר לאבטחה של צופן הצפנה היא שגודל מרחב המפתחות — במילים אחרות, מספר המפתחות הנבדלים האפשריים שמשתמש באלגוריתם יכל לבחור מהם — הוא גדול מאוד.

**עמידות בפני התקפת קריפטניליזה:** הדרישה הבסיסית השנייה לצופן, סימטרי או אחר, היא שהוא יכול לייצר ciphertexts ש<DefinitionTooltip definition="התכונה של הנתונים המוצפנים להיות קשים ביותר להבנה, אם לא בלתי אפשריים מעשית, ללא מפתח הפענוח הנכון.">בלתי ניתנים לפענוח</DefinitionTooltip> מבחינת מידע. לשם כך, תנאי הכרחי אך לא מספיק מנקודת מבט תיאורית מידע הוא שה-[ciphertexts צריכים להיות מאופיינים על ידי אנטרופיה גבוהה](https://en.wikipedia.org/w/index.php?title=Entropy_(information_theory)&oldid=1154931420), מה שהופך אותם לבלתי ניתנים להבחנה מטקסט אקראי ללא דפוסים או קורלציות ניתנים לזיהוי. בכך, תוקף לא יכול לקבל מידע על הטקסט הברור או המפתח הסודי על ידי ניסיון לנתח את ה-ciphertext באמצעות ניתוח תדרים או טכניקות סטטיסטיות אחרות.

עמידות בפני צורות כלליות של התקפות קריפטאנליזה מספיקה להבטחת [אבטחה סמנטית](https://en.wikipedia.org/w/index.php?title=Semantic_security&oldid=1133491450) מפורשת דרך המושג [*אי-הבחנה*](https://en.wikipedia.org/w/index.php?title=Ciphertext_indistinguishability&oldid=1130147047). בעוד שישנן מספר גרסאות של אי-הבחנה עם דרישות שונות, מערכת קריפטוגרפית סימטרית נחשבת מאובטחת סמנטית אם היא מספקת את הקריטריון של [*אי-הבחנה תחת התקפת טקסט ברור מוכר*](https://en.wikipedia.org/w/index.php?title=Ciphertext_indistinguishability&oldid=1130147047) (IND-CPA). משמעות הדבר היא שתוקף לא יכול להבחין בין ההצפנות של שתי הודעות שונות גם אם מורשה לשלוח טקסטות ברורות מרובות לבחירתו לאלגוריתם ולראות את ה-ciphertexts המתאימים.

כפי שנשקול בהמשך, IND-CPA מחייב בדרך כלל שימוש באקראיות כדי להבטיח שבכל פעם שטקסט ברור נתון מוצפן עם מפתח סודי נתון, ה-ciphertext המתקבל יהיה שונה בצורה בלתי צפויה עבור כל הצפנה.

**מצבי כשל של צופנים קלאסיים:** לפני הופעת הקריפטוגרפיה המודרנית בשנות ה-70, רוב הצופנים הקלאסיים בשימוש מעשי לא עמדו באחת משתי הדרישות לעיל או בשתיהן. לדוגמה, צופני החלפה מוקדמים כגון [צופן ההזזה החד-אלפביתי של קיסר](https://en.wikipedia.org/w/index.php?title=Caesar_cipher&oldid=1151194063) אופיינו הן בגודל מרחב מפתחות קטן (ראה <a href="#Table 1">טבלה 1</a>) והן ב-ciphertext בעל אנטרופיה נמוכה, מה שהפך אותם לא מאובטחים בפני מגוון התקפות קריפטאנליטיות כגון התקפות כוח גס, ניתוח תדרים והתקפות טקסט ברור מוכר (KPT).

צופני החלפה רב-אלפביתיים מאוחרים יותר כגון [צופן Vigenère](https://en.wikipedia.org/w/index.php?title=Vigen%C3%A8re_cipher&oldid=1155012941) ו[צופן מכונת האניגמה](https://en.wikipedia.org/w/index.php?title=Enigma_machine&oldid=1154440070) הציגו גדלי מרחב מפתחות גדולים למעשה, מה שהפך אותם לעמידים בפני התקפות כוח גס, אך היו פגיעים לניתוח תדרים ולהתקפות KPT, בהתאמה. בדומה לצופני ההחלפה, [צופני ערבוב קלאסיים](https://en.wikipedia.org/w/index.php?title=Transposition_cipher&oldid=1151499771), המסדרים מחדש אותיות בהודעה במקום להחליף אותן, נפגעים גם הם על ידי מגוון התקפות כגון אנאגרמינג, ניתוח סטטיסטי, כוח גס והתקפות KPT, בין היתר.

תיאורטית, צופן החלפה רב-אלפביתי המכונה [one-time pad](https://en.wikipedia.org/w/index.php?title=One-time_pad&oldid=1155039884) (OTP) ידוע כמאובטח קריפטוגרפית. OTP מציג מפתח סודי שצריך להיות (1) מורכב מאותיות או ביטים שנבחרו באקראי, (2) ארוך לפחות כמו הטקסט הברור המקורי, ו-(3) בשימוש פעם אחת בלבד. OTP אינו מעשי ליישומים בפועל מכיוון שאם ניתן היה לשתף בצורה מאובטחת את המפתח הסודי — הנדרש להיות ארוך כמו הטקסט הברור וניתן לשימוש פעם אחת בלבד — אז ניתן היה לשתף גם את הטקסט הברור המקורי. OTP במקום זאת ממחיש את השימוש באקראיות ביצירת ciphertexts מאובטחים.

תוקף המנסה לבצע חיפוש כוח גס דרך מרחב המפתחות כדי למצוא מפתח שמפענח את ההודעה חייב לבצע מספר פעולות פרופורציונלי לגודל מרחב המפתחות.

לכן, מרחב מפתחות גדול מספק עמידות בפני התקפות כוח גס על ידי הפיכתן לחיסרון חישובי. <a href="#Table 1">טבלה 1</a> מפרטת את גדלי מרחב המפתחות של כמה צופנים ידועים.
### טבלה 1: גדלי מרחב מפתחות של כמה צופנים סימטריים {#table-1-key-space-sizes-of-some-symmetric-ciphers}

|--------------|------------------|-------------------------------------------|
| זזי קיסר | 1                | גודל-האלפבית                             |
| Vigenere     | n                | גודל-האלפבית$^\mathrm{n}$                |
| One-time-pad | אורך-הטקסט-הברור | גודל-האלפבית$^\mathrm{אורך-הטקסט-הברור}$ |
| DES          | 56               | 2$^\mathrm{56}$                           |
| AES-128      | 128              | 2$^\mathrm{128}$                          |
| AES-192      | 192              | 2$^\mathrm{192}$                          |
| AES-256      | 256              | 2$^\mathrm{256}$                          |
| ChaCha20     | 256              | 2$^\mathrm{256}$                          |
שיטות הצפנת מפתח סימטרי מודרניות מתגברות במידה רבה על מגבלות הצופנים הקלאסיים. הן מייצרות ciphertext עמיד בפני <DefinitionTooltip definition="ניתוח מערכות קריפטוגרפיות לזיהוי חולשותיהן, פגיעויותיהן או מידע מוסתר ללא גישה למפתחות הסודיים; המטרה הסופית היא לעקוף את אמצעי האבטחה שהמערכת מיישמת. קריפטאנליזה לינארית שואפת למצוא קירוב לפונקציית הצפנה, בעוד שקריפטאנליזה דיפרנציאלית עוקבת אחר רשת של טרנספורמציות שונות.">קריפטאנליזה</DefinitionTooltip> ומציגות גדלי מרחב מפתחות גדולים תוך שהן יעילות הרבה יותר מבחינה מעשית מ<DefinitionTooltip definition="One-time pad. המפתח ארוך יותר מההודעה לשליחה, כאשר כל תו בטקסט הברור מחובר לערך המפתח.">OTP</DefinitionTooltip>.

**צופני בלוק**: מחלקה אחת של צופנים מודרניים — כגון [DES](https://en.wikipedia.org/w/index.php?title=Data_Encryption_Standard&oldid=1154751166) ו-[AES](https://www.nist.gov/publications/development-advanced-encryption-standard) — משיגה אבטחה על ידי שילוב עקרונות הבלבול והפיזור [שהוצגו במקור על ידי קלוד שאנון](https://en.wikipedia.org/wiki/A_Mathematical_Theory_of_Communication#:~:text=%22A%20Mathematical%20Theory%20of%20Communication,the%20generality%20of%20this%20work). נדון במושגים אלה בהגדרה שבה שיטות הצפנה עובדות עם ייצוגים בינאריים של הודעות:

- **בלבול**: בלבול הוא המאפיין לפיו כל ביט ב-ciphertext תלוי בביטים מרובים של המפתח הסודי. הוא מבטיח שגם שינוי קטן במפתח הסודי ישנה כמעט את כל הביטים של ה-ciphertext, ומטשטש את הקשר בין ה-ciphertext למפתח הסודי.

- **פיזור**: פיזור הוא המאפיין לפיו היפוך ביט אחד בטקסט הברור צריך לשנות בערך חצי מהביטים ב-ciphertext ולהיפך. הפיזור מסתיר קשרים סטטיסטיים בין הטקסט הברור ל-ciphertext. לצופנים עם פיזור מספיק מתממש מה שנקרא <DefinitionTooltip definition="תהליך של פיזור השפעת אלמנטים בודדים של הטקסט הברור על חלק גדול של ה-ciphertext במהלך ההצפנה. מטרת הפיזור היא להבטיח שגם שינוי בביט אחד של הטקסט הברור יגרום לשינויים בביטים מרובים ב-ciphertext.">*קריטריון המפולת*</DefinitionTooltip> של הקריפטוגרפיה.

צופני בלוק מיישמים בלבול ופיזור באמצעות מבנים קריפטוגרפיים הידועים כ[רשתות החלפה-תמורה](https://en.wikipedia.org/w/index.php?title=Substitution%E2%80%93permutation_network&oldid=1098155951) (SPNs) הפועלות על בלוקים דיסקרטיים של נתונים. <DefinitionTooltip definition="רשת החלפה-תמורה. סוג של צופן בלוק שבו ביטים א) מוחלפים ו-ב) מעורבבים מספר איטרציות, שבהן גם המפתח עצמו עובר טרנספורמציה.">SPN</DefinitionTooltip> מקבל בלוק טקסט ברור ואת המפתח הסודי כקלט ומבצע מספר מוגדר של [*סיבובים*](https://en.wikipedia.org/w/index.php?title=Round_(cryptography)&oldid=1149554107) של טרנספורמציות כדי לייצר בלוק ciphertext. כל סיבוב מורכב ממבנים מתמטיים מתחלפים הידועים כתיבות החלפה [(S-boxes)](https://en.wikipedia.org/w/index.php?title=S-box&oldid=1154458964) ותיבות תמורה [(P-boxes)](https://en.wikipedia.org/w/index.php?title=Permutation_box&oldid=1151700196) או פעולות שוות ערך.

אלה מיישמות בהתאמה טרנספורמציות לא-לינאריות מורכבות ולינאריות על בלוקי הקלט, מה שמוביל ל[*אפקטי מפולת*](https://en.wikipedia.org/w/index.php?title=Avalanche_effect&oldid=1148010716) ב-ciphertext.

SPNs מתוכננות בצורה שבה הגדלת מספר ה<DefinitionTooltip definition="מספר הפעמים שטרנספורמציה חוזרת על עצמה על ידי אלגוריתם.">סיבובים</DefinitionTooltip> בדרך כלל מגדילה את אבטחת הצופן. דבר זה מוביל למושג [*שולי האבטחה*](https://pmc.ncbi.nlm.nih.gov/articles/PMC4878865/).
**שולי אבטחה:** שולי האבטחה של צופן קריפטוגרפי מבוסס SPN הוא ההפרש בין מספר הסיבובים ביישום המלא של הצופן לבין המספר המרבי של סיבובים שידועים כניתנים לפריצה באמצעות ההתקפה הידועה הטובה ביותר בעולם האמיתי.

לדוגמה, כרגע ההתקפות הטובות ביותר הידועות המהירות מחיפוש כוח גס נגד <DefinitionTooltip definition="תקן ההצפנה המתקדם הוא מפרט להצפנת נתונים, שנקבע על ידי NIST בשנת 2001. הוא נוצר מהצופנים של Rijndael שפותחו על ידי Joan Daemen ו-Vincent Rijmen, שני קריפטוגרפים בלגיים. הוא מוגדר ב-FIBS 197 ו-ISO/IEC 18033-3.">AES-256</DefinitionTooltip> יכולות לפרוץ עד 9 סיבובים מתוך 14 הסיבובים הכוללים בצופן המלא כאשר נמצאים במצב הפעולה הסטנדרטי המכונה *Electronic Codebook* (ECB). לכן, [שולי האבטחה הנוכחיים של AES-256 הם 5 סיבובים](https://doi.org/10.13154/tosc.v2019.i2.55-93).

**צופני זרם:** כחלופה לצופני בלוק, קריפטולוגים מודרניים תכננו גם [*צופני זרם*](https://en.wikipedia.org/w/index.php?title=Stream_cipher&oldid=1143750236) מאובטחים מעשית כגון [Chacha20](https://en.wikipedia.org/w/index.php?title=Salsa20&oldid=1145010249). צופנים אלה משתמשים באקראיות כחלק יסודי מעיצובם ופועלים על <DefinitionTooltip definition="זרם של תווים אקראיים המשמש להצפנת טקסט ברור. משמש בצופן זרם באופן דומה לאופן שבו one-time pad כתוב יד שימש.">*מחרוזות מפתח*</DefinitionTooltip> פסאודו-אקראיות של ביטים במקום בלוקים דיסקרטיים של נתונים.

בהתאם לכך, צופני זרם משלבים מפתח סודי ו[וקטור אתחול](https://en.wikipedia.org/w/index.php?title=Initialization_vector&oldid=1136156102) (IV) כדי לאתחל [מחולל מספרים פסאודו-אקראיים](https://en.wikipedia.org/w/index.php?title=Pseudorandom_number_generator&oldid=1153255020) (PRNG) לייצור <DefinitionTooltip definition="זרם של תווים אקראיים המשמש להצפנת טקסט ברור. משמש בצופן זרם באופן דומה לאופן שבו one-time pad כתוב יד שימש.">מחרוזת מפתח</DefinitionTooltip> של ביטים אקראיים המשולבים לאחר מכן עם הטקסט הברור הנתון לייצור ה-ciphertext. במובן זה, צופני זרם דומים ל<DefinitionTooltip definition="One-time pad. המפתח ארוך יותר מההודעה לשליחה, כאשר כל תו בטקסט הברור מחובר לערך המפתח. מימושים מוקדמים היו מבוססי נייר.">one-time pad</DefinitionTooltip> (OTP) אך מציגים אורכי מפתח סודי קצרים יותר ומפתחות לשימוש חוזר, מה שהופך אותם למעשיים יותר. עם זאת, מאותה סיבה, הם אינם מבטיחים סודיות מושלמת, בשונה מ-OTP.

**אבטחה סמנטית**: נסכם תת-חלק זה בחזרה למושג האבטחה הסמנטית או <DefinitionTooltip definition="אי-הבחנה תחת התקפת plaintext מוכר - מתייחסת למשחק היפותטי של יריב מנחש איזו מהודעות קבוצה הוצפנה עם מפתח נתון. בעצם ההתקפה מצליחה אם אין מורכבות/אקראיות מספיקה באלגוריתם הצפנה.">אבטחת IND-CPA</DefinitionTooltip> שהוצגו לעיל. הפעולות הבסיסיות שמיישמים [צופני בלוק](https://en.wikipedia.org/w/index.php?title=Block_cipher_mode_of_operation&oldid=1154901199#cite_note-23) כגון S-box ו-P-box הן דטרמיניסטיות. משמעות הדבר היא שבמצבי הפעולה הסטנדרטיים כגון <DefinitionTooltip definition="מצב Electronic Codebook. מצב הצפנה פשוט הפועל על בלוק בכל פעם. טקסט ברור נתון תמיד יגרום לאותו ciphertext. הוא אינו מסתיר נתונים היטב.">ECB</DefinitionTooltip>, זוג טקסט ברור-מפתח נתון תמיד מניב אותו ciphertext, מצב שפגיע להתקפות plaintext מוכר.

כדי להשיג אבטחת IND-CPA, צופני בלוק צריכים לפעול במצב המשתמש באקראיות המוכנסת דרך <DefinitionTooltip definition="וקטור אתחול. בדרך כלל ערך אקראי שנבחר בצורה בלתי צפויה המשמש לאתחול אלגוריתם קריפטוגרפי. דומה מאוד ל-nonce, ובמידה מה גם ל-salt.">וקטור אתחול</DefinitionTooltip> (IV) פסאודו-אקראי עם הדרישה הנוספת שאף שתי פעולות הצפנה לא ישתמשו באותו זוג מפתח-IV. AES תומך במספר מצבי פעולה, כגון [cipher block chaining](https://patents.google.com/patent/US4074066A/en) (CBC), שהם IND-CPA מאובטחים. דרישה דומה חלה גם על צופני זרם שבהם אותו זוג מפתח-IV לא צריך לשמש לאתחול ה-PRNG יותר מפעם אחת אם IND-CPA הוא רצוי.
## אלגוריתמי מפתח סימטרי פופולריים {#popular-symmetric-key-algorithms}

לאחר שהצגנו כמה עקרונות בסיסיים של SKC, נפרט כעת כמה אלגוריתמי מפתח סימטרי פופולריים כדי להדגים מגוון הגישות הנחקרות במערכות קריפטוגרפיות מודרניות. הן צופני בלוק מודרניים והן צופני זרם משמשים בהקשרים שונים כמומחש להלן.

1. [**תקן ההצפנה המתקדם:**](https://www.nist.gov/publications/development-advanced-encryption-standard) AES, שכבר הוצג לעיל, הוא כיום התקן de facto ל-SKC בשל מאפייני האבטחה, היעילות והביצועים שלו. AES מציג גדלי מפתח קבועים של 128, 192 ו-256 ביטים ומשתמש ב<DefinitionTooltip definition="רשת החלפה-תמורה. סוג של צופן בלוק שבו ביטים א) מוחלפים ו-ב) מעורבבים מספר איטרציות, שבהן גם המפתח עצמו עובר טרנספורמציה.">רשת החלפה-תמורה</DefinitionTooltip> (SPN) רב-סיבובית. AES ידוע כעמיד בפני מגוון רחב של התקפות קריפטאנליטיות. AES הוכרז כתקן עיבוד מידע פדרלי (FIPS) להצפנת מפתח סימטרי בארצות הברית בשנת 2001.

2. [**תקן ההצפנה של הנתונים (DES) ו-Triple DES (3DES):**](https://en.wikipedia.org/w/index.php?title=Data_Encryption_Standard&oldid=1154751166) DES היה צופן בלוק שהומצא במקור על ידי הורסט פייסטל ועמיתיו ב-IBM&reg; בשנות ה-70 והשתמש ב-SPN עם מפתח קצר יחסית של 56 ביטים. DES אומץ כ<DefinitionTooltip definition="תקנים לעיבוד מידע פדרלי - תקנים שפותחו על ידי NIST בארה&quot;ב לעיבוד מחשב.">FIPS</DefinitionTooltip> להצפנת מפתח סימטרי בארצות הברית עד סוף שנות ה-90 כאשר הוכח שניתן לפרוץ אותו באמצעות התקפות כוח גס עם חומרה ייעודית בשל גודל מרחב המפתחות הקטן שלו. לאחר מכן, 3DES הוצג כתחליף ומיישם את אלגוריתם DES שלוש פעמים עם מפתחות שונים, ומגדיל את אורך המפתח ל-168 ביטים. עם זאת, 3DES הוחלף בעיקר על ידי AES.

3. [**Blowfish ו-Twofish:**](https://en.wikipedia.org/w/index.php?title=Blowfish_(cipher)&oldid=1120053771) Blowfish ויורשו Twofish הם צופני בלוק שהוצעו על ידי הקריפטוגרף ברוס שניר בשנות ה-90. Blowfish ו-Twofish מאפשרים אורכי מפתח משתנים של עד 448 ביטים ו-256 ביטים בהתאמה, ומציעים גמישות מסוימת בפשרה בין אבטחה לביצועים. בשונה מ-AES, הם גם מציגים <DefinitionTooltip definition="תיבת החלפה. ממפה קבוצת קלטים של n ביטים לפלטים של m ביטים דרך טבלת חיפוש. משמשת להוספת אי-בהירות (בלבול) לאלגוריתמי הצפנת מפתח סימטרי.">S-boxes</DefinitionTooltip> תלויות מפתח. Twofish היה אחד המועמדים הסופיים בתחרות NIST לבחירת תקן ההצפנה המתקדם אך בסופו של דבר לא נבחר. שני האלגוריתמים נחשבים כיום מאובטחים.

4. [**צופנים של Rivest (RC2, RC4, RC5 ו-RC6):**](https://en.wikipedia.org/w/index.php?title=RC_algorithm&oldid=1072978900) משפחת האלגוריתמים של Rivest Cipher (RC) תוכננה על ידי רון ריבסט החל משנות ה-80. RC2 היה צופן בלוק מוקדם של 64 ביטים ואילו RC4 היה צופן זרם שהיה בשימוש נרחב בפרוטוקולי אבטחה הקשורים לתעבורת רשת בשל פשטותו ומהירותו. אף אחד מהם אינו נחשב כיום מאובטח. RC5 ו-RC6 הם צופני בלוק מבוססי SPN שתוכננו עם גודל בלוק, גודל מפתח ומספר סיבובים הניתנים להתאמה. כמו Twofish לעיל, RC6 היה מועמד סופי בתחרות NIST AES ונחשב מאובטח.

5. [**Salsa20 ו-ChaCha20:**](https://en.wikipedia.org/w/index.php?title=Salsa20&oldid=1145010249) Salsa20 ו-ChaCha20 מתייחסים למשפחה קשורה של צופני זרם שתוכנן על ידי הקריפטוגרף דניאל ברנשטיין בשנות ה-2000. Salsa20 הוא חלק מתיק פרופיל-1 של פרויקט אימות הקריפטוגרפי האירופי eSTREAM. ChaCha20, שינוי של Salsa20, תוכנן להגביר <DefinitionTooltip definition="פיזור הוא המאפיין לפיו היפוך ביט אחד בטקסט הברור צריך לשנות בערך חצי מהביטים ב-ciphertext ולהיפך. הפיזור מסתיר קשרים סטטיסטיים בין הטקסט הברור ל-ciphertext. לצופנים עם פיזור מספיק מתממש קריטריון המפולת של הקריפטוגרפיה.">פיזור</DefinitionTooltip> ביצועים. כרגע ChaCha20 נחשב מאובטח ומציע ביצועים טובים יותר בהיעדר האצת חומרת AES ייעודית. לכן, ChaCha20 מוצא שימוש בהגדרות מסוימות כגון פרוטוקולי רשת כמו <DefinitionTooltip definition="חיבורי UDP מהירים באינטרנט. פותח על ידי Google לחיבורים יעילים יותר. כיום תוקנן, ומשמש בתוך מחסנית HTTP/3 בין מקומות אחרים.">QUIC</DefinitionTooltip> ומכשירים ניידים עם מעבדי ARM.

## יתרונות קריפטוגרפיית מפתח סימטרי {#advantages-of-symmetric-key-cryptography}

לאחר תיאור תכונות המערכות הקריפטוגרפיות של מפתח סימטרי וחלק מהעקרונות העומדים בבסיס פיתוחן, נפרט כעת כמה מהיתרונות העיקריים של [SKC](https://books.google.com/books?id=01vfjgEACAAJ) ביחס לקריפטוגרפיית מפתח א-סימטרי. האחרונה תידון בשיעורים הבאים.

1. **מהירות ויעילות:** אלגוריתמי מפתח סימטרי מתאימים יותר להצפנת נפחי נתונים גדולים או לשימוש בתרחישי תקשורת בזמן אמת מכיוון שהם בדרך כלל מהירים יותר ופחות צורכי משאבים מאשר עמיתיהם הא-סימטריים. אלגוריתמי SKC כגון AES גדלים בצורה לינארית עם גודל הטקסט הברור ואינם כוללים פעולות מתמטיות אינטנסיביות מבחינה אלגברית. עיין ב-[Tomoiaga et al.](https://doi.org/10.1109/ICSNC.2010.33) לסקירה מפורטת של מאפייני הביצועים של AES.

2. **מדרגיות:** בשל העלות החישובית הנמוכה יחסית, אלגוריתמי מפתח סימטרי מתמדגים היטב עם מספר המשתמשים וכמות הנתונים המוצפנים.

3. **פשטות:** פרוטוקולי הצפנה סימטריים לרוב קלים יותר ליישום ולהבנה בהשוואה לגישות מפתח א-סימטרי, מה שהופך אותם לאטרקטיביים עבור מפתחים ומשתמשים.
## אתגרים ומגבלות של קריפטוגרפיית מפתח סימטרי {#challenges-and-limitations-of-symmetric-key-cryptography}

למרות היתרונות, לקריפטוגרפיית מפתח סימטרי יש גם כמה אתגרים ומגבלות:

1. **הפצה וניהול מפתחות:** ב-SKC, הן השולח והן המקבל של הודעה חייבים לקבל גישה לאותו מפתח, שחייב להישמר סודי מגורמים בלתי מורשים. אם המפתח מיורט בצורה כלשהי או נפגע על ידי צד שלישי, אז גם האבטחה של הנתונים המוצפנים תאבד. לכן, ההפצה המאובטחת וניהול המפתח הסודי הם אתגר מרכזי. עם זאת, הפתרון לאתגר זה נמצא מחוץ ל-SKC עצמה.

2. **היעדר אי-כפירה:** [אי-כפירה](https://en.wikipedia.org/w/index.php?title=Non-repudiation&oldid=1148337707) מתייחסת ליכולת להוכיח שצד ספציפי שלח הודעה. ב-SKC, מכיוון שאותו מפתח משמש הן להצפנה והן לפענוח, לא ניתן לקבוע איזה צד יצר ciphertext מסוים. לעומת זאת, קריפטוגרפיית מפתח א-סימטרי מספקת אי-כפירה באמצעות שימוש ב<DefinitionTooltip definition="עקבה המשמשת לאימות אותנטיות, שלמות ומקור של מסמכים/נתונים דיגיטליים; מספקת דרך להבטיח שאובייקט לא שונה ומגיע מהשולח המיועד.">חתימה דיגיטלית</DefinitionTooltip>.

כדי להתמודד עם אתגרים אלה, קריפטוגרפיית מפתח סימטרי משמשת לרוב בשילוב עם קריפטוגרפיית מפתח א-סימטרי. לדוגמה, לעתים קרובות משתמשים בהצפנת מפתח א-סימטרי להעברה מאובטחת של מפתח סוד משותף קצר יחסית בין השולח למקבל. זה מאפשר שימוש לאחר מכן בהצפנת מפתח סימטרי להעברת נתונים והודעות גדולות הרבה יותר ביעילות.
## מחשוב קוונטי והצפנת מפתח סימטרי: סיכונים ומיתיגציה {#quantum-computing-and-symmetric-key-encryption-risks-and-mitigation}

הקריפטוגרפיה הקוונטית מציעה נתיב מבטיח למיתיגציית סיכונים בעידן הדיגיטלי, כאשר אימוץ מוצרים עמידים בפני קוונטום עומד לאבטח את המידע שלנו מפני האיום המתקרב של התקדמות המחשוב הקוונטי.

בהמשך נדון בסיכונים שמחשבים קוונטיים מציבים לשיטות הצפנת מפתח סימטרי שהוצגו בחלק הקודם ונתאר כמה נתיבים פוטנציאליים להפחתת הסיכונים.
### התקפות קריפטוגרפיות קוונטיות {#quantum-cryptographic-attacks}

ישנן שתי מחלקות נפרדות של איומים קוונטיים על אלגוריתמי קריפטוגרפיה מסורתיים:

1. **התקפות כוח גס קוונטיות**: אלה מתייחסות למצבים שבהם התוקף משתמש במחשב קוונטי להרצת אלגוריתם קוונטי מיוחד לביצוע חיפוש כוח גס דרך מרחב המפתחות של צופן סימטרי. הפרימיטיב הקוונטי הרלוונטי ביותר לאפשור סוג זה של התקפה הוא [אלגוריתם גרובר](https://dl.acm.org/doi/10.1145/237814.237866).

2. **התקפות קריפטאנליטיות קוונטיות**: אלה מתייחסות למצבים שבהם מחשבים קוונטיים נפרסים להרצת התקפות קריפטאנליטיות שמטרתן לשחזר את המפתח הסודי או הטקסט הברור ביעילות רבה יותר מחיפוש כוח גס. האפשרות לביצוע התקפות <DefinitionTooltip definition="ניתוח מערכות קריפטוגרפיות לזיהוי חולשותיהן, פגיעויותיהן או מידע מוסתר ללא גישה למפתחות הסודיים; המטרה הסופית היא לעקוף את אמצעי האבטחה שהמערכת מיישמת. קריפטאנליזה לינארית שואפת למצוא קירוב לפונקציית הצפנה, בעוד שקריפטאנליזה דיפרנציאלית עוקבת אחר רשת של טרנספורמציות שונות.">קריפטאנליטיות</DefinitionTooltip> קוונטיות מוצלחות תלויה בגורמים רבים הקשורים למבנה המתמטי של הצופן המנותח וכן לנקודות חולשה פוטנציאליות ביישומים ספציפיים.
### אסטרטגיות מיתיגציית סיכון להתקפות קוונטיות {#risk-mitigation-strategies-for-quantum-attacks}

לפני שנדון באסטרטגיות מיתיגציית סיכון להתקפות קוונטיות, נציג את מושג [רמת האבטחה](https://en.wikipedia.org/w/index.php?title=Security_level&oldid=1131789113) של צופן קריפטוגרפי:

**רמת האבטחה** היא מדד לקושי לפרוץ צופן, נמדדת במונחים של מספר הפעולות החישוביות שפריצה מוצלחת של הצופן תדרוש.

בדרך כלל, רמת האבטחה מבוטאת בביטים; כלומר, בכלל, צופן מציע אבטחה של N ביטים אם הוא דורש $\mathcal{O}(2^{N})$ פעולות לפריצתו. במחשבים קלאסיים, בהנחה שצופן סימטרי מאובטח קריפטוגרפית אחרת, רמת האבטחה נרדפת בערך לאורך המפתח.

לדוגמה, רמת האבטחה של AES-128, המציג מפתח של 128 ביטים, נחשבת בדרך כלל כ-128 ביטים מכיוון שתדרוש בסדר גודל של 2$^{128}$ פעולות עבור תוקף המשתמש במחשב קלאסי לניסוי כל מפתחות ה-128 ביטים האפשריים במרחב המפתחות.

### התקפות כוח גס ומיתיגציה {#brute-force-attacks-and-mitigation}

**סיכון התקפת כוח גס קוונטית:** [התקפת כוח גס קוונטית](https://inria.hal.science/hal-01237242) משנה את ההערכה הנ"ל מכיוון ש<DefinitionTooltip definition="אלגוריתם גרובר הוא אלגוריתם קוונטי לחיפוש בנתונים לא-מובנים. בעוד שאלגוריתם קלאסי עשוי להצריך N/2 שלבים לחיפוש ב-N פריטים, עם אלגוריתם גרובר זה מצטמצם ל-$sqrt(N)$.">אלגוריתם גרובר</DefinitionTooltip> מאפשר לתוקף עם מחשב קוונטי מתאים לחפש דרך מרחב המפתחות של צופן <DefinitionTooltip definition="ריבועית מהר יותר מתייחס להאצה המתרחשת ביחס לריבוע מספר הערכים, לדוגמה חיפוש, בהשוואה לבדיקה לינארית פשוטה של כל ערך.">מהר יותר בריבוע</DefinitionTooltip> מכל מחשב קלאסי.

לדוגמה, אותה התקפת כוח גס על AES-128 עם אלגוריתם גרובר יכולה להיות מושגת עם 2$^{64}$ פעולות בלבד. לכן רמת האבטחה של AES-128 מצטמצמת מ-128 ביטים ל-64 ביטים כאשר מתמודדים עם יריב קוונטי הרץ חיפוש גרובר. מכיוון שכוח החישוב גדל באופן מסורתי אקספוננציאלית עם הזמן, כרגע רמת אבטחה של 64 ביטים נחשבת לא מאובטחת, מה שאומר שברגע שיממשו מחשבים קוונטיים בעלי יכולת מספקת, AES-128 יצטרך להינטש.

אותו סוג חישוב חל על צופני בלוק או זרם סימטריים אחרים שבהם רמת האבטחה לאורך מפתח נתון מוצטמצמת למעשה בחצי על ידי אלגוריתם גרובר.

**מיתיגציית סיכון התקפת כוח גס קוונטית:** השיקולים לעיל רומזים שהדרך הברורה לעמוד בפני התקפות כוח גס קוונטיות היא לפחות [הכפלת אורכי המפתח המינימליים המשמשים להצפנת מפתח סימטרי](https://inria.hal.science/hal-01237242).

לכן, כדי להבטיח אבטחה של 128 ביטים ביחס להתקפות כוח גס קוונטיות, ניתן פשוט להשתמש בצופנים כגון AES-256 או ChaCha20 המשתמשים במפתחות של 256 ביטים. הדבר נחשב מאובטח מכיוון שגם עם מחשבים קוונטיים, ביצוע 2$^{128}$ פעולות לפריצת צופנים אינו ישים בעתיד הנראה לעין.

למרות שהפתרון המוצע של הכפלת גדלי מפתח הוא פשוט מבחינה תיאורטית, אין הוא נטול עלויות, שכן מפתחות ארוכים יותר מרמזים על עלות חישובית נוספת עבור משימות הצפנה-פענוח שגרתיות, לצד ביצועים איטיים יותר, דרישת זיכרון גבוהה יותר ושימוש נוסף באנרגיה.
### התקפות קריפטאנליטיות ומיתיגציה {#cryptanalytic-attacks-and-mitigation}

**סיכון התקפות קריפטאנליטיות קוונטיות:** הסיכון שהתקפות קריפטאנליטיות קוונטיות מציבות למערכות קריפטוגרפיות של מפתח סימטרי נחקר כיום [באופן פעיל על ידי קריפטוגרפים](https://www.sciencedirect.com/science/article/pii/S0045790622003743). השילוב של מחשוב קלאסי וקוונטי מרחיב פוטנציאלית את מגוון הכלים הזמינים לתוקפים לבחינת חולשות במבנה המתמטי של צופנים, ומגוון רחב של <DefinitionTooltip definition="התקפה קריפטאנליטית קוונטית. מתייחסת למצבים שבהם מחשבים קוונטיים נפרסים להרצת התקפות קריפטאנליטיות שמטרתן לשחזר את המפתח הסודי או הטקסט הברור ביעילות רבה יותר מחיפוש כוח גס.">התקפות קריפטאנליטיות קוונטיות</DefinitionTooltip> חדשות מוצעות כיום. אלה כוללות <DefinitionTooltip definition="קוונטיזציה היא תהליך של מיפוי ערכים מקבוצה גדולה לקבוצה קטנה יותר. לדוגמה דגימת אודיו של 16 ביטים ממפה מערך תדר רציף לערך המיוצג ב-16 ביטים. במחשוב קוונטי ניתן להמיר מספר עם נקודה צפה לקביט קוונטי מסובך. לאחר מכן ניתן להשתמש בו באלגוריתמי חיפוש כגון אלגוריתם גרובר.">קוונטיזציות</DefinitionTooltip> של טכניקות קלאסיות ידועות כגון [קריפטאנליזה לינארית ודיפרנציאלית](https://inria.hal.science/hal-01237242) וכן מצבי התקפה חדשים ללא עמיתים קלאסיים.

[מחקר קריפטאנליטי קוונטי אחרון של תקן ההצפנה המתקדם (AES)](https://doi.org/10.13154/tosc.v2019.i2.55-93) מצא שהצופן נותר עמיד בפני התקפות <DefinitionTooltip definition="התקפה קריפטאנליטית קוונטית. מתייחסת למצבים שבהם מחשבים קוונטיים נפרסים להרצת התקפות קריפטאנליטיות שמטרתן לשחזר את המפתח הסודי או הטקסט הברור ביעילות רבה יותר מחיפוש כוח גס.">קריפטאנליטיות קוונטיות</DefinitionTooltip> ידועות שונות והמשיך להציג <DefinitionTooltip definition="מדד לכמה עוד צריך לשפר את ניתוח הצופן כדי לפרוץ אותו.">שולי אבטחה</DefinitionTooltip> פוסט-קוונטיים מספקים. עם זאת, מחקרים מסוימים מצאו שצופנים סימטריים שונים הנחשבים מאובטחים קלאסית נפגעים בקלות על ידי מה שנקרא [*התקפת plaintext מוכר קוונטית*](https://doi.org/10.1007/978-3-319-56617-7_3). לכן, פרימיטיבים חדשים להצפנת מפתח סימטרי שתוכננו במיוחד לעידן הפוסט-קוונטי הוצעו גם הם.

**מיתיגציית סיכון התקפות קריפטאנליטיות קוונטיות:** בהינתן שהקריפטאנליזה הקוונטית כדיסציפלינה היא בחיתוליה, ייתכן שקריפטוגרפיית מפתח סימטרי פוסט-קוונטית תעבור אבולוציה מהירה ככל שיתעוררו התקפות קריפטאנליטיות קוונטיות חדשות וככל שיוצעו ויוערכו צופנים חדשים העמידים בפניהם. לכן, האסטרטגיה הטובה ביותר להפחתת סיכון ההתקפות הקריפטאנליטיות הקוונטיות בעתיד הנראה לעין היא [*זריזות קריפטוגרפית*](https://en.wikipedia.org/w/index.php?title=Cryptographic_agility&oldid=1155073704) (או crypto-agility). זריזות קריפטוגרפית מתייחסת ליכולת של מערכת מידע לאמץ במהירות ובקלות פרימיטיבים קריפטוגרפיים חלופיים ללא שינויים משבשים בתשתית המערכת.

זריזות קריפטוגרפית דורשת את היכולת להחליף אלגוריתמים מיושנים המשמשים להצפנה, פענוח, חתימות דיגיטליות או פונקציות קריפטוגרפיות אחרות במאמץ ובשיבוש מינימליים. מערכות crypto-agile יהיו ממוקמות היטב לניהול המעבר לקריפטוגרפיית מפתח סימטרי פוסט-קוונטית.
## סיכום {#summary}

קריפטוגרפיית מפתח סימטרי מספקת פתרונות חזקים ויעילים לאבטחת מידע דיגיטלי. פשטות השימוש באותו מפתח הן להצפנה והן לפענוח מאפשרת ביצועים גבוהים ומדרגיות, מה שהופך SKC למתאימה למגוון רחב של יישומים.

אבטחת SKC מסתמכת על עמידות אלגוריתמית בפני התקפות קריפטוגרפיות וכן על ניהול מפתח סוד נאות. מערכות קריפטוגרפיות מפתח סימטרי מודרניות משלבות את עקרונות ה<DefinitionTooltip definition="בלבול הוא המאפיין לפיו כל ביט ב-ciphertext תלוי בביטים מרובים של המפתח הסודי. הוא מבטיח שגם שינוי קטן במפתח הסודי ישנה כמעט את כל הביטים של ה-ciphertext, ומטשטש את הקשר בין ה-ciphertext למפתח הסודי.">בלבול</DefinitionTooltip>, הפיזור והאקראיות, בשילוב עם גדלי מפתח מספיקים, כדי להשיג <DefinitionTooltip definition="עם ה-ciphertext בלבד, וכל מידע ציבורי רלוונטי (כגון מפתח ציבורי לקריפטוגרפיה א-סימטרית), יריב לא יכול לאחזר אפילו מידע חלקי מהטקסט הברור.">אבטחה סמנטית</DefinitionTooltip>. ניהול מפתח סוד, אמנם קריטי, אינו ניתן להשגה עם SKC בלבד.

הבנת התכונות והמגבלות של SKC תאפשר למפתחים לתכנן, ליישם ולפרוס פתרונות טכנולוגיית מידע מאובטחים תוך שימוש בגישות הכוללות גדלי מפתח ארוכים יותר לפי הצורך ושימוש באלגוריתמים חדשים.

ההתקדמות במחשוב קוונטי ולמידה קוונטית מציגה ממד חדש בקריפטוגרפיית מפתח סימטרי. למחשבים קוונטיים יש פוטנציאל לפרום את האבטחה שמספקים אלגוריתמי מפתח סימטרי קלאסיים, מה שמעלה את הצורך בגישות קריפטוגרפיות עמידות בפני קוונטום להבטחת פרטיות ואבטחת הנתונים בפני נוף טכנולוגי מתפתח.

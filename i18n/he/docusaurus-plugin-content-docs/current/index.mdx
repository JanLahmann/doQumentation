## תוכן קוד פתוח של IBM Quantum {#ibm-quantums-open-source-content}

IBM מספקת שפע של חומרי למידה למחשוב קוונטי — כולם בקוד פתוח תחת רישיון CC BY-SA 4.0:

- **[Learning](https://quantum.cloud.ibm.com/learning)** — קורסים מובנים מיסודות הקוונטים ועד נושאים מתקדמים
- **[Tutorials](https://quantum.cloud.ibm.com/docs/en/tutorials)** — למעלה מ־40 מדריכים על transpilation, הפחתת שגיאות ועוד
- **[Documentation](https://quantum.cloud.ibm.com/docs/en/guides)** — מדריכי עזר ומידע API עבור Qiskit
- **[Source repo](https://github.com/Qiskit/documentation)** — כל התוכן ב־GitHub

ה־[Quantum Platform](https://quantum.cloud.ibm.com) שלהם מעודכן תמיד ומעוצב היטב — המקום הטוב ביותר לקריאה, למידה ועיון.

## מה הפרויקט הזה מוסיף {#what-this-project-adds}

ה[תיעוד של Qiskit](https://github.com/Qiskit/documentation) של IBM הוא קוד פתוח (CC BY-SA 4.0), אבל אפליקציית הווב שלהם לא. doQumentation מוסיף ממשק קוד פתוח עם הרצת קוד חיה, הזרקת אישורים אוטומטית ומצב סימולטור.

ניתן לפריסה בכל מקום — מ־[GitHub Pages](https://doqumentation.org) דרך [Docker](https://github.com/JanLahmann/doQumentation/pkgs/container/doqumentation) ועד [RasQberry](https://rasqberry.org/). ראו [את כל התכונות](/features).

## הרצת קוד {#code-execution}

לחץ על **Run** בכל בלוק קוד. הלחיצה הראשונה מפעילה ליבת Jupyter חינמית דרך [Binder](https://mybinder.org) (דקה עד שתיים). לאחר מכן, ההרצות מיידיות.

- **מצב סימולטור (ללא צורך בחשבון)** — הפעל [מצב סימולטור](/jupyter-settings#simulator-mode) להרצת כל המחברות עם AerSimulator או FakeBackends עם מודל רעש. אפס התקנה נדרשת.

- **חומרה קוונטית של IBM Quantum** — הזן את ה־API token וה־CRN שלך ב[הגדרות](/jupyter-settings#ibm-quantum) פעם אחת — האישורים מוזרקים אוטומטית בכל הרצה. [צור חשבון חינמי](https://quantum.cloud.ibm.com/registration) אם אין לך כזה.

<details>
<summary>מנועי הרצה זמינים</summary>

לכל מדריך יש בלוקי קוד להרצה. לחץ על **Run** להרצתם באמצעות אחד משלושת המנועים:

1. **Binder** (ברירת מחדל ב־GitHub Pages) — ליבת Jupyter מרוחקת בחינם דרך [mybinder.org](https://mybinder.org)
2. **Jupyter מקומי (Docker / RasQberry)** — מתחבר לשרת Jupyter המקומי עם Qiskit מותקן מראש
3. **שרת מותאם אישית** — הפנה לכל נקודת קצה Jupyter ב[הגדרות](/jupyter-settings#advanced)

</details>

<details>
<summary>אפשרויות פריסה</summary>

doQumentation זמין כ:

- **[GitHub Pages](https://doqumentation.org)** — אתר סטטי עם [Binder](https://mybinder.org) להרצת קוד מרוחקת
- **[Docker](https://github.com/JanLahmann/doQumentation/pkgs/container/doqumentation)** — מערכת מלאה מקומית, ניתנת לעבודה לא מקוונת
- **[RasQberry](https://rasqberry.org/)** — אירוח עצמי על Raspberry Pi עם ליבת Jupyter מקומית

</details>

<details>
<summary>הרצה מקומית עם Podman / Docker</summary>

התקן [Podman Desktop](https://podman-desktop.io/) או [Docker Desktop](https://www.docker.com/products/docker-desktop/) (Mac / Windows / Linux), ולאחר מכן הרץ:

```bash
# מערכת מלאה: אתר + Jupyter + Qiskit (~3 GB)
podman run -p 8080:80 -p 8888:8888 ghcr.io/janlahmann/doqumentation:jupyter
```

פתח [http://localhost:8080](http://localhost:8080) — הרצת קוד עובדת מקומית, ללא זמן המתנה ל־Binder.

לגרסה קלת משקל ללא הרצת קוד מקומית (~60 MB):

```bash
# אתר סטטי בלבד — הרצת קוד עדיין עובדת דרך Binder
podman run -p 8080:80 ghcr.io/janlahmann/doqumentation:latest
```

משתמש ב־Docker במקום? פשוט החלף `podman` ב־`docker` — הפקודות זהות. התמונות תומכות במספר ארכיטקטורות (`linux/amd64` + `linux/arm64`), כך ש־Mac עם Apple Silicon עובדים באופן טבעי.

</details>

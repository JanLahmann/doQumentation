### 1א: הגדרת ההמילטוניאן שלנו {#1a-setting-up-our-hamiltonian}

אנו משתמשים במודל איזינג על שרשרת של 10 אתרים:

$$
\hat{\mathcal{H}}_{\text{Ising}} = \sum_{i=1}^{9} J_{i,(i+1)} Z_i Z_{(i+1)} + \sum_{i=1}^{10} h_i X_i \, ,
$$

כאשר $J$ הוא עוצמת הצימוד בין שני אתרים ו-$h$ הוא השדה המגנטי החיצוני.
החבילה [qiskit_addon_utils](https://qiskit.github.io/qiskit-addon-utils/) מספקת פונקציות שימושיות לשימוש חוזר למטרות שונות.

### 1ב: נוסחאות רב-מכפלה {#1b-multi-product-formulas}

נוסחאות MPF מצמצמות את שגיאת הטרוטר בדינמיקת המילטוניאן באמצעות שילוב משוקלל של מספר ריצות מעגלים.

כדי להמחיש זאת, נגדיר MPF כך:

$$
\mu(t) = \sum_{j} x_j \rho^{k_j}_{j}\left(\frac{t}{k_j}\right) + \text{some remaining Trotter error} \, ,
$$

כאשר $x_j$ הם מקדמי המשקל שלנו, $\rho^{k_j}_j$ היא מטריצת הצפיפות המתאימה למצב הטהור המתקבל על ידי אבולוציה של המצב ההתחלתי עם נוסחת המכפלה $S^{k_j}$ הכוללת $k_j$ צעדי טרוטר, ו-$j$ הוא אינדקס מספר נוסחאות המכפלה המרכיבות את ה-MPF.

הנקודה המרכזית היא ששגיאת הטרוטר הנותרת קטנה יותר מהשגיאה שהיינו מקבלים באמצעות שימוש פשוט בערך $k_j$ הגדול ביותר!

ניתן להסתכל על התועלת של MPF משתי זוויות:

1. עבור תקציב קבוע של צעדי טרוטר שניתן לבצע, ניתן לקבל תוצאות עם שגיאת טרוטר קטנה יותר בסך הכול.
2. עבור מספר צעדי טרוטר המייצרים מעגלים עמוקים, ניתן להשתמש ב-MPF כדי למצוא מספר מעגלים בעומק נמוך יותר שיניבו שגיאת טרוטר דומה.
#### מבוא ל-MPF סטטיות {#an-introduction-to-static-mpfs}

MPF _סטטיות_ הן אלה שבהן ערכי $x_j$ **אינם** תלויים בזמן האבולוציה, $t$.

קביעת מקדמי ה-MPF הסטטית עבור קבוצה נתונה של ערכי $k_j$ שקולה לפתרון מערכת משוואות לינארית:
$Ax=b$, כאשר $x$ הם המקדמים הנדרשים, $A$ היא מטריצה התלויה ב-$k_j$ ובסוג ה-PF שבו אנו משתמשים ($S$), ו-$b$ הוא וקטור אילוצים.
לשם קיצור, לא נרחיב כאן ובמקום זאת נפנה אותך לתיעוד של [LSE](https://qiskit.github.io/qiskit-addon-mpf/apidocs/qiskit_addon_mpf.costs.html#qiskit_addon_mpf.costs.LSE).

ניתן למצוא פתרון ל-$x$ אנליטית כ-$x = A^{-1}b$, ראה למשל [Carrera Vazquez et al., 2023] או [Zhuk et al., 2023].
אולם פתרון מדויק זה עלול להיות _"בעייתי מבחינת קונדיציה"_ ולהוביל לנורמות L1 גדולות מאוד של המקדמים $x$, מה שעלול לגרום לביצועים גרועים של ה-MPF.
חלופה לכך היא קבלת פתרון מקורב המינימיזציה את נורמת ה-L1 של $x$ במטרה לייעל את התנהגות ה-MPF.

בהמשך תלמד/י כיצד לעשות את כל זאת.

[Carrera Vazquez et al., 2023]: https://quantum-journal.org/papers/q-2023-07-25-1067/
[Zhuk et al., 2023]: https://journals.aps.org/prresearch/abstract/10.1103/PhysRevResearch.6.033309
#### בחירת $k_j$ {#choosing-k_j}

הבחירה של $k_j$ נתונה לשיקול המשתמש.
באופן עקרוני ניתן לבחור כל ערכים, אך ישנם ערכי $k_j$ שיובילו להגברת רעש גדולה יותר בהתקנים אמיתיים בהשוואה לאחרים.
לכן חשוב לנסות ולמצוא ערכים _"טובים"_ של $k_j$.

כאן פשוט נבחר כמה ערכים קבועים ל-$k_j$.
הערך הקטן ביותר מונחה על ידי זמן האבולוציה המטרה $t=8.0$, שבדרך כלל מחייב לעמוד בתנאי $t/k_{\text{min}} \lt 1$, אך אמפירית אנו יודעים שהגדרתו שווה ל-$1$ בדרך כלל עובדת אף היא.
אם ברצונך ללמוד עוד על כך וכיצד לבחור את שאר ערכי $k_j$, עיין במדריך הרלוונטי: [How to choose the Trotter steps for an MPF](https://qiskit.github.io/qiskit-addon-mpf/how_tos/choose_trotter_steps.html).

```python
time = 8.0
trotter_steps = (8, 12, 19)
```

#### הגדרת ה-LSE {#setting-up-the-lse}

כעת שבחרנו את ערכי $k_j$, עלינו תחילה לבנות את ה-LSE, $Ax=b$ כפי שהוסבר לעיל.
המטריצה $A$ תלויה לא רק ב-$k_j$ אלא גם בבחירת נוסחת המכפלה (PF) שלנו -- ובפרט ב_סדר_ שלה.
בנוסף, ניתן לקחת בחשבון האם ה-PF סימטרי או לא (ראה [Carrera Vazquez et al., 2023]), על ידי הגדרת `symmetric=True`.
אולם זה אינו חובה כפי שמוצג ב-[Zhuk et al., 2023].

כאן נשתמש בנוסחת סוזוקי-טרוטר מסדר שני המניבה `order=2` ונגדיר `symmetric=True`.

[Carrera Vazquez et al., 2023]: https://quantum-journal.org/papers/q-2023-07-25-1067/
[Zhuk et al., 2023]: https://journals.aps.org/prresearch/abstract/10.1103/PhysRevResearch.6.033309

```python
from qiskit_addon_mpf.static import setup_static_lse

lse = setup_static_lse(trotter_steps, order=2, symmetric=True)
print(lse)
```

```text
LSE(A=array([[1.00000000e+00, 1.00000000e+00, 1.00000000e+00],
       [1.56250000e-02, 6.94444444e-03, 2.77008310e-03],
       [2.44140625e-04, 4.82253086e-05, 7.67336039e-06]]), b=array([1., 0., 0.]))
```

#### פתרון אנליטי של $x$ {#solving-for-x-analytically}

כפי שהוזכר קודם, ניתן למצוא את $x$ אנליטית:

```python
import numpy as np

coeffs_analytical = lse.solve()
print(coeffs_analytical)
```

```text
[ 0.17239057 -1.19447005  2.02207947]
```

#### אופטימיזציה של $x$ באמצעות מודל מדויק {#optimizing-for-x-using-an-exact-model}

כחלופה לחישוב $x=A^{-1}b$, ניתן גם להשתמש ב-[setup_exact_problem](https://qiskit.github.io/qiskit-addon-mpf/stubs/qiskit_addon_mpf.costs.setup_exact_problem.html) לבניית מופע [cvxpy.Problem](https://www.cvxpy.org/api_reference/cvxpy.problems.html#cvxpy.Problem) המשתמש ב-LSE כאילוצים ושהפתרון האופטימלי שלו יניב את $x$.

בחלק הבא יתברר מדוע ממשק זה קיים.

```python
from qiskit_addon_mpf.costs import setup_exact_problem

model_exact, coeffs_exact = setup_exact_problem(lse)
model_exact.solve()
print(coeffs_exact.value)
```

```text
[ 0.17239057 -1.19447005  2.02207947]
```

כמדד לשאלה האם MPF שנבנית עם מקדמים אלו תניב תוצאות טובות, ניתן להשתמש בנורמת ה-L1 (ראה גם [Carrera Vazquez et al., 2023]).

[Carrera Vazquez et al., 2023]: https://quantum-journal.org/papers/q-2023-07-25-1067/

```python
print(np.linalg.norm(coeffs_exact.value, ord=1))
```

```text
3.3889400921655914
```

#### אופטימיזציה של $x$ באמצעות מודל מקורב {#optimizing-for-x-using-an-approximate-model}

ייתכן שנורמת ה-L1 עבור קבוצת ערכי $k_j$ הנבחרת תיחשב גבוהה מדי.
אם כך הוא המצב ולא ניתן לבחור קבוצת ערכי $k_j$ שונה, ניתן להשתמש בפתרון מקורב ל-LSE במקום בפתרון מדויק.

לשם כך, פשוט השתמש/י ב-[setup_sum_of_squares_problem](https://qiskit.github.io/qiskit-addon-mpf/stubs/qiskit_addon_mpf.costs.setup_sum_of_squares_problem.html) לבניית מופע [cvxpy.Problem](https://www.cvxpy.org/api_reference/cvxpy.problems.html#cvxpy.Problem) שונה המגביל את נורמת ה-L1 לסף נבחר תוך מינימיזציה של ההפרש בין $Ax$ ל-$b$.

```python
from qiskit_addon_mpf.costs import setup_sum_of_squares_problem

model_approx, coeffs_approx = setup_sum_of_squares_problem(lse, max_l1_norm=3.0)
model_approx.solve()
print(coeffs_approx.value)
print(np.linalg.norm(coeffs_approx.value, ord=1))
```

```text
[-0.40454257  0.57553173  0.8290123 ]
1.8090865903790838
```

שים/י לב שיש לך חופש מלא על אופן פתרון בעיית האופטימיזציה, כלומר ניתן לשנות את ה-solver, סף ההתכנסות שלו, וכן הלאה.
עיין/י במדריך הרלוונטי [How to use the approximate model](https://qiskit.github.io/qiskit-addon-mpf/how_tos/using_approximate_model.html).
### 1ג: הגדרת מעגלי הטרוטר {#1c-setting-up-the-trotter-circuits}

בשלב זה מצאנו את מקדמי הפיתוח $x$, וכל שנותר הוא לייצר את מעגלי הקוונטום בשיטת טרוטר.
שוב, המודול [qiskit_addon_utils.problem_generators](https://qiskit.github.io/qiskit-addon-utils/stubs/qiskit_addon_utils.problem_generators.html) בא לעזרתנו בדיוק לשם כך:

```python
from qiskit.synthesis import SuzukiTrotter

        hamiltonian,
        synthesis=SuzukiTrotter(order=2, reps=k),
        time=time,

    circuits.append(circ)
```

## שלב 2: אופטימיזציה של הבעיה {#step-2-optimize-the-problem}

בדרך כלל, זהו השלב בתבנית שבו מבצעים אופטימיזציה של המעגלים לביצוע על חומרה.
כאן, מכיוון שאנו משתמשים רק בסימולטור נטול רעש, פשוט נבצע Transpile למעגלים שלנו עבור [GenericBackendV2](https://quantum.cloud.ibm.com/docs/api/qiskit/qiskit.providers.fake_provider.GenericBackendV2).

```python
from qiskit.providers.fake_provider import GenericBackendV2
from qiskit.transpiler import generate_preset_pass_manager

backend = GenericBackendV2(num_qubits=10)
transpiler = generate_preset_pass_manager(optimization_level=2, backend=backend)

transpiled_circuits = [transpiler.run(circ) for circ in circuits]
```

## שלב 3: ביצוע ניסויים קוונטיים {#step-3-execute-quantum-experiments}

כפי שהוסבר בתחילה, נדלג על שלב האופטימיזציה 2 מכיוון שפשוט נחשב את ערכי הציפייה של האובזרבבל המטרה שלנו באמצעות סימולטור נטול רעש, ה-[StatevectorEstimator](https://quantum.cloud.ibm.com/docs/api/qiskit/qiskit.primitives.StatevectorEstimator).

```python
from qiskit.primitives import StatevectorEstimator

estimator = StatevectorEstimator()
job = estimator.run([(circ, observable) for circ in transpiled_circuits])
result = job.result()
```

## שלב 4: שחזור תוצאות {#step-4-reconstruct-results}

תחילה, נחלץ את ערכי הציפייה הבודדים שהתקבלו עבור כל אחד ממעגלי הטרוטר:

```python
evs = [res.data.evs for res in result]
print(evs)
```

```text
[array(0.23799162), array(0.35754312), array(0.38649906)]
```

לאחר מכן, פשוט נשלב אותם עם מקדמי ה-MPF שלנו כדי לקבל את ערכי הציפייה הכוללים של ה-MPF. להלן נעשה זאת עבור כל אחת מהדרכים השונות שבהן חישבנו את $x$.

```python
print("Analytical    solution:", evs @ coeffs_analytical)
print("Exact model   solution:", evs @ coeffs_exact.value)
print("Approx. model solution:", evs @ coeffs_approx.value)
```

```text
Analytical    solution: 0.3954847855980006
Exact model   solution: 0.39548478559800204
Approx. model solution: 0.42991214253489807
```

לבסוף, עבור בעיה קטנה זו ניתן לחשב את ערך הייחוס המדויק באמצעות [scipy.linalg.expm](https://docs.scipy.org/doc/scipy/reference/generated/scipy.linalg.expm.html) כדלקמן:

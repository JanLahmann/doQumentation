## תחילת העבודה עם Estimator {#start-estimator}

{/*Verified the v2 examples 2/29/24 - updated 10/29/24*/}

### 1. אתחול החשבון {#initialize-the-account}

מכיוון ש-Qiskit Runtime Estimator הוא שירות מנוהל, עליך תחילה לאתחל את החשבון שלך. לאחר מכן תוכל לבחור את ה-QPU שבו ברצונך להשתמש לחישוב ערך הציפייה.

פעל לפי השלבים ב-[נושא ההתקנה וההגדרה](install-qiskit) אם אין לך עדיין חשבון.

```python
from qiskit_ibm_runtime import QiskitRuntimeService

service = QiskitRuntimeService()
backend = service.least_busy(
    operational=True, simulator=False, min_num_qubits=127
)

print(backend.name)
```

```text
ibm_torino
```

### 2. יצירת Circuit ו-observable {#create-a-circuit-and-an-observable}

אתה צריך לפחות Circuit אחד ו-observable אחד כקלט ל-Estimator primitive.

```python
from qiskit.circuit.library import qaoa_ansatz
from qiskit.quantum_info import SparsePauliOp

entanglement = [tuple(edge) for edge in backend.coupling_map.get_edges()]
observable = SparsePauliOp.from_sparse_list(
    [("ZZ", [i, j], 0.5) for i, j in entanglement],
    num_qubits=backend.num_qubits,
)
circuit = qaoa_ansatz(observable, reps=2)
# the circuit is parametrized, so we will define the parameter values for execution
param_values = [0.1, 0.2, 0.3, 0.4]

print(f">>> Observable: {observable.paulis}")
```

ה-Circuit וה-observable צריכים להיות מומרים לשימוש רק בהוראות הנתמכות על ידי ה-QPU (המכונות מעגלים של *ארכיטקטורת מערכת הוראות (ISA)*). נשתמש ב-Transpiler לכך.

```python
from qiskit.transpiler import generate_preset_pass_manager

pm = generate_preset_pass_manager(optimization_level=1, backend=backend)
isa_circuit = pm.run(circuit)
isa_observable = observable.apply_layout(isa_circuit.layout)
print(f">>> Circuit ops (ISA): {isa_circuit.count_ops()}")
```

### 3. אתחול Qiskit Runtime Estimator {#initialize-qiskit-runtime-estimator}

בעת אתחול ה-Estimator, השתמש בפרמטר `mode` כדי לציין את המצב שבו ברצונך להריץ אותו. הערכים האפשריים הם אובייקטים מסוג `batch`, `session`, או `backend` עבור מצבי הרצה של אצווה, סשן ועבודה, בהתאמה. לקבלת מידע נוסף, ראה [Introduction to Qiskit Runtime execution modes.](execution-modes)

```python
from qiskit_ibm_runtime import EstimatorV2 as Estimator

estimator = Estimator(mode=backend)
```

### 4. הפעלת ה-Estimator וקבלת תוצאות {#invoke-the-estimator-and-get-results}

בשלב הבא, הפעל את המתודה `run()` לחישוב ערכי ציפייה עבור ה-Circuits וה-observables שהוזנו. ה-Circuit, ה-observable וערכי הפרמטרים האופציונליים מוזנים כ-tuples של *primitive unified bloc* (PUB).

```python
job = estimator.run([(isa_circuit, isa_observable, param_values)])
print(f">>> Job ID: {job.job_id()}")
print(f">>> Job Status: {job.status()}")
```

result = job.result()
print(f">>> {result}")

## תחילת העבודה עם Sampler {#start-sampler}

### 1. אתחול החשבון {#initialize-the-account-sampler}

מכיוון ש-Qiskit Runtime Sampler הוא שירות מנוהל, עליך תחילה לאתחל את החשבון שלך. לאחר מכן תוכל לבחור את ה-QPU שבו ברצונך להשתמש לחישוב ערך הציפייה.

פעל לפי השלבים ב-[נושא ההתקנה וההגדרה](install-qiskit) אם אין לך עדיין חשבון מוגדר.

```python
from qiskit_ibm_runtime import QiskitRuntimeService

service = QiskitRuntimeService()
backend = service.least_busy(
    operational=True, simulator=False, min_num_qubits=127
)
```

### 2. יצירת Circuit {#create-a-circuit}

אתה צריך לפחות Circuit אחד כקלט ל-Sampler primitive.

```python
import numpy as np
from qiskit.circuit.library import efficient_su2

circuit = efficient_su2(127, entanglement="linear")
circuit.measure_all()
# The circuit is parametrized, so we will define the parameter values for execution
param_values = np.random.rand(circuit.num_parameters)
```

השתמש ב-Transpiler כדי לקבל ISA Circuit.

```python
from qiskit.transpiler import generate_preset_pass_manager

pm = generate_preset_pass_manager(optimization_level=1, backend=backend)
isa_circuit = pm.run(circuit)
print(f">>> Circuit ops (ISA): {isa_circuit.count_ops()}")
```

```text
>>> Circuit ops (ISA): OrderedDict([('sx', 3089), ('rz', 3036), ('cz', 1092), ('measure', 127), ('barrier', 1)])
```

### 3. אתחול Qiskit Runtime Sampler {#initialize-the-qiskit-runtime-sampler}

בעת אתחול ה-Sampler, השתמש בפרמטר `mode` כדי לציין את המצב שבו ברצונך להריץ אותו. הערכים האפשריים הם אובייקטים מסוג `batch`, `session`, או `backend` עבור מצבי הרצה של אצווה, סשן ועבודה, בהתאמה. לקבלת מידע נוסף, ראה [Introduction to Qiskit Runtime execution modes.](execution-modes) שים לב שמשתמשי Open Plan אינם יכולים לשלוח עבודות סשן.

```python
from qiskit_ibm_runtime import SamplerV2 as Sampler

sampler = Sampler(mode=backend)
```

### 4. הפעלת ה-Sampler וקבלת תוצאות {#invoke-the-sampler-and-get-results}

בשלב הבא, הפעל את המתודה `run()` ליצירת הפלט. ה-Circuit וערכי הפרמטרים האופציונליים מוזנים כ-tuples של *primitive unified bloc* (PUB).

```python
job = sampler.run([(isa_circuit, param_values)])
print(f">>> Job ID: {job.job_id()}")
print(f">>> Job Status: {job.status()}")
```

## תחילת העבודה עם backend primitives {#backend}

בניגוד ל-primitives ספציפיים לספק, backend primitives הם מימושים גנריים שניתן להשתמש בהם עם כל אובייקט `backend` שרירותי, כל עוד הוא מממש את ממשק [`Backend`](https://docs.quantum.ibm.com/api/qiskit/qiskit.providers.Backend).

- ה-Sampler primitive ניתן להרצה עם כל ספק על ידי שימוש ב-[`qiskit.primitives.BackendSamplerV2`](https://docs.quantum.ibm.com/api/qiskit/qiskit.primitives.BackendSamplerV2).
- ה-Estimator primitive ניתן להרצה עם כל ספק על ידי שימוש ב-[`qiskit.primitives.BackendEstimatorV2`](../api/qiskit/qiskit.primitives.BackendEstimatorV2).

חלק מהספקים מממשים primitives באופן ילידי. ראה את [דף Qiskit Ecosystem](https://qiskit.github.io/ecosystem#provider) לפרטים.
### דוגמה: BackendEstimator {#example-backendestimator}

```python
from qiskit.primitives import BackendEstimatorV2
from <some_qiskit_provider> import QiskitProvider

provider = QiskitProvider()
backend = provider.get_backend('backend_name')
estimator = BackendEstimatorV2(backend)
```

### דוגמה: BackendSampler {#example-backendsampler}

```python
from qiskit.primitives import BackendSamplerV2
from <some_qiskit_provider> import QiskitProvider

provider = QiskitProvider()
backend = provider.get_backend('backend_name')
sampler = BackendSamplerV2(backend)
```
### דמיון והבדלים בין backend ל-Runtime primitives {#similarities-and-differences-between-backend-and-runtime-primitives}

- הקלטים והפלטים של [`qiskit.primitives.BackendSamplerV2`](https://docs.quantum.ibm.com/api/qiskit/qiskit.primitives.BackendSamplerV2) ו-[`qiskit.primitives.BackendEstimatorV2`](../api/qiskit/qiskit.primitives.BackendEstimatorV2)
עוקבים אחר אותו פורמט PUB כמו ה-primitives ב-Qiskit Runtime. ראה [Primitive inputs and outputs](primitive-input-output) לפרטים.
עם זאת, עשויים להיות הבדלים בשדות של המטא-דאטה המוחזר.

- מחלקת [`qiskit.primitives.BackendEstimatorV2`](https://docs.quantum.ibm.com/api/qiskit/qiskit.primitives.BackendEstimatorV2) אינה מציעה מימושי צמצום שגיאות מדידה או Gate מובנים, מכיוון שה-backend primitives מתוכננים לרוץ באופן מקומי על מחשב המשתמש.

- מחלקת [`qiskit.primitives.BackendSamplerV2`](https://docs.quantum.ibm.com/api/qiskit/qiskit.primitives.BackendSamplerV2) דורשת Backend התומך באפשרות `memory`.

- ממשקי ה-backend primitive חושפים `Options` מותאמים אישית של [`SamplerV2`](https://docs.quantum.ibm.com/api/qiskit/qiskit.primitives.BackendSamplerV2) ו-[`EstimatorV2`](https://docs.quantum.ibm.com/api/qiskit/qiskit.primitives.BackendEstimatorV2) השונים מהמימושים של Runtime.

# הבנת אחריותך בעת שימוש ב-Qiskit Runtime {#understanding-your-responsibilities-when-using-qiskit-runtime}

עיין בסעיפים הבאים לגבי האחריויות הספציפיות שלך ושל IBM&reg; בעת שימוש ב-Qiskit Runtime. לתנאי השימוש הכלליים, ראה [Terms and Notices](https://cloud.ibm.com/docs/overview/terms-of-use?topic=overview-terms) עבור IBM Cloud&reg;.

## ניהול אירועים ותפעול {#incident-and-operations-management}

ניהול אירועים ותפעול כולל משימות כגון ניטור, ניהול אירועים, זמינות גבוהה, זיהוי בעיות, שחזור, וגיבוי ושחזור מלאים.

| משימה | אחריות IBM | האחריות שלך |
| --- | --- | --- |
|ניהול תקינות תשתית|<ul><li>לספק ברציפות תכונות חדשות, עדכונים ותיקוני באגים לפי הצורך, בצורה שקופה.</li><li>לספק התראות על תחזוקה מתוכננת, עלוני אבטחה, או הפסקות בלתי מתוכננות.</li></ul> | <ul><li>[הגדר העדפות לקבלת דוא"ל](https://cloud.ibm.com/user/notifications) על התראות שירות.</li><li>עקוב אחר דף [IBM Cloud status](https://cloud.ibm.com/status) להודעות כלליות.</li></ul> |
|ניטור תקינות תשתית והתראות| <ul><li>לתקן את כל בעיות סביבת התשתית.</li><li>להודיע ​​ללקוחות על אירועים רלוונטיים.</li></ul> | <ul><li>להעריך את ההשפעה של כל אירוע מדווח.</li><li>לפנות ל-[IBM Support](/guides/support) לפי הצורך.</li></ul>|
|שמירה על SLA של זמינות גבוהה ב-IBM Cloud| להפעיל את Qiskit Runtime בהתאם ל-[IBM Cloud Public Service Level Agreements](https://cloud.ibm.com/docs/overview?topic=overview-slas) (SLAs). | לפעול לפי שיטות עבודה מומלצות לזמינות גבוהה. |

## ניהול שינויים {#change-management}

ניהול שינויים כולל משימות כגון פריסה, תצורה, שדרוגים, תיקוני קוד, שינויי תצורה ומחיקה.

| משימה | אחריות IBM | האחריות שלך |
| --- | --- | --- |
| שדרוגי גרסאות מרכזיות | לספק זמינות וכלים לעדכוני גרסאות מרכזיות. | לעבור לגרסה נתמכת לפני סיום התמיכה בגרסה ישנה. |
| שינויי גרסאות API | גרסאות ה-REST APIs. | שילוב עם הגרסה הרצויה של REST APIs. |
| עדכונים, תיקונים ותכונות חדשות | <ul><li>לספק עדכונים שוטפים, תיקוני באגים ותכונות חדשות, בצורה שקופה לפי מודל מסירה רציפה.</li><li>לפרסם הודעות על שינויים המשפיעים על לקוחות.</li></ul> | <ul><li>הגדר העדפות ל[קבלת התראות בדוא"ל](https://cloud.ibm.com/user/notifications) ועקוב אחר דף [IBM Cloud status](https://cloud.ibm.com/status) להודעות כלליות.</li><li>קרא הודעות שנוצרו אוטומטית ו[הכרזות פלטפורמה,](https://quantum.cloud.ibm.com/announcements) ונקוט פעולה מתאימה.</li></ul> |

## ניהול זהות וגישה {#identity-and-access-management}

ניהול זהות וגישה כולל משימות כגון אימות, הרשאה, ניהול מדיניות בקרת גישה, ואישור, מתן והסרת גישה.

| משימה | אחריות IBM | האחריות שלך |
| --- | --- | --- |
| זהות ובקרת גישה | לספק את האפשרות לנהל ולהגביל גישה למשאבים דרך IBM Cloud. | להעניק, להסיר ולנהל גישה למשאבים דרך [Cloud Identity and Access Management](/guides/access-groups) (IAM). |
| אימות שירות | להבטיח שרק משתמשים מאומתים יוכלו לגשת לשירותי ומשאבי Qiskit Runtime שלך. | <ul><li>[שמור על מפתח ה-API שלך](/guides/cloud-setup-untrusted#initialize-the-service-in-an-untrusted-environment) מגישה לא מורשית.</li><li>סב ועדכן את [מפתחות ה-API](https://cloud.ibm.com/iam/apikeys) שלך בהתאם לדרישות מדיניות האבטחה שלך.</li></ul> |
| ניצפות | לאפשר שילוב עם IBM Cloud Activity Tracker Event Routing לביקורת פעולות משתמש. | הגדר [IBM Cloud Activity Tracker Event Routing](https://cloud.ibm.com/docs/vmware-service?topic=vmware-service-at_events) או פונקציות אחרות למעקב אחר פעילות משתמשים. |

## אבטחה ותאימות לתקנות {#security-and-regulation-compliance}

אבטחה ותאימות לתקנות כוללת משימות כגון יישום בקרות אבטחה והסמכת תאימות.

| משימה | אחריות IBM | האחריות שלך |
| --- | --- | --- |
| כללי | <ul><li>לשמור על בקרות התואמות לסטנדרטים שונים של ציות לתעשייה.</li><li>לנטר, לבודד ולשחזר את פונקציונליות השירות.</li><li>לנטר ולדווח על תקינות ממשקי השירות.</li></ul> | להבטיח שלא מסופקים נתונים אישיים ומוסדרים. |
| פגיעויות | לנטר לאיתור בעיות פגיעות ותאימות אבטחה. | <ul><li>להישאר מעודכן לגבי פגיעויות ובעיות אבטחה אפשריות דרך עלוני אבטחה המספקים פעולות תיקון פוטנציאליות.</li><li>בדוק את [סעיף האבטחה של מאגר GitHub של Qiskit.](https://github.com/Qiskit/qiskit/security) כל פגיעויות האבטחה מפורסמות שם.</li><li>בדוק את [אתר מצב IBM Cloud](https://cloud.ibm.com/status) למציאת הכרזות ועלוני אבטחה על אירועים מרכזיים המשפיעים על פלטפורמת IBM Cloud, תשתית ושירותים מרכזיים.</li></ul> |
| הצפנה | להצפין נתונים בעת העברתם על כל רשתות ציבוריות ובתוך רשת מרכז הנתונים הפרטי של שירות הענן. שיטות הצפנה כגון HTTPS ו-TLS משמשות להגנה על נתונים בתנועה. | להבטיח, לפי הצורך, שהחיבור שלך מוצפן מקצה לקצה. |
| אבטחה | <ul><li>להבטיח את אבטחת הנתונים על הדיסק ונתונים בתנועה בתוך התשתית שלה.</li><li>לספק שירות מאובטח התואם לסטנדרטים מרכזיים.</li></ul> | הגבל [גישת משתמש](/guides/access-groups) למשאבים ולקבוצות משאבים המתאימות. |

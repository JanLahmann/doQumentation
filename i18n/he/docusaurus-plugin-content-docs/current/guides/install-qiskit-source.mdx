# התקנת Qiskit SDK ו-Qiskit Runtime מקוד המקור {#install-the-qiskit-sdk-and-qiskit-runtime-from-source}

התקנת Qiskit SDK מקוד המקור מאפשרת לך לגשת לגרסת הפיתוח הנוכחית, במקום להשתמש בגרסה שנמצאת במאגר Python Package Index‏ (PyPI). כך תוכל לבחון ולהרחיב את הגרסה העדכנית ביותר של קוד Qiskit ביתר יעילות.

## יצירה והפעלה של סביבה וירטואלית חדשה {#create-and-activate-a-new-virtual-environment}

1. נווט לתיקיית הפרויקט שלך וצור סביבה מינימלית עם Python בלבד מותקן בה.

          </TabItem>

          <TabItem value="win" label="Windows">

## התקנת מהדר Rust {#install-the-rust-compiler}

על מנת לקמפל את Qiskit, חייב להיות מותקן מהדר Rust במערכת שלך. להתקנת מהדר Rust, השתמש במתקין הרב-פלטפורמי [`rustup`](https://rustup.rs/) או [בשיטת התקנה אחרת.](https://forge.rust-lang.org/infra/other-installation-methods.html)

## התקנת Qiskit {#install-qiskit}

עקוב אחר השלבים הבאים להתקנת Qiskit:

1. שכפל את מאגר Qiskit.

```bash
git clone https://github.com/Qiskit/qiskit.git
```

2. עבור לתיקיית `qiskit`.

```bash
cd qiskit
```

3. (אופציונלי) אם ברצונך להריץ בדיקות או בדיקות lint, התקן את דרישות המפתח.

```bash
pip install -r requirements-dev.txt
```

4. התקן את `qiskit`.

  * **התקנה רגילה**:

    ```bash
    pip install .
    ```

  * **מצב עריכה**: במצב זה אינך צריך להתקין מחדש את Qiskit כשיש שינויי קוד בפרויקט.

    ```bash
    pip install -e .
    ```

    במצב עריכה, ההרחבות המקומפלות נבנות ב-_מצב ניפוי שגיאות_ ללא אופטימיזציות. הדבר משפיע על ביצועי זמן הריצה של הקוד המקומפל. כדי לבנות את ההרחבות המקומפלות עם אופטימיזציות מופעלות, הרץ את הפקודה הבאה כדי לבנות מחדש את הבינארי ב-_מצב שחרור_:

    ```bash
    python setup.py build_rust --release --inplace
    ```

    <Admonition type="note">
    אם אתה עובד על קוד Rust ב-Qiskit, עליך לבנות מחדש את קוד ההרחבה בכל פעם שאתה מבצע שינוי מקומי. במצב עריכה, הרחבת Rust נבנית רק כשפקודת ההתקנה מורצת, לכן שינויים מקומיים שאתה מבצע בקוד Rust אינם באים לידי ביטוי בחבילה המותקנת אלא אם כן תבנה מחדש את ההרחבה על ידי הרצה חוזרת של `build_rust` (עם `--release` או בלעדיו, בהתאם לכך אם ברצונך לבנות במצב שחרור או מצב ניפוי שגיאות).
    </Admonition>

## התקנת Qiskit Runtime {#install-qiskit-runtime}

עקוב אחר השלבים הבאים אם ברצונך להתקין את Qiskit Runtime:

1. שכפל את מאגר Qiskit Runtime.

```bash
git clone https://github.com/Qiskit/qiskit-ibm-runtime.git
```

2. עבור לתיקיית `qiskit-ibm-runtime`.

```bash
cd qiskit-ibm-runtime
```

3. התקן את `qiskit-runtime`. אנו ממליצים להשתמש ב[סביבה וירטואלית](https://docs.python.org/3/library/venv.html) כדי להימנע מזיהום התקנת Python הגלובלית שלך.

  * **התקנה רגילה**:

    ```bash
    pip install .
    ```

  * **מצב עריכה**: במצב זה אינך צריך להתקין מחדש את Qiskit כשיש שינויי קוד בפרויקט.

    ```bash
    pip install -e .
    ```

    במצב עריכה, ההרחבות המקומפלות נבנות ב-_מצב ניפוי שגיאות_ ללא אופטימיזציות.

4. (אופציונלי) אם ברצונך להריץ בדיקות או בדיקות lint, התקן את דרישות המפתח. אנו ממליצים להשתמש ב[סביבה וירטואלית](https://docs.python.org/3/library/venv.html) כדי להימנע מזיהום התקנת Python הגלובלית שלך.

```bash
pip install -e ".[dev]"
```

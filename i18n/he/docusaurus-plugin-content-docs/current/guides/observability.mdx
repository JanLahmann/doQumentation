# אירועי מעקב פעילות עבור Qiskit Runtime {#activity-tracking-events-for-qiskit-runtime}

שירותי IBM Cloud&reg;, כגון Qiskit Runtime, מייצרים אירועי מעקב פעילות.

אירועי מעקב פעילות מדווחים על פעולות המשנות את מצב השירות ב-IBM Cloud. ניתן להשתמש באירועים כדי לחקור פעילות חריגה ופעולות קריטיות ולעמוד בדרישות ביקורת רגולטוריות.

ניתן להשתמש ב-IBM Cloud Activity Tracker Event Routing, שירות פלטפורמה, לניתוב אירועי ביקורת בחשבון שלך ליעדים שתבחר על ידי הגדרת יעדים ומסלולים המגדירים לאן נשלחים אירועי המעקב. למידע נוסף, ראה [אודות ניתוב אירועי IBM Cloud Activity Tracker.](https://cloud.ibm.com/docs/atracker?topic=atracker-about)

ניתן להשתמש ב-IBM Cloud Logs לצפייה בהתראות על אירועים שנוצרים בחשבון שלך ומנותבים על ידי IBM Cloud Activity Tracker Event Routing למופע IBM Cloud Logs.

## מיקומים שבהם נוצרים אירועי מעקב פעילות {#locations-where-activity-tracking-events-are-generated}

Qiskit Runtime שולח אירועי מעקב פעילות באזורים הבאים:

- וושינגטון ("us-east")
- פרנקפורט ("eu-de")

## צפייה באירועי מעקב פעילות עבור Qiskit Runtime {#viewing-activity-tracking-events-for-qiskit-runtime}

ניתן להשתמש ב-IBM Cloud Logs לצפייה ולהתראות על אירועים שנוצרים בחשבון שלך ומנותבים על ידי IBM Cloud Activity Tracker Event Routing למופע IBM Cloud Logs.

### הפעלת IBM Cloud Logs מדף ה-Observability {#launching-ibm-cloud-logs-from-the-observability-page}

למידע על הפעלת ממשק המשתמש של IBM Cloud Logs, ראה [הפעלת ממשק המשתמש](https://cloud.ibm.com/docs/cloud-logs?topic=cloud-logs-instance-launch#instance-launch-cloud-ui) בתיעוד IBM Cloud Logs.

## סקירה כללית {#overview}

פונקציית Circuit של IBM&reg; מקבלת [PUBs מופשטים](/guides/primitive-input-output#pubs) כקלט ומחזירה ערכי ציפייה מותאמים כפלט. פונקציית Circuit זו כוללת צינור עיבוד אוטומטי ומותאם אישית כדי לאפשר לחוקרים להתמקד בגילוי אלגוריתמים ויישומים.

## תיאור {#description}

לאחר שתגיש את ה-PUB שלך, המעגלים המופשטים והאובזרבלים שלך יעברו טרנספילציה אוטומטית, הרצה על חומרה ועיבוד-לאחר כדי להחזיר ערכי ציפייה מותאמים. לשם כך, הפונקציה משלבת את הכלים הבאים:

- [שירות Qiskit Transpiler](/guides/qiskit-transpiler-service), כולל בחירה אוטומטית של מעברי טרנספילציה מונחי בינה מלאכותית והיוריסטיים לתרגום המעגלים המופשטים שלך למעגלי ISA מותאמי חומרה
- [טכניקות דיכוי ומיתון שגיאות הנדרשות לחישוב בקנה מידה שימושי](/guides/error-mitigation-and-suppression-techniques), כולל twirling של מדידות ושערים, ניתוק דינמי, Twirled Readout Error eXtinction‏ (TREX),‏ Zero-Noise Extrapolation‏ (ZNE) ו-Probabilistic Error Amplification‏ (PEA)
- [Qiskit Runtime Estimator](/guides/get-started-with-estimator), להרצת PUBs מסוג ISA על חומרה והחזרת ערכי ציפייה מותאמים

![IBM Circuit function](/docs/images/guides/ibm-circuit-function/ibm-circuit-function.svg)

## דוגמאות {#examples}

כדי להתחיל, נסה את הדוגמה הבסיסית הזו:

```python
from qiskit.circuit.random import random_circuit
from qiskit_ibm_runtime import QiskitRuntimeService

# You can skip this step if you have a target backend, e.g.
# backend_name = "ibm_brisbane"
# You'll need to specify the credentials when initializing QiskitRuntimeService, if they were not previously saved.
service = QiskitRuntimeService()
backend = service.least_busy(operational=True, simulator=False)

circuit = random_circuit(num_qubits=2, depth=2, seed=42)
observable = "Z" * circuit.num_qubits
pubs = [(circuit, observable)]

job = function.run(
    # Use `backend_name=backend_name` if you didn't initialize a backend object
    backend_name=backend.name,
    pubs=pubs,
)
```

בדוק את [הסטטוס](/guides/functions#check-job-status) של עומס העבודה של פונקציית Qiskit שלך או אחזר את [התוצאות](/guides/functions#retrieve-results) כדלקמן:

```python
print(job.status())
result = job.result()
```

```text
QUEUED
```

לתוצאות יש את אותו פורמט כמו [תוצאת Estimator](/guides/estimator-input-output):

```python
print(f"The result of the submitted job had {len(result)} PUB\n")

    f"The expectation values measured from this PUB are: \n{result[0].data.evs}"
)
```

### דוגמאות לרמת מיתון {#mitigation-level-examples}

הדוגמה הבאה מדגימה הגדרת רמת המיתון:

```python
options = {"mitigation_level": 2}

job = function.run(backend_name=backend.name, pubs=pubs, options=options)
```

בדוגמה הבאה, הגדרת רמת המיתון ל-1 מכבה בתחילה את מיתון ZNE, אך הגדרת `zne_mitigation` ל-`True` עוקפת את ההגדרה הרלוונטית מ-`mitigation_level`.

```python
options = {"mitigation_level": 1, "resilience": {"zne_mitigation": True}}
```

### דוגמת פלט {#output-example}

קטע הקוד הבא מתאר את פורמט ה-`PrimitiveResult` (וה-`PubResult` המשויך).

```python
print(f"The result of the submitted job had {len(result)} PUB")
print(
    f"The expectation values measured from this PUB are: \n{result[0].data.evs}"
)
print(f"And the associated metadata is: \n{result[0].metadata}")
```


### אחזור הודעות שגיאה {#fetch-error-messages}

אם סטטוס עומס העבודה שלך הוא `ERROR`, השתמש ב-`job.result()` כדי לאחזר את הודעת השגיאה לצורך ניפוי באגים:

```python
job = function.run(
    backend_name="bad_backend_name", pubs=pubs, options=options
)

print(job.result())
```

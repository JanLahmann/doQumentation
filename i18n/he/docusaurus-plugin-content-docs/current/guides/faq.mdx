## כיצד לצטט מחשב IBM Quantum במחקר שלי? {#how-do-i-cite-an-ibm-quantum-computer-in-my-research}

למאמרי מחקר, אנו ממליצים למחברים להודות ל-IBM Quantum&reg; בנוסח הבא:

>    We acknowledge the use of IBM Quantum services for this work. The views expressed are those of the authors, and do not reflect the official policy or position of IBM or the IBM Quantum team.

הפניות ביבליוגרפיות במאמר צריכות להשתמש בציטוט הבא:

> IBM Quantum. https://<span>quantum.cloud.ibm.com</span>/, YYYY

יחידות עיבוד קוונטיות במאמר צריכות להיות מוזכרות לפי שמן הייחודי (למשל `ibmq_vigo`), ואפשר להוסיף גם את גרסתן (למשל `ibmq_vigo` v1.0.2). אנו ממליצים גם להפנות למעבד עצמו. לדוגמה:

> In this paper we used `ibmq_vigo`, which is one of the IBM Quantum Canary processors.

דוגמה לציטוט תוכנית IBM Quantum:

> IBM Quantum (2025). Estimator primitive (Version x.y.z) [computer software]. https://<span>quantum.cloud.ibm.com</span>/

נסה לבחור חשבון אחר דרך מתג החשבונות בכותרת ממשק [IBM Quantum Platform](https://quantum.cloud.ibm.com).

## אני משתמש רק בתוכנית החינמית (Open Plan), אך נדרשתי להזין פרטי כרטיס אשראי. האם אחויב? {#im-just-using-the-open-plan-but-i-had-to-put-in-my-credit-card-will-i-be-charged}

לא תחויב אלא אם תצור [מופע](/guides/instances) מסוג Pay-As-You-Go ותשתמש בו לשליחת עומסי עבודה ל-QPU. ראה [ניהול עלויות](/guides/manage-cost) להוראות על קביעת מגבלת עלות ודרכים נוספות לניהול עלויות.

## מה ההבדל בין API key ל-API token? {#whats-the-difference-between-an-api-key-and-an-api-token}

אין הבדל. המונחים API key ו-API token משמשים לסירוגין. תמצא את שני המונחים בתיעוד מאחר שממשקים שונים משתמשים בטרמינולוגיה שונה.

<span id="bearer"></span>
## מה ההבדל בין API token ל-bearer token? {#whats-the-difference-between-an-api-token-and-a-bearer-token}

API token הוא אישור לטווח ארוך המשמש ל[אימות מול שירות Qiskit Runtime.](/guides/hello-world#install-and-authenticate) Bearer token הוא אישור זמני המשמש ל[אימות מול REST API בסביבה לא מהימנה.](/guides/cloud-setup-untrusted#authenticate-to-the-rest-api-in-an-untrusted-environment) לא ניתן להשתמש ב-bearer token לאימות מול שירות Qiskit Runtime.

## האם עלי לבחור את האזור שבו אני גר? {#do-i-have-to-choose-the-region-that-i-live-in}

לא. אתה יכול להשתמש בכל אחד מהאזורים. האזור קובע היכן עובדות מורצות והיכן נשמרים הנתונים. בנוסף, תוכל לראות ולהשתמש רק במופעים שנוצרו באזור שאליו אתה מחובר. שים לב שרק האזור us-east תומך בתוכנית החינמית (Open Plan).

## האם ניתן להתחבר דרך שרת פרוקסי? {#can-i-connect-through-a-proxy-server}

כן, אך עליך להשתמש ב-Qiskit Runtime גרסה 0.44.0 ומעלה.

## מהי מדיניות שמירת הנתונים? {#what-is-the-data-retention-policy}

IBM שומרת את נתוני העבודות והשימוש שלך (שלא נמחקו) למשך שלוש שנים, כל עוד אתה ממשיך להשתמש בשירות Qiskit Runtime.

## מה קרה להפצות Python כמו Qiskit Aer, Qiskit Nature ו-Qiskit Experiments? {#what-happened-to-python-distributions-such-as-qiskit-aer-qiskit-nature-and-qiskit-experiments}

מאז שחרור Qiskit SDK v1.0, הפקודה `pip install qiskit` תתקין **רק** את [Qiskit SDK](https://github.com/Qiskit/qiskit). המשמעות היא שחלק מהמודולים שהשתמשת בהם בעבר, כגון `qiskit-aer` או `qiskit-nature`, כעת יש להתקין בנפרד.

לפני שחרור Qiskit SDK v1.0, Qiskit כלל הפצות Python רבות ושונות, כגון `qiskit-terra`, ליבת המהדר; `qiskit-aer`, הסימולטור בעל הביצועים הגבוהים; הספק המקורי של IBM Quantum&reg;; ומספר חבילות שכיום אינן בשימוש, שסיפקו תכונות אלגוריתמיות ניסיוניות או הרצת ניסויים.

חבילות התוכנה הנפרדות הללו אוחדו למה שנקרא ה-*metapackage* של Qiskit, שאפשר למשתמשים להתקין את כל הרכיבים יחד דרך `pip install qiskit`. ארכיטקטורת ה-metapackage הוסרה עם שחרור Qiskit SDK v1.0, וכעת כל הפצת תוכנה בנפרד צריכה להיות מותקנת בנפרד.

אם ברצונך להתקין את החבילות שנכללו בעבר ב-metapackage של Qiskit, בקר ב[מערכת האקולוגית של Qiskit](https://www.ibm.com/quantum/ecosystem) למציאת מגוון חבילות שיתאימו לצרכיך. תוכל גם לקרוא את [מדריך המעבר ל-v1.0](/guides/metapackage-migration) לקבלת מידע נוסף על שינוי זה.

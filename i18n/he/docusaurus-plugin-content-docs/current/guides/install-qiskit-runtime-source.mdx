# התקנת לקוח Qiskit Runtime מהקוד המקור {#install-the-qiskit-runtime-client-from-source}

כשמתקינים את Qiskit Runtime מהקוד המקור, אפשר לגשת לגרסת הפיתוח הנוכחית, במקום להשתמש בגרסה שבמאגר Python Package Index (PyPI). לכן אפשר לבחון ולהרחיב את הגרסה האחרונה של הקוד בצורה יעילה יותר.

1. נווט לספריית הפרויקט שלך וצור סביבה מינימלית עם Python בלבד.

## התקנת Qiskit Runtime {#install-qiskit-runtime}

בצע את השלבים הבאים כדי להתקין את Qiskit Runtime:

1. שכפל את מאגר Qiskit Runtime.

```bash
git clone https://github.com/Qiskit/qiskit-ibm-runtime.git
```

2. עבור לספריית `qiskit-ibm-runtime`.

```bash
cd qiskit-ibm-runtime
```

3. התקן את `qiskit-runtime`. אנחנו ממליצים להשתמש ב[סביבה וירטואלית](https://docs.python.org/3/library/venv.html) כדי להימנע מזיהום ההתקנה הגלובלית של Python שלך.

  * **התקנה סטנדרטית**:

    ```bash
    pip install .
    ```

  * **מצב עריכה**: במצב זה, אינך צריך להתקין מחדש את Qiskit כשיש שינויי קוד בפרויקט.

    ```bash
    pip install -e .
    ```

    במצב עריכה, ההרחבות המורכבות נבנות ב_מצב ניפוי באגים_ ללא אופטימיזציות.

4. (אופציונלי) אם אתה רוצה להריץ בדיקות או בדיקות איכות קוד, התקן את דרישות המפתח. אנחנו ממליצים להשתמש ב[סביבה וירטואלית](https://docs.python.org/3/library/venv.html) כדי להימנע מזיהום ההתקנה הגלובלית של Python שלך.

```bash
pip install -e ".[dev]"
```

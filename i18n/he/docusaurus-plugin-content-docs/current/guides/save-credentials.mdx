# שמירת פרטי הכניסה שלך {#save-your-login-credentials}
אם אתה עובד בסביבת Python מהימנה (כגון מחשב נייד אישי או תחנת עבודה), אתה יכול להשתמש במתודה `save_account()` כדי לשמור את פרטי הכניסה שלך באופן מקומי, ולאחר מכן להשתמש בהם לאתחול השירות.

<Admonition type="note" title="הערות">
      * אם אתם משתמשים במחשב ציבורי או בסביבה לא מהימנה אחרת, עקבו במקום זאת אחרי ההוראות ב[אתחול השירות בסביבה לא מהימנה](/guides/cloud-setup-untrusted).
      * עקבו אחרי [ההוראות האלה](/guides/cloud-setup-rest-api) אם אתם רוצים להתחבר דרך REST API במקום דרך Qiskit.
      * אם צריך, השתמשו ב[מידע הזה](/guides/quickstart-steps-org#firewall) כדי להגדיר את חומת האש שלכם לאפשר גישה לנקודות הקצה של IBM Quantum API.
</Admonition>

## לפני שמתחילים {#before-you-begin}

1. ודא שיש לך [חשבון IBM Cloud.](https://quantum.cloud.ibm.com/registration)
1. ודא שאתה עובד בסביבת Python פעילה עם [Qiskit SDK ו-Qiskit Runtime מותקנים](/guides/install-qiskit#local).
1. הפעל את סביבת ה-virtual Python ו-Python בסביבה הוירטואלית שלך.
1. התחבר ל-[IBM Quantum Platform](https://quantum.cloud.ibm.com) עם IBMid או חשבון Google.

<span id="find-credentials"></span>
## מציאת פרטי הגישה שלך {#find-your-access-credentials}

1. ודא שהחשבון והאזור הנכונים נבחרו במתג החשבונות בכותרת.
1. מצא את מפתח ה-API שלך. מה-[dashboard](https://quantum.cloud.ibm.com/), צור את מפתח ה-API שלך, העתק אותו למיקום מאובטח כדי שתוכל להשתמש בו לאימות. שים לב שאתה יכול להשתמש באותו מפתח API כדי להתחבר לכל אזור.
1. אופציונלי: מצא את ה-instance שברצונך להשתמש בו מדף [Instances](https://quantum.cloud.ibm.com/instances). רחף מעל ה-CRN שלו, לחץ על הסמל כדי להעתיק אותו, ולאחר מכן שמור אותו במיקום מאובטח כדי שתוכל להשתמש בו לזיהוי ה-instance.

<span id="cloud-save"></span>
## שמירת פרטי הגישה שלך {#save-your-access-credentials}

שמור את פרטי הכניסה שלך על ידי הרצת הקוד המתאים פעם אחת לכל חשבון שברצונך לשמור. לאחר שמירת פרטי הכניסה שלך, טען אותם על ידי ביצוע השלבים ב[אתחול שירות Qiskit Runtime.](/guides/initialize-account)

<span id="save-instance"></span>
### שמירת פרטי כניסה לגישה ל-instance ספציפי: {#save-credentials-that-access-a-specific-instance}
אם יש לך מספר instances וברצונך לציין ל-Qiskit Runtime איזה instance להשתמש, שמור פרטי כניסה הכוללים CRN של instance.

    ```python
    from qiskit_ibm_runtime import QiskitRuntimeService

    QiskitRuntimeService.save_account(
      # For `token`, use the 44-character API_KEY you created 
      # and saved from the IBM Quantum Platform Home dashboard:
      token="<your-api-key>", 
      name="<account-name>", # Optional
      instance="<IBM Cloud CRN or instance name>", # Optional
      set_as_default=True, # Optional
      overwrite=True, # Optional
    )
    ```

<span id="auto-select"></span>
### שמירת פרטי כניסה לבחירת instance אוטומטית: {#save-credentials-for-automatic-instance-selection}
אם אינך מספק CRN של instance ומעביר פרטי כניסה אלה ל-Qiskit Runtime, instance מתאים ייבחר עבורך בהתאם לאפשרויות שצוינו.

    ```python
    from qiskit_ibm_runtime import QiskitRuntimeService

    QiskitRuntimeService.save_account(
      # For `token`, use the 44-character API_KEY you created 
      # and saved from the IBM Quantum Platform Home dashboard:
      token="<your=api-key>", 
      name="<account-name>", # Optional
      instance="<instance-to-use>", # Optional
      plans_preference="<plan_types>", # Optional
      region="<region>", # Optional
      tags="<instance-tags>", # Optional
      set_as_default=True, # Optional
      overwrite=True, # Optional
    )
   ```

### אפשרויות זמינות {#available-options}

- **`token`**: מפתח API של IBM Cloud. הטוקן שלך סודי. אל תשתף את הטוקן שלך בקוד ציבורי.
- **`instance`**: ציין אופציונלית את ה-instance לשימוש דרך ה-IBM Cloud CRN או שם ה-instance שלו.
- **`plans_preference`**: הגדר אופציונלית את סוגי התוכניות שיש לתת להן עדיפות. אפשרות זו מתעלמת אם ה-instance מצוין. האפשרויות הזמינות הן `open`, `pay-as-you-go`, `flex`, `premium` ו-`on-prem`. Instances מסוג תוכנית מסוים נשללים אם שם התוכנית אינו מצוין. לדוגמה, אם מועבר [`open`], רק instances של Open Plan זמינים. אפשרות זו מתעלמת אם `instance` מצוין.
- **`region`**: הגדר אופציונלית את האזור לשימוש. הערכים המקובלים הם `us-east` ו-`eu-de`. אפשרות זו מתעלמת אם `instance` מצוין.
- **`tags`**: ציין אופציונלית את תגיות ה-instance. מקבל רשימה של מחרוזות שם תגית. אפשרות זו מתעלמת אם `instance` מצוין.
- **`name`**: שם אופציונלי לסט פרטי כניסה זה לחשבון.
- **`set_as_default`**: הגדר את הערך ל-`True` כדי לשמור אלה כפרטי ברירת המחדל שלך. אם אתה שומר רק חשבון אחד, הוא נקבע אוטומטית כברירת המחדל.
- **`overwrite`**: הגדר ערך זה ל-`True` כדי לעדכן את פרטי ברירת המחדל שלך.

## דוגמאות {#examples}

### דוגמה 1 {#example-1}
דוגמה זו שומרת פרטי כניסה ל-instances ספציפיים, המאפשרים גישה open ו-premium. פרטי הכניסה הפתוחים נקבעים כברירת מחדל.

```python
from qiskit_ibm_runtime import QiskitRuntimeService

QiskitRuntimeService.save_account(token="<API_TOKEN>", 
                                  instance="<CRN_for_premium_instance>",
                                  name="premium")
QiskitRuntimeService.save_account(token="<API_TOKEN>", 
                                  instance="<CRN_for_open_instance>",
                                  name="open", set_as_default=True)
 ```

### דוגמה 2 {#example-2}
דוגמה זו שומרת פרטי כניסה לבחירת instance אוטומטית. Qiskit Runtime יסתכל רק על instances premium באזור ה-EU הזמינים לחשבון. פרטי כניסה אלה הופכים לברירת המחדל החדשה.

```python
QiskitRuntimeService.save_account(
    token="your-api-key",
    set_as_default=True,
    overwrite = True,

    # Set instance "filters" instead of specifying the instance:
    region="eu-de",
    plans_preference=["premium"]
  )
 ```

## צפייה בפרטי כניסה שמורים {#view-saved-credentials}

כדי לצפות בכל פרטי הכניסה שמרת, הרץ `service.saved_accounts()`. שים לב שאם שמרת פרטי כניסה כברירת מחדל ולא נתת להם שם, הם ייקראו `default-ibm-quantum-platform`. אם נתת שם לפרטי ברירת המחדל שלך, תראה `"is_default_account": true` בפלט.

## שיקולים {#considerations}

* אם אתה שומר מספר חשבונות, השתמש בפרמטר `name` כדי להבחין ביניהם.
* פרטי הכניסה נשמרים ב-`$HOME/.qiskit/qiskit-ibm.json`. אל תערוך קובץ זה ידנית.
* אם אינך שומר את פרטי הכניסה שלך, עליך לציין אותם בכל פעם שאתה מתחיל session חדש (מאתחל את שירות Qiskit Runtime).
* אם אתה מציין ידנית את פרטי הכניסה שלך, חשבון שמור לא ישמש.

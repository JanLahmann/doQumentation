כדי לאפשר פיתוח ושימוש חוזר בקוד transpilation מותאם אישית על-ידי קהילת משתמשי Qiskit הרחבה, ה-Qiskit SDK תומך בממשק תוספים המאפשר לחבילות Python של צד שלישי להצהיר שהן מספקות פונקציונליות transpilation מורחבת הנגישה דרך Qiskit.

כיום, תוספים של צד שלישי יכולים לספק פונקציונליות transpilation מורחבת בשלוש דרכים:

- [תוסף שלב Transpiler](https://docs.quantum.ibm.com/api/qiskit/transpiler_plugins) מספק pass manager שניתן להשתמש בו במקום אחד מ-[6 השלבים](transpiler-stages) של pass manager מדורג מוגדר מראש: `init`, `layout`, `routing`, `translation`, `optimization`, ו-`scheduling`.
- [תוסף סינתזת יוניטרי](https://docs.quantum.ibm.com/api/qiskit/qiskit.transpiler.passes.synthesis.plugin.UnitarySynthesisPlugin) מספק פונקציונליות מורחבת לסינתזת שערי יוניטרי.
- [תוסף סינתזה ברמה גבוהה](https://docs.quantum.ibm.com/api/qiskit/qiskit.transpiler.passes.synthesis.plugin.HighLevelSynthesisPlugin) מספק פונקציונליות מורחבת לסינתזת "אובייקטים ברמה גבוהה" כגון פונקציות לינאריות או אופרטורי Clifford. אובייקטים ברמה גבוהה מיוצגים על-ידי מחלקות משנה של מחלקת [Operation](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.Operation).

שאר הדף מתאר כיצד לרשום תוספים זמינים, להתקין חדשים ולהשתמש בהם.
## רשימת תוספים זמינים והתקנת חדשים {#list-available-plugins-and-install-new-ones}

Qiskit כבר כולל כמה תוספים מובנים לצורך transpilation. כדי להתקין עוד, ניתן להשתמש במנהל החבילות של Python. לדוגמה, אפשר להריץ `pip install qiskit-toqm` כדי להתקין את תוסף שלב הניתוב [Qiskit TOQM](https://github.com/qiskit-toqm/qiskit-toqm). מספר תוספים של צד שלישי הם חלק מ-[מערכת האקולוגית של Qiskit](https://qiskit.github.io/ecosystem/#transpiler_plugin).

## שימוש בתוסף {#use-a-plugin}

בחלק זה נראה כיצד להשתמש בתוספי Transpiler. בדוגמאות הקוד אנו משתמשים בתוספים שמגיעים עם Qiskit, אך תוספים שמותקנים מחבילות צד שלישי משמשים באותה הדרך.

### שימוש בתוסף שלב Transpiler {#use-a-transpiler-stage-plugin}

כדי להשתמש בתוסף שלב Transpiler, יש לציין את שמו עם הארגומנט המתאים ל-[`generate_preset_pass_manager`](https://docs.quantum.ibm.com/api/qiskit/qiskit.transpiler.generate_preset_pass_manager#qiskit.transpiler.generate_preset_pass_manager) או ל-[`transpile`](https://docs.quantum.ibm.com/api/qiskit/compiler#qiskit.compiler.transpile). הארגומנט נוצר על-ידי הוספת `_method` לשם שלב ה-transpilation. לדוגמה, כדי להשתמש בתוסף הניתוב `lookahead`, נציין `lookahead` עבור ארגומנט `routing_method`:

```python
from qiskit.transpiler import generate_preset_pass_manager
from qiskit_ibm_runtime import QiskitRuntimeService

service = QiskitRuntimeService()
backend = service.backend("ibm_fez")

pass_manager = generate_preset_pass_manager(
    optimization_level=3, backend=backend, routing_method="lookahead"
)
```

### שימוש בתוסף סינתזת יוניטרי {#use-a-unitary-synthesis-plugin}

כדי להשתמש בתוסף סינתזת יוניטרי, יש לציין את שמו כארגומנט `unitary_synthesis_method` ל-[`generate_preset_pass_manager`](https://docs.quantum.ibm.com/api/qiskit/qiskit.transpiler.generate_preset_pass_manager#qiskit.transpiler.generate_preset_pass_manager) או ל-[`transpile`](https://docs.quantum.ibm.com/api/qiskit/compiler#qiskit.compiler.transpile):

```python
pass_manager = generate_preset_pass_manager(
    optimization_level=3,
    backend=backend,
    unitary_synthesis_method="sk",
    unitary_synthesis_plugin_config=dict(
        basis_gates=["cz", "id", "rz", "sx", "x"]
    ),
)
```

סינתזת יוניטרי משמשת בשלבי `init`, `translation` ו-`optimization` של ה-pass manager המדורג שמוחזר על-ידי [`generate_preset_pass_manager`](https://docs.quantum.ibm.com/api/qiskit/qiskit.transpiler.generate_preset_pass_manager#qiskit.transpiler.generate_preset_pass_manager) או שמשמש ב-[`transpile`](https://docs.quantum.ibm.com/api/qiskit/compiler#qiskit.compiler.transpile). ראה [שלבי Transpiler](transpiler-stages) לתיאור שלבים אלו.

השתמש בארגומנט `unitary_synthesis_plugin_config`, מילון חופשי, כדי להעביר אפשרויות לשיטת סינתזת היוניטרי. התיעוד של שיטת הסינתזה אמור להסביר את האפשרויות שהיא תומכת בהן. ראה [רשימה זו](https://docs.quantum.ibm.com/api/qiskit/transpiler_synthesis_plugins#unitary-synthesis-plugins-2) לקישורים לתיעוד של תוספי סינתזת היוניטרי המובנים.
### שימוש בתוסף סינתזה ברמה גבוהה {#use-a-high-level-synthesis-plugin}

ראשית, צור [HLSConfig](https://docs.quantum.ibm.com/api/qiskit/qiskit.transpiler.passes.HLSConfig) כדי
לאחסן את שמות התוספים לשימוש עבור אובייקטים שונים ברמה גבוהה. לדוגמה:

```python
from qiskit.transpiler.passes import HLSConfig

hls_config = HLSConfig(clifford=["layers"], linear_function=["pmh"])
```

תא קוד זה יוצר הגדרת סינתזה ברמה גבוהה שמשתמשת בתוסף `layers`
לסינתזת אובייקטי [Clifford](https://docs.quantum.ibm.com/api/qiskit/qiskit.quantum_info.Clifford#clifford) ובתוסף `pmh` לסינתזת
אובייקטי [LinearFunction](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.library.LinearFunction#linearfunction). שמות הארגומנטים של מילות המפתח תואמים לתכונה
[`name`](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.Operation#name) של מחלקת [Operation](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.Operation) המייצגת את סוג האובייקט שמסנתזים.
עבור כל אובייקט ברמה גבוהה, רשימת התוספים הנתונה מנוסה ברצף עד שאחד מהם מצליח (בדוגמה לעיל, כל רשימה מכילה רק תוסף בודד).

בנוסף לציון תוסף לפי שמו, ניתן גם להעביר טאפל `(name, options)`, כאשר האלמנט השני בטאפל הוא מילון המכיל אפשרויות לתוסף. התיעוד של שיטת הסינתזה אמור להסביר את האפשרויות שהיא תומכת בהן. ראה [רשימה זו](https://docs.quantum.ibm.com/api/qiskit/transpiler_synthesis_plugins#high-level-synthesis-plugins-2) לקישורים לתיעוד של תוספי הסינתזה ברמה גבוהה המובנים.

לאחר שיצרת את האובייקט `HLSConfig`, העבר אותו כארגומנט
`hls_config` ל-[`generate_preset_pass_manager`](https://docs.quantum.ibm.com/api/qiskit/qiskit.transpiler.generate_preset_pass_manager#qiskit.transpiler.generate_preset_pass_manager) או ל-[`transpile`](https://docs.quantum.ibm.com/api/qiskit/compiler#qiskit.compiler.transpile):

```python
pass_manager = generate_preset_pass_manager(
    optimization_level=3, backend=backend, hls_config=hls_config
)
```

סינתזה ברמה גבוהה משמשת בשלבי `init`, `translation` ו-`optimization` של ה-pass manager המדורג שמוחזר על-ידי [`generate_preset_pass_manager`](https://docs.quantum.ibm.com/api/qiskit/qiskit.transpiler.generate_preset_pass_manager#qiskit.transpiler.generate_preset_pass_manager) או שמשמש ב-[`transpile`](https://docs.quantum.ibm.com/api/qiskit/compiler#qiskit.compiler.transpile). ראה [שלבי Transpiler](transpiler-stages) לתיאור שלבים אלו.

# אתחול השירות בסביבה לא מהימנה {#initialize-the-service-in-an-untrusted-environment}

עקוב אחר ההוראות האלה אם אתה עובד עם IBM Cloud או עם Qiskit Runtime REST API של IBM בסביבה לא מהימנה, כגון מחשב ציבורי.

**הימנע מהרצת קוד על מחשב לא מהימן או בסביבת Python בענן חיצוני כדי למזער סיכוני אבטחה.** אם אתה חייב להשתמש בסביבה לא מהימנה (למשל, על מחשב משותף או ציבורי), שנה את מפתח ה-API שלך לאחר כל שימוש על ידי מחיקתו בדף [מפתחות API](https://cloud.ibm.com/iam/apikeys) ויצירת מפתח חדש.

    <Admonition type="caution">
      **הגן על מפתח ה-API שלך!** לעולם אל תכלול את המפתח שלך בקוד מקור, סקריפט Python, או קובץ מחברת. כאשר אתה משתף קוד עם אחרים, וודא שמפתח ה-API שלך אינו מוטמע ישירות בתוך סקריפט ה-Python. במקום זאת, שתף את הסקריפט ללא האסימון וספק הוראות להגדרתו בצורה מאובטחת.

      אם שיתפת בטעות את המפתח שלך עם מישהו או כללת אותו בבקרת גרסאות כמו Git, בטל מיד את המפתח שלך על ידי ביצוע ההוראות בנושא [מחיקת מפתח API](https://cloud.ibm.com/docs/account?topic=account-userapikey&interface=ui#delete_user_key) ב-IBM Cloud.
    </Admonition>

<span id="access-credentials"></span>
## מצא את פרטי הגישה שלך {#find-your-access-credentials}

1. מצא את מפתח ה-API שלך (המכונה גם _אסימון API_). מה[לוח הבקרה](https://quantum.cloud.ibm.com/), צור את מפתח ה-API שלך, ולאחר מכן העתק אותו למקום מאובטח כדי שתוכל להשתמש בו לאימות. האסימון לא יהיה גלוי שוב. שים לב שניתן להשתמש במפתח API יחיד להתחברות לכל אזור.
2. אופציונלי: מצא את המופע שברצונך להשתמש בו מדף [המופעים](https://quantum.cloud.ibm.com/instances). העבר את העכבר מעל ה-CRN שלו, לחץ על הסמל כדי להעתיק אותו, ולאחר מכן שמור אותו במקום מאובטח כדי שתוכל להשתמש בו לזיהוי המופע.

## אימות ל-IBM Cloud בסביבה לא מהימנה {#authenticate-to-ibm-cloud-in-an-untrusted-environment}

כדי לאתחל את השירות במצב זה, השתמש בקוד כמו הבא:

    ```python
    from qiskit_ibm_runtime import QiskitRuntimeService

    # The credentials you enter will override any saved
    # account credentials that might be available locally.
    service = QiskitRuntimeService(
      # Use the 44-character API_KEY you created and saved from
      # the IBM Quantum Platform dashboard, and then delete
      # the key on the API keys page after entering this code:
      token="<your-API-key>",

      # Optionally specify an instance to use
      instance="<IBM Cloud CRN or instance name>"
      )
    ```

<span id="rest-untrusted"></span>
## אימות ל-REST API בסביבה לא מהימנה {#authenticate-to-the-rest-api-in-an-untrusted-environment}

עקוב אחר הצעדים האלה כדי להשתמש במפתח ה-API שלך ישירות לאימות מול Qiskit Runtime REST API, כאשר `<your_API_KEY>` הוא ה-API_KEY בן 44 התווים שיצרת ושמרת מלוח הבית של IBM Quantum Platform.

1. השתמש במפתח ה-API שלך כדי ליצור [bearer token](https://docs.quantum.ibm.com/api/qiskit-runtime-rest#authentication). _bearer token_ הוא פרטי גישה זמניים שפגים לאחר שעה לכל היותר. לאחר שה-token שנרכש פג, עליך ליצור חדש כדי להמשיך לקרוא ל-IBM Cloud או לממשקי API אחרים של שירותים. :
  ```bash
  curl -X POST 'https://iam.cloud.ibm.com/identity/token' \
  -H 'Content-Type: application/x-www-form-urlencoded' \
  -d 'grant_type=urn:ibm:params:oauth:grant-type:apikey&apikey=your_API_KEY'
  ```
  העתק ושמור את ה-bearer token שהוחזר: `"access_token": "<NEW_BEARER_TOKEN>"`
1. אמת בקשות ל-Qiskit Runtime REST API על ידי הכללת ה-CRN וה-bearer token בכותרות הבקשה.

  ```bash
  curl -X 'GET' \
     'https://quantum.cloud.ibm.com/api/v1/usage' \
     '-H accept: application/json' \
     '-H authorization: Bearer <BEARER_TOKEN>' \
     '-H Service-CRN: <INSTANCE_CRN>'
  ```

1. שנה את מפתח ה-API שלך לאחר כל שימוש על ידי יצירת מפתח API חדש בדף הבית של IBM Quantum Platform, או על ידי ביצוע הוראות [מחיקת מפתח API](https://cloud.ibm.com/docs/account?topic=account-userapikey&interface=ui#delete_user_key) ב-IBM Cloud.

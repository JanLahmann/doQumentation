# שימוש ב-IBM Cloud Platform APIs לגישה ל-instances {#use-ibm-cloud-platform-apis-to-access-instances}

מדריך זה מיועד לכל מי שמשתמש ב-IBM Cloud Platform APIs כדי ליצור, לעדכן או לנהל גישה ל-instances. הוא מספק פרטים על ה-[פרמטרים](#parameters) שאפשר להגדיר בעת יצירת instance, וכן על קבוצת [שדות ההרחבה](#extensions) המוחזרים בעת גישה ל-instance.

<Admonition type="note">
לפרטים נוספים, עיין/י בתיעוד IBM Cloud&reg; ResourceController API.

דפים רלוונטיים:
- [יצירה (provisioning) של resource instance חדש](https://cloud.ibm.com/apidocs/resource-controller/resource-controller#create-resource-instance)
- [עדכון resource instance](https://cloud.ibm.com/apidocs/resource-controller/resource-controller#update-resource-instance)
- [קבלת resource instance](https://cloud.ibm.com/apidocs/resource-controller/resource-controller#get-resource-instance)
- [קבלת רשימה של כל resource instances](https://cloud.ibm.com/apidocs/resource-controller/resource-controller#list-resource-instances)

    בעת קבלת רשימה של כל resource instances, שים/י לב שאפשר לסנן רק את ה-instances של Qiskit Runtime עם `resource_id = "b6049020-80f4-11eb-a0f7-e35ec9b4054f"`

</Admonition>

## `parameters` {#parameters}

בגוף הבקשה בעת יצירה או עדכון של instance:

- **instance_limit_seconds** (string או null, `^[0-9]+$`): המגבלה להגדרה על ה-instance. עבור Open Plan: אם לא צוין, ברירת המחדל היא 10 דקות.
.
- **usage_allocation_seconds** (string או null, `^[0-9]+$`): הזמן המוקצה ל-instance, שבו משתמש ה-fair-share scheduler כדי לקבוע את העדיפות בתור, בהתאם לשימוש בכל ה-QPUs. (לא חל על instances מסוג Pay-As-You-Go.)
- **backends** (מערך של רשימות backends (strings או null) או מערך עם "ANY"): רשימת ההיתרים של שמות ה-backends הזמינים ל-instance זה. ברירת מחדל: `["ANY"]`, כלומר כל ה-backends הזמינים בתוכנית זו. `[]` פירושו ללא backends.

## `extensions` {#extensions}

בתגובה בעת יצירה או בעת קבלת פרטי instance:

מאפייני instance נוספים שהשירות תורם, המיוצגים כזוגות מפתח-ערך. מאפיינים אלה נשמרים עם ה-instance תחת ה-property‏ extensions.

- **instance_limit_seconds** (integer או null, `>= 0`): מגבלת הזמן להגדרה על ה-instance.
- **usage_allocation_seconds** (integer או null, `>= 0`): הזמן המוקצה ל-instance, שבו משתמש ה-fair-share scheduler כדי לקבוע את העדיפות בתור, בהתאם לשימוש בכל ה-QPUs.
- **backends** (נדרש; מערך של רשימות backends (strings) או מערך עם "ANY" (strings)): רשימת ההיתרים של שמות ה-backends הזמינים ל-instance זה. ברירת מחדל: `["ANY"]`, כלומר כל ה-backends זמינים בתוכנית זו. `[]` פירושו ללא backends.

## Construa o circuito {#build-the-circuit}

```python
# Added by doQumentation — required packages for this notebook
```

```python
from qiskit import QuantumCircuit

qc = QuantumCircuit(2)
qc.h(0)
qc.cx(0, 1)
qc.measure_all()

qc.draw(output="mpl")
```

## Opção 1: Simulação ideal (não precisa de conta) {#option-1-ideal-simulation-no-account-needed}

Usa `StatevectorSampler` — um simulador local com resultados perfeitos, sem ruído.

```python
from qiskit.primitives import StatevectorSampler

result = StatevectorSampler().run([qc], shots=1024).result()
counts = result[0].data.meas.get_counts()
counts
```

```python
from qiskit.visualization import plot_histogram
plot_histogram(counts)
```

## Opção 2: Simulação com ruído (não precisa de conta) {#option-2-noisy-simulation-no-account-needed}

Usa `FakeManilaV2` — um simulador local que imita um dispositivo quântico IBM real, incluindo suas características de ruído. O circuito deve primeiro ser transpilado (adaptado) para o conjunto de portas e conectividade de qubits do dispositivo.

```python
from qiskit_ibm_runtime import SamplerV2
from qiskit_ibm_runtime.fake_provider import FakeManilaV2
from qiskit.transpiler.preset_passmanagers import generate_preset_pass_manager

backend = FakeManilaV2()
pm = generate_preset_pass_manager(backend=backend, optimization_level=1)
isa_qc = pm.run(qc)

result = SamplerV2(mode=backend).run([isa_qc], shots=1024).result()
counts = result[0].data.meas.get_counts()
counts
```

```python
plot_histogram(counts)
```

## Opção 3: Hardware quântico real {#option-3-real-quantum-hardware}

Requer uma conta IBM Quantum. Veja [Configurando sua conta IBM Quantum](https://janlahmann.github.io/Qiskit-documentation/#setting-up-your-ibm-quantum-account) para detalhes.

Se você ainda não salvou suas credenciais nesta sessão do Binder, execute isso primeiro:

```python
from qiskit_ibm_runtime import QiskitRuntimeService

QiskitRuntimeService.save_account(
    channel="ibm_quantum_platform",
    token="<your-api-key>",
    instance="<your-crn>",
    set_as_default=True,
    overwrite=True,
)
```

**Observação:** Jobs em hardware real podem levar algum tempo dependendo do tempo de fila. Se a célula ainda estiver em execução, você pode verificar o status do seu job e ver os resultados em [quantum.cloud.ibm.com/workloads](https://quantum.cloud.ibm.com/workloads?user=me).

```python
from qiskit_ibm_runtime import QiskitRuntimeService, SamplerV2
from qiskit.transpiler.preset_passmanagers import generate_preset_pass_manager

service = QiskitRuntimeService()
backend = service.least_busy(operational=True, simulator=False)
print(f"Running on {backend.name}")

pm = generate_preset_pass_manager(backend=backend, optimization_level=1)
isa_qc = pm.run(qc)

result = SamplerV2(mode=backend).run([isa_qc], shots=1024).result()
counts = result[0].data.meas.get_counts()
counts
```

```python
plot_histogram(counts)
```

## Visão geral {#overview}

A função de circuito IBM&reg; recebe [PUBs abstratos](/guides/primitive-input-output#pubs) como entradas e retorna valores de expectativa mitigados como saídas. Essa função de circuito inclui um pipeline automatizado e personalizado para permitir que pesquisadores se concentrem na descoberta de algoritmos e aplicações.

## Descrição {#description}

Após enviar seu PUB, seus circuitos abstratos e observáveis são automaticamente transpilados, executados em hardware e pós-processados para retornar valores de expectativa mitigados. Para isso, combina as seguintes ferramentas:

- [Qiskit Transpiler Service](/guides/qiskit-transpiler-service), incluindo seleção automática de passes de transpilação baseados em IA e heurísticos para traduzir seus circuitos abstratos em circuitos ISA otimizados para hardware
- [Supressão e mitigação de erros necessárias para computação em escala utilitária](/guides/error-mitigation-and-suppression-techniques), incluindo twirling de medições e de portas, desacoplamento dinâmico, Twirled Readout Error eXtinction (TREX), Zero-Noise Extrapolation (ZNE) e Probabilistic Error Amplification (PEA)
- [Qiskit Runtime Estimator](/guides/get-started-with-estimator), para executar PUBs ISA em hardware e retornar valores de expectativa mitigados

## Exemplos {#examples}

Para começar, experimente este exemplo básico:

```python
from qiskit.circuit.random import random_circuit
from qiskit_ibm_runtime import QiskitRuntimeService

# You can skip this step if you have a target backend, e.g.
# backend_name = "ibm_brisbane"
# You'll need to specify the credentials when initializing QiskitRuntimeService, if they were not previously saved.
service = QiskitRuntimeService()
backend = service.least_busy(operational=True, simulator=False)

circuit = random_circuit(num_qubits=2, depth=2, seed=42)
observable = "Z" * circuit.num_qubits
pubs = [(circuit, observable)]

job = function.run(
    # Use `backend_name=backend_name` if you didn't initialize a backend object
    backend_name=backend.name,
    pubs=pubs,
)
```

Verifique o [status](/guides/functions#check-job-status) da sua carga de trabalho da Qiskit Function ou obtenha os [resultados](/guides/functions#retrieve-results) da seguinte forma:

```python
print(job.status())
result = job.result()
```

```text

```

Os resultados têm o mesmo formato de um [resultado do Estimator](/guides/estimator-input-output):

```python

    f"The expectation values measured from this PUB are: \n{result[0].data.evs}"
)
```

### Exemplos de nível de mitigação {#mitigation-level-examples}

O exemplo a seguir demonstra como definir o nível de mitigação:

```python
options = {"mitigation_level": 2}

job = function.run(backend_name=backend.name, pubs=pubs, options=options)
```

No exemplo a seguir, definir o nível de mitigação como 1 desativa inicialmente a mitigação ZNE, mas definir `zne_mitigation` como `True` substitui a configuração relevante do `mitigation_level`.

```python
options = {"mitigation_level": 1, "resilience": {"zne_mitigation": True}}
```

### Exemplo de saída {#output-example}

O trecho de código a seguir descreve o formato do `PrimitiveResult` (e do `PubResult` associado).

### Buscar mensagens de erro {#fetch-error-messages}

Se o status da sua carga de trabalho for `ERROR`, use `job.result()` para obter a mensagem de erro e ajudar na depuração da seguinte forma:

```python
job = function.run(
    backend_name="bad_backend_name", pubs=pubs, options=options
)

print(job.result())
```

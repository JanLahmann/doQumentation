### Inicializar o Estimator {#initialize-estimator}

Instancie um [`qiskit.primitives.StatevectorEstimator`](../api/qiskit/qiskit.primitives.StatevectorEstimator).

### Executar e obter resultados {#run-and-get-results}

Este exemplo usa apenas um circuito (do tipo [`QuantumCircuit`](../api/qiskit/qiskit.circuit.QuantumCircuit)) e um
observГЎvel.

Execute a estimativa chamando o mГ©todo [`StatevectorEstimator.run`](../api/qiskit/qiskit.primitives.StatevectorEstimator#run), que retorna uma instГўncia de um objeto [`PrimitiveJob`](https://docs.quantum.ibm.com/api/qiskit/qiskit.primitives.PrimitiveJob). VocГӘ pode obter os resultados do job (como um objeto [`qiskit.primitives.PrimitiveResult`](../api/qiskit/qiskit.primitives.PrimitiveResult))
com o mГ©todo [`qiskit.primitives.PrimitiveJob.result`](../api/qiskit/qiskit.primitives.PrimitiveJob#result).



#### Obter o valor esperado do resultado {#get-the-expected-value-from-the-result}

Os resultados das primitivas retornam um array de objetos [`PubResult`](https://docs.quantum.ibm.com/api/qiskit/qiskit.primitives.PubResult#pubresult), onde cada item do array Г© um objeto `PubResult` que contГ©m em seus dados o array de avaliaГ§Гөes correspondente a cada combinaГ§ГЈo circuito-observГЎvel no PUB.

Para recuperar os valores esperados e os metadados da primeira (e, neste caso, Гәnica) avaliaГ§ГЈo do circuito, precisamos acessar os [`data`](https://docs.quantum.ibm.com/api/qiskit/qiskit.primitives.PubResult#data) de avaliaГ§ГЈo do PUB 0:

### Definir opГ§Гөes de execuГ§ГЈo do Estimator {#set-estimator-run-options}

Por padrГЈo, o Estimator de referГӘncia realiza um cГЎlculo exato de vetor de estado baseado na
classe [`quantum_info.Statevector`](../api/qiskit/qiskit.quantum_info.Statevector).
No entanto, isso pode ser modificado para introduzir o efeito da sobrecarga de amostragem (tambГ©m conhecida como "ruГӯdo de shot").

O Estimator aceita um argumento `precision` que expressa as barras de erro que a
implementaГ§ГЈo da primitiva deve almejar para estimativas de valores esperados. Isso Г© a sobrecarga de amostragem e Г© definida exclusivamente no mГ©todo `.run()`. Isso permite ajustar a opГ§ГЈo atГ© o nГӯvel do PUB.

```python

    [(circuit, observable, parameter_values)], precision=0.05
```


As primitivas aceitam mГәltiplos PUBs como entradas, e cada PUB recebe seu prГіprio resultado. Portanto, vocГӘ pode executar circuitos diferentes com vГЎrias combinaГ§Гөes de parГўmetros/observГЎveis e recuperar os resultados dos PUBs:

```python

# execute 2 circuits using Sampler

pub_result_2 = job.result()[1]

```

```text
> Result class: <class 'qiskit.primitives.containers.sampler_pub_result.SamplerPubResult'>
```

### Alterar opГ§Гөes de execuГ§ГЈo {#change-run-options}


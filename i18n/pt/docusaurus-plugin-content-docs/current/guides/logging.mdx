# Logging para o Qiskit Runtime {#logging-for-qiskit-runtime}

Os serviços do IBM Cloud&reg;, como o Qiskit Runtime, geram logs de plataforma que você pode usar para investigar atividades anômalas e ações críticas na sua conta, além de solucionar problemas.

Você pode usar o [IBM Cloud Logs Routing,](https://cloud.ibm.com/docs/logs-router?topic=logs-router-about) que é um serviço de plataforma, para rotear os logs de plataforma da sua conta para um destino de sua escolha configurando um tenant que define para onde os logs de plataforma são enviados. Para mais informações, consulte [Sobre o Logs Routing.](https://cloud.ibm.com/docs/logs-router?topic=logs-router-about)

Você pode usar o [IBM Cloud Logs](https://cloud.ibm.com/docs/cloud-logs) para visualizar e configurar alertas sobre os logs de plataforma gerados na sua conta e roteados pelo IBM Cloud Logs Routing para uma instância do IBM Cloud Logs.

## Locais onde os logs de plataforma são gerados {#locations-where-platform-logs-are-generated}

O IBM Cloud Container Registry gera logs de plataforma que são exibidos nas suas instâncias de logging. A tabela a seguir lista os locais onde a coleta automática de logs do serviço Container Registry está habilitada.

| Local                          | Os logs de serviços de plataforma estão disponíveis? |
|--------------------------------|------------------------------------------------------|
| Washington, D.C. (us-east)     |                        Sim                           |
| Frankfurt (eu-de)              |                        Sim                           |

## Logs de plataforma gerados {#platform-logs-that-are-generated}

O IBM Qiskit Runtime gera logs de serviços de plataforma que são exibidos nas suas instâncias de logging para os seguintes casos específicos:

- Quando o estado de um job muda.

## Visualizando logs pela página de Observabilidade {#viewing-logs-from-the-observability-page}

Para informações sobre como abrir a interface do IBM Cloud Logs, consulte **Iniciando a UI** na [documentação do IBM Cloud Logs.](https://cloud.ibm.com/docs/cloud-logs?topic=cloud-logs-instance-launch)

## Campos por tipo de log {#fields-by-log-type}

Para informações sobre os campos incluídos em todos os logs de plataforma, consulte [Campos incluídos nos logs de plataforma.](https://cloud.ibm.com/docs/logs-router?topic=logs-router-about-platform-logs#about-platform-logs-2)

Os campos a seguir estão incluídos no registro de log.

| Campo           | Tipo       | Descrição                                                                                   |
|-----------------|------------|---------------------------------------------------------------------------------------------|
| logSourceCRN    | Obrigatório | Define a conta e a instância de log de fluxo onde o log é publicado.                       |
| saveServiceCopy | Obrigatório | Define se a IBM salva uma cópia do registro para fins operacionais.                         |
| message         | Obrigatório | Descrição do log gerado.                                                                    |

# Instalar o cliente Qiskit Runtime a partir do código-fonte {#install-the-qiskit-runtime-client-from-source}

Ao instalar o Qiskit Runtime a partir do código-fonte, você pode acessar a versão de desenvolvimento atual, em vez de usar a versão no repositório Python Package Index (PyPI). Portanto, você pode inspecionar e estender a versão mais recente do código com mais eficiência.

## Criar e ativar um novo ambiente virtual {#create-and-activate-a-new-virtual-environment}

1. Navegue até o diretório do seu projeto e crie um ambiente mínimo com apenas o Python instalado nele.

        <OperatingSystemTabs>
          <TabItem value="mac" label="macOS">
            ```shell
            python3 -m venv .venv
            ```
          </TabItem>

          <TabItem value="linux" label="Linux">
            ```shell
            python3 -m venv .venv
            ```
          </TabItem>

          <TabItem value="win" label="Windows">
            ```text
            python -m venv .venv
            ```
          </TabItem>
         </OperatingSystemTabs>

1. Ative seu novo ambiente.

         <OperatingSystemTabs>
           <TabItem value="mac" label="macOS">
             ```shell
             source .venv/bin/activate
             ```
           </TabItem>

           <TabItem value="linux" label="Linux">
             ```shell
             source .venv/bin/activate
             ```

             .venv\Scripts\Activate.ps1

## Instalar o Qiskit Runtime {#install-qiskit-runtime}

Siga estes passos para instalar o Qiskit Runtime:

1. Clone o repositório Qiskit Runtime.

```bash
git clone https://github.com/Qiskit/qiskit-ibm-runtime.git
```

2. Mude para o diretório `qiskit-ibm-runtime`.

```bash
cd qiskit-ibm-runtime
```

3. Instale o `qiskit-runtime`. Recomendamos usar um [ambiente virtual](https://docs.python.org/3/library/venv.html) para evitar poluir sua instalação global do Python.

  * **Instalação padrão**:

    ```bash
    pip install .
    ```

  * **Modo editável**: Neste modo, você não precisa reinstalar o Qiskit quando houver alterações de código no projeto.

    ```bash
    pip install -e .
    ```

    No modo editável, as extensões compiladas são construídas em _modo de depuração_ sem otimizações.

4. (Opcional) Se você quiser executar testes ou verificações de lint, instale os requisitos do desenvolvedor. Recomendamos usar um [ambiente virtual](https://docs.python.org/3/library/venv.html) para evitar poluir sua instalação global do Python.

```bash
pip install -e ".[dev]"
```

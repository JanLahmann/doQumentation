# Migrar do `BackendV1` para o `BackendV2` {#migrate-from-backendv1-to-backendv2}

A classe [`BackendV1`](https://docs.quantum.ibm.com/api/qiskit/1.4/qiskit.providers.BackendV1) do Qiskit foi descontinuada e será removida do serviço. Este guia de migração descreve os pequenos ajustes que você precisa fazer caso use um provedor que atualizou do [`BackendV1`](https://docs.quantum.ibm.com/api/qiskit/1.4/qiskit.providers.BackendV1) para o [`BackendV2`](https://docs.quantum.ibm.com/api/qiskit/qiskit.providers.BackendV2).

<Admonition type="note">
 Se você usa exclusivamente `qiskit_ibm_runtime` e `qiskit_aer`, nenhuma ação é necessária. O pacote `qiskit_ibm_runtime` sempre usou `BackendV2`, e o `qiskit_aer` usa `BackendV2` desde a versão 0.13.
 </Admonition>

## Mudanças gerais no `BackendV2` {#high-level-changes-in-backendv2}

O modelo [`Backend`](https://docs.quantum.ibm.com/api/qiskit/qiskit.providers.Backend) do Qiskit foi projetado para fornecer ao
Qiskit SDK uma camada de abstração que permite raciocinar sobre computadores quânticos dentro do escopo do SDK. A primeira iteração
do modelo foi introduzida com a classe [`BackendV1`](https://docs.quantum.ibm.com/api/qiskit/1.4/qiskit.providers.BackendV1). Essa classe armazenava as informações do backend em uma série
de contêineres de dados, nomeadamente as classes [`BackendConfiguration`](https://docs.quantum.ibm.com/api/qiskit/1.4/qiskit.providers.models.BackendConfiguration) e
[`BackendProperties`](https://docs.quantum.ibm.com/api/qiskit/1.4/qiskit.providers.models.BackendProperties).

A classe [`BackendV2`](https://docs.quantum.ibm.com/api/qiskit/qiskit.providers.BackendV2) redefiniu
o acesso do usuário à maioria das propriedades do backend para que funcionem com as estruturas de dados nativas do Qiskit e tenham padrões
de acesso mais simples. O núcleo do modelo [`BackendV2`](https://docs.quantum.ibm.com/api/qiskit/qiskit.providers.BackendV2) é a classe
[`Target`](https://docs.quantum.ibm.com/api/qiskit/qiskit.transpiler.Target), uma representação do QPU que contém as restrições de transpilação que o Qiskit pode usar para otimizar circuitos para execução.

O SDK do Qiskit foi atualizado para trabalhar exclusivamente com entradas [`BackendV2`](https://docs.quantum.ibm.com/api/qiskit/qiskit.providers.BackendV2),
e a maioria dos provedores migrou do [`BackendV1`](https://docs.quantum.ibm.com/api/qiskit/1.4/qiskit.providers.BackendV1) para o [`BackendV2`](https://docs.quantum.ibm.com/api/qiskit/qiskit.providers.BackendV2). Espera-se que os provedores existentes descontinuem o acesso antigo onde for possível, oferecendo uma migração gradual, mas eventualmente os usuários vão precisar ajustar seu código.

O princípio por trás do [`BackendV2`](https://docs.quantum.ibm.com/api/qiskit/qiskit.providers.BackendV2) é
que a maior parte das informações sobre um backend está contida em seu
objeto [`Target`](https://docs.quantum.ibm.com/api/qiskit/qiskit.transpiler.Target), e os atributos do backend frequentemente consultam
o atributo `BackendV2.target` para retornar informações. No entanto, em muitos casos os atributos fornecem
apenas um subconjunto das informações que o Target pode conter. Por exemplo, `backend.coupling_map`
retorna um objeto [`CouplingMap`](https://docs.quantum.ibm.com/api/qiskit/qiskit.transpiler.CouplingMap) construído a partir do
[`Target`](https://docs.quantum.ibm.com/api/qiskit/qiskit.transpiler.Target) acessível pelo
atributo `BackendV2.target`. No entanto, o Target pode conter
instruções que operam em mais de dois qubits (o que não pode ser representado em um
[`CouplingMap`](https://docs.quantum.ibm.com/api/qiskit/qiskit.transpiler.CouplingMap)) ou pode ter instruções que operam apenas em
um subconjunto de qubits (ou em links de dois qubits, para uma instrução de dois qubits), o que não será
detalhado no coupling map completo retornado por
`BackendV2.coupling_map`. Portanto, dependendo do seu caso de uso,
pode ser necessário ir além do acesso equivalente com
[`BackendV2`](https://docs.quantum.ibm.com/api/qiskit/qiskit.providers.BackendV2).

## Mudanças específicas no `BackendV2` {#specific-changes-in-backendv2}

A maioria dos atributos tem uma substituição direta, simplificando os esforços de migração. O único ponto de divergência entre as interfaces está no [`CouplingMap`](https://docs.quantum.ibm.com/api/qiskit/qiskit.transpiler.CouplingMap).

A seguir há uma tabela com exemplos de padrões de acesso no [`BackendV1`](https://docs.quantum.ibm.com/api/qiskit/1.4/qiskit.providers.BackendV1) e a nova forma
com [`BackendV2`](https://docs.quantum.ibm.com/api/qiskit/qiskit.providers.BackendV2).

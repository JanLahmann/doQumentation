# Eventos de rastreamento de atividade para o serviço IBM Quantum System {#activity-tracking-events-for-the-ibm-quantum-system-service}

Os serviços do IBM Cloud&reg;, como o serviço IBM&reg; Quantum System, geram eventos de rastreamento de atividade.

Os eventos de rastreamento de atividade relatam atividades que alteram o estado de um serviço no IBM Cloud. Você pode usar os eventos para investigar atividades anômalas e ações críticas, além de cumprir requisitos de auditoria regulatória.

Você pode usar o IBM Cloud Activity Tracker Event Routing, um serviço de plataforma, para rotear eventos de auditoria na sua conta para destinos de sua escolha, configurando destinos e rotas que definem para onde os eventos de rastreamento de atividade são enviados. Para mais informações, veja [Sobre o IBM Cloud Activity Tracker Events Routing](https://cloud.ibm.com/docs/atracker?topic=atracker-about).

Você pode usar o IBM Cloud Logs para visualizar e emitir alertas sobre eventos gerados na sua conta e roteados pelo IBM Cloud Activity Tracker Event Routing para uma instância do IBM Cloud Logs.

## Visualizando eventos de rastreamento de atividade para o serviço IBM Quantum System {#viewing-activity-tracking-events-for-the-ibm-quantum-system-service}

Você pode usar o IBM Cloud Logs para visualizar e emitir alertas sobre eventos gerados na sua conta e roteados pelo IBM Cloud Activity Tracker Event Routing para uma instância do IBM Cloud Logs.

### Iniciando o IBM Cloud Logs pela página de Observabilidade {#launching-ibm-cloud-logs-from-the-observability-page}

Para informações sobre como iniciar a interface do IBM Cloud Logs, veja [Iniciando a interface](https://cloud.ibm.com/docs/cloud-logs?topic=cloud-logs-instance-launch#instance-launch-cloud-ui) na documentação do IBM Cloud Logs.

| Ação | Acionado por |
| -- | -- |
| `quantum-computing.direct-access-backend-configuration.read` | Obter configuração do Backend (`GET /v1/backends/{backend-name}/configuration`) |
| `quantum-computing.direct-access-backend-properties.read` | Obter propriedades do Backend (`GET /v1/backends/{backend-name}/properties`) |
| `quantum-computing.direct-access-backend.list` | Obter lista de Backends (`GET /v1/backends`) |
| `quantum-computing.direct-access-backend.read` | Obter detalhes do Backend (`GET /v1/backends/{backend-name}`) |
| `quantum-computing.direct-access-job.cancel` | Cancelar um job (`POST /v1/jobs/{job-id}/cancel`) |
| `quantum-computing.direct-access-job.create` | Executar um job (`POST /v1/jobs`) |
| `quantum-computing.direct-access-job.delete` | Excluir um job (`DELETE /v1/jobs/{job-id}`) |
| `quantum-computing.direct-access-job.list` | Obter jobs (`GET /v1/jobs`) |
| `quantum-computing.direct-access-job.read` | Obter job por ID (`GET /v1/jobs/{job-id}`) |

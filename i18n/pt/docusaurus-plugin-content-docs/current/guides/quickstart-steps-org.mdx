# Configurar o IBM Quantum Platform para uma organização {#configure-ibm-quantum-platform-for-an-organization}

Siga estes passos para configurar o IBM Quantum Platform.

<span id="iam-org"></span>
## 1. Configurar as definições do IAM {#1-configure-iam-settings}

Primeiro, configure as seguintes definições na conta IAM do administrador. Para rever e configurar essas definições, acesse [Manage → IAM → Settings](https://cloud.ibm.com/iam/settings).

* **User list visibility** determina se os usuários podem ver uns aos outros, independentemente da atribuição de projetos. A configuração `enabled` restringe a visibilidade dos usuários. Ou seja, os usuários na sua conta não conseguem ver uns aos outros, mesmo que tenham acesso aos mesmos recursos. Escolha o valor adequado para o seu ambiente. Consulte [Controlling user visibility](https://cloud.ibm.com/docs/account?topic=account-iam-user-setting) para mais informações.
* **API key creation** controla se os usuários podem criar chaves de API. No IBM Quantum Platform, é comum usar chaves de API. Se as chaves de API estiverem sendo utilizadas, escolha `disabled`. Como alternativa, você pode conceder permissões específicas a cada usuário.

<span id="crt-rsc-grp-org"></span>
## 2. (Opcional) Criar grupos de recursos {#2-optional-create-resource-groups}

Pule esta etapa se você estiver usando grupos de acesso que referenciam diretamente instâncias do serviço IBM Quantum.

Se você optou por usar grupos de recursos, acesse [Manage → Account → Resource groups (in Account resources)](https://cloud.ibm.com/account/resource-groups) e clique em **Create**.

<span id="create-instance-org"></span>
## 3. Criar instâncias do serviço IBM Quantum {#3-create-ibm-quantum-service-instances}

Se você já criou instâncias do serviço IBM Quantum, pule esta etapa.

* Se você estiver usando grupos de recursos, certifique-se de criar as instâncias de serviço no grupo de recursos adequado.
* O nome da instância de serviço, como `QR-ml`, é necessário para referências em grupos de acesso.

Siga os passos em [Set up to use IBM Cloud](/guides/cloud-setup#cloud) para obter instruções.

<span id="create-group-org"></span>
## 4. Criar grupos de acesso para projetos {#4-create-access-groups-for-projects}

Primeiro, [crie uma função personalizada](/guides/custom-roles#custom) que permita aos usuários realizar ações para trabalhar com instâncias do serviço IBM Quantum.
Em seguida, [crie um grupo de acesso](/guides/access-groups) para cada projeto e conceda a esse grupo o conjunto mínimo de permissões necessárias para trabalhar com os recursos do projeto.

<span id="setup-idp-org"></span>
## 5. Configurar seu provedor de identidade e atribuir usuários {#5-set-up-your-id-provider-and-assign-users}

Siga os passos no tópico adequado, de acordo com o provedor de identidade escolhido:

* [Use o IBM Cloud&reg; como provedor de identidade para usuários que possuem contas IBM Cloud](/guides/manage-cloud-users)
* [Use o IBM Cloud como provedor de identidade para usuários que não possuem contas IBM Cloud](/guides/manage-appid)
* [Use um provedor de identidade diferente do IBM Cloud para usuários que não possuem contas IBM Cloud](/guides/manage-appid)
* [Convide usuários e gerencie seus acessos](/guides/invite-and-manage-users)

<span id="firewall"></span>
## 6. Configurar seu firewall {#6-configure-your-firewall}

Para configurar seu firewall e permitir o acesso aos endpoints da API IBM Quantum, adicione as seguintes URLs à sua lista de permissões:

- Serviços globais do IBM Cloud:
    ```text
    cloud.ibm.com
    iam.cloud.ibm.com
    api.global-search-tagging.cloud.ibm.com
    globalcatalog.cloud.ibm.com
    resource-controller.cloud.ibm.com
    ```
- Serviços IBM Quantum Qiskit Runtime - região us-east:
    ```text
    quantum.cloud.ibm.com
    s3.us-east.cloud-object-storage.appdomain.cloud
    ```
- Serviços IBM Quantum Qiskit Runtime - região eu-de:
    ```text
    eu-de.quantum.cloud.ibm.com
    s3.direct.eu-de.cloud-object-storage.appdomain.cloud
    ```
- Serviço de prévia do IBM Quantum Qiskit Functions:
    ```text
    qiskit-serverless.quantum.ibm.com
    ```

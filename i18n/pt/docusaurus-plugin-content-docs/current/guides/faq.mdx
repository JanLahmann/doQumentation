# Perguntas frequentes (FAQ) {#frequently-asked-questions-faq}

Para perguntas frequentes específicas sobre os modos de execução, consulte o guia de [perguntas frequentes sobre modos de execução do Qiskit Runtime](/guides/execution-modes-faq).

## Como cito o Qiskit na minha pesquisa? {#how-do-i-cite-qiskit-in-my-research}

Cite o Qiskit usando o [arquivo BibTeX](https://github.com/Qiskit/qiskit/blob/main/CITATION.bib) incluído.

## Como cito um computador IBM Quantum na minha pesquisa? {#how-do-i-cite-an-ibm-quantum-computer-in-my-research}

Para artigos de pesquisa, incentivamos os autores a reconhecer a IBM Quantum&reg; usando:

>    Reconhecemos o uso dos serviços IBM Quantum neste trabalho. As opiniões expressas são dos autores e não refletem a política ou posição oficial da IBM ou da equipe IBM Quantum.

As referências bibliográficas devem usar a seguinte citação:

> IBM Quantum. https://<span>quantum.cloud.ibm.com</span>/, AAAA

As unidades de processamento quântico no artigo devem ser referenciadas pelo nome exclusivo (por exemplo, `ibmq_vigo`) e, opcionalmente, com a versão (por exemplo, `ibmq_vigo` v1.0.2). Também incentivamos a referência ao processador.
Por exemplo:

> Neste artigo, utilizamos `ibmq_vigo`, que é um dos processadores IBM Quantum Canary.

Exemplo de citação de um programa IBM Quantum:

> IBM Quantum (2025). Estimator primitive (Versão x.y.z) [software de computador]. https://<span>quantum.cloud.ibm.com</span>/

## Como cito o IBM Quantum Composer na minha pesquisa? {#how-do-i-cite-ibm-quantum-composer-in-my-research}

Exemplo de citação para o IBM Quantum Composer:

> IBM Quantum Composer. 2025. url: https://<span>quantum.cloud.ibm.com/composer</span>

## Não estou vendo algo na interface do usuário, como minhas tarefas de administração. {#i-dont-see-something-in-the-user-interface-such-as-my-administration-tasks}

Tente escolher uma conta diferente no seletor de contas no cabeçalho da interface da [IBM Quantum Platform](https://quantum.cloud.ibm.com).

## Estou usando apenas o Plano Open, mas precisei informar meu cartão de crédito. Serei cobrado? {#im-just-using-the-open-plan-but-i-had-to-put-in-my-credit-card-will-i-be-charged}

Você não será cobrado a menos que crie uma [instância](/guides/instances) Pay-As-You-Go e a use para enviar cargas de trabalho a QPUs. Consulte [Gerenciar custos](/guides/manage-cost) para obter instruções sobre como definir um limite de custo e conhecer outras formas de gerenciar custos.

## Qual é a diferença entre uma chave de API e um token de API? {#whats-the-difference-between-an-api-key-and-an-api-token}

Não há diferença. Os termos chave de API e token de API são usados de forma intercambiável. Você verá ambos os termos na documentação porque interfaces diferentes usam termos diferentes.

<span id="bearer"></span>
## Qual é a diferença entre um token de API e um bearer token? {#whats-the-difference-between-an-api-token-and-a-bearer-token}

Um token de API é uma credencial de longa duração usada para [autenticar no serviço Qiskit Runtime.](/guides/hello-world#install-and-authenticate) Um bearer token é uma credencial temporária usada para [autenticar na API REST em um ambiente não confiável.](/guides/cloud-setup-untrusted#authenticate-to-the-rest-api-in-an-untrusted-environment) Um bearer token não pode ser usado para autenticar no serviço Qiskit Runtime.

## Preciso escolher a região em que moro? {#do-i-have-to-choose-the-region-that-i-live-in}

Não. Você pode usar qualquer uma das regiões. A região determina onde os jobs são executados e onde os dados são armazenados. Além disso, você só pode ver e usar instâncias criadas na instância em que está conectado. Observe que apenas a região us-east oferece suporte ao Plano Open.

## Posso me conectar por meio de um servidor proxy? {#can-i-connect-through-a-proxy-server}

Sim, mas você deve usar o Qiskit Runtime v0.44.0 ou posterior.

## Qual é a política de retenção de dados? {#what-is-the-data-retention-policy}

A IBM retém seus dados de jobs e uso (não excluídos) por três anos enquanto você continuar usando o serviço Qiskit Runtime.

## O que aconteceu com as distribuições Python como Qiskit Aer, Qiskit Nature e Qiskit Experiments? {#what-happened-to-python-distributions-such-as-qiskit-aer-qiskit-nature-and-qiskit-experiments}

Desde o lançamento do Qiskit SDK v1.0, o comando `pip install qiskit` instala **apenas** o [Qiskit SDK](https://github.com/Qiskit/qiskit). Isso significa que alguns módulos que você usou no passado, como `qiskit-aer` ou `qiskit-nature`, agora precisam ser instalados separadamente.

Antes do lançamento do Qiskit SDK v1.0, o Qiskit era composto por muitas distribuições Python diferentes, como `qiskit-terra`, o núcleo do compilador; `qiskit-aer`, o simulador de alto desempenho; o provedor IBM Quantum&reg; original; e vários pacotes atualmente obsoletos que forneciam recursos exploratórios específicos de algoritmos ou execução de experimentos.

Esses pacotes de software separados foram combinados no que era chamado de *metapacote* do Qiskit, que permitia aos usuários instalar todos esses componentes juntos por meio de `pip install qiskit`. Essa arquitetura de metapacote foi removida com o lançamento do Qiskit SDK v1.0, e agora cada distribuição de software individual precisa ser instalada separadamente.

Se você quiser instalar os pacotes que estavam anteriormente incluídos no metapacote do Qiskit, visite o [ecossistema Qiskit](https://www.ibm.com/quantum/ecosystem) para encontrar uma variedade de pacotes que atendam às suas necessidades. Você também pode ler o [guia de migração v1.0](/guides/metapackage-migration) para obter mais informações sobre essa mudança.

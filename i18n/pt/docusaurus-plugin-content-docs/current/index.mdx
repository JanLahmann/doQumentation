## Conteúdo de código aberto da IBM Quantum {#ibm-quantums-open-source-content}

A IBM fornece uma grande quantidade de material de aprendizado sobre computação quântica — todo em código aberto sob CC BY-SA 4.0:

- **[Learning](https://quantum.cloud.ibm.com/learning)** — Cursos estruturados desde os fundamentos quânticos até tópicos avançados
- **[Tutorials](https://quantum.cloud.ibm.com/docs/en/tutorials)** — Mais de 40 tutoriais sobre transpilação, mitigação de erros e muito mais
- **[Documentation](https://quantum.cloud.ibm.com/docs/en/guides)** — Guias e referência de API para Qiskit
- **[Source repo](https://github.com/Qiskit/documentation)** — Todo o conteúdo no GitHub

Sua [Quantum Platform](https://quantum.cloud.ibm.com) está sempre atualizada e bem projetada — o melhor lugar para leitura, aprendizado e referência.

## O que este projeto adiciona {#what-this-project-adds}

A [documentação do Qiskit](https://github.com/Qiskit/documentation) da IBM é de código aberto (CC BY-SA 4.0), mas seu aplicativo web não é. doQumentation adiciona um frontend de código aberto com execução de código ao vivo, injeção automática de credenciais e modo simulador.

Pode ser implantado em qualquer lugar — de [GitHub Pages](https://doqumentation.org) a [Docker](https://github.com/JanLahmann/doQumentation/pkgs/container/doqumentation) a [RasQberry](https://rasqberry.org/). Veja [todos os recursos](/features).

## Execução de código {#code-execution}

Clique em **Run** em qualquer bloco de código. O primeiro clique inicia um kernel Jupyter gratuito via [Binder](https://mybinder.org) (1–2 min). Depois disso, as execuções são instantâneas.

- **Modo Simulador (não precisa de conta)** — Habilite o [Modo Simulador](/jupyter-settings#simulator-mode) para executar todos os notebooks com AerSimulator ou FakeBackends com modelo de ruído. Nenhuma configuração necessária.

- **Hardware IBM Quantum** — Insira seu token de API e CRN nas [Configurações](/jupyter-settings#ibm-quantum) uma vez — as credenciais são auto-injetadas em cada execução. [Crie uma conta gratuita](https://quantum.cloud.ibm.com/registration) se você não tiver uma.

<details>
<summary>Backends de execução disponíveis</summary>

Todo tutorial tem blocos de código executáveis. Clique em **Run** para executá-los usando um dos três backends:

1. **Binder** (padrão no GitHub Pages) — Kernel Jupyter remoto gratuito via [mybinder.org](https://mybinder.org)
2. **Jupyter local (Docker / RasQberry)** — Conecta ao servidor Jupyter local com Qiskit pré-instalado
3. **Servidor customizado** — Aponte para qualquer endpoint Jupyter nas [Configurações](/jupyter-settings#advanced)

</details>

<details>
<summary>Opções de implantação</summary>

doQumentation está disponível como:

- **[GitHub Pages](https://doqumentation.org)** — Site estático com [Binder](https://mybinder.org) para execução remota de código
- **[Docker](https://github.com/JanLahmann/doQumentation/pkgs/container/doqumentation)** — Pilha completa localmente, capaz de funcionar offline
- **[RasQberry](https://rasqberry.org/)** — Auto-hospedado em Raspberry Pi com kernel Jupyter local

</details>

<details>
<summary>Execute localmente com Podman / Docker</summary>

Instale [Podman Desktop](https://podman-desktop.io/) ou [Docker Desktop](https://www.docker.com/products/docker-desktop/) (Mac / Windows / Linux), então execute:

```bash
# Pilha completa: site + Jupyter + Qiskit (~3 GB)
podman run -p 8080:80 -p 8888:8888 ghcr.io/janlahmann/doqumentation:jupyter
```

Abra [http://localhost:8080](http://localhost:8080) — a execução de código funciona localmente, sem tempo de espera do Binder.

Para uma versão leve sem execução local de código (~60 MB):

```bash
# Apenas site estático — a execução de código ainda funciona via Binder
podman run -p 8080:80 ghcr.io/janlahmann/doqumentation:latest
```

Usando Docker em vez disso? Apenas substitua `podman` por `docker` — os comandos são idênticos. As imagens são multi-arquitetura (`linux/amd64` + `linux/arm64`), então Macs Apple Silicon funcionam nativamente.

</details>

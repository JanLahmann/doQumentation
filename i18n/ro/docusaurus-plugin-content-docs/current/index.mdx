## Conținutul open-source IBM Quantum {#ibm-quantums-open-source-content}

IBM oferă o mulțime de materiale de învățare despre calculul cuantic — toate open-source sub CC BY-SA 4.0:

- **[Learning](https://quantum.cloud.ibm.com/learning)** — Cursuri structurate de la bazele cuantice până la subiecte avansate
- **[Tutorials](https://quantum.cloud.ibm.com/docs/en/tutorials)** — Peste 40 de tutoriale despre transpilare, atenuarea erorilor și altele
- **[Documentation](https://quantum.cloud.ibm.com/docs/en/guides)** — Ghiduri și referință API pentru Qiskit
- **[Source repo](https://github.com/Qiskit/documentation)** — Tot conținutul pe GitHub

[Quantum Platform](https://quantum.cloud.ibm.com) lor este mereu actualizat și bine proiectat — cel mai bun loc pentru citit, învățat și consultare.

## Ce adaugă acest proiect {#what-this-project-adds}

[Documentația Qiskit](https://github.com/Qiskit/documentation) a IBM este open-source (CC BY-SA 4.0), dar aplicația lor web nu este. doQumentation adaugă un frontend open-source cu execuție de cod live, injectare automată a credențialelor și mod simulator.

Poate fi implementat oriunde — de la [GitHub Pages](https://doqumentation.org) la [Docker](https://github.com/JanLahmann/doQumentation/pkgs/container/doqumentation) la [RasQberry](https://rasqberry.org/). Vezi [toate funcțiile](/features).

## Execuția codului {#code-execution}

Apasă **Run** pe orice bloc de cod. Prima apăsare pornește un kernel Jupyter prin [Binder](https://mybinder.org) sau [IBM Code Engine](/jupyter-settings#code-engine). După aceea, rulările sunt instantanee.

- **Modul Simulator (fără cont)** — Activează [Modul Simulator](/jupyter-settings#simulator-mode) ca să rulezi toate notebook-urile cu AerSimulator sau FakeBackends cu model de zgomot. Zero configurare necesară.

- **Hardware IBM Quantum** — Introdu token-ul API și CRN-ul în [Setări](/jupyter-settings#ibm-quantum) o singură dată — credențialele sunt injectate automat la fiecare rulare. [Creează un cont gratuit](https://quantum.cloud.ibm.com/registration) dacă nu ai unul.

<details>
<summary>Backend-uri de execuție disponibile</summary>

Fiecare tutorial are blocuri de cod executabile. Apasă **Run** ca să le rulezi folosind unul din cele patru backend-uri:

1. **Binder** (implicit pe GitHub Pages) — Kernel Jupyter la distanță gratuit prin [mybinder.org](https://mybinder.org)
2. **IBM Code Engine** — Jupyter serverless pe contul tău IBM Cloud. Cold start în secunde în loc de minute. [Configurează](/jupyter-settings#code-engine)
3. **Jupyter Local (Docker / RasQberry)** — Se conectează la serverul Jupyter local cu Qiskit preinstalat
4. **Server personalizat** — Indică orice endpoint Jupyter în [Setări](/jupyter-settings#advanced)

Când sunt disponibile mai multe backend-uri, alege care să fie folosit din [Setări](/jupyter-settings#backend-selection).

</details>

<details>
<summary>Opțiuni de implementare</summary>

doQumentation este disponibil ca:

- **[GitHub Pages](https://doqumentation.org)** — Site static cu [Binder](https://mybinder.org) sau [IBM Code Engine](/jupyter-settings#code-engine) pentru execuție de cod la distanță
- **[Docker](https://github.com/JanLahmann/doQumentation/pkgs/container/doqumentation)** — Full stack local, capabil să funcționeze offline
- **[RasQberry](https://rasqberry.org/)** — Self-hosted pe Raspberry Pi cu kernel Jupyter local

</details>

<details>
<summary>Rulează local cu Podman / Docker</summary>

Instalează [Podman Desktop](https://podman-desktop.io/) sau [Docker Desktop](https://www.docker.com/products/docker-desktop/) (Mac / Windows / Linux), apoi rulează:

```bash
# Full stack: site + Jupyter + Qiskit (~3 GB)
podman run -p 8080:80 -p 8888:8888 ghcr.io/janlahmann/doqumentation:jupyter
```

Deschide [http://localhost:8080](http://localhost:8080) — execuția codului funcționează local, fără timp de așteptare Binder.

Pentru o versiune ușoară fără execuție locală de cod (~60 MB):

```bash
# Static site only — code execution still works via Binder
podman run -p 8080:80 ghcr.io/janlahmann/doqumentation:latest
```

Folosești Docker în schimb? Înlocuiește `podman` cu `docker` — comenzile sunt identice. Imaginile sunt multi-arch (`linux/amd64` + `linux/arm64`), deci Mac-urile Apple Silicon funcționează nativ.

</details>

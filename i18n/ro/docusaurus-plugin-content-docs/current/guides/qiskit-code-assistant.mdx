# Qiskit Code Assistant

Qiskit Code Assistant își propune să facă informatica cuantică mai accesibilă noilor utilizatori Qiskit și să îmbunătățească experiența de programare pentru utilizatorii actuali. Este un asistent de cod bazat pe AI generativ, alimentat de [watsonx](https://www.ibm.com/products/watsonx-ai). A fost antrenat folosind milioane de tokeni de text din Qiskit SDK, ani de exemple de cod Qiskit și funcționalități IBM Quantum&reg;. Qiskit Code Assistant poate sprijini fluxul tău de lucru în dezvoltarea cuantică oferind sugestii generate de LLM bazate pe [modelele IBM Granite](https://www.ibm.com/products/watsonx-ai/foundation-models) care încorporează cele mai recente funcționalități și caracteristici de la IBM&reg;.

<Admonition type="note" title="Note">
- Aceasta este o funcționalitate experimentală disponibilă utilizatorilor IBM Quantum Premium Plan înregistrați pe noua [platformă IBM Quantum](https://quantum.cloud.ibm.com/).
- Qiskit Code Assistant se află în stadiu de previzualizare și poate suferi modificări.
- Dacă ai feedback sau dorești să contactezi echipa de dezvoltare, folosește [canalul Qiskit Slack Workspace](https://qiskit.enterprise.slack.com/archives/C07LYA6PL83) sau depozitele publice GitHub aferente.
</Admonition>

## Funcționalități {#features}

Următoarele funcționalități sunt incluse în extensiile pentru [Visual Studio Code](https://code.visualstudio.com/) (VS Code) și editoarele compatibile, precum și pentru [JupyterLab](https://jupyterlab.readthedocs.io/en/latest/):

* Accelerează generarea de cod Qiskit valorificând AI generativ bazat pe modele specializate în generarea de cod Qiskit.
* Permite formularea de prompturi abstracte și specifice pentru a genera recomandări.
* Prezintă sugestii pe care le poți revizui, accepta sau respinge.
* Suportă fișiere cu cod Python și fișiere Jupyter notebook.
* Include măsuri de siguranță pentru a evita răspunsurile la întrebări ce reprezintă un risc potențial pentru utilizatori, cum ar fi discursul instigator la ură.

Pentru instrucțiuni de integrare a Qiskit Code Assistant direct în mediul tău de dezvoltare, urmează instrucțiunile din subiectul corespunzător:
- [JupyterLab](/guides/qiskit-code-assistant-jupyterlab)
- [VS Code](/guides/qiskit-code-assistant-vscode)

## Modelul lingvistic de mari dimensiuni (LLM) din spatele Qiskit Code Assistant {#the-large-language-model-llm-behind-qiskit-code-assistant}

Pentru a oferi sugestii de cod, Qiskit Code Assistant folosește un Model Lingvistic de Mari Dimensiuni (LLM). În acest caz, Qiskit Code Assistant se bazează în prezent pe modelul `mistral-small-3.2-24b-qiskit`, construit pe [modelul Mistral-Small-3.2-24B-Qiskit](https://huggingface.co/Qiskit/mistral-small-3.2-24b-qiskit). Modelul `mistral-small-3.2-24b-qiskit` îmbunătățește capacitățile de generare a codului modelului `Mistral-Small-3.2-24B-Instruct-2506` pentru Qiskit prin preantrenare extinsă și ajustare fină pe date Qiskit de înaltă calitate, precum și pe commit-uri Python și conversații. Pentru mai multe informații despre familia de modele Mistral AI, consultă [documentația Mistral AI](https://docs.mistral.ai/getting-started/models). Pentru mai multe detalii despre modelele `.*-qiskit`, vezi [Qiskit Code Assistant: Training LLMs for generating Quantum Computing Code](https://arxiv.org/abs/2405.19495).
2. [qiskit/qwen2.5-coder-14b-qiskit](https://huggingface.co/Qiskit/qwen2.5-coder-14b-qiskit) - Released June 2025

## Benchmark-urile Qiskit HumanEval și Qiskit HumanEval Hard {#the-qiskit-humaneval-and-qiskit-humaneval-hard-benchmarks}

Pentru a testa modelul `mistral-small-3.2-24b-qiskit` și alte modele, am colaborat cu Qiskit Advocates și experți pentru a crea benchmark-urile bazate pe execuție numite Qiskit HumanEval (QHE) și Qiskit HumanEval Hard (QHE Hard), pe care le-am rulat pe modele. Aceste benchmark-uri sunt similare cu [HumanEval](https://arxiv.org/abs/2107.03374), incluzând multiple probleme de cod dificile de rezolvat, toate bazate pe bibliotecile oficiale Qiskit.

Benchmark-urile sunt compuse din aproximativ 150 de teste, fiecare construit dintr-o definiție de funcție, urmată de un docstring care detaliază sarcina pe care modelul trebuie să o rezolve. Fiecare exemplu include, de asemenea, o soluție canonică de referință, precum și teste unitare, pentru a evalua corectitudinea soluțiilor generate. Există trei niveluri de dificultate pentru teste: de bază, intermediar și dificil. Benchmark-ul Qiskit HumanEval Hard este o variație a celui Qiskit HumanEval, dar elimină informațiile legate de importurile de cod, astfel încât LLM-ul trebuie să determine importurile corecte de metode sau clase. Această modificare face setul de date mult mai dificil pentru LLM-uri, conform testelor și rezultatelor noastre inițiale.

Seturile de date pentru Qiskit HumanEval și Qiskit HumanEval Hard sunt disponibile pe aceste site-uri: [Qiskit HumanEval](https://huggingface.co/datasets/Qiskit/qiskit_humaneval) și [Qiskit HumanEval](https://huggingface.co/datasets/Qiskit/qiskit_humaneval_hard). Poți contribui la dezvoltarea acestor benchmark-uri la [depozitul GitHub](https://github.com/qiskit-community/qiskit-human-eval).

### Pornire rapidă (recomandat) {#quick-start-recommended}

Cel mai simplu mod de a începe cu Qiskit Code Assistant în modul local este să folosești scripturile de configurare automată pentru extensia VS Code sau JupyterLab. Aceste scripturi vor instala automat [Ollama](https://ollama.com/) pentru a rula LLM-urile, vor descărca modelul recomandat și vor configura extensia pentru tine.

#### Configurarea extensiei VS Code {#vs-code-extension-setup}

Rulează următoarea comandă în terminal:

```bash
bash <(curl -fsSL https://raw.githubusercontent.com/Qiskit/qiskit-code-assistant-vscode/main/setup_local.sh)
```

Acest script efectuează următorii pași:
- Instalează Ollama (dacă nu este deja instalat)
- Descarcă și configurează modelul Qiskit Code Assistant recomandat
- Configurează extensia VS Code pentru a lucra cu implementarea ta locală

#### Configurarea extensiei JupyterLab {#jupyterlab-extension-setup}

Rulează următoarea comandă în terminal:

```bash
bash <(curl -fsSL https://raw.githubusercontent.com/Qiskit/qiskit-code-assistant-jupyterlab/main/setup_local.sh)
```

Acest script va:
- Instala Ollama (dacă nu este deja instalat)
- Descărca și configura modelul Qiskit Code Assistant recomandat
- Configura extensia JupyterLab pentru a lucra cu implementarea ta locală

##### Configurează Ollama cu un model GGUF Qiskit Code Assistant descărcat manual {#set-up-ollama-with-a-manually-downloaded-qiskit-code-assistant-gguf-model}

Dacă ai descărcat manual un model GGUF, de exemplu https://huggingface.co/Qiskit/Qwen2.5-Coder-14B-Qiskit-GGUF, și dorești să experimentezi cu diferite template-uri și parametri, poți urma acești pași pentru a-l încărca în aplicația ta locală Ollama.

1. Creează un `Modelfile` introducând următorul conținut și asigură-te că actualizezi `<PATH-TO-GGUF-FILE>` cu calea reală a modelului descărcat.

    ```
    FROM <PATH-TO-GGUF-FILE>
    TEMPLATE """{{ if .System }}
    System:
    {{ .System }}

    {{ end }}{{ if .Prompt }}Question:
    {{ .Prompt }}

    {{ end }}Answer:
    ```python{{ .Response }}
    """

    PARAMETER stop "Question:"
    PARAMETER stop "Answer:"
    PARAMETER stop "System:"
    PARAMETER stop "```"

    PARAMETER temperature 0
    PARAMETER top_k 1
    ```

1. Run the following command to create a custom model instance based on the `Modelfile`.

    ```
    ollama create Qwen2.5-Coder-14B-Qiskit -f ./path-to-model-file
    ```

    <Admonition type="note">
        This process may take some time for Ollama to read the model file, initialize the model instance, and configure it according to the specifications provided.
    </Admonition>

<AccordionItem title="Manually deploy the Qiskit Code Assistant models in local through the `llama-cpp-python` package">

<span id="use-llama-cpp-python"></span>
An alternative to the Ollama application is the `llama-cpp-python` package, which is a Python binding for `llama.cpp`. It gives you more control and flexibility to run the GGUF model locally, and is ideal for users who wish to integrate the local model in their workflows and Python applications.

1. Install [`llama-cpp-python`](https://pypi.org/project/llama-cpp-python/)
2. Interact with the model from within your application using `llama_cpp`. For example:

```python
from llama_cpp import Llama

model_path = <PATH-TO-GGUF-FILE>

model = Llama(
        model_path,
        seed=17,
        n_ctx=10000,
        n_gpu_layers=37, # to offload in gpu, but put 0 if all in cpu
    )

input = 'Generate a quantum circuit with 2 qubits'
raw_pred = model(input)["choices"][0]["text"]
```

You can also add text generation parameters to the model to customize the inference:

```python
generation_kwargs = {
        "max_tokens": 512,
        "echo": False, # Echo the prompt in the output
        "top_k": 1
    }

raw_pred = model(input, **generation_kwargs)["choices"][0]["text"]
```

</AccordionItem>

#### Use the `llama.cpp` library

Another alternative is to use `llama.cpp`, an open-source library for performing LLM inference on a CPU with minimal setup.
It provides low-level control over the model execution and is typically run from the command line, pointing to a local GGUF model file.

There are several ways to install `llama.cpp` on your machine:

- Install llama.cpp using [brew, nix, or winget](https://github.com/ggml-org/llama.cpp/blob/master/docs/install.md)
- Run with Docker: See out the [Docker documentation by `llama.cpp` team](https://github.com/ggml-org/llama.cpp/blob/master/docs/docker.md)
- Download pre-built binaries from the [releases page](https://github.com/ggml-org/llama.cpp/releases)
- Build from source by [cloning this repository](https://github.com/ggml-org/llama.cpp/blob/master/docs/build.md)

Once installed, you can use `llama.cpp` to interact with GGUF models in conversation mode as follows:
```bash
# Use a local model file
llama-cli -m my_model.gguf -cnv

# Or download and run a model directly from Hugging Face
llama-cli -hf Qiskit/Qwen2.5-Coder-14B-Qiskit-GGUF -cnv
```

You can also launch an OpenAI-compatible API server for the model in the following way:
```bash
llama-server -hf Qiskit/Qwen2.5-Coder-14B-Qiskit-GGUF
```

##### Advanced parameters
With the `llama-cli` program, you can control the model generation using command-line options. For example, you can provide an initial “system” prompt using the `-p/--prompt` flag. In conversation mode (`-cnv`), this initial prompt acts as the system message. Otherwise, you can simply prepend any desired instruction to your prompt text. You can also adjust sampling parameters - for instance: temperature (`--temp`), top-k (`--top-k`), top-p (`--top-p`), repetition penalty (`--repeat-penalty`), and the seed to use (`--seed`). The following is an example invocation using these options:

```bash
llama-cli -hf Qiskit/Qwen2.5-Coder-14B-Qiskit-GGUF \
  -p "You are a friendly assistant." -cnv \
  --temp 0.7 \
  --top-k 50 \
  --top-p 0.95 \
  --repeat-penalty 1.1 \
  --seed 42
```

Pentru a asigura funcționarea corectă a modelelor noastre Qiskit, recomandăm folosirea promptului de sistem furnizat în depozitele noastre HF GGUF: promptul de sistem pentru [mistral-small-3.2-24b-qiskit-GGUF](https://huggingface.co/Qiskit/mistral-small-3.2-24b-qiskit-GGUF/blob/main/system), [Qwen2.5-Coder-14B-Qiskit-GGUF](https://huggingface.co/Qiskit/Qwen2.5-Coder-14B-Qiskit-GGUF/blob/main/system), [granite-3.3-8b-qiskit-GGUF](https://huggingface.co/Qiskit/granite-3.3-8b-qiskit-GGUF/blob/main/system) și [granite-3.2-8b-qiskit-GGUF](https://huggingface.co/Qiskit/granite-3.2-8b-qiskit-GGUF/blob/main/system).

<AccordionItem title="Conectează manual extensiile la implementarea locală">

Folosește extensia VS Code și extensia JupyterLab pentru Qiskit Code Assistant pentru a trimite prompturi modelului Qiskit Code Assistant implementat local. Odată ce ai aplicația Ollama [configurată cu modelul](#using-the-ollama-application), poți configura extensiile pentru a se conecta la serviciul local.

#### Conectează-te cu extensia Qiskit Code Assistant VS Code {#connect-with-the-qiskit-code-assistant-vs-code-extension}

Cu extensia Qiskit Code Assistant VS Code, poți interacționa cu modelul și efectua completarea codului în timp ce scrii codul. Aceasta poate funcționa bine pentru utilizatorii care caută asistență în scrierea codului Qiskit pentru aplicațiile lor Python.

1. Instalează [extensia Qiskit Code Assistant VS Code](/guides/qiskit-code-assistant-vscode).
1. În VS Code, mergi la **User Settings** și setează **Qiskit Code Assistant: Url** la URL-ul implementării tale locale Ollama (de exemplu, `http://localhost:11434`).
1. Reîncarcă VS Code mergând la **View > Command Palette...** și selectând **Developer: Reload Window**.

Modelul Qiskit Code Assistant configurat în Ollama ar trebui să apară în bara de stare și este gata de utilizare.

#### Conectează-te cu extensia Qiskit Code Assistant JupyterLab {#connect-with-the-qiskit-code-assistant-jupyterlab-extension}

Cu extensia Qiskit Code Assistant JupyterLab, poți interacționa cu modelul și efectua completarea codului direct în Jupyter Notebook. Utilizatorii care lucrează predominant cu Jupyter Notebooks pot profita de această extensie pentru a-și îmbunătăți experiența de scriere a codului Qiskit.

1. Instalează [extensia Qiskit Code Assistant JupyterLab](/guides/qiskit-code-assistant-jupyterlab).
1. În JupyterLab, mergi la **Settings Editor** și setează **Qiskit Code Assistant Service API** la URL-ul implementării tale locale Ollama (de exemplu, `http://localhost:11434`).

Modelul Qiskit Code Assistant configurat în Ollama ar trebui să apară în bara de stare și este gata de utilizare.

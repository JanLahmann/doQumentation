Formulele multi-produs (MPF) pot fi utilizate pentru a simula mai precis dinamica unui sistem cuantic, cu costul unor execuții suplimentare de Circuit. Aceasta este o tehnică de post-procesare care reduce eroarea valorilor de așteptare pentru stările evoluate în timp.

Calculul clasic este utilizat pentru a rezolva un sistem de ecuații liniare care furnizează coeficienți pentru o combinație ponderată a mai multor execuții de Circuit. Folosind această combinație ponderată se poate reduce eroarea asociată simulării evoluției în timp, dată o selecție bună a pașilor Trotter. Instrumentul MPF va prelua o selecție de date --- inclusiv numărul de pași Trotter și ordinul aproximației Trotter --- pentru a pregăti și rezolva (sau a aproxima soluția) sistemul asociat de ecuații liniare, pe care îl poți folosi ulterior pentru a post-procesa măsurătorile valorilor de așteptare ale unei stări evoluate în timp.
## Instalarea pachetului MPF {#install-the-mpf-package}

Există două modalități de a instala pachetul MPF: prin PyPI și prin compilare din sursă. Este recomandat să instalezi într-un [mediu virtual](https://docs.python.org/3.10/tutorial/venv.html) pentru a asigura separarea dintre dependențele pachetelor.

### Instalare din PyPI {#install-from-pypi}

Cea mai simplă modalitate de a instala pachetul `qiskit-addon-mpf` este prin PyPI.
```bash
pip install qiskit-addon-mpf
```

### Compilare din sursă {#build-from-source}
Utilizatorii care doresc să dezvolte în depozit sau care vor să îl instaleze manual pot face acest lucru clonând mai întâi depozitul:
```bash
git clone git@github.com:Qiskit/qiskit-addon-mpf.git
```

și instalând pachetul prin `pip`. Depozitul conține, de asemenea, o serie de dependențe opționale care activează anumite funcționalități.

Ajustează opțiunile pentru a corespunde nevoilor tale.
```bash
pip install tox notebook -e '.[notebook-dependencies,dev]'
```
## Context teoretic {#theoretical-background}

MPF-urile pot reduce eroarea de aproximare Trotter asociată simulării dinamicii sistemelor cuantice printr-o combinație ponderată a mai multor execuții de Circuit. Această sumă ponderată este definită ca:

$$ \mu(t) = \sum_j x_j\rho_j^{k_j}\left(\frac{t}{k_j}\right) + \text{some remaining Trotter error}, $$

unde $x_j$ sunt coeficienții de ponderare, $\rho_j^{k_j}$ este matricea de densitate care corespunde stării pure obținute prin evoluția stării inițiale cu ajutorul unei formule produs $S^{k_j}$ care aproximează operatorul de evoluție temporală folosind $k_j$ pași Trotter, iar $j$ indexează fiecare formulă produs utilizată în sumă.

![Mai multe Circuit-uri cu un număr variabil de pași Trotter sunt folosite pentru a calcula observabila țintă](/docs/images/guides/qiskit-addons/mpf-diagram.avif)

În general, însă, scopul simulării dinamicii cuantice este de a măsura o anumită observabilă $\mathcal{O}(t)$, care este o funcție de timp. Când se folosesc MPF-uri, mai multe Circuit-uri -- fiecare folosind $k_j$ pași Trotter -- sunt executate pentru a obține mai multe măsurători ale observabilei țintă $\mathcal{O}_{k_j}(t)$. Măsurătoarea observabilei țintă este apoi obținută calculând:

$$ \langle \mathcal{O}(t) \rangle = \sum_j x_j(t) \langle \mathcal{O}_{k_j}(t) \rangle. $$

În esență, poți reduce eroarea totală Trotter aproximând operatorul de evoluție temporală cu mai multe formule produs cu un număr variabil de pași Trotter în loc de o singură formulă produs. Construiești un Circuit pentru fiecare termen din suma ponderată, care evoluează sistemul conform fiecăruia dintre cei $k_j$ pași Trotter. Fiecare Circuit este apoi executat separat pe un QPU pentru a reconstitui rezultatele într-un pas de post-procesare. Utilitatea acestei tehnici poate fi privită din două perspective:
1. Pentru un număr fix de pași Trotter pe care îi execuți, poți obține rezultate cu o eroare Trotter mai mică în total.
2. Pentru un număr de pași Trotter care duc la Circuit-uri adânci, poți folosi MPF pentru a găsi mai multe Circuit-uri de adâncime mai mică de rulat, ceea ce duce la o eroare de aproximare Trotter similară.

## Determinarea coeficienților MPF {#determine-mpf-coefficients}

Funcționalitatea de bază a pachetului `qiskit-addon-mpf` constă în determinarea coeficienților MPF $x_j(t)$ (care pot fi dependenți de timp). Procesul de obținere a fiecărui $x_j(t)$ implică rezolvarea unui sistem de ecuații liniare $Ax=b$, unde $x$ este vectorul coeficienților ce trebuie determinați, $A$ este o matrice care depinde de fiecare $k_j$ și de formula produs utilizată $S$ (adică ordinul aproximării și numărul de pași Trotter), iar $b$ este un vector de constrângeri. Acest sistem de ecuații poate fi rezolvat fie exact, fie cu un model aproximativ care minimizează norma 1 a coeficienților. De asemenea, alegerea fiecărui $k_j$ este un proces euristic, dar poate fi limitată de următoarele constrângeri:

1. Cea mai **mare** valoare $k_j$ este limitată de adâncimea maximă a Circuit-ului care poate fi executat în mod fiabil
2. Cel mai **mic** $k_j$ ar trebui să satisfacă $dt = t/k_j < 1$, deoarece acolo eroarea Trotter se comportă cel mai bine
3. Niciun coeficient $x_j$ nu ar trebui să fie aproape de $0$, deoarece acest lucru ar implica că nu contribuie mult la MPF
4. De asemenea, coeficientul asociat cu cea mai mare valoare $k_j$ nu ar trebui să domine, deoarece acest lucru implică că folosești o singură formulă produs
5. În fine, norma coeficienților $x_j$ obținuți ar trebui să fie mică, deoarece acest lucru indică un MPF bine condiționat [1](#references)

# Folosește endpoint-uri private virtuale pentru VPC pentru a te conecta în mod privat la IBM Quantum Platform {#use-virtual-private-endpoints-for-vpc-to-privately-connect-to-ibm-quantum-platform}

Folosește Virtual Private Endpoints (VPE) pentru VPC pentru a te conecta la IBM Quantum&reg; din rețeaua ta VPC, utilizând adresa IP specificată de tine, alocată dintr-un subnet din VPC-ul tău.

VPE-urile sunt interfețe IP virtuale legate de un gateway de endpoint creat pentru fiecare serviciu în parte. Gateway-ul de endpoint este o funcție virtualizată care scalează orizontal, este redundantă și înalt disponibilă și acoperă toate zonele de disponibilitate din VPC-ul tău. Gateway-urile de endpoint permit comunicarea dintre instanțele de server virtual din VPC-ul tău și serviciul IBM Cloud&reg; prin intermediul rețelei private. VPE pentru VPC îți permite să controlezi toate adresările private din cloud-ul tău. Pentru mai multe informații, consultă [About virtual private endpoint gateways](https://cloud.ibm.com/docs/vpc?topic=vpc-about-vpe).

În cadrul IBM Quantum Platform, toate datele clienților sunt transmise prin rețeaua privată, indiferent dacă sunt accesate printr-un endpoint public sau VPE.

Conectarea la IBM Quantum Platform prin endpoint-urile publice transmite toate datele de cerere și răspuns prin internetul public. Pentru a te conecta la IBM Quantum Platform folosind un VPE, trebuie să utilizezi IBM Quantum Platform API sau SDK. IBM Quantum Platform poate fi accesată doar prin rețeaua publică.

## Înainte de a începe {#before-you-begin}

Înainte de a configura un VPE pentru IBM Quantum Platform, parcurge următorii pași:

* [Creează un Virtual Private Cloud](https://cloud.ibm.com/docs/vpc?topic=vpc-getting-started).
* [Planifică topologia de rețea](https://cloud.ibm.com/docs/vpc?topic=vpc-planning-considerations) pentru conectarea la VPE-uri.
* [Configurează controalele de acces](https://cloud.ibm.com/docs/vpc?topic=vpc-configure-acls-sgs-endpoint-gateways) pentru VPE-ul tău.
* Înțelege [limitările](https://cloud.ibm.com/docs/vpc?topic=vpc-limitations-vpe) unui VPE.
* Înțelege cum să [vizualizezi detaliile VPE](https://cloud.ibm.com/docs/vpc?topic=vpc-vpe-viewing-details-of-an-endpoint-gateway).

## Configurează un VPE pentru IBM Quantum Platform {#set-up-a-vpe-for-ibm-quantum-platform}

Există mai multe moduri de a crea un gateway VPE. Dacă folosești [CLI](https://cloud.ibm.com/docs/vpc?topic=vpc-vpc-reference&interface=cli#vpe-clis) sau [API](https://cloud.ibm.com/apidocs/vpc/latest#create-endpoint-gateway), trebuie să specifici [Cloud Resource Name (CRN)](https://cloud.ibm.com/docs/account?topic=account-crn) al regiunii în care dorești să te conectezi la IBM Quantum Platform. Consultă tabelul următor pentru regiunile disponibile și CRN-urile aferente.

| Regiune | Planuri   | Fully Qualified Domain Name (FDQN) | Cloud Resource Name (CRN) |
|-----------------|-----------------|-----------------|-----------------|
| `us-east` | `Standard` | `private.us-east.quantum.cloud.ibm.com` | `crn:v1:bluemix:public:quantum-computing:us-east:::endpoint:private.us-east.quantum.cloud.ibm.com` |
| `eu-de` | `Standard` | `private.eu-de.quantum.cloud.ibm.com` | `crn:v1:bluemix:public:quantum-computing:eu-de:::endpoint:private.eu-de.quantum.cloud.ibm.com` |

### Configurarea unui gateway de endpoint {#configuring-an-endpoint-gateway}

Pentru a configura un gateway VPE, urmează acești pași:

1. Listează serviciile disponibile, inclusiv serviciile de infrastructură IBM Cloud disponibile (implicit) pentru toți utilizatorii VPC.
1. [Creează un gateway de endpoint](https://cloud.ibm.com/docs/vpc?topic=vpc-ordering-endpoint-gateway) pentru IBM Quantum Platform pe care dorești să îl faci disponibil în mod privat în VPC.
1. [Leagă o adresă IP rezervată](https://cloud.ibm.com/docs/vpc?topic=vpc-bind-unbind-reserved-ip) de gateway-ul de endpoint.
1. [Vizualizează gateway-urile VPE create](https://cloud.ibm.com/docs/vpc?topic=vpc-vpe-viewing-details-of-an-endpoint-gateway) asociate cu IBM Quantum Platform.

Acum instanțele tale de server virtual din VPC pot accesa în mod privat instanța ta IBM Quantum Platform.

## Folosește VPE-ul pentru IBM Quantum Platform {#use-your-vpe-for-ibm-quantum-platform}

După ce ai creat un gateway de endpoint pentru IBM Quantum Platform, urmează acești pași:

### Folosește VPE cu `qiskit-ibm-runtime` (Python SDK) {#use-the-vpe-with-qiskit-ibm-runtime-python-sdk}

<Admonition type="note">
Suportul VPE necesită `qiskit-ibm-runtime` v0.24.0 sau o versiune ulterioară.
</Admonition>

Când instanțiezi `QiskitRuntimeService`, specifică `private_endpoint=True`.

```python
service = QiskitRuntimeService(token="your-API_KEY", # Use the 44-character API_KEY you created and saved from the IBM Quantum Platform Home dashboard
                              instance="INSTANCE_CRN",
                              channel="ibm_cloud",
                              private_endpoint=True)
```

### Folosește VPE cu IBM Quantum API {#use-the-vpe-with-the-ibm-quantum-api}

După crearea unui gateway de endpoint pentru IBM Quantum Platform, folosește FQDN-ul endpoint-ului de serviciu pentru regiunea țintă.

```bash
  curl -X POST https://private.us-east.quantum.cloud.ibm.com/api/v1/jobs -H "Authorization: Bearer $BEARER_TOKEN" -H "Service-CRN: $SERVICE_INSTANCE_CRN" -d '{
    "backend": "backend",
    "program_id": "sampler"
  }'
```

# Întrebări frecvente (FAQ) {#frequently-asked-questions-faq}

Pentru întrebările frecvente specifice modurilor de execuție, consultă ghidul [FAQ pentru modurile de execuție Qiskit Runtime](/guides/execution-modes-faq).

## Cum citez Qiskit în cercetarea mea? {#how-do-i-cite-qiskit-in-my-research}

Citează Qiskit folosind [fișierul BibTeX](https://github.com/Qiskit/qiskit/blob/main/CITATION.bib) inclus.

## Cum citez un calculator IBM Quantum în cercetarea mea? {#how-do-i-cite-an-ibm-quantum-computer-in-my-research}

Pentru articolele de cercetare, îi încurajăm pe autori să recunoască IBM Quantum&reg; folosind:

>    Recunoaștem utilizarea serviciilor IBM Quantum pentru această lucrare. Opiniile exprimate aparțin autorilor și nu reflectă politica oficială sau poziția IBM sau a echipei IBM Quantum.

Referințele din articol ar trebui să folosească următoarea citare:

> IBM Quantum. https://<span>quantum.cloud.ibm.com</span>/, YYYY

Unitățile de procesare cuantică din articol ar trebui referențiate prin numele lor unic (de exemplu, `ibmq_vigo`) și opțional
adăugând versiunea (de exemplu, `ibmq_vigo` v1.0.2). Îi încurajăm, de asemenea, să referențieze procesorul.
De exemplu:

> În această lucrare am folosit `ibmq_vigo`, care este unul dintre procesoarele IBM Quantum Canary.

Exemplu de citare a unui program IBM Quantum:

> IBM Quantum (2025). Estimator primitive (Version x.y.z) [computer software]. https://<span>quantum.cloud.ibm.com</span>/

## Cum citez IBM Quantum Composer în cercetarea mea? {#how-do-i-cite-ibm-quantum-composer-in-my-research}

Exemplu de citare pentru IBM Quantum Composer:

> IBM Quantum Composer. 2025. url: https://<span>quantum.cloud.ibm.com/composer</span>

## Nu văd ceva în interfața de utilizare, cum ar fi sarcinile mele administrative. {#i-dont-see-something-in-the-user-interface-such-as-my-administration-tasks}

Încearcă să alegi un alt cont din selectorul de conturi din antetul interfeței [IBM Quantum Platform](https://quantum.cloud.ibm.com).

## Folosesc doar Planul Open, dar a trebuit să introduc cardul de credit. Voi fi taxat? {#im-just-using-the-open-plan-but-i-had-to-put-in-my-credit-card-will-i-be-charged}

Nu vei fi taxat dacă nu creezi o [instanță](/guides/instances) Pay-As-You-Go și nu o folosești pentru a trimite sarcini către QPU-uri. Consultă [Gestionarea costurilor](/guides/manage-cost) pentru instrucțiuni despre cum să setezi o limită de cost și pentru a afla alte modalități de a gestiona costurile.

## Care este diferența dintre o cheie API și un token API? {#whats-the-difference-between-an-api-key-and-an-api-token}

Nu există nicio diferență. Termenii cheie API și token API sunt folosiți interschimbabil. Vei vedea ambii termeni în documentație deoarece diferite interfețe folosesc termeni diferiți.

<span id="bearer"></span>
## Care este diferența dintre un token API și un token bearer? {#whats-the-difference-between-an-api-token-and-a-bearer-token}

Un token API este o acreditare de lungă durată folosită pentru [autentificarea la serviciul Qiskit Runtime.](/guides/hello-world#install-and-authenticate) Un token bearer este o acreditare temporară folosită pentru [autentificarea la REST API într-un mediu neîncrezut.](/guides/cloud-setup-untrusted#authenticate-to-the-rest-api-in-an-untrusted-environment) Un token bearer nu poate fi folosit pentru autentificarea la serviciul Qiskit Runtime.

## Trebuie să aleg regiunea în care locuiesc? {#do-i-have-to-choose-the-region-that-i-live-in}

Nu. Poți folosi oricare regiune. Regiunea determină unde sunt rulate lucrările și unde sunt păstrate datele. În plus, poți vedea și folosi doar instanțele care au fost create în instanța în care ești autentificat. Reține că doar regiunea us-east suportă Planul Open.

## Pot să mă conectez printr-un server proxy? {#can-i-connect-through-a-proxy-server}

Da, dar trebuie să folosești Qiskit Runtime v0.44.0 sau o versiune ulterioară.

## Care este politica de retenție a datelor? {#what-is-the-data-retention-policy}

IBM păstrează datele tale despre lucrări și utilizare (neșterse) timp de trei ani, cât timp continui să folosești serviciul Qiskit Runtime.

## Ce s-a întâmplat cu distribuțiile Python cum ar fi Qiskit Aer, Qiskit Nature și Qiskit Experiments? {#what-happened-to-python-distributions-such-as-qiskit-aer-qiskit-nature-and-qiskit-experiments}

Începând cu lansarea Qiskit SDK v1.0, comanda `pip install qiskit` va instala **doar** [Qiskit SDK](https://github.com/Qiskit/qiskit). Aceasta înseamnă că unele module pe care le-ai folosit în trecut, cum ar fi `qiskit-aer` sau `qiskit-nature`, trebuie acum instalate separat.

Înainte de lansarea Qiskit SDK v1.0, Qiskit era alcătuit din multe distribuții Python diferite, cum ar fi `qiskit-terra`, nucleul compilatorului; `qiskit-aer`, simulatorul de înaltă performanță; furnizorul original IBM Quantum&reg;; și mai multe pachete acum obsolete care ofereau funcționalități algoritmice exploratorii sau de rulare a experimentelor.

Aceste pachete software separate au fost combinate în ceea ce se numea *metapachetul* Qiskit, care le permitea utilizatorilor să instaleze toate aceste componente împreună prin `pip install qiskit`. Această arhitectură de metapachet a fost eliminată odată cu lansarea Qiskit SDK v1.0, iar acum fiecare dintre distribuțiile software individuale trebuie instalată separat.

Dacă vrei să instalezi pachetele care erau incluse anterior în metapachetul Qiskit, vizitează [ecosistemul Qiskit](https://www.ibm.com/quantum/ecosystem) pentru a găsi o gamă de pachete care să corespundă nevoilor tale. Poți citi, de asemenea, [ghidul de migrare v1.0](/guides/metapackage-migration) pentru mai multe informații despre această schimbare.

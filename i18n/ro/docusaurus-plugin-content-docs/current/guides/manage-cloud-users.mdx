# Gestionează utilizatorii IBM Cloud {#manage-ibm-cloud-users}

Acest ghid demonstrează cum să folosești IBM Cloud&reg; pentru a activa utilizatori care au conturi IBM Cloud și oferă instrucțiuni pentru ca utilizatorii să acceseze mediul.

Pentru a gestiona în schimb utilizatorii furnizorului de identitate, urmează instrucțiunile din subiectul [Gestionează furnizorii de identitate](/guides/manage-appid).

## Pași {#steps}

Urmează acești pași la nivel înalt pentru a gestiona utilizatorii IBM Cloud:

1. Asigură-te că utilizatorii pe care vrei să îi inviți au conturi IBM Cloud.
2. Accesează IAM și [invită utilizatori](https://cloud.ibm.com/iam/users/invite_users)
   <Admonition type="note">
   Utilizatorii nu pot fi gestionați până nu acceptă invitația și nu se conectează cel puțin o dată.
   </Admonition>
3. Opțional: Modifică atribuirile de proiecte și grupurile de acces ale utilizatorilor din [IAM.](https://cloud.ibm.com/iam/users)

<span id="user-org"></span>
## Fluxul utilizatorului {#user-flow}

1. Utilizatorul acceptă o invitație, apoi se conectează prin [portalul IBM Cloud](https://cloud.ibm.com/).
2. Utilizatorul creează o cheie API accesând ([Manage → Access (IAM) → API keys](https://cloud.ibm.com/iam/apikeys)).
3. Pentru informații suplimentare, utilizatorii pot consulta [Instalează Qiskit.](/guides/install-qiskit)

## Scenariu exemplu {#example-scenario}

Acest exemplu creează următoarea configurare:

* Există două proiecte, `ml` și `finance`.
   * Proiectul `ml` ar trebui să aibă acces la instanțele de serviciu `QR-ml` și `QR-common`.
   * Proiectul `finance` ar trebui să aibă acces la instanțele de serviciu `QR-finance` și `QR-common`.
* Există trei utilizatori:
   * Fatima ar trebui să aibă acces la proiectul `ml`.
   * Ravi ar trebui să aibă acces la proiectul `finance`.
   * Amyra ar trebui să aibă acces la ambele proiecte.
* Folosește grupuri de acces fără grupuri de resurse.
* Utilizatorii sunt definiți în IBM Cloud, iar atribuirile de proiecte se fac tot acolo.
* Utilizatorii ar trebui să poată șterge joburi.

Pașii pentru implementarea acestei configurări sunt:

2. Administratorul Cloud creează trei instanțe de serviciu: `QR-ml`, `QR finance` și `QR-common`.
3. Administratorul Cloud creează o regulă personalizată care include acțiunea `quantum-computing.job.delete`.
1. Administratorul Cloud creează două grupuri de acces:
   * Grupul de acces `ml` poate accesa `QR-ml` și `QR-common`.
   * Grupul de acces `finance` poate accesa `QR-finance` și `QR-common`.
1. Administratorul Cloud invită utilizatorii cloud la proiectul corespunzător. Mai precis, îi invită și îi atribuie unui grup de acces care include proiectul.
   * Fatima este adăugată în grupul de acces "ml".
   * Ravi este adăugat în grupul de acces "finance".
   * Amyra este adăugată în ambele grupuri de acces, "ml" și "finance".
1. Utilizatorii se pot conecta prin portalul IBM Cloud, pot crea chei API și pot lucra cu instanțele de serviciu ale proiectelor lor.

# Utilizează Qiskit Code Assistant în modul local {#use-qiskit-code-assistant-in-local-mode}

Află cum să instalezi, să configurezi și să folosești oricare dintre modelele Qiskit Code Assistant pe mașina ta locală.

<Admonition type="note" title="Note">
    - Qiskit Code Assistant este în stadiu de previzualizare și poate suferi modificări.
    - Dacă ai feedback sau dorești să contactezi echipa de dezvoltare, folosește [canalul Qiskit Slack Workspace](https://qiskit.enterprise.slack.com/archives/C07LYA6PL83) sau depozitele publice GitHub aferente.
</Admonition>

## Pornire rapidă (recomandat) {#quick-start-recommended}

Cel mai simplu mod de a începe cu Qiskit Code Assistant în modul local este să folosești scripturile de configurare automată pentru extensia VS Code sau JupyterLab. Aceste scripturi vor instala automat [Ollama](https://ollama.com/) pentru a rula LLM-urile, vor descărca modelul recomandat și vor configura extensia pentru tine.

### Configurarea extensiei VS Code {#vs-code-extension-setup}

Rulează următoarea comandă în terminal:

```bash
bash <(curl -fsSL https://raw.githubusercontent.com/Qiskit/qiskit-code-assistant-vscode/main/setup_local.sh)
```

Acest script efectuează următorii pași:
- Instalează Ollama (dacă nu este deja instalat)
- Descarcă și configurează modelul Qiskit Code Assistant recomandat
- Configurează extensia VS Code pentru a funcționa cu implementarea ta locală

### Configurarea extensiei JupyterLab {#jupyterlab-extension-setup}

Rulează următoarea comandă în terminal:

```bash
bash <(curl -fsSL https://raw.githubusercontent.com/Qiskit/qiskit-code-assistant-jupyterlab/main/setup_local.sh)
```

Acest script va:
- Instala Ollama (dacă nu este deja instalat)
- Descărca și configura modelul Qiskit Code Assistant recomandat
- Configura extensia JupyterLab pentru a funcționa cu implementarea ta locală

<summary>Implementare manuală a modelelor Qiskit Code Assistant local prin Ollama</summary>

Există mai multe moduri de a implementa și interacționa cu modelul Qiskit Code Assistant descărcat. Acest ghid demonstrează utilizarea [Ollama](https://ollama.com) astfel: fie cu [aplicația Ollama](#using-the-ollama-application) prin integrarea Hugging Face Hub sau cu un model local, fie cu [pachetul `llama-cpp-python`](#use-llama-cpp-python).

### Utilizarea aplicației Ollama {#using-the-ollama-application}

Aplicația Ollama oferă o soluție simplă pentru a rula LLM-urile local. Este ușor de utilizat, cu un CLI care simplifică întregul proces de configurare, gestionare a modelelor și interacțiune. Este ideală pentru experimentare rapidă și pentru utilizatorii care doresc să gestioneze mai puține detalii tehnice.

#### Instalează Ollama {#install-ollama}

1. Descarcă [aplicația Ollama](https://ollama.com/download)
1. Instalează fișierul descărcat
1. Lansează aplicația Ollama instalată

    <Admonition type="info">Aplicația rulează cu succes atunci când pictograma Ollama apare în bara de meniu a desktopului. Poți verifica, de asemenea, că serviciul rulează accesând `http://localhost:11434/`.</Admonition>

1. Testează Ollama în terminal și începe să rulezi modele. De exemplu:

    ```
    ollama run hf.co/Qiskit/Qwen2.5-Coder-14B-Qiskit
    ```

#### Configurează Ollama folosind integrarea Hugging Face Hub {#set-up-ollama-using-the-hugging-face-hub-integration}

[Integrarea Ollama/Hugging Face Hub](https://huggingface.co/docs/hub/ollama) oferă o modalitate de a interacționa cu modelele găzduite pe Hugging Face Hub fără a fi necesară crearea unui nou modelfile sau descărcarea manuală a fișierelor GGUF sau safetensors. Fișierele implicite `template` și `params` sunt deja incluse pentru model pe Hugging Face Hub.

1. Asigură-te că aplicația Ollama rulează.

1. Accesează pagina modelului dorit și copiază URL-ul. De exemplu, https://huggingface.co/Qiskit/Qwen2.5-Coder-14B-Qiskit-GGUF.

1. Din terminal, rulează comanda:

    ```
    ollama run hf.co/Qiskit/Qwen2.5-Coder-14B-Qiskit
    ```

Poți folosi modelul `hf.co/Qiskit/Qwen2.5-Coder-14B-Qiskit` sau oricare dintre celelalte modele GGUF oficiale recomandate în prezent: `hf.co/Qiskit/mistral-small-3.2-24b-qiskit-GGUF` sau `hf.co/Qiskit/granite-3.3-8b-qiskit-GGUF`.

#### Configurează Ollama cu un model Qiskit Code Assistant GGUF descărcat manual {#set-up-ollama-with-a-manually-downloaded-qiskit-code-assistant-gguf-model}

Dacă ai descărcat manual un model GGUF, cum ar fi https://huggingface.co/Qiskit/Qwen2.5-Coder-14B-Qiskit-GGUF, și dorești să experimentezi cu diferite șabloane și parametri, poți urmări acești pași pentru a-l încărca în aplicația Ollama locală.

1. Creează un `Modelfile` cu următorul conținut și asigură-te că înlocuiești `<PATH-TO-GGUF-FILE>` cu calea reală a modelului tău descărcat.

    ```
    FROM <PATH-TO-GGUF-FILE>
    TEMPLATE """{{ if .System }}
    System:
    {{ .System }}

    {{ end }}{{ if .Prompt }}Question:
    {{ .Prompt }}

    {{ end }}Answer:
    ```python{{ .Response }}
    """

    PARAMETER stop "Question:"
    PARAMETER stop "Answer:"
    PARAMETER stop "System:"
    PARAMETER stop "```"

    PARAMETER temperature 0
    PARAMETER top_k 1
    ```

1. Run the following command to create a custom model instance based on the `Modelfile`.

    ```
    ollama create Qwen2.5-Coder-14B-Qiskit -f ./path-to-model-file
    ```

    <Admonition type="note">This process may take some time for Ollama to read the model file, initialize the model instance, and configure it according to the specifications provided.</Admonition>

#### Run the Qiskit Code Assistant model manually downloaded in Ollama

After the `Qwen2.5-Coder-14B-Qiskit` model has been set up in Ollama, run the following command to launch the model and interact with it in the terminal (in chat mode).

```
ollama run Qwen2.5-Coder-14B-Qiskit
```

Some useful commands:

- `ollama list` - List models on your computer
- `ollama rm Qwen2.5-Coder-14B-Qiskit` - Delete the model
- `ollama show Qwen2.5-Coder-14B-Qiskit` - Show model information
- `ollama stop Qwen2.5-Coder-14B-Qiskit` - Stop a model that is currently running
- `ollama ps` - List which models are currently loaded

<summary>Manually deploy the Qiskit Code Assistant models in local through the `llama-cpp-python` package</summary>

<span id="use-llama-cpp-python"></span>
An alternative to the Ollama application is the `llama-cpp-python` package, which is a Python binding for `llama.cpp`. It gives you more control and flexibility to run the GGUF model locally, and is ideal for users who wish to integrate the local model in their workflows and Python applications.

1. Install [`llama-cpp-python`](https://pypi.org/project/llama-cpp-python/)
2. Interact with the model from within your application using `llama_cpp`. For example:

```python
from llama_cpp import Llama

model_path = <PATH-TO-GGUF-FILE>

model = Llama(
        model_path,
        seed=17,
        n_ctx=10000,
        n_gpu_layers=37, # to offload in gpu, but put 0 if all in cpu
    )

input = 'Generate a quantum circuit with 2 qubits'
raw_pred = model(input)["choices"][0]["text"]
```

You can also add text generation parameters to the model to customize the inference:

```python
generation_kwargs = {
        "max_tokens": 512,
        "echo": False, # Echo the prompt in the output
        "top_k": 1
    }

raw_pred = model(input, **generation_kwargs)["choices"][0]["text"]
```

### Use the `llama.cpp` library

Another alternative is to use `llama.cpp`, an open-source library for performing LLM inference on a CPU with minimal setup.
It provides low-level control over the model execution and is typically run from the command line, pointing to a local GGUF model file.

There are several ways to install `llama.cpp` on your machine:

- Install llama.cpp using [brew, nix, or winget](https://github.com/ggml-org/llama.cpp/blob/master/docs/install.md)
- Run with Docker: See out the [Docker documentation by `llama.cpp` team](https://github.com/ggml-org/llama.cpp/blob/master/docs/docker.md)
- Download pre-built binaries from the [releases page](https://github.com/ggml-org/llama.cpp/releases)
- Build from source by [cloning this repository](https://github.com/ggml-org/llama.cpp/blob/master/docs/build.md)

Once installed, you can use `llama.cpp` to interact with GGUF models in conversation mode as follows:
```bash
# Use a local model file
llama-cli -m my_model.gguf -cnv

# Or download and run a model directly from Hugging Face
llama-cli -hf Qiskit/Qwen2.5-Coder-14B-Qiskit-GGUF -cnv
```

You can also launch an OpenAI-compatible API server for the model in the following way:
```bash
llama-server -hf Qiskit/Qwen2.5-Coder-14B-Qiskit-GGUF
```

#### Advanced parameters
With the `llama-cli` program, you can control the model generation using command-line options. For example, you can provide an initial “system” prompt using the `-p/--prompt` flag. In conversation mode (`-cnv`), this initial prompt acts as the system message. Otherwise, you can simply prepend any desired instruction to your prompt text. You can also adjust sampling parameters - for instance: temperature (`--temp`), top-k (`--top-k`), top-p (`--top-p`), repetition penalty (`--repeat-penalty`), and the seed to use (`--seed`). The following is an example invocation using these options:

```bash
llama-cli -hf Qiskit/Qwen2.5-Coder-14B-Qiskit-GGUF \
  -p "You are a friendly assistant." -cnv \
  --temp 0.7 \
  --top-k 50 \
  --top-p 0.95 \
  --repeat-penalty 1.1 \
  --seed 42
```

Pentru a asigura funcționarea corectă a modelelor noastre Qiskit, recomandăm utilizarea promptului de sistem furnizat în depozitele noastre HF GGUF: promptul de sistem pentru [mistral-small-3.2-24b-qiskit-GGUF](https://huggingface.co/Qiskit/mistral-small-3.2-24b-qiskit-GGUF/blob/main/system), [Qwen2.5-Coder-14B-Qiskit-GGUF](https://huggingface.co/Qiskit/Qwen2.5-Coder-14B-Qiskit-GGUF/blob/main/system), [granite-3.3-8b-qiskit-GGUF](https://huggingface.co/Qiskit/granite-3.3-8b-qiskit-GGUF/blob/main/system) și [granite-3.2-8b-qiskit-GGUF.](https://huggingface.co/Qiskit/granite-3.2-8b-qiskit-GGUF/blob/main/system)

<summary>Conectare manuală a extensiilor la implementarea locală</summary>

Folosește extensia VS Code și extensia JupyterLab pentru Qiskit Code Assistant pentru a transmite prompturi modelului Qiskit Code Assistant implementat local. Odată ce ai [configurat aplicația Ollama cu modelul](#using-the-ollama-application), poți configura extensiile pentru a se conecta la serviciul local.

### Conectare cu extensia Qiskit Code Assistant pentru VS Code {#connect-with-the-qiskit-code-assistant-vs-code-extension}

Cu extensia Qiskit Code Assistant pentru VS Code, poți interacționa cu modelul și efectua completarea codului în timp ce scrii. Aceasta funcționează bine pentru utilizatorii care caută asistență la scrierea codului Qiskit pentru aplicațiile Python.

1. Instalează [extensia Qiskit Code Assistant pentru VS Code](/guides/qiskit-code-assistant-vscode).
1. În VS Code, mergi la **User Settings** și setează **Qiskit Code Assistant: Url** la URL-ul implementării tale locale Ollama (de exemplu, `http://localhost:11434`).
1. Reîncarcă VS Code mergând la **View > Command Palette...** și selectând **Developer: Reload Window**.

Modelul Qiskit Code Assistant configurat în Ollama ar trebui să apară în bara de stare și este gata de utilizare.

### Conectare cu extensia Qiskit Code Assistant pentru JupyterLab {#connect-with-the-qiskit-code-assistant-jupyterlab-extension}

Cu extensia Qiskit Code Assistant pentru JupyterLab, poți interacționa cu modelul și efectua completarea codului direct în Jupyter Notebook. Utilizatorii care lucrează predominant cu Jupyter Notebooks pot beneficia de această extensie pentru a-și îmbunătăți experiența de scriere a codului Qiskit.

1. Instalează [extensia Qiskit Code Assistant pentru JupyterLab](/guides/qiskit-code-assistant-jupyterlab).
1. În JupyterLab, mergi la **Settings Editor** și setează **Qiskit Code Assistant Service API** la URL-ul implementării tale locale Ollama (de exemplu, `http://localhost:11434`).

Modelul Qiskit Code Assistant configurat în Ollama ar trebui să apară în bara de stare și este gata de utilizare.

</details>

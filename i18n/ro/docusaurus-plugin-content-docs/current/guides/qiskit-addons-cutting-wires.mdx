Acest ghid demonstrează un exemplu funcțional de tăieturi de fire cu pachetul `qiskit-addon-cutting`. Acoperă reconstrucția valorilor de așteptare ale unui Circuit cu șapte Qubiți folosind tăieturi de fire.

O tăietură de fir este reprezentată în acest pachet ca o instrucțiune de două Qubiți [`Move`](https://docs.quantum.ibm.com/api/qiskit-addon-cutting/instructions-move), definită ca o resetare a celui de-al doilea Qubit pe care acționează instrucțiunea, urmată de un schimb al ambilor Qubiți. Această operație este echivalentă cu transferul stării primului Qubit către al doilea Qubit, renunțând simultan la starea de intrare a celui de-al doilea Qubit.

Pachetul este proiectat să fie consistent cu modul în care trebuie să tratezi tăieturile de fire atunci când acționezi pe Qubiți fizici. De exemplu, o tăietură de fir poate prelua starea Qubitului fizic $n$ și o poate continua ca Qubit fizic $m$ după tăietură. Te poți gândi la „tăierea instrucțiunilor" ca la un cadru unificat pentru a considera atât tăieturile de fire, cât și tăieturile de Gate-uri în același formalism (deoarece o tăietură de fir este doar o instrucțiune [`Move`](https://docs.quantum.ibm.com/api/qiskit-addon-cutting/instructions-move) tăiată). Folosirea acestui cadru pentru tăierea firelor permite, de asemenea, reutilizarea Qubiților, lucru explicat în secțiunea despre [tăierea firelor manual](#cut-wires-using-the-low-level-move-instruction).

Instrucțiunea de un singur Qubit [`CutWire`](https://docs.quantum.ibm.com/api/qiskit-addon-cutting/instructions-cut-wire) acționează ca o interfață mai abstractă și mai simplă pentru lucrul cu tăieturile de fire. Îți permite să indici unde în Circuit trebuie tăiat un fir la un nivel înalt și să lași add-on-ul de tăiere a circuitelor să insereze instrucțiunile [`Move`](https://docs.quantum.ibm.com/api/qiskit-addon-cutting/instructions-move) corespunzătoare pentru tine.

Exemplul următor demonstrează reconstrucția valorii de așteptare după tăierea firelor. Vei crea un Circuit cu mai multe Gate-uri non-locale și vei defini observabile de estimat.


```python
import numpy as np
from qiskit import QuantumCircuit
from qiskit.transpiler import generate_preset_pass_manager
from qiskit.quantum_info import SparsePauliOp
from qiskit_ibm_runtime.fake_provider import FakeManilaV2
from qiskit_ibm_runtime import SamplerV2, Batch
from qiskit_aer.primitives import EstimatorV2

from qiskit_addon_cutting.instructions import Move, CutWire
from qiskit_addon_cutting import (
    partition_problem,
    generate_cutting_experiments,
    cut_wires,
    expand_observables,
    reconstruct_expectation_values,
)

qc_0 = QuantumCircuit(7)
for i in range(7):
    qc_0.rx(np.pi / 4, i)
qc_0.cx(0, 3)
qc_0.cx(1, 3)
qc_0.cx(2, 3)
qc_0.cx(3, 4)
qc_0.cx(3, 5)
qc_0.cx(3, 6)
qc_0.cx(0, 3)
qc_0.cx(1, 3)
qc_0.cx(2, 3)

# Define observable
observable = SparsePauliOp(["ZIIIIII", "IIIZIII", "IIIIIIZ"])

# Draw circuit
qc_0.draw("mpl")
```

## Taie firele folosind instrucțiunea de nivel înalt `CutWire` {#cut-wires-using-the-high-level-cutwire-instruction}

Urmează să faci tăieturi de fire folosind instrucțiunea de un singur Qubit [`CutWire`](https://docs.quantum.ibm.com/api/qiskit-addon-cutting/instructions-cut-wire) pe Qubit-ul $q_3$. Odată ce subexperimentele sunt pregătite pentru a fi executate, folosește funcția [`cut_wires()`](https://docs.quantum.ibm.com/api/qiskit-addon-cutting/qiskit-addon-cutting#cut_wires) pentru a transforma instrucțiunile `CutWire` în instrucțiuni [`Move`](https://docs.quantum.ibm.com/api/qiskit-addon-cutting/instructions-move) pe Qubiți nou alocați.

```python
qc_1 = QuantumCircuit(7)
for i in range(7):
    qc_1.rx(np.pi / 4, i)
qc_1.cx(0, 3)
qc_1.cx(1, 3)
qc_1.cx(2, 3)
qc_1.append(CutWire(), [3])
qc_1.cx(3, 4)
qc_1.cx(3, 5)
qc_1.cx(3, 6)
qc_1.append(CutWire(), [3])
qc_1.cx(0, 3)
qc_1.cx(1, 3)
qc_1.cx(2, 3)

qc_1.draw("mpl")
```

![Output of the previous code cell](/docs/images/guides/qiskit-addons-cutting-wires/extracted-outputs/9bac1915-316b-49d0-a1a1-145047679530-0.svg)

<Admonition type="info" title="Notă despre extinderea observabilelor">
    Când un Circuit este extins prin una sau mai multe tăieturi de fire, observabilul trebuie actualizat pentru a ține cont de Qubiții suplimentari introduși. Pachetul `qiskit-addon-cutting` are o funcție de conveniență [`expand_observables()`](https://docs.quantum.ibm.com/api/qiskit-addon-cutting/qiskit-addon-cutting#expand_observables), care primește ca argumente obiecte `PauliList` și Circuitele originale și extinse, și returnează un nou `PauliList`.

    Acest `PauliList` returnat nu va conține nicio informație despre coeficienții observabilului original, dar aceștia pot fi ignorați până la reconstrucția valorii finale de așteptare.
</Admonition>

```python
# Transform CutWire instructions to Move instructions
qc_2 = cut_wires(qc_1)

# Expand the observable to match the new circuit size
expanded_observable = expand_observables(observable.paulis, qc_0, qc_2)
print(f"Expanded Observable: {expanded_observable}")
qc_2.draw("mpl")
```

```text

```

### Partiționează Circuit-ul și observabilul {#partition-the-circuit-and-observable}

Acum problema poate fi separată în partiții. Acest lucru se realizează folosind funcția [`partition_problem()`](https://docs.quantum.ibm.com/api/qiskit-addon-cutting/qiskit-addon-cutting#partition_problem) cu un set opțional de etichete de partiție pentru a specifica cum să se separe Circuit-ul. Qubiții care au o etichetă de partiție comună sunt grupați împreună, iar orice Gate-uri non-locale care acoperă mai mult de o partiție sunt tăiate.

Dacă nu sunt furnizate etichete de partiție, partiționarea va fi determinată automat pe baza conectivității Circuit-ului. Citește secțiunea următoare despre [tăierea firelor manual](#cut-wires-using-the-low-level-move-instruction) pentru mai multe informații despre includerea etichetelor de partiție.

```python
partitioned_problem = partition_problem(
    circuit=qc_2,
    observables=expanded_observable,
)
subcircuits = partitioned_problem.subcircuits
subobservables = partitioned_problem.subobservables
bases = partitioned_problem.bases

print(f"Subobservables to measure: \n{subobservables}\n")
print(f"Sampling overhead: {np.prod([basis.overhead for basis in bases])}")
subcircuits[0].draw("mpl")
```

```text

{0: PauliList(['IIIII', 'ZIIII', 'IIIIZ']), 1: PauliList(['ZIII', 'IIII', 'IIII'])}

Sampling overhead: 256.0
```

### Generează subexperimente de executat și post-procesează rezultatele {#generate-subexperiments-to-execute-and-post-process-results}

Pentru a estima valoarea de așteptare a Circuit-ului complet, mai multe subexperimente sunt generate din distribuția cvasi-probabilistică comună a Gate-urilor descompuse și apoi executate pe unul (sau mai multe) QPU-uri. Metoda [`generate_cutting_experiments`](https://docs.quantum.ibm.com/api/qiskit-addon-cutting/qiskit-addon-cutting#generate_cutting_experiments) face acest lucru primind ca argumente dicționarele `subcircuits` și `subobservables` create mai sus, precum și numărul de eșantioane de extras din distribuție.

<Admonition type="note" title="Notă despre numărul de eșantioane">
    Argumentul `num_samples` specifică câte eșantioane să fie extrase din distribuția cvasi-probabilistică și determină acuratețea coeficienților folosiți pentru reconstrucție. Transmiterea valorii infinit (`np.inf`) asigură că toți coeficienții sunt calculați exact. Citește documentația API despre [generarea ponderilor](https://docs.quantum.ibm.com/api/qiskit-addon-cutting/qpd#generate_qpd_weights) și [generarea experimentelor de tăiere](https://docs.quantum.ibm.com/api/qiskit-addon-cutting/qiskit-addon-cutting#generate_cutting_experiments) pentru mai multe informații.
</Admonition>

```python
# Generate subexperiments
subexperiments, coefficients = generate_cutting_experiments(
    circuits=subcircuits, observables=subobservables, num_samples=np.inf
)

# Set a backend to use and transpile the subexperiments
backend = FakeManilaV2()
pass_manager = generate_preset_pass_manager(
    optimization_level=1, backend=backend
)
isa_subexperiments = {
    label: pass_manager.run(partition_subexpts)
    for label, partition_subexpts in subexperiments.items()
}

# Submit each partition's subexperiments to the Qiskit Runtime Sampler
# primitive, in a single batch so that the jobs will run back-to-back.
with Batch(backend=backend) as batch:
    sampler = SamplerV2(mode=batch)
    jobs = {
        label: sampler.run(subsystem_subexpts, shots=2**12)
        for label, subsystem_subexpts in isa_subexperiments.items()
    }
    # Retrieve results
    results = {label: job.result() for label, job in jobs.items()}
```

În final, valoarea de așteptare a Circuit-ului complet poate fi reconstruită folosind metoda [`reconstruct_expectation_values()`](https://docs.quantum.ibm.com/api/qiskit-addon-cutting/qiskit-addon-cutting#reconstruct_expectation_values).

Blocul de cod de mai jos reconstruiește rezultatele și le compară cu valoarea exactă de așteptare.

```python
reconstructed_expval_terms = reconstruct_expectation_values(
    results,
    coefficients,
    subobservables,
)
# Apply the coefficients of the original observable
reconstructed_expval = np.dot(reconstructed_expval_terms, observable.coeffs)

# Compute the exact expectation value using the `qiskit_aer` package.
estimator = EstimatorV2()
exact_expval = estimator.run([(qc_0, observable)]).result()[0].data.evs
print(
    f"Reconstructed expectation value: {np.real(np.round(reconstructed_expval, 8))}"
)
print(f"Exact expectation value: {np.round(exact_expval, 8)}")
print(
    f"Error in estimation: {np.real(np.round(reconstructed_expval-exact_expval, 8))}"
)
print(
    f"Relative error in estimation: {np.real(np.round((reconstructed_expval-exact_expval) / exact_expval, 8))}"
)
```

```text
Reconstructed expectation value: 1.45965266
Exact expectation value: 1.59099026
Error in estimation: -0.1313376
Relative error in estimation: -0.08255085
```

<Admonition type="caution" title="Notă despre coeficienții observabilului">
    Pentru a reconstrui corect valoarea de așteptare, coeficienții observabilului original (care sunt diferiți de rezultatul lui `generate_cutting_experiments()`) trebuie aplicați rezultatului reconstrucției, deoarece această informație s-a pierdut la generarea experimentelor de tăiere sau la extinderea observabilului.

    De obicei, acești coeficienți pot fi aplicați prin `numpy.dot()` după cum s-a arătat anterior.
</Admonition>
## Taie firele folosind instrucțiunea de nivel scăzut `Move` {#cut-wires-using-the-low-level-move-instruction}

O limitare a utilizării instrucțiunii de nivel înalt `CutWire` este că nu permite reutilizarea Qubiților. Dacă acest lucru este dorit pentru un experiment de tăiere, poți plasa manual instrucțiuni [`Move`](https://docs.quantum.ibm.com/api/qiskit-addon-cutting/instructions-move). Cu toate acestea, deoarece instrucțiunea `Move` renunță la starea Qubit-ului destinație, este important ca acest Qubit să nu împărtășească nicio întrețesere cu restul sistemului. Altfel, operația de resetare va cauza colapsul parțial al stării Circuit-ului după tăietura de fir.

Blocul de cod de mai jos efectuează o tăietură de fir pe Qubit-ul $q_3$ pentru același Circuit de exemplu ca cel prezentat anterior. Diferența aici este că poți reutiliza un Qubit inversând operația `Move` acolo unde s-a efectuat a doua tăietură de fir (cu toate acestea, acest lucru nu este întotdeauna posibil și depinde de Circuit-ul tăiat).

```python
qc_1 = QuantumCircuit(8)
for i in [*range(4), *range(5, 8)]:
    qc_1.rx(np.pi / 4, i)
qc_1.cx(0, 3)
qc_1.cx(1, 3)
qc_1.cx(2, 3)
qc_1.append(Move(), [3, 4])
qc_1.cx(4, 5)
qc_1.cx(4, 6)
qc_1.cx(4, 7)
qc_1.append(Move(), [4, 3])
qc_1.cx(0, 3)
qc_1.cx(1, 3)
qc_1.cx(2, 3)

# Expand observable
observable_expanded = SparsePauliOp(["ZIIIIIII", "IIIIZIII", "IIIIIIIZ"])
qc_1.draw("mpl")
```

Circuit-ul de mai sus poate fi acum partiționat și experimentele de tăiere pot fi generate. Pentru a specifica explicit cum trebuie partiționat Circuit-ul, poți adăuga etichete de partiție la funcția `partition_problem()`. Qubiții care au o etichetă de partiție comună sunt grupați împreună, iar orice Gate-uri non-locale care acoperă mai mult de o partiție sunt tăiate. Cheile dicționarului returnat de `partition_problem()` vor corespunde celor specificate în șirul de etichete.

```python
partitioned_problem = partition_problem(
    circuit=qc_1,
    partition_labels="AAAABBBB",
    observables=observable_expanded.paulis,
)
subcircuits = partitioned_problem.subcircuits
subobservables = partitioned_problem.subobservables
bases = partitioned_problem.bases

print(f"Subobservables to measure: \n{subobservables}\n")
print(f"Sampling overhead: {np.prod([basis.overhead for basis in bases])}")
subcircuits["A"].draw("mpl")
```

## Starea implicită

O *stare de referință* reprezintă punctul de start inițial, fix, pentru problema noastră. Pentru a pregăti o stare de referință, trebuie să aplicăm unitarul nepametrizat adecvat $U_R$ la începutul circuitului nostru cuantic, astfel încât $|\rho\rangle = U_R |0\rangle$. Dacă ai o estimare informată sau un punct de date dintr-o soluție optimă existentă, algoritmul variațional va converge probabil mai rapid dacă îl folosești ca punct de plecare.

Cea mai simplă stare de referință posibilă este starea implicită, în care folosim starea de start a unui circuit cuantic cu $n$ qubiți: $|0\rangle^{\otimes n}$. Pentru starea implicită, operatorul unitar este $U_R \equiv I$. Datorită simplității sale, starea implicită este o stare de referință validă utilizată în multe scenarii.
## Starea de referință clasică

Să presupunem că ai un sistem cu trei qubiți și vrei să pornești din starea $|001\rangle$ în loc de starea implicită $|000\rangle$. Acesta este un exemplu de stare de referință pur clasică, iar pentru a o construi trebuie pur și simplu să aplici o [poartă X](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.library.XGate) qubitului $0$ (urmând ordinea qubiților din Qiskit), deoarece $|001\rangle = X_0 |000\rangle$.

În acest caz, operatorul nostru unitar este $U_R \equiv X_0$, ceea ce conduce la starea de referință $|\rho\rangle \equiv |001\rangle$.

## Starea de referință cuantică

Să presupunem că vrei să pornești dintr-o stare mai complexă, care implică superpoziție și/sau entanglement, cum ar fi $\frac{1}{\sqrt{2}}(|100\rangle+|111\rangle)$.

Pentru a obține această stare din $|000\rangle$, o abordare este să folosești o [poartă Hadamard](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.library.HGate) pe qubit $0$ ($H_0$), o poartă [CNOT (CX)](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.library.CXGate) cu qubit $0$ ca qubit de control și qubit $1$ ca qubit țintă ($CNOT_{01}$), și în final o poartă $X$ aplicată qubitului $2$ ($X_2$).

În acest scenariu, operatorul nostru unitar este $U_{R} \equiv X_2CNOT_{01}H_0|000\rangle$, iar starea de referință este $|\rho\rangle \equiv \frac{1}{\sqrt{2}}(|100\rangle+|111\rangle)$.

```python
qc = QuantumCircuit(3)
qc.h(0)
qc.cx(0, 1)
qc.x(2)

qc.draw("mpl")
```

## Construirea stărilor de referință cu circuite șablon

Putem folosi, de asemenea, diverse circuite șablon, cum ar fi [`TwoLocal`](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.library.TwoLocal), care permite exprimarea cu ușurință a mai multor parametri reglabili și entanglement-uri. Vom acoperi aceste circuite șablon în mai mult detaliu în lecția următoare, dar le putem folosi pentru stările noastre de referință *dacă* legăm parametrii:

```python
from qiskit.circuit.library import TwoLocal
from math import pi

reference_circuit = TwoLocal(2, "rx", "cz", entanglement="linear", reps=1)
theta_list = [pi / 2, pi / 3, pi / 3, pi / 2]

reference_circuit = reference_circuit.assign_parameters(theta_list)

reference_circuit.decompose().draw("mpl")
```

## Stări de referință specifice aplicației
### Învățare automată cuantică

În contextul unui clasificator cuantic variațional (VQC), datele de antrenament sunt codificate într-o stare cuantică cu un circuit parametrizat cunoscut sub numele de *hartă de caracteristici*, unde fiecare valoare a parametrului reprezintă un punct de date din setul de antrenament. `zz_feature_map` este un tip de circuit parametrizat care poate fi utilizat pentru a transmite punctele noastre de date ($x$) acestei hărți de caracteristici.

```python
from qiskit.circuit.library import zz_feature_map

data = [0.1, 0.2]

zz_feature_map_reference = zz_feature_map(feature_dimension=2, reps=2)
zz_feature_map_reference = zz_feature_map_reference.assign_parameters(data)
zz_feature_map_reference.decompose().draw("mpl")
```

## VQE personalizat {#custom-vqe}

Mai întâi vom explora cum să construiești o instanță VQE manual pentru a găsi cea mai mică valoare proprie a lui $\hat{O}_1$. Aceasta va incorporate o varietate de tehnici pe care le-am acoperit de-a lungul acestui curs.

```python
def cost_func_vqe(params, ansatz, hamiltonian, estimator):
    """Return estimate of energy from estimator

    Parameters:
        params (ndarray): Array of ansatz parameters
        ansatz (QuantumCircuit): Parameterized ansatz circuit
        hamiltonian (SparsePauliOp): Operator representation of Hamiltonian
        estimator (Estimator): Estimator primitive instance

    Returns:
        float: Energy estimate
    """
    pub = (ansatz, hamiltonian, params)
    cost = estimator.run([pub]).result()[0].data.evs

    return cost
```

## Exemplu VQD {#vqd-example}
Aici implementăm explicit framework-ul Qiskit patterns.
### Pasul 1: Maparea intrărilor clasice la o problemă cuantică {#step-1-map-classical-inputs-to-a-quantum-problem}

Acum, în loc să căutăm doar cea mai mică valoare proprie a observabililor noștri, vom căuta toate $4$, (unde $k=4$).

Ține minte că funcțiile de cost ale VQD sunt:

$$
C_{l}(\vec{\theta}) :=
\langle \psi(\vec{\theta}) | \hat{H} | \psi(\vec{\theta})\rangle +
\sum_{j=0}^{l-1}\beta_j |\langle \psi(\vec{\theta})| \psi(\vec{\theta^j})\rangle  |^2
\quad \forall l\in\{1,\cdots,k\}=\{1,\cdots,4\}
$$

Acest lucru este deosebit de important deoarece un vector $\vec{\beta}=(\beta_0,\cdots,\beta_{k-1})$ (în acest caz $(\beta_0, \beta_1, \beta_2, \beta_3)$) trebuie transmis ca argument atunci când definim obiectul `VQD`.

De asemenea, în implementarea Qiskit a VQD, în loc să considerăm observabilii efectivi descriși în notebook-ul anterior, fidelitățile $|\langle \psi(\vec{\theta})| \psi(\vec{\theta^j})\rangle  |^2$ sunt calculate direct prin algoritmul `ComputeUncompute`, care folosește un Sampler primitiv pentru a eșantiona probabilitatea de a obține $|0\rangle$ pentru Circuit-ul
$U_A^\dagger(\vec{\theta})U_A(\vec{\theta^j})$. Aceasta funcționează tocmai pentru că această probabilitate este

$$
\begin{aligned}

p_0

& = |\langle 0|U_A^\dagger(\vec{\theta})U_A(\vec{\theta^j})|0\rangle|^2 \\[1mm]

& = |\big(\langle 0|U_A^\dagger(\vec{\theta})\big)\big(U_A(\vec{\theta^j})|0\rangle\big)|^2 \\[1mm]

& = |\langle \psi(\vec{\theta}) |\psi(\vec{\theta^j}) \rangle|^2 \\[1mm]

\end{aligned}
$$

Vom folosi următoarea funcție pentru a calcula penalizarea de suprapunere. Reține că aceasta face încă parte din maparea problemei pe circuite cuantice. Totuși, după cum am discutat în lecția anterioară, această funcție calculează suprapunerea dintre circuitul variațional curent și circuitul optimizat dintr-o stare anterioară cu energie/cost mai mic. Noul circuit generat trebuie și el transpilat pentru a rula pe hardware real. Am văzut această funcție anterior, folosită pe un simulator. Aici trebuie să luăm deja în calcul transpilarea și optimizările aferente pentru momentul în care folosim un Backend real, de aceea apar liniile în jurul `if realbackend == 1`. Acest lucru amestecă un pic pasul 2, dar pasul 2 va fi menționat explicit mai târziu.

```python
import numpy as np
from qiskit.transpiler.preset_passmanagers import generate_preset_pass_manager

def calculate_overlaps(
    ansatz, prev_circuits, parameters, sampler, realbackend, backend
):
    def create_fidelity_circuit(circuit_1, circuit_2):
        if len(circuit_1.clbits) > 0:
            circuit_1.remove_final_measurements()
        if len(circuit_2.clbits) > 0:
            circuit_2.remove_final_measurements()

        circuit = circuit_1.compose(circuit_2.inverse())
        circuit.measure_all()
        return circuit

    overlaps = []

    for prev_circuit in prev_circuits:
        fidelity_circuit = create_fidelity_circuit(ansatz, prev_circuit)
        if realbackend == 1:
            pm = generate_preset_pass_manager(backend=backend, optimization_level=3)
            fidelity_circuit = pm.run(fidelity_circuit)
        sampler_job = sampler.run([(fidelity_circuit, parameters)])
        meas_data = sampler_job.result()[0].data.meas

        counts_0 = meas_data.get_int_counts().get(0, 0)
        shots = meas_data.num_shots
        overlap = counts_0 / shots
        overlaps.append(overlap)

    return np.array(overlaps)
```

Acum adăugăm funcția de cost a VQD. Față de lecția anterioară, avem acum două argumente suplimentare (`realbackend` și `backend`) care ne ajută cu transpilarea când folosim Backend-uri reale.

```python
def cost_func_vqd(
    parameters,
    ansatz,
    prev_states,
    step,
    betas,
    estimator,
    sampler,
    hamiltonian,
    realbackend,
    backend,
):
    estimator_job = estimator.run([(ansatz, hamiltonian, [parameters])])

    total_cost = 0

    if step > 1:
        overlaps = calculate_overlaps(
            ansatz, prev_states, parameters, sampler, realbackend, backend
        )
        total_cost = np.sum(
            [np.real(betas[state] * overlap) for state, overlap in enumerate(overlaps)]
        )

    estimator_result = estimator_job.result()[0]

    value = estimator_result.data.evs[0] + total_cost

    return value
```

Dacă nu reușim să obținem o energie exactă a stării fundamentale (sau dacă avem o eroare de câteva procente), atunci nu reușim nici să obținem un vector exact al stării fundamentale! Astfel, atunci când impunem ca starea excitată să fie ortogonală față de prima stare găsită, nu impunem ortogonalitatea față de adevărata stare fundamentală, ci față de o aproximare a acesteia (uneori o aproximare slabă). Prin urmare, starea excitată nu a fost forțată să fie ortogonală față de adevărata stare fundamentală, iar estimările noastre de energie pentru stările excitate au fost de fapt foarte apropiate de energia stării fundamentale.

Aceasta va fi întotdeauna o preocupare în VQD. Dar, în principiu, poate fi corectată prin creșterea numărului maxim de iterații pentru optimizatorul clasic, impunând o toleranță mai mică pentru optimizatorul clasic și, eventual, încercând un alt ansatz dacă ratăm în mod repetat adevărata stare fundamentală. Așa cum am văzut, poate fi necesară și modificarea penalizărilor de suprapunere (betas). Dar aceasta este cu adevărat o problemă separată. Nicio penalizare pentru suprapunere nu te va ține departe de adevărata stare fundamentală dacă nu ai găsit o estimare bună a acesteia pentru circuitul de suprapunere.
## Optimizare: Max-Cut {#optimization-max-cut}

Problema tăieturii maxime (MaxCut) este o problemă de optimizare combinatorie care implică împărțirea vârfurilor unui graf în două mulțimi disjuncte astfel încât numărul de muchii dintre cele două mulțimi să fie maximizat. Mai formal, dat un graf neorientat $G=(V,E)$, unde $V$ este mulțimea vârfurilor și $E$ este mulțimea muchiilor, problema MaxCut cere partiționarea vârfurilor în două submulțimi disjuncte, $S$ și $T$, astfel încât numărul de muchii cu un capăt în $S$ și celălalt în $T$ să fie maximizat.

Putem aplica MaxCut pentru a rezolva diverse probleme, cum ar fi clusterizarea, proiectarea rețelelor și tranzițiile de fază. Vom începe prin crearea unui graf al problemei:

```python
import rustworkx as rx
from rustworkx.visualization import mpl_draw

n = 4
G = rx.PyGraph()
G.add_nodes_from(range(n))
# The edge syntax is (start, end, weight)
edges = [(0, 1, 1.0), (0, 2, 1.0), (0, 3, 1.0), (1, 2, 1.0), (2, 3, 1.0)]
G.add_edges_from(edges)

mpl_draw(
    G, pos=rx.shell_layout(G), with_labels=True, edge_labels=str, node_color="#1192E8"
)
```

![Output of the previous code cell](/learning/images/courses/variational-algorithm-design/examples-and-applications/extracted-outputs/3eb71b79-a988-4807-aa88-7fb25a78a236-0.avif)

Această problemă poate fi exprimată ca o problemă de optimizare binară. Pentru fiecare nod $0 \leq i < n$, unde $n$ este numărul de noduri ale grafului (în acest caz $n=4$), vom lua în considerare variabila binară $x_i$. Această variabilă va avea valoarea $1$ dacă nodul $i$ face parte din unul din grupurile pe care le vom eticheta $1$, și $0$ dacă se află în celălalt grup, pe care îl vom eticheta $0$. Vom nota de asemenea cu $w_{ij}$ (elementul $(i,j)$ al matricei de adiacență $w$) ponderea muchiei care leagă nodul $i$ de nodul $j$. Deoarece graful este neorientat, $w_{ij}=w_{ji}$. Putem astfel formula problema noastră ca maximizarea următoarei funcții de cost:

$$
\begin{aligned}
C(\vec{x})
& =\sum_{i,j=0}^n w_{ij} x_i(1-x_j)\\[1mm]

& = \sum_{i,j=0}^n w_{ij} x_i - \sum_{i,j=0}^n w_{ij} x_ix_j\\[1mm]

& = \sum_{i,j=0}^n w_{ij} x_i - \sum_{i=0}^n \sum_{j=0}^i 2w_{ij} x_ix_j
\end{aligned}
$$

Pentru a rezolva această problemă cu un calculator cuantic, vom exprima funcția de cost ca valoarea așteptată a unui observabil. Totuși, observabilele pe care Qiskit le acceptă în mod nativ constau din operatori Pauli, care au valorile proprii $1$ și $-1$ în loc de $0$ și $1$. De aceea vom efectua următoarea schimbare de variabilă:

Unde $\vec{x}=(x_0,x_1,\cdots ,x_{n-1})$. Putem folosi matricea de adiacență $w$ pentru a accesa comod ponderile tuturor muchiilor. Aceasta va fi folosită pentru a obține funcția noastră de cost:

$$
z_i = 1-2x_i \rightarrow x_i = \frac{1-z_i}{2}
$$

Aceasta implică că:

$$
\begin{array}{lcl} x_i=0 & \rightarrow & z_i=1 \\ x_i=1 & \rightarrow & z_i=-1.\end{array}
$$

Prin urmare, noua funcție de cost pe care dorim să o maximizăm este:

$$
\begin{aligned}
C(\vec{z})
& = \sum_{i,j=0}^n w_{ij} \bigg(\frac{1-z_i}{2}\bigg)\bigg(1-\frac{1-z_j}{2}\bigg)\\[1mm]

& = \sum_{i,j=0}^n \frac{w_{ij}}{4} - \sum_{i,j=0}^n \frac{w_{ij}}{4} z_iz_j\\[1mm]

& = \sum_{i=0}^n \sum_{j=0}^i \frac{w_{ij}}{2} -  \sum_{i=0}^n \sum_{j=0}^i \frac{w_{ij}}{2} z_iz_j
\end{aligned}
$$

În plus, tendința naturală a unui calculator cuantic este să găsească minime (de obicei cea mai mică energie) în loc de maxime, deci în loc să maximizăm $C(\vec{z})$, vom minimiza:

$$
-C(\vec{z}) =  \sum_{i=0}^n \sum_{j=0}^i \frac{w_{ij}}{2} z_iz_j -  \sum_{i=0}^n \sum_{j=0}^i \frac{w_{ij}}{2}
$$

Acum că avem o funcție de cost de minimizat ale cărei variabile pot lua valorile $-1$ și $1$, putem face următoarea analogie cu operatorul Pauli $Z$:

$$
z_i \equiv Z_i = \overbrace{I}^{n-1}\otimes ... \otimes \overbrace{Z}^{i} \otimes ... \otimes \overbrace{I}^{0}
$$

Cu alte cuvinte, variabila $z_i$ va fi echivalentă cu un Gate $Z$ aplicat pe qubitul $i$. Mai mult:

$$
Z_i|x_{n-1}\cdots x_0\rangle = z_i|x_{n-1}\cdots x_0\rangle \rightarrow \langle x_{n-1}\cdots x_0 |Z_i|x_{n-1}\cdots x_0\rangle = z_i
$$

Atunci observabilul pe care îl vom considera este:

$$
\hat{H} = \sum_{i=0}^n \sum_{j=0}^i \frac{w_{ij}}{2} Z_iZ_j
$$

la care va trebui să adăugăm ulterior termenul independent:

$$
\texttt{offset} = - \sum_{i=0}^n \sum_{j=0}^i \frac{w_{ij}}{2}
$$

Operatorul este o combinație liniară de termeni cu operatori Z pe nodurile conectate printr-o muchie (reamintim că qubitul 0 este cel mai din dreapta): $IIZZ + IZIZ + IZZI + ZIIZ + ZZII$. Odată ce operatorul este construit, ansatz-ul pentru algoritmul QAOA poate fi ușor construit folosind circuitul `QAOAAnsatz` din biblioteca de circuite Qiskit.

```python
    #    cost = estimator.run(ansatz, hamiltonian, parameter_values=params).result().values[0]
    return cost
```

Orice optimizator clasic poate fi folosit pentru a minimiza funcția de cost. Pe un sistem cuantic real, un optimizator proiectat pentru peisaje de funcții de cost non-liniare funcționează de obicei mai bine. Aici folosim [rutina COBYLA](https://docs.scipy.org/doc/scipy/reference/optimize.minimize-cobyla.html#optimize-minimize-cobyla) din SciPy prin intermediul funcției minimize.

Deoarece executăm iterativ multe apeluri către Runtime, folosim o sesiune pentru a executa toate apelurile în cadrul unui singur bloc. Mai mult, pentru QAOA, soluția este codificată în distribuția de ieșire a circuitului ansatz legat cu parametrii optimi din minimizare. Prin urmare, vom avea nevoie de un primitiv Sampler, pe care îl vom instanția cu aceeași sesiune.
Și rulăm rutina noastră de minimizare:

```python
result = minimize(
    cost_func, x0, args=(max_ansatz, max_hamiltonian, estimator), method="COBYLA"
)
```

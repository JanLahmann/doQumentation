## Gestionarea așteptărilor

Multe seturi de date utilizate în aplicațiile QML descrise în literatură sunt „feature engineered", ceea ce înseamnă că un set de date este selectat sau generat special pentru a arăta un caz de utilizare restrâns în care calculul cuantic este util. Dacă ți se pare că aceasta este o trișare, atunci înțelegem greșit sarcina în cauză. __Nu__ este cazul că unele hărți de caracteristici cuantice ne permit să rezolvăm toate sau multe sarcini de clasificare mai eficient sau mai scalabil decât algoritmii clasici de machine learning. Mai degrabă, unele hărți de caracteristici cuantice (nu toate) se comportă diferit față de hărțile de caracteristici clasice. Sarcina constă atunci în a explora Circuit-uri cuantice în contextul structurilor de date complexe. Câteva întrebări specifice la care trebuie să răspundem sunt:
1.	Ce Circuit-uri cuantice sunt cel mai probabil să se comporte în moduri noi, comparativ cu alternativele clasice?
2.	Există probleme din lumea reală care implică date cu proprietăți cel mai bine explorate folosind astfel de Circuit-uri cuantice noi?
3.	Aceste Circuit-uri cuantice scalează pe calculatoarele cuantice de generație actuală?

### Explicație insuficientă

Deseori întâlnești o explicație simplificată a modului în care calculul cuantic poate fi puternic. Aceasta sună cam așa:

La fel cum calculatoarele clasice folosesc biți de informație, calculatoarele cuantice folosesc Qubit-uri. Dat fiind un număr de biți, să zicem 4, un calculator clasic poate lua oricare dintre $2^4 = 16$ stări posibile, în timp ce un calculator cuantic poate exista într-o superpoziție a tuturor celor 16 stări simultan, iar operațiile pot fi efectuate pe întreaga superpoziție. În unele cazuri, aceasta ne permite în mod natural să concepem algoritmi de învățare potențial interesanți bazați pe mapări în spații de dimensiuni superioare.

Aceasta este o afirmație adevărată, dar este inadecvată și puțin înșelătoare, după cum vom explica. Se remarcă, de asemenea, diferențele dintre coeficienții complecși și cei reali, ca în:

Un sistem clasic probabilist în care un sistem poate fi descris ca având anumite probabilități de a fi în stări diferite poate fi descris astfel.
$$
|s\rangle = a|0000\rangle+b|0001\rangle+c|0010\rangle+... a, b, c \in \reals
$$
Într-un astfel de sistem, coeficienții $a$, $b$, $c$ și așa mai departe pot fi semnificativi doar dacă sunt numere reale pozitive. Stările din calculatoarele cuantice sunt descrise de amplitudini de probabilitate care pot fi numere complexe.

$$
|\psi \rangle = A|0000\rangle+B|0001\rangle+C|0010\rangle+... A, B, C \in \mathbb{C}
$$

Afirmațiile de mai sus au fost formulate cu mare atenție astfel încât să fie adevărate (multe afirmații superficial similare sunt incorecte). Dar aceste afirmații corecte nu explică puterea calculului cuantic în machine learning. În primul rând, orice aplicare a calculului cuantic în machine learning va implica măsurători, iar un Qubit nu poate fi măsurat ca aflându-se în mai multe stări simultan. Putem pregăti un Qubit într-o superpoziție de tipul $|\psi\rangle = \frac{1}{\sqrt{2}}\left(|0\rangle+|1\rangle\right)$, dar o măsurătoare va produce fie $|0\rangle$, fie $|1\rangle$. Prin urmare, la modul cel mai elementar, această poveste despre creșterea dimensionalității este incompletă. Mai mult, în contextul kernel-urilor, dimensionalitatea ridicată din calculul cuantic nu poate fi o condiție suficientă pentru putere computațională superioară alternativelor clasice, deoarece kernel-urile Gaussiene sunt infinite dimensionale. Există subtilități în această privință: aplicațiile Gaussiene sunt folosite doar împreună cu „trucul kernel" care ocolește necesitatea de a calcula vreodată un vector mapat de dimensiune infinită. Dar concluzia rămâne:

__Dimensionalitatea ridicată a stărilor cuantice întricate nu este paralelism exponențial și nu este o condiție suficientă pentru putere sporită în machine learning.__

În lecțiile care urmează, prezentăm fluxuri de lucru pentru incorporarea Circuit-urilor cuantice în sarcini de machine learning, și facem acest lucru cu scopul explicit de a facilita explorarea puterii calculului cuantic. Nicio aplicație de tip feature map sau algoritm din acest curs nu este prezentat ca o cale rapidă spre rezultate mai bune în machine learning pentru probleme generale, deoarece nu există un astfel de feature map sau algoritm. În schimb, prezentăm o gamă largă de instrumente cuantice care să fie folosite în explorarea calculului cuantic util.

### Decuantizare

Decuantizarea se referă la înlocuirea unui algoritm cuantic dat cu unul clasic care performează similar cu algoritmul cuantic pentru un anumit set de sarcini, incluzând de obicei scalarea. Conform unor definiții, algoritmul clasic ar trebui să performeze cu cel mult un ordin polinomial mai lent decât algoritmul cuantic.

Mai mulți algoritmi QML care inițial se credea că oferă speedup-uri semnificative față de algoritmii clasici au fost decuantizați în ultimii ani. Acest proces de decuantizare a condus la insight-uri importante privind avantajele potențiale și limitările abordărilor cuantice în machine learning.

Unul dintre cele mai notabile rezultate de decuantizare a venit din [lucrarea lui Ewin Tang despre sistemele de recomandare](https://arxiv.org/abs/1807.04271). Tang a descoperit un algoritm clasic capabil să efectueze sarcini de recomandare la viteze considerate anterior ca fiind realizabile doar de calculatoarele cuantice. Această descoperire a contestat presupunerea că algoritmii cuantici aveau un avantaj exponențial pentru această problemă. Lucrări mai recente ale [Shin et al.](https://journals.aps.org/prresearch/abstract/10.1103/PhysRevResearch.6.023218) s-au concentrat pe identificarea condițiilor de decuantizabilitate a clasei de funcții a unui model variațional de QML.

O abordare comună a decuantizării (deși nu singurul truc) constă în luarea în considerare a overhead-ului de încărcare a datelor. Adică, orice algoritm cuantic aplicat datelor clasice va include un pas în care datele clasice sunt codificate în calculatorul cuantic. Dacă un algoritm cuantic presupune un punct de pornire în care datele cuantice sunt deja disponibile, atunci se ascunde efectiv timpul necesar pentru codificare. Există contexte în care presupunerea disponibilității datelor cuantice poate fi rezonabilă, dar multe aplicații de interes vor porni de la date clasice. Unele cazuri de decuantizare au arătat că atunci când acest timp de codificare este inclus și când încărcarea clasică a datelor poate fi realizată eficient, algoritmul cuantic nu mai depășește omologul său clasic.

Chiar dacă un algoritm nu poate fi decuantizat, asta nu înseamnă că este mai eficient sau mai scalabil decât toți algoritmii clasici. Ca exemplu extrem și artificial: imaginează-ți un algoritm pentru a selecta cei mai mari j elemente dintr-o mulțime de dimensiune k. S-ar putea scrie un algoritm cuantic care folosește algoritmul lui Shor pentru a factoriza fiecare dintre cele k elemente în factori primi, iar apoi să determine elementele cele mai mari folosind factorii primi. Un astfel de algoritm probabil nu poate fi decuantizat, dar este drastic mai puțin eficient decât algoritmii clasici pentru a realiza aceeași selecție a celor mai mari elemente (deși nu și pentru partea inutilă de factorizare).

## Dovada de existență

În 2021, cercetătorii IBM Quantum&reg; Yunchao Liu, Srinivasan Arunachalam și Kristan Temme au publicat un articol în Nature, [A rigorous and robust quantum speed-up in supervised machine learning.](https://www.nature.com/articles/s41567-021-01287-z) În concordanță cu avertismentele de mai sus, o problemă de clasificare a fost aleasă cu grijă pentru această lucrare, astfel încât să fie (1) demonstrat că este dificilă clasic și (2) potrivită pentru algoritmii cuantici care să ofere un speedup.

Articolul abordează clasificarea datelor bazată pe logaritmi discreți. Citând articolul, „Pentru un număr prim mare $p$ și un generator $g$ al lui $\mathbb{Z}^*_p = {1, 2, . . . , p − 1}$, este o conjectură larg acceptată că niciun algoritm clasic nu poate calcula $\text{log}_g(x)$ pentru un input $x \in \mathbb{Z}^*_p$, în timp polinomial în $n = \lceil{\text{log}_2(p)}\rceil$, numărul de biți necesari pentru a reprezenta $p$." În contrast, [algoritmul lui Shor](https://epubs.siam.org/doi/10.1137/S0097539795293172) este cunoscut că rezolvă problema logaritmului discret în timp polinomial. Această alegere a problemelor satisface astfel simultan criteriile de mai sus: dificultate clasică (puțin probabil să fie decuantizată) și adecvare cunoscutî pentru algoritmii cuantici.

Prin această alegere judicioasă a problemei de clasificare, autorii au reușit să demonstreze un speedup exponențial folosind metode de kernel cuantic (schițate pe scurt mai jos și discutate în lecțiile ulterioare) care este atât end-to-end, cât și robust. Aici, „end-to-end" se referă la ipotezele despre pornirea cu date clasice; autorii în acest caz includ și timpul pentru codificarea datelor. Iar „robust" se referă la faptul că datele de clasificat sunt separate de o marjă largă cu algoritmul cuantic, astfel că succesul clasificării este robust față de considerente din lumea reală, cum ar fi eroarea de eșantionare finită.

Toate acestea sunt pentru a spune că există probleme în care kernel-urile cuantice pot oferi un speedup exponențial. Dar stadiul actual al științei este că astfel de probleme sunt selectate pe baza observațiilor sau a justificării teoretice că ar trebui să fie pretabile la algoritmii cuantici. Nu este realist să te aștepți la un speedup cuantic pentru sarcini de machine learning pe care calculatoarele clasice le realizează deja foarte bine.

Identificarea unor astfel de cazuri ideale pentru explorarea utilității cuantice este o responsabilitate enormă pentru cei care studiază acest curs. Și nu este o sarcină care poate fi îndeplinită într-un curs de acest gen. Acea explorare este o sarcină pentru întreaga Rețea IBM Quantum, alcătuită din cercetători ca tine. Acest curs va demonstra fluxuri de lucru QML și strategii de codificare, astfel încât să poți începe să explorezi utilitatea cuantică în domeniul tău de expertiză.

Sperăm că această introducere a clarificat câteva lucruri despre quantum machine learning:
1. Algoritmii cuantici pot oferi un speedup exponențial față de algoritmii clasici pentru probleme foarte specifice care sunt dificile clasic și bine adaptate algoritmilor cuantici.
2. Dimensionalitatea ridicată a stărilor întricate în calculul cuantic contează, dar nu este suficient să obții pur și simplu un avantaj față de algoritmii clasici.
3. Găsirea problemelor bine adaptate algoritmilor cuantici este o sarcină extrem de dificilă și una care va reveni în mare măsură celor care studiază acest curs.

```python
# Import some qiskit packages required for setting up our quantum circuits.
from qiskit.circuit import Parameter, ParameterVector, QuantumCircuit
from qiskit.circuit.library import unitary_overlap

# Import StatevectorSampler as our sampler.
from qiskit.primitives import StatevectorSampler

# Step 1: Map classical inputs to a quantum problem:

# Start by getting some appropriate data. The data imported below consist of 128 rows or data points.
# Each row has 14 columns that correspond to data features, and a 15th column with a label (+/-1).
!wget https://raw.githubusercontent.com/qiskit-community/prototype-quantum-kernel-training/main/data/dataset_graph7.csv

# Import some required packages, and write a function to pull some training data out of the csv file you got above.
import pandas as pd
import numpy as np

def get_training_data():
    """Read the training data."""
    df = pd.read_csv("dataset_graph7.csv", sep=",", header=None)
    training_data = df.values[:20, :]
    ind = np.argsort(training_data[:, -1])
    X_train = training_data[ind][:, :-1]

    return X_train

# Prepare training data
X_train = get_training_data()

# Empty kernel matrix
num_samples = np.shape(X_train)[0]

# Prepare feature map for computing overlap between two data points.
# This could be pre-built feature maps like ZZFeatureMap, or a custom quantum circuit, as shown here.
num_features = np.shape(X_train)[1]
num_qubits = int(num_features / 2)
entangler_map = [[0, 2], [3, 4], [2, 5], [1, 4], [2, 3], [4, 6]]
fm = QuantumCircuit(num_qubits)
training_param = Parameter("θ")
feature_params = ParameterVector("x", num_qubits * 2)
fm.ry(training_param, fm.qubits)
for cz in entangler_map:
    fm.cz(cz[0], cz[1])
for i in range(num_qubits):
    fm.rz(-2 * feature_params[2 * i + 1], i)
    fm.rx(-2 * feature_params[2 * i], i)

# Pick two data points, here 14 and 19, and assign the features to the circuits as parameters.
x1 = 14
x2 = 19
unitary1 = fm.assign_parameters(list(X_train[x1]) + [np.pi / 2])
unitary2 = fm.assign_parameters(list(X_train[x2]) + [np.pi / 2])

# Create the overlap circuit
```

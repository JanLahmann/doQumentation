## 2. Backend și Target

Qiskit oferă un API pentru a obține informații, atât statice cât și dinamice, despre un dispozitiv cuantic. Folosim o instanță Backend pentru a interacționa cu un dispozitiv, care include o instanță Target — un model abstract al mașinii ce rezumă caracteristicile relevante, cum ar fi arhitectura setului de instrucțiuni (ISA) și orice proprietăți sau constrângeri asociate acestuia.
Să folosim aceste instanțe Backend pentru a obține o parte din informațiile afișate pe pagina [Compute resources](https://quantum.cloud.ibm.com/computers)
de pe IBM Quantum&reg; Platform. Mai întâi, creăm o instanță Backend pentru un dispozitiv de interes. În cele ce urmează, alegem „ibm_kyoto", „ibm_kawasaki" sau mașina Eagle cu cel mai redus grad de utilizare. Accesul tău la QPU-uri poate fi diferit; actualizează numele Backend-ului corespunzător.

```python
service = QiskitRuntimeService()
# backend = service.backend("ibm_kawasaki") # an Eagle, if you have access to ibm_kawasaki
backend = service.least_busy(
    operational=True, simulator=False, min_num_qubits=127
)  # Eagle
backend.name
```

```text
'ibm_strasbourg'
```

Începem cu câteva informații de bază (statice) despre dispozitiv.

```python
print(
    f"""
{backend.name}, {backend.num_qubits} qubits
processor type = {backend.processor_type}
basis gates = {backend.basis_gates}
"""
)
```

```text
ibm_strasbourg, 127 qubits
processor type = {'family': 'Eagle', 'revision': 3} 
basis gates = ['ecr', 'id', 'rz', 'sx', 'x']
```

## IBM Quantum sien open-source Inholt {#ibm-quantums-open-source-content}

IBM stellt en Masse Lernmaterial för Quantencomputing to'r Verfögung — allens open source ünner CC BY-SA 4.0:

- **[Learning](https://quantum.cloud.ibm.com/learning)** — Struktureerde Kürse vun de Quantengrundlagen bit to de vörankamen Themen
- **[Tutorials](https://quantum.cloud.ibm.com/docs/en/tutorials)** — Över 40 Tutorials över Transpilation, Fehlerminnerung un mehr
- **[Documentation](https://quantum.cloud.ibm.com/docs/en/guides)** — Anleidungen un API-Referenz för Qiskit
- **[Source repo](https://github.com/Qiskit/documentation)** — All Inholt op GitHub

Ehr [Quantum Platform](https://quantum.cloud.ibm.com) is jümmer aktuell un good afstellt — de beste Steed för Lesen, Lehren un Nakieken.

## Wat dit Projekt dortosett {#what-this-project-adds}

IBM sien [Qiskit-Dokumentatschoon](https://github.com/Qiskit/documentation) is open source (CC BY-SA 4.0), aver ehr Webawendung is dat nich. doQumentation sett en open-source Frontend mit live Code-Utföhrung, automaatsche Credential-Injektschoon un Simulator-Modus dor to.

Överal to bruken — vun [GitHub Pages](https://doqumentation.org) över [Docker](https://github.com/JanLahmann/doQumentation/pkgs/container/doqumentation) bit to [RasQberry](https://rasqberry.org/). Kiekt [all Funkschoonen](/features) an.

## Code-Utföhrung {#code-execution}

Klickt bi jeed Codeblock op **Run**. De eerst Klick start en köstenlos Jupyter-Kernel över [Binder](https://mybinder.org) (1–2 Min.). Dorna geiht dat direkt.

- **Simulator-Modus (keen Konto nödig)** — Aktiveert den [Simulator-Modus](/jupyter-settings#simulator-mode), dormit ji all Notebooks mit AerSimulator oder FakeBackends mit Ruschmodell utföhren köönt. Null Inrichtopwand nödig.

- **IBM Quantum Hardware** — Geevt jo API-Token un CRN eenmal in de [Instellungen](/jupyter-settings#ibm-quantum) in — de Togangsdaten warrt bi jeed Utföhrung automaatsch inföögt. [Maak en köstenlos Konto](https://quantum.cloud.ibm.com/registration), wenn ji noch keen hebbt.

<details>
<summary>Verfögbore Utföhrungs-Backends</summary>

Jeed Tutorial hett utföhrbore Codeblöck. Klickt op **Run**, dormit ji se mit een vun dree Backends utföhren köönt:

1. **Binder** (Standard op GitHub Pages) — Köstenlos Remote-Jupyter-Kernel över [mybinder.org](https://mybinder.org)
2. **Lokaal Jupyter (Docker / RasQberry)** — Verbindt juch mit den lokaalen Jupyter-Server mit vorinstalleerd Qiskit
3. **Brukerdefineert Server** — Wiest in de [Instellungen](/jupyter-settings#advanced) op en beleevig Jupyter-Endpunkt

</details>

<details>
<summary>Bereedstellungsoptschoonen</summary>

doQumentation is verfögbor as:

- **[GitHub Pages](https://doqumentation.org)** — Statsche Website mit [Binder](https://mybinder.org) för Remote-Code-Utföhrung
- **[Docker](https://github.com/JanLahmann/doQumentation/pkgs/container/doqumentation)** — Vollständig Stack lokaal, offline-fähig
- **[RasQberry](https://rasqberry.org/)** — Sülvst host op en Raspberry Pi mit lokaalen Jupyter-Kernel

</details>

<details>
<summary>Lokaal utföhren mit Podman / Docker</summary>

Installeert [Podman Desktop](https://podman-desktop.io/) oder [Docker Desktop](https://www.docker.com/products/docker-desktop/) (Mac / Windows / Linux), denn föhrt ut:

```bash
# Vollständig Stack: Website + Jupyter + Qiskit (~3 GB)
podman run -p 8080:80 -p 8888:8888 ghcr.io/janlahmann/doqumentation:jupyter
```

Maakt [http://localhost:8080](http://localhost:8080) op — Code-Utföhrung funkschioneert lokaal, keen Binder-Waardtied.

För en leichtgewichtig Verschoon ahn lokaale Code-Utföhrung (~60 MB):

```bash
# Bloots statsche Website — Code-Utföhrung funkschioneert wiederhen över Binder
podman run -p 8080:80 ghcr.io/janlahmann/doqumentation:latest
```

Bruukt ji Docker? Ersett eenfach `podman` dör `docker` — de Befehlen sünd identsch. De Images sünd Multi-Arch (`linux/amd64` + `linux/arm64`), sodass Apple Silicon Macs nativ funkschooneren.

</details>

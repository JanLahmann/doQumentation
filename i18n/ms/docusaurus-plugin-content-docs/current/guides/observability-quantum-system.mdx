# Peristiwa penjejakan aktiviti untuk perkhidmatan IBM Quantum System {#activity-tracking-events-for-the-ibm-quantum-system-service}

Perkhidmatan IBM Cloud&reg;, seperti perkhidmatan IBM&reg; Quantum System, menjana peristiwa penjejakan aktiviti.

Peristiwa penjejakan aktiviti melaporkan aktiviti yang mengubah keadaan perkhidmatan dalam IBM Cloud. Kamu boleh menggunakan peristiwa ini untuk menyiasat aktiviti luar biasa dan tindakan kritikal serta mematuhi keperluan audit kawal selia.

Kamu boleh menggunakan IBM Cloud Activity Tracker Event Routing, perkhidmatan platform, untuk menghalakan peristiwa pengauditan dalam akaun kamu ke destinasi pilihan kamu dengan mengkonfigurasi sasaran dan laluan yang menentukan ke mana peristiwa penjejakan aktiviti dihantar. Untuk maklumat lanjut, lihat [Mengenai IBM Cloud Activity Tracker Events Routing](https://cloud.ibm.com/docs/atracker?topic=atracker-about).

Kamu boleh menggunakan IBM Cloud Logs untuk memvisualisasi dan memberi amaran tentang peristiwa yang dijana dalam akaun kamu dan dihalakan oleh IBM Cloud Activity Tracker Event Routing ke instans IBM Cloud Logs.

## Melihat peristiwa penjejakan aktiviti untuk perkhidmatan IBM Quantum System {#viewing-activity-tracking-events-for-the-ibm-quantum-system-service}

Kamu boleh menggunakan IBM Cloud Logs untuk memvisualisasi dan memberi amaran tentang peristiwa yang dijana dalam akaun kamu dan dihalakan oleh IBM Cloud Activity Tracker Event Routing ke instans IBM Cloud Logs.

### Melancarkan IBM Cloud Logs dari halaman Observability {#launching-ibm-cloud-logs-from-the-observability-page}

Untuk maklumat tentang melancarkan UI IBM Cloud Logs, lihat [Melancarkan UI](https://cloud.ibm.com/docs/cloud-logs?topic=cloud-logs-instance-launch#instance-launch-cloud-ui) dalam dokumentasi IBM Cloud Logs.

| Tindakan | Dicetuskan oleh |
| -- | -- |
| `quantum-computing.direct-access-backend-configuration.read` | Dapatkan konfigurasi Backend (`GET /v1/backends/{backend-name}/configuration`) |
| `quantum-computing.direct-access-backend-properties.read` | Dapatkan sifat Backend (`GET /v1/backends/{backend-name}/properties`) |
| `quantum-computing.direct-access-backend.list` | Dapatkan senarai Backend (`GET /v1/backends`) |
| `quantum-computing.direct-access-backend.read` | Dapatkan butiran Backend (`GET /v1/backends/{backend-name}`) |
| `quantum-computing.direct-access-job.cancel` | Batalkan kerja (`POST /v1/jobs/{job-id}/cancel`) |
| `quantum-computing.direct-access-job.create` | Jalankan kerja (`POST /v1/jobs`) |
| `quantum-computing.direct-access-job.delete` | Padam kerja (`DELETE /v1/jobs/{job-id}`) |
| `quantum-computing.direct-access-job.list` | Dapatkan kerja (`GET /v1/jobs`) |
| `quantum-computing.direct-access-job.read` | Dapatkan Kerja mengikut ID (`GET /v1/jobs/{job-id}`) |

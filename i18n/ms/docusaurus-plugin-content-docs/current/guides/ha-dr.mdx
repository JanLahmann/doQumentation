# Memahami ketersediaan tinggi dan pemulihan bencana untuk Qiskit Runtime {#understanding-high-availability-and-disaster-recovery-for-qiskit-runtime}

Ketersediaan tinggi (HA) adalah keupayaan sesuatu perkhidmatan untuk kekal beroperasi dan boleh diakses semasa berlakunya kegagalan yang tidak dijangka. Pemulihan bencana adalah proses memulihkan contoh perkhidmatan ke keadaan berfungsi. Pelanggan Qiskit Runtime tidak bertanggungjawab untuk perancangan atau tindakan pemulihan bencana.

Qiskit Runtime adalah perkhidmatan serantau yang memenuhi [Objektif Tahap Perkhidmatan (SLO)](https://cloud.ibm.com/docs/resiliency?topic=resiliency-slo) yang ditetapkan dengan sebarang pelan berbayar. Untuk maklumat lanjut, lihat [Perjanjian Tahap Perkhidmatan (SLA)](https://www.ibm.com/support/customer/csol/terms/?id=i126-9268&lc=en) untuk IBM Cloud&reg;. Untuk maklumat lanjut tentang rantau dan pusat data IBM Cloud yang tersedia untuk Qiskit Runtime, lihat [Ketersediaan perkhidmatan dan infrastruktur mengikut lokasi.](https://cloud.ibm.com/docs/overview?topic=overview-services_region)

## Seni bina ketersediaan tinggi {#high-availability-architecture}

### Ciri-ciri ketersediaan tinggi {#high-availability-features}

Perkhidmatan ini disediakan merentas tiga zon dalam rantau berbilang zon tanpa titik kegagalan tunggal. Permintaan API dihalakan melalui pengimbang beban ke nod contoh HA, setiap satu di zon yang berbeza.

Jika nod contoh HA atau zon gagal, perkhidmatan terus menjalankan permintaan API melalui nod contoh HA yang masih bertahan. Tiada penurunan perkhidmatan yang ketara berlaku semasa kegagalan zon.

Ia tersedia di rantau berikut:

- Washington DC (us-east)
- Frankfurt (eu-de)

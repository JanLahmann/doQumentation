# Soalan lazim (FAQ) {#frequently-asked-questions-faq}

Untuk soalan lazim khusus tentang mod pelaksanaan, lihat panduan [FAQ mod pelaksanaan Qiskit Runtime](/guides/execution-modes-faq).

## Bagaimana saya memetik Qiskit dalam penyelidikan saya? {#how-do-i-cite-qiskit-in-my-research}

Petik Qiskit dengan menggunakan [fail BibTeX](https://github.com/Qiskit/qiskit/blob/main/CITATION.bib) yang disertakan.

## Bagaimana saya memetik komputer IBM Quantum dalam penyelidikan saya? {#how-do-i-cite-an-ibm-quantum-computer-in-my-research}

Untuk kertas penyelidikan, kami menggalakkan penulis mengakui IBM Quantum&reg; dengan menggunakan:

>    Kami mengakui penggunaan perkhidmatan IBM Quantum untuk kerja ini. Pandangan yang dinyatakan adalah milik penulis, dan tidak mencerminkan dasar atau pendirian rasmi IBM atau pasukan IBM Quantum.

Rujukan kertas harus menggunakan petikan berikut:

> IBM Quantum. https://<span>quantum.cloud.ibm.com</span>/, YYYY

Unit pemprosesan kuantum dalam kertas harus dirujuk dengan nama uniknya (contohnya, `ibmq_vigo`) dan secara pilihan menambahkan versi (contohnya, `ibmq_vigo` v1.0.2). Kami juga menggalakkan merujuk kepada pemproses tersebut.
Contohnya:

> Dalam kertas ini kami menggunakan `ibmq_vigo`, yang merupakan salah satu pemproses IBM Quantum Canary.

Contoh memetik program IBM Quantum:

> IBM Quantum (2025). Primitif Estimator (Versi x.y.z) [perisian komputer]. https://<span>quantum.cloud.ibm.com</span>/

## Bagaimana saya memetik IBM Quantum Composer dalam penyelidikan saya? {#how-do-i-cite-ibm-quantum-composer-in-my-research}

Contoh petikan untuk IBM Quantum Composer:

> IBM Quantum Composer. 2025. url: https://<span>quantum.cloud.ibm.com/composer</span>

## Saya tidak nampak sesuatu dalam antara muka pengguna, seperti tugas pentadbiran saya. {#i-dont-see-something-in-the-user-interface}

Cuba pilih akaun yang berbeza daripada penukar akaun di pengepala antara muka [IBM Quantum Platform](https://quantum.cloud.ibm.com).

## Saya hanya menggunakan Open Plan, tetapi saya terpaksa memasukkan kad kredit saya. Adakah saya akan dicaj? {#im-just-using-the-open-plan-but-i-had-to-put-in-my-credit-card}

Anda tidak akan dicaj melainkan anda mencipta [instans](/guides/instances) Pay-As-You-Go dan menggunakannya untuk menghantar beban kerja ke QPU. Lihat [Urus kos](/guides/manage-cost) untuk arahan menetapkan had kos dan untuk mengetahui cara-cara lain mengurus kos.

## Apakah perbezaan antara kunci API dan token API? {#whats-the-difference-between-an-api-key-and-an-api-token}

Tiada perbezaan. Istilah kunci API dan token API digunakan secara bergantian. Anda akan melihat kedua-dua istilah dalam dokumentasi kerana antara muka yang berbeza menggunakan istilah yang berbeza.

<span id="bearer"></span>
## Apakah perbezaan antara token API dan token bearer? {#whats-the-difference-between-an-api-token-and-a-bearer-token}

Token API ialah kelayakan jangka panjang yang digunakan untuk [mengesahkan diri kepada perkhidmatan Qiskit Runtime.](/guides/hello-world#install-and-authenticate) Token bearer ialah kelayakan sementara yang digunakan untuk [mengesahkan diri kepada REST API dalam persekitaran yang tidak dipercayai.](/guides/cloud-setup-untrusted#authenticate-to-the-rest-api-in-an-untrusted-environment) Token bearer tidak boleh digunakan untuk mengesahkan diri kepada perkhidmatan Qiskit Runtime.

## Adakah saya perlu memilih rantau tempat saya tinggal? {#do-i-have-to-choose-the-region-that-i-live-in}

Tidak. Anda boleh menggunakan mana-mana rantau. Rantau menentukan di mana job dijalankan dan di mana data disimpan. Selain itu, anda hanya boleh melihat dan menggunakan instans yang dicipta dalam instans yang anda log masuk. Perlu diingat bahawa hanya rantau us-east yang menyokong Open Plan.

## Bolehkah saya sambung melalui pelayan proksi? {#can-i-connect-through-a-proxy-server}

Ya, tetapi anda mesti menggunakan Qiskit Runtime v0.44.0 atau lebih baharu.

## Apakah dasar pengekalan data? {#what-is-the-data-retention-policy}

IBM menyimpan data job dan penggunaan anda (yang tidak dipadam) selama tiga tahun selagi anda terus menggunakan perkhidmatan Qiskit Runtime.

## Apa yang berlaku kepada pengedaran Python seperti Qiskit Aer, Qiskit Nature, dan Qiskit Experiments? {#what-happened-to-python-distributions}

Sejak keluaran Qiskit SDK v1.0, arahan `pip install qiskit` hanya akan memasang [Qiskit SDK](https://github.com/Qiskit/qiskit) **sahaja**. Ini bermakna beberapa modul yang pernah anda gunakan sebelum ini, seperti `qiskit-aer` atau `qiskit-nature`, kini perlu dipasang secara berasingan.

Sebelum keluaran Qiskit SDK v1.0, Qiskit terdiri daripada banyak pengedaran Python yang berbeza, seperti `qiskit-terra`, teras pengkompil; `qiskit-aer`, simulator berprestasi tinggi; pembekal IBM Quantum&reg; asal; dan beberapa pakej yang kini lapuk yang menyediakan ciri algoritma atau eksperimen tertentu yang bersifat penerokaan.

Pakej perisian yang berasingan ini digabungkan menjadi apa yang dikenali sebagai *metapackage* Qiskit, yang membolehkan pengguna memasang semua komponen ini bersama-sama melalui `pip install qiskit`. Seni bina metapackage ini dihapuskan dengan keluaran Qiskit SDK v1.0, dan kini setiap pengedaran perisian individu perlu dipasang secara berasingan.

Jika anda ingin memasang pakej yang sebelum ini disertakan dalam metapackage Qiskit, lawati [ekosistem Qiskit](https://www.ibm.com/quantum/ecosystem) untuk mencari pelbagai pakej yang sesuai dengan keperluan anda. Anda juga boleh membaca [panduan migrasi v1.0](/guides/metapackage-migration) untuk maklumat lanjut tentang perubahan ini.

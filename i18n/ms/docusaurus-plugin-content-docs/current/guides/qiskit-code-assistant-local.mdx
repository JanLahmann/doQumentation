# Gunakan Qiskit Code Assistant dalam mod tempatan {#use-qiskit-code-assistant-in-local-mode}

Ketahui cara memasang, mengkonfigurasi, dan menggunakan mana-mana model Qiskit Code Assistant pada mesin tempatan anda.

<Admonition type="note" title="Nota">
    - Qiskit Code Assistant berada dalam status keluaran pratonton dan tertakluk kepada perubahan.
    - Jika anda mempunyai maklum balas atau ingin menghubungi pasukan pembangun, gunakan [saluran Ruang Kerja Qiskit Slack](https://qiskit.enterprise.slack.com/archives/C07LYA6PL83) atau repositori GitHub awam yang berkaitan.
</Admonition>

## Permulaan pantas (disyorkan) {#quick-start-recommended}

Cara termudah untuk bermula dengan Qiskit Code Assistant dalam mod tempatan adalah dengan menggunakan skrip persediaan automatik untuk sama ada sambungan VS Code atau JupyterLab. Skrip ini akan memasang [Ollama](https://ollama.com/) secara automatik untuk menjalankan LLM, memuat turun model yang disyorkan, dan mengkonfigurasi sambungan untuk anda.

### Persediaan sambungan VS Code {#vs-code-extension-setup}

Jalankan arahan berikut dalam terminal anda:

```bash
bash <(curl -fsSL https://raw.githubusercontent.com/Qiskit/qiskit-code-assistant-vscode/main/setup_local.sh)
```

Skrip ini melakukan langkah-langkah berikut:
- Memasang Ollama (jika belum dipasang)
- Memuat turun dan mengkonfigurasi model Qiskit Code Assistant yang disyorkan
- Menyediakan sambungan VS Code untuk berfungsi dengan penggunaan tempatan anda

### Persediaan sambungan JupyterLab {#jupyterlab-extension-setup}

Jalankan arahan berikut dalam terminal anda:

```bash
bash <(curl -fsSL https://raw.githubusercontent.com/Qiskit/qiskit-code-assistant-jupyterlab/main/setup_local.sh)
```

Skrip ini akan:
- Memasang Ollama (jika belum dipasang)
- Memuat turun dan mengkonfigurasi model Qiskit Code Assistant yang disyorkan
- Menyediakan sambungan JupyterLab untuk berfungsi dengan penggunaan tempatan anda

### Menggunakan aplikasi Ollama {#using-the-ollama-application}

Aplikasi Ollama menyediakan penyelesaian mudah untuk menjalankan LLM secara tempatan. Ia mudah digunakan, dengan CLI yang menjadikan keseluruhan proses persediaan, pengurusan model, dan interaksi agak mudah. Ia sesuai untuk eksperimen pantas dan bagi pengguna yang mahu mengendalikan lebih sedikit butiran teknikal.

#### Pasang Ollama {#install-ollama}

1. Muat turun [aplikasi Ollama](https://ollama.com/download)
1. Pasang fail yang dimuat turun
1. Lancarkan aplikasi Ollama yang telah dipasang

    <Admonition type="info">Aplikasi berjalan dengan berjaya apabila ikon Ollama muncul dalam bar menu desktop. Anda juga boleh mengesahkan perkhidmatan sedang berjalan dengan pergi ke `http://localhost:11434/`.</Admonition>

1. Cuba Ollama dalam terminal anda dan mula menjalankan model. Contohnya:

    ```
    ollama run hf.co/Qiskit/Qwen2.5-Coder-14B-Qiskit
    ```

#### Sediakan Ollama menggunakan integrasi Hugging Face Hub {#set-up-ollama-using-the-hugging-face-hub-integration}

[Integrasi Ollama/Hugging Face Hub](https://huggingface.co/docs/hub/ollama) menyediakan cara untuk berinteraksi dengan model yang dihoskan di Hugging Face Hub tanpa perlu mencipta modelfile baharu atau memuat turun fail GGUF atau safetensors secara manual. Fail `template` dan `params` lalai sudah disertakan untuk model di Hugging Face Hub.

1. Pastikan aplikasi Ollama sedang berjalan.

1. Pergi ke halaman model yang dikehendaki, dan salin URL. Contohnya, https://huggingface.co/Qiskit/Qwen2.5-Coder-14B-Qiskit-GGUF.

1. Dari terminal anda, jalankan arahan:

    ```
    ollama run hf.co/Qiskit/Qwen2.5-Coder-14B-Qiskit
    ```

Anda boleh menggunakan model `hf.co/Qiskit/Qwen2.5-Coder-14B-Qiskit` atau mana-mana model GGUF rasmi yang disyorkan pada masa ini `hf.co/Qiskit/mistral-small-3.2-24b-qiskit-GGUF` atau `hf.co/Qiskit/granite-3.3-8b-qiskit-GGUF`.

#### Sediakan Ollama dengan model GGUF Qiskit Code Assistant yang dimuat turun secara manual {#set-up-ollama-with-a-manually-downloaded-qiskit-code-assistant-gguf-model}

Jika anda telah memuat turun model GGUF secara manual seperti https://huggingface.co/Qiskit/Qwen2.5-Coder-14B-Qiskit-GGUF dan anda ingin bereksperimen dengan templat dan parameter yang berbeza, anda boleh mengikuti langkah-langkah ini untuk memuatnya ke dalam aplikasi Ollama tempatan anda.

1. Cipta `Modelfile` dengan memasukkan kandungan berikut dan pastikan untuk mengemas kini `<PATH-TO-GGUF-FILE>` kepada laluan sebenar model yang anda muat turun.

    ```
    FROM <PATH-TO-GGUF-FILE>
    TEMPLATE """{{ if .System }}
    System:
    {{ .System }}

    {{ end }}{{ if .Prompt }}Question:
    {{ .Prompt }}

    {{ end }}Answer:
    ```python{{ .Response }}
    """

    PARAMETER stop "Question:"
    PARAMETER stop "Answer:"
    PARAMETER stop "System:"
    PARAMETER stop "```"

    PARAMETER temperature 0
    PARAMETER top_k 1
    ```

1. Run the following command to create a custom model instance based on the `Modelfile`.

    ```
    ollama create Qwen2.5-Coder-14B-Qiskit -f ./path-to-model-file
    ```

    <Admonition type="note">This process may take some time for Ollama to read the model file, initialize the model instance, and configure it according to the specifications provided.</Admonition>

#### Run the Qiskit Code Assistant model manually downloaded in Ollama

After the `Qwen2.5-Coder-14B-Qiskit` model has been set up in Ollama, run the following command to launch the model and interact with it in the terminal (in chat mode).

```
ollama run Qwen2.5-Coder-14B-Qiskit
```

Some useful commands:

- `ollama list` - List models on your computer
- `ollama rm Qwen2.5-Coder-14B-Qiskit` - Delete the model
- `ollama show Qwen2.5-Coder-14B-Qiskit` - Show model information
- `ollama stop Qwen2.5-Coder-14B-Qiskit` - Stop a model that is currently running
- `ollama ps` - List which models are currently loaded

<summary>Manually deploy the Qiskit Code Assistant models in local through the `llama-cpp-python` package</summary>

<span id="use-llama-cpp-python"></span>
An alternative to the Ollama application is the `llama-cpp-python` package, which is a Python binding for `llama.cpp`. It gives you more control and flexibility to run the GGUF model locally, and is ideal for users who wish to integrate the local model in their workflows and Python applications.

1. Install [`llama-cpp-python`](https://pypi.org/project/llama-cpp-python/)
2. Interact with the model from within your application using `llama_cpp`. For example:

```python
from llama_cpp import Llama

model_path = <PATH-TO-GGUF-FILE>

model = Llama(
        model_path,
        seed=17,
        n_ctx=10000,
        n_gpu_layers=37, # to offload in gpu, but put 0 if all in cpu
    )

input = 'Generate a quantum circuit with 2 qubits'
raw_pred = model(input)["choices"][0]["text"]
```

You can also add text generation parameters to the model to customize the inference:

```python
generation_kwargs = {
        "max_tokens": 512,
        "echo": False, # Echo the prompt in the output
        "top_k": 1
    }

raw_pred = model(input, **generation_kwargs)["choices"][0]["text"]
```

### Use the `llama.cpp` library

Another alternative is to use `llama.cpp`, an open-source library for performing LLM inference on a CPU with minimal setup.
It provides low-level control over the model execution and is typically run from the command line, pointing to a local GGUF model file.

There are several ways to install `llama.cpp` on your machine:

- Install llama.cpp using [brew, nix, or winget](https://github.com/ggml-org/llama.cpp/blob/master/docs/install.md)
- Run with Docker: See out the [Docker documentation by `llama.cpp` team](https://github.com/ggml-org/llama.cpp/blob/master/docs/docker.md)
- Download pre-built binaries from the [releases page](https://github.com/ggml-org/llama.cpp/releases)
- Build from source by [cloning this repository](https://github.com/ggml-org/llama.cpp/blob/master/docs/build.md)

Once installed, you can use `llama.cpp` to interact with GGUF models in conversation mode as follows:
```bash
# Use a local model file
llama-cli -m my_model.gguf -cnv

# Or download and run a model directly from Hugging Face
llama-cli -hf Qiskit/Qwen2.5-Coder-14B-Qiskit-GGUF -cnv
```

You can also launch an OpenAI-compatible API server for the model in the following way:
```bash
llama-server -hf Qiskit/Qwen2.5-Coder-14B-Qiskit-GGUF
```

#### Advanced parameters
With the `llama-cli` program, you can control the model generation using command-line options. For example, you can provide an initial “system” prompt using the `-p/--prompt` flag. In conversation mode (`-cnv`), this initial prompt acts as the system message. Otherwise, you can simply prepend any desired instruction to your prompt text. You can also adjust sampling parameters - for instance: temperature (`--temp`), top-k (`--top-k`), top-p (`--top-p`), repetition penalty (`--repeat-penalty`), and the seed to use (`--seed`). The following is an example invocation using these options:

```bash
llama-cli -hf Qiskit/Qwen2.5-Coder-14B-Qiskit-GGUF \
  -p "You are a friendly assistant." -cnv \
  --temp 0.7 \
  --top-k 50 \
  --top-p 0.95 \
  --repeat-penalty 1.1 \
  --seed 42
```

Untuk memastikan fungsi yang betul bagi model Qiskit kami, kami mengesyorkan menggunakan gesaan sistem yang disediakan dalam repositori HF GGUF kami: gesaan sistem untuk [mistral-small-3.2-24b-qiskit-GGUF](https://huggingface.co/Qiskit/mistral-small-3.2-24b-qiskit-GGUF/blob/main/system), [Qwen2.5-Coder-14B-Qiskit-GGUF](https://huggingface.co/Qiskit/Qwen2.5-Coder-14B-Qiskit-GGUF/blob/main/system), [granite-3.3-8b-qiskit-GGUF](https://huggingface.co/Qiskit/granite-3.3-8b-qiskit-GGUF/blob/main/system), dan [granite-3.2-8b-qiskit-GGUF.](https://huggingface.co/Qiskit/granite-3.2-8b-qiskit-GGUF/blob/main/system)

<summary>Sambungkan sambungan ke penggunaan tempatan secara manual</summary>

Gunakan sambungan VS Code dan sambungan JupyterLab untuk Qiskit Code Assistant untuk menggesaan model Qiskit Code Assistant yang digunakan secara tempatan. Setelah anda mempunyai aplikasi Ollama [yang disediakan dengan model](#using-the-ollama-application), anda boleh mengkonfigurasi sambungan untuk berhubung ke perkhidmatan tempatan.

### Sambung dengan sambungan VS Code Qiskit Code Assistant {#connect-with-the-qiskit-code-assistant-vs-code-extension}

Dengan sambungan VS Code Qiskit Code Assistant, anda boleh berinteraksi dengan model dan melakukan penyempurnaan kod semasa menulis kod anda. Ini boleh berfungsi dengan baik untuk pengguna yang mencari bantuan menulis kod Qiskit untuk aplikasi Python mereka.

1. Pasang [sambungan VS Code Qiskit Code Assistant](/guides/qiskit-code-assistant-vscode).
1. Dalam VS Code, pergi ke **User Settings** dan tetapkan **Qiskit Code Assistant: Url** ke URL penggunaan Ollama tempatan anda (contohnya, `http://localhost:11434`).
1. Muat semula VS Code dengan pergi ke **View > Command Palette...** dan memilih **Developer: Reload Window**.

Model Qiskit Code Assistant yang dikonfigurasi dalam Ollama sepatutnya muncul dalam bar status dan sedia digunakan.

### Sambung dengan sambungan JupyterLab Qiskit Code Assistant {#connect-with-the-qiskit-code-assistant-jupyterlab-extension}

Dengan sambungan JupyterLab Qiskit Code Assistant, anda boleh berinteraksi dengan model dan melakukan penyempurnaan kod terus dalam Jupyter Notebook anda. Pengguna yang kebanyakannya bekerja dengan Jupyter Notebooks boleh memanfaatkan sambungan ini untuk meningkatkan lagi pengalaman mereka menulis kod Qiskit.

1. Pasang [sambungan JupyterLab Qiskit Code Assistant](/guides/qiskit-code-assistant-jupyterlab).
1. Dalam JupyterLab, pergi ke **Settings Editor** dan tetapkan **Qiskit Code Assistant Service API** ke URL penggunaan Ollama tempatan anda (contohnya, `http://localhost:11434`).

Model Qiskit Code Assistant yang dikonfigurasi dalam Ollama sepatutnya muncul dalam bar status dan sedia digunakan.

</details>

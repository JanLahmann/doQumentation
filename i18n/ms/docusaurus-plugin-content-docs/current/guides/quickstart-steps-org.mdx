# Konfigurasi IBM Quantum Platform untuk sesebuah organisasi {#configure-ibm-quantum-platform-for-an-organization}

Ikuti langkah-langkah ini untuk menyediakan IBM Quantum Platform.

<span id="iam-org"></span>
## 1. Konfigurasi tetapan IAM {#1-configure-iam-settings}

Pertama, konfigurasi tetapan berikut dalam akaun IAM pentadbir. Untuk menyemak dan mengkonfigurasi tetapan ini, pergi ke [Manage → IAM → Settings](https://cloud.ibm.com/iam/settings).

* **Keterlihatan senarai pengguna** menentukan sama ada pengguna boleh melihat antara satu sama lain, tanpa mengira tugasan projek. Tetapan `enabled` mengehadkan keterlihatan pengguna. Maksudnya, pengguna dalam akaun anda tidak dapat melihat antara satu sama lain, walaupun mereka boleh mengakses sumber yang sama. Pilih nilai yang sesuai untuk persekitaran anda. Lihat [Mengawal keterlihatan pengguna](https://cloud.ibm.com/docs/account?topic=account-iam-user-setting) untuk maklumat lanjut.
* **Penciptaan kunci API** mengawal sama ada pengguna boleh mencipta kunci API. Dalam IBM Quantum Platform, kunci API biasa digunakan. Jika kunci API digunakan, pilih `disabled`. Sebagai alternatif, anda boleh memberi kebenaran khusus kepada setiap pengguna.

<span id="crt-rsc-grp-org"></span>
## 2. (Pilihan) Cipta kumpulan sumber {#2-optional-create-resource-groups}

Langkah ini boleh dilangkau jika anda menggunakan kumpulan akses yang merujuk terus kepada instans perkhidmatan IBM Quantum.

Jika anda memilih untuk menggunakan kumpulan sumber, pergi ke [Manage → Account → Resource groups (dalam Account resources)](https://cloud.ibm.com/account/resource-groups) dan klik **Create**.

<span id="create-instance-org"></span>
## 3. Cipta instans perkhidmatan IBM Quantum {#3-create-ibm-quantum-service-instances}

Jika anda telah mencipta instans perkhidmatan IBM Quantum, langkau langkah ini.

* Jika anda menggunakan kumpulan sumber, pastikan anda mencipta instans perkhidmatan pada kumpulan sumber yang sesuai.
* Nama instans perkhidmatan, seperti `QR-ml`, diperlukan untuk rujukan kumpulan akses.

Ikuti langkah-langkah dalam [Sediakan untuk menggunakan IBM Cloud](/guides/cloud-setup#cloud) untuk arahan.

<span id="create-group-org"></span>
## 4. Cipta kumpulan akses untuk projek {#4-create-access-groups-for-projects}

Pertama, [cipta peranan tersuai](/guides/custom-roles#custom) yang membolehkan pengguna melakukan tindakan untuk bekerja dengan instans perkhidmatan IBM Quantum.
Seterusnya, [cipta kumpulan akses](/guides/access-groups) untuk setiap projek dan berikan kumpulan tersebut set kebenaran minimum yang diperlukan untuk bekerja dengan sumber projek.

<span id="setup-idp-org"></span>
## 5. Sediakan penyedia ID anda dan tugaskan pengguna {#5-set-up-your-id-provider-and-assign-users}

Ikuti langkah-langkah dalam topik yang sesuai, bergantung pada penyedia ID yang anda pilih:

* [Gunakan IBM Cloud sebagai penyedia ID untuk pengguna yang tidak mempunyai akaun IBM Cloud](/guides/manage-appid)

<span id="firewall"></span>
## 6. Konfigurasi tembok api anda {#6-configure-your-firewall}

Untuk mengkonfigurasi tembok api anda bagi membolehkan akses ke titik akhir API IBM Quantum, tambahkan URL berikut ke senarai benarkan anda:

- Perkhidmatan global IBM Cloud:
    ```text
    cloud.ibm.com
    iam.cloud.ibm.com
    api.global-search-tagging.cloud.ibm.com
    globalcatalog.cloud.ibm.com
    resource-controller.cloud.ibm.com
    ```
- Perkhidmatan IBM Quantum Qiskit Runtime - rantau us-east:
    ```text
    quantum.cloud.ibm.com
    s3.us-east.cloud-object-storage.appdomain.cloud
    ```
- Perkhidmatan IBM Quantum Qiskit Runtime - rantau eu-de:
    ```text
    eu-de.quantum.cloud.ibm.com
    s3.direct.eu-de.cloud-object-storage.appdomain.cloud
    ```
- Perkhidmatan pratonton IBM Quantum Qiskit Functions:
    ```text
    qiskit-serverless.quantum.ibm.com
    ```

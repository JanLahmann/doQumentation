# Peristiwa penjejakan aktiviti untuk Qiskit Runtime {#activity-tracking-events-for-qiskit-runtime}

Perkhidmatan IBM Cloud&reg;, seperti Qiskit Runtime, menjana peristiwa penjejakan aktiviti.

Peristiwa penjejakan aktiviti melaporkan aktiviti yang mengubah keadaan perkhidmatan dalam IBM Cloud. Anda boleh menggunakan peristiwa ini untuk menyiasat aktiviti luar biasa dan tindakan kritikal serta untuk mematuhi keperluan audit kawal selia.

Anda boleh menggunakan Penghalaan Peristiwa Penjejak Aktiviti IBM Cloud, perkhidmatan platform, untuk menghalakan peristiwa audit dalam akaun anda ke destinasi pilihan anda dengan mengkonfigurasi sasaran dan laluan yang menentukan ke mana peristiwa penjejakan aktiviti dihantar. Untuk maklumat lanjut, lihat [Tentang Penghalaan Peristiwa Penjejak Aktiviti IBM Cloud.](https://cloud.ibm.com/docs/atracker?topic=atracker-about)

Anda boleh menggunakan IBM Cloud Logs untuk menggambarkan dan memberi amaran tentang peristiwa yang dijana dalam akaun anda dan dihalakan oleh Penghalaan Peristiwa Penjejak Aktiviti IBM Cloud ke instance IBM Cloud Logs.

## Lokasi di mana peristiwa penjejakan aktiviti dijana {#locations-where-activity-tracking-events-are-generated}

Qiskit Runtime menghantar peristiwa penjejakan aktiviti di rantau berikut:

- Washington ("us-east")
- Frankfurt ("eu-de")

## Melihat peristiwa penjejakan aktiviti untuk Qiskit Runtime {#viewing-activity-tracking-events-for-qiskit-runtime}

Anda boleh menggunakan IBM Cloud Logs untuk menggambarkan dan memberi amaran tentang peristiwa yang dijana dalam akaun anda dan dihalakan oleh Penghalaan Peristiwa Penjejak Aktiviti IBM Cloud ke instance IBM Cloud Logs.

### Melancarkan IBM Cloud Logs dari halaman Kebolehperhatian {#launching-ibm-cloud-logs-from-the-observability-page}

Untuk maklumat tentang melancarkan UI IBM Cloud Logs, lihat [Melancarkan UI](https://cloud.ibm.com/docs/cloud-logs?topic=cloud-logs-instance-launch#instance-launch-cloud-ui) dalam dokumentasi IBM Cloud Logs.

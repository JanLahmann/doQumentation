# Qiskit Code Assistant {#qiskit-code-assistant}

Qiskit Code Assistant bertujuan untuk menjadikan pengkomputeran kuantum lebih mudah diakses oleh pengguna Qiskit baharu dan meningkatkan pengalaman pengekodan bagi pengguna semasa. Ia adalah pembantu kod AI generatif yang dikuasakan oleh [watsonx](https://www.ibm.com/products/watsonx-ai). Ia dilatih menggunakan jutaan token teks dari Qiskit SDK, bertahun-tahun contoh kod Qiskit, dan ciri IBM Quantum&reg;. Qiskit Code Assistant boleh membantu aliran kerja pembangunan kuantum anda dengan menawarkan cadangan yang dijana oleh LLM berdasarkan [model IBM Granite](https://www.ibm.com/products/watsonx-ai/foundation-models), yang menggabungkan ciri dan fungsi terkini dari IBM&reg;.

<Admonition type="note" title="Nota">
- Ini adalah ciri eksperimental yang tersedia untuk pengguna IBM Quantum Premium Plan yang berdaftar pada [IBM Quantum Platform](https://quantum.cloud.ibm.com/) yang baharu.
- Qiskit Code Assistant sedang dalam status keluaran pratonton dan boleh berubah.
- Jika anda ada maklum balas atau ingin menghubungi pasukan pembangun, gunakan [saluran Qiskit Slack Workspace](https://qiskit.enterprise.slack.com/archives/C07LYA6PL83) atau repositori GitHub awam yang berkaitan.
</Admonition>

## Ciri-ciri {#features}

Ciri-ciri berikut disertakan dalam sambungan [Visual Studio Code](https://code.visualstudio.com/) (VS Code) dan editor yang serasi, serta sambungan [JupyterLab](https://jupyterlab.readthedocs.io/en/latest/):

* Mempercepat penjanaan kod Qiskit dengan memanfaatkan AI generatif berdasarkan model yang pakar dalam menjana kod Qiskit.
* Membenarkan arahan abstrak dan spesifik untuk menjana cadangan.
* Mempersembahkan cadangan yang boleh anda semak, terima, atau tolak.
* Menyokong fail kod Python dan Jupyter notebook.
* Termasuk pagar keselamatan untuk mengelak menjawab soalan yang mewakili risiko berpotensi bagi pengguna, seperti pertuturan membenci.

Untuk arahan mengintegrasikan Qiskit Code Assistant terus ke dalam persekitaran pembangunan anda, ikuti arahan dalam topik yang sesuai:
- [JupyterLab](/guides/qiskit-code-assistant-jupyterlab)
- [VS Code](/guides/qiskit-code-assistant-vscode)

## Model Bahasa Besar (LLM) di sebalik Qiskit Code Assistant {#the-large-language-model-llm-behind-qiskit-code-assistant}

Untuk memberikan cadangan kod, Qiskit Code Assistant menggunakan Model Bahasa Besar (LLM). Dalam kes ini, Qiskit Code Assistant kini bergantung pada model `mistral-small-3.2-24b-qiskit`, yang dibina atas [model Mistral-Small-3.2-24B-Qiskit](https://huggingface.co/Qiskit/mistral-small-3.2-24b-qiskit). Model `mistral-small-3.2-24b-qiskit` meningkatkan keupayaan penjanaan kod model `Mistral-Small-3.2-24B-Instruct-2506` untuk Qiskit melalui pra-latihan lanjutan dan penalaan halus pada data Qiskit berkualiti tinggi, serta komit Python dan chat. Untuk maklumat lanjut tentang keluarga model Mistral AI, rujuk [dokumentasi Mistral AI](https://docs.mistral.ai/getting-started/models). Untuk maklumat lanjut tentang model `.*-qiskit`, lihat [Qiskit Code Assistant: Training LLMs for generating Quantum Computing Code](https://arxiv.org/abs/2405.19495).

LLM khusus kami untuk Qiskit juga tersedia sebagai model sumber terbuka. Semak semua model yang tersedia di https://huggingface.co/Qiskit.

## Penanda aras Qiskit HumanEval dan Qiskit HumanEval Hard {#the-qiskit-humaneval-and-qiskit-humaneval-hard-benchmarks}

Untuk menguji `mistral-small-3.2-24b-qiskit` dan model-model lain, kami bekerjasama dengan Qiskit Advocates dan pakar untuk mencipta penanda aras berasaskan pelaksanaan yang dipanggil Qiskit HumanEval (QHE) dan Qiskit HumanEval Hard (QHE Hard), dan menjalankannya pada model-model tersebut. Penanda aras ini serupa dengan [HumanEval](https://arxiv.org/abs/2107.03374), termasuk pelbagai masalah kod yang mencabar untuk diselesaikan, semuanya berdasarkan pustaka Qiskit rasmi.

Penanda aras ini terdiri daripada kira-kira 150 ujian, setiap satu dibuat dari definisi fungsi, diikuti oleh docstring yang memperincikan tugas yang diperlukan model untuk diselesaikan. Setiap contoh juga termasuk penyelesaian kanonik rujukan, serta ujian unit, untuk menilai ketepatan penyelesaian yang dihasilkan. Terdapat tiga tahap kesukaran untuk ujian: asas, pertengahan, dan sukar. Penanda aras Qiskit HumanEval Hard ialah variasi penanda aras Qiskit HumanEval, tetapi mengeluarkan maklumat berkaitan import kod, supaya LLM perlu mengetahui import kaedah atau kelas yang betul. Perubahan ini menjadikan set data jauh lebih mencabar bagi LLM, menurut ujian dan keputusan awal kami.

Set data untuk Qiskit HumanEval dan Qiskit HumanEval Hard tersedia di laman web ini: [Qiskit HumanEval](https://huggingface.co/datasets/Qiskit/qiskit_humaneval) dan [Qiskit HumanEval](https://huggingface.co/datasets/Qiskit/qiskit_humaneval_hard). Anda boleh menyumbang kepada pembangunan penanda aras ini di [repositori GitHub](https://github.com/qiskit-community/qiskit-human-eval).

### Permulaan pantas (disyorkan) {#quick-start-recommended}

Cara paling mudah untuk bermula dengan Qiskit Code Assistant dalam mod tempatan ialah menggunakan skrip persediaan automatik untuk sambungan VS Code atau JupyterLab. Skrip ini akan memasang [Ollama](https://ollama.com/) secara automatik untuk menjalankan LLM, memuat turun model yang disyorkan, dan mengkonfigurasi sambungan untuk anda.

#### Persediaan sambungan VS Code {#vs-code-extension-setup}

Jalankan arahan berikut dalam terminal anda:

```bash
bash <(curl -fsSL https://raw.githubusercontent.com/Qiskit/qiskit-code-assistant-vscode/main/setup_local.sh)
```

Skrip ini melaksanakan langkah-langkah berikut:
- Memasang Ollama (jika belum dipasang)
- Memuat turun dan mengkonfigurasi model Qiskit Code Assistant yang disyorkan
- Menyediakan sambungan VS Code untuk berfungsi dengan penggunaan tempatan anda

#### Persediaan sambungan JupyterLab {#jupyterlab-extension-setup}

Jalankan arahan berikut dalam terminal anda:

```bash
bash <(curl -fsSL https://raw.githubusercontent.com/Qiskit/qiskit-code-assistant-jupyterlab/main/setup_local.sh)
```

Skrip ini akan:
- Memasang Ollama (jika belum dipasang)
- Memuat turun dan mengkonfigurasi model Qiskit Code Assistant yang disyorkan
- Menyediakan sambungan JupyterLab untuk berfungsi dengan penggunaan tempatan anda

#### Menggunakan aplikasi Ollama {#using-the-ollama-application}

Aplikasi Ollama menyediakan penyelesaian mudah untuk menjalankan LLM secara tempatan. Ia mudah digunakan, dengan CLI yang menjadikan keseluruhan proses persediaan, pengurusan model, dan interaksi agak mudah. Ia sesuai untuk eksperimen pantas dan untuk pengguna yang mahu mengendalikan butiran teknikal yang lebih sedikit.

##### Pasang Ollama {#install-ollama}

1. Muat turun [aplikasi Ollama](https://ollama.com/download)
1. Pasang fail yang dimuat turun
1. Lancarkan aplikasi Ollama yang dipasang

    <Admonition type="info">
        Aplikasi berjalan dengan jayanya apabila ikon Ollama muncul dalam bar menu desktop. Anda juga boleh mengesahkan perkhidmatan sedang berjalan dengan pergi ke `http://localhost:11434/`.
    </Admonition>

1. Cuba Ollama dalam terminal anda dan mula menjalankan model. Sebagai contoh:

    ```
    ollama run hf.co/Qiskit/Qwen2.5-Coder-14B-Qiskit
    ```

##### Sediakan Ollama menggunakan integrasi Hugging Face Hub {#set-up-ollama-using-the-hugging-face-hub-integration}

[Integrasi Ollama/Hugging Face Hub](https://huggingface.co/docs/hub/ollama) menyediakan cara untuk berinteraksi dengan model yang dihoskan di Hugging Face Hub tanpa perlu mencipta modelfile baharu atau memuat turun fail GGUF atau safetensors secara manual. Fail `template` dan `params` lalai sudah disertakan untuk model di Hugging Face Hub.

1. Pastikan aplikasi Ollama sedang berjalan.

1. Pergi ke halaman model yang dikehendaki, dan salin URL. Sebagai contoh, https://huggingface.co/Qiskit/Qwen2.5-Coder-14B-Qiskit-GGUF.

1. Dari terminal anda, jalankan arahan:

    ```
    ollama run hf.co/Qiskit/Qwen2.5-Coder-14B-Qiskit
    ```

Anda boleh menggunakan model `hf.co/Qiskit/Qwen2.5-Coder-14B-Qiskit` atau mana-mana model GGUF rasmi yang disyorkan masa kini `hf.co/Qiskit/mistral-small-3.2-24b-qiskit-GGUF` atau `hf.co/Qiskit/granite-3.3-8b-qiskit-GGUF`.

##### Sediakan Ollama dengan model GGUF Qiskit Code Assistant yang dimuat turun secara manual {#set-up-ollama-with-a-manually-downloaded-qiskit-code-assistant-gguf-model}

Jika anda telah memuat turun model GGUF secara manual seperti https://huggingface.co/Qiskit/Qwen2.5-Coder-14B-Qiskit-GGUF dan anda ingin bereksperimen dengan templat dan parameter yang berbeza, anda boleh ikuti langkah-langkah ini untuk memuatkannya ke dalam aplikasi Ollama tempatan anda.

1. Cipta `Modelfile` dengan memasukkan kandungan berikut dan pastikan anda mengemas kini `<PATH-TO-GGUF-FILE>` kepada laluan sebenar model yang anda muat turun.

    ```
    FROM <PATH-TO-GGUF-FILE>
    TEMPLATE """{{ if .System }}
    System:
    {{ .System }}

    {{ end }}{{ if .Prompt }}Question:
    {{ .Prompt }}

    {{ end }}Answer:
    ```python{{ .Response }}
    """

    PARAMETER stop "Question:"
    PARAMETER stop "Answer:"
    PARAMETER stop "System:"
    PARAMETER stop "```"

    PARAMETER temperature 0
    PARAMETER top_k 1
    ```

1. Jalankan arahan berikut untuk mencipta contoh model tersuai berdasarkan `Modelfile`.

    ```
    ollama create Qwen2.5-Coder-14B-Qiskit -f ./path-to-model-file
    ```

    <Admonition type="note">
        Proses ini mungkin mengambil sedikit masa untuk Ollama membaca fail model, memulakan contoh model, dan mengkonfigurasinya mengikut spesifikasi yang diberikan.
    </Admonition>

##### Jalankan model Qiskit Code Assistant yang dimuat turun secara manual dalam Ollama {#run-the-qiskit-code-assistant-model-manually-downloaded-in-ollama}

Selepas model `Qwen2.5-Coder-14B-Qiskit` disediakan dalam Ollama, jalankan arahan berikut untuk melancarkan model dan berinteraksi dengannya dalam terminal (dalam mod chat).

```
ollama run Qwen2.5-Coder-14B-Qiskit
```

Beberapa arahan berguna:

- `ollama list` - Senaraikan model pada komputer anda
- `ollama rm Qwen2.5-Coder-14B-Qiskit` - Padam model
- `ollama show Qwen2.5-Coder-14B-Qiskit` - Tunjukkan maklumat model
- `ollama stop Qwen2.5-Coder-14B-Qiskit` - Hentikan model yang sedang berjalan
- `ollama ps` - Senaraikan model yang sedang dimuatkan

</AccordionItem>

<AccordionItem title="Gunakan model Qiskit Code Assistant secara tempatan melalui pakej `llama-cpp-python` secara manual">

<span id="use-llama-cpp-python"></span>
Alternatif kepada aplikasi Ollama ialah pakej `llama-cpp-python`, iaitu pengikatan Python untuk `llama.cpp`. Ia memberi anda lebih kawalan dan fleksibiliti untuk menjalankan model GGUF secara tempatan, dan sesuai untuk pengguna yang ingin mengintegrasikan model tempatan dalam aliran kerja dan aplikasi Python mereka.

1. Pasang [`llama-cpp-python`](https://pypi.org/project/llama-cpp-python/)
2. Berinteraksi dengan model dari dalam aplikasi anda menggunakan `llama_cpp`. Sebagai contoh:

```python
from llama_cpp import Llama

model_path = <PATH-TO-GGUF-FILE>

model = Llama(
        model_path,
        seed=17,
        n_ctx=10000,
        n_gpu_layers=37, # to offload in gpu, but put 0 if all in cpu
    )

input = 'Generate a quantum circuit with 2 qubits'
raw_pred = model(input)["choices"][0]["text"]
```

Anda juga boleh menambah parameter penjanaan teks pada model untuk menyesuaikan inferens:

```python
generation_kwargs = {
        "max_tokens": 512,
        "echo": False, # Echo the prompt in the output
        "top_k": 1
    }

raw_pred = model(input, **generation_kwargs)["choices"][0]["text"]
```

</AccordionItem>

<AccordionItem title="Gunakan model Qiskit Code Assistant secara tempatan melalui llama.cpp secara manual">
#### Gunakan pustaka `llama.cpp` {#use-the-llamacpp-library}

Alternatif lain ialah menggunakan `llama.cpp`, pustaka sumber terbuka untuk melaksanakan inferens LLM pada CPU dengan persediaan yang minimum.
Ia menyediakan kawalan peringkat rendah ke atas pelaksanaan model dan biasanya dijalankan dari baris arahan, menunjuk ke fail model GGUF tempatan.

Terdapat beberapa cara untuk memasang `llama.cpp` pada mesin anda:

- Pasang llama.cpp menggunakan [brew, nix, atau winget](https://github.com/ggml-org/llama.cpp/blob/master/docs/install.md)
- Jalankan dengan Docker: Lihat [dokumentasi Docker oleh pasukan `llama.cpp`](https://github.com/ggml-org/llama.cpp/blob/master/docs/docker.md)
- Muat turun binari pra-bina dari [halaman keluaran](https://github.com/ggml-org/llama.cpp/releases)
- Bina dari sumber dengan [mengklon repositori ini](https://github.com/ggml-org/llama.cpp/blob/master/docs/build.md)

Setelah dipasang, anda boleh menggunakan `llama.cpp` untuk berinteraksi dengan model GGUF dalam mod perbualan seperti berikut:
```bash
# Use a local model file
llama-cli -m my_model.gguf -cnv

# Or download and run a model directly from Hugging Face
llama-cli -hf Qiskit/Qwen2.5-Coder-14B-Qiskit-GGUF -cnv
```

Anda juga boleh melancarkan pelayan API yang serasi dengan OpenAI untuk model dengan cara berikut:
```bash
llama-server -hf Qiskit/Qwen2.5-Coder-14B-Qiskit-GGUF
```

##### Parameter lanjutan {#advanced-parameters}
Dengan program `llama-cli`, anda boleh mengawal penjanaan model menggunakan pilihan baris arahan. Sebagai contoh, anda boleh memberikan gesaan “system” awal menggunakan bendera `-p/--prompt`. Dalam mod perbualan (`-cnv`), gesaan awal ini bertindak sebagai mesej sistem. Jika tidak, anda boleh sekadar menambah sebarang arahan yang dikehendaki ke teks gesaan anda. Anda juga boleh melaraskan parameter persampelan - sebagai contoh: temperature (`--temp`), top-k (`--top-k`), top-p (`--top-p`), penalti pengulangan (`--repeat-penalty`), dan seed yang digunakan (`--seed`). Berikut adalah contoh seruan menggunakan pilihan ini:

```bash
llama-cli -hf Qiskit/Qwen2.5-Coder-14B-Qiskit-GGUF \
  -p "You are a friendly assistant." -cnv \
  --temp 0.7 \
  --top-k 50 \
  --top-p 0.95 \
  --repeat-penalty 1.1 \
  --seed 42
```

Untuk memastikan fungsi model Qiskit kami berfungsi dengan betul, kami mengesyorkan menggunakan gesaan sistem yang disediakan dalam repositori HF GGUF kami: gesaan sistem untuk [mistral-small-3.2-24b-qiskit-GGUF](https://huggingface.co/Qiskit/mistral-small-3.2-24b-qiskit-GGUF/blob/main/system), [Qwen2.5-Coder-14B-Qiskit-GGUF](https://huggingface.co/Qiskit/Qwen2.5-Coder-14B-Qiskit-GGUF/blob/main/system), [granite-3.3-8b-qiskit-GGUF](https://huggingface.co/Qiskit/granite-3.3-8b-qiskit-GGUF/blob/main/system), dan [granite-3.2-8b-qiskit-GGUF](https://huggingface.co/Qiskit/granite-3.2-8b-qiskit-GGUF/blob/main/system).

<AccordionItem title="Sambungkan sambungan ke penggunaan tempatan secara manual">

Gunakan sambungan VS Code dan sambungan JupyterLab untuk Qiskit Code Assistant bagi menggesa model Qiskit Code Assistant yang digunakan secara tempatan. Setelah anda mempunyai aplikasi Ollama [disediakan dengan model](#using-the-ollama-application), anda boleh mengkonfigurasi sambungan untuk menyambung ke perkhidmatan tempatan.

#### Sambung dengan sambungan VS Code Qiskit Code Assistant {#connect-with-the-qiskit-code-assistant-vs-code-extension}

Dengan sambungan VS Code Qiskit Code Assistant, anda boleh berinteraksi dengan model dan melaksanakan penyiapan kod semasa menulis kod anda. Ini berfungsi dengan baik untuk pengguna yang mencari bantuan menulis kod Qiskit untuk aplikasi Python mereka.

1. Pasang [sambungan VS Code Qiskit Code Assistant](/guides/qiskit-code-assistant-vscode).
1. Dalam VS Code, pergi ke **User Settings** dan tetapkan **Qiskit Code Assistant: Url** kepada URL penggunaan Ollama tempatan anda (sebagai contoh, `http://localhost:11434`).
1. Muat semula VS Code dengan pergi ke **View > Command Palette...** dan memilih **Developer: Reload Window**.

Model Qiskit Code Assistant yang dikonfigurasi dalam Ollama sepatutnya muncul dalam bar status dan kemudian sedia untuk digunakan.

#### Sambung dengan sambungan JupyterLab Qiskit Code Assistant {#connect-with-the-qiskit-code-assistant-jupyterlab-extension}

Dengan sambungan JupyterLab Qiskit Code Assistant, anda boleh berinteraksi dengan model dan melaksanakan penyiapan kod terus dalam Jupyter Notebook anda. Pengguna yang kebanyakannya bekerja dengan Jupyter Notebook boleh memanfaatkan sambungan ini untuk terus meningkatkan pengalaman menulis kod Qiskit mereka.

1. Pasang [sambungan JupyterLab Qiskit Code Assistant](/guides/qiskit-code-assistant-jupyterlab).
1. Dalam JupyterLab, pergi ke **Settings Editor** dan tetapkan **Qiskit Code Assistant Service API** kepada URL penggunaan Ollama tempatan anda (sebagai contoh, `http://localhost:11434`).

Model Qiskit Code Assistant yang dikonfigurasi dalam Ollama sepatutnya muncul dalam bar status dan kemudian sedia untuk digunakan.

# Urus pengguna IBM Cloud {#manage-ibm-cloud-users}

Panduan ini menunjukkan cara menggunakan IBM Cloud&reg; untuk mendayakan pengguna yang mempunyai akaun IBM Cloud dan memberikan arahan untuk pengguna mengakses persekitaran.

Untuk mengurus pengguna penyedia ID sebaliknya, ikut arahan dalam topik [Urus penyedia ID](/guides/manage-appid).

## Langkah-langkah {#steps}

Ikut langkah-langkah peringkat tinggi ini untuk mengurus pengguna IBM Cloud:

1. Pastikan pengguna yang ingin anda jemput mempunyai akaun IBM Cloud.
2. Pergi ke IAM dan [jemput pengguna](https://cloud.ibm.com/iam/users/invite_users)
   <Admonition type="note">
   Pengguna tidak boleh diurus sehingga mereka menerima jemputan dan log masuk sekurang-kurangnya sekali.
   </Admonition>
3. Pilihan: Ubah suai tugasan projek dan kumpulan akses pengguna daripada [IAM.](https://cloud.ibm.com/iam/users)

<span id="user-org"></span>
## Aliran pengguna {#user-flow}

1. Pengguna menerima jemputan, kemudian log masuk melalui [portal IBM Cloud](https://cloud.ibm.com/).
2. Pengguna mencipta kunci API dengan pergi ke ([Urus → Akses (IAM) → Kunci API](https://cloud.ibm.com/iam/apikeys)).
3. Untuk maklumat lanjut, pengguna boleh semak [Pasang Qiskit.](/guides/install-qiskit)

<span id="steps-cloud-org"></span>
## Senario contoh {#example-scenario}

Contoh ini mencipta persediaan berikut:

* Terdapat dua projek, `ml` dan `finance`.
   * Projek `ml` sepatutnya mempunyai akses ke instans perkhidmatan `QR-ml` dan `QR-common`.
   * Projek `finance` sepatutnya mempunyai akses ke instans perkhidmatan `QR-finance` dan `QR-common`.
* Terdapat tiga pengguna:
   * Fatima sepatutnya mempunyai akses ke projek `ml`.
   * Ravi sepatutnya mempunyai akses ke projek `finance`.
   * Amyra sepatutnya mempunyai akses ke kedua-dua projek.
* Ia menggunakan kumpulan akses tanpa kumpulan sumber.
* Pengguna ditentukan dalam IBM Cloud dan tugasan projek juga dilakukan di sana.
* Pengguna sepatutnya boleh memadam kerja.

Langkah-langkah untuk melaksanakan persediaan ini adalah:

2. Pentadbir Cloud mencipta tiga instans perkhidmatan: `QR-ml`, `QR finance`, dan `QR-common`.
3. Pentadbir Cloud mencipta peraturan kustom yang menyertakan tindakan `quantum-computing.job.delete`.
1. Pentadbir Cloud mencipta dua kumpulan akses:
   * Kumpulan akses `ml` boleh mengakses `QR-ml` dan `QR-common`.
   * Kumpulan akses `finance` boleh mengakses `QR-finance` dan `QR-common`.
1. Pentadbir Cloud menjemput pengguna awan ke projek yang sesuai. Secara khusus, mereka menjemput dan menugaskan pengguna ke kumpulan akses yang menyertakan projek.
   * Fatima ditambahkan ke kumpulan akses "ml".
   * Ravi ditambahkan ke kumpulan akses "finance".
   * Amyra ditambahkan ke kedua-dua kumpulan akses "ml" dan "finance".
1. Pengguna boleh log masuk melalui portal IBM Cloud, mencipta kunci API, dan bekerja dengan instans perkhidmatan projek mereka.

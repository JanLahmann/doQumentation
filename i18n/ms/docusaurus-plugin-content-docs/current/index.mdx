## Kandungan sumber terbuka IBM Quantum {#ibm-quantums-open-source-content}

IBM menyediakan banyak bahan pembelajaran pengkomputeran kuantum — semuanya sumber terbuka di bawah CC BY-SA 4.0:

- **[Learning](https://quantum.cloud.ibm.com/learning)** — Kursus berstruktur dari asas kuantum hingga topik lanjutan
- **Develop locally** — [install Qiskit](https://docs.quantum.ibm.com/guides/install-qiskit) on your own laptop for serious development work.
- **[Tutorials](https://quantum.cloud.ibm.com/docs/en/tutorials)** — Lebih 40 tutorials tentang transpilation, error mitigation dan lain-lain
- **[Documentation](https://quantum.cloud.ibm.com/docs/en/guides)** — Panduan dan rujukan API untuk Qiskit
- **[Source repo](https://github.com/Qiskit/documentation)** — Semua kandungan di GitHub

[Quantum Platform](https://quantum.cloud.ibm.com) mereka sentiasa terkini dan direka dengan baik — tempat terbaik untuk membaca, belajar dan rujukan.

## Apa yang projek ini tambah {#what-this-project-adds}

[Dokumentasi Qiskit](https://github.com/Qiskit/documentation) IBM adalah sumber terbuka (CC BY-SA 4.0), tetapi aplikasi web mereka tidak. doQumentation menambah frontend sumber terbuka dengan pelaksanaan kod langsung, suntikan kelayakan automatik dan mod simulator.

Boleh digunakan di mana-mana — dari [GitHub Pages](https://doqumentation.org) ke [Docker](https://github.com/JanLahmann/doQumentation/pkgs/container/doqumentation) ke [RasQberry](https://rasqberry.org/). Lihat [semua ciri](/features).

## Bila dah ready nak lebih {#when-youre-ready-for-more}

doQumentation bantu korang buat **langkah pertama dengan Qiskit** — baca, jalankan kod, dan terokai — tanpa perlu install apa-apa. Bila dah maju, pergi ke platform rasmi IBM:

- **Jalankan pada perkakasan sebenar** — cipta [akaun IBM Cloud](https://quantum.cloud.ibm.com/registration) percuma dan instans Open Plan untuk laksanakan litar pada komputer kuantum sebenar.
- **Kandungan terkini** — [Platform pembelajaran Qiskit](https://quantum.cloud.ibm.com/learning) IBM ada kursus, tutorial, dan tambahan terbaru (sesetengah mungkin belum ada dalam doQumentation).
- **Bangun secara tempatan** — [pasang Qiskit](https://docs.quantum.ibm.com/guides/install-qiskit) pada laptop sendiri untuk kerja pembangunan yang serius.

*Dibina berdasarkan [dokumentasi sumber terbuka IBM Quantum](https://github.com/Qiskit/documentation). Terima kasih kepada pasukan IBM Quantum kerana menjadikan kandungan ini tersedia secara percuma di bawah CC BY-SA 4.0.*

## Pelaksanaan kod {#code-execution}

Klik **Run** pada mana-mana blok kod. Klik pertama memulakan kernel Jupyter melalui [Binder](https://mybinder.org) atau [IBM Code Engine](/jupyter-settings#code-engine). Selepas itu, pelaksanaan adalah serta-merta.

- **Mod Simulator (tiada akaun diperlukan)** — Aktifkan [Mod Simulator](/jupyter-settings#simulator-mode) untuk jalankan semua notebook dengan AerSimulator atau FakeBackends model hingar. Tiada persediaan diperlukan.

- **Perkakasan IBM Quantum** — Masukkan token API dan CRN anda dalam [Tetapan](/jupyter-settings#ibm-quantum) sekali sahaja — kelayakan disuntik secara automatik pada setiap pelaksanaan. [Cipta akaun percuma](https://quantum.cloud.ibm.com/registration) jika belum ada.

<details>
<summary>Backend pelaksanaan yang tersedia</summary>

Setiap tutorial mempunyai blok kod yang boleh dilaksanakan. Klik **Run** untuk melaksanakannya menggunakan salah satu daripada empat backend:

1. **Binder** (lalai pada GitHub Pages) — Kernel Jupyter jauh percuma melalui [mybinder.org](https://mybinder.org)
2. **IBM Code Engine** — Jupyter tanpa pelayan pada akaun IBM Cloud anda. Cold start dalam beberapa saat berbanding minit. [Sediakan](/jupyter-settings#code-engine)
3. **Jupyter Tempatan (Docker / RasQberry)** — Sambung ke pelayan Jupyter tempatan dengan Qiskit diprapasang
4. **Pelayan tersuai** — Tunjuk ke mana-mana endpoint Jupyter dalam [Tetapan](/jupyter-settings#advanced)

  <strong>No IBM Quantum account?</strong> Enable <a href="/jupyter-settings#execution-mode">Simulator Mode</a> in Settings to run all code without signing up.

Apabila berbilang backend tersedia, pilih yang mana satu untuk digunakan dari [Tetapan](/jupyter-settings#backend-selection).

</details>

<details>
<summary>Pilihan penggunaan</summary>

doQumentation tersedia sebagai:

- **[GitHub Pages](https://doqumentation.org)** — Laman statik dengan [Binder](https://mybinder.org) atau [IBM Code Engine](/jupyter-settings#code-engine) untuk pelaksanaan kod jauh
- **[Docker](https://github.com/JanLahmann/doQumentation/pkgs/container/doqumentation)** — Full stack secara tempatan, mampu beroperasi luar talian
- **[RasQberry](https://rasqberry.org/)** — Self-hosted pada Raspberry Pi dengan kernel Jupyter tempatan

</details>

<details>
<summary>Jalankan secara tempatan dengan Podman / Docker</summary>

Pasang [Podman Desktop](https://podman-desktop.io/) atau [Docker Desktop](https://www.docker.com/products/docker-desktop/) (Mac / Windows / Linux), kemudian jalankan:

```bash
# Full stack: site + Jupyter + Qiskit (~3 GB)
podman run -p 8080:80 -p 8888:8888 ghcr.io/janlahmann/doqumentation:jupyter
```

Buka [http://localhost:8080](http://localhost:8080) — pelaksanaan kod berfungsi secara tempatan, tiada masa tunggu Binder.

Untuk versi ringan tanpa pelaksanaan kod tempatan (~60 MB):

```bash
# Static site only — code execution still works via Binder
podman run -p 8080:80 ghcr.io/janlahmann/doqumentation:latest
```

Guna Docker sebaliknya? Gantikan sahaja `podman` dengan `docker` — arahan adalah sama. Image adalah multi-arch (`linux/amd64` + `linux/arm64`), jadi Apple Silicon Mac berfungsi secara native.

</details>

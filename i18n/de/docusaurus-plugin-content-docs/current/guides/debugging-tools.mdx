# Einführung in Debugging-Tools

Sie können Ihre Quantenprogramme testen, indem Sie sie auf simulierten Geräten ausführen und ihre Leistung unter realistischen Geräterauschmodellen untersuchen. Dadurch können Sie sie debuggen, bevor Sie sie an eine Quantenprozessoreinheit (QPU) senden.

Quantensimulatoren können verwendet werden, um Programme zu entwickeln und zu testen, bevor sie feinabgestimmt und an Quantenhardware gesendet werden. Lokale Simulatoren können dies mit guter Leistung und Effizienz durchführen.

Da die Kosten für die klassische Simulation von Quantenschaltkreisen exponentiell mit der Anzahl der Qubits skalieren, können Schaltkreise mit mehr als 50 Qubits oder so im Allgemeinen nicht auf Simulatoren ausgeführt werden. Für solche Schaltkreise können Sie:

- Kleinere Versionen der Schaltkreise testen, die klassisch simuliert werden können.
- Die Schaltkreise so modifizieren, dass sie klassisch simulierbar werden, obwohl sie weniger genau sind.

Stabilizer-Schaltkreise, auch als Clifford-Schaltkreise bekannt, sind ein nützliches Werkzeug, um dieses letztere Ziel zu erreichen. Dies sind eine eingeschränkte Klasse von Quantenschaltkreisen, die effizient klassisch simuliert werden können. Spezialisierte Simulatoren können Stabilizer-Schaltkreise mit Tausenden von Qubits problemlos simulieren. Siehe [Effiziente Simulation von Stabilizer-Schaltkreisen mit Qiskit Aer Primitives](./simulate-stabilizer-circuits) für weitere Informationen.

Für allgemeine Quantenschaltkreise stehen die folgenden Tools zum Testen und Debuggen Ihrer Quantenprogramme zur Verfügung:

- Zum Simulieren von Schaltkreisen bei Verwendung von Qiskit Runtime verwenden Sie den [Local Testing Mode](local-testing-mode).
- Für die exakte Simulation kleiner Quantenschaltkreise können Sie die mit Qiskit enthaltenen Referenz-Primitives verwenden. Siehe [Exakte Simulation mit Qiskit Primitives](./simulate-with-qiskit-sdk-primitives).
- Für leistungsstärkere Simulation, die größere Schaltkreise bewältigen kann, oder um Rauschmodelle in Ihre Simulation einzubeziehen, verwenden Sie [Qiskit Aer](https://qiskit.org/ecosystem/aer/), ein Projekt, das Teil des [Qiskit Ecosystem](https://qiskit.github.io/ecosystem/) ist. Siehe [Exakte und rauschbehaftete Simulation mit Qiskit Aer Primitives](simulate-with-qiskit-aer).
- Um benutzerdefinierte Rauschmodelle zu erstellen, verwenden Sie das [`noise`](https://qiskit.org/ecosystem/aer/apidocs/aer_noise.html)-Modul von Qiskit Aer. Siehe [Rauschmodelle erstellen](./build-noise-models).
- Für Qiskit Runtime Estimator-Jobs verwenden Sie die Qiskit Runtime [`Neat`](https://docs.quantum.ibm.com/api/qiskit-ibm-runtime/debug-tools)-Klasse, um Ihre Jobs zu debuggen und zu analysieren. Siehe [Qiskit Runtime Jobs debuggen](/guides/debug-qiskit-runtime-jobs) für Details.

## Hardware-Überlegungen

Mehrere Faktoren beeinflussen, wie viel Speicher die Quantensimulation benötigt, sodass es keine exakten Hardwareanforderungen für die Simulation gibt, aber es gibt einige Richtlinien, die Sie befolgen können.

- Die einzige Anforderung zum Ausführen von Qiskit ist eine funktionierende Python-Umgebung. Einzelheiten finden Sie im Abschnitt [Betriebssystemunterstützung](./install-qiskit#operating-system-support).
- Da die Anforderungen für die Simulation von Quantenschaltkreisen exponentiell mit der Anzahl der Qubits skalieren, begrenzt die verfügbare Hardware die Anzahl der Qubits, die simuliert werden können. Beispielsweise kann ein System mit 4 GB RAM ungefähr 27 Qubits simulieren.
- Mehr oder weniger verfügbarer Speicher führt nicht zu mehr oder weniger genauen Ergebnissen (vorausgesetzt, Ergebnisse werden zurückgegeben), obwohl mehr Speicher Ergebnisse schneller zurückgeben oder Ihnen ermöglichen könnte, mehr Qubits zu simulieren.
- Um den größten Nutzen aus Ihrer Hardware zu ziehen, verwenden Sie wenn möglich lokale Stabilizer- (Clifford-) Schaltkreissimulation. Siehe [Clifford-Simulation mit Qiskit Runtime Local Testing Mode](local-testing-mode#clifford-simulation) oder [Effiziente Simulation von Stabilizer-Schaltkreisen mit Qiskit Aer Primitives](./simulate-stabilizer-circuits) für Beispiele.
- Sie können Ihre Rechenleistung erhöhen, indem Sie diesen Anweisungen folgen, um [mit mehreren GPUs, Knoten oder beidem auszuführen](https://qiskit.github.io/qiskit-aer/howtos/running_gpu.html).

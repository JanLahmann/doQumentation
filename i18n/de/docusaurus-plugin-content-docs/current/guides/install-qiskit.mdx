# Qiskit installieren

Egal ob Sie lokal oder in einer Cloud-Umgebung arbeiten werden, der erste Schritt für alle Benutzer ist die Installation von Qiskit.

<Admonition type="caution" title="Upgrade von Qiskit 0.x auf Qiskit 1.0 und höher">
  (Falls Sie Qiskit zum ersten Mal installieren, fahren Sie mit dem Abschnitt [Installation und Einrichtung](#local) fort. Dieser Hinweis ist nur für Benutzer relevant, die Qiskit bereits zuvor installiert haben.)

  **Für diejenigen, die von Version 0.x auf 1.0 oder höher upgraden**: Beachten Sie, dass Sie aufgrund der neuen Paketstruktur von Qiskit v1.0 **nicht** `pip install -U qiskit` verwenden können, um von einer Qiskit 0.x-Version auf 1.0 zu upgraden.

  Siehe den [Qiskit 1.0 Migrationsleitfaden](/guides/qiskit-1.0) für Details und Anweisungen.

  Zukünftige Updates ab Qiskit 1.0 ermöglichen In-Place-Upgrades.
</Admonition>

<span id="local"></span>
## Qiskit SDK und Qiskit Runtime-Client installieren

1. Installieren Sie Python. Überprüfen Sie den Abschnitt "Programming Language" auf der [Qiskit PyPI-Projektseite](https://pypi.org/project/qiskit/), um festzustellen, welche Python-Versionen von der neuesten Version unterstützt werden. Download-Anweisungen finden Sie im [Python Beginners Guide.](https://wiki.python.org/moin/BeginnersGuide/Download)

    Es wird empfohlen, [virtuelle Python-Umgebungen](https://docs.python.org/3.10/tutorial/venv.html) zu verwenden, um Qiskit von anderen Anwendungen zu trennen.

    <Admonition type="note">
    Diese Anweisungen verwenden die Standard-Python-Distribution von [pypi.org](https://pypi.org/). Sie können jedoch auch andere Python-Distributionen wie [Anaconda](https://docs.anaconda.com/anaconda/) oder [miniconda](https://docs.anaconda.com/miniconda/) zusammen mit anderen Abhängigkeitsverwaltungs-Workflows wie [Poetry](https://python-poetry.org/docs/) verwenden.
    </Admonition>

    <details>
    <summary>
    Falls Sie neu bei virtuellen Umgebungen sind, klicken Sie hier für weitere Informationen.
    </summary>
    Eine virtuelle Python-Umgebung ist ein isolierter Raum, um mit Python für einen bestimmten Zweck zu arbeiten – sodass Sie beliebige Pakete installieren, Bibliotheken, Abhängigkeiten usw. einrichten können, ohne die "Basis"-Python-Umgebung auf Ihrem Computer zu beeinflussen.

    Ein wichtiger Vorteil einer virtuellen Umgebung ist, dass Sie, wenn Ihre Python-Umgebung irgendwann beschädigt wird, die virtuelle Umgebung einfach löschen und von vorne beginnen können!

    Wählen Sie einen bevorzugten Speicherort, an dem Sie Informationen über Ihre virtuellen Umgebungen speichern möchten. Normalerweise werden sie in einem Verzeichnis namens `.venv` innerhalb jedes Projektverzeichnisses gespeichert, an dem Sie arbeiten.
    </details>

    Navigieren Sie zuerst zu Ihrem Projektverzeichnis und erstellen Sie eine minimale Umgebung, in der nur Python installiert ist.

        <OperatingSystemTabs>
          <TabItem value="mac" label="macOS">
            ```shell
            python3 -m venv .venv
            ```
          </TabItem>

          <TabItem value="linux" label="Linux">
            ```shell
            python3 -m venv .venv
            ```
          </TabItem>

          <TabItem value="win" label="Windows">
            ```text
            python -m venv .venv
            ```
          </TabItem>
         </OperatingSystemTabs>

    Als Nächstes aktivieren Sie Ihre neue Umgebung.

         <OperatingSystemTabs>
           <TabItem value="mac" label="macOS">
             ```shell
             source .venv/bin/activate
             ```
           </TabItem>

           <TabItem value="linux" label="Linux">
             ```shell
             source .venv/bin/activate
             ```
           </TabItem>

           <TabItem value="win" label="Windows">
              Falls Sie PowerShell verwenden:

             ```text
             .venv\Scripts\Activate.ps1
             ```
             Falls Sie Git Bash verwenden:

             ```text
             source .venv/scripts/activate
             ```
              Falls Sie die Eingabeaufforderung verwenden:

             ```text
             .venv\Scripts\activate
             ```
           </TabItem>
         </OperatingSystemTabs>

1. [Installieren Sie pip](https://pip.pypa.io/en/stable/installation/), falls es noch nicht in Ihrer Umgebung installiert ist. Pip ist ein Python-Paketmanager, den Sie zum Installieren von Qiskit und anderen Python-Paketen verwenden. Verwenden Sie `pip list`, um zu sehen, was sich in Ihrer virtuellen Umgebung befindet. In den meisten Python-Umgebungen ist pip bereits installiert.

1. Installieren Sie das Qiskit SDK. Falls Sie planen, Jobs auf Quantenhardware auszuführen, installieren Sie auch Qiskit Runtime.

      ```shell
      pip install qiskit
      ```

      ```shell
      pip install qiskit-ibm-runtime
      ```

      <Admonition type="tip" title="Visualisierungsunterstützung hinzufügen">
      Falls Sie Visualisierungsfunktionalität oder Jupyter-Notebooks verwenden möchten, installieren Sie auch das `qiskit.visualization`-Modul. Siehe die [Visualizations API-Referenz](https://docs.quantum.ibm.com/api/qiskit/visualization) für weitere Informationen.
      </Admonition>

<span id="local-jupyter"></span>
1. Falls Sie ein Jupyter-Notebook mit den gerade installierten Qiskit-Paketen ausführen möchten, müssen Sie Jupyter in Ihrer Umgebung installieren.

      ```shell
      pip install jupyter
      ```
    Öffnen Sie dann Ihr Notebook wie folgt:

      ```shell
      jupyter notebook <path/to/notebook.ipynb>
      ```

Falls Sie planen, lokal zu arbeiten und in Qiskit eingebaute Simulatoren zu verwenden, ist Ihre Installation abgeschlossen. Falls Sie Jobs auf IBM QPUs ausführen möchten, fahren Sie fort mit [Richten Sie Ihr IBM Cloud-Konto ein](cloud-setup).

<Admonition type="note" title="Bleiben Sie auf dem neuesten Stand mit den neuesten Versionen">
    Überprüfen Sie regelmäßig die [Qiskit Release Notes](../api/qiskit/release-notes) und die [Qiskit Runtime Release Notes](../api/qiskit-ibm-runtime/release-notes), um neue Releases zu sehen. Wir empfehlen, Ihre Anforderungen für `qiskit` und `qiskit-ibm-runtime` häufig zu aktualisieren, indem Sie beispielsweise die Versionen in `requirements.txt` auf die neuesten Versionen ändern und dann `pip install -r requirements.txt` oder den entsprechenden Befehl für Ihren Abhängigkeitsverwaltungs-Workflow ausführen.
</Admonition>

## Fehlerbehebung

<details>
  <summary>
    "No Module 'qiskit'"-Fehler mit Jupyter Notebook
  </summary>
  Falls Sie ``pip install qiskit`` verwendet und Ihre virtuelle Umgebung in
  Anaconda eingerichtet haben, erhalten Sie möglicherweise den ``No Module 'qiskit'``-Fehler, wenn Sie ein Tutorial
  in Jupyter Notebook ausführen. Falls Sie Qiskit nicht installiert oder Ihre
  virtuelle Umgebung eingerichtet haben, können Sie den [Installationsschritten](#qiskit-install) folgen.

  Der Fehler entsteht, wenn versucht wird, das Qiskit-Paket in einer
  Umgebung zu importieren, in der Qiskit nicht installiert ist. Falls Sie Jupyter Notebook
  vom Anaconda-Navigator gestartet haben, ist es möglich, dass Jupyter Notebook
  in der Basis-(Root-)Umgebung läuft, anstatt in Ihrer virtuellen
  Umgebung. Wählen Sie eine virtuelle Umgebung im Anaconda-Navigator aus dem
  **Applications on**-Dropdown-Menü. In diesem Menü können Sie
  alle virtuellen Umgebungen innerhalb von Anaconda sehen, und Sie können
  die Umgebung auswählen, in der Sie Qiskit installiert haben, um Jupyter
  Notebook zu starten.
</details>

<details>
  <summary>
    Kompilierungsfehler während der Installation
  </summary>
  Qiskit hängt von einer Reihe anderer Open-Source-Python-Pakete ab, die
  automatisch installiert werden, wenn Sie ``pip install qiskit`` ausführen. Abhängig von
  der Plattform und Python-Version Ihres Systems ist es möglich, dass ein bestimmtes
  Paket keine vorgefertigten Binärdateien für Ihr System bereitstellt. Sie können sich auf
  [Betriebssystemunterstützung](#operating-system-support) beziehen für eine Liste der von Qiskit unterstützten Plattformen, von denen
  einige möglicherweise einen zusätzlichen Compiler benötigen. In Fällen, in denen
  keine vorkompilierten Binärdateien verfügbar sind, wird ``pip`` versuchen, das Paket
  aus dem Quellcode zu kompilieren, was wiederum einige zusätzliche Abhängigkeiten erfordern kann, die manuell
  installiert werden müssen.

  Falls die Ausgabe von ``pip install qiskit`` ähnliche Zeilen enthält wie:

  ```
  Failed building wheel for SOME_PACKAGE
  ...
  build/temp.linux-x86_64-3.5/_openssl.c:498:30: fatal error
  compilation terminated.
  error: command 'x86_64-linux-gnu-gcc' failed with exit status 1
  ```

  überprüfen Sie bitte die Dokumentation des Pakets, dessen Installation fehlgeschlagen ist (im
  Beispielcode ``SOME_PACKAGE``), für Informationen zur Installation der
  für die Kompilierung aus dem Quellcode benötigten Bibliotheken.
</details>

## Betriebssystemunterstützung

Qiskit strebt an, so viele Betriebssysteme wie möglich zu unterstützen, aber aufgrund von Einschränkungen bei verfügbaren Testressourcen und Betriebssystemverfügbarkeit können nicht alle Betriebssysteme unterstützt werden. Die Betriebssystemunterstützung für Qiskit ist in drei Stufen mit unterschiedlichen Unterstützungsebenen für jede Stufe unterteilt. Für Plattformen außerhalb dieser, wie FreeBSD oder WebAssembly (WASI), kann Qiskit möglicherweise noch installierbar sein, aber es ist nicht getestet und Sie müssen Qiskit (und wahrscheinlich Qiskits Abhängigkeiten) aus dem Quellcode erstellen.

Zusätzlich unterstützt Qiskit nur die CPython-Implementierung der Python-Sprache. Die Ausführung mit anderen Python-Interpretern wie PyPy wird nicht unterstützt.

### Qiskit v2.x

In der Qiskit v2.x-Release-Serie sind die unterstützten Plattformen:

<details>
  <summary>
    Tier 1
  </summary>
  Tier-1-Betriebssysteme werden als Teil der Entwicklungsprozesse vollständig getestet, um sicherzustellen, dass jede vorgeschlagene Änderung korrekt funktioniert. Vorkompilierte Binärdateien werden im Rahmen des Release-Prozesses erstellt, getestet und auf PyPI veröffentlicht. Normalerweise kann Qiskit auf diesen Betriebssystemen installiert werden, ohne weitere Abhängigkeiten installieren zu müssen, solange eine funktionierende Python-Umgebung installiert ist.

  Tier-1-Betriebssysteme:

  - Linux x86_64 (Distributionen kompatibel mit der [manylinux 2014](https://www.python.org/dev/peps/pep-0599/)-Paketspezifikation).
  - macOS ARM64 (11.0 oder neuer)
  - Windows 64-Bit (Windows 10 und später unterstützt)
  - Linux AArch64 (Distributionen kompatibel mit der [manylinux 2014](https://www.python.org/dev/peps/pep-0599/)-Paketspezifikation)
</details>

<details>
  <summary>
    Tier 2
  </summary>
  Tier-2-Betriebssysteme werden nicht als Teil des Entwicklungsprozesses getestet. Vorkompilierte Binärdateien werden jedoch im Rahmen des Release-Prozesses erstellt, getestet und auf PyPI veröffentlicht, und diese Pakete können voraussichtlich nur mit einer funktionierenden Python-Umgebung installiert werden.
  Es kann zu einer Verzögerung bei der Veröffentlichung von Paketen für diese Systeme kommen, da Testfehler möglicherweise erst viel später erkannt werden und Fehler während der Veröffentlichung die Publikation des Pakets für Tier-1-Systeme nicht blockieren.

  Tier-2-Betriebssysteme:

  - macOS x86_64 (10.12 oder später)
</details>

<details>
  <summary>
    Tier 3
  </summary>
  Tier-3-Betriebssysteme werden nicht als Teil des Entwicklungsprozesses getestet. Vorkompilierte Binärdateien werden im Rahmen des Release-Prozesses erstellt und auf PyPI veröffentlicht, aber nicht getestet. Sie sind möglicherweise nicht nur mit einer funktionierenden Python-Umgebung installierbar und erfordern möglicherweise einen C/C++-Compiler oder zusätzliche Programme, um Abhängigkeiten aus dem Quellcode als Teil des Installationsprozesses zu erstellen. Die Unterstützung für diese Betriebssysteme erfolgt nur nach bestem Bemühen.

  Tier-3-Betriebssysteme:

  - Linux ppc64le (Distributionen kompatibel mit der [manylinux 2014](https://www.python.org/dev/peps/pep-0599/)-Paketspezifikation)
  - Linux s390x (Distributionen kompatibel mit der [manylinux 2014](https://www.python.org/dev/peps/pep-0599/)-Paketspezifikation)
</details>

Ab Qiskit v2.0.0 werden nur 64-Bit-Plattformen unterstützt und 32-Bit-Plattformen werden nicht unterstützt. Sie können auch nicht aus dem Quellcode auf
32-Bit-Plattformen erstellen, da der Qiskit Rust-Code intern eine 64-Bit-Zeigerbreite annimmt.

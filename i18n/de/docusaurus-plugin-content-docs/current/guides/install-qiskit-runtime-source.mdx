# Qiskit Runtime Client aus dem Quellcode installieren {#install-the-qiskit-runtime-client-from-source}

Wenn du Qiskit Runtime aus dem Quellcode installierst, hast du Zugang zur aktuellen Entwicklungsversion, anstatt die Version aus dem Python Package Index (PyPI) Repository zu verwenden. So kannst du die neueste Version des Codes effizienter untersuchen und erweitern.

## Neue virtuelle Umgebung erstellen und aktivieren {#create-and-activate-a-new-virtual-environment}

1. Navigiere zu deinem Projektverzeichnis und erstelle eine minimale Umgebung, in der nur Python installiert ist.

        <OperatingSystemTabs>
          <TabItem value="mac" label="macOS">
            ```shell
            python3 -m venv .venv
            ```
          </TabItem>

          <TabItem value="linux" label="Linux">
            ```shell
            python3 -m venv .venv
            ```
          </TabItem>

          <TabItem value="win" label="Windows">
            ```text
            python -m venv .venv
            ```
          </TabItem>
         </OperatingSystemTabs>

1. Aktiviere deine neue Umgebung.

         <OperatingSystemTabs>
           <TabItem value="mac" label="macOS">
             ```shell
             source .venv/bin/activate
             ```

           </TabItem>

           <TabItem value="linux" label="Linux">

## Qiskit Runtime installieren {#install-qiskit-runtime}

Folge diesen Schritten, um Qiskit Runtime zu installieren:

1. Klone das Qiskit Runtime Repository.

```bash
git clone https://github.com/Qiskit/qiskit-ibm-runtime.git
```

2. Wechsle in das Verzeichnis `qiskit-ibm-runtime`.

```bash
cd qiskit-ibm-runtime
```

3. Installiere `qiskit-runtime`. Wir empfehlen die Verwendung einer [virtuellen Umgebung](https://docs.python.org/3/library/venv.html), um deine globale Python-Installation nicht zu beeinträchtigen.

  * **Standardinstallation**:

    ```bash
    pip install .
    ```

  * **Bearbeitbarer Modus**: In diesem Modus musst du Qiskit nicht neu installieren, wenn sich der Code des Projekts ändert.

    ```bash
    pip install -e .
    ```

    Im bearbeitbaren Modus werden die kompilierten Erweiterungen im _Debug-Modus_ ohne Optimierungen erstellt.

4. (Optional) Wenn du Tests oder Linting-Prüfungen ausführen möchtest, installiere die Entwickleranforderungen. Wir empfehlen die Verwendung einer [virtuellen Umgebung](https://docs.python.org/3/library/venv.html), um deine globale Python-Installation nicht zu beeinträchtigen.

```bash
pip install -e ".[dev]"
```

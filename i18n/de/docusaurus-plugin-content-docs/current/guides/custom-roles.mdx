# Benutzerdefinierte Rollen einrichten {#set-up-custom-roles}

Wenn Kontoinhaber und Administratoren den Benutzerzugriff für ein Konto einrichten, weisen sie Rollen (z. B. Editor, Viewer, Operator usw.) [Zugriffsrichtlinien und Zugriffsgruppen](/guides/access-groups) zu und weisen dann Benutzer diesen Richtlinien oder Gruppen zu. Ein Benutzer kann alle Aktionen ausführen, die der Rolle oder den Rollen zugeordnet sind, die seiner Zugriffsgruppe oder Richtlinie zugewiesen wurden.

Zusätzlich zu den vorkonfigurierten Rollen, die auf der Seite [Roles](https://cloud.ibm.com/iam/roles) aufgeführt sind, können Kontoinhaber und Administratoren Rollen erstellen, die ihren Anforderungen besser entsprechen. Sie können beispielsweise eine benutzerdefinierte Rolle erstellen, die einem Benutzer Zugriff zum Anzeigen von Kontoverbrauchsanalysen gewährt, ohne anderen Verwaltungszugriff zu gewähren. Der nächste Abschnitt zeigt Schritt für Schritt eine Demonstration dieses Beispiels.

<span id="custom"></span>
## Beispiel: Eine benutzerdefinierte Rolle erstellen, die es Benutzern ermöglicht, Aktionen zur Arbeit mit IBM Quantum-Service-Instanzen auszuführen {#example-create-a-custom-role-that-allows-users-to-perform-actions-to-work-with-ibm-quantum-service-instances}

Diese benutzerdefinierte Rolle gewährt quantenspezifische Rollen an Benutzer.

1. Gehen Sie zu [Manage → IAM → Roles](https://cloud.ibm.com/iam/roles) und klicken Sie auf `Create`.
2. Geben Sie einen Namen, eine ID, eine Beschreibung ein und wählen Sie `Qiskit Runtime` für den Service.
3. Wählen Sie die folgenden Rollen aus und klicken Sie dann auf **Create**.
    * quantum-computing.device.read
    * quantum-computing.job.cancel
    * quantum-computing.job.create
    * quantum-computing.job.read
    * quantum-computing.program.delete
    * quantum-computing.program.read
    * quantum-computing.user.logout
    * Wählen Sie quantum-computing.job.delete, wenn Sie Benutzern erlauben möchten, Jobs zu löschen.

## Beispiel: Eine benutzerdefinierte Rolle erstellen, damit ein Benutzer nur Analysen anzeigen kann {#example-create-a-custom-role-for-a-user-to-view-analytics-only}

1. Gehen Sie zu [Manage → IAM → Roles](https://cloud.ibm.com/iam/roles) in IBM Cloud. Wählen Sie den Qiskit Runtime-Service aus dem Dropdown-Menü, um die Rollen anzuzeigen, die derzeit für diesen Service existieren.

2. Klicken Sie auf die Schaltfläche **Create +**.

3. Geben Sie einen Namen für die benutzerdefinierte Rolle ein, z. B. `Analytics Viewer`.

4. Geben Sie eine ID ein. Sie können für diese ID jede eindeutige Zeichenfolge eingeben, solange sie mit einem Großbuchstaben beginnt und nur alphanumerische Zeichen enthält (keine Leerzeichen) - zum Beispiel `AnalyticsViewer`.

5. Geben Sie eine Beschreibung für die benutzerdefinierte Rolle ein. In diesem Beispiel könnte eine passende Beschreibung lauten: "Als Analytics Viewer können Sie nur Kontoanalysen anzeigen."

6. Wählen Sie im Service-Dropdown-Menü Qiskit Runtime. Es erscheint eine Tabelle, die alle Aktionen enthält, die Sie der Rolle zuordnen können.

7. Suchen Sie für dieses Beispiel nach allen Aktionen, die sich auf Analysen beziehen, wie "Read usage filters for analytics" und "Read usage for analytics". Fügen Sie auch die Aktion "Read account configuration" hinzu.
Um die Aktionen zu Ihrer benutzerdefinierten Rolle hinzuzufügen, klicken Sie am Ende der Zeile der Aktion auf Add.

8. Sobald Sie alle Aktionen hinzugefügt haben, die Sie der Rolle zuordnen möchten, klicken Sie auf Create.

9. Gehen Sie zu [Manage → IAM → Access groups](https://cloud.ibm.com/iam/groups). Klicken Sie auf **Create +**, um eine neue Zugriffsrolle zu erstellen. (Es wird empfohlen, speziell für diese Rolle eine neue Zugriffsgruppe zu erstellen, anstatt die Rolle zu Ihrer Public Access-Gruppe hinzuzufügen, es sei denn, Sie möchten, dass alle Ihre Benutzer den durch Ihre benutzerdefinierte Rolle gewährten Zugriff haben.) Sobald Sie auf **Create +** geklickt und Ihre neue Gruppe benannt haben (oder im Falle der Bearbeitung einer bestehenden Zugriffsgruppe, sobald Sie auf den Namen der zu bearbeitenden Zugriffsgruppe geklickt haben), klicken Sie auf die Registerkarte Access. Klicken Sie auf **Assign access +**.

10. Wählen Sie unter Service Qiskit Runtime aus und klicken Sie dann auf Next.

11. Wählen Sie unter Resources All resources aus und klicken Sie dann auf Next. (Beachten Sie, dass die Analytics Viewer-Rolle keine Analysen von allen Instanzen sehen kann, wenn Sie den Zugriff nicht auf alle Ressourcen beschränken.)

12. Aktivieren Sie unter Roles and actions das Kontrollkästchen Viewer (unter Platform) und das Kontrollkästchen Analytics Viewer (unter Custom access) und klicken Sie dann auf Next.

13. Klicken Sie auf **Add >** und dann auf Assign. Sie haben jetzt eine Richtlinie erstellt, die Ihre benutzerdefinierte Rolle sowie die Viewer-Rolle enthält.

14. Zeigen Sie die Zugriffsgruppe an und klicken Sie auf die Registerkarte Users, um dieser Gruppe einen Benutzer hinzuzufügen. Dieser Benutzer kann nun die Aktionen in Ihrer benutzerdefinierten Rolle ausführen (sowie die anderen Rollen, die Sie der Richtlinie zugewiesen haben).

Weitere Informationen finden Sie in den Themen [Creating custom roles](https://cloud.ibm.com/docs/account?topic=account-custom-roles) und [What are IAM policies and who can assign them?](https://cloud.ibm.com/docs/account?topic=account-iamusermanpol).

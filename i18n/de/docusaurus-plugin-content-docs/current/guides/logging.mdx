# Logging für Qiskit Runtime {#logging-for-qiskit-runtime}

IBM Cloud&reg;-Dienste wie Qiskit Runtime erzeugen Plattform-Logs, die du nutzen kannst, um ungewöhnliche Aktivitäten und kritische Aktionen in deinem Konto zu untersuchen und Probleme zu beheben.

Du kannst [IBM Cloud Logs Routing](https://cloud.ibm.com/docs/logs-router?topic=logs-router-about) verwenden – einen Plattformdienst –, um Plattform-Logs in deinem Konto an ein Ziel deiner Wahl weiterzuleiten. Dazu konfigurierst du einen Tenant, der festlegt, wohin die Plattform-Logs gesendet werden. Weitere Informationen findest du unter [About Logs Routing.](https://cloud.ibm.com/docs/logs-router?topic=logs-router-about)

Du kannst [IBM Cloud Logs](https://cloud.ibm.com/docs/cloud-logs) nutzen, um Plattform-Logs, die in deinem Konto erzeugt und von IBM Cloud Logs Routing an eine IBM Cloud Logs-Instanz weitergeleitet werden, zu visualisieren und darauf basierend Benachrichtigungen einzurichten.

## Standorte, an denen Plattform-Logs erzeugt werden {#locations-where-platform-logs-are-generated}

IBM Cloud Container Registry erzeugt Plattform-Logs, die in deinen Logging-Instanzen angezeigt werden. Die folgende Tabelle listet die Standorte auf, an denen die automatische Erfassung von Container Registry-Dienst-Logs aktiviert ist.

| Standort                     | Plattformdienst-Logs verfügbar? |
|------------------------------|----------------------------------|
| Washington, D.C. (us-east)   |               Ja                 |
| Frankfurt (eu-de)            |               Ja                 |

## Erzeugte Plattform-Logs {#platform-logs-that-are-generated}

IBM Qiskit Runtime erzeugt Plattformdienst-Logs, die in deinen Logging-Instanzen für die folgenden spezifischen Fälle angezeigt werden:

- Wenn sich der Status eines Jobs ändert.

## Logs über die Observability-Seite anzeigen {#viewing-logs-from-the-observability-page}

Informationen zum Starten der IBM Cloud Logs-Oberfläche findest du unter **Launching the UI** in der [IBM Cloud Logs-Dokumentation.](https://cloud.ibm.com/docs/cloud-logs?topic=cloud-logs-instance-launch)

## Felder nach Log-Typ {#fields-by-log-type}

Informationen zu Feldern, die in jedem Plattform-Log enthalten sind, findest du unter [Fields that are included in platform logs.](https://cloud.ibm.com/docs/logs-router?topic=logs-router-about-platform-logs#about-platform-logs-2)

Die folgenden Felder sind im Log-Datensatz enthalten.

| Feld            | Typ        | Beschreibung                                                                         |
|-----------------|------------|--------------------------------------------------------------------------------------|
| logSourceCRN    | Erforderlich | Definiert das Konto und die Flow-Log-Instanz, in der das Log veröffentlicht wird.   |
| saveServiceCopy | Erforderlich | Legt fest, ob IBM eine Kopie des Datensatzes für betriebliche Zwecke speichert.     |
| message         | Erforderlich | Beschreibung des erzeugten Logs.                                                     |

# Kosten im Pay-As-You-Go-Plan verwalten {#manage-cost-on-the-pay-as-you-go-plan}

Diese Anleitung beschreibt, wie du ein Kostenlimit für deine Pay-As-You-Go-Plan-Instanz festlegst. Du kannst Kosten auch verwalten, indem du die [QPU-Nutzung einer Arbeitslast minimierst](/guides/minimize-time) und eine [maximale Ausführungszeit](/guides/max-execution-time) für deine Arbeitslasten festlegst.

Das _Kostenlimit_ bezieht sich auf die Gesamtkosten aller mit dieser Instanz seit ihrer Erstellung ausgeführten Arbeitslasten und ist immer größer oder gleich den Gesamtkosten. Sobald die Instanz die angegebene Gesamtzahl an Sekunden erreicht, können keine weiteren Arbeitslasten ausgeführt werden und es entstehen keine weiteren Kosten.

    <Admonition type="note">
      Das Kostenlimit wird immer in US-Dollar (USD) angegeben und dann in Laufzeitsekunden umgerechnet. Für monatliche Abrechnungszwecke wird dir jedoch die in deinem IBM Cloud&reg;-Konto angegebene Landeswährung berechnet. Da Wechselkurse schwanken können, können die Kosten für X Laufzeitsekunden beim erstmaligen Berechnen in USD von den Kosten abweichen, die dir tatsächlich in deiner Landeswährung berechnet werden. Wenn deine Landeswährung nicht USD ist, kann der Gesamtbetrag, der für die in diesem Feld angegebene Anzahl von Sekunden berechnet wird, daher vom angegebenen Dollarbetrag abweichen.
    </Admonition>

<span id="cost-limit"></span>
## Kostenlimit festlegen oder ändern {#set-or-change-a-cost-limit}

Es gibt mehrere Möglichkeiten, ein Limit für die Ausgaben festzulegen oder zu ändern:

* **IBM Quantum Platform**: Öffne die Seite [Instances](https://quantum.cloud.ibm.com/instances). Suche die Instanz, die du begrenzen möchtest, klicke auf das Aktionsmenü am Ende der Zeile und dann auf **Edit details**. Lege das Gesamtkostenlimit oder das Nutzungslimit fest und klicke anschließend auf **Save changes**.
   <Admonition type="note">
   Du kannst das Kosten- oder Nutzungslimit auch beim Erstellen einer neuen Instanz im Pay-As-You-Go-Plan festlegen. Wenn du die Pay-As-You-Go-Kachel auswählst, erscheint ein Feld mit der Bezeichnung „Total cost limit (Optional)". In diesem Feld kannst du ein Gesamtkostenlimit eingeben.
   </Admonition>

* **API**: [Lege das Instanzlimit in Sekunden fest](https://docs.quantum.ibm.com/api/qiskit-runtime-rest/tags/instances#tags__instances__operations__replace_instance_data).

   ```api
   PUT /v1/instances/configuration "instance_limit":<seconds>
   ```

* **IBM Cloud CLI**: Eine Anleitung zur Verwendung der CLI findest du unter [Getting started with the IBM Cloud CLI](https://cloud.ibm.com/docs/cli?topic=cli-getting-started).

   ```cli
      ibmcloud resource service-instance-update <instance_CRN>
      -p '{"instance_limit_seconds":"<seconds>"}'
   ```

## Kostenlimit anzeigen {#view-the-cost-limit}

Es gibt mehrere Möglichkeiten, das aktuelle Kostenlimit anzuzeigen:

* **IBM Quantum Platform**: Das Gesamtkostenlimit wird auf der Registerkarte Pay-As-You-Go der Tabelle [Instances](https://quantum.cloud.ibm.com/instances) angezeigt.

* **API**:

   ```api
   GET /v1/instances/configuration
   ```
   Weitere Informationen findest du in der [REST-API-Dokumentation](https://docs.quantum.ibm.com/api/qiskit-runtime-rest/tags/instances).

* **IBM Cloud CLI**: Eine Anleitung zur Verwendung der CLI findest du unter [Getting started with the IBM Cloud CLI](https://cloud.ibm.com/docs/cli?topic=cli-getting-started).

   ```cli
   ibmcloud resource service-instance-update <instance_CRN> -p
   '{"instance_limit_seconds":null}'
   ```

## Kostenlimit entfernen {#remove-a-cost-limit}

Du kannst das Kostenlimit auf verschiedene Arten entfernen:

* **IBM Quantum Platform**: Öffne die Tabelle [Instances](https://quantum.cloud.ibm.com/instances). Suche die Instanz, die du aktualisieren möchtest, klicke auf das Überlaufmenü am Ende der Zeile und dann auf **Edit details**. Lösche das Gesamtkostenlimit oder die Nutzung und klicke anschließend auf **Save changes**.

* **API**: [Setze das Instanzlimit auf `null`](https://docs.quantum.ibm.com/api/qiskit-runtime-rest/tags/instances).

   ```api
   PUT /v1/instances/configuration "instance_limit":null
   ```

* **IBM Cloud CLI**: Eine Anleitung zur Verwendung der CLI findest du unter [Getting started with the IBM Cloud CLI](https://cloud.ibm.com/docs/cli?topic=cli-getting-started).

   ```cli
   ibmcloud resource service-instance-update <instance_CRN> -p
   '{"instance_limit_seconds":null}'
   ```
<Admonition type="note">
Nur Administratoren und Eigentümer können die Instanz bearbeiten. Informationen zur Zuweisung von Zugriffsrechten findest du im Leitfaden [Zugriffsrichtlinien und Zugriffsgruppen erstellen](/guides/access-groups).
</Admonition>

### Was passiert, wenn das Kostenlimit erreicht wird {#what-happens-when-the-cost-limit-is-reached}

Wenn das Kostenlimit der Instanz erreicht wird, wird der aktuell laufende Job gestoppt. Sein Status wird auf „Abgebrochen" mit dem Grund „Zu lange gelaufen" gesetzt. Alle verfügbaren Teilergebnisse werden gespeichert.

Es können keine weiteren Jobs über diese Instanz übermittelt werden, bis das Kostenlimit erhöht wird.

## Wie du siehst, was dir berechnet wird {#how-to-see-what-youre-being-charged}

Du erhältst eine monatliche Rechnung mit Einzelheiten zu deinen Ressourcenkosten. Du kannst jederzeit auf der Seite [IBM Cloud Billing and usage](https://cloud.ibm.com/billing) prüfen, wie viel bereits ausgegeben wurde.

Die Seite [Analytics](https://quantum.cloud.ibm.com/analytics) zeigt die Nutzung nach den von dir gesetzten Filtern an. Außerdem kannst du jederzeit die Kosten pro Instanz oder pro Job auf der Seite [Instances](https://quantum.cloud.ibm.com/instances) ermitteln, wie nachfolgend beschrieben.

### Instanznutzung anzeigen {#view-instance-usage}

Um zu ermitteln, wie viel einer Instanz berechnet wurde, klicke auf der Seite [Instances](https://quantum.cloud.ibm.com/instances) auf die Instanz, um deren Detailseite zu öffnen. Das Feld **Instance usage** zeigt die gesamte QPU-Nutzung dieser Instanz seit ihrer Erstellung an.

Deinen Abrechnungszeitraum kannst du auf der [Billing and usage page](https://cloud.ibm.com/billing) einsehen.

### Arbeitslast-Nutzung anzeigen {#view-workload-usage}

Um zu ermitteln, wie viel jeder mit einer Instanz verknüpften Arbeitslast berechnet wurde, klicke auf der Seite [Instances](https://quantum.cloud.ibm.com/instances) auf die Instanz, um deren Detailseite zu öffnen. Klicke dann auf den Wert unter **Total workloads (all types)**.

Die Spalte **Usage** zeigt die von dem Job verbrauchte QPU-Menge an.

## Ausgabenbenachrichtigungen einrichten {#set-up-spending-notifications}

Du kannst Ausgabenbenachrichtigungen einrichten, um benachrichtigt zu werden, wenn dein Konto oder ein bestimmter Dienst einen von dir festgelegten Ausgabenschwellenwert erreicht. Weitere Informationen findest du auf der Seite [IBM Cloud Account types](https://cloud.ibm.com/docs/account?topic=account-accounts). IBM Cloud-Ausgabenbenachrichtigungen müssen aus mehreren Gründen in Verbindung mit anderen Kostenverwaltungsmethoden verwendet werden:

*   Die Benachrichtigungen werden erst ausgelöst, *nachdem* die Kosten das angegebene Limit überschritten haben.
*   Die Kosten werden stündlich an das Abrechnungssystem übermittelt. Daher kann es zu einer erheblichen Verzögerung zwischen der Job-Übermittlung und dem Versand der Ausgabenbenachrichtigung kommen.
*   Das Abrechnungssystem kann mehrere Tage benötigen, um Informationen an das Rechnungsstellungssystem zu übermitteln, was zu weiteren Verzögerungen bei den Benachrichtigungen führen kann. Weitere Informationen zur Funktionsweise des IBM Cloud-Abrechnungssystems findest du unter [Setting spending notifications](https://cloud.ibm.com/docs/billing-usage?topic=billing-usage-spending).

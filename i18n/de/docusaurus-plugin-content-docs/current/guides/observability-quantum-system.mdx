# Aktivitätsverfolgungsereignisse für den IBM Quantum System-Dienst {#activity-tracking-events-for-the-ibm-quantum-system-service}

IBM Cloud&reg;-Dienste, wie der IBM&reg; Quantum System-Dienst, erzeugen Aktivitätsverfolgungsereignisse.

Aktivitätsverfolgungsereignisse berichten über Aktivitäten, die den Zustand eines Dienstes in IBM Cloud ändern. Du kannst die Ereignisse nutzen, um ungewöhnliche Aktivitäten und kritische Aktionen zu untersuchen sowie regulatorische Prüfanforderungen zu erfüllen.

Du kannst IBM Cloud Activity Tracker Event Routing, einen Plattformdienst, verwenden, um Prüfereignisse in deinem Konto an Ziele deiner Wahl weiterzuleiten, indem du Ziele und Routen konfigurierst, die festlegen, wohin Aktivitätsverfolgungsereignisse gesendet werden. Weitere Informationen findest du unter [About IBM Cloud Activity Tracker Events Routing](https://cloud.ibm.com/docs/atracker?topic=atracker-about).

Du kannst IBM Cloud Logs verwenden, um Ereignisse zu visualisieren und Benachrichtigungen zu erhalten, die in deinem Konto erzeugt und von IBM Cloud Activity Tracker Event Routing an eine IBM Cloud Logs-Instanz weitergeleitet werden.

## Aktivitätsverfolgungsereignisse für den IBM Quantum System-Dienst anzeigen {#viewing-activity-tracking-events-for-the-ibm-quantum-system-service}

Du kannst IBM Cloud Logs verwenden, um Ereignisse zu visualisieren und Benachrichtigungen zu erhalten, die in deinem Konto erzeugt und von IBM Cloud Activity Tracker Event Routing an eine IBM Cloud Logs-Instanz weitergeleitet werden.

### IBM Cloud Logs über die Observability-Seite starten {#launching-ibm-cloud-logs-from-the-observability-page}

Informationen zum Starten der IBM Cloud Logs-Benutzeroberfläche findest du unter [Launching the UI](https://cloud.ibm.com/docs/cloud-logs?topic=cloud-logs-instance-launch#instance-launch-cloud-ui) in der IBM Cloud Logs-Dokumentation.

| Aktion | Ausgelöst durch |
| -- | -- |
| `quantum-computing.direct-access-backend-configuration.read` | Backend-Konfiguration abrufen (`GET /v1/backends/{backend-name}/configuration`) |
| `quantum-computing.direct-access-backend-properties.read` | Backend-Eigenschaften abrufen (`GET /v1/backends/{backend-name}/properties`) |
| `quantum-computing.direct-access-backend.list` | Liste der Backends abrufen (`GET /v1/backends`) |
| `quantum-computing.direct-access-backend.read` | Backend-Details abrufen (`GET /v1/backends/{backend-name}`) |
| `quantum-computing.direct-access-job.cancel` | Einen Job abbrechen (`POST /v1/jobs/{job-id}/cancel`) |
| `quantum-computing.direct-access-job.create` | Einen Job ausführen (`POST /v1/jobs`) |
| `quantum-computing.direct-access-job.delete` | Einen Job löschen (`DELETE /v1/jobs/{job-id}`) |
| `quantum-computing.direct-access-job.list` | Jobs abrufen (`GET /v1/jobs`) |
| `quantum-computing.direct-access-job.read` | Job nach ID abrufen (`GET /v1/jobs/{job-id}`) |
| `quantum-computing.direct-access-lane-configuration.list` | Backend-Lane-Konfiguration abrufen (`GET /v1/backends/{backend-name}/lanes`) |

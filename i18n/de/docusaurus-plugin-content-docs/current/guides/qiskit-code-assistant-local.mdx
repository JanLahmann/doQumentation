# Qiskit Code Assistant im lokalen Modus verwenden {#use-qiskit-code-assistant-in-local-mode}

Erfahre, wie du eines der Qiskit Code Assistant-Modelle auf deinem lokalen Rechner installierst, konfigurierst und verwendest.

<Admonition type="note" title="Hinweise">
    - Qiskit Code Assistant befindet sich im Vorschau-Releasestatus und kann sich noch ändern.
    - Wenn du Feedback hast oder das Entwicklerteam kontaktieren möchtest, nutze den [Qiskit Slack Workspace-Kanal](https://qiskit.enterprise.slack.com/archives/C07LYA6PL83) oder die entsprechenden öffentlichen GitHub-Repositories.
</Admonition>

## Schnellstart (empfohlen) {#quick-start-recommended}

Der einfachste Weg, mit dem Qiskit Code Assistant im lokalen Modus zu beginnen, ist die Verwendung der automatisierten Einrichtungsskripte für die VS Code- oder JupyterLab-Erweiterung. Diese Skripte installieren automatisch [Ollama](https://ollama.com/), um die LLMs auszuführen, laden das empfohlene Modell herunter und konfigurieren die Erweiterung für dich.

### Einrichtung der VS Code-Erweiterung {#vs-code-extension-setup}

Führe den folgenden Befehl in deinem Terminal aus:

```bash
bash <(curl -fsSL https://raw.githubusercontent.com/Qiskit/qiskit-code-assistant-vscode/main/setup_local.sh)
```

Dieses Skript führt die folgenden Schritte aus:
- Installiert Ollama (falls noch nicht installiert)
- Lädt das empfohlene Qiskit Code Assistant-Modell herunter und konfiguriert es
- Richtet die VS Code-Erweiterung für die Verwendung mit deiner lokalen Bereitstellung ein

### Einrichtung der JupyterLab-Erweiterung {#jupyterlab-extension-setup}

Führe den folgenden Befehl in deinem Terminal aus:

```bash
bash <(curl -fsSL https://raw.githubusercontent.com/Qiskit/qiskit-code-assistant-jupyterlab/main/setup_local.sh)
```

Dieses Skript wird:
- Ollama installieren (falls noch nicht installiert)
- Das empfohlene Qiskit Code Assistant-Modell herunterladen und konfigurieren
- Die JupyterLab-Erweiterung für die Verwendung mit deiner lokalen Bereitstellung einrichten

<summary>Qiskit Code Assistant-Modelle lokal manuell über Ollama bereitstellen</summary>

Es gibt mehrere Möglichkeiten, das heruntergeladene Qiskit Code Assistant-Modell bereitzustellen und damit zu interagieren. Diese Anleitung zeigt die Verwendung von [Ollama](https://ollama.com) auf folgende Weise: entweder mit der [Ollama-Anwendung](#using-the-ollama-application) über die Hugging Face Hub-Integration oder ein lokales Modell, oder mit dem [`llama-cpp-python`-Paket](#use-llama-cpp-python).

### Die Ollama-Anwendung verwenden {#using-the-ollama-application}

Die Ollama-Anwendung bietet eine einfache Lösung, um LLMs lokal auszuführen. Sie ist benutzerfreundlich, mit einer CLI, die den gesamten Einrichtungsprozess, die Modellverwaltung und die Interaktion recht unkompliziert gestaltet. Sie eignet sich ideal für schnelle Experimente und für Benutzer, die sich mit wenigen technischen Details befassen möchten.

#### Ollama installieren {#install-ollama}

1. Lade die [Ollama-Anwendung](https://ollama.com/download) herunter
1. Installiere die heruntergeladene Datei
1. Starte die installierte Ollama-Anwendung

    <Admonition type="info">Die Anwendung läuft erfolgreich, wenn das Ollama-Symbol in der Desktop-Menüleiste erscheint. Du kannst auch überprüfen, ob der Dienst läuft, indem du `http://localhost:11434/` aufrufst.</Admonition>

1. Probiere Ollama in deinem Terminal aus und starte das Ausführen von Modellen. Zum Beispiel:

    ```
    ollama run hf.co/Qiskit/Qwen2.5-Coder-14B-Qiskit
    ```

#### Ollama mit der Hugging Face Hub-Integration einrichten {#set-up-ollama-using-the-hugging-face-hub-integration}

Die [Ollama/Hugging Face Hub-Integration](https://huggingface.co/docs/hub/ollama) bietet eine Möglichkeit, mit auf dem Hugging Face Hub gehosteten Modellen zu interagieren, ohne eine neue Modelfile erstellen oder die GGUF- oder Safetensors-Dateien manuell herunterladen zu müssen. Die Standard-`template`- und `params`-Dateien sind für das Modell auf dem Hugging Face Hub bereits enthalten.

1. Stelle sicher, dass die Ollama-Anwendung läuft.

1. Gehe zur gewünschten Modellseite und kopiere die URL. Zum Beispiel: https://huggingface.co/Qiskit/Qwen2.5-Coder-14B-Qiskit-GGUF.

1. Führe den folgenden Befehl in deinem Terminal aus:

    ```
    ollama run hf.co/Qiskit/Qwen2.5-Coder-14B-Qiskit
    ```

Du kannst das Modell `hf.co/Qiskit/Qwen2.5-Coder-14B-Qiskit` oder eines der anderen derzeit empfohlenen offiziellen GGUF-Modelle `hf.co/Qiskit/mistral-small-3.2-24b-qiskit-GGUF` oder `hf.co/Qiskit/granite-3.3-8b-qiskit-GGUF` verwenden.

#### Ollama mit einem manuell heruntergeladenen Qiskit Code Assistant GGUF-Modell einrichten {#set-up-ollama-with-a-manually-downloaded-qiskit-code-assistant-gguf-model}

Wenn du ein GGUF-Modell wie https://huggingface.co/Qiskit/Qwen2.5-Coder-14B-Qiskit-GGUF manuell heruntergeladen hast und mit verschiedenen Templates und Parametern experimentieren möchtest, kannst du diese Schritte befolgen, um es in deine lokale Ollama-Anwendung zu laden.

1. Erstelle eine `Modelfile` mit folgendem Inhalt und aktualisiere `<PATH-TO-GGUF-FILE>` mit dem tatsächlichen Pfad deines heruntergeladenen Modells.

    ```
    FROM <PATH-TO-GGUF-FILE>
    TEMPLATE """{{ if .System }}
    System:
    {{ .System }}

    {{ end }}{{ if .Prompt }}Question:
    {{ .Prompt }}

    {{ end }}Answer:
    ```python{{ .Response }}
    """

    PARAMETER stop "Question:"
    PARAMETER stop "Answer:"
    PARAMETER stop "System:"
    PARAMETER stop "```"

    PARAMETER temperature 0
    PARAMETER top_k 1
    ```

1. Führe den folgenden Befehl aus, um eine benutzerdefinierte Modellinstanz basierend auf der `Modelfile` zu erstellen.

    ```
    ollama create Qwen2.5-Coder-14B-Qiskit -f ./path-to-model-file
    ```

    <Admonition type="note">Dieser Vorgang kann einige Zeit in Anspruch nehmen, da Ollama die Modelldatei liest, die Modellinstanz initialisiert und sie gemäß den angegebenen Spezifikationen konfiguriert.</Admonition>

#### Das manuell in Ollama heruntergeladene Qiskit Code Assistant-Modell ausführen {#run-the-qiskit-code-assistant-model-manually-downloaded-in-ollama}

Nachdem das `Qwen2.5-Coder-14B-Qiskit`-Modell in Ollama eingerichtet wurde, führe den folgenden Befehl aus, um das Modell zu starten und im Terminal damit zu interagieren (im Chat-Modus).

```
ollama run Qwen2.5-Coder-14B-Qiskit
```

Einige nützliche Befehle:

- `ollama list` – Modelle auf deinem Computer auflisten
- `ollama rm Qwen2.5-Coder-14B-Qiskit` – Das Modell löschen
- `ollama show Qwen2.5-Coder-14B-Qiskit` – Modellinformationen anzeigen
- `ollama stop Qwen2.5-Coder-14B-Qiskit` – Ein aktuell laufendes Modell stoppen
- `ollama ps` – Auflisten, welche Modelle aktuell geladen sind

<summary>Qiskit Code Assistant-Modelle lokal manuell über das `llama-cpp-python`-Paket bereitstellen</summary>

<span id="use-llama-cpp-python"></span>
Eine Alternative zur Ollama-Anwendung ist das `llama-cpp-python`-Paket, eine Python-Bindung für `llama.cpp`. Es bietet dir mehr Kontrolle und Flexibilität beim lokalen Ausführen des GGUF-Modells und ist ideal für Benutzer, die das lokale Modell in ihre Workflows und Python-Anwendungen integrieren möchten.

1. Installiere [`llama-cpp-python`](https://pypi.org/project/llama-cpp-python/)
2. Interagiere mit dem Modell aus deiner Anwendung heraus mit `llama_cpp`. Zum Beispiel:

```python
from llama_cpp import Llama

model_path = <PATH-TO-GGUF-FILE>

model = Llama(
        model_path,
        seed=17,
        n_ctx=10000,
        n_gpu_layers=37, # to offload in gpu, but put 0 if all in cpu
    )

input = 'Generate a quantum circuit with 2 qubits'
raw_pred = model(input)["choices"][0]["text"]
```

Du kannst dem Modell auch Textgenerierungsparameter hinzufügen, um die Inferenz anzupassen:

```python
generation_kwargs = {
        "max_tokens": 512,
        "echo": False, # Echo the prompt in the output
        "top_k": 1
    }

raw_pred = model(input, **generation_kwargs)["choices"][0]["text"]
```

</details>

### Die `llama.cpp`-Bibliothek verwenden {#use-the-llamacpp-library}

Eine weitere Alternative ist die Verwendung von `llama.cpp`, einer Open-Source-Bibliothek zur Durchführung von LLM-Inferenz auf einer CPU mit minimalem Einrichtungsaufwand.
Sie bietet Low-Level-Kontrolle über die Modellausführung und wird typischerweise über die Befehlszeile mit Verweis auf eine lokale GGUF-Modelldatei ausgeführt.

Es gibt mehrere Möglichkeiten, `llama.cpp` auf deinem Rechner zu installieren:

- llama.cpp mit [brew, nix oder winget](https://github.com/ggml-org/llama.cpp/blob/master/docs/install.md) installieren
- Mit Docker ausführen: Sieh dir die [Docker-Dokumentation des `llama.cpp`-Teams](https://github.com/ggml-org/llama.cpp/blob/master/docs/docker.md) an
- Vorkompilierte Binärdateien von der [Releases-Seite](https://github.com/ggml-org/llama.cpp/releases) herunterladen
- Aus dem Quellcode kompilieren durch [Klonen dieses Repositories](https://github.com/ggml-org/llama.cpp/blob/master/docs/build.md)

Nach der Installation kannst du `llama.cpp` verwenden, um im Konversationsmodus mit GGUF-Modellen zu interagieren:
```bash
# Use a local model file
llama-cli -m my_model.gguf -cnv

# Or download and run a model directly from Hugging Face
llama-cli -hf Qiskit/Qwen2.5-Coder-14B-Qiskit-GGUF -cnv
```

Du kannst auch einen OpenAI-kompatiblen API-Server für das Modell auf folgende Weise starten:
```bash
llama-server -hf Qiskit/Qwen2.5-Coder-14B-Qiskit-GGUF
```

#### Erweiterte Parameter {#advanced-parameters}
Mit dem `llama-cli`-Programm kannst du die Modellgenerierung über Befehlszeilenoptionen steuern. Zum Beispiel kannst du einen initialen „System"-Prompt mit dem Flag `-p/--prompt` angeben. Im Konversationsmodus (`-cnv`) fungiert dieser initiale Prompt als Systemnachricht. Andernfalls kannst du einfach eine gewünschte Anweisung deinem Prompttext voranstellen. Du kannst auch Sampling-Parameter anpassen – zum Beispiel: Temperatur (`--temp`), Top-k (`--top-k`), Top-p (`--top-p`), Wiederholungsstrafe (`--repeat-penalty`) und den zu verwendenden Seed (`--seed`). Das Folgende ist ein Beispielaufruf mit diesen Optionen:

```bash
llama-cli -hf Qiskit/Qwen2.5-Coder-14B-Qiskit-GGUF \
  -p "You are a friendly assistant." -cnv \
  --temp 0.7 \
  --top-k 50 \
  --top-p 0.95 \
  --repeat-penalty 1.1 \
  --seed 42
```

Um die ordnungsgemäße Funktionsweise unserer Qiskit-Modelle sicherzustellen, empfehlen wir die Verwendung des System-Prompts aus unseren HF-GGUF-Repositories: System-Prompt für [mistral-small-3.2-24b-qiskit-GGUF](https://huggingface.co/Qiskit/mistral-small-3.2-24b-qiskit-GGUF/blob/main/system), [Qwen2.5-Coder-14B-Qiskit-GGUF](https://huggingface.co/Qiskit/Qwen2.5-Coder-14B-Qiskit-GGUF/blob/main/system), [granite-3.3-8b-qiskit-GGUF](https://huggingface.co/Qiskit/granite-3.3-8b-qiskit-GGUF/blob/main/system) und [granite-3.2-8b-qiskit-GGUF.](https://huggingface.co/Qiskit/granite-3.2-8b-qiskit-GGUF/blob/main/system)

<summary>Erweiterungen manuell mit der lokalen Bereitstellung verbinden</summary>

Verwende die VS Code-Erweiterung und die JupyterLab-Erweiterung für den Qiskit Code Assistant, um das lokal bereitgestellte Qiskit Code Assistant-Modell zu verwenden. Sobald du die Ollama-Anwendung [mit dem Modell eingerichtet hast](#using-the-ollama-application), kannst du die Erweiterungen so konfigurieren, dass sie eine Verbindung zum lokalen Dienst herstellen.

### Mit der Qiskit Code Assistant VS Code-Erweiterung verbinden {#connect-with-the-qiskit-code-assistant-vs-code-extension}

Mit der Qiskit Code Assistant VS Code-Erweiterung kannst du mit dem Modell interagieren und während du deinen Code schreibst Code-Vervollständigung durchführen. Dies eignet sich gut für Benutzer, die beim Schreiben von Qiskit-Code für ihre Python-Anwendungen Unterstützung suchen.

1. Installiere die [Qiskit Code Assistant VS Code-Erweiterung](/guides/qiskit-code-assistant-vscode).
1. Gehe in VS Code zu den **Benutzereinstellungen** und setze die **Qiskit Code Assistant: Url** auf die URL deiner lokalen Ollama-Bereitstellung (zum Beispiel `http://localhost:11434`).
1. Lade VS Code neu, indem du zu **Ansicht > Befehlspalette...** gehst und **Entwickler: Fenster neu laden** auswählst.

Das in Ollama konfigurierte Qiskit Code Assistant-Modell sollte in der Statusleiste erscheinen und ist dann einsatzbereit.

### Mit der Qiskit Code Assistant JupyterLab-Erweiterung verbinden {#connect-with-the-qiskit-code-assistant-jupyterlab-extension}

Mit der Qiskit Code Assistant JupyterLab-Erweiterung kannst du mit dem Modell interagieren und Code-Vervollständigung direkt in deinem Jupyter Notebook durchführen. Benutzer, die vorwiegend mit Jupyter Notebooks arbeiten, können diese Erweiterung nutzen, um ihre Erfahrung beim Schreiben von Qiskit-Code weiter zu verbessern.

1. Installiere die [Qiskit Code Assistant JupyterLab-Erweiterung](/guides/qiskit-code-assistant-jupyterlab).
1. Gehe in JupyterLab zum **Einstellungs-Editor** und setze die **Qiskit Code Assistant Service API** auf die URL deiner lokalen Ollama-Bereitstellung (zum Beispiel `http://localhost:11434`).

Das in Ollama konfigurierte Qiskit Code Assistant-Modell sollte in der Statusleiste erscheinen und ist dann einsatzbereit.

</details>

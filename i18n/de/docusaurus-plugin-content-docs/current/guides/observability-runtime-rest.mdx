# Aktivitätsverfolgungsereignisse für Qiskit Runtime {#activity-tracking-events-for-qiskit-runtime}

IBM Cloud&reg;-Dienste wie Qiskit Runtime erzeugen Aktivitätsverfolgungsereignisse.

Aktivitätsverfolgungsereignisse berichten über Aktivitäten, die den Zustand eines Dienstes in IBM Cloud ändern. Du kannst die Ereignisse nutzen, um ungewöhnliche Aktivitäten und kritische Aktionen zu untersuchen und um gesetzliche Prüfanforderungen zu erfüllen.

Du kannst IBM Cloud Activity Tracker Event Routing, einen Plattformdienst, verwenden, um Prüfereignisse in deinem Konto an Ziele deiner Wahl weiterzuleiten, indem du Ziele und Routen konfigurierst, die festlegen, wohin Aktivitätsverfolgungsereignisse gesendet werden. Weitere Informationen findest du unter [About IBM Cloud Activity Tracker Events Routing.](https://cloud.ibm.com/docs/atracker?topic=atracker-about)

Du kannst IBM Cloud Logs verwenden, um Ereignisse zu visualisieren und Benachrichtigungen dafür einzurichten, die in deinem Konto erzeugt und von IBM Cloud Activity Tracker Event Routing an eine IBM Cloud Logs-Instanz weitergeleitet werden.

## Standorte, an denen Aktivitätsverfolgungsereignisse erzeugt werden {#locations-where-activity-tracking-events-are-generated}

Qiskit Runtime sendet Aktivitätsverfolgungsereignisse in folgenden Regionen:

- Washington ("us-east")
- Frankfurt ("eu-de")

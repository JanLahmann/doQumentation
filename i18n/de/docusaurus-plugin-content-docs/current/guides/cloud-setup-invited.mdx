# Richten Sie Ihr IBM Cloud-Konto ein - eingeladene Benutzer

Wenn Sie keine Einladung von einem Administrator erhalten haben, um einem Konto beizutreten, das ein Administrator eingerichtet hat, kontaktieren Sie Ihren Administrator. Wenn Sie Ihr eigenes Konto einrichten müssen, folgen Sie den Schritten in [Richten Sie Ihr IBM Cloud-Konto ein](/guides/cloud-setup).

Sie können Workloads auf IBM&reg; Quantenprozessoren (QPUs) ausführen, indem Sie ein Konto auf IBM Cloud&reg; einrichten. Sie müssen auch [Qiskit und Qiskit Runtime installieren und einrichten](/guides/install-qiskit#local).

Ihr Benutzerkonto ist mit einer oder mehreren [Instanzen](./instances) verknüpft, die durch _Cloud Resource Names_ (CRNs) identifiziert werden, die Zugriff auf IBM Quantum-Dienste gewähren. Zusätzlich wird jedem Konto ein eindeutiger API-Token (auch Schlüssel genannt) zugewiesen, der den IBM Quantum-Zugriff von Qiskit ermöglicht. Anweisungen zur Auswahl einer bestimmten Instanz finden Sie unter [Mit einer Instanz verbinden](./instances#connect-instance).

    <Admonition type="note">
      Das [IBM Quantum Platform-Dashboard](https://quantum.cloud.ibm.com) und die [Instanzen-Seite](https://quantum.cloud.ibm.com/instances) listen die Instanzen auf, auf die Sie zugreifen können.
    </Admonition>

## Einrichtung zur Verwendung von IBM Cloud

### Bevor Sie beginnen

1. Akzeptieren Sie die E-Mail-Einladung.
1. Melden Sie sich mit einem IBMid- oder Google-Konto an. Beachten Sie, dass das Profil, das Sie zur Anmeldung bei IBM Cloud oder IBM Quantum Platform in diesem Schritt verwenden, nicht Ihr IBM Cloud-Konto ist. Bestätigen Sie nach der Anmeldung Ihre Informationen und akzeptieren Sie die Einladung, dem Konto beizutreten.
1. Navigieren Sie zurück zur [IBM Quantum Platform.](https://quantum.cloud.ibm.com)
1. Stellen Sie sicher, dass das richtige Konto und die richtige Region im Kontowechsler im Header ausgewählt sind, wie in der folgenden Abbildung dargestellt.
    * Wählen Sie das Konto aus, das in der E-Mail eingeladen wurde.
    * Die Region steuert, wo Ihre Jobs ausgeführt werden und wo die Job-Daten aufbewahrt werden. Sie können auf beide Regionen mit demselben API-Schlüssel zugreifen, aber Sie können nur die Instanzen sehen und darauf zugreifen, die in der Region erstellt wurden, in der Sie angemeldet sind.

    ![Der IBM Quantum Platform-Header wird angezeigt. Der Kontowechsler befindet sich rechts von der Suchleiste. Der Regionswechsler befindet sich rechts vom Kontowechsler.](/docs/images/guides/cloud-setup/IQP-Header.svg 'IBM Quantum Platform-Header')

### 2. Optional: Speichern Sie Ihre Zugangsdaten

Wenn Sie eine vertrauenswürdige Python-Umgebung verwenden, wie z.B. einen persönlichen Laptop, wird empfohlen, die Methode `save_account()` zu verwenden, um [Ihre Anmeldedaten lokal zu speichern.](/guides/save-credentials)

### 3. Verbinden Sie Qiskit mit Ihrer Dienstinstanz

- [Initialisieren Sie den Dienst](/guides/initialize-account) in einer **vertrauenswürdigen Python-Umgebung**
- [Initialisieren Sie den Dienst in einer **nicht vertrauenswürdigen Umgebung**](/guides/cloud-setup-untrusted) (wie einem öffentlichen Computer)

# Initialisieren Sie Ihr Qiskit Runtime Service-Konto

Bevor Sie Qiskit Runtime verwenden, müssen Sie Ihr Konto initialisieren (instanziieren), indem Sie Anmeldedaten übermitteln (laden). Diese Anmeldedaten können jedes Mal manuell übermittelt werden, wenn Sie den Qiskit Runtime-Dienst initialisieren, oder Sie können sie zur Wiederverwendung speichern.

<Admonition type="note" title="Notes">
      * Wenn Sie einen öffentlichen Computer oder eine andere nicht vertrauenswürdige Umgebung verwenden, folgen Sie stattdessen den Anweisungen unter [Initialize the service in an untrusted environment](/guides/cloud-setup-untrusted).
      * Folgen Sie [diesen Anweisungen](/guides/cloud-setup-rest-api), wenn Sie sich über die REST-API anstelle von Qiskit verbinden möchten.
      * Falls erforderlich, verwenden Sie [diese Informationen](/guides/quickstart-steps-org#firewall), um Ihre Firewall zu konfigurieren und den Zugriff auf die IBM Quantum API-Endpunkte zu ermöglichen.
</Admonition>

## Bevor Sie beginnen

Stellen Sie sicher, dass Sie diese Schritte abgeschlossen haben:

1. Stellen Sie sicher, dass Sie Mitglied eines IBM Cloud-Kontos sind. Anleitungen finden Sie unter [Set up your IBM Cloud account](/guides/cloud-setup).
2. Erstellen Sie (oder haben Sie Zugriff auf) mindestens eine Instanz. Befolgen Sie diese Schritte zur Überprüfung:
    1. Melden Sie sich bei [IBM Quantum Platform](https://quantum.cloud.ibm.com) an.
    1. Stellen Sie sicher, dass das richtige Konto und die richtige Region im Konto-Umschalter in der Kopfzeile ausgewählt sind.
    1. Wenn Sie eine oder mehrere Instanzen angezeigt bekommen, sind Sie mit diesem Schritt fertig. Andernfalls [erstellen Sie eine Instanz.](/guides/instances#add-instance)
1. Stellen Sie sicher, dass Sie in einer aktiven Python-Umgebung mit [installiertem Qiskit SDK und Qiskit Runtime](/guides/install-qiskit#local) arbeiten.
1. Aktivieren Sie die virtuelle Python-Umgebung und führen Sie Python in Ihrer virtuellen Umgebung aus.

<span id="access-credentials"></span>
## Finden Sie Ihre Zugangsanmeldedaten

1. Finden Sie Ihren API-Schlüssel (auch als _API-Token_ bezeichnet). Erstellen Sie vom [Dashboard](https://quantum.cloud.ibm.com/) aus Ihren API-Schlüssel, kopieren Sie ihn dann an einen sicheren Ort, damit Sie ihn zur Authentifizierung verwenden können. Das Token wird nicht erneut sichtbar sein. Beachten Sie, dass Sie einen einzelnen API-Schlüssel verwenden können, um sich mit jeder Region zu verbinden.
2. Optional: Finden Sie die Instanz, die Sie verwenden möchten, auf der Seite [Instances](https://quantum.cloud.ibm.com/instances). Bewegen Sie den Mauszeiger über die CRN, klicken Sie auf das Symbol zum Kopieren und speichern Sie sie an einem sicheren Ort, damit Sie sie zur Identifizierung der Instanz verwenden können.

## Verbinden Sie Qiskit mit Ihrer Qiskit Runtime Service-Instanz

<Admonition type="note">
Diese Anweisungen sind für `qiskit_ibm_runtime` v0.42 oder höher konzipiert. Einige Funktionen sind in früheren Versionen von `qiskit_ibm_runtime` nicht aktiviert. In diesen Fällen sollten Sie immer einen Wert für `channel`, `token` und `instance` angeben, entweder explizit oder über ein gespeichertes Konto.
</Admonition>

Der grundlegende Code zum Verbinden von Qiskit mit Ihrer Qiskit Runtime Service-Instanz folgt. Es gibt jedoch mehrere verschiedene Möglichkeiten, die `QiskitRuntimeService`-Optionen anzupassen, abhängig von Ihren Anforderungen. Diese Optionen werden in den folgenden Abschnitten beschrieben.

```python
from qiskit_ibm_runtime import QiskitRuntimeService

service = QiskitRuntimeService(channel=<channel>,
            token=<your-API_KEY>, # Use the 44-character API_KEY you created and saved from the IBM Quantum Platform Home dashboard
            instance=<instance_CRN>)
```

<Admonition type="note">
Der Standardkanal ist `ibm_quantum_platform`. Da dies fast immer der geeignete Kanal ist, ist er in den Beispielen nicht enthalten.
</Admonition>

<span id="explicit"></span>
### Schnellstartpfad: Anmeldedaten explizit angeben

Der schnellste Weg, eine laufende `QiskitRuntimeService`-Instanz zu erhalten, ist die direkte Instanziierungsmethode: Geben Sie jedes Mal explizit den API-Token (Schlüssel) und die CRN (Instanzkennung) an, wenn Sie den Qiskit Runtime-Dienst initialisieren müssen. Siehe [Finden Sie Ihre Zugangsanmeldedaten](#access-credentials), falls erforderlich.

```python
from qiskit_ibm_runtime import QiskitRuntimeService

service = QiskitRuntimeService(token=<cloud_api_key>, # Use the 44-character API_KEY you created and saved from the IBM Quantum Platform Home dashboard
            instance=<instance_CRN>)
```

Dieser Pfad ist zuverlässig, kann aber mühsam sein, wenn Sie dieselben Details wiederholt laden müssen. Um das mehrfache Laden Ihrer Anmeldedaten zu vermeiden, können Sie gespeicherte Konto-Anmeldedaten verwenden, wenn Sie in einer vertrauenswürdigen Python-Umgebung arbeiten (z. B. auf einem persönlichen Laptop oder einer Arbeitsstation), wie im folgenden Abschnitt beschrieben.

Obwohl der Eingabeparameter `instance` optional ist, wird empfohlen, diese Informationen immer anzugeben, es sei denn, Sie möchten einen einzelnen Dienst für die Arbeit mit mehreren Instanzen verwenden. In dieser Situation siehe Abschnitt [automatische Instanzauswahl](#auto-select).

<span id="saved"></span>
### Schnellstartpfad: Gespeicherte Anmeldedaten angeben

Wenn Sie Ihre Anmeldedaten bereits gespeichert haben, verwenden Sie den folgenden Code, um Ihre Standard-Anmeldedaten anzuwenden. Anweisungen zum Speichern von Anmeldedaten finden Sie unter [Save your access credentials.](/guides/save-credentials)

```python
from qiskit_ibm_runtime import QiskitRuntimeService

# run every time you need the service
service = QiskitRuntimeService()
...
```

Wenn Sie einen oder mehrere Sätze von Anmeldedaten benannt haben - zum Beispiel für Open- und Premium-Zugriff - verwenden Sie den folgenden Code, um einen Satz benannter Anmeldedaten zu verwenden.

```python
from qiskit_ibm_runtime import QiskitRuntimeService

# run every time you need the service
service = QiskitRuntimeService(name="<name_of_saved_credentials>")
...
```

#### Überlegungen beim Laden gespeicherter Anmeldedaten

- Wenn Sie einen Dienst mit `token` und `name` initialisieren, wird das `token` ignoriert und die gespeicherten Details aus dem `name`-Konto werden geladen.

    Im folgenden Beispiel lädt der Dienst die Details von `account_A` und verwendet `token_B` nicht:
    ```python
    from qiskit_ibm_runtime import QiskitRuntimeService

    service = QiskitRuntimeService(token="token_B", name="account_A")
    ```
- Wenn Sie beim Initialisieren eines Dienstes einen `name` und eine `instance` angeben, versucht der Dienst, das `name`-Konto **und** eine Verbindung zur angegebenen `instance` zu laden. Bei Konflikten wird eine Warnung ausgegeben.

    Im folgenden Beispiel versucht der Dienst, die Anmeldedaten für `account_A` zu laden und die Instanz `CRN_B` zu verwenden - auch wenn in `account_A` eine andere Instanz angegeben ist:
    ```python
    from qiskit_ibm_runtime import QiskitRuntimeService

    service = QiskitRuntimeService(instance="CRN_B", name="account_A")
    ```
- Der Dienst versucht, das standardmäßig gespeicherte Konto nur zu laden, wenn Sie beim Initialisieren eines Dienstes kein `token` oder `name` angeben. Wenn jedoch eine `instance` explizit angegeben wird, versucht der Dienst, sich mit dieser Instanz unter Verwendung der Standard-Anmeldedaten zu verbinden. Bei Konflikten wird eine Warnung ausgegeben.

    Im folgenden Beispiel versucht der Dienst, die Standard-Anmeldedaten zu laden und die Instanz `CRN_B` zu verwenden - auch wenn im Standardkonto eine andere Instanz angegeben ist:
    ```python
    from qiskit_ibm_runtime import QiskitRuntimeService

    service = QiskitRuntimeService(instance="CRN_B")
    ```
- Wenn Sie mehrere Sätze von Anmeldedaten speichern, aber beim Initialisieren des Dienstes keinen angeben und kein standardmäßig gespeichertes Konto vorhanden ist, wird derjenige verwendet, dessen Name alphabetisch zuletzt kommt.

    Im folgenden Beispiel hat der Benutzer Konten namens "my_premium" und "my_open" gespeichert, aber keines als Standard markiert. Anschließend haben sie den Dienst mit folgendem Code initialisiert. In diesem Fall werden die "my_premium"-Anmeldedaten verwendet:
    ```python
    from qiskit_ibm_runtime import QiskitRuntimeService

    service = QiskitRuntimeService()
    ```
- Wenn Sie dauerhaft den Fehler "401 Unauthorized" erhalten, versuchen Sie möglicherweise, ein [Bearer-Token](/guides/cloud-setup-untrusted#authenticate-to-the-rest-api-in-an-untrusted-environment) anstelle Ihres API-Schlüssels zu verwenden.

<span id="auto-select"></span>
### Automatische Auswahl der Instanz

Wenn Sie beim Instanziieren des Dienstes ein Token angeben, aber **keine Instanz-CRN angeben**, authentifiziert sich `QiskitRuntimeService` beim durch das Token identifizierten Konto und verwendet die automatische Instanzauswahl, um die relevanteste Instanz für die angeforderte Aufgabe auszuwählen. Wenn Sie mehrere Instanzen in Ihrem Konto verfügbar haben, wählt der Dienst automatisch aus den verfügbaren Instanzen aus, abhängig von der angeforderten Ressource und diesen `QiskitRuntimeService`-Optionen (falls gesetzt): `plans_preference`, `region`, `tags`.

- **`plans_preference`**: Die Arten von Instanzplänen, die priorisiert werden sollen. Wenn z. B. [`open`] übergeben wird, sind nur Open Plan-Instanzen verfügbar. Akzeptierte Werte sind `open`, `premium`, `pay-as-you-go`, `flex` und `on-prem`. Wenn `plans_preference` nicht angegeben wird, werden kostenlose Pläne gegenüber kostenpflichtigen Plänen priorisiert.
- **`region`**: Die Instanzregion. Akzeptierte Werte sind `us-east` und `eu-de`.
- **`tags`**: Die Instanz-Tags. Akzeptiert eine Liste von Tag-Namen-Strings.

<Admonition type="note">
Sie können Ihre [gespeicherten Anmeldedaten](/guides/save-credentials) so einrichten, dass automatisch eine Instanz ausgewählt wird.
</Admonition>

Die Instanz wird in dieser Reihenfolge gefunden und verwendet:

1. Wenn Ihr Konto nur Zugriff auf eine Instanz hat, wird sie standardmäßig ausgewählt.
1. Wenn gespeicherte Anmeldedaten verwendet werden und eine Instanz mit den Anmeldedaten angegeben wurde, wird diese Instanz verwendet. Siehe [Überlegungen beim Laden gespeicherter Anmeldedaten.](#considerations-when-loading-saved-credentials)
3. Wenn Ihr Konto mehrere Instanzen hat, die auf die angeforderte QPU zugreifen können, verwendet das System Ihre angegebenen Planpräferenzen, um einen Plan und eine Instanz auszuwählen. Für Qiskit Runtime v0.42 und höher werden kostenlose Pläne standardmäßig priorisiert.
2. Wenn Ihr Konto Zugriff auf mehrere Instanzen hat, aber nur eine auf die angeforderte QPU zugreifen kann, wird die Instanz mit Zugriff ausgewählt. Wenn diese Instanz nicht mit einem kostenlosen Plan verknüpft ist, fallen Kosten an.

#### Beispiele

In diesem Beispiel durchsucht der Dienst alle dem Konto verfügbaren Instanzen, um diejenigen zu finden, die auf das angegebene Backend zugreifen können:

```python
service = QiskitRuntimeService(token=<your-API_KEY>)
```

In diesem Beispiel durchsucht der Dienst alle dem Konto in der EU-Region verfügbaren Instanzen, um diejenigen zu finden, die auf das angegebene Backend zugreifen können:

```python
service = QiskitRuntimeService(token=<your-API_KEY>, region="eu-de")
```

In diesem Beispiel durchsucht der Dienst alle dem Konto verfügbaren Instanzen, die als `services` getaggt sind, um diejenigen zu finden, die auf das angegebene Backend zugreifen können:

```python
service = QiskitRuntimeService(token=<your-API_KEY>, tags=['services'])
```

In diesem Beispiel durchsucht der Dienst alle Premium- und Open-Instanzen, die dem Konto zur Verfügung stehen, um diejenigen zu finden, die auf das angegebene Backend zugreifen können. Wenn ein Backend sowohl in einer Premium- als auch in einer Open Plan-Instanz vorhanden ist, wird die Premium Plan-Instanz priorisiert, da `premium` in `plans_preference` zuerst angegeben ist.

```python
service = QiskitRuntimeService(token=<your-API_KEY>, plans_preference=['premium', 'open'])
```

#### Bestimmen, welche Instanz ausgewählt wurde
Die ausgewählte Instanz wird als Warnung zurückgegeben. Zusätzlich können Sie `service.active_instance()` ausführen, um die aktive Instanz zu ermitteln.

### Lokales Testen mit Qiskit Runtime

Die Klasse `QiskitRuntimeService` kann mit `channel=local` initialisiert werden, um lokale Simulation durchzuführen. In diesem Fall ist keine Authentifizierung erforderlich und es ist nicht erforderlich, Werte für `token` oder `instance` anzugeben. Daher wird nicht empfohlen, ein lokales Konto zu speichern. Stattdessen können Sie eine direkte Instanziierung durchführen:

```python
from qiskit_ibm_runtime import QiskitRuntimeService

# Initialize for local testing

service = QiskitRuntimeService(channel="local")
```

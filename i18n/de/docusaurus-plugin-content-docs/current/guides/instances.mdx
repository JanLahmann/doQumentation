# Instanzen erstellen und verwalten

Der Zugriff auf IBM Quantum Platform-Dienste wird durch die **Instanzen** gesteuert, denen Sie zugewiesen sind. Benutzer können mehreren Instanzen angehören.

Instanzen sind virtuelle Server (Deployments von Qiskit Runtime), die die Ausführung Ihrer Workloads verwalten, einschließlich der Ausführung von Quantenprogrammen und klassischen Berechnungsaufgaben (wie der Verarbeitung von Fehlerminderung). Instanzen werden durch ihren *Cloud Resource Name* (CRN) identifiziert. Dieser CRN kann beim Senden von Workloads aus Ihren Qiskit-Programmen verwendet werden.

Sie können die Instanzen, auf die Sie Zugriff haben, im Dashboard sehen, oder indem Sie auf die Registerkarte "Instances" im Dashboard klicken. Jede Instanz wird mit ihrer CRN-Kennung aufgelistet.

Sie benötigen für jeden Serviceplan (wie Open oder Pay-As-You-Go), den Sie verwenden, in jeder Region, die Sie nutzen möchten, eine andere Instanz. Der Account-Manager und jeder mit ausreichender Berechtigung kann eine Instanz erstellen und ihre Konfiguration definieren, wie z.B. ihre Zuweisungen, Nutzungslimits, Quantencomputer und Benutzerberechtigungen.

Das IBM Quantum Platform-[Dashboard](https://quantum.cloud.ibm.com/) zeigt die Instanzen an, auf die Sie Zugriff haben. Je nach Ihrem Zugriff können Sie vollständige Details anzeigen, Instanzen bearbeiten und löschen auf der [Instances-Seite.](https://quantum.cloud.ibm.com/instances)

Es ist wichtig, dass Sie verstehen, welche Instanz Sie verwenden, denn alle Instanzen, die nicht mit dem Open Plan verbunden sind, verursachen Kosten.

<span id="open-plan"></span>
## Open Plan-Instanz

Open Plan-Benutzer können Quantenschaltungen auf den weltweit besten QPUs kostenlos ausführen (bis zu 10 Minuten Quantenzeit pro rollendem 28-Tage-Fenster). Die QPU-Nutzung kann auf dem [Platform-Dashboard](https://quantum.cloud.ibm.com/) und den [Workloads](https://quantum.cloud.ibm.com/workloads)-Seiten verfolgt werden.

<span id="standard-plan"></span>
## Pay-As-You-Go Plan-Instanz

Der Pay-As-You-Go Plan ist ein kostenpflichtiger Plan. Um diesen Plan zu nutzen, müssen Sie eine Instanz erstellen und sie dem Plan zuweisen. Alle Workloads, die mit diesem Plan an QPUs gesendet werden, verursachen Kosten.

Die Jobs, die Sie ausführen, und die Jobs, die von anderen Mitarbeitern in derselben Instanz ausgeführt werden, zählen zur gemeldeten Nutzung für diese Instanz. Sie können die Nutzung einer Instanz auf der [Instances](https://quantum.cloud.ibm.com/instances)-Seite oder, für diejenigen mit entsprechender Berechtigung, auf der [Analytics](https://quantum.cloud.ibm.com/analytics)-Seite anzeigen. Beachten Sie, dass die Seiten möglicherweise unterschiedliche Nutzungszahlen anzeigen, da die Instances-Seite die Nutzung in einem rollenden 28-Tage-Fenster vor dem aktuellen Zeitstempel anzeigt, während die Analytics-Seite vollständige Tagesgrenzen verwendet.

Wenn eine Instanz von Ihrem Administrator als "limited" markiert wurde (Sie sehen eine "Remaining"-Spalte im Nutzungsbereich) und die Instanz ihre Zuweisung (definiert von Ihrem Administrator) innerhalb des rollenden 28-Tage-Fensters überschreitet, wird jede aktive Workload weiterhin ausgeführt (einschließlich Sessions), aber ausstehende Workloads verbleiben in der Warteschlange, bis mehr Zeit verfügbar ist. Wenn eine Instanz nicht begrenzt ist und die Instanz ihre Zuweisung überschreitet, werden Jobs, die mit dieser Instanz ausgeführt werden, wahrscheinlich mit niedrigerer Priorität ausgeführt und erfahren längere Wartezeiten in der Warteschlange.

Eine Warnung wird bei der Nutzung einer Instanz angezeigt, wenn sie ihre Zuweisung überschritten hat.

<span id="connect-instance"></span>
## Angabe einer Instanz in Ihrem Code

Sie können den CRN oder den Instanznamen verwenden, um eine zu verwendende Instanz anzugeben - entweder bei der Initialisierung des Service oder beim Senden der Workload an eine QPU (zum Beispiel mit [`QiskitRuntimeServicee.backend()`](https://docs.quantum.ibm.com/api/qiskit-ibm-runtime/qiskit-runtime-service#backend)). Anweisungen zur Angabe einer Instanz in Ihrem Code finden Sie unter [Initialisieren Sie Ihr Konto.](/guides/initialize-account)

Wenn eine Instanz übergeben wird, sind nur Backends und Jobs aus dieser Instanz verfügbar (erlaubt in Qiskit Runtime v0.40.1 und später).

Wenn beim Initialisieren des Service keine Instanz angegeben wird, sind alle Backends und Jobs über alle Instanzen in Ihrem Konto hinweg verfügbar. In diesem Fall wird, wenn ein Backend angegeben wird, eine Instanz gefunden und verwendet, die auf das Backend zugreifen kann, wie in [Automatische Auswahl der Instanz](/guides/initialize-account#auto-select) beschrieben.

<Admonition type="important" title="Erforderliche Versionen">
Qiskit Runtime v0.40.1 oder später ist erforderlich für Folgendes:

    - Verwendung des Instanznamens zur Angabe einer Instanz
    - Keine Übergabe einer Instanz
    - Festlegen eines bevorzugten Instanztyps und einer Region

Für Qiskit Runtime v0.42 oder später werden kostenlose Pläne standardmäßig priorisiert, wenn die [automatische Instanzauswahl](/guides/initialize-account#auto-select) verwendet wird.
</Admonition>

<span id="add-instance"></span>
## Erstellen einer Instanz

Befolgen Sie diese Schritte, um eine Instanz zu erstellen und sie zu Ihrem Konto hinzuzufügen. Wenn Sie ein Administrator sind, der Qiskit Runtime on Cloud für Ihre Organisation einrichten muss, lesen Sie [Planen Sie Qiskit Runtime für eine Organisation](/guides/considerations-set-up-runtime).

<span id="switcher"></span>

1. Um eine Instanz zu Ihrem Konto hinzuzufügen, stellen Sie zunächst sicher, dass Sie das richtige Konto und die richtige Region im Kontoumschalter im Header ausgewählt haben.
        * Sie können mehrere IBM Cloud-Konten haben (oder Teil davon sein). Wenn Sie nicht die erwarteten Instanzen oder Jobs sehen, versuchen Sie ein anderes Konto.
        * Sie können auf beide Regionen mit demselben API-Schlüssel zugreifen, aber Sie können nur die Instanzen sehen und darauf zugreifen, die in der Region erstellt wurden, in der Sie angemeldet sind.
        * Wenn Sie eine Open-Instanz erstellen möchten, müssen Sie die Region us-east wählen.

        ![The IBM Quantum Platform header is shown.  The account switcher is immediately to the right of the search bar.  The region switcher is immediately to the right of the account switcher.](/docs/images/guides/cloud-setup/IQP-Header.svg 'IBM Quantum Platform header')

   <span id="regions"></span>

   <Admonition type="note" title="Über Regionen">
   Die Instanz wird in der ausgewählten Region erstellt. Diese Region bestimmt diese wichtigen Eigenschaften:
     * Welche QPUs verfügbar sind.
     * Wo die klassische Berechnung der Jobs, wie z.B. Kompilierung, stattfindet.
     * Wo die Workflow-Daten des Benutzers verbleiben.

    Die Workflow-Daten umfassen die Eingabeschaltungen, Schaltungsparameter und Quantenberechnungsergebnisse. Sie umfassen keine Metadaten wie Anzahl der Shots, Größe der Schaltungen oder andere Job-Statistiken.

   Um die Offenlegung der Workflow-Daten des Benutzers gegenüber einer globalen Distributed Denial of Service-Schutzschicht zu vermeiden, können Sie [Virtual Private Endpoints](https://cloud.ibm.com/docs/overview?topic=overview-endpoints-support) verwenden. `qiskit-ibm-runtime` unterstützt dies über den [`private_endpoint`-Parameter](https://docs.quantum.ibm.com/api/qiskit-ibm-runtime/qiskit-runtime-service).
   </Admonition>

1. Klicken Sie im Dashboard-Bereich "Instances" auf **View all**, oder gehen Sie vom Hauptmenü zur [Instances](https://quantum.cloud.ibm.com/instances)-Seite.
1. Klicken Sie auf der Instances-Seite auf **Create instance**.
    * Geben Sie einen Namen ein und fügen Sie optional Tags hinzu.
    * Wenn Sie Zugriff auf mehr als eine Ressourcengruppe in IBM Cloud&reg; haben, können Sie wählen, die Ressourcengruppe zu ändern, zu der diese Instanz gehört. Siehe [Verwalten von Ressourcengruppen](https://cloud.ibm.com/docs/account?topic=account-rgs&interface=ui) und [Zugriff auf Ressourcen in Ressourcengruppen gewähren](https://cloud.ibm.com/docs/account?topic=account-rgs_manage_access) für weitere Informationen.
1. Wählen Sie den Preisplan aus, mit dem diese Instanz verknüpft ist (zum Beispiel Pay-As-You-Go). Je nach ausgewähltem Preisplan können Sie weitere Optionen angeben, wie z.B. das Kostenlimit oder die Anzahl der Minuten, die dieser Instanz zugewiesen werden sollen. Beachten Sie, dass Kosten anfallen, wenn diese Instanz nicht mit einem kostenlosen Plan verknüpft ist und zur Ausführung von Workloads verwendet wird.
1. Klicken Sie auf **Next**. Eine Liste der QPUs, auf die die Instanz zugreifen kann, wird angezeigt.
    - Wenn Sie möchten, dass die Instanz Zugriff auf alle aufgelisteten QPUs hat (und alle QPUs, die diesem Plan in Zukunft hinzugefügt werden), klicken Sie auf **Next**.
    - Wenn Sie keine Open-Instanz einrichten, können Sie anpassen, auf welche QPUs mit dieser Instanz zugegriffen werden kann. Klicken Sie auf den "Customize allocated compute resources"-Schalter. Wählen Sie spezifische QPUs aus, auf die diese Instanz zugreifen kann, und klicken Sie dann auf **Next**.
        <Admonition type="note">
      Wenn Sie die zugewiesenen Rechenressourcen anpassen, werden die verfügbaren QPUs niemals automatisch aktualisiert, unabhängig von Änderungen am übergeordneten Plan. Sie können jedoch später manuell QPUs hinzufügen oder entfernen. Wenn Sie sie nicht anpassen, haben Sie immer Zugriff auf jede QPU, auf die das Konto im Plan Zugriff hat. Wenn also in Zukunft eine neue QPU zum Plan hinzugefügt wird, hat die Instanz automatisch Zugriff darauf.
        </Admonition>
1. Der Bereich "Access group" wird geöffnet. Eine "collaborators"-Zugriffsgruppe wird automatisch für diese Instanz erstellt. Benutzer, die dieser Zugriffsgruppe hinzugefügt werden, können die dieser Instanz zugewiesene Zeit nutzen. Sie können [Zugriffsgruppen ändern oder zusätzliche erstellen](/guides/access-groups), indem Sie die [IBM Cloud-Konsole](https://cloud.ibm.com/iam/groups) verwenden. Siehe [Einrichten von Zugriffsgruppen](https://cloud.ibm.com/docs/account?topic=account-groups&interface=ui) für Anweisungen.
1. Klicken Sie auf **Create instance**. Sie können die Instanz jetzt auf der [Instances](https://quantum.cloud.ibm.com/instances)-Seite anzeigen. Wenn Sie mehrere Pläne in Ihrem Konto eingerichtet haben, hat jeder Plantyp seine eigene Registerkarte in der Instances-Tabelle.

## Bearbeiten einer Instanz und ihrer Zuweisung

Suchen Sie die Instanz auf der [Instances](https://quantum.cloud.ibm.com/instances)-Seite. Klicken Sie auf das Überlaufmenü am Ende der Zeile der Instanz und wählen Sie "Edit details".

<Admonition type="note">
Nur Administratoren und Eigentümer können die Instanz bearbeiten. Siehe den Leitfaden [Zugriffsrichtlinien und Zugriffsgruppen erstellen](/guides/access-groups) für Informationen zum Zuweisen von Zugriff.
</Admonition>

# Einrichtung zur Verwendung von IBM Quantum Platform mit REST-API {#set-up-to-use-ibm-quantum-platform-with-rest-api}

Du kannst auf Quantenprozessoren mit REST-APIs zugreifen, sodass du mit QPUs in jeder Programmiersprache oder jedem Framework arbeiten kannst.

### 1. Zugang erhalten {#1-get-access}

1. Falls du noch kein Benutzerkonto hast, erstelle eines auf der [IBM Quantum-Anmeldeseite.](https://quantum.cloud.ibm.com/login)
1. Erstelle einen API-Schlüssel (auch Token genannt) auf dem [Dashboard.](https://quantum.cloud.ibm.com) Beachte, dass derselbe API-Schlüssel für beide Regionen verwendet werden kann.
1. Generiere ein IBM Cloud Identity and Access Management (IAM) Bearer-Token. Dies ist ein kurzlebiges Token, das zur Authentifizierung von Anfragen an die REST-API verwendet wird. Um eines zu generieren, rufe die [IAM Identity Services API](https://cloud.ibm.com/apidocs/iam-identity-token-api#create-api-key) auf, wie in der folgenden Beispielanfrage gezeigt:
<Tabs>

   <TabItem value="Curl" label="Curl">
   ```bash
   curl -X POST 'https://iam.cloud.ibm.com/identity/token' \
   -H 'Content-Type: application/x-www-form-urlencoded' \
   -d 'grant_type=urn:ibm:params:oauth:grant-type:apikey&apikey=MY_APIKEY'
   ```
   Erwartete Antwort
   ```text
   {
      "access_token": "eyJhbGciOiJIUz......sgrKIi8hdFs",
      "refresh_token": "SPrXw5tBE3......KBQ+luWQVY=",
      "token_type": "Bearer",
      "expires_in": 3600,
      "expiration": 1473188353
   }
   ```
   </TabItem>

   <TabItem value="Python" label="Python">
   ```python
   # Use 'service' to invoke operations.
   import requests
   import json

   url = 'https://iam.cloud.ibm.com/identity/token'
   api_key = 'MY_APIKEY'
   headers = {
    'Content-Type': 'application/x-www-form-urlendcoded',
   }
   data = f'grant_type=urn:ibm:params:oauth:grant-type:apikey&apikey={api_key}'
   response = requests.post(url, headers=headers, data=data)

   # Bearer token to authorize requests to the REST API
   bearer_token = response.json()['access_token']
   ```
   </TabItem>
</Tabs>

### 2. Wähle eine Authentifizierungsmethode {#2-choose-an-authentication-method}

Wähle die geeignete Authentifizierungsmethode, abhängig von deiner Arbeitsumgebung:

* [Erstelle eine Umgebungsvariable für deinen API-Schlüssel (vertrauenswürdige Python-Umgebungen)](#environment_variable)
* [Verwende deinen API-Schlüssel direkt (nicht vertrauenswürdige Umgebung)](/guides/cloud-setup-untrusted#rest-untrusted)

<span id="environment_variable"></span>
#### Erstelle eine Umgebungsvariable (vertrauenswürdige Umgebung) {#create-an-environment-variable-trusted-environment}

1. Um die IQP_API_TOKEN-Umgebungsvariable in deinem System festzulegen, kannst du die folgende Zeile zu deinem Shell-Profil hinzufügen (z.B. .bashrc oder .zshrc) oder sie direkt in deinem Terminal setzen:

    ```shell
    export IQP_API_TOKEN=<your-API_KEY> # Use the 44-character API_KEY you created and saved from the IBM Quantum Platform Home dashboard
    ```
    Wenn du die Umgebungsvariable in deinem Code aufrufst, füge `import os` hinzu, wie in diesem Beispiel:

    ```python
    import os
    api_token = os.environ['IQP_API_TOKEN']
    ```
    Beachte, dass beim Erstellen einer Umgebungsvariablen dein API-Schlüssel immer noch lokal im Klartext gespeichert ist und geschützt werden sollte.

1. Authentifiziere Anfragen an die Qiskit Runtime REST-API, indem du den CRN und das Bearer-Token in die Request-Header aufnimmst.
```bash
curl -X 'GET' \
   'https://quantum.cloud.ibm.com/api/v1/usage' \
   '-H accept: application/json' \
   '-H authorization: Bearer <BEARER_TOKEN>' \
   '-H Service-CRN: <INSTANCE_CRN>'
```

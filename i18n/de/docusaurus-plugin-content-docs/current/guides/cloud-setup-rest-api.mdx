# Einrichtung zur Verwendung von IBM Quantum Platform mit REST-API {#set-up-to-use-ibm-quantum-platform-with-rest-api}

Sie können auf Quantenprozessoren mit REST-APIs zugreifen, sodass Sie mit QPUs in jeder Programmiersprache oder jedem Framework arbeiten können.

### 1. Zugang erhalten {#1-get-access}

1. Falls Sie noch kein Benutzerkonto haben, erstellen Sie eines auf der [IBM Quantum-Anmeldeseite.](https://quantum.cloud.ibm.com/login)
1. Erstellen Sie einen API-Schlüssel (auch Token genannt) auf dem [Dashboard.](https://quantum.cloud.ibm.com) Beachten Sie, dass derselbe API-Schlüssel für beide Regionen verwendet werden kann.
1. Generieren Sie ein IBM Cloud Identity and Access Management (IAM) Bearer-Token. Dies ist ein kurzlebiges Token, das zur Authentifizierung von Anfragen an die REST-API verwendet wird. Um eines zu generieren, rufen Sie die [IAM Identity Services API](https://cloud.ibm.com/apidocs/iam-identity-token-api#create-api-key) auf, wie in der folgenden Beispielanfrage gezeigt:
<Tabs>

   <TabItem value="Curl" label="Curl">
   ```bash
   curl -X POST 'https://iam.cloud.ibm.com/identity/token' \
   -H 'Content-Type: application/x-www-form-urlencoded' \
   -d 'grant_type=urn:ibm:params:oauth:grant-type:apikey&apikey=MY_APIKEY'
   ```
   Erwartete Antwort
   ```text
   {
      "access_token": "eyJhbGciOiJIUz......sgrKIi8hdFs",
      "refresh_token": "SPrXw5tBE3......KBQ+luWQVY=",
      "token_type": "Bearer",
      "expires_in": 3600,
      "expiration": 1473188353
   }
   ```
   </TabItem>

   <TabItem value="Python" label="Python">
   ```python
   # Use 'service' to invoke operations.
   import requests
   import json

   url = 'https://iam.cloud.ibm.com/identity/token'
   api_key = 'MY_APIKEY'
   headers = {
    'Content-Type': 'application/x-www-form-urlendcoded',
   }
   data = f'grant_type=urn:ibm:params:oauth:grant-type:apikey&apikey={api_key}'
   response = requests.post(url, headers=headers, data=data)

   # Bearer token to authorize requests to the REST API
   bearer_token = response.json()['access_token']
   ```
   </TabItem>
</Tabs>

### 2. Wählen Sie eine Authentifizierungsmethode {#2-choose-an-authentication-method}

Wählen Sie die geeignete Authentifizierungsmethode, abhängig von Ihrer Arbeitsumgebung:

* [Erstellen Sie eine Umgebungsvariable für Ihren API-Schlüssel (vertrauenswürdige Python-Umgebungen)](#environment_variable)
* [Verwenden Sie Ihren API-Schlüssel direkt (nicht vertrauenswürdige Umgebung)](/guides/cloud-setup-untrusted#rest-untrusted)

<span id="environment_variable"></span>
#### Erstellen Sie eine Umgebungsvariable (vertrauenswürdige Umgebung) {#create-an-environment-variable-trusted-environment}

1. Um die IQP_API_TOKEN-Umgebungsvariable in Ihrem System festzulegen, können Sie die folgende Zeile zu Ihrem Shell-Profil hinzufügen (z.B. .bashrc oder .zshrc) oder sie direkt in Ihrem Terminal setzen:

    ```shell
    export IQP_API_TOKEN=<your-API_KEY> # Use the 44-character API_KEY you created and saved from the IBM Quantum Platform Home dashboard
    ```
    Wenn Sie die Umgebungsvariable in Ihrem Code aufrufen, fügen Sie `import os` hinzu, wie in diesem Beispiel:

    ```python
    import os
    api_token = os.environ['IQP_API_TOKEN']
    ```
    Beachten Sie, dass beim Erstellen einer Umgebungsvariablen Ihr API-Schlüssel immer noch lokal im Klartext gespeichert ist und geschützt werden sollte.

1. Authentifizieren Sie Anfragen an die Qiskit Runtime REST-API, indem Sie den CRN und das Bearer-Token in die Request-Header aufnehmen.
```bash
curl -X 'GET' \
   'https://quantum.cloud.ibm.com/api/v1/usage' \
   '-H accept: application/json' \
   '-H authorization: Bearer <BEARER_TOKEN>' \
   '-H Service-CRN: <INSTANCE_CRN>'
```

# Instanz-Zuweisungslimits festlegen

Ein Konto kann zugewiesene Zeit im Premium- oder Flex-Plan haben. Da jedes Konto viele Instanzen in einem Plan haben kann, helfen Zuweisungslimits dabei zu kontrollieren, welche Instanzen diese Zeit verbrauchen dürfen. Beachten Sie, dass die Instanz mehr als ihre zugewiesene Zeit verbrauchen kann, wenn die Zuweisung nicht als Limit festgelegt ist.

Die Zeit, die Sie Instanzen zuweisen, wird vom Fair-Share-Algorithmus verwendet, um die Priorität in der Warteschlange zu bestimmen, basierend auf der Nutzung aller QPUs über ein rollendes Zeitfenster von 28 Tagen (für Premium-Plan-Nutzer) oder über den Abonnementzeitraum (für Flex-Plan-Nutzer).

Die Zuweisung wird beim Erstellen der Instanz festgelegt. Sie können jedoch die zugewiesene Zeit auf der [Instanzen-Seite](https://quantum.cloud.ibm.com/instances) anzeigen und ändern.

## Was passiert, wenn das Limit erreicht wird

Sie können optional die Zeitzuweisung als Limit für die Instanz festlegen. Folgendes geschieht, wenn eine Instanz ihr Limit erreicht:

- Aktive Workloads laufen weiter.
- Ausstehende Jobs bleiben in der Warteschlange.
- Keine Workloads werden abgebrochen.
- Mitarbeiter können neue Workloads einreichen, die in der Warteschlange bleiben, bis mehr Zeit verfügbar ist.
- Für den Flex-Plan kann die Instanz keine weiteren Jobs verarbeiten, es sei denn, das Limit wurde erhöht oder, für den Premium-Plan, nach Beginn des nächsten Nutzungsfensters.

## Unterschiedliche Werte für Zuweisungslimit und Nutzungslimit festlegen

Wenn Sie das Zuweisungslimit für eine Instanz in der IBM Quantum Platform festlegen, wird derselbe Wert als Nutzungslimit der Instanz festgelegt. Wenn Sie möchten, dass Benutzer weiterhin Jobs einreichen und ausführen können, nachdem das Zuweisungslimit der Instanz erreicht wurde, können Sie einen höheren Wert für das Nutzungslimit festlegen, indem Sie den Parameter [usage_allocation_seconds](/guides/access-instances-platform-apis#parameters) in der API [Update a resource instance](https://cloud.ibm.com/apidocs/resource-controller/resource-controller#update-resource-instance) verwenden.

# Initialisieren Sie den Dienst in einer nicht vertrauenswürdigen Umgebung {#initialize-the-service-in-an-untrusted-environment}

Folgen Sie diesen Anweisungen, wenn Sie mit IBM Cloud oder der IBM Qiskit Runtime REST-API in einer nicht vertrauenswürdigen Umgebung arbeiten, wie einem öffentlichen Computer.

**Vermeiden Sie die Ausführung von Code auf einem nicht vertrauenswürdigen Computer oder einer externen Cloud-Python-Umgebung, um Sicherheitsrisiken zu minimieren.** Wenn Sie eine nicht vertrauenswürdige Umgebung verwenden müssen (beispielsweise auf einem gemeinsam genutzten oder öffentlichen Computer), ändern Sie Ihren API-Schlüssel nach jeder Verwendung, indem Sie ihn auf der [API-Schlüssel-Seite](https://cloud.ibm.com/iam/apikeys) löschen und einen neuen erstellen.

    <Admonition type="caution">
      **Schützen Sie Ihren API-Schlüssel!** Fügen Sie Ihren Schlüssel niemals in Quellcode, Python-Skript oder Notebook-Datei ein. Wenn Sie Code mit anderen teilen, stellen Sie sicher, dass Ihr API-Schlüssel nicht direkt im Python-Skript eingebettet ist. Teilen Sie stattdessen das Skript ohne das Token und geben Sie Anweisungen zur sicheren Einrichtung.

      Wenn Sie Ihren Schlüssel versehentlich mit jemandem teilen oder ihn in der Versionskontrolle wie Git einbinden, widerrufen Sie Ihren Schlüssel sofort, indem Sie das Thema [Löschen eines API-Schlüssels](https://cloud.ibm.com/docs/account?topic=account-userapikey&interface=ui#delete_user_key) auf IBM Cloud befolgen.
    </Admonition>

<span id="access-credentials"></span>
## Finden Sie Ihre Zugangsdaten {#find-your-access-credentials}

1. Finden Sie Ihren API-Schlüssel (auch als _API-Token_ bezeichnet). Erstellen Sie auf dem [Dashboard](https://quantum.cloud.ibm.com/) Ihren API-Schlüssel und kopieren Sie ihn an einen sicheren Ort, damit Sie ihn zur Authentifizierung verwenden können. Das Token wird nicht erneut sichtbar sein. Beachten Sie, dass Sie einen einzelnen API-Schlüssel verwenden können, um sich mit jeder Region zu verbinden.
2. Optional: Finden Sie die Instanz, die Sie verwenden möchten, auf der [Instanzen](https://quantum.cloud.ibm.com/instances)-Seite. Fahren Sie mit der Maus über ihren CRN, klicken Sie auf das Symbol zum Kopieren und speichern Sie ihn an einem sicheren Ort, damit Sie ihn zur Identifizierung der Instanz verwenden können.

## Authentifizierung bei IBM Cloud in einer nicht vertrauenswürdigen Umgebung {#authenticate-to-ibm-cloud-in-an-untrusted-environment}

Um den Dienst in dieser Situation zu initialisieren, verwenden Sie Code wie den folgenden:

    ```python
    from qiskit_ibm_runtime import QiskitRuntimeService

    # The credentials you enter will override any saved
    # account credentials that might be available locally.
    service = QiskitRuntimeService(
      # Use the 44-character API_KEY you created and saved from
      # the IBM Quantum Platform dashboard, and then delete
      # the key on the API keys page after entering this code:
      token="<your-API-key>",

      # Optionally specify an instance to use
      instance="<IBM Cloud CRN or instance name>"
      )
    ```

<span id="rest-untrusted"></span>
## Authentifizierung bei der REST-API in einer nicht vertrauenswürdigen Umgebung {#authenticate-to-the-rest-api-in-an-untrusted-environment}

Folgen Sie diesen Schritten, um Ihren API-Schlüssel direkt zur Authentifizierung bei der Qiskit Runtime REST-API zu verwenden, wobei `<your_API_KEY>` der 44-stellige API_KEY ist, den Sie erstellt und von Ihrem IBM Quantum Platform Home-Dashboard gespeichert haben.

1. Verwenden Sie Ihren API-Schlüssel, um ein [Bearer-Token](https://docs.quantum.ibm.com/api/qiskit-runtime-rest#authentication) zu generieren. Ein _Bearer-Token_ ist eine temporäre Anmeldeinformation, die nach maximal einer Stunde abläuft. Nachdem das erworbene Token abgelaufen ist, müssen Sie ein neues generieren, um weiterhin IBM Cloud oder andere Dienst-APIs aufzurufen:
  ```bash
  curl -X POST 'https://iam.cloud.ibm.com/identity/token' \
  -H 'Content-Type: application/x-www-form-urlencoded' \
  -d 'grant_type=urn:ibm:params:oauth:grant-type:apikey&apikey=your_API_KEY'
  ```
  Kopieren und speichern Sie das zurückgegebene Bearer-Token: `"access_token": "<NEW_BEARER_TOKEN>"`
1. Authentifizieren Sie Anfragen an die Qiskit Runtime REST-API, indem Sie den CRN und das Bearer-Token in die Request-Header aufnehmen.

  ```bash
  curl -X 'GET' \
     'https://quantum.cloud.ibm.com/api/v1/usage' \
     '-H accept: application/json' \
     '-H authorization: Bearer <BEARER_TOKEN>' \
     '-H Service-CRN: <INSTANCE_CRN>'
  ```

1. Ändern Sie Ihren API-Schlüssel nach jeder Verwendung, indem Sie einen neuen API-Schlüssel auf der IBM Quantum Platform-Startseite generieren oder indem Sie die Anweisungen zum [Löschen eines API-Schlüssels](https://cloud.ibm.com/docs/account?topic=account-userapikey&interface=ui#delete_user_key) auf IBM Cloud befolgen.

# Installation des Qiskit SDK und Qiskit Runtime aus dem Quellcode {#install-the-qiskit-sdk-and-qiskit-runtime-from-source}

Die Installation des Qiskit SDK aus dem Quellcode ermöglicht Ihnen den Zugriff auf die aktuelle Entwicklungsversion, anstatt die Version aus dem Python Package Index (PyPI)-Repository zu verwenden. Dies ermöglicht es Ihnen, die neueste Version des Qiskit-Codes effizienter zu überprüfen und zu erweitern.

## Erstellen und Aktivieren einer neuen virtuellen Umgebung {#create-and-activate-a-new-virtual-environment}

1. Navigieren Sie zu Ihrem Projektverzeichnis und erstellen Sie eine minimale Umgebung, in der nur Python installiert ist.

        <OperatingSystemTabs>
          <TabItem value="mac" label="macOS">
            ```shell
            python3 -m venv .venv
            ```
          </TabItem>

          <TabItem value="linux" label="Linux">
            ```shell
            python3 -m venv .venv
            ```
          </TabItem>

          <TabItem value="win" label="Windows">
            ```text
            python -m venv .venv
            ```
          </TabItem>
         </OperatingSystemTabs>

1. Aktivieren Sie Ihre neue Umgebung.

         <OperatingSystemTabs>
           <TabItem value="mac" label="macOS">
             ```shell
             source .venv/bin/activate
             ```
           </TabItem>

           <TabItem value="linux" label="Linux">
             ```shell
             source .venv/bin/activate
             ```
           </TabItem>

           <TabItem value="win" label="Windows">
             ```text
             .venv\Scripts\Activate.ps1
             ```
           </TabItem>
         </OperatingSystemTabs>

## Installation des Rust-Compilers {#install-the-rust-compiler}

Ein Rust-Compiler muss auf Ihrem System installiert sein, um Qiskit zu kompilieren. Um den Rust-Compiler zu installieren, verwenden Sie den plattformübergreifenden Rust-Installer [`rustup`](https://rustup.rs/) oder [eine andere Installationsmethode.](https://forge.rust-lang.org/infra/other-installation-methods.html)

## Installation von Qiskit {#install-qiskit}

Befolgen Sie diese Schritte, um Qiskit zu installieren:

1. Klonen Sie das Qiskit-Repository.

```bash
git clone https://github.com/Qiskit/qiskit.git
```

2. Wechseln Sie in das Verzeichnis `qiskit`.

```bash
cd qiskit
```

3. (Optional) Wenn Sie Tests oder Linting-Überprüfungen ausführen möchten, installieren Sie die Entwickler-Anforderungen.

```bash
pip install -r requirements-dev.txt
```

4. Installieren Sie `qiskit`.

  * **Standard-Installation**:

    ```bash
    pip install .
    ```

  * **Bearbeitbarer Modus**: In diesem Modus müssen Sie Qiskit nicht neu installieren, wenn Codeänderungen am Projekt vorgenommen werden.

    ```bash
    pip install -e .
    ```

    Im bearbeitbaren Modus werden die kompilierten Erweiterungen im _Debug-Modus_ ohne Optimierungen erstellt. Dies beeinflusst die Laufzeitleistung des kompilierten Codes. Um die kompilierten Erweiterungen mit aktivierten Optimierungen zu erstellen, führen Sie den folgenden Befehl aus, um die Binärdatei im _Release-Modus_ neu zu erstellen:

    ```bash
    python setup.py build_rust --release --inplace
    ```

    <Admonition type="note">
    Wenn Sie am Rust-Code in Qiskit arbeiten, müssen Sie den Erweiterungscode jedes Mal neu erstellen, wenn Sie eine lokale Änderung vornehmen. Im bearbeitbaren Modus wird die Rust-Erweiterung nur erstellt, wenn der Installationsbefehl ausgeführt wird. Lokale Änderungen, die Sie am Rust-Code vornehmen, werden daher nicht im installierten Paket widergespiegelt, es sei denn, Sie erstellen die Erweiterung neu, indem Sie `build_rust` erneut ausführen (mit oder ohne `--release`, je nachdem, ob Sie im Release- oder Debug-Modus erstellen möchten).
    </Admonition>

## Installation von Qiskit Runtime {#install-qiskit-runtime}

Befolgen Sie diese Schritte, wenn Sie Qiskit Runtime installieren möchten:

1. Klonen Sie das Qiskit Runtime-Repository.

```bash
git clone https://github.com/Qiskit/qiskit-ibm-runtime.git
```

2. Wechseln Sie in das Verzeichnis `qiskit-ibm-runtime`.

```bash
cd qiskit-ibm-runtime
```

3. Installieren Sie `qiskit-runtime`. Wir empfehlen die Verwendung einer [virtuellen Umgebung](https://docs.python.org/3/library/venv.html), um zu vermeiden, dass Ihre globale Python-Installation beeinträchtigt wird.

  * **Standard-Installation**:

    ```bash
    pip install .
    ```

  * **Bearbeitbarer Modus**: In diesem Modus müssen Sie Qiskit nicht neu installieren, wenn Codeänderungen am Projekt vorgenommen werden.

    ```bash
    pip install -e .
    ```

    Im bearbeitbaren Modus werden die kompilierten Erweiterungen im _Debug-Modus_ ohne Optimierungen erstellt.

4. (Optional) Wenn Sie Tests oder Linting-Überprüfungen ausführen möchten, installieren Sie die Entwickler-Anforderungen. Wir empfehlen die Verwendung einer [virtuellen Umgebung](https://docs.python.org/3/library/venv.html), um zu vermeiden, dass Ihre globale Python-Installation beeinträchtigt wird.

```bash
pip install -e ".[dev]"
```

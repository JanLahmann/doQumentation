## Klassen in Operators umwandeln {#convert-classes-to-operators}

Mehrere andere Klassen in Qiskit kГ¶nnen mithilfe der Operator-Initialisierungsmethode direkt in ein `Operator`-Objekt umgewandelt werden. Zum Beispiel:

* `Pauli`-Objekte
* `Gate`- und `Instruction`-Objekte
* `QuantumCircuit`-Objekte

Beachte, dass der letzte Punkt bedeutet, dass du die `Operator`-Klasse als unitГӨren Simulator verwenden kannst, um die abschlieГҹende unitГӨre Matrix fГјr einen Quantum Circuit zu berechnen, ohne einen Simulator-Backend aufrufen zu mГјssen. Falls der Circuit nicht unterstГјtzte Operationen enthГӨlt, wird eine Ausnahme ausgelГ¶st. Nicht unterstГјtzte Operationen sind: Messen, ZurГјcksetzen, bedingte Operationen oder ein Gate, das keine Matrixdefinition oder Zerlegung in Gates mit Matrixdefinitionen besitzt.

## Operators in Circuits verwenden {#use-operators-in-circuits}

UnitГӨre `Operators` kГ¶nnen mithilfe der `QuantumCircuit.append`-Methode direkt in einen `QuantumCircuit` eingefГјgt werden. Dabei wird der `Operator` in ein `UnitaryGate`-Objekt umgewandelt, das dem Circuit hinzugefГјgt wird.

Falls der Operator nicht unitГӨr ist, wird eine Ausnahme ausgelГ¶st. Dies kann mit der Funktion `Operator.is_unitary()` ГјberprГјft werden, die `True` zurГјckgibt, wenn der Operator unitГӨr ist, und andernfalls `False`.

```python
# Create an operator
XX = Operator(Pauli("XX"))

# Add to a circuit
circ = QuantumCircuit(2, 2)
circ.append(XX, [0, 1])
circ.measure([0, 1], [0, 1])
circ.draw("mpl")
```

## Operators kombinieren {#combine-operators}

Operators kГ¶nnen auf verschiedene Weisen miteinander kombiniert werden.

### Tensorprodukt {#tensor-product}

Zwei Operatoren $A$ und $B$ kГ¶nnen mithilfe der Funktion `Operator.tensor` zu einem Tensorprodukt-Operator $A\otimes B$ kombiniert werden. Wenn sowohl $A$ als auch $B$ Einzel-Qubit-Operatoren sind, gilt: `A.tensor(B)` = $A\otimes B$ hat die Teilsysteme so indiziert, dass Matrix $B$ auf Teilsystem 0 und Matrix $A$ auf Teilsystem 1 liegt.

```python
A = Operator(Pauli("X"))
B = Operator(Pauli("Z"))
A.tensor(B)
```

```text
Operator([[ 0.+0.j,  0.+0.j,  1.+0.j,  0.+0.j],
          [ 0.+0.j, -0.+0.j,  0.+0.j, -1.+0.j],
          [ 1.+0.j,  0.+0.j,  0.+0.j,  0.+0.j],
          [ 0.+0.j, -1.+0.j,  0.+0.j, -0.+0.j]],
         input_dims=(2, 2), output_dims=(2, 2))
```

### Tensor-Expansion {#tensor-expansion}

Eine eng verwandte Operation ist `Operator.expand`, die wie ein Tensorprodukt funktioniert, jedoch in umgekehrter Reihenfolge. FГјr zwei Operatoren $A$ und $B$ gilt daher: `A.expand(B)` = $B\otimes A$, wobei die Teilsysteme so indiziert sind, dass Matrix $A$ auf Teilsystem 0 und Matrix $B$ auf Teilsystem 1 liegt.

```python
A = Operator(Pauli("X"))
B = Operator(Pauli("Z"))
A.expand(B)
```

```text
Operator([[ 0.+0.j,  1.+0.j,  0.+0.j,  0.+0.j],
          [ 1.+0.j,  0.+0.j,  0.+0.j,  0.+0.j],
          [ 0.+0.j,  0.+0.j, -0.+0.j, -1.+0.j],
          [ 0.+0.j,  0.+0.j, -1.+0.j, -0.+0.j]],
         input_dims=(2, 2), output_dims=(2, 2))
```

### Komposition {#composition}

Du kannst zwei Operatoren $A$ und $B$ auch mittels der Methode `Operator.compose` zur Matrixmultiplikation verknГјpfen. `A.compose(B)` gibt den Operator mit Matrix $B.A$ zurГјck:

```python
A = Operator(Pauli("X"))
B = Operator(Pauli("Z"))
A.compose(B)
```

```text
Operator([[ 0.+0.j,  1.+0.j],
          [-1.+0.j,  0.+0.j]],
         input_dims=(2,), output_dims=(2,))
```

Du kannst die Komposition auch in umgekehrter Reihenfolge durchfГјhren, indem du $B$ vor $A$ anwendest, mithilfe des `front`-Kwargs von `compose`: `A.compose(B, front=True)` = $A.B$:

```python
A = Operator(Pauli("X"))
B = Operator(Pauli("Z"))
A.compose(B, front=True)
```

```text
Operator([[ 0.+0.j, -1.+0.j],
          [ 1.+0.j,  0.+0.j]],
         input_dims=(2,), output_dims=(2,))
```

### Teilsystem-Komposition {#subsystem-composition}

Beachte, dass bei der vorherigen Komposition die gesamte Ausgabedimension des ersten Operators $A$ gleich der gesamten Eingabedimension des komponierten Operators $B$ sein muss (und entsprechend muss die Ausgabedimension von $B$ gleich der Eingabedimension von $A$ sein, wenn mit `front=True` komponiert wird).

Du kannst auch einen kleineren Operator mit einer Auswahl von Teilsystemen eines grГ¶Гҹeren Operators verknГјpfen, indem du das `qargs`-Kwarg von `compose` verwendest вҖ“ mit oder ohne `front=True`. In diesem Fall mГјssen die relevanten Eingabe- und Ausgabedimensionen der komponierten Teilsysteme Гјbereinstimmen. *Beachte, dass der kleinere Operator immer das Argument der* `compose`*-Methode sein muss.*

Um beispielsweise ein Zwei-Qubit-Gate mit einem Drei-Qubit-Operator zu verknГјpfen:

```python
# Compose XZ with a 3-qubit identity operator
op = Operator(np.eye(2**3))
XZ = Operator(Pauli("XZ"))
op.compose(XZ, qargs=[0, 2])
```

### Linearkombinationen {#linear-combinations}

Operators kГ¶nnen auch mithilfe von Standard-Linearoperatoren fГјr Addition, Subtraktion und skalare Multiplikation mit komplexen Zahlen kombiniert werden.

```python
XX = Operator(Pauli("XX"))
YY = Operator(Pauli("YY"))
ZZ = Operator(Pauli("ZZ"))

op = 0.5 * (XX + YY - 3 * ZZ)
op
```

```text
Operator([[-1.5+0.j,  0. +0.j,  0. +0.j,  0. +0.j],
          [ 0. +0.j,  1.5+0.j,  1. +0.j,  0. +0.j],
          [ 0. +0.j,  1. +0.j,  1.5+0.j,  0. +0.j],
          [ 0. +0.j,  0. +0.j,  0. +0.j, -1.5+0.j]],
         input_dims=(2, 2), output_dims=(2, 2))
```

Ein wichtiger Punkt ist, dass `tensor`, `expand` und `compose` die UnitaritГӨt unitГӨrer Operatoren erhalten, Linearkombinationen jedoch nicht. Das Addieren zweier unitГӨrer Operatoren ergibt daher im Allgemeinen einen nicht-unitГӨren Operator:

```python
op.is_unitary()
```

```text
False
```

### Implizite Umwandlung zu Operators {#implicit-conversion-to-operators}

Beachte, dass bei allen folgenden Methoden ein zweites Objekt, das noch kein `Operator`-Objekt ist, von der Methode implizit in eines umgewandelt wird. Das bedeutet, dass Matrizen direkt Гјbergeben werden kГ¶nnen, ohne sie zuvor explizit in einen `Operator` umzuwandeln. Falls die Umwandlung nicht mГ¶glich ist, wird eine Ausnahme ausgelГ¶st.

```python
# Compose with a matrix passed as a list
Operator(np.eye(2)).compose([[0, 1], [1, 0]])
```

```text
Operator([[0.+0.j, 1.+0.j],
          [1.+0.j, 0.+0.j]],
         input_dims=(2,), output_dims=(2,))
```

## Operators vergleichen {#compare-operators}

Operators implementieren eine Gleichheitsmethode, mit der geprГјft werden kann, ob zwei Operatoren nГӨherungsweise gleich sind.

```python
Operator(Pauli("X")) == Operator(XGate())
```

```text
True
```

Beachte, dass dabei jedes Matrixelement der Operatoren nГӨherungsweise verglichen wird; zwei unitГӨre Operatoren, die sich nur durch eine globale Phase unterscheiden, gelten nicht als gleich:

```python
Operator(XGate()) == np.exp(1j * 0.5) * Operator(XGate())
```

```text
False
```

### Prozesstreue {#process-fidelity}

Du kannst Operatoren auch mithilfe der Funktion `process_fidelity` aus dem Modul *Quantum Information* vergleichen. Dies ist eine informationstheoretische GrГ¶Гҹe, die angibt, wie nah sich zwei QuantenkanГӨle sind. Bei unitГӨren Operatoren hГӨngt sie nicht von der globalen Phase ab.

```python
# Two operators which differ only by phase
op_a = Operator(XGate())
op_b = np.exp(1j * 0.5) * Operator(XGate())

# Compute process fidelity
F = process_fidelity(op_a, op_b)
print("Process fidelity =", F)
```

```text
Process fidelity = 1.0
```

Beachte, dass die Prozesstreue im Allgemeinen nur dann ein gГјltiges NГӨherungsmaГҹ ist, wenn die Eingabeoperatoren unitГӨr sind (oder CP im Fall von QuantenkanГӨlen), und eine Ausnahme ausgelГ¶st wird, wenn die Eingaben nicht CP sind.

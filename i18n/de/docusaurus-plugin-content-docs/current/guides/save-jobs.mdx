Quantenworkflows dauern oft eine Weile und können über mehrere Sitzungen laufen. Wenn du deinen Python-Kernel neu startest, gehen alle im Arbeitsspeicher gespeicherten Ergebnisse verloren. Um Datenverlust zu vermeiden, kannst du Ergebnisse in einer Datei speichern und Ergebnisse früherer Jobs von IBM Quantum&reg; abrufen, damit deine nächste Sitzung dort weitermachen kann, wo du aufgehört hast.
## Job-Ergebnisse von IBM Quantum abrufen {#retrieve-job-results-from-ibm-quantum}

IBM Quantum speichert automatisch die Ergebnisse jedes Jobs, damit du sie zu einem späteren Zeitpunkt abrufen kannst. Nutze diese Funktion, um Quantenprogramme über Kernel-Neustarts hinweg fortzusetzen und frühere Ergebnisse einzusehen. Du kannst die ID eines Jobs programmatisch über seine `job_id`-Methode ermitteln oder alle deine eingereichten Jobs und ihre IDs auf der [Workloads-Seite](https://quantum.cloud.ibm.com/workloads) einsehen.

Um einen Job programmatisch zu finden, verwende die Methode [`QiskitRuntimeService.jobs`](https://docs.quantum.ibm.com/api/qiskit-ibm-runtime/qiskit-runtime-service#jobs). Standardmäßig werden die zuletzt eingereichten Jobs zurückgegeben, aber du kannst Jobs auch nach Backend-Name, Erstellungsdatum und mehr filtern. Die folgende Zelle findet alle Jobs, die in den letzten drei Monaten eingereicht wurden. Das Argument `created_after` muss ein [`datetime.datetime`](https://docs.python.org/3.8/library/datetime.html#datetime.datetime)-Objekt sein.

```python
# Added by doQumentation — required packages for this notebook
```

```python
import datetime
from qiskit_ibm_runtime import QiskitRuntimeService

three_months_ago = datetime.datetime.now() - datetime.timedelta(days=90)

service = QiskitRuntimeService()
jobs_in_last_three_months = service.jobs(created_after=three_months_ago)
jobs_in_last_three_months[:3]  # show first three jobs
```

Du kannst auch nach Backend, Job-Status, Sitzung und mehr auswählen. Weitere Informationen findest du in der API-Dokumentation unter [`QiskitRuntimeService.jobs`](https://docs.quantum.ibm.com/api/qiskit-ibm-runtime/qiskit-runtime-service#jobs).

Sobald du die Job-ID hast, verwende die Methode [`QiskitRuntimeService.job`](https://docs.quantum.ibm.com/api/qiskit-ibm-runtime/qiskit-runtime-service#job), um den Job abzurufen.

## Ergebnisse auf der Festplatte speichern {#save-results-to-disk}

Du kannst Ergebnisse auch auf der Festplatte speichern. Verwende dazu Pythons integrierte JSON-Bibliothek mit Encodern aus Qiskit Runtime.

```python
import json
from qiskit_ibm_runtime import RuntimeEncoder

with open("result.json", "w") as file:
    json.dump(retrieved_job.result(), file, cls=RuntimeEncoder)
```

Du kannst dieses Array dann in einem separaten Kernel von der Festplatte laden.

```python
from qiskit_ibm_runtime import RuntimeDecoder

with open("result.json", "r") as file:
    result = json.load(file, cls=RuntimeDecoder)

result
```

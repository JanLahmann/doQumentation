## IBM Quantums Open-Source-Inhalte {#ibm-quantums-open-source-content}

IBM stellt eine Fülle von Lernmaterialien zum Quantencomputing bereit – alle Open Source unter CC BY-SA 4.0:

- **[Learning](https://quantum.cloud.ibm.com/learning)** — Strukturierte Kurse von Quantengrundlagen bis zu fortgeschrittenen Themen
- **[Tutorials](https://quantum.cloud.ibm.com/docs/en/tutorials)** — Über 40 Tutorials zu Transpilation, Fehlerminderung und mehr
- **[Documentation](https://quantum.cloud.ibm.com/docs/en/guides)** — Anleitungen und API-Referenz für Qiskit
- **[Source repo](https://github.com/Qiskit/documentation)** — Alle Inhalte auf GitHub

Ihre [Quantum Platform](https://quantum.cloud.ibm.com) ist stets aktuell und gut gestaltet – der beste Ort zum Lesen, Lernen und Nachschlagen.

## Was dieses Projekt hinzufügt {#what-this-project-adds}

IBMs [Qiskit-Dokumentation](https://github.com/Qiskit/documentation) ist Open Source (CC BY-SA 4.0), aber ihre Webanwendung ist es nicht. doQumentation fügt ein Open-Source-Frontend mit Live-Code-Ausführung, automatischer Credential-Injektion und Simulator-Modus hinzu.

Überall einsetzbar – von [GitHub Pages](https://doqumentation.org) über [Docker](https://github.com/JanLahmann/doQumentation/pkgs/container/doqumentation) bis zu [RasQberry](https://rasqberry.org/). Siehe [alle Funktionen](/features).

## Code-Ausführung {#code-execution}

Klicken Sie bei jedem Codeblock auf **Run**. Der erste Klick startet einen kostenlosen Jupyter-Kernel über [Binder](https://mybinder.org) (1–2 Min.). Danach erfolgen Ausführungen sofort.

- **Simulator-Modus (kein Konto erforderlich)** — Aktivieren Sie den [Simulator-Modus](/jupyter-settings#simulator-mode), um alle Notebooks mit AerSimulator oder FakeBackends mit Rauschmodell auszuführen. Null Einrichtungsaufwand erforderlich.

- **IBM Quantum Hardware** — Geben Sie Ihren API-Token und CRN einmal in den [Einstellungen](/jupyter-settings#ibm-quantum) ein — die Zugangsdaten werden bei jeder Ausführung automatisch eingefügt. [Erstellen Sie ein kostenloses Konto](https://quantum.cloud.ibm.com/registration), falls Sie noch keines haben.

<details>
<summary>Verfügbare Ausführungs-Backends</summary>

Jedes Tutorial hat ausführbare Codeblöcke. Klicken Sie auf **Run**, um sie mit einem von drei Backends auszuführen:

1. **Binder** (Standard auf GitHub Pages) — Kostenloser Remote-Jupyter-Kernel über [mybinder.org](https://mybinder.org)
2. **Lokales Jupyter (Docker / RasQberry)** — Verbindet sich mit dem lokalen Jupyter-Server mit vorinstalliertem Qiskit
3. **Benutzerdefinierter Server** — Verweisen Sie in den [Einstellungen](/jupyter-settings#advanced) auf einen beliebigen Jupyter-Endpunkt

</details>

<details>
<summary>Bereitstellungsoptionen</summary>

doQumentation ist verfügbar als:

- **[GitHub Pages](https://doqumentation.org)** — Statische Website mit [Binder](https://mybinder.org) für Remote-Code-Ausführung
- **[Docker](https://github.com/JanLahmann/doQumentation/pkgs/container/doqumentation)** — Vollständiger Stack lokal, offline-fähig
- **[RasQberry](https://rasqberry.org/)** — Selbst gehostet auf Raspberry Pi mit lokalem Jupyter-Kernel

</details>

<details>
<summary>Lokal ausführen mit Podman / Docker</summary>

Installieren Sie [Podman Desktop](https://podman-desktop.io/) oder [Docker Desktop](https://www.docker.com/products/docker-desktop/) (Mac / Windows / Linux), dann führen Sie aus:

```bash
# Vollständiger Stack: Website + Jupyter + Qiskit (~3 GB)
podman run -p 8080:80 -p 8888:8888 ghcr.io/janlahmann/doqumentation:jupyter
```

Öffnen Sie [http://localhost:8080](http://localhost:8080) — Code-Ausführung funktioniert lokal, keine Binder-Wartezeit.

Für eine leichtgewichtige Version ohne lokale Code-Ausführung (~60 MB):

```bash
# Nur statische Website — Code-Ausführung funktioniert weiterhin über Binder
podman run -p 8080:80 ghcr.io/janlahmann/doqumentation:latest
```

Sie verwenden Docker? Ersetzen Sie einfach `podman` durch `docker` — die Befehle sind identisch. Die Images sind Multi-Arch (`linux/amd64` + `linux/arm64`), sodass Apple Silicon Macs nativ funktionieren.

</details>

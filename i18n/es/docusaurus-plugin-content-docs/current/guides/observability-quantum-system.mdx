# Eventos de seguimiento de actividad para el servicio IBM Quantum System {#activity-tracking-events-for-the-ibm-quantum-system-service}

Los servicios de IBM Cloud&reg;, como el servicio IBM&reg; Quantum System, generan eventos de seguimiento de actividad.

Los eventos de seguimiento de actividad informan sobre las actividades que cambian el estado de un servicio en IBM Cloud. Puedes usar los eventos para investigar actividades anómalas y acciones críticas, y para cumplir con los requisitos de auditoría regulatoria.

Puedes usar IBM Cloud Activity Tracker Event Routing, un servicio de plataforma, para enrutar eventos de auditoría en tu cuenta hacia los destinos que elijas configurando destinos y rutas que definen hacia dónde se envían los eventos de seguimiento de actividad. Para más información, consulta [About IBM Cloud Activity Tracker Events Routing](https://cloud.ibm.com/docs/atracker?topic=atracker-about).

Puedes usar IBM Cloud Logs para visualizar y generar alertas sobre eventos generados en tu cuenta y enrutados por IBM Cloud Activity Tracker Event Routing hacia una instancia de IBM Cloud Logs.

## Ver eventos de seguimiento de actividad para el servicio IBM Quantum System {#viewing-activity-tracking-events-for-the-ibm-quantum-system-service}

Puedes usar IBM Cloud Logs para visualizar y generar alertas sobre eventos generados en tu cuenta y enrutados por IBM Cloud Activity Tracker Event Routing hacia una instancia de IBM Cloud Logs.

### Iniciar IBM Cloud Logs desde la página de Observabilidad {#launching-ibm-cloud-logs-from-the-observability-page}

Para obtener información sobre cómo iniciar la interfaz de usuario de IBM Cloud Logs, consulta [Launching the UI](https://cloud.ibm.com/docs/cloud-logs?topic=cloud-logs-instance-launch#instance-launch-cloud-ui) en la documentación de IBM Cloud Logs.

| Acción | Activada por |
| -- | -- |
| `quantum-computing.direct-access-backend-configuration.read` | Obtener configuración del backend (`GET /v1/backends/{backend-name}/configuration`) |
| `quantum-computing.direct-access-backend-properties.read` | Obtener propiedades del backend (`GET /v1/backends/{backend-name}/properties`) |
| `quantum-computing.direct-access-backend.list` | Obtener lista de backends (`GET /v1/backends`) |
| `quantum-computing.direct-access-backend.read` | Obtener detalles del backend (`GET /v1/backends/{backend-name}`) |
| `quantum-computing.direct-access-job.cancel` | Cancelar un trabajo (`POST /v1/jobs/{job-id}/cancel`) |
| `quantum-computing.direct-access-job.create` | Ejecutar un trabajo (`POST /v1/jobs`) |
| `quantum-computing.direct-access-job.delete` | Eliminar un trabajo (`DELETE /v1/jobs/{job-id}`) |
| `quantum-computing.direct-access-job.list` | Obtener trabajos (`GET /v1/jobs`) |
| `quantum-computing.direct-access-job.read` | Obtener trabajo por ID (`GET /v1/jobs/{job-id}`) |
| `quantum-computing.direct-access-lane-configuration.list` | Obtener configuración de carriles del backend (`GET /v1/backends/{backend-name}/lanes`) |

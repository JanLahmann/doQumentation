# Configurar límites de asignación de instancias {#set-instance-allocation-limits}

Una cuenta podría tener tiempo asignado en el plan Premium o Flex. Dado que cada cuenta puede tener muchas instancias en un plan, configurar límites de asignación te ayuda a controlar qué instancias tienen permitido consumir ese tiempo. Ten en cuenta que si la asignación no se configura como un límite, la instancia puede usar más de su tiempo asignado.

El tiempo que asignas a las instancias es utilizado por el algoritmo de reparto equitativo (fair-share scheduler) para determinar la prioridad en la cola, en función del uso de todas las QPUs en una ventana de tiempo móvil de 28 días (para usuarios del Plan Premium) o durante el período de suscripción (para usuarios del Plan Flex).

La asignación se configura al crear la instancia. Sin embargo, puedes visualizar y cambiar el tiempo asignado en la [página de instancias.](https://quantum.cloud.ibm.com/instances)

## Qué sucede cuando se alcanza el límite {#what-happens-when-the-limit-is-reached}

Opcionalmente, puedes configurar la asignación de tiempo como un límite para la instancia. Lo siguiente ocurre cuando una instancia alcanza su límite:

- Las cargas de trabajo activas continúan ejecutándose.
- Los trabajos (jobs) pendientes permanecen en cola.
- No se cancela ninguna carga de trabajo.
- Los colaboradores pueden enviar nuevas cargas de trabajo, las cuales permanecerán en cola hasta que haya más tiempo disponible.
- Para el Plan Flex, la instancia no puede procesar más trabajos a menos que el límite se haya aumentado o, para el Plan Premium, después de que comience la próxima ventana de uso.

## Establecer diferentes valores para el límite de asignación y el límite de uso {#set-different-values-for-the-allocation-limit-and-usage-limit}

Cuando configuras el límite de asignación para una instancia en IBM Quantum Platform, el mismo valor se establece como el límite de uso de la instancia. Si deseas que los usuarios puedan continuar enviando y ejecutando trabajos después de que se haya alcanzado el límite de asignación de la instancia, puedes establecer un valor mayor para el límite de uso utilizando el parámetro [usage_allocation_seconds](/guides/access-instances-platform-apis#parameters) en la API [Update a resource instance](https://cloud.ibm.com/apidocs/resource-controller/resource-controller#update-resource-instance).

# Introducción a los primitivos {#introduction-to-primitives}

Los primitivos se crearon para simplificar las tareas más comunes de las computadoras cuánticas: concretamente, el muestreo de estados cuánticos y el cálculo de valores de expectación. Los primeros primitivos de Qiskit Runtime ([`EstimatorV2`](https://docs.quantum.ibm.com/api/qiskit-ibm-runtime/estimator-v2) y [`SamplerV2`](https://docs.quantum.ibm.com/api/qiskit-ibm-runtime/sampler-v2)) son implementaciones de las [clases base de primitivos](/guides/primitives) de Qiskit. Proporcionan una implementación más sofisticada (por ejemplo, con mitigación de errores incluida) como servicio en la nube y se utilizan para acceder al hardware de IBM Quantum&reg;.

El primitivo más reciente de Qiskit Runtime, [Executor](https://docs.quantum.ibm.com/api/qiskit-ibm-runtime/executor) (que está en versión beta), ofrece una interfaz de nivel inferior que brinda mayor visibilidad y control sin sacrificar el rendimiento.

# Complementos de Qiskit (addons) {#qiskit-addons}

Los complementos de Qiskit (addons) son una colección de capacidades de investigación para permitir el descubrimiento de algoritmos a escala de utilidad. Estas capacidades se basan en la base de alto rendimiento de herramientas de Qiskit para crear y ejecutar algoritmos cuánticos. Se proporcionan como componentes modulares de software que pueden integrarse en un [flujo de trabajo (workflow)](/guides/intro-to-patterns) para escalar o diseñar nuevos algoritmos cuánticos.

## Complementos para el mapeo {#addons-for-mapping}

### Compilación cuántica aproximada con redes tensoriales {#approximate-quantum-compilation-with-tensor-networks}

La compilación cuántica aproximada con redes tensoriales (AQC-Tensor) permite la construcción de circuitos de alta fidelidad con una profundidad reducida.

- Visita el [repositorio de GitHub.](https://github.com/Qiskit/qiskit-addon-aqc-tensor)
- Lee la [documentación.](/guides/qiskit-addons-aqc)
- Lee el tutorial sobre el uso de AQC para [mejorar la evolución temporal trotterizada.](/tutorials/approximate-quantum-compilation-for-time-evolution)

### Fórmulas multiproducto {#multi-product-formulas}

Las fórmulas multiproducto (MPF) reducen el error de Trotter de la dinámica Hamiltoniana a través de una combinación ponderada de varias ejecuciones del circuito.

- Visita el [repositorio de GitHub.](https://github.com/Qiskit/qiskit-addon-mpf)
- Lee la [documentación.](https://qiskit.github.io/qiskit-addon-mpf)

### Optimizador de mapeo {#optimization-mapper}

El complemento Optimization Mapper contiene la funcionalidad para modelar problemas de optimización formulándolos en modelos abstractos y luego convirtiéndolos en representaciones que una computadora cuántica puede entender.

- Lee la [documentación](https://qiskit.github.io/qiskit-addon-opt-mapper/) para guías prácticas y referencias de la API.
- Visita el repositorio de [GitHub](https://github.com/qiskit/qiskit-addon-opt-mapper).

## Complementos para optimización {#addons-for-optimizing}

### Retropropagación de operadores {#operator-backpropagation}

La retropropagación de operadores (OBP, Operator Backpropagation) reduce la profundidad del circuito recortando operaciones del final a costa de más mediciones de operadores.

- Visita el [repositorio de GitHub.](https://github.com/Qiskit/qiskit-addon-obp)
- Lee la [documentación.](/guides/qiskit-addons-obp)
- Lee el tutorial sobre el uso de [OBP para mejorar los valores esperados.](/tutorials/operator-back-propagation)

### Corte de circuitos {#circuit-cutting}

El corte de circuitos reduce la profundidad de los circuitos transpilados descomponiendo las puertas entrelazadas entre qubits no adyacentes.

- Visita el [repositorio de GitHub.](https://github.com/Qiskit/qiskit-addon-cutting)
- Lee la [documentación.](https://qiskit.github.io/qiskit-addon-cutting)

## Complementos para posprocesamiento {#addons-for-post-processing}

### Diagonalización cuántica basada en muestras {#sample-based-quantum-diagonalization}

La diagonalización cuántica basada en muestras (SQD) posprocesa clásicamente las muestras cuánticas con ruido para generar estimaciones de autovalores más precisas de los Hamiltonianos de sistemas cuánticos, por ejemplo en aplicaciones de química.

- Visita el [repositorio de GitHub.](https://github.com/Qiskit/qiskit-addon-sqd)
- Lee la [documentación.](/guides/qiskit-addons-sqd)
- Lee el tutorial sobre la [Diagonalización cuántica basada en muestras de un Hamiltoniano de química.](/tutorials/sample-based-quantum-diagonalization)

### SQD para HPC {#sqd-for-hpc}

Este complemento es una implementación de SQD lista para Computación de Alto Rendimiento (HPC). Está escrito en los estándares modernos C++17 y está diseñado para crear un único binario compilado para usar con MPI.

- Visita la documentación de la [referencia de la API](https://qiskit.github.io/qiskit-addon-sqd-hpc/).
- Ve el repositorio de [GitHub](https://github.com/qiskit/qiskit-addon-sqd-hpc).

## Complementos para la mitigación de errores {#addons-for-error-mitigation}

### M3 (mthree) {#m3-mthree}

La Mitigación de Mediciones sin Matrices (M3) es un paquete para la mitigación escalable de errores de medición cuántica que puede calcularse en paralelo.

- Visita la [documentación](https://qiskit.github.io/qiskit-addon-mthree/) para guías prácticas, tutoriales y referencias de la API.
- Visita la página de [GitHub](https://github.com/Qiskit/qiskit-addon-mthree) del paquete.

### Conos de luz sombreados {#shaded-lightcones}

El complemento de conos de luz sombreados (shaded lightcones) usa la propagación de Pauli para reducir la cantidad de términos de error en un modelo de ruido que necesitan ser mitigados, de acuerdo con las especificaciones del observable objetivo. Esto tiene el efecto de reducir la sobrecarga de muestreo para los flujos de trabajo de cancelación probabilística de errores (PEC, Probabilistic Error Cancellation).

- Visita la [documentación](https://qiskit.github.io/qiskit-addon-slc/) para guías prácticas, tutoriales y referencias de la API.

### Absorción de ruido propagado {#propagated-noise-absorption}

La absorción de ruido propagado (PNA) usa la propagación de Pauli para absorber la información de un modelo de ruido en un observable objetivo. Medir este observable modificado tiene el efecto de mitigar el ruido tal como lo representa el modelo.

- Visita la [documentación](https://qiskit.github.io/qiskit-addon-pna/apidocs/qiskit_addon_pna.html) para guías prácticas, tutoriales y referencias de la API.

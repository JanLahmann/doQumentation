# Preguntas frecuentes (FAQ) {#frequently-asked-questions-faq}

Para preguntas frecuentes específicas sobre los modos de ejecución, consulta la guía [FAQ sobre modos de ejecución de Qiskit Runtime](/guides/execution-modes-faq).

## ¿Cómo cito Qiskit en mi investigación? {#how-do-i-cite-qiskit-in-my-research}

Cita Qiskit utilizando el [archivo BibTeX](https://github.com/Qiskit/qiskit/blob/main/CITATION.bib) incluido.

## ¿Cómo cito un computador IBM Quantum en mi investigación? {#how-do-i-cite-an-ibm-quantum-computer-in-my-research}

Para artículos de investigación, animamos a los autores a reconocer IBM Quantum&reg; usando:

>    Reconocemos el uso de los servicios de IBM Quantum para este trabajo. Las opiniones expresadas son las de los autores y no reflejan la política oficial ni la posición de IBM o del equipo de IBM Quantum.

Las referencias bibliográficas del artículo deben usar la siguiente cita:

> IBM Quantum. https://<span>quantum.cloud.ibm.com</span>/, YYYY

Las unidades de procesamiento cuántico en el artículo deben referenciarse por su nombre único (por ejemplo, `ibmq_vigo`) y, opcionalmente, añadir la versión (por ejemplo, `ibmq_vigo` v1.0.2). También recomendamos hacer referencia al procesador. Por ejemplo:

> En este artículo utilizamos `ibmq_vigo`, que es uno de los procesadores IBM Quantum Canary.

Ejemplo de cita de un programa de IBM Quantum:

> IBM Quantum (2025). Estimator primitive (Version x.y.z) [computer software]. https://<span>quantum.cloud.ibm.com</span>/

## ¿Cómo cito IBM Quantum Composer en mi investigación? {#how-do-i-cite-ibm-quantum-composer-in-my-research}

Ejemplo de cita para IBM Quantum Composer:

> IBM Quantum Composer. 2025. url: https://<span>quantum.cloud.ibm.com/composer</span>

## No veo algo en la interfaz de usuario, como mis tareas de administración. {#i-dont-see-something-in-the-user-interface-such-as-my-administration-tasks}

Prueba a elegir una cuenta diferente desde el selector de cuentas en el encabezado de la interfaz de [IBM Quantum Platform](https://quantum.cloud.ibm.com).

## Solo uso el Plan Gratuito, pero tuve que ingresar mi tarjeta de crédito. ¿Me cobrarán? {#im-just-using-the-open-plan-but-i-had-to-put-in-my-credit-card-will-i-be-charged}

No se te cobrará a menos que crees una [instancia](/guides/instances) de pago por uso (Pay-As-You-Go) y la uses para enviar cargas de trabajo a las QPUs. Consulta [Gestionar costos](/guides/manage-cost) para obtener instrucciones sobre cómo establecer un límite de costo y conocer otras formas de gestionar los costos.

## ¿Cuál es la diferencia entre una clave API y un token API? {#whats-the-difference-between-an-api-key-and-an-api-token}

No hay diferencia. Los términos clave API y token API se usan indistintamente. Verás ambos términos en la documentación porque distintas interfaces usan distintos términos.

<span id="bearer"></span>
## ¿Cuál es la diferencia entre un token API y un token bearer? {#whats-the-difference-between-an-api-token-and-a-bearer-token}

Un token API es una credencial de larga duración que se usa para [autenticarse en el servicio de Qiskit Runtime.](/guides/hello-world#install-and-authenticate) Un token bearer es una credencial temporal que se usa para [autenticarse en la API REST en un entorno no confiable.](/guides/cloud-setup-untrusted#authenticate-to-the-rest-api-in-an-untrusted-environment) Un token bearer no se puede usar para autenticarse en el servicio de Qiskit Runtime.

## ¿Tengo que elegir la región en la que vivo? {#do-i-have-to-choose-the-region-that-i-live-in}

No. Puedes usar cualquiera de las dos regiones. La región determina dónde se ejecutan los trabajos y dónde se almacenan los datos. Además, solo puedes ver y usar las instancias que fueron creadas en la región en la que has iniciado sesión. Ten en cuenta que solo la región us-east es compatible con el Plan Gratuito.

## ¿Puedo conectarme a través de un servidor proxy? {#can-i-connect-through-a-proxy-server}

Sí, pero debes usar Qiskit Runtime v0.44.0 o posterior.

## ¿Cuál es la política de retención de datos? {#what-is-the-data-retention-policy}

IBM conserva tus datos de trabajos y uso (no eliminados) durante tres años mientras continúes usando el servicio de Qiskit Runtime.

## ¿Qué pasó con las distribuciones de Python como Qiskit Aer, Qiskit Nature y Qiskit Experiments? {#what-happened-to-python-distributions-such-as-qiskit-aer-qiskit-nature-and-qiskit-experiments}

Desde el lanzamiento de Qiskit SDK v1.0, el comando `pip install qiskit` instalará **únicamente** el [Qiskit SDK](https://github.com/Qiskit/qiskit). Esto significa que algunos módulos que usabas en el pasado, como `qiskit-aer` o `qiskit-nature`, ahora deben instalarse por separado.

Antes del lanzamiento de Qiskit SDK v1.0, Qiskit estaba compuesto por muchas distribuciones de Python diferentes, como `qiskit-terra`, el núcleo del compilador; `qiskit-aer`, el simulador de alto rendimiento; el proveedor original de IBM Quantum&reg;; y varios paquetes ahora obsoletos que ofrecían características algorítmicas exploratorias o de ejecución de experimentos.

Estos paquetes de software separados se combinaron en lo que se llamó el *metapaquete* de Qiskit, que permitía a los usuarios instalar todos estos componentes juntos mediante `pip install qiskit`. Esta arquitectura de metapaquete se eliminó con el lanzamiento de Qiskit SDK v1.0, y ahora cada una de las distribuciones de software individuales debe instalarse por separado.

Si quieres instalar los paquetes que antes estaban incluidos en el metapaquete de Qiskit, visita el [ecosistema de Qiskit](https://www.ibm.com/quantum/ecosystem) para encontrar una variedad de paquetes que se adapten a tus necesidades. También puedes leer la [guía de migración a v1.0](/guides/metapackage-migration) para obtener más información sobre este cambio.

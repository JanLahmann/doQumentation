# Usar las APIs de IBM Cloud Platform para acceder a instancias {#use-ibm-cloud-platform-apis-to-access-instances}

Esta guía está dirigida a cualquiera que use las APIs de IBM Cloud Platform para aprovisionar, actualizar o gestionar el acceso a instancias. Proporciona detalles sobre los [parámetros](#parameters) que puedes configurar al aprovisionar una instancia, así como el conjunto de [campos de extensión](#extensions) devueltos cuando accedes a una instancia.

<Admonition type="note">
Consulta la documentación de la API ResourceController de IBM Cloud&reg; para más información.

Algunas páginas relevantes:
- [Crear (aprovisionar) una nueva instancia de recurso](https://cloud.ibm.com/apidocs/resource-controller/resource-controller#create-resource-instance)
- [Actualizar una instancia de recurso](https://cloud.ibm.com/apidocs/resource-controller/resource-controller#update-resource-instance)
- [Obtener una instancia de recurso](https://cloud.ibm.com/apidocs/resource-controller/resource-controller#get-resource-instance)
- [Obtener una lista de todas las instancias de recurso](https://cloud.ibm.com/apidocs/resource-controller/resource-controller#list-resource-instances)  
    
    Al obtener una lista de todas las instancias de recurso, ten en cuenta que puedes filtrar solo por instancias de Qiskit Runtime con `resource_id = "b6049020-80f4-11eb-a0f7-e35ec9b4054f"`

</Admonition>

## `parameters` {#parameters}

En el cuerpo de la petición al aprovisionar o actualizar una instancia:

- **instance_limit_seconds** (cadena o nulo, `^[0-9]+$`): El límite a establecer en la instancia. Para el Plan Open: si se omite, el valor predeterminado es 10 minutos.
.
- **usage_allocation_seconds** (cadena o nulo, `^[0-9]+$`): Tiempo asignado para la instancia, usado por el programador de recursos compartidos (fair-share scheduler) para determinar la prioridad en la cola, en función del uso de todas las QPUs. (Esto no se aplica a las instancias Pay-As-You-Go).
- **backends** (matriz de listas de backends (cadenas o nulo) o matriz con "ANY"): la lista de permitidos de nombres de backends que están disponibles para esta instancia. Predeterminado: `["ANY"]`, significa que todos los backends están disponibles en este plan. `[]` significa ningún backend.

## `extensions` {#extensions}

En la respuesta al aprovisionar o al obtener los detalles de una instancia:

Propiedades adicionales de la instancia aportadas por el servicio que se representan como pares clave-valor. Estas propiedades persisten con la instancia bajo la propiedad extensions.

- **instance_limit_seconds** (entero o nulo, `>= 0`): El límite de tiempo a establecer en la instancia.
- **usage_allocation_seconds** (entero o nulo, `>= 0`): Tiempo asignado para la instancia, usado por el programador de recursos compartidos (fair-share scheduler) para determinar la prioridad en la cola, en función del uso de todas las QPUs.
- **backends** (Requerido; matriz de listas de backends (cadenas) o matriz con "ANY" (cadenas)): La lista de permitidos de nombres de backends que están disponibles para esta instancia. Predeterminado: `["ANY"]`, significa que todos los backends están disponibles en este plan. `[]` significa ningún backend.

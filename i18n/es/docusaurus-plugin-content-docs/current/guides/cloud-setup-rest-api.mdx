# Configurar para usar IBM Quantum Platform con la API REST {#set-up-to-use-ibm-quantum-platform-with-rest-api}

Puedes acceder a procesadores cuánticos usando APIs REST, lo que te permite trabajar con las QPUs utilizando cualquier lenguaje de programación o framework.

### 1. Obtener acceso {#get-access}

1. Si aún no tienes una cuenta de usuario, obtén una en la [página de inicio de sesión de IBM Quantum.](https://quantum.cloud.ibm.com/login)
2. Crea una clave API (también llamada token) en el [panel principal.](https://quantum.cloud.ibm.com) Ten en cuenta que la misma clave API se puede usar para ambas regiones.
3. Genera un token al portador (bearer token) de IBM Cloud Identity and Access Management (IAM). Este es un token de corta duración que se utiliza para autenticar solicitudes a la API REST. Para generar uno, llama a la [API de Servicios de Identidad de IAM](https://cloud.ibm.com/apidocs/iam-identity-token-api#create-api-key) como se muestra en la siguiente solicitud de ejemplo:

<Tabs>

   <TabItem value="Curl" label="Curl">
   ```bash
   curl -X POST 'https://iam.cloud.ibm.com/identity/token' \
   -H 'Content-Type: application/x-www-form-urlencoded' \
   -d 'grant_type=urn:ibm:params:oauth:grant-type:apikey&apikey=MI_APIKEY'
   ```
   Respuesta Esperada
   ```text
   {
      "access_token": "eyJhbGciOiJIUz......sgrKIi8hdFs",
      "refresh_token": "SPrXw5tBE3......KBQ+luWQVY=",
      "token_type": "Bearer",
      "expires_in": 3600,
      "expiration": 1473188353
   }
   ```
   </TabItem>

   <TabItem value="Python" label="Python">
   ```python
   # Utiliza 'service' para invocar operaciones.
   import requests
   import json

   url = 'https://iam.cloud.ibm.com/identity/token'
   api_key = 'MI_APIKEY'
   headers = {
    'Content-Type': 'application/x-www-form-urlendcoded',
   }
   data = f'grant_type=urn:ibm:params:oauth:grant-type:apikey&apikey={api_key}'
   response = requests.post(url, headers=headers, data=data)

   # Bearer token (token al portador) para autorizar solicitudes a la API de REST
   bearer_token = response.json()['access_token']
   ```
   </TabItem>
</Tabs>

<span id="environment_variable"></span>
#### Crear una variable de entorno (entorno de confianza) {#create-an-environment-variable-trusted-environment}

1. Para configurar la variable de entorno `IQP_API_TOKEN` en tu sistema, puedes agregar la siguiente línea a tu perfil de shell (por ejemplo, `.bashrc` o `.zshrc`) o estableciéndola directamente en tu terminal:

    ```shell
    export IQP_API_TOKEN=<tu-CLAVE_API> # Usa la clave API_KEY de 44 caracteres que creaste y guardaste desde el panel de inicio (Home dashboard) de IBM Quantum Platform
    ```
    Cuando invoques la variable de entorno en tu código, incluye `import os`, como en este ejemplo:

    ```python
    import os
    api_token = os.environ['IQP_API_TOKEN']
    ```
    Ten en cuenta que al crear una variable de entorno, tu clave API aún se almacena localmente en texto sin formato (plain text), y debe ser salvaguardada.

2. Autentica las solicitudes a la API REST de Qiskit Runtime al incluir el CRN y el token al portador (bearer token) en los encabezados (headers) de la solicitud.
```bash
curl -X 'GET' \
   'https://quantum.cloud.ibm.com/api/v1/usage' \
   '-H accept: application/json' \
   '-H authorization: Bearer <BEARER_TOKEN>' \
   '-H Service-CRN: <INSTANCE_CRN>'
```

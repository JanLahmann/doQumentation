## Contenido de código abierto de IBM Quantum {#ibm-quantums-open-source-content}

IBM proporciona una gran cantidad de material de aprendizaje sobre computación cuántica, todo de código abierto bajo CC BY-SA 4.0:

- **[Learning](https://quantum.cloud.ibm.com/learning)** — Cursos estructurados desde los fundamentos cuánticos hasta temas avanzados
- **[Tutorials](https://quantum.cloud.ibm.com/docs/en/tutorials)** — Más de 40 tutoriales sobre transpilación, mitigación de errores y más
- **[Documentation](https://quantum.cloud.ibm.com/docs/en/guides)** — Guías y referencia de API para Qiskit
- **[Source repo](https://github.com/Qiskit/documentation)** — Todo el contenido en GitHub

Su [Quantum Platform](https://quantum.cloud.ibm.com) está siempre actualizada y bien diseñada: el mejor lugar para leer, aprender y consultar.

## Lo que este proyecto añade {#what-this-project-adds}

La [documentación de Qiskit](https://github.com/Qiskit/documentation) de IBM es de código abierto (CC BY-SA 4.0), pero su aplicación web no lo es. doQumentation añade un frontend de código abierto con ejecución de código en vivo, inyección automática de credenciales y modo simulador.

Desplegable en cualquier lugar: desde [GitHub Pages](https://doqumentation.org) hasta [Docker](https://github.com/JanLahmann/doQumentation/pkgs/container/doqumentation) y [RasQberry](https://rasqberry.org/). Vea [todas las funciones](/features).

## Ejecución de código {#code-execution}

Haga clic en **Run** en cualquier bloque de código. El primer clic inicia un kernel Jupyter gratuito a través de [Binder](https://mybinder.org) (1-2 min). Después de eso, las ejecuciones son instantáneas.

- **Modo Simulador (sin cuenta necesaria)** — Active el [Modo Simulador](/jupyter-settings#simulator-mode) para ejecutar todos los notebooks con AerSimulator o FakeBackends con modelos de ruido. Sin configuración requerida.

- **Hardware Cuántico de IBM** — Ingrese su token de API y CRN en [Configuración](/jupyter-settings#ibm-quantum) una vez: las credenciales se inyectan automáticamente en cada ejecución. [Cree una cuenta gratuita](https://quantum.cloud.ibm.com/registration) si no tiene una.

<details>
<summary>Backends de ejecución disponibles</summary>

Cada tutorial tiene bloques de código ejecutables. Haga clic en **Run** para ejecutarlos usando uno de tres backends:

1. **Binder** (predeterminado en GitHub Pages) — Kernel Jupyter remoto gratuito a través de [mybinder.org](https://mybinder.org)
2. **Jupyter Local (Docker / RasQberry)** — Se conecta al servidor Jupyter local con Qiskit preinstalado
3. **Servidor personalizado** — Apunte a cualquier endpoint Jupyter en [Configuración](/jupyter-settings#advanced)

</details>

<details>
<summary>Opciones de despliegue</summary>

doQumentation está disponible como:

- **[GitHub Pages](https://doqumentation.org)** — Sitio estático con [Binder](https://mybinder.org) para ejecución de código remota
- **[Docker](https://github.com/JanLahmann/doQumentation/pkgs/container/doqumentation)** — Stack completo localmente, capaz de funcionar sin conexión
- **[RasQberry](https://rasqberry.org/)** — Auto-hospedado en Raspberry Pi con kernel Jupyter local

</details>

<details>
<summary>Ejecutar localmente con Podman / Docker</summary>

Instale [Podman Desktop](https://podman-desktop.io/) o [Docker Desktop](https://www.docker.com/products/docker-desktop/) (Mac / Windows / Linux), luego ejecute:

```bash
# Stack completo: sitio + Jupyter + Qiskit (~3 GB)
podman run -p 8080:80 -p 8888:8888 ghcr.io/janlahmann/doqumentation:jupyter
```

Abra [http://localhost:8080](http://localhost:8080): la ejecución de código funciona localmente, sin tiempo de espera de Binder.

Para una versión ligera sin ejecución de código local (~60 MB):

```bash
# Solo sitio estático: la ejecución de código aún funciona a través de Binder
podman run -p 8080:80 ghcr.io/janlahmann/doqumentation:latest
```

¿Usa Docker en su lugar? Simplemente reemplace `podman` por `docker`: los comandos son idénticos. Las imágenes son multi-arquitectura (`linux/amd64` + `linux/arm64`), por lo que las Mac con Apple Silicon funcionan nativamente.

</details>

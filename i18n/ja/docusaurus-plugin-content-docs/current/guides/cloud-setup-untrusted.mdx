# 信頼できない環境でのサービス初期化 {#initialize-the-service-in-an-untrusted-environment}

公共のコンピューターなど、信頼できない環境でIBM CloudまたはIBM Qiskit Runtime REST APIを使用する場合は、以下の手順に従ってください。

**セキュリティリスクを最小限に抑えるため、信頼できないマシンや外部のクラウドPython環境でコードを実行することは避けてください。** やむを得ず信頼できない環境（共有コンピューターや公共のコンピューターなど）を使用する場合は、使用のたびに[APIキーのページ](https://cloud.ibm.com/iam/apikeys)でAPIキーを削除し、新しいものを作成してください。

    <Admonition type="caution">
      **APIキーを守ってください！** ソースコード、Pythonスクリプト、またはノートブックファイルにキーを含めないでください。他の人とコードを共有する際は、APIキーがPythonスクリプトに直接埋め込まれていないことを確認してください。代わりに、トークンを含まないスクリプトを共有し、安全な設定方法を案内してください。

      誤ってキーを他の人と共有したり、Gitなどのバージョン管理システムに含めてしまった場合は、IBM Cloudの[APIキーの削除](https://cloud.ibm.com/docs/account?topic=account-userapikey&interface=ui#delete_user_key)のトピックに従って、すぐにキーを失効させてください。
    </Admonition>

<span id="access-credentials"></span>
## アクセス資格情報の確認 {#find-your-access-credentials}

1. APIキー（_APIトークン_とも呼ばれます）を確認します。[ダッシュボード](https://quantum.cloud.ibm.com/)からAPIキーを作成し、認証に使用できるよう安全な場所にコピーしてください。トークンは再表示されません。なお、1つのAPIキーでどのリージョンにも接続できます。
2. 省略可能：使用するインスタンスを[インスタンス](https://quantum.cloud.ibm.com/instances)ページで確認します。CRNにカーソルを合わせ、コピーアイコンをクリックして、インスタンスの識別に使用できるよう安全な場所に保存してください。

## 信頼できない環境でのIBM Cloudへの認証 {#authenticate-to-ibm-cloud-in-an-untrusted-environment}

この状況でサービスを初期化するには、次のようなコードを使用してください：

    ```python
    from qiskit_ibm_runtime import QiskitRuntimeService

    # The credentials you enter will override any saved
    # account credentials that might be available locally.
    service = QiskitRuntimeService(
      # Use the 44-character API_KEY you created and saved from
      # the IBM Quantum Platform dashboard, and then delete
      # the key on the API keys page after entering this code:
      token="<your-API-key>",

      # Optionally specify an instance to use
      instance="<IBM Cloud CRN or instance name>"
      )
    ```

<span id="rest-untrusted"></span>
## 信頼できない環境でのREST APIへの認証 {#authenticate-to-the-rest-api-in-an-untrusted-environment}

Qiskit Runtime REST APIへの認証にAPIキーを直接使用するには、以下の手順に従ってください。ここで`<your_API_KEY>`は、IBM Quantum Platformのホームダッシュボードで作成・保存した44文字のAPI_KEYです。

1. APIキーを使用して[ベアラートークン](https://docs.quantum.ibm.com/api/qiskit-runtime-rest#authentication)を生成します。_ベアラートークン_は、最大1時間で有効期限が切れる一時的な資格情報です。取得したトークンが期限切れになった場合は、IBM Cloudやその他のサービスAPIの呼び出しを続けるために新しいトークンを生成する必要があります。:
  ```bash
  curl -X POST 'https://iam.cloud.ibm.com/identity/token' \
  -H 'Content-Type: application/x-www-form-urlencoded' \
  -d 'grant_type=urn:ibm:params:oauth:grant-type:apikey&apikey=your_API_KEY'
  ```
  返されたベアラートークンをコピーして保存してください：`"access_token": "<NEW_BEARER_TOKEN>"`
1. リクエストのヘッダーにCRNとベアラートークンを含めることで、Qiskit Runtime REST APIへのリクエストを認証します。

  ```bash
  curl -X 'GET' \
     'https://quantum.cloud.ibm.com/api/v1/usage' \
     '-H accept: application/json' \
     '-H authorization: Bearer <BEARER_TOKEN>' \
     '-H Service-CRN: <INSTANCE_CRN>'
  ```

1. IBM Quantum Platformのホームページで新しいAPIキーを生成するか、IBM Cloudの[APIキーの削除](https://cloud.ibm.com/docs/account?topic=account-userapikey&interface=ui#delete_user_key)の手順に従って、使用のたびにAPIキーを変更してください。

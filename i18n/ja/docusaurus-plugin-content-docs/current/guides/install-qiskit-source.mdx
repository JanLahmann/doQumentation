# ソースからQiskit SDKをインストールする {#install-the-qiskit-sdk-from-source}

ソースからQiskit SDKをインストールすると、Python Package Index（PyPI）リポジトリのバージョンではなく、現在の開発版にアクセスできます。そのため、Qiskitコードの最新バージョンをより効率的に確認・拡張できます。

## 新しい仮想環境を作成して有効化する {#create-and-activate-a-new-virtual-environment}

1. プロジェクトディレクトリに移動し、Pythonのみがインストールされた最小限の環境を作成します。

        <OperatingSystemTabs>
          <TabItem value="mac" label="macOS">
            ```shell
            python3 -m venv .venv
            ```
          </TabItem>

          <TabItem value="linux" label="Linux">
            ```shell
            python3 -m venv .venv
            ```
          </TabItem>

          <TabItem value="win" label="Windows">
            ```text
            python -m venv .venv
            ```
          </TabItem>
         </OperatingSystemTabs>

1. 新しい環境を有効化します。

         <OperatingSystemTabs>
           <TabItem value="mac" label="macOS">
             ```shell
             source .venv/bin/activate
             ```
           </TabItem>

           <TabItem value="linux" label="Linux">
             ```shell
             source .venv/bin/activate
             ```
           </TabItem>

           <TabItem value="win" label="Windows">

## Rustコンパイラをインストールする {#install-the-rust-compiler}

Qiskitをコンパイルするには、システムにRustコンパイラがインストールされている必要があります。Rustコンパイラをインストールするには、クロスプラットフォームのRustインストーラー[`rustup`](https://rustup.rs/)、または[その他のインストール方法](https://forge.rust-lang.org/infra/other-installation-methods.html)を使用してください。

## Qiskitをインストールする {#install-qiskit}

以下の手順でQiskitをインストールします。

1. Qiskitリポジトリをクローンします。

```bash
git clone https://github.com/Qiskit/qiskit.git
```

2. `qiskit`ディレクトリに移動します。

```bash
cd qiskit
```

3. （オプション）テストまたはリントチェックを実行する場合は、開発者向け要件をインストールします。

```bash
pip install -r requirements-dev.txt
```

4. `qiskit`をインストールします。

  * **標準インストール**:

    ```bash
    pip install .
    ```

  * **編集可能モード**: このモードでは、プロジェクトのコードが変更されてもQiskitを再インストールする必要はありません。

    ```bash
    pip install -e .
    ```

    編集可能モードでは、コンパイル済み拡張機能は最適化なしの_デバッグモード_でビルドされます。これはコンパイル済みコードの実行時パフォーマンスに影響します。最適化を有効にしてコンパイル済み拡張機能をビルドするには、次のコマンドを実行してバイナリを_リリースモード_で再ビルドしてください。

    ```bash
    python setup.py build_rust --release --inplace
    ```

    <Admonition type="note">
    QiskitのRustコードを変更している場合、ローカルで変更を加えるたびに拡張コードを再ビルドする必要があります。編集可能モードでは、Rust拡張機能はインストールコマンドの実行時にのみビルドされるため、Rustコードへのローカルの変更は、`build_rust`を再実行して拡張機能を再ビルドしない限りインストール済みパッケージに反映されません（リリースモードとデバッグモードのどちらでビルドするかに応じて、`--release`の有無を選択してください）。
    </Admonition>

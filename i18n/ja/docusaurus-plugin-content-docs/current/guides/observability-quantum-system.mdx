# IBM Quantum System サービスのアクティビティー・トラッキング・イベント {#activity-tracking-events-for-the-ibm-quantum-system-service}

IBM&reg; Quantum System サービスなどの IBM Cloud&reg; サービスは、アクティビティー・トラッキング・イベントを生成します。

アクティビティー・トラッキング・イベントは、IBM Cloud 上のサービスの状態を変更するアクティビティーを記録します。これらのイベントを使用して、異常なアクティビティーや重要なアクションを調査したり、規制上の監査要件に準拠したりすることができます。

IBM Cloud Activity Tracker Event Routing（プラットフォーム・サービス）を使用すると、ターゲットとルートを構成することで、アカウント内の監査イベントを任意の宛先にルーティングできます。詳細については、[IBM Cloud Activity Tracker Events Routing について](https://cloud.ibm.com/docs/atracker?topic=atracker-about)を参照してください。

IBM Cloud Logs を使用すると、アカウントで生成され、IBM Cloud Activity Tracker Event Routing によって IBM Cloud Logs インスタンスにルーティングされたイベントを可視化し、アラートを設定できます。

## IBM Quantum System サービスのアクティビティー・トラッキング・イベントの表示 {#viewing-activity-tracking-events-for-the-ibm-quantum-system-service}

IBM Cloud Logs を使用すると、アカウントで生成され、IBM Cloud Activity Tracker Event Routing によって IBM Cloud Logs インスタンスにルーティングされたイベントを可視化し、アラートを設定できます。

### オブザーバビリティー・ページからの IBM Cloud Logs の起動 {#launching-ibm-cloud-logs-from-the-observability-page}

IBM Cloud Logs UI の起動方法については、IBM Cloud Logs ドキュメントの[UI の起動](https://cloud.ibm.com/docs/cloud-logs?topic=cloud-logs-instance-launch#instance-launch-cloud-ui)を参照してください。

| アクション | トリガー |
| -- | -- |
| `quantum-computing.direct-access-backend-configuration.read` | Backend 構成の取得（`GET /v1/backends/{backend-name}/configuration`） |
| `quantum-computing.direct-access-backend-properties.read` | Backend プロパティーの取得（`GET /v1/backends/{backend-name}/properties`） |
| `quantum-computing.direct-access-backend.list` | Backend 一覧の取得（`GET /v1/backends`） |
| `quantum-computing.direct-access-backend.read` | Backend 詳細の取得（`GET /v1/backends/{backend-name}`） |
| `quantum-computing.direct-access-job.cancel` | ジョブのキャンセル（`POST /v1/jobs/{job-id}/cancel`） |
| `quantum-computing.direct-access-job.create` | ジョブの実行（`POST /v1/jobs`） |
| `quantum-computing.direct-access-job.delete` | ジョブの削除（`DELETE /v1/jobs/{job-id}`） |
| `quantum-computing.direct-access-job.list` | ジョブ一覧の取得（`GET /v1/jobs`） |
| `quantum-computing.direct-access-job.read` | ID によるジョブの取得（`GET /v1/jobs/{job-id}`） |
| `quantum-computing.direct-access-lane-configuration.list` | バックエンドのレーン設定の取得（`GET /v1/backends/{backend-name}/lanes`） |

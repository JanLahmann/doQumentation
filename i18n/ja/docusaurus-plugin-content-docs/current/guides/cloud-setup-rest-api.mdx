# REST APIを使用してIBM Quantum Platformを利用するための設定 {#set-up-to-use-ibm-quantum-platform-with-rest-api}

REST APIを使用して量子プロセッサーにアクセスできます。これにより、任意のプログラミング言語やフレームワークを使ってQPUを操作することが可能になります。

### 1. アクセスの取得 {#1-get-access}

1. ユーザーアカウントをお持ちでない場合は、[IBM Quantumのログインページ](https://quantum.cloud.ibm.com/login)でアカウントを作成してください。
1. [ダッシュボード](https://quantum.cloud.ibm.com)でAPIキー（トークンとも呼ばれます）を作成します。同じAPIキーはどちらのリージョンでも使用できます。
1. IBM Cloud Identity and Access Management（IAM）のBearerトークンを生成します。これはREST APIへのリクエストを認証するために使用される有効期限の短いトークンです。生成するには、以下のサンプルリクエストのように[IAM Identity Services API](https://cloud.ibm.com/apidocs/iam-identity-token-api#create-api-key)を呼び出してください。
<Tabs>

   <TabItem value="Curl" label="Curl">
   ```bash
   curl -X POST 'https://iam.cloud.ibm.com/identity/token' \
   -H 'Content-Type: application/x-www-form-urlencoded' \
   -d 'grant_type=urn:ibm:params:oauth:grant-type:apikey&apikey=MY_APIKEY'
   ```
   期待されるレスポンス
   ```text
   {
      "access_token": "eyJhbGciOiJIUz......sgrKIi8hdFs",
      "refresh_token": "SPrXw5tBE3......KBQ+luWQVY=",
      "token_type": "Bearer",
      "expires_in": 3600,
      "expiration": 1473188353
   }
   ```
   </TabItem>

   <TabItem value="Python" label="Python">
   ```python
   # Use 'service' to invoke operations.
   import requests
   import json

   url = 'https://iam.cloud.ibm.com/identity/token'
   api_key = 'MY_APIKEY'
   headers = {
    'Content-Type': 'application/x-www-form-urlendcoded',
   }
   data = f'grant_type=urn:ibm:params:oauth:grant-type:apikey&apikey={api_key}'
   response = requests.post(url, headers=headers, data=data)

   # Bearer token to authorize requests to the REST API
   bearer_token = response.json()['access_token']
   ```
   </TabItem>
</Tabs>

### 2. 認証方法の選択 {#2-choose-an-authentication-method}

作業環境に応じて、適切な認証方法を選択してください。

* [APIキーの環境変数を作成する（信頼済みのPython環境）](#environment_variable)
* [APIキーを直接使用する（信頼されていない環境）](/guides/cloud-setup-untrusted#rest-untrusted)

<span id="environment_variable"></span>
#### 環境変数の作成（信頼済み環境） {#create-an-environment-variable-trusted-environment}

1. システムにIQP_API_TOKEN環境変数を設定するには、シェルプロファイル（例：.bashrcや.zshrc）に以下の行を追加するか、ターミナルで直接設定してください。

    ```shell
    export IQP_API_TOKEN=<your-API_KEY> # Use the 44-character API_KEY you created and saved from the IBM Quantum Platform Home dashboard
    ```
    コード内で環境変数を呼び出す際は、この例のように `import os` を含めてください。

    ```python
    import os
    api_token = os.environ['IQP_API_TOKEN']
    ```
    環境変数を作成する場合でも、APIキーはローカルに平文で保存されるため、適切に管理してください。

1. CRNとBearerトークンをリクエストのヘッダーに含めることで、Qiskit Runtime REST APIへのリクエストを認証します。
```bash
curl -X 'GET' \
   'https://quantum.cloud.ibm.com/api/v1/usage' \
   '-H accept: application/json' \
   '-H authorization: Bearer <BEARER_TOKEN>' \
   '-H Service-CRN: <INSTANCE_CRN>'
```

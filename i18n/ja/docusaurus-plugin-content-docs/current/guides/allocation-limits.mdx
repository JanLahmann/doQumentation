# インスタンスの割り当て制限を設定する {#set-instance-allocation-limits}

アカウントには、PremiumプランまたはFlexプランで使用できる時間が割り当てられている場合があります。各アカウントはプランに複数のインスタンスを持つことができるため、割り当て制限を設定することで、どのインスタンスがその時間を消費できるかを管理できます。なお、割り当てが制限として設定されていない場合、インスタンスは割り当て時間を超えて使用できます。

インスタンスに割り当てた時間は、フェアシェアアルゴリズムによってキューの優先順位を決定するために使用されます。優先順位は、28日間のローリング時間枠（Premiumプランユーザーの場合）またはサブスクリプション期間（Flexプランユーザーの場合）にわたるすべてのQPUの使用量に基づいて算出されます。

割り当ては、インスタンス作成時に設定されます。ただし、割り当て時間は[インスタンスページ](https://quantum.cloud.ibm.com/instances)で確認および変更できます。

## 制限に達した場合の動作 {#what-happens-when-the-limit-is-reached}

時間の割り当てをインスタンスの制限として任意に設定できます。インスタンスが制限に達した場合、以下の動作が発生します。

- アクティブなワークロードは引き続き実行されます。
- 保留中のジョブはキューに残り続けます。
- ワークロードはキャンセルされません。
- コラボレーターは新しいワークロードを送信でき、利用可能な時間が増えるまでキューに残ります。
- Flexプランの場合、制限が引き上げられない限り、またPremiumプランの場合は次の使用期間が開始されるまで、インスタンスはジョブを処理できません。

## 割り当て制限と使用制限に異なる値を設定する {#set-different-values-for-the-allocation-limit-and-usage-limit}

IBM Quantum Platformでインスタンスの割り当て制限を設定すると、同じ値がインスタンスの使用制限としても設定されます。インスタンスの割り当て制限に達した後もユーザーがジョブを送信・実行し続けられるようにするには、[Update a resource instance](https://cloud.ibm.com/apidocs/resource-controller/resource-controller#update-resource-instance) APIの[usage_allocation_seconds](/guides/access-instances-platform-apis#parameters)パラメーターを使用して、使用制限により高い値を設定できます。

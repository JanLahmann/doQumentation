# モニタリング、キャリブレーション、およびベンチマーキング {#monitoring-calibrations-and-benchmarking}

量子コンピューターのキャリブレーションには、量子 Gate および読み出し操作を動作させる電気信号を定義するさまざまなパラメーターの最適化が必要です。量子コンピューターがリリースされる前の初期キャリブレーションプロセスでは、量子コンピューターの想定される作業負荷に関連するベンチマークに基づいて可能な限り最高のパフォーマンスを達成するために、各パラメーターが徹底的に調整されます。量子コンピューターがリリースされた後の主な目標は、デバイスの寿命を通じて一貫したパフォーマンスを維持することです。キャリブレーションされたパラメーターの多くの最適値は無期限に安定していますが、量子プロセッサーチップ上の二準位系（TLS）の環境変化、データセンターの周囲条件（温度など）の変化、または制御システム内の不安定性などの要因により、時間とともに変動するものもあります。

一貫したパフォーマンスを確保するため、IBM&reg; の量子コンピューターは、経時的に変動する可能性のあるパラメーターを追跡するために頻繁にモニタリングされ、必要に応じてキャリブレーションが実行され、毎日のベンチマーキングが行われます。このページでは、IBM の量子コンピューターフリートが可能な限り安定した、予測可能な状態でユーザーが利用できるよう連携して動作する、モニタリング、キャリブレーション、ベンチマーキングという3つのプロセスについて詳しく説明します。

## モニタリング {#monitoring}

### パラメーターモニタリング {#parameter-monitoring}

簡単なパラメーターモニタリングジョブが約1時間ごとに実行され、完全な Qiskit Runtime ソフトウェアスタックを使用してユーザージョブと自動的に交互に実行されます。これらのジョブの結果は、パラメーターが許容範囲から逸脱し始めていないかを確認するために分析されます。これにより、問題がパフォーマンスに目に見えるほど影響するほど深刻になる前に、理想的なタイミングで検出できます。

モニタリングされるパラメーターの一部には以下が含まれます：

* 読み出し角度、振幅、および識別器閾値。正確な状態識別、低リーケージ、および安定した動作を確保します。これには量子制限増幅器の動作パラメーターも含まれます。
* 単一および2 Qubit Gate 操作。正しい回転角度を維持し、位相および振幅エラーを最小化するために期待どおりに動作していることを確認します。
* TLS 活動のシグニチャー。

これらのモニタリングジョブの結果が期待されるパフォーマンスから適度に逸脱していることを示した場合、適切なキャリブレーションジョブが起動されます。深刻な TLS 活動が検出された場合、影響を受ける Qubit に関連する Gate のキャリブレーション戦略が自動的に変更される可能性があり（キャリブレーションの一時停止を含む場合がある）、TLS 活動が許容レベルまで低下するまで続きます。

### ホリスティックモニタリング {#holistic-monitoring}

個々のパラメーターをモニタリングするジョブに加えて、生成された Bell 状態の忠実度を調べるテストや、それらの機能をサポートする量子コンピューターでの分数 Gate および動的 Circuit のテストなど、量子コンピューターのパフォーマンスをより総合的にモニタリングするジョブもあります。これらのテストは、完全な Qiskit Runtime スタックを通じてユーザージョブと交互に実行され、ハードウェアとソフトウェアの全体的な動作を効率的に検証することを目的としています。これらのテストでパフォーマンスの大幅な低下が検出された場合、問題が解決されるまで量子コンピューターは自動的にジョブキューを一時停止し、デバイスが期待どおりのパフォーマンスを発揮するまでユーザージョブが実行されないことを保証します。

## キャリブレーション {#calibrating}

キャリブレーションは、モニタリングジョブがパルス振幅や角度などのパラメーターが理想的な値から逸脱していることを示したときにトリガーされます。ユーザージョブの合間に1日を通じて実行されるため、キャリブレーションが開始・終了する固定時間帯はありません。これらはパラメーターモニタリングが特定の問題を識別した Qubit/Gate のみで実行され、特定のバッチ処理ルールに従って同時に実行する必要がある Qubit も含まれます。Heron QPU では、キャリブレーションに費やされる合計時間は通常1日あたり2時間未満です。

### 単一 Qubit 操作 {#single-qubit-operations}

これらのキャリブレーションは、単一 Qubit Gate（sx、x、rx（分数））の正確な実装を確保します。以下を調整します：

* Qubit 周波数
* パルス振幅と位相

これらのキャリブレーションは、影響を受ける Qubit 全体でバッチ処理され、適切な場合は並行して実行されます。バッチ処理戦略はキャリブレーションタイプごとに調整されます。

### 2 Qubit 操作 {#two-qubit-operations}

* CZ および RZZ Gate の振幅と位相（Heron および Nighthawk プロセッサー向け）
* ECR Gate の振幅と位相（Eagle プロセッサー向け）

これらのキャリブレーションは、クロストークを最小化するために最近傍でない Qubit のバッチで実行されます。

### 読み出し {#readout}

* 読み出しパルス角度
* 測定識別パラメーター

これらのキャリブレーションは、キャリブレーションが必要な Qubit 上で同時に実行されます。

### キャリブレーションのスケジュール方法 {#how-calibrations-are-scheduled}

* キャリブレーションジョブは、ジョブまたは Session の実行中に同時に実行することはできません。
* したがって、長い Session 中は、キャリブレーションの遅延または頻度の低下により、量子コンピューターの実効的な安定性が低下する可能性があります。
* 同時に送信された2つのジョブは、タイミングによって異なるキャリブレーションセットで実行される場合があります。

## ベンチマーキング {#benchmarking}

毎日のベンチマーキングにより、量子コンピューターのパフォーマンスの包括的なビューが得られ、Qiskit を通じてユーザーに送信されるメトリクスが生成されます。これらはユーザーが Qubit を選択し、コンパイルを最適化し、期待される Circuit パフォーマンスをより適切に予測するのに役立ちます。報告された数値は、プログラムで確認するか、[Compute resources](https://quantum.cloud.ibm.com/computers) ページ（任意の QPU をクリックして詳細情報カードを開く）で確認できます。各メトリクスの詳細は[ドキュメント](/guides/qpu-information)をご覧ください。

<Admonition type="tip" title="Device drift">
デバイスドリフト（時間とともにハードウェアパフォーマンスが低下すること）の影響は、最後にキャリブレーションが実行された時期、実行された特定の実験、TLS 活動など、多くの要因によって異なります。特定のワークロードがデバイスパラメーターのエラー値に非常に敏感な場合、IBM Quantum Learning の[このチュートリアル](/tutorials/real-time-benchmarking-for-qubit-selection)に従ってリアルタイムのデバイスパラメーターベンチマーキングを実行できます。
</Admonition>

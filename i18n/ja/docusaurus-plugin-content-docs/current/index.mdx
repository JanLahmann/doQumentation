## IBM Quantumのオープンソースコンテンツ {#ibm-quantums-open-source-content}

IBMは、量子コンピューティング学習教材を豊富に提供しています。すべてCC BY-SA 4.0ライセンスのオープンソースです:

- **[Learning](https://quantum.cloud.ibm.com/learning)** — 量子基礎から高度なトピックまでの構造化されたコース
- **[Tutorials](https://quantum.cloud.ibm.com/docs/en/tutorials)** — トランスパイル、エラー緩和などに関する40以上のチュートリアル
- **[Documentation](https://quantum.cloud.ibm.com/docs/en/guides)** — QiskitのガイドとAPIリファレンス
- **[Source repo](https://github.com/Qiskit/documentation)** — GitHubのすべてのコンテンツ

彼らの[Quantum Platform](https://quantum.cloud.ibm.com)は常に最新で洗練されたデザインです。読書、学習、リファレンスに最適な場所です。

## このプロジェクトが追加するもの {#what-this-project-adds}

IBMの[Qiskitドキュメント](https://github.com/Qiskit/documentation)はオープンソース(CC BY-SA 4.0)ですが、Webアプリケーションはオープンソースではありません。doQumentationは、ライブコード実行、自動認証情報インジェクション、シミュレーターモードを備えたオープンソースフロントエンドを追加します。

どこでもデプロイ可能 — [GitHub Pages](https://doqumentation.org)から[Docker](https://github.com/JanLahmann/doQumentation/pkgs/container/doqumentation)、[RasQberry](https://rasqberry.org/)まで。[すべての機能](/features)をご覧ください。

## コード実行 {#code-execution}

任意のコードブロックで**Run**をクリックします。最初のクリックで[Binder](https://mybinder.org)経由で無料のJupyterカーネルが起動します(1〜2分)。その後、実行は即座に行われます。

- **シミュレーターモード(アカウント不要)** — [シミュレーターモード](/jupyter-settings#simulator-mode)を有効にすると、AerSimulatorまたはノイズモデルFakeBackendsですべてのノートブックを実行できます。セットアップは一切不要です。

- **IBM Quantumハードウェア** — APIトークンとCRNを[設定](/jupyter-settings#ibm-quantum)に一度入力すれば、認証情報はすべての実行時に自動注入されます。アカウントをお持ちでない場合は、[無料アカウントを作成](https://quantum.cloud.ibm.com/registration)してください。

<details>
<summary>利用可能な実行バックエンド</summary>

すべてのチュートリアルには実行可能なコードブロックがあります。**Run**をクリックして、3つのバックエンドのいずれかを使用して実行します:

1. **Binder** (GitHub Pagesのデフォルト) — [mybinder.org](https://mybinder.org)経由の無料リモートJupyterカーネル
2. **ローカルJupyter (Docker / RasQberry)** — QiskitがプリインストールされたローカルJupyterサーバーに接続
3. **カスタムサーバー** — [設定](/jupyter-settings#advanced)で任意のJupyterエンドポイントを指定

</details>

<details>
<summary>デプロイオプション</summary>

doQumentationは以下として利用可能です:

- **[GitHub Pages](https://doqumentation.org)** — リモートコード実行用の[Binder](https://mybinder.org)を備えた静的サイト
- **[Docker](https://github.com/JanLahmann/doQumentation/pkgs/container/doqumentation)** — ローカルで完全スタック、オフライン対応
- **[RasQberry](https://rasqberry.org/)** — ローカルJupyterカーネルを備えたRaspberry Pi上のセルフホスト

</details>

<details>
<summary>PodmanまたはDockerでローカル実行</summary>

[Podman Desktop](https://podman-desktop.io/)または[Docker Desktop](https://www.docker.com/products/docker-desktop/) (Mac / Windows / Linux)をインストールして、以下を実行します:

```bash
# フルスタック: サイト + Jupyter + Qiskit (~3 GB)
podman run -p 8080:80 -p 8888:8888 ghcr.io/janlahmann/doqumentation:jupyter
```

[http://localhost:8080](http://localhost:8080)を開く — コード実行はローカルで動作し、Binderの待ち時間はありません。

ローカルコード実行なしの軽量版(~60 MB):

```bash
# 静的サイトのみ — コード実行はBinderを介して引き続き機能
podman run -p 8080:80 ghcr.io/janlahmann/doqumentation:latest
```

Dockerを使用する場合は、`podman`を`docker`に置き換えるだけです。コマンドは同じです。イメージはマルチアーキテクチャ(`linux/amd64` + `linux/arm64`)なので、Apple Silicon Macでもネイティブに動作します。

</details>

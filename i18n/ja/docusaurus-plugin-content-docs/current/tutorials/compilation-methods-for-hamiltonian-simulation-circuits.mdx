## 背景 {#background}

量子Circuit のコンパイルは、高レベルの量子アルゴリズムをターゲットハードウェアの制約に準拠した物理回路に変換します。効果的なコンパイルにより、回路の深さとGate数を大幅に削減でき、これは近未来の量子デバイスにおける結果の品質に直接影響します。

このチュートリアルでは、`PauliEvolutionGate`を用いて構築されたハミルトニアンシミュレーション回路について、3つのコンパイル手法をベンチマークします。これらの回路はQubit間のペアワイズ相互作用（$ZZ$、$XX$、$YY$項など）をモデル化しており、量子化学、凝縮系物理学、材料科学において一般的に使用されます。

ベンチマーク回路は[Hamlib](https://github.com/SRI-International/QC-App-Oriented-Benchmarks/tree/master/qedcbench/hamlib#hamlib-simulation---benchmark-program)コレクションから取得し、[Benchpress](https://github.com/Qiskit/benchpress)リポジトリを通じてアクセスします。Hamlibは代表的なハミルトニアンの標準化されたセットを提供しており、現実的なシミュレーションワークロードでコンパイル戦略を比較することができます。

### コンパイル手法の概要 {#compilation-methods-overview}

#### SABREを用いたQiskit Transpiler {#qiskit-transpiler-with-sabre}
Qiskit TranspilerはSABRE（SWAP-based BidiREctional heuristic search）アルゴリズムを使用して回路のレイアウトとルーティングを最適化します。SABREはハードウェアの接続性制約に準拠しながら、SWAPゲートとそれらが回路の深さに与える影響を最小化することに焦点を当てています。汎用的な手法であり、性能とコンパイル時間のバランスが優れています。詳細については[\[1\]](https://arxiv.org/abs/2409.08368)を参照してください。SABREの利点とパラメータ探索については、別の[チュートリアル](/tutorials/transpilation-optimizations-with-sabre)で詳しく扱っています。

#### AI搭載トランスパイラ {#ai-powered-transpiler}
AI搭載トランスパイラは機械学習を使用して、回路構造とハードウェア制約のパターンを分析することで、最適なトランスパイル戦略を予測します。また、パウリネットワーク回路を強化学習ベースの合成アプローチで対象とする`AIPauliNetworkSynthesis`パスを適用することもできます。詳細については[\[2\]](https://arxiv.org/abs/2405.13196)および[\[3\]](https://arxiv.org/abs/2503.14448)を参照してください。

#### Rustiqプラグイン {#rustiq-plugin}
Rustiqプラグインは、トロッター化ダイナミクスで一般的に使用されるパウリ回転を表す`PauliEvolutionGate`演算に特化した高度な合成技術を提供します。ハミルトニアンシミュレーションワークロードに対して低深さの回路分解を生成するように設計されています。詳細については[\[4\]](https://arxiv.org/abs/2404.03280)を参照してください。

### 主要なメトリクス {#key-metrics}

3つの手法を以下のメトリクスで比較します：
- **2量子ビット深さ**：2量子ビットGateのみをカウントした回路の深さ。実際のハードウェアにおける忠実度のボトルネックとなることが多いです。
- **回路サイズ（総Gate数）**：トランスパイルされた回路の総Gate数。
- **実行時間**：トランスパイルに要するウォールクロック時間。

### ヘルパー関数の定義 {#define-helper-functions}

以下の関数は、指定されたパスマネージャーを使用してCircuit のリストをトランスパイルし、各回路の主要なメトリクス（2量子ビット深さ、回路サイズ、実行時間）を記録します。

```python
def capture_transpilation_metrics(
    results, pass_manager, circuits, method_name
):
    """
    Transpile circuits and append one metrics record per circuit to
    ``results``.

    Args:
        results (list): List of dicts to append the metrics records to.
        pass_manager: Pass manager used for transpilation.
        circuits (list): List of quantum circuits to transpile.
        method_name (str): Name of the transpilation method.

    Returns:
        list: List of transpiled circuits.
    """
    transpiled_circuits = []

    for i, qc in enumerate(circuits):
        start_time = time.time()
        transpiled_qc = pass_manager.run(qc)
        end_time = time.time()

        # Decompose swaps for consistency across methods
        transpiled_qc = transpiled_qc.decompose(gates_to_decompose=["swap"])

        transpilation_time = end_time - start_time
        two_qubit_depth = transpiled_qc.depth(
            lambda x: x.operation.num_qubits == 2
        )
        circuit_size = transpiled_qc.size()

        results.append(
            {
                "method": method_name,
                "qc_name": qc.name,
                "qc_index": i,
                "num_qubits": qc.num_qubits,
                "two_qubit_depth": two_qubit_depth,
                "size": circuit_size,
                "runtime": transpilation_time,
            }
        )
        transpiled_circuits.append(transpiled_qc)
        print(
            f"[{method_name}] Circuit {i} ({qc.name}): "
            f"2Q depth={two_qubit_depth}, size={circuit_size}, "
            f"time={transpilation_time:.2f}s"
        )

    return transpiled_circuits
```

```python
def _method_order(results):
    """Return the distinct method names in their first-seen order."""
    order = []
    for r in results:
        if r["method"] not in order:
            order.append(r["method"])
    return order

def print_summary_table(results):
    """
    Print the mean and standard deviation of each metric per compilation
    method, followed by the mean percent improvement relative to SABRE.
    """
    metrics = [
        ("two_qubit_depth", "2Q Depth"),
        ("size", "Gate Count"),
        ("runtime", "Runtime (s)"),
    ]
    methods = _method_order(results)
    by_method = {m: [r for r in results if r["method"] == m] for m in methods}
    sabre_by_index = {r["qc_index"]: r for r in by_method.get("SABRE", [])}

    col_w = 22
    name_w = max(len(m) for m in methods)
    header = f"{'Method':<{name_w}}" + "".join(
        f"  {label:>{col_w}}" for _, label in metrics
    )

    print("Mean +/- std per compilation method")
    print(header)
    print("-" * len(header))
    for method in methods:
        cells = []
        for key, _ in metrics:
            values = [r[key] for r in by_method[method]]
            std = stdev(values) if len(values) > 1 else 0.0
            cells.append(f"{mean(values):,.1f} +/- {std:,.1f}")
        print(
            f"{method:<{name_w}}" + "".join(f"  {c:>{col_w}}" for c in cells)
        )

    others = [m for m in methods if m != "SABRE"]
    if others and sabre_by_index:
        print()
        print("Mean % improvement vs SABRE (positive = better than SABRE)")
        print(header)
        print("-" * len(header))
        for method in others:
            cells = []
            for key, _ in metrics:
                pct = [
                    (sabre_by_index[r["qc_index"]][key] - r[key])
                    / sabre_by_index[r["qc_index"]][key]
                    * 100
                    for r in by_method[method]
                    if sabre_by_index.get(r["qc_index"])
                    and sabre_by_index[r["qc_index"]][key]
                ]
                if pct:
                    std = stdev(pct) if len(pct) > 1 else 0.0
                    cells.append(f"{mean(pct):+.1f}% +/- {std:.1f}%")
                else:
                    cells.append("n/a")
            print(
                f"{method:<{name_w}}"
                + "".join(f"  {c:>{col_w}}" for c in cells)
            )
```

```python
def print_per_circuit_comparison(results, num_rows=5):
    """
    Print a per-metric comparison of the compilation methods for the
    first ``num_rows`` circuits (sorted by qubit count). The best
    (lowest) value for each metric is marked with an asterisk.
    """
    metrics = [
        ("two_qubit_depth", "2Q Depth"),
        ("size", "Gate Count"),
        ("runtime", "Runtime (s)"),
    ]
    methods = _method_order(results)

    by_index = {}
    for r in results:
        by_index.setdefault(r["qc_index"], {})[r["method"]] = r
    ordered = sorted(
        by_index.items(),
        key=lambda kv: (next(iter(kv[1].values()))["num_qubits"], kv[0]),
    )[:num_rows]

    for key, label in metrics:
        print(f"{label} (first {num_rows} circuits by qubit count); * = best")
        header = f"{'Idx':>3}  {'Circuit':<16} {'Q':>3}" + "".join(
            f"{m:>9}" for m in methods
        )
        print(header)
        print("-" * len(header))
        for idx, method_map in ordered:
            any_record = next(iter(method_map.values()))
            present = {
                m: method_map[m][key] for m in methods if m in method_map
            }
            best = min(present.values())
            line = (
                f"{idx:>3}  {any_record['qc_name'][:16]:<16} "
                f"{any_record['num_qubits']:>3}"
            )
            for m in methods:
                value = method_map[m][key]
                text = f"{value:.2f}" if key == "runtime" else f"{int(value)}"
                if value == best:
                    text += "*"
                line += f"{text:>9}"
            print(line)
        print()
```

### HamlibからハミルトニアンCircuit をロードする {#load-hamiltonian-circuits-from-hamlib}

Benchpressリポジトリから代表的なハミルトニアンのセットをロードし、`PauliEvolutionGate`回路を構築します。Backend のQubit数を超えるCircuit は除外し、分解後のサイズが1,500Gateを超える回路も（トランスパイル時間を現実的な範囲に保つために）除外します。

```python
# Obtain the Hamiltonian JSON from the benchpress repository
url = "https://raw.githubusercontent.com/Qiskit/benchpress/e7b29ef7be4cc0d70237b8fdc03edbd698908eff/benchpress/hamiltonian/hamlib/100_representative.json"
response = requests.get(url)
response.raise_for_status()
ham_records = json.loads(response.text)

# Remove circuits that are too large for the backend
ham_records = [
    h for h in ham_records if h["ham_qubits"] <= backend.num_qubits
]

# Build PauliEvolutionGate circuits
qc_ham_list = []
for h in ham_records:
    terms = h["ham_hamlib_hamiltonian_terms"]
    coeff = h["ham_hamlib_hamiltonian_coefficients"]
    num_qubits = h["ham_qubits"]
    name = h["ham_problem"]

    evo_gate = PauliEvolutionGate(SparsePauliOp(terms, coeff))
    qc = QuantumCircuit(num_qubits)
    qc.name = name
    qc.append(evo_gate, range(num_qubits))
    qc_ham_list.append(qc)

# Remove circuits whose decomposed size exceeds 1500 gates so that transpilation completes in a reasonable time frame
qc_ham_list = [qc for qc in qc_ham_list if qc.decompose().size() <= 1500]

print(f"Total Hamiltonian circuits loaded: {len(qc_ham_list)}")
print(
    f"Qubit range: {min(qc.num_qubits for qc in qc_ham_list)} to {max(qc.num_qubits for qc in qc_ham_list)}"
)
```

#### メトリクスごとに最良の手法 {#best-performing-method-by-metric}

以下のグラフは、各メトリクスで最良値（最小値）を達成した手法の割合を示しています。タイも可能です：単純な回路では、複数の手法が同じ最適な2量子ビット深さやGate数に到達できます。タイが発生した場合、すべてのタイした手法がクレジットを受けるため、あるメトリクスのパーセンテージの合計が100%を超える場合があります。

```python
def plot_best_method_bars(results, metrics_list=None):
    """
    Plot a grouped bar chart showing the percentage of circuits
    where each method achieved the best (lowest) value for each metric.

    Ties are counted for all tied methods, so percentages per metric
    can sum to more than 100%.
    """
    if metrics_list is None:
        metrics_list = ["two_qubit_depth", "size", "runtime"]

    labels = {
        "two_qubit_depth": "2Q Depth",
        "size": "Gate Count",
        "runtime": "Runtime",
    }
    methods = _method_order(results)
    palette = {"SABRE": "#1f77b4", "AI": "#ff7f0e", "Rustiq": "#2ca02c"}

    by_index = {}
    for r in results:
        by_index.setdefault(r["qc_index"], []).append(r)
    n_circuits = len(by_index)

    win_data = {m: [] for m in methods}
    tie_counts = []
    metric_labels = []

    for metric in metrics_list:
        metric_labels.append(
            labels.get(metric, metric.replace("_", " ").title())
        )
        counts = Counter()
        ties = 0
        for group in by_index.values():
            min_val = min(r[metric] for r in group)
            best = [r["method"] for r in group if r[metric] == min_val]
            if len(best) > 1:
                ties += 1
            counts.update(best)
        tie_counts.append(ties)
        for m in methods:
            win_data[m].append(counts.get(m, 0) / n_circuits * 100)

    x = np.arange(len(metric_labels))
    width = 0.22
    fig, ax = plt.subplots(figsize=(8, 5))

    for i, method in enumerate(methods):
        bars = ax.bar(
            x + i * width,
            win_data[method],
            width,
            label=method,
            color=palette.get(method, None),
            edgecolor="black",
            linewidth=0.5,
        )
        for bar in bars:
            height = bar.get_height()
            if height > 0:
                ax.text(
                    bar.get_x() + bar.get_width() / 2,
                    height + 1.5,
                    f"{height:.0f}%",
                    ha="center",
                    va="bottom",
                    fontsize=9,
                )

    # Annotate tie counts below each metric label
    for j, ties in enumerate(tie_counts):
        if ties > 0:
            ax.text(
                x[j] + width,
                -8,
                f"({ties} tie{'s' if ties != 1 else ''})",
                ha="center",
                va="top",
                fontsize=8,
                color="gray",
            )

    ax.set_xticks(x + width)
    ax.set_xticklabels(metric_labels, fontsize=11)
    ax.set_ylabel("Circuits with best value (%)", fontsize=11)
    ax.set_title(
        "Best-Performing Method by Metric (ties counted for all tied methods)",
        fontsize=12,
        fontweight="bold",
    )
    ax.legend(frameon=True, fontsize=10)
    ax.set_ylim(-12, 120)
    ax.yaxis.set_major_formatter(ticker.PercentFormatter())
    ax.grid(axis="y", linestyle="--", alpha=0.4)

    plt.tight_layout()
    plt.show()
```

### ステップ3：Qiskitプリミティブを使用した実行 {#step-3-execute-using-qiskit-primitives}

ノイズ下でのトランスパイル品質が実行にどう影響するかを評価するために、**ミラー回路**技術を使用します。各トランスパイル済みCircuit $U$について、その逆回路$U^\dagger$を付加し、組み合わせた回路$U^\dagger U$が理論的に恒等演算となるようにします。$|0\rangle$状態から始めた場合、完璧な（ノイズなし）実行では全ゼロのビット列が確率1で返されます。

実際には、回路全体でGateエラーが蓄積するため、$|0\rangle^{\otimes n}$を回復する確率が低下します。より浅い回路とより少ないGateを生成するコンパイル手法は、蓄積するノイズが少なくなります。

ミラー回路アプローチは魅力的なほどシンプルで、任意の回路サイズにスケールします。期待される出力は常に$|0\rangle^{\otimes n}$であり、理想状態の古典的シミュレーションは不要だからです。ただし、以下の注意点があります：ミラー回路は実際の回路のプロキシです（回路そのものではありません）、Gate数が2倍になります（ノイズの影響を誇張します）、ノイズがミラー境界を対称的にキャンセルする場合、特定のエラーを過小評価する可能性があります。

```python
def remap_to_contiguous(tqc):
    """Remap a transpiled circuit to use contiguous qubit indices.

    Transpiled circuits target specific physical qubits (e.g., qubit 45, 67)
    on a large backend. This remaps them to 0, 1, 2, ... so Aer only
    simulates the active qubits.
    """
    active = sorted(
        {tqc.find_bit(q).index for inst in tqc.data for q in inst.qubits}
    )
    qubit_map = {old: new for new, old in enumerate(active)}
    new_qc = QuantumCircuit(len(active))
    for inst in tqc.data:
        old_indices = [tqc.find_bit(q).index for q in inst.qubits]
        new_qc.append(inst.operation, [qubit_map[i] for i in old_indices])
    return new_qc

def build_mirror_circuit(tqc):
    """Build a mirror circuit: U followed by U-dagger, with measurements.

    The combined circuit U-dagger @ U should be the identity, so measuring
    all zeros indicates a noise-free execution.
    """
    tqc_compact = remap_to_contiguous(tqc)
    mirror = tqc_compact.compose(tqc_compact.inverse())
    mirror.measure_all()
    return mirror

# Build a simple depolarizing noise model
noise_model = NoiseModel()
noise_model.add_all_qubit_quantum_error(
    depolarizing_error(0.001, 1),
    ["sx", "x", "rz"],  # ~0.1% per 1Q gate
)
noise_model.add_all_qubit_quantum_error(
    depolarizing_error(0.01, 2),
    ["cx", "ecr"],  # ~1% per 2Q gate
)

aer_sim = AerSimulator(noise_model=noise_model)

shots = 10000
fidelities = {}

for method, tqc in tqc_methods_small.items():
    mirror = build_mirror_circuit(tqc)

    sampler = SamplerV2(mode=aer_sim)
    job = sampler.run([mirror], shots=shots)
    result = job.result()
    counts = result[0].data.meas.get_counts()

    # Fidelity = fraction of all-zeros (error-free) outcomes
    n_qubits = mirror.num_qubits - mirror.num_clbits  # active qubits
    all_zeros = "0" * mirror.num_qubits
    fidelity = counts.get(all_zeros, 0) / shots
    fidelities[method] = fidelity

        f"{method:8s}  P(|00...0>) = {fidelity:.4f}  ({counts.get(all_zeros, 0)}/{shots})"
    )
```

## コンパイル結果の分析 {#analysis-of-compilation-results}

上記のベンチマークでは、Hamlibコレクションのハミルトニアンシミュレーション回路について、小規模と大規模の両方のスケールでSABRE、AI搭載トランスパイラ、Rustiqを比較しています。

### 2量子ビット深さとGate数 {#two-qubit-depth-and-gate-count}

大規模においては、SABREとAI搭載トランスパイラが最も強力な2手法であり、それぞれ異なるメトリクスでリードしています。*メトリクスごとの最良手法*グラフが示すように、SABREは大部分の回路で最もGate数が少なく、ほぼすべての回路で最速の手法です。これは挿入されたSWAPゲートを最小化するように設計されたヒューリスティックと、最近のレイアウトおよびルーティングの最適化と一致しています。AI搭載トランスパイラは大部分の回路で最も低い2量子ビット深さを達成しており、これは回路深さをターゲットとする強化学習の目標関数の一部と一致しています。要約表も同じ分断を反映しています：SABREはより低い平均Gate数を、AIトランスパイラはより低い平均2量子ビット深さを持っています。両手法とも回路全体にわたって一貫性があり、信頼性があります。

`PauliEvolutionGate`合成のために特別に構築されたRustiqは、大規模回路の一部でのみ単独での最良結果を生成します。その平均メトリクスは、コンパイル比較プロットに大きなスパイクとして見える少数の重大な外れ値によって大きく歪められており、Rustiqが他の手法よりも大幅に高い深さとGate数を生成しています。これらの外れ値なしでは、平均性能はSABREおよびAI搭載トランスパイラにはるかに近くなるでしょう。

重要な観察点は、すべての回路で単一の手法が支配しているわけではないということです。各手法は特定のケースで他の手法を上回るため、利用可能なすべてのツールを試し、各回路に対して最良の結果を選択する価値があります。

### 実行時間 {#runtime}

SABREは一貫して最速の手法です。Rustiqは一般的に同様の速度で動作しますが、コンパイルに著しく長い時間がかかる外れ値が発生することがあります。これは特に大規模な結果で顕著であり、いくつかの回路がRustiqの実行時間を急増させます。これらの外れ値は平均実行時間に大きな影響を与えるため、中央値がRustiqのより代表的なまとめになる可能性があります。AI搭載トランスパイラは3つの中で最も遅く、より大きく複雑な回路では実行時間が著しく増加します。

### ミラー回路の結果 {#mirror-circuit-results}

ミラー回路実験は期待されるトレンドを確認しています：ノイズ下では、2量子ビット深さが低くGate数が少ない手法が高い忠実度を達成します。これはノイズを含むシミュレータ（小規模）と実際のハードウェア（大規模）の両方で成立します。

各ミラー回路プロットは集計ではなく単一の回路を反映していることに留意してください。上記のハードウェア例では1つの26Qubit`tfim`回路を使用しており、これはSABREがAI搭載トランスパイラとRustiqよりもはるかに高い2量子ビット深さを生成するケースであるため、その忠実度は相応に大幅に低くなっています。これは広範な結果を代表するものではありません：大規模回路の全セットにわたって、SABREの2量子ビット深さは通常AI搭載トランスパイラに近く、2つの手法はそれぞれ異なるメトリクスでリードしています（AI搭載トランスパイラは2量子ビット深さで、SABREはGate数と実行時間で）。単一のミラー結果は1つの回路の2倍バージョンをテストするものであり、全体的な手法の品質に対する評決として読むべきではありません。

### 推奨事項 {#recommendations}

すべての回路に対して単一の最良のトランスパイル戦略は存在しません。最良の選択は回路の構造、最適化の目標、および利用可能なコンパイル時間の予算に依存します：

- **SABRE**は推奨されるデフォルトです。高速で信頼性が高く、幅広い回路で優れた結果を生成します。さらなるチューニングには、ユーザーがレイアウトとルーティングのトライアルを増やすことができます（[SABREの最適化チュートリアル](/tutorials/transpilation-optimizations-with-sabre)を参照）。
- **AI搭載トランスパイラ**は、コンパイル時間が制約でない場合、特に2量子ビット深さの最小化が優先事項のときに試す価値があります：このベンチマークの大規模回路の大部分で最も低い2量子ビット深さを達成しました。
- **Rustiq**は`PauliEvolutionGate`回路のために特別に構築されており、特に小規模回路で非常に低い深さと低いGate数の解を見つけることができます。大規模回路では時に大幅に大きな結果を生成することがあるため、デフォルトとして使用するのではなく、試すべき複数の手法の1つとして使用するのが最善です。

実際には、利用可能なすべての手法を実行し、各回路に対して最良の結果を選択することが最善のアプローチです。複数の手法を試すコンパイルオーバーヘッドは、実際のハードウェアでの実行品質の潜在的な改善と比べれば小さいものです。

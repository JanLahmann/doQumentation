## Latar Belakang {#background}
Tutorial ini memaparkan *Pauli Correlation Encoding* (PCE) [\[1\]](#references), sebuah pendekatan yang dirancang untuk mengkodekan masalah optimasi ke dalam qubit dengan efisiensi lebih tinggi untuk komputasi kuantum. PCE memetakan variabel klasik dalam masalah optimasi ke korelasi matriks Pauli multi-bodi, sehingga menghasilkan kompresi polinomial pada kebutuhan ruang masalah tersebut. Dengan menggunakan PCE, jumlah qubit yang diperlukan untuk pengkodean berkurang, sehingga sangat menguntungkan bagi perangkat kuantum near-term yang memiliki sumber daya qubit terbatas. Selain itu, secara analitis telah dibuktikan bahwa PCE secara inheren memitigasi barren plateaus, memberikan ketahanan super-polinomial terhadap fenomena ini. Fitur bawaan ini memungkinkan performa yang belum pernah ada sebelumnya dalam solver optimasi kuantum.
### Ikhtisar {#overview}
Pendekatan PCE terdiri dari tiga langkah utama, seperti yang diilustrasikan pada Gambar 1 dari [\[1\]](#references) di bawah ini:
1. Mengkodekan masalah optimasi ke dalam ruang korelasi Pauli.
2. Menyelesaikan masalah menggunakan solver optimasi kuantum-klasik.
3. Mendekodekan solusi kembali ke ruang optimasi asli.
Pendekatan PCE dapat diadaptasi ke solver optimasi kuantum apa pun yang mampu memproses matriks korelasi Pauli.
![Overview of PCE.](/docs/images/tutorials/solving-maxcut-with-reduced-qubit-requirements-using-pauli-correlation-encoding/af2cb835-88db-4a3d-9c86-51424b1a4bd3.avif)
Pada Gambar 1 dari [\[1\]](#references), masalah [max-cut](/tutorials/quantum-approximate-optimization-algorithm) digunakan sebagai contoh untuk mengilustrasikan pendekatan PCE. Masalah max-cut dengan $m=9$ simpul dikodekan ke dalam ruang korelasi Pauli, merepresentasikan masalah optimasi sebagai matriks korelasi &mdash; khususnya, korelasi matriks Pauli dua-bodi pada $n=3$ qubit $(Q_1, Q_2, Q_3)$. Warna simpul menunjukkan string Pauli yang digunakan untuk setiap simpul yang dikodekan.
Misalnya, simpul 1 yang bersesuaian dengan variabel biner $x_1$ dikodekan oleh nilai ekspektasi dari $Z_1 \otimes Z_2 \otimes I_3$, sedangkan $x_8$ dikodekan oleh $I_1 \otimes Y_2 \otimes Y_3$.
Ini bersesuaian dengan kompresi $m$ variabel masalah menjadi $n = O(m^{1/2})$ qubit. Secara lebih umum, korelasi $k$-bodi memungkinkan kompresi polinomial berorde $k$, dengan $k>1$. Set Pauli yang dipilih terdiri dari tiga subset string Pauli yang saling komutasi, sehingga semua $m$ korelasi dapat diestimasi secara eksperimental hanya dengan tiga pengaturan pengukuran.

Sebuah fungsi kerugian $\mathcal{L}$ dari nilai ekspektasi Pauli yang meniru fungsi objektif max-cut asli dikonstruksi. Fungsi kerugian kemudian dioptimalkan menggunakan solver optimasi kuantum-klasik, seperti [Variational Quantum Eigensolver (VQE)](/learning/courses/quantum-diagonalization-algorithms/vqe).

Setelah optimasi selesai, solusi didekodekan kembali ke ruang optimasi asli, menghasilkan solusi max-cut yang optimal.

```python
    for edge0, edge1 in graph.edge_list():
        if edge0 in partition0 and edge1 in partition1:
```

#### Masalah max-cut {#the-max-cut-problem}
Masalah max-cut adalah masalah optimasi kombinatorial yang didefinisikan pada graf $G = (V, E)$, di mana $V$ adalah himpunan simpul dan $E$ adalah himpunan sisi. Tujuannya adalah membagi simpul menjadi dua himpunan, $S$ dan $V \setminus S$, sedemikian sehingga jumlah sisi antara kedua himpunan tersebut dimaksimalkan.
Untuk deskripsi lengkap masalah max-cut, silakan rujuk tutorial [Quantum approximate optimization algorithm](/tutorials/quantum-approximate-optimization-algorithm).
Masalah max-cut juga digunakan sebagai contoh dalam tutorial [Advanced techniques for QAOA](/tutorials/advanced-techniques-for-qaoa).
Dalam tutorial-tutorial tersebut, algoritma QAOA digunakan untuk menyelesaikan masalah max-cut.

Kita mengkodekan graf dengan 100 simpul ke dalam korelasi matriks Pauli dua-bodi pada sembilan qubit (lihat penjelasan di bawah). Graf direpresentasikan sebagai matriks korelasi, di mana setiap simpul dikodekan oleh sebuah string Pauli. Tanda nilai ekspektasi string Pauli menunjukkan partisi simpul tersebut. Misalnya, simpul 0 dikodekan oleh string Pauli, $\prod_0 = I_{8} \otimes ... I_2 \otimes X_1 \otimes X_0$. Tanda nilai ekspektasi string Pauli ini menunjukkan partisi simpul 0. Kita mendefinisikan sebuah *Pauli-correlation encoding* (PCE) relatif terhadap $\prod$ sebagai

$$ x_i \coloneqq \textit{sgn}(\langle\prod_i \rangle), $$

di mana $x_i$ adalah partisi simpul $i$ dan $\langle \prod_i \rangle \coloneqq  \langle \psi |\prod_i| \psi \rangle $ adalah nilai ekspektasi string Pauli yang mengkodekan simpul $i$ atas keadaan kuantum $|\psi \rangle$.
Sekarang, mari kita kodekan graf ke dalam Hamiltonian menggunakan PCE.
Kita membagi simpul menjadi tiga himpunan: $S_1$, $S_2$, dan $S_3$.
Kemudian, kita mengkodekan simpul-simpul di setiap himpunan menggunakan string Pauli dengan $X$, $Y$, dan $Z$, masing-masing.
Kita perlu mengekstrak hubungan antara jumlah simpul dan qubit yang dibutuhkan untuk mengkodekan semua simpul. Menggunakan semua permutasi yang mungkin untuk pengkodean menghasilkan:

$$
m=3\binom{n}{k}.
$$

Dalam contoh ini kita menggunakan $k=2$, sehingga,

$$
m  = \frac{3}{2} n(n-1).
$$

Oleh karena itu, jumlah qubit $n$ yang dibutuhkan untuk mengekspresikan sejumlah simpul $m$ adalah:

$$
n = \left\lceil \frac{1 + \sqrt{1 + \tfrac{8}{3}m}}{2} \right\rceil.
$$

*Perhatikan bahwa simbol $\lceil \cdot \rceil$ mewakili fungsi ceiling, yang membulatkan bilangan real apa pun ke atas ke bilangan bulat berikutnya. Ini memastikan bahwa jumlah qubit merupakan bilangan bulat.*

```python
num_qubits = int(np.ceil((1 + np.sqrt(1 + (8 / 3) * num_nodes)) / 2))

list_size = num_nodes // 3
node_x = [i for i in range(list_size)]
node_y = [i for i in range(list_size, 2 * list_size)]
node_z = [i for i in range(2 * list_size, num_nodes)]
```

```python
def build_pauli_correlation_encoding(pauli, node_list, n, k=2):
    pauli_correlation_encoding = []
    for idx, c in enumerate(combinations(range(n), k)):
        if idx >= len(node_list):
            break
        paulis = ["I"] * n
        paulis[c[0]], paulis[c[1]] = pauli, pauli
        pauli_correlation_encoding.append(("".join(paulis)[::-1], 1))

    hamiltonian = []
    for pauli, weight in pauli_correlation_encoding:
        hamiltonian.append(SparsePauliOp.from_list([(pauli, weight)]))

    return hamiltonian

pauli_correlation_encoding_x = build_pauli_correlation_encoding(
    "X", node_x, num_qubits
)
pauli_correlation_encoding_y = build_pauli_correlation_encoding(
    "Y", node_y, num_qubits
)
pauli_correlation_encoding_z = build_pauli_correlation_encoding(
    "Z", node_z, num_qubits
)
```

#### Fungsi Loss {#loss-function}
Untuk fungsi loss $\mathcal{L}$, kita menggunakan relaksasi dari fungsi objektif max-cut sebagaimana dijelaskan dalam [\[1\]](#references), yang didefinisikan sebagai $\mathcal{V}(\mathbf{x}) \coloneqq \sum_{(i, j) \in E} W_{i, j}(1-x_i x_j)$. Di sini, $W_{i, j}$ menunjukkan bobot dari sisi $(i, j)$, dan $x_i$ merepresentasikan partisi node $i$.
Fungsi loss $\mathcal{L}$ diberikan oleh:

$\mathcal{L}\coloneqq \sum_{(i, j) \in E} W_{i, j} \text{tanh} (\alpha \langle\prod_i \rangle) \text{tanh} (\alpha \langle\prod_j \rangle) + \mathcal{L}^{(\text{reg})},$

di mana fungsi objektif max-cut digantikan oleh tangen hiperbolik yang mulus dari nilai ekspektasi string Pauli yang mengenkode node-node tersebut. Suku regularisasi $\mathcal{L}^{(\text{reg})}$ dan faktor penskalaan ulang $\alpha$, yang proporsional terhadap jumlah qubit, diperkenalkan untuk meningkatkan performa solver.

Suku regularisasi didefinisikan sebagai:

$\mathcal{L}^{(\text{reg})}$ didefinisikan sebagai $\mathcal{L}^{(\text{reg})} \coloneqq \beta \nu \lbrack \frac{1}{m} \sum_{i \in V} \text{tanh} (\alpha \langle\prod_i \rangle)^2 \rbrack ^2$

di mana $\beta=1/2$, $\nu = |E|/2 + (m -1) /4$, $|E|$ adalah jumlah sisi, dan $m$ adalah jumlah node dalam graph.

```python
def loss_func_estimator(x, ansatz, hamiltonian, estimator, graph):
    """
    Calculates the specified loss function for the given ansatz, Hamiltonian,
    and graph.

    The expectation values of each Pauli string in the Hamiltonian are first
    obtained by running the ansatz on the quantum backend. These
    expectation values are then passed through the nonlinear function
    tanh(alpha * prod_i). The loss function is
    subsequently computed from these transformed values.
    """
    job = estimator.run(
        [
            (ansatz, hamiltonian[0], x),
            (ansatz, hamiltonian[1], x),
            (ansatz, hamiltonian[2], x),
        ]
    )
    result = job.result()

    # calculate the loss function
    node_exp_map = {}
    idx = 0
    for r in result:
        for ev in r.data.evs:
            node_exp_map[idx] = ev
            idx += 1

    loss = 0
    alpha = num_qubits
    for edge0, edge1 in graph.edge_list():
        loss += np.tanh(alpha * node_exp_map[edge0]) * np.tanh(
            alpha * node_exp_map[edge1]
        )

    regulation_term = 0
    for i in range(len(graph.nodes())):
        regulation_term += np.tanh(alpha * node_exp_map[i]) ** 2
    regulation_term = regulation_term / len(graph.nodes())
    regulation_term = regulation_term**2
    beta = 1 / 2
    v = len(graph.edges()) / 2 + (len(graph.nodes()) - 1) / 4
    regulation_term = beta * v * regulation_term

    loss = loss + regulation_term

    global experiment_result
    print(f"Iter {len(experiment_result)}: {loss}")
    experiment_result.append({"loss": loss, "exp_map": node_exp_map})
    return loss
```

```python
# Calculate the partitions based on the final expectation values
# If the expectation value is positive, the node belongs to partition 0 (par0)
# Otherwise, the node belongs to partition 1 (par1)
def get_partitions(experiment_result):
    par0, par1 = set(), set()
    best_index = min(
        range(len(experiment_result)),
        key=lambda i: experiment_result[i]["loss"],

        if experiment_result[best_index]["exp_map"][i] >= 0:
            par0.add(i)
```

```python
# Swap the partitions and calculate the cut size

def swap_partitions(graph, cur_bits):
    best_cut = 0
    best_bits = []
    for edge0, edge1 in graph.edge_list():
        swapped_bits = cur_bits.copy()
        swapped_bits[edge0], swapped_bits[edge1] = (
            swapped_bits[edge1],
            swapped_bits[edge0],
        )

        cur_partition = [set(), set()]
        for i, bit in enumerate(swapped_bits):
            if bit > 0:
                cur_partition[0].add(i)
            else:
                cur_partition[1].add(i)
        cut_size = calc_cut_size(graph, cur_partition[0], cur_partition[1])
        if best_cut < cut_size:
            best_cut = cut_size
            best_bits = swapped_bits
    return best_cut, best_bits

best_cut, best_bits = swap_partitions(graph, cur_bits)
print(best_cut, best_bits)
```

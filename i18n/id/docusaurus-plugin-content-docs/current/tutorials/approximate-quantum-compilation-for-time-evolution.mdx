## Latar Belakang {#background}

Tutorial ini menunjukkan cara mengimplementasikan **Kompilasi Kuantum Aproksimasi** menggunakan tensor network (AQC-Tensor) dengan Qiskit untuk meningkatkan performa sirkuit kuantum. AQC-Tensor mengompresi sirkuit Trotter yang dalam menjadi sirkuit yang lebih dangkal dan ramah perangkat keras sambil mempertahankan akurasi simulasi.

### Cara kerja AQC-Tensor {#how-aqc-tensor-works}

Pertimbangkan simulasi Hamiltonian $H$ untuk waktu total $t$ menggunakan $k$ langkah Trotter. Sirkuit Trotter penuh adalah:

$$
U_{\text{full}} = \left[U_{\text{Trotter}}(t/k)\right]^k
$$

Pendekatan naif menggunakan sedikit langkah Trotter untuk menjaga kedalaman sirkuit tetap terkelola, tetapi ini memperkenalkan error Trotter yang signifikan. AQC-Tensor menyelesaikan ketegangan ini dengan memisahkan akurasi dari kedalaman:

1. **Sirkuit target (akurasi tinggi, dalam):** Bangun sirkuit Trotter dengan banyak langkah&mdash;misalnya, $10k$&mdash;untuk waktu evolusi yang sama. Sirkuit ini memiliki error Trotter yang jauh lebih sedikit, tetapi terlalu dalam untuk perangkat keras. Karena hanya disimulasikan secara klasik sebagai matrix product state (MPS), kedalaman bukanlah masalah.

2. **Sirkuit ansatz (kedalaman rendah, terparametrisasi):** Definisikan sirkuit terparametrisasi $V(\theta)$ dengan struktur yang sama seperti sirkuit Trotter satu langkah. Inisialisasi sehingga $V(\theta_{\text{init}}) = U_{\text{Trotter}}(t/k)$, lalu iteratif optimalkan $\theta$ sehingga $V(\theta)$ mereproduksi state target berkeakurasi tinggi sedekat mungkin.

Hasilnya adalah sirkuit yang mempertahankan kedalaman satu langkah Trotter tetapi mencapai akurasi banyak langkah, membuatnya layak untuk perangkat keras kuantum near-term.

### Kapan menggunakan AQC-Tensor {#when-to-use-aqc-tensor}

AQC-Tensor paling efektif ketika:

- **Kedalaman sirkuit melebihi waktu koherensi perangkat keras.** Jika simulasi Trotter membutuhkan lebih banyak langkah Trotter daripada yang dapat didukung perangkat, AQC-Tensor dapat mengompresi evolusi menjadi sirkuit yang lebih dangkal.
- **Entanglement tetap dapat dihitung secara klasik.** Total entanglement dalam state yang berevolusi waktu bergantung terutama pada waktu evolusi $t$, bukan jumlah langkah Trotter $k$. Ini berarti sirkuit target dengan $10k$ langkah biasanya tidak lebih sulit untuk direpresentasikan sebagai MPS daripada dengan $k$ langkah, selama $t$ cukup pendek agar dimensi bond tetap terkelola.
- **Ada ansatz alami.** Karena ansatz mencerminkan struktur sirkuit Trotter, ansatz memberikan titik awal yang termotivasi secara fisik dengan parameter awal yang terdefinisi dengan baik, menghindari masalah konvergensi yang dapat menghantui ansatz variasional sewenang-wenang.

Pendekatan ini berbeda dengan kompresi sirkuit generik: alih-alih mencoba mengaproksimasi unitary sewenang-wenang dengan lebih sedikit gate, AQC-Tensor mempertahankan struktur gate yang sama dan mengoptimalkan parameternya untuk mengurangi error Trotter. Lihat [dokumentasi AQC-Tensor](https://qiskit.github.io/qiskit-addon-aqc-tensor/) untuk informasi lebih lanjut.

Tutorial ini memandu kamu melalui alur kerja persiapan state AQC-Tensor secara lengkap: mendefinisikan Hamiltonian, membuat sirkuit Trotter, mengompresnya melalui optimasi tensor network, dan mengeksekusi hasilnya di perangkat keras IBM Quantum&reg;.

## Contoh simulator skala kecil {#small-scale-simulator-example}

Bagian ini menggunakan sistem 10-situs untuk mengilustrasikan alur kerja AQC-Tensor langkah demi langkah. Kita mensimulasikan dinamika rantai spin XXZ 10-situs, model yang banyak dipelajari untuk mengkaji interaksi spin dan sifat magnetik.

Hamiltoniannya adalah sebagai berikut:
$$
\hat{\mathcal{H}}_{XXZ} = \sum_{i=1}^{L-1} J_{i,(i+1)}\left(X_i X_{(i+1)}+Y_i Y_{(i+1)}+ 2\cdot Z_i Z_{(i+1)} \right) \, ,
$$

di mana $J_{i,(i+1)}$ adalah koefisien acak untuk edge $(i, i+1)$ dan $L=10$.
### Langkah 1: Petakan input klasik ke masalah kuantum {#step-1-map-classical-inputs-to-a-quantum-problem}

Pada langkah ini, kita:
1. Mendefinisikan Hamiltonian, observabel, dan state awal.
2. Menghitung nilai ekspektasi yang tepat secara klasik untuk perbandingan nantinya.
3. Membuat sirkuit Trotter berkeakurasi tinggi (target AQC) dan mengompresnya menjadi ansatz berkedalaman rendah menggunakan AQC-Tensor.
#### Menyiapkan Hamiltonian, observabel, dan state awal {#set-up-the-hamiltonian-observable-and-initial-state}

```python
# L is the number of sites in the 1D spin chain
L = 10

# Generate the coupling map
edge_list = [(i - 1, i) for i in range(1, L)]
even_edges = edge_list[::2]
odd_edges = edge_list[1::2]
coupling_map = CouplingMap(edge_list)

# Generate random coefficients for our XXZ Hamiltonian
np.random.seed(0)
Js = np.random.rand(L - 1) + 0.5 * np.ones(L - 1)
hamiltonian = SparsePauliOp(Pauli("I" * L))
for i, edge in enumerate(even_edges + odd_edges):
    hamiltonian += SparsePauliOp.from_sparse_list(
        [
            ("XX", (edge), Js[i] / 2),
            ("YY", (edge), Js[i] / 2),
            ("ZZ", (edge), Js[i]),
        ],
        num_qubits=L,
    )

# Generate a ZZ observable between the two middle qubits
observable = SparsePauliOp.from_sparse_list(
    [("ZZ", (L // 2 - 1, L // 2), 1.0)], num_qubits=L
)

# Generate an initial Néel state |1010101010⟩
initial_state_circuit = QuantumCircuit(L)
for i in range(L):
    if i % 2:
        initial_state_circuit.x(i)

print("Hamiltonian:", hamiltonian)
print("Observable:", observable)
graphviz_draw(coupling_map.graph, method="circo")
```

#### Menghitung nilai ekspektasi yang tepat {#compute-the-exact-expectation-value}

Untuk sistem seukuran ini, kita dapat menghitung nilai ekspektasi yang tepat dari waktu-evolusi secara langsung menggunakan eksponen matriks. Ini berfungsi sebagai kebenaran dasar kita untuk mengevaluasi akurasi sirkuit AQC.

```python
aqc_evolution_time = 0.2

# Each baseline Trotter step covers dt = aqc_evolution_time / 3
# The subsequent (uncompressed) step covers 1 additional dt
subsequent_evolution_time = aqc_evolution_time / 3
total_evolution_time = aqc_evolution_time + subsequent_evolution_time

# Compute exact expectation value via matrix exponentiation
H_matrix = hamiltonian.to_matrix()
U_exact = expm(-1j * H_matrix * total_evolution_time)

# Build the initial state vector (Néel state)
initial_state_vec = np.zeros(2**L)
state_idx = sum(2**i for i in range(L) if i % 2)
initial_state_vec[state_idx] = 1.0

# Evolve and compute expectation value
evolved_state = U_exact @ initial_state_vec
obs_matrix = observable.to_matrix()
exact_expval = (evolved_state.conj() @ obs_matrix @ evolved_state).real

print(f"AQC evolution time: {aqc_evolution_time}")
print(f"Subsequent evolution time: {subsequent_evolution_time:.6f}")
print(f"Total evolution time: {total_evolution_time:.6f}")
print(f"Exact expectation value: {exact_expval:.6f}")
```

#### Membuat sirkuit target AQC {#generate-the-aqc-target-circuit}

Kita sekarang membangun sirkuit Trotter yang akan berfungsi sebagai target AQC. Sirkuit ini menggunakan banyak langkah Trotter (32) untuk akurasi yang tinggi. Karena hanya akan disimulasikan secara klasik sebagai MPS&mdash;bukan dieksekusi di perangkat keras&mdash;kedalaman yang besar bukanlah masalah.

```python
aqc_target_num_trotter_steps = 32

aqc_target_circuit = initial_state_circuit.copy()
aqc_target_circuit.compose(
    generate_time_evolution_circuit(
        hamiltonian,
        synthesis=SuzukiTrotter(reps=aqc_target_num_trotter_steps),
        time=aqc_evolution_time,
    ),
    inplace=True,
)
```

#### Membuat ansatz, parameter awal, sirkuit berikutnya, dan sirkuit baseline {#generate-an-ansatz-initial-parameters-subsequent-circuit-and-a-baseline-circuit}

Selanjutnya, kita membangun sirkuit "baik" dengan waktu evolusi yang sama dengan target AQC tetapi jauh lebih sedikit langkah Trotter (hanya satu). Kita meneruskan sirkuit ini ke `generate_ansatz_from_circuit`, yang mengembalikan:

1. Sirkuit **ansatz** terparametrisasi yang umum dengan konektivitas dua-qubit yang sama.
2. **Parameter awal** yang mereproduksi sirkuit input ketika dimasukkan ke dalam ansatz.

Kita juga membangun:
- Sebuah **sirkuit berikutnya** dengan satu langkah Trotter yang akan ditambahkan (tidak dikompres) setelah bagian yang dioptimalkan AQC, mengikuti pendekatan dalam [tutorial state awal AQC-Tensor](https://qiskit.github.io/qiskit-addon-aqc-tensor/tutorials/01_initial_state_aqc.html).
- Sebuah **sirkuit Trotter baseline** menggunakan empat langkah Trotter selama waktu evolusi penuh (`aqc_evolution_time + subsequent_evolution_time`). Ini berfungsi sebagai pembanding: mewakili apa yang akan kamu jalankan di perangkat keras tanpa AQC. Ansatz AQC (3 langkah terkompres + 1 langkah tidak terkompres) mencapai akurasi yang lebih baik pada kedalaman yang lebih rendah.

```python
aqc_ansatz_num_trotter_steps = 1

aqc_good_circuit = initial_state_circuit.copy()
aqc_good_circuit.compose(
    generate_time_evolution_circuit(
        hamiltonian,
        synthesis=SuzukiTrotter(reps=aqc_ansatz_num_trotter_steps),
        time=aqc_evolution_time,
    ),
    inplace=True,
)

aqc_ansatz, aqc_initial_parameters = generate_ansatz_from_circuit(
    aqc_good_circuit
)

# Subsequent circuit: 1 non-compressed Trotter step appended after AQC
subsequent_num_trotter_steps = 1
subsequent_circuit = generate_time_evolution_circuit(
    hamiltonian,
    synthesis=SuzukiTrotter(reps=subsequent_num_trotter_steps),
    time=subsequent_evolution_time,
)

# Baseline Trotter circuit: 4 Trotter steps over total evolution time, no AQC
baseline_num_trotter_steps = 4
baseline_circuit = initial_state_circuit.copy()
baseline_circuit.compose(
    generate_time_evolution_circuit(
        hamiltonian,
        synthesis=SuzukiTrotter(reps=baseline_num_trotter_steps),
        time=total_evolution_time,
    ),
    inplace=True,
)

print(
    f"Target circuit:      depth {aqc_target_circuit.depth(lambda x: x.operation.num_qubits == 2)}"
)
print(
    f"Baseline circuit:    depth {baseline_circuit.depth(lambda x: x.operation.num_qubits == 2)} ({baseline_num_trotter_steps} Trotter steps, time={total_evolution_time:.4f})"
)
print(
    f"Subsequent circuit:  depth {subsequent_circuit.depth(lambda x: x.operation.num_qubits == 2)} ({subsequent_num_trotter_steps} Trotter step, time={subsequent_evolution_time:.4f})"
)
print(
    f"Ansatz circuit:      depth {aqc_ansatz.depth(lambda x: x.operation.num_qubits == 2)}, with {len(aqc_initial_parameters)} parameters"
)
aqc_ansatz.draw("mpl", fold=-1)
```

#### Menyiapkan simulasi tensor network dan membangun MPS target {#set-up-tensor-network-simulation-and-build-the-target-mps}

Kita menggunakan simulator sirkuit matrix-product state (MPS) [quimb](https://github.com/jcmgray/quimb), dengan JAX menyediakan diferensiasi otomatis untuk optimasi berbasis gradient. Kita kemudian membangun representasi MPS dari state target dan mengevaluasi fidelitas awal antara ansatz awal dan target. Karena contoh masalahnya cukup kecil, fidelitas awal dimulai cukup tinggi.

```python
simulator_settings = QuimbSimulator(
    quimb.tensor.CircuitMPS, autodiff_backend="jax"
)

    aqc_target_circuit, simulator_settings

starting_fidelity = abs(compute_overlap(good_mps, aqc_target_mps)) ** 2
```

#### Mengoptimalkan parameter ansatz {#optimize-the-ansatz-parameters}

Kita meminimalkan fungsi biaya `MaximizeStateFidelity` menggunakan optimizer L-BFGS-B. Optimizer secara iteratif menyesuaikan parameter ansatz untuk memaksimalkan fidelitas antara sirkuit ansatz dan MPS target.

```python
aqc_stopping_fidelity = 1
aqc_max_iterations = 500

stopping_point = 1.0 - aqc_stopping_fidelity
objective = MaximizeStateFidelity(
    aqc_target_mps, aqc_ansatz, simulator_settings
)

def callback(intermediate_result: OptimizeResult):
    fidelity = 1 - intermediate_result.fun
    print(
        f"{datetime.datetime.now()} Intermediate result: Fidelity {fidelity:.8f}"
    )
    if intermediate_result.fun < stopping_point:
        raise StopIteration

result = minimize(
    objective,
    aqc_initial_parameters,
    method="L-BFGS-B",
    jac=True,
    options={"maxiter": aqc_max_iterations},
    callback=callback,
)
if result.status not in (0, 1, 99):
    raise RuntimeError(
        f"Optimization failed: {result.message} (status={result.status})"
    )

print(f"Done after {result.nit} iterations.")
aqc_final_parameters = result.x
```

### Langkah 2: Optimalkan masalah untuk eksekusi perangkat keras kuantum {#step-2-optimize-problem-for-quantum-hardware-execution}

Untuk contoh skala kecil ini, kita menggunakan backend palsu (`FakeKyiv`) untuk mensimulasikan eksekusi perangkat keras secara lokal. Kita melakukan transpilasi sirkuit yang dioptimalkan AQC (`aqc_final_circuit`) dan sirkuit Trotter baseline (`baseline_circuit`, empat langkah Trotter selama waktu evolusi penuh, tanpa AQC) ke instruction set architecture (ISA) backend, dengan `optimization_level=3` untuk lebih mengurangi kedalaman sirkuit.

```python
backend = FakeKyiv()

    backend=backend, optimization_level=3

# Transpile the AQC-optimized circuit (compressed + subsequent step)
isa_circuit = pass_manager.run(aqc_final_circuit)
isa_observable = observable.apply_layout(isa_circuit.layout)
print(
    "AQC circuit depth:",
    isa_circuit.depth(lambda x: x.operation.num_qubits == 2),
)

# Transpile the baseline Trotter circuit (no AQC optimization)
isa_baseline_circuit = pass_manager.run(baseline_circuit)
isa_baseline_observable = observable.apply_layout(isa_baseline_circuit.layout)
print(
    "Baseline Trotter circuit depth:",
    isa_baseline_circuit.depth(lambda x: x.operation.num_qubits == 2),
)
```

## Contoh perangkat keras skala besar {#large-scale-hardware-example}

Kita sekarang meningkatkan skala ke model XXZ 50-situs untuk mendemonstrasikan AQC-Tensor pada ukuran masalah yang lebih realistis. Alur kerjanya sama dengan contoh skala kecil: kita mengompresi tiga langkah Trotter melalui AQC dan menambahkan satu langkah tidak terkompres.

Untuk sistem seukuran ini, eksponen matriks tidak layak dilakukan ($2^{50}$ dimensi), sehingga kita menghitung nilai ekspektasi referensi langsung dari MPS berkeakurasi tinggi yang berevolusi selama waktu penuh.

### Langkah 1–4 digabungkan {#steps-1-4-combined}

```python
# -------------------------Step 1-------------------------

# Define the 50-site spin chain
L = 50
edge_list = [(i - 1, i) for i in range(1, L)]
even_edges = edge_list[::2]
odd_edges = edge_list[1::2]
coupling_map = CouplingMap(edge_list)

# Random XXZ Hamiltonian
np.random.seed(0)
Js = np.random.rand(L - 1) + 0.5 * np.ones(L - 1)
hamiltonian = SparsePauliOp(Pauli("I" * L))
for i, edge in enumerate(even_edges + odd_edges):
    hamiltonian += SparsePauliOp.from_sparse_list(
        [
            ("XX", (edge), Js[i] / 2),
            ("YY", (edge), Js[i] / 2),
            ("ZZ", (edge), Js[i]),
        ],
        num_qubits=L,
    )

observable = SparsePauliOp.from_sparse_list(
    [("ZZ", (L // 2 - 1, L // 2), 1.0)], num_qubits=L
)

# Initial Néel state
initial_state_circuit = QuantumCircuit(L)
for i in range(L):
    if i % 2:
        initial_state_circuit.x(i)

# Time parameters
aqc_evolution_time = 0.2
subsequent_evolution_time = aqc_evolution_time / 3
total_evolution_time = aqc_evolution_time + subsequent_evolution_time

# AQC target circuit (high-accuracy, 32 Trotter steps for AQC portion)
aqc_target_num_trotter_steps = 32

aqc_target_circuit = initial_state_circuit.copy()
aqc_target_circuit.compose(
    generate_time_evolution_circuit(
        hamiltonian,
        synthesis=SuzukiTrotter(reps=aqc_target_num_trotter_steps),
        time=aqc_evolution_time,
    ),
    inplace=True,
)

# Generate ansatz from 1-step Trotter circuit
aqc_good_circuit = initial_state_circuit.copy()
aqc_good_circuit.compose(
    generate_time_evolution_circuit(
        hamiltonian,
        synthesis=SuzukiTrotter(reps=1),
        time=aqc_evolution_time,
    ),
    inplace=True,
)

aqc_ansatz, aqc_initial_parameters = generate_ansatz_from_circuit(
    aqc_good_circuit
)

# Subsequent circuit: 1 non-compressed Trotter step
subsequent_circuit = generate_time_evolution_circuit(
    hamiltonian,
    synthesis=SuzukiTrotter(reps=1),
    time=subsequent_evolution_time,
)

# Baseline Trotter circuit: 4 Trotter steps over total evolution time, no AQC
baseline_num_trotter_steps = 4
baseline_circuit = initial_state_circuit.copy()
baseline_circuit.compose(
    generate_time_evolution_circuit(
        hamiltonian,
        synthesis=SuzukiTrotter(reps=baseline_num_trotter_steps),
        time=total_evolution_time,
    ),
    inplace=True,
)
print(
    f"Target circuit:  depth {aqc_target_circuit.depth(lambda x: x.operation.num_qubits == 2)}"
)
print(
    f"Ansatz circuit:  depth {aqc_ansatz.depth(lambda x: x.operation.num_qubits == 2)}, with {len(aqc_initial_parameters)} parameters"
)
print(
    f"Subsequent circuit: depth {subsequent_circuit.depth(lambda x: x.operation.num_qubits == 2)}"
)
print(
    f"Baseline circuit:   depth {baseline_circuit.depth(lambda x: x.operation.num_qubits == 2)} ({baseline_num_trotter_steps} steps, time={total_evolution_time:.4f})"
)

# Build target MPS and compute reference expectation value
simulator_settings = QuimbSimulator(
    quimb.tensor.CircuitMPS, autodiff_backend="jax"
)
aqc_target_mps = tensornetwork_from_circuit(
    aqc_target_circuit, simulator_settings
)
print("Target MPS maximum bond dimension:", aqc_target_mps.psi.max_bond())

# For the reference expectation value, we need the full evolution (AQC + subsequent)
# Build a high-accuracy full circuit for MPS reference
full_target_circuit = initial_state_circuit.copy()
full_target_circuit.compose(
    generate_time_evolution_circuit(
        hamiltonian,
        synthesis=SuzukiTrotter(reps=aqc_target_num_trotter_steps),
        time=total_evolution_time,
    ),
    inplace=True,
)
full_target_mps = tensornetwork_from_circuit(
    full_target_circuit, simulator_settings
)
exact_expval = full_target_mps.local_expectation(
    quimb.pauli("Z") & quimb.pauli("Z"), (L // 2 - 1, L // 2)
).real.item()
print(f"Reference expectation value (from MPS): {exact_expval:.6f}")

# Optimize ansatz parameters
objective = MaximizeStateFidelity(
    aqc_target_mps, aqc_ansatz, simulator_settings
)

def callback(intermediate_result: OptimizeResult):
    fidelity = 1 - intermediate_result.fun
    print(
        f"{datetime.datetime.now()} Intermediate result: Fidelity {fidelity:.8f}"
    )

result = minimize(
    objective,
    aqc_initial_parameters,
    method="L-BFGS-B",
    jac=True,
    options={"maxiter": 500},
    callback=callback,
)
if result.status not in (0, 1, 99):
    raise RuntimeError(
        f"Optimization failed: {result.message} (status={result.status})"
    )
print(f"Done after {result.nit} iterations.")

# Assemble the final AQC circuit: optimized ansatz + subsequent Trotter step
aqc_final_circuit = aqc_ansatz.assign_parameters(result.x)
aqc_final_circuit.compose(subsequent_circuit, inplace=True)

# -------------------------Step 2-------------------------

service = QiskitRuntimeService()
backend = service.least_busy(min_num_qubits=127)
print(backend)

pass_manager = generate_preset_pass_manager(
    backend=backend, optimization_level=3
)
isa_circuit = pass_manager.run(aqc_final_circuit)
isa_observable = observable.apply_layout(isa_circuit.layout)
print(
    "AQC circuit depth:",
    isa_circuit.depth(lambda x: x.operation.num_qubits == 2),
)

# Also transpile the baseline Trotter circuit (4 Trotter steps, no AQC)
isa_baseline_circuit = pass_manager.run(baseline_circuit)
isa_baseline_observable = observable.apply_layout(isa_baseline_circuit.layout)
print(
    "Baseline Trotter circuit depth:",
    isa_baseline_circuit.depth(lambda x: x.operation.num_qubits == 2),
)

# -------------------------Step 3-------------------------

# Submit both circuits in a single job
estimator = Estimator(backend)
estimator.options.environment.job_tags = ["TUT_AQCTE"]

job = estimator.run(
    [
        (isa_circuit, isa_observable),
        (isa_baseline_circuit, isa_baseline_observable),
    ]
)
print("Job ID:", job.job_id())
```

# Memahami high availability dan disaster recovery untuk Qiskit Runtime {#understanding-high-availability-and-disaster-recovery-for-qiskit-runtime}

High availability (HA) adalah kemampuan suatu layanan untuk tetap beroperasi dan dapat diakses meski terjadi kegagalan yang tidak terduga. Disaster recovery adalah proses memulihkan instans layanan ke kondisi yang berfungsi kembali. Pelanggan Qiskit Runtime tidak bertanggung jawab atas perencanaan atau tindakan disaster recovery.

Qiskit Runtime adalah layanan regional yang memenuhi [Service Level Objectives (SLO)](https://cloud.ibm.com/docs/resiliency?topic=resiliency-slo) yang telah ditetapkan dengan paket berbayar apa pun. Untuk info lebih lanjut, lihat [Service Level Agreement (SLA)](https://www.ibm.com/support/customer/csol/terms/?id=i126-9268&lc=en) untuk IBM Cloud&reg;. Untuk info lebih lanjut tentang region dan pusat data IBM Cloud yang tersedia untuk Qiskit Runtime, lihat [Ketersediaan layanan dan infrastruktur berdasarkan lokasi.](https://cloud.ibm.com/docs/overview?topic=overview-services_region)

## Arsitektur high availability {#high-availability-architecture}

### Fitur high availability {#high-availability-features}

Layanan ini disediakan di tiga zona dalam region multi-zona tanpa titik kegagalan tunggal. Permintaan API dirutekan melalui load balancer ke node instans HA, masing-masing di zona yang berbeda.

Jika node instans HA atau zona mengalami kegagalan, layanan tetap melanjutkan permintaan API ke node instans HA yang masih aktif. Tidak ada penurunan layanan yang terasa selama kegagalan zona.

Layanan ini tersedia di region berikut:

- Washington DC (us-east)
- Frankfurt (eu-de)

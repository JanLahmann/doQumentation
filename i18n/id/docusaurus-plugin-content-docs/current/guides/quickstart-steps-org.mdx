# Konfigurasi IBM Quantum Platform untuk organisasi {#configure-ibm-quantum-platform-for-an-organization}

Ikuti langkah-langkah berikut untuk menyiapkan IBM Quantum Platform.

<span id="iam-org"></span>
## 1. Konfigurasi pengaturan IAM {#1-konfigurasi-pengaturan-iam}

Pertama, konfigurasi pengaturan berikut di akun IAM administrator. Untuk meninjau dan mengonfigurasi pengaturan ini, buka [Manage → IAM → Settings](https://cloud.ibm.com/iam/settings).

* **Visibilitas daftar pengguna** menentukan apakah pengguna bisa saling melihat satu sama lain, terlepas dari penugasan proyek. Pengaturan `enabled` membatasi visibilitas pengguna. Artinya, pengguna di akunmu tidak bisa saling melihat, meskipun mereka bisa mengakses sumber daya yang sama. Pilih nilai yang sesuai untuk lingkunganmu. Lihat [Controlling user visibility](https://cloud.ibm.com/docs/account?topic=account-iam-user-setting) untuk informasi lebih lanjut.
* **Pembuatan API key** mengontrol apakah pengguna bisa membuat API key. Di IBM Quantum Platform, penggunaan API key adalah hal yang umum. Jika API key digunakan, pilih `disabled`. Atau, kamu bisa memberikan izin khusus kepada setiap pengguna.

<span id="crt-rsc-grp-org"></span>
## 2. (Opsional) Buat grup sumber daya {#2-opsional-buat-grup-sumber-daya}

Lewati langkah ini jika kamu menggunakan grup akses yang langsung merujuk ke instans layanan IBM Quantum.

Jika kamu memilih untuk menggunakan grup sumber daya, buka [Manage → Account → Resource groups (in Account resources)](https://cloud.ibm.com/account/resource-groups) dan klik **Create**.

<span id="create-instance-org"></span>
## 3. Buat instans layanan IBM Quantum {#3-buat-instans-layanan-ibm-quantum}

Jika kamu sudah membuat instans layanan IBM Quantum, lewati langkah ini.

* Jika kamu menggunakan grup sumber daya, pastikan untuk membuat instans layanan di grup sumber daya yang sesuai.
* Nama instans layanan, seperti `QR-ml`, diperlukan untuk referensi grup akses.

Ikuti langkah-langkah di [Set up to use IBM Cloud](/guides/cloud-setup#cloud) untuk petunjuknya.

<span id="create-group-org"></span>
## 4. Buat grup akses untuk proyek {#4-buat-grup-akses-untuk-proyek}

Pertama, [buat peran kustom](/guides/custom-roles#custom) yang memungkinkan pengguna melakukan tindakan untuk bekerja dengan instans layanan IBM Quantum.
Selanjutnya, [buat grup akses](/guides/access-groups) untuk setiap proyek dan berikan grup tersebut kumpulan izin minimum yang diperlukan untuk bekerja dengan sumber daya proyek.

<span id="setup-idp-org"></span>
## 5. Siapkan penyedia ID-mu dan tetapkan pengguna {#5-siapkan-penyedia-idmu-dan-tetapkan-pengguna}

Ikuti langkah-langkah di topik yang sesuai, tergantung pada penyedia ID yang kamu pilih untuk digunakan:

* [Gunakan IBM Cloud&reg; sebagai penyedia ID untuk pengguna yang memiliki akun IBM Cloud](/guides/manage-cloud-users)
* [Gunakan IBM Cloud sebagai penyedia ID untuk pengguna yang tidak memiliki akun IBM Cloud](/guides/manage-appid)
* [Gunakan penyedia ID selain IBM Cloud untuk pengguna yang tidak memiliki akun IBM Cloud](/guides/manage-appid)
* [Undang pengguna dan kelola aksesnya](/guides/invite-and-manage-users)

<span id="firewall"></span>
## 6. Konfigurasi firewall-mu {#6-konfigurasi-firewallmu}

Untuk mengonfigurasi firewall-mu agar bisa mengakses endpoint API IBM Quantum, tambahkan URL berikut ke daftar yang diizinkan:

- Layanan global IBM Cloud:
    ```text
    cloud.ibm.com
    iam.cloud.ibm.com
    api.global-search-tagging.cloud.ibm.com
    globalcatalog.cloud.ibm.com
    resource-controller.cloud.ibm.com
    ```
- Layanan IBM Quantum Qiskit Runtime - region us-east:
    ```text
    quantum.cloud.ibm.com
    s3.us-east.cloud-object-storage.appdomain.cloud
    ```
- Layanan IBM Quantum Qiskit Runtime - region eu-de:
    ```text
    eu-de.quantum.cloud.ibm.com
    s3.direct.eu-de.cloud-object-storage.appdomain.cloud
    ```
- Layanan pratinjau IBM Quantum Qiskit Functions:
    ```text
    qiskit-serverless.quantum.ibm.com
    ```

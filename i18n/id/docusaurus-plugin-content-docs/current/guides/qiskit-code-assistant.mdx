# Qiskit Code Assistant

Qiskit Code Assistant bertujuan membuat komputasi kuantum lebih mudah diakses oleh pengguna Qiskit baru dan meningkatkan pengalaman coding bagi pengguna saat ini. Ini adalah asisten kode AI generatif yang didukung oleh [watsonx](https://www.ibm.com/products/watsonx-ai). Asisten ini dilatih menggunakan jutaan token teks dari Qiskit SDK, bertahun-tahun contoh kode Qiskit, dan fitur-fitur IBM Quantum&reg;. Qiskit Code Assistant bisa membantu alur kerja pengembangan kuantummu dengan menawarkan saran yang dihasilkan LLM berdasarkan [model IBM Granite](https://www.ibm.com/products/watsonx-ai/foundation-models), yang mencakup fitur dan fungsionalitas terbaru dari IBM&reg;.

<Admonition type="note" title="Catatan">
- Ini adalah fitur eksperimental yang tersedia untuk pengguna IBM Quantum Premium Plan yang terdaftar di [IBM Quantum Platform](https://quantum.cloud.ibm.com/) yang baru.
- Qiskit Code Assistant dalam status rilis pratinjau dan dapat berubah sewaktu-waktu.
- Kalau kamu punya masukan atau ingin menghubungi tim pengembang, gunakan [saluran Qiskit Slack Workspace](https://qiskit.enterprise.slack.com/archives/C07LYA6PL83) atau repositori GitHub publik terkait.
</Admonition>

## Fitur {#features}

Fitur-fitur berikut tersedia dalam ekstensi [Visual Studio Code](https://code.visualstudio.com/) (VS Code) dan editor yang kompatibel, serta [JupyterLab](https://jupyterlab.readthedocs.io/en/latest/):

* Mempercepat pembuatan kode Qiskit dengan memanfaatkan AI generatif berdasarkan model yang dikhususkan untuk menghasilkan kode Qiskit.
* Memungkinkan prompt abstrak dan spesifik untuk menghasilkan rekomendasi.
* Menampilkan saran yang bisa kamu tinjau, terima, atau tolak.
* Mendukung file kode Python dan Jupyter notebook.
* Dilengkapi guardrail untuk menghindari menjawab pertanyaan yang berpotensi membahayakan pengguna, seperti ujaran kebencian.

Untuk petunjuk mengintegrasikan Qiskit Code Assistant langsung ke lingkungan pengembanganmu, ikuti instruksi di topik yang sesuai:
- [JupyterLab](/guides/qiskit-code-assistant-jupyterlab)
- [VS Code](/guides/qiskit-code-assistant-vscode)

## Large Language Model (LLM) di balik Qiskit Code Assistant {#the-large-language-model-llm-behind-qiskit-code-assistant}

Untuk memberikan saran kode, Qiskit Code Assistant menggunakan Large Language Model (LLM). Dalam hal ini, Qiskit Code Assistant saat ini mengandalkan model `mistral-small-3.2-24b-qiskit`, yang dibangun di atas [model Mistral-Small-3.2-24B-Qiskit](https://huggingface.co/Qiskit/mistral-small-3.2-24b-qiskit). Model `mistral-small-3.2-24b-qiskit` meningkatkan kemampuan pembuatan kode model `Mistral-Small-3.2-24B-Instruct-2506` untuk Qiskit melalui pelatihan tambahan yang diperluas dan fine-tuning pada data Qiskit berkualitas tinggi, serta commit Python dan chat. Untuk informasi lebih lanjut tentang keluarga model Mistral AI, lihat [dokumentasi Mistral AI](https://docs.mistral.ai/getting-started/models). Untuk detail lebih lanjut tentang model `.*-qiskit`, lihat [Qiskit Code Assistant: Training LLMs for generating Quantum Computing Code](https://arxiv.org/abs/2405.19495).
2. [qiskit/qwen2.5-coder-14b-qiskit](https://huggingface.co/Qiskit/qwen2.5-coder-14b-qiskit) - Released June 2025
3. [qiskit/granite-3.3-8b-qiskit](https://huggingface.co/Qiskit/granite-3.3-8b-qiskit) - Released June 2025

LLM yang kami khususkan untuk Qiskit juga tersedia sebagai model open-source. Lihat semua model yang tersedia di https://huggingface.co/Qiskit.

## Benchmark Qiskit HumanEval dan Qiskit HumanEval Hard {#the-qiskit-humaneval-and-qiskit-humaneval-hard-benchmarks}

Untuk menguji `mistral-small-3.2-24b-qiskit` dan model lainnya, kami berkolaborasi dengan Qiskit Advocates dan para ahli untuk membuat benchmark berbasis eksekusi yang disebut Qiskit HumanEval (QHE) dan Qiskit HumanEval Hard (QHE Hard), lalu menjalankannya pada model-model tersebut. Benchmark ini mirip dengan [HumanEval](https://arxiv.org/abs/2107.03374), mencakup berbagai masalah kode yang menantang untuk diselesaikan, semuanya berbasis library Qiskit resmi.

Benchmark ini terdiri dari sekitar 150 tes, masing-masing dibuat dari definisi fungsi, diikuti oleh docstring yang merinci tugas yang harus diselesaikan model. Setiap contoh juga mencakup solusi kanonik referensi, serta unit test, untuk mengevaluasi kebenaran solusi yang dihasilkan. Ada tiga tingkat kesulitan tes: dasar, menengah, dan sulit. Benchmark Qiskit HumanEval Hard adalah variasi dari Qiskit HumanEval, tetapi menghapus informasi terkait impor kode, sehingga LLM perlu mencari tahu impor metode atau kelas yang tepat. Perubahan ini membuat dataset jauh lebih menantang bagi LLM, menurut pengujian dan hasil awal kami.

Dataset untuk Qiskit HumanEval dan Qiskit HumanEval Hard tersedia di situs-situs ini: [Qiskit HumanEval](https://huggingface.co/datasets/Qiskit/qiskit_humaneval) dan [Qiskit HumanEval](https://huggingface.co/datasets/Qiskit/qiskit_humaneval_hard). Kamu bisa berkontribusi pada pengembangan benchmark ini di [repositori GitHub](https://github.com/qiskit-community/qiskit-human-eval).

### Mulai cepat (direkomendasikan) {#quick-start-recommended}

Cara termudah untuk mulai menggunakan Qiskit Code Assistant dalam mode lokal adalah dengan memakai skrip setup otomatis untuk ekstensi VS Code atau JupyterLab. Skrip ini akan otomatis menginstal [Ollama](https://ollama.com/) untuk menjalankan LLM, mengunduh model yang direkomendasikan, dan mengonfigurasi ekstensi buat kamu.

#### Setup ekstensi VS Code {#vs-code-extension-setup}

Jalankan perintah berikut di terminalmu:

```bash
bash <(curl -fsSL https://raw.githubusercontent.com/Qiskit/qiskit-code-assistant-vscode/main/setup_local.sh)
```

Skrip ini melakukan langkah-langkah berikut:
- Menginstal Ollama (kalau belum terinstal)
- Mengunduh dan mengonfigurasi model Qiskit Code Assistant yang direkomendasikan
- Menyiapkan ekstensi VS Code agar bekerja dengan deployment lokalmu

#### Setup ekstensi JupyterLab {#jupyterlab-extension-setup}

Jalankan perintah berikut di terminalmu:

```bash
bash <(curl -fsSL https://raw.githubusercontent.com/Qiskit/qiskit-code-assistant-jupyterlab/main/setup_local.sh)
```

Skrip ini akan:
- Menginstal Ollama (kalau belum terinstal)
- Mengunduh dan mengonfigurasi model Qiskit Code Assistant yang direkomendasikan
- Menyiapkan ekstensi JupyterLab agar bekerja dengan deployment lokalmu

#### Menggunakan aplikasi Ollama {#using-the-ollama-application}

Aplikasi Ollama menyediakan solusi sederhana untuk menjalankan LLM secara lokal. Aplikasi ini mudah digunakan, dengan CLI yang membuat seluruh proses setup, manajemen model, dan interaksi cukup mudah. Ini ideal untuk eksperimen cepat dan untuk pengguna yang ingin menangani lebih sedikit detail teknis.

1. Instal file yang telah diunduh
1. Luncurkan aplikasi Ollama yang sudah terinstal

    <Admonition type="info">

1. Pastikan aplikasi Ollama sedang berjalan.

##### Menyiapkan Ollama dengan model GGUF Qiskit Code Assistant yang diunduh secara manual {#set-up-ollama-with-a-manually-downloaded-qiskit-code-assistant-gguf-model}

Kalau kamu sudah mengunduh model GGUF secara manual seperti https://huggingface.co/Qiskit/Qwen2.5-Coder-14B-Qiskit-GGUF dan kamu ingin bereksperimen dengan template dan parameter yang berbeda, kamu bisa mengikuti langkah-langkah ini untuk memuatnya ke aplikasi Ollama lokalmu.

1. Buat sebuah `Modelfile` dengan memasukkan konten berikut dan pastikan memperbarui `<PATH-TO-GGUF-FILE>` ke path sebenarnya dari model yang kamu unduh.

    ```
    FROM <PATH-TO-GGUF-FILE>
    TEMPLATE """{{ if .System }}
    System:
    {{ .System }}

    {{ end }}{{ if .Prompt }}Question:
    {{ .Prompt }}

    {{ end }}Answer:
    ```python{{ .Response }}
    """

    PARAMETER stop "Question:"
    PARAMETER stop "Answer:"
    PARAMETER stop "System:"
    PARAMETER stop "```"

    PARAMETER temperature 0
    PARAMETER top_k 1
    ```

1. Jalankan perintah berikut untuk membuat instance model kustom berdasarkan `Modelfile` tersebut.

    ```
    ollama create Qwen2.5-Coder-14B-Qiskit -f ./path-to-model-file
    ```

    <Admonition type="note">
        Proses ini mungkin memakan waktu bagi Ollama untuk membaca file model, menginisialisasi instance model, dan mengonfigurasinya sesuai spesifikasi yang diberikan.
    </Admonition>

<AccordionItem title="Deploy model Qiskit Code Assistant secara lokal melalui paket `llama-cpp-python` secara manual">

<span id="use-llama-cpp-python"></span>
Alternatif dari aplikasi Ollama adalah paket `llama-cpp-python`, yang merupakan Python binding untuk `llama.cpp`. Paket ini memberimu kontrol dan fleksibilitas lebih untuk menjalankan model GGUF secara lokal, dan ideal untuk pengguna yang ingin mengintegrasikan model lokal ke dalam alur kerja dan aplikasi Python mereka.

1. Instal [`llama-cpp-python`](https://pypi.org/project/llama-cpp-python/)
2. Berinteraksilah dengan model dari dalam aplikasimu menggunakan `llama_cpp`. Misalnya:

```python
from llama_cpp import Llama

model_path = <PATH-TO-GGUF-FILE>

model = Llama(
        model_path,
        seed=17,
        n_ctx=10000,
        n_gpu_layers=37, # to offload in gpu, but put 0 if all in cpu
    )

input = 'Generate a quantum circuit with 2 qubits'
raw_pred = model(input)["choices"][0]["text"]
```

Kamu juga bisa menambahkan parameter pembuatan teks ke model untuk menyesuaikan inferensi:

```python
generation_kwargs = {
        "max_tokens": 512,
        "echo": False, # Echo the prompt in the output
        "top_k": 1
    }

raw_pred = model(input, **generation_kwargs)["choices"][0]["text"]
```

</AccordionItem>

<AccordionItem title="Deploy model Qiskit Code Assistant secara lokal melalui llama.cpp secara manual">
#### Menggunakan library `llama.cpp` {#use-the-llamacpp-library}

Alternatif lainnya adalah menggunakan `llama.cpp`, sebuah library open-source untuk melakukan inferensi LLM pada CPU dengan setup minimal.
Library ini memberikan kontrol tingkat rendah atas eksekusi model dan biasanya dijalankan dari command line, mengarah ke file model GGUF lokal.

Ada beberapa cara untuk menginstal `llama.cpp` di mesinmu:

- Instal llama.cpp menggunakan [brew, nix, atau winget](https://github.com/ggml-org/llama.cpp/blob/master/docs/install.md)
- Jalankan dengan Docker: Lihat [dokumentasi Docker dari tim `llama.cpp`](https://github.com/ggml-org/llama.cpp/blob/master/docs/docker.md)
- Unduh binary pra-bangun dari [halaman rilis](https://github.com/ggml-org/llama.cpp/releases)
- Bangun dari sumber dengan [meng-clone repositori ini](https://github.com/ggml-org/llama.cpp/blob/master/docs/build.md)

Setelah terinstal, kamu bisa menggunakan `llama.cpp` untuk berinteraksi dengan model GGUF dalam mode percakapan sebagai berikut:
```bash
# Use a local model file
llama-cli -m my_model.gguf -cnv

# Or download and run a model directly from Hugging Face
llama-cli -hf Qiskit/Qwen2.5-Coder-14B-Qiskit-GGUF -cnv
```

Kamu juga bisa meluncurkan server API yang kompatibel dengan OpenAI untuk model tersebut dengan cara berikut:
```bash
llama-server -hf Qiskit/Qwen2.5-Coder-14B-Qiskit-GGUF
```

##### Parameter lanjutan {#advanced-parameters}
Dengan program `llama-cli`, kamu bisa mengontrol pembuatan model menggunakan opsi command-line. Misalnya, kamu bisa memberikan prompt “system” awal menggunakan flag `-p/--prompt`. Dalam mode percakapan (`-cnv`), prompt awal ini berfungsi sebagai pesan system. Selain itu, kamu cukup menambahkan instruksi apa pun yang diinginkan di awal teks prompt-mu. Kamu juga bisa menyesuaikan parameter sampling - misalnya: temperature (`--temp`), top-k (`--top-k`), top-p (`--top-p`), repetition penalty (`--repeat-penalty`), dan seed yang digunakan (`--seed`). Berikut adalah contoh invokasi menggunakan opsi-opsi ini:

```bash
llama-cli -hf Qiskit/Qwen2.5-Coder-14B-Qiskit-GGUF \
  -p "You are a friendly assistant." -cnv \
  --temp 0.7 \
  --top-k 50 \
  --top-p 0.95 \
  --repeat-penalty 1.1 \
  --seed 42
```

Untuk memastikan model Qiskit kami berfungsi dengan baik, kami merekomendasikan menggunakan system prompt yang disediakan di repositori HF GGUF kami: system prompt untuk [mistral-small-3.2-24b-qiskit-GGUF](https://huggingface.co/Qiskit/mistral-small-3.2-24b-qiskit-GGUF/blob/main/system), [Qwen2.5-Coder-14B-Qiskit-GGUF](https://huggingface.co/Qiskit/Qwen2.5-Coder-14B-Qiskit-GGUF/blob/main/system), [granite-3.3-8b-qiskit-GGUF](https://huggingface.co/Qiskit/granite-3.3-8b-qiskit-GGUF/blob/main/system), dan [granite-3.2-8b-qiskit-GGUF](https://huggingface.co/Qiskit/granite-3.2-8b-qiskit-GGUF/blob/main/system).

</AccordionItem>

<AccordionItem title="Hubungkan ekstensi ke deployment lokal secara manual">

Gunakan ekstensi VS Code dan ekstensi JupyterLab untuk Qiskit Code Assistant guna memberi prompt ke model Qiskit Code Assistant yang di-deploy secara lokal. Setelah kamu memiliki aplikasi Ollama yang [disiapkan dengan model](#using-the-ollama-application), kamu bisa mengonfigurasi ekstensi untuk terhubung ke layanan lokal.

#### Terhubung dengan ekstensi VS Code Qiskit Code Assistant {#connect-with-the-qiskit-code-assistant-vs-code-extension}

Dengan ekstensi VS Code Qiskit Code Assistant, kamu bisa berinteraksi dengan model dan melakukan penyelesaian kode sambil menulis kodemu. Ini bisa berfungsi baik untuk pengguna yang mencari bantuan menulis kode Qiskit untuk aplikasi Python mereka.

1. Instal [ekstensi VS Code Qiskit Code Assistant](/guides/qiskit-code-assistant-vscode).
1. Di VS Code, buka **User Settings** dan atur **Qiskit Code Assistant: Url** ke URL deployment Ollama lokalmu (misalnya, `http://localhost:11434`).
1. Muat ulang VS Code dengan membuka **View > Command Palette...** dan memilih **Developer: Reload Window**.

Model Qiskit Code Assistant yang dikonfigurasi di Ollama seharusnya muncul di status bar dan kemudian siap digunakan.

#### Terhubung dengan ekstensi JupyterLab Qiskit Code Assistant {#connect-with-the-qiskit-code-assistant-jupyterlab-extension}

Dengan ekstensi JupyterLab Qiskit Code Assistant, kamu bisa berinteraksi dengan model dan melakukan penyelesaian kode langsung di Jupyter Notebook-mu. Pengguna yang sebagian besar bekerja dengan Jupyter Notebook bisa memanfaatkan ekstensi ini untuk lebih meningkatkan pengalaman menulis kode Qiskit mereka.

1. Instal [ekstensi JupyterLab Qiskit Code Assistant](/guides/qiskit-code-assistant-jupyterlab).
1. Di JupyterLab, buka **Settings Editor** dan atur **Qiskit Code Assistant Service API** ke URL deployment Ollama lokalmu (misalnya, `http://localhost:11434`).

Model Qiskit Code Assistant yang dikonfigurasi di Ollama seharusnya muncul di status bar dan kemudian siap digunakan.

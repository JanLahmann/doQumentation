# Pertanyaan yang sering diajukan (FAQ) {#frequently-asked-questions-faq}

Untuk FAQ khusus tentang mode eksekusi, lihat panduan [FAQ mode eksekusi Qiskit Runtime](/guides/execution-modes-faq).

## Bagaimana cara mengutip Qiskit dalam penelitian saya? {#bagaimana-cara-mengutip-qiskit-dalam-penelitian-saya}

Kutip Qiskit dengan menggunakan [file BibTeX](https://github.com/Qiskit/qiskit/blob/main/CITATION.bib) yang disertakan.

## Bagaimana cara mengutip komputer IBM Quantum dalam penelitian saya? {#bagaimana-cara-mengutip-komputer-ibm-quantum-dalam-penelitian-saya}

Untuk makalah penelitian, kami mendorong penulis untuk mengakui IBM Quantum&reg; dengan menggunakan:

>    We acknowledge the use of IBM Quantum services for this work. The views expressed are those of the authors, and do not reflect the official policy or position of IBM or the IBM Quantum team.

Referensi makalah harus menggunakan kutipan berikut:

> IBM Quantum. https://<span>quantum.cloud.ibm.com</span>/, YYYY

Unit pemrosesan kuantum dalam makalah harus direferensikan dengan nama uniknya (misalnya, `ibmq_vigo`) dan secara opsional menambahkan versinya (misalnya, `ibmq_vigo` v1.0.2). Kami juga mendorong untuk mereferensikan prosesornya. Contoh:

> In this paper we used `ibmq_vigo`, which is one of the IBM Quantum Canary processors.

Contoh mengutip program IBM Quantum:

> IBM Quantum (2025). Estimator primitive (Version x.y.z) [computer software]. https://<span>quantum.cloud.ibm.com</span>/

## Bagaimana cara mengutip IBM Quantum Composer dalam penelitian saya? {#bagaimana-cara-mengutip-ibm-quantum-composer-dalam-penelitian-saya}

Contoh kutipan untuk IBM Quantum Composer:

> IBM Quantum Composer. 2025. url: https://<span>quantum.cloud.ibm.com/composer</span>

## Saya tidak melihat sesuatu di antarmuka pengguna, seperti tugas administrasi saya. {#saya-tidak-melihat-sesuatu-di-antarmuka-pengguna-seperti-tugas-administrasi-saya}

Coba pilih akun lain dari pemilih akun di header antarmuka [IBM Quantum Platform](https://quantum.cloud.ibm.com).

## Saya hanya menggunakan Open Plan, tapi saya harus memasukkan kartu kredit. Apakah saya akan ditagih? {#saya-hanya-menggunakan-open-plan-tapi-saya-harus-memasukkan-kartu-kredit-apakah-saya-akan-ditagih}

Kamu tidak akan ditagih kecuali jika kamu membuat [instance](/guides/instances) Pay-As-You-Go dan menggunakannya untuk mengirim workload ke QPU. Lihat [Kelola biaya](/guides/manage-cost) untuk instruksi menetapkan batas biaya dan untuk mengetahui cara lain mengelola biaya.

## Apa perbedaan antara API key dan API token? {#apa-perbedaan-antara-api-key-dan-api-token}

Tidak ada perbedaan. Istilah API key dan API token digunakan secara bergantian. Kamu akan melihat kedua istilah dalam dokumentasi karena antarmuka yang berbeda menggunakan istilah yang berbeda.

<span id="bearer"></span>
## Apa perbedaan antara API token dan bearer token? {#apa-perbedaan-antara-api-token-dan-bearer-token}

API token adalah kredensial yang bertahan lama yang digunakan untuk [mengautentikasi ke layanan Qiskit Runtime.](/guides/hello-world#install-and-authenticate) Bearer token adalah kredensial sementara yang digunakan untuk [mengautentikasi ke REST API di lingkungan yang tidak dipercaya.](/guides/cloud-setup-untrusted#authenticate-to-the-rest-api-in-an-untrusted-environment) Bearer token tidak bisa digunakan untuk mengautentikasi ke layanan Qiskit Runtime.

## Apakah saya harus memilih region tempat saya tinggal? {#apakah-saya-harus-memilih-region-tempat-saya-tinggal}

Tidak. Kamu bisa menggunakan region mana pun. Region menentukan di mana job dijalankan dan di mana data disimpan. Selain itu, kamu hanya bisa melihat dan menggunakan instance yang dibuat di instance yang kamu masuki. Perlu diperhatikan bahwa hanya region us-east yang mendukung Open Plan.

## Bisakah saya terhubung melalui proxy server? {#bisakah-saya-terhubung-melalui-proxy-server}

Bisa, tapi kamu harus menggunakan Qiskit Runtime v0.44.0 atau yang lebih baru.

## Apa kebijakan retensi data? {#apa-kebijakan-retensi-data}

IBM menyimpan data job dan penggunaan kamu (yang tidak dihapus) selama tiga tahun selama kamu terus menggunakan layanan Qiskit Runtime.

## Apa yang terjadi pada distribusi Python seperti Qiskit Aer, Qiskit Nature, dan Qiskit Experiments? {#apa-yang-terjadi-pada-distribusi-python-seperti-qiskit-aer-qiskit-nature-dan-qiskit-experiments}

Sejak rilis Qiskit SDK v1.0, perintah `pip install qiskit` **hanya** akan menginstal [Qiskit SDK](https://github.com/Qiskit/qiskit). Ini berarti beberapa modul yang pernah kamu gunakan sebelumnya, seperti `qiskit-aer` atau `qiskit-nature`, sekarang harus diinstal secara terpisah.

Sebelum rilis Qiskit SDK v1.0, Qiskit terdiri dari banyak distribusi Python yang berbeda, seperti `qiskit-terra`, inti compiler; `qiskit-aer`, simulator berperforma tinggi; provider IBM Quantum&reg; asli; dan beberapa paket yang kini sudah usang yang menyediakan fitur algoritmik eksploratif atau eksekusi eksperimen tertentu.

Paket-paket perangkat lunak terpisah ini digabungkan menjadi apa yang disebut Qiskit *metapackage*, yang memungkinkan pengguna menginstal semua komponen ini sekaligus melalui `pip install qiskit`. Arsitektur metapackage ini dihapus dengan rilis Qiskit SDK v1.0, dan sekarang setiap distribusi perangkat lunak individual perlu diinstal secara terpisah.

Jika kamu ingin menginstal paket yang sebelumnya termasuk dalam Qiskit metapackage, kunjungi [ekosistem Qiskit](https://www.ibm.com/quantum/ecosystem) untuk menemukan berbagai paket yang sesuai dengan kebutuhanmu. Kamu juga bisa membaca [panduan migrasi v1.0](/guides/metapackage-migration) untuk informasi lebih lanjut tentang perubahan ini.

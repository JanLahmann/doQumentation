# Melindungi sumber daya Qiskit Runtime Service dengan pembatasan berbasis konteks {#protecting-qiskit-runtime-service-resources-with-context-based-restrictions}

Pembatasan berbasis konteks memberi pemilik akun dan administrator kemampuan untuk mendefinisikan dan menerapkan pembatasan akses terhadap sumber daya IBM Cloud&reg; berdasarkan konteks permintaan akses. Akses ke sumber daya Qiskit Runtime Service dapat dikontrol dengan pembatasan berbasis konteks dan kebijakan identity and access management (IAM).

Pembatasan ini bekerja bersama kebijakan IAM tradisional yang berbasis identitas, untuk memberikan lapisan perlindungan tambahan. Tidak seperti kebijakan IAM, pembatasan berbasis konteks tidak memberikan hak akses. Pembatasan berbasis konteks memeriksa apakah permintaan akses berasal dari konteks yang diizinkan sesuai konfigurasi kamu. Karena baik akses IAM maupun pembatasan berbasis konteks sama-sama menerapkan kontrol akses, pembatasan berbasis konteks tetap memberikan perlindungan meskipun kredensial bocor atau salah kelola. Untuk informasi lebih lanjut, lihat topik [Apa itu pembatasan berbasis konteks](https://cloud.ibm.com/docs/account?topic=account-context-restrictions-whatis).

Pengguna harus memiliki peran Administrator pada layanan Qiskit Runtime Service untuk membuat, memperbarui, atau menghapus aturan. Pengguna juga harus memiliki peran Editor atau Administrator pada layanan Context-based restrictions untuk membuat, memperbarui, atau menghapus zona jaringan. Pengguna dengan peran Viewer pada layanan Context-based restrictions hanya dapat menambahkan zona jaringan ke suatu aturan.

Semua peristiwa Activity Tracker atau log audit yang dihasilkan berasal dari layanan context-based restrictions, bukan dari Qiskit Runtime Service. Untuk informasi lebih lanjut, lihat halaman [Memantau pembatasan berbasis konteks](https://cloud.ibm.com/docs/account?topic=account-cbr-monitor).

Untuk mulai melindungi sumber daya Qiskit Runtime Service kamu dengan pembatasan berbasis konteks, lihat tutorial [Memanfaatkan pembatasan berbasis konteks untuk mengamankan sumber daya kamu](https://cloud.ibm.com/docs/account?topic=account-context-restrictions-tutorial).

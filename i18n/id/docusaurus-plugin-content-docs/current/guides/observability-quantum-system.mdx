# Event pelacakan aktivitas untuk layanan IBM Quantum System {#activity-tracking-events-for-the-ibm-quantum-system-service}

Layanan IBM Cloud&reg;, seperti layanan IBM&reg; Quantum System, menghasilkan event pelacakan aktivitas.

Event pelacakan aktivitas melaporkan aktivitas yang mengubah status layanan di IBM Cloud. Kamu bisa menggunakan event ini untuk menginvestigasi aktivitas yang mencurigakan, tindakan kritis, dan untuk memenuhi persyaratan audit regulasi.

Kamu bisa menggunakan IBM Cloud Activity Tracker Event Routing, layanan platform, untuk merutekan event auditing di akunmu ke tujuan yang Anda pilih dengan mengonfigurasi target dan rute yang menentukan ke mana event pelacakan aktivitas dikirim. Untuk info lebih lanjut, lihat [Tentang IBM Cloud Activity Tracker Events Routing](https://cloud.ibm.com/docs/atracker?topic=atracker-about).

Kamu bisa menggunakan IBM Cloud Logs untuk memvisualisasikan dan memberi peringatan pada event yang dihasilkan di akunmu dan diroute oleh IBM Cloud Activity Tracker Event Routing ke instance IBM Cloud Logs.

## Melihat event pelacakan aktivitas untuk layanan IBM Quantum System {#viewing-activity-tracking-events-for-the-ibm-quantum-system-service}

Kamu bisa menggunakan IBM Cloud Logs untuk memvisualisasikan dan memberi peringatan pada event yang dihasilkan di akunmu dan diroute oleh IBM Cloud Activity Tracker Event Routing ke instance IBM Cloud Logs.

### Meluncurkan IBM Cloud Logs dari halaman Observability {#launching-ibm-cloud-logs-from-the-observability-page}

Untuk info tentang cara meluncurkan UI IBM Cloud Logs, lihat [Meluncurkan UI](https://cloud.ibm.com/docs/cloud-logs?topic=cloud-logs-instance-launch#instance-launch-cloud-ui) di dokumentasi IBM Cloud Logs.

| Aksi | Dipicu oleh |
| -- | -- |
| `quantum-computing.direct-access-backend-configuration.read` | Ambil konfigurasi backend (`GET /v1/backends/{backend-name}/configuration`) |
| `quantum-computing.direct-access-backend-properties.read` | Ambil properti backend (`GET /v1/backends/{backend-name}/properties`) |
| `quantum-computing.direct-access-backend.list` | Ambil daftar backend (`GET /v1/backends`) |
| `quantum-computing.direct-access-backend.read` | Ambil detail backend (`GET /v1/backends/{backend-name}`) |
| `quantum-computing.direct-access-job.cancel` | Batalkan job (`POST /v1/jobs/{job-id}/cancel`) |
| `quantum-computing.direct-access-job.create` | Jalankan job (`POST /v1/jobs`) |
| `quantum-computing.direct-access-job.delete` | Hapus job (`DELETE /v1/jobs/{job-id}`) |
| `quantum-computing.direct-access-job.list` | Ambil daftar job (`GET /v1/jobs`) |
| `quantum-computing.direct-access-job.read` | Ambil job berdasarkan ID (`GET /v1/jobs/{job-id}`) |
| `quantum-computing.direct-access-lane-configuration.list` | Ambil konfigurasi lane backend (`GET /v1/backends/{backend-name}/lanes`) |

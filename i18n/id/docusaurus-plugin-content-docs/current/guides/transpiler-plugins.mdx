Untuk memudahkan pengembangan dan penggunaan kembali kode transpilasi kustom oleh komunitas pengguna Qiskit yang lebih luas, Qiskit SDK mendukung antarmuka plugin yang memungkinkan paket Python pihak ketiga untuk mendeklarasikan bahwa mereka menyediakan fungsionalitas transpilasi yang diperluas dan dapat diakses melalui Qiskit.

Saat ini, plugin pihak ketiga dapat menyediakan fungsionalitas transpilasi yang diperluas dengan tiga cara:

- [Plugin tahap transpiler](https://docs.quantum.ibm.com/api/qiskit/transpiler_plugins) menyediakan pass manager yang dapat digunakan sebagai pengganti salah satu dari [6 tahap](transpiler-stages) staged pass manager preset: `init`, `layout`, `routing`, `translation`, `optimization`, dan `scheduling`.
- [Plugin sintesis uniter](https://docs.quantum.ibm.com/api/qiskit/qiskit.transpiler.passes.synthesis.plugin.UnitarySynthesisPlugin) menyediakan fungsionalitas yang diperluas untuk sintesis gate uniter.
- [Plugin sintesis tingkat tinggi](https://docs.quantum.ibm.com/api/qiskit/qiskit.transpiler.passes.synthesis.plugin.HighLevelSynthesisPlugin) menyediakan fungsionalitas yang diperluas untuk mensintesis "objek tingkat tinggi" seperti fungsi linear atau operator Clifford. Objek tingkat tinggi direpresentasikan oleh subkelas dari kelas [Operation](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.Operation).

Sisa halaman ini menjelaskan cara membuat daftar plugin yang tersedia, menginstal yang baru, dan menggunakannya.
## Daftar plugin yang tersedia dan instal yang baru {#list-available-plugins-and-install-new-ones}

Qiskit sudah menyertakan beberapa plugin bawaan untuk transpilasi. Untuk menginstal lebih banyak, kamu bisa menggunakan package manager Python-mu. Misalnya, kamu bisa menjalankan `pip install qiskit-toqm` untuk menginstal plugin tahap routing [Qiskit TOQM](https://github.com/qiskit-toqm/qiskit-toqm). Sejumlah plugin pihak ketiga merupakan bagian dari [ekosistem Qiskit](https://qiskit.github.io/ecosystem/#transpiler_plugin).

### Daftar plugin tahap transpiler yang tersedia {#list-available-transpiler-stage-plugins}

Gunakan fungsi [list_stage_plugins](https://docs.quantum.ibm.com/api/qiskit/transpiler_plugins#qiskit.transpiler.preset_passmanagers.plugin.list_stage_plugins), dengan memberikan nama tahap yang plugin-nya ingin kamu daftarkan.

```python
# Added by doQumentation — required packages for this notebook
!pip install -q qiskit qiskit-ibm-runtime
```

```python
from qiskit.transpiler.preset_passmanagers.plugin import list_stage_plugins

list_stage_plugins("layout")
```

```text
['default', 'dense', 'sabre', 'trivial']
```

```python
list_stage_plugins("routing")
```

```text
['basic', 'default', 'lookahead', 'none', 'sabre']
```

 Jika `qiskit-toqm` sudah terinstal, maka `toqm` akan muncul dalam daftar plugin `routing`.
### Daftar plugin sintesis uniter yang tersedia {#list-available-unitary-synthesis-plugins}

Gunakan fungsi [unitary_synthesis_plugin_names](https://docs.quantum.ibm.com/api/qiskit/qiskit.transpiler.passes.synthesis.plugin.unitary_synthesis_plugin_names).

```python
from qiskit.transpiler.passes.synthesis import unitary_synthesis_plugin_names

unitary_synthesis_plugin_names()
```

```text
['aqc', 'clifford', 'default', 'gridsynth', 'sk']
```

### Daftar plugin sintesis tingkat tinggi yang tersedia {#list-available-high-level-synthesis-plugins}

Gunakan fungsi [high_level_synthesis_plugin_names](https://docs.quantum.ibm.com/api/qiskit/qiskit.transpiler.passes.synthesis.plugin.high_level_synthesis_plugin_names), dengan memberikan nama jenis "objek tingkat tinggi" yang akan disintesis. Nama tersebut sesuai dengan atribut [`name`](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.Operation#name) dari kelas [Operation](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.Operation) yang merepresentasikan jenis objek yang sedang disintesis.

```python
from qiskit.transpiler.passes.synthesis import (
    high_level_synthesis_plugin_names,
)

high_level_synthesis_plugin_names("clifford")
```

```text
['ag', 'bm', 'default', 'greedy', 'layers', 'lnn', 'rb_default']
```

## Gunakan plugin {#use-a-plugin}

Di bagian ini, kita akan menunjukkan cara menggunakan plugin transpiler. Dalam contoh kode, kita menggunakan plugin yang sudah disertakan bersama Qiskit, tapi plugin yang diinstal dari paket pihak ketiga digunakan dengan cara yang sama.

### Gunakan plugin tahap transpiler {#use-a-transpiler-stage-plugin}

Untuk menggunakan plugin tahap transpiler, tentukan namanya dengan argumen yang sesuai pada [`generate_preset_pass_manager`](https://docs.quantum.ibm.com/api/qiskit/qiskit.transpiler.generate_preset_pass_manager#qiskit.transpiler.generate_preset_pass_manager) atau [`transpile`](https://docs.quantum.ibm.com/api/qiskit/compiler#qiskit.compiler.transpile). Argumen tersebut dibentuk dengan menambahkan `_method` pada nama tahap transpilasi. Misalnya, untuk menggunakan plugin routing `lookahead`, kita akan menetapkan `lookahead` untuk argumen `routing_method`:

```python
from qiskit.transpiler import generate_preset_pass_manager
from qiskit_ibm_runtime import QiskitRuntimeService

service = QiskitRuntimeService()
backend = service.backend("ibm_fez")

pass_manager = generate_preset_pass_manager(
    optimization_level=3, backend=backend, routing_method="lookahead"
)
```

### Gunakan plugin sintesis uniter {#use-a-unitary-synthesis-plugin}

Untuk menggunakan plugin sintesis uniter, tentukan namanya sebagai argumen `unitary_synthesis_method` pada [`generate_preset_pass_manager`](https://docs.quantum.ibm.com/api/qiskit/qiskit.transpiler.generate_preset_pass_manager#qiskit.transpiler.generate_preset_pass_manager) atau [`transpile`](https://docs.quantum.ibm.com/api/qiskit/compiler#qiskit.compiler.transpile):

```python
pass_manager = generate_preset_pass_manager(
    optimization_level=3,
    backend=backend,
    unitary_synthesis_method="sk",
    unitary_synthesis_plugin_config=dict(
        basis_gates=["cz", "id", "rz", "sx", "x"]
    ),
)
```

Sintesis uniter digunakan dalam tahap `init`, `translation`, dan `optimization` dari staged pass manager yang dikembalikan oleh [`generate_preset_pass_manager`](https://docs.quantum.ibm.com/api/qiskit/qiskit.transpiler.generate_preset_pass_manager#qiskit.transpiler.generate_preset_pass_manager) atau yang digunakan dalam [`transpile`](https://docs.quantum.ibm.com/api/qiskit/compiler#qiskit.compiler.transpile). Lihat [Tahap Transpiler](transpiler-stages) untuk deskripsi tahap-tahap ini.

Gunakan argumen `unitary_synthesis_plugin_config`, sebuah dictionary bebas, untuk meneruskan opsi pada metode sintesis uniter. Dokumentasi metode sintesis seharusnya menjelaskan opsi yang didukungnya. Lihat [daftar ini](https://docs.quantum.ibm.com/api/qiskit/transpiler_synthesis_plugins#unitary-synthesis-plugins-2) untuk tautan ke dokumentasi plugin sintesis uniter bawaan.
### Gunakan plugin sintesis tingkat tinggi {#use-a-high-level-synthesis-plugin}

Pertama, buat [HLSConfig](https://docs.quantum.ibm.com/api/qiskit/qiskit.transpiler.passes.HLSConfig) untuk
menyimpan nama plugin yang akan digunakan untuk berbagai objek tingkat tinggi. Misalnya:

```python
from qiskit.transpiler.passes import HLSConfig

hls_config = HLSConfig(clifford=["layers"], linear_function=["pmh"])
```

Baris kode ini membuat konfigurasi sintesis tingkat tinggi yang menggunakan plugin `layers`
untuk mensintesis objek [Clifford](https://docs.quantum.ibm.com/api/qiskit/qiskit.quantum_info.Clifford#clifford) dan plugin `pmh` untuk mensintesis
objek [LinearFunction](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.library.LinearFunction#linearfunction). Nama argumen kata kunci sesuai dengan
atribut [`name`](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.Operation#name) dari kelas [Operation](https://docs.quantum.ibm.com/api/qiskit/qiskit.circuit.Operation) yang merepresentasikan jenis objek yang sedang disintesis.
Untuk setiap objek tingkat tinggi, daftar plugin yang diberikan dicoba secara berurutan hingga salah satunya
berhasil (dalam contoh di atas, setiap daftar hanya berisi satu plugin).

Selain menentukan
plugin berdasarkan namanya, kamu juga bisa meneruskan tuple `(nama, opsi)`, di mana elemen kedua tuple adalah dictionary yang berisi opsi untuk plugin tersebut. Dokumentasi metode sintesis seharusnya menjelaskan opsi yang didukungnya. Lihat [daftar ini](https://docs.quantum.ibm.com/api/qiskit/transpiler_synthesis_plugins#high-level-synthesis-plugins-2) untuk tautan ke dokumentasi plugin sintesis tingkat tinggi bawaan.

Setelah kamu membuat objek `HLSConfig`, berikan sebagai argumen
`hls_config` pada [`generate_preset_pass_manager`](https://docs.quantum.ibm.com/api/qiskit/qiskit.transpiler.generate_preset_pass_manager#qiskit.transpiler.generate_preset_pass_manager) atau [`transpile`](https://docs.quantum.ibm.com/api/qiskit/compiler#qiskit.compiler.transpile):

```python
pass_manager = generate_preset_pass_manager(
    optimization_level=3, backend=backend, hls_config=hls_config
)
```

Sintesis tingkat tinggi digunakan dalam tahap `init`, `translation`, dan `optimization` dari staged pass manager yang dikembalikan oleh [`generate_preset_pass_manager`](https://docs.quantum.ibm.com/api/qiskit/qiskit.transpiler.generate_preset_pass_manager#qiskit.transpiler.generate_preset_pass_manager) atau yang digunakan dalam [`transpile`](https://docs.quantum.ibm.com/api/qiskit/compiler#qiskit.compiler.transpile). Lihat [Tahap Transpiler](transpiler-stages) untuk deskripsi tahap-tahap ini.

# Gunakan virtual private endpoints untuk VPC agar bisa terhubung secara privat ke IBM Quantum Platform {#use-virtual-private-endpoints-for-vpc-to-privately-connect-to-ibm-quantum-platform}

Gunakan Virtual Private Endpoints (VPE) untuk VPC agar bisa terhubung ke IBM Quantum&reg; dari jaringan VPC kamu menggunakan alamat IP yang kamu tentukan, yang dialokasikan dari subnet dalam VPC kamu.

VPE adalah antarmuka IP virtual yang terikat pada endpoint gateway yang dibuat per layanan. Endpoint gateway adalah fungsi virtual yang dapat diskalakan secara horizontal, redundan dan sangat tersedia, serta mencakup semua zona ketersediaan VPC kamu. Endpoint gateway memungkinkan komunikasi dari virtual server instance dalam VPC kamu dan layanan IBM Cloud&reg; melalui jaringan backbone privat. VPE untuk VPC memungkinkan kamu mengontrol semua pengalamatan privat dalam cloud kamu. Untuk informasi lebih lanjut, lihat [Tentang virtual private endpoint gateways](https://cloud.ibm.com/docs/vpc?topic=vpc-about-vpe).

Di dalam IBM Quantum Platform, semua data pelanggan ditransmisikan melalui jaringan privat terlepas dari apakah diakses melalui public endpoint atau VPE.

Menghubungkan ke IBM Quantum Platform melalui public endpoints akan mentransmisikan semua data permintaan dan respons melalui internet publik. Untuk terhubung ke IBM Quantum Platform menggunakan VPE, kamu harus menggunakan IBM Quantum Platform API atau SDK. IBM Quantum Platform hanya bisa diakses melalui jaringan publik.

## Sebelum memulai {#before-you-begin}

Sebelum menargetkan VPE untuk IBM Quantum Platform, selesaikan langkah-langkah berikut:

* [Buat Virtual Private Cloud](https://cloud.ibm.com/docs/vpc?topic=vpc-getting-started).
* [Rencanakan topologi jaringan](https://cloud.ibm.com/docs/vpc?topic=vpc-planning-considerations) untuk terhubung ke VPE.
* [Atur kontrol akses](https://cloud.ibm.com/docs/vpc?topic=vpc-configure-acls-sgs-endpoint-gateways) untuk VPE kamu.
* Pahami [batasan](https://cloud.ibm.com/docs/vpc?topic=vpc-limitations-vpe) dari penggunaan VPE.
* Pahami cara [melihat detail VPE](https://cloud.ibm.com/docs/vpc?topic=vpc-vpe-viewing-details-of-an-endpoint-gateway).

## Siapkan VPE untuk IBM Quantum Platform {#set-up-a-vpe-for-ibm-quantum-platform}

Ada beberapa cara untuk membuat VPE gateway. Jika kamu menggunakan [CLI](https://cloud.ibm.com/docs/vpc?topic=vpc-vpc-reference&interface=cli#vpe-clis) atau [API](https://cloud.ibm.com/apidocs/vpc/latest#create-endpoint-gateway), kamu harus menentukan [Cloud Resource Name (CRN)](https://cloud.ibm.com/docs/account?topic=account-crn) dari region tempat kamu ingin terhubung ke IBM Quantum Platform. Tinjau tabel berikut untuk region dan CRN yang tersedia.

| Region | Plans   | Fully Qualified Domain Name (FDQN) | Cloud Resource Name (CRN) |
|-----------------|-----------------|-----------------|-----------------|
| `us-east` | `Standard` | `private.us-east.quantum.cloud.ibm.com` | `crn:v1:bluemix:public:quantum-computing:us-east:::endpoint:private.us-east.quantum.cloud.ibm.com` |
| `eu-de` | `Standard` | `private.eu-de.quantum.cloud.ibm.com` | `crn:v1:bluemix:public:quantum-computing:eu-de:::endpoint:private.eu-de.quantum.cloud.ibm.com` |

### Mengonfigurasi endpoint gateway {#configuring-an-endpoint-gateway}

Untuk mengonfigurasi VPE gateway, ikuti langkah-langkah berikut:

1. Daftar layanan yang tersedia, termasuk layanan infrastruktur IBM Cloud yang tersedia (secara default) untuk semua pengguna VPC.
1. [Buat endpoint gateway](https://cloud.ibm.com/docs/vpc?topic=vpc-ordering-endpoint-gateway) untuk IBM Quantum Platform yang ingin kamu jadikan tersedia secara privat ke VPC.
1. [Ikat alamat IP cadangan](https://cloud.ibm.com/docs/vpc?topic=vpc-bind-unbind-reserved-ip) ke endpoint gateway.
1. [Lihat VPE gateway yang dibuat](https://cloud.ibm.com/docs/vpc?topic=vpc-vpe-viewing-details-of-an-endpoint-gateway) yang terkait dengan IBM Quantum Platform.

Sekarang virtual server instance kamu di VPC dapat mengakses instance IBM Quantum Platform kamu secara privat.

## Gunakan VPE untuk IBM Quantum Platform {#use-your-vpe-for-ibm-quantum-platform}

Setelah membuat endpoint gateway untuk IBM Quantum Platform, ikuti langkah-langkah berikut:

### Gunakan VPE dengan `qiskit-ibm-runtime` (Python SDK) {#use-the-vpe-with-qiskit-ibm-runtime-python-sdk}

<Admonition type="note">
Dukungan VPE memerlukan `qiskit-ibm-runtime` v0.24.0 atau yang lebih baru.
</Admonition>

Saat membuat instance `QiskitRuntimeService`, tentukan `private_endpoint=True`.

```python
service = QiskitRuntimeService(token="your-API_KEY", # Use the 44-character API_KEY you created and saved from the IBM Quantum Platform Home dashboard
                              instance="INSTANCE_CRN",
                              channel="ibm_cloud",
                              private_endpoint=True)
```

### Gunakan VPE dengan IBM Quantum API {#use-the-vpe-with-the-ibm-quantum-api}

Setelah membuat endpoint gateway untuk IBM Quantum Platform, gunakan FQDN endpoint layanan untuk region yang dituju.

```bash
  curl -X POST https://private.us-east.quantum.cloud.ibm.com/api/v1/jobs -H "Authorization: Bearer $BEARER_TOKEN" -H "Service-CRN: $SERVICE_INSTANCE_CRN" -d '{
    "backend": "backend",
    "program_id": "sampler"
  }'
```

### Operasi Dasar pada Qubit dan Pengukuran {#basic-operations-on-qubits-and-measurements}

### Menuliskan state single-qubit {#writing-down-single-qubit-states}
Mari kita mulai dengan melihat satu qubit. Perbedaan utama antara bit klasik, yang hanya bisa mengambil nilai 0 dan 1, adalah bahwa bit kuantum, atau **qubit**, bisa berada di state $\vert0\rangle$, $\vert1\rangle$, maupun kombinasi linier dari kedua state tersebut. Fitur ini dikenal sebagai superposisi, dan memungkinkan kita menuliskan state paling umum dari sebuah qubit sebagai: 

$$\vert\psi\rangle = \sqrt{1-p}\vert0\rangle + e^{i \phi} \sqrt{p}  \vert1\rangle$$

Jika kita mengukur state qubit ini, kita akan menemukan hasil $1$ dengan probabilitas $p$, dan hasil $0$ dengan probabilitas $1-p$. Seperti yang kamu lihat, total probabilitasnya adalah $1$, artinya kita memang akan mengukur $0$ atau $1$, dan tidak ada hasil lain yang mungkin.

Selain $p$, kamu mungkin memperhatikan parameter lain di atas. Variabel $\phi$ menunjukkan fase kuantum relatif antara kedua state $\vert0\rangle$ dan $\vert1\rangle$. Seperti yang akan kita temukan nanti, fase relatif ini cukup penting. Untuk saat ini, cukup diketahui bahwa fase kuantum adalah yang memungkinkan interferensi antar state kuantum, sehingga kita bisa menulis algoritma kuantum untuk menyelesaikan tugas-tugas tertentu.

### Memvisualisasikan state kuantum {#visualizing-quantum-states}
Kita memvisualisasikan state kuantum di seluruh latihan ini menggunakan apa yang dikenal sebagai `qsphere`. Berikut tampilan `qsphere` untuk state $\vert0\rangle$ dan $\vert1\rangle$ secara berurutan. Perhatikan bahwa bagian paling atas bola mewakili state $\vert0\rangle$, sementara bagian bawah mewakili $\vert1\rangle$.

```python
#visualize |0>
sv=Statevector([1, 0])
plot_state_qsphere(sv)
```

Kamu bisa membuat QSphere yang sama persis menggunakan circuit kuantum. Statevector yang digunakan di sini berasal dari state $|0\rangle$. Di Qiskit, qubit diinisialisasi dalam state $|0\rangle$. Coba jalankan circuit di bawah dan lihat apakah kamu bisa mendapatkan QSphere yang sama.

```python
qc1 = QuantumCircuit(1)
sv=Statevector(qc1)
plot_state_qsphere(sv)
```

Sekarang mari kita visualisasikan state $|1\rangle$. 
Tidak mengherankan bahwa state superposisi dengan fase kuantum $\phi = 0$ dan probabilitas $p = 1/2$ (artinya kemungkinan sama untuk mengukur 0 dan 1) ditampilkan pada `qsphere` dengan dua titik. Namun, perhatikan juga bahwa ukuran lingkaran di dua titik tersebut lebih kecil dari saat kita hanya memiliki $\vert0\rangle$ dan $\vert1\rangle$ di atas. Ini karena ukuran lingkaran sebanding dengan probabilitas mengukur masing-masing, yang kini berkurang setengahnya.

```python

```

Dalam kasus state superposisi, di mana fase kuantum tidak nol, qsphere memungkinkan kita memvisualisasikan fase tersebut dengan mengubah warna blob yang bersangkutan. Misalnya, state dengan $\phi = 90^\circ$ (derajat) dan probabilitas $p = 1/2$ ditampilkan di `qsphere` di bawah ini.

### Memanipulasi qubit {#manipulating-qubits}
Qubit dimanipulasi dengan menerapkan gate kuantum. Mari kita tinjau berbagai gate yang akan kita pertimbangkan dalam latihan-latihan berikut.

Pertama, mari kita jelaskan bagaimana kita bisa mengubah nilai $p$ untuk state kuantum umum kita. Untuk melakukannya, kita akan menggunakan dua gate:

1. **$X$-gate**: Gate ini membalik antara dua state $\vert0\rangle$ dan $\vert1\rangle$. Operasi ini sama dengan gate NOT klasik. Akibatnya, $X$-gate kadang disebut bit flip atau NOT gate. Secara matematis, $X$ gate mengubah $p$ menjadi $1-p$, sehingga khususnya dari 0 ke 1, dan sebaliknya.

2. **$H$-gate**: Gate ini memungkinkan kita pergi dari state $\vert0\rangle$ ke state $\frac{1}{\sqrt{2}}\left(\vert0\rangle + \vert1\rangle\right)$. State ini juga dikenal sebagai $\vert+\rangle$. Secara matematis, ini berarti berpindah dari $p=0, \phi=0$ ke $p=1/2, \phi=0$. Karena state akhir qubit adalah superposisi $\vert0\rangle$ dan $\vert1\rangle$, Hadamard gate mewakili operasi kuantum yang sesungguhnya.

## 4. Kuis Kuantum dengan Multi-Qubit Gates {#4-quantum-quiz-with-multi-qubit-gates}

Bagus sekali! Sekarang kamu sudah paham tentang single-qubit gates, mari kita lihat gates yang beroperasi pada beberapa qubit. Di sini kamu akan diminta menyelesaikan 4 kuis keadaan kuantum dengan menggabungkan single-qubit gates dan multi-qubit gates. Gates dasar pada dua qubit adalah:

    qc.cx(c,t)       # controlled-X (= CNOT) gate dengan qubit kontrol c dan qubit target t
    qc.cz(c,t)       # controlled-Z gate dengan qubit kontrol c dan qubit target t
    qc.swap(a,b)     # SWAP gate yang menukar keadaan qubit a dan qubit b

Kalau kamu ingin membaca lebih lanjut tentang berbagai multi-qubit gates dan hubungannya, kunjungi [Quantum Information](https://quantum.cloud.ibm.com/learning/en/courses/basics-of-quantum-information/multiple-systems/quantum-information) dari sistem ganda, di Kursus `Basics of Quantum Information` milik John.

Perhatikan bahwa untuk dua qubit, keadaan umum berbentuk $a|00\rangle + b |01\rangle + c |10\rangle + d|11\rangle$, di mana $a$, $b$, $c$, dan $d$ adalah bilangan kompleks yang nilai absolutnya dikuadratkan memberikan peluang mengukur keadaan masing-masing; misalnya, $|a|^2$ adalah peluang berakhir di keadaan '0' pada kedua qubit. Ini berarti kita sekarang bisa memiliki hingga empat titik pada qsphere.

Kita mulai dengan multi-qubit gate kanonik, yaitu controlled-NOT (juga disebut CNOT atau CX) gate. Di sini, seperti semua controlled two-qubit gates, satu qubit dilabeli sebagai "kontrol", sementara yang lain disebut "target". Jika qubit kontrol berada di keadaan $|0\rangle$, ia menerapkan gate identitas $I$ pada target, yaitu tidak ada operasi yang dilakukan. Sebaliknya, jika qubit kontrol berada di keadaan $|1\rangle$, X-gate diterapkan pada qubit target. Oleh karena itu, dengan kedua qubit di salah satu dari dua keadaan klasik, $|0\rangle$ atau $|1\rangle$, CNOT gate terbatas pada operasi klasik.

Situasi ini berubah secara dramatis ketika kita menerapkan Hadamard gate pada qubit kontrol terlebih dahulu, sehingga membawanya ke keadaan superposisi $|+\rangle$. Aksi CNOT gate pada input non-klasik ini dapat menghasilkan keadaan yang sangat entangled antara qubit kontrol dan target. Jika qubit target awalnya berada di keadaan $|0\rangle$, keadaan yang dihasilkan dilambangkan $|\Phi^+\rangle$, dan merupakan salah satu dari [Bell states](https://quantum.cloud.ibm.com/learning/en/courses/basics-of-quantum-information/multiple-systems/quantum-information#bell-states).

#### i) Konstruksi Bell state $|\Phi^+\rangle = \frac{1}{\sqrt{2}}\left(|00\rangle + |11\rangle\right)$. {#i-construct-the-bell-state}

Untuk keadaan ini kita akan memiliki peluang $\frac{1}{2}$ untuk mengukur "00" dan peluang $\frac{1}{2}$ untuk mengukur "11". Dengan demikian, hasil kedua qubit berkorelasi sempurna.

```python

def create_circuit5():
    qc = QuantumCircuit(2)
    qc.h(0)
    qc.cx(0, 1)
    return qc

qc6 = create_circuit5()
state = Statevector(qc6) # determine final state after running the circuit
display(array_to_latex(state))
display(plot_state_qsphere(state, show_state_labels=True, show_state_phases = True, use_degrees = True))
qc6.draw(output='mpl') # we draw the circuit
```

Selanjutnya, coba buat keadaan qubit yang berkorelasi sempurna secara anti. Perhatikan tanda minus di sini, yang menunjukkan fase relatif antara kedua keadaan.

#### ii) Konstruksi Bell state $\vert\Psi^-\rangle = \frac{1}{\sqrt{2}}\left(\vert01\rangle - \vert10\rangle\right)$. {#ii-construct-the-bell-state}

```python

def create_circuit6():

    qc.x(1)
```

### iii) Kamu diberikan quantum circuit yang dijelaskan dalam fungsi di bawah ini. Tukar keadaan qubit pertama dan kedua untuk mendapatkan QSphere ini. {#iii-swap-the-states}

```python
def create_circuit7():
```

## 5. Jalankan circuit-mu dan lihat hasil pengukurannya dengan komputer kuantum nyata {#5-run-your-circuit}

Sekarang kita sudah tahu cara membangun quantum circuit untuk memanipulasi statevector dari qubit. Dan inilah saatnya bagian paling seru: menjalankannya dan melihat hasilnya! Di sini kita akan belajar cara modern dan efektif untuk mengeksekusi circuit kita menggunakan Qiskit.

Pola Qiskit adalah kerangka umum untuk memecah masalah spesifik domain dan mengontekstualisasikan kapabilitas yang diperlukan dalam beberapa tahap. Ini memungkinkan komposisi yang mulus dari kapabilitas baru yang dikembangkan oleh para peneliti IBM Quantum (dan lainnya) serta memungkinkan masa depan di mana tugas-tugas komputasi kuantum dilakukan oleh infrastruktur komputasi heterogen yang kuat (CPU/GPU/QPU).

Empat langkah dari pola Qiskit adalah sebagai berikut:

1. **Map** masalah ke quantum circuits dan operator
2. **Optimize** untuk hardware target
3. **Execute** pada hardware target
4. **Post-process** hasil

Kita baru saja menyelesaikan `Langkah 1: Mapping`, dengan membangun quantum circuits untuk menghasilkan keadaan kuantum yang diinginkan. Sekarang, mari kita telusuri langkah-langkah berikutnya untuk melihat hasilnya.

### Optimize {#optimize}

Di sini kita akan mengatur Backend untuk menjalankan circuits — kamu bisa memilih qpu `least busy` di antara grup qpu yang bisa diakses, atau cukup pilih simulator jika tidak punya cukup sisa waktu GPU. Setelah memilih backend, `pass_manager` akan `transpile` circuit-mu ke set gate native dari backend yang kamu pilih dan mengoptimalkannya agar kamu mendapatkan hasil yang lebih baik. Kamu bisa dengan mudah mendeklarasikan `pass_manager` menggunakan `generate_preset_pass_manager` dan mengatur `optimization_level`, di mana angka yang lebih tinggi menunjukkan lebih banyak langkah optimasi.

Langkah berikutnya sangat menarik — kita akan menjalankan quantum circuit menggunakan Qiskit Runtime!

Kita akan melakukannya menggunakan dua [primitif Qiskit](https://quantum.cloud.ibm.com/docs/guides/primitives):
1. **Sampler** mengambil sampel dari register output dari eksekusi satu atau lebih quantum circuits. Outputnya adalah counts dari pengukuran per-shot.
2. **Estimator** menghitung nilai ekspektasi dari satu atau lebih observabel terhadap keadaan yang dihasilkan oleh quantum circuit. Outputnya terdiri dari nilai ekspektasi beserta kesalahan standarnya.

Di sini, kita akan menggunakan `Sampler` untuk mengeksekusi circuit kita. Cell kode di bawah ini menunjukkan cara mendefinisikan backend dan pass manager untuk itu terlebih dahulu. Kemudian, ia akan menambahkan `measurement` ke semua circuits dan membuat array dari quantum circuits (`pub`) untuk diteruskan ke Sampler.

```python

backend=service.least_busy()
#backend=AerSimulator()

pm = generate_preset_pass_manager(optimization_level=3, backend=backend)
sampler = Sampler(mode=backend)

pub = []
for qc in circ:
    qc.measure_all()
    pub.append(pm.run(qc))
```

### Execute {#execute}

Mari eksekusi circuit kita. Jika ada banyak antrian menunggu di cloud, cetak dan simpan job_id untuk digunakan nanti dan periksa job_status. Setelah kamu melihat status job berubah menjadi `Done`, ambil hasil job tersebut.

```python
job = sampler.run(pub)
job_id = job.job_id()
print(job_id)
job.status()
```

```python
job_retrived = service.job(job_id)
result = job.result()
```

### Post-process {#post-process}

Langkah terakhir adalah menginterpretasikan visualisasi untuk memahami keadaan kuantum yang telah kita buat. Sebelum membuat beberapa grafik, mari ambil semua counts dari semua circuits. Kemudian kita akan membuat 4 plot dengan mengategorikan circuits.

```python
result = job.result()
counts_all = [result[k].data.meas.get_counts() for k in range(9)]
```

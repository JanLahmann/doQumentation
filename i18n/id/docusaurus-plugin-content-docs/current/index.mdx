## Konten sumber terbuka IBM Quantum {#ibm-quantums-open-source-content}

IBM menyediakan banyak materi pembelajaran komputasi kuantum — semua sumber terbuka di bawah CC BY-SA 4.0:

- **[Learning](https://quantum.cloud.ibm.com/learning)** — Kursus terstruktur dari dasar kuantum hingga topik lanjutan
- **[Tutorials](https://quantum.cloud.ibm.com/docs/en/tutorials)** — Lebih dari 40 tutorials tentang transpilation, error mitigation, dan lainnya
- **[Documentation](https://quantum.cloud.ibm.com/docs/en/guides)** — Panduan dan referensi API untuk Qiskit
- **[Source repo](https://github.com/Qiskit/documentation)** — Semua konten di GitHub

[Quantum Platform](https://quantum.cloud.ibm.com) mereka selalu up-to-date dan dirancang dengan baik — tempat terbaik untuk membaca, belajar, dan referensi.

## Apa yang ditambahkan proyek ini {#what-this-project-adds}

[Dokumentasi Qiskit](https://github.com/Qiskit/documentation) IBM adalah sumber terbuka (CC BY-SA 4.0), tetapi aplikasi web mereka tidak. doQumentation menambahkan frontend sumber terbuka dengan eksekusi kode langsung, injeksi kredensial otomatis, dan mode simulator.

Bisa di-deploy di mana saja — dari [GitHub Pages](https://doqumentation.org) hingga [Docker](https://github.com/JanLahmann/doQumentation/pkgs/container/doqumentation) hingga [RasQberry](https://rasqberry.org/). Lihat [semua fitur](/features).

## Eksekusi kode {#code-execution}

Klik **Run** pada blok kode mana pun. Klik pertama memulai kernel Jupyter melalui [Binder](https://mybinder.org) atau [IBM Code Engine](/jupyter-settings#code-engine). Setelah itu, eksekusi langsung.

- **Mode Simulator (tidak perlu akun)** — Aktifkan [Mode Simulator](/jupyter-settings#simulator-mode) untuk menjalankan semua notebook dengan AerSimulator atau FakeBackends model noise. Tidak perlu setup sama sekali.

- **Hardware IBM Quantum** — Masukkan token API dan CRN Anda di [Pengaturan](/jupyter-settings#ibm-quantum) sekali — kredensial disuntikkan otomatis setiap kali dijalankan. [Buat akun gratis](https://quantum.cloud.ibm.com/registration) jika belum punya.

<details>
<summary>Backend eksekusi yang tersedia</summary>

Setiap tutorial punya blok kode yang bisa dieksekusi. Klik **Run** untuk menjalankannya menggunakan salah satu dari empat backend:

1. **Binder** (default di GitHub Pages) — Kernel Jupyter jarak jauh gratis melalui [mybinder.org](https://mybinder.org)
2. **IBM Code Engine** — Jupyter serverless di akun IBM Cloud Anda. Cold start dalam detik, bukan menit. [Setup](/jupyter-settings#code-engine)
3. **Jupyter Lokal (Docker / RasQberry)** — Terhubung ke server Jupyter lokal dengan Qiskit sudah terinstal
4. **Server kustom** — Arahkan ke endpoint Jupyter mana pun di [Pengaturan](/jupyter-settings#advanced)

Saat beberapa backend tersedia, pilih yang mana dari [Pengaturan](/jupyter-settings#backend-selection).

</details>

<details>
<summary>Opsi deployment</summary>

doQumentation tersedia sebagai:

- **[GitHub Pages](https://doqumentation.org)** — Situs statis dengan [Binder](https://mybinder.org) atau [IBM Code Engine](/jupyter-settings#code-engine) untuk eksekusi kode jarak jauh
- **[Docker](https://github.com/JanLahmann/doQumentation/pkgs/container/doqumentation)** — Full stack secara lokal, bisa offline
- **[RasQberry](https://rasqberry.org/)** — Self-hosted di Raspberry Pi dengan kernel Jupyter lokal

</details>

<details>
<summary>Jalankan secara lokal dengan Podman / Docker</summary>

Instal [Podman Desktop](https://podman-desktop.io/) atau [Docker Desktop](https://www.docker.com/products/docker-desktop/) (Mac / Windows / Linux), lalu jalankan:

```bash
# Full stack: site + Jupyter + Qiskit (~3 GB)
podman run -p 8080:80 -p 8888:8888 ghcr.io/janlahmann/doqumentation:jupyter
```

Buka [http://localhost:8080](http://localhost:8080) — eksekusi kode berjalan secara lokal, tidak perlu menunggu Binder.

Untuk versi ringan tanpa eksekusi kode lokal (~60 MB):

```bash
# Static site only — code execution still works via Binder
podman run -p 8080:80 ghcr.io/janlahmann/doqumentation:latest
```

Pakai Docker? Cukup ganti `podman` dengan `docker` — perintahnya identik. Image bersifat multi-arch (`linux/amd64` + `linux/arm64`), sehingga Apple Silicon Mac berfungsi secara native.

</details>
